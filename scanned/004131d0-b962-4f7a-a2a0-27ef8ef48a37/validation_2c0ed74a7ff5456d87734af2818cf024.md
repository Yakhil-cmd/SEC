### Title
Byzantine Peer Can Inject Duplicate `StrippedIngressMessage` Entries to Abort Block Assembly and Stall Consensus Participation — (`rs/p2p/artifact_downloader/src/fetch_stripped_artifact/types/stripped.rs`)

---

### Summary

`TryFrom<pb::StrippedBlockProposal> for StrippedBlockProposal` performs no deduplication on the `ingress_messages` repeated field. A Byzantine subnet peer can craft a proto with N identical `StrippedIngressMessage` entries for the same ingress. This causes `BlockProposalAssembler` to spawn N redundant `get_or_fetch` tasks; the second insertion returns `InsertionError::AlreadyInserted`, which causes `assemble_message` to return `AssembleResult::Unwanted`. The consensus manager's `process_slot_update` then waits indefinitely for all peers to stop advertising the block before it can retry — meaning the victim node never delivers the block to the unvalidated pool while honest peers keep advertising it.

### Impact Explanation

A single Byzantine subnet peer can prevent a victim replica from processing a targeted block proposal for the duration of that proposal's advertisement by honest peers. The victim node's consensus layer never receives the block, causing it to fall behind. Recovery requires state sync or catch-up packages. This is a targeted per-node liveness violation exploitable by any single Byzantine subnet member, with no threshold requirement.

---

### Likelihood Explanation

The attack requires only subnet membership (a Byzantine replica). The crafted proto is trivially constructable — just repeat the same `StrippedIngressMessage` N times. No cryptographic material, no threshold, no privileged access is needed. The deserialization path has no guard against duplicates, and the code itself acknowledges the `AlreadyInserted` path as "a bug" (line 265), confirming this is an unintended reachable state.

### Recommendation

Add a deduplication check in `TryFrom<pb::StrippedBlockProposal> for StrippedBlockProposal` immediately after collecting `ingress_messages`:

```rust
let ingress_messages: Vec<SignedIngressId> = value
    .ingress_messages
    .into_iter()
    .map(SignedIngressId::try_from)
    .collect::<Result<Vec<_>, _>>()?;

// Reject proposals with duplicate ingress IDs at deserialization time.
let unique_count = ingress_messages.iter().collect::<std::collections::HashSet<_>>().len();
if unique_count != ingress_messages.len() {
    return Err(ProxyDecodeError::Other(
        "Duplicate ingress message IDs in StrippedBlockProposal".into()
    ));
}
``` [1](#0-0) 

The same check should be applied to `stripped_idkg_dealings` for the same reason.
