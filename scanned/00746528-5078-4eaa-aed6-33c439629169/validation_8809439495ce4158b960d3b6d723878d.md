### Title
ckBTC Minter `get_withdrawal_account` and `get_btc_address` Bypass `ReadOnly` Mode Guard, Mutating State During Emergency Pause - (File: `rs/bitcoin/ckbtc/minter/src/updates/get_withdrawal_account.rs`, `rs/bitcoin/ckbtc/minter/src/updates/get_btc_address.rs`)

---

### Summary

The ckBTC minter implements a `Mode` enum (`ReadOnly`, `RestrictedTo`, `DepositsRestrictedTo`, `GeneralAvailability`) as a guarded-launch / emergency-pause mechanism. `ReadOnly` is explicitly documented as "the minter does not allow any state modifications." However, two public update-call endpoints — `get_withdrawal_account` and `get_btc_address` — perform no mode check and unconditionally call `init_ecdsa_public_key()`, which **mutates replicated state** by writing the ECDSA public key into the minter's stable state. This is a direct analog to M-08: public functions that bypass the pause flag and change contract state during an emergency pause.

### Impact Explanation

**State mutation during ReadOnly pause:** When the minter is placed in `ReadOnly` mode (e.g., due to an emergency), any unprivileged caller can invoke `get_withdrawal_account` or `get_btc_address`. If the ECDSA public key has not yet been initialized (e.g., immediately after a fresh deployment or upgrade), these calls will write `ecdsa_public_key` into the minter's replicated state — a state modification that `ReadOnly` mode is explicitly designed to prevent.

**Stuck-funds state created during pause:** The old withdrawal flow requires the user to call `get_withdrawal_account`, transfer ckBTC to the returned account on the ledger, then call `retrieve_btc`. In `ReadOnly` mode, `retrieve_btc` is blocked, but `get_withdrawal_account` is not. A user (or an automated script) can therefore:
1. Call `get_withdrawal_account` (succeeds — no mode check).
2. Transfer ckBTC to the returned minter subaccount on the ledger.
3. Call `retrieve_btc` (fails — mode check blocks it).

The ckBTC is now locked in the minter's subaccount. This is a **new balance state created during the emergency pause** that did not exist before the pause was activated. Recovery operators must account for these in-flight balances when reasoning about the minter's state post-pause, complicating safe restart.

**`DepositsRestrictedTo` bypass:** In `DepositsRestrictedTo` mode, `get_btc_address` is callable by any principal (no mode check), so non-whitelisted users can obtain a deposit address, send BTC on-chain, and then find `update_balance` blocked. The BTC sits unprocessed in the deposit address until the mode changes, creating an unintended state.

---

### Likelihood Explanation

The `ReadOnly` and `DepositsRestrictedTo` modes are operator-activated emergency controls. Any unprivileged ingress sender can reach `get_withdrawal_account` and `get_btc_address` at any time — no special role or key is required. The attack path is a single direct update call to the minter canister. The likelihood of accidental triggering is high (users naturally call these endpoints as part of the normal withdrawal/deposit flow), and the likelihood of deliberate exploitation during a pause is also high since the endpoints are publicly documented.

---

### Recommendation

Apply the mode guard at the top of both `get_withdrawal_account` and `get_btc_address`, mirroring the pattern used in `update_balance` and `retrieve_btc`:

```rust
// In get_withdrawal_account:
pub async fn get_withdrawal_account() -> Account {
    let caller = PrincipalId(ic_cdk::api::msg_caller());
    state::read_state(|s| s.mode.is_withdrawal_available_for(&caller.0))
        .unwrap_or_else(|e| ic_cdk::trap(&e));
    init_ecdsa_public_key().await;
    // ...
}

// In get_btc_address:
pub async fn get_btc_address(args: GetBtcAddressArgs) -> String {
    let owner = args.owner.unwrap_or_else(ic_cdk::api::msg_caller);
    state::read_state(|s| s.mode.is_deposit_available_for(&owner))
        .unwrap_or_else(|e| ic_cdk::trap(&e));
    // ...
}
```

Additionally, validate all public/update-call endpoints in the minter DID against the full mode matrix to ensure no other endpoints bypass the mode guard.

### Proof of Concept

1. Deploy the ckBTC minter canister.
2. Upgrade it with `Mode::ReadOnly` via `UpgradeArgs { mode: Some(Mode::ReadOnly), .. }`.
3. Confirm that `update_balance` and `retrieve_btc` return `TemporarilyUnavailable` (as verified by the existing test `test_upgrade_read_only` at `rs/bitcoin/ckbtc/minter/tests/tests.rs:410`). [9](#0-8) 

4. Call `get_withdrawal_account` as any unprivileged principal — it **succeeds** and returns an account, with no mode check performed.
5. Call `get_btc_address` as any unprivileged principal — it **succeeds** and returns a Bitcoin address, with no mode check performed.
6. If the ECDSA public key was not yet initialized, step 4 or 5 will have written it into replicated state — a state modification that occurred while the minter was in `ReadOnly` mode.
7. Transfer ckBTC to the account returned in step 4 via the ckBTC ledger. The ckBTC is now locked in the minter's subaccount with no way to retrieve it while `ReadOnly` mode is active, creating unintended state during the emergency pause.

    /// Returns Ok if the specified principal can convert BTC to ckBTC.
    pub fn is_deposit_available_for(&self, p: &Principal) -> Result<(), String> {
        match self {
            Self::GeneralAvailability => Ok(()),
            Self::ReadOnly => Err("the minter is in read-only mode".to_string()),
            Self::RestrictedTo(allow_list) => {
                if !allow_list.contains(p) {
                    return Err("access to the minter is temporarily restricted".to_string());
                }
                Ok(())
            }
            Self::DepositsRestrictedTo(allow_list) => {
                if !allow_list.contains(p) {
                    return Err("BTC deposits are temporarily restricted".to_string());
                }
                Ok(())
            }
        }
    }

    /// Returns Ok if the specified principal can convert ckBTC to BTC.
    pub fn is_withdrawal_available_for(&self, p: &Principal) -> Result<(), String> {
        match self {
            Self::GeneralAvailability | Self::DepositsRestrictedTo(_) => Ok(()),
            Self::ReadOnly => Err("the minter is in read-only mode".to_string()),
            Self::RestrictedTo(allow_list) => {
                if !allow_list.contains(p) {
                    return Err("BTC withdrawals are temporarily restricted".to_string());
                }
                Ok(())
            }
        }
    }

pub async fn init_ecdsa_public_key() -> ECDSAPublicKey {
    if let Some(key) = read_state(|s| s.ecdsa_public_key.clone()) {
        return key;
    };
    let key_name = read_state(|s| s.ecdsa_key_name.clone());
    log!(
        Priority::Debug,
        "Fetching the ECDSA public key {}",
        &key_name
    );
    let ecdsa_public_key =
        crate::management::ecdsa_public_key(key_name, DerivationPath::new(vec![]))
            .await
            .unwrap_or_else(|e| ic_cdk::trap(format!("failed to retrieve ECDSA public key: {e}")));
    log!(
        Priority::Debug,
        "ECDSA public key set to {}, chain code to {}",
        hex::encode(&ecdsa_public_key.public_key),
        hex::encode(&ecdsa_public_key.chain_code)
    );
    mutate_state(|s| {
        s.ecdsa_public_key = Some(ecdsa_public_key.clone());
    });
    ecdsa_public_key
}

#[test]
fn test_upgrade_read_only() {
    let env = new_state_machine();
    let ledger_id = install_ledger(&env);
    let minter_id = install_minter(&env, ledger_id);

    let authorized_principal =
        Principal::from_str("k2t6j-2nvnp-4zjm3-25dtz-6xhaa-c7boj-5gayf-oj3xs-i43lp-teztq-6ae")
            .unwrap();

    // upgrade
    let upgrade_args = UpgradeArgs {
        mode: Some(Mode::ReadOnly),
        ..Default::default()
    };
    let minter_arg = MinterArg::Upgrade(Some(upgrade_args));
    env.upgrade_canister(minter_id, minter_wasm(), Encode!(&minter_arg).unwrap())
        .expect("Failed to upgrade the minter canister");

    // when the mode is ReadOnly then the minter should reject all update calls.

    // 1. update_balance
    let update_balance_args = UpdateBalanceArgs {
        owner: None,
        subaccount: None,
    };
    let res = env
        .execute_ingress_as(
            authorized_principal.into(),
            minter_id,
            "update_balance",
            Encode!(&update_balance_args).unwrap(),
        )
        .expect("Failed to call update_balance");
    let res = Decode!(&res.bytes(), Result<Vec<UtxoStatus>, UpdateBalanceError>).unwrap();
    assert!(
        matches!(res, Err(UpdateBalanceError::TemporarilyUnavailable(_))),
        "unexpected result: {res:?}"
    );

    // 2. retrieve_btc
    let retrieve_btc_args = RetrieveBtcArgs {
        amount: 10,
        address: "".into(),
    };
    let res = env
        .execute_ingress_as(
            authorized_principal.into(),
            minter_id,
            "retrieve_btc",
            Encode!(&retrieve_btc_args).unwrap(),
        )
        .expect("Failed to call retrieve_btc");
    let res = Decode!(&res.bytes(), Result<RetrieveBtcOk, RetrieveBtcError>).unwrap();
    assert!(
        matches!(res, Err(RetrieveBtcError::TemporarilyUnavailable(_))),
        "unexpected result: {res:?}"
    );
}
