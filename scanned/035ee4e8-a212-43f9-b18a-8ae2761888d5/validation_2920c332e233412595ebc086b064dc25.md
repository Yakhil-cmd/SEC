### Title
Wrong Field Assigned to `max_neurons_fund_participation_icp_e8s` in `From` Conversion — (File: rs/sns/swap/src/neurons_fund.rs)

### Summary

In `rs/sns/swap/src/neurons_fund.rs`, the `From<ValidatedNeuronsFundParticipationConstraints<F>> for NeuronsFundParticipationConstraintsPb` implementation incorrectly copies `value.min_direct_participation_threshold_icp_e8s` into the `max_neurons_fund_participation_icp_e8s` field instead of `value.max_neurons_fund_participation_icp_e8s`. This is a direct analog to the ForceDAO `setProfitSharingNumerator` bug: a critical numeric parameter controlling fund distribution is silently assigned the wrong value, with no validation or error surfaced.

### Finding Description

The `From` impl at lines 502–526 of `rs/sns/swap/src/neurons_fund.rs` constructs a `NeuronsFundParticipationConstraintsPb` from a validated struct. The field assignment is:

```rust
max_neurons_fund_participation_icp_e8s: Some(
    value.min_direct_participation_threshold_icp_e8s,  // BUG: should be value.max_neurons_fund_participation_icp_e8s
),
``` [1](#0-0) 

The correct source field `value.max_neurons_fund_participation_icp_e8s` exists on `ValidatedNeuronsFundParticipationConstraints` and is correctly populated by the `TryFrom` path (lines 416–421), but is silently discarded in the reverse `From` direction. [2](#0-1) 

The `max_neurons_fund_participation_icp_e8s` field is the hard cap used in `MatchedParticipationFunction::apply` to limit total Neurons' Fund ICP disbursement:

```rust
let hard_cap_icp = rescale_to_icp(self.max_neurons_fund_participation_icp_e8s)?;
let effective_icp = hard_cap_icp.min(intercept_icp.saturating_add(...));
``` [3](#0-2) 

When the `From` impl is invoked, the hard cap is silently set to the minimum participation threshold (e.g., 36,000 ICP) instead of the actual maximum (e.g., 45,000 ICP), causing the SNS Swap canister to enforce a far lower ceiling on Neurons' Fund contributions than was computed and intended.

### Impact Explanation

The `max_neurons_fund_participation_icp_e8s` field is the sole hard cap applied to Neurons' Fund ICP disbursement in every SNS swap that uses Matched Funding. When this field is set to `min_direct_participation_threshold_icp_e8s` instead of the actual maximum:

- The Neurons' Fund contributes less ICP than the NNS governance computed and intended.
- SNS swaps that depend on Neurons' Fund participation to reach their minimum ICP target may fail.
- Neurons' Fund maturity is not correctly converted to ICP participation, constituting a ledger conservation violation: the NNS governance allocates maturity for a swap, but the swap canister caps disbursement at the wrong (lower) value.

### Likelihood Explanation

Every SNS swap that requests Neurons' Fund participation (`neurons_fund_participation: true`) and triggers the `From<ValidatedNeuronsFundParticipationConstraints<F>> for NeuronsFundParticipationConstraintsPb` conversion is affected. This conversion is a standard Rust `From`/`Into` impl in the production SNS Swap crate. Any `CreateServiceNervousSystem` proposal adopted by the NNS that includes Neurons' Fund participation will exercise this path. No privileged access, key compromise, or threshold attack is required.

### Recommendation

Fix the field assignment in the `From` impl:

```rust
max_neurons_fund_participation_icp_e8s: Some(
    value.max_neurons_fund_participation_icp_e8s,  // correct field
),
```

Add a round-trip test that constructs a `ValidatedNeuronsFundParticipationConstraints` with distinct `min_direct_participation_threshold_icp_e8s` and `max_neurons_fund_participation_icp_e8s` values, converts to `NeuronsFundParticipationConstraintsPb` via `From`, and asserts that `max_neurons_fund_participation_icp_e8s` in the output equals the original max, not the min.

### Proof of Concept

Given a `ValidatedNeuronsFundParticipationConstraints` with:
- `min_direct_participation_threshold_icp_e8s = 36_000 * E8`
- `max_neurons_fund_participation_icp_e8s = 45_000 * E8`

(matching the test data in `rs/nns/governance/src/governance/test_data.rs` lines 205–233) [4](#0-3) 

After conversion via `NeuronsFundParticipationConstraintsPb::from(validated)`:

```
result.max_neurons_fund_participation_icp_e8s == Some(36_000 * E8)  // wrong: min threshold
// should be:
result.max_neurons_fund_participation_icp_e8s == Some(45_000 * E8)  // correct: actual max
```

The SNS Swap canister then calls `apply(direct_participation)` and caps Neurons' Fund ICP at 36,000 ICP instead of 45,000 ICP, silently under-disbursing up to 9,000 ICP per swap from the Neurons' Fund. [5](#0-4)
