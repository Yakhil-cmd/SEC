### Title
NNS Governance Canister Uses Hardcoded Fallback ICP/XDR Rate for Neurons' Fund SNS Swap Participation Limits - (File: rs/nns/governance/src/heap_governance_data.rs)

### Summary
The NNS Governance canister hardcodes a fallback ICP/XDR rate of `xdr_permyriad_per_icp: 10_000` (1 ICP = 1 XDR) when the cached conversion rate is absent or partially missing. This fallback is consumed by `icp_xdr_rate()` → `try_derive_neurons_fund_participation_limits` to convert XDR-denominated Neurons' Fund thresholds into ICP amounts at the moment a `CreateServiceNervousSystem` proposal executes. If the actual ICP market price is higher than 1 XDR, the Neurons' Fund will over-contribute ICP to the SNS swap by a proportional factor.

### Finding Description
In `rs/nns/governance/src/heap_governance_data.rs` the conversion from the optional proto field to the internal `XdrConversionRate` struct silently falls back to a hardcoded value:

```rust
impl From<Option<ApiXdrConversionRate>> for XdrConversionRate {
    fn from(src: Option<ApiXdrConversionRate>) -> Self {
        let timestamp_seconds = src.as_ref().and_then(|x| x.timestamp_seconds);
        let xdr_permyriad_per_icp = src.as_ref().and_then(|x| x.xdr_permyriad_per_icp);

        match (timestamp_seconds, xdr_permyriad_per_icp) {
            (Some(timestamp_seconds), Some(xdr_permyriad_per_icp)) => Self { ... },
            _ => Self {
                timestamp_seconds: 0,
                xdr_permyriad_per_icp: 10_000,   // ← hardcoded 1 ICP = 1 XDR
            },
        }
    }
}

This value is stored in `HeapGovernanceData::xdr_conversion_rate` and read by `Governance::icp_xdr_rate()` in `rs/nns/governance/src/governance.rs`. That rate is then passed directly into `try_derive_neurons_fund_participation_limits_impl` in `rs/nns/governance/src/neurons_fund.rs`:

```rust
let icp_xdr_rate = self.icp_xdr_rate();
Self::try_derive_neurons_fund_participation_limits_impl(
    neurons_fund_economics,
    icp_xdr_rate,
)
``` [2](#0-1) 

Inside that function the rate is clamped between `minimum_icp_xdr_rate` (default 1 XDR, i.e. `10_000` basis-points) and `maximum_icp_xdr_rate` (default 100 XDR), and then used to convert every XDR-denominated threshold into ICP:

```rust
let icp_xdr_rate = icp_xdr_rate.clamp(minimum_icp_xdr_rate, maximum_icp_xdr_rate);
let xdr_icp_rate = icp_xdr_rate.inv();
let convert_xdr_to_icp = |amount_xdr| amount_xdr.checked_mul(xdr_icp_rate) ...
``` [3](#0-2) 

The fallback value of `10_000` (1 XDR/ICP) equals the governance-configured minimum, so it represents the worst-case scenario: the Neurons' Fund participation limits are computed as if ICP is worth only 1 XDR, regardless of the actual market price.

The default `NeuronsFundEconomics` thresholds are denominated in XDR:

```rust
max_theoretical_neurons_fund_participation_amount_xdr: "750_000.0"
contribution_threshold_xdr:                            "75_000.0"
one_third_participation_milestone_xdr:                 "225_000.0"
full_participation_milestone_xdr:                      "375_000.0"
``` [4](#0-3) 

At the fallback rate (1 XDR/ICP) these convert to ICP amounts that are N× larger than they should be, where N is the actual XDR/ICP market rate.

### Impact Explanation
**Ledger conservation / governance authorization bug.** If the fallback rate is active when a `CreateServiceNervousSystem` proposal executes, the Neurons' Fund participation limits are inflated by a factor equal to the actual ICP price in XDR. For example, if ICP trades at 10 XDR:

| Threshold | Correct ICP (10 XDR/ICP) | Fallback ICP (1 XDR/ICP) | Over-contribution |
|---|---|---|---|
| `max_theoretical` | 75,000 ICP | 750,000 ICP | 10× |
| `contribution_threshold` | 7,500 ICP | 75,000 ICP | 10× |

The Neurons' Fund can commit up to 750,000 ICP of neuron maturity to a single SNS swap instead of the intended 75,000 ICP. This is a direct, irreversible economic loss to all Neurons' Fund participants (NNS neuron holders who opted in).

### Likelihood Explanation
The fallback is active whenever `heap_data.xdr_conversion_rate` is absent or has a `None` sub-field. This occurs:

1. **Fresh governance deployment** — before the CMC has pushed its first rate via `update_icp_xdr_conversion_rate`. The CMC only pushes a rate after its own heartbeat fires and the XRC responds, which can take several minutes.
2. **State migration** — any upgrade that does not explicitly preserve `xdr_conversion_rate` in stable storage resets the field to the fallback.
3. **Partial proto deserialization** — if either `timestamp_seconds` or `xdr_permyriad_per_icp` is missing from the stored proto, the entire struct falls back.

A malicious SNS proposer who monitors governance state can submit a `CreateServiceNervousSystem` proposal timed to execute during the window when the fallback is active (e.g., immediately after a governance upgrade). The proposal passes through the normal NNS voting process, so no privileged access is required.

### Recommendation
1. **Refuse to compute participation limits when no rate is available.** Return an error from `try_derive_neurons_fund_participation_limits` if `xdr_conversion_rate` has `timestamp_seconds == 0` (the sentinel for "never set"), forcing the proposal execution to fail safely rather than silently using a wrong rate.
2. **Validate the rate age before use.** Reject rates whose `timestamp_seconds` is older than a configurable staleness threshold (e.g., 48 hours) and abort the SNS proposal execution.
3. **Persist the rate across upgrades.** Ensure `xdr_conversion_rate` is always serialized to and deserialized from stable storage so that upgrades do not reset it to the fallback.

### Proof of Concept
1. Observe that `heap_data.xdr_conversion_rate` is initialized from the governance proto via `From<Option<ApiXdrConversionRate>> for XdrConversionRate`; if the proto field is absent, `xdr_permyriad_per_icp` is set to `10_000`.
2. Perform a governance canister upgrade that does not preserve `xdr_conversion_rate` in stable storage (or observe a fresh deployment).
3. Before the CMC heartbeat fires and pushes a fresh rate, submit a `CreateServiceNervousSystem` proposal with a large direct-participation target.
4. The proposal executes; `icp_xdr_rate()` returns `Decimal::from(1)` (1 XDR/ICP).
5. `try_derive_neurons_fund_participation_limits_impl` computes `max_theoretical_neurons_fund_participation_amount_icp = 750_000 XDR × (1 ICP/XDR) = 750_000 ICP` instead of the intended ~75,000 ICP at a 10 XDR/ICP market rate.
6. The Neurons' Fund commits up to 750,000 ICP of maturity to the swap — a 10× over-contribution relative to the protocol's intent. [1](#0-0) [5](#0-4) [6](#0-5)

impl Governance {
    fn try_derive_neurons_fund_participation_limits_impl(
        neurons_fund_economics: &NeuronsFundEconomicsPb,
        icp_xdr_rate: Decimal,
    ) -> Result<NeuronsFundParticipationLimits, String> {
        let NeuronsFundEconomics {
            max_theoretical_neurons_fund_participation_amount_xdr,
            contribution_threshold_xdr,
            one_third_participation_milestone_xdr,
            full_participation_milestone_xdr,
            minimum_icp_xdr_rate,
            maximum_icp_xdr_rate,
        } = NeuronsFundEconomics::try_from(neurons_fund_economics)?;

        if icp_xdr_rate <= minimum_icp_xdr_rate {
            println!(
                "{}WARNING: icp_xdr_rate ({}) is being clamped at the lower bound ({}).",
                governance::LOG_PREFIX,
                icp_xdr_rate,
                minimum_icp_xdr_rate,
            );
        }
        if icp_xdr_rate >= maximum_icp_xdr_rate {
            println!(
                "{}WARNING: icp_xdr_rate ({}) is being clamped at the upper bound ({}).",
                governance::LOG_PREFIX,
                icp_xdr_rate,
                maximum_icp_xdr_rate,
            );
        }
        let icp_xdr_rate = icp_xdr_rate.clamp(minimum_icp_xdr_rate, maximum_icp_xdr_rate);

        if icp_xdr_rate.is_zero() {
            // We don't expect this to ever happen in practice.
            return Err("icp_xdr_rate must be greater than zero.".to_string());
        }
        let xdr_icp_rate = icp_xdr_rate.inv();

        let convert_xdr_to_icp = |amount_xdr: Decimal| -> Result<Decimal, String> {
            amount_xdr.checked_mul(xdr_icp_rate).ok_or_else(|| {
                format!(
                    "Cannot convert {amount_xdr} XDR to ICP due to a Decimal overflow. xdr_icp_rate = {xdr_icp_rate}.",
                )
            })
        };

        let max_theoretical_neurons_fund_participation_amount_icp =
            convert_xdr_to_icp(max_theoretical_neurons_fund_participation_amount_xdr)?;

        let contribution_threshold_icp = convert_xdr_to_icp(contribution_threshold_xdr)?;

        let one_third_participation_milestone_icp =
            convert_xdr_to_icp(one_third_participation_milestone_xdr)?;

        let full_participation_milestone_icp =
            convert_xdr_to_icp(full_participation_milestone_xdr)?;

        Ok(NeuronsFundParticipationLimits {
            max_theoretical_neurons_fund_participation_amount_icp,
            contribution_threshold_icp,
            one_third_participation_milestone_icp,
            full_participation_milestone_icp,
        })
    }

    pub fn try_derive_neurons_fund_participation_limits(
        &self,
    ) -> Result<NeuronsFundParticipationLimits, String> {
        let Some(ref economics) = self.heap_data.economics else {
            return Err("Network Economics must be specified.".to_string());
        };

        // The initial values are expected to be populated in `canister_post_upgrade`.
        let Some(ref neurons_fund_economics) = economics.neurons_fund_economics else {
            return Err("Neurons' Fund economics must be specified.".to_string());
        };

        let icp_xdr_rate = self.icp_xdr_rate();

        Self::try_derive_neurons_fund_participation_limits_impl(
            neurons_fund_economics,
            icp_xdr_rate,
        )
    }
