### Title
Duplicate `NeuronPermission` Entries in SNS Neurons for Neurons' Fund Participants — (`rs/sns/governance/src/types.rs`)

---

### Summary

`NeuronRecipe::construct_permissions` in `rs/sns/governance/src/types.rs` builds the initial `Vec<NeuronPermission>` for SNS neurons created during swap finalization without any deduplication check. For Neurons' Fund (NF) participants, it unconditionally pushes one entry for `nns_neuron_controller` and then one entry per hotkey. If a user has registered their own controller principal as a hotkey on their NNS neuron, the same `PrincipalId` appears twice in the resulting permissions Vec. The neuron is then stored with duplicate `NeuronPermission` entries, causing `remove_permissions_for_principal` to leave a stale second entry intact after the first is removed, permanently retaining permissions that should have been revoked.

---

### Finding Description

`construct_permissions` in `rs/sns/governance/src/types.rs` (lines 2353–2407) builds the permissions Vec by unconditional `push` calls:

```
permissions.push(controller_entry);          // NeuronRecipe.controller
permissions.push(nns_neuron_controller_entry); // NeuronsFund.nns_neuron_controller
for hotkey in nns_neuron_hotkeys {
    permissions.push(hotkey_entry);          // each hotkey, no dedup check
}
``` [1](#0-0) 

There is no check that `nns_neuron_controller` is absent from `nns_neuron_hotkeys` before pushing. If the same `PrincipalId` appears in both positions, the resulting `Vec<NeuronPermission>` contains two entries with the same `principal` field.

The validation step in `NeuronRecipe::validate` only checks the raw length of the constructed permissions Vec against `max_number_of_principals_per_neuron`:

```rust
if permissions.len() > max_number_of_principals_per_neuron as usize {
    defects.push(...)
}
``` [2](#0-1) 

It does not check for duplicate principals. If the total count (including duplicates) is still within the limit, the recipe passes validation and the neuron is created with duplicate entries via `claim_swap_neurons`: [3](#0-2) 

The downstream `remove_permissions_for_principal` uses `iter().position()` to find only the **first** matching entry: [4](#0-3) 

After removing all permission types from the first entry and calling `swap_remove`, the second duplicate entry for the same principal remains in the Vec. The function returns `AllPermissionTypesRemoved`, causing the caller to also remove the principal from the `principal_to_neuron_ids_index`, but the neuron's own `permissions` Vec still contains the stale entry. The principal retains effective permissions on the neuron.

Additionally, `add_neuron_permissions` checks `neuron.permissions.len() == max_number_of_principals_per_neuron` to gate new additions: [5](#0-4) 

With inflated length from duplicates, this limit is hit prematurely, blocking legitimate permission grants.

---

### Impact Explanation

An NF participant's SNS neuron is created with duplicate `NeuronPermission` entries for the same principal. After a `RemoveNeuronPermissions` call removes all permission types from the first entry, the second entry survives, granting the principal continued access (e.g., `Vote`, `DisburseMaturity`, `ManagePrincipals`) that governance intended to revoke. The `principal_to_neuron_ids_index` is incorrectly updated to reflect removal, creating an inconsistency between the index and the neuron's actual state. The inflated `permissions.len()` also prematurely blocks other principals from being added to the neuron.

---

### Likelihood Explanation

Any NNS neuron holder who has joined the Neurons' Fund can trigger this by adding their own controller `PrincipalId` as a hotkey. NNS governance's `add_hot_key` only checks for duplicates within the existing hotkeys list, not against the neuron's controller: [6](#0-5) 

The `pick_most_important_hotkeys` function in `rs/nns/governance/src/neurons_fund.rs` deduplicates within the hotkeys list but does not filter out principals that match `nns_neuron_controller`: [7](#0-6) 

So the controller-as-hotkey survives into the `NeuronsFund.nns_neuron_hotkeys` field passed to `construct_permissions`. This is a deliberate, low-effort user action requiring no privileged access.

---

### Recommendation

In `construct_permissions`, collect all principals into a `HashSet` before building the Vec, or check for membership before each `push`:

```rust
let mut seen = HashSet::new();
let mut permissions = vec![];

let push_if_new = |seen: &mut HashSet<PrincipalId>, permissions: &mut Vec<NeuronPermission>, p, perms| {
    if seen.insert(p) {
        permissions.push(NeuronPermission::new(&p, perms));
    }
};

push_if_new(&mut seen, &mut permissions, *controller, neuron_claimer_permissions.permissions);
// ... similarly for nns_neuron_controller and each hotkey
```

Alternatively, add a validation step in `NeuronRecipe::validate` that checks for duplicate principals in the constructed permissions Vec and rejects the recipe if any are found.

---

### Proof of Concept

1. Alice holds an NNS neuron with controller principal `P`.
2. Alice calls `manage_neuron` → `AddHotKey { new_hot_key: P }` on her NNS neuron. This succeeds because `add_hot_key` only checks for duplicates within `hot_keys`, not against the controller.
3. Alice's NNS neuron joins the Neurons' Fund.
4. An SNS swap commits. The swap canister calls `claim_swap_neurons` with a `NeuronRecipe` where `nns_neuron_controller = P` and `nns_neuron_hotkeys = [P, ...]`.
5. `construct_permissions` pushes: `[..., NeuronPermission{principal: P, ...}, NeuronPermission{principal: P, ...}]` — two entries for `P`.
6. The neuron is stored with `permissions.len() == N+1` instead of `N` (where `N` is the expected unique count).
7. A governance action calls `RemoveNeuronPermissions` for principal `P` with all permission types. `remove_permissions_for_principal` finds the first entry at position `i`, removes all its types, calls `swap_remove(i)`, and returns `AllPermissionTypesRemoved`.
8. The second entry for `P` remains in `neuron.permissions`. Alice retains all permissions associated with that entry and can continue to vote, disburse maturity, or manage the neuron. [8](#0-7) [9](#0-8)

    pub fn remove_permissions_for_principal(
        &mut self,
        principal_id: PrincipalId,
        permission_types_to_remove: Vec<i32>,
    ) -> Result<RemovePermissionsStatus, GovernanceError> {
        // Get the position as it will reduce search time in the future.
        let existing_permission_position = self
            .permissions
            .iter()
            .position(|p| p.principal == Some(principal_id))
            .ok_or_else(|| {
                GovernanceError::new_with_message(
                    ErrorType::AccessControlList,
                    format!(
                        "PrincipalId {} does not have any permissions in Neuron {}",
                        principal_id,
                        self.id.as_ref().expect("Neuron must have a NeuronId")
                    ),
                )
            })?;

        let existing_permission = self
            .permissions
            .get_mut(existing_permission_position)
            .expect("Expected permission to exist");

        // Initialize a structure to efficiently remove provided permission_types from
        // existing permission_types.
        let mut remaining_permission_types: HashSet<i32> =
            HashSet::from_iter(existing_permission.permission_type.iter().cloned());

        // Initialize a structure to track if permission_types were present in the existing NeuronPermission
        let mut missing_permissions = HashSet::new();
        for permission_type in &permission_types_to_remove {
            let permission_type_is_present = remaining_permission_types.remove(permission_type);
            if !permission_type_is_present {
                missing_permissions.insert(NeuronPermissionType::try_from(*permission_type).ok());
            }
        }

        if !missing_permissions.is_empty() {
            return Err(GovernanceError::new_with_message(
                ErrorType::AccessControlList,
                format!(
                    "PrincipalId {principal_id} was missing permissions {missing_permissions:?} when removing {permission_types_to_remove:?}"
                ),
            ));
        }

        // If there are no remaining permissions after removing the requested permissions, remove
        // the NeuronPermission entry from the neuron.
        if remaining_permission_types.is_empty() {
            self.permissions.swap_remove(existing_permission_position);
            return Ok(RemovePermissionsStatus::AllPermissionTypesRemoved);
        // If not, update the existing permission with what is left in the remaining permissions.
        } else {
            existing_permission.permission_type = Vec::from_iter(remaining_permission_types);
        }

        Ok(RemovePermissionsStatus::SomePermissionTypesRemoved)
    }

    pub fn pick_most_important_hotkeys(hotkeys: &Vec<PrincipalId>) -> Vec<PrincipalId> {
        // Remove duplicates while preserving the order.
        let mut unique_hotkeys = vec![];
        let mut non_self_auth_hotkeys = vec![];
        let mut observed = HashSet::new();
        for hotkey in hotkeys {
            if !observed.contains(hotkey) {
                observed.insert(*hotkey);
                // Collect hotkeys that are self-authenticating; save non_self_auth_hotkeys for
                // later, in case there is still space for some of them.
                if hotkey.is_self_authenticating() {
                    unique_hotkeys.push(*hotkey);
                } else {
                    non_self_auth_hotkeys.push(*hotkey);
                }
            }
            // Limit how many hotkeys may be collected.
            if unique_hotkeys.len() == MAX_HOTKEYS_FROM_NEURONS_FUND_NEURON {
                break;
            }
        }

        // If there is space in `unique_hotkeys`, fill it up using `non_self_auth_hotkeys`.
        while unique_hotkeys.len() < MAX_HOTKEYS_FROM_NEURONS_FUND_NEURON
            && !non_self_auth_hotkeys.is_empty()
        {
            let non_self_authenticating_hotkey = non_self_auth_hotkeys.remove(0);
            unique_hotkeys.push(non_self_authenticating_hotkey);
        }

        unique_hotkeys
    }
