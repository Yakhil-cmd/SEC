### Title
Neuron Split Resets `voting_power_refreshed_timestamp_seconds`, Allowing Deciding Voting Power Recovery Without Participation - (`File: rs/nns/governance/src/governance.rs`)

---

### Summary

The NNS `split_neuron` operation creates a child neuron whose `voting_power_refreshed_timestamp_seconds` is always set to the current time (`now`), regardless of how stale the parent neuron is. A controller of a neuron whose deciding voting power has been reduced (or zeroed) due to inactivity can split it to create a child neuron with full deciding voting power, partially or fully recovering governance influence without ever casting a vote.

---

### Finding Description

The NNS governance canister implements a deciding voting power mechanism: when a neuron's `voting_power_refreshed_timestamp_seconds` is more than 6 months in the past, its deciding voting power begins declining linearly to zero over the following month. After 7 months of inactivity, deciding voting power reaches 0 and following is cleared.

In `split_neuron`, the child neuron is constructed via `NeuronBuilder::new` with `created_timestamp_seconds` as its fifth argument: [1](#0-0) 

The `NeuronBuilder` uses `created_timestamp_seconds` as the default value for `voting_power_refreshed_timestamp_seconds`. No `.with_voting_power_refreshed_timestamp_seconds(...)` call is made to inherit the parent's (potentially stale) timestamp. The child therefore always starts with a fresh `voting_power_refreshed_timestamp_seconds = now`.

### Impact Explanation

A neuron controller whose neuron has been inactive for ≥ 7 months (deciding voting power = 0) can call `split_neuron` to create a child neuron with `voting_power_refreshed_timestamp_seconds = now`. The child inherits the parent's `dissolve_state_and_age` (age bonus preserved) and a proportional share of stake, but starts with full deciding voting power.

By splitting into N child neurons (limited only by the minimum stake requirement), the attacker can recover up to their full potential deciding voting power without voting on any proposal. This directly undermines the NNS mechanism designed to reduce the governance influence of inactive participants.

- **Governance accounting integrity**: The deciding VP staleness mechanism is bypassed.
- **Voting power inflation**: A coalition of inactive large neuron holders can recover full deciding VP on demand before a critical vote.
- **Minimum stake bound**: The NNS minimum stake (e.g., 1 ICP) limits the number of splits, but large neurons (e.g., 10,000 ICP) can split into thousands of child neurons.

---

### Likelihood Explanation

- Entry path: any NNS neuron controller can call `manage_neuron` with a `Split` command via ingress message — no privileged access required.
- The operation is cheap (one ledger transfer) and the split can be done immediately before a governance vote.
- The parent neuron's deciding VP is not restored, but the child's is, so the net effect is a partial-to-full recovery depending on split ratio.
- No threshold corruption, admin key, or social engineering is required.

---

### Recommendation

When constructing the child neuron in `split_neuron`, inherit the parent's `voting_power_refreshed_timestamp_seconds` rather than defaulting to `created_timestamp_seconds`:

```rust
let child_neuron = NeuronBuilder::new(
    child_nid,
    to_subaccount,
    *caller,
    parent_neuron.dissolve_state_and_age(),
    created_timestamp_seconds,
)
// Add this:
.with_voting_power_refreshed_timestamp_seconds(
    parent_neuron.voting_power_refreshed_timestamp_seconds()
)
// ... rest of builder
.build();
```

This ensures that the child neuron carries the same staleness as the parent, preserving the economic intent of the deciding voting power mechanism. The child can then legitimately refresh its voting power by voting or calling `refresh_voting_power`.

---

### Proof of Concept

1. Create NNS neuron N with stake 1000 ICP, set dissolve delay ≥ 6 months.
2. Wait 7+ months without voting. Neuron N now has `deciding_voting_power = 0`.
3. Call `manage_neuron` with `Split { amount_e8s: 500_000_000_000, memo: 1 }` (500 ICP).
4. Observe child neuron C: `voting_power_refreshed_timestamp_seconds = now`, `deciding_voting_power = potential_voting_power` (full).
5. Repeat splits on C to further subdivide and recover more deciding VP.
6. Total deciding VP recovered ≈ `split_fraction × potential_voting_power` per split, without casting a single vote.

The existing test at `rs/nns/governance/tests/governance.rs` line 5602–5616 already documents and asserts this behavior (child VP refresh > parent VP refresh), confirming the root cause is present in production code. [1](#0-0) [3](#0-2) [2](#0-1)

        now_seconds: u64,
    ) -> (u64, u64) {
        let stake_e8s = self.stake_e8s();
        let boost = dissolve_delay_bonus_multiplier(self.dissolve_delay_seconds(now_seconds))
            * age_bonus_multiplier(self.age_seconds(now_seconds));
        let mut potential_voting_power = Decimal::from(stake_e8s) * boost;

        // 8 Year Gang bonus. Cap the bonus base to the current stake because
        // rejection fees can cause the bonus base to exceed stake_e8s.
        if is_mission_70_voting_rewards_enabled() {
            let eight_year_gang_bonus_base_e8s = self.eight_year_gang_bonus_base_e8s.min(stake_e8s);
            potential_voting_power +=
                Decimal::from(eight_year_gang_bonus_base_e8s) / Decimal::from(10) * boost;
        }

        // For DECIDING voting power.

            let time_since_last_refreshed = Duration::from_secs(
                now_seconds.saturating_sub(self.voting_power_refreshed_timestamp_seconds),
            );

            voting_power_economics
                .deciding_voting_power_adjustment_factor(time_since_last_refreshed)
        };

        let deciding_voting_power = adjustment_factor * potential_voting_power.floor();
