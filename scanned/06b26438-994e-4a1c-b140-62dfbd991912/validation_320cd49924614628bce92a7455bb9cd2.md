### Title
SNS Governance Proposal Queue Saturation Enables Denial-of-Service Against Legitimate Proposals - (File: rs/sns/governance/src/governance.rs)

### Summary
The SNS governance canister enforces a global cap of 700 unsettled proposals (`MAX_NUMBER_OF_PROPOSALS_WITH_BALLOTS`). There is no per-neuron or per-proposer limit on open proposals. A well-funded attacker controlling neurons with sufficient aggregate stake can fill this queue with spam proposals, blocking all non-emergency governance actions (including `ManageNervousSystemParameters`) for the full voting period of each wave.

### Finding Description
In `rs/sns/governance/src/governance.rs`, the `make_proposal` function enforces a single global ceiling check:

```rust
if self
    .proto
    .proposals
    .values()
    .filter(|data| !data.ballots.is_empty())
    .count()
    >= MAX_NUMBER_OF_PROPOSALS_WITH_BALLOTS   // hardcoded 700
    && !proposal.allowed_when_resources_are_low()
{
    return Err(GovernanceError::new_with_message(
        ErrorType::ResourceExhausted,
        "Reached maximum number of proposals that have not yet \
        been taken into account for voting rewards. \
        Please try again later.",
    ));
}
``` [1](#0-0) 

The constant is defined as:

```rust
pub const MAX_NUMBER_OF_PROPOSALS_WITH_BALLOTS: usize = 700;
``` [2](#0-1) 

The only proposals exempt from this cap are those whose action returns `true` from `allowed_when_resources_are_low()`:

```rust
pub(crate) fn allowed_when_resources_are_low(&self) -> bool {
    match self {
        Action::UpgradeSnsControlledCanister(_) => true,
        Action::UpgradeSnsToNextVersion(_) => true,
        Action::ExecuteGenericNervousSystemFunction(_) => true,
        _ => false,
    }
}
``` [3](#0-2) 

This means `ManageNervousSystemParameters`, `Motion`, `TransferSnsTreasuryFunds`, `MintSnsTokens`, `RegisterDappCanisters`, `DeregisterDappCanisters`, `ManageSnsMetadata`, and `ManageLedgerParameters` proposals are all blocked when the queue is full.

The only economic deterrent is the `reject_cost_e8s` (default: 1 governance token = `E8S_PER_TOKEN`), which is charged upfront as `neuron_fees_e8s` and burned on rejection:

```rust
neuron.neuron_fees_e8s += proposal_data.reject_cost_e8s;
``` [4](#0-3) 

The stake check only verifies that `proposer.stake_e8s() >= reject_cost_e8s` at submission time:

```rust
if proposer.stake_e8s() < reject_cost_e8s {
    return Err(GovernanceError::new_with_message(
        ErrorType::PreconditionFailed,
        "Neuron doesn't have enough stake to submit proposal.",
    ));
}
``` [5](#0-4) 

Because `stake_e8s()` = `cached_neuron_stake_e8s - neuron_fees_e8s`, a single neuron with N tokens can submit N proposals before being blocked. An attacker distributing stake across multiple neurons can submit 700 proposals in parallel, saturating the queue.

The default voting period is 4 days, extendable to 30 days via wait-for-quiet:

```rust
initial_voting_period_seconds: Some(4 * ONE_DAY_SECONDS), // 4d
``` [6](#0-5) 

Ballots are only cleared after the voting period ends and rewards are settled, so the queue remains blocked for the full duration. The attacker can immediately re-fill cleared slots.

Critically, the community cannot pass a `ManageNervousSystemParameters` proposal to raise `reject_cost_e8s` while the queue is full, because that proposal type is not exempt from the cap.

### Impact Explanation
A well-funded attacker can permanently block all non-emergency SNS governance proposals for any SNS instance. This prevents the SNS community from:
- Changing governance parameters (including raising the proposal fee to deter the attack)
- Transferring treasury funds
- Minting SNS tokens
- Registering or deregistering dapp canisters
- Updating SNS metadata or ledger parameters

The only proposals that can still pass are canister upgrades and generic nervous system functions. The SNS is effectively governance-frozen for all other actions for as long as the attacker continues to refill the queue.

### Likelihood Explanation
The attack requires 700 governance tokens (at default `reject_cost_e8s = 1 token`) distributed across neurons with sufficient dissolve delay. For a newly launched SNS with low token value, this cost may be trivial for a motivated competitor or adversary. The attacker loses these tokens when proposals are rejected, but can repeat the attack indefinitely if the token cost remains low relative to the damage caused. There is no rate limiting, no per-neuron proposal cap, and no mechanism for the community to remove spam proposals from the queue.

### Recommendation
- **Short term**: Enforce a per-neuron limit on the number of simultaneously open proposals (e.g., no more than 5 open proposals per neuron). This prevents any single actor from monopolizing the queue regardless of stake.
- **Short term**: Add `ManageNervousSystemParameters` to the `allowed_when_resources_are_low` exemption list so the community can always raise `reject_cost_e8s` to deter spam even when the queue is full.
- **Long term**: Implement a proposal removal mechanism (e.g., a supermajority vote to immediately close a proposal and burn its fee) to allow the community to clear spam without waiting for the full voting period.

### Proof of Concept
1. Attacker acquires 700 SNS governance tokens and distributes them across 700 neurons (or fewer neurons with more tokens each), each with dissolve delay ≥ `neuron_minimum_dissolve_delay_to_vote_seconds`.
2. Attacker calls `manage_neuron` with `Command::MakeProposal(Motion { motion_text: "spam" })` 700 times across their neurons.
3. Each call passes the stake check (`stake_e8s() >= reject_cost_e8s = 1 token`) and the global count check (count < 700).
4. After 700 submissions, `self.proto.proposals.values().filter(|data| !data.ballots.is_empty()).count() == 700 >= MAX_NUMBER_OF_PROPOSALS_WITH_BALLOTS`.
5. Any subsequent `ManageNervousSystemParameters` proposal from a legitimate community member returns `ResourceExhausted`.
6. The queue remains full for 4+ days (default voting period). As proposals expire and are settled, the attacker immediately submits new ones.
7. The community cannot pass a `ManageNervousSystemParameters` proposal to raise `reject_cost_e8s` because that action is not in `allowed_when_resources_are_low`. [2](#0-1) [1](#0-0) [7](#0-6)
