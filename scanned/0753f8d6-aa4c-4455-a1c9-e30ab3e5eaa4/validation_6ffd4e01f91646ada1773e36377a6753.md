### Title
NNS `disburse_neuron` Burns All Fees Including Refundable Open-Proposal Fees, Causing Permanent ICP Loss — (`rs/nns/governance/src/governance.rs`)

---

### Summary

NNS `disburse_neuron` unconditionally burns the entire `neuron_fees_e8s` balance and zeroes the field. If the neuron has open proposals whose `reject_cost_e8s` would be refunded on adoption, those fees are burned prematurely and the subsequent refund path silently no-ops, causing the neuron owner to permanently lose ICP equal to the sum of `reject_cost_e8s` for every open proposal that is later adopted.

SNS governance fixed this identical bug in proposal 137687 (CHANGELOG entry: *"Fixed a bug that could allow an SNS Neuron to burn fees that would have been refunded after proposal acceptance"*).

---

### Finding Description

**NNS burn path — burns 100% of fees:**

`fees_amount_e8s` is read directly from `neuron.neuron_fees_e8s` with no deduction for open proposals. [1](#0-0) 

The full amount is burned and `neuron_fees_e8s` is hard-set to `0`: [2](#0-1) 

**NNS proposal-adoption refund path — silently skipped:**

When a proposal is adopted, NNS tries to refund `reject_cost_e8s` by decrementing `neuron_fees_e8s`, but only if `neuron_fees_e8s >= rejection_cost`. After disburse has zeroed the field, this guard always fails and the refund is silently dropped: [3](#0-2) 

**SNS fixed path — only burns the safe portion:**

SNS computes `max_burnable_fee = neuron_fees_e8s - Σ(reject_cost_e8s for open proposals)` and burns only that, leaving the rest in `neuron_fees_e8s` so the refund can succeed later: [4](#0-3) 

The helper that computes the safe burn ceiling: [5](#0-4) 

---

### Impact Explanation

For every open proposal that is later adopted, the neuron owner loses `reject_cost_e8s` ICP permanently. On NNS mainnet the current `reject_cost_e8s` is 10 ICP. A neuron with N open proposals that are all adopted loses N × 10 ICP. The loss is irreversible: the tokens were burned on the ledger and the governance refund path silently no-ops.

---

### Likelihood Explanation

The preconditions are realistic and reachable by any unprivileged neuron controller:

1. Submit one or more proposals while the neuron is dissolving (or before starting to dissolve). `neuron_fees_e8s` is incremented by `reject_cost_e8s` per proposal.
2. Wait for the neuron to reach the `Dissolved` state.
3. Call `ManageNeuron::Disburse` — a standard, permissionless user action.
4. One or more of the open proposals is later adopted.

There is no warning in the UI or the API that disbursing while holding open proposals will forfeit the potential refund. A user who submits a governance proposal, starts dissolving, and disburses as soon as the neuron is dissolved will trigger this silently.

---

### Recommendation

Port the SNS fix to NNS. Before burning, compute the maximum safe burn amount:

```
max_burnable = neuron_fees_e8s
             - Σ reject_cost_e8s for all open proposals where proposer == this neuron
```

Burn only `max_burnable`, and subtract only `max_burnable` from `neuron_fees_e8s` (using `saturating_sub`), mirroring the SNS implementation at `rs/sns/governance/src/governance.rs:1243-1268`. [5](#0-4) 

---

### Proof of Concept

**State setup:**
- Neuron: `cached_neuron_stake_e8s = 1_100_000_000` (11 ICP), `neuron_fees_e8s = 1_000_000_000` (10 ICP, from one rejected proposal), dissolved.
- Open proposal: `reject_cost_e8s = 1_000_000_000`, proposer = this neuron, `decided_timestamp_seconds = 0` (still open).

**Step 1 — call `disburse_neuron`:**
- `fees_amount_e8s = 1_000_000_000` (full fees, no open-proposal deduction)
- Ledger burn: 1_000_000_000 e8s destroyed
- `neuron_fees_e8s` set to `0`
- User receives `~100_000_000 - tx_fee` e8s (the remaining stake)

**Step 2 — proposal is adopted (`process_proposal`):**
```
rejection_cost = 1_000_000_000
neuron.neuron_fees_e8s = 0
guard: 0 >= 1_000_000_000  →  false  →  refund silently skipped
```

**Result:** User burned 10 ICP that should have been refunded. Net loss = 10 ICP.

**SNS behaviour with identical state:** `max_burnable_fee = 1_000_000_000 - 1_000_000_000 = 0`. No burn occurs. After proposal adoption, `neuron_fees_e8s -= 1_000_000_000` succeeds, and the user's effective stake is restored. No ICP lost. [6](#0-5)

    fn maximum_burnable_fees_for_neuron(&self, neuron: &Neuron) -> Result<u64, GovernanceError> {
        let neuron_id = neuron.id.as_ref().ok_or_else(|| {
            GovernanceError::new_with_message(ErrorType::NotFound, "Neuron does not have an ID")
        })?;

        // Calculate the total reject costs from all open proposals submitted by this neuron
        let total_open_proposal_reject_costs = self
            .proto
            .proposals
            .values()
            .filter(|proposal_data| {
                // Only consider open proposals where this neuron is the proposer
                proposal_data.proposer.as_ref() == Some(neuron_id)
                    && proposal_data.status() == ProposalDecisionStatus::Open
            })
            .map(|proposal_data| proposal_data.reject_cost_e8s)
            .sum::<u64>();

        // The maximum burnable amount is the total fees minus any fees that are
        // tied up in open proposals (which could potentially be refunded)
        let max_burnable = neuron
            .neuron_fees_e8s
            .saturating_sub(total_open_proposal_reject_costs);

        Ok(max_burnable)
    }
