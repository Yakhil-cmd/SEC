### Title
SNS Governance Marks `ExecuteGenericNervousSystemFunction` Proposals as Executed Regardless of Application-Level Return Value - (File: rs/sns/governance/src/canister_control.rs)

---

### Summary

`perform_execute_generic_nervous_system_function_call` in the SNS governance canister unconditionally treats any non-rejected inter-canister reply as `Ok(())`, permanently marking the adopted proposal as `Executed` even when the target canister returns an application-level error. This is the direct IC analog of the Ethereum `Timelock::executeTransaction()` bug: queued/adopted state is irrevocably consumed without verifying that the intended operation actually succeeded.

---

### Finding Description

When an SNS governance proposal with action `ExecuteGenericNervousSystemFunction` is adopted, `perform_action` dispatches to `perform_execute_generic_nervous_system_function`, which calls `perform_execute_generic_nervous_system_function_call`. [1](#0-0) 

The function makes an inter-canister call to the registered `target_canister_id` / `target_method`. The `Ok(_reply)` branch **discards the reply bytes entirely** and returns `Ok(())`. The code even contains a self-documenting TODO acknowledging the missing check:

```
// For now, any reply is considered a success.
Ok(())
```

The documented method signature for a `GenericNervousSystemFunction` target is `Result<(), String>`: [2](#0-1) [3](#0-2) 

When `perform_execute_generic_nervous_system_function_call` returns `Ok(())`, `set_proposal_execution_status` sets `proposal.executed_timestamp_seconds` to the current time: [4](#0-3) 

`ProposalDecisionStatus::Executed` is a **terminal state**: [5](#0-4) 

Once set, the proposal cannot be re-executed. The SNS community must submit a new proposal and repeat the full governance voting cycle.

---

### Impact Explanation

An adopted `ExecuteGenericNervousSystemFunction` proposal is permanently finalized as `Executed` even when the target canister's method returns `Err(...)` (application-level failure). The intended on-chain operation never takes effect, but the governance record shows success. The SNS community loses the proposal slot and must re-propose, re-vote, and wait through the full voting period again — with no guarantee the re-submitted proposal will not suffer the same fate.

---

### Likelihood Explanation

The target method signature is explicitly `Result<(), String>`. Any target canister that returns `Err(...)` — due to a transient error, a precondition failure, or a deliberately adversarial implementation — triggers this path. Because `GenericNervousSystemFunction` targets are arbitrary canisters, this is a realistic and reachable condition. The attacker-controlled entry path is: any SNS token holder with sufficient voting power submits an `ExecuteGenericNervousSystemFunction` proposal targeting a canister whose method returns `Err(...)`. No privileged access is required beyond normal SNS participation.

---

### Recommendation

Decode the reply bytes and propagate application-level errors as `Err(GovernanceError)`. The code itself suggests the fix:

```rust
Ok(reply) => {
    match candid::Decode!(&reply, Result<(), String>) {
        Ok(Ok(())) => Ok(()),
        Ok(Err(e)) => Err(GovernanceError::new_with_message(
            ErrorType::External,
            format!("Target canister returned application error: {e}"),
        )),
        Err(e) => Err(GovernanceError::new_with_message(
            ErrorType::External,
            format!("Failed to decode reply: {e}"),
        )),
    }
}
``` [2](#0-1)

### Proof of Concept

1. Register a `GenericNervousSystemFunction` whose `target_method` always returns `Err("intentional failure".to_string())`.
2. Submit and adopt an `ExecuteGenericNervousSystemFunction` proposal targeting that function.
3. After execution, query the proposal: `status()` returns `ProposalDecisionStatus::Executed` and `executed_timestamp_seconds > 0`.
4. The target canister's state is unchanged (the operation never succeeded).
5. The proposal is in a terminal state and cannot be re-executed.

The call chain is:

```
perform_action (governance.rs:2172)
  → perform_execute_generic_nervous_system_function (governance.rs:2531)
    → perform_execute_generic_nervous_system_function_call (canister_control.rs:278)
      → env.call_canister(...).await → Ok(Err("failure"))
      → Ok(_reply) branch → Ok(())   ← reply discarded
  → set_proposal_execution_status(pid, Ok(()))
    → proposal.executed_timestamp_seconds = now  ← permanently Executed
``` [6](#0-5) [7](#0-6) [8](#0-7) [9](#0-8)

pub async fn perform_execute_generic_nervous_system_function_call(
    env: &dyn Environment,
    function: NervousSystemFunction,
    call: ExecuteGenericNervousSystemFunction,
) -> Result<(), GovernanceError> {
    // Get the canister id and the method against which we execute the proposal.
    let valid_function = ValidGenericNervousSystemFunction::try_from(&function)
        .map_err(|e| GovernanceError::new_with_message(ErrorType::InvalidProposal, e))?;

    let result = env
        .call_canister(
            valid_function.target_canister_id,
            &valid_function.target_method,
            call.payload,
        )
        .await;

    // Convert result.
    match result {
        Err(err) => Err(GovernanceError::new_with_message(
            ErrorType::External,
            format!("Canister method call to execute proposal failed: {err:?}"),
        )),

        Ok(_reply) => {
            // TODO: Do something with reply. E.g. store it in the proposal,

            //
            // This could then be converted into a Result<(), GovernanceError>.
            // For now, any reply is considered a success.
            Ok(())
        }
    }
}
