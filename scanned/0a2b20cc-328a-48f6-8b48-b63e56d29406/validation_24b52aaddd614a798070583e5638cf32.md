### Title
Unbounded `transaction_fee_e8s` and `reject_cost_e8s` in SNS `NervousSystemParameters` Allows Governance Freeze via `ManageNervousSystemParameters` Proposal ‚Äî (File: `rs/sns/governance/src/types.rs`)

---

### Summary

The SNS governance canister accepts `ManageNervousSystemParameters` proposals that can set `transaction_fee_e8s` and `reject_cost_e8s` to any `u64` value. The validation functions for these two fields check only that the fields are `Some(...)` ‚Äî they impose no upper bound. A governance majority (or a single large token holder) can set either value to `u64::MAX`, permanently freezing SNS ledger operations or blocking all future proposals.

---

### Finding Description

`NervousSystemParameters` in the SNS governance canister contains two fee fields that are updatable via on-chain proposals:

- `transaction_fee_e8s`: the fee charged for every SNS ledger transaction (staking, disbursing, transfers).
- `reject_cost_e8s`: the ICP/token cost deducted from a neuron's stake when it submits a proposal.

The `validate()` method called at both proposal submission time and execution time delegates to two sub-validators:

**`validate_transaction_fee_e8s`** ‚Äî only checks presence: [1](#0-0) 

**`validate_reject_cost_e8s`** ‚Äî only checks presence: [2](#0-1) 

Both functions return `Ok(value)` for any non-`None` `u64`, including `u64::MAX`. No ceiling constant exists for either field, unlike parameters such as `max_proposals_to_keep_per_action` which has `MAX_PROPOSALS_TO_KEEP_PER_ACTION_CEILING`: [3](#0-2) 

The full `validate()` call chain is: [4](#0-3) 

The proposal execution path calls `perform_manage_nervous_system_parameters`, which calls `new_params.validate()` and, if it passes, writes the new parameters: [5](#0-4) 

The proposal validation path calls `validate_and_render_manage_nervous_system_parameters`, which also calls `validate()`: [6](#0-5) 

Both paths accept `transaction_fee_e8s = u64::MAX` without error.

### Impact Explanation

**Scenario A ‚Äî `transaction_fee_e8s` set to `u64::MAX`:**  
Every SNS ledger transaction (staking a neuron, disbursing, transferring tokens) requires paying `transaction_fee_e8s`. If this is set to `u64::MAX`, no account can ever hold enough tokens to pay the fee. All ledger operations fail. Neurons cannot be staked, dissolved, or disbursed. The SNS ledger is permanently frozen.

**Scenario B ‚Äî `reject_cost_e8s` set to `u64::MAX`:**  
Before submitting a proposal, the governance canister checks that the proposing neuron's stake is at least `reject_cost_e8s`: [7](#0-6) 

With `reject_cost_e8s = u64::MAX`, no neuron can ever have sufficient stake to submit a proposal. Governance is permanently frozen ‚Äî no further proposals (including a corrective one) can be submitted.

Both scenarios are irreversible without an NNS-level canister upgrade intervention.

---

### Likelihood Explanation

Many SNS deployments launch with the founding team holding a supermajority of voting power. A single large token holder, or a small coordinating group, can pass a `ManageNervousSystemParameters` proposal unilaterally. A typo (e.g., entering `u64::MAX` instead of `10_000`) or a malicious insider with majority stake can trigger this. The attack requires no special cryptographic capability, no subnet compromise, and no NNS majority ‚Äî only an SNS governance majority, which is a realistic threshold for many deployed SNS instances.

---

### Recommendation

Add upper-bound ceiling constants for `transaction_fee_e8s` and `reject_cost_e8s` in `NervousSystemParameters`, analogous to the existing ceilings for other parameters. For example:

```rust
pub const TRANSACTION_FEE_E8S_CEILING: u64 = 1_000_000_000; // 10 tokens
pub const REJECT_COST_E8S_CEILING: u64 = 1_000_000_000_000; // 10,000 tokens
```

Enforce these in `validate_transaction_fee_e8s` and `validate_reject_cost_e8s`. Additionally, consider requiring that `reject_cost_e8s` never exceed `neuron_minimum_stake_e8s`, so that any neuron meeting the minimum stake requirement can always submit a proposal.

---

### Proof of Concept

1. Deploy an SNS with default parameters (`transaction_fee_e8s = 10_000`, `reject_cost_e8s = 100_000_000`).
2. As a neuron holder with majority voting power, submit a `ManageNervousSystemParameters` proposal:
   ```
   NervousSystemParameters {
       transaction_fee_e8s: Some(u64::MAX),  // 18_446_744_073_709_551_615
       ..Default::default()
   }
   ```
3. The proposal passes `validate_and_render_manage_nervous_system_parameters` ‚Üí `validate()` ‚Üí `validate_transaction_fee_e8s()` which returns `Ok(u64::MAX)` with no error. [8](#0-7) 
4. The proposal is adopted and executed. `perform_manage_nervous_system_parameters` writes `transaction_fee_e8s = u64::MAX` to state.
5. All subsequent SNS ledger transactions fail because no account balance can cover the fee. The SNS is frozen.

    fn perform_manage_nervous_system_parameters(
        &mut self,
        proposed_params: NervousSystemParameters,
    ) -> Result<(), GovernanceError> {
        // Only set `self.proto.parameters` if "applying" the proposed params to the
        // current params results in valid params
        let new_params = proposed_params.inherit_from(self.nervous_system_parameters_or_panic());

        log!(
            INFO,
            "Setting Governance nervous system params to: {:?}",
            &new_params
        );

        match new_params.validate() {
            Ok(()) => {
                self.proto.parameters = Some(new_params);
                Ok(())
            }

            // Even though proposals are validated when they are first made, this is still
            // possible, because the inner value of a ManageNervousSystemParameters
            // proposal is only valid with respect to the current
            // nervous_system_parameters() at the time when the proposal was first
            // made. If nervous_system_parameters() changed (by another proposal) since
            // the current proposal was first made, the current proposal might have become
            // invalid. Basically, this might occur if there are conflicting (concurrent)
            // proposals, but we expect this to be highly unusual in practice.
            Err(msg) => Err(GovernanceError::new_with_message(
                ErrorType::PreconditionFailed,
                format!(
                    "Failed to perform ManageNervousSystemParameters action, proposed \
                        parameters would lead to invalid NervousSystemParameters: {msg}"
                ),
            )),
        }
    }
