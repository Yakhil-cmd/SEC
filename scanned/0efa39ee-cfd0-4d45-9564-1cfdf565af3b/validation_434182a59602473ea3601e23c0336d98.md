### Title
Hardcoded Default Authorized Caller in Engine Controller Canister Regains Privileges After Any Upgrade Without Explicit Override - (`File: rs/engine_controller/canister/canister.rs`)

### Summary

The `engine_controller` canister hard-codes a single privileged principal (`DEFAULT_AUTHORIZED_CALLER`) that is unconditionally restored as the sole authorized caller on every `post_upgrade` invocation that does not explicitly supply an `authorized_caller` override. This mirrors the Axelar `validateSender` flaw: just as that contract implicitly trusted any address matching the `InterchainTokenService` address regardless of source chain, the engine controller implicitly trusts a fixed EOA-equivalent principal regardless of any prior governance decision to replace it. The result is that the hardcoded principal can never be permanently stripped of the ability to create CloudEngine subnets, modify their admin sets, delete them, or deploy GuestOS to their nodes.

### Finding Description

`rs/engine_controller/canister/canister.rs` hard-codes a single authorized principal:

```rust
const DEFAULT_AUTHORIZED_CALLER: &str =
    "bct5z-vccu4-6q4t2-3lb6l-wm43p-ulppt-o5sqq-w6het-rthdz-qp4yn-fqe";
``` [1](#0-0) 

The `apply_init_args` helper, called from both `init` and `post_upgrade`, falls back to this constant whenever the upgrade argument is `None` or its `authorized_caller` field is `None`:

```rust
fn apply_init_args(args: Option<EngineControllerInitArgs>) {
    let args = args.unwrap_or_default();
    let authorized = args
        .authorized_caller
        .unwrap_or_else(default_authorized_caller);   // ← always reverts
    AUTHORIZED_CALLER.with(|c| *c.borrow_mut() = authorized);
    ...
}
``` [2](#0-1) 

`post_upgrade` unconditionally delegates to `apply_init_args`:

```rust
#[post_upgrade]
fn post_upgrade(args: Option<EngineControllerInitArgs>) {
    apply_init_args(args);
}
``` [3](#0-2) 

The test suite explicitly documents and verifies this reversion:

```
// Upgrade with no override: default principal becomes authorized again.
``` [4](#0-3) 

Every privileged endpoint (`create_engine`, `delete_engine`, `update_subnet`, `deploy_guestos_to_all_subnet_nodes`) is gated solely by `ensure_authorized`, which compares `msg_caller()` against the in-memory `AUTHORIZED_CALLER`: [5](#0-4) 

Additionally, `normalize_subnet_admins` — called inside `update_subnet` — unconditionally injects the current `AUTHORIZED_CALLER` into every subnet admin list it processes, meaning the hardcoded principal is silently embedded as a permanent subnet admin in every CloudEngine subnet touched while it holds authority: [6](#0-5) 

### Impact Explanation

An operator who wishes to transfer authority to a governance multisig upgrades the canister with `authorized_caller = Some(multisig_principal)`. This works correctly until the next upgrade that omits the field — a routine NNS proposal that does not explicitly re-specify the override. At that point `DEFAULT_AUTHORIZED_CALLER` silently regains sole authority. The hardcoded principal can then:

1. Call `create_engine` → submit `registry.create_subnet` for a new CloudEngine subnet, with itself injected as a subnet admin.
2. Call `update_subnet` → modify the admin set of any existing CloudEngine subnet (and inject itself via `normalize_subnet_admins`).
3. Call `delete_engine` → delete a CloudEngine subnet.
4. Call `deploy_guestos_to_all_subnet_nodes` → roll out arbitrary GuestOS to all nodes of a CloudEngine subnet.

The registry canister enforces that only governance or the engine controller may call `create_subnet`/`update_subnet` for CloudEngine subnets: [7](#0-6) [8](#0-7) 

So the hardcoded principal, once it regains authority, has the same effective power over CloudEngine subnets as NNS governance itself.

### Likelihood Explanation

The trigger is a standard NNS upgrade proposal for the engine controller that omits `authorized_caller` from the upgrade argument — a realistic operational mistake, especially as the canister evolves and upgrade templates are reused. The hardcoded principal is a single-key EOA-equivalent, not a multisig. If that key is compromised (or its holder acts maliciously), the attacker needs only to wait for any such upgrade. The probability is low but the impact is critical, matching the Axelar judge's reasoning for Medium severity.

### Recommendation

Remove the unconditional fallback to `DEFAULT_AUTHORIZED_CALLER` in `post_upgrade`. Instead, persist the current `AUTHORIZED_CALLER` in stable memory and restore it on upgrade, only accepting an explicit override when one is provided. This ensures that once authority is transferred to a governance multisig it cannot be silently reverted by a careless upgrade argument.

```rust
#[post_upgrade]
fn post_upgrade(args: Option<EngineControllerInitArgs>) {
    // Restore from stable memory first, then apply any explicit override.
    let current = load_authorized_caller_from_stable_memory();
    AUTHORIZED_CALLER.with(|c| *c.borrow_mut() = current);
    if let Some(args) = args {
        if let Some(caller) = args.authorized_caller {
            AUTHORIZED_CALLER.with(|c| *c.borrow_mut() = caller);
        }
    }
}
```

### Proof of Concept

1. Engine controller is deployed; `AUTHORIZED_CALLER` = `DEFAULT_AUTHORIZED_CALLER` (hardcoded EOA).
2. NNS proposal upgrades the canister with `authorized_caller = Some(governance_multisig)`. The EOA is now locked out.
3. A subsequent NNS upgrade proposal (e.g., a routine version bump) is submitted without specifying `authorized_caller`. The proposal passes.
4. `post_upgrade(None)` executes → `apply_init_args(None)` → `authorized_caller.unwrap_or_else(default_authorized_caller)` → `AUTHORIZED_CALLER` is reset to the hardcoded EOA.
5. The EOA holder calls `create_engine` with a chosen set of nodes and themselves as `subnet_admins`. The registry creates a new CloudEngine subnet with the EOA as a permanent admin.
6. The EOA holder calls `update_subnet` targeting an existing CloudEngine subnet; `normalize_subnet_admins` injects the EOA into the admin list, giving persistent subnet-level control even after the engine controller is re-locked. [9](#0-8) [3](#0-2) [10](#0-9) [6](#0-5)

/// Ensures that the configured `AUTHORIZED_CALLER` (the engine controller's
/// "super admin") is always present in the resulting admin list, even if the

fn normalize_subnet_admins(admins: Vec<PrincipalId>) -> Vec<PrincipalId> {
    let super_admin = PrincipalId(AUTHORIZED_CALLER.with(|c| *c.borrow()));
    let mut admins = admins;
    if !admins.contains(&super_admin) {
        admins.push(super_admin);
