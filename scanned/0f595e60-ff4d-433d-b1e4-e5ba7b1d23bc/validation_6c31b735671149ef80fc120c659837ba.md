### Title
SNS Neuron Stake Permanently Locked When All Permissions Are Removed - (`File: rs/sns/governance/src/governance.rs`)

### Summary

The SNS governance canister allows a neuron owner to call `RemoveNeuronPermissions` to strip all permissions from all principals associated with a neuron. Once this is done, the staked governance tokens held in the neuron's ledger subaccount become permanently inaccessible — no principal can disburse, dissolve, or otherwise recover the stake. This is the direct IC analog of the NFTR "lost name after NFT burn" pattern: a valuable asset (staked tokens) becomes permanently orphaned when the owning entity loses all controlling principals.

### Finding Description

The SNS `manage_neuron` endpoint exposes a `RemoveNeuronPermissions` command. The `remove_neuron_permissions` function in `rs/sns/governance/src/governance.rs` removes permission entries for a given `PrincipalId` from a neuron's ACL. The function explicitly documents that removing all permissions from all principals is permitted and results in a neuron that can no longer be modified:

> "If all the permissions are removed from the Neuron i.e. by removing all permissions for all PrincipalIds, the Neuron is not deleted. This is a dangerous operation as it is possible to remove all permissions for a neuron and no longer be able to modify its state, i.e. disbursing the neuron back into the governance token."

The code performs no guard to prevent this outcome. After the call succeeds, the neuron persists in governance state with `permissions = []`, but no principal holds `Disburse`, `ManagePrincipals`, `ConfigureDissolveState`, or any other permission. The staked tokens remain locked in the neuron's ledger subaccount indefinitely.

The same pattern is acknowledged in the Protobuf definition and the generated Rust bindings: [1](#0-0) [2](#0-1) 

The `remove_neuron_permissions` function performs no check that at least one principal retains `Disburse` or `ManagePrincipals` before completing: [3](#0-2) 

An integration test confirms this is reachable and succeeds: [4](#0-3) 

### Impact Explanation

Once all permissions are removed, the staked SNS governance tokens are permanently locked in the neuron's ledger subaccount (controlled by the SNS governance canister). No principal can call `Disburse`, `DisburseMaturity`, `StartDissolving`, or any other neuron management command because every such command requires the caller to hold a specific `NeuronPermissionType`. The tokens are not burned — they remain on the ledger — but they are irrecoverable without an SNS governance upgrade proposal. For a user who self-inflicts this, the loss is permanent. For a malicious actor who tricks a neuron owner into signing a `RemoveNeuronPermissions` call (e.g., via a rogue dapp frontend), the victim's stake is permanently frozen.

**Impact:** Permanent loss of user funds (staked governance tokens) with no on-chain recovery path available to the affected user.

### Likelihood Explanation

The attack surface is reachable by any unprivileged ingress sender who holds `ManagePrincipals` permission on their own neuron (which is granted by default at neuron creation). A user can self-inflict this accidentally. A malicious dapp or wallet frontend can craft a `RemoveNeuronPermissions` payload targeting the user's own principal with all permission types and submit it as an ingress message. No privileged role, key, or majority is required. The SNS governance canister is a production canister on mainnet.

### Recommendation

Add a guard inside `remove_neuron_permissions` (or at the neuron mutation layer) that checks whether the resulting neuron would have zero principals holding `Disburse` permission (or at minimum `ManagePrincipals`). If so, reject the operation with an appropriate `GovernanceError`. Alternatively, enforce that a neuron must always retain at least one principal with `ManagePrincipals` before the removal is committed.

### Proof of Concept

1. User stakes SNS tokens and claims a neuron. The claimer is granted all `NeuronPermissionType` permissions by default.
2. User (or a malicious dapp acting on their behalf) calls `manage_neuron` with:
   ```
   ManageNeuron {
     subaccount: <neuron_subaccount>,
     command: RemoveNeuronPermissions {
       principal_id: <user_principal>,
       permissions_to_remove: NeuronPermissionList { permissions: NeuronPermissionType::all() }
     }
   }
   ```
3. `remove_neuron_permissions` in `rs/sns/governance/src/governance.rs` (line 4659) executes, removes the last permission entry, and returns `Ok(())`.
4. The neuron now has `permissions = []`. Any subsequent `manage_neuron` call (Disburse, StartDissolving, etc.) fails with `AccessControlList` error because no principal is authorized.
5. The staked tokens remain locked in the neuron's ledger subaccount forever, with no user-accessible recovery path.

This exact flow is demonstrated by the existing integration test `test_neuron_remove_all_permissions_of_self` at: [4](#0-3) 

which confirms the neuron ends up with `permissions.len() == 0` and no error is returned.

#[test]
fn test_neuron_remove_all_permissions_of_self() {
    local_test_on_sns_subnet(|runtime| async move {
        let user = Sender::from_keypair(&TEST_USER1_KEYPAIR);
        let account_identifier = Account {
            owner: user.get_principal_id().0,
            subaccount: None,
        };
        let alloc = Tokens::from_tokens(1000).unwrap();

        let system_params = NervousSystemParameters {
            neuron_claimer_permissions: Some(NeuronPermissionList {
                permissions: NeuronPermissionType::all(),
            }),
            ..NervousSystemParameters::with_default_values()
        };

        let sns_init_payload = SnsTestsInitPayloadBuilder::new()
            .with_ledger_account(account_identifier, alloc)
            .with_nervous_system_parameters(system_params)
            .build();

        let sns_canisters = SnsCanisters::set_up(&runtime, sns_init_payload).await;

        let neuron_id = sns_canisters.stake_and_claim_neuron(&user, None).await;
        let neuron = sns_canisters.get_neuron(&neuron_id).await;
        let subaccount = neuron.subaccount().expect("Error creating the subaccount");

        // Assert that the Claimer has been granted all permissions
        assert_eq!(neuron.permissions.len(), 1);
        let mut neuron_permission =
            get_neuron_permission_from_neuron(&neuron, &user.get_principal_id());
        // .sort() emits () and needs to be called outside of the assert!
        neuron_permission.permission_type.sort_unstable();
        assert_eq!(
            neuron_permission.permission_type,
            NeuronPermissionType::all(),
        );

        sns_canisters
            .remove_neuron_permissions_or_panic(
                &user,
                &subaccount,
                &user.get_principal_id(),
                NeuronPermissionType::all(),
            )
            .await;

        let neuron = sns_canisters.get_neuron(&neuron_id).await;
        assert_eq!(neuron.permissions.len(), 0);

        Ok(())
    });
