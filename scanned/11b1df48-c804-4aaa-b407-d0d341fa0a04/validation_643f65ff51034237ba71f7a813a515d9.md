### Title
SNS Governance `compute_ballots_for_new_proposal` Lacks Voting Power Spike Detection, Enabling Pre-Proposal Stake Manipulation - (File: rs/sns/governance/src/governance.rs)

### Summary
The SNS governance canister's `compute_ballots_for_new_proposal` function computes each eligible neuron's voting power using the live current timestamp (`now_seconds`) with no historical snapshot or spike-detection guard. An attacker who rapidly stakes a large amount of SNS tokens immediately before a proposal is created receives full, unattenuated voting power in the resulting ballots, potentially swinging the outcome of any governance vote. The NNS governance canister has an explicit `VotingPowerSnapshots` mechanism and spike-detection logic to defend against exactly this class of attack; SNS governance has neither.

### Finding Description
`compute_ballots_for_new_proposal` in `rs/sns/governance/src/governance.rs` iterates over every neuron in `self.proto.neurons` and calls `neuron.voting_power(now_seconds, …)` to populate each ballot:


There is no comparison against any historical baseline. By contrast, NNS governance maintains a `VotingPowerSnapshots` store (up to 7 daily snapshots) and, in `compute_ballots_for_standard_proposal`, checks whether the current total potential voting power exceeds 1.5× the minimum seen in recent history; if so, it falls back to the older snapshot's ballots:


SNS governance has no equivalent of `VotingPowerSnapshots`, `SnapshotVotingPowerTask`, or `previous_ballots_if_voting_power_spike_detected`.

### Impact Explanation
An attacker who controls a large supply of SNS tokens can stake them into a neuron with the minimum required dissolve delay immediately before submitting (or anticipating) a governance proposal. Because `compute_ballots_for_new_proposal` reads live state, the attacker's neuron receives a ballot whose `voting_power` reflects the full freshly-staked amount. Once the ballot is created the voting power is frozen in the ballot, so the attacker can cast a decisive vote on a critical proposal (e.g., `TransferSnsTreasuryFunds`, `UpgradeSnsControlledCanister`, `ExecuteGenericNervousSystemFunction`) and then begin dissolving. The result is that a single actor can unilaterally pass or block any SNS governance proposal without the stake having been present long enough to represent genuine long-term alignment with the protocol.

### Likelihood Explanation
The attack requires a meaningful amount of SNS tokens and willingness to lock them for the minimum dissolve delay (typically months). It is not a flash-loan attack. However, for SNS DAOs with low circulating supply or concentrated token distribution, the capital requirement may be modest relative to the governance impact. The entry path is fully unprivileged: any principal can stake tokens and submit or time a proposal via the public `manage_neuron` / `make_proposal` Candid endpoints. No admin key, oracle, or network-level compromise is required.

### Recommendation
Port the NNS `VotingPowerSnapshots` pattern to SNS governance:
1. Add a recurring timer task (analogous to `SnapshotVotingPowerTask`) that records a daily snapshot of total potential voting power for each SNS governance canister.
2. In `compute_ballots_for_new_proposal`, compare the current total potential voting power against the minimum seen in recent snapshots; if the ratio exceeds a configurable threshold (e.g., 1.5×), use the snapshot's ballot set instead of the live computation.
3. Expose the spike-detection threshold as a `NervousSystemParameters` field so each SNS DAO can tune it.

### Proof of Concept

**Attacker-controlled entry path (unprivileged ingress):**

1. Attacker holds a large amount of SNS token `T`.
2. Attacker calls `manage_neuron { command: Stake … }` to stake `T` into a new neuron with dissolve delay ≥ `neuron_minimum_dissolve_delay_to_vote_seconds`. This is a standard public endpoint.
3. In the same round (or the next), attacker calls `make_proposal` to submit a `TransferSnsTreasuryFunds` proposal draining the SNS treasury to an attacker-controlled account.
4. `compute_ballots_for_new_proposal` runs at `now_seconds`, sees the freshly-staked neuron, and assigns it a ballot with full voting power proportional to `T`.
5. Because `T` is large relative to existing staked supply, the attacker's ballot alone exceeds the simple-majority threshold; the attacker votes YES and the proposal passes immediately via early adoption.
6. Attacker begins dissolving the neuron; after the dissolve delay the tokens are returned.

```rust
        let max_age_bonus_percentage = nervous_system_parameters
            .max_age_bonus_percentage
            .expect("NervousSystemParameters must have max_age_bonus_percentage");

        let min_dissolve_delay_for_vote = nervous_system_parameters
            .neuron_minimum_dissolve_delay_to_vote_seconds
            .expect("NervousSystemParameters must have min_dissolve_delay_for_vote");

        let mut electoral_roll = BTreeMap::<String, Ballot>::new();
        let mut total_power: u128 = 0;

        for (k, v) in self.proto.neurons.iter() {
            // If this neuron is eligible to vote, record its
            // voting power at the time of proposal creation (now).
```

```rust
/// A task to snapshot the voting power every day, so that the snapshot can be used to disable
/// early adoption of proposals if such proposals have unusually high voting power.
pub(super) struct SnapshotVotingPowerTask {
    governance: &'static LocalKey<RefCell<Governance>>,
    snapshots: &'static LocalKey<RefCell<VotingPowerSnapshots>>,
}

const VOTING_POWER_SNAPSHOT_INTERVAL: Duration = Duration::from_secs(60 * 60 * 24);

impl SnapshotVotingPowerTask {
    pub fn new(
        governance: &'static LocalKey<RefCell<Governance>>,
        snapshots: &'static LocalKey<RefCell<VotingPowerSnapshots>>,
    ) -> Self {
        Self {
            governance,
            snapshots,
        }
    }
}

impl RecurringSyncTask for SnapshotVotingPowerTask {
    fn execute(self) -> (Duration, Self) {
        let now_seconds = self
            .governance
            .with_borrow(|governance| governance.env.now());
        if self
            .snapshots
            .with_borrow(|snapshots| snapshots.is_latest_snapshot_a_spike(now_seconds))
        {
            return (VOTING_POWER_SNAPSHOT_INTERVAL, self);
        }

        let voting_power_snapshot = self.governance.with_borrow_mut(|governance| {
            let voting_power_economics = governance.voting_power_economics();
            governance
                .neuron_store
                .compute_voting_power_snapshot_for_standard_proposal(
                    voting_power_economics,
                    now_seconds,
                )
                .expect("Voting power snapshot failed")
        });

        self.snapshots.with_borrow_mut(|snapshots| {
            snapshots.record_voting_power_snapshot(now_seconds, voting_power_snapshot);
        });

        (VOTING_POWER_SNAPSHOT_INTERVAL, self)
```
