### Title
Missing `min_received_amount` and Deadline in `withdraw_eth` Allows Unexpected ETH Loss Due to Gas Fee Spikes - (`rs/ethereum/cketh/minter/src/main.rs`)

---

### Summary

The `withdraw_eth` function in the ckETH minter burns ckETH immediately and irrevocably at call time, but defers the actual Ethereum transaction creation — and therefore the fee deduction — to a background timer task. Because `WithdrawalArg` accepts no `max_fee` / `min_received_amount` parameter and no deadline, a user has no protection against receiving significantly less ETH than expected if Ethereum gas fees spike between the `withdraw_eth` call and when the minter processes the queued request.

---

### Finding Description

When a user calls `withdraw_eth`:

1. The full `amount` of ckETH is burned from the caller's account immediately via `burn_from`.
2. An `EthWithdrawalRequest` is queued in the minter's state.
3. A background timer task later calls `create_transaction()`, which computes `max_transaction_fee` from the **current** gas fee estimate at that moment and deducts it from `withdrawal_amount` to determine the ETH value sent on-chain. [1](#0-0) 

The `WithdrawalArg` type accepted by `withdraw_eth` contains only `amount`, `recipient`, and `from_subaccount` — no `max_fee`, `min_received_amount`, or `deadline` field: [2](#0-1) 

Inside `create_transaction`, the fee is computed fresh from the current `gas_fee_estimate` at processing time, not at call time: [3](#0-2) 

The gas fee estimate used is cached and can be up to **60 seconds stale** (`MAX_AGE_NS = 60_000_000_000`), and the minter queues requests for up to `max_time_in_queue_nanos` before batching them: [4](#0-3) 

The `eip_1559_transaction_price` query endpoint does expose a timestamped fee estimate that users can read before calling `withdraw_eth`, but there is no mechanism to pass that estimate as a constraint into the withdrawal call itself: [5](#0-4) 

---

### Impact Explanation

A user who calls `withdraw_eth(1 ETH)` after observing a fee estimate of 0.001 ETH may find that by the time the minter processes the request, Ethereum gas fees have spiked and the actual deduction is 0.05 ETH — receiving only 0.95 ETH instead of the ~0.999 ETH they expected. The ckETH burn is irreversible; the user cannot cancel the request. In the extreme case where fees exceed the withdrawal amount, `create_transaction` returns `InsufficientTransactionFee` and a reimbursement is scheduled, but the user still loses the ckETH ledger transaction fee and suffers the delay. [6](#0-5) 

---

### Likelihood Explanation

Ethereum EIP-1559 base fees can increase by up to ~12.5% per block and can spike dramatically during periods of high network activity (NFT mints, DeFi liquidation cascades, etc.). The minter's `max_time_in_queue_nanos` parameter means requests can sit pending for a configurable window before being batched and processed. This is a realistic, non-adversarial scenario: no malicious actor is required — ordinary Ethereum network congestion is sufficient to trigger the discrepancy. [7](#0-6) 

---

### Recommendation

Add a `max_fee` (or equivalently `min_received_amount`) field to `WithdrawalArg`. Before burning ckETH, or at transaction creation time, compare the current `max_transaction_fee` against the user-supplied limit and reject (with reimbursement) if the fee exceeds it. Optionally, add a `deadline_ns` field: if the request has not been processed by that IC timestamp, cancel it and reimburse the user.

```
type WithdrawalArg = record {
    recipient          : text;
    amount             : nat;
    from_subaccount    : opt blob;
    // NEW: reject if fee would exceed this value
    max_transaction_fee : opt nat;
    // NEW: cancel and reimburse if not processed by this IC time
    deadline_ns        : opt nat64;
};
```

---

### Proof of Concept

1. User queries `eip_1559_transaction_price` → observes `max_transaction_fee = 0.001 ETH`.
2. User calls `withdraw_eth({ amount = 1 ETH, recipient = "0x..." })`. ckETH is burned immediately.
3. Ethereum network congestion spikes; base fee increases 10×.
4. Minter's background task fires, calls `create_transaction` with the new gas estimate → `max_transaction_fee = 0.01 ETH`.
5. User receives only `1 - 0.01 = 0.99 ETH` instead of the expected `~0.999 ETH`, with no recourse.
6. If fees spike to `> 1 ETH`, `create_transaction` returns `InsufficientTransactionFee`; user is reimbursed minus the ckETH ledger fee, having gained nothing but lost time and the ledger fee. [8](#0-7) [1](#0-0)

/// Estimate price of EIP-1559 transaction based on the
/// `base_fee_per_gas` included in the last finalized block.
#[query]
async fn eip_1559_transaction_price(
    token: Option<Eip1559TransactionPriceArg>,
) -> Eip1559TransactionPrice {
    let gas_limit = match token {
        None => CKETH_WITHDRAWAL_TRANSACTION_GAS_LIMIT,
        Some(Eip1559TransactionPriceArg { ckerc20_ledger_id }) => {
            match read_state(|s| s.find_ck_erc20_token_by_ledger_id(&ckerc20_ledger_id)) {
                Some(_) => CKERC20_WITHDRAWAL_TRANSACTION_GAS_LIMIT,
                None => {
                    if ckerc20_ledger_id == read_state(|s| s.cketh_ledger_id) {
                        CKETH_WITHDRAWAL_TRANSACTION_GAS_LIMIT
                    } else {
                        ic_cdk::trap(format!(
                            "ERROR: Unsupported ckERC20 token ledger {ckerc20_ledger_id}"
                        ))
                    }
                }
            }
        }
    };
    match read_state(|s| s.last_transaction_price_estimate.clone()) {
        Some((ts, estimate)) => {
            let mut result = Eip1559TransactionPrice::from(estimate.to_price(gas_limit));
            result.timestamp = Some(ts);
            result
        }
        None => ic_cdk::trap("ERROR: last transaction price estimate is not available"),
    }
}

#[update]
async fn withdraw_eth(
    WithdrawalArg {
        amount,
        recipient,
        from_subaccount,
    }: WithdrawalArg,
) -> Result<RetrieveEthRequest, WithdrawalError> {
    let caller = validate_caller_not_anonymous();
    let _guard = retrieve_withdraw_guard(caller).unwrap_or_else(|e| {
        ic_cdk::trap(format!(
            "Failed retrieving guard for principal {caller}: {e:?}"
        ))
    });

    let destination = validate_address_as_destination(&recipient).map_err(|e| match e {
        AddressValidationError::Invalid { .. } | AddressValidationError::NotSupported(_) => {
            ic_cdk::trap(e.to_string())
        }
        AddressValidationError::Blocked(address) => WithdrawalError::RecipientAddressBlocked {
            address: address.to_string(),
        },
    })?;

    let amount = Wei::try_from(amount).expect("failed to convert Nat to u256");

    let minimum_withdrawal_amount = read_state(|s| s.cketh_minimum_withdrawal_amount);
    if amount < minimum_withdrawal_amount {
        return Err(WithdrawalError::AmountTooLow {
            min_withdrawal_amount: minimum_withdrawal_amount.into(),
        });
    }

    let client = read_state(LedgerClient::cketh_ledger_from_state);
    let now = ic_cdk::api::time();
    log!(INFO, "[withdraw]: burning {:?}", amount);
    match client
        .burn_from(
            Account {
                owner: caller,
                subaccount: from_subaccount,
            },
            amount,
            BurnMemo::Convert {
                to_address: destination,
            },
        )
        .await
    {
        Ok(ledger_burn_index) => {
            let withdrawal_request = EthWithdrawalRequest {
                withdrawal_amount: amount,
                destination,
                ledger_burn_index,
                from: caller,
                from_subaccount: from_subaccount.and_then(LedgerSubaccount::from_bytes),
                created_at: Some(now),
            };

            log!(
                INFO,
                "[withdraw]: queuing withdrawal request {:?}",
                withdrawal_request,
            );

            mutate_state(|s| {
                process_event(
                    s,
                    EventType::AcceptedEthWithdrawalRequest(withdrawal_request.clone()),
                );
            });
            Ok(RetrieveEthRequest::from(withdrawal_request))
        }
        Err(e) => Err(WithdrawalError::from(e)),
    }
}

pub fn create_transaction(
    withdrawal_request: &WithdrawalRequest,
    nonce: TransactionNonce,
    gas_fee_estimate: GasFeeEstimate,
    gas_limit: GasAmount,
    ethereum_network: EthereumNetwork,
) -> Result<Eip1559TransactionRequest, CreateTransactionError> {
    assert!(
        gas_limit > GasAmount::ZERO,
        "BUG: gas limit should be non-zero"
    );
    match withdrawal_request {
        WithdrawalRequest::CkEth(request) => {
            let transaction_price = gas_fee_estimate.to_price(gas_limit);
            let max_transaction_fee = transaction_price.max_transaction_fee();
            let tx_amount = match request.withdrawal_amount.checked_sub(max_transaction_fee) {
                Some(tx_amount) => tx_amount,
                None => {
                    return Err(CreateTransactionError::InsufficientTransactionFee {
                        cketh_ledger_burn_index: request.ledger_burn_index,
                        allowed_max_transaction_fee: request.withdrawal_amount,
                        actual_max_transaction_fee: max_transaction_fee,
                    });
                }
            };
            Ok(Eip1559TransactionRequest {
                chain_id: ethereum_network.chain_id(),
                nonce,
                max_priority_fee_per_gas: transaction_price.max_priority_fee_per_gas,
                max_fee_per_gas: transaction_price.max_fee_per_gas,
                gas_limit: transaction_price.gas_limit,
                destination: request.destination,
                amount: tx_amount,
                data: Vec::new(),
                access_list: Default::default(),
            })

pub async fn lazy_refresh_gas_fee_estimate() -> Option<GasFeeEstimate> {
    const MAX_AGE_NS: u64 = 60_000_000_000_u64; //60 seconds

    async fn do_refresh() -> Option<GasFeeEstimate> {
        let _guard = match TimerGuard::new(TaskType::RefreshGasFeeEstimate) {
            Ok(guard) => guard,
            Err(e) => {
                log!(
                    DEBUG,
                    "[refresh_gas_fee_estimate]: Failed retrieving guard: {e:?}",
                );
                return None;
            }
        };

        let fee_history = match eth_fee_history().await {
            Ok(fee_history) => fee_history,
            Err(e) => {
                log!(
                    INFO,
                    "[refresh_gas_fee_estimate]: Failed retrieving fee history: {e:?}",
                );
                return None;
            }
        };

        let gas_fee_estimate = match estimate_transaction_fee(&fee_history) {
            Ok(estimate) => {
                mutate_state(|s| {
                    s.last_transaction_price_estimate =
                        Some((ic_cdk::api::time(), estimate.clone()));
                });
                estimate
            }
            Err(e) => {
                log!(
                    INFO,
                    "[refresh_gas_fee_estimate]: Failed estimating gas fee: {e:?}",
                );
                return None;
            }
        };
        log!(
            INFO,
            "[refresh_gas_fee_estimate]: Estimated transaction fee: {:?}",
            gas_fee_estimate,
        );
        Some(gas_fee_estimate)
    }

    async fn eth_fee_history() -> Result<FeeHistory, MultiCallError<FeeHistory>> {
        read_state(rpc_client)
            .fee_history((5_u8, BlockTag::Latest))
            .with_reward_percentiles(vec![20])
            .with_cycles(MIN_ATTACHED_CYCLES)
            .try_send()
            .await
            .reduce_with_strategy(StrictMajorityByKey::new(|fee_history: &FeeHistory| {
                Nat::from(fee_history.oldest_block.clone())
            }))
    }

    let now_ns = ic_cdk::api::time();
    match read_state(|s| s.last_transaction_price_estimate.clone()) {
        Some((last_estimate_timestamp_ns, estimate))
            if now_ns < last_estimate_timestamp_ns.saturating_add(MAX_AGE_NS) =>
        {
            Some(estimate)
        }
        _ => do_refresh().await,
    }
}
