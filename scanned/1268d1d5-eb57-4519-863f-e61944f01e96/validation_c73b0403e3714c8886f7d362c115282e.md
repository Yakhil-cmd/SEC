### Title
Unprotected `swap_node_in_subnet_directly` Registry Update Method Allows Unauthorized Subnet Membership Modification - (File: rs/registry/canister/canister/canister.rs)

### Summary
The Registry canister exposes `swap_node_in_subnet_directly` as a public `canister_update` method with no caller authorization check at the canister dispatch level and no documented internal authorization guard, allowing any unprivileged ingress sender or canister to swap nodes in subnets — a privileged registry mutation that normally requires Governance approval.

### Finding Description
In `rs/registry/canister/canister/canister.rs`, the `swap_node_in_subnet_directly` function is exported as a `canister_update` endpoint with no `check_caller_is_*` guard:

```rust
#[unsafe(export_name = "canister_update swap_node_in_subnet_directly")]
fn swap_node_in_subnet_directly() {
    over(candid_one, |payload: SwapNodeInSubnetDirectlyPayload| {
        swap_node_in_subnet_directly_(payload)
    });
}
``` [1](#0-0) 

Compare this to every other privileged registry mutation, which all call a `check_caller_is_*` guard before dispatching:

```rust
fn update_node_operator_config() {
    check_caller_is_governance_and_log("update_node_operator_config");
    ...
}
fn remove_nodes() {
    check_caller_is_governance_and_log("remove_nodes");
    ...
}
``` [2](#0-1) [3](#0-2) 

The "directly" methods that are intentionally open to any caller carry an explicit comment `// This method can be called by anyone` and have internal authorization checks (e.g., `update_node_directly` verifies the caller is the node itself via `NodeId::from(caller)`; `update_node_ipv4_config_directly` calls `check_caller_is_node_operator`; `update_node_operator_config_directly` verifies the caller equals the node provider principal). `swap_node_in_subnet_directly` has neither the comment nor any discoverable internal authorization guard. [4](#0-3) [5](#0-4) [6](#0-5) 

The same pattern applies to `migrate_node_operator_directly`, which also has no caller check:

```rust
#[unsafe(export_name = "canister_update migrate_node_operator_directly")]
fn migrate_node_operator_directly() {
    over(candid_one, |payload: MigrateNodeOperatorPayload| {
        migrate_node_operator_directly_(payload)
    });
}
``` [7](#0-6) 

Both methods are listed in the public Candid interface: [8](#0-7) 

### Impact Explanation
The Registry canister is the authoritative source of truth for the IC network topology. `swap_node_in_subnet_directly` modifies subnet membership records — the same data that determines which nodes participate in consensus on each subnet. An attacker who can arbitrarily swap nodes in subnet records can:

- Replace legitimate subnet members with attacker-controlled or non-existent nodes, degrading or halting consensus on targeted subnets.
- Disrupt the NNS subnet itself if the NNS subnet's membership record is targeted.
- Corrupt the certified registry state that replicas and boundary nodes rely on for routing and topology discovery.

`migrate_node_operator_directly` allows reassigning node operator records, which controls which principals can add/remove nodes — enabling an attacker to seize node operator authority over existing infrastructure.

### Likelihood Explanation
The Registry canister is reachable via standard ingress from any Internet Computer user. No special role, key, or threshold is required. The attacker only needs to craft a valid `SwapNodeInSubnetDirectlyPayload` (two optional principal fields) and submit it as an update call. The method is publicly listed in the Candid interface. Exploitation requires no privileged access and is straightforward.

### Recommendation
Add a `check_caller_is_governance_or_engine_controller_and_log` (or equivalent) guard to `swap_node_in_subnet_directly` and `migrate_node_operator_directly`, consistent with the pattern used by all other privileged registry mutation endpoints:

```rust
#[unsafe(export_name = "canister_update swap_node_in_subnet_directly")]
fn swap_node_in_subnet_directly() {
    check_caller_is_governance_or_engine_controller_and_log("swap_node_in_subnet_directly");
    over(candid_one, |payload: SwapNodeInSubnetDirectlyPayload| {
        swap_node_in_subnet_directly_(payload)
    });
}
``` [9](#0-8) 

### Proof of Concept
Any ingress sender can call the Registry canister's `swap_node_in_subnet_directly` update method with an arbitrary `SwapNodeInSubnetDirectlyPayload`:

```
dfx canister --network ic call <registry_canister_id> swap_node_in_subnet_directly \
  '(record { old_node_id = opt principal "<victim_node>"; new_node_id = opt principal "<attacker_node>" })'
```

Because no `check_caller_is_*` guard exists at the dispatch site and no internal authorization check is present, the call proceeds to `do_swap_node_in_subnet_directly`, mutates the subnet membership record in the Registry, and triggers `recertify_registry()` — making the unauthorized change part of the certified state served to all replicas and boundary nodes.
