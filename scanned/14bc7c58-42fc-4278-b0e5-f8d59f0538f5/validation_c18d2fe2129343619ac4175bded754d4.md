### Title
Stale `maturity_modulation` Silently Blocks Neuron Spawning and Maturity Disbursement Finalization - (`rs/nns/governance/src/governance.rs`, `rs/nns/governance/src/governance/disburse_maturity.rs`)

---

### Summary

The NNS Governance canister's `maybe_spawn_neurons` and `finalize_maturity_disbursement` functions both gate their execution on the presence of a non-`None` `maturity_modulation.current_value_permyriad` field. This field is populated by a periodic background task (`UpdateIcpXdrRateRelatedData`) that fetches ICP/XDR rates from the Exchange Rate Canister (XRC). If the XRC is unavailable, returns zero rates, or the background task fails to populate the field (e.g., on a fresh canister or after an upgrade that resets the field), all neurons in the `Spawning` state and all pending `DisburseMaturity` disbursements that have passed their `finalize_disbursement_timestamp_seconds` are silently skipped indefinitely ŌĆö with no error returned to the user and no recovery path except waiting for the background task to succeed.

---

### Finding Description

**Spawning path (`maybe_spawn_neurons`):**

In `rs/nns/governance/src/governance.rs`, `maybe_spawn_neurons` reads `heap_data.maturity_modulation.current_value_permyriad`. If it is `None`, the function returns immediately without processing any spawning neurons:

```rust
let maturity_modulation = match self
    .heap_data
    .maturity_modulation
    .as_ref()
    .and_then(|m| m.current_value_permyriad)
{
    None => return,   // ŌåÉ silently aborts all spawning
    Some(value) => value,
};
```

**Maturity disbursement finalization path (`finalize_maturity_disbursement`):**

In `rs/nns/governance/src/governance/disburse_maturity.rs`, `next_maturity_disbursement_to_finalize` similarly requires `maturity_modulation_basis_points` to be `Some`:

```rust
let maturity_modulation_basis_points = maturity_modulation_basis_points
    .ok_or(FinalizeMaturityDisbursementError::NoMaturityModulation)?;
```

This error causes `finalize_maturity_disbursement` to log and retry after `RETRY_INTERVAL`, but the disbursement is never completed until the modulation field is populated.

**The background task that populates the field:**

`UpdateIcpXdrRateRelatedData` (in `rs/nns/governance/src/timer_tasks/update_icp_xdr_rate_related_data.rs`) fetches ICP/XDR rates from the XRC and calls `update_maturity_modulation`. If the XRC returns no usable rate for any day in the lookback window, `compute_maturity_modulation_permyriad` returns `Err` and the field is left unchanged (or `None`):

```rust
Err(reason) => {
    println!(
        "{}update_maturity_modulation: skipping update: {}; leaving prior modulation \
         unchanged",
        LOG_PREFIX, reason
    );
}
```

On a fresh canister, `maturity_modulation` is initialized with `current_value_permyriad: Some(0)` and `updated_at_days_since_epoch: None` ŌĆö but after an upgrade or state reset, the field may be `None` until the background task succeeds.

The analog to the external report is exact: just as `epoch[tigAsset]` not being updated to the current day blocks BondNFT transfers, `maturity_modulation.current_value_permyriad` being `None` blocks neuron spawning and maturity disbursement finalization ŌĆö both of which are time-sensitive operations that users have already committed to (neurons are in `Spawning` state, maturity has been deducted).

---

### Impact Explanation

1. **Neuron spawning permanently stalled**: A neuron in `Spawning` state has already had its maturity zeroed and `cached_neuron_stake_e8s` set. If `maturity_modulation` is `None`, `maybe_spawn_neurons` silently returns without minting ICP to the neuron's subaccount. The neuron remains in a limbo state ŌĆö maturity gone, stake not minted ŌĆö until the background task eventually succeeds.

2. **Maturity disbursement permanently stalled**: A user who called `DisburseMaturity` has already had their maturity deducted from the neuron. The disbursement sits in `maturity_disbursements_in_progress` indefinitely if `maturity_modulation` is `None`, with no ICP minted to the destination account.

3. **No user-facing error**: Neither path returns an error to the user. The user has no way to know their funds are stuck or to trigger recovery.

---

### Likelihood Explanation

The `maturity_modulation` field can be `None` or stale in realistic scenarios:

- **Fresh canister deployment or upgrade**: The field starts as `None` until the XRC background task completes its first successful run.
- **XRC unavailability**: If the XRC canister is unavailable or returns zero rates for all days in the lookback window, `compute_maturity_modulation_permyriad` returns `Err` and the field is not updated.
- **Canister snapshot restore**: The NNS supports `LoadCanisterSnapshot` proposals; restoring a snapshot taken before the field was populated would reset it to `None`.

The XRC is an external dependency. Its unavailability is a realistic operational scenario, not a theoretical one.

---

### Recommendation

1. **For spawning**: Instead of silently returning when `maturity_modulation` is `None`, use a fallback value of `0` (neutral modulation) so that spawning proceeds. This matches the behavior of `initialize_governance`, which defaults `current_value_permyriad` to `Some(0)`.

2. **For disbursement finalization**: Similarly, treat a missing modulation as `0` rather than blocking finalization entirely.

3. **Alternatively**: Ensure `maturity_modulation` is always initialized to `Some(MaturityModulation { current_value_permyriad: Some(0), ... })` on canister init and upgrade, so the field is never `None` in production.

4. **Add monitoring**: Emit a metric or log a warning when spawning or disbursement is skipped due to missing modulation, so operators can detect the condition.

---

### Proof of Concept

The existing test `test_finalize_maturity_disbursement_no_maturity_modulation` in `rs/nns/governance/src/governance/disburse_maturity_tests.rs` directly demonstrates the blocking behavior:

```rust
TEST_GOVERNANCE.with_borrow_mut(|governance| {
    governance.heap_data.maturity_modulation = None;
});
// ... initiate disbursement, advance time past deadline ...
let result = try_finalize_maturity_disbursement(&TEST_GOVERNANCE)
    .now_or_never()
    .unwrap();
assert_eq!(
    result,
    Err(FinalizeMaturityDisbursementError::NoMaturityModulation)
);
```

For spawning, the analogous path is in `maybe_spawn_neurons`:

```rust
// heap_data.maturity_modulation = None  ŌåÆ  function returns immediately
// Neurons in Spawning state are never processed
```

The test helper `run_periodic_tasks_often_enough_to_update_maturity_modulation` in `rs/nns/governance/tests/governance.rs` explicitly notes this gap by manually setting the field:

```rust
gov.heap_data.maturity_modulation = Some(MaturityModulation {
    current_value_permyriad: Some(100),
    updated_at_days_since_epoch: Some(gov.env.now() / ONE_DAY_SECONDS),
});
```
