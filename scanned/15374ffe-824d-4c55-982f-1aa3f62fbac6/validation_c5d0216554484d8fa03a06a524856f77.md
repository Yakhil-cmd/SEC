### Title
Predictable `RuleId` UUID Generation via Time-Only PRNG Seed Enables Enumeration of Undisclosed Rate-Limit Rules — (File: `rs/boundary_node/rate_limits/canister/random.rs`)

### Summary
The rate-limit canister's custom `getrandom` implementation seeds its ChaCha20 PRNG using only `ic_cdk::api::time()` (the IC consensus block time) for the first 8 bytes, with the remaining 24 bytes hardcoded to `0x42`. Because the IC consensus time is publicly observable on-chain, the entire PRNG state at canister initialization is predictable. This PRNG is the sole source of randomness for `generate_random_uuid()`, which mints the `RuleId` identifiers for confidential rate-limit rules. An unprivileged caller can compute all possible `RuleId` values and probe `get_rule_by_id` to enumerate undisclosed rules and their associated metadata.

### Finding Description

In `rs/boundary_node/rate_limits/canister/random.rs`, the thread-local RNG is initialized as:

```rust
thread_local! {
  static RNG: RefCell<ChaCha20Rng> = {
    let mut seed = [42; 32];          // 24 bytes are constant 0x42
    seed[..8].copy_from_slice(&time().to_le_bytes()); // only 8 bytes from IC time
    RefCell::new(ChaCha20Rng::from_seed(seed))
  };
}
``` [1](#0-0) 

This `RNG` is registered as the canister's `getrandom` backend: [2](#0-1) 

`generate_random_uuid()` in `add_config.rs` calls `getrandom::getrandom`, which routes to this RNG on `wasm32-unknown-unknown`: [3](#0-2) 

Every new rate-limit rule receives a `RuleId` minted by this function: [4](#0-3) 

The seed has two components:
- **Bytes 0–7**: `ic_cdk::api::time()` — the IC consensus time in nanoseconds, which is a deterministic, publicly observable value recorded in every block.
- **Bytes 8–31**: the constant `0x42` — zero additional entropy.

The IC initialization time of any canister is observable via `read_state` or block explorers. An attacker who knows the canister's initialization block time can reconstruct the exact 32-byte seed and replay the ChaCha20 stream to predict every `RuleId` UUID the canister will ever generate.

The `get_rule_by_id` query endpoint is publicly accessible without authentication: [5](#0-4) 

For unauthorized callers, the `RuleGetter` redacts `rule_raw` and `description` but still returns `Ok` with the rule's `incident_id`, version metadata, and disclosure status: [6](#0-5) 

### Impact Explanation

The rate-limit canister is designed to keep undisclosed rules confidential — their `rule_raw` (the actual rate-limit policy) and `description` are hidden from the public until explicitly disclosed. The `RuleId` is the only access key to probe for a specific undisclosed rule.

Because `RuleId` values are predictable from the publicly observable canister initialization time, an unprivileged attacker can:
1. Retrieve the canister's initialization block time from the IC state.
2. Reconstruct the ChaCha20 seed and generate the sequence of UUIDs.
3. Call `get_rule_by_id` for each predicted UUID to confirm which rules exist.
4. Learn the `incident_id` and version metadata of undisclosed rules before they are officially disclosed.

This breaks the confidentiality model of the rate-limit canister: security incidents and their associated rate-limit rules are meant to be secret until disclosed, but their existence and incident linkage become enumerable.

### Likelihood Explanation

The attack requires no privileged access. The IC consensus time is a deterministic, publicly observable value. Any caller can query `read_state` to obtain the canister's initialization time with nanosecond precision. The ChaCha20 stream is then fully determined. The `get_rule_by_id` query is unauthenticated and callable by any principal. The only cost is the number of `get_rule_by_id` calls needed to enumerate rules, which is bounded by the number of `add_config` calls (a small number in practice).

### Recommendation

Replace the time-seeded RNG with a seed derived from `ic00::raw_rand`, which provides cryptographically unpredictable randomness from the subnet's threshold BLS randomness beacon. Since `raw_rand` is asynchronous, the canister should call it during `init`/`post_upgrade` and store the seed, similar to how NNS governance handles this in `rs/nns/governance/src/timer_tasks/seeding.rs`: [7](#0-6) 

Alternatively, seed the RNG by XOR-ing `raw_rand` output with the current time, ensuring that even if one source is predictable, the combined seed is not.

### Proof of Concept

```
1. Query the IC state for the rate-limit canister's initialization block time T (nanoseconds).
2. Construct seed: seed[0..8] = T.to_le_bytes(); seed[8..32] = [42u8; 24]
3. Initialize ChaCha20Rng::from_seed(seed).
4. For each add_config call (observable on-chain), advance the RNG by 16 bytes to predict the next RuleId UUID.
5. Call get_rule_by_id(predicted_uuid) — a 200 Ok response with incident_id confirms the undisclosed rule exists.
6. Repeat to enumerate all undisclosed rules and their incident linkages before official disclosure.
```

fn generate_random_uuid() -> Result<Uuid, anyhow::Error> {
    let mut buf = [0_u8; 16];
    getrandom::getrandom(&mut buf)
        .map_err(|e| anyhow::anyhow!(e))
        .context("Failed to generate random bytes")?;
    let uuid = Uuid::from_slice(&buf).context("Failed to create UUID from bytes")?;
    Ok(uuid)
}

        let rule_id = RuleId::try_from(rule_id.clone())

        let stored_rule = self
            .canister_api
            .get_rule(&rule_id)
            .ok_or_else(|| GetEntityError::NotFound(rule_id.0.to_string()))?;

        let output_rule = OutputRuleMetadata {
            id: rule_id,
            incident_id: stored_rule.incident_id,
            rule_raw: Some(stored_rule.rule_raw),
            description: Some(stored_rule.description),
            disclosed_at: stored_rule.disclosed_at,
            added_in_version: stored_rule.added_in_version,
            removed_in_version: stored_rule.removed_in_version,
        };

        let is_authorized_viewer = self.access_resolver.get_access_level()
            == AccessLevel::FullAccess
            || self.access_resolver.get_access_level() == AccessLevel::FullRead;

        if is_authorized_viewer {
            return Ok(output_rule.into());
        }

        // Hide non-disclosed rules from unauthorized viewers.
        let output_rule = self.formatter.format(output_rule);

        Ok(output_rule.into())
    }
