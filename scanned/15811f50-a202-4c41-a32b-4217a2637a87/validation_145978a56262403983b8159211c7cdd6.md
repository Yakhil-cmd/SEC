### Title
SNS Governance Lacks Time-Weighted Voting Power Decay, Allowing Stale Neurons to Retain Full Influence Indefinitely - (File: rs/sns/governance/src/neuron.rs)

---

### Summary

The SNS governance canister's `Neuron::voting_power` function and `compute_ballots_for_new_proposal` contain no time-based decay or refresh requirement. Unlike the NNS governance, which introduced `deciding_voting_power` with linear decay after 6 months of inactivity (NNS proposal 132411), SNS neurons maintain their full voting power indefinitely regardless of participation. Any unprivileged SNS token holder who locked tokens and became inactive retains full governance influence permanently, with no on-chain incentive to remain engaged.

---

### Finding Description

**Root cause in SNS governance:**

The SNS `Neuron::voting_power` function computes voting power purely from stake, dissolve-delay bonus, age bonus, and `voting_power_percentage_multiplier`: [1](#0-0) 

There is no `voting_power_refreshed_timestamp_seconds` field on SNS neurons, no `VotingPowerEconomics` decay parameters, and no `deciding_voting_power_adjustment_factor` equivalent. The ballot-creation path in `compute_ballots_for_new_proposal` calls `voting_power` directly for every eligible neuron: [2](#0-1) 

**Contrast with NNS governance (the fixed path):**

The NNS governance explicitly addressed this exact class of issue. It introduced `VotingPowerEconomics` with two decay parameters: [3](#0-2) 

Each NNS neuron carries `voting_power_refreshed_timestamp_seconds`: [4](#0-3) 

And `potential_and_deciding_voting_power` applies a linear adjustment factor that reduces `deciding_voting_power` from 100 % to 0 % over the decay window: [5](#0-4) 

The NNS ballot-creation path uses `deciding_voting_power` (the decayed value), not `potential_voting_power`: [6](#0-5) 

The SNS has none of these mechanisms. The `VotingPowerEconomics` struct and the `refresh_voting_power` call exist only in the NNS codebase: [7](#0-6) [8](#0-7) 

---

### Impact Explanation

- SNS neurons locked years ago retain 100 % voting power with zero active participation.
- Large inactive token holders can permanently dominate SNS proposal outcomes.
- No on-chain penalty for inactivity; no incentive to maintain long-term governance engagement.
- Governance can be captured by stale neurons whose holders have lost interest or whose keys are dormant.
- Misalignment between the token-locking commitment and actual governance influence, undermining the purpose of dissolve-delay bonuses.

---

### Likelihood Explanation

Every deployed SNS is affected by default. The condition requires only that an SNS neuron holder locks tokens and then stops interacting. This is a common real-world scenario (early investors, team allocations, forgotten wallets). No special privilege, key compromise, or majority collusion is required — the attacker simply does nothing and retains full influence.

---

### Recommendation

Port the NNS `VotingPowerEconomics` decay mechanism to SNS governance:

1. Add `voting_power_refreshed_timestamp_seconds` to the SNS `Neuron` protobuf.
2. Add decay parameters (`start_reducing_voting_power_after_seconds`, `clear_following_after_seconds`) to `NervousSystemParameters`.
3. Implement a `deciding_voting_power` method on SNS `Neuron` that applies a linear adjustment factor analogous to `deciding_voting_power_adjustment_factor` in `VotingPowerEconomics`.
4. Update `compute_ballots_for_new_proposal` to use `deciding_voting_power` instead of `voting_power` when building the electoral roll.
5. Expose a `RefreshVotingPower` manage-neuron command so neuron holders can signal active participation.

---

### Proof of Concept

1. An SNS token holder locks 1 000 000 tokens for the maximum dissolve delay, gaining maximum voting power at time T₀.
2. The holder becomes completely inactive — no votes, no following updates, no refresh calls — for 2 years.
3. At time T₀ + 2 years, a critical SNS proposal is submitted.
4. `compute_ballots_for_new_proposal` calls `v.voting_power(now_seconds, ...)` for the stale neuron. Because there is no decay factor, the returned value is identical to what it was at T₀ (modulo age-bonus growth, which actually *increases* it).
5. The stale neuron's ballot carries full voting power. An actively participating neuron with the same stake receives the same ballot weight.
6. The inactive holder can vote at the last moment with full influence, with no on-chain record of their prolonged absence and no reduction in their deciding power.

This is a direct analog of the reported `GaugeController` issue: votes maintain their full weight until explicitly changed, with no automatic reduction in voting power over time. [9](#0-8)

    pub fn voting_power(
        &self,
        now_seconds: u64,
        max_dissolve_delay_seconds: u64,
        max_neuron_age_for_age_bonus: u64,
        max_dissolve_delay_bonus_percentage: u64,
        max_age_bonus_percentage: u64,
    ) -> u64 {
        // We compute the stake adjustments in u128.
        let stake = self.voting_power_stake_e8s() as u128;
        // Dissolve delay is capped to max_dissolve_delay_seconds, but we cap it
        // again here to make sure, e.g., if this changes in the future.
        let d = std::cmp::min(
            self.dissolve_delay_seconds(now_seconds),
            max_dissolve_delay_seconds,
        ) as u128;
        // 'd_stake' is the stake with bonus for dissolve delay.
        let d_stake = stake
            + if max_dissolve_delay_seconds > 0 {
                (stake * d * max_dissolve_delay_bonus_percentage as u128)
                    / (100 * max_dissolve_delay_seconds as u128)
            } else {
                0
            };
        // Sanity check.
        assert!(d_stake <= stake + (stake * (max_dissolve_delay_bonus_percentage as u128) / 100));
        // The voting power is also a function of the age of the
        // neuron, giving a bonus of up to max_age_bonus_percentage at max_neuron_age_for_age_bonus.
        let a = std::cmp::min(self.age_seconds(now_seconds), max_neuron_age_for_age_bonus) as u128;
        let ad_stake = d_stake
            + if max_neuron_age_for_age_bonus > 0 {
                (d_stake * a * max_age_bonus_percentage as u128)
                    / (100 * max_neuron_age_for_age_bonus as u128)
            } else {
                0
            };
        // Final stake 'ad_stake' has is not more than max_age_bonus_percentage above 'd_stake'.
        assert!(ad_stake <= d_stake + (d_stake * (max_age_bonus_percentage as u128) / 100));

        // Convert the multiplier to u128. The voting_power_percentage_multiplier represents
        // a percent and will always be within the range 0 to 100.
        let v = self.voting_power_percentage_multiplier as u128;

        // Apply the multiplier to 'ad_stake' and divide by 100 to have the same effect as
        // multiplying by a percent.
        let vad_stake = ad_stake
            .checked_mul(v)
            .expect("Overflow detected when calculating voting power")
            .checked_div(100)
            .expect("Underflow detected when calculating voting power");

        // The final voting power is the stake adjusted by both age,
        // dissolve delay, and voting power multiplier. If the stake is is greater than
        // u64::MAX divided by 2.5, the voting power may actually not
        // fit in a u64.
        std::cmp::min(vad_stake, u64::MAX as u128) as u64

            .max_dissolve_delay_seconds

            .max_neuron_age_for_age_bonus
            .expect("NervousSystemParameters must have max_neuron_age_for_age_bonus");

            .max_dissolve_delay_bonus_percentage
            .expect("NervousSystemParameters must have max_dissolve_delay_bonus_percentage");

        let max_age_bonus_percentage = nervous_system_parameters
            .max_age_bonus_percentage
            .expect("NervousSystemParameters must have max_age_bonus_percentage");

        let min_dissolve_delay_for_vote = nervous_system_parameters
            .neuron_minimum_dissolve_delay_to_vote_seconds
            .expect("NervousSystemParameters must have min_dissolve_delay_for_vote");

        let mut electoral_roll = BTreeMap::<String, Ballot>::new();
        let mut total_power: u128 = 0;

        for (k, v) in self.proto.neurons.iter() {
            // If this neuron is eligible to vote, record its
            // voting power at the time of proposal creation (now).
            if v.dissolve_delay_seconds(now_seconds) < min_dissolve_delay_for_vote {
                // Not eligible due to dissolve delay.
                continue;

                max_dissolve_delay,
                max_age_bonus,
                max_dissolve_delay_bonus_percentage,
                max_age_bonus_percentage,

                    vote: Vote::Unspecified as i32,

                    cast_timestamp_seconds: 0,

    /// Returns 1 if a neuron has refreshed (its voting power/following)
    /// recently.
    ///
    /// Otherwise, if a neuron has not refreshed for >
    /// start_reducing_voting_power_after_seconds, returns < 1 (but >= 0).
    ///
    /// Once a neuron has not refresehd for
    /// start_reducing_voting_power_after_seconds +
    /// clear_following_after_seconds, this returns 0.
    ///
    /// Between these two points, the decrease is linear.
    pub fn deciding_voting_power_adjustment_factor(
        &self,
        time_since_last_voting_power_refreshed: Duration,
    ) -> Decimal {
        self.deciding_voting_power_adjustment_factor_function()
            .apply(time_since_last_voting_power_refreshed.as_secs())
            .clamp(Decimal::from(0), Decimal::from(1))
    }

    fn deciding_voting_power_adjustment_factor_function(&self) -> LinearMap {
        let from_range = {
            let begin = self.get_start_reducing_voting_power_after_seconds();
            let end = begin.saturating_add(self.get_clear_following_after_seconds());

            begin..end
        };

        #[allow(clippy::reversed_empty_ranges)]
        let to_range = 1..0;

        LinearMap::new(from_range, to_range)
    }
