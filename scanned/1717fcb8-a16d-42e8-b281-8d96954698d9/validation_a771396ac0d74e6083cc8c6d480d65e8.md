### Title
Stale `API_BOUNDARY_NODE_PRINCIPALS` Grants Indefinite `FullRead` Access to Decommissioned Boundary Nodes During Registry Poll Failures — (`rs/boundary_node/rate_limits/canister/canister.rs`, `state.rs`, `access_control.rs`)

---

### Summary

When the periodic registry poll in `periodically_poll_api_boundary_nodes` fails (network partition, registry canister unavailability), the in-memory `API_BOUNDARY_NODE_PRINCIPALS` `HashSet` is never cleared or updated. A boundary node that has been decommissioned from the registry retains its principal in the stale set indefinitely. Because `get_rule_by_id`, `get_rules_by_incident_id`, and `get_config` are `#[query]` methods that bypass `inspect_message`, the decommissioned node can call them directly, receive `FullRead` access from `AccessLevelResolver`, and obtain unredacted `rule_raw` and `description` for all undisclosed (confidential) rate-limit rules.

---

### Finding Description

**Step 1 — The set is non-stable and only updated on successful polls.**

`API_BOUNDARY_NODE_PRINCIPALS` is declared as a non-stable, in-memory `thread_local`: [1](#0-0) 

It starts empty on every canister init or upgrade. It is populated only when `periodically_poll_api_boundary_nodes` succeeds: [2](#0-1) 

`set_api_boundary_nodes_principals` performs a **full replacement** of the set: [3](#0-2) 

**Step 2 — Failed polls leave the set completely unchanged.**

Both error branches in `periodically_poll_api_boundary_nodes` only log and return; they do not clear or modify `API_BOUNDARY_NODE_PRINCIPALS`: [4](#0-3) 

There is no TTL, no expiry, and no fallback clearing. A decommissioned node's principal persists in the set for the entire duration of the poll outage.

**Step 3 — `inspect_message` does not protect query methods.**

On the Internet Computer, `inspect_message` is invoked only for ingress **update** messages. The three read methods are all `#[query]`: [5](#0-4) [6](#0-5) [7](#0-6) 

They bypass `inspect_message` entirely. The `REPLICATED_QUERY_METHOD` guard inside `inspect_message` only applies when `get_config` is invoked as an update (replicated query), which is not the normal call path.

**Step 4 — `AccessLevelResolver` grants `FullRead` based solely on the stale set.** [8](#0-7) 

`is_api_boundary_node_principal` reads directly from the stale `HashSet`: [9](#0-8) 

**Step 5 — `FullRead` bypasses all confidentiality formatting.**

In `RuleGetter::get`, when `is_authorized_viewer` is `true`, the raw unredacted content is returned without calling `formatter.format`: [10](#0-9) 

The formatter that would redact `rule_raw` and `description` for undisclosed rules is skipped: [11](#0-10)

### Impact Explanation

A decommissioned (or compromised) API boundary node operator who still holds the node's private key can call `get_rule_by_id` or `get_config` as a regular query during any period when registry polls are failing. They receive the full unredacted `rule_raw` and `description` of every undisclosed rate-limit rule — including active incident-response countermeasures that are intentionally kept confidential to prevent evasion. This breaks the core confidentiality invariant of the rate-limit canister.

---

### Likelihood Explanation

Registry poll failures are a realistic operational condition (transient network partitions, registry canister upgrades, subnet issues). The window of exposure is the entire duration of the failure. A decommissioned node operator who anticipates decommissioning can deliberately time their queries to coincide with a known or induced poll failure. No privileged access beyond possession of the node's own private key is required after decommissioning.

---

### Recommendation

On every failed poll, clear or invalidate the `API_BOUNDARY_NODE_PRINCIPALS` set (or entries older than a configurable TTL). A minimal fix in the error branches of `periodically_poll_api_boundary_nodes`:

```rust
Ok((Err(err),)) => {
    log!(...);
    // Clear stale principals on poll failure
    canister_api.set_api_boundary_nodes_principals(vec![]);
    ("failure", "calling_canister_method_failed")
}
Err(err) => {
    log!(...);
    canister_api.set_api_boundary_nodes_principals(vec![]);
    ("failure", "canister_call_rejected")
}
```

Alternatively, store a `last_successful_poll_timestamp` alongside each principal and reject `FullRead` grants for entries older than `2 × polling_interval`.

fn periodically_poll_api_boundary_nodes(interval: u64, canister_api: Arc<dyn CanisterApi>) {
    let interval = Duration::from_secs(interval);

    ic_cdk_timers::set_timer_interval(interval, move || {
        let canister_api = canister_api.clone();

        async move {
            let canister_id = Principal::from(REGISTRY_CANISTER_ID);

            let (call_status, message) = match call::<
                _,
                (Result<Vec<ApiBoundaryNodeIdRecord>, String>,),
            >(
                canister_id,
                REGISTRY_CANISTER_METHOD,
                (&GetApiBoundaryNodeIdsRequest {},),
            )
            .await
            {
                Ok((Ok(api_bn_records),)) => {
                    // Set authorized readers of the rate-limit config.
                    canister_api.set_api_boundary_nodes_principals(
                        api_bn_records.into_iter().filter_map(|n| n.id).collect(),
                    );
                    // Update metric.
                    let current_time = ic_cdk::api::time() as i64;
                    METRICS.with(|cell| {
                        let mut cell = cell.borrow_mut();
                        cell.last_successful_registry_poll_time
                            .borrow_mut()
                            .set(current_time);
                    });
                    ("success", "")
                }
                Ok((Err(err),)) => {
                    log!(
                        P0,
                        "[poll_api_boundary_nodes]: failed to fetch nodes from registry {err:?}",
                    );
                    ("failure", "calling_canister_method_failed")
                }
                Err(err) => {
                    log!(
                        P0,
                        "[poll_api_boundary_nodes]: failed to fetch nodes from registry {err:?}",
                    );
                    ("failure", "canister_call_rejected")
                }
            };

            // Update metric.
            METRICS.with(|cell| {
                let mut cell = cell.borrow_mut();
                cell.registry_poll_calls
                    .borrow_mut()
                    .with_label_values(&[call_status, message])
                    .inc();
            });
        }
    });
}
