### Title
SNS Swap Canister Accepts Unverified `transaction_fee_e8s` and `neuron_minimum_stake_e8s` in `Init`, Enabling Silent Neuron-Creation Failures at Finalization - (File: `rs/sns/swap/src/types.rs`, `rs/sns/swap/proto/ic_sns_swap/pb/v1/swap.proto`)

---

### Summary

The SNS Swap canister's `Init` struct accepts `transaction_fee_e8s` and `neuron_minimum_stake_e8s` as caller-supplied arguments that are explicitly documented as never being cross-checked against the actual SNS ledger and SNS governance canister values. These unverified values are then used as the authoritative inputs to `Params::validate()`, which determines whether `min_participant_icp_e8s` is large enough to guarantee that every participant can form valid SNS neurons. If the supplied values are lower than the real on-chain values, the validation passes with a `min_participant_icp_e8s` that is actually insufficient, causing neuron-creation transfers to fail silently during swap finalization and locking participant ICP inside the swap canister.

---

### Finding Description

The `Init` message for the SNS Swap canister carries two fields whose correctness is never enforced:

```
// Same as SNS ledger. Must hold the same value as SNS ledger. Whether the
// values match is not checked. If they don't match things will break.
optional uint64 transaction_fee_e8s = 13;

// Same as SNS governance. Must hold the same value as SNS governance. Whether
// the values match is not checked. If they don't match things will break.
optional uint64 neuron_minimum_stake_e8s = 14;
``` [1](#0-0) 

The `Init::validate()` function only checks that these fields are *present*; it never queries the SNS ledger or SNS governance canister to confirm the values match:

```rust
if self.transaction_fee_e8s.is_none() {
    // The value itself is not checked; only that it is supplied. Needs to
    // match the value in SNS ledger though.
    return Err("transaction_fee_e8s is required.".to_string());
}
if self.neuron_minimum_stake_e8s.is_none() {
    // As with transaction_fee_e8s, the value itself is not checked; only
    // that it is supplied. Needs to match the value in SNS governance though.
    return Err("neuron_minimum_stake_e8s is required.".to_string());
}
``` [2](#0-1) 

These unverified values are then consumed directly by `Params::validate()` to compute the minimum viable participant contribution:

```rust
let transaction_fee_e8s = init.transaction_fee_e8s.expect("...");
let neuron_minimum_stake_e8s = init.neuron_minimum_stake_e8s.expect("...");
...
let min_participant_icp_e8s_big_enough = min_participant_sns_e8s
    >= neuron_basket_count * (neuron_minimum_stake_e8s + transaction_fee_e8s) as u128;
``` [3](#0-2) 

If `Init.transaction_fee_e8s` is set lower than the actual SNS ledger fee, the inequality above passes with a `min_participant_icp_e8s` that is too small. When the swap finalizes and the canister attempts to transfer SNS tokens into each participant's neuron subaccount, the actual ledger fee causes each transfer to fall below `neuron_minimum_stake_e8s`, and the transfer fails. The `SweepResult` records these as `failure` entries, the participant's ICP remains locked in the swap canister, and the neuron is never created.

---

### Impact Explanation

- **Ledger conservation bug**: Participant ICP is accepted and locked by the swap canister based on a `min_participant_icp_e8s` floor that was computed using a fictitiously low fee. When finalization sweeps occur, the actual ledger fee makes the per-neuron SNS amount fall below `neuron_minimum_stake_e8s`, causing transfer failures.
- **Neuron creation silently fails**: The `SweepResult` records `failure` counts but the swap lifecycle still transitions to `Committed`. Affected participants lose their ICP for the duration of the swap and may never receive SNS neurons.
- **Misleading state**: The swap canister's `sns_token_e8s` invariant (`state.sns_token_e8s <= token_ledger.balance_of(<swap-canister>)`) is documented but never enforced at initialization, mirroring the original `VestingEscrowSimple` pattern of trusting an argument rather than reading the actual balance. [4](#0-3) 

---

### Likelihood Explanation

In the standard SNS-WASM deployment path, `transaction_fee_e8s` and `neuron_minimum_stake_e8s` are copied from the same `SnsInitPayload` into both the swap `Init` and the SNS ledger/governance canisters, so they normally match. However:

1. **Direct swap canister deployment**: Any principal can deploy a swap canister WASM directly (bypassing SNS-WASM) and supply arbitrary `Init` values. The canister itself imposes no cross-canister check.
2. **SNS-WASM deployment with crafted payload**: An NNS governance proposal can specify `transaction_fee_e8s` in `SnsInitPayload` that is intentionally low. The SNS ledger is initialized with the same low fee, but if the ledger is later upgraded or the fee is changed via governance, the swap canister retains the stale value with no mechanism to detect the divergence.
3. **Acknowledged design gap**: The code comment "Whether the values match is not checked. If they don't match things will break." is an explicit developer acknowledgment that this invariant is unenforceable at the canister level.

---

### Recommendation

1. **Cross-check at `open` time**: When the swap transitions from `Pending` to `Adopted`/`Open`, perform an inter-canister call to `sns_ledger.icrc1_fee()` and `sns_governance.get_nervous_system_parameters()` and assert that the returned values equal `Init.transaction_fee_e8s` and `Init.neuron_minimum_stake_e8s` respectively. Reject the `open` call if they diverge.
2. **Store actual balance, not argument**: Analogous to the recommendation in the original report, the swap canister should query `sns_ledger.icrc1_balance_of(swap_canister_account)` at open time and use the observed balance as the authoritative `sns_token_e8s`, rather than trusting the caller-supplied value.
3. **Remove the "not checked" escape hatch**: The `Init::validate()` function should validate the numeric values of `transaction_fee_e8s` and `neuron_minimum_stake_e8s` against the live canisters, not merely their presence.

---

### Proof of Concept

1. Deploy an SNS swap canister directly (or via a crafted NNS proposal) with `Init { transaction_fee_e8s: Some(0), neuron_minimum_stake_e8s: Some(1), ... }` while the actual SNS ledger has `transfer_fee = 10_000`.
2. `Init::validate()` passes because both fields are `Some(...)`.
3. `Params::validate()` computes `min_participant_icp_e8s` using `transaction_fee_e8s = 0`, so the floor is lower than it should be.
4. A participant contributes exactly `min_participant_icp_e8s` ICP.
5. At finalization, the swap canister attempts to transfer `amount_sns_e8s` to the participant's neuron subaccount. The actual ledger deducts `10_000` e8s as fee, leaving the neuron subaccount with `amount_sns_e8s - 10_000` which is below `neuron_minimum_stake_e8s` as enforced by the real SNS governance canister.
6. The neuron claim fails; the participant's ICP is swept to SNS governance but no neuron is created, resulting in a permanent loss of the participant's contribution relative to the promised SNS tokens. [5](#0-4) [6](#0-5)

            .count as u128;

            * self.sns_token_e8s as u128
