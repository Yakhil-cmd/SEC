### Title
`StableNeuronStore::delete` Fails to Clear `maturity_disbursements_map`, Causing Permanent DoS of the Finalization Timer — (`rs/nns/governance/src/storage/neurons.rs`)

---

### Summary

`StableNeuronStore::delete` omits clearing `maturity_disbursements_map` when a neuron is removed. Because `disburse_neuron` (stake disbursal) has no guard against pending maturity disbursements, an attacker can initiate a `DisburseMaturity`, then dissolve and disburse the neuron's stake, leaving orphaned `(neuron_id, index)` entries in `maturity_disbursements_map` and the `MaturityDisbursementIndex`. The finalization timer then loops forever on `NeuronNotFound`, blocking all users' maturity disbursement finalizations.

---

### Finding Description

**Root cause — `StableNeuronStore::delete` omits `maturity_disbursements_map` cleanup:** [1](#0-0) 

Every other auxiliary map is cleared on deletion:

```
update_repeated_field(neuron_id, vec![], &mut self.hot_keys_map);
update_repeated_field(neuron_id, vec![], &mut self.recent_ballots_map);
self.update_followees(neuron_id, hashmap![]);
update_singleton_field(neuron_id, None, &mut self.known_neuron_data_map);
update_singleton_field(neuron_id, None, &mut self.transfer_map);
```

`maturity_disbursements_map` is never touched. The `MaturityDisbursementIndex` (in `stable_neuron_indexes`) is similarly not cleaned up for the deleted neuron.

**Enabling condition — `disburse_neuron` has no guard for pending maturity disbursements:** [2](#0-1) 

The only checks are: caller is controller, neuron state is `Dissolved`, and KYC verified. There is no check for `maturity_disbursements_in_progress`. A neuron with pending maturity disbursements can be fully deleted via `Disburse`.

**Finalization timer behavior on orphaned entries:** [3](#0-2) 

`next_maturity_disbursement_to_finalize` reads from the `MaturityDisbursementIndex` to find neuron IDs ready to finalize, then calls `neuron_store.with_neuron(&neuron_id, ...)`. For a deleted neuron, this returns `NeuronStoreError::NeuronNotFound`, which maps to `FinalizeMaturityDisbursementError::NeuronNotFound`. [4](#0-3) 

On any error, `finalize_maturity_disbursement` logs the error and returns `RETRY_INTERVAL`. The timer reschedules itself, finds the same orphaned entry, fails again, and loops forever — permanently blocking all other users' maturity disbursement finalizations.

---

### Impact Explanation

- **Permanent DoS of the maturity disbursement finalization timer**: The timer processes one disbursement per invocation. A single orphaned entry causes it to retry indefinitely on `NeuronNotFound`, starving all legitimate pending disbursements for all users.
- **Loss of maturity**: The maturity was already deducted from the neuron at `DisburseMaturity` initiation time and is never minted, resulting in permanent loss of the attacker's own maturity (self-inflicted, but the DoS impact on others is the primary concern).
- **No neuron-ID-reuse ICP minting**: If a new neuron is created with the same ID, `StableNeuronStore::create` calls `update_repeated_field` with the new neuron's (empty) disbursements, which would overwrite the orphaned entries in `maturity_disbursements_map`. So the ICP-minting-to-unintended-account scenario is not reachable in practice.

---

### Likelihood Explanation

The attack requires only:
1. Owning a neuron with sufficient maturity (≥ `MINIMUM_DISBURSEMENT_E8S`).
2. Calling `DisburseMaturity` (standard ingress).
3. Dissolving the neuron and calling `Disburse` (standard ingress, requires neuron to be dissolved).

This is fully reachable by any unprivileged neuron owner. No privileged access, key compromise, or majority attack is required.

---

### Recommendation

1. **Fix `StableNeuronStore::delete`**: Add the missing cleanup call:
   ```rust
   update_repeated_field(neuron_id, vec![], &mut self.maturity_disbursements_map);
   ``` [5](#0-4) 

2. **Guard `disburse_neuron` against pending maturity disbursements**: Before allowing stake disbursal (which deletes the neuron), check `neuron.maturity_disbursements_in_progress.is_empty()` and return an error if not. [6](#0-5) 

3. **Make the finalization timer resilient**: If `NeuronNotFound` is returned for an index entry, remove the stale entry from the index rather than retrying indefinitely. [4](#0-3) 

---

### Proof of Concept

State-machine test sequence:
1. Create a neuron with maturity ≥ `MINIMUM_DISBURSEMENT_E8S` and a dissolve delay.
2. Call `DisburseMaturity` — verify `maturity_disbursements_in_progress` has one entry and `MaturityDisbursementIndex` has one entry.
3. Start dissolving; advance time past the dissolve delay.
4. Call `Disburse` (stake) — neuron is deleted.
5. Assert `maturity_disbursements_map` still contains the orphaned `(neuron_id, 0)` entry (bug confirmed at storage layer).
6. Advance time past `finalize_disbursement_timestamp_seconds`.
7. Run the finalization timer — assert it returns `RETRY_INTERVAL` (error path) and that no ICP was minted.
8. Run the timer again — assert it still returns `RETRY_INTERVAL` (infinite loop confirmed).
9. Assert `MaturityDisbursementIndex` still contains the orphaned entry (no self-healing). [1](#0-0) [7](#0-6) [3](#0-2) [8](#0-7)
