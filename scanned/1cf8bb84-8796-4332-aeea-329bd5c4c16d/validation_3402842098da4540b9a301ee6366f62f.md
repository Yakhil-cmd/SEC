### Title
SNS Neuron Permanently Bricked by Removing All Permissions — (`rs/sns/governance/src/governance.rs`)

---

### Summary

In SNS Governance, a neuron controller holding `ManagePrincipals` permission can call `RemoveNeuronPermissions` to strip all permissions from every principal on a neuron, including themselves. Once all permissions are removed, no principal can disburse, configure, or re-grant permissions on the neuron. The staked SNS tokens are permanently frozen with no recovery path. The code itself acknowledges this danger in a comment but provides no guard against it.

---

### Finding Description

The `remove_neuron_permissions` function in `rs/sns/governance/src/governance.rs` allows a caller with `ManagePrincipals` to remove any set of permissions from any principal on a neuron, including removing all permissions from themselves: [1](#0-0) 

The authorization check in `check_principal_authorized_to_change_permissions` only verifies that the caller currently holds `ManagePrincipals` (or `ManageVotingPermission` for voting-only changes) at the time of the call. It does not verify that at least one principal will retain `ManagePrincipals` after the operation completes: [2](#0-1) 

The underlying `remove_permissions_for_principal` function will happily remove the last permission entry from the neuron's permission list, leaving `neuron.permissions` empty: [3](#0-2) 

This is confirmed by the integration test `test_neuron_remove_all_permissions_of_self`, which explicitly demonstrates that a user can remove all their own permissions, leaving `neuron.permissions.len() == 0`: [4](#0-3) 

The attack sequence mirrors the Biconomy pauser-brick pattern exactly:

1. Alice holds `ManagePrincipals` + `Disburse` + `ConfigureDissolveState` on a neuron with staked tokens.
2. Alice calls `manage_neuron` → `RemoveNeuronPermissions` to remove all permissions from every other principal (if any).
3. Alice calls `manage_neuron` → `RemoveNeuronPermissions` to remove all her own permissions.
4. The neuron now has zero permissions. No principal can call `Disburse`, `ConfigureDissolveState`, `AddNeuronPermissions`, or any other operation.

The `disburse_neuron` function requires `NeuronPermissionType::Disburse` and will reject any caller once permissions are empty: [5](#0-4) 

There is no administrative override. SNS Root and NNS Governance have no special bypass for neuron permission operations. The neuron persists in governance state indefinitely with its staked tokens locked in the corresponding ledger subaccount.

---

### Impact Explanation

**Ledger conservation bug / governance authorization bug.** The staked SNS tokens held in the neuron's ledger subaccount are permanently inaccessible. No disburse, split, or maturity operation can be executed. The neuron cannot be dissolved or reconfigured. The tokens are effectively burned without a burn event. If the neuron held significant voting power, governance participation is also permanently lost for that stake.

---

### Likelihood Explanation

The entry path is a standard ingress `update` call to the SNS Governance canister's `manage_neuron` endpoint, available to any neuron holder. No special privilege beyond holding `ManagePrincipals` on the target neuron is required — and `ManagePrincipals` is included in the default `neuron_claimer_permissions` granted at neuron creation. A malicious or compromised neuron controller can execute this in two sequential transactions with no time constraint. The code comment explicitly acknowledges the danger, confirming the path is reachable and the outcome is known to be irreversible. [6](#0-5) 

---

### Recommendation

Add a post-condition check inside `remove_neuron_permissions` that rejects the operation if it would leave the neuron with no principal holding `ManagePrincipals`. Concretely, after computing the would-be resulting permission set, verify that at least one `NeuronPermission` entry still contains `NeuronPermissionType::ManagePrincipals`. If not, return an error such as `ErrorType::PreconditionFailed` with a message like `"Cannot remove the last ManagePrincipals permission from a neuron"`. This mirrors the recommended fix from the Biconomy report: require the contract not to be in a bricked state when renouncing the privileged role.

---

### Proof of Concept

```
// Attacker: Alice, sole principal on neuron N with all permissions.
// Neuron N has cached_neuron_stake_e8s > 0.

// Step 1: Remove all permissions from self.
manage_neuron(
    ManageNeuron {
        subaccount: neuron_N_subaccount,
        command: Some(Command::RemoveNeuronPermissions(RemoveNeuronPermissions {
            principal_id: Some(alice_principal_id),
            permissions_to_remove: Some(NeuronPermissionList {
                permissions: NeuronPermissionType::all(),  // all 10 permission types
            }),
        })),
    },
    sender = alice,
)
// Returns Ok(RemoveNeuronPermissionsResponse {})

// Step 2: Verify neuron is now bricked.
get_neuron(GetNeuron { neuron_id: Some(neuron_N_id) })
// Returns Neuron { permissions: [], cached_neuron_stake_e8s: <non-zero>, ... }

// Step 3: Attempt to disburse — fails permanently.
manage_neuron(
    ManageNeuron {
        subaccount: neuron_N_subaccount,
        command: Some(Command::Disburse(Disburse { amount: None, to_account: None })),
    },
    sender = alice,
)
// Returns GovernanceError { error_type: NotAuthorized, ... }
// No recovery is possible. Tokens are permanently frozen.
```

    pub(crate) fn check_principal_authorized_to_change_permissions(
        &self,
        caller: &PrincipalId,
        permissions_to_change: NeuronPermissionList,
    ) -> Result<(), GovernanceError> {
        // If the permissions to change are exclusively voting related,
        // ManagePrincipals or ManageVotingPermission is sufficient.
        // Otherwise, only ManagePrincipals is sufficient.
        let sufficient_permissions = if permissions_to_change.is_exclusively_voting_related() {
            vec![
                NeuronPermissionType::ManagePrincipals,
                NeuronPermissionType::ManageVotingPermission,
            ]
        } else {
            vec![NeuronPermissionType::ManagePrincipals]
        };

        // The caller is authorized if they have any of the sufficient permissions
        let caller_authorized = sufficient_permissions
            .iter()
            .any(|sufficient_permission| self.is_authorized(caller, *sufficient_permission));

        if caller_authorized {
            Ok(())
        } else {
            let caller_permissions = self.permissions_for_principal(caller);
            Err(GovernanceError::new_with_message(
                ErrorType::NotAuthorized,
                format!(
                    "Caller '{caller:?}' is not authorized to modify permissions {permissions_to_change} for neuron '{}' as it does not have any of {sufficient_permissions:?}. (Caller's permissions are {caller_permissions})",
                    self.id.as_ref().expect("Neuron must have a NeuronId"),
                ),
            ))
        }
    }
