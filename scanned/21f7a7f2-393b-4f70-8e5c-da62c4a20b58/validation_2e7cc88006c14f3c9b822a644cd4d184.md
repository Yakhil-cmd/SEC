### Title
Stale Gas Fee Estimate in `withdraw_erc20` Causes Permanently Stuck Withdrawals with No Reimbursement Path - (File: `rs/ethereum/cketh/minter/src/main.rs`)

---

### Summary

The ckETH minter's `withdraw_erc20` endpoint burns both ckETH (as gas fee) and ckERC20 tokens based on a potentially stale (up to 60-second-old) gas fee estimate. If Ethereum gas fees spike between the time of burning and the time the minter's background task attempts to create the Ethereum transaction, the withdrawal is rescheduled indefinitely with **no reimbursement path**, permanently locking the user's burned tokens.

---

### Finding Description

**Step 1 — Stale fee estimate used to burn tokens**

`withdraw_erc20` calls `estimate_erc20_transaction_fee()`, which delegates to `lazy_refresh_gas_fee_estimate()`: [1](#0-0) 

This function caches the gas fee estimate for up to **60 seconds** (`MAX_AGE_NS = 60_000_000_000`). If the cached estimate is fresh, it is returned without querying the EVM RPC. The resulting `erc20_tx_fee` is then used to burn ckETH from the user: [2](#0-1) 

After the ckETH burn succeeds, the ckERC20 tokens are also burned: [3](#0-2) 

Both burns are **irreversible** at this point. The `Erc20WithdrawalRequest` is queued with `max_transaction_fee = erc20_tx_fee` (the stale estimate).

**Step 2 — Transaction creation fails silently when fees have risen**

The background task `create_transactions_batch` later calls `create_transaction()` with the **current** gas fee estimate. For ckERC20 requests, `create_transaction` checks whether the current minimum `max_fee_per_gas` exceeds what was burned: [4](#0-3) 

If the current fee exceeds the burned amount, it returns `CreateTransactionError::InsufficientTransactionFee`. The handler in `create_transactions_batch` only **reschedules** the request — no reimbursement is issued: [5](#0-4) 

**Step 3 — No reimbursement path exists for this scenario**

The ckERC20 reimbursement logic only triggers when an Ethereum transaction is finalized with a failure receipt: [6](#0-5) 

If the transaction is **never created** (stuck in queue), `record_finalized_transaction` is never called, so no reimbursement is ever scheduled. There is no timeout or maximum retry count that would trigger reimbursement for permanently stuck requests.

---

### Impact Explanation

A user who calls `withdraw_erc20` during a period of rising Ethereum gas fees can have both their ckETH gas fee and their ckERC20 withdrawal amount permanently burned with no recovery. The ckERC20 tokens are destroyed on the ledger at withdrawal time; if the Ethereum transaction is never created, those tokens are gone. The ckETH burned as gas fee is similarly unrecoverable in this path. The withdrawal request loops in the pending queue indefinitely.

---

### Likelihood Explanation

Ethereum gas fees can spike dramatically within seconds (e.g., during high-demand events). The 60-second cache window in `lazy_refresh_gas_fee_estimate` creates a meaningful window where the burned fee estimate can become insufficient. Any unprivileged user calling `withdraw_erc20` is exposed to this race condition without any warning or protection. The DFINITY team acknowledged a related issue in the August 2024 minter upgrade: [7](#0-6) 

However, the current code still reschedules without reimbursing when `create_transaction` fails with `InsufficientTransactionFee`, leaving the permanent-stuck scenario unaddressed.

---

### Recommendation

When `create_transaction` returns `CreateTransactionError::InsufficientTransactionFee` for a ckERC20 request and the request has been rescheduled beyond a configurable maximum retry count or age threshold, the minter should:

1. Emit a `FailedErc20WithdrawalRequest` event to schedule reimbursement of the burned ckETH gas fee (minus the ledger transfer fee).
2. Schedule reimbursement of the burned ckERC20 tokens via a `ReimbursementRequest` on the ckERC20 ledger.

Alternatively, enforce a minimum withdrawal amount that accounts for the maximum possible fee increase across multiple resubmissions, and validate this at `withdraw_erc20` call time using a freshly fetched (non-cached) fee estimate before burning any tokens.

---

### Proof of Concept

1. Ethereum gas fees are at `X` wei/gas. The minter's cached estimate is `X`.
2. User calls `withdraw_erc20` with `amount = 1 ckUSDC`. The minter burns `X * 65_000` ckETH as gas fee and burns `1_000_000` ckUSDC.
3. Within 60 seconds, Ethereum gas fees spike to `10X` wei/gas.
4. `create_transactions_batch` runs. `create_transaction` computes `actual_min_max_fee_per_gas = 10X` which exceeds `request_max_fee_per_gas = X`. It returns `InsufficientTransactionFee`.
5. The handler calls `reschedule_withdrawal_request` — the request goes to the back of the queue.
6. Gas fees remain elevated. Steps 4–5 repeat indefinitely.
7. The user's `X * 65_000` ckETH and `1_000_000` ckUSDC are permanently burned with no reimbursement. [8](#0-7) [9](#0-8)

pub async fn lazy_refresh_gas_fee_estimate() -> Option<GasFeeEstimate> {
    const MAX_AGE_NS: u64 = 60_000_000_000_u64; //60 seconds

    async fn do_refresh() -> Option<GasFeeEstimate> {
        let _guard = match TimerGuard::new(TaskType::RefreshGasFeeEstimate) {
            Ok(guard) => guard,
            Err(e) => {
                log!(
                    DEBUG,
                    "[refresh_gas_fee_estimate]: Failed retrieving guard: {e:?}",
                );
                return None;
            }
        };

        let fee_history = match eth_fee_history().await {
            Ok(fee_history) => fee_history,
            Err(e) => {
                log!(
                    INFO,
                    "[refresh_gas_fee_estimate]: Failed retrieving fee history: {e:?}",
                );
                return None;
            }
        };

        let gas_fee_estimate = match estimate_transaction_fee(&fee_history) {
            Ok(estimate) => {
                mutate_state(|s| {
                    s.last_transaction_price_estimate =
                        Some((ic_cdk::api::time(), estimate.clone()));
                });
                estimate
            }
            Err(e) => {
                log!(
                    INFO,
                    "[refresh_gas_fee_estimate]: Failed estimating gas fee: {e:?}",
                );
                return None;
            }
        };
        log!(
            INFO,
            "[refresh_gas_fee_estimate]: Estimated transaction fee: {:?}",
            gas_fee_estimate,
        );
        Some(gas_fee_estimate)
    }

    async fn eth_fee_history() -> Result<FeeHistory, MultiCallError<FeeHistory>> {
        read_state(rpc_client)
            .fee_history((5_u8, BlockTag::Latest))
            .with_reward_percentiles(vec![20])
            .with_cycles(MIN_ATTACHED_CYCLES)
            .try_send()
            .await
            .reduce_with_strategy(StrictMajorityByKey::new(|fee_history: &FeeHistory| {
                Nat::from(fee_history.oldest_block.clone())
            }))
    }

    let now_ns = ic_cdk::api::time();
    match read_state(|s| s.last_transaction_price_estimate.clone()) {
        Some((last_estimate_timestamp_ns, estimate))
            if now_ns < last_estimate_timestamp_ns.saturating_add(MAX_AGE_NS) =>
        {
            Some(estimate)
        }
        _ => do_refresh().await,
    }
}

fn create_transactions_batch(gas_fee_estimate: GasFeeEstimate) {
    for request in read_state(|s| {
        s.eth_transactions
            .withdrawal_requests_batch(WITHDRAWAL_REQUESTS_BATCH_SIZE)
    }) {
        log!(DEBUG, "[create_transactions_batch]: processing {request:?}",);
        let ethereum_network = read_state(State::ethereum_network);
        let nonce = read_state(|s| s.eth_transactions.next_transaction_nonce());
        let gas_limit = estimate_gas_limit(&request);
        match create_transaction(
            &request,
            nonce,
            gas_fee_estimate.clone(),
            gas_limit,
            ethereum_network,
        ) {
            Ok(transaction) => {
                log!(
                    DEBUG,
                    "[create_transactions_batch]: created transaction {transaction:?}",
                );

                mutate_state(|s| {
                    process_event(
                        s,
                        EventType::CreatedTransaction {
                            withdrawal_id: request.cketh_ledger_burn_index(),
                            transaction,
                        },
                    );
                });
            }
            Err(CreateTransactionError::InsufficientTransactionFee {
                cketh_ledger_burn_index: ledger_burn_index,
                allowed_max_transaction_fee: withdrawal_amount,
                actual_max_transaction_fee: max_transaction_fee,
            }) => {
                log!(
                    INFO,
                    "[create_transactions_batch]: Withdrawal request with burn index {ledger_burn_index} has insufficient amount {withdrawal_amount:?} to cover transaction fees: {max_transaction_fee:?}. Request moved back to end of queue."
                );
                mutate_state(|s| s.eth_transactions.reschedule_withdrawal_request(request));
            }
        };
    }
