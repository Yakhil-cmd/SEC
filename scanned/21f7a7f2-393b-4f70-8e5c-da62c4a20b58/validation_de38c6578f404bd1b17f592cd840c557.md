### Title
Non-Whitelisted Unclaimed GTC Neurons Are Permanently Locked With No Recovery Mechanism - (File: rs/nns/gtc/src/lib.rs)

### Summary
The Genesis Token Canister (GTC) is the IC's genesis token distribution mechanism — the direct analog of an airdrop contract. GTC accounts that are **not** in the forwarding whitelist and whose owners never call `claim_neurons` or `donate_account` are permanently locked. There is no canister-level function to recover or redirect those neurons, and the ICP staked in them is irrecoverable without an NNS governance upgrade.

### Finding Description
The GTC was initialized at IC genesis with neurons pre-created on behalf of Seed Round (SR) and Early Contributor Token (ECT) holders. Each account owner can:

1. Call `claim_neurons` to take ownership of their neurons.
2. Call `donate_account` to voluntarily transfer them to a custodian.
3. Be forwarded via `forward_whitelisted_unclaimed_accounts` — but **only if** their address appears in `whitelisted_accounts_to_forward`.

The forwarding function in `rs/nns/gtc/src/lib.rs` contains the critical whitelist gate:

```rust
for (gtc_address, account) in self.accounts.iter_mut() {
    if !account.has_claimed
        && !account.has_donated
        && !account.has_forwarded
        && forward_whitelist.contains(gtc_address)   // ← non-whitelisted accounts skipped
    {
        match account.transfer(custodian_neuron_id).await { … }
    }
}
``` [1](#0-0) 

Accounts not in `whitelisted_accounts_to_forward` are silently skipped. The GTC canister exposes exactly four update methods — `claim_neurons`, `donate_account`, `forward_whitelisted_unclaimed_accounts`, and `get_build_metadata` — and none of them can touch a non-whitelisted, unclaimed account. [2](#0-1) 

The whitelist itself is a hard-coded compile-time constant (`FORWARD_WHITELIST`) injected only during `canister_post_upgrade`, meaning accounts absent from that list at deployment time have no forwarding path ever. [3](#0-2) 

The `AccountState` struct tracks `has_claimed`, `has_donated`, and `has_forwarded`, but there is no `has_recovered` or equivalent escape hatch. [4](#0-3) 

The integration test explicitly notes that forwarding is whitelist-scoped and that non-whitelisted accounts are intentionally left behind: [5](#0-4) 

### Impact Explanation
**Impact: High.** Any ICP staked in non-whitelisted, unclaimed GTC neurons is permanently locked inside NNS Governance under the GTC canister's controller identity. The GTC has no `dissolve`, `disburse`, or `transfer` method callable by the NNS root or any other privileged principal. Recovery requires an NNS governance proposal to upgrade the GTC canister with a new method — a non-trivial governance action that may never be proposed or pass. The locked ICP is effectively removed from circulation indefinitely.

### Likelihood Explanation
**Likelihood: Low.** The GTC was deployed at IC genesis in May 2021. Any SR or ECT holder who has not claimed, donated, or been forwarded by now is unlikely to do so. Key loss, death, or simple inaction are realistic causes. The 188-day forwarding window (`SECONDS_UNTIL_FORWARD_WHITELISTED_UNCLAIMED_ACCOUNTS_CAN_BE_CALLED`) has long since elapsed, so the only remaining path for non-whitelisted accounts is `claim_neurons` — which requires the original private key. [6](#0-5) 

### Recommendation
Add a permissioned canister method (callable only by the NNS root or a governance-controlled principal) that forwards **all** remaining unclaimed, non-whitelisted accounts to the existing custodian neuron after a sufficiently long grace period — mirroring the existing `forward_whitelisted_unclaimed_accounts` logic but without the whitelist filter. This is the direct IC analog of the M-01 recommendation to "add a function that allows the owner to withdraw unclaimed airdrop tokens after a certain period."

### Proof of Concept

1. At genesis, account `0xABCD…` (a non-whitelisted SR holder) is initialized in the GTC with 1 200 ICP worth of neurons.
2. The owner never calls `claim_neurons` (key lost, deceased, or inactive).
3. After 188 days, any caller invokes `forward_whitelisted_unclaimed_accounts`. The function iterates all accounts but skips `0xABCD…` because `forward_whitelist.contains("0xABCD…")` is `false`.
4. No other GTC method can touch `0xABCD…`'s neurons.
5. The 1 200 ICP remains staked under the GTC's controller identity in NNS Governance with no dissolution or transfer path, permanently locked. [7](#0-6) [8](#0-7)

    pub async fn forward_whitelisted_unclaimed_accounts(&mut self) -> Result<(), String> {
        self.assert_forward_whitelisted_unclaimed_accounts_can_be_called()?;
        let mut forward_whitelist = HashSet::new();

        for gtc_address in &self.whitelisted_accounts_to_forward {
            forward_whitelist.insert(gtc_address.to_string());
        }

        let custodian_neuron_id = self.forward_whitelisted_unclaimed_accounts_recipient_neuron_id;

        for (gtc_address, account) in self.accounts.iter_mut() {
            if !account.has_claimed
                && !account.has_donated
                && !account.has_forwarded
                && forward_whitelist.contains(gtc_address)
            {
                match account.transfer(custodian_neuron_id).await {
                    Ok(_) => account.has_forwarded = true,
                    Err(error) => {
                        println!(
                            "Error forwarding gtc account: {}. Error: {}",
                            gtc_address, error
                        );
                    }
                }
            }
        }

        Ok(())
    }
