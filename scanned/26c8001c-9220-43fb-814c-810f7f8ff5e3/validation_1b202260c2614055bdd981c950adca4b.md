### Title
Missing Removal Function for NNS Canister Registry Records - (File: `rs/nns/handlers/root/impl/src/canister_management.rs`)

### Summary
The NNS root canister exposes `add_nns_canister` to register new NNS canisters into the on-chain `NnsCanisterRecords` registry, but provides no corresponding removal function. Once a canister is registered, it can never be deregistered from the NNS canister records, even by governance. This is a direct analog to the Gearbox `ContractsRegister` issue where pools and managers could be added but never removed.

### Finding Description
`do_add_nns_canister` in `rs/nns/handlers/root/impl/src/canister_management.rs` writes a new `NnsCanisterRecord` entry into the registry under the `make_nns_canister_records_key()` key. The function inserts the canister name and ID into the `NnsCanisterRecords` protobuf map and commits it via `mutate_registry`. There is no corresponding `remove_nns_canister` or `do_remove_nns_canister` function anywhere in the NNS root canister codebase (confirmed by exhaustive grep). The canister's public interface (`canister.rs`) exposes `add_nns_canister` (governance-gated) but no removal counterpart. The code even contains a `TODO(NNS-81)` comment acknowledging that cleanup on failure is unimplemented.

The `NnsCanisterRecords` map therefore grows monotonically. Governance has no on-chain path to remove a stale, decommissioned, or compromised canister entry from this registry.

### Impact Explanation
Any canister added via `add_nns_canister` is permanently recorded in the NNS registry. If such a canister is later stopped, deleted, or found to be compromised, governance can stop or upgrade it via `stop_or_start_nns_canister` / `change_nns_canister`, but cannot remove its record from `NnsCanisterRecords`. The stale record persists indefinitely in the registry, polluting the authoritative NNS canister list. Any downstream logic that enumerates NNS canisters (e.g., `get_sns_canisters_summary`-style queries, canister status aggregation, or future governance tooling) will include the dead or compromised entry. This also means the name slot is permanently occupied, preventing re-use of the same canister name for a replacement.

### Likelihood Explanation
NNS canisters are expected to evolve over time. The IC already has a `TODO(NNS-81)` acknowledging the need for cleanup. As the NNS grows and canisters are added experimentally or need replacement, the inability to remove records becomes a practical operational and security limitation. The likelihood of needing removal is non-trivial given the long-lived nature of the protocol.

### Recommendation
Add a `remove_nns_canister` update method to the NNS root canister, callable only by the governance canister (mirroring the access control on `add_nns_canister`). The implementation should issue a `Type::Update` or `Type::Delete` registry mutation to remove the named entry from `NnsCanisterRecords`. Additionally, resolve `TODO(NNS-81)` to ensure partial-failure cleanup during `do_add_nns_canister` is handled correctly.

### Proof of Concept

1. Governance submits a proposal that calls `add_nns_canister` on the NNS root canister with a new canister name and WASM.
2. `do_add_nns_canister` executes, inserting the entry into `NnsCanisterRecords` in the registry.
3. The newly added canister is later found to be compromised or needs to be decommissioned.
4. Governance attempts to remove it — no such proposal type or root canister method exists.
5. The entry remains permanently in `NnsCanisterRecords`; the name slot is permanently occupied and the dead canister continues to appear in all NNS canister enumerations.

pub async fn do_add_nns_canister(request: AddCanisterRequest) {
    let key = make_nns_canister_records_key().into_bytes();
    let name = request.name.clone();

    // We first need to claim the name of this new canister. Indeed, even though we
    // don't yet know its ID, if we create it first and then find out the name
    // is already taken, that would be bad. So, first, we create a record.

    // First, just to make sure that the key exists, because the libraries around
    // the registry don't have good ways to deal with defaults. So we try to
    // insert, but do nothing with the results.
    let _ = mutate_registry(
        vec![RegistryMutation {
            mutation_type: Type::Insert as i32,
            key: key.clone(),
            value: Vec::<u8>::new(),
        }],
        Vec::<Precondition>::new(),
    )
    .await;

    // Now the key should exist. Let's get its value.
    let (mut nns_canister_records, nns_canister_records_version): (NnsCanisterRecords, u64) =
        get_value(&key, None).await.unwrap();

    // Make sure the name is free
    let old_record = nns_canister_records
        .canisters
        .insert(name.clone(), NnsCanisterRecord::default());
    assert!(
        old_record.is_none(),
        "Trying to add an NNS canister called '{}', but we already have \
             a record for that name: '{:?}'",
        request.name,
        old_record
    );

    // Commit, so as to reserve the name
    let name_reserved_version = mutate_registry(
        vec![RegistryMutation {
            mutation_type: Type::Update as i32,
            key: key.clone(),
            value: nns_canister_records.encode_to_vec(),
        }],
        vec![Precondition {
            key: key.clone(),
            expected_version: nns_canister_records_version,
        }],
    )
    .await
    .unwrap();

    let id_or_error = try_to_create_and_install_canister(request).await;
    let id = id_or_error.unwrap();
    // TODO(NNS-81): If it did not work, remove the name from the registry

    nns_canister_records.canisters.insert(
        name.clone(),
        NnsCanisterRecord {
            id: Some(pb::CanisterId::from(id)),
        },
    );

    mutate_registry(
        vec![RegistryMutation {
            mutation_type: Type::Update as i32,
            key: key.clone(),
            value: nns_canister_records.encode_to_vec(),
        }],
        vec![Precondition {
            key: key.clone(),
            expected_version: name_reserved_version,
        }],
    )
    .await
    .unwrap();
    // TODO(NNS-81): Handle failure in the case we couldn't write the canister id
    // into the registry
}
