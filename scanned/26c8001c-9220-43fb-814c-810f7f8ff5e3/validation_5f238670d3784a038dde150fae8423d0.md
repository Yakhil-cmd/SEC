### Title
Missing Staleness Check on Cached ICP/XDR Rate in `tokens_to_cycles()` Before Cycles Minting - (File: rs/nns/cmc/src/main.rs)

---

### Summary

The Cycles Minting Canister (CMC) caches the ICP/XDR conversion rate fetched from the Exchange Rate Canister (XRC). The `tokens_to_cycles()` function, which is called on every `notify_top_up` and `notify_create_canister` ingress, reads `state.icp_xdr_conversion_rate.xdr_permyriad_per_icp` and uses it directly to mint cycles without ever checking whether the cached rate's `timestamp_seconds` is recent. If the XRC becomes unavailable for an extended period (or the CMC's update mechanism enters `Disabled` state), the CMC silently continues minting cycles at an arbitrarily stale rate.

---

### Finding Description

`tokens_to_cycles()` in `rs/nns/cmc/src/main.rs` reads the cached `icp_xdr_conversion_rate` and extracts only `xdr_permyriad_per_icp`, discarding `timestamp_seconds` entirely:

```rust
fn tokens_to_cycles(amount: Tokens) -> Result<Cycles, NotifyError> {
    with_state(|state| {
        let xdr_permyriad_per_icp = state
            .icp_xdr_conversion_rate
            .as_ref()
            .map(|rate| rate.xdr_permyriad_per_icp);   // timestamp_seconds silently dropped
        match xdr_permyriad_per_icp {
            Some(xdr_permyriad_per_icp) => Ok(TokensToCycles {
                xdr_permyriad_per_icp,
                cycles_per_xdr: state.cycles_per_xdr,
            }.to_cycles(amount)),
            None => Err(...)
        }
    })
}
``` [1](#0-0) 

The `IcpXdrConversionRate` struct carries a `timestamp_seconds` field that records when the market data was queried: [2](#0-1) 

The CMC's `update_exchange_rate()` heartbeat task refreshes this rate every `REFRESH_RATE_INTERVAL_SECONDS = 5 * ONE_MINUTE_SECONDS`: [3](#0-2) 

However, the `validate_exchange_rate()` function called during ingestion only validates the number of data sources — it contains no check that the returned rate's timestamp is recent relative to the current canister time: [4](#0-3) 

When the XRC is unavailable (e.g., XRC subnet degradation) or when the CMC's update state is set to `Disabled` (via a governance proposal with `DivergedRate` reason), the heartbeat stops refreshing the rate: [5](#0-4) 

In either case, `tokens_to_cycles()` continues to use the last cached rate indefinitely, with no maximum-age guard at the point of use.

The same stale rate is also consumed without a staleness check by the SNS token valuation library, which calls `get_average_icp_xdr_conversion_rate` on the CMC and uses `response.data.xdr_permyriad_per_icp` directly: [6](#0-5) 

---

### Impact Explanation

Cycles are the fundamental economic unit of the Internet Computer. `tokens_to_cycles()` is invoked on every `notify_top_up` and `notify_create_canister` call, which are the primary mechanisms by which ICP is burned and cycles are minted. If the cached rate is stale:

- **Rate too low (ICP price has risen):** Users receive more cycles per ICP than the current market rate warrants — cycles are over-minted, inflating the cycles supply and effectively subsidizing callers at the protocol's expense.
- **Rate too high (ICP price has fallen):** Users receive fewer cycles per ICP than they are entitled to — cycles are under-minted, causing economic loss to users.

For the SNS treasury valuation path, a stale rate causes incorrect XDR valuations of SNS treasuries, which can either block legitimate large-transfer proposals or allow transfers that exceed the intended safety limits. [7](#0-6) 

---

### Likelihood Explanation

The XRC is a system canister on a dedicated subnet. Any subnet-level degradation, upgrade, or temporary unavailability causes the CMC's heartbeat to fail to refresh the rate. The CMC's own `UpdateExchangeRateState::Disabled` path (reachable via a governance proposal) also freezes the rate indefinitely. The `notify_top_up` and `notify_create_canister` endpoints are publicly callable by any principal without special permissions, making the stale-rate window exploitable by any user who monitors the rate age and times their calls accordingly. [8](#0-7) 

---

### Recommendation

1. In `tokens_to_cycles()`, check that `now_seconds() - icp_xdr_conversion_rate.timestamp_seconds` does not exceed a defined maximum acceptable age (e.g., 30 minutes or 1 hour) before using the rate. Return a retriable `NotifyError` if the rate is too old.
2. In `validate_exchange_rate()`, add a `max_age_seconds` parameter and reject rates whose `timestamp` is older than the threshold relative to the caller's current time.
3. Apply the same staleness guard in `CmcBased30DayMovingAverageXdrsPerIcpClient::get()` in the SNS token valuation library.

---

### Proof of Concept

1. The CMC's XRC update heartbeat is running normally, last updating `icp_xdr_conversion_rate` at time `T` with rate `R`.
2. The XRC subnet experiences degradation; all subsequent `update_exchange_rate()` calls return `FailedToRetrieveRate`.
3. Time advances by several hours. The real ICP/XDR rate moves significantly from `R` to `R'`.
4. An attacker (or any user) calls `notify_top_up` with an ICP transfer. The CMC calls `tokens_to_cycles()`, which reads `state.icp_xdr_conversion_rate` — still `R` from step 1 — and mints cycles at the stale rate without any age check.
5. If `R' > R` (ICP price rose), the attacker receives more cycles than the current market rate warrants. If `R' < R`, the user is shortchanged. [1](#0-0) [9](#0-8)

pub async fn update_exchange_rate(
    safe_state: &'static LocalKey<RefCell<Option<State>>>,
    env: &impl Environment,
    xrc_client: &impl ExchangeRateCanisterClient,
) -> Result<(), UpdateExchangeRateError> {
    let now_timestamp_seconds = env.now_timestamp_seconds();
    let current_minute_seconds =
        round_down_to_multiple_of(now_timestamp_seconds, ONE_MINUTE_SECONDS);

    UpdateExchangeRateGuard::with_guard(safe_state, current_minute_seconds, async {
        let call_xrc_result = xrc_client.get_icp_to_xdr_exchange_rate(None).await;
        // Check if updating the rate via the exchange rate canister was disabled while retrieving the rate.
        // If it has, exit early.
        let is_updating_rate_disabled = read_state(safe_state, |state| {
            state
                .update_exchange_rate_canister_state
                .unwrap_or_default()
                == UpdateExchangeRateState::Disabled
        });
        if is_updating_rate_disabled {
            return Err(UpdateExchangeRateError::Disabled);
        }

        match call_xrc_result {
            Ok(exchange_rate) => {
                validate_exchange_rate(&exchange_rate)
                    .map_err(|error| UpdateExchangeRateError::InvalidRate(error.to_string()))?;
                let icp_xdr_conversion_rate = IcpXdrConversionRate::from(exchange_rate);
                if let Err(error) =
                    do_set_icp_xdr_conversion_rate(safe_state, env, icp_xdr_conversion_rate)
                {
                    return Err(UpdateExchangeRateError::FailedToSetRate(error));
                }
            }
            Err(error) => {
                return Err(UpdateExchangeRateError::FailedToRetrieveRate(
                    error.to_string(),
                ));
            }
        };

        Ok(())
    })
    .await
}

pub fn validate_exchange_rate(
    exchange_rate: &ExchangeRate,
) -> Result<(), ValidateExchangeRateError> {
    if exchange_rate.metadata.base_asset_num_received_rates < MINIMUM_ICP_SOURCES {
        return Err(ValidateExchangeRateError::NotEnoughIcpSources {
            received: exchange_rate.metadata.base_asset_num_received_rates,
            queried: exchange_rate.metadata.base_asset_num_queried_sources,
        });
    }

    if exchange_rate.metadata.quote_asset_num_received_rates < MINIMUM_CXDR_SOURCES {
        return Err(ValidateExchangeRateError::NotEnoughCxdrSources {
            received: exchange_rate.metadata.quote_asset_num_received_rates,
            queried: exchange_rate.metadata.quote_asset_num_queried_sources,
        });
    }

    Ok(())
}
