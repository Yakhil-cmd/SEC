### Title
`settle_neurons_fund_participation` Authorizes Any SNS Swap Canister to Settle Any Proposal's Neurons' Fund — (`rs/nns/governance/src/governance.rs`)

### Summary

`Governance::settle_neurons_fund_participation` in NNS governance checks only that the caller is *any* deployed SNS swap canister, not the specific swap canister that was created by the `CreateServiceNervousSystem` proposal being settled. This is the direct IC analog of the DeBridge bug: the wrong context identifier is used for the authorization lookup. The code even contains an explicit acknowledgment of the flaw in a comment.

### Finding Description

When an SNS swap concludes, its swap canister calls `settle_neurons_fund_participation` on NNS governance, passing the `nns_proposal_id` of the `CreateServiceNervousSystem` proposal that created it. NNS governance then either mints ICP to the SNS treasury (Committed) or refunds reserved Neurons' Fund maturity (Aborted).

The authorization check in `settle_neurons_fund_participation` is:

```rust
// Check authorization. Note that a Swap could settle each other's participation.
let target_canister_id: CanisterId = caller.try_into()...?;
if let Err(err_msg) =
    is_canister_id_valid_swap_canister_id(target_canister_id, &*self.env).await
{ ... }
``` [1](#0-0) 

`is_canister_id_valid_swap_canister_id` queries SNS-W's `list_deployed_snses` and checks whether the caller appears as *any* swap canister in the full list:

```rust
let is_swap = list_deployed_snses_response
    .instances
    .iter()
    .any(|sns| sns.swap_canister_id == Some(target_canister_id.into()));
``` [2](#0-1) 

There is no check that the caller is the swap canister *specifically associated with `request.nns_proposal_id`*. The proposal data does not store the swap canister ID — `execute_create_service_nervous_system_proposal` receives the `DeployNewSnsResponse` (which contains the swap canister ID) but only prints it and never persists it to `ProposalData`: [3](#0-2) 

Consequently, SNS-A's swap canister can call `settle_neurons_fund_participation` supplying SNS-B's `nns_proposal_id`. The check passes because SNS-A is a valid swap canister. The caller also supplies `sns_governance_canister_id` in the `Committed` payload — this is the address to which ICP is minted — and can set it to SNS-A's governance canister: [4](#0-3) 

The `Committed` payload structure that carries the destination: [5](#0-4) 

### Impact Explanation

**Committed path:** SNS-A's swap canister calls with SNS-B's `nns_proposal_id`, `result = Committed`, and `sns_governance_canister_id = SNS_A_GOVERNANCE`. NNS governance mints the Neurons' Fund ICP reserved for SNS-B and sends it to SNS-A's treasury. SNS-B's proposal lifecycle is marked `Committed`, preventing any future legitimate settlement. SNS-B never receives the ICP it was entitled to.

**Aborted path:** SNS-A's swap canister calls with SNS-B's `nns_proposal_id` and `result = Aborted`. NNS governance refunds all Neurons' Fund maturity reserved for SNS-B back to individual neurons. SNS-B's proposal lifecycle is marked `Aborted`. Even if SNS-B's swap later commits legitimately, the Neurons' Fund ICP is gone — the funds are permanently unavailable to SNS-B's treasury.

Both paths cause irreversible loss or misdirection of ICP/maturity from the Neurons' Fund.

### Likelihood Explanation

The attacker must control a deployed SNS swap canister. SNS swap canisters are deployed by SNS-W under NNS governance, so the attacker must have had an SNS proposal pass. This is a non-trivial but realistic bar: dozens of SNSes have been deployed on mainnet. Any one of them whose swap canister is compromised, buggy, or maliciously coded could exploit this. The code comment at line 7018 explicitly acknowledges the gap ("a Swap could settle each other's participation"), confirming the developers are aware but have not closed it.

### Recommendation

1. When `execute_create_service_nervous_system_proposal` receives `DeployNewSnsResponse`, persist the swap canister ID into `ProposalData`.
2. In `settle_neurons_fund_participation`, after retrieving `proposal_data`, assert that `caller == proposal_data.swap_canister_id`.
3. In the `Committed` branch, validate that the caller-supplied `sns_governance_canister_id` matches the one recorded in `proposal_data` (or derive it from the stored swap canister ID via a registry lookup), rather than accepting it as a free parameter.

### Proof of Concept

1. Deploy SNS-A legitimately via NNS governance (proposal P-A). SNS-A's swap canister ID is `SWAP_A`.
2. Deploy SNS-B legitimately via NNS governance (proposal P-B, with Neurons' Fund participation enabled). NNS governance reserves maturity for SNS-B.
3. From `SWAP_A`, call NNS governance's `settle_neurons_fund_participation` with:
   - `nns_proposal_id = P-B`
   - `result = Committed { sns_governance_canister_id = SNS_A_GOVERNANCE, total_direct_participation_icp_e8s = MAX }`
4. `is_canister_id_valid_swap_canister_id(SWAP_A)` returns `Ok` because `SWAP_A` is in `list_deployed_snses`.
5. NNS governance mints Neurons' Fund ICP and sends it to `SNS_A_GOVERNANCE`. SNS-B's proposal lifecycle is set to `Committed`.
6. SNS-B's legitimate swap canister later calls `settle_neurons_fund_participation` for P-B; the call fails because the lifecycle is already terminal. SNS-B's treasury receives nothing. [1](#0-0) [6](#0-5) [3](#0-2)

/// Given a target_canister_id, is it a CanisterId of a deployed SNS recorded by
/// the SNS-W canister.
async fn is_canister_id_valid_swap_canister_id(
    target_canister_id: CanisterId,
    env: &dyn Environment,
) -> Result<(), String> {
    let list_deployed_snses_response = env
        .call_canister_method(
            SNS_WASM_CANISTER_ID,
            "list_deployed_snses",
            Encode!(&ListDeployedSnsesRequest {}).expect(""),
        )
        .await
        .map_err(|err| {
            format!(
                "Failed to call the list_deployed_snses method on sns_wasm ({SNS_WASM_CANISTER_ID}): {err:?}",
            )
        })?;

    let list_deployed_snses_response =
        Decode!(&list_deployed_snses_response, ListDeployedSnsesResponse).map_err(|err| {
            format!(
                "Unable to decode response as ListDeployedSnsesResponse: {err}. reply_bytes = {list_deployed_snses_response:#?}",
            )
        })?;

    let is_swap = list_deployed_snses_response
        .instances
        .iter()
        .any(|sns| sns.swap_canister_id == Some(target_canister_id.into()));
    if !is_swap {
        return Err(format!(
            "target_swap_canister_id is not the ID of any swap canister known to sns_wasm: {target_canister_id}"
        ));
    }

    Ok(())
}
