### Title
`is_funded()` Omits `staked_maturity_e8s_equivalent`, Causing Dissolved Neurons With Only Staked Maturity to Be Classified Inactive and Skipped by the Unstake-Maturity Timer Task — (`rs/nns/governance/src/neuron/types.rs`)

### Summary

`Neuron::is_funded()` does not account for `staked_maturity_e8s_equivalent`. A neuron that holds only staked maturity (zero ICP stake, zero regular maturity) is therefore considered "not funded," and once it has been dissolved for more than two weeks it satisfies every condition of `is_inactive()`. The periodic timer task that moves staked maturity to regular maturity iterates exclusively over active neurons via `with_active_neurons_iter_sections()`, so such a neuron is permanently skipped. The staked maturity is never converted to spendable maturity, violating the protocol invariant documented in both the code comments and the existing proptest.

### Finding Description

`Neuron::is_funded()` computes:

```rust
let amount_e8s = self.stake_e8s().saturating_add(self.maturity_e8s_equivalent);
amount_e8s > 0
``` [1](#0-0) 

`staked_maturity_e8s_equivalent` is absent. `is_inactive()` returns `true` when `is_funded()` is `false` **and** the neuron dissolved more than two weeks ago. [2](#0-1) 

`with_active_neurons_iter_sections()` filters the full neuron set to those where `!n.is_inactive(now)`: [6](#0-5) 

`list_neurons_ready_to_unstake_maturity()` — the only discovery path for the timer task — calls that filtered iterator: [4](#0-3) 

`ready_to_unstake_maturity()` correctly identifies the neuron (dissolved + staked maturity > 0): [7](#0-6) 

But the neuron is never reached because it is filtered out one level above.

**Reachable scenario (no privileged access required):**

1. User creates a neuron with ICP stake and stakes some maturity via `StakeMaturity`.
2. User starts dissolving; dissolve delay > 2 weeks.
3. Neuron reaches `Dissolved` state.
4. User calls `Disburse` to withdraw the ICP stake. `cached_neuron_stake_e8s` becomes 0.
5. More than two weeks elapse after dissolution (or the disburse happens after the two-week mark).
6. State: `cached_neuron_stake_e8s = 0`, `maturity_e8s_equivalent = 0`, `staked_maturity_e8s_equivalent > 0`, dissolved > 2 weeks ago.
7. `is_funded()` → `false`; `is_inactive()` → `true`.
8. Every subsequent run of `UnstakeMaturityOfDissolvedNeuronsTask` skips this neuron.
9. `staked_maturity_e8s_equivalent` is never moved to `maturity_e8s_equivalent`; the user cannot spawn or disburse it.

The existing proptest explicitly encodes the intended invariant (staked maturity counts toward "funded") and would fail for this exact input combination: [8](#0-7) 

### Impact Explanation

The user's staked maturity is inaccessible: it cannot be spawned, disbursed, or otherwise converted. Recovery requires the user to either (a) transfer additional ICP to the neuron's ledger account and call `RefreshNeuron` to raise `cached_neuron_stake_e8s` above zero, or (b) call `IncreaseDissolveDelay` (which re-locks the neuron and requires waiting out a new dissolve delay before the maturity can be accessed). Neither recovery path is obvious or documented. For users who have already disbursed their stake and are unaware of the bug, the staked maturity appears permanently inaccessible.

### Likelihood Explanation

The scenario is reachable by any governance participant who (1) staked maturity while their neuron was non-dissolving, (2) dissolved the neuron, and (3) disbursed the ICP stake more than two weeks after dissolution. This is a natural usage pattern (stake maturity for voting power, dissolve, disburse ICP, expect maturity to be available). The proptest mismatch confirms the implementation diverges from the intended invariant.

### Recommendation

Include `staked_maturity_e8s_equivalent` in `is_funded()`:

```rust
pub fn is_funded(&self) -> bool {
    let amount_e8s = self
        .stake_e8s()
        .saturating_add(self.staked_maturity_e8s_equivalent.unwrap_or(0))
        .saturating_add(self.maturity_e8s_equivalent);
    amount_e8s > 0
}
```

This aligns the implementation with the proptest invariant and ensures that any neuron holding staked maturity is never classified as inactive, guaranteeing the timer task will eventually process it.

        if self.is_seed_neuron() || self.is_ect_neuron() {

        if self.is_a_neurons_fund_member() {

        true
