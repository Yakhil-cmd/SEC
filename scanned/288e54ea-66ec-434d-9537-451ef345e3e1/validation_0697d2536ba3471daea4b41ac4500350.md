### Title
ICRC-2 Allowance Granted to Extension Canister Before Code Is Installed Leaves Treasury Funds Exposed If Installation Fails - (`File: rs/sns/governance/src/extensions.rs`)

### Summary

In `rs/sns/governance/src/extensions.rs`, the SNS Governance canister's `approve_treasury_manager` function grants an ICRC-2 allowance over SNS treasury tokens (SNS token and ICP) to a `treasury_manager_canister_id` that is fully controlled by the proposal submitter at the time the approval is issued. The approval is granted **before** the blessed Wasm is installed on the extension canister. If the subsequent `upgrade_non_root_canister` call fails (e.g., due to a transient error), the allowance remains live on the ledger for up to one hour, pointing to a canister that still has no code installed (or whose code was not yet replaced). The cleanup path (`clean_up_failed_register_extension`) only removes the canister from Root's registry — it does **not** revoke the ICRC-2 allowance. An attacker who controls the extension canister can exploit this window to call `icrc2_transfer_from` on the SNS or ICP ledger and drain the approved amount from the SNS treasury.

### Finding Description

The execution sequence in `ValidatedRegisterExtension::execute` is:

1. `register_extension_with_root` — Root takes control of the canister.
2. `ensure_no_code_is_installed` — Checks the canister is empty.
3. `approve_treasury_manager` — **Issues ICRC-2 allowances** to the extension canister on both the SNS token ledger and the ICP ledger.
4. `upgrade_non_root_canister` — Installs the blessed Wasm. [1](#0-0) 

`approve_treasury_manager` calls `icrc2_approve` on both ledgers with `expected_allowance: None`, granting the extension canister principal an allowance of `treasury_allocation_sns_e8s` / `treasury_allocation_icp_e8s` that expires in one hour: [2](#0-1) 

If step 4 (`upgrade_non_root_canister`) fails for any reason, the error handler calls `clean_up_failed_register_extension`, which only asks Root to deregister the canister: [3](#0-2) [4](#0-3) 

**The ICRC-2 allowances are never revoked.** For up to `ONE_HOUR_SECONDS`, the extension canister principal retains a live allowance over the SNS treasury subaccount on both ledgers.

The same pattern is repeated in `execute_treasury_manager_deposit` (the `ExecuteExtensionOperation` / deposit path): `approve_treasury_manager` is called first, then `deposit` is called on the extension canister. If `deposit` fails or the extension canister misbehaves, the allowance is again left live with no revocation: [5](#0-4) 

The `ALLOWED_EXTENSIONS` map is currently **empty in production** (KongSwap was removed on April 6, 2026), so no new `RegisterExtension` proposals can currently pass `validate_extension_wasm`. However, the `ExecuteExtensionOperation` deposit path is reachable for any **already-registered** extension, and the same allowance-before-action ordering applies there. [6](#0-5) 

### Impact Explanation

If the `upgrade_non_root_canister` call fails after `approve_treasury_manager` succeeds, the extension canister (which the attacker still controls, since Root only took control in step 1 and the install failed) holds a live ICRC-2 allowance. The attacker can call `icrc2_transfer_from` on the SNS token ledger and/or ICP ledger to transfer up to the approved amount out of the SNS treasury subaccount before the one-hour expiry. The maximum loss per incident is bounded by the approved amount (up to 50% of the treasury balance per the validation check), but this can be repeated across multiple proposals. [7](#0-6) 

### Likelihood Explanation

This requires a governance proposal to pass (SNS token majority), which is a significant barrier. However, the SNS governance model allows any token holder with sufficient stake to submit proposals, and a malicious actor who accumulates enough voting power (or exploits a low-participation SNS) can submit a `RegisterExtension` or `ExecuteExtensionOperation` proposal with a canister they control. The one-hour window is generous. The `ExecuteExtensionOperation` deposit path is the more realistic attack vector since it does not require a new extension to be blessed — it only requires an already-registered extension canister to be present.

### Recommendation

1. **Revoke allowances in the cleanup path.** `clean_up_failed_register_extension` and the error handler in `execute_treasury_manager_deposit` should call `icrc2_approve` with `amount = 0` on both ledgers to revoke the allowance before returning an error.

2. **Invert the order of operations.** Install the Wasm first, then grant the allowance. This eliminates the window entirely for the `RegisterExtension` path.

3. **Use `expected_allowance`** when issuing approvals so that a second approval cannot silently overwrite a first one, preventing double-spend across two concurrent proposals. [8](#0-7) 

### Proof of Concept

**Setup:** An SNS with a registered extension canister `E` (controlled by attacker `A`). The SNS treasury holds 1,000,000 SNS tokens and 500,000 ICP.

1. Attacker `A` submits an `ExecuteExtensionOperation` proposal with `operation_name = "deposit"`, `treasury_allocation_sns_e8s = 500_000_00` (≤50%), `treasury_allocation_icp_e8s = 250_000_00` (≤50%). The proposal passes.

2. SNS Governance executes `execute_treasury_manager_deposit`:
   - Calls `approve_treasury_manager(E, 500_000_00, 250_000_00)`.
   - Both ledgers now record: `allowance(governance_treasury → E) = 500_000_00 SNS` and `250_000_00 ICP`, expiring in 1 hour.
   - Calls `E.deposit(...)`. Attacker's canister `E` returns an error (or the call traps).

3. `execute_treasury_manager_deposit` returns `Err`. No allowance revocation occurs.

4. Within the one-hour window, attacker calls `icrc2_transfer_from` on the SNS ledger: `from = governance_treasury_subaccount`, `to = attacker_wallet`, `amount = 500_000_00`. The ledger accepts because the allowance is still live.

5. Repeat for ICP ledger. [9](#0-8) [2](#0-1)

async fn validate_deposit_operation_impl(
    governance: &Governance,
    value: Option<Precise>,
) -> Result<ValidatedDepositOperationArg, String> {
    let structurally_valid = ValidatedDepositOperationArg::try_from(value)?;

    let sns_subaccount = governance.sns_treasury_subaccount();
    let icp_subaccount = governance.icp_treasury_subaccount();

    // Fail if either is asking for more than 50% of current balance.  The balance could have changed
    // since the proposal was created, and we don't assume that the proposal should work
    let sns_balance = governance
        .ledger
        .account_balance(Account {
            owner: governance.env.canister_id().get().0,
            subaccount: sns_subaccount,
        })
        .await
        .map_err(|e| format!("Failed to get SNS treasury balance: {e:?}"))?;
    let icp_balance = governance
        .nns_ledger
        .account_balance(Account {
            owner: governance.env.canister_id().get().0,
            subaccount: icp_subaccount,
        })
        .await
        .map_err(|e| format!("Failed to get ICP treasury balance: {e:?}"))?;

    let icp_requested = Tokens::from_e8s(structurally_valid.treasury_allocation_icp_e8s);
    let sns_requested = Tokens::from_e8s(structurally_valid.treasury_allocation_sns_e8s);

    // Unwrap is safe, only fails if divisor is zero, which we don't do.
    if sns_requested > sns_balance.checked_div(2).unwrap() {
        return Err(format!(
            "SNS treasury deposit request of {sns_requested} exceeds 50% of current SNS Token balance of {sns_balance}"
        ));
    }

    if icp_requested > icp_balance.checked_div(2).unwrap() {
        return Err(format!(
            "ICP treasury deposit request of {icp_requested} exceeds 50% of current ICP balance of {icp_balance}"
        ));
    }

    Ok(structurally_valid)

    async fn clean_up_failed_register_extension(&self, extension_canister_id: CanisterId) {
        let main = async || -> Result<(), String> {
            let request = Encode!(&CleanUpFailedRegisterExtensionRequest {
                canister_id: Some(extension_canister_id.get()),
            })
            .map_err(|err| {
                format!("Could not encode Root.clean_up_failed_register_extension request: {err:?}")
            })?;

            // Call clean_up_failed_register_extension method of the SNS Root canister.
            let reply = self
                .env
                .call_canister(
                    self.proto.root_canister_id_or_panic(),
                    "clean_up_failed_register_extension",
                    request,
                )
                .await
                .map_err(|err| {
                    format!("Failed to call Root.clean_up_failed_register_extension: {err:?}")
                })?;

            // Make sure reply does not indicate any problems.
            let CleanUpFailedRegisterExtensionResponse { result } =
                Decode!(&reply, CleanUpFailedRegisterExtensionResponse).map_err(|err| {
                    format!(
                        "Failed to decode Root.clean_up_failed_register_extension response: {err:?}"
                    )
                })?;
            if let Some(clean_up_failed_register_extension_response::Result::Err(
                CanisterCallError { code, description },
            )) = result
            {
                let code = if let Some(code) = code {
                    code.to_string()
                } else {
                    "<no code>".to_string()
                };

                return Err(format!(
                    "Root.clean_up_failed_register_extension failed with code {code}: {description}"
                ));
            }

            Ok(())
        };

    async fn approve_treasury_manager(
        &self,
        treasury_manager_canister_id: CanisterId,
        sns_amount_e8s: u64,
        icp_amount_e8s: u64,
    ) -> Result<(), GovernanceError> {
        let to = Account {
            owner: treasury_manager_canister_id.get().0,
            subaccount: None,
        };

        let expiry_time_sec = self.env.now().saturating_add(ONE_HOUR_SECONDS);
        let expiry_time_nsec = expiry_time_sec.saturating_mul(NANO_SECONDS_PER_SECOND);

        // If expected_allowance is None, the ledger *blindly* overwrites any existing
        // allowance (even if non-zero). Therefore, there is no risk of double spending.

        self.ledger
            .icrc2_approve(
                to,
                sns_amount_e8s,
                Some(expiry_time_nsec),
                self.transaction_fee_e8s_or_panic(),
                self.sns_treasury_subaccount(),
                None,
            )
            .await
            .map(|_| ())
            .map_err(|e| {
                GovernanceError::new_with_message(
                    ErrorType::External,
                    format!("Error making SNS Token treasury transfer: {e}"),
                )
            })?;

        self.nns_ledger
            .icrc2_approve(
                to,
                icp_amount_e8s,
                Some(expiry_time_nsec),
                icp_ledger::DEFAULT_TRANSFER_FEE.get_e8s(),
                self.icp_treasury_subaccount(),
                None,
            )
            .await
            .map(|_| ())
            .map_err(|e| {
                GovernanceError::new_with_message(
                    ErrorType::External,
                    format!("Error making ICP Token treasury transfer: {e}"),
                )
            })?;

        Ok(())
    }

/// Execute a treasury manager deposit operation
async fn execute_treasury_manager_deposit(
    governance: &Governance,
    extension_canister_id: CanisterId,
    arg: ValidatedDepositOperationArg,
) -> Result<(), GovernanceError> {
    let ValidatedDepositOperationArg {
        treasury_allocation_sns_e8s,
        treasury_allocation_icp_e8s,
        original,
    } = arg;

    let context = governance.treasury_manager_deposit_context().await?;
    let arg_blob =
        construct_treasury_manager_deposit_payload(context, original).map_err(|err| {
            GovernanceError::new_with_message(
                ErrorType::PreconditionFailed,
                format!("Failed to construct treasury manager deposit payload: {err}"),
            )
        })?;

    // 1. Transfer funds from treasury to treasury manager
    governance
        .approve_treasury_manager(
            extension_canister_id,
            treasury_allocation_sns_e8s,
            treasury_allocation_icp_e8s,
        )
        .await?;

    // 2. Call deposit on treasury manager
    let balances = governance
        .env
        .call_canister(extension_canister_id, "deposit", arg_blob)
        .await
        .map_err(|(code, err)| {
            GovernanceError::new_with_message(
                ErrorType::External,
                format!(
                    "Canister method call {extension_canister_id}.deposit failed with code {code:?}: {err}"
                ),
            )
        })
        .and_then(|blob| {
            Decode!(&blob, sns_treasury_manager::TreasuryManagerResult).map_err(|err| {
                GovernanceError::new_with_message(
                    ErrorType::External,
                    format!("Error decoding TreasuryManager.deposit response: {err:?}"),
                )
            })
        })?
        .map_err(|err| {
            GovernanceError::new_with_message(
                ErrorType::External,
                format!("TreasuryManager.deposit failed: {err:?}"),
            )
        })?;

    log!(
        INFO,
        "TreasuryManager.deposit succeeded with response: {:?}",
        balances
    );

    Ok(())
}
