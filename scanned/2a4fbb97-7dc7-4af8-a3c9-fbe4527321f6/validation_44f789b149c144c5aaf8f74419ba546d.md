### Title
Lack of Expiry/Cancellation in `sender_info` Canister Signatures — (`rs/validator/src/ingress_validation.rs`, `rs/types/types/src/messages/http.rs`)

---

### Summary

The IC's `sender_info` feature allows a signing canister (e.g., Internet Identity) to attach a canister-signed attestation blob to any ingress message. Unlike `Delegation`, which carries an `expiration` field that is validated at the replica, the `sender_info` signed content (`SenderInfoContent`) contains only the raw `info` bytes with no expiry, no message binding, and no canister-target binding. Once a signing canister issues a `sender_info` canister signature, that signature remains valid indefinitely — across any ingress message to any canister — until the signing canister explicitly revokes it by mutating its certified state. There is no protocol-level mechanism for a user or the signing canister to time-bound or cancel a previously issued `sender_info` attestation.

---

### Finding Description

**Root cause — no expiry in the signed content:**

`SenderInfoContent` is defined as a newtype wrapper over raw bytes only: [1](#0-0) 

The `write_signed_bytes_without_domain_separator` implementation appends only the raw `info` bytes — no timestamp, no expiry, no message ID, no canister target: [2](#0-1) 

The `SignedSenderInfo` struct likewise carries no expiry field: [3](#0-2) 

**Validation path — no expiry check:**

`validate_sender_info` in the ingress validator only checks (1) that `sender_pubkey` is a valid canister signature public key, (2) that the canister ID in `sender_pubkey` matches the declared `signer`, and (3) that the canister signature over the `info` bytes is cryptographically valid: [4](#0-3) 

There is no check analogous to `validate_sender_delegation_expiry`, which explicitly rejects expired delegations: [5](#0-4) 

**Contrast with `Delegation`:**

`Delegation` carries an `expiration` field that is part of the signed bytes and is validated at every replica: [6](#0-5) 

`sender_info` has no equivalent.

**Reuse across messages:**

Because the `sender_info` canister signature is over only the `info` bytes (not bound to a specific `MessageId`, `ingress_expiry`, or target canister), a single valid `sender_info` signature can be attached to any number of ingress messages to any canister, for as long as the canister signature remains valid in the signing canister's certified state.

---

### Impact Explanation

**Impact: Medium**

A signing canister (e.g., Internet Identity) cannot issue time-limited `sender_info` attestations. Once issued, an attestation is valid indefinitely until the signing canister explicitly revokes it by mutating its certified state tree. Concrete consequences:

1. **Stale privilege escalation**: If a user's attested attributes change (e.g., an admin flag is revoked, a KYC status expires, or an account is suspended), the old `sender_info` signature remains usable by the user until the signing canister explicitly removes the corresponding entry from its state. Any canister that gates access on `msg_caller_info` data will continue to see the old (now-invalid) attributes.

2. **No short-lived attestations**: The signing canister cannot issue a `sender_info` that is valid only for a limited time window (e.g., "valid for the next 5 minutes"), because there is no expiry field in the signed content. The only revocation mechanism is a full state mutation, which is coarser-grained and requires the signing canister to track and actively expire all issued attestations.

3. **Cross-message reuse**: The same `sender_info` signature can be replayed across any ingress message to any canister, amplifying the window of exposure.

---

### Likelihood Explanation

**Likelihood: Medium**

The `sender_info` feature is production code, validated in the ingress path for both update calls and queries. Any canister that reads `msg_caller_info` and makes authorization decisions based on it is exposed. The attack requires only that a user retain a previously issued `sender_info` signature after their attributes have changed — a realistic scenario whenever attribute revocation is not instantaneous or the signing canister does not proactively purge old signatures.

---

### Recommendation

Add an `expiry` (or `expiration`) field to the `sender_info` signed content, analogous to the `expiration` field in `Delegation`. The replica's `validate_sender_info` should reject any `sender_info` whose expiry is in the past, mirroring `validate_sender_delegation_expiry`. This allows signing canisters to issue time-bounded attestations and gives users a protocol-level guarantee that old `sender_info` signatures cannot be replayed indefinitely.

---

### Proof of Concept

1. Signing canister C issues a `sender_info` canister signature over `info = [admin: true]` for user U, using seed S. The canister signature is valid because C's certified state contains `hash(info)` at path `[seed=S]`.

2. User U's admin privilege is revoked. C issues a new `sender_info` for U with `info = [admin: false]` using a different seed S'. The old signature (seed S) is **not** removed from C's state.

3. User U constructs an ingress message with `sender_info = {info: [admin: true], signer: C, sig: old_sig}`. The replica calls `validate_sender_info`, which verifies the canister signature is still valid (C's state still certifies it) and passes. The target canister reads `msg_caller_info` and sees `[admin: true]`.

4. There is no expiry check that would have rejected this replayed attestation. The only defense is for C to have explicitly removed the old entry from its state, which is not enforced by the protocol.

impl SignedBytesWithoutDomainSeparator for Delegation {
    fn write_signed_bytes_without_domain_separator(&self, bytes: &mut Vec<u8>) {
        use RawHttpRequestVal::*;

        let mut map = btreemap! {
            "pubkey" => Bytes(self.pubkey.0.as_slice()),
            "expiration" => U64(self.expiration.as_nanos_since_unix_epoch()),
        };
        if let Some(targets) = &self.targets {
            map.insert(
                "targets",
                Array(targets.iter().map(|t| Bytes(t.0.as_slice())).collect()),
            );
        }

        bytes.extend_from_slice(&hash_of_map(&map, |key, value| hash_key_val(key, value)));
    }
}

fn validate_sender_info<C: HttpRequestContent, R: RootOfTrustProvider>(
    request: &HttpRequest<C>,
    ingress_signature_verifier: &dyn IngressSigVerifier,
    root_of_trust_provider: &R,
) -> Result<(), RequestValidationError>
where
    R::Error: std::error::Error,
{
    let Some(sender_info) = request.sender_info() else {
        return Ok(());
    };

    // Per the spec, the sender_info signature must verify using the
    // envelope-level sender_pubkey as a canister signature public key.
    let sender_pubkey = match request.authentication() {
        Authentication::Authenticated(sig) => &sig.signer_pubkey,
        Authentication::Anonymous => {
            return Err(InvalidSenderInfo(
                "sender_info requires an authenticated request with sender_pubkey".to_string(),
            ));
        }
    };

    verify_sender_info_canister_sig(
        sender_info,
        sender_pubkey,
        ingress_signature_verifier,
        root_of_trust_provider,
    )
}

/// Verifies that the sender_info canister signature is valid:
/// 1. The envelope-level sender_pubkey is a valid canister signature public key.
/// 2. The canister ID encoded in sender_pubkey matches the signer field.
/// 3. The signature over the info blob is valid against the root of trust.
fn verify_sender_info_canister_sig<R: RootOfTrustProvider>(
    sender_info: &SignedSenderInfo,
    sender_pubkey_bytes: &[u8],
    validator: &dyn IngressSigVerifier,
    root_of_trust_provider: &R,
) -> Result<(), RequestValidationError>
where
    R::Error: std::error::Error,
{
    // Parse the envelope-level sender_pubkey DER to extract the raw
    // public key bytes and verify it's a valid canister signature public key.
    let pk_bytes = public_key_bytes_from_der(sender_pubkey_bytes).map_err(|e| {
        InvalidSenderInfo(format!(
            "sender_pubkey is not a valid canister signature public key: {e}"
        ))
    })?;

    // Extract the canister ID from the parsed public key and verify
    // it matches the declared signer.
    let parsed_pk = ic_crypto_iccsa::types::PublicKey::try_from(&pk_bytes)
        .map_err(|e| InvalidSenderInfo(format!("invalid canister sig public key: {e:?}")))?;
    let pubkey_canister_id = parsed_pk.signing_canister_id();
    if pubkey_canister_id != sender_info.signer {
        return Err(InvalidSenderInfo(format!(
            "signer {} does not match canister ID {} in sender_pubkey",
            sender_info.signer, pubkey_canister_id
        )));
    }

    // Construct the UserPublicKey for verification.
    let public_key = UserPublicKey {
        key: pk_bytes.0,
        algorithm_id: AlgorithmId::IcCanisterSignature,
    };

    // Construct the signable content (domain = "ic-sender-info")
    let sender_info_content = SenderInfoContent(&sender_info.info);
    let canister_sig = CanisterSigOf::from(CanisterSig(sender_info.sig.clone()));

    verify_canister_sig_with_fallback!(
        validator,
        &canister_sig,
        &sender_info_content,
        &public_key,
        root_of_trust_provider,
        |e| InvalidSenderInfo(format!("signature verification failed: {e}")),
        |e: <R as RootOfTrustProvider>::Error| InvalidSenderInfo(format!(
            "failed to get root of trust: {e}"
        ))
    );
    Ok(())
}

fn validate_sender_delegation_expiry(
    sender_delegation: &Option<Vec<SignedDelegation>>,
    current_time: Time,
) -> Result<(), RequestValidationError> {
    if let Some(delegations) = &sender_delegation {
        for delegation in delegations.iter() {
            let expiry = delegation.delegation().expiration();
            if delegation.delegation().expiration() < current_time {
                return Err(InvalidDelegationExpiry(format!(
                    "Specified sender delegation has expired:\n\
                     Provided expiry:    {expiry}\n\
                     Local replica time: {current_time}",
                )));
            }
        }
    }
    Ok(())
}
