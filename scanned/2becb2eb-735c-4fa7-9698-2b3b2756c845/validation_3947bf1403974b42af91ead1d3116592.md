### Title
Permanent Unvalidated-Pool Pollution via `start_block_height=u64::MAX` DKG Message — (`rs/artifact_pool/src/dkg_pool.rs`)

---

### Summary

Two independent code properties compose into a permanent memory leak. A Byzantine subnet peer can insert a `dkg::Message` whose `content.dkg_id.start_block_height` is `u64::MAX`. The validator silently skips (never invalidates) the message because its height never matches the current DKG interval. The purge filter `id.height < height` can never be satisfied for `u64::MAX` at any finite purge height. The underlying `PoolSection` is an unbounded `BTreeMap`. The message occupies the unvalidated pool forever, and because each message has a unique crypto-hash key, a Byzantine node can accumulate arbitrarily many such entries.

---

### Finding Description

**Bug 1 — Purge filter never fires for `height = u64::MAX`**

`DkgMessageId::from` copies `msg.content.dkg_id.start_block_height` directly into `DkgMessageId.height` with no bounds check: [1](#0-0) 

`DkgPoolImpl::purge` removes entries only where `id.height < height`: [2](#0-1) [3](#0-2) 

For `id.height = u64::MAX`, the predicate `u64::MAX < height` is false for every representable `height`. The message is never selected for removal.

**Bug 2 — Validator skips (does not invalidate) wrong-height messages**

`validate_dealings_for_dealer` returns `Mutations::new()` (empty — no action) when the message's `start_block_height` does not match the current DKG interval: [4](#0-3) 

`Mutations::new()` means the message is left in the unvalidated pool. It is not moved to validated, not removed, not flagged invalid. Every subsequent call to `on_state_change` repeats the same skip.

**Bug 3 — No size limit on the pool section**

`PoolSection` is a plain `BTreeMap` with no capacity cap: [5](#0-4) 

**Bug 4 — Insert performs no height validation**

`DkgPoolImpl::insert` accepts any `UnvalidatedArtifact` and stores it immediately: [6](#0-5) 

---

### Impact Explanation

Each crafted message occupies a unique slot in the unvalidated `BTreeMap` (keyed by `CryptoHashOf<Message>` + `Height`). A Byzantine node can send an unbounded stream of distinct messages (varying dealing bytes or signer identity) all with `start_block_height = u64::MAX`. None are ever purged; none are ever invalidated. Honest nodes accumulate these entries across all DKG intervals, leading to unbounded heap growth and eventual OOM / replica crash — a subnet-level denial of service.

---

### Likelihood Explanation

The attacker is a Byzantine subnet node (P2P peer below the consensus fault threshold), which is an explicitly in-scope threat actor. No privileged key, governance vote, or external infrastructure is required. The crafted message requires only a valid protobuf structure with an arbitrary `start_block_height` field set to `u64::MAX`; no valid cryptographic dealing or signature is needed to reach the unvalidated pool. The path is entirely local-testable.

---

### Recommendation

Apply **both** fixes:

1. **Purge**: Change the filter to `id.height <= height` (strict less-than misses the boundary) **or** use `!=` semantics: remove any entry whose height does not equal the current start height. This ensures `u64::MAX`-height entries are evicted at the next purge cycle.

2. **Validator**: When `start_block_height` does not match the current interval **and** is not a plausible future interval (e.g., exceeds finalized height by more than one interval length), emit `HandleInvalid` rather than `Mutations::new()`.

3. **Admission**: Add a pool-size cap or a per-peer rate limit at the P2P ingress layer for DKG artifacts.

---

### Proof of Concept

```rust
// 1. Craft a message with start_block_height = u64::MAX
let dkg_id = NiDkgId {
    start_block_height: Height::from(u64::MAX),
    dealer_subnet: subnet_test_id(100),
    dkg_tag: NiDkgTag::HighThreshold,
    target_subnet: NiDkgTargetSubnet::Local,
};
let msg = dkg::Message {
    content: dkg::DealingContent::new(dummy_dealing(0), dkg_id),
    signature: BasicSignature::fake(node_test_id(1)),
};

// 2. Insert into pool (no height validation at insert)
pool.insert(UnvalidatedArtifact { message: msg, peer_id: node_test_id(1), timestamp: UNIX_EPOCH });
assert_eq!(pool.get_unvalidated().count(), 1);

// 3. Purge at u64::MAX - 1: message survives
pool.apply(vec![ChangeAction::Purge(Height::from(u64::MAX - 1))]);
assert_eq!(pool.get_unvalidated().count(), 1); // still present

// 4. Purge at u64::MAX: message STILL survives (u64::MAX < u64::MAX == false)
pool.apply(vec![ChangeAction::Purge(Height::from(u64::MAX))]);
assert_eq!(pool.get_unvalidated().count(), 1); // never evicted
```

impl<K: Ord, V: HasLabel> PoolSection<K, V> {
    pub(crate) fn new(metrics_registry: MetricsRegistry, pool: &str, pool_type: &str) -> Self {
        Self {
            messages: Default::default(),
            metrics: PoolMetrics::new(metrics_registry, pool, pool_type),
        }
    }

    pub(crate) fn insert(&mut self, key: K, value: V) -> Option<V> {
        self.metrics
            .observe_insert(MESSAGE_SIZE_BYTES, value.label());
        let replaced = self.messages.insert(key, value);
        if let Some(replaced) = &replaced {
            self.metrics
                .observe_duplicate(MESSAGE_SIZE_BYTES, replaced.label());
        }
        replaced
    }
