### Title
SNS Governance `disburse_maturity` Defaults Receiver to Caller, Enabling Authorized Delegates to Redirect Maturity to Themselves - (`rs/sns/governance/src/governance.rs`)

---

### Summary

In SNS Governance, `disburse_maturity` accepts calls from any principal holding `NeuronPermissionType::DisburseMaturity` permission — not just the neuron controller. When the optional `to_account` field is omitted, the maturity is sent to the **caller's** account rather than the neuron controller's account. A malicious or compromised delegate who holds this permission can drain a neuron's entire maturity to their own ledger account by simply omitting `to_account`.

---

### Finding Description

`Governance::disburse_maturity` in SNS governance performs two distinct steps:

**Step 1 — Authorization check** (`rs/sns/governance/src/governance.rs:1615-1616`):

```rust
let neuron = self.get_neuron_result(id)?;
neuron.check_authorized(caller, NeuronPermissionType::DisburseMaturity)?;
```

`check_authorized` resolves to `is_authorized`, which only checks whether the caller's principal appears in the neuron's `permissions` list with the `DisburseMaturity` bit set. It does **not** require the caller to be the neuron controller. [1](#0-0) 

**Step 2 — Default receiver resolution** (`rs/sns/governance/src/governance.rs:1618-1623`):

```rust
// If no account was provided, transfer to the caller's account.
let to_account: Account = match disburse_maturity.to_account.as_ref() {
    None => Account {
        owner: caller.0,
        subaccount: None,
    },
```

When `to_account` is `None`, the destination is set to `caller.0` — the **caller's** principal — not the neuron controller's principal. [2](#0-1) 

Any principal can be granted `DisburseMaturity` permission via `AddNeuronPermissions`. A neuron owner might legitimately grant this to a service canister or co-manager to automate disbursements. That delegate can then call `manage_neuron` with `DisburseMaturity { percentage_to_disburse: 100, to_account: None }` and receive all maturity at their own account. [3](#0-2) 

**Contrast with NNS Governance**, which correctly restricts `DisburseMaturity` to the neuron controller only:

```rust
if !is_neuron_controlled_by_caller {
    return Err(InitiateMaturityDisbursementError::CallerIsNotNeuronController);
}
``` [4](#0-3) 

NNS governance also defaults to the caller's account when no destination is specified, but since only the controller can call it, the caller and the neuron owner are always the same principal. [5](#0-4) 

SNS governance has no such restriction. The `DisburseMaturity` proto message only carries `percentage_to_disburse` and an optional `to_account`: [6](#0-5) 

---

### Impact Explanation

A principal with `DisburseMaturity` permission on an SNS neuron — but who is not the controller — can call `manage_neuron` with `DisburseMaturity { percentage_to_disburse: 100, to_account: None }`. The SNS Governance canister will mint and transfer up to 100% of the neuron's accumulated maturity to the caller's own ledger account. The neuron controller loses all maturity with no recourse, since the disbursement is irreversible once finalized after the 7-day delay window. [7](#0-6) 

---

### Likelihood Explanation

The attack requires the neuron owner to have previously granted `DisburseMaturity` permission to the attacker's principal. This is a realistic scenario: neuron owners commonly delegate operational permissions to service canisters, bots, or co-managers. The SNS permission model explicitly supports this pattern. The `NeuronPermissionType::DisburseMaturity` permission is listed as a grantable permission type, and the `AddNeuronPermissions` command is available to any principal with `ManagePrincipals` permission. A neuron owner who grants `DisburseMaturity` to a third party expecting it to only trigger disbursements to the owner's account is exposed to this theft. [8](#0-7) 

---

### Recommendation

When `to_account` is `None` in SNS `disburse_maturity`, the default destination should be the **neuron controller's** account, not the caller's account. The controller's principal is available on the neuron object. Alternatively, restrict `disburse_maturity` to the neuron controller only (as NNS governance does), or require that non-controller callers always supply an explicit `to_account`.

```rust
// Proposed fix: default to controller's account, not caller's account
None => Account {
    owner: neuron.controller().0,
    subaccount: None,
},
``` [9](#0-8) 

---

### Proof of Concept

1. Alice owns an SNS neuron with 1,000 SNS tokens of accumulated maturity.
2. Alice grants `DisburseMaturity` permission to Bob's principal via `AddNeuronPermissions`.
3. Bob (attacker) sends an ingress `manage_neuron` call to the SNS Governance canister:
   ```
   ManageNeuron {
     subaccount: <alice_neuron_subaccount>,
     command: DisburseMaturity {
       percentage_to_disburse: 100,
       to_account: None,   // defaults to Bob's account
     }
   }
   ```
4. `check_authorized` passes because Bob holds `DisburseMaturity` permission.
5. `to_account` resolves to `Account { owner: Bob.principal, subaccount: None }`.
6. After the 7-day delay, the SNS ledger mints and transfers all of Alice's maturity to Bob's account.
7. Alice's neuron maturity is zero; Bob has received all the tokens. [10](#0-9) [3](#0-2)

    /// Checks whether a given principal has the permission to perform a certain action on
    /// the neuron.
    pub(crate) fn check_authorized(
        &self,
        principal: &PrincipalId,
        permission: NeuronPermissionType,
    ) -> Result<(), GovernanceError> {
        if !self.is_authorized(principal, permission) {
            return Err(GovernanceError::new_with_message(
                ErrorType::NotAuthorized,
                format!(
                    "Caller '{:?}' is not authorized to perform action: '{:?}' on neuron '{}'.",
                    principal,
                    permission,
                    self.id.as_ref().expect("Neuron must have a NeuronId"),
                ),
            ));
        }

        Ok(())
    }

    /// Returns true if the principalId has the permission to act on this neuron (i.e., self).
    pub(crate) fn is_authorized(
        &self,
        principal: &PrincipalId,
        permission: NeuronPermissionType,
    ) -> bool {
        let found_neuron_permission = self
            .permissions
            .iter()
            .find(|neuron_permission| neuron_permission.principal == Some(*principal));

        if let Some(p) = found_neuron_permission {
            return p.permission_type.contains(&(permission as i32));
        }

        false
    }
