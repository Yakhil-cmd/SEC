### Title
Vesting SNS Neurons Can Disburse Accumulated Maturity (Voting Rewards) to Liquid Accounts While Still Vesting - (`rs/sns/governance/src/governance.rs`)

### Summary
The `check_command_is_valid_if_neuron_is_vesting` function in SNS governance explicitly permits `DisburseMaturity` for vesting neurons. This allows a developer or team member whose neuron is still within its vesting period to extract all accumulated voting-reward maturity as liquid tokens, directly undermining the commitment guarantee that vesting is designed to enforce.

### Finding Description
SNS vesting neurons are designed to prove long-term commitment: they cannot dissolve, cannot start dissolving, and cannot disburse their stake until the vesting period elapses. The gating function `check_command_is_valid_if_neuron_is_vesting` enforces this by blocking `Disburse` and `Split` for vesting neurons.

However, `DisburseMaturity` — which sends accumulated voting-reward maturity to any arbitrary liquid ledger account — is explicitly placed in the **allowed** arm of the match:

```rust
Follow(_)
| SetFollowing(_)
| MakeProposal(_)
| RegisterVote(_)
| ClaimOrRefresh(_)
| MergeMaturity(_)
| DisburseMaturity(_)   // ← allowed for vesting neurons
| AddNeuronPermissions(_)
| RemoveNeuronPermissions(_)
| StakeMaturity(_) => Ok(()),
``` [1](#0-0) 

The `disburse_maturity` function itself performs no independent vesting check; it only verifies the caller holds `NeuronPermissionType::DisburseMaturity` permission. [2](#0-1) 

After the call, maturity is deducted from the neuron immediately and queued for transfer to the specified account after a 7-day delay (`MATURITY_DISBURSEMENT_DELAY_SECONDS`). [3](#0-2) 

The `is_vesting` predicate confirms a neuron is vesting when `created_timestamp_seconds + vesting_period_seconds >= now`, meaning the neuron is still within its declared commitment window. [4](#0-3) 

The vesting period documentation explicitly states its purpose is to allow developers to "prove their long-term commitment to the community": [5](#0-4) 

### Impact Explanation
A developer neuron created at SNS genesis with a 3-year vesting period and a 1-year dissolve delay accumulates maturity (voting rewards) throughout the vesting window. By calling `manage_neuron` with `DisburseMaturity`, the developer can continuously extract that maturity as liquid SNS tokens — selling them on the open market — while the neuron still appears fully committed (non-dissolving, vesting). Other token holders who do not or cannot perform this operation are disadvantaged: their maturity remains locked in their neurons. The commitment signal that vesting is supposed to provide to the community is hollow.

**Vulnerability class**: Governance authorization bug — reward accounting bypass via missing vesting guard on `DisburseMaturity`.

### Likelihood Explanation
This is reachable by any unprivileged ingress sender who controls a vesting SNS neuron with `DisburseMaturity` permission. Developer neurons at SNS genesis are the primary affected population. The call path is a standard `manage_neuron` update call — no special privileges, no threshold corruption, no social engineering required. The 7-day disbursement delay does not prevent the attack; it only delays receipt.

### Recommendation
Add `DisburseMaturity` to the blocked commands for vesting neurons inside `check_command_is_valid_if_neuron_is_vesting`, consistent with how `Disburse` and `Split` are already blocked:

```rust
Disburse(_) => err("Disburse"),
Split(_) => err("Split"),
DisburseMaturity(_) => err("DisburseMaturity"),  // add this
``` [6](#0-5) 

`MergeMaturity` and `StakeMaturity` can remain allowed because they keep the maturity locked inside the neuron (merged into stake or staked maturity), which is consistent with the vesting commitment.

### Proof of Concept
1. At SNS genesis, developer neuron `N` is created with `vesting_period_seconds = 3 years`, `dissolve_delay_seconds = 1 year`, and `maturity_e8s_equivalent = 0`.
2. Over time, `N` votes on proposals and accumulates maturity, e.g., `maturity_e8s_equivalent = 10_000_000_000` (100 SNS tokens equivalent).
3. `N.is_vesting(now)` returns `true` — the neuron is still within its vesting window.
4. The developer calls:
   ```
   manage_neuron({
     subaccount: N.subaccount,
     command: DisburseMaturity({
       percentage_to_disburse: 100,
       to_account: developer_liquid_account,
     })
   })
   ```
5. `check_command_is_valid_if_neuron_is_vesting` is called, reaches the `DisburseMaturity(_) => Ok(())` arm, and returns no error.
6. `disburse_maturity` deducts `10_000_000_000` e8s from `N.maturity_e8s_equivalent` and enqueues a disbursement to `developer_liquid_account`.
7. After 7 days, `maybe_finalize_disburse_maturity` mints the modulated amount to `developer_liquid_account`.
8. The developer now holds liquid SNS tokens while the neuron remains non-dissolving and vesting — the community sees a "committed" neuron, but the developer has already extracted all voting rewards. [7](#0-6) [8](#0-7)

    /// Returns an error if the given neuron is vesting and the given command cannot be called by
    /// a vesting neuron
    fn check_command_is_valid_if_neuron_is_vesting(
        &self,
        neuron_id: &NeuronId,
        command: &manage_neuron::Command,
    ) -> Result<(), GovernanceError> {
        use manage_neuron::{Command::*, configure::Operation::*};

        // If this is a "claim" call, the neuron doesn't exist yet, so we return (because no checks
        // can be made). A "refresh" call can be made on a vesting neuron, so in this case also
        // results in returning Ok.
        if let ClaimOrRefresh(_) = command {
            return Ok(());
        }

        let neuron = self.get_neuron_result(neuron_id)?;

        if !neuron.is_vesting(self.env.now()) {
            return Ok(());
        }

        let err = |op: &str| -> Result<(), GovernanceError> {
            Err(GovernanceError::new_with_message(
                ErrorType::PreconditionFailed,
                format!("Neuron {neuron_id} is vesting and cannot call {op}"),
            ))
        };

        match command {
            Configure(configure) => match configure.operation {
                Some(IncreaseDissolveDelay(_)) => err("IncreaseDissolveDelay"),
                Some(StartDissolving(_)) => err("StartDissolving"),
                Some(StopDissolving(_)) => err("StopDissolving"),
                Some(SetDissolveTimestamp(_)) => err("SetDissolveTimestamp"),
                Some(ChangeAutoStakeMaturity(_)) => Ok(()),
                None => Ok(()),
            },
            Disburse(_) => err("Disburse"),
            Split(_) => err("Split"),
            Follow(_)
            | SetFollowing(_)
            | MakeProposal(_)
            | RegisterVote(_)
            | ClaimOrRefresh(_)
            | MergeMaturity(_)
            | DisburseMaturity(_)
            | AddNeuronPermissions(_)
            | RemoveNeuronPermissions(_)
            | StakeMaturity(_) => Ok(()),
        }
    }
