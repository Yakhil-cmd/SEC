### Title
Missing Lower Bound on `max_dissolve_delay_seconds` Allows SNS Governance Parameter Mispricing — (`File: rs/sns/governance/src/types.rs`, `rs/sns/init/src/lib.rs`)

---

### Summary

The `max_dissolve_delay_seconds` parameter in SNS `NervousSystemParameters` is validated only for presence (not `None`), but never for a meaningful lower bound. An unprivileged SNS creator can set `max_dissolve_delay_seconds = 0` (and correspondingly `neuron_minimum_dissolve_delay_to_vote_seconds = 0`), which passes all validation gates and causes the SNS to launch with its dissolve-delay governance incentive completely disabled and with dissolved neurons eligible to vote.

---

### Finding Description

Both the `SnsInitPayload` pre/post-execution validation path and the runtime `NervousSystemParameters::validate()` path check `max_dissolve_delay_seconds` only for presence:

**`rs/sns/governance/src/types.rs` (runtime validation):**
```rust
fn validate_max_dissolve_delay_seconds(&self) -> Result<u64, String> {
    self.max_dissolve_delay_seconds.ok_or_else(|| {
        "NervousSystemParameters.max_dissolve_delay_seconds must be set".to_string()
    })
}
``` [1](#0-0) 

**`rs/sns/init/src/lib.rs` (init-payload validation):**
```rust
fn validate_max_dissolve_delay_seconds(&self) -> Result<(), String> {
    let _max_dissolve_delay_seconds = self
        .max_dissolve_delay_seconds
        .ok_or("Error: max_dissolve_delay_seconds must be specified")?;
    Ok(())
}
``` [2](#0-1) 

The downstream `validate_neuron_minimum_dissolve_delay_to_vote_seconds` only enforces an **upper** bound (≤ `max_dissolve_delay_seconds`), not a lower bound:

```rust
if neuron_minimum_dissolve_delay_to_vote_seconds > max_dissolve_delay_seconds {
    Err(...)
} else {
    Ok(())
}
``` [3](#0-2) 

This mirrors the external report exactly: `maxTick` had only an upper-bound check; here `max_dissolve_delay_seconds` has only a presence check with no lower bound.

The same pattern applies to `validate_max_neuron_age_for_age_bonus`, which also only checks presence: [4](#0-3) 

---

### Impact Explanation

When `max_dissolve_delay_seconds = 0` and `neuron_minimum_dissolve_delay_to_vote_seconds = 0` are submitted together:

1. **All neurons (including fully dissolved ones) become eligible to vote**, because the minimum dissolve delay to vote is 0. The eligibility check in `compute_ballots_for_new_proposal` is `dissolve_delay_seconds(now) < min_dissolve_delay_for_vote`, which is never true when `min_dissolve_delay_for_vote = 0`. [5](#0-4) 

2. **The dissolve-delay voting power bonus is silently zeroed out** for every neuron. The SNS `voting_power` function guards against divide-by-zero with `if max_dissolve_delay_seconds > 0`, but this means the bonus is simply `0` — the entire incentive mechanism is disabled. [6](#0-5) 

3. **The SNS governance security model is undermined**: the design intent is that long-term token holders (with high dissolve delay) have proportionally more voting power. Setting `max_dissolve_delay_seconds = 0` collapses all voting power to raw stake, and allows short-term or dissolved token holders to control governance equally.

---

### Likelihood Explanation

The entry point is the `CreateServiceNervousSystem` NNS proposal, which any NNS neuron holder with sufficient dissolve delay can submit. The `SnsInitPayload::validate_pre_execution()` and `validate_post_execution()` both call `validate_max_dissolve_delay_seconds()` which only checks presence. [7](#0-6) 

The `validate_neuron_minimum_dissolve_delay_to_vote_seconds` in `SnsInitPayload` uses the **default** `max_dissolve_delay_seconds` (not `self`'s value), so it does not catch the inconsistency: [8](#0-7) 

The runtime `NervousSystemParameters::validate()` also passes because `neuron_minimum_dissolve_delay_to_vote_seconds (0) <= max_dissolve_delay_seconds (0)` is satisfied. No privileged access is required; any NNS neuron holder can submit this proposal.

---

### Recommendation

- **Short term:** Add a lower bound check on `max_dissolve_delay_seconds` in both `SnsInitPayload` and `NervousSystemParameters` validation. A sensible minimum (e.g., one week or one month) should be enforced to prevent the dissolve-delay incentive from being disabled. Similarly add a lower bound to `max_neuron_age_for_age_bonus`.
- **Long term:** Cross-validate `neuron_minimum_dissolve_delay_to_vote_seconds` against the **submitted** `max_dissolve_delay_seconds` (not the default) in `SnsInitPayload::validate_neuron_minimum_dissolve_delay_to_vote_seconds`, and enforce a non-zero minimum for `neuron_minimum_dissolve_delay_to_vote_seconds` to ensure dissolved neurons cannot vote.

---

### Proof of Concept

An attacker submits a `CreateServiceNervousSystem` NNS proposal with:
```
max_dissolve_delay_seconds = 0
neuron_minimum_dissolve_delay_to_vote_seconds = 0
```

**Validation trace:**
1. `SnsInitPayload::validate_max_dissolve_delay_seconds()` → `Some(0)` is present → **passes**
2. `SnsInitPayload::validate_neuron_minimum_dissolve_delay_to_vote_seconds()` → compares `0` against the **default** max (e.g., 252460800) → `0 <= 252460800` → **passes**
3. `NervousSystemParameters::validate_max_dissolve_delay_seconds()` → `Some(0)` is present → **passes**
4. `NervousSystemParameters::validate_neuron_minimum_dissolve_delay_to_vote_seconds()` → `0 <= 0` → **passes**

**Runtime effect:**
- `compute_ballots_for_new_proposal`: `min_dissolve_delay_for_vote = 0`, so every neuron (including dissolved ones with `dissolve_delay_seconds = 0`) receives a ballot.
- `voting_power(...)`: `max_dissolve_delay_seconds = 0` → dissolve delay bonus branch is skipped → all neurons receive only base stake as voting power, with no long-term commitment incentive.

    fn validate_neuron_minimum_dissolve_delay_to_vote_seconds(&self) -> Result<(), String> {
        // As this is not currently configurable, pull the default value from
        let max_dissolve_delay_seconds = *NervousSystemParameters::with_default_values()
            .max_dissolve_delay_seconds
            .as_ref()
            .unwrap();

        let neuron_minimum_dissolve_delay_to_vote_seconds = self
            .neuron_minimum_dissolve_delay_to_vote_seconds
            .ok_or_else(|| {
                "Error: neuron-minimum-dissolve-delay-to-vote-seconds must be specified".to_string()
            })?;

        if neuron_minimum_dissolve_delay_to_vote_seconds > max_dissolve_delay_seconds {
            return Err(format!(
                "The minimum dissolve delay to vote ({neuron_minimum_dissolve_delay_to_vote_seconds}) cannot be greater than the max \
                dissolve delay ({max_dissolve_delay_seconds})"
            ));
        }

        Ok(())
    }
