### Title
SNS Swap Canister Has No Recovery Mechanism for SNS Tokens Locked After Aborted Swap - (File: `rs/sns/swap/src/swap.rs`)

### Summary
The SNS Swap canister provides `error_refund_icp` to recover accidentally sent ICP (the non-primary token), but has no equivalent mechanism for SNS tokens (the primary token). When a swap is aborted, the SNS tokens sent to the swap canister by the SNS governance canister are permanently locked with no built-in recovery path — directly analogous to the NounsDAO stream contract where `rescueERC20` blocks the stream token, leaving excess stream tokens irrecoverable without canceling the stream.

### Finding Description

The SNS Swap canister lifecycle is: `PENDING → ADOPTED → OPEN → COMMITTED or ABORTED`. During the PENDING/ADOPTED phase, the SNS governance canister transfers `sns_token_e8s` SNS tokens to the swap canister's default account. The proto invariant documents this explicitly:

```
// Invariant for the OPEN state:
// state.sns_token_e8s <= token_ledger.balance_of(<swap-canister>)
``` [1](#0-0) 

When the swap is aborted (e.g., minimum ICP participation not reached), `finalize_inner` calls `sweep_icp` to return ICP to buyers, then hits `should_restore_dapp_control()` which returns `true` for aborted swaps and causes an **early return** — `sweep_sns` is never called: [2](#0-1) 

`sweep_sns` itself enforces that it only runs in the `COMMITTED` state, making it impossible to use for token recovery in the `ABORTED` state: [3](#0-2) 

The `error_refund_icp` function — the canister's only explicit "rescue" mechanism — is hardcoded to the ICP ledger and operates only on buyer subaccounts, not the swap canister's default account where SNS tokens are held: [4](#0-3) [5](#0-4) 

There is no `error_refund_sns` or equivalent function in the canister's public API. Integration tests confirm that after an aborted swap, the full SNS token balance remains stranded in the swap canister: [6](#0-5) 

Additionally, any user can permissionlessly send SNS tokens to the swap canister's default account on the SNS ledger at any time (before, during, or after the swap). Those tokens are equally irrecoverable.

### Impact Explanation

SNS tokens sent to the swap canister — whether as part of the normal governance-initiated swap flow or accidentally by any user — are permanently locked when the swap is aborted. Every SNS project with a failed decentralization swap loses its entire `sns_token_e8s` allocation with no built-in recovery. The only escape is an NNS governance upgrade proposal to add a recovery function, which is a drastic, slow, and governance-dependent action analogous to "canceling the stream" in the NounsDAO report.

### Likelihood Explanation

Aborted swaps are a routine, expected outcome of the SNS lifecycle (minimum ICP participation not reached, swap due date exceeded without sufficient participants, etc.). The condition is not adversarial — it is a normal protocol path. Any SNS project that fails its decentralization sale triggers this loss. Additionally, any unprivileged user can send SNS tokens to the swap canister's default account at any time, permanently locking them.

### Recommendation

Add a `return_sns_tokens` (or `error_refund_sns`) function callable by the SNS governance canister (or the fallback controllers) after the swap reaches a terminal state (`ABORTED` or `COMMITTED`). The function should transfer the remaining SNS token balance from the swap canister's default account back to the SNS governance treasury. Analogous to the NounsDAO recommendation:

```rust
pub async fn return_sns_tokens_to_governance(
    &self,
    self_canister_id: CanisterId,
    sns_ledger: &dyn ICRC1Ledger,
) -> Result<u64, String> {
    // Only callable after swap is terminal
    if self.lifecycle() != Lifecycle::Aborted && self.lifecycle() != Lifecycle::Committed {
        return Err("Can only return SNS tokens after swap is terminal".to_string());
    }
    let balance = sns_ledger.account_balance(Account {
        owner: self_canister_id.into(),
        subaccount: None,
    }).await?;
    let sns_governance = self.init_or_panic().sns_governance_or_panic();
    // Transfer balance minus fee back to SNS governance treasury
    sns_ledger.transfer_funds(
        balance.get_e8s().saturating_sub(transaction_fee),
        transaction_fee,
        None,
        Account { owner: sns_governance.get().0, subaccount: None },
        0,
    ).await
}
```

### Proof of Concept

1. SNS governance canister sends `N` SNS tokens to the swap canister's default account (normal swap initialization).
2. The swap opens and runs until the due date, but minimum ICP participation is not reached.
3. `try_abort` transitions the swap to `ABORTED`.
4. `finalize_swap` → `finalize_inner` is called:
   - `sweep_icp` returns ICP to buyers ✓
   - `should_restore_dapp_control()` returns `true` → early return ✓
   - `sweep_sns` is **never called** ✗
5. The swap canister's SNS token balance remains at `N` tokens.
6. `error_refund_icp` cannot recover SNS tokens (wrong ledger, wrong account type).
7. `sweep_sns` rejects calls in `ABORTED` state with a global failure.
8. The `N` SNS tokens are permanently locked in the swap canister with no built-in recovery path. [7](#0-6) [8](#0-7)

    pub async fn finalize_inner(
        &mut self,
        now_fn: fn(bool) -> u64,
        environment: &mut impl CanisterEnvironment,
    ) -> FinalizeSwapResponse {
        let mut finalize_swap_response = FinalizeSwapResponse::default();

        if let Err(e) = self.can_finalize() {
            finalize_swap_response.set_error_message(e);
            return finalize_swap_response;
        }

        // Transfer the ICP tokens from the Swap canister.
        finalize_swap_response
            .set_sweep_icp_result(self.sweep_icp(now_fn, environment.icp_ledger()).await);
        if finalize_swap_response.has_error_message() {
            return finalize_swap_response;
        }

        // Settle the Neurons' Fund participation in the token swap.
        finalize_swap_response.set_settle_neurons_fund_participation_result(
            self.settle_neurons_fund_participation(environment.nns_governance_mut())
                .await,
        );
        if finalize_swap_response.has_error_message() {
            return finalize_swap_response;
        }

        if self.should_restore_dapp_control() {
            // Restore controllers of dapp canisters to their original
            // owners (i.e. self.init.fallback_controller_principal_ids).
            finalize_swap_response.set_set_dapp_controllers_result(
                self.restore_dapp_controllers_for_finalize(environment.sns_root_mut())
                    .await,
            );

            // In the case of returning control of the dapp(s) to the fallback
            // controllers, finalize() need not do any more work, so always return
            // and end execution.
            return finalize_swap_response;
        }

        // Create the SnsNeuronRecipes based on the contribution of direct and NF participants
        finalize_swap_response
            .set_create_sns_neuron_recipes_result(self.create_sns_neuron_recipes());
        if finalize_swap_response.has_error_message() {
            return finalize_swap_response;
        }

        // Transfer the SNS tokens from the Swap canister.
        finalize_swap_response
            .set_sweep_sns_result(self.sweep_sns(now_fn, environment.sns_ledger()).await);
        if finalize_swap_response.has_error_message() {
            return finalize_swap_response;
        }

        // Once SNS tokens have been distributed to the correct accounts, claim
        // them as neurons on behalf of the Swap participants.
        finalize_swap_response.set_claim_neuron_result(
            self.claim_swap_neurons(environment.sns_governance_mut())
                .await,
        );
        if finalize_swap_response.has_error_message() {
            return finalize_swap_response;
        }

        finalize_swap_response.set_set_mode_call_result(
            Self::set_sns_governance_to_normal_mode(environment.sns_governance_mut()).await,
        );

        // The following step is non-critical, so we'll do it after we set
        // governance to normal mode, but only if there were no errors.
        if !finalize_swap_response.has_error_message() {
            finalize_swap_response.set_set_dapp_controllers_result(
                self.take_sole_control_of_dapp_controllers_for_finalize(environment.sns_root_mut())
                    .await,
            );
        }

        finalize_swap_response
    }
