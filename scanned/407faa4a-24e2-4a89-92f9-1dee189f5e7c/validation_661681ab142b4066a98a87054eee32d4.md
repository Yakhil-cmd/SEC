### Title
Missing `contextual_check_header_auxpow` call in legacy (non-AuxPow) path of `validate_auxpow_header` allows version-obsolete and post-fork legacy blocks to bypass enforcement — (`rs/bitcoin/validation/src/doge.rs`)

### Summary
`validate_auxpow_header` has two branches. The AuxPow branch correctly calls `contextual_check_header_auxpow`, which enforces both the version-obsolescence rule and the post-fork legacy-block prohibition. The legacy branch (`aux_pow == None`) calls only `validate_header`, which performs PoW/difficulty/timestamp/checkpoint checks but has no version check. Any legacy `DogecoinHeader` with `version < 3` at `height >= bip66_height` (or `version < 4` at `height >= bip65_height`) passes the legacy branch without triggering `VersionObsolete`.

### Finding Description
In `rs/bitcoin/validation/src/doge.rs`, `validate_auxpow_header` (lines 366–409):

```
if let Some(aux_pow) = header.aux_pow.as_ref() {
    ...
    self.contextual_check_header_auxpow(...)  // ← version check present
    ...
} else {
    // header.aux_pow == None
    self.validate_header(...)                 // ← version check ABSENT
}
```

`contextual_check_header_auxpow` (lines 342–362) is the sole location of:
1. The `VersionObsolete` guard (lines 355–359)
2. The `LegacyBlockNotAllowed` guard (lines 347–349)

Neither guard is reachable from the `else` branch. An attacker who submits a `DogecoinHeader` with `aux_pow: None`, `has_auxpow_bit() == false`, `version == 1`, and a valid Scrypt PoW at `height >= bip66_height` will have the header accepted by the adapter.

### Impact Explanation
The Dogecoin adapter is the chain-fusion component that tracks the Dogecoin chain for ckDOGE. Accepting a version-obsolete or post-fork legacy block causes the adapter's internal chain to diverge from the canonical Dogecoin chain. Downstream effects include incorrect ckDOGE deposit/withdrawal accounting and potential double-spend or loss of funds in the chain-fusion bridge.

### Likelihood Explanation
Exploitation requires the attacker to produce a `DogecoinHeader` that:
1. Has a valid `prev_blockhash` pointing to a block already in the adapter's store.
2. Carries the correct difficulty target for that height.
3. Satisfies Scrypt PoW at that target.

This is computationally expensive (requires mining-level resources) but not cryptographically impossible, especially for a well-resourced adversary or a mining pool. The bug is also trivially unit-testable without any PoW: a mock store can be constructed to return a known prev-header, and the test can assert that `validate_auxpow_header` with `version=1` at `height >= bip66_height` returns `VersionObsolete` — it currently does not.

### Recommendation
In the `else` branch of `validate_auxpow_header`, after `validate_header` succeeds, call `contextual_check_header_auxpow` with the height returned from `contextual_check_header`. Concretely:

```rust
} else {
    if header.has_auxpow_bit() {
        return Err(ValidateAuxPowHeaderError::InconsistentAuxPowBitSet);
    }
    let (_, height) = self.contextual_check_header(store, &header.pure_header)?;
    self.contextual_check_header_auxpow(&header.pure_header, height)?;
    self.validate_header(store, &header.pure_header)?;
}
```

Note that `contextual_check_header` is already called inside `validate_header` (line 162), so this introduces a second call; alternatively, refactor `validate_header` to accept a pre-computed height, or extract the height from the store directly.

    fn validate_auxpow_header(
        &self,
        store: &impl HeaderStore,
        header: &DogecoinHeader,
    ) -> Result<(), ValidateAuxPowHeaderError> {
        if !header.is_legacy()
            && self.strict_chain_id()
            && header.extract_chain_id() != self.auxpow_chain_id()
        {
            return Err(ValidateAuxPowHeaderError::InvalidChainId);
        }

        if let Some(aux_pow) = header.aux_pow.as_ref() {
            if !header.has_auxpow_bit() {
                return Err(ValidateAuxPowHeaderError::InconsistentAuxPowBitSet);
            }

            let (target, height) = self.contextual_check_header(store, &header.pure_header)?;

            self.contextual_check_header_auxpow(&header.pure_header, height)?;

            if !target.is_met_by(aux_pow.parent_block_header.block_hash_with_scrypt()) {
                return Err(ValidateAuxPowHeaderError::InvalidParentPoW);
            }
            if aux_pow
                .check(
                    header.block_hash(),
                    header.extract_chain_id(),
                    self.strict_chain_id(),
                )
                .is_err()
            {
                return Err(ValidateAuxPowHeaderError::InvalidAuxPoW);
            }
        } else {
            if header.has_auxpow_bit() {
                return Err(ValidateAuxPowHeaderError::InconsistentAuxPowBitSet);
            }

            self.validate_header(store, &header.pure_header)?;
        }

        Ok(())
    }
