### Title
Unbounded `sender_info.info` in `EntryKey` Enables Cache-Eviction DoS on Replica Query Cache — (`rs/execution_environment/src/query_handler/query_cache.rs`)

---

### Summary

`validate_sender_info` enforces cryptographic validity of the `sender_info` blob but imposes **no upper bound on `sender_info.info` size**. Because `SenderInfo::deterministic_heap_bytes()` returns `self.info.len()` verbatim, a large info blob inflates the `EntryKey` size that `LruCache` uses for capacity accounting. An attacker who controls any IC canister can certify arbitrarily large info blobs and send queries whose keys are large enough to evict all legitimate cache entries on every request.

---

### Finding Description

**Step 1 — No size guard in `validate_sender_info`**

`rs/validator/src/ingress_validation.rs:459-544` verifies only that the canister signature over `sender_info.info` is cryptographically valid. There is no check of the form `if sender_info.info.len() > MAX { return Err(...) }`. [1](#0-0) 

**Step 2 — `SenderInfo::deterministic_heap_bytes()` is unbounded**

The manual `DeterministicHeapBytes` implementation for `SenderInfo` returns the raw byte length of the info blob with no cap: [2](#0-1) 

**Step 3 — `EntryKey` propagates this size into LRU accounting**

`EntryKey` derives `DeterministicHeapBytes` and includes `sender_info: Option<SenderInfo>`, so the full info blob length flows into the size reported to `LruCache`: [3](#0-2) 

**Step 4 — `LruCache::push()` uses that size to evict existing entries**

`LruCache::push()` adds the new entry's size to `memory_size` and then calls `evict()`, which pops LRU entries until `memory_size <= memory_capacity`. A single entry whose key size alone approaches the 200 MiB capacity will cause the eviction of every other entry in the cache: [4](#0-3) [5](#0-4) 

**Step 5 — Cache capacity is 200 MiB; no per-entry size limit exists** [6](#0-5) 

---

### Impact Explanation

An attacker who controls any IC canister (a low barrier — anyone with cycles can deploy one) can:

1. Certify a large info blob (e.g., 1–2 MB, bounded only by the HTTP request body limit) in the canister's state tree via a single update call.
2. Construct a valid canister signature proof over that blob.
3. Send a query with `sender_info.info = <large_blob>`. The resulting `EntryKey` is large, causing `LruCache::push()` to evict many legitimate cache entries.
4. Repeat with varying info blobs to maintain continuous cache pressure.

Every evicted legitimate entry forces a full re-execution of the corresponding query on the next request, degrading replica query throughput proportionally to the eviction rate. The cache itself stays within its 200 MiB bound, but its effective hit rate collapses to near zero under sustained attack.

---

### Likelihood Explanation

The barrier is low: any IC canister controller can execute this attack. Certifying a new info blob requires one update call per distinct blob; the cycles cost is negligible. The attack is repeatable and requires no privileged access beyond a deployed canister.

---

### Recommendation

Enforce a maximum size on `sender_info.info` **before** the `EntryKey` is constructed — ideally in `validate_sender_info` alongside the existing signature check, or as an explicit guard in `EntryKey::new`. A reasonable limit (e.g., 256 bytes or 1 KB) is consistent with the intended use case (identity metadata) and eliminates the disproportionate cache-key inflation.

Additionally, consider adding a per-entry size guard in `LruCache::push()` that rejects (or skips caching) any entry whose size alone exceeds a configurable fraction of the total capacity.

---

### Proof of Concept

```
1. Deploy canister C on the IC.
2. In C, certify H("ic-sender-info" || <2MB_blob>) via certified variables.
3. Construct RawSignedSenderInfo { info: <2MB_blob>, signer: C, sig: <proof> }.
4. Send HTTP query:
     POST /api/v2/canister/<target>/query
     { ..., sender_info: { info: <2MB_blob>, signer: C, sig: <proof> } }
5. Observe execution_query_cache_evicted_entries_total spike and
   execution_query_cache_count_bytes drop to near zero.
6. Repeat with a different 2MB blob every few seconds to maintain eviction pressure.
7. Measure: all subsequent legitimate queries become cache misses
   (execution_query_cache_misses_total increases monotonically).
```

    request: &HttpRequest<C>,

    root_of_trust_provider: &R,

        return Ok(());
    };

    // Per the spec, the sender_info signature must verify using the
    // envelope-level sender_pubkey as a canister signature public key.

        Authentication::Authenticated(sig) => &sig.signer_pubkey,
        Authentication::Anonymous => {
            return Err(InvalidSenderInfo(
                "sender_info requires an authenticated request with sender_pubkey".to_string(),
            ));
        }
    };

    verify_sender_info_canister_sig(
        sender_info,
        sender_pubkey,
        ingress_signature_verifier,
        root_of_trust_provider,
    )
}

    pub fn push(&mut self, key: K, value: V) -> Vec<(K, V)> {
        let memory_size = key.deterministic_heap_bytes() + value.deterministic_heap_bytes();
        assert!(memory_size <= MAX_SIZE);
        let disk_size = key.disk_bytes() + value.disk_bytes();
        assert!(disk_size <= MAX_SIZE);

        let removed_entry = self.cache.push(key, value);
        if let Some((removed_key, removed_value)) = &removed_entry {
            self.pop_inner(removed_key, removed_value);
        }
        // This cannot overflow because we know that
        // `self.memory_size <= self.memory_capacity <= MAX_SIZE`
        // and `memory_size <= MAX_SIZE == usize::MAX / 2`.
        self.memory_size += memory_size;
        // Similar as for `memory_size``.
        self.disk_size += disk_size;
        let mut evicted_entries = self.evict();
        self.check_invariants();

        evicted_entries.extend(removed_entry);
        evicted_entries
    }

    fn evict(&mut self) -> Vec<(K, V)> {
        let mut ret = vec![];
        while self.memory_size > self.memory_capacity || self.disk_size > self.disk_capacity {
            match self.cache.pop_lru() {
                Some((key, value)) => {
                    self.pop_inner(&key, &value);
                    ret.push((key, value));
                }
                None => break,
            }
        }
        self.check_invariants();
        ret
    }
