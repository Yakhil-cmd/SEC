**Key findings:**

1. `PublicKey::deserialize_raw` explicitly does NOT check for small-subgroup membership — confirmed by both the docstring warning and the test `public_key_accepts_but_can_detect_keys_with_torsion_component` which shows 18 torsion-component keys are accepted.

3. `verify_public_key` in `rs/crypto/internal/crypto_lib/basic_sig/ed25519/src/api.rs` **does** call `is_torsion_free()`, but this function is **not** in the ingress verification path.

### Title
Ed25519 Small-Subgroup Public Key Forgery via ZIP215 Cofactor Verification — (`packages/ic-ed25519/src/lib.rs`)

### Summary
`verify_basic_sig_by_public_key` for `AlgorithmId::Ed25519` accepts small-subgroup (low-order) public keys and uses ZIP215 cofactor-based verification, allowing an unprivileged attacker to forge a valid signature for any message under any of the 8 known small-subgroup Ed25519 points without knowledge of any private key.

### Finding Description

The ingress verification path is:

`CryptoComponentImpl::verify_basic_sig_by_public_key` → `ic_crypto_standalone_sig_verifier::verify_basic_sig_by_public_key` → `ic_ed25519::PublicKey::deserialize_raw` → `pk.verify_signature` [1](#0-0) 

`deserialize_raw` calls `VerifyingKey::from_bytes` but explicitly does **not** check `is_torsion_free()`: [2](#0-1) 

The test suite confirms small-subgroup keys pass deserialization: [3](#0-2) 

`verify_signature` uses ZIP215 cofactor multiplication: [4](#0-3) 

The check `(recomputed_r − R)·8 == identity` is satisfied trivially when A is a small-subgroup point (8·A = 0), s = 0, and R is any small-subgroup point, for **any** message M:

- `recomputed_r = 0·B + k·(−A) = −k·A` (also small-subgroup, since 8·(−k·A) = 0)
- `8·(recomputed_r − R) = 8·(−k·A − R) = 0 − 0 = 0` ✓

The `s`-canonicality check in `Signature::from_slice` accepts `s = 0` (it is canonical): [5](#0-4) 

By contrast, `verify_public_key` (not called in the ingress path) correctly rejects small-subgroup keys via `is_torsion_free()`: [6](#0-5) 

The delegation validation path also uses `verify_basic_sig_by_public_key` directly: [7](#0-6) 

### Impact Explanation

The 8 small-subgroup Ed25519 points are publicly known constants. Any attacker can:

1. Derive a principal from a small-subgroup public key (e.g., the identity point `[0u8; 32]`).
2. Construct a forged signature `(R = small-subgroup-point, s = 0)` — 64 bytes of trivially computable data.
3. Submit any ingress message authenticated as that principal; `verify_basic_sig_by_public_key` returns `Ok(())`.

Since the small-subgroup keys are public constants, **any** canister whose controller principal is derived from one of these keys is controllable by any party on the internet. Additionally, any delegation chain that routes through a small-subgroup key can be forged for any target message.

### Likelihood Explanation

The attack requires no privileged access, no key material, and no cryptographic computation beyond hashing. The 8 small-subgroup points are documented in academic literature and in the IC codebase itself. The forged signature is 64 bytes of trivially chosen values. The only precondition is that a principal derived from a small-subgroup key exists and has been granted some permission (e.g., canister controller).

### Recommendation

In `verify_basic_sig_by_public_key` (or in `ic_ed25519::PublicKey::deserialize_raw` when used for authentication), add a `is_torsion_free()` check immediately after deserialization and return `CryptoError::MalformedPublicKey` if it fails: [8](#0-7) 

This mirrors the guard already present in `verify_public_key` and in node key validation: [9](#0-8)
