The evidence is conclusive. Let me confirm one more detail about the proxy URL forwarding.### Title
Cross-Version Cache Collision: QueryV2 Response Served for QueryV3 Request — (`rs/boundary_node/ic_boundary/src/routes.rs`, `src/http/middleware/cache.rs`)

---

### Summary

The boundary node's `RequestContext` cache key omits `request_type` from both `Hash` and `PartialEq`. Because `QueryV2` and `QueryV3` both pass the bypass check (`is_query()` returns `true` for both), a cached V2 response is unconditionally returned as a cache hit for a structurally identical V3 request. The V2 response carries a **flat** NNS delegation (canister ranges in flat format), while V3 clients expect a **tree-format** delegation. The client receives a semantically wrong response that will fail V3 certificate validation.

---

### Finding Description

**Root cause — `RequestContext::Hash` and `PartialEq` exclude `request_type`:** [1](#0-0) 

Both implementations hash/compare only `(canister_id, sender, method_name, ingress_expiry, arg/http_request)`. The `request_type` field — which distinguishes `QueryV2` from `QueryV3` — is **never included**.

**Bypass check passes for both versions:** [2](#0-1) [3](#0-2) 

`is_query()` returns `true` for `QueryV2`, `QueryV3`, and `QuerySubnetV3`. Neither version is bypassed.

**Cache key extractor uses `Arc<RequestContext>` directly:** [4](#0-3) 

**The boundary node forwards to version-specific replica URLs:** [5](#0-4) 

`Node::build_url` routes `QueryV2` to `/api/v2/canister/{id}/query` and `QueryV3` to `/api/v3/canister/{id}/query`. These are **different replica endpoints** that produce structurally different responses.

**V2 vs V3 replica responses differ in NNS delegation format:** [6](#0-5) 

- V2: `CanisterRangesFilter::Flat` — flat canister range list in the NNS delegation
- V3: `CanisterRangesFilter::Tree(canister_id)` — Merkle-tree canister ranges

Both versions sign the response with a node key: [7](#0-6) 

The signature is over `QueryResponseHash` which covers the response body and query content. A V2 response cached and replayed for a V3 request carries a **valid signature over the V2 body**, but the V2 body contains a flat-format NNS delegation. A V3 client that validates the delegation tree format will fail.

**The DFINITY test suite itself acknowledges this risk:** [8](#0-7) 

The comment "check that we don't return incorrect cached response" confirms the developers are aware of the cross-version collision risk — but this guard exists only at the replica level, not at the boundary node cache layer where the actual collision occurs.

---

### Impact Explanation

Any client using the V3 query endpoint (`/api/v3/canister/{id}/query`) that hits the boundary node cache after a V2 request with identical `(canister_id, sender, method_name, ingress_expiry, arg)` fields receives a V2-format response. The embedded NNS delegation uses flat canister ranges instead of the tree format V3 clients expect. Certificate validation against the V3 delegation format will fail, causing silent integrity loss or spurious validation errors for any dApp relying on V3 node-signature + delegation verification.

---

### Likelihood Explanation

- Both endpoints are active in production.
- Anonymous queries (the default for many dApps) are cached by default.
- The attacker needs only to send a V2 query before a V3 query with the same parameters — trivially achievable by any unprivileged API client, or even by two independent clients making the same query through different API versions.
- No special privileges, keys, or network position required.

---

### Recommendation

Include `request_type` in both `Hash` and `PartialEq` for `RequestContext`:

```rust
// In Hash::hash:
self.request_type.hash(state);

// In PartialEq::eq:
self.request_type == other.request_type && ...
``` [9](#0-8)

/// Hash and Eq are implemented for request caching
/// They should both work on the same fields so that
/// k1 == k2 && hash(k1) == hash(k2)
impl Hash for RequestContext {
    fn hash<H: Hasher>(&self, state: &mut H) {
        self.canister_id.hash(state);
        self.sender.hash(state);
        self.method_name.hash(state);
        self.ingress_expiry.hash(state);

        // Hash http_request if it's present, arg otherwise
        // They're mutually exclusive
        if self.http_request.is_some() {
            self.http_request.hash(state);
        } else {
            self.arg.hash(state);
        }
    }
}

impl PartialEq for RequestContext {
    fn eq(&self, other: &Self) -> bool {
        let r = self.canister_id == other.canister_id
            && self.sender == other.sender
            && self.method_name == other.method_name
            && self.ingress_expiry == other.ingress_expiry;

        // Same as in hash()
        if self.http_request.is_some() {
            r && self.http_request == other.http_request
        } else {
            r && self.arg == other.arg
        }
    }
}
impl Eq for RequestContext {}
