### Title
Neuron Fees Not Burned But Governance State Cleared in `disburse_neuron` — (`File: rs/nns/governance/src/governance.rs`)

### Summary

In the NNS Governance canister's `disburse_neuron`, when a neuron's `neuron_fees_e8s` is positive but does not exceed `transaction_fee_e8s`, the ledger burn is skipped — yet the governance state unconditionally clears `neuron_fees_e8s` to zero and decrements `cached_neuron_stake_e8s` by the fee amount. This leaves the fee tokens permanently stranded in the neuron's ledger subaccount while governance believes they have been burned, creating a persistent ledger-governance state divergence.

### Finding Description

In `disburse_neuron`, the fee burn is gated on `fees_amount_e8s > transaction_fee_e8s`: [1](#0-0) 

However, the state mutation that clears `neuron_fees_e8s` and decrements `cached_neuron_stake_e8s` runs **unconditionally** immediately after, regardless of whether the burn occurred: [2](#0-1) 

When `fees_amount_e8s <= transaction_fee_e8s` (i.e., the burn is skipped):
- `neuron.neuron_fees_e8s` is set to `0`
- `neuron.cached_neuron_stake_e8s` is decremented by `fees_amount_e8s`
- But no ledger burn occurs — the `fees_amount_e8s` tokens remain in the neuron's subaccount on the ICP ledger

After the subsequent stake disbursal transfer, the neuron's ledger subaccount retains exactly `fees_amount_e8s` tokens that governance no longer tracks. Since governance records `cached_neuron_stake_e8s = 0` and `neuron_fees_e8s = 0`, there is no governance-level path to recover or burn these tokens.

**Contrast with SNS Governance**, which correctly scopes the state update inside the conditional: [3](#0-2) 

In SNS, `neuron_fees_e8s` and `cached_neuron_stake_e8s` are only updated when the burn actually executes.

The NNS TLA+ model (`rs/nns/governance/tla/Disburse_Neuron.tla`) also reflects this behavior — `update_fees` is called in the `else` branch (when fees ≤ transaction fee) without a corresponding ledger burn: [4](#0-3) 

### Impact Explanation

Any NNS neuron with `0 < neuron_fees_e8s <= transaction_fee_e8s` (i.e., 1–10,000 e8s, accumulated from rejected proposals) will permanently lose those tokens when the neuron is dissolved and disbursed. The tokens are stranded in the neuron's governance subaccount on the ICP ledger with no mechanism to recover them. This is a **ledger conservation bug**: ICP tokens are removed from governance accounting without being burned on the ledger, violating the invariant that `sum(neuron subaccount balances) == sum(cached_neuron_stake_e8s)`.

The maximum loss per neuron is `transaction_fee_e8s` = 10,000 e8s = 0.0001 ICP. Across many neurons this accumulates into permanently locked, unrecoverable ICP.

### Likelihood Explanation

This is triggered by any unprivileged user who:
1. Submits a proposal that is rejected (accumulating `neuron_fees_e8s`)
2. Accumulates fees in the range `(0, transaction_fee_e8s]`
3. Dissolves their neuron and calls `disburse_neuron`

This is a realistic, low-barrier scenario. The `reject_cost_e8s` for NNS proposals is configurable and can be set to values within this range. The path is fully reachable via the standard `manage_neuron` ingress endpoint.

### Recommendation

Move the `neuron_fees_e8s` and `cached_neuron_stake_e8s` state update inside the `if fees_amount_e8s > transaction_fee_e8s` block, mirroring the SNS governance implementation. When fees are too small to burn, they should either be left in place (preserving the ledger-governance invariant) or explicitly handled (e.g., written off with a corresponding note that the tokens remain on-chain). The current unconditional clearing is the root cause.

```rust
if fees_amount_e8s > transaction_fee_e8s {
    // burn via ledger
    ...
    // Only update state when burn actually happened
    self.with_neuron_mut(id, |neuron| {
        if neuron.cached_neuron_stake_e8s > fees_amount_e8s {
            neuron.cached_neuron_stake_e8s -= fees_amount_e8s;
        } else {
            neuron.cached_neuron_stake_e8s = 0;
        }
        neuron.neuron_fees_e8s = 0;
    })?;
}
```

### Proof of Concept

1. Create a neuron with stake > minimum, submit a proposal that gets rejected, accumulating `neuron_fees_e8s = 5_000` (below `transaction_fee_e8s = 10_000`).
2. Dissolve the neuron and call `disburse_neuron`.
3. Observe: the ledger burn is skipped (`fees_amount_e8s=5000 <= transaction_fee_e8s=10000`), but governance sets `neuron_fees_e8s = 0` and decrements `cached_neuron_stake_e8s` by 5,000.
4. After disbursal, the neuron's ledger subaccount retains 5,000 e8s. Governance records `cached_neuron_stake_e8s = 0`, `neuron_fees_e8s = 0`. The 5,000 e8s are permanently unrecoverable. [5](#0-4)
