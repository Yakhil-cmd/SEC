### Title
Byzantine Peer CUP Displaces Registry Recovery CUP via `max_by_key` Last-Element Tie-Breaking — (`rs/orchestrator/src/catch_up_package_provider.rs`)

---

### Summary

The `get_latest_cup` function selects the best CUP using `max_by_key(get_cup_proto_height)` over `[local_cup, registry_cup, subnet_cup]`. Because Rust's `max_by_key` returns the **last** element on ties, a Byzantine peer that serves a legitimately threshold-signed CUP at the same height as the registry recovery CUP will always win the selection. The `get_peer_cup` pre-filter only guards against the **local** CUP's height/registry-version, not the registry CUP's. The result is that the orchestrator persists and boots from the peer's CUP, targeting the wrong `state_hash` for state sync.

---

### Finding Description

**Exact code path:**

`get_latest_cup` builds the candidate list and selects the winner:

```rust
let latest_cup_proto = vec![local_cup, registry_cup, subnet_cup]
    .into_iter()
    .flatten()
    .max_by_key(get_cup_proto_height)   // height-only comparison; last wins on tie
    ...
``` [1](#0-0) 

`get_cup_proto_height` decodes only the block height — it carries no state-hash or registry-version information:

```rust
fn get_cup_proto_height(cup: &pb::CatchUpPackage) -> Option<Height> {
    pb::CatchUpContent::decode(cup.content.as_slice())
        .ok()
        .and_then(|content| content.block)
        .map(|block| Height::from(block.height))
}
``` [2](#0-1) 

The `get_peer_cup` pre-filter compares the peer CUP only against `param`, which is derived from the **locally persisted** CUP, not from the registry CUP:

```rust
let param = current_cup
    .map(CatchUpPackageParam::try_from)
    .and_then(Result::ok);
...
if Some(CatchUpPackageParam::from(&cup)) > param {
    return Some(proto);
}
``` [3](#0-2) 

`CatchUpPackageParam` ordering is `(height, registry_version)`:

```
C1 > C2 iff
  C1.height > C2.height ||
  C1.height == C2.height && C1.registry_version > C2.registry_version
``` [4](#0-3) 

**Attack preconditions (both realistic):**

1. The subnet legitimately produced a threshold-signed CUP at height H with `state_hash = S1` during normal operation.
2. The NNS subsequently published a recovery CUP at the same height H with `state_hash = S2 ≠ S1` (standard recovery procedure — the recovery height is the last known CUP height).

**Two exploitable entry states:**

| State | `param` | Peer CUP `{H, R_old}` passes filter? | `max_by_key` winner |
|---|---|---|---|
| No local CUP | `None` | Always (`Some(_) > None`) | `subnet_cup` (last at H) |
| Local CUP at H-1 | `Some{H-1, R_local}` | Yes, since H > H-1 and R_old ≥ R_local | `subnet_cup` (last at H) |

The only safe state is when the local CUP is **already** the recovery CUP at height H with `R_recovery > R_old`, which blocks the peer CUP in the filter. But this is the state the node is trying to reach — it is not the starting state.

**Persist condition is satisfied:**

```rust
if height > local_cup_height || height == local_cup_height && !latest_cup.is_signed() {
    self.persist_cup(&latest_cup_proto)?;
}
``` [5](#0-4) 

In both entry states above, `height > local_cup_height` is true (or `local_cup_height` is `None`), so the peer's CUP with `state_hash = S1` is written to disk.

**Signature verification is not a barrier.** The old CUP at height H was legitimately signed by a threshold of subnet nodes. A single Byzantine peer can replay it verbatim; `verify_combined_threshold_sig_by_public_key` will succeed: [6](#0-5) 

---

### Impact Explanation

The orchestrator persists the peer's CUP (`state_hash = S1`) and returns it to the caller. The replica is started against `S1`. State sync fetches the wrong checkpoint. The node diverges from the recovered subnet, is unable to participate in consensus, and is effectively stalled — precisely during the fragile recovery window when the subnet needs all nodes to converge on the correct state.

---

### Likelihood Explanation

- Requires one Byzantine node in the subnet (below the `f`-fault threshold — explicitly within the IC threat model).
- The old CUP at the recovery height exists whenever the subnet ran normally before recovery; this is the standard recovery scenario.
- No cryptographic forgery is needed; the attacker replays an existing, valid artifact.
- The victim node need only be in the common state of "no local CUP" or "local CUP below recovery height."

---

### Recommendation

In `get_latest_cup`, after obtaining `registry_cup`, give it explicit priority over `subnet_cup` when both have the same height. One approach: replace the flat `max_by_key` with a two-phase selection — first pick the max by `CatchUpPackageParam` (height + registry_version) rather than height alone, and break remaining ties in favour of `registry_cup` over `subnet_cup`. Alternatively, after `max_by_key` selects a winner, check whether `registry_cup` exists at the same height and, if so, prefer it unconditionally.

---

### Proof of Concept

Unit test sketch (within the existing test harness in `catch_up_package_provider.rs`):

1. Construct `registry_cup` at height 100 with `state_hash = [0xAA; 32]` and `registry_version = 50` (recovery CUP).
2. Construct `peer_cup` at height 100 with `state_hash = [0xBB; 32]` and `registry_version = 40` (old legitimate CUP, valid threshold signature via `CryptoReturningOk`).
3. Set `local_cup = None`.
4. Call `get_latest_cup` with a mock `get_peer_cup` that returns `peer_cup`.
5. Assert that the returned CUP has `state_hash = [0xAA; 32]` (registry CUP wins).
6. Observe the test **fails** — the returned CUP has `state_hash = [0xBB; 32]` — confirming the bug.

/// The parameters used to request [`CatchUpPackage`] (by orchestrator).
///
/// We make use of the [`Ord`] trait to determine if one [`CatchUpPackage`] is newer
/// than the other:
///
/// ```ignore
/// C1 > C2 iff
///   C1.height > C2.height ||
///   C1.height == C2.height && C1.registry_version > C2.registry_version
/// ```
#[derive(Copy, Clone, Eq, PartialEq, Debug, Deserialize, Serialize)]
pub struct CatchUpPackageParam {
    height: Height,
    registry_version: RegistryVersion,
}

/// The [`PartialOrd`] instance is explicitly given below to avoid relying on
/// the ordering of the struct fields.
impl PartialOrd for CatchUpPackageParam {
    fn partial_cmp(&self, other: &CatchUpPackageParam) -> Option<Ordering> {
        match (
            self.height.cmp(&other.height),
            self.registry_version.partial_cmp(&other.registry_version),
        ) {
            // If height is less, registry version needs to be less or equal
            (Ordering::Less, Some(x)) if x != Ordering::Greater => Some(Ordering::Less),
            // If height is equal, registry version decides ordering
            (Ordering::Equal, Some(x)) => Some(x),
            // If height is greater, registry version needs to be equal or greater
            (Ordering::Greater, Some(x)) if x != Ordering::Less => Some(Ordering::Greater),
            // All other combinations of height and registry versions are incomparable
            // This also covers the case, that the registry versions themselves are incomparable
            _ => None,
        }
    }
