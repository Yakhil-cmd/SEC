### Title
SNS Swap `finalize_swap` Has No Emergency Exit for SNS Tokens When Finalization Is Permanently Blocked - (File: `rs/sns/swap/src/swap.rs`)

### Summary
The SNS Swap canister's `finalize_swap` function makes multiple sequential inter-canister calls. If any call permanently fails after `sweep_icp` has already succeeded (ICP transferred to SNS governance), SNS tokens remain locked in the swap canister with no emergency exit mechanism for users. Additionally, the code explicitly acknowledges that a panic in `finalize_inner` will permanently freeze the `finalize_swap_in_progress` lock, blocking all future finalization attempts.

### Finding Description
The `finalize_inner` function executes a strict sequential pipeline of inter-canister calls:

1. `sweep_icp` — transfers ICP to SNS governance (committed) or back to buyers (aborted)
2. `settle_neurons_fund_participation` — calls NNS governance canister
3. `create_sns_neuron_recipes` — local computation
4. `sweep_sns` — transfers SNS tokens to neuron accounts
5. `claim_swap_neurons` — calls SNS governance canister
6. `set_sns_governance_to_normal_mode` — calls SNS governance canister [1](#0-0) 

Each step gates the next: if any step sets an error message, `finalize_inner` returns early and the remaining steps are skipped. [2](#0-1) 

If `settle_neurons_fund_participation` (step 2) permanently fails — for example, due to a bug introduced in an NNS governance upgrade that breaks this specific endpoint — finalization halts after step 1 has already completed. ICP has been swept to SNS governance and is gone from the swap canister's subaccounts. SNS tokens remain in the swap canister. There is no `error_refund_sns` function or equivalent emergency exit for SNS tokens.

The only existing emergency exit, `error_refund_icp`, explicitly refuses to refund ICP that is still "in escrow" (i.e., `transfer_success_timestamp_seconds == 0`), and covers only ICP — not SNS tokens: [3](#0-2) 

A second, compounding issue: the code explicitly acknowledges that a panic in `finalize_inner` will permanently freeze the `finalize_swap_in_progress` lock: [4](#0-3) 

The lock is a plain boolean in canister state with no `scopeguard`-style drop protection: [5](#0-4) 

If the lock is frozen, every subsequent call to `finalize_swap` returns immediately with an error, permanently blocking finalization without any user-accessible recovery path. [6](#0-5) 

The `finalize_swap` endpoint itself has no access control — any unprivileged principal can call it: [7](#0-6) 

### Impact Explanation
In a committed SNS swap where `sweep_icp` has succeeded but a subsequent inter-canister call permanently fails, buyers lose access to their SNS tokens. ICP has already been transferred to SNS governance and cannot be recovered via `error_refund_icp`. SNS tokens remain in the swap canister with no user-accessible withdrawal path. The only recovery requires a governance-approved canister upgrade to either fix the broken dependency or add an emergency exit — a process that takes days and is not guaranteed to succeed.

### Likelihood Explanation
The likelihood is low-to-medium. The NNS governance canister is upgraded via proposals and could introduce a breaking change to the `settle_neurons_fund_participation` endpoint. The SNS governance canister (called in steps 5 and 6) is similarly upgradeable and could break `claim_swap_neurons`. A panic in `finalize_inner` — while the code attempts to avoid it — is acknowledged as possible in the source comments. Each SNS launch triggers `finalize_swap`, making this path exercised repeatedly across the ecosystem.

### Recommendation
1. **Add an emergency SNS token exit**: Implement an `error_refund_sns` function analogous to `error_refund_icp` that allows buyers to reclaim SNS tokens from the swap canister's accounts when finalization has been permanently stuck for a configurable timeout period.
2. **Protect the lock against panics**: Wrap the lock acquisition in a `scopeguard` (as already used in `rs/bitcoin/ckbtc/minter/src/reimbursement/mod.rs`) so the lock is always released even if `finalize_inner` panics.
3. **Decouple `sweep_icp` from downstream failures**: Consider making `sweep_icp` and `sweep_sns` independently retryable without requiring all prior steps to succeed, so a failure in `settle_neurons_fund_participation` does not block SNS token distribution.

### Proof of Concept
1. A committed SNS swap reaches the `Committed` lifecycle state with buyers holding ICP in the swap canister's subaccounts.
2. An unprivileged principal calls `finalize_swap`.
3. `finalize_inner` executes `sweep_icp` successfully — ICP is transferred to SNS governance.
4. `settle_neurons_fund_participation` calls NNS governance. The NNS governance canister has been upgraded with a breaking change that causes this endpoint to permanently return an error.
5. `finalize_inner` returns early with an error message. The lock is released.
6. All subsequent calls to `finalize_swap` reach step 4 and fail identically.
7. SNS tokens remain in the swap canister. `error_refund_icp` refuses to act because buyers' `transfer_success_timestamp_seconds` is 0 (ICP was swept, not refunded). There is no `error_refund_sns`.
8. Buyers have permanently lost access to both their ICP (sent to SNS governance) and their SNS tokens (stuck in swap canister). [8](#0-7)

    pub async fn finalize_inner(
        &mut self,
        now_fn: fn(bool) -> u64,
        environment: &mut impl CanisterEnvironment,
    ) -> FinalizeSwapResponse {
        let mut finalize_swap_response = FinalizeSwapResponse::default();

        if let Err(e) = self.can_finalize() {
            finalize_swap_response.set_error_message(e);
            return finalize_swap_response;
        }

        // Transfer the ICP tokens from the Swap canister.
        finalize_swap_response
            .set_sweep_icp_result(self.sweep_icp(now_fn, environment.icp_ledger()).await);
        if finalize_swap_response.has_error_message() {
            return finalize_swap_response;
        }

        // Settle the Neurons' Fund participation in the token swap.
        finalize_swap_response.set_settle_neurons_fund_participation_result(
            self.settle_neurons_fund_participation(environment.nns_governance_mut())
                .await,
        );
        if finalize_swap_response.has_error_message() {
            return finalize_swap_response;
        }

        if self.should_restore_dapp_control() {
            // Restore controllers of dapp canisters to their original
            // owners (i.e. self.init.fallback_controller_principal_ids).
            finalize_swap_response.set_set_dapp_controllers_result(
                self.restore_dapp_controllers_for_finalize(environment.sns_root_mut())
                    .await,
            );

            // In the case of returning control of the dapp(s) to the fallback
            // controllers, finalize() need not do any more work, so always return
            // and end execution.
            return finalize_swap_response;
        }

        // Create the SnsNeuronRecipes based on the contribution of direct and NF participants
        finalize_swap_response
            .set_create_sns_neuron_recipes_result(self.create_sns_neuron_recipes());
        if finalize_swap_response.has_error_message() {
            return finalize_swap_response;
        }

        // Transfer the SNS tokens from the Swap canister.
        finalize_swap_response
            .set_sweep_sns_result(self.sweep_sns(now_fn, environment.sns_ledger()).await);
        if finalize_swap_response.has_error_message() {
            return finalize_swap_response;
        }

        // Once SNS tokens have been distributed to the correct accounts, claim
        // them as neurons on behalf of the Swap participants.
        finalize_swap_response.set_claim_neuron_result(
            self.claim_swap_neurons(environment.sns_governance_mut())
                .await,
        );
        if finalize_swap_response.has_error_message() {
            return finalize_swap_response;
        }

        finalize_swap_response.set_set_mode_call_result(
            Self::set_sns_governance_to_normal_mode(environment.sns_governance_mut()).await,
        );
