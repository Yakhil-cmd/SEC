### Title
Unbounded Synchronous Follow-Cascade BFS in SNS `cast_vote_and_cascade_follow` Enables Voting DOS - (File: `rs/sns/governance/src/governance.rs`)

---

### Summary

SNS governance's `cast_vote_and_cascade_follow` executes an unbounded breadth-first search (BFS) over the entire neuron follow graph in a single synchronous message execution, with no instruction-limit guard. Any unprivileged user who creates enough SNS neurons and sets up a sufficiently large follow chain can cause every subsequent `register_vote` or `make_proposal` call to trap due to exceeding the IC's per-message instruction limit, permanently DOSing voting for the affected SNS.

---

### Finding Description

The SNS governance function `cast_vote_and_cascade_follow` performs a BFS over the follow graph to propagate votes: [1](#0-0) 

The outer `while !induction_votes.is_empty()` loop has **no instruction-limit check and no mechanism to resume across multiple messages**. It must complete entirely within the single message execution that triggered it.

This function is called synchronously in two user-reachable paths:

1. **`make_proposal`** — called when any eligible neuron submits a proposal; the proposer's initial `Yes` vote is immediately cascaded: [2](#0-1) 

2. **`register_vote`** — called when any eligible neuron directly casts a vote: [3](#0-2) 

By contrast, the NNS governance equivalent (`rs/nns/governance/src/voting.rs`) explicitly checks `is_over_instructions_limit()` at each step and resumes cascade processing across multiple timer ticks: [4](#0-3) 

The SNS has no analogous multi-round mechanism. Additionally, `compute_ballots_for_new_proposal`, called during `make_proposal`, also iterates over **all** neurons in `self.proto.neurons` with no bound: [5](#0-4) 

---

### Impact Explanation

If the BFS traversal (or the ballot-computation loop) exceeds the IC's per-message instruction limit (~40 billion instructions for update calls), the message traps. Because both `make_proposal` and `register_vote` are synchronous update calls that invoke `cast_vote_and_cascade_follow` inline, a sufficiently large follow graph causes **every vote and every proposal submission to trap**, permanently halting governance for the SNS. No state is committed on a trap, so the DOS is persistent until the follow graph is reduced — which itself requires governance to function.

---

### Likelihood Explanation

SNS neuron creation is permissionless: any user with sufficient SNS tokens can stake and create neurons. Follow relationships are also freely configurable by neuron owners. An attacker needs only to create a linear follow chain (A→B→C→…→N) of sufficient depth. The IC's instruction limit is finite and the BFS processes one neuron per iteration with non-trivial per-neuron work (BTreeMap lookups, string conversions). With tens of thousands of neurons in a chain, the limit is reachable. SNS DAOs with open neuron creation and no cap on neuron count are directly exposed.

---

### Recommendation

Apply the same pattern used by NNS governance:

1. Add an `is_over_instructions_limit()` guard inside the BFS loop in `cast_vote_and_cascade_follow`, returning early when the limit is approached.
2. Persist the in-progress cascade state and resume it in subsequent timer ticks, as NNS governance does via its `VotingStateMachine` and `continue_processing`.
3. Enforce a maximum neuron count or a maximum follow-graph depth for SNS instances to bound worst-case cascade size.

---

### Proof of Concept

1. Deploy an SNS with permissionless neuron creation.
2. Create N neurons (e.g., N = 50,000) each staking the minimum amount.
3. Configure a linear follow chain: neuron[1] follows neuron[0], neuron[2] follows neuron[1], …, neuron[N] follows neuron[N-1], all on the same `function_id`.
4. Have neuron[0] submit a proposal via `manage_neuron { MakeProposal(...) }`.
5. The `cast_vote_and_cascade_follow` BFS must traverse all N neurons in one message. With N large enough, the message traps.
6. Repeat for `register_vote`: any direct vote by neuron[0] also triggers the full cascade and traps.
7. Result: no neuron can successfully vote or submit proposals; SNS governance is permanently halted.

        while !induction_votes.is_empty() {
            // This will be populated with the followers of neurons in the
            // current BFS tier, who might be swayed to indirectly vote, thus
            // forming the next tier in the BFS.
            let mut follower_neuron_ids = BTreeSet::new();

            // Process the current tier in the BFS.
            for (current_neuron_id, current_new_vote) in &induction_votes {
                let current_ballot = match ballots.get_mut(current_neuron_id) {
                    Some(b) => b,
                    None => {
                        // neuron_id has no (blank) ballot, which means they
                        // were not eligible when the proposal was first
                        // created. This is fairly unusual, but does not
                        // indicate a bug (therefore, no log).
                        continue;
                    }
                };

                // Only fill in "blank" ballots. I.e. those with vote ==
                // Unspecified. This check could just as well be done before
                // current_neuron_id is added to induction_votes.
                if current_ballot.vote != (Vote::Unspecified as i32) {
                    continue;
                }

                // Fill in current_ballot.
                assert_ne!(*current_new_vote, Vote::Unspecified);
                current_ballot.vote = *current_new_vote as i32;
                current_ballot.cast_timestamp_seconds = now_seconds;

                // Take note of the followers of current_neuron_id, and add them
                // to the next "tier" in the BFS.

                if let Some(new_follower_neuron_ids) = topic_followers
                    .and_then(|topic_followers| topic_followers.get(current_neuron_id))
                {
                    for follower_neuron_id in new_follower_neuron_ids {
                        follower_neuron_ids.insert(follower_neuron_id.clone());
                    }
                }

                if let Some(new_follower_neuron_ids) =
                    neuron_id_to_follower_neuron_ids.get(current_neuron_id)
                {
                    for follower_neuron_id in new_follower_neuron_ids {
                        follower_neuron_ids.insert(follower_neuron_id.clone());
                    }
                }
            }

            // Prepare for the next iteration of the (outer most) loop by
            // constructing the next BFS tier (from follower_neuron_ids).
            induction_votes.clear();
            for follower_neuron_id in follower_neuron_ids {
                let Some(follower_neuron) = neurons.get(&follower_neuron_id.to_string()) else {
                    // This is a highly suspicious, because currently, we do not
                    // delete neurons, which means that we have an invalid NeuronId
                    // floating around in the system, which indicates that we have a
                    // bug. For now, we deal with that by logging, and pretending like
                    // we did not see follower_neuron_id.
                    log!(
                        ERROR,
                        "Missing neuron {} while trying to record (and cascade) \
                            a vote on proposal {:#?}.",
                        follower_neuron_id,
                        proposal_id,
                    );
                    continue;
                };

                let follower_vote = follower_neuron.vote_from_ballots_following(
                    function_id,
                    topic,
                    ballots,
                    proposal_id,
                );

                if follower_vote != Vote::Unspecified {
                    // follower_neuron would be swayed by its followees!
                    //
                    // This is the other (earlier) point at which we could
                    // consider whether a neuron is already locked in, and that
                    // no recursion is needed.
                    induction_votes.insert(follower_neuron_id.to_string(), follower_vote);
                }
            }
        }

    fn continue_processing(
        &mut self,
        neuron_store: &mut NeuronStore,
        ballots: &mut HashMap<u64, Ballot>,
        is_over_instructions_limit: fn() -> bool,
    ) {
        let voting_finished = self.is_voting_finished();

        if !voting_finished {
            while let Some(neuron_id) = self.neurons_to_check_followers.pop_first() {
                self.add_followers_to_check(neuron_store, neuron_id, self.topic);

                // Before we check the next one, see if we're over the limit.
                if is_over_instructions_limit() {
                    return;
                }
            }

            // Memory optimization, will not cause tests to fail if removed
            retain_neurons_with_castable_ballots(&mut self.followers_to_check, ballots);

            while let Some(follower) = self.followers_to_check.pop_first() {
                let vote = match neuron_store
                    .neuron_would_follow_ballots(follower, self.topic, ballots)
                {
                    Ok(vote) => vote,
                    Err(e) => {
                        // This is a bad inconsistency, but there is
                        // nothing that can be done about it at this
                        // place.  We somehow have followers recorded that don't exist.
                        eprintln!(
                            "error in cast_vote_and_cascade_follow when gathering induction votes: {:?}",
                            e
                        );
                        Vote::Unspecified
                    }
                };
                // Casting vote immediately might affect other follower votes, which makes
                // voting resolution take fewer iterations.
                // Vote::Unspecified is ignored by cast_vote.
                self.cast_vote(ballots, follower, vote);

                if is_over_instructions_limit() {
                    return;
                }
            }
        } else {
            while let Some((neuron_id, vote)) = self.recent_neuron_ballots_to_record.pop_first() {
                match neuron_store.record_neuron_vote(neuron_id, self.topic, self.proposal_id, vote)
                {
                    Ok(_) => {}
                    Err(e) => {
                        // This is a bad inconsistency, but there is
                        // nothing that can be done about it at this
                        // place.  We somehow have followers recorded that don't exist.
                        eprintln!(
                            "error in cast_vote_and_cascade_follow when gathering induction votes: {:?}",
                            e
                        );
                    }
                };

                if is_over_instructions_limit() {
                    return;
                }
            }
        }
    }
