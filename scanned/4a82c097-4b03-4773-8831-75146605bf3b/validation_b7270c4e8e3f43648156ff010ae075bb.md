### Title
Unconditional `neuron_ids` Removal on Failed Transfer Permanently Orphans ICP Stake — (`rs/nns/gtc/src/lib.rs`)

### Summary

`AccountState::transfer` removes a `neuron_id` from `self.neuron_ids` **unconditionally** before checking whether `GovernanceCanister::transfer_gtc_neuron` succeeded. If the governance (or ledger) call fails transiently, the neuron is evicted from `neuron_ids`, placed in `failed_transferred_neurons`, and the function still returns `Ok(())`. The caller then sets `has_donated = true` (or `has_forwarded = true`), permanently sealing the account. The neuron remains in governance under GTC control with no GTC-side recovery path.

### Finding Description

The defect is at line 192 of `rs/nns/gtc/src/lib.rs`:

```rust
for neuron_id in neuron_ids {
    let result =
        GovernanceCanister::transfer_gtc_neuron(neuron_id, custodian_neuron_id).await;

    self.neuron_ids.retain(|id| id != &neuron_id);   // ← unconditional, before result check

    match result {
        Ok(_) => self.successfully_transferred_neurons.push(donated_neuron),
        Err(e) => {
            donated_neuron.error = Some(e.to_string());
            self.failed_transferred_neurons.push(donated_neuron)  // no retry, no re-add
        }
    }
}

Ok(())   // ← always Ok, even if every neuron failed
``` [1](#0-0) 

The `retain` call removes the `neuron_id` from `self.neuron_ids` regardless of the result. After `transfer()` returns `Ok(())`, the callers permanently lock the account:

- `donate_account` sets `account.has_donated = true`
- `forward_whitelisted_unclaimed_accounts` sets `account.has_forwarded = true` [2](#0-1) 

The guards at the top of `transfer()` then prevent any future invocation: [3](#0-2) 

On the governance side, `transfer_gtc_neuron` performs a ledger transfer before deleting the donor neuron. If the ledger call fails, the neuron is **not** deleted from governance — it remains GTC-controlled: [4](#0-3) 

So after a failed transfer: the neuron still exists in governance under GTC control, but `neuron_ids` no longer contains it, `has_donated`/`has_forwarded` is `true`, and `failed_transferred_neurons` has no retry mechanism.

### Impact Explanation

Any neuron whose `transfer_gtc_neuron` call fails is permanently orphaned:
- It is no longer in `neuron_ids` → not claimable, not re-donatable, not re-forwardable.
- It is not in `successfully_transferred_neurons` → the recipient neuron did not receive the stake.
- It remains in governance under GTC control → no user can ever control it again.
- The ICP stake locked in that neuron is permanently inaccessible.

### Likelihood Explanation

The trigger is a transient inter-canister call failure, which is realistic on the IC:
- Governance canister upgrade in progress (canister stopped/frozen).
- Ledger temporarily unavailable or message queue full.
- Cycles exhaustion on the GTC canister mid-loop.
- Any governance-side error (e.g., recipient neuron not found, donor neuron stake below transaction fee).

The `donate_account` entrypoint is callable by any GTC account owner (unprivileged ingress). `forward_whitelisted_unclaimed_accounts` is callable by **anyone** after 188 days. [5](#0-4) 

### Recommendation

Only remove `neuron_id` from `self.neuron_ids` on success:

```rust
match result {
    Ok(_) => {
        self.neuron_ids.retain(|id| id != &neuron_id);
        self.successfully_transferred_neurons.push(donated_neuron);
    }
    Err(e) => {
        donated_neuron.error = Some(e.to_string());
        self.failed_transferred_neurons.push(donated_neuron);
        // neuron_id stays in self.neuron_ids for retry
    }
}
```

Additionally, `transfer()` should return `Err` (or a partial-failure type) if any neuron failed, so callers do not set `has_donated`/`has_forwarded = true` prematurely.

### Proof of Concept

1. Set up a GTC account with two neuron IDs: `[N1, N2]`.
2. Mock `GovernanceCanister::transfer_gtc_neuron` to succeed for `N1` and return `Err` for `N2`.
3. Call `donate_account`.
4. Assert: `transfer()` returns `Ok(())`.
5. Assert: `account.neuron_ids` is empty (both removed unconditionally).
6. Assert: `account.failed_transferred_neurons` contains `N2` with an error.
7. Assert: `account.has_donated == true`.
8. Assert: Calling `donate_account` again returns `Err("Account has already donated its funds")`.
9. Assert: `N2` still exists in governance under GTC control with its original stake intact.
10. Confirm: No GTC method can ever reach `N2` again — the ICP stake is permanently orphaned. [6](#0-5)
