### Title
Unsanitized Memo Injection into GenericDisplayMessage Markdown Enables Consent-Message Spoofing - (`packages/icrc-ledger-types/src/icrc21/responses.rs`)

### Summary
The `add_memo` function embeds attacker-controlled memo bytes directly into the `GenericDisplayMessage` markdown string without escaping markdown-special characters. A backtick in the memo closes the opening code-span delimiter, allowing arbitrary markdown to be injected after it. A wallet UI that renders `GenericDisplayMessage` as markdown will display attacker-injected fake fields (e.g., `**Amount:**`, `**To:**`) that are visually indistinguishable from the real ledger-generated fields, enabling a user to be deceived about the transaction they are authorizing.

### Finding Description
`add_memo` in `responses.rs` converts the raw memo bytes to a UTF-8 string when valid, then interpolates it verbatim into the markdown message:

```rust
message.push_str(&format!("\n\n**Memo:**\n`{memo_str}`"));
``` [1](#0-0) 

No escaping or sanitization of `memo_str` is performed before this interpolation. A grep across the entire `icrc21` package confirms there is no sanitization step anywhere in the pipeline.

The call chain is:

1. Attacker calls `icrc21_canister_call_consent_message` (public ingress endpoint) with a `TransferArg` whose `memo` field contains crafted UTF-8 bytes.
2. In `lib.rs`, the memo is decoded and passed to `display_message_builder.with_memo(GenericMemo::Icrc1Memo(memo.0))` with no validation beyond UTF-8 well-formedness. [2](#0-1) 

3. `build()` calls `message.add_memo(...)`, which reaches the unsanitized interpolation above.

A memo of `` `\n\n**Amount:** `999999 ICP`\n\n**To:** `attacker_account` `` produces the following raw string appended to the message:

```
**Memo:**
``

**Amount:** `999999 ICP`

**To:** `attacker_account`
```

The first backtick in the memo closes the opening backtick, producing an empty code span (` `` `), and everything after it is free markdown rendered by the wallet.

### Impact Explanation
A wallet UI that renders `GenericDisplayMessage` as markdown (which is the intended use — the entire message is authored in markdown: `# Send …`, `**From:**`, `` `account` ``, etc.) will display the injected fake fields with the same visual weight as the real ledger-generated fields. A user reviewing the consent screen sees a fabricated `Amount` and `To` field and authorizes a transfer believing it goes to a different recipient or for a different amount than the actual on-chain transaction. [3](#0-2) 

### Likelihood Explanation
- The endpoint is fully public; no privilege is required.
- The only precondition is that the memo is valid UTF-8, which is trivially satisfied.
- The ICRC-21 `GenericDisplayMessage` variant is explicitly designed to be rendered as markdown by wallet UIs; the ledger itself uses markdown syntax throughout the message construction.
- The exploit is deterministic and locally testable with a single canister call.

### Recommendation
Before interpolating `memo_str` into the markdown string, escape all characters that have special meaning in a CommonMark code span — at minimum, replace every backtick with its hex-encoded representation or with a Unicode lookalike, and strip or escape newlines. A safe alternative is to use a fenced code block with a delimiter that cannot appear in the memo (e.g., triple backticks with a unique info string), or to always hex-encode the memo when `GenericDisplayMessage` is the target format, reserving the UTF-8 display for `FieldsDisplayMessage` where the value is a structured `Value::Text` field that wallets render safely.

    pub fn add_memo(&mut self, memo: GenericMemo) {
        match memo {
            GenericMemo::Icrc1Memo(memo) => {
                // Check if the memo is a valid UTF-8 string and display it as such if it is.
                let memo_str = match std::str::from_utf8(memo.as_slice()) {
                    Ok(valid_str) => valid_str.to_string(),
                    Err(_) => hex::encode(memo.as_slice()),
                };

                    ConsentMessage::GenericDisplayMessage(message) => {
                        message.push_str(&format!("\n\n**Memo:**\n`{memo_str}`"));
                    }

                        .fields
