### Title
Missing Upper-Bound Validation on `transfer_fee` in `ManageLedgerParameters` Proposal Allows Breaking SNS Neuron Operations - (`rs/sns/governance/src/proposal.rs`)

### Summary

The `validate_and_render_manage_ledger_parameters` function in SNS governance accepts any `u64` value for `transfer_fee` without checking it against `neuron_minimum_stake_e8s`. A governance proposal can set the ledger's transfer fee to a value equal to or exceeding `neuron_minimum_stake_e8s`, violating the required system invariant and permanently breaking neuron operations such as `split_neuron`, `disburse_to_neuron`, and `disburse_maturity` for all SNS participants.

### Finding Description

**Root cause — missing bounds check in proposal validation:**

`validate_and_render_manage_ledger_parameters` in `rs/sns/governance/src/proposal.rs` accepts any `transfer_fee` value without validating it against `neuron_minimum_stake_e8s`:

```rust
if let Some(transfer_fee) = transfer_fee {
    render += &format!("# Set token transfer fee: {transfer_fee} token-quantums. \n",);
    change = true;
}
```

No upper-bound check is performed. [1](#0-0) 

**Invariant that is bypassed:**

`NervousSystemParameters::validate_neuron_minimum_stake_e8s` enforces that `neuron_minimum_stake_e8s > transaction_fee_e8s`. This invariant is enforced when `ManageNervousSystemParameters` proposals are executed, but is **not** enforced when `ManageLedgerParameters` proposals are executed. [2](#0-1) 

**Execution path that writes the unchecked value:**

When the proposal executes, `perform_manage_ledger_parameters` unconditionally writes the new fee into `NervousSystemParameters.transaction_fee_e8s` after the ledger upgrade succeeds:

```rust
if let Some(nervous_system_parameters) = self.proto.parameters.as_mut()
    && let Some(transfer_fee) = manage_ledger_parameters.transfer_fee
{
    nervous_system_parameters.transaction_fee_e8s = Some(transfer_fee);
}
``` [3](#0-2) 

**Downstream breakage — `split_neuron` and `disburse_to_neuron`:**

Both functions compute the minimum required amount as `min_stake + transaction_fee_e8s`. If `transaction_fee_e8s` is set to a value close to `u64::MAX`, this addition overflows (wraps in release mode), making the guard meaningless. If set to a value >= `neuron_minimum_stake_e8s`, the minimum required split amount becomes `>= 2 * neuron_minimum_stake_e8s`, which may exceed any neuron's actual stake. [4](#0-3) [5](#0-4) 

**Downstream breakage — `disburse_maturity`:**

`disburse_maturity` rejects disbursements when `worst_case_maturity_modulation < transaction_fee_e8s`. If `transaction_fee_e8s` is set to a large value (e.g., `u64::MAX`), this condition is always true and **all** maturity disbursements for all neurons in the SNS are permanently blocked. [6](#0-5) 

**The `ManageLedgerParameters` proposal type:**

`transfer_fee` is typed as `Option<u64>` with no documented or enforced upper bound. [7](#0-6) 

### Impact Explanation

A successfully executed `ManageLedgerParameters` proposal with `transfer_fee >= neuron_minimum_stake_e8s` (or near `u64::MAX`) causes:

1. **`split_neuron` and `disburse_to_neuron` become impossible or arithmetically broken** — the minimum required amount overflows or exceeds any realistic neuron stake.
2. **`disburse_maturity` is permanently blocked for all neurons** — the fee threshold check always fails.
3. **The SNS token ecosystem is effectively frozen** for these core governance operations, with no recovery path short of another governance proposal (which itself requires paying the now-inflated fee).

This is a direct analog to the Cally.sol bug: a privileged actor sets a fee parameter to an extreme value, causing all downstream operations that depend on that fee to revert or behave incorrectly.

### Likelihood Explanation

The attack requires a `ManageLedgerParameters` governance proposal to pass with a majority vote. In the SNS context, this is achievable by:

- A malicious actor who accumulates sufficient SNS voting power.
- A well-intentioned governance majority that accidentally sets the fee too high (e.g., confusing e8s with tokens).

The absence of any validation in `validate_and_render_manage_ledger_parameters` means there is no on-chain protection against this misconfiguration. The `ManageNervousSystemParameters` path has this protection (via `NervousSystemParameters::validate()`), but `ManageLedgerParameters` does not.

### Recommendation

Add a bounds check in `validate_and_render_manage_ledger_parameters` to ensure the proposed `transfer_fee` is strictly less than the current `neuron_minimum_stake_e8s`:

```rust
if let Some(transfer_fee) = transfer_fee {
    let min_stake = current_parameters
        .neuron_minimum_stake_e8s
        .unwrap_or(u64::MAX);
    if *transfer_fee >= min_stake {
        return Err(format!(
            "transfer_fee ({transfer_fee}) must be less than \
             neuron_minimum_stake_e8s ({min_stake})"
        ));
    }
    render += &format!("# Set token transfer fee: {transfer_fee} token-quantums. \n");
    change = true;
}
```

Additionally, add a re-validation of the full `NervousSystemParameters` invariant inside `perform_manage_ledger_parameters` before writing the new fee, mirroring the guard already present in `perform_manage_nervous_system_parameters`. [8](#0-7) 

### Proof of Concept

1. An SNS governance majority submits a `ManageLedgerParameters` proposal with `transfer_fee = u64::MAX` (or any value >= `neuron_minimum_stake_e8s`).
2. `validate_and_render_manage_ledger_parameters` accepts the proposal — no bounds check exists. [9](#0-8) 
3. The proposal passes and `perform_manage_ledger_parameters` upgrades the ledger with the new fee, then writes `nervous_system_parameters.transaction_fee_e8s = Some(u64::MAX)`. [10](#0-9) 
4. Any user calling `disburse_maturity` now receives `PreconditionFailed` because `worst_case_maturity_modulation < u64::MAX` is always true. [6](#0-5) 
5. Any user calling `split_neuron` with a realistic amount finds `split.amount_e8s < (min_stake + u64::MAX)` wraps to a small number, making the guard unreliable; or with a moderate extreme fee, the minimum required amount exceeds any neuron's stake. [11](#0-10)

        Ok(render)
