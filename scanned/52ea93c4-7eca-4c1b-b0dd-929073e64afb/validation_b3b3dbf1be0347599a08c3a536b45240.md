### Title
Authorized-Caller State Not Persisted to Stable Memory — Silent Revert to Hardcoded Default on Upgrade - (File: rs/engine_controller/canister/canister.rs)

---

### Summary
The `engine_controller` canister stores its sole authorized caller in `thread_local!` heap memory that is never written to stable storage. On every `post_upgrade`, the authorized caller is unconditionally re-derived from the upgrade argument, falling back to a hardcoded default principal when no argument is supplied. There is no runtime method to transfer or rotate the authorized caller. This means any canister upgrade that omits the `authorized_caller` field silently restores the hardcoded default principal as the sole entity permitted to create subnets, delete subnets, update subnet settings, and deploy replica versions — regardless of what principal was previously configured.

---

### Finding Description
In `rs/engine_controller/canister/canister.rs`, the authorized caller is held exclusively in a `thread_local!` `RefCell`:

```rust
thread_local! {
    static AUTHORIZED_CALLER: RefCell<Principal> = RefCell::new(default_authorized_caller());
}
```

`thread_local!` storage is heap memory. It is **not** serialized to stable memory at `pre_upgrade` and is **not** read from stable memory at `post_upgrade`. Instead, `post_upgrade` calls `apply_init_args`, which unconditionally overwrites `AUTHORIZED_CALLER` from the upgrade argument, defaulting to the hardcoded constant `DEFAULT_AUTHORIZED_CALLER` (`bct5z-vccu4-6q4t2-3lb6l-wm43p-ulppt-o5sqq-w6het-rthdz-qp4yn-fqe`) when the argument is `None` or when `authorized_caller` is `None` inside the argument:

```rust
fn apply_init_args(args: Option<EngineControllerInitArgs>) {
    let args = args.unwrap_or_default();
    let authorized = args
        .authorized_caller
        .unwrap_or_else(default_authorized_caller);   // ← silent revert
    AUTHORIZED_CALLER.with(|c| *c.borrow_mut() = authorized);
    ...
}

#[post_upgrade]
fn post_upgrade(args: Option<EngineControllerInitArgs>) {
    apply_init_args(args);
}
```

There is no `update`-callable method to change the authorized caller at runtime. The only mechanism is a canister upgrade with an explicit `authorized_caller` field. If any upgrade proposal omits that field — even accidentally — the hardcoded default principal immediately regains exclusive control over all privileged operations.

The test suite explicitly documents and validates this revert behavior:

```rust
// Upgrade with no override: default principal becomes authorized again.
pic.upgrade_canister(..., Encode!(&None::<EngineControllerInitArgs>).unwrap(), ...)
    .await.expect("upgrade should succeed");
// custom caller must be rejected after upgrade with default
```

---

### Impact Explanation
The engine controller canister is the sole non-NNS-governance path to:
- `create_engine` → calls `registry.create_subnet` to provision a new `CloudEngine` subnet
- `delete_engine` → calls `registry.delete_subnet`
- `update_subnet` → modifies `subnet_admins` and `is_halted` on any engine subnet
- `deploy_guestos_to_all_subnet_nodes` → deploys a replica version to all nodes of an engine subnet

If the hardcoded default principal (`bct5z-vccu4-6q4t2-3lb6l-wm43p-ulppt-o5sqq-w6het-rthdz-qp4yn-fqe`) was intentionally rotated away via a prior upgrade, a subsequent upgrade that omits `authorized_caller` silently reinstates it. The reinstated principal can immediately:
1. Create unauthorized engine subnets, consuming node resources and polluting the registry.
2. Delete existing engine subnets, causing service disruption.
3. Halt engine subnets via `is_halted`, denying service to all canisters on those subnets.
4. Deploy an arbitrary (but NNS-blessed) replica version to engine subnet nodes.

Because the registry canister accepts `create_subnet` and `delete_subnet` calls from the engine controller without additional governance approval, the impact is direct and immediate upon the IC's subnet topology.

---

### Likelihood Explanation
The likelihood is **medium**. The revert requires a canister upgrade — which on mainnet requires an NNS governance proposal. However:
- The `EngineControllerInitArgs` struct has `authorized_caller: Option<Principal>`, making it easy to accidentally pass `None` or omit the field in a proposal payload.
- The test suite itself demonstrates the revert with `None` args as the expected behavior, indicating it is a known and accepted code path, not a guarded one.
- There is no on-chain check or warning that prevents an upgrade from silently reverting the authorized caller.
- The hardcoded default principal is a long-lived identity whose key material must remain accessible indefinitely to serve as the fallback, increasing the window for key compromise.

---

### Recommendation
1. **Persist the authorized caller to stable memory.** Serialize `AUTHORIZED_CALLER` in `pre_upgrade` and restore it in `post_upgrade` before applying any override from the upgrade argument. This ensures the previously configured caller is preserved across upgrades that do not explicitly change it.

2. **Add a runtime `set_authorized_caller` update method** callable only by the current authorized caller (or the canister's NNS-controlled controller), analogous to a `transferOwnership` function. This allows key rotation without requiring a full canister upgrade.

3. **Emit a certified log entry or canister event** whenever the authorized caller changes, so the change is auditable on-chain.

4. **Validate in `post_upgrade`** that if no `authorized_caller` override is supplied, the previously persisted value is used rather than the hardcoded default.

---

### Proof of Concept

**Scenario: Silent revert reinstates a previously-rotated-away default principal.**

1. The engine controller is deployed with a custom authorized caller `CUSTOM` via:
   ```
   EngineControllerInitArgs { authorized_caller: Some(CUSTOM), ... }
   ```
2. `CUSTOM` is the only principal that can call `create_engine`, `delete_engine`, etc.
3. An NNS upgrade proposal is submitted for the engine controller canister. The proposal payload encodes `None::<EngineControllerInitArgs>` (or `EngineControllerInitArgs { authorized_caller: None, ... }`).
4. After the upgrade, `post_upgrade` calls `apply_init_args(None)`, which executes:
   ```rust
   let authorized = args.authorized_caller.unwrap_or_else(default_authorized_caller);
   // → DEFAULT_AUTHORIZED_CALLER is now the authorized caller
   ```
5. `CUSTOM` is now rejected. `DEFAULT_AUTHORIZED_CALLER` is silently reinstated.
6. The holder of `DEFAULT_AUTHORIZED_CALLER`'s key can now call:
   ```
   create_engine({ node_ids: [...], subnet_admins: [], replica_version_id: "..." })
   ```
   which forwards directly to `registry.create_subnet`, creating an unauthorized engine subnet on the IC.

This is confirmed by the existing test at `rs/engine_controller/tests/tests.rs:416–432`:
```rust
// Upgrade with no override: default principal becomes authorized again.
pic.upgrade_canister(ENGINE_CONTROLLER_CANISTER_ID.into(), wasm.bytes(),
    Encode!(&None::<EngineControllerInitArgs>).unwrap(), Some(Principal::anonymous()))
    .await.expect("upgrade should succeed");

let err = call_create_engine(&pic, custom_caller, &args).await.unwrap_err();
assert!(err.contains("not authorized"), ...);
// → default principal is now authorized again

#[update]
async fn create_engine(args: CreateEngineArgs) -> Result<NewSubnet, String> {
    let caller = ensure_authorized()?;

    // Validate node list.
    if args.node_ids.len() < REQUIRED_NODE_COUNT {
        return Err(format!(
            "Expected at least {REQUIRED_NODE_COUNT} node ids, got {}",
            args.node_ids.len()
        ));
    }
    let mut seen: HashSet<Principal> = HashSet::new();
    for n in &args.node_ids {
        if !seen.insert(*n) {
            return Err(format!("Duplicate node id supplied: {n}"));
        }
    }

    // Make sure the caller is part of the subnet admins.
    let mut subnet_admins: Vec<PrincipalId> =
        args.subnet_admins.into_iter().map(PrincipalId).collect();
    let caller_pid = PrincipalId(caller);
    if !subnet_admins.contains(&caller_pid) {
        subnet_admins.push(caller_pid);
    }

    let node_ids: Vec<NodeId> = args
        .node_ids
        .into_iter()
        .map(|p| NodeId::from(PrincipalId(p)))
        .collect();

    let initial_dkg_subnet_id = INITIAL_DKG_SUBNET_ID.with(|c| *c.borrow());

    let payload = CreateSubnetPayload {
        node_ids,
        subnet_admins: Some(subnet_admins),
        replica_version_id: args.replica_version_id,
        subnet_type: SubnetType::CloudEngine,
        initial_dkg_subnet_id: Some(initial_dkg_subnet_id),
        dkg_interval_length: 499,
        dkg_dealings_per_block: 1,
        initial_notary_delay_millis: 300,
        max_block_payload_size: 4 * 1024 * 1024, // 4 MiB
        max_ingress_bytes_per_message: 2 * 1024 * 1024, // 2 MiB
        max_ingress_messages_per_block: 1000,
        unit_delay_millis: 1000,
        canister_cycles_cost_schedule: Some(CanisterCyclesCostSchedule::Free),
        features: SubnetFeatures {
            canister_sandboxing: false,
            http_requests: true,
            sev_enabled: Some(false),
        },
        ..Default::default()
    };

    let response: Result<NewSubnet, String> =
        Call::unbounded_wait(REGISTRY_CANISTER_ID.into(), "create_subnet")
            .with_arg(payload)
            .await
            .map_err(|e| format!("registry.create_subnet call failed: {e:?}"))?
            .candid()
            .map_err(|e| format!("Failed to decode registry response: {e}"))?;

    response
}
