### Title
Double-Minting of ICP Node Provider Rewards via Repeated `RewardNodeProviders` Proposal Execution - (`rs/nns/governance/src/governance.rs`)

### Summary
`mint_monthly_node_provider_rewards()` in NNS Governance does not enforce the monthly period constraint when invoked through a `RewardNodeProviders` governance proposal with `use_registry_derived_rewards = true`. Two such proposals adopted and executed within the same month will both successfully mint ICP to all node providers, violating the ledger conservation invariant.

### Finding Description

The `reward_node_providers_from_proposal` function calls `mint_monthly_node_provider_rewards()` directly when `use_registry_derived_rewards == Some(true)`: [1](#0-0) 

`mint_monthly_node_provider_rewards()` only guards against *concurrent* calls via a `thread_local!` lock: [2](#0-1) 

After the lock is released (on drop at function exit), the function mints rewards and updates the timestamp: [3](#0-2) 

The time-period guard `is_time_to_mint_monthly_node_provider_rewards()` is **only** checked in `run_periodic_tasks()`, not inside `mint_monthly_node_provider_rewards()` itself: [4](#0-3) 

The period check function: [5](#0-4) 

Because the lock is a `thread_local!` `RefCell<Option<u64>>` that is released (set to `None`) when `release_on_drop` is dropped at the end of each call, sequential proposal executions in separate canister messages each acquire the lock successfully. There is no persistent idempotency guard keyed on the reward period.

The `RewardNodeProviders` proposal type with `use_registry_derived_rewards`: [6](#0-5) 

### Impact Explanation

Two `RewardNodeProviders` proposals with `use_registry_derived_rewards = true` adopted and executed within the same calendar month will each independently call `mint_reward_to_neuron_or_account` for every registered node provider: [7](#0-6) 

Each execution mints fresh ICP from the governance minting account to node provider accounts or neurons. Node providers receive double their intended monthly ICP allocation. This is a direct ledger conservation violation — ICP supply is inflated beyond the protocol-intended monthly reward budget.

### Likelihood Explanation

The NNS uses liquid democracy where many neurons auto-follow known neurons, causing proposals to be adopted rapidly. A realistic scenario:

1. A `RewardNodeProviders` proposal (Proposal A) is submitted and adopted.
2. Due to execution uncertainty (e.g., the proposal shows `Adopted` but not yet `Executed` during the async inter-canister call phase), a second identical proposal (Proposal B) is submitted.
3. Proposal B is also adopted and executed.
4. Both proposals independently mint the full monthly reward to all node providers.

This mirrors the BribeVault scenario exactly: the admin (here, the NNS community) sends two transactions with the same calldata because the first appeared stuck. The NNS has no mechanism to prevent two `RewardNodeProviders` proposals from being adopted in the same month.

### Recommendation

Add the period check inside `mint_monthly_node_provider_rewards()` itself, not only in `run_periodic_tasks()`:

```rust
async fn mint_monthly_node_provider_rewards(&mut self) -> Result<(), GovernanceError> {
    // Enforce the monthly period regardless of call path.
    if !self.is_time_to_mint_monthly_node_provider_rewards() {
        return Err(GovernanceError::new_with_message(
            ErrorType::PreconditionFailed,
            "Monthly node provider rewards were already distributed recently.",
        ));
    }
    // ... existing lock and minting logic
}
```

Alternatively, add an analogous `prevTimestamp` field to the `RewardNodeProviders` proposal payload (analogous to the `prevUpdateCount` fix in the report) so that the proposal can only execute if the current `most_recent_monthly_node_provider_rewards.timestamp` matches the expected value at proposal creation time.

### Proof of Concept

1. At time T (within a reward month), NNS neuron holder submits `RewardNodeProviders { rewards: [], use_registry_derived_rewards: Some(true) }` — Proposal A.
2. Proposal A is adopted and executed. `mint_monthly_node_provider_rewards()` acquires the lock, fetches rewards, calls `reward_node_providers(...)` minting ICP to all node providers, then calls `update_most_recent_monthly_node_provider_rewards(...)` and releases the lock.
3. At time T+1 (same month), a second identical proposal — Proposal B — is submitted (e.g., due to confusion about whether Proposal A succeeded, since it was in `Adopted` state during async execution).
4. Proposal B is adopted and executed. `mint_monthly_node_provider_rewards()` is called again. The lock is `None` (released after step 2). No period check is performed. The function mints the full monthly reward to all node providers a second time.
5. All node providers receive 2× their intended monthly ICP reward. The ICP ledger total supply is inflated by one full month's node provider reward budget. [1](#0-0) [8](#0-7)

    async fn mint_reward_to_neuron_or_account(
        &mut self,
        np_principal: &PrincipalId,
        reward: &RewardNodeProvider,
    ) -> Result<(), GovernanceError> {
        let now = self.env.now();
        match reward.reward_mode.as_ref() {
            None => Err(GovernanceError::new_with_message(
                ErrorType::PreconditionFailed,
                "Reward node provider proposal must have a reward mode.",
            )),
            Some(RewardMode::RewardToNeuron(reward_to_neuron)) => {
                let to_subaccount = Subaccount(self.randomness.random_byte_array()?);
                let _block_height = self
                    .ledger
                    .transfer_funds(
                        reward.amount_e8s,
                        0, // Minting transfers don't pay transaction fees.
                        None,
                        neuron_subaccount(to_subaccount),
                        now,
                    )
                    .await?;
                let nid = self.neuron_store.new_neuron_id(&mut *self.randomness)?;
                let dissolve_delay_seconds = std::cmp::min(
                    reward_to_neuron.dissolve_delay_seconds,
                    max_dissolve_delay_seconds(),
                );

                let dissolve_state_and_age = if dissolve_delay_seconds > 0 {
                    DissolveStateAndAge::NotDissolving {
                        dissolve_delay_seconds,
                        aging_since_timestamp_seconds: now,
                    }
                } else {
                    DissolveStateAndAge::DissolvingOrDissolved {
                        when_dissolved_timestamp_seconds: now,
                    }
                };

                // Transfer successful.
                let neuron = NeuronBuilder::new(
                    nid,
                    to_subaccount,
                    *np_principal,
                    dissolve_state_and_age,
                    now,
                )
                .with_followees(self.heap_data.default_followees.clone())
                .with_cached_neuron_stake_e8s(reward.amount_e8s)
                .with_kyc_verified(true)
                .build();

                self.add_neuron(nid.id, neuron)
            }
            Some(RewardMode::RewardToAccount(reward_to_account)) => {
                // We are not creating a neuron, just transferring funds.
                let to_account = match &reward_to_account.to_account {
                    Some(to_account) => AccountIdentifier::try_from(to_account).map_err(|e| {
                        GovernanceError::new_with_message(
                            ErrorType::InvalidCommand,
                            format!("The recipient's subaccount is invalid due to: {e}"),
                        )
                    })?,
                    None => AccountIdentifier::new(*np_principal, None),
                };

                self.ledger
                    .transfer_funds(
                        reward.amount_e8s,
                        0, // Minting transfers don't pay transaction fees.
                        None,
                        to_account,
                        now,
                    )
                    .await
                    .map(|_| ())
                    .map_err(|e| {
                        GovernanceError::new_with_message(
                            ErrorType::PreconditionFailed,
                            format!(
                                "Couldn't perform minting transfer: {}",
                                GovernanceError::from(e)
                            ),
                        )
                    })
            }
        }
    }

    async fn mint_monthly_node_provider_rewards(&mut self) -> Result<(), GovernanceError> {
        // Return immediately if another call is already in progress.
        thread_local! {
            static LOCK: RefCell<Option<u64>> = const { RefCell::new(None) };
        }
        let release_on_drop = ic_nervous_system_lock::acquire(&LOCK, self.env.now());
        if let Err(earlier_call_start_timestamp) = release_on_drop {
            // Log, but not too frequently (at most once every 5 minutes).
            thread_local! {
                static LAST_LOGGED_UNAVAILABLE_TIMESTAMP_SECONDS: RefCell<u64> = const { RefCell::new(0) };
            }
            let time_since_logged_seconds = LAST_LOGGED_UNAVAILABLE_TIMESTAMP_SECONDS
                .with(|t| self.env.now().saturating_sub(*t.borrow()));
            if time_since_logged_seconds > 5 * 60 {
                println!(
                    "{}Another mint_monthly_node_provider_rewards call (started at \
                     {} seconds since the UNIX epoch) is already in progress.",
                    LOG_PREFIX, earlier_call_start_timestamp,
                );
                LAST_LOGGED_UNAVAILABLE_TIMESTAMP_SECONDS.with(|t| {
                    *t.borrow_mut() = self.env.now();
                });
            }

            return Ok(());
        }

        let monthly_node_provider_rewards = if are_performance_based_rewards_enabled() {
            self.get_node_providers_rewards().await?
        } else {
            self.get_monthly_node_provider_rewards().await?
        };

        let _ = self
            .reward_node_providers(&monthly_node_provider_rewards.rewards)
            .await;
        self.update_most_recent_monthly_node_provider_rewards(monthly_node_provider_rewards);

        // Commit the minting status by making a canister call.
        let _unused_canister_status_response = self
            .env
            .call_canister_method(
                GOVERNANCE_CANISTER_ID,
                "get_build_metadata",
                Encode!().unwrap_or_default(),
            )
            .await;

        Ok(())
    }
