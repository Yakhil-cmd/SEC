### Title
SNS Governance `max_number_of_neurons` Hard Cap Exhaustion Enables Neuron Creation DOS - (File: `rs/sns/governance/src/governance.rs`)

---

### Summary

SNS Governance enforces a hard cap on the total number of neurons via `max_number_of_neurons`. There is no rate limiting on neuron creation (unlike NNS Governance). An attacker holding sufficient SNS tokens can fill all available neuron slots with minimum-stake, zero-dissolve-delay neurons, permanently blocking legitimate users from creating neurons until the attacker voluntarily disbursed. The capital cost is bounded by `max_number_of_neurons × neuron_minimum_stake_e8s` and is fully recoverable, making the attack sustainable at near-zero net cost.

---

### Finding Description

`check_neuron_population_can_grow()` in SNS Governance enforces a strict integer ceiling:

```rust
if (self.proto.neurons.len() as u64) + 1 > max_number_of_neurons {
    return Err(GovernanceError::new_with_message(
        ErrorType::PreconditionFailed,
        "Cannot add neuron. Max number of neurons reached.",
    ));
}
``` [1](#0-0) 

This check is called unconditionally from `add_neuron()`, which is invoked by `claim_neuron()`:

```rust
// This also verifies that there are not too many neurons already.
self.add_neuron(neuron.clone())?;
``` [2](#0-1) 

The `claim_neuron` flow allows a neuron to be created with `dissolve_delay = 0`, placing it immediately in the `Dissolved` state:

```rust
dissolve_state: Some(DissolveState::DissolveDelaySeconds(0)),
``` [3](#0-2) 

A dissolved neuron can be immediately disbursed via `disburse_neuron()`, which requires only `NeuronState::Dissolved`:

```rust
if state != NeuronState::Dissolved {
    return Err(GovernanceError::new_with_message(
        ErrorType::PreconditionFailed,
        format!("Neuron {id} is NOT dissolved. It is in state {state:?}"),
    ));
}
``` [4](#0-3) 

The ceiling constant is `MAX_NUMBER_OF_NEURONS_CEILING = 200_000`: [5](#0-4) 

Unlike NNS Governance, which has `MAX_SUSTAINED_NEURONS_PER_HOUR = 15` and a spike cap of 300: [6](#0-5) 

SNS Governance has **no rate limiting** on `claim_neuron`. There is no per-principal neuron count limit, no time-based throttle, and no minimum dissolve delay required to claim a neuron.

---

### Impact Explanation

Once `max_number_of_neurons` is reached, every subsequent `ClaimOrRefresh` call from any principal returns `PreconditionFailed: "Cannot add neuron. Max number of neurons reached."` This is confirmed by the integration test: [7](#0-6) 

Legitimate token holders cannot stake or participate in SNS governance. Voting power is frozen at the attacker-controlled distribution. The SNS community cannot pass governance proposals to raise `max_number_of_neurons` because the neurons needed to vote on such a proposal may themselves be blocked from creation.

---

### Likelihood Explanation

The attack cost is:
- **Capital**: `max_number_of_neurons × neuron_minimum_stake_e8s` tokens, held in dissolved neurons. This capital is **fully recoverable** by disbursing.
- **Transaction fees**: `max_number_of_neurons × transaction_fee_e8s` to create neurons (one-time, non-recoverable).

For an SNS with `max_number_of_neurons = 10_000` and `neuron_minimum_stake_e8s = 1_000` (1000 e8s), the non-recoverable cost is only `10_000 × transaction_fee_e8s`. The locked capital is returned on disbursal. An attacker can maintain the DOS indefinitely at near-zero ongoing cost, since the capital is not destroyed. The attack is feasible for any party that can acquire the SNS token in quantity, including a competing protocol or a large token holder.

---

### Recommendation

1. **Impose a minimum dissolve delay for neuron creation** (analogous to the report's recommendation of a minimum delegation lifetime). Neurons with `dissolve_delay = 0` contribute no governance value and should not be allowed to occupy neuron slots. Require `dissolve_delay >= neuron_minimum_dissolve_delay_to_vote_seconds` at claim time, or at minimum a non-zero floor.

2. **Add per-principal neuron creation rate limiting** in SNS Governance, mirroring the NNS `MAX_SUSTAINED_NEURONS_PER_HOUR` mechanism. [8](#0-7) 

3. **Raise `neuron_minimum_stake_e8s`** to a value that makes filling `max_number_of_neurons` slots economically prohibitive relative to the SNS token's market value.

---

### Proof of Concept

```
Attacker controls principal P with sufficient SNS tokens.

1. For i in 0..max_number_of_neurons:
   a. Transfer neuron_minimum_stake_e8s tokens to governance subaccount(P, nonce=i)
   b. Call manage_neuron { ClaimOrRefresh { MemoAndController { memo: i, controller: P } } }
   c. Neuron is created with dissolve_delay=0 (immediately Dissolved state)

2. Neuron count == max_number_of_neurons.

3. Legitimate user L calls manage_neuron { ClaimOrRefresh { ... } }
   → Returns: GovernanceError { PreconditionFailed, "Cannot add neuron. Max number of neurons reached." }

4. Attacker recovers capital:
   For each neuron_id:
     Call manage_neuron { Disburse { amount: None, to_account: P } }
     → Tokens returned to P minus transaction_fee_e8s per neuron.

Net cost to attacker: max_number_of_neurons × transaction_fee_e8s (non-recoverable).
Net effect: SNS neuron creation blocked for all legitimate users during attack window.
```

fn test_claim_neuron_fails_when_max_number_of_neurons_is_reached() {
    local_test_on_sns_subnet(|runtime| async move {
        // Set up an SNS with a ledger account for two users
        let user1 = UserInfo::new(Sender::from_keypair(&TEST_USER1_KEYPAIR));
        let user2 = UserInfo::new(Sender::from_keypair(&TEST_USER2_KEYPAIR));
        let alloc = Tokens::from_tokens(1000).unwrap();

        let sys_params = NervousSystemParameters {
            neuron_claimer_permissions: Some(NeuronPermissionList {
                permissions: NeuronPermissionType::all(),
            }),
            max_number_of_neurons: Some(1),
            ..NervousSystemParameters::with_default_values()
        };

        let sns_init_payload = SnsTestsInitPayloadBuilder::new()
            .with_ledger_account(user1.sender.get_principal_id().0.into(), alloc)
            .with_ledger_account(user2.sender.get_principal_id().0.into(), alloc)
            .with_nervous_system_parameters(sys_params.clone())
            .build();

        let sns_canisters = SnsCanisters::set_up(&runtime, sns_init_payload).await;

        // Successfully STAKE and CLAIM user1's neuron, reaching the configured maximum number of
        // neurons allowed in the system
        sns_canisters
            .stake_and_claim_neuron(&user1.sender, Some(ONE_YEAR_SECONDS as u32))
            .await;

        // Only STAKE for user2.
        sns_canisters
            .stake_neuron_account(&user2.sender, &user2.subaccount, 1)
            .await;

        let manage_neuron_command = ManageNeuron {
            subaccount: user2.subaccount.to_vec(),
            command: Some(Command::ClaimOrRefresh(ClaimOrRefresh {
                by: Some(By::MemoAndController(MemoAndController {
                    memo: NONCE,
                    controller: None,
                })),
            })),
        };

        // Try claiming the Neuron for user2 (via memo and controller). This should fail due to
        // the max_number_of_neurons being reached.
        let response: ManageNeuronResponse = sns_canisters
            .governance
            .update_from_sender(
                "manage_neuron",
                candid_one,
                manage_neuron_command.clone(),
                &user2.sender,
            )
            .await
            .expect("Error calling the manage_neuron api.");

        // assert that the error_type is PreconditionFailed.
        let error = match response.command.unwrap() {
            CommandResponse::Error(error) => error,
            CommandResponse::ClaimOrRefresh(_) => {
                panic!(
                    "User should not have been able to claim a neuron due to reaching max_number_of_neurons"
                )
            }
            _ => panic!("Unexpected command response when claiming neuron."),
        };
        assert_eq!(error.error_type, ErrorType::PreconditionFailed as i32);
