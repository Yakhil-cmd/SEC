### Title
Partial-Failure in `AccountState::transfer` Permanently Locks GTC Neurons on Transient Governance Errors — (`rs/nns/gtc/src/lib.rs`)

### Summary
`AccountState::transfer` unconditionally removes every neuron from `neuron_ids` and always returns `Ok(())` regardless of per-neuron `transfer_gtc_neuron` failures. `Gtc::donate_account` then sets `has_donated = true` on that `Ok(())`. Any neuron whose governance call failed is moved to `failed_transferred_neurons`, still exists in governance under GTC control, but is permanently unreachable: `has_donated = true` blocks every future `claim_neurons` and `donate_account` call on the account.

### Finding Description

**Step 1 — `transfer()` swallows per-neuron errors** [1](#0-0) 

For every `neuron_id` in the cloned list:
- `self.neuron_ids.retain(|id| id != &neuron_id)` is called **unconditionally** at line 192, whether the governance call succeeded or failed.
- On failure the neuron is pushed to `failed_transferred_neurons` (line 204), not back into `neuron_ids`.
- The function returns `Ok(())` at line 209 regardless of how many neurons failed.

**Step 2 — `donate_account` sets `has_donated` unconditionally** [2](#0-1) 

Because `transfer()` never returns `Err`, the `?` on line 89 never short-circuits, and `has_donated = true` is always written.

**Step 3 — `has_donated` permanently blocks all recovery paths** [3](#0-2) [4](#0-3) 

Both `claim_neurons` and `transfer` (called by `donate_account`) gate on `has_donated`. Once set, neither can be called again.

**Step 4 — Failed neurons remain in governance under GTC control**

`transfer_gtc_neuron` in governance only removes the donor neuron after a successful ledger transfer: [5](#0-4) 

If the ledger transfer fails (line 1911 `?`), the donor neuron is **not** removed. It persists in governance with the GTC canister as controller, but no GTC method can ever touch it again.

**Step 5 — No admin recovery path exists**

The GTC canister's public interface exposes no method to retry `failed_transferred_neurons`: [6](#0-5) 

### Impact Explanation
Any GTC account whose `donate_account` call experiences a partial governance failure (even one neuron out of N) ends up with:
- `has_donated = true` — account permanently locked
- `neuron_ids` empty — no record of what to retry
- Failed neurons in `failed_transferred_neurons` — still in governance under GTC control, inaccessible to anyone

The ICP staked in those neurons is permanently frozen. Neither the original owner nor the custodian can ever access it.

### Likelihood Explanation
The trigger is a transient error from governance or the ledger during a `donate_account` call. Realistic causes include:
- A ledger transfer failing for one neuron (e.g., insufficient cached stake after a fee change, or a transient ledger reject)
- Governance canister being mid-upgrade when the Nth inter-canister call is dispatched
- Message queue saturation causing a reject for a subset of the sequential calls

An account owner with many neurons (SR accounts have 48 neurons, ECT accounts have 12) makes 48 or 12 sequential inter-canister calls; the probability of at least one transient failure is non-negligible.

### Recommendation
1. **`transfer()` should return `Err` if any neuron fails**, or at minimum propagate a partial-failure signal back to `donate_account`.
2. **`donate_account` should only set `has_donated = true` when `failed_transferred_neurons` is empty** after the loop.
3. Alternatively, keep failed neurons in `neuron_ids` so a retry is possible, and only set `has_donated` once all neurons have been successfully transferred.
4. Add a privileged admin method to retry `failed_transferred_neurons` for a given account.

### Proof of Concept
```
1. Create a GTC account with 3 neurons [N1, N2, N3].
2. Mock governance to succeed for N1, fail for N2, succeed for N3.
3. Call donate_account.
4. transfer() loop:
     N1 → Ok  → successfully_transferred_neurons, removed from neuron_ids
     N2 → Err → failed_transferred_neurons,       removed from neuron_ids
     N3 → Ok  → successfully_transferred_neurons, removed from neuron_ids
   transfer() returns Ok(()).
5. donate_account sets has_donated = true.
6. N2 still exists in governance under GTC control.
7. Call claim_neurons  → Err("Account has previously donated its funds")
8. Call donate_account → transfer() returns Err("Account has already donated its funds")
9. N2's ICP is permanently inaccessible.
```
