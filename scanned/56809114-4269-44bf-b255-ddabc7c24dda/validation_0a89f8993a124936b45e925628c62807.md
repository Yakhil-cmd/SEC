### Title
Missing Anonymous Principal Validation in `disburse_maturity` Destination - (File: `rs/nns/governance/src/governance/disburse_maturity.rs`)

### Summary
The NNS governance `disburse_maturity` flow does not validate that the `to_account` owner is not the anonymous principal. A neuron controller who specifies the anonymous principal as the disbursement recipient will permanently lose their ICP maturity after the 7-day disbursement delay, with no ability to cancel once initiated.

### Finding Description
In `rs/nns/governance/src/governance/disburse_maturity.rs`, the `Destination::try_new` function validates that `Account.owner` is not `None` and that the subaccount is 32 bytes, but it does **not** check whether `owner` is the anonymous principal (`2vxsx-fae`). [1](#0-0) 

The `Icrc1Account::try_from` conversion only rejects a missing owner: [2](#0-1) 

`Destination::try_new` calls `try_into_account_identifier` for validation, which in turn calls `Icrc1Account::try_from`.

impl TryFrom<Account> for Icrc1Account {
    type Error = String;

    fn try_from(account: Account) -> Result<Self, Self::Error> {
        let Account { owner, subaccount } = account;
        let Some(owner) = owner else {
            return Err("Owner is required".to_string());
        };
        let subaccount: Option<[u8; 32]> = subaccount
            .map(|s| s.subaccount.try_into())
            .transpose()
            .map_err(|_| "Subaccount must be 32 bytes".to_string())?;

        Ok(Icrc1Account {
            owner: owner.0,
            subaccount,
        })
    }
}
