### Title
Missing New-Owner Validation in `do_update_node_operator_config_directly` Allows Unauthorized Node Operator Record Ownership Transfer - (File: `rs/registry/canister/src/mutations/do_update_node_operator_config_directly.rs`)

---

### Summary

`do_update_node_operator_config_directly` verifies that the caller equals the current `node_provider_principal_id` of a `NodeOperatorRecord`, but never validates that the new `node_provider_id` supplied in the payload also equals the caller. Any registered node provider can therefore silently transfer ownership of their node operator record ŌĆö and the ICP node rewards attributed to it ŌĆö to any arbitrary principal without that principal's consent.

---

### Finding Description

The canister entry point `update_node_operator_config_directly` is explicitly open to any caller: [1](#0-0) 

Inside the implementation, the only authorization check is:

```rust
// 2. Make sure that the caller is authorized to make the requested changes to node_operator_record.
if caller != PrincipalId::try_from(&node_operator_record.node_provider_principal_id).unwrap() {
    return Err(format!(
        "Caller {caller} not equal to the node_provider_princpal_id for this record."
    ));
}
``` [2](#0-1) 

After passing this check, the function unconditionally writes the caller-supplied `payload.node_provider_id` into the record:

```rust
node_operator_record.node_provider_principal_id = node_provider_id.to_vec();
``` [3](#0-2) 

There is no check that `payload.node_provider_id == caller`. The only other guard is that `node_provider_id != node_operator_id`, which is unrelated to ownership. [4](#0-3) 

The `node_provider_principal_id` field in `NodeOperatorRecord` is the direct input to the node reward calculation pipeline: [5](#0-4) 

Monthly ICP rewards (minted tokens) are distributed to whichever principal is stored in that field at reward time: [6](#0-5) 

---

### Impact Explanation

A malicious node provider (Alice) who controls a `NodeOperatorRecord` can call `update_node_operator_config_directly` with `node_provider_id = <victim_principal>`. After the call:

1. The victim's principal becomes the `node_provider_principal_id` for Alice's node operator record.
2. All future ICP node rewards for Alice's nodes are minted and sent to the victim's account ŌĆö the victim did not consent and may not be aware.
3. Alice can use this to offload a poorly-performing node operator record (low uptime, penalized nodes) onto a victim, reducing the victim's aggregate reward rate.
4. The victim cannot prevent this; they can only undo it by calling the same function themselves (if they discover the change).
5. Alice can also use this to exit her node operator obligations by transferring the record to an unsuspecting principal, leaving that principal associated with nodes they do not control.

The `node_provider_principal_id` is the sole field used to route minted ICP rewards, making this a direct, on-chain financial redirection primitive available to any registered node provider. [7](#0-6) 

---

### Likelihood Explanation

- The entry point is explicitly open to any caller with no pre-consensus rejection.
- Any existing node provider (a real-world role with known principals on mainnet) can trigger this with a single update call.
- No privileged access, governance majority, or key compromise is required.
- The attack is silent: the victim receives no notification and the registry change is not surfaced to them directly.

---

### Recommendation

Add a check that the new `node_provider_id` in the payload equals the caller, mirroring the pattern used in `do_migrate_node_operator_directly`:

```rust
// After the existing caller == current_node_provider check:
if node_provider_id != caller {
    return Err(format!(
        "New node_provider_id {node_provider_id} must equal the caller {caller}."
    ));
}
```

This ensures a node provider can only set themselves as the new provider (a no-op) or that any transfer requires a governance proposal via `update_node_operator_config` (which is restricted to the governance canister). [8](#0-7) 

---

### Proof of Concept

**Setup:**
- Alice is a registered node provider with `node_provider_principal_id = Alice` on a `NodeOperatorRecord` for `node_operator_id = NO_1`.
- Bob is any arbitrary principal (may or may not be a registered node provider).

**Attack:**
```
Alice calls registry canister:
  update_node_operator_config_directly({
    node_operator_id: Some(NO_1),
    node_provider_id: Some(Bob),   // ŌåÉ arbitrary victim principal
  })
```

**Result:**
1. Check passes: `Alice == current node_provider_principal_id` Ō£ō
2. No check on `Bob == Alice` ŌĆö missing.
3. Registry writes `node_provider_principal_id = Bob` for `NO_1`.
4. At next monthly reward cycle, governance reads `node_provider_principal_id = Bob` from the registry and mints ICP rewards to Bob's account.
5. Alice's nodes' rewards are permanently redirected to Bob without Bob's knowledge or consent. [9](#0-8)

    fn do_update_node_operator_config_directly_(
        &mut self,
        payload: UpdateNodeOperatorConfigDirectlyPayload,
        caller: PrincipalId,
        now: SystemTime,
    ) -> Result<(), String> {
        println!("{LOG_PREFIX}do_update_node_operator_config_directly: {payload:?}");

        // 1. Look up the record of the requested target NodeOperatorRecord.
        let node_operator_id = payload
            .node_operator_id
            .ok_or("No Node Operator specified in the payload".to_string())?;

        let node_operator_record_key = make_node_operator_record_key(node_operator_id).into_bytes();
        let node_operator_record_vec = &self
            .get(&node_operator_record_key, self.latest_version())
            .ok_or(format!(
                "Node Operator record with ID {node_operator_id} not found in the registry."
            ))?
            .value;

        let mut node_operator_record =
            NodeOperatorRecord::decode(node_operator_record_vec.as_slice())
                .map_err(|e| format!("{e:?}"))?;

        // 2. Make sure that the caller is authorized to make the requested changes to node_operator_record.
        if caller
            != PrincipalId::try_from(&node_operator_record.node_provider_principal_id).unwrap()
        {
            return Err(format!(
                "Caller {caller} not equal to the node_provider_princpal_id for this record."
            ));
        }

        // 3. Check Rate Limits
        let current_node_provider = caller;
        let reservation =
            self.try_reserve_capacity_for_node_provider_operation(now, current_node_provider, 1)?;

        // 4. Check that the Node Provider is not being set with the same ID as the Node Operator
        let node_provider_id = payload
            .node_provider_id
            .ok_or("No Node Provider specified in the payload".to_string())?;

        if node_provider_id == node_operator_id {
            return Err(format!(
                "The Node Operator ID cannot be the same as the Node Provider ID: {node_operator_id}"
            ));
        }

        node_operator_record.node_provider_principal_id = node_provider_id.to_vec();

        // 5. Set and execute the mutation
        let mutations = vec![RegistryMutation {
            mutation_type: registry_mutation::Type::Update as i32,
            key: node_operator_record_key,
            value: node_operator_record.encode_to_vec(),
        }];

        // Check invariants before applying mutations
        self.maybe_apply_mutation_internal(mutations);

        if let Err(e) = self.commit_used_capacity_for_node_provider_operation(now, reservation) {
            println!("{LOG_PREFIX}Error committing Rate Limit usage: {e}");
        }

        Ok(())
    }
