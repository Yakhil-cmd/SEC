### Title
Missing Minimum Cycles Output Protection in ICP-to-Cycles Conversion — (`File: rs/nns/cmc/src/main.rs`)

### Summary

The Cycles Minting Canister (CMC) exposes three public update endpoints — `notify_top_up`, `notify_mint_cycles`, and `notify_create_canister` — that convert a committed ICP ledger payment into cycles using the CMC's internally stored `icp_xdr_conversion_rate`. None of these endpoints accept a caller-supplied minimum cycles floor. Because the ICP transfer (step 1) and the notify call (step 2) are two separate, non-atomic operations, the ICP/XDR rate can change between them, and the caller has no on-chain mechanism to abort or bound the loss.

### Finding Description

The ICP-to-cycles conversion flow is a two-step commit-then-convert protocol:

1. The user sends ICP to a CMC subaccount on the ICP ledger (irreversible once included in a block).
2. The user calls `notify_top_up` / `notify_mint_cycles` / `notify_create_canister` to trigger the conversion.

At step 2, `tokens_to_cycles` reads the CMC's current `state.icp_xdr_conversion_rate` and applies it unconditionally:

```rust
fn tokens_to_cycles(amount: Tokens) -> Result<Cycles, NotifyError> {
    with_state(|state| {
        let xdr_permyriad_per_icp = state
            .icp_xdr_conversion_rate
            .as_ref()
            .map(|rate| rate.xdr_permyriad_per_icp);
        match xdr_permyriad_per_icp {
            Some(xdr_permyriad_per_icp) => Ok(TokensToCycles {
                xdr_permyriad_per_icp,
                cycles_per_xdr: state.cycles_per_xdr,
            }
            .to_cycles(amount)),
            ...
        }
    })
}

### Impact Explanation

A user who:
1. Queries `get_icp_xdr_conversion_rate` and observes a favorable rate,
2. Sends ICP to the CMC subaccount (committed, irreversible),
3. Calls `notify_top_up` / `notify_mint_cycles` after the rate has dropped,

will receive substantially fewer cycles than anticipated with no recourse. The ICP is burned regardless of the resulting cycle count. There is no on-chain mechanism for the caller to express "give me at least X cycles or refund me." The `NotifyError::Refunded` path is only triggered by deposit/deposit-target failures, not by an unfavorable rate.

**Cycles/resource accounting bug**: users permanently lose ICP value (converted to fewer cycles than expected) with no ability to set a minimum acceptable output.

### Likelihood Explanation

- The ICP/XDR rate is updated every 5 minutes by the exchange rate canister. In volatile market conditions, a 5–10% rate swing within a single update window is realistic.
- Any unprivileged ingress sender who uses the two-step CMC flow is exposed. This is the standard, documented path for all cycle purchases.
- No adversary needs to manipulate the rate; ordinary market movement between step 1 and step 2 is sufficient to cause material loss.
- The window is bounded (5 minutes) but not zero, and the ICP transfer in step 1 has its own finality latency.

### Recommendation

Add an optional `min_cycles: Option<Nat>` (or `min_cycles: Option<u128>`) field to `NotifyMintCyclesArg`, `NotifyTopUp`, and `NotifyCreateCanister`. In `process_mint_cycles` / `process_top_up` / `process_create_canister`, after computing `cycles = tokens_to_cycles(amount)`, check:

```rust
if let Some(min) = min_cycles {
    if cycles < min {
        // refund ICP and return NotifyError::Refunded
    }
}
```

This mirrors the `minReceivedAmount` mitigation recommended in the original report and gives callers a slippage floor without breaking backward compatibility (the field is optional).

### Proof of Concept

1. User calls `get_icp_xdr_conversion_rate` → observes rate R (e.g., 50,000 XDR/ICP → 5T cycles/ICP).
2. User sends 10 ICP to `CMC_PRINCIPAL / subaccount(user)` on the ICP ledger. Transfer is finalized.
3. Exchange rate canister heartbeat fires; NNS governance proposal or XRC update drops rate to R' = 25,000 XDR/ICP (2.5T cycles/ICP).
4. User calls `notify_mint_cycles { block_index, to_subaccount: None, deposit_memo: None }`.
5. `tokens_to_cycles(10 ICP)` uses R' → mints 25T cycles instead of the expected 50T cycles.
6. ICP is burned; user receives 50% fewer cycles than anticipated with no recourse.

The two-step flow is the only supported path for `notify_mint_cycles` and `notify_top_up`: [8](#0-7) [9](#0-8)

/// The periodic task for collecting the ICP/XDR rate from the Exchange Rate Canister.
/// To avoid having multiple calls sent to the Exchange Rate Canister,
/// this function contains a guard to ensure multiple calls cannot be made until
/// the prior call is complete.
pub async fn update_exchange_rate(
    safe_state: &'static LocalKey<RefCell<Option<State>>>,
    env: &impl Environment,
    xrc_client: &impl ExchangeRateCanisterClient,
) -> Result<(), UpdateExchangeRateError> {
    let now_timestamp_seconds = env.now_timestamp_seconds();
    let current_minute_seconds =
        round_down_to_multiple_of(now_timestamp_seconds, ONE_MINUTE_SECONDS);

    UpdateExchangeRateGuard::with_guard(safe_state, current_minute_seconds, async {
        let call_xrc_result = xrc_client.get_icp_to_xdr_exchange_rate(None).await;
        // Check if updating the rate via the exchange rate canister was disabled while retrieving the rate.
        // If it has, exit early.
        let is_updating_rate_disabled = read_state(safe_state, |state| {
            state
                .update_exchange_rate_canister_state
                .unwrap_or_default()
                == UpdateExchangeRateState::Disabled
        });
        if is_updating_rate_disabled {
            return Err(UpdateExchangeRateError::Disabled);
        }

        match call_xrc_result {
            Ok(exchange_rate) => {
                validate_exchange_rate(&exchange_rate)
                    .map_err(|error| UpdateExchangeRateError::InvalidRate(error.to_string()))?;
                let icp_xdr_conversion_rate = IcpXdrConversionRate::from(exchange_rate);
                if let Err(error) =
                    do_set_icp_xdr_conversion_rate(safe_state, env, icp_xdr_conversion_rate)
                {
                    return Err(UpdateExchangeRateError::FailedToSetRate(error));
                }
            }
            Err(error) => {
                return Err(UpdateExchangeRateError::FailedToRetrieveRate(
                    error.to_string(),
                ));
            }
        };

        Ok(())
    })
    .await
