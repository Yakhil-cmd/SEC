### Title
Stale Maturity Modulation Applied to ICP Minting Without Freshness Check — (File: `rs/nns/governance/src/governance/disburse_maturity.rs`, `rs/nns/governance/src/governance.rs`)

---

### Summary

`try_finalize_maturity_disbursement` and `maybe_spawn_neurons` in NNS Governance read `current_value_permyriad` from the cached `MaturityModulation` struct and apply it to compute the ICP amount to mint, without ever checking `updated_at_days_since_epoch` to verify the modulation is recent. If the `UpdateIcpXdrRateRelatedData` daily timer is delayed or the Exchange Rate Canister (XRC) is persistently unavailable, the cached modulation can be arbitrarily stale at the moment of settlement, causing incorrect ICP to be minted for all pending disbursements and neuron spawns.

---

### Finding Description

The NNS Governance canister introduced a new Mission 70 maturity modulation system (Proposal 141779). A daily timer task (`UpdateIcpXdrRateRelatedData`) fetches ICP/XDR rates from XRC, maintains a 365-day price history, and computes a maturity modulation value stored in `heap_data.maturity_modulation.current_value_permyriad`. [1](#0-0) 

This value is consumed at settlement time in two places:

**1. `try_finalize_maturity_disbursement`** — reads `current_value_permyriad` with no freshness check: [2](#0-1) 

It then passes this directly to `apply_maturity_modulation` to compute the ICP to mint: [3](#0-2) 

**2. `maybe_spawn_neurons`** — reads `current_value_permyriad` with no freshness check: [4](#0-3) 

And applies it to mint neuron stake: [5](#0-4) 

The `MaturityModulation` struct carries `updated_at_days_since_epoch` precisely to track when the value was last computed: [6](#0-5) 

Neither settlement path checks this field before using `current_value_permyriad`. If the daily timer fires late (e.g., XRC is offline for multiple days, or the timer task itself encounters a persistent error), the cached modulation reflects a price from days or weeks ago, not the price at the time of settlement.

The disbursement delay is 7 days: [7](#0-6) 

A user initiates disbursement at time T. The modulation is locked in at time T+7 days (finalization), using whatever `current_value_permyriad` happens to be cached at that moment — which may reflect ICP prices from days before T+7 if the timer was delayed.

---

### Impact Explanation

When the maturity modulation is stale:

- **Overbonding (positive stale modulation)**: If `current_value_permyriad` is stuck at the maximum of +200 permyriad (+2%) while the actual ICP price has since dropped, the protocol mints 2% more ICP than the current price warrants — a direct ICP supply inflation loss to the protocol.
- **Underbonding (negative stale modulation)**: If `current_value_permyriad` is stuck at the minimum of -1000 permyriad (-10%) while the actual ICP price has since risen, users receive 10% less ICP than they are entitled to — a direct financial loss to neuron owners.

The modulation range is bounded at `[-1000, +200]` permyriad: [8](#0-7) 

For large maturity disbursements (e.g., large NNS neuron holders), a 10% error represents a material ICP amount. The error affects all neurons whose finalization window coincides with the stale period.

---

### Likelihood Explanation

The `UpdateIcpXdrRateRelatedData` timer depends on XRC availability. The changelog explicitly notes that XRC failures were a real operational concern: [9](#0-8) 

If XRC is unavailable for more than 7 days (the disbursement delay), every maturity disbursement finalized during that window uses a modulation that does not reflect the current ICP price. The IC network has experienced outages historically, and XRC is an external dependency. The timer's error retry interval is only 60 seconds, meaning a persistent XRC failure will leave the modulation frozen at its last computed value indefinitely. [10](#0-9) 

Any unprivileged neuron owner can trigger this path by calling `disburse_maturity` or `spawn_neuron` — no special privileges required.

---

### Recommendation

Before applying `current_value_permyriad` in `try_finalize_maturity_disbursement` and `maybe_spawn_neurons`, check `updated_at_days_since_epoch` against `now / ONE_DAY_SECONDS`. If the modulation is older than a configurable threshold (e.g., 2–3 days), either:

1. **Block finalization** and retry later (conservative), or
2. **Fall back to a neutral modulation of 0** (safe default already used at init), or
3. **Log a warning and proceed** with the stale value, accepting the design tradeoff explicitly.

The `MaturityModulation` struct already carries the necessary timestamp to implement this check: [11](#0-10) 

---

### Proof of Concept

1. Neuron owner calls `disburse_maturity` at time T. Disbursement is recorded with `finalize_disbursement_timestamp_seconds = T + 7 days`.
2. XRC becomes unavailable at time T+1. `UpdateIcpXdrRateRelatedData` fails on every tick; `current_value_permyriad` is frozen at its last computed value (e.g., +200 permyriad, reflecting a high ICP price).
3. ICP price drops 20% over the next 7 days while XRC remains offline.
4. At time T+7 days, `FinalizeMaturityDisbursementsTask` fires. `try_finalize_maturity_disbursement` reads `current_value_permyriad = 200` (stale, reflecting the old high price) with no freshness check.
5. `apply_maturity_modulation(original_maturity_e8s, 200)` mints 2% more ICP than the current price warrants, inflating ICP supply at the protocol's expense.
6. Simultaneously, `maybe_spawn_neurons` applies the same stale +200 permyriad to all neurons ready to spawn, compounding the inflation.

The entry path is fully unprivileged: any neuron owner on mainnet can initiate a disbursement, and the finalization is automatic. [12](#0-11) [13](#0-12) [14](#0-13)

fn update_maturity_modulation(
    icp_price_history: &IcpPriceHistory,
    maturity_modulation: &mut MaturityModulation,
    current_day: u64,
) {
    if maturity_modulation.updated_at_days_since_epoch == Some(current_day) {
        return;
    }

    let previous = match (
        maturity_modulation.current_value_permyriad,
        maturity_modulation.updated_at_days_since_epoch,
    ) {
        (Some(p), Some(d)) => Some((p as i64, d)),
        _ => None,
    };

    match compute_maturity_modulation_permyriad(
        &icp_price_history.icp_xdr_rates,
        current_day,
        previous,
    ) {
        Ok(new_permyriad) => {
            maturity_modulation.current_value_permyriad = Some(new_permyriad as i32);
            maturity_modulation.updated_at_days_since_epoch = Some(current_day);
        }
        Err(reason) => {
            // Reaches this branch only when the buffer has no rate at or before any day in the
            // recent window (e.g., a fresh canister where every backfill fetch has failed so far,
            // or every fetched rate was zero). Log and leave the prior modulation untouched —
            // subsequent rounds will retry the missing days.
            println!(
                "{}update_maturity_modulation: skipping update: {}; leaving prior modulation \
                 unchanged",
                LOG_PREFIX, reason
            );
        }
    }
}
