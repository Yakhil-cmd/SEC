### Title
SNS Governance Lacks Voting Power Snapshot Protection Against Stake-Inflation Attacks at Proposal Creation Time - (File: rs/sns/governance/src/governance.rs)

### Summary
The SNS governance canister computes ballot voting power at the exact moment a proposal is created using the current live stake of every neuron, with no historical snapshot or spike-detection mechanism. An unprivileged actor who controls a large SNS neuron can temporarily inflate their stake immediately before submitting a proposal, obtain an inflated ballot, then withdraw the stake, achieving disproportionate voting power on that proposal. The NNS governance canister has an explicit defense against this (the `VotingPowerSnapshots` / spike-detection system), but SNS governance has no equivalent.

### Finding Description
**NNS governance** (the reference implementation) maintains a rolling window of up to 7 daily `VotingPowerSnapshot` records in `VOTING_POWER_SNAPSHOTS`. When a new proposal is created, `compute_ballots_for_standard_proposal` compares the current total potential voting power against the minimum seen in those snapshots. If the current value exceeds 1.5× the historical minimum, it falls back to the snapshot with the minimum total, preventing a sudden stake inflation from dominating a ballot. [1](#0-0) [2](#0-1) 

**SNS governance** has no such mechanism. Its `compute_ballots_for_new_proposal` iterates directly over `self.proto.neurons` and calls `v.voting_power(now_seconds, ...)` for each neuron, recording the live stake at the instant of proposal creation: [3](#0-2) 

There is no snapshot store, no historical comparison, and no spike-detection guard anywhere in the SNS governance canister. The ballot voting power is purely a function of `cached_neuron_stake_e8s` at the moment `make_proposal` is called. [4](#0-3) [5](#0-4) 

The SNS `make_proposal` entry point calls `compute_ballots_for_new_proposal` synchronously with no pre-proposal snapshot check: [6](#0-5) 

### Impact Explanation
An attacker who controls a large SNS neuron (or can temporarily acquire one via flash-stake: transfer tokens → `claim_or_refresh` → submit proposal → dissolve/disburse) can:

1. Inflate `cached_neuron_stake_e8s` to a very large value immediately before calling `make_proposal`.
2. Obtain a ballot with inflated `voting_power` that is locked into the proposal's `ballots` map at creation time.
3. Reduce their stake after the proposal is created (the ballot is already fixed).
4. Use the inflated ballot to pass or block proposals that would otherwise require broader consensus.

Because the proposer's neuron automatically casts a `Yes` vote at proposal creation time (via `cast_vote_and_cascade_follow`), the inflated voting power is immediately exercised. If the inflated stake exceeds 50% of total voting power, the proposal can be adopted in the same transaction. [7](#0-6) 

### Likelihood Explanation
The attack requires the attacker to hold or temporarily acquire a large SNS token balance. SNS tokens are publicly tradeable; an attacker with sufficient capital can acquire tokens, stake a neuron, submit a proposal, and immediately begin dissolving. The SNS ledger's `claim_or_refresh` mechanism allows stake to be updated at any time. The attack is fully permissionless — no privileged role is required. The only friction is the minimum dissolve delay required to vote, but the ballot is fixed at proposal creation, so the attacker only needs the dissolve delay to be above the minimum at the moment of proposal submission, not throughout the voting period.

### Recommendation
Implement a voting power snapshot mechanism for SNS governance analogous to the NNS `VotingPowerSnapshots` system:

1. Periodically (e.g., daily) record a snapshot of total voting power across all eligible SNS neurons.
2. At proposal creation time, compare the current total voting power against the historical minimum in the snapshot window.
3. If a spike is detected (e.g., current > 1.5× historical minimum), use the snapshot with the minimum total to construct ballots instead of the live state.

Alternatively, at minimum, record the total voting power at proposal creation and reject proposals where the submitting neuron's voting power exceeds a configurable fraction of the total.

### Proof of Concept

**NNS (protected):** `compute_ballots_for_standard_proposal` in `rs/nns/governance/src/governance.rs` checks `VOTING_POWER_SNAPSHOTS.previous_ballots_if_voting_power_spike_detected(...)` before using the current snapshot. [8](#0-7) 

**SNS (unprotected):** `compute_ballots_for_new_proposal` in `rs/sns/governance/src/governance.rs` uses only `now_seconds` live state with no historical guard. [9](#0-8) 

Attack steps (permissionless ingress):
1. Acquire SNS tokens on the open market.
2. Call `manage_neuron { ClaimOrRefresh }` to stake a large neuron with dissolve delay ≥ `min_dissolve_delay_for_vote`.
3. Immediately call `manage_neuron { MakeProposal }` — the ballot is created with the inflated stake.
4. The proposer's neuron auto-votes `Yes` with the inflated voting power.
5. If inflated power > 50% of total, the proposal is adopted immediately.
6. Begin dissolving the neuron to recover tokens.

The NNS spike-detection timer task (`SnapshotVotingPowerTask`) and `VotingPowerSnapshots` structure have no counterpart in the SNS codebase. [10](#0-9) [11](#0-10)

    /// Computes the total potential voting power of the governance canister and ballots.
    fn compute_ballots_for_new_proposal(&self) -> Result<(u64, BTreeMap<String, Ballot>), String> {
        let now_seconds = self.env.now();

        let nervous_system_parameters = self.nervous_system_parameters_or_panic();

        // Voting power bonus parameters.
        let max_dissolve_delay = nervous_system_parameters
            .max_dissolve_delay_seconds
            .expect("NervousSystemParameters must have max_dissolve_delay_seconds");

        let max_age_bonus = nervous_system_parameters
            .max_neuron_age_for_age_bonus
            .expect("NervousSystemParameters must have max_neuron_age_for_age_bonus");

        let max_dissolve_delay_bonus_percentage = nervous_system_parameters
            .max_dissolve_delay_bonus_percentage
            .expect("NervousSystemParameters must have max_dissolve_delay_bonus_percentage");

        let max_age_bonus_percentage = nervous_system_parameters
            .max_age_bonus_percentage
            .expect("NervousSystemParameters must have max_age_bonus_percentage");

        let min_dissolve_delay_for_vote = nervous_system_parameters
            .neuron_minimum_dissolve_delay_to_vote_seconds
            .expect("NervousSystemParameters must have min_dissolve_delay_for_vote");

        let mut electoral_roll = BTreeMap::<String, Ballot>::new();
        let mut total_power: u128 = 0;

        for (k, v) in self.proto.neurons.iter() {
            // If this neuron is eligible to vote, record its
            // voting power at the time of proposal creation (now).
            if v.dissolve_delay_seconds(now_seconds) < min_dissolve_delay_for_vote {
                // Not eligible due to dissolve delay.
                continue;
            }

            let voting_power = v.voting_power(
                now_seconds,
                max_dissolve_delay,
                max_age_bonus,
                max_dissolve_delay_bonus_percentage,
                max_age_bonus_percentage,
            );

            total_power += voting_power as u128;
            electoral_roll.insert(
                k.clone(),
                Ballot {
                    vote: Vote::Unspecified as i32,
                    voting_power,
                    cast_timestamp_seconds: 0,
                },
            );
        }

        if total_power >= (u64::MAX as u128) {
            // The way the neurons are configured, the total voting
            // power on this proposal would overflow a u64!
            return Err("Voting power overflow.".to_string());
        }
        if electoral_roll.is_empty() {
            // Cannot make a proposal with no eligible voters.  This
            // is a precaution that shouldn't happen as we check that
            // the voter is allowed to vote.
            return Err("No eligible voters.".to_string());
        }

    pub fn voting_power(
        &self,
        now_seconds: u64,
        max_dissolve_delay_seconds: u64,
        max_neuron_age_for_age_bonus: u64,
        max_dissolve_delay_bonus_percentage: u64,
        max_age_bonus_percentage: u64,
    ) -> u64 {
        // We compute the stake adjustments in u128.
        let stake = self.voting_power_stake_e8s() as u128;
        // Dissolve delay is capped to max_dissolve_delay_seconds, but we cap it
        // again here to make sure, e.g., if this changes in the future.
        let d = std::cmp::min(
            self.dissolve_delay_seconds(now_seconds),
            max_dissolve_delay_seconds,
        ) as u128;
        // 'd_stake' is the stake with bonus for dissolve delay.
        let d_stake = stake
            + if max_dissolve_delay_seconds > 0 {
                (stake * d * max_dissolve_delay_bonus_percentage as u128)
                    / (100 * max_dissolve_delay_seconds as u128)
            } else {
                0
            };
        // Sanity check.
        assert!(d_stake <= stake + (stake * (max_dissolve_delay_bonus_percentage as u128) / 100));
        // The voting power is also a function of the age of the
        // neuron, giving a bonus of up to max_age_bonus_percentage at max_neuron_age_for_age_bonus.
        let a = std::cmp::min(self.age_seconds(now_seconds), max_neuron_age_for_age_bonus) as u128;
        let ad_stake = d_stake
            + if max_neuron_age_for_age_bonus > 0 {
                (d_stake * a * max_age_bonus_percentage as u128)
                    / (100 * max_neuron_age_for_age_bonus as u128)
            } else {
                0
            };
        // Final stake 'ad_stake' has is not more than max_age_bonus_percentage above 'd_stake'.
        assert!(ad_stake <= d_stake + (d_stake * (max_age_bonus_percentage as u128) / 100));

        // Convert the multiplier to u128. The voting_power_percentage_multiplier represents
        // a percent and will always be within the range 0 to 100.
        let v = self.voting_power_percentage_multiplier as u128;

        // Apply the multiplier to 'ad_stake' and divide by 100 to have the same effect as
        // multiplying by a percent.
        let vad_stake = ad_stake
            .checked_mul(v)
            .expect("Overflow detected when calculating voting power")
            .checked_div(100)
            .expect("Underflow detected when calculating voting power");

        // The final voting power is the stake adjusted by both age,
        // dissolve delay, and voting power multiplier. If the stake is is greater than
        // u64::MAX divided by 2.5, the voting power may actually not
        // fit in a u64.
        std::cmp::min(vad_stake, u64::MAX as u128) as u64

/// A task to snapshot the voting power every day, so that the snapshot can be used to disable
/// early adoption of proposals if such proposals have unusually high voting power.
pub(super) struct SnapshotVotingPowerTask {
    governance: &'static LocalKey<RefCell<Governance>>,
    snapshots: &'static LocalKey<RefCell<VotingPowerSnapshots>>,
}

const VOTING_POWER_SNAPSHOT_INTERVAL: Duration = Duration::from_secs(60 * 60 * 24);

impl SnapshotVotingPowerTask {
    pub fn new(
        governance: &'static LocalKey<RefCell<Governance>>,
        snapshots: &'static LocalKey<RefCell<VotingPowerSnapshots>>,
    ) -> Self {
        Self {
            governance,
            snapshots,
        }
    }
}

impl RecurringSyncTask for SnapshotVotingPowerTask {
    fn execute(self) -> (Duration, Self) {
        let now_seconds = self
            .governance
            .with_borrow(|governance| governance.env.now());
        if self
            .snapshots
            .with_borrow(|snapshots| snapshots.is_latest_snapshot_a_spike(now_seconds))
        {
            return (VOTING_POWER_SNAPSHOT_INTERVAL, self);
        }

        let voting_power_snapshot = self.governance.with_borrow_mut(|governance| {
            let voting_power_economics = governance.voting_power_economics();
            governance
                .neuron_store
                .compute_voting_power_snapshot_for_standard_proposal(
                    voting_power_economics,
                    now_seconds,
                )
                .expect("Voting power snapshot failed")
        });

        self.snapshots.with_borrow_mut(|snapshots| {
            snapshots.record_voting_power_snapshot(now_seconds, voting_power_snapshot);
        });

        (VOTING_POWER_SNAPSHOT_INTERVAL, self)
