### Title
Missing Signer Membership Pre-Check in IDKG Signature Share Validation Enables Sustained Resource Exhaustion by Malicious Subnet Node - (File: rs/consensus/idkg/src/signer.rs)

---

### Summary

The IC's IDKG threshold signing protocol (`validate_signature_share`) does not pre-check whether the claimed signer is a member of the expected receiver set before invoking `crypto_verify_sig_share`. Unlike the dealing-support path, which explicitly gates on receiver membership before any crypto work, the signature-share path proceeds directly to cryptographic verification for any claimed `NodeId`. Combined with the complete absence of any penalty or rate-limiting mechanism for nodes that send invalid IDKG messages, a single malicious subnet node (below the Byzantine fault threshold) can flood the unvalidated pool with crafted signature shares bearing arbitrary signer IDs, forcing all honest nodes to perform cryptographic verification for every injected share.

---

### Finding Description

In `validate_signature_share` (`rs/consensus/idkg/src/signer.rs`, lines 249–326), the function performs three cheap pre-checks (cache dedup, threshold saturation, pool dedup) and then immediately calls `crypto_verify_sig_share`:

```rust
let signer = share.signer();          // attacker-controlled NodeId
// ... cache / dedup checks only ...
match self.crypto_verify_sig_share(inputs, share, idkg_pool.stats()) {
``` [1](#0-0) 

There is **no check** that `signer` is actually a member of the receiver set embedded in `ThresholdSigInputs` before the crypto call is made. The crypto layer will eventually return `InvalidArgumentMissingSignerInTranscript` for an unknown signer, but only after the verification attempt is dispatched.

By contrast, `validate_dealing_support` in `rs/consensus/idkg/src/pre_signer.rs` explicitly gates on receiver membership before any crypto work:

```rust
if !transcript_params_ref.receivers.contains(&signer) {
    self.metrics.pre_sign_errors_inc("unexpected_support");
    return Some(IDkgChangeAction::HandleInvalid(
        id,
        format!("Support from unexpected node: {support}"),
    ));
}
``` [2](#0-1) 

The `ThresholdSigInputs` passed into `validate_signature_share` already contains the receiver set (via the key transcript), so the same early-exit pattern is feasible but absent. The `MaliciousFlags` struct documents the IC's awareness of malicious node behaviors, yet no analogous flag or penalty path exists for nodes that flood invalid signature shares. [3](#0-2) 

The `validated_sig_share_signers` in-memory cache only tracks *validated* signers per `RequestId`; it does not prevent repeated attempts with distinct fake `NodeId` values, each of which bypasses the cache check and reaches the crypto layer. [4](#0-3) 

---

### Impact Explanation

A malicious subnet node can:

1. Craft `EcdsaSigShare` / `SchnorrSigShare` / `VetKdKeyShare` messages with `signer_id` set to arbitrary `NodeId` values not present in the subnet's receiver set.
2. Inject these into the IDKG unvalidated pool via the P2P gossip layer (any subnet node can gossip IDKG artifacts).
3. Force all honest nodes to invoke `crypto_verify_sig_share` for each crafted share inside the parallel thread pool.
4. Cause sustained CPU consumption and thread-pool saturation, delaying the processing of legitimate signature shares and potentially causing threshold signing requests to time out.

Because `validate_signature_shares` collects all unvalidated shares and dispatches them in parallel:

```rust
shares.into_par_iter().filter_map(|(id, share)| {
    match Action::new(...) {
        Action::Process(sig_inputs) => {
            self.validate_signature_share(idkg_pool, id.clone(), share, sig_inputs)
        }
``` [5](#0-4) 

a flood of crafted shares saturates the thread pool, starving legitimate share processing. The availability impact on threshold signing (tECDSA, tSchnorr, vetKD) is high; integrity is not directly affected.

---

### Likelihood Explanation

Any subnet node can gossip IDKG artifacts to peers. A single compromised or malicious node below the fault threshold `f = floor((n-1)/3)` can sustain this attack indefinitely. The IC has no rate-limiting, quota, or penalty mechanism for nodes that repeatedly send invalid IDKG messages — the only consequence is that the crafted share is silently removed from the unvalidated pool via `HandleInvalid`. There is no NNS-level automatic detection or slashing; removal requires manual governance action, leaving a window for sustained disruption.

---

### Recommendation

Add an explicit signer membership pre-check in `validate_signature_share` before dispatching to `crypto_verify_sig_share`, mirroring the pattern already used in `validate_dealing_support`. The `ThresholdSigInputs` provides access to the receiver set; a non-member signer should be immediately returned as `HandleInvalid` without invoking the crypto layer. Additionally, consider tracking per-node invalid-share counts in metrics to enable detection and governance-level response.

---

### Proof of Concept

1. Malicious subnet node M constructs `EcdsaSigShare { signer_id: fake_node_id, request_id: active_request_id, sig_share_raw: <garbage> }` for many distinct `fake_node_id` values not in the subnet.
2. M gossips these to all honest nodes via the P2P layer; they enter each node's unvalidated IDKG pool.
3. On the next `on_state_change` tick, `validate_signature_shares` collects all unvalidated shares and dispatches them in parallel.
4. For each crafted share, `validate_signature_share` passes the cache/dedup checks (new fake signer ID each time) and calls `crypto_verify_sig_share`.
5. The crypto layer returns `InvalidArgumentMissingSignerInTranscript`; the share is marked `HandleInvalid` and removed — but the CPU cost is already incurred.
6. M repeats continuously, sustaining thread-pool saturation and delaying legitimate threshold signing operations. [6](#0-5) [7](#0-6)

        &self,
        idkg_pool: &dyn IDkgPool,
        state_snapshot: &dyn CertifiedStateSnapshot<State = ReplicatedState>,
    ) -> IDkgChangeSet {
        let sig_inputs_map = state_snapshot
            .get_state()
            .signature_request_contexts()
            .iter()
            .map(|(id, c)| {
                let inputs = build_signature_inputs(*id, c).map_err(|err| if err.is_fatal() {
                    warn!(every_n_seconds => 15, self.log,
                        "validate_signature_shares(): failed to build signatures inputs: {:?}",
                        err
                    );
                    self.metrics.sign_errors_inc("signature_inputs_malformed");
                }).ok();
                (*id, inputs)
            })
            .collect::<BTreeMap<_, _>>();

        let shares: Vec<_> = idkg_pool.unvalidated().signature_shares().collect();

        self.thread_pool.install(|| {
            // Iterate over all signature shares of all schemes
            shares
                .into_par_iter()
                .filter_map(|(id, share)| {
                    match Action::new(
                        &sig_inputs_map,
                        &share.request_id(),
                        state_snapshot.get_height(),

                        Action::Process(sig_inputs) => {
                            self.validate_signature_share(idkg_pool, id.clone(), share, sig_inputs)
                        }
                        Action::Drop => Some(IDkgChangeAction::RemoveUnvalidated(id)),
                        Action::Defer => None,
                    }
                })
                .collect()
        })
    }

    fn validate_signature_share(
        &self,
        idkg_pool: &dyn IDkgPool,
        id: IDkgMessageId,
        share: SigShare,
        inputs: &ThresholdSigInputs,
    ) -> Option<IDkgChangeAction> {
        {
            let valid_sig_share_signers = self.validated_sig_share_signers.read().unwrap();
            let maybe_signers = valid_sig_share_signers.get(&share.request_id());
            if maybe_signers.is_some_and(|signers| signers.contains(&share.signer())) {
                self.metrics
                    .sign_errors_inc("duplicate_sig_share_cache_hit");
                return Some(IDkgChangeAction::RemoveUnvalidated(id));
            }

            if Self::inputs_already_have_enough_shares(inputs, maybe_signers) {
                // We already have enough valid shares for this request
                return Some(IDkgChangeAction::RemoveUnvalidated(id));
            }
        }

        let signer = share.signer();
        let request_id = share.request_id();
        let scheme = share.scheme();
        if Self::signer_has_issued_share(idkg_pool, &signer, &request_id, scheme) {
            // The node already sent a valid share for this request
            self.metrics.sign_errors_inc("duplicate_sig_share");
            return Some(IDkgChangeAction::RemoveUnvalidated(id));
        }

        let share_string = share.to_string();
        match self.crypto_verify_sig_share(inputs, share, idkg_pool.stats()) {
            Err(error) if error.is_reproducible() => {
                self.metrics.sign_errors_inc("verify_sig_share_permanent");
                Some(IDkgChangeAction::HandleInvalid(
                    id,
                    format!(
                        "Signature share validation(permanent error): {share_string}, error = {error:?}"
                    ),
                ))
            }
            Err(error) => {
                // Defer in case of transient errors
                warn!(
                    every_n_seconds => 10,
                    self.log,
                    "Signature share validation(transient error): {}, error = {:?}",
                    share_string,
                    error
                );
                let label = if error.is_nidkg_transcript_not_loaded() {
                    "verify_sig_share_nidkg_transcript_not_loaded"
                } else {
                    "verify_sig_share_transient"
                };
                self.metrics.sign_errors_inc(label);
                None
            }
            Ok(share) => {
                self.metrics.sign_metrics_inc("sig_shares_received");

                // Although we already checked the cache for duplicate shares above, it could happen that a
                // different thread validated a share for the same request_id in the meantime, after we
                // released the read lock. Therefore, we acquire the write lock here to check again with
                // exclusive access.
                let mut valid_sig_share_signers = self.validated_sig_share_signers.write().unwrap();
                let signers = valid_sig_share_signers.entry(request_id).or_default();
                if !signers.insert(signer) {
                    self.metrics
                        .sign_errors_inc("duplicate_sig_share_cache_miss");
                    Some(IDkgChangeAction::RemoveUnvalidated(id))
                } else {
                    Some(IDkgChangeAction::MoveToValidated(share))
                }
            }
        }
    }
