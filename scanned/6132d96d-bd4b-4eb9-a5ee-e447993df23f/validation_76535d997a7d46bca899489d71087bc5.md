### Title
Minter Canister ID Accepted as Deposit Beneficiary Causes Permanent ETH Lock — (`rs/ethereum/cketh/minter/src/eth_logs/mod.rs`, `rs/ethereum/cketh/minter/src/deposit.rs`)

---

### Summary

`parse_principal_from_slice` explicitly blocks only the management canister and the anonymous principal. It does not block opaque canister principals such as the ckETH minter canister ID itself. When such a principal is encoded as the `bytes32` beneficiary in an Ethereum deposit, the minter accepts it, records it as a valid deposit event, and then enters a permanent retry loop because the ICRC1 ledger rejects the mint (treating it as a burn from the zero-balance minting account). The deposited ETH is already in the minter's Ethereum address and cannot be returned, causing permanent loss.

---

### Finding Description

**Step 1 — `parse_principal_from_slice` accepts canister IDs** [1](#0-0) 

The function's own doc comment states it only blocks the management canister (`num_bytes == 0`) and the anonymous principal (`[4]`). Any other valid principal — including an opaque canister ID ending in `0x01` with 10 bytes (e.g., the minter `sv3dd-oaaaa-aaaar-qacoa-cai`) — passes all checks and is returned as `Ok(Principal)`.

**Step 2 — No beneficiary validation before recording the deposit** [2](#0-1) 

`register_deposit_events` only checks whether the Ethereum *sender* address is on the blocklist. There is no check that the decoded beneficiary principal is not the minter canister or the ledger canister.

**Step 3 — `mint()` calls the ledger with the minting account as `to`** [3](#0-2) 

The minter calls `icrc1_transfer` with `to: event.beneficiary()`. If the beneficiary is the minter canister's default account (`Account { owner: minter_id, subaccount: None }`), this equals the ledger's `minting_account`.

**Step 4 — ICRC1 ledger rejects the call permanently** [4](#0-3) 

When `to == minting_account`, the ledger interprets the call as a *burn* from `from_account` (the minting account itself). The minting account has zero balance, so `apply_transaction` returns `InsufficientFunds`. The mint fails.

**Step 5 — The event is never quarantined; the minter retries forever** [5](#0-4) 

On `Ok(Err(err))`, the guard is defused and the event remains in `events_to_mint`. The minter reschedules a retry. Because the ledger will always reject this transfer, the event is permanently stuck and the ETH is permanently locked in the minter's Ethereum address.

---

### Impact Explanation

The deposited ETH is transferred to the minter's Ethereum address at the Ethereum layer before the IC ever processes the log. Once the minter records the event, it cannot return the ETH. The ckETH mint will fail on every attempt. Without a canister upgrade that explicitly quarantines or handles this case, the ETH is permanently unrecoverable.

---

### Likelihood Explanation

The minter canister ID (`sv3dd-oaaaa-aaaar-qacoa-cai`) is public. Encoding it as a `bytes32` principal is straightforward using the documented encoding scheme. An attacker can deliberately trigger this, or an innocent user can accidentally encode the wrong principal. The attack requires only a standard Ethereum transaction to the helper contract — no privileged access.

---

### Recommendation

1. In `parse_principal_from_slice`, add explicit rejection of the minter canister ID and the ckETH ledger canister ID (and any other system-critical canister IDs).
2. Alternatively, add a guard in `mint()` or `register_deposit_events()` that checks `event.beneficiary().owner != ic_cdk::id()` and rejects/quarantines the event before attempting the ledger call.
3. Add a unit test asserting that `parse_principal_from_slice` rejects the minter canister ID bytes.

---

### Proof of Concept

1. Obtain the minter canister ID bytes: `sv3dd-oaaaa-aaaar-qacoa-cai` → raw bytes (10 bytes, opaque principal).
2. Encode as `bytes32`: `[10, b0, b1, ..., b9, 0, 0, ..., 0]` (10-byte length prefix + principal bytes + zero padding).
3. Call `depositEth(encoded_principal, 0x0)` on the Ethereum helper contract with any ETH amount.
4. The minter scrapes the log, `parse_principal_from_slice` returns `Ok(minter_principal)`.
5. The event is recorded in `events_to_mint`.
6. `mint()` calls `icrc1_transfer { to: Account { owner: minter_id, subaccount: None }, amount: X }`.
7. The ICRC1 ledger sees `to == minting_account`, attempts a burn from the minting account (zero balance), returns `InsufficientFunds`.
8. The minter logs the error, defuses the guard, and reschedules. The ETH is permanently locked.

/// Decode a candid::Principal from a slice of at most 32 bytes
/// encoded as follows
/// - the first byte is the number of bytes in the principal
/// - the next N bytes are the principal
/// - the remaining bytes are zero
///
/// Any other encoding will return an error.
/// Some specific valid [`Principal`]s are also not allowed
/// since the decoded principal will be used to receive ckETH:
/// * the management canister principal
/// * the anonymous principal
///
/// This method MUST never panic (decode bytes from untrusted sources).
fn parse_principal_from_slice(slice: &[u8]) -> Result<Principal, String> {
    const ANONYMOUS_PRINCIPAL_BYTES: [u8; 1] = [4];

    if slice.is_empty() {
        return Err("slice too short".to_string());
    }
    if slice.len() > 32 {
        return Err(format!("Expected at most 32 bytes, got {}", slice.len()));
    }
    let num_bytes = slice[0] as usize;
    if num_bytes == 0 {
        return Err("management canister principal is not allowed".to_string());
    }
    if num_bytes > 29 {
        return Err(format!(
            "invalid number of bytes: expected a number in the range [1,29], got {num_bytes}",
        ));
    }
    if slice.len() < 1 + num_bytes {
        return Err("slice too short".to_string());
    }
    let (principal_bytes, trailing_zeroes) = slice[1..].split_at(num_bytes);
    if !trailing_zeroes
        .iter()
        .all(|trailing_zero| *trailing_zero == 0)
    {
        return Err("trailing non-zero bytes".to_string());
    }
    if principal_bytes == ANONYMOUS_PRINCIPAL_BYTES {
        return Err("anonymous principal is not allowed".to_string());
    }
    Principal::try_from_slice(principal_bytes).map_err(|err| err.to_string())
}
