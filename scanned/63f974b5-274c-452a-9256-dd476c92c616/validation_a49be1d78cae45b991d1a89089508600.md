### Title
VetKD Key Share Verification Accepts Undeserializable Content, Allowing a Single Byzantine Node to Block Key Derivation â€” (`rs/crypto/src/vetkd/mod.rs`)

### Summary

`verify_encrypted_key_share_internal` only validates the Ed25519 signature over the raw bytes of `encrypted_key_share`. It never attempts to deserialize the content. A Byzantine subnet node can submit a `VetKdEncryptedKeyShare` with `encrypted_key_share = VetKdEncryptedKeyShareContent(vec![])` and a valid signature over those empty bytes. The share passes verification and enters the validated pool. When `combine_encrypted_key_shares_internal` later processes all validated shares, it calls `EncryptedKeyShare::deserialize` on the empty bytes, which fails immediately and propagates as `InvalidArgumentEncryptedKeyShare`, causing the entire combination to fail and the key derivation request to be silently dropped.

### Finding Description

**Gap in `verify_encrypted_key_share_internal`:** [1](#0-0) 

The function only calls `BasicSigVerifierInternal::verify_basic_sig` over `key_share.encrypted_key_share`. There is no attempt to deserialize or structurally validate the content of `encrypted_key_share`. A valid Ed25519 signature over empty bytes passes this check.

**Deserialization failure in `combine_encrypted_key_shares_internal`:** [2](#0-1) 

All shares are deserialized via `.collect::<Result<_, _>>()?`. If any single share fails deserialization, the entire combination fails immediately with `InvalidArgumentEncryptedKeyShare` â€” before `combine_all` or the `combine_valid_shares` fallback is ever reached.

**`EncryptedKeyShare::deserialize` rejects wrong-length input:** [3](#0-2) 

Empty bytes (length 0) fail the `val.len() != Self::BYTES` check immediately.

**The combiner includes ALL validated shares, not just threshold-many:** [4](#0-3) 

`shares.vetkd_shares.get(callback_id)` returns the full `BTreeMap<NodeId, VetKdEncryptedKeyShare>` collected from all validated pool entries. There is no pre-filtering to exclude the Byzantine node's share.

**The combination failure is silently dropped:** [5](#0-4) 

`InvalidArgumentEncryptedKeyShare` is not an `UnsatisfiedReconstructionThreshold` error, so it falls into the `Err(err)` branch: a warning is logged and `None` is returned â€” no agreement is produced for that request.

**The share enters the validated pool via the signer:** [6](#0-5) 

`VetKdProtocol::verify_encrypted_key_share` is the only gate before a share is added to the validated pool as `IDkgMessage::VetKdKeyShare`. Since it only checks the signature, the Byzantine share passes.

### Impact Explanation

A single Byzantine subnet node (below the BFT fault threshold) can permanently block any VetKD key derivation request for the duration of the DKG interval by submitting one signed-but-empty share per request. The request is silently dropped from every block payload until it expires. The node can repeat this for every new request, effectively denying the VetKD service to all users of the subnet.

### Likelihood Explanation

Any subnet node with a valid registered Ed25519 key can execute this. No threshold corruption, governance majority, or privileged access is required. The attack is deterministic and repeatable. The only natural limit is the DKG interval expiry, but the attacker can re-attack each new request.

### Recommendation

Add deserialization validation inside `verify_encrypted_key_share_internal` in `rs/crypto/src/vetkd/mod.rs`. After the signature check passes, attempt `EncryptedKeyShare::deserialize(&key_share.encrypted_key_share.0)` and return a `VetKdKeyShareVerificationError` (e.g., a new `InvalidEncryptedKeyShareContent` variant, or reuse `VerificationError`) if deserialization fails. This closes the gap between what is verified and what is later consumed by the combiner.

### Proof of Concept

1. Byzantine node holds a valid registered Ed25519 key pair `(sk, pk)`.
2. Construct `VetKdEncryptedKeyShare { encrypted_key_share: VetKdEncryptedKeyShareContent(vec![]), node_signature: Ed25519Sign(sk, b"") }`.
3. Broadcast this as a `VetKdKeyShare` P2P artifact for any active VetKD request.
4. Other nodes call `verify_encrypted_key_share` â†’ signature over empty bytes is valid â†’ `Ok(())` â†’ share enters validated pool.
5. Combiner calls `combine_encrypted_key_shares` with all validated shares including the Byzantine one.
6. `EncryptedKeyShare::deserialize(&[])` fails â†’ `collect::<Result<_, _>>()?` returns `Err(InvalidArgumentEncryptedKeyShare)`.
7. `combine_shares` returns `Err(CombineSharesError::VetKd(...))` â†’ not an `UnsatisfiedReconstructionThreshold` â†’ logged as warning, `None` returned â†’ no agreement produced â†’ request is dropped.
