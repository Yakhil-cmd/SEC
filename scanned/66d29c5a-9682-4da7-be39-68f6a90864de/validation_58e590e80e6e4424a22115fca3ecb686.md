### Title
`max_number_of_neurons` Can Be Reduced Below Current Neuron Count via `ManageNervousSystemParameters` - (File: `rs/sns/governance/src/types.rs`)

### Summary
The `validate_max_number_of_neurons()` function in SNS Governance validates the `max_number_of_neurons` parameter only against static absolute bounds, never against the current live neuron count. A `ManageNervousSystemParameters` governance proposal can therefore set `max_number_of_neurons` to a value strictly below the number of neurons that already exist, creating a permanently inconsistent state in which no new neuron can ever be created.

### Finding Description
`NervousSystemParameters.max_number_of_neurons` is the governance-controlled cap on how many neurons an SNS may hold. When a `ManageNervousSystemParameters` proposal is submitted, `validate_and_render_manage_nervous_system_parameters()` calls `new_parameters.inherit_from(current_parameters).validate()`. [1](#0-0) 

`validate()` in turn calls `validate_max_number_of_neurons()`: [2](#0-1) 

`validate_max_number_of_neurons()` only rejects values that are `0` or exceed `MAX_NUMBER_OF_NEURONS_CEILING` (200,000): [3](#0-2) 

There is no check that the proposed value is `>= self.proto.neurons.len()`. The execution path `perform_manage_nervous_system_parameters()` also only calls `new_params.validate()` and has no access to the live neuron count: [4](#0-3) 

The enforcement point that actually uses `max_number_of_neurons` is `check_neuron_population_can_grow()`: [5](#0-4) 

Once `max_number_of_neurons` is set below the current `neurons.len()`, the condition `(self.proto.neurons.len() as u64) + 1 > max_number_of_neurons` is permanently true, and no new neuron can ever be added.

### Impact Explanation
After the inconsistent parameter is committed, every call path that invokes `check_neuron_population_can_grow()` â€” including `ClaimOrRefresh`, `Split`, and `claim_swap_neurons` â€” returns `PreconditionFailed`. The SNS governance canister is permanently frozen for new participants: no new neuron can be staked or claimed. Existing neurons and their voting power are unaffected, but the SNS can never grow its participant base again without a corrective governance proposal (which itself requires existing neuron holders to act). This is a governance-level denial-of-service for new participants.

### Likelihood Explanation
Exploiting this requires a `ManageNervousSystemParameters` proposal to pass with a `max_number_of_neurons` value below the current neuron count. This can happen either accidentally (a governance majority misconfiguring the parameter) or intentionally (a governance majority with adversarial intent). Because SNS governance proposals require a supermajority, the likelihood is low but non-zero, and the missing validation means the canister provides no safety net against the mistake.

### Recommendation
Add a runtime check in `perform_manage_nervous_system_parameters()` (which has access to `self.proto.neurons.len()`) that rejects any proposed `max_number_of_neurons` value that is less than the current neuron count:

```rust
if new_max < self.proto.neurons.len() as u64 {
    return Err(GovernanceError::new_with_message(
        ErrorType::PreconditionFailed,
        format!(
            "Cannot set max_number_of_neurons ({new_max}) below the current \
             neuron count ({})",
            self.proto.neurons.len()
        ),
    ));
}
```

Alternatively, add a `validate_against_state(&self, governance: &Governance)` method to `NervousSystemParameters` that performs live-state checks and is called from both the proposal validator and the executor.

### Proof of Concept

1. An SNS governance canister has **N = 100** neurons (e.g., after a successful token swap).
2. A token holder with sufficient voting power submits a `ManageNervousSystemParameters` proposal setting `max_number_of_neurons = 50`.
3. `validate_and_render_manage_nervous_system_parameters` accepts the proposal because `50 > 0` and `50 < 200_000`. [6](#0-5) 
4. The proposal passes and `perform_manage_nervous_system_parameters` commits `max_number_of_neurons = 50`. [7](#0-6) 
5. Any subsequent call to `check_neuron_population_can_grow` evaluates `(100 as u64) + 1 > 50` â†’ `true` â†’ `PreconditionFailed`. [8](#0-7) 
6. No new neuron can ever be created. The SNS is permanently closed to new participants unless a corrective governance proposal is passed by the existing neuron holders.

/// Validates and renders a proposal with action ManageNervousSystemParameters.
fn validate_and_render_manage_nervous_system_parameters(
    new_parameters: &NervousSystemParameters,
    current_parameters: &NervousSystemParameters,
) -> Result<String, String> {
    if new_parameters == &NervousSystemParameters::default() {
        return Err("NervousSystemParameters: at least one field must be set.".to_string());
    }

    new_parameters.inherit_from(current_parameters).validate()?;

    Ok(format!(
        r"# Proposal to change nervous system parameters:
## Current nervous system parameters:

{:#?}

## New nervous system parameters:

{:#?}",
        &current_parameters, new_parameters
    ))
}

    /// Validates that the nervous system parameter max_number_of_neurons is well-formed.
    fn validate_max_number_of_neurons(&self) -> Result<(), String> {
        let max_number_of_neurons = self.max_number_of_neurons.ok_or_else(|| {
            "NervousSystemParameters.max_number_of_neurons must be set".to_string()
        })?;

        if max_number_of_neurons > Self::MAX_NUMBER_OF_NEURONS_CEILING {
            Err(format!(
                "NervousSystemParameters.max_number_of_neurons must be less than {}",
                Self::MAX_NUMBER_OF_NEURONS_CEILING
            ))
        } else if max_number_of_neurons == 0 {
            Err("NervousSystemParameters.max_number_of_neurons must be greater than 0".to_string())
        } else {
            Ok(())
        }
    }

    fn perform_manage_nervous_system_parameters(
        &mut self,
        proposed_params: NervousSystemParameters,
    ) -> Result<(), GovernanceError> {
        // Only set `self.proto.parameters` if "applying" the proposed params to the
        // current params results in valid params
        let new_params = proposed_params.inherit_from(self.nervous_system_parameters_or_panic());

        log!(
            INFO,
            "Setting Governance nervous system params to: {:?}",
            &new_params
        );

        match new_params.validate() {
            Ok(()) => {
                self.proto.parameters = Some(new_params);
                Ok(())
            }

            // Even though proposals are validated when they are first made, this is still
            // possible, because the inner value of a ManageNervousSystemParameters
            // proposal is only valid with respect to the current
            // nervous_system_parameters() at the time when the proposal was first
            // made. If nervous_system_parameters() changed (by another proposal) since
            // the current proposal was first made, the current proposal might have become
            // invalid. Basically, this might occur if there are conflicting (concurrent)
            // proposals, but we expect this to be highly unusual in practice.
            Err(msg) => Err(GovernanceError::new_with_message(
                ErrorType::PreconditionFailed,
                format!(
                    "Failed to perform ManageNervousSystemParameters action, proposed \
                        parameters would lead to invalid NervousSystemParameters: {msg}"
                ),
            )),
        }
    }

    /// Checks whether new neurons can be added or whether the maximum number of neurons,
    /// as defined in the nervous system parameters, has already been reached.
    fn check_neuron_population_can_grow(&self) -> Result<(), GovernanceError> {
        let max_number_of_neurons = self
            .nervous_system_parameters_or_panic()
            .max_number_of_neurons
            .expect("NervousSystemParameters must have max_number_of_neurons");

        if (self.proto.neurons.len() as u64) + 1 > max_number_of_neurons {
            return Err(GovernanceError::new_with_message(
                ErrorType::PreconditionFailed,
                "Cannot add neuron. Max number of neurons reached.",
            ));
        }

        Ok(())
    }
