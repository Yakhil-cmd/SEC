### Title
Integer Overflow in Neuron Age Calculation During Merge Produces Incorrect Voting Power - (`File: rs/nns/governance/src/governance/merge_neurons.rs`)

### Summary

Both `ValidSourceNeuron::try_new` and `ValidTargetNeuron::try_new` compute neuron age using plain subtraction (`now_seconds - aging_since_timestamp_seconds`) instead of the saturating variant used everywhere else in the codebase. When `aging_since_timestamp_seconds > now_seconds` — a known valid legacy state explicitly tested in the IC codebase — the subtraction wraps around in Wasm, producing a near-`u64::MAX` age value. This corrupts the merged neuron's `aging_since_timestamp_seconds`, granting it a massively inflated age bonus and therefore incorrect voting power.

### Finding Description

In `ValidSourceNeuron::try_new` and `ValidTargetNeuron::try_new`, the neuron age is computed as:

```rust
age_seconds: now_seconds - aging_since_timestamp_seconds,
``` [1](#0-0) [2](#0-1) 

The canonical age computation throughout the rest of the codebase uses `saturating_sub`:

```rust
now_seconds.saturating_sub(aging_since_timestamp_seconds)
``` [3](#0-2) 

The IC's own test suite explicitly acknowledges that a `NotDissolving` neuron can have `aging_since_timestamp_seconds > now_seconds` (e.g., `NOW + 1`), and that the correct result in that case is `0`: [4](#0-3) 

When `aging_since_timestamp_seconds > now_seconds`, the plain `-` wraps around in Wasm (defined wrapping behavior), producing a value near `u64::MAX`. This overflowed `age_seconds` is then fed into `combine_aged_stakes`: [5](#0-4) 

`combine_aged_stakes` uses `saturating_mul`/`saturating_add` internally, but the saturated intermediate result still produces a wildly incorrect `new_target_age_seconds`, which is then written back as the merged neuron's `aging_since_timestamp_seconds`: [6](#0-5) 

The corrupted `aging_since_timestamp_seconds` persists in governance state and directly inflates the age bonus used in all subsequent voting power calculations for the merged neuron: [7](#0-6) 

### Impact Explanation

A neuron with a corrupted (near-zero) `aging_since_timestamp_seconds` would receive the maximum age bonus (up to 1.25×) on its full stake, regardless of its actual age. This inflates both `potential_voting_power` and `deciding_voting_power`, giving the attacker disproportionate influence over NNS governance proposals and inflated voting rewards. The corruption is permanent — it persists in stable state until the neuron is dissolved or otherwise modified.

**Impact: Medium** — Incorrect voting power for a single merged neuron; does not break consensus or allow arbitrary minting, but distorts governance outcomes and reward distribution.

### Likelihood Explanation

The `from_api` conversion for `DissolveStateAndAge::NotDissolving` imposes no constraint that `aging_since_timestamp_seconds <= now_seconds`: [8](#0-7) 

Legacy neurons from early IC deployments may carry `aging_since_timestamp_seconds` values in the future. The test suite explicitly covers this case, confirming the IC team is aware it can occur. Any user who controls (or can obtain hotkey access to) such a neuron and any compatible partner neuron can trigger the merge via the public `manage_neuron` endpoint: [9](#0-8) 

**Likelihood: Medium** — Requires a neuron with `aging_since_timestamp_seconds > now_seconds`, which is a known possible state but not universally present.

### Recommendation

Replace the plain subtraction with `saturating_sub` in both `ValidSourceNeuron::try_new` and `ValidTargetNeuron::try_new`, consistent with `DissolveStateAndAge::age_seconds`:

```diff
- age_seconds: now_seconds - aging_since_timestamp_seconds,
+ age_seconds: now_seconds.saturating_sub(aging_since_timestamp_seconds),
```

Apply this fix at both:
- `rs/nns/governance/src/governance/merge_neurons.rs` line 438 (`ValidSourceNeuron`)
- `rs/nns/governance/src/governance/merge_neurons.rs` line 480 (`ValidTargetNeuron`)

### Proof of Concept

1. Create a `NotDissolving` neuron `A` with `aging_since_timestamp_seconds = now + 1` (a future timestamp, a known valid legacy state).
2. Create a second compatible `NotDissolving` neuron `B` with the same controller.
3. Call `manage_neuron` → `Merge` with `A` as source and `B` as target.
4. In `ValidSourceNeuron::try_new`, `now_seconds - (now_seconds + 1)` wraps to `u64::MAX`.
5. `combine_aged_stakes(target_stake, target_age, amount, u64::MAX)` saturates, producing a near-maximal `new_target_age_seconds`.
6. The merged neuron `B` receives `aging_since_timestamp_seconds ≈ now_seconds - u64::MAX ≈ 0`, effectively claiming the maximum possible age and the full 1.25× age bonus on its entire stake for all future voting power calculations. [10](#0-9) [11](#0-10)
