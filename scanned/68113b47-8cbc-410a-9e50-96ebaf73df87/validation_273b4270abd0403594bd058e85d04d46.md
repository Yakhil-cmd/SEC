### Title
HTTP Outcall Full-Fee Upfront Charge With No Actual-Usage Refund Under Locked Legacy Pricing — (`rs/types/management_canister_types/src/http.rs`, `rs/https_outcalls/pricing/src/legacy.rs`, `rs/execution_environment/src/execution_environment.rs`)

---

### Summary

Every canister HTTP outcall on the IC is forced onto legacy pricing, which charges the full fee upfront — including a term proportional to the caller-declared maximum response size — and never refunds any portion based on actual bytes received. Pay-as-you-go pricing (which charges only a base fee upfront and refunds the remainder) is implemented but is explicitly excluded from the allowed-versions list, so no canister can opt into it. The result is a systematic, per-request cycles overpayment that grows with the gap between `max_response_bytes` and the actual response.

---

### Finding Description

**Step 1 — All requests are silently locked to legacy pricing.**

`ALLOWED_HTTP_OUTCALLS_PRICING_VERSIONS` contains only `PRICING_VERSION_LEGACY`: [1](#0-0) 

In `CanisterHttpRequestContext::generate_from_args`, any caller-supplied `pricing_version` that is not in the allowed set is silently replaced with the default (legacy): [2](#0-1) 

**Step 2 — Legacy pricing charges the full fee upfront, including the max-response-size term.**

`http_request_fee` computes the legacy fee as:

```
(linear_baseline + quadratic_baseline × subnet_size
 + per_byte_request × request_size
 + per_byte_response × response_size)   ← response_size = max_response_bytes or MAX (2 MB)
× subnet_size
``` [3](#0-2) 

When `max_response_bytes` is absent, `response_size` defaults to `MAX_CANISTER_HTTP_RESPONSE_BYTES` (2 MB): [4](#0-3) 

In `try_add_http_context_to_replicated_state`, the full legacy fee is deducted from the request payment immediately: [5](#0-4) 

The code comment explicitly acknowledges that nothing will actually be refunded for legacy pricing: [6](#0-5) 

**Step 3 — The adapter-side tracker never produces a refund.**

`PricingFactory::new_tracker` always constructs a `LegacyTracker` regardless of `context.pricing_version`, with a TODO acknowledging the gap: [7](#0-6) 

`LegacyTracker::create_payment_receipt` unconditionally returns a zero-refund receipt: [8](#0-7) 

**Step 4 — Pay-as-you-go is implemented but unreachable.**

The pay-as-you-go branch exists in the execution environment and would charge only the base fee upfront and refund the remainder: [9](#0-8) 

But because `ALLOWED_HTTP_OUTCALLS_PRICING_VERSIONS` excludes `PRICING_VERSION_PAY_AS_YOU_GO`, this branch is dead code for any externally submitted request. [10](#0-9) 

---

### Impact Explanation

On a 13-node application subnet with the production fee schedule (`http_response_per_byte_fee = 800` cycles/byte):

| Scenario | Cycles charged | Cycles for actual work |
|---|---|---|
| No `max_response_bytes` (defaults to 2 MB), 1 KB actual response | ≈ 21.85 B | ≈ 60 M |
| `max_response_bytes = 1 MB`, 10 KB actual response | ≈ 10.9 B | ≈ 160 M |

The overpayment per request is `(max_response_bytes − actual_bytes) × 800 × subnet_size`. For a canister making 1 000 outcalls per day with a 2 MB limit and 1 KB responses, the daily overpayment is ≈ 21.8 trillion cycles. This is a direct, measurable cycles drain on every canister that uses HTTP outcalls, with no mechanism for the canister to recover the excess.

The impact class is **cycles/resource accounting bug**: cycles are permanently burned in excess of the actual cost of the service rendered, with no proportional refund path available to any caller.

---

### Likelihood Explanation

- **All** HTTP outcalls on the IC currently use legacy pricing — there is no opt-out.
- Many canister developers omit `max_response_bytes`, triggering the 2 MB default and the maximum possible overpayment.
- The entry path (`ic00::HttpRequest`) is accessible to any canister on any application subnet without any privileged role.
- The overpayment is deterministic and repeatable on every request.

---

### Recommendation

1. Add `PRICING_VERSION_PAY_AS_YOU_GO` to `ALLOWED_HTTP_OUTCALLS_PRICING_VERSIONS` and update `PricingFactory::new_tracker` to dispatch on `context.pricing_version` (resolving `TODO(IC-1937)`).
2. Make pay-as-you-go the default (`DEFAULT_HTTP_OUTCALLS_PRICING_VERSION`) so that canisters that do not specify a version benefit from proportional refunds automatically.
3. Until pay-as-you-go is live, document the overpayment behaviour prominently so developers can set tight `max_response_bytes` values to limit excess charges.

---

### Proof of Concept

1. Any canister calls `ic00::HttpRequest` with `max_response_bytes: None` and attaches ≥ legacy fee in cycles.
2. `generate_from_args` sets `pricing_version = PricingVersion::Legacy` (the only allowed version).
3. `try_add_http_context_to_replicated_state` deducts `legacy_fee ≈ 21.85 B cycles` from `request.payment`.
4. The HTTP adapter fetches the URL; the server returns 1 KB.
5. `LegacyTracker::create_payment_receipt()` returns `CanisterHttpPaymentReceipt::default()` — zero refund.
6. The canister's callback fires; the remaining `request.payment` (everything above the legacy fee) is returned, but the ≈ 21.79 B cycles overpayment for the unused 2 MB − 1 KB response budget is permanently burned.
7. Repeating this 1 000 times per day burns ≈ 21.8 trillion cycles in excess fees with no recourse.

    pub fn http_request_fee(
        &self,
        request_size: NumBytes,
        response_size_limit: Option<NumBytes>,
        subnet_cycles_config: CyclesAccountManagerSubnetConfig,
    ) -> CompoundCycles<HTTPOutcalls> {
        let subnet_size = subnet_cycles_config.subnet_size;
        let response_size = match response_size_limit {
            Some(response_size) => response_size.get(),
            // Defaults to maximum response size.
            None => MAX_CANISTER_HTTP_RESPONSE_BYTES,
        };

        let amount = (self.config.http_request_linear_baseline_fee
            + self.config.http_request_quadratic_baseline_fee * (subnet_size as u64)
            + self.config.http_request_per_byte_fee * request_size.get()
            + self.config.http_response_per_byte_fee * response_size)
            * (subnet_size as u64);

        CompoundCycles::new(amount, subnet_cycles_config.cost_schedule)
    }
