### Title
Missing Combined Constraint on `max_dissolve_delay_bonus_percentage` and `max_age_bonus_percentage` Enables Governance DoS - (File: rs/sns/governance/src/types.rs)

### Summary
`NervousSystemParameters` in the SNS governance canister validates `max_dissolve_delay_bonus_percentage` and `max_age_bonus_percentage` independently against their individual ceilings (900 and 400 respectively), but enforces no combined constraint on their multiplicative effect. When both are set to their maximum values via separate `ManageNervousSystemParameters` proposals, the combined voting power multiplier reaches 50×. If total neuron stake is large enough, aggregate voting power overflows `u64::MAX`, causing `compute_ballots_for_new_proposal` to permanently return an error and preventing any proposal from being submitted to the SNS governance.

### Finding Description
In `rs/sns/governance/src/types.rs`, `NervousSystemParameters::validate()` calls two independent validators:

```rust
self.validate_max_dissolve_delay_bonus_percentage()?;
self.validate_max_age_bonus_percentage()?;
```

`validate_max_dissolve_delay_bonus_percentage` only checks `max_dissolve_delay_bonus_percentage <= MAX_DISSOLVE_DELAY_BONUS_PERCENTAGE_CEILING` (900), and `validate_max_age_bonus_percentage` only checks `max_age_bonus_percentage <= MAX_AGE_BONUS_PERCENTAGE_CEILING` (400). No combined constraint is enforced. [1](#0-0) [2](#0-1) 

The `Neuron::voting_power()` function in `rs/sns/governance/src/neuron.rs` applies these bonuses **multiplicatively**:

1. `d_stake = stake * (1 + max_dissolve_delay_bonus_percentage / 100)` → up to **10× stake** at ceiling 900
2. `ad_stake = d_stake * (1 + max_age_bonus_percentage / 100)` → up to **5× d_stake = 50× stake** at ceiling 400 [3](#0-2) 

In `compute_ballots_for_new_proposal()` in `rs/sns/governance/src/governance.rs`, total voting power is accumulated in `u128` and checked:

```rust
if total_power >= (u64::MAX as u128) {
    return Err("Voting power overflow.".to_string());
}
```

If this check triggers, the function returns an error and **no proposals can be submitted**. [4](#0-3) 

The `ManageNervousSystemParameters` proposal validation path calls `new_parameters.inherit_from(current_parameters).validate()`, which runs the same independent per-field checks and has no combined constraint: [5](#0-4) [6](#0-5) 

### Impact Explanation
If both parameters are set to their maximum values (900 and 400) via separate `ManageNervousSystemParameters` proposals — each individually valid — and total neuron stake exceeds approximately `u64::MAX / 50 ≈ 3.68 × 10^9` tokens (in e8s: `~3.68 × 10^17`), the SNS governance permanently loses the ability to create any proposal. The SNS community cannot upgrade canisters, change parameters, or take any governance action. This is a governance DoS with no recovery path short of a subnet-level intervention.

### Likelihood Explanation
Low. Two conditions must hold simultaneously:
1. Two separate governance proposals must set both parameters to their respective ceilings (900 and 400). Each proposal passes validation individually, so no on-chain check prevents this accidental misconfiguration.
2. Total neuron stake must exceed ~3.68 billion tokens. For most SNS projects this is unrealistic, but for a large SNS with high token supply it is theoretically reachable.

The vulnerability class is real and the missing combined constraint is the root cause; the high stake threshold is what keeps likelihood low.

### Recommendation
Add a combined constraint in `NervousSystemParameters::validate()` (or in a dedicated helper) that bounds the product of the two bonus multipliers:

```rust
fn validate_combined_bonus_percentages(&self) -> Result<(), String> {
    let dd = self.max_dissolve_delay_bonus_percentage.unwrap_or(0);
    let age = self.max_age_bonus_percentage.unwrap_or(0);
    // Combined multiplier = (1 + dd/100) * (1 + age/100)
    // Expressed in integer arithmetic: (100 + dd) * (100 + age) <= COMBINED_CEILING * 10_000
    let combined = (100u128 + dd as u128) * (100u128 + age as u128);
    if combined > SOME_COMBINED_CEILING {
        return Err(format!(
            "Combined bonus multiplier ({combined}) exceeds allowed ceiling"
        ));
    }
    Ok(())
}
```

This mirrors the recommendation in the original report: validate the combined effect, not just each parameter in isolation.

### Proof of Concept
1. Neuron holder submits `ManageNervousSystemParameters { max_dissolve_delay_bonus_percentage: Some(900), ..Default::default() }`. Proposal passes `validate()` — individual ceiling is 900. ✓
2. Neuron holder submits `ManageNervousSystemParameters { max_age_bonus_percentage: Some(400), ..Default::default() }`. Proposal passes `validate()` — individual ceiling is 400. ✓
3. Both proposals are executed. `NervousSystemParameters` now has `max_dissolve_delay_bonus_percentage = 900`, `max_age_bonus_percentage = 400`.
4. For an SNS with total neuron stake ≥ ~3.68 × 10^9 tokens, every call to `compute_ballots_for_new_proposal` returns `Err("Voting power overflow.")`.
5. `make_proposal` fails for every caller. The SNS governance is permanently DoS'd. [7](#0-6) [6](#0-5)
