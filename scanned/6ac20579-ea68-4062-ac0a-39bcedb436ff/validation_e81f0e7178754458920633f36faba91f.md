### Title
Information Disclosure of Private Neuron `maturity_e8s_equivalent` via Pre-Authorization Error Ordering — (`rs/nns/governance/src/governance/disburse_maturity.rs`)

### Summary
`initiate_maturity_disbursement` checks `DisbursementTooSmall` before `CallerIsNotNeuronController`. The `DisbursementTooSmall` error message embeds the exact computed `disbursement_maturity_e8s` value. Any unprivileged caller can recover the `maturity_e8s_equivalent` of any private neuron with maturity below 100 ICP using a single `manage_neuron` update call.

### Finding Description
In `initiate_maturity_disbursement`: [6](#0-5) 

The function computes `disbursement_maturity_e8s = floor(maturity_e8s_equivalent * percentage / 100)` and immediately checks it against `MINIMUM_DISBURSEMENT_E8S` (1 ICP = 100,000,000 e8s) **before** checking whether the caller is the neuron controller. If the computed amount is below the minimum, the error returned is:

```
"Disbursement (X) is too small. The amount should be at least: 100000000 e8s"
```

where `X` is the exact `disbursement_maturity_e8s`. With `percentage_to_disburse = 1`, this reveals `floor(maturity_e8s_equivalent / 100)`, from which the attacker recovers `maturity_e8s_equivalent` to within 1%.

This only triggers when `maturity_e8s_equivalent < 10,000,000,000` (100 ICP), i.e., when 1% of maturity is below the 1 ICP minimum. For neurons with maturity ≥ 100 ICP, all percentages produce a value ≥ 1 ICP, so the attacker receives `CallerIsNotNeuronController` for every probe and learns nothing new.

### Impact Explanation
`maturity_e8s_equivalent` is deliberately excluded from the public `NeuronInfo` type returned by `get_neuron_info` for private neurons. The `get_full_neuron` endpoint that does expose it is gated to controller/hotkey. This side-channel bypasses that privacy boundary for any private neuron holding less than 100 ICP of liquid maturity, leaking exact financial information about the neuron holder to any anonymous caller.

### Likelihood Explanation
The attack requires a single `manage_neuron` update call with `DisburseMaturity { percentage_to_disburse: 1, ... }` targeting any known neuron ID. Neuron IDs are enumerable via `get_neuron_ids` (a public query). The call costs a trivial amount of cycles. No privileged access, social engineering, or threshold corruption is required.

### Recommendation
Move the `CallerIsNotNeuronController` check (and `NeuronSpawning` check) to occur **before** the `DisbursementTooSmall` check. Authorization should always be the first substantive gate after input validation. Additionally, consider whether the `DisbursementTooSmall` error message should omit the exact `disbursement_maturity_e8s` value when the caller is not authorized.
