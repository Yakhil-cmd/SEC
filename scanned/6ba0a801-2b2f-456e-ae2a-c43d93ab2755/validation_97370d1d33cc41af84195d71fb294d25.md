### Title
Orchestrator `get_latest_cup` Uses Height-Only `max_by_key` Comparison, Allowing Byzantine Peer to Displace Recovery CUP at Equal Height — (`rs/orchestrator/src/catch_up_package_provider.rs`)

---

### Summary

The orchestrator's `get_latest_cup` function selects the "best" CUP from `[local_cup, registry_cup, subnet_cup]` using `max_by_key(get_cup_proto_height)`, which compares only by block height. When a peer-served signed CUP and the registry recovery CUP share the same height but differ in `registry_version`, the peer CUP wins by tie-breaking position (Rust's `max_by_key` returns the **last** maximum element). A Byzantine subnet peer can exploit this by serving a legitimately signed but older CUP at the recovery height, causing the orchestrator to start the replica on the pre-recovery state.

---

### Finding Description

**The `CatchUpPackageParam` ordering** is explicitly defined to break ties at equal height using `registry_version`: [1](#0-0) 

```
C1 > C2 iff
  C1.height > C2.height ||
  C1.height == C2.height && C1.registry_version > C2.registry_version
```

**The final CUP selection in `get_latest_cup`** ignores this ordering entirely: [2](#0-1) 

`get_cup_proto_height` extracts only the raw block height from the protobuf bytes: [3](#0-2) 

Because Rust's `Iterator::max_by_key` returns the **last** element when keys are equal, and the vector is ordered `[local_cup, registry_cup, subnet_cup]`, the `subnet_cup` (peer-provided) always wins over `registry_cup` when both have the same height.

**The peer CUP passes all prior guards:**

In `get_peer_cup`, the filter baseline is the *local* CUP (height H-1), not the registry CUP (height H): [4](#0-3) 

A peer CUP at height H with `registry_version R1` satisfies `(H, R1) > (H-1, R_local)` since `H > H-1`, so it passes.

**Signature verification** in `fetch_and_verify_catch_up_package` verifies the threshold signature using the CUP's own `block.context.registry_version`: [5](#0-4) 

An old but legitimately signed CUP (produced by the subnet during normal operation before recovery) carries a valid combined threshold signature and passes this check.

**The registry CUP is unsigned** (empty signature bytes): [6](#0-5) 

---

### Impact Explanation

In a subnet recovery scenario:
- Registry CUP: height H, `registry_version R2` (new DKG transcripts, updated membership, unsigned)
- Byzantine peer CUP: height H, `registry_version R1 < R2` (old DKG transcripts, old membership, validly signed)

`max_by_key(get_cup_proto_height)` selects the peer CUP. The orchestrator persists and starts the replica on the old CUP. The replica uses `R1`-era DKG transcripts and subnet membership, potentially including nodes that the recovery was intended to remove. Those nodes can participate in consensus under the old membership, defeating the recovery's security intent.

The `ConsensusPoolImpl::init_genesis` correctly uses `CatchUpPackageParam` ordering: [7](#0-6) 

But the orchestrator's `get_latest_cup` does not, creating an inconsistency.

---

### Likelihood Explanation

- Requires a subnet recovery scenario (uncommon but a defined operational procedure on the IC).
- The Byzantine peer must be a node still listed in the subnet's node records at the queried registry version — realistic if the recovery CUP and the removal registry update are at the same height.
- No key forgery or threshold corruption required: the attacker serves a legitimately signed CUP from the subnet's prior operation.
- A single Byzantine node below the fault threshold suffices.

---

### Recommendation

Replace `max_by_key(get_cup_proto_height)` with a comparator that uses `CatchUpPackageParam` ordering as a tiebreaker when heights are equal. Specifically, when two CUPs have the same height, prefer the one with the higher `registry_version`. This aligns with the existing `CatchUpPackageParam::partial_cmp` semantics already used in `init_genesis` and `get_peer_cup`.

---

### Proof of Concept

Differential test:

```rust
// Two CUPs at the same height, different registry_version
let registry_cup = make_cup_proto(height: H, registry_version: R2); // unsigned, from registry
let peer_cup     = make_cup_proto(height: H, registry_version: R1); // signed, R1 < R2

// Current behavior (height-only):
let selected = vec![registry_cup.clone(), peer_cup.clone()]
    .into_iter()
    .max_by_key(get_cup_proto_height);
assert_eq!(selected, Some(peer_cup)); // peer CUP wins — WRONG

// Expected behavior (CatchUpPackageParam ordering):
let selected_param = vec![registry_cup.clone(), peer_cup.clone()]
    .into_iter()
    .max_by(|a, b| {
        CatchUpPackageParam::try_from(a).unwrap()
            .partial_cmp(&CatchUpPackageParam::try_from(b).unwrap())
            .unwrap_or(Ordering::Equal)
    });
assert_eq!(selected_param, Some(registry_cup)); // registry CUP wins — CORRECT
```

impl PartialOrd for CatchUpPackageParam {
    fn partial_cmp(&self, other: &CatchUpPackageParam) -> Option<Ordering> {
        match (
            self.height.cmp(&other.height),
            self.registry_version.partial_cmp(&other.registry_version),
        ) {
            // If height is less, registry version needs to be less or equal
            (Ordering::Less, Some(x)) if x != Ordering::Greater => Some(Ordering::Less),
            // If height is equal, registry version decides ordering
            (Ordering::Equal, Some(x)) => Some(x),
            // If height is greater, registry version needs to be equal or greater
            (Ordering::Greater, Some(x)) if x != Ordering::Less => Some(Ordering::Greater),
            // All other combinations of height and registry versions are incomparable
            // This also covers the case, that the registry versions themselves are incomparable
            _ => None,
        }
    }
