### Title
`ClaimedSwapNeuronStatus::AlreadyExists` Incorrectly Mapped to `ClaimedStatus::Invalid`, Permanently Halting SNS Swap Finalization - (`File: rs/sns/swap/src/types.rs`)

---

### Summary

In the SNS Swap canister's finalization flow, when SNS Governance returns `ClaimedSwapNeuronStatus::AlreadyExists` for a neuron (meaning it was already successfully created in a prior partial execution), the Swap canister maps this to `ClaimedStatus::Invalid` instead of `ClaimedStatus::Success`. This causes `is_successful_sweep()` to return `false`, which triggers `set_claim_neuron_result` to set an error message and halt finalization. On subsequent retries, the recipe's `claimed_status` is now `Invalid`, causing `to_neuron_recipe()` to permanently skip it as `ConversionError::Invalid`, incrementing `sweep_result.invalid` again. The SNS Governance canister is permanently stuck in restricted mode (never reaching `Normal` mode), bricking all governance operations for SNS participants until a manual canister upgrade intervention.

---

### Finding Description

**Root cause — wrong status mapping in `rs/sns/swap/src/types.rs`:**

```rust
impl From<ClaimedSwapNeuronStatus> for ClaimedStatus {
    fn from(claimed_swap_neuron_status: ClaimedSwapNeuronStatus) -> Self {
        match claimed_swap_neuron_status {
            ClaimedSwapNeuronStatus::Success       => ClaimedStatus::Success,
            ClaimedSwapNeuronStatus::Unspecified   => ClaimedStatus::Failed,
            ClaimedSwapNeuronStatus::MemoryExhausted => ClaimedStatus::Failed,
            ClaimedSwapNeuronStatus::Invalid       => ClaimedStatus::Invalid,
            ClaimedSwapNeuronStatus::AlreadyExists => ClaimedStatus::Invalid,  // ← BUG
        }
    }
}
```

`AlreadyExists` means the neuron **was already successfully created** in SNS Governance. It is semantically equivalent to `Success` from the Swap canister's perspective. Mapping it to `Invalid` is incorrect. [1](#0-0) 

**Step-by-step failure chain:**

1. `finalize_swap` is called (publicly callable by any principal via `rs/sns/swap/canister/canister.rs`). [2](#0-1) 

2. `finalize_inner` calls `claim_swap_neurons`, which calls `batch_claim_swap_neurons` in batches. [3](#0-2) 

3. A batch is sent to SNS Governance. SNS Governance successfully claims the neurons and commits state. However, a `CanisterCallError` occurs on the response path (e.g., response too large, system-level issue). The Swap canister increments `global_failures` and returns early — the neurons in that batch remain `ClaimedStatus::Pending`. [4](#0-3) 

4. On retry, `to_neuron_recipe()` sees `ClaimedStatus::Pending` and includes those neurons in the next batch. SNS Governance, having already committed them, returns `ClaimedSwapNeuronStatus::AlreadyExists`. [5](#0-4) 

5. `process_swap_neuron` maps `AlreadyExists` → `ClaimedStatus::Invalid` → increments `sweep_result.invalid`. The recipe's `claimed_status` is written as `ClaimedStatus::Invalid`. [6](#0-5) 

6. `is_successful_sweep()` returns `false` because `invalid > 0`. [7](#0-6) 

7. `set_claim_neuron_result` sets an error message and halts finalization — `set_sns_governance_to_normal_mode` is never called. [8](#0-7) 

8. On every subsequent retry, `to_neuron_recipe()` sees `ClaimedStatus::Invalid` and returns `ConversionError::Invalid`, incrementing `sweep_result.invalid` again — **permanently blocking finalization**. [9](#0-8) 

---

### Impact Explanation

SNS Governance is initialized in a restricted mode during the swap and is only transitioned to `Normal` mode by `set_sns_governance_to_normal_mode` at the end of `finalize_inner`. If finalization is permanently halted, SNS Governance **never exits restricted mode**. All SNS governance operations (proposals, voting, neuron management) are blocked for all participants. The SNS is effectively bricked. Recovery requires a manual canister upgrade to reset the affected `SnsNeuronRecipe.claimed_status` fields from `Invalid` back to `Pending`. [10](#0-9) 

---

### Likelihood Explanation

The trigger condition — a `CanisterCallError` where SNS Governance already committed the batch — is a realistic failure mode in the IC's asynchronous messaging model. The IC does not guarantee that a `CanisterCallError` implies the callee rolled back; the code itself acknowledges this with "we hope that the canister being called rolls back." Any transient system-level issue (response encoding failure, subnet overload, canister memory pressure during response delivery) can produce this scenario. No privileged access, no threshold attack, and no social engineering is required. Any caller of `finalize_swap` can trigger the retry path. [11](#0-10) 

---

### Recommendation

Map `ClaimedSwapNeuronStatus::AlreadyExists` to `ClaimedStatus::Success` in `rs/sns/swap/src/types.rs`, since a neuron that already exists in SNS Governance is, from the Swap canister's perspective, successfully claimed:

```rust
ClaimedSwapNeuronStatus::AlreadyExists => ClaimedStatus::Success,
```

This mirrors the fix described in the external report: treat "already claimed" as a silent success rather than a permanent failure, allowing the batch to continue and finalization to complete. [12](#0-11) 

---

### Proof of Concept

The existing test `test_process_swap_neuron_successful_cases` in `rs/sns/swap/src/swap.rs` already demonstrates the incorrect behavior: it asserts that `ClaimedSwapNeuronStatus::AlreadyExists` produces `SweepResult { invalid: 1, .. }` and sets `claimed_status` to `ClaimedStatus::Invalid`. This test itself encodes the bug as expected behavior. [13](#0-12) 

A concrete reproduction sequence:

1. Deploy an SNS swap in `Committed` state with N neuron recipes.
2. Call `finalize_swap`. Arrange for the SNS Governance `claim_swap_neurons` inter-canister call to succeed in execution but return a `CanisterCallError` to the Swap canister (e.g., by causing a response-path trap). The neurons remain `Pending` in the Swap journal.
3. Call `finalize_swap` again. The Swap canister re-sends the same neurons. SNS Governance returns `AlreadyExists`. The Swap canister maps this to `Invalid`, sets `claimed_status = Invalid`, and halts with error "Claiming SNS Neurons did not complete fully."
4. All subsequent calls to `finalize_swap` encounter the same `Invalid` recipes, permanently halting finalization. SNS Governance never reaches `Normal` mode. [14](#0-13) [15](#0-14)
