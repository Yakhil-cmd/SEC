### Title
Floor Division Truncation in `calculate_split_neuron_effect` Allows Splitting Neuron Stake Without Proportional Maturity Transfer - (File: rs/nns/governance/src/governance/split_neuron.rs)

### Summary
`calculate_split_neuron_effect` uses integer floor division to compute the proportional maturity, staked maturity, and eight-year gang bonus base to transfer during an NNS neuron split. When the product `maturity × split_amount` is less than `stake`, the division truncates to zero. A neuron controller can repeatedly split off minimum-stake child neurons that receive stake but zero maturity, while the parent retains all maturity despite losing proportional stake.

### Finding Description
In `calculate_split_neuron_effect` (`rs/nns/governance/src/governance/split_neuron.rs:20–42`), all three proportional quantities are computed with the same floor-division pattern:

```rust
let transfer_maturity_e8s: u64 = (source_neuron_maturity_e8s as u128)
    .checked_mul(split_amount_e8s as u128)
    .expect("Two u64s can't overflow when multiplied")
    .checked_div(source_neuron_stake_e8s as u128)   // floor division
    .expect("The input source_neuron_stake_e8s should be greater than zero")
    .try_into()
    .expect("...");
```

The same pattern applies to `transfer_staked_maturity_e8s` (lines 28–34) and `transfer_eight_year_gang_bonus_base_e8s` (lines 35–42).

When `source_neuron_maturity_e8s × split_amount_e8s < source_neuron_stake_e8s`, integer division truncates `transfer_maturity_e8s` to 0. The denominator used is `minted_stake_e8s` (i.e., `cached_neuron_stake_e8s − neuron_fees_e8s`), which does **not** include staked maturity, making the denominator potentially much larger than the numerator.

**Concrete trigger condition.** For NNS, the minimum split amount is `neuron_minimum_stake_e8s + transaction_fee_e8s = 100_000_000 + 10_000 = 100_010_000` e8s. Truncation to zero occurs when:

```
maturity × 100_010_000 < stake
⟺ maturity < stake / 100_010_000
```

For a neuron with stake = 1,000,000,000,000 e8s (10,000 ICP), truncation occurs whenever `maturity < ~9,999 e8s` (≈ 0.0001 ICP). This window applies to any neuron that has recently been created, recently spawned all maturity, or recently disbursed maturity.

**Exploit path.** A neuron controller:
1. Holds a neuron with large `minted_stake_e8s` and small `maturity_e8s_equivalent` (or `staked_maturity_e8s_equivalent`).
2. Calls `split_neuron` with the minimum allowed `amount_e8s` repeatedly (each call is a valid ingress message to the NNS governance canister).
3. Each split: `transfer_maturity_e8s = 0`, so the child neuron receives stake but zero maturity.
4. The parent retains all maturity despite losing proportional stake.
5. The controller then changes the child neuron's controller principal to a buyer, who receives a neuron with stake but no maturity backing.

The call path is: ingress `manage_neuron { command: Split { amount_e8s, memo } }` → `Governance::split_neuron` (governance.rs:2134) → `calculate_split_neuron_effect` (split_neuron.rs:13) → floor division at line 23.

### Impact Explanation
The maturity accounting invariant is violated: after a split, the sum of `(parent_maturity + child_maturity)` should equal the original `parent_maturity`, with each proportional to stake. With truncation, the child receives 0 maturity while the parent retains 100%. If the child neuron's controller is subsequently changed to another principal (a legitimate NNS operation), that principal holds a neuron with real ICP stake but zero maturity — they receive no ICP when spawning maturity, despite holding a proportional share of the original neuron's stake. The original controller retains all maturity and receives a disproportionately large ICP payout when spawning. The same truncation applies to `staked_maturity_e8s_equivalent` and `eight_year_gang_bonus_base_e8s`, compounding the accounting error.

### Likelihood Explanation
Low-to-medium. The truncation window (maturity < stake / min_split) is narrow for neurons that have been accumulating rewards for a long time, but is reliably present for: (a) newly created neurons before significant maturity accumulates, (b) neurons immediately after a `spawn_neuron` or `disburse_maturity` call that zeroed out maturity, and (c) neurons whose `staked_maturity_e8s_equivalent` is small relative to their `cached_neuron_stake_e8s`. Any unprivileged NNS neuron controller can trigger this via standard ingress calls with no special permissions.

### Recommendation
In `calculate_split_neuron_effect`, add a guard: if `source_neuron_maturity_e8s > 0` but the computed `transfer_maturity_e8s == 0`, either revert with an error (analogous to the Sablier fix) or use ceiling division to ensure at least 1 e8s is transferred. The same guard should apply to `transfer_staked_maturity_e8s` and `transfer_eight_year_gang_bonus_base_e8s`. Alternatively, enforce a minimum maturity threshold before allowing splits when maturity is non-zero.

//   Parent: stake = 100 ICP (minimum), maturity = 1 e8s  (all retained)

// Victim holds 1 ICP stake, spawns maturity → receives 0 ICP.
// Original controller spawns parent maturity → receives full 1 e8s worth.
