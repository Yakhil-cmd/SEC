### Title
CMC `remove_subnet_from_authorized_subnet_list` Does Not Remove Deleted Subnets from `default_subnets`, Leaving Stale Entries That Route Canister Creation to Non-Existent Subnets — (File: rs/nns/cmc/src/main.rs)

---

### Summary

The Cycles Minting Canister (CMC) tracks subnets eligible for canister creation in two separate data structures: `authorized_subnets` (a per-principal `BTreeMap<PrincipalId, Vec<SubnetId>>`) and `default_subnets` (a global `Vec<SubnetId>`). The function `remove_subnet_from_authorized_subnet_list`, which is the only removal hook callable by the registry canister, only removes the subnet from `authorized_subnets` and silently ignores `default_subnets`. When a subnet is deleted from the registry, the stale entry in `default_subnets` persists, causing canister creation requests to be routed to a non-existent subnet, wasting cycles and causing guaranteed failures for any caller who explicitly targets that subnet.

---

### Finding Description

In `rs/nns/cmc/src/main.rs`, the CMC state holds two independent lists of subnets:

```
default_subnets: Vec<SubnetId>          // global fallback list
authorized_subnets: BTreeMap<PrincipalId, Vec<SubnetId>>  // per-principal list
``` [1](#0-0) 

The `remove_subnet_from_authorized_subnet_list` function, restricted to the registry canister caller, only iterates over `authorized_subnets` and retains entries that do not match the subnet to remove:

```rust
with_state_mut(|state| {
    state
        .authorized_subnets
        .values_mut()
        .for_each(|subnet_list| subnet_list.retain(|subnet| *subnet != subnet_to_remove))
});
``` [2](#0-1) 

`default_subnets` is never touched. There is no corresponding removal path for `default_subnets` in this function or anywhere else triggered by subnet deletion.

The `do_delete_subnet` mutation in the registry canister removes the subnet from `subnet_list`, the subnet record, the catch-up package, the threshold signing key, and the routing table — but does not call `remove_subnet_from_authorized_subnet_list` on the CMC at all: [3](#0-2) 

Even if it did call that function, `default_subnets` would still retain the stale entry.

The stale entry in `default_subnets` is then consumed by `do_create_canister`. When `subnet_selection` is `None`, `get_subnets_for` returns `default_subnets` and a random subnet is chosen from it. When `subnet_selection` is `SubnetSelection::Subnet { subnet }`, the CMC explicitly validates the requested subnet against `default_subnets.contains(&subnet)` and permits the call if it matches: [4](#0-3) 

In both cases, a deleted subnet that remains in `default_subnets` passes the guard and is forwarded to `ic00::create_canister`, which will fail because the subnet no longer exists in the replicated state.

---

### Impact Explanation

Any unprivileged ingress caller invoking `create_canister`, `notify_create_canister`, or `notify_top_up` on the CMC after a subnet deletion may have their request routed to the deleted subnet. The concrete impacts are:

1. **Cycles/ICP loss**: The caller pays cycles or ICP before the CMC attempts creation. If the downstream call to `ic00` fails and the CMC's error-handling path does not fully refund (e.g., due to a trap or a partial-refund edge case), the caller loses funds.
2. **Targeted denial of service**: A caller who explicitly specifies the deleted subnet via `SubnetSelection::Subnet` is guaranteed to fail, because `default_subnets.contains(&subnet)` returns `true` for the stale entry, bypassing the "subnet not authorized" guard. This is a resource-accounting bug: the caller is charged for an operation that cannot succeed.
3. **Misleading query responses**: `get_default_subnets()` returns the stale list, so any client querying available subnets receives incorrect information, leading to repeated failed creation attempts. [5](#0-4) 

---

### Likelihood Explanation

The scenario requires: (a) a subnet to be present in `default_subnets` (set via a governance proposal calling `set_authorized_subnetwork_list` with `who = None`), and (b) that subnet to subsequently be deleted via `do_delete_subnet`. Currently `do_delete_subnet` is restricted to `SubnetType::CloudEngine` subnets: [6](#0-5) 

As CloudEngine subnets are deployed and potentially added to `default_subnets` for general canister creation, the window for this inconsistency grows. The governance proposals required are legitimate administrative actions, not adversarial. Any user calling the CMC after such a sequence is an unprivileged ingress sender who triggers the bug without any special access.

---

### Recommendation

1. Update `remove_subnet_from_authorized_subnet_list` to also remove the subnet from `default_subnets`:
   ```rust
   state.default_subnets.retain(|s| *s != subnet_to_remove);
   state.authorized_subnets.values_mut()
       .for_each(|list| list.retain(|s| *s != subnet_to_remove));
   ```
2. Update `do_delete_subnet` in the registry canister to call `remove_subnet_from_authorized_subnet_list` on the CMC as part of the deletion flow, analogous to how it already removes routing table shards.
3. Consider renaming `remove_subnet_from_authorized_subnet_list` to `remove_subnet_from_all_subnet_lists` to make its intended scope clear, mirroring the suggestion in the external report to rename `removeMarket` to `deactivateMarket`.

---

### Proof of Concept

1. Governance proposal A: call `set_authorized_subnetwork_list(who = None, subnets = [subnet_X])` on the CMC — `subnet_X` is now in `default_subnets`.
2. Governance proposal B: call `do_delete_subnet(subnet_id = subnet_X)` on the registry canister — `subnet_X` is removed from the registry's `subnet_list` and its record is deleted. `remove_subnet_from_authorized_subnet_list` is **not** called; `default_subnets` still contains `subnet_X`.
3. Any user calls `create_canister` on the CMC with no `subnet_selection`. `get_subnets_for` returns `default_subnets` including `subnet_X`. If `subnet_X` is randomly selected, the call to `ic00::create_canister` on the deleted subnet fails.
4. Alternatively, a user calls `create_canister` with `subnet_selection = SubnetSelection::Subnet { subnet: subnet_X }`. The CMC checks `default_subnets.contains(&subnet_X)` → `true`, proceeds, and the downstream call fails unconditionally.
5. `get_default_subnets()` query still returns `subnet_X` as a valid deployment target, misleading all clients. [1](#0-0) [7](#0-6) [8](#0-7)

    pub fn do_delete_subnet(&mut self, payload: DeleteSubnetPayload) -> Result<(), String> {
        println!("{LOG_PREFIX}do_delete_subnet: {payload:?}");

        let DeleteSubnetPayload { subnet_id } = payload;
        let subnet_id_ = SubnetId::from(PrincipalId::from(subnet_id));
        let subnet_record = self.get_subnet(subnet_id_, self.latest_version())?;

        // Currently, only CloudEngines can be deleted.
        if subnet_record.subnet_type != i32::from(SubnetType::CloudEngine) {
            return Err("Only CloudEngines may be deleted".to_string());
        }

        // Remove from `subnet_list`.
        let mut subnet_list = self.get_subnet_list_record().subnets;
        let len_before = subnet_list.len();
        subnet_list.retain(|s| s != subnet_id.as_slice());
        if subnet_list.len() > len_before - 1 {
            println!(
                "{LOG_PREFIX}do_delete_subnet: Subnet {} was not present in subnet_list.",
                subnet_id
            );
        }
        let new_subnet_list_record = SubnetListRecord {
            subnets: subnet_list,
        };
        let subnet_list_mutation = update(
            make_subnet_list_record_key().as_bytes(),
            new_subnet_list_record.encode_to_vec(),
        );

        // Remove catch up package.
        let subnet_dkg_mutation = delete(make_catch_up_package_contents_key(subnet_id_).as_bytes());

        // Remove pubkey.
        let subnet_threshold_signing_pubkey_mutation =
            delete(make_crypto_threshold_signing_pubkey_key(subnet_id_).as_bytes());

        // Remove subnet record.
        let remove_subnet_mutation = delete(make_subnet_record_key(subnet_id_).into_bytes());

        // Remove routing table shards.
        let mut remove_from_routing_table_mutations =
            self.remove_subnet_from_routing_table(self.latest_version(), subnet_id_);
        let mut mutations = vec![
            subnet_list_mutation,
            subnet_dkg_mutation,
            subnet_threshold_signing_pubkey_mutation,
            remove_subnet_mutation,
        ];
        mutations.append(&mut remove_from_routing_table_mutations);

        // Check invariants before applying mutations
        self.maybe_apply_mutation_internal(mutations);
        Ok(())
    }
