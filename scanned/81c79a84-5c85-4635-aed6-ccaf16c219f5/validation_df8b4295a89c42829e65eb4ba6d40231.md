### Title
Silent Discard of Neurons' Fund Maturity Refund Failure Causes Permanent Maturity Loss When Neuron Is Disbursed During Active SNS Swap - (`rs/nns/governance/src/governance.rs`)

---

### Summary

When a `CreateServiceNervousSystem` proposal executes, the NNS Governance draws maturity from Neurons' Fund (NF) neurons and reserves it for the SNS swap. If a participating neuron is disbursed (removed from the store) between the draw and the swap's settlement, the subsequent `refund_maturity_to_neurons_fund` call silently fails and the reserved-but-unused maturity is permanently destroyed. The error is explicitly discarded with `let _ = ...` in `settle_neurons_fund_participation`.

---

### Finding Description

**Step 1 – Maturity draw (proposal execution):**
When a `CreateServiceNervousSystem` proposal is executed, `draw_maturity_from_neurons_fund` is called, which reduces `maturity_e8s_equivalent` on each participating NF neuron. This call propagates errors with `?`, so it cannot silently fail. [1](#0-0) 

**Step 2 – Neuron disbursement (user action during swap):**
SNS swaps run for days or weeks. During this window, any NF neuron owner can call `disburse` on their neuron. The neuron's stake is sent to the owner and the neuron is removed from the stable neuron store. No guard prevents disbursement while NF maturity is reserved. The neuron's `maturity_e8s_equivalent` was already reduced in Step 1, so the owner receives their stake but the drawn maturity is now unattached to any live neuron.

**Step 3 – Refund silently fails (swap settlement):**
When the swap concludes, `settle_neurons_fund_participation` computes the refund (reserved minus actually-used maturity) and calls `refund_maturity_to_neurons_fund`. This calls `apply_neurons_fund_snapshot`, which iterates over the snapshot and calls `modify_neuron_maturity` for each neuron. [2](#0-1) 

`modify_neuron_maturity` calls `with_main_part_mut` on the stable neuron store. If the neuron was disbursed, it no longer exists, and a `NeuronStoreError` is returned. [3](#0-2) 

The error is collected into `neurons_fund_action_error` and returned as `Err(String)` from `apply_neurons_fund_snapshot`. Back in `settle_neurons_fund_participation`, this error is **explicitly discarded**: [4](#0-3) 

The developer comment reads: *"If refunding failed for whatever reason, we opt for providing data to the SNS Swap canister … merely logging the error for human inspection."* The maturity that was drawn but not used is never returned to anyone and is permanently destroyed.

---

### Impact Explanation

This is a **ledger conservation bug**. Maturity drawn from a Neurons' Fund neuron that is subsequently disbursed before swap settlement is permanently lost. Maturity has direct monetary value: it can be spawned into ICP tokens. The loss is proportional to the difference between the reserved (maximum) participation amount and the actual participation amount for that neuron, which can be substantial for large NF neurons. The lost maturity is not credited to any account or neuron — it simply disappears from the system's accounting.

---

### Likelihood Explanation

- SNS swaps run for days to weeks, providing a wide window.
- Any NNS neuron holder whose neuron is in the Neurons' Fund can disburse their neuron at any time; no protocol-level lock prevents this during an active swap.
- The action (disbursing a neuron) is a standard, unprivileged ingress call available to any neuron controller.
- The scenario is reachable without any privileged access, social engineering, or threshold corruption.

---

### Recommendation

1. **Propagate the refund error** instead of discarding it. If `refund_maturity_to_neurons_fund` fails, the settlement should either retry or surface the failure so that a governance hot-fix can be applied before the maturity is lost.
2. **Guard disbursement while NF maturity is reserved.** Before allowing a neuron to be fully disbursed, check whether it appears in any active `initial_neurons_fund_participation` snapshot and reject the disbursement (or require the NF participation to be settled first).
3. **Alternatively**, record the failed refund amount in a dedicated recovery ledger so that a subsequent NNS proposal can manually restore the maturity to the neuron owner's account.

---

### Proof of Concept

1. NF neuron N has `maturity_e8s_equivalent = 1_000_000_000`.
2. A `CreateServiceNervousSystem` proposal executes; `draw_maturity_from_neurons_fund` reduces N's maturity to `800_000_000` (reserving `200_000_000` e8s).
3. The SNS swap opens. During the swap window, N's controller calls `disburse` on N. N is removed from the neuron store.
4. The swap commits with only `50_000_000` e8s of actual NF participation from N. The refund snapshot contains `150_000_000` e8s owed back to N.
5. `settle_neurons_fund_participation` calls `refund_maturity_to_neurons_fund`. `modify_neuron_maturity` returns `NeuronStoreError` (neuron not found). The error is logged and discarded via `let _ = ...`.
6. `150_000_000` e8s of maturity (~1.5 ICP equivalent) is permanently destroyed. The neuron owner receives no compensation.

fn apply_neurons_fund_snapshot(
    neuron_store: &mut NeuronStore,
    snapshot: &NeuronsFundSnapshot,
    action: NeuronsFundAction,
) -> Result<(), String> {
    let mut neurons_fund_action_error = vec![];
    for (neuron_id, neuron_delta) in snapshot.neurons().iter() {
        let action_result = neuron_store.modify_neuron_maturity(neuron_id, |old_maturity| {
            action
                .checked_apply(old_maturity, neuron_delta.amount_icp_e8s)
                .map_err(|verb| {
                    let maturity_delta_e8s = neuron_delta.amount_icp_e8s;
                    format!(
                        "u64 overflow while {verb} maturity from {neuron_id:?} \
                            (*kept* original maturity e8s = {old_maturity}; \
                            requested maturity delta e8s = {maturity_delta_e8s})."
                    )
                })
        });
        if let Err(with_neuron_mut_error) = action_result {
            neurons_fund_action_error.push(with_neuron_mut_error.to_string());
        }

    pub fn modify_neuron_maturity(
        &mut self,
        neuron_id: &NeuronId,
        modify: impl FnOnce(u64) -> Result<u64, String>,
    ) -> Result<(), NeuronStoreError> {
        with_stable_neuron_store_mut(|stable_neuron_store| {
            stable_neuron_store
                .with_main_part_mut(*neuron_id, |neuron| -> Result<(), String> {
                    let new_maturity = modify(neuron.maturity_e8s_equivalent)?;
                    neuron.maturity_e8s_equivalent = new_maturity;
                    Ok(())
                })?
                .map_err(|e| NeuronStoreError::InvalidData { reason: e })?;
            Ok(())
        })
    }
