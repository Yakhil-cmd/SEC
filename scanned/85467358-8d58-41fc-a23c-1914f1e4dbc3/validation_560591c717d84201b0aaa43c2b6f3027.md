### Title
Unauthenticated `notify_top_up` Allows Front-Running to Bypass Subnet Rental Rate Limit - (File: `rs/nns/cmc/src/main.rs`)

### Summary

The `notify_top_up` endpoint in the Cycles Minting Canister (CMC) has no caller-based access control. Any unprivileged ingress sender or canister can call it with any valid `block_index` and `canister_id`. The rate-limiter selection is derived from caller identity at call time, not from the ledger block. An attacker who front-runs the subnet rental canister's notification causes the top-up to execute under the base rate limit (150T cycles/hour) instead of the privileged subnet rental limit (500,000T cycles/month). For large top-ups this produces a permanent `NotifyError::Refunded`, which is non-retriable, so the block is permanently poisoned and the subnet rental canister cannot recover without sending a fresh ICP transfer.

### Finding Description

`notify_top_up` is a public `#[update]` method with no caller restriction: [1](#0-0) 

The rate-limiter is chosen based solely on `caller()`: [2](#0-1) 

`fetch_transaction` validates only that the ledger block's destination subaccount matches `Subaccount::from(&canister_id)` and that the memo is `MEMO_TOP_UP_CANISTER`. It does not check who is calling: [3](#0-2) 

After `fetch_transaction` returns, the block is atomically claimed with `NotificationStatus::Processing`: [4](#0-3) 

Because the IC scheduler can interleave async continuations, an attacker who submits `notify_top_up(N, SUBNET_RENTAL_CANISTER_ID)` before the subnet rental canister's own call resumes after its ledger query will win the race and claim the block first — with `CyclesMintingLimiterSelector::BaseLimit` instead of `SubnetRentalLimit`.

When `process_top_up` then fails due to the base-limit cap, the result is stored as `NotifiedTopUp(Err(Refunded {...}))`: [5](#0-4) 

`NotifyError::Refunded` is explicitly documented as non-retriable: [6](#0-5) 

Because `is_transient_error` returns `false` for `Refunded`, the block entry is **not** removed: [7](#0-6) 

The subnet rental canister's subsequent call hits the `Entry::Occupied` branch and receives the cached `Err(Refunded {...})`. It cannot retry the same block.

The subnet rental canister's privileged limit is `SUBNET_RENTAL_CYCLES_MINTING_LIMIT` (500,000T cycles/month), far above the base limit of 150T cycles/hour, as confirmed by the integration test: [8](#0-7) 

### Impact Explanation

A sustained attacker can permanently invalidate every large ICP transfer the subnet rental canister makes to the CMC. Each poisoned block forces the subnet rental canister to pay a new ICP transfer fee and retry. Because the subnet rental canister manages subnet provisioning — a critical NNS function — repeated disruption can delay or block subnet creation. The attacker pays only the cost of submitting ingress messages; the victim loses ICP transfer fees on every failed attempt.

### Likelihood Explanation

The ICP ledger is a public canister. Any observer can poll `query_blocks` to detect a transfer to `AccountIdentifier(CMC_ID, Subaccount::from(SUBNET_RENTAL_CANISTER_ID))` with `MEMO_TOP_UP_CANISTER`. The subnet rental canister's ID is a well-known constant. The attacker submits `notify_top_up(block_index, SUBNET_RENTAL_CANISTER_ID)` in the next consensus round. Because the subnet rental canister's own `notify_top_up` call must also traverse an async ledger query before it can claim the block, the attacker has a realistic window to win the race. The attack is fully automatable with no privileged access required.

### Recommendation

Restrict the `SubnetRentalLimit` path so that it is only reachable when the caller is actually the subnet rental canister. Concretely, add a guard before `fetch_transaction`:

```rust
if canister_id.get() == SUBNET_RENTAL_CANISTER_ID.get()
    && caller() != SUBNET_RENTAL_CANISTER_ID.get()
{
    return Err(NotifyError::Other {
        error_code: NotifyErrorCode::Unauthorized as u64,
        error_message: "Only the subnet rental canister may notify top-ups for itself".into(),
    });
}
```

Alternatively, derive the rate-limiter from the ledger block's source account rather than from the caller, so that the privilege cannot be stripped by a third-party notification.

### Proof of Concept

1. Subnet rental canister sends 5 000 ICP to `AccountIdentifier(CMC, Subaccount::from(SUBNET_RENTAL_CANISTER_ID))` with `MEMO_TOP_UP_CANISTER`. At 100 XDR/ICP this equals ≈ 500 000T cycles, within `SubnetRentalLimit` but far above `BaseLimit`.
2. Attacker observes ledger block `N` via `query_blocks`.
3. Attacker submits ingress `notify_top_up({ block_index: N, canister_id: SUBNET_RENTAL_CANISTER_ID })`.
4. Attacker's async continuation resumes first; `limiter_to_use = BaseLimit`; `process_top_up` hits the 150T/hour cap and returns `Err(Refunded { ... })`.
5. `blocks_notified[N]` is set to `NotifiedTopUp(Err(Refunded { ... }))` and is **not** cleared (non-transient error).
6. Subnet rental canister's continuation resumes, finds `Entry::Occupied(NotifiedTopUp(Err(Refunded)))`, and returns the cached failure.
7. Subnet rental canister must issue a new ICP transfer; attacker repeats from step 2. [9](#0-8)

#[update]
async fn notify_top_up(
    NotifyTopUp {
        block_index,
        canister_id,
    }: NotifyTopUp,
) -> Result<Cycles, NotifyError> {
    let caller = caller();

    let src_canister_principal = SUBNET_RENTAL_CANISTER_ID.get();
    let limiter_to_use =
        if caller == src_canister_principal && canister_id.get() == src_canister_principal {
            // caller and destination needs to be src_canister_principal to get alternate limiter
            CyclesMintingLimiterSelector::SubnetRentalLimit
        } else {
            CyclesMintingLimiterSelector::BaseLimit
        };

    let (amount, from) = fetch_transaction(
        block_index,
        Subaccount::from(&canister_id),
        MEMO_TOP_UP_CANISTER,
    )
    .await?;

    // Try to set the status of this block to Processing. In order for this to
    // succeed, two conditions must hold:
    //
    //     1. It must not already have a status.
    //
    //     2. The block is "sufficiently recent". More precisely, it must be
    //        more recent than last_purged_notification. (To avoid unbounded
    //        growth of the blocks_notified.)
    let maybe_early_result = with_state_mut(|state| {
        state.purge_old_notifications(MAX_NOTIFY_HISTORY);

        if block_index <= state.last_purged_notification {
            return Some(Err(NotifyError::TransactionTooOld(
                state.last_purged_notification + 1,
            )));
        }

        match state.blocks_notified.entry(block_index) {
            Entry::Occupied(entry) => match entry.get() {
                NotificationStatus::Processing => Some(Err(NotifyError::Processing)),

                // If the user makes a duplicate request, we respond as though
                // the current request is the original one.
                NotificationStatus::NotifiedTopUp(result) => Some(result.clone()),
                NotificationStatus::NotifiedCreateCanister(_) => {
                    Some(Err(NotifyError::InvalidTransaction(
                        "The same payment is already processed as create canister request".into(),
                    )))
                }
                NotificationStatus::NotifiedMint(_) => Some(Err(NotifyError::InvalidTransaction(
                    "The same payment is already processed as mint request".into(),
                ))),
                NotificationStatus::NotMeaningfulMemo(_) => {
                    Some(Err(NotifyError::InvalidTransaction(
                        "The same payment is already processed as automatic refund".into(),
                    )))
                }
            },
            Entry::Vacant(entry) => {
                entry.insert(NotificationStatus::Processing);
                None
            }
        }
    });

    match maybe_early_result {
        Some(result) => result,
        None => {
            let result = process_top_up(canister_id, from, amount, limiter_to_use).await;

            with_state_mut(|state| {
                state.blocks_notified.insert(
                    block_index,
                    NotificationStatus::NotifiedTopUp(result.clone()),
                );
                if is_transient_error(&result) {
                    state.blocks_notified.remove(&block_index);
                }
            });

            result
        }
    }
}

async fn fetch_transaction(
    block_index: BlockIndex,
    expected_to_subaccount: Subaccount,
    expected_memo: Memo,
) -> Result<(Tokens, AccountIdentifier), NotifyError> {
    let ledger_id = with_state(|state| state.ledger_canister_id);

    let block = query_block(block_index, ledger_id).await?;

    let (from, to, amount) = match block.transaction().operation {
        Operation::Transfer {
            from, to, amount, ..
        } => (from, to, amount),
        _ => {
            return Err(NotifyError::InvalidTransaction(
                "Notification transaction must be of type Transfer".into(),
            ));
        }
    };

    let expected_to = AccountIdentifier::new(
        PrincipalId::from(ic_cdk::api::id()),
        Some(expected_to_subaccount),
    );
    if to != expected_to {
        return Err(NotifyError::InvalidTransaction(format!(
            "Destination account in the block ({to}) different than in the notification ({expected_to_subaccount})",
        )));
    }

    issue_automatic_refund_if_memo_not_offerred(
        block_index,
        expected_to_subaccount,
        block.transaction().as_ref(),
    )
    .await?;

    transaction_has_expected_memo(block.transaction().as_ref(), expected_memo)?;

    Ok((amount, from))
}

#[test]
fn test_cmc_subnet_rental_topups_use_separate_limit() {
    let state_machine = state_machine_builder_for_nns_tests().build();

    let unprivilieged_user_account = AccountIdentifier::new(*TEST_USER1_PRINCIPAL, None);
    let subnet_rental_canister_account =
        AccountIdentifier::new(SUBNET_RENTAL_CANISTER_ID.get(), None);
    // The only requirement here is to have sufficient funds to run the test. Other than that,
    // the precise number here does not matter.
    let balance = Tokens::new(1e6 as u64, 0).unwrap();
    let nns_init_payloads = NnsInitPayloadsBuilder::new()
        .with_test_neurons()
        .with_ledger_account(subnet_rental_canister_account, balance)
        .with_ledger_account(unprivilieged_user_account, balance)
        .build();
    setup_nns_canisters(&state_machine, nns_init_payloads);

    // Conversion rate in tests is 100 XDR per ICP, and cycles cost 1 XDR per trillion cycles.
    // To get the ICP needed to mint maximum cycles, we divide by 1 trillion and then by 100.
    let subnet_rental_limit_cost_icp =
        SUBNET_RENTAL_CYCLES_MINTING_LIMIT as u64 / ONE_TRILLION / 100;
    let base_limit_cost_icp = CYCLES_MINTING_LIMIT as u64 / ONE_TRILLION / 100;

    // First top-up should succeed as it's 500,000T cycles, which is at the limit for SRC.
    let cycles = notify_top_up_as(
        &state_machine,
        SUBNET_RENTAL_CANISTER_ID,
        Tokens::new(subnet_rental_limit_cost_icp, 0).unwrap(),
        SUBNET_RENTAL_CANISTER_ID.get(),
    )
    .unwrap();
    assert_eq!(cycles, Cycles::new(SUBNET_RENTAL_CYCLES_MINTING_LIMIT));
