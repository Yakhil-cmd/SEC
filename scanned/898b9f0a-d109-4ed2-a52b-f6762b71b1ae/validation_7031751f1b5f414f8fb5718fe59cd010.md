### Title
NNS Governance `disburse_neuron` Burns All Neuron Fees Without Checking Open Proposals, Permanently Destroying Refundable Reject Costs - (`File: rs/nns/governance/src/governance.rs`)

### Summary

The NNS governance `disburse_neuron` function burns the entire `neuron_fees_e8s` balance unconditionally when a neuron is dissolved and disbursed. It does not check whether any portion of those fees is tied to open (still-voting) proposals whose reject cost would be refunded upon adoption. This mirrors the Taiko H-02 pattern: a bond/fee is permanently destroyed even though the underlying action (the proposal) is ultimately proven correct (adopted). The SNS governance canister already identified and fixed this exact bug in proposal 137687 (2025-08-01), but the NNS governance canister retains the vulnerable code path.

---

### Finding Description

**Vulnerability class:** Ledger conservation bug — funds that should be conditionally returned to a correct participant are instead permanently burned because the history of open proposals is not consulted before burning.

**Root cause in NNS governance `disburse_neuron`:**

When a neuron is dissolved and the owner calls `disburse_neuron`, the function reads the full `neuron.neuron_fees_e8s` as `fees_amount_e8s` with no filtering: [1](#0-0) 

It then burns that entire amount on the ICP ledger and resets `neuron_fees_e8s` to zero unconditionally: [2](#0-1) 

There is no call to any equivalent of `maximum_burnable_fees_for_neuron()` before burning. The NNS governance `disburse_neuron` does not inspect `self.heap_data.proposals` to find open proposals submitted by this neuron whose `reject_cost_e8s` would be refunded on adoption.

**Contrast with the fixed SNS governance:**

The SNS governance `disburse_neuron` was patched (SNS proposal 137687, CHANGELOG entry): [3](#0-2) 

The SNS fix introduced `maximum_burnable_fees_for_neuron()`, which subtracts the sum of `reject_cost_e8s` of all open proposals submitted by the neuron before burning: [4](#0-3) 

The SNS `disburse_neuron` now only burns `max_burnable_fee` and updates `neuron_fees_e8s` by that partial amount: [5](#0-4) 

The NNS governance has no analogous guard.

**Fee-return on adoption is a no-op after premature burn:**

When a proposal is adopted, the NNS governance returns the reject cost by decrementing `neuron_fees_e8s`. But if `disburse_neuron` already set `neuron_fees_e8s = 0` and burned the ICP, the decrement is a no-op (saturating subtraction from 0). No ICP is ever minted back to the neuron owner. The ICP is permanently destroyed. [6](#0-5) 

---

### Impact Explanation

A neuron owner who submits a proposal and then disburses their dissolved neuron before the proposal concludes loses the `reject_cost_e8s` amount of ICP permanently, even if the proposal is subsequently adopted. The ICP is burned on the ledger and the governance accounting field is zeroed, so the adoption-time refund path has nothing to restore. This is a direct, irreversible loss of ICP tokens for the neuron owner.

The `reject_cost_e8s` in NNS is a governance-controlled parameter (currently 10 ICP). Any neuron that submits a proposal and disburses before the voting period ends (including wait-for-quiet extensions) is affected.

---

### Likelihood Explanation

The scenario is reachable by any unprivileged NNS neuron holder:

1. A neuron with a dissolve delay that is about to expire submits a proposal (incrementing `neuron_fees_e8s` by `reject_cost_e8s`).
2. The neuron's dissolve delay expires (state becomes `Dissolved`).
3. The owner calls `manage_neuron` → `Disburse` while the proposal voting period is still open (possible because wait-for-quiet can extend voting periods, or because the neuron dissolved quickly after submitting).
4. `disburse_neuron` burns all fees and sets `neuron_fees_e8s = 0`.
5. The proposal is later adopted; the refund path is a no-op.

This is not a contrived race condition — it is a normal operational sequence for any neuron that submits a proposal near the end of its dissolve delay. The NNS voting period is up to several days, and wait-for-quiet can extend it further, making the overlap window non-trivial. [7](#0-6) 

---

### Recommendation

Apply the same fix that was applied to SNS governance. Before burning fees in NNS `disburse_neuron`, compute the maximum safely burnable amount by subtracting the sum of `reject_cost_e8s` of all open proposals submitted by the neuron:

```rust
// Compute total reject costs locked in open proposals
let locked_fees: u64 = self.heap_data.proposals.values()
    .filter(|p| p.proposer == Some(*id) && p.status() == ProposalStatus::Open)
    .map(|p| p.reject_cost_e8s)
    .sum();

let fees_amount_e8s = neuron.neuron_fees_e8s.saturating_sub(locked_fees);
```

Only burn `fees_amount_e8s` (the reduced amount), and update `neuron_fees_e8s` by subtracting only the burned amount rather than zeroing it. This mirrors the SNS fix at: [5](#0-4) 

---

### Proof of Concept

1. Create an NNS neuron with dissolve delay set to expire soon (e.g., 6 months + 1 day).
2. Submit a proposal from that neuron. `neuron_fees_e8s` is incremented by `reject_cost_e8s` (10 ICP).
3. Advance time past the dissolve delay so the neuron enters `Dissolved` state.
4. Call `manage_neuron` → `Disburse` while the proposal voting period is still open.
5. Observe: `disburse_neuron` burns all 10 ICP of fees on the ledger and sets `neuron_fees_e8s = 0`.
6. Have enough neurons vote `Yes` to adopt the proposal.
7. Observe: `process_proposal` attempts `neuron_fees_e8s -= reject_cost_e8s` but `neuron_fees_e8s` is already 0; the saturating subtraction is a no-op. No ICP is returned.
8. The neuron owner has permanently lost 10 ICP that should have been refunded.

The existing NNS governance test `test_cant_disburse_without_paying_fees` confirms the burn path is exercised but does not test the open-proposal scenario: [8](#0-7)

    /// Calculate the reject_cost_e8s of a proposal. This value is set in `ProposalData` and
    /// is the amount reimbursed to the proposing neuron if the proposal passes.
    fn reject_cost_e8s(&self, proposal: &Proposal) -> Result<u64, GovernanceError> {
        let action = proposal.action.as_ref().ok_or_else(|| {
            GovernanceError::new_with_message(
                ErrorType::InvalidProposal,
                format!("Proposal lacks an action: {proposal:?}"),
            )
        })?;
        match *action {
            // We don't return proposal submission fee for ManageNeuron proposals.
            // if we did, there would be no cost to creating a bunch of ManageNeuron
            // proposals, because you could always vote to adopt them and get the
            // fee back. Therefore, we set this value to 0 and if the proposal
            // is adopted, 0 e8s is reimbursed to the proposing neuron.
            Action::ManageNeuron(_) => Ok(0),
            // For all other proposals, we return the proposal submission fee.
            _ => self.proposal_submission_fee(proposal),
        }
    }

    fn maximum_burnable_fees_for_neuron(&self, neuron: &Neuron) -> Result<u64, GovernanceError> {
        let neuron_id = neuron.id.as_ref().ok_or_else(|| {
            GovernanceError::new_with_message(ErrorType::NotFound, "Neuron does not have an ID")
        })?;

        // Calculate the total reject costs from all open proposals submitted by this neuron
        let total_open_proposal_reject_costs = self
            .proto
            .proposals
            .values()
            .filter(|proposal_data| {
                // Only consider open proposals where this neuron is the proposer
                proposal_data.proposer.as_ref() == Some(neuron_id)
                    && proposal_data.status() == ProposalDecisionStatus::Open
            })
            .map(|proposal_data| proposal_data.reject_cost_e8s)
            .sum::<u64>();

        // The maximum burnable amount is the total fees minus any fees that are
        // tied up in open proposals (which could potentially be refunded)
        let max_burnable = neuron
            .neuron_fees_e8s
            .saturating_sub(total_open_proposal_reject_costs);

        Ok(max_burnable)
    }
