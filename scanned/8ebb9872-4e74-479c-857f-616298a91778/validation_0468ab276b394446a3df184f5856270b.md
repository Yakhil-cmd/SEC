### Title
Unbounded `IngressQueue` with O(n) Per-Round Expiry Purge Enables Liveness DoS on System Subnets — (`rs/replicated_state/src/canister_state/queues/queue.rs`, `rs/execution_environment/src/scheduler.rs`)

---

### Summary

The `IngressQueue` is explicitly unbounded. On system subnets (NNS), the `ingress_history_max_messages` induction guard is bypassed entirely. The scheduler's `purge_expired_ingress_messages` function, called unconditionally at the start of every round, iterates over every expired message and calls `set_status` once per message — O(n) work with no per-round cap. An unprivileged user can flood the NNS ingress queue with free subnet-method messages timed to expire simultaneously, forcing a single round to process an arbitrarily large batch of expirations, delaying or stalling block production.

---

### Finding Description

**Step 1 — Unbounded queue, explicitly documented.** [1](#0-0) 

The `IngressQueue` struct carries no capacity field and no push-time rejection path. [2](#0-1) 

`push()` unconditionally appends; it never returns an error.

**Step 2 — System subnets bypass the only induction-time guard.**

`ValidSetRuleImpl::enqueue` checks `ingress_history_max_messages` only for non-system subnets: [3](#0-2) 

The NNS is `SubnetType::System`, so the guard is skipped unconditionally. The test suite explicitly verifies this bypass: [4](#0-3) 

**Step 3 — O(n) expiry purge runs every round with no per-round cap.**

`purge_expired_ingress_messages` is called at the start of every execution round: [5](#0-4) 

The function collects all expired messages across the subnet queue and every hot canister, then calls `ingress_history_writer.set_status` once per expired message — a BTreeMap insert plus potential `forget_terminal_statuses` traversal — with no bound on the batch size: [6](#0-5) 

`retain_ingress_messages` itself is O(n) over all messages in the queue: [7](#0-6) 

**Step 4 — Attacker-controlled entry path.**

Ingress messages addressed to subnet methods are inducted at zero cost on system subnets (`IngressInductionCost::Free`): [8](#0-7) 

An unprivileged user submits a large volume of signed ingress messages targeting any free NNS management method (e.g. `canister_status`, `raw_rand`), each with the maximum allowed expiry (`MAX_INGRESS_TTL = 5 minutes`): [9](#0-8) 

Over 5 minutes at ~1 block/second with the default block ingress limit (~1 000 messages/block), up to ~300 000 messages accumulate. When the TTL elapses, all expire in the same round, forcing `purge_expired_ingress_messages` to execute O(300 000) `set_status` calls in a single round with no instruction-limit guard on the scheduler path.

---

### Impact Explanation

The scheduler runs in native Rust outside the Wasm instruction-limit sandbox. An O(300 000) expiry purge in one round causes the replica to spend excessive wall-clock time in round preparation, causing it to miss its consensus slot. Repeated across rounds (the attacker can re-flood after each TTL window), this degrades NNS block production throughput and can stall governance proposal execution, canister upgrades, and subnet management operations that depend on timely NNS rounds.

---

### Likelihood Explanation

The attack requires no privileged access, no cycles (free subnet methods on NNS), and no threshold corruption. It requires only the ability to submit valid signed ingress messages — available to any boundary-node user. The 5-minute accumulation window is realistic. The `ingress_history_max_messages` bypass for system subnets is an intentional design choice (to avoid DoS-ing system canisters), making it unlikely to be closed as a side-effect of other changes.

---

### Recommendation

1. **Bound the per-round expiry purge.** Process at most `K` expired messages per round in `purge_expired_ingress_messages`, deferring the remainder to subsequent rounds. This mirrors the external report's recommendation to process roots in batches.

2. **Apply an ingress-queue size cap to system subnets.** Even if the `ingress_history_max_messages` guard is intentionally bypassed for system subnets, a separate per-canister or per-subnet ingress-queue depth limit should be enforced at induction time to prevent unbounded accumulation.

3. **Rate-limit free subnet-method ingress per sender.** Analogous to the external report's recommendation to restrict the `send` function by requiring a minimum interval between calls from the same origin.

---

### Proof of Concept

```
1. Attacker submits ~300 000 signed ingress messages to the NNS management
   canister (e.g. method `canister_status` with a valid canister ID argument),
   each with expiry_time = now + MAX_INGRESS_TTL (5 min).

2. ValidSetRuleImpl::enqueue skips the ingress_history_max_messages guard
   (SubnetType::System branch), so all messages are inducted into the
   IngressQueue, which has no upper bound.

3. After 5 minutes, all messages expire simultaneously.

4. The next call to execute_round() calls purge_expired_ingress_messages(),
   which:
     a. Calls retain_ingress_messages() — O(300 000) scan of the queue.
     b. Iterates expired_ingress_messages (300 000 entries) and calls
        set_status() for each — O(300 000) BTreeMap operations.

5. The round takes far longer than the consensus slot deadline, causing the
   NNS replica to miss its block-making window and slowing or stalling
   NNS governance operations.
```

    pub(super) fn retain_messages<F>(&mut self, mut predicate: F) -> Vec<Arc<Ingress>>
    where
        F: FnMut(&Ingress) -> bool,
    {
        let mut filtered_messages = vec![];
        for canister_ingress_queue in self.queues.values_mut() {
            canister_ingress_queue.retain_mut(|item| {
                if predicate(item) {
                    return true;
                }
                // Empty `canister_ingress_queues` and their corresponding schedule entry
                // are pruned below.
                filtered_messages.push(Arc::clone(item));
                self.size_bytes -= Self::ingress_size_bytes(&(*item));
                self.total_ingress_count -= 1;
                false
            });
        }

        self.schedule
            .retain_mut(|canister_id| match self.queues.entry(*canister_id) {
                Entry::Occupied(entry) if entry.get().is_empty() => {
                    entry.remove();
                    self.size_bytes -= Self::PER_CANISTER_QUEUE_OVERHEAD_BYTES;
                    false
                }
                Entry::Occupied(_) => true,
                Entry::Vacant(_) => unreachable!(),
            });

        filtered_messages
    }
