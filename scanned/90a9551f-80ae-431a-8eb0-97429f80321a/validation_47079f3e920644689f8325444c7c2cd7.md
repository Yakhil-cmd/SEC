### Title
Silent `wasm_memory_threshold` Mutation in `ManageDappCanisterSettings` Proposal Rendering — (File: `rs/sns/governance/src/proposal.rs`)

---

### Summary

`validate_and_render_manage_dapp_canister_settings` omits `wasm_memory_threshold` from both the "at least one field must be set" guard and the human-readable proposal rendering, while the field is still forwarded to the SNS Root canister and executed. Any SNS neuron holder can craft a `ManageDappCanisterSettings` proposal that visibly changes one setting (e.g., `compute_allocation`) while silently also setting `wasm_memory_threshold` to an arbitrary value — a mutation that voters never see in the proposal text.

---

### Finding Description

In `rs/sns/governance/src/proposal.rs`, `validate_and_render_manage_dapp_canister_settings` iterates over every optional field of `ManageDappCanisterSettings` to:

1. Detect whether at least one field is set (`no_change` guard, line 1866).
2. Build the human-readable rendering shown to voters before they cast ballots.

Every optional field — `compute_allocation`, `memory_allocation`, `freezing_threshold`, `reserved_cycles_limit`, `log_visibility`, `snapshot_visibility`, and `wasm_memory_limit` — is handled in both checks. The field `wasm_memory_threshold` is **absent from both**. [1](#0-0) 

The struct definition confirms `wasm_memory_threshold` is a first-class optional field: [2](#0-1) 

At execution time, `perform_manage_dapp_canister_settings` converts the governance struct directly into a `ManageDappCanisterSettingsRequest` and dispatches it to the SNS Root canister: [3](#0-2) 

The root-side `ManageDappCanisterSettingsRequest` also carries `wasm_memory_threshold`: [4](#0-3) 

Consequently:

- A proposal that sets **only** `wasm_memory_threshold` is incorrectly rejected with `"must change at least one value, all values are None"` — a false negative that silently discards legitimate governance actions.
- A proposal that sets `wasm_memory_threshold` **alongside any other visible field** passes validation, is rendered without mentioning `wasm_memory_threshold`, yet executes the hidden mutation on adoption.

---

### Impact Explanation

An SNS neuron holder submits a `ManageDappCanisterSettings` proposal that visibly proposes, say, a `compute_allocation` change. The rendered text shown to all voters contains only that change. Voters approve. On execution, `wasm_memory_threshold` is also silently applied to every listed dapp canister.

`wasm_memory_threshold` controls the point at which a canister's Wasm memory is considered critically low and triggers low-memory callbacks or operational alerts. Setting it to `u64::MAX` (near or above the actual Wasm memory limit) causes the canister to permanently signal a low-memory condition, potentially disrupting memory-aware logic, triggering unintended upgrade or freeze behavior, or degrading canister availability — all without any voter awareness.

This is a **governance authorization bypass via proposal rendering mismatch**: the on-chain vote is cast on an incomplete description of the state change being authorized.

---

### Likelihood Explanation

The attacker must hold an SNS neuron with sufficient stake and dissolve delay to submit proposals, and the proposal must pass a majority vote. However, since the hidden field is absent from the rendered text, voters have no mechanism to detect the deception from the proposal alone. The attack is straightforward to construct and requires no special privileges beyond normal SNS neuron ownership. Likelihood is **medium**.

---

### Recommendation

Add `wasm_memory_threshold` to both the `no_change` tracking block and the rendered proposal text in `validate_and_render_manage_dapp_canister_settings`, consistent with how every other optional field is handled:

```rust
if let Some(wasm_memory_threshold) = &manage_dapp_canister_settings.wasm_memory_threshold {
    render += &format!("# Set Wasm memory threshold to: {wasm_memory_threshold}\n");
    no_change = false;
}
```

Add a unit test analogous to the existing `validate_and_render_manage_dapp_canister_settings_no_changes` test that verifies a proposal with only `wasm_memory_threshold` set is accepted, and that the rendered text includes the threshold value. [5](#0-4) 

---

### Proof of Concept

Submit the following `ManageDappCanisterSettings` proposal via `manage_neuron` on any SNS governance canister:

```
ManageDappCanisterSettings {
    canister_ids: [<registered_dapp_canister_id>],
    compute_allocation: Some(50),   // visible in rendered text → voters see this
    wasm_memory_threshold: Some(u64::MAX),  // NOT in rendered text → voters never see this
    // all other fields: None
}
```

**Observed**: Proposal passes `validate_and_render_manage_dapp_canister_settings`. Rendered text shows only `"# Set compute allocation to: 50%"`. On adoption, `perform_manage_dapp_canister_settings` forwards both fields to the root canister, which applies `wasm_memory_threshold = u64::MAX` to the dapp canister — a mutation no voter was shown. [6](#0-5)

fn validate_and_render_manage_dapp_canister_settings(
    manage_dapp_canister_settings: &ManageDappCanisterSettings,
) -> Result<String, String> {
    if manage_dapp_canister_settings.canister_ids.is_empty() {
        return Err("ManageDappCanisterSettings must specify at least one canister".to_string());
    }

    if manage_dapp_canister_settings.canister_ids.len() > MAX_NUMBER_OF_DAPPS_TO_MANAGE_PER_PROPOSAL
    {
        return Err(format!(
            "ManageDappCanisterSettings cannot specify more than \
             {MAX_NUMBER_OF_DAPPS_TO_MANAGE_PER_PROPOSAL} canister ids"
        ));
    }

    let num_canisters = manage_dapp_canister_settings.canister_ids.len();
    let canister_list = manage_dapp_canister_settings
        .canister_ids
        .iter()
        .map(|id| format!("  - {id}"))
        .collect::<Vec<_>>()
        .join("\n");
    let mut render = format!(
        "# Proposal to manage settings for {num_canisters} dapp canister{plural}: \n\
        ## Canister ids: \n\
        {canister_list}\n",
        plural = if num_canisters > 1 { "s" } else { "" },
    );

    let mut no_change = true;
    if let Some(compute_allocation) = &manage_dapp_canister_settings.compute_allocation {
        render += &format!("# Set compute allocation to: {compute_allocation}%\n");
        no_change = false;
    }
    if let Some(memory_allocation) = &manage_dapp_canister_settings.memory_allocation {
        render += &format!("# Set memory allocation to: {memory_allocation} bytes\n");
        no_change = false;
    }
    if let Some(freezing_threshold) = &manage_dapp_canister_settings.freezing_threshold {
        render += &format!("# Set freezing threshold to: {freezing_threshold} seconds\n");
        no_change = false;
    }
    if let Some(reserved_cycles_limit) = &manage_dapp_canister_settings.reserved_cycles_limit {
        render += &format!("# Set reserved cycles limit to: {reserved_cycles_limit} \n");
        no_change = false;
    }
    if let Some(log_visibility) = &manage_dapp_canister_settings.log_visibility {
        render += &format!(
            "# Set log visibility to: {:?} \n",
            LogVisibility::try_from(*log_visibility).unwrap_or_default()
        );
        no_change = false;
    }
    if let Some(snapshot_visibility) = &manage_dapp_canister_settings.snapshot_visibility {
        render += &format!(
            "# Set snapshot visibility to: {:?} \n",
            SnapshotVisibility::try_from(*snapshot_visibility).unwrap_or_default()
        );
        no_change = false;
    }
    if let Some(wasm_memory_limit) = &manage_dapp_canister_settings.wasm_memory_limit {
        render += &format!("# Set Wasm memory limit to: {wasm_memory_limit}\n");
        no_change = false;
    }

    if no_change {
        Err(String::from(
            "ManageDappCanisterSettings must change at least one value, all values are None",
        ))
    } else {
        Ok(render)
    }
}

    async fn perform_manage_dapp_canister_settings(
        &self,
        manage_dapp_canister_settings: ManageDappCanisterSettings,
    ) -> Result<(), GovernanceError> {
        let request = ManageDappCanisterSettingsRequest::from(manage_dapp_canister_settings);
        let payload = candid::Encode!(&request).map_err(|err| {
            GovernanceError::new_with_message(
                ErrorType::InvalidProposal,
                format!("Could not encode ManageDappCanisterSettings: {err:?}"),
            )
        })?;
        self.env
            .call_canister(
                self.proto.root_canister_id_or_panic(),
                "manage_dapp_canister_settings",
                payload,
            )
            .await
            .map_err(|err| {
                GovernanceError::new_with_message(
                    ErrorType::External,
                    format!("Canister method call failed: {err:?}"),
                )
            })
            .and_then(
                |reply| match Decode!(&reply, ManageDappCanisterSettingsResponse) {
                    Ok(ManageDappCanisterSettingsResponse { failure_reason }) => failure_reason
                        .map_or(Ok(()), |failure_reason| {
                            Err(GovernanceError::new_with_message(
                                ErrorType::InvalidProposal,
                                format!(
                                    "Failed to manage dapp canister settings: {failure_reason}"
                                ),
                            ))
                        }),
                    Err(error) => Err(GovernanceError::new_with_message(
                        ErrorType::External,
                        format!("Could not decode ManageDappCanisterSettingsResponse: {error}"),
                    )),
                },
            )
    }
