### Title
SNS Treasury Manager `DepositRequest` Interface Lacks Slippage Protection, Enabling Front-Running of DAO Treasury Deposits — (`rs/sns/treasury_manager/treasury_manager.did`, `rs/sns/governance/src/extensions.rs`)

---

### Summary

The `DepositRequest` type defined in the IC's `treasury_manager.did` interface contains no slippage-protection fields (no minimum LP tokens out, no deadline, no price bounds). The SNS governance canister's `execute_treasury_manager_deposit` function constructs and dispatches this request with only token allowance amounts. Because the interface structurally prevents slippage constraints from ever being communicated to the treasury manager, any treasury manager implementation that deposits into a DEX liquidity pool does so with effectively zero tolerance — mirroring the root cause in the original report.

---

### Finding Description

The `DepositRequest` Candid type is defined as:

```candid
type DepositRequest = record {
  allowances : vec Allowance;
};
``` [1](#0-0) 

It carries only token allowance amounts. There are no fields for `min_lp_tokens_out`, `max_price_impact`, or any deadline. The IC codebase itself acknowledges this as a known risk:

> "Some liquidity pools do not implement slippage protection for deposits. In other words, the price ratio at the time of execution may differ from the ratio at the time the proposal was approved." [2](#0-1) 

The same warning appears in the SNS governance proposal renderer:

> "Some Decentralized Exchanges lack slippage protection during deposits… This can expose liquidity pool adaptors to mispricing, making them vulnerable to front-running or sandwich attacks." [3](#0-2) 

The `ValidatedDepositOperationArg` struct — the only validated representation of a deposit operation in governance — contains only `treasury_allocation_sns_e8s`, `treasury_allocation_icp_e8s`, and the raw `original` Precise value. No slippage field exists or is parsed: [4](#0-3) 

`construct_treasury_manager_deposit_payload` builds the `DepositRequest` from these allowances alone: [5](#0-4) 

`execute_treasury_manager_deposit` then approves ICRC-2 allowances and calls `deposit` on the treasury manager canister with this slippage-free payload: [6](#0-5) 

---

### Impact Explanation

When an SNS DAO executes a `TreasuryManagerDeposit` governance proposal, the governance canister approves ICRC-2 allowances and calls `deposit` on the treasury manager (e.g., a KongSwap adaptor). The treasury manager then calls the DEX canister to add liquidity. Because the `DepositRequest` carries no minimum-output constraint, the treasury manager has no basis on which to reject an unfavorable execution price.

On the IC, inter-canister calls are asynchronous. Between the governance canister's `deposit` call and the treasury manager's actual DEX interaction, other messages can be processed. Any canister or user who observes the public governance proposal (proposals are publicly readable) can submit a transaction to the DEX pool to skew the price before the treasury manager's deposit executes, then reverse it afterward — capturing the difference at the expense of the SNS treasury. The SNS treasury holds DAO-owned ICP and SNS tokens; losses are permanent and affect all token holders.

---

### Likelihood Explanation

- Governance proposals are publicly visible on-chain before execution; the deposit amount and target pool are known in advance.
- The KongSwap adaptor was previously a registered extension (its Wasm hashes appear in the IC codebase), and the framework is designed to support future treasury manager implementations.
- Any canister that can interact with the target DEX pool is a sufficient attacker — no privileged access is required.
- The IC's async inter-canister call model creates the ordering window needed for the attack. [7](#0-6) 

---

### Recommendation

Add slippage-protection fields to `DepositRequest` in `treasury_manager.did`:

```candid
type DepositRequest = record {
  allowances : vec Allowance;
  min_lp_tokens_out : opt nat;       // minimum acceptable LP tokens
  max_price_impact_bps : opt nat16;  // e.g. 50 = 0.5%
  deadline_ns : opt nat64;           // reject if executed after this timestamp
};
```

Correspondingly, extend `ValidatedDepositOperationArg` in `rs/sns/governance/src/extensions.rs` to parse and validate these fields from the governance proposal's `Precise` map, and propagate them through `construct_treasury_manager_deposit_payload`. Treasury manager implementations must then enforce these constraints before calling the DEX. [8](#0-7) [1](#0-0) 

---

### Proof of Concept

1. An SNS DAO registers a KongSwap treasury manager extension and passes a `TreasuryManagerDeposit` proposal to deposit 1,000 ICP + 500,000 SNS tokens into a KongSwap ICP/SNS pool.
2. The proposal is publicly visible. An attacker observes it and, before the governance canister's `deposit` call reaches the treasury manager, submits a large swap on the KongSwap pool to skew the ICP/SNS price ratio.
3. `execute_treasury_manager_deposit` approves ICRC-2 allowances and calls `deposit` with a `DepositRequest` containing only `allowances` — no minimum LP output.
4. The treasury manager calls KongSwap's `add_liquidity` at the manipulated price. The DAO receives significantly fewer LP tokens than the fair-price equivalent.
5. The attacker reverses their swap, capturing the spread. The SNS treasury has permanently lost value with no on-chain mechanism to detect or prevent it. [9](#0-8)

/// Execute a treasury manager deposit operation
async fn execute_treasury_manager_deposit(
    governance: &Governance,
    extension_canister_id: CanisterId,
    arg: ValidatedDepositOperationArg,
) -> Result<(), GovernanceError> {
    let ValidatedDepositOperationArg {
        treasury_allocation_sns_e8s,
        treasury_allocation_icp_e8s,
        original,
    } = arg;

    let context = governance.treasury_manager_deposit_context().await?;
    let arg_blob =
        construct_treasury_manager_deposit_payload(context, original).map_err(|err| {
            GovernanceError::new_with_message(
                ErrorType::PreconditionFailed,
                format!("Failed to construct treasury manager deposit payload: {err}"),
            )
        })?;

    // 1. Transfer funds from treasury to treasury manager
    governance
        .approve_treasury_manager(
            extension_canister_id,
            treasury_allocation_sns_e8s,
            treasury_allocation_icp_e8s,
        )
        .await?;

    // 2. Call deposit on treasury manager
    let balances = governance
        .env
        .call_canister(extension_canister_id, "deposit", arg_blob)
        .await
        .map_err(|(code, err)| {
            GovernanceError::new_with_message(
                ErrorType::External,
                format!(
                    "Canister method call {extension_canister_id}.deposit failed with code {code:?}: {err}"
                ),
            )
        })
        .and_then(|blob| {
            Decode!(&blob, sns_treasury_manager::TreasuryManagerResult).map_err(|err| {
                GovernanceError::new_with_message(
                    ErrorType::External,
                    format!("Error decoding TreasuryManager.deposit response: {err:?}"),
                )
            })
        })?
        .map_err(|err| {
            GovernanceError::new_with_message(
                ErrorType::External,
                format!("TreasuryManager.deposit failed: {err:?}"),
            )
        })?;

    log!(
        INFO,
        "TreasuryManager.deposit succeeded with response: {:?}",
        balances
    );

    Ok(())
}

/// Validated deposit operation arguments
#[derive(Debug, Clone)]
pub struct ValidatedDepositOperationArg {
    /// Amount of SNS tokens to allocate from treasury
    pub treasury_allocation_sns_e8s: u64,
    /// Amount of ICP tokens to allocate from treasury
    pub treasury_allocation_icp_e8s: u64,
    /// Original Precise value with all fields
    pub original: Precise,
}

impl TryFrom<Option<Precise>> for ValidatedDepositOperationArg {
    type Error = String;

    fn try_from(value: Option<Precise>) -> Result<Self, Self::Error> {
        let Some(original) = value else {
            return Err("Deposit operation arguments must be provided".to_string());
        };

        let map = match &original.value {
            Some(precise::Value::Map(PreciseMap { map })) => map,
            _ => return Err("Deposit operation arguments must be a PreciseMap".to_string()),
        };

        let treasury_allocation_sns_e8s = map
            .get("treasury_allocation_sns_e8s")
            .and_then(|p| match &p.value {
                Some(precise::Value::Nat(n)) => Some(*n),
                _ => None,
            })
            .ok_or_else(|| "treasury_allocation_sns_e8s must be a Nat value".to_string())?;

        let treasury_allocation_icp_e8s = map
            .get("treasury_allocation_icp_e8s")
            .and_then(|p| match &p.value {
                Some(precise::Value::Nat(n)) => Some(*n),
                _ => None,
            })
            .ok_or_else(|| "treasury_allocation_icp_e8s must be a Nat value".to_string())?;

        Ok(Self {
            treasury_allocation_sns_e8s,
            treasury_allocation_icp_e8s,
            original,
        })
    }
