### Title
Timer-Driven Migration Processing Continues When Canister Is Paused (`disable_api`) - (File: `rs/migration_canister/src/lib.rs`)

### Summary

The migration canister exposes a `disable_api` endpoint to pause all migration activity. The `migrate_canister` ingress endpoint correctly checks the `DISABLED` flag and rejects new requests. However, the timer-driven processing pipeline — which drives already-accepted requests through all subsequent irreversible steps — never checks `migrations_disabled()`. Any request that entered `REQUESTS` stable storage before the disable continues to be processed to completion, including canister deletion and routing-table mutation, even while the canister is nominally paused.

### Finding Description

The `disable_api` / `enable_api` endpoints in `rs/migration_canister/src/privileged.rs` toggle a stable `DISABLED` cell: [1](#0-0) 

`migrations_disabled()` is read and enforced only in the single public ingress endpoint `migrate_canister`: [2](#0-1) 

The `start_timers()` function in `rs/migration_canister/src/lib.rs` registers nine independent 1-second timer callbacks that drive every subsequent processing stage: [3](#0-2) 

None of these callbacks — `process_accepted`, `process_controllers_changed`, `process_stopped`, `process_renamed`, `process_updated`, `process_routing_table`, `process_migrated_canister_deleted`, `process_all_succeeded`, `process_all_failed` — call `migrations_disabled()` before executing. The processing functions themselves also contain no such check: [4](#0-3) [5](#0-4) 

The irreversible steps that continue despite the disable include:

- `set_exclusive_controller` — strips the user of controller rights over their own canisters
- `rename_canister` — renames the replaced canister on the target subnet
- `migrate_canister` (registry call) — updates the routing table, moving the canister ID to a new subnet
- `delete_canister` — permanently deletes the migrated canister from the source subnet [6](#0-5) [7](#0-6) 

### Impact Explanation

When an operator calls `disable_api()` — the intended emergency brake, e.g., upon discovering a bug in the processing pipeline — all requests already present in `REQUESTS` stable storage continue to be driven to completion by the background timers. This means:

1. The migration canister continues to strip users of controller rights over their canisters.
2. The routing table continues to be mutated, redirecting canister IDs to new subnets.
3. The migrated canister continues to be permanently deleted from the source subnet.

These are irreversible on-chain state changes that the operator explicitly intended to halt. The `DISABLED` flag provides a false sense of safety: operators believe they have stopped all migration activity, but the processing pipeline is unaffected.

### Likelihood Explanation

Any user who is a controller of both a migrated and a replaced canister can call `migrate_canister` to submit a request. Once the request is stored in `REQUESTS`, it is processed regardless of the `DISABLED` flag. The scenario is not a narrow race: any request accepted at any point before the disable — including requests submitted days earlier that are still progressing through the multi-step state machine — will continue to completion. The operator has no mechanism to halt in-flight migrations.

### Recommendation

Add a `migrations_disabled()` guard at the top of each timer callback in `start_timers()`, or inside `process_all_by_predicate`, so that when the canister is disabled, the timers become no-ops:

```rust
// In start_timers(), wrap each timer body:
set_timer_interval(interval, async || {
    if migrations_disabled() { return; }
    process_all_by_predicate(
        "accepted",
        |r| matches!(r, RequestState::Accepted { .. }),
        process_accepted,
    ).await
});
```

Alternatively, add the check inside `process_all_by_predicate` itself so all processing stages are covered uniformly. [8](#0-7) 

### Proof of Concept

1. User A (controller of both `migrated_canister` and `replaced_canister`) calls `migrate_canister`. The request is validated and stored as `RequestState::Accepted` in stable `REQUESTS`.
2. An operator discovers a bug and calls `disable_api()`. `DISABLED` is set to `true`.
3. The 1-second timer fires. `process_all_by_predicate("accepted", ...)` runs. It does **not** check `migrations_disabled()`. It calls `process_accepted`, which calls `set_exclusive_controller` on both canisters, stripping User A of controller rights.
4. Subsequent timer ticks advance the request through `ControllersChanged → StoppedAndReady → RenamedReplacedCanister → UpdatedRoutingTable → RoutingTableChangeAccepted → MigratedCanisterDeleted → RestoredControllers`, all without checking `DISABLED`.
5. The migrated canister is permanently deleted from the source subnet, and the routing table is mutated — both irreversible — despite the operator having called `disable_api()`. [3](#0-2) [9](#0-8)

pub async fn process_all_by_predicate<F>(
    tag: &str,
    predicate: impl Fn(&RequestState) -> bool,
    processor: impl Fn(RequestState) -> F,
) where
    F: Future<Output = ProcessingResult<RequestState, RequestState>>,
{
    // Ensures this method runs only once at any given time.
    let Ok(_guard) = MethodGuard::new(tag) else {
        return;
    };
    let mut tasks = vec![];
    let requests = list_by(predicate);
    if requests.is_empty() {
        return;
    }
    println!(
        "Entering `{}` with {} pending requests",
        tag,
        requests.len()
    );
    for request in requests.iter() {
        tasks.push(processor(request.clone()));
    }
    let results = join_all(tasks).await;
    let mut success_counter = 0;
    for (req, res) in zip(requests, results) {
        if res.is_success() {
            success_counter += 1;
        }
        res.transition(req);
    }
    println!(
        "Exiting `{}` with {} successful transitions.",
        tag, success_counter
    );
}

pub async fn process_accepted(
    request: RequestState,
) -> ProcessingResult<RequestState, RequestState> {
    let RequestState::Accepted { request } = request else {
        println!("Error: list_by Accepted returned bad variant");
        return ProcessingResult::NoProgress;
    };

    // Set controller of migrated canister
    let res = set_exclusive_controller(request.migrated_canister)
        .await
        .map_success(|_| RequestState::ControllersChanged {
            request: request.clone(),
        })
        .map_failure(|reason| RequestState::Failed {
            request: request.clone(),
            recovery_state: RecoveryState::new(),
            reason,
        });
    if !res.is_success() {
        return res;
    }

    // Set controller of replaced canister
    set_exclusive_controller(request.replaced_canister)
        .await
        .map_success(|_| RequestState::ControllersChanged {
            request: request.clone(),
        })
        .map_failure(|reason| RequestState::Failed {
            request,
            recovery_state: RecoveryState::new(),
            reason,
        })
}
