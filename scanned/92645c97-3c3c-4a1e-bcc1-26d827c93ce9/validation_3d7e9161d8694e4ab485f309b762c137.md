### Title
Registry Canister Node-Swap Caller/Subnet Allowlist Permanently Bypassed in Production — (`File: rs/registry/canister/src/flags.rs`)

### Summary

The `do_swap_node_in_subnet_directly` endpoint of the NNS registry canister enforces caller and subnet allowlists (`NODE_SWAPPING_CALLERS_POLICY`, `NODE_SWAPPING_SUBNETS_POLICY`) as part of a documented phased rollout. However, the only functions that can set these policies are gated behind `#[cfg(any(test, feature = "test"))]` and are never called in the production binary. Both policies are permanently hardcoded to `AccessList::allow_all()`, making the intended access control completely inoperative on mainnet.

### Finding Description

In `rs/registry/canister/src/flags.rs`, the two phased-rollout access-control policies are initialized as open:

```rust
static NODE_SWAPPING_CALLERS_POLICY: RefCell<AccessList<PrincipalId>> =
    RefCell::new(AccessList::allow_all());   // line 18

static NODE_SWAPPING_SUBNETS_POLICY: RefCell<AccessList<SubnetId>> =
    RefCell::new(AccessList::allow_all());   // line 20
``` [1](#0-0) 

The only setters for these policies live in the `temporary_overrides` module, which is compiled only under `#[cfg(any(test, feature = "test"))]`:

```rust
#[cfg(any(test, feature = "test"))]
pub mod temporary_overrides {
    pub fn test_set_swapping_whitelisted_callers(override_callers: Vec<PrincipalId>) { … }
    pub fn test_set_swapping_enabled_subnets(override_subnets: Vec<SubnetId>) { … }
}
``` [2](#0-1) 

In `canister_init`, the init-payload fields `swapping_whitelisted_callers` and `swapping_enabled_subnets` are read but only applied inside a `#[cfg(feature = "test")]` block:

```rust
#[cfg(feature = "test")]
{
    test_set_swapping_whitelisted_callers(…);
    test_set_swapping_enabled_subnets(…);
}
``` [3](#0-2) 

The `RegistryCanisterInitPayload` struct itself documents this explicitly:

> "Note: in `src/flags.rs` are the default values for all of these arguments and these **shouldn't be provided when deploying to mainnet** and should be left behind the test configuration." [4](#0-3) 

Yet the code comment on the flags also states:

> "These are needed for the phased rollout approach in order to allow granular rolling out of the feature to specific subnets to specific subset of callers." [1](#0-0) 

The enforcement functions `swapping_enabled_for_caller` and `swapping_allowed_on_subnet` call `is_node_swapping_enabled_for_caller` and `is_node_swapping_enabled_on_subnet`, which read the permanently-`allow_all()` policies: [5](#0-4) 

### Impact Explanation

Any node operator (unprivileged ingress sender) can call `swap_node_in_subnet_directly` on the production registry canister and swap nodes on **any** subnet, bypassing the intended phased-rollout caller and subnet restrictions. The only remaining check is that the caller must be the registered node operator of both nodes. The governance-level control intended to gate which operators and which subnets participate in the feature is completely non-functional in production.

**Impact: Medium** — subnet membership can be altered by any node operator without the intended governance-controlled rollout gating.

### Likelihood Explanation

**Likelihood: Medium** — the `swap_node_in_subnet_directly` endpoint is directly reachable by any node operator via ingress. No privileged access is required beyond owning the two nodes being swapped. The feature is live (`IS_NODE_SWAPPING_ENABLED = true` by default) and the allowlists are permanently open. [6](#0-5) 

### Recommendation

Move the policy-setting logic out of the `#[cfg(any(test, feature = "test"))]` gate and into a production-accessible path. Either:

1. Apply the `swapping_whitelisted_callers` and `swapping_enabled_subnets` fields from `RegistryCanisterInitPayload` unconditionally in `canister_init` (not only under `#[cfg(feature = "test")]`), or
2. Add a governance-gated update method (callable only by the governance canister) that can set `NODE_SWAPPING_CALLERS_POLICY` and `NODE_SWAPPING_SUBNETS_POLICY` at runtime.

The `RegistryCanisterInitPayload` fields and `RegistryCanisterInitPayloadBuilder` methods (`whitelist_swapping_feature_caller`, `enable_swapping_feature_for_subnet`) already exist and are correct — the only missing piece is applying them in the production binary path. [7](#0-6) 

### Proof of Concept

1. Deploy the production registry canister (without `feature = "test"`). Both `NODE_SWAPPING_CALLERS_POLICY` and `NODE_SWAPPING_SUBNETS_POLICY` are `allow_all()`.
2. As any node operator who owns two nodes (one assigned to any subnet, one unassigned), send an ingress call to `swap_node_in_subnet_directly` with `old_node_id` and `new_node_id`.
3. `swapping_enabled_for_caller` calls `is_node_swapping_enabled_for_caller(caller)` → returns `true` (policy is `allow_all()`).
4. `swapping_allowed_on_subnet` calls `is_node_swapping_enabled_on_subnet(subnet_id)` → returns `true` (policy is `allow_all()`).
5. The swap proceeds on any subnet, regardless of whether that subnet or that operator was intended to be part of the phased rollout. [8](#0-7)

    /// Check if the caller is whitelisted to use this feature.
    fn swapping_enabled_for_caller(caller: PrincipalId) -> Result<(), SwapError> {
        if !is_node_swapping_enabled_for_caller(caller) {
            return Err(SwapError::FeatureDisabledForCaller { caller });
        }

        Ok(())
    }

    /// Map the `old_node_id` to a subnet and error if it is
    /// not a member of any subnet.
    fn find_subnet_for_old_node(&self, old_node_id: PrincipalId) -> Result<SubnetId, SwapError> {
        find_subnet_for_node(
            self,
            NodeId::new(old_node_id),
            &self.get_subnet_list_record(),
        )
        .ok_or(SwapError::SubnetNotFoundForNode { old_node_id })
    }

    fn swapping_allowed_on_subnet(subnet_id: SubnetId) -> Result<(), SwapError> {
        if !is_node_swapping_enabled_on_subnet(subnet_id) {
            return Err(SwapError::FeatureDisabledOnSubnet { subnet_id });

        Ok(())
    }
