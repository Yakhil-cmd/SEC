### Title
Byzantine Peer Can Craft `ingress_bytes_hash` in `StrippedBlockProposal` to Force Infinite Peer RPC Retries and Block Assembly Stall — (`rs/p2p/artifact_downloader/src/fetch_stripped_artifact/types.rs`)

---

### Summary

The `TryFrom<pb::StrippedIngressMessage> for SignedIngressId` conversion performs no length or content validation on the `ingress_bytes_hash` field. A Byzantine peer can send a `StrippedBlockProposal` with an arbitrary (e.g., empty) `ingress_bytes_hash` for each ingress entry. This causes two independent equality checks — one in `get_or_fetch` (assembler-side pool lookup) and one in `parse_response` (download-side verification) — to permanently fail, trapping the block assembly task in an infinite retry loop until the bouncer eventually marks the block unwanted.

---

### Finding Description

**Deserialization — no hash validation:** [1](#0-0) 

`CryptoHashOf::from(CryptoHash(value.ingress_bytes_hash))` wraps whatever bytes the peer supplied — including `[]` or any arbitrary blob — into a typed hash with no length or content check.

**Pool-lookup guard in `get_or_fetch`:** [2](#0-1) 

The pool is queried by `ingress_message_id` (which the peer can supply correctly), but the guard `SignedIngressId::from(&ingress_message) == signed_ingress_id` computes the real SHA-256 hash of the local copy and compares it against the peer-supplied `ingress_bytes_hash`. With a crafted hash the comparison always fails, so the code falls through to `download_stripped_message`.

**Server-side hash guard (responding peer):** [3](#0-2) 

The serving peer performs the same `SignedIngressId::from(&ingress_message) == *signed_ingress_id` check. With a crafted hash in the request it returns `NOT_FOUND`.

**Client-side parse guard:** [4](#0-3) 

Even if a peer returns `200 OK` with the real bytes, `derived_ingress_id = SignedIngressId::from(&ingress)` recomputes the real hash and the comparison `derived_ingress_id == *ingress_id` fails, yielding `MessageIdMismatch`.

**Infinite retry loop:** [5](#0-4) 

`with_max_elapsed_time(None)` means the loop retries forever. The only escape is the bouncer marking the block unwanted: [6](#0-5) 

---

### Impact Explanation

For every ingress entry in the crafted `StrippedBlockProposal`:
- The local ingress pool is bypassed (pool hit is discarded due to hash mismatch).
- An outbound RPC is issued to a random peer; the peer returns `NOT_FOUND` (or `200` with real bytes that fail `parse_response`).
- The RPC is retried with exponential backoff, indefinitely, until the bouncer fires.

Concrete effects:
- **Assembly latency degradation**: the block assembly task is stalled for the full bouncer timeout on every crafted proposal.
- **Bandwidth exhaustion**: one RPC per ingress per backoff interval, multiplied by the number of ingress messages in the block (up to ~1000 per `PayloadAssembler` comment).
- **CPU waste**: repeated serialization/deserialization cycles on both sides.

A single Byzantine peer below the consensus fault threshold can target any honest node continuously by replaying crafted `StrippedBlockProposal`s for each new advertised block.

---

### Likelihood Explanation

- The `StrippedBlockProposal` is a P2P-layer artifact received from peers before any consensus-layer authentication of its internal fields.
- The `unstripped_consensus_message_id` (the artifact's key) is publicly observable from block advertisements; the Byzantine peer only needs to copy it correctly.
- The `ingress_bytes_hash` fields are not covered by any cryptographic commitment in the `StrippedConsensusMessageId`.
- No privilege is required; any subnet peer can send crafted artifacts.

---

### Recommendation

In `TryFrom<pb::StrippedIngressMessage> for SignedIngressId`, validate that `value.ingress_bytes_hash` has exactly the expected length for the hash algorithm in use (32 bytes for SHA-256) and reject the artifact with `ProxyDecodeError` otherwise:

```rust
const EXPECTED_HASH_LEN: usize = 32;
if value.ingress_bytes_hash.len() != EXPECTED_HASH_LEN {
    return Err(ProxyDecodeError::Other(format!(
        "ingress_bytes_hash has invalid length {}",
        value.ingress_bytes_hash.len()
    )));
}
let ingress_bytes_hash = CryptoHashOf::from(CryptoHash(value.ingress_bytes_hash));
```

The same guard should be applied in `TryFrom<pb::GetIngressMessageInBlockRequest> for GetIngressMessageInBlockRequest` at [7](#0-6) .

### Proof of Concept

```rust
#[test]
fn crafted_empty_hash_forces_peer_fetch() {
    // Build a real ingress message and put it in the pool.
    let ingress = SignedIngressBuilder::new().nonce(1).build();
    let real_id = SignedIngressId::from(&ingress);

    // Craft a SignedIngressId with an empty ingress_bytes_hash via the proto path.
    let proto = pb::StrippedIngressMessage {
        stripped: Some(real_id.ingress_message_id.clone().into()),
        ingress_bytes_hash: vec![], // crafted: empty
    };
    let crafted_id = SignedIngressId::try_from(proto).unwrap();

    // The crafted id does NOT equal the real id.
    assert_ne!(crafted_id, real_id);

    // Simulate the get_or_fetch pool-lookup guard:
    // pool returns the ingress, but the hash check fails.
    assert!(SignedIngressId::from(&ingress) != crafted_id);
    // => falls through to download_stripped_message, which retries forever.
}
```
