### Title
Stale Cached `transaction_fee_e8s` in SNS Governance Not Updated on Ledger Upgrade Timeout — (`rs/sns/governance/src/governance.rs`)

---

### Summary

The SNS governance canister maintains a cached copy of the ledger's transfer fee in `NervousSystemParameters.transaction_fee_e8s`. When a `ManageLedgerParameters` proposal upgrades the ledger with a new fee, the governance canister polls `canister_info` to confirm the upgrade. If this confirmation loop times out (after 5 minutes), the function returns an error **without** updating the cached `transaction_fee_e8s`. The ledger now has the new fee, but governance still holds the old value. All subsequent neuron operations (`disburse_neuron`, `split_neuron`, `disburse_maturity`) that read the cached fee and pass it to the ledger will be rejected with `BadFee`, permanently locking neuron holders out of their funds until a corrective governance proposal is passed.

---

### Finding Description

`perform_manage_ledger_parameters` in `rs/sns/governance/src/governance.rs` upgrades the SNS ledger canister with new parameters (including a new transfer fee), then enters a polling loop to confirm the upgrade via `canister_info`: [1](#0-0) 

After the upgrade call returns, the confirmation loop runs for up to 5 minutes: [2](#0-1) 

The cached `transaction_fee_e8s` is updated **only** inside the success branch of the confirmation loop. If the loop times out, the function returns an error: [3](#0-2) 

At this point the ledger has the new fee, but `NervousSystemParameters.transaction_fee_e8s` still holds the old value. Every subsequent call to `transaction_fee_e8s_or_panic()` returns the stale value: [4](#0-3) 

This stale value is used directly in three critical neuron operations:

- `disburse_neuron` (line 1138) passes it as the ledger fee: [5](#0-4) [6](#0-5) 

- `split_neuron` (line 1309) uses it for minimum-stake validation and the ledger transfer: [7](#0-6) 

- `disburse_maturity` (line 1653) uses it to gate whether maturity disbursement is allowed: [8](#0-7) 

The ICRC-1 ledger enforces an exact fee match; any mismatch causes a `BadFee` rejection. The two values — the ledger's actual fee (`icrc1_fee()`) and governance's cached fee (`transaction_fee_e8s`) — are now permanently inconsistent, mirroring the `DOMAIN_SEPARATOR()` vs `_domainSeparatorV4()` split in the original report.

---

### Impact Explanation

After the inconsistency is established:

- Every `disburse_neuron` call passes the stale fee to the ledger → ledger returns `BadFee` → neuron holders **cannot recover their staked tokens**.
- Every `split_neuron` call fails similarly → neurons cannot be split.
- `disburse_maturity` may incorrectly gate or fail maturity disbursements.
- The SNS is stuck until a new `ManageNervousSystemParameters` proposal is passed to re-sync the cached fee, which itself requires a governance majority and takes days.

This is a **ledger conservation / funds-locking bug**: legitimate token holders lose access to their staked funds through no fault of their own.

---

### Likelihood Explanation

The trigger condition — the `canister_info` polling loop timing out after 5 minutes — can occur due to:

- Subnet congestion causing inter-canister calls to queue beyond the 5-minute window.
- A transient bug or panic in the ledger canister's `post_upgrade` hook that delays the canister becoming available.
- Any condition that prevents the governance canister from receiving a `canister_info` response for 5 consecutive minutes.

The `ManageLedgerParameters` proposal type is a standard, publicly-proposable SNS action. Any SNS community that changes its ledger fee is exposed to this race condition. Likelihood is **low-to-medium** given the 5-minute window, but the impact (permanent fund lock until corrective governance action) is severe.

---

### Recommendation

1. **Persist the intended fee before the upgrade**: Store the new `transfer_fee` in governance state at the start of `perform_manage_ledger_parameters`, before the upgrade call. If the confirmation loop times out, the cached value is already correct.

2. **Alternatively, always re-query the ledger**: Replace `transaction_fee_e8s_or_panic()` with an async call to `icrc1_fee()` at the point of use in `disburse_neuron`, `split_neuron`, and `disburse_maturity`, eliminating the cached copy entirely.

3. **On timeout, still update the cache**: If the upgrade call itself succeeded (returned `Ok`), update `transaction_fee_e8s` before returning the timeout error, so the cache reflects the new reality even if confirmation could not be verified.

---

### Proof of Concept

1. Deploy an SNS with default ledger fee (10,000 e8s).
2. Pass a `ManageLedgerParameters` proposal setting `transfer_fee = 34`.
3. Simulate subnet congestion so that `canister_info` inter-canister calls queue for > 5 minutes during the confirmation loop.
4. The proposal execution returns `Err("Upgrade marked as failed...")`.
5. The ledger now charges 34 e8s; governance still stores 10,000 e8s in `NervousSystemParameters.transaction_fee_e8s`.
6. Any neuron holder calling `disburse_neuron` triggers a ledger transfer with `fee = 10_000`; the ledger rejects with `BadFee { expected_fee: 34 }`.
7. The neuron holder's stake is inaccessible until a corrective `ManageNervousSystemParameters` proposal (requiring days of governance voting) re-syncs the cached fee. [9](#0-8) [10](#0-9)

    async fn perform_manage_ledger_parameters(
        &mut self,
        proposal_id: u64,
        manage_ledger_parameters: ManageLedgerParameters,
    ) -> Result<(), GovernanceError> {
        self.check_no_upgrades_in_progress(Some(proposal_id))?;

        let current_version = self.get_or_reset_deployed_version().await.map_err(|err| {
            GovernanceError::new_with_message(
                ErrorType::External,
                format!("Could not execute proposal: {err}"),
            )
        })?;

        let ledger_canister_id = self.proto.ledger_canister_id_or_panic();

        let ledger_canister_info = self.env
            .call_canister(
                CanisterId::ic_00(),
                "canister_info",
                candid::encode_one(
                    CanisterInfoRequest::new(
                        ledger_canister_id,
                        Some(1),
                    )
                ).map_err(|e| GovernanceError::new_with_message(ErrorType::External, format!("Could not execute proposal. Error encoding canister_info request.\n{e}")))?
            )
            .await
            .map(|b| {
                candid::decode_one::<CanisterInfoResponse>(&b)
                .map_err(|e| GovernanceError::new_with_message(ErrorType::External, format!("Could not execute proposal. Error decoding canister_info response.\n{e}")))
            })
            .map_err(|err: (Option<i32>, String)| GovernanceError::new_with_message(ErrorType::External, format!("Canister method call canister_info failed: {err:?}")))??;

        let ledger_canister_info_version_number_before_upgrade: u64 =
            ledger_canister_info
            .changes()
            .last().ok_or(GovernanceError::new_with_message(ErrorType::External, "Could not execute proposal. Error finding current ledger canister_info version number".to_string()))?
            .canister_version();

        let ledger_wasm = get_wasm(
            &*self.env,
            current_version.ledger_wasm_hash.clone(),
            SnsCanisterType::Ledger,
        )
        .await
        .map_err(|e| {
            GovernanceError::new_with_message(
                ErrorType::External,
                format!("Could not execute proposal. Error getting ledger canister wasm: {e}"),
            )
        })?
        .wasm;

        use ic_icrc1_ledger::{LedgerArgument, UpgradeArgs};
        let ledger_upgrade_arg = candid::encode_one(Some(LedgerArgument::Upgrade(Some(
            UpgradeArgs::from(manage_ledger_parameters.clone()),
        ))))
        .unwrap();

        self.upgrade_non_root_canister(
            ledger_canister_id,
            Wasm::Bytes(ledger_wasm),
            ledger_upgrade_arg,
            CanisterInstallMode::Upgrade,
        )
        .await?;

        // If this operation takes 5 minutes, there is very likely a real failure, and other intervention will
        // be required
        let mark_failed_at_seconds = self.env.now() + 5 * 60;

        loop {
            let ledger_canister_info = self.env
                .call_canister(
                    CanisterId::ic_00(),
                    "canister_info",
                    candid::encode_one(
                        CanisterInfoRequest::new(
                            ledger_canister_id,
                            Some(20), // Get enough to ensure we did not miss the relevant change
                        )
                    ).map_err(|e| GovernanceError::new_with_message(ErrorType::External, format!("Could not check if ledger upgrade succeeded. Error encoding canister_info request.\n{e}")))?
                )
                .await
                .map(|b| {
                    candid::decode_one::<CanisterInfoResponse>(&b)
                        .map_err(|e| GovernanceError::new_with_message(ErrorType::External, format!("Could not check if ledger upgrade succeeded. Error decoding canister_info response.\n{e}")))
                })
                .map_err(|e| GovernanceError::new_with_message(ErrorType::External, format!("Could not check if ledger upgrade succeeded. Canister method call canister_info failed: {e:?}")))??;

            for canister_change in ledger_canister_info.changes().iter().rev() {
                if canister_change.canister_version()
                    > ledger_canister_info_version_number_before_upgrade
                    && let CanisterChangeDetails::CanisterCodeDeployment(code_deployment) =
                        canister_change.details()
                    && let CanisterInstallMode::Upgrade = code_deployment.mode()
                    && code_deployment.module_hash()[..] == current_version.ledger_wasm_hash[..]
                {
                    // success
                    // update nervous-system-parameters transaction_fee if the fee is changed.
                    if let Some(nervous_system_parameters) = self.proto.parameters.as_mut()
                        && let Some(transfer_fee) = manage_ledger_parameters.transfer_fee
                    {
                        nervous_system_parameters.transaction_fee_e8s = Some(transfer_fee);
                    }
                    return Ok(());
                }
            }

            if self.env.now() > mark_failed_at_seconds {
                let error = format!(
                    "Upgrade marked as failed at {}. \
                     Did not find an upgrade in the ledger's canister_info recent_changes.",
                    format_timestamp_for_humans(self.env.now()),
                );
                return Err(GovernanceError::new_with_message(
                    ErrorType::External,
                    error,
                ));
            }
        }
    }
