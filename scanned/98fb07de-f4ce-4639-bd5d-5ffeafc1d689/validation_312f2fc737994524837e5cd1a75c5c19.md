### Title
SNS Swap `min_participants` Decentralization Check Bypassed via Multiple Principals - (`rs/sns/swap/canister/canister.rs`)

---

### Summary

The SNS Swap canister's `refresh_buyer_tokens` endpoint accepts an arbitrary `buyer` principal supplied by the caller without verifying it matches `msg_caller()`. Because `min_participation_reached()` counts unique entries in the `buyers` BTreeMap, a single real-world entity controlling multiple IC principals can artificially satisfy the `min_participants` threshold, forcing a swap to commit without genuine independent participation.

---

### Finding Description

The `refresh_buyer_tokens` update method in the swap canister reads the buyer principal from the request argument rather than enforcing it equals the caller:

```rust
// rs/sns/swap/canister/canister.rs
async fn refresh_buyer_tokens(arg: RefreshBuyerTokensRequest) -> RefreshBuyerTokensResponse {
    let p: PrincipalId = if arg.buyer.is_empty() {
        caller_principal_id()
    } else {
        PrincipalId::from_str(&arg.buyer).unwrap()  // ← no caller == buyer check
    };
    swap_mut()
        .refresh_buyer_token_e8s(p, arg.confirmation_text, this_canister_id(), &icp_ledger)
        .await
``` [1](#0-0) 

Inside `refresh_buyer_token_e8s`, the swap canister checks the ICP ledger balance of the **subaccount derived from `buyer`** (not from `caller`), then inserts `buyer` as a new key in `self.buyers`: [2](#0-1) 

The `min_participation_reached()` function counts unique keys in `self.buyers` and compares against `params.min_participants`:

```rust
// rs/sns/swap/src/swap.rs
(self.buyers.len() as u32) >= params.min_participants
``` [3](#0-2)

---

### Impact Explanation

A single entity (e.g., the SNS project team) can force a swap to commit by:

1. Generating N self-controlled principals P₁…Pₙ where N ≥ `min_participants`.
2. Transferring `min_participant_icp_e8s` ICP to the swap canister subaccount `principal_to_subaccount(Pᵢ)` for each Pᵢ on the ICP ledger.
3. Calling `refresh_buyer_tokens` with `buyer = Pᵢ` for each i — from any single caller principal, since the endpoint does not validate `arg.buyer == caller`.
4. Each call inserts a distinct key into `self.buyers`, so `buyers.len()` reaches `min_participants`.
5. `sufficient_participation()` returns `true`; the swap commits.

The SNS decentralization guarantee — that a minimum number of **independent** participants must buy tokens — is entirely defeated. The attacker receives all SNS tokens allocated to the fake participants and the SNS governance is effectively controlled by a single entity from inception.

---

### Likelihood Explanation

The attack is straightforward and requires only:
- Control of N IC principals (trivially generated via `dfx identity new`).
- Enough ICP to fund N subaccounts at `min_participant_icp_e8s` each (the attacker recovers this ICP if the swap aborts, and retains the SNS tokens if it commits).
- A single ingress call per fake principal to `refresh_buyer_tokens`.

No privileged access, no threshold corruption, and no social engineering is required. The `arg.buyer` field is part of the public Candid interface and is accepted from any unauthenticated ingress sender. [1](#0-0) 

---

### Recommendation

Enforce that the `buyer` principal in `refresh_buyer_tokens` must equal `msg_caller()`. If third-party notification on behalf of another principal is needed, require an explicit allowlist or a signed delegation. Concretely:

```rust
async fn refresh_buyer_tokens(arg: RefreshBuyerTokensRequest) -> RefreshBuyerTokensResponse {
    let caller = caller_principal_id();
    let p: PrincipalId = if arg.buyer.is_empty() {
        caller
    } else {
        let requested = PrincipalId::from_str(&arg.buyer).unwrap();
        // Enforce caller == buyer to prevent sybil participation.
        if requested != caller {
            panic!("buyer must match caller");
        }
        requested
    };
    ...
}
```

Alternatively, if third-party notification is intentional (e.g., a relayer notifying on behalf of a user who already transferred ICP), document it explicitly and accept the decentralization trade-off, or add an on-chain identity/KYC layer.

    pub fn min_participation_reached(&self) -> bool {
        if let (Some(params), Some(init)) = (&self.params, &self.init) {
            if init.neurons_fund_participation.is_some() {
                // Only count direct participants for determining swap's success.
                // Note that a valid Swap Init should either have `neurons_fund_participation` or
                // `cf_participants`, but not both at the same time; here, we defensively perform
                // the check again anyway.
                if !self.cf_participants.is_empty() {
                    log!(
                        ERROR,
                        "Inconsistent Swap Init: cf_participants has {} elements (starting with \
                        {:?}) while neurons_fund_participation is set.",
                        self.cf_participants.len(),
                        self.cf_participants[0],
                    );
                }
                (self.buyers.len() as u32) >= params.min_participants
            } else {
                (self.cf_participants.len().saturating_add(self.buyers.len()) as u32)
                    >= params.min_participants
            }
        } else {
            false
        }
    }
