### Title
Unchecked Index Panic on Empty `Vec<EnvelopePair>` in `construction_parse` — (`rs/rosetta-api/icp/src/request_handler/construction_parse.rs`)

---

### Summary

The `construction_parse` handler unconditionally indexes `updates[0]` on a `Vec<EnvelopePair>` that is deserialized directly from attacker-controlled CBOR bytes with no non-empty invariant enforced. A single crafted POST to the public `/construction/parse` endpoint triggers an index-out-of-bounds panic.

---

### Finding Description

`construction_parse` at line 42 of `rs/rosetta-api/icp/src/request_handler/construction_parse.rs` iterates over `signed_transaction.requests` and, for each `(RequestType, Vec<EnvelopePair>)` tuple, blindly accesses element zero:

```rust
.map(
    |(request_type, updates)| match updates[0].update.content.clone() {
        HttpCallContent::Call { update } => (request_type.clone(), update),
    },
)
``` [1](#0-0) 

The `SignedTransaction` type is defined as:

```rust
pub struct SignedTransaction {
    pub requests: Vec<Request>,
}
pub type Request = (RequestType, Vec<EnvelopePair>);
``` [2](#0-1) 

`ParsedTransaction::try_from` performs only CBOR deserialization — no structural validation that any `Vec<EnvelopePair>` is non-empty:

```rust
Ok(ParsedTransaction::Signed(
    serde_cbor::from_slice(&from_hex(&value.transaction)?).map_err(|e| {
        ApiError::invalid_request(format!("Could not decode signed transaction: {e}"))
    })?,
))
``` [3](#0-2) 

The HTTP route is registered publicly with no authentication:

```rust
#[post("/construction/parse")]
async fn construction_parse(
    msg: web::Json<ConstructionParseRequest>,
    req_handler: web::Data<RosettaRequestHandler>,
) -> HttpResponse {
    let res = req_handler.construction_parse(msg.into_inner());
    to_rosetta_response(res, &req_handler.rosetta_metrics())
}
``` [4](#0-3) 

There is no panic-catching middleware in the actix-web `App` configuration. [5](#0-4) 

---

### Impact Explanation

A panic in an actix-web handler propagates through the worker thread. Actix-web will restart the crashed worker, but during the restart window all in-flight requests on that worker fail. A single persistent attacker sending this request repeatedly can keep all worker threads in a restart loop, rendering the Rosetta API unavailable. This is a non-volumetric, single-request-per-worker DoS against the ICP Rosetta service. The IC protocol, consensus, and canisters are unaffected; only the Rosetta API process availability is impacted.

---

### Likelihood Explanation

The endpoint is unauthenticated and publicly reachable. The exploit requires only the ability to POST JSON to `/construction/parse` with `signed: true` and a hex-encoded CBOR payload. No privileged access, key material, or protocol knowledge beyond the Rosetta API spec is needed. The CBOR structure is straightforward to craft.

---

### Recommendation

Add a bounds check before accessing `updates[0]`. Return an `ApiError::invalid_request` if the vec is empty:

```rust
.map(|(request_type, updates)| {
    let first = updates.first().ok_or_else(|| {
        ApiError::invalid_request("EnvelopePair list is empty")
    })?;
    match first.update.content.clone() {
        HttpCallContent::Call { update } => Ok((request_type.clone(), update)),
    }
})
.collect::<Result<Vec<_>, _>>()?
```

The same pattern should be audited in `construction_hash.rs` at line 27 (`&envelope_pairs[0].update`), which has an identical unchecked index on attacker-controlled input. [6](#0-5)

    fn from_str(s: &str) -> Result<Self, Self::Err> {
        let bytes = hex::decode(s).map_err(|err| format!("{err:?}"))?;
        serde_cbor::from_slice(bytes.as_slice()).or_else(|first_err| {
            serde_cbor::from_slice::<LegacySignedTransaction>(bytes.as_slice())
                .map(|legacy_requests| SignedTransaction {
                    requests: legacy_requests,
                })
                .map_err(|_| format!("{first_err:?}"))
        })
    }
}

}
