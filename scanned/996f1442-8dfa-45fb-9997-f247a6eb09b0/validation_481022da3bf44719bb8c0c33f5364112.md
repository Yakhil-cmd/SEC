### Title
SNS `ExecuteGenericNervousSystemFunction` Proposal Marked Executed Despite Application-Level Failure in Target Canister Reply - (File: rs/sns/governance/src/canister_control.rs)

### Summary
In the SNS governance canister, when an `ExecuteGenericNervousSystemFunction` proposal is executed, the cross-canister call to the target canister is made and **any reply** — including a Candid-encoded application-level error — is treated as a success. The proposal is then permanently marked as `Executed` even though the target canister's method may have returned `Err(...)`. This is a direct analog to the reported TON bug where a non-executed order is marked as executed.

### Finding Description
In `rs/sns/governance/src/canister_control.rs`, the function `perform_execute_generic_nervous_system_function_call` makes a cross-canister call to the registered target canister and method. The result handling explicitly acknowledges the problem with a TODO comment but leaves it unresolved:

Once `executed_timestamp_seconds != 0`, the proposal status is irreversibly `Executed` and cannot be re-executed. The `set_proposal_execution_status` function in the SNS governance canister does not guard against re-execution after a failed application-level reply: [5](#0-4) 

The `NervousSystemFunction` target method signature is documented as `<method_name>(proposal_data: ProposalData) -> Result<(), String>`, meaning the target canister is expected to signal application-level failures via `Err(...)` in the reply body: [6](#0-5) 

### Impact Explanation
Any SNS `ExecuteGenericNervousSystemFunction` proposal whose target canister method returns a Candid-encoded `Err(...)` will be permanently marked as `Executed` in the SNS governance state, even though the intended on-chain action was not performed. This:

1. **Permanently blocks re-execution**: Once `executed_timestamp_seconds` is set, the proposal cannot be retried. The SNS community must submit a new proposal to retry the action, burning additional proposal fees and requiring another voting round.
2. **Misleads governance participants**: Observers querying proposal status via `get_proposal` will see `ProposalDecisionStatus::Executed`, implying the action succeeded when it did not.
3. **Breaks SNS dapp governance integrity**: For SNS DAOs that rely on `ExecuteGenericNervousSystemFunction` proposals to manage dapp state (e.g., parameter updates, treasury operations via custom canisters), a silently-failed proposal can leave the dapp in an inconsistent state while governance believes the action was applied.

### Likelihood Explanation
**Medium.** The condition is triggered whenever:
- An SNS has a registered `GenericNervousSystemFunction` (common for SNS DAOs that extend governance with custom actions).
- The target canister's method returns `Err(...)` in its Candid reply (e.g., due to a precondition failure, state inconsistency, or parameter validation error at execution time).

This is reachable by any SNS token holder with sufficient voting power to pass a proposal. No privileged access is required. The target canister returning an error is a normal operational scenario (e.g., the target canister's state changed between proposal submission and execution).

### Recommendation
- **Short term**: In `perform_execute_generic_nervous_system_function_call`, decode the reply as `Result<(), String>` (or the documented return type) and propagate `Err(...)` as a `GovernanceError` so that `set_proposal_execution_status` marks the proposal as `Failed` rather than `Executed`.
- **Long term**: Document the expected reply type for `GenericNervousSystemFunction` target methods and enforce it during `AddGenericNervousSystemFunction` proposal validation. Consider storing the raw reply bytes in `ProposalData` for auditability, as the existing TODO suggests.

### Proof of Concept
1. Register a `GenericNervousSystemFunction` (function ID ≥ 1000) whose `target_method` returns `Encode!(&Err::<(), String>("precondition failed".to_string()))`.
2. Submit and pass an `ExecuteGenericNervousSystemFunction` proposal targeting that function.
3. The governance canister calls the target method, receives the `Err(...)` reply, and — because `Ok(_reply) => Ok(())` — calls `set_proposal_execution_status(proposal_id, Ok(()))`.
4. Query the proposal: `executed_timestamp_seconds > 0`, `failed_timestamp_seconds == 0`, `failure_reason == None`, status = `Executed`.
5. The target canister's intended action was never applied, but the proposal is permanently and irreversibly marked as successfully executed. [7](#0-6) [8](#0-7)

/// Executes a generic nervous system function (i.e., a non-native SNS proposal).
pub async fn perform_execute_generic_nervous_system_function_call(
    env: &dyn Environment,
    function: NervousSystemFunction,
    call: ExecuteGenericNervousSystemFunction,
) -> Result<(), GovernanceError> {
    // Get the canister id and the method against which we execute the proposal.
    let valid_function = ValidGenericNervousSystemFunction::try_from(&function)
        .map_err(|e| GovernanceError::new_with_message(ErrorType::InvalidProposal, e))?;

    let result = env
        .call_canister(
            valid_function.target_canister_id,
            &valid_function.target_method,
            call.payload,
        )
        .await;

    // Convert result.
    match result {
        Err(err) => Err(GovernanceError::new_with_message(
            ErrorType::External,
            format!("Canister method call to execute proposal failed: {err:?}"),
        )),

        Ok(_reply) => {
            // TODO: Do something with reply. E.g. store it in the proposal,
            // and/or deserialize it so that we can detect whether there was an
            // application-level error, as opposed to a communication
            // error. Detecting application error could be done as follows:
            //
            //   candid::!Decode(&reply, Result<String, String>)
            //
            // This could then be converted into a Result<(), GovernanceError>.
            // For now, any reply is considered a success.
            Ok(())
        }
    }
}

    /// Sets a proposal's status to 'executed' or 'failed' depending on the given result that
    /// was returned by the method that was supposed to execute the proposal.
    ///
    /// The proposal ID 'pid' is taken as a raw integer to avoid
    /// lifetime issues.
    ///
    /// Pre-conditions:
    /// - The proposal's decision status is ProposalStatusAdopted
    pub fn set_proposal_execution_status(&mut self, pid: u64, result: Result<(), GovernanceError>) {
        match self.proto.proposals.get_mut(&pid) {
            Some(proposal) => {
                // The proposal has to be adopted before it is executed.
                assert_eq!(proposal.status(), ProposalDecisionStatus::Adopted);
                match result {
                    Ok(_) => {
                        log!(INFO, "Execution of proposal: {} succeeded.", pid);
                        // The proposal was executed 'now'.
                        proposal.executed_timestamp_seconds = self.env.now();
                        // If the proposal was executed it has not failed,
                        // thus we set the failed_timestamp_seconds to zero
                        // (it should already be zero, but let's be defensive).
                        proposal.failed_timestamp_seconds = 0;
                        proposal.failure_reason = None;
                    }
                    Err(error) => {
                        log!(
                            ERROR,
                            "Execution of proposal: {} failed. Reason: {:?}",
                            pid,
                            error
                        );
                        // To ensure that we don't update the failure timestamp
                        // if there has been success, check if executed_timestamp_seconds
                        // is set to a non-zero value (this should not happen).
                        // Then, record that the proposal failed 'now' with the
                        // given error.
                        if proposal.executed_timestamp_seconds == 0 {
                            proposal.failed_timestamp_seconds = self.env.now();
                            proposal.failure_reason = Some(error);
                        }
                    }
                }
            }
            None => {
                // The proposal ID was not found. Something is wrong:
                // just log this information to aid debugging.
                log!(
                    ERROR,
                    "Proposal {:?} not found when attempt to set execution result to {:?}",
                    pid,
                    result
                );
            }
        }
    }

    /// Executes a (non-native) nervous system function as a result of an adopted proposal.
    async fn perform_execute_generic_nervous_system_function(
        &self,
        call: ExecuteGenericNervousSystemFunction,
    ) -> Result<(), GovernanceError> {
        match self
            .proto
            .id_to_nervous_system_functions
            .get(&call.function_id)
        {
            None => Err(GovernanceError::new_with_message(
                ErrorType::NotFound,
                format!(
                    "There is no generic NervousSystemFunction with id: {}",
                    call.function_id
                ),
            )),
            Some(function) => {
                perform_execute_generic_nervous_system_function_call(
                    &*self.env,
                    function.clone(),
                    call,
                )
                .await
            }
        }
    }
