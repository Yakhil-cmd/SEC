### Title
`ManageLedgerParameters.transfer_fee` Lacks Input Validation in SNS Governance - (File: `rs/sns/governance/src/proposal.rs`)

### Summary
The `validate_and_render_manage_ledger_parameters` function in SNS governance validates `token_name`, `token_symbol`, and `token_logo` fields via dedicated validation helpers, but performs **no bounds or sanity check on `transfer_fee`**. Any `u64` value — including `0` — is silently accepted. Upon proposal execution, the unvalidated value is written directly to the live SNS ledger and simultaneously back-propagated into `NervousSystemParameters.transaction_fee_e8s`, making the SNS ledger's spam-prevention mechanism ineffective and corrupting the governance parameter that guards minimum neuron stake.

### Finding Description
`validate_and_render_manage_ledger_parameters` destructures all four fields of `ManageLedgerParameters` and calls `ledger_validation::validate_token_name`, `validate_token_symbol`, and `validate_token_logo` for the string fields, but the `transfer_fee` branch only formats a render string with no validation:

```rust
if let Some(transfer_fee) = transfer_fee {
    render += &format!("# Set token transfer fee: {transfer_fee} token-quantums. \n",);
    change = true;   // ← no minimum, no maximum, no sanity check
}
``` [1](#0-0) 

After the proposal is adopted and executed, `perform_manage_ledger_parameters` upgrades the SNS ledger with the raw value and then unconditionally overwrites `NervousSystemParameters.transaction_fee_e8s`:

```rust
if let Some(nervous_system_parameters) = self.proto.parameters.as_mut()
    && let Some(transfer_fee) = manage_ledger_parameters.transfer_fee
{
    nervous_system_parameters.transaction_fee_e8s = Some(transfer_fee);
}
``` [2](#0-1) 

The ICRC-1 ledger's `upgrade` path also applies the value without a floor check:

```rust
if let Some(transfer_fee) = args.transfer_fee {
    self.transfer_fee = Tokens::try_from(transfer_fee.clone()).unwrap_or_else(|e| {
        ic_cdk::trap(...)
    });
}
``` [3](#0-2) 

`Tokens::try_from` only traps on arithmetic overflow; `0` converts successfully. There is no floor anywhere in the chain.

The secondary corruption: `NervousSystemParameters.validate_transaction_fee_e8s` only checks the field is `Some`, not that it is positive:

```rust
fn validate_transaction_fee_e8s(&self) -> Result<u64, String> {
    self.transaction_fee_e8s
        .ok_or_else(|| "NervousSystemParameters.transaction_fee_e8s must be set".to_string())
}
``` [4](#0-3) 

With `transaction_fee_e8s = 0`, the guard `neuron_minimum_stake_e8s > transaction_fee_e8s` degenerates to `> 0`, so any positive stake passes, and the economic floor on neuron creation is effectively removed. [5](#0-4) 

### Impact Explanation
Setting `transfer_fee = 0` via a `ManageLedgerParameters` proposal:

1. **Ledger spam / resource exhaustion** — every principal can submit unlimited zero-cost ICRC-1 transfers, filling the ledger's transaction log and exhausting archive cycles with no economic disincentive.
2. **Governance parameter corruption** — `NervousSystemParameters.transaction_fee_e8s` is silently set to `0`, removing the economic floor that prevents dust-neuron spam.
3. **Irreversibility window** — the ledger upgrade is asynchronous; between the upgrade and a corrective proposal, the SNS ledger operates without fee protection.

Impact: **4 / 10** — affects all SNS token holders and ledger integrity, but does not directly steal funds.

### Likelihood Explanation
A `ManageLedgerParameters` proposal is a standard SNS governance action reachable by any neuron holder with sufficient voting power. The misconfiguration can occur accidentally (e.g., a proposer intending to set a fee in raw e8s but omitting the `× 10^8` factor, yielding a near-zero value) or deliberately by a coalition. No privileged key or subnet-majority is required beyond normal SNS governance participation.

Likelihood: **3 / 10** — requires proposal adoption, but the absence of any guard makes accidental misconfiguration realistic.

### Recommendation
Add an explicit minimum-value check inside `validate_and_render_manage_ledger_parameters` before accepting `transfer_fee`:

```rust
if let Some(transfer_fee) = transfer_fee {
    const MIN_TRANSFER_FEE: u64 = 1_000; // or a protocol-defined constant
    if *transfer_fee < MIN_TRANSFER_FEE {
        return Err(format!(
            "transfer_fee ({transfer_fee}) must be at least {MIN_TRANSFER_FEE} token-quantums"
        ));
    }
    render += &format!("# Set token transfer fee: {transfer_fee} token-quantums. \n");
    change = true;
}
```

Additionally, `validate_transaction_fee_e8s` in `NervousSystemParameters` should enforce a positive lower bound rather than only checking `is_some`.

### Proof of Concept

1. Deploy a standard SNS (governance + ledger) with a non-zero `transaction_fee_e8s`.
2. As any SNS neuron holder with majority voting power, submit:
   ```
   ManageLedgerParameters { transfer_fee: Some(0), .. }
   ```
3. Observe that `validate_and_render_manage_ledger_parameters` returns `Ok(...)` — the proposal is accepted.
4. After the proposal executes, call `icrc1_fee` on the SNS ledger and observe `0`.
5. Call `get_nervous_system_parameters` on SNS governance and observe `transaction_fee_e8s = Some(0)`.
6. Submit unlimited zero-fee ICRC-1 transfers; all succeed, confirming the spam-prevention mechanism is disabled. [6](#0-5) [2](#0-1) [3](#0-2)

fn validate_and_render_manage_ledger_parameters(
    manage_ledger_parameters: &ManageLedgerParameters,
) -> Result<String, String> {
    let mut change = false;
    let mut render = "# Proposal to change ledger parameters:\n".to_string();
    let ManageLedgerParameters {
        transfer_fee,
        token_name,
        token_symbol,
        token_logo,
    } = manage_ledger_parameters;

    if let Some(transfer_fee) = transfer_fee {
        render += &format!("# Set token transfer fee: {transfer_fee} token-quantums. \n",);
        change = true;
    }
    if let Some(token_name) = token_name {
        ledger_validation::validate_token_name(token_name)?;
        render += &format!("# Set token name: {token_name}. \n",);
        change = true;
    }
    if let Some(token_symbol) = token_symbol {
        ledger_validation::validate_token_symbol(token_symbol)?;
        render += &format!("# Set token symbol: {token_symbol}. \n",);
        change = true;
    }
    if let Some(token_logo) = token_logo {
        ledger_validation::validate_token_logo(token_logo)?;
        render += &format!("# Set token logo: {token_logo}. \n",);
        change = true;
    }
    if !change {
        Err(String::from(
            "ManageLedgerParameters must change at least one value, all values are None",
        ))
    } else {
        Ok(render)
    }
}
