### Title
ckETH `withdraw_eth` Burns User Funds Without a User-Specified Maximum Gas Fee Cap - (`rs/ethereum/cketh/minter/src/main.rs`)

### Summary
The ckETH minter's `withdraw_eth` endpoint burns the user's full ckETH amount upfront but deducts the Ethereum transaction fee (`max_transaction_fee = max_fee_per_gas * gas_limit`) from the received ETH at execution time, with no mechanism for the user to specify a maximum acceptable gas fee. The overcharged portion (`max_tx_fee_estimate - actual_tx_fee`) is never reimbursed to the user. On transaction resubmission under rising gas prices, the user's received ETH amount is further reduced via `ResubmissionStrategy::ReduceEthAmount`.

### Finding Description
`WithdrawalArg` accepts only `amount`, `recipient`, and `from_subaccount`:

```
type WithdrawalArg = record {
    amount : nat;
    recipient : text;
    from_subaccount : opt Subaccount;
};
```

There is no `max_fee_per_gas` or `max_transaction_fee` field. [1](#0-0) 

In `withdraw_eth`, the minter immediately burns the full `amount` from the user's ckETH ledger account: [2](#0-1) 

Later, when the minter processes the queued withdrawal, `create_transaction` deducts the current `max_transaction_fee` (computed from the live Ethereum gas estimate) from `withdrawal_amount`:

```rust
let max_transaction_fee = transaction_price.max_transaction_fee();
let tx_amount = match request.withdrawal_amount.checked_sub(max_transaction_fee) {
    Some(tx_amount) => tx_amount,
    None => return Err(CreateTransactionError::InsufficientTransactionFee { ... }),
};
``` [3](#0-2) 

If the transaction is not mined and must be resubmitted at a higher gas price, `ResubmissionStrategy::ReduceEthAmount` further reduces the ETH amount sent to the user on each resubmission: [4](#0-3) 

The documentation explicitly confirms the overcharge is never returned: *"Total unspent transaction fees: the sum of all `max_tx_fee_estimate - actual_tx_fee` for all withdrawals. This represents an overestimate of the actual transaction fees that were charged to the user but in retrospect not needed to mine the sent transaction."* [5](#0-4) 

### Impact Explanation
A user who calls `withdraw_eth(amount)` during a period of normal Ethereum gas prices has their ckETH burned immediately and irrevocably. If Ethereum gas prices spike before the minter processes the transaction (up to ~6 minutes later, or longer during congestion), the user receives substantially less ETH than they expected. The overcharged portion (`max_tx_fee_estimate - actual_tx_fee`) accumulates in the minter's ETH balance and is never returned. On repeated resubmissions under rising gas, the user's received amount is reduced further with each bump. There is no on-chain mechanism for the user to cancel the withdrawal or set a gas price ceiling after the ckETH burn.

### Likelihood Explanation
Ethereum gas prices are highly volatile and can spike 5–50× within minutes during NFT mints, DeFi liquidation cascades, or network congestion events. The minter processes withdrawals on a periodic timer, meaning a withdrawal submitted at low gas prices can be executed at peak gas prices. Any unprivileged user calling `withdraw_eth` is exposed to this risk on every withdrawal. The minter's `eip_1559_transaction_price` query endpoint provides only a point-in-time estimate with no binding commitment. [6](#0-5) 

### Recommendation
Add an optional `max_fee_per_gas: opt nat` field to `WithdrawalArg`. In `create_transaction`, if the user-supplied cap is present and the current `gas_fee_estimate.min_max_fee_per_gas()` exceeds it, reject the transaction creation and reimburse the user's ckETH rather than proceeding at an unacceptable gas price. This mirrors the protection already present for `CkErc20` withdrawals, which carry a `max_transaction_fee` field and enforce it in `create_transaction`: [7](#0-6) 

### Proof of Concept

1. User calls `withdraw_eth({ amount = 100_000_000_000_000_000; recipient = "0x..."; from_subaccount = null })` — 0.1 ckETH is burned immediately.
2. Ethereum gas prices spike 10× before the minter's next processing cycle.
3. The minter calls `create_transaction` with the current high `gas_fee_estimate`; `max_transaction_fee` is now, say, `0.05 ETH` instead of the expected `0.002 ETH`.
4. The user receives only `0.05 ETH` at the destination instead of the expected `~0.098 ETH`.
5. The `0.048 ETH` overcharge (`max_tx_fee_estimate - actual_tx_fee`) remains in the minter's ETH balance and is never reimbursed.
6. If the transaction is not mined and is resubmitted with a 10% fee bump, `ResubmissionStrategy::ReduceEthAmount` deducts the new higher fee from the user's received amount again. [8](#0-7)

#[derive(Clone, Eq, PartialEq, Debug)]
pub enum ResubmissionStrategy {
    ReduceEthAmount { withdrawal_amount: Wei },
    GuaranteeEthAmount { allowed_max_transaction_fee: Wei },
}

impl ResubmissionStrategy {
    pub fn allowed_max_transaction_fee(&self) -> Wei {
        match self {
            ResubmissionStrategy::ReduceEthAmount { withdrawal_amount } => *withdrawal_amount,
            ResubmissionStrategy::GuaranteeEthAmount {
                allowed_max_transaction_fee,
            } => *allowed_max_transaction_fee,
        }
    }
}

#[derive(Clone, Eq, PartialEq, Debug)]
pub enum ResubmitTransactionError {
    InsufficientTransactionFee {
        allowed_max_transaction_fee: Wei,
        actual_max_transaction_fee: Wei,
    },
}

impl SignedTransactionRequest {
    pub fn resubmit(
        &self,
        new_gas_fee: GasFeeEstimate,
    ) -> Result<Option<Eip1559TransactionRequest>, ResubmitTransactionError> {
        let transaction_request = self.transaction.transaction();
        let last_tx_price = transaction_request.transaction_price();
        let new_tx_price = last_tx_price
            .clone()
            .resubmit_transaction_price(new_gas_fee);
        if new_tx_price == last_tx_price {
            return Ok(None);
        }

        if new_tx_price.max_transaction_fee() > self.resubmission.allowed_max_transaction_fee() {
            return Err(ResubmitTransactionError::InsufficientTransactionFee {
                allowed_max_transaction_fee: self.resubmission.allowed_max_transaction_fee(),
                actual_max_transaction_fee: new_tx_price.max_transaction_fee(),
            });
        }
        let new_amount = match self.resubmission {
            ResubmissionStrategy::ReduceEthAmount { withdrawal_amount } => {
                withdrawal_amount.checked_sub(new_tx_price.max_transaction_fee())
                    .expect("BUG: withdrawal_amount covers new transaction fee because it was checked before")
            }
            ResubmissionStrategy::GuaranteeEthAmount { .. } => transaction_request.amount,
        };
        Ok(Some(Eip1559TransactionRequest {
            max_priority_fee_per_gas: new_tx_price.max_priority_fee_per_gas,
            max_fee_per_gas: new_tx_price.max_fee_per_gas,
            gas_limit: new_tx_price.gas_limit,
            amount: new_amount,
            ..transaction_request.clone()
        }))
    }
