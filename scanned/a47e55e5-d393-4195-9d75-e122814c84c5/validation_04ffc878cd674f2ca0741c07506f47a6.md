### Title
Unvalidated `buyer` Parameter in `refresh_buyer_tokens` Allows Third-Party Forced Participation Bypassing Confirmation Text Consent - (File: `rs/sns/swap/canister/canister.rs`)

---

### Summary

The SNS Swap canister's `refresh_buyer_tokens` endpoint accepts a caller-supplied `buyer` principal that is never validated against the actual message caller. Any unprivileged ingress sender can specify an arbitrary victim principal in the `buyer` field, supply the publicly-known `confirmation_text`, and force-register that victim's ICP participation in the swap — permanently locking their funds and bypassing the explicit consent mechanism the confirmation text was designed to enforce.

---

### Finding Description

In `rs/sns/swap/canister/canister.rs`, the `refresh_buyer_tokens` update method resolves the buyer identity as follows:

```rust
let p: PrincipalId = if arg.buyer.is_empty() {
    caller_principal_id()
} else {
    PrincipalId::from_str(&arg.buyer).unwrap()  // ← no caller check
};
``` [1](#0-0) 

When `arg.buyer` is non-empty, the resolved principal `p` is passed directly to `refresh_buyer_token_e8s` without any check that `p == caller`. The inner function then:

1. Validates the `confirmation_text` against the SNS-configured text — but **not** against the caller's identity:

```rust
self.validate_confirmation_text(confirmation_text)?;
``` [2](#0-1) 

2. Queries the ICP ledger balance of `principal_to_subaccount(&buyer)` under the swap canister:

```rust
let account = Account {
    owner: this_canister.get().0,
    subaccount: Some(principal_to_subaccount(&buyer)),
};
icp_ledger.account_balance(account).await ...
``` [3](#0-2) 

3. If the balance meets the minimum, registers the buyer's participation in `self.buyers`:

```rust
self.buyers
    .entry(buyer.to_string())
    .or_insert_with(|| BuyerState::new(0))
    .set_amount_icp_e8s(new_balance_e8s);
``` [4](#0-3) 

The `confirmation_text` validation only checks that the provided string matches the SNS-configured text — it is a public, human-readable string stored in `Init.confirmation_text` and visible to anyone querying the swap state:

```rust
(Some(expected_text), Some(text)) => {
    if &text != expected_text {
        Err("The value of `confirmation_text` does not match...")
    } else { Ok(()) }
}
``` [5](#0-4) 

The proto comment confirms the design intent allows third-party callers: `"If not specified, the caller is used."` [6](#0-5) 

Once a victim's participation is registered via `refresh_buyer_tokens`, `error_refund_icp` is blocked for that principal while their ICP is in escrow:

```rust

    if let Some(transfer) = &buyer_state.icp
        && transfer.transfer_success_timestamp_seconds == 0

        return ErrorRefundIcpResponse::new_precondition_error(format!(
            "ICP cannot be refunded as principal {} has {} ICP (e8s) in escrow",
``` [7](#0-6) 

Furthermore, `error_refund_icp` is only callable after the swap reaches `Aborted` or `Committed` state — not during `Open`:

```rust
if !(self.lifecycle() == Lifecycle::Aborted || self.lifecycle() == Lifecycle::Committed) {
    return ErrorRefundIcpResponse::new_precondition_error(
        "Error refunds can only be performed when the swap is ABORTED or COMMITTED",

``` [8](#0-7)

### Impact Explanation

An attacker who knows the SNS swap's `confirmation_text` (publicly readable from swap state) can call `refresh_buyer_tokens` with any victim principal that has ICP sitting in the swap's subaccount. This:

1. **Permanently locks the victim's ICP** into the swap during the `Open` lifecycle — `error_refund_icp` is unavailable until the swap closes, and once registered, the escrow check blocks refund while `transfer_success_timestamp_seconds == 0`.
2. **Bypasses the confirmation text consent mechanism** — the text is meant to be an explicit opt-in by the buyer, but a third party can satisfy it on the victim's behalf.
3. **Causes financial loss** if the swap commits: the victim's ICP is swept to SNS governance and they receive SNS tokens instead, potentially at a loss relative to ICP market value.
4. **Prevents the victim from withdrawing** ICP they may have sent accidentally or reconsidered participating with, for the entire duration of the swap.

The `error_refund_icp` escape hatch is specifically designed for principals who **never** called `refresh_buyer_tokens`. Once the attacker forces registration, that escape hatch is closed until sweep completes. [9](#0-8) 

---

### Likelihood Explanation

- **Entry path**: Any unprivileged ingress sender. No special role, key, or majority is required.
- **Precondition**: The victim must have ICP in their swap subaccount. This is a realistic scenario — users may send ICP to the swap subaccount and then reconsider, or send it accidentally.
- **Information needed**: The `confirmation_text` is a public string stored in the swap's `Init` state, readable by anyone via `get_state` query.
- **Execution**: A single `update` call to `refresh_buyer_tokens` with `buyer = <victim_principal>` and the correct `confirmation_text` is sufficient.

Likelihood is **medium-high**: it requires a victim to have ICP in the subaccount, but that is the normal first step of participation, and the attack window is the entire `Open` lifecycle (which can last days to weeks).

---

### Recommendation

1. **Enforce caller == buyer**: When `arg.buyer` is non-empty, require that `PrincipalId::from_str(&arg.buyer).unwrap() == caller_principal_id()`, or reject non-empty `buyer` fields from non-privileged callers entirely.
2. **Alternatively, remove the `buyer` override field** and always use `caller_principal_id()`. Third-party notification of participation is not a necessary feature and introduces this consent bypass.
3. If third-party notification is intentionally supported (e.g., for automation), the `confirmation_text` requirement should be tied to the **buyer's** explicit submission, not the caller's — for example, by requiring the buyer to pre-register their consent via a separate signed call before a third party can trigger `refresh_buyer_tokens` on their behalf.

---

### Proof of Concept

1. SNS swap is in `Open` state with `confirmation_text = "I agree to participate in this SNS swap"`.
2. Victim sends 10 ICP to `principal_to_subaccount(victim_principal)` on the ICP ledger, targeting the swap canister. They then reconsider and want to withdraw.
3. Attacker queries `get_state` to read the public `confirmation_text`.
4. Attacker submits ingress:
   ```
   refresh_buyer_tokens({
     buyer: "<victim_principal_text>",
     confirmation_text: Some("I agree to participate in this SNS swap")
   })
   ```
5. The swap canister resolves `p = victim_principal`, validates the confirmation text (passes), queries the victim's subaccount balance (10 ICP), and registers `BuyerState { amount_icp_e8s: 10_ICP }` for the victim.
6. Victim attempts `error_refund_icp` — rejected with `"ICP cannot be refunded as principal X has Y ICP (e8s) in escrow"`.
7. When the swap commits, victim's 10 ICP is swept to SNS governance and they receive SNS tokens, having never explicitly consented. [1](#0-0) [10](#0-9) [11](#0-10)

    pub async fn refresh_buyer_token_e8s(
        &mut self,
        buyer: PrincipalId,
        confirmation_text: Option<String>,
        this_canister: CanisterId,
        icp_ledger: &dyn ICRC1Ledger,
    ) -> Result<RefreshBuyerTokensResponse, String> {
        use swap_participation::*;

        // These two checks need to be repeated after awaiting the response from the ICP ledger.
        self.validate_lifecycle_is_open()
            .map_err(context_before_awaiting_icp_ledger_response)?;
        self.validate_possibility_of_direct_participation()
            .map_err(context_before_awaiting_icp_ledger_response)?;

        // User input validation doesn't expire after await, so this check doesn't need repetition.
        self.validate_confirmation_text(confirmation_text)?;

            let account = Account {
                owner: this_canister.get().0,
                subaccount: Some(principal_to_subaccount(&buyer)),
            };
            icp_ledger
                .account_balance(account)
                .await
                .map_err(|x| x.to_string())?
                .get_e8s()
        };
