### Title
Missing Sanity Check on `transfer_fee` in `ManageLedgerParameters` Proposal Validation — (File: `rs/sns/governance/src/proposal.rs`)

---

### Summary

The SNS governance canister's `validate_and_render_manage_ledger_parameters` function accepts any `transfer_fee` value without checking it against the current `neuron_minimum_stake_e8s`. A governance-approved `ManageLedgerParameters` proposal that sets `transfer_fee >= neuron_minimum_stake_e8s` silently breaks the required invariant `neuron_minimum_stake_e8s > transaction_fee_e8s`, making neuron split/disburse operations permanently fail and blocking future `ManageNervousSystemParameters` proposals from executing.

---

### Finding Description

`NervousSystemParameters` enforces a critical invariant at initialization and on every `ManageNervousSystemParameters` proposal: `neuron_minimum_stake_e8s > transaction_fee_e8s`. This is validated in `validate_neuron_minimum_stake_e8s`: [1](#0-0) 

However, the separate `ManageLedgerParameters` proposal action — which can also change the ledger's transfer fee — has its own validation function that performs **no such cross-check**: [2](#0-1) 

When a `ManageLedgerParameters` proposal is executed, the governance canister updates `NervousSystemParameters.transaction_fee_e8s` to match the new ledger fee: [3](#0-2) 

If the proposed `transfer_fee` is `>= neuron_minimum_stake_e8s`, the invariant is broken post-execution. The `neuron_minimum_stake_e8s` field is never updated by this proposal path.

---

### Impact Explanation

Once `transaction_fee_e8s >= neuron_minimum_stake_e8s`:

1. **Neuron split is broken**: The check `split.amount_e8s < min_stake + transaction_fee_e8s` now requires an amount of at least `2 * neuron_minimum_stake_e8s` (or more), which is impossible for neurons staked near the minimum. [4](#0-3) 

2. **`disburse_to_neuron` is broken**: Same arithmetic check applies. [5](#0-4) 

3. **Future `ManageNervousSystemParameters` proposals are blocked**: Any attempt to update nervous system parameters will fail `validate_neuron_minimum_stake_e8s`, which checks `neuron_minimum_stake_e8s > transaction_fee_e8s`. This permanently locks out governance parameter updates unless a separate `ManageLedgerParameters` proposal is passed to lower the fee again. [6](#0-5) 

4. **Neuron staking/claiming is degraded**: Neurons with stake equal to `neuron_minimum_stake_e8s` can no longer be validly split or disbursed.

---

### Likelihood Explanation

Any SNS token holder with sufficient voting power can submit a `ManageLedgerParameters` proposal. The proposal validation passes for any numeric `transfer_fee` value — there is no upper-bound check. An honest governance mistake (e.g., setting the fee in the wrong unit, e.g., ICP instead of e8s) or a malicious proposal from a governance majority can trigger this. The SNS governance model is designed to allow token holders to adjust parameters, and the absence of a cross-parameter sanity check makes this a realistic governance misconfiguration path.

---

### Recommendation

In `validate_and_render_manage_ledger_parameters`, when `transfer_fee` is `Some(fee)`, retrieve the current `neuron_minimum_stake_e8s` from `NervousSystemParameters` and reject the proposal if `fee >= neuron_minimum_stake_e8s`. This mirrors the existing check in `validate_neuron_minimum_stake_e8s`: [7](#0-6) 

The function signature should be extended to accept the current `NervousSystemParameters` (or at minimum `neuron_minimum_stake_e8s`) so the cross-parameter check can be performed at proposal submission time.

---

### Proof of Concept

1. Deploy an SNS with default parameters: `neuron_minimum_stake_e8s = 100_000_000` (1 SNS token), `transaction_fee_e8s = 10_000`.
2. Submit a `ManageLedgerParameters` proposal with `transfer_fee = Some(200_000_000)` (2 SNS tokens, greater than `neuron_minimum_stake_e8s`).
3. Observe that `validate_and_render_manage_ledger_parameters` returns `Ok(...)` — no rejection. [8](#0-7) 

4. After the proposal executes, `NervousSystemParameters.transaction_fee_e8s` becomes `200_000_000` while `neuron_minimum_stake_e8s` remains `100_000_000`.
5. Attempt to split a neuron with stake `150_000_000`: the check `150_000_000 < 100_000_000 + 200_000_000` fails — split is rejected.
6. Attempt to submit a `ManageNervousSystemParameters` proposal: `validate_neuron_minimum_stake_e8s` fails because `100_000_000 <= 200_000_000` — proposal is rejected at submission. [9](#0-8)

```rust
    /// Validates that the nervous system parameter neuron_minimum_stake_e8s is well-formed.
    fn validate_neuron_minimum_stake_e8s(&self) -> Result<(), String> {
        let transaction_fee_e8s = self.validate_transaction_fee_e8s()?;

        let neuron_minimum_stake_e8s = self.neuron_minimum_stake_e8s.ok_or_else(|| {
            "NervousSystemParameters.neuron_minimum_stake_e8s must be set".to_string()
        })?;

        if neuron_minimum_stake_e8s <= transaction_fee_e8s {
            Err(format!(
                "NervousSystemParameters.neuron_minimum_stake_e8s ({neuron_minimum_stake_e8s}) must be greater than \
                NervousSystemParameters.transaction_fee_e8s ({neuron_minimum_stake_e8s})"
            ))
        } else {
            Ok(())
        }
    }
```

```rust
fn validate_and_render_manage_ledger_parameters(
    manage_ledger_parameters: &ManageLedgerParameters,
) -> Result<String, String> {
    let mut change = false;
    let mut render = "# Proposal to change ledger parameters:\n".to_string();
    let ManageLedgerParameters {
        transfer_fee,
        token_name,
        token_symbol,
        token_logo,
    } = manage_ledger_parameters;

    if let Some(transfer_fee) = transfer_fee {
        render += &format!("# Set token transfer fee: {transfer_fee} token-quantums. \n",);
        change = true;
    }
    if let Some(token_name) = token_name {
        ledger_validation::validate_token_name(token_name)?;
        render += &format!("# Set token name: {token_name}. \n",);
        change = true;
    }
    if let Some(token_symbol) = token_symbol {
        ledger_validation::validate_token_symbol(token_symbol)?;
        render += &format!("# Set token symbol: {token_symbol}. \n",);
        change = true;
    }
    if let Some(token_logo) = token_logo {
        ledger_validation::validate_token_logo(token_logo)?;
        render += &format!("# Set token logo: {token_logo}. \n",);
        change = true;
    }
    if !change {
        Err(String::from(
            "ManageLedgerParameters must change at least one value, all values are None",
        ))
    } else {
        Ok(render)
    }
}
```
