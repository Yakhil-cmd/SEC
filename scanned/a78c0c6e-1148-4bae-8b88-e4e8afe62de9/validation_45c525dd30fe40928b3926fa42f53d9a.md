### Title
Any Unprivileged Caller Can Reset SNS Swap Periodic Task Timers — (File: rs/sns/swap/canister/canister.rs)

### Summary
The `reset_timers` `#[update]` endpoint in the SNS swap canister carries no caller authorization check. Any unprivileged ingress sender can invoke it to clear and restart the periodic task timer that drives all automatic swap lifecycle transitions (`try_open`, `try_commit`, `try_abort`, `try_auto_finalize`), delaying those transitions by up to 60 seconds per invocation, callable once every 10 minutes.

### Finding Description
`reset_timers` is exposed as a plain `#[update]` endpoint with no `caller()` guard: [1](#0-0) 

Its only protection is a 600-second cooldown checked against `timers.last_reset_timestamp_seconds`. When called, it invokes `init_timers()`, which:

1. Clears the existing `ic_cdk_timers` interval timer via `ic_cdk_timers::clear_timer`.
2. Schedules a fresh 60-second interval timer for `run_periodic_tasks`. [2](#0-1) 

`run_periodic_tasks` is the sole driver of the swap's automatic state machine: [3](#0-2) 

By resetting the timer just before it would fire, an attacker delays the next execution of these transitions by up to 60 seconds. Repeating this every 600 seconds yields a sustained delay of up to ~8,640 seconds (~2.4 hours) per day.

The cooldown check itself has a subtle bypass: if `swap_mut().timers` is `None` or `timers.last_reset_timestamp_seconds` is `None`, the entire cooldown guard short-circuits and the reset proceeds unconditionally: [4](#0-3) 

This is directly analogous to the reported `extendTime` missing `onlyOwner` modifier — a publicly reachable state-modifying function that should be restricted to an authorized caller (NNS governance or the swap's controllers) but is callable by anyone.

### Impact Explanation
An attacker can delay:
- **Auto-open**: The swap remains in `ADOPTED` state longer than intended, blocking participation.
- **Auto-abort**: A swap that has reached its due date without sufficient participation is not aborted on schedule, keeping participants' ICP locked in the canister beyond the protocol's time guarantees.
- **Auto-commit / auto-finalize**: SNS neuron creation and ICP disbursement to the SNS governance canister are delayed.

`can_abort` and `can_commit` both depend on `swap_due(now_seconds)`: [5](#0-4) 

A motivated attacker calling `reset_timers` every 600 seconds can delay the abort of a failing swap by hours, keeping participants' ICP locked and preventing timely refunds.

### Likelihood Explanation
The entry path is a plain ingress update call to the swap canister requiring no special role, no elevated cycles beyond the call fee, and no prior state. The cooldown is enforced in canister state observable via `get_timers`. The attack is trivially scriptable by any external user.

### Recommendation
Add a caller authorization check to `reset_timers` restricting it to the NNS governance canister or the swap canister's controllers, consistent with how `open` is gated to NNS governance only. The SNS governance and root canisters expose the same pattern and should be reviewed similarly.

### Proof of Concept
```rust
// Attacker script — runs every 600 seconds from any principal
loop {
    swap_canister.reset_timers(ResetTimersRequest {}).await;
    sleep(Duration::from_secs(600));
}
// Each call clears the existing 60-second interval timer and starts a new one,
// delaying the next run_periodic_tasks() by up to 60 seconds.
// Over 24 hours: 144 resets × 60 s = 8,640 s of cumulative lifecycle delay.
// A failing swap that should auto-abort at T can be kept alive until T + 2.4 hours.
```

fn init_timers() {
    let last_reset_timestamp_seconds = Some(now_seconds());
    let requires_periodic_tasks = swap().requires_periodic_tasks();

    swap_mut().timers.replace(Timers {
        requires_periodic_tasks: Some(requires_periodic_tasks),
        last_reset_timestamp_seconds,
        ..Default::default()
    });

    if requires_periodic_tasks {
        let new_timer_id =
            ic_cdk_timers::set_timer_interval(RUN_PERIODIC_TASKS_INTERVAL, async || {
                run_periodic_tasks().await
            });
        TIMER_ID.with(|saved_timer_id| {
            let mut saved_timer_id = saved_timer_id.borrow_mut();
            if let Some(saved_timer_id) = *saved_timer_id {
                ic_cdk_timers::clear_timer(saved_timer_id);
            }
            saved_timer_id.replace(new_timer_id);
        });
    } else {
        log!(
            INFO,
            "Periodic tasks are not required for this Swap anymore."
        );
    }
}

#[update]
fn reset_timers(_request: ResetTimersRequest) -> ResetTimersResponse {
    let reset_timers_cool_down_interval_seconds = RESET_TIMERS_COOL_DOWN_INTERVAL.as_secs();

    if let Some(timers) = swap_mut().timers
        && let Some(last_reset_timestamp_seconds) = timers.last_reset_timestamp_seconds
        && now_seconds().saturating_sub(last_reset_timestamp_seconds)
            < reset_timers_cool_down_interval_seconds
    {
        panic!(
            "Reset has already been called within the past {reset_timers_cool_down_interval_seconds:?} seconds"
        );
    }

    init_timers();

    ResetTimersResponse {}
}

    pub fn can_commit(&self, now_seconds: u64) -> bool {
        if self.lifecycle() != Lifecycle::Open {
            return false;
