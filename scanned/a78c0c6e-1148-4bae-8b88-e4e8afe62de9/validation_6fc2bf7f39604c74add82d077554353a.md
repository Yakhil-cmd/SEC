### Title
Missing Validation of `authorized_caller` in `apply_init_args` Permanently Locks Engine Controller Privileged Role - (File: rs/engine_controller/canister/canister.rs)

---

### Summary

The `apply_init_args` function in the engine controller canister accepts an `authorized_caller: Option<Principal>` at `init` and `post_upgrade` without validating that the supplied principal is not `Principal::anonymous()`. If a governance upgrade proposal passes with `authorized_caller = Some(Principal::anonymous())`, the single-principal authorization gate is set to a principal that can never send an update call on the IC, permanently locking every privileged method (`create_engine`, `delete_engine`, `update_subnet`, `deploy_guestos_to_all_subnet_nodes`). Recovery requires a second governance proposal — an exact IC analog of the UniswapV3Factory "costly redeploy" scenario.

---

### Finding Description

`apply_init_args` in `rs/engine_controller/canister/canister.rs` unconditionally writes the caller-supplied principal into the thread-local `AUTHORIZED_CALLER` with no guard against the anonymous principal or any other sentinel value:

```rust
fn apply_init_args(args: Option<EngineControllerInitArgs>) {
    let args = args.unwrap_or_default();
    let authorized = args
        .authorized_caller
        .unwrap_or_else(default_authorized_caller);
    AUTHORIZED_CALLER.with(|c| *c.borrow_mut() = authorized); // ← no validation
    ...
}
``` [1](#0-0) 

Both `init` and `post_upgrade` delegate to this function:

```rust
#[init]
fn init(args: Option<EngineControllerInitArgs>) { apply_init_args(args); }

#[post_upgrade]
fn post_upgrade(args: Option<EngineControllerInitArgs>) { apply_init_args(args); }
``` [2](#0-1) 

Every privileged entry point is gated by `ensure_authorized`, which performs an exact equality check against `AUTHORIZED_CALLER`:

```rust
fn ensure_authorized() -> Result<Principal, String> {
    let caller = msg_caller();
    let expected = AUTHORIZED_CALLER.with(|c| *c.borrow());
    if caller != expected {
        return Err(format!("Caller {caller} is not authorized to call this canister"));
    }
    Ok(caller)
}
``` [3](#0-2) 

The four privileged update methods all call `ensure_authorized()` as their first step: [4](#0-3) [5](#0-4) 

The `EngineControllerInitArgs` type exposes `authorized_caller` as a plain `Option<Principal>` with no Candid-level constraint:

```rust
pub struct EngineControllerInitArgs {
    pub authorized_caller: Option<Principal>,
    pub initial_dkg_subnet_id: Option<Principal>,
}
``` [6](#0-5) 

The IC protocol forbids anonymous principals from sending **update** calls (they may only issue unsigned query calls). Therefore, if `AUTHORIZED_CALLER` is set to `Principal::anonymous()`, the equality check in `ensure_authorized` can never be satisfied by any real update-call sender, and every privileged method is permanently unreachable.

The canister continues to exist and respond to query calls, so the misconfiguration may not be noticed immediately — directly mirroring the UniswapV3Factory pattern where `_enableFeeAmount` calls make the contract appear functional while the owner role is silently locked.

---

### Impact Explanation

With `AUTHORIZED_CALLER = Principal::anonymous()`, the following production operations become permanently impossible without a recovery upgrade:

| Method | Effect of lock |
|---|---|
| `create_engine` | Cannot provision new cloud-engine subnets |
| `delete_engine` | Cannot decommission engine subnets |
| `update_subnet` | Cannot update subnet admins or halt/unhalt subnets |
| `deploy_guestos_to_all_subnet_nodes` | Cannot roll out OS updates to engine subnet nodes |

The engine controller is the sole non-governance path for cloud-engine subnet lifecycle management. Locking it stalls all cloud-engine operations until a second NNS governance proposal is executed to restore a valid `authorized_caller` — a costly, time-consuming recovery analogous to the UniswapV3Factory redeploy scenario.

**Vulnerability class:** governance authorization bug / missing input validation on privileged role assignment.

---

### Likelihood Explanation

The trigger is a governance upgrade proposal that supplies `authorized_caller = Some(Principal::anonymous())` in the Candid-encoded init argument. This can occur through:

1. **Honest operator error** — a proposer constructs the upgrade payload programmatically and accidentally encodes the anonymous principal (e.g., a zero-byte principal blob) instead of the intended DFINITY key. The NNS voting period may not catch a subtle encoding mistake in a binary payload.
2. **Tooling bug** — a script or CI pipeline that generates upgrade proposals could emit the wrong principal if a configuration variable is unset or defaults to empty.

The likelihood is low but non-zero. The UniswapV3Factory report explicitly identifies the "passing an incorrect address to the constructor" case as one of four realistic scenarios; the IC analog is passing an incorrect principal in an upgrade proposal.

---

### Recommendation

1. **Reject anonymous and management-canister principals in `apply_init_args`:**

```rust
fn apply_init_args(args: Option<EngineControllerInitArgs>) {
    let args = args.unwrap_or_default();
    let authorized = args.authorized_caller.unwrap_or_else(default_authorized_caller);
    assert!(
        !authorized.is_anonymous(),
        "authorized_caller must not be the anonymous principal"
    );
    AUTHORIZED_CALLER.with(|c| *c.borrow_mut() = authorized);
    ...
}
```

2. **Implement a two-step ownership-change process** analogous to the report's recommendation: require the new `authorized_caller` to confirm acceptance before the old one is overwritten.

3. **Add a Candid-level constraint** or a pre-upgrade invariant check that logs and rejects an anonymous `authorized_caller` before state is mutated.

4. **Mirror fix in the rate-limit canister** (`rs/boundary_node/rate_limits/canister/canister.rs` lines 76–78), which has the identical pattern: `state.set_authorized_principal(principal)` is called without checking `principal.is_anonymous()`, permanently locking `add_config` and `disclose_rules` if the anonymous principal is supplied. [7](#0-6)

### Proof of Concept

1. Construct a governance proposal to upgrade the engine controller canister with the following Candid-encoded init argument:
   ```
   (opt record { authorized_caller = opt principal "2vxsx-fae"; initial_dkg_subnet_id = null })
   ```
   (`2vxsx-fae` is `Principal::anonymous()` in text form.)

2. The proposal passes NNS voting (voters may not scrutinize the binary payload).

3. `post_upgrade(Some(EngineControllerInitArgs { authorized_caller: Some(Principal::anonymous()), ... }))` is executed.

4. `apply_init_args` sets `AUTHORIZED_CALLER = Principal::anonymous()` with no error.

5. Any subsequent call to `create_engine`, `delete_engine`, `update_subnet`, or `deploy_guestos_to_all_subnet_nodes` from any real principal hits:
   ```
   Err("Caller <real_principal> is not authorized to call this canister")
   ```

6. No update call from any real principal can ever satisfy `caller == Principal::anonymous()`.

7. All cloud-engine subnet lifecycle operations are permanently blocked until a recovery governance proposal is executed. [1](#0-0) [3](#0-2) [6](#0-5)
