### Title
Unchecked Subtraction on `cached_neuron_stake_e8s` in SNS `split_neuron` Enables Silent Stake Inflation via Concurrent Modification - (File: rs/sns/governance/src/governance.rs)

### Summary
The SNS governance `split_neuron` function performs a plain arithmetic subtraction (`-=`) on `cached_neuron_stake_e8s` at line 1429 without locking the parent neuron during the cross-canister ledger transfer await. Because the parent neuron is unlocked during the await, a concurrent ingress message can reduce the parent's stake below the split amount between the pre-check and the subtraction. In Wasm, unsigned integer underflow wraps silently, inflating the parent neuron's cached stake to a near-`u64::MAX` value.

1. **Clone and check** (line 1313, 1333): A clone of the parent neuron is obtained and its `stake_e8s()` is verified to be ≥ `min_stake + split.amount_e8s`.
2. **Lock only the child** (line 1388): `lock_neuron_for_command` is called for `child_nid` only — the parent neuron is **not** locked.
3. **Await ledger transfer** (lines 1397–1406): An inter-canister call to the ledger is awaited. During this await the IC canister scheduler can deliver other ingress messages.
