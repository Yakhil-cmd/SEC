### Title
Single Canister Can Exhaust Subnet-Wide HTTP Outcall In-Flight Pool, Blocking All Other Canisters - (File: `rs/execution_environment/src/execution_environment.rs`)

### Summary
The Internet Computer enforces a single subnet-wide limit (`MAX_CANISTER_HTTP_REQUESTS_IN_FLIGHT = 3000`) on concurrent canister HTTP outcalls, with no per-canister fairness quota. A single malicious canister can monopolize all 3000 slots by issuing requests to URLs that never respond, blocking every other canister on the subnet from making any HTTP outcall for the full 60-second timeout window.

### Finding Description

The function `try_add_http_context_to_replicated_state` in `rs/execution_environment/src/execution_environment.rs` enforces the in-flight limit with a single global count:

```rust
if state
    .metadata
    .subnet_call_context_manager
    .canister_http_request_contexts
    .len()
    >= self.config.max_canister_http_requests_in_flight
{
    return Err(UserError::new(
        ErrorCode::CanisterRejectedMessage,
        format!(
            "max number ({}) of http requests in-flight reached.",
            self.config.max_canister_http_requests_in_flight
        ),
    ));
}
``` [1](#0-0) 

The global cap is set to 3000: [2](#0-1) 

There is no per-canister sub-limit anywhere in this path. The entire `canister_http_request_contexts` map is a flat, unpartitioned pool shared across all canisters on the subnet. [3](#0-2) 

Requests remain in the pool until a consensus response arrives or the 60-second `CANISTER_HTTP_TIMEOUT_INTERVAL` elapses: [4](#0-3) 

### Impact Explanation

A malicious canister issues 3000 `HttpRequest` calls to `ic00` targeting URLs that will never respond (e.g., a non-routable IPv6 address). Each call is accepted and inserted into `canister_http_request_contexts`. Once the pool is full, every subsequent `HttpRequest` from any canister on the subnet — including legitimate production canisters — is rejected with `CanisterRejectedMessage`. This denial of service lasts for the full 60-second timeout window. After the attacker's requests time out and are cleared from the pool, the attacker can immediately repeat the cycle. The impact is a sustained, renewable denial of the HTTP outcall feature for all canisters on the affected subnet.

### Likelihood Explanation

The attack requires only a deployed canister with sufficient cycles to pay the HTTP request fee for 3000 calls per 60-second window. The fee formula scales with subnet size: [5](#0-4) 

On subnets where `CanisterCyclesCostSchedule::Free` applies, the fee is zero, making the attack completely free. On normal-cost subnets the fee is non-trivial but finite and predictable, making the attack economically feasible for a motivated adversary. No privileged access, governance majority, or threshold corruption is required — only a deployed canister.

### Recommendation

1. **Enforce a per-canister sub-limit** inside `try_add_http_context_to_replicated_state`: count how many entries in `canister_http_request_contexts` share the same `request.sender` and reject if that count exceeds a per-canister cap (e.g., `MAX_CANISTER_HTTP_REQUESTS_IN_FLIGHT / expected_active_canisters`).
2. **Alternatively, use a fair-share quota**: track per-canister usage and apply the same leaky-bucket pattern already used in `rs/nervous_system/rate_limits/src/lib.rs` to prevent any single canister from consuming a disproportionate share of the pool.
3. **Increase the economic cost** of holding slots by charging cycles proportional to the time a request remains in-flight (not just at submission), making prolonged slot-squatting expensive.

### Proof of Concept

```
1. Deploy canister A on a subnet with http_requests enabled.
2. In a single update call, canister A calls ic00::HttpRequest 3000 times,
   each targeting "https://[::1]:1/", a URL that will never respond.
3. Observe: canister_http_request_contexts.len() == 3000.
4. Any other canister B on the same subnet calls ic00::HttpRequest.
5. Observe: B receives CanisterRejectedMessage:
   "max number (3000) of http requests in-flight reached."
6. After 60 seconds, all of A's requests time out and are cleared.
7. A repeats step 2, sustaining the DOS indefinitely.
```

The test `http_request_bound_holds` in `rs/execution_environment/src/execution_environment/tests.rs` already demonstrates that the global cap is enforced and that a single caller can fill it: [6](#0-5)

#[test]
fn http_request_bound_holds() {
    let own_subnet = subnet_test_id(1);
    let caller_canister = canister_test_id(10);
    let mut test = ExecutionTestBuilder::new()
        .with_own_subnet_id(own_subnet)
        .with_caller(own_subnet, caller_canister)
        // set number of max in-flight calls to 10
        .with_max_canister_http_requests_in_flight(10)
        .build();
    test.state_mut().metadata.own_subnet_features.http_requests = true;

    // Create payload of the request.
    let url = "https://".to_string();
    let response_size_limit = 1000_u64;
    let transform_method_name = "transform".to_string();
    let transform_context = vec![0, 1, 2];
    let args = CanisterHttpRequestArgs {
        url: url.clone(),
        max_response_bytes: Some(response_size_limit),
        headers: BoundedHttpHeaders::new(vec![]),
        body: None,
        method: HttpMethod::GET,
        transform: Some(TransformContext {
            function: TransformFunc(candid::Func {
                principal: caller_canister.get().0,
                method: transform_method_name.clone(),
            }),
            context: transform_context.clone(),
        }),
        is_replicated: None,
        pricing_version: None,
    };

    // Create request to HTTP_REQUEST method.
    let payload = args.clone().encode();
    test.inject_call_to_ic00(Method::HttpRequest, payload, Cycles::new(1_000_000_000));
    test.execute_all();
    // Check that the SubnetCallContextManager contains the request.
    let canister_http_request_contexts = &test
        .state()
        .metadata
        .subnet_call_context_manager
        .canister_http_request_contexts;
    assert_eq!(canister_http_request_contexts.len(), 1);

    // Now we try to inject more than the maximum number of requests we allow to be in-flight

    for _ in 0..15 {
        let payload = args.clone().encode();
        test.inject_call_to_ic00(Method::HttpRequest, payload, Cycles::new(1_000_000_000));
    }

    test.execute_all();
    let canister_http_request_contexts = &test
        .state()
        .metadata
        .subnet_call_context_manager
        .canister_http_request_contexts;

    // Check that the SubnetCallContextManager contains the maximum number but not more
    assert_eq!(canister_http_request_contexts.len(), 10);
}
