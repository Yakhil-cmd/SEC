### Title
Unprivileged Caller Can Commit Another User's ICP to SNS Swap Without Consent, Bypassing Confirmation Text - (File: rs/sns/swap/canister/canister.rs)

### Summary
The `refresh_buyer_tokens` endpoint in the SNS Swap canister accepts an arbitrary `buyer` principal in its request argument and performs no check that the caller equals the specified buyer. Any unprivileged ingress sender can call this function on behalf of any user who has already transferred ICP to the swap canister's subaccount, irrevocably committing that user's ICP to the swap — including bypassing the SNS-configured `confirmation_text` consent mechanism.

### Finding Description
In `rs/sns/swap/canister/canister.rs`, the `refresh_buyer_tokens` update method resolves the target principal from the caller-supplied `arg.buyer` field rather than from `ic_cdk::caller()`:

```rust
#[update]
async fn refresh_buyer_tokens(arg: RefreshBuyerTokensRequest) -> RefreshBuyerTokensResponse {
    let p: PrincipalId = if arg.buyer.is_empty() {
        caller_principal_id()
    } else {
        PrincipalId::from_str(&arg.buyer).unwrap()   // ← attacker-controlled
    };
    ...
    swap_mut()
        .refresh_buyer_token_e8s(p, arg.confirmation_text, this_canister_id(), &icp_ledger)
        .await
``` [1](#0-0) 

There is no assertion that `p == caller_principal_id()`. The inner function `refresh_buyer_token_e8s` then:
1. Validates the `confirmation_text` against the SNS-configured consent string.
2. Reads the ICP balance of the **attacker-supplied** buyer's subaccount on the swap canister.
3. Permanently records that ICP as the buyer's committed participation. [2](#0-1) 

The `confirmation_text` is a public parameter readable from the swap state. An attacker can supply it trivially. Once `refresh_buyer_token_e8s` succeeds, the victim's ICP is locked in the swap; it cannot be reclaimed until the swap closes (commits or aborts). [3](#0-2) 

### Impact Explanation
- A victim who has transferred ICP to the swap canister's subaccount (the required first step) but has not yet called `refresh_buyer_tokens` has their participation committed without their explicit action.
- The `confirmation_text` mechanism — designed to require the user to explicitly acknowledge swap terms — is rendered ineffective: the attacker reads the public text and supplies it on the victim's behalf.
- If the swap commits, the victim receives SNS tokens instead of ICP, a financial outcome they did not consent to at the time of the call.
- The victim has no recourse until the swap lifecycle ends.

### Likelihood Explanation
The SNS swap is a publicly accessible canister on mainnet. The `refresh_buyer_tokens` endpoint is a standard update call reachable by any ingress sender. The attack requires only knowledge of the victim's principal (observable on-chain from the ICP ledger transfer) and the swap's public `confirmation_text`. No privileged access, key material, or majority corruption is needed. The window of opportunity exists between the victim's ICP transfer and their own `refresh_buyer_tokens` call — a race that is straightforward to win by monitoring the ICP ledger for transfers to the swap canister's subaccounts.

### Recommendation
Add a caller-equals-buyer guard in `refresh_buyer_tokens` before dispatching to `refresh_buyer_token_e8s`. Only allow a non-caller `buyer` if the caller is an explicitly authorized canister (e.g., the NNS governance canister for Neurons' Fund participation):

```rust
async fn refresh_buyer_tokens(arg: RefreshBuyerTokensRequest) -> RefreshBuyerTokensResponse {
    let caller = caller_principal_id();
    let p: PrincipalId = if arg.buyer.is_empty() {
        caller
    } else {
        let p = PrincipalId::from_str(&arg.buyer).unwrap();
        // Only allow acting on behalf of another principal if caller is authorized
        if p != caller {
            panic!("Caller is not authorized to refresh tokens on behalf of another buyer");
        }
        p
    };
    ...
}
```

### Proof of Concept
1. Victim calls `icrc1_transfer` on the ICP ledger, sending `N` ICP to `Account { owner: swap_canister_id, subaccount: Some(principal_to_subaccount(victim_principal)) }` with memo `MEMO_OPEN_SALE`.
2. Attacker observes this transfer on the ICP ledger (public state).
3. Attacker reads the swap's `confirmation_text` via `get_sale_parameters` (public query).
4. Attacker submits an ingress message to the swap canister:
   ```
   refresh_buyer_tokens({
       buyer: "<victim_principal_text>",
       confirmation_text: Some("<public_confirmation_text>")
   })
   ```
5. The swap canister resolves `p = victim_principal`, validates the confirmation text (passes, since the attacker supplied the correct public string), reads the victim's subaccount balance, and records the victim's ICP as committed participation — all before the victim's own call executes.
6. The victim's ICP is now locked in the swap. If the swap commits, the victim receives SNS tokens they did not explicitly choose to acquire at that moment, and the `confirmation_text` consent gate has been bypassed. [4](#0-3) [5](#0-4)

    pub async fn refresh_buyer_token_e8s(
        &mut self,
        buyer: PrincipalId,
        confirmation_text: Option<String>,
        this_canister: CanisterId,
        icp_ledger: &dyn ICRC1Ledger,
    ) -> Result<RefreshBuyerTokensResponse, String> {
        use swap_participation::*;

        // These two checks need to be repeated after awaiting the response from the ICP ledger.
        self.validate_lifecycle_is_open()
            .map_err(context_before_awaiting_icp_ledger_response)?;
        self.validate_possibility_of_direct_participation()
            .map_err(context_before_awaiting_icp_ledger_response)?;

        // User input validation doesn't expire after await, so this check doesn't need repetition.
        self.validate_confirmation_text(confirmation_text)?;

        // Look for the token balance of the specified principal's subaccount on 'this' canister.
        let e8s = {
            let account = Account {
                owner: this_canister.get().0,
                subaccount: Some(principal_to_subaccount(&buyer)),
            };
            icp_ledger
                .account_balance(account)
                .await
                .map_err(|x| x.to_string())?
                .get_e8s()
        };
