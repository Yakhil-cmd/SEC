### Title
Missing Anonymous Principal Check in SNS Governance `disburse_maturity` Allows Maturity Tokens to Be Claimed by Anyone - (File: rs/sns/governance/src/governance.rs)

### Summary

The `disburse_maturity` function in SNS governance does not validate that the caller-supplied `to_account.owner` is not the anonymous principal (`2vxsx-fae`). A neuron owner (or any principal holding `DisburseMaturity` permission) can direct maturity disbursement to the anonymous principal's ICRC-1 account. Because the IC ICRC-1 ledger permits transfers *from* the anonymous principal's account when called anonymously, any unprivileged user can subsequently drain those tokens. The neuron's maturity is permanently deducted with no recourse.

### Finding Description

In `rs/sns/governance/src/governance.rs`, `disburse_maturity` resolves the destination account as follows:

```rust
let to_account: Account = match disburse_maturity.to_account.as_ref() {
    None => Account { owner: caller.0, subaccount: None },
    Some(account) => Account::try_from(account.clone()).map_err(|e| { ... })?,
};
``` [1](#0-0) 

`Account::try_from` only validates the subaccount length; it does **not** reject `Principal::anonymous()` as the owner. The anonymous principal is a structurally valid ICRC-1 principal, so the conversion succeeds silently.

The maturity is then immediately deducted from the neuron and queued in `disburse_maturity_in_progress`:

```rust
neuron.maturity_e8s_equivalent = neuron
    .maturity_e8s_equivalent
    .saturating_sub(maturity_to_deduct);
neuron.disburse_maturity_in_progress.push(disbursement_in_progress);
``` [2](#0-1) 

After the 7-day delay, `finalize_maturity_disbursements` mints tokens directly to the stored `to_account` via a minting transfer:

```rust
let transfer_result = self.ledger.transfer_funds(
    maturity_to_disburse_after_modulation_e8s,
    0,    // Minting transfers don't pay a fee.
    None, // This is a minting transfer, no 'from' account is needed
    to_account,
    self.env.now(),
).await;
``` [3](#0-2) 

The ICRC-1 ledger test suite explicitly confirms that transfers **from** the anonymous principal's account succeed when the caller is anonymous:

```rust
// Transfer from the account of the anonymous principal
transfer(&env, canister_id, anon.0, p1.0, TRANSFER_AMOUNT).expect("transfer failed");
``` [4](#0-3) 

The same missing check exists in `disburse_neuron` for the stake path:

```rust
Some(ai_pb) => Account::try_from(ai_pb.clone()).map_err(|e| { ... })?,
``` [5](#0-4) 

By contrast, other SNS/NNS operations that involve principal-based destinations explicitly reject the anonymous principal. For example, `locally_validate_and_render_transfer_sns_treasury_funds` and `locally_validate_and_render_mint_sns_tokens` both check:

```rust
if to_principal == PrincipalId::new_anonymous() {
    defects.push("to_principal must not be anonymous.".to_string());
}
``` [6](#0-5) [7](#0-6) 

And `icrc152_mint` explicitly rejects the anonymous principal as a mint target:

```rust
if args.to.owner == Principal::anonymous() {
    return Err(Icrc152MintError::InvalidAccount(
        "anonymous principal is not allowed".to_string(),
    ));
}
``` [8](#0-7) 

`disburse_maturity` and `disburse_neuron` are the only disbursement paths that omit this guard.

### Impact Explanation

**Ledger conservation bug + governance authorization bypass.**

1. A neuron owner (or any principal granted `NeuronPermissionType::DisburseMaturity`) calls `manage_neuron` → `DisburseMaturity` with `to_account = { owner: Principal::anonymous(), subaccount: None }`.
2. The neuron's `maturity_e8s_equivalent` is immediately and permanently reduced.
3. After 7 days, the SNS governance canister mints the modulated amount to the anonymous principal's ICRC-1 account on the SNS ledger.
4. Any unprivileged user makes an anonymous ingress call to the SNS ledger's `icrc1_transfer`, spending from the anonymous principal's account to their own account — no authentication required.

The neuron owner's maturity is permanently lost; the minted tokens are claimable by any observer of the ledger. The same path applies to `disburse_neuron` for the staked principal.

### Likelihood Explanation

**Medium.** The attacker must hold `DisburseMaturity` (or `Disburse`) permission on the target neuron. In SNS governance, neuron permissions are explicitly granted and can be delegated. An attacker who is granted this permission (e.g., as a hot key or co-controller) can silently redirect maturity to the anonymous account. Accidental self-harm by a neuron owner who mistakenly supplies an empty/zero principal is also plausible. The 7-day delay provides a window for detection but no on-chain mechanism to cancel the queued disbursement once initiated.

### Recommendation

Add an explicit anonymous principal check immediately after resolving `to_account` in both `disburse_maturity` and `disburse_neuron`, consistent with the pattern already used in `locally_validate_and_render_transfer_sns_treasury_funds`:

```rust
if to_account.owner == Principal::anonymous() {
    return Err(GovernanceError::new_with_message(
        ErrorType::InvalidCommand,
        "The recipient account must not be the anonymous principal.",
    ));
}
```

Apply the same guard in `disburse_neuron` after the `to_account` is resolved.

### Proof of Concept

1. Deploy an SNS with a neuron that has accumulated maturity.
2. Call `manage_neuron` as the neuron controller:
   ```
   DisburseMaturity {
       percentage_to_disburse: 100,
       to_account: Some(Account {
           owner: Some(PrincipalId::new_anonymous()),
           subaccount: None,
       }),
   }
   ```
3. Observe that `disburse_maturity` succeeds: `maturity_e8s_equivalent` drops to 0, and a `DisburseMaturityInProgress` entry is created with `account_to_disburse_to.owner = anonymous`.
4. Advance time by 7+ days and trigger `run_periodic_tasks`. The SNS ledger mints the maturity amount to the anonymous principal's account.
5. From any client, make an **anonymous** `icrc1_transfer` call to the SNS ledger, transferring the minted tokens from the anonymous principal's account to any chosen destination. The transfer succeeds because the ICRC-1 ledger permits anonymous-principal-as-sender calls (confirmed by `test_anonymous_transfers`). [9](#0-8) [10](#0-9) [11](#0-10)
