### Title
Missing Zero-Address Check in ckETH/ckERC20 Helper Contract Constructors Allows Deployment with Null Minter Address - (File: rs/ethereum/cketh/minter/EthDepositHelper.sol, rs/ethereum/cketh/minter/ERC20DepositHelper.sol, rs/ethereum/cketh/minter/DepositHelperWithSubaccount.sol)

---

### Summary

Three Solidity helper contracts in the IC chain-fusion system (`CkEthDeposit`, `CkErc20Deposit`, `CkDeposit`) accept the minter address as a constructor argument without validating it against `address(0)`. If any of these contracts is deployed with a null minter address, user ETH or ERC-20 deposits are permanently burned to `address(0)` while the ckETH minter canister — which scrapes events from the helper contract — still observes the `ReceivedEth`/`ReceivedErc20`/`ReceivedEthOrErc20` events and mints ckETH or ckERC20 tokens, creating an unbacked token supply.

---

### Finding Description

All three production Solidity helper contracts store the minter address as an `immutable` field set once in the constructor, with no zero-address guard:

**`EthDepositHelper.sol` — `CkEthDeposit`:**
```solidity
constructor(address _cketh_minter_main_address) {
    cketh_minter_main_address = payable(_cketh_minter_main_address);
}
``` [1](#0-0) 

**`ERC20DepositHelper.sol` — `CkErc20Deposit`:**
```solidity
constructor(address _cketh_minter_main_address) {
    cketh_minter_main_address = _cketh_minter_main_address;
}
``` [2](#0-1) 

**`DepositHelperWithSubaccount.sol` — `CkDeposit`:**
```solidity
constructor(address _minterAddress) {
    minterAddress = payable(_minterAddress);
}
``` [3](#0-2) 

Notably, `CkDeposit` already defines `ZERO_ADDRESS = address(0)` and uses it to guard `depositErc20` against a zero ERC-20 token address, but applies no equivalent guard to `_minterAddress` in the constructor: [4](#0-3) 

The deposit functions unconditionally forward funds to `minterAddress` and emit events:

```solidity
function deposit(bytes32 _principal) public payable {
    emit ReceivedEth(msg.sender, msg.value, _principal);
    cketh_minter_main_address.transfer(msg.value);  // burns ETH if address is 0
}
``` [5](#0-4) 

The ckETH minter canister scrapes events from the helper contract address (configured separately in minter state) and mints ckETH upon observing a valid `ReceivedEth` event, regardless of whether the ETH actually arrived at the minter's Ethereum address: [6](#0-5) 

---

### Impact Explanation

If any helper contract is deployed with `_minterAddress = address(0)`:

1. **ETH/ERC-20 is permanently burned**: `address(0).transfer(msg.value)` destroys the ETH; `safeTransferFrom(msg.sender, address(0), amount)` destroys ERC-20 tokens. Funds are irrecoverable.
2. **ckETH/ckERC20 is still minted**: The minter canister observes the emitted event (e.g., `ReceivedEth`) and mints the corresponding ckETH to the user's IC principal, because event scraping is decoupled from verifying that the minter's Ethereum balance actually increased.
3. **Unbacked token supply**: The ckETH ledger supply grows without a corresponding ETH reserve, breaking the 1:1 backing invariant that the entire chain-fusion system depends on.

This is a **chain-fusion mint/burn bug** with direct ledger conservation impact.

---

### Likelihood Explanation

Deployment of these contracts is controlled by NNS governance proposals. A governance proposal that accidentally or maliciously passes `address(0)` as the constructor argument — or a deployment script bug — would trigger this condition. Because the `immutable` field cannot be changed post-deployment, the only remediation would be redeployment and reconfiguration of the minter canister, during which all user deposits to the misconfigured contract would be lost. The absence of a constructor-level guard means there is no on-chain safety net to catch this class of deployment error.

---

### Recommendation

Add a zero-address check in all three constructors, consistent with the pattern already used for `erc20Address` in `depositErc20`:

```solidity
// EthDepositHelper.sol
constructor(address _cketh_minter_main_address) {
    require(_cketh_minter_main_address != address(0), "minter address cannot be zero");
    cketh_minter_main_address = payable(_cketh_minter_main_address);
}

// ERC20DepositHelper.sol
constructor(address _cketh_minter_main_address) {
    require(_cketh_minter_main_address != address(0), "minter address cannot be zero");
    cketh_minter_main_address = _cketh_minter_main_address;
}

// DepositHelperWithSubaccount.sol (CkDeposit)
constructor(address _minterAddress) {
    require(_minterAddress != ZERO_ADDRESS, "minter address cannot be zero");
    minterAddress = payable(_minterAddress);
}
```

### Proof of Concept

1. Deploy `CkEthDeposit` with `_cketh_minter_main_address = address(0)`.
2. Configure the ckETH minter canister to scrape events from this contract address (via `ethereum_contract_address` in `InitArg` or upgrade).
3. Call `deposit(encodedPrincipal)` with 1 ETH.
4. Observe: `ReceivedEth(msg.sender, 1 ETH, encodedPrincipal)` is emitted; ETH is transferred to `address(0)` and burned.
5. The ckETH minter canister scrapes the event, validates the principal via `parse_principal_from_slice`, and mints 1 ckETH (minus fees) to the IC principal.
6. Result: 1 ckETH exists on the IC ledger with no ETH backing it in the minter's Ethereum account. [7](#0-6) [8](#0-7) [9](#0-8)

/// Decode a candid::Principal from a slice of at most 32 bytes
/// encoded as follows
/// - the first byte is the number of bytes in the principal
/// - the next N bytes are the principal
/// - the remaining bytes are zero
///
/// Any other encoding will return an error.
/// Some specific valid [`Principal`]s are also not allowed
/// since the decoded principal will be used to receive ckETH:
/// * the management canister principal
/// * the anonymous principal
///
/// This method MUST never panic (decode bytes from untrusted sources).
fn parse_principal_from_slice(slice: &[u8]) -> Result<Principal, String> {
    const ANONYMOUS_PRINCIPAL_BYTES: [u8; 1] = [4];

    if slice.is_empty() {
        return Err("slice too short".to_string());
    }
    if slice.len() > 32 {
        return Err(format!("Expected at most 32 bytes, got {}", slice.len()));
    }
    let num_bytes = slice[0] as usize;
    if num_bytes == 0 {
        return Err("management canister principal is not allowed".to_string());
    }
    if num_bytes > 29 {
        return Err(format!(
            "invalid number of bytes: expected a number in the range [1,29], got {num_bytes}",
        ));
    }
    if slice.len() < 1 + num_bytes {
        return Err("slice too short".to_string());
    }
    let (principal_bytes, trailing_zeroes) = slice[1..].split_at(num_bytes);
    if !trailing_zeroes
        .iter()
        .all(|trailing_zero| *trailing_zero == 0)
    {
        return Err("trailing non-zero bytes".to_string());
    }
    if principal_bytes == ANONYMOUS_PRINCIPAL_BYTES {
        return Err("anonymous principal is not allowed".to_string());
    }
    Principal::try_from_slice(principal_bytes).map_err(|err| err.to_string())
}
