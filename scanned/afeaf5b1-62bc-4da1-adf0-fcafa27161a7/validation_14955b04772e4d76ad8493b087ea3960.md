### Title
SNS Treasury Manager Deposit Lacks Slippage Protection, Enabling Price-Manipulation Attacks on DAO Treasury Funds - (File: `rs/sns/governance/src/extensions.rs`, `rs/sns/treasury_manager/treasury_manager.did`)

---

### Summary

The SNS governance `execute_treasury_manager_deposit` function transfers treasury funds (SNS tokens + ICP) into a DEX liquidity pool via the Treasury Manager extension without any slippage control or minimum-return check. Because governance proposals are public during their voting period, an attacker can observe a pending deposit proposal and manipulate the DEX price before execution, causing the SNS treasury to receive far fewer LP tokens than expected. The codebase itself acknowledges this as a "Known Security Risk."

---

### Finding Description

When an SNS governance proposal of type `ExecuteExtensionOperation` with operation `deposit` is adopted, `execute_treasury_manager_deposit` in `rs/sns/governance/src/extensions.rs` is called. It:

1. Approves a transfer of `treasury_allocation_sns_e8s` SNS tokens and `treasury_allocation_icp_e8s` ICP to the Treasury Manager canister.
2. Calls `deposit` on the Treasury Manager canister with a `DepositRequest` containing only `allowances` (token amounts).
3. Logs the returned `Balances` response but performs **no check on the actual LP tokens received**. [1](#0-0) 

The `DepositRequest` type contains only `allowances: Vec<Allowance>` — there is no `min_lp_tokens_out`, no minimum return, and no slippage tolerance field. [2](#0-1) 

The proposal validation (`validate_deposit_operation_impl`) only checks that the requested amount does not exceed 50% of the current treasury balance. It performs no price or slippage check. [3](#0-2) 

The `ValidatedDepositOperationArg` struct only carries `treasury_allocation_sns_e8s`, `treasury_allocation_icp_e8s`, and the raw `original` Precise value — no slippage parameter is parsed or enforced anywhere in the execution path. [4](#0-3) 

The `construct_deposit_allowances` helper confirms that only two fields are accepted from the proposal: `treasury_allocation_sns_e8s` and `treasury_allocation_icp_e8s`. [5](#0-4)

---

### Impact Explanation

An attacker who can manipulate the DEX price (e.g., by providing or removing liquidity, or executing large swaps) before the governance proposal executes can cause the SNS treasury to deposit funds at a severely unfavorable ratio. The SNS DAO receives far fewer LP tokens than the deposited value warrants, permanently losing treasury value. Up to 50% of the SNS token treasury and 50% of the ICP treasury (the maximum allowed per proposal) can be affected in a single proposal execution.

---

### Likelihood Explanation

SNS governance proposals are public and have a multi-day voting period. Any market participant with sufficient capital to move the DEX price can observe a pending deposit proposal and front-run it. On the Internet Computer, canister calls are ordered within rounds, and an attacker can submit DEX manipulation transactions in the same or immediately preceding rounds before the governance heartbeat executes the proposal. No privileged access is required — only the ability to trade on the DEX.

---

### Recommendation

1. Add a `min_lp_tokens_out` (or equivalent minimum-return) field to `DepositRequest` and `ValidatedDepositOperationArg`, specified at proposal creation time based on the current DEX price.
2. In `execute_treasury_manager_deposit`, after calling `deposit`, verify that the returned `external_custodian` balance in `Balances` meets the minimum specified in the proposal. Revert (or trigger a withdrawal) if it does not.
3. Alternatively, require Treasury Manager implementations to enforce slippage protection internally and make this a mandatory requirement for NNS blessing of any Treasury Manager extension.

---

### Proof of Concept

1. An SNS DAO submits a governance proposal: deposit 1,000,000 SNS e8s + 500,000 ICP e8s into a KongSwap liquidity pool via the Treasury Manager.
2. The proposal enters its voting period (publicly visible on-chain).
3. An attacker observes the proposal and, before the governance heartbeat executes it, submits a large swap on the DEX that skews the SNS/ICP price ratio significantly.
4. The governance heartbeat fires `execute_treasury_manager_deposit`:
   - `approve_treasury_manager` grants the allowance.
   - `call_canister(..., "deposit", arg_blob)` is called with no minimum-return constraint.
   - The Treasury Manager deposits at the manipulated price, receiving e.g. 30% fewer LP tokens than fair value.
   - The returned `Balances` is only logged; no minimum check is performed.
5. The attacker reverses their DEX position, profiting from the price impact at the SNS treasury's expense.
6. The SNS DAO has permanently lost treasury value with no on-chain recourse. [7](#0-6) [8](#0-7)

async fn validate_deposit_operation_impl(
    governance: &Governance,
    value: Option<Precise>,
) -> Result<ValidatedDepositOperationArg, String> {
    let structurally_valid = ValidatedDepositOperationArg::try_from(value)?;

    let sns_subaccount = governance.sns_treasury_subaccount();
    let icp_subaccount = governance.icp_treasury_subaccount();

    // Fail if either is asking for more than 50% of current balance.  The balance could have changed
    // since the proposal was created, and we don't assume that the proposal should work
    let sns_balance = governance
        .ledger
        .account_balance(Account {
            owner: governance.env.canister_id().get().0,
            subaccount: sns_subaccount,
        })
        .await
        .map_err(|e| format!("Failed to get SNS treasury balance: {e:?}"))?;
    let icp_balance = governance
        .nns_ledger
        .account_balance(Account {
            owner: governance.env.canister_id().get().0,
            subaccount: icp_subaccount,
        })
        .await
        .map_err(|e| format!("Failed to get ICP treasury balance: {e:?}"))?;

    let icp_requested = Tokens::from_e8s(structurally_valid.treasury_allocation_icp_e8s);
    let sns_requested = Tokens::from_e8s(structurally_valid.treasury_allocation_sns_e8s);

    // Unwrap is safe, only fails if divisor is zero, which we don't do.
    if sns_requested > sns_balance.checked_div(2).unwrap() {
        return Err(format!(
            "SNS treasury deposit request of {sns_requested} exceeds 50% of current SNS Token balance of {sns_balance}"
        ));
    }

    if icp_requested > icp_balance.checked_div(2).unwrap() {
        return Err(format!(
            "ICP treasury deposit request of {icp_requested} exceeds 50% of current ICP balance of {icp_balance}"
        ));
    }

    Ok(structurally_valid)
}

/// Validated deposit operation arguments
#[derive(Debug, Clone)]
pub struct ValidatedDepositOperationArg {
    /// Amount of SNS tokens to allocate from treasury
    pub treasury_allocation_sns_e8s: u64,
    /// Amount of ICP tokens to allocate from treasury
    pub treasury_allocation_icp_e8s: u64,
    /// Original Precise value with all fields
    pub original: Precise,
}

impl TryFrom<Option<Precise>> for ValidatedDepositOperationArg {
    type Error = String;

    fn try_from(value: Option<Precise>) -> Result<Self, Self::Error> {
        let Some(original) = value else {
            return Err("Deposit operation arguments must be provided".to_string());
        };

        let map = match &original.value {
            Some(precise::Value::Map(PreciseMap { map })) => map,
            _ => return Err("Deposit operation arguments must be a PreciseMap".to_string()),
        };

        let treasury_allocation_sns_e8s = map
            .get("treasury_allocation_sns_e8s")
            .and_then(|p| match &p.value {
                Some(precise::Value::Nat(n)) => Some(*n),
                _ => None,
            })
            .ok_or_else(|| "treasury_allocation_sns_e8s must be a Nat value".to_string())?;

        let treasury_allocation_icp_e8s = map
            .get("treasury_allocation_icp_e8s")
            .and_then(|p| match &p.value {
                Some(precise::Value::Nat(n)) => Some(*n),
                _ => None,
            })
            .ok_or_else(|| "treasury_allocation_icp_e8s must be a Nat value".to_string())?;

        Ok(Self {
            treasury_allocation_sns_e8s,
            treasury_allocation_icp_e8s,
            original,
        })
    }
