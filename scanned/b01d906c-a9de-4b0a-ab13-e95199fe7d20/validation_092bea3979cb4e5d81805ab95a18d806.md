### Title
SNS Governance Permanently Locked in `PreInitializationSwap` Mode After Aborted Swap — No Admin Escape Hatch - (File: `rs/sns/governance/src/governance.rs`)

### Summary
When an SNS token swap is aborted, the SNS Governance canister remains permanently locked in `MODE_PRE_INITIALIZATION_SWAP`. The only entity authorized to call `set_mode` to transition governance to `Normal` is the swap canister itself, and `set_mode` only accepts `Normal` as a target. After an aborted swap, the swap canister never calls `set_mode(Normal)`, leaving governance frozen indefinitely. No admin, no NNS governance proposal, and no other privileged principal can override this. This is the direct IC analog of the reported "ProxyAdmin causes DoS for SYS_ADMIN" pattern: a system-wide mode lock with no privileged bypass.

### Finding Description

SNS Governance initializes in `MODE_PRE_INITIALIZATION_SWAP` and is designed to transition to `MODE_NORMAL` only after a successful swap finalization. The `set_mode` function enforces two hard constraints:

1. **Only the swap canister may call it** — any other caller causes a `panic!`.
2. **Only `Normal` mode may be entered** — any other target mode causes a `panic!`. [1](#0-0) 

The swap canister calls `set_sns_governance_to_normal_mode` only in the **committed** finalization path (`finalize_inner`), after neurons are claimed: [2](#0-1) 

In the **aborted** finalization path, `finalize_inner` returns early after restoring dapp controllers, and `set_mode` is never called: [3](#0-2) 

This is confirmed by the integration test suite, which explicitly asserts that after an aborted swap, governance mode remains `PreInitializationSwap`: [4](#0-3) 

In `PreInitializationSwap` mode, the following `manage_neuron` commands are **unconditionally blocked** for all callers, including the SNS developer/admin neuron holders:

- `Configure` (start/stop dissolving, increase dissolve delay)
- `Disburse`
- `Split`
- `MergeMaturity`
- `DisburseMaturity`
- `StakeMaturity` [5](#0-4) 

The blocked commands are explicitly enumerated in tests: [6](#0-5) 

Additionally, the following proposal actions are blocked for all neuron holders:

- `ManageNervousSystemParameters`
- `TransferSnsTreasuryFunds`
- `MintSnsTokens`
- `UpgradeSnsControlledCanister`
- `RegisterDappCanisters`
- `DeregisterDappCanisters` [7](#0-6) 

There is no privileged bypass: `set_mode` panics for any non-swap-canister caller, and the swap canister itself never calls `set_mode(Normal)` after an abort. [8](#0-7) 

### Impact Explanation

After a failed SNS swap, the SNS Governance canister is permanently frozen in `PreInitializationSwap` mode. Developer neurons cannot dissolve, disburse, or manage their staked tokens. Treasury funds cannot be transferred. Dapp canisters cannot be upgraded or re-registered. The SNS is effectively bricked from a governance perspective, with no on-chain recovery path available to any principal — including the original developers. The only recovery is an out-of-band canister upgrade (which itself requires NNS governance approval), introducing significant operational friction and delay.

**Impact:** Governance authorization bug / resource lock — all SNS neuron holders and the SNS developer team are permanently denied access to staking operations and treasury management after a swap abort.

### Likelihood Explanation

SNS swap aborts are a normal, expected lifecycle event (insufficient participation, swap deadline reached without meeting minimum ICP). Any SNS project whose swap fails enters this state automatically. The condition is reachable by any unprivileged actor who simply participates insufficiently in a swap, or by the passage of time past the swap deadline. No special access is required to trigger the abort; the swap canister's `run_periodic_tasks` calls `try_abort` autonomously. [9](#0-8) 

### Recommendation

Add an escape hatch in `set_mode` that allows a trusted privileged principal (e.g., NNS Root, or the SNS's own root canister) to transition governance out of `PreInitializationSwap` mode when the swap has been aborted. Alternatively, the swap canister's aborted finalization path should explicitly call `set_mode(Normal)` after completing ICP refunds, so that developer neurons retain full operational capability even after a failed swap. The current design intentionally preserves `PreInitializationSwap` mode after abort (as shown by the integration test assertion), but provides no recovery mechanism for the locked neuron holders.

### Proof of Concept

1. An SNS is deployed; governance initializes in `MODE_PRE_INITIALIZATION_SWAP`. [10](#0-9) 

2. The swap opens, runs, and fails to meet minimum participation. `try_abort` is called by `run_periodic_tasks`, setting lifecycle to `Aborted`. [9](#0-8) 

3. `finalize` is called (auto or manual). The aborted path restores dapp controllers and returns — `set_mode` is never called. [11](#0-10) 

4. Any developer neuron holder attempts `manage_neuron` with `Disburse` or `Configure`. The call reaches `manage_neuron_internal`, which calls `allows_manage_neuron_command_or_err` and returns `Err(PreconditionFailed)` because mode is still `PreInitializationSwap`. [12](#0-11) 

5. Any attempt to call `set_mode(Normal)` from any principal other than the swap canister panics unconditionally. [8](#0-7) 

6. Result: governance is permanently locked. Developer neurons cannot dissolve or disburse. Treasury is inaccessible. No on-chain recovery path exists.

    pub fn set_mode(&mut self, mode: i32, caller: PrincipalId) {
        let mode =
            governance::Mode::try_from(mode).unwrap_or_else(|_| panic!("Unknown mode: {mode}"));

        if !self.is_swap_canister(caller) {
            panic!("Caller must be the swap canister.");
        }

        // As of Aug, 2022, the only use-case we have for set_mode is to enter
        // Normal mode (from PreInitializationSwap). Therefore, this is here
        // just to make sure we do not proceed with unexpected operations.
        if mode != governance::Mode::Normal {
            panic!("Entering {mode:?} mode is not allowed.");
        }

        self.proto.mode = mode as i32;
    }

    fn manage_neuron_command_is_allowed_in_pre_initialization_swap_or_err(
        command: &manage_neuron::Command,
        caller_is_swap_canister: bool,
    ) -> Result<(), GovernanceError> {
        use manage_neuron::Command as C;
        let ok = match command {
            C::Follow(_)
            | C::MakeProposal(_)
            | C::RegisterVote(_)
            | C::AddNeuronPermissions(_)
            | C::RemoveNeuronPermissions(_) => true,

            C::ClaimOrRefresh(_) => caller_is_swap_canister,

            _ => false,
        };

        if ok {
            return Ok(());
        }

        Err(GovernanceError::new_with_message(
            ErrorType::PreconditionFailed,
            format!(
                "Because governance is currently in PreInitializationSwap mode, \
                 manage_neuron commands of this type are not allowed \
                 (caller_is_swap_canister={caller_is_swap_canister}). command: {command:#?}",
            ),
        ))
    }
