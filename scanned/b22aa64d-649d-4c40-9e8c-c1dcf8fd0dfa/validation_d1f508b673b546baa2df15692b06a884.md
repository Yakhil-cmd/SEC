### Title
Unbounded Iteration Over All Neurons in SNS Governance `maybe_finalize_disburse_maturity` Can Exhaust Instructions and Prevent Maturity Disbursements - (File: rs/sns/governance/src/governance.rs)

### Summary
The `maybe_finalize_disburse_maturity` function in SNS governance iterates over **all** neurons in `self.proto.neurons.values()` without any instruction-limit guard. Any SNS participant who creates many neurons with pending-but-not-yet-due disbursements causes the synchronous `filter_map` scan to grow proportionally, and the subsequent per-neuron async ledger calls compound the cost. If the aggregate instruction budget is exhausted, `run_periodic_tasks` traps, no maturity is disbursed in that round, and the pattern repeats every heartbeat — effectively disabling maturity disbursement for all SNS participants.

### Finding Description
`maybe_finalize_disburse_maturity` is called unconditionally from `run_periodic_tasks`:

```
self.maybe_finalize_disburse_maturity().await;
```

Inside, it performs two unbounded passes over the full neuron map:

**Pass 1 — synchronous scan of every neuron:**
```rust
let neuron_id_and_disbursements: Vec<...> = self
    .proto
    .neurons          // entire BTreeMap<String, Neuron>
    .values()
    .filter_map(|neuron| {
        let first_disbursement = neuron.disburse_maturity_in_progress.first()?;
        if now_seconds >= finalize_disbursement_timestamp_seconds {
            Some(...)
        } else {
            None          // ← skipped, but still iterated
        }
    })
    .collect();
```

**Pass 2 — async ledger call per ready neuron:**
```rust
for (neuron_id, disbursement) in neuron_id_and_disbursements.into_iter() {
    ...
    let _neuron_lock = match self.lock_neuron_for_command(...) {
        Ok(lock) => lock,
        Err(_) => continue,   // ← locked neurons silently skipped
    };
    let transfer_result = self.ledger.transfer_funds(...).await;
    ...
}
```

**Attacker-controlled skip condition (analogous to `_getAssignableAmount == 0`):**
A neuron is silently skipped when:
1. Its first `disburse_maturity_in_progress` entry has `finalize_disbursement_timestamp_seconds` in the future — a condition the neuron owner controls by calling `DisburseMaturity` repeatedly, each call appending a new entry with a timestamp 7 days out.
2. The neuron is locked (`in_flight_commands` contains the neuron ID) — a condition the owner can sustain by keeping another operation in flight.

An unprivileged SNS participant can therefore:
- Stake tokens into many neurons (no privileged role required).
- Call `DisburseMaturity` on each, scheduling disbursements far in the future.
- The `filter_map` must visit every neuron on every heartbeat, paying stable-memory read costs per neuron even for those that are skipped.
- With enough neurons, Pass 1 alone exhausts the per-message instruction budget (40 B instructions on application subnets), causing `run_periodic_tasks` to trap before any disbursement is processed.

This is structurally identical to the ACOToken pattern: a loop over a collection where entries can be made to evaluate as "no-op" by user-controlled state, and enough no-ops exhaust execution resources.

### Impact Explanation
- **Maturity disbursement disabled**: Every invocation of `run_periodic_tasks` traps before completing `maybe_finalize_disburse_maturity`. All SNS participants with legitimately due disbursements are blocked indefinitely.
- **Heartbeat DoS**: Because `run_periodic_tasks` is the SNS governance heartbeat, other periodic work (reward distribution, upgrade checks, maturity modulation updates) that follows `maybe_finalize_disburse_maturity` in the call sequence is also blocked.
- **No recovery path without upgrade**: The attacker's neurons remain in the neuron map; the loop cannot be shortened without a canister upgrade.

### Likelihood Explanation
- Any SNS participant with enough governance tokens to stake many neurons can mount this attack. No privileged access, no threshold corruption, no social engineering is required.
- The cost is proportional to the number of neurons needed to exhaust 40 B instructions. Given that each stable-memory neuron read costs on the order of millions of instructions, a few thousand neurons with pending disbursements suffice.
- The attack is persistent: the attacker's neurons remain until dissolved and disbursed, which cannot happen while the loop is broken.
- Likelihood is **medium**: requires capital to stake many neurons, but the barrier is purely economic, not technical.

### Recommendation
1. **Add an instruction-limit guard inside the loop**, analogous to the pattern already used in `prune_some_following` and `distribute_pending_rewards`:
   ```rust
   use ic_nervous_system_long_message::is_message_over_threshold;
   const MAX_INSTRUCTIONS: u64 = 1_000_000_000; // 1 B
   for (neuron_id, disbursement) in neuron_id_and_disbursements {
       if is_message_over_threshold(MAX_INSTRUCTIONS) { break; }
       ...
   }
   ```
2. **Replace the full-scan `filter_map`** with an index-driven lookup (analogous to NNS governance's `MaturityDisbursementIndex` / `get_neuron_ids_ready_to_finalize_maturity_disbursement`) so only neurons with due disbursements are visited.
3. **Process at most N neurons per heartbeat** and carry a cursor across invocations, as done in `groom_some_neurons`.

### Proof of Concept
1. Acquire enough SNS governance tokens to stake, say, 5 000 neurons.
2. On each neuron call `manage_neuron { DisburseMaturity { percentage_to_disburse: 1, ... } }`. Each call appends one `DisburseMaturityInProgress` entry with `finalize_disbursement_timestamp_seconds = now + 7 days`.
3. Wait for the next SNS governance heartbeat (`run_periodic_tasks`).
4. Observe that `maybe_finalize_disburse_maturity` iterates all 5 000 neurons in `filter_map`, paying stable-memory read cost per neuron; the message traps with `CanisterInstructionLimitExceeded`.

    async fn maybe_finalize_disburse_maturity(&mut self) {
        if !self.can_finalize_disburse_maturity() {
            return;
        }

        let maturity_modulation_basis_points =
            match self.proto.effective_maturity_modulation_basis_points() {
                Ok(maturity_modulation_basis_points) => maturity_modulation_basis_points,
                Err(message) => {
                    log!(ERROR, "{}", message.error_message);
                    return;
                }
            };

        self.proto.is_finalizing_disburse_maturity = Some(true);
        let now_seconds = self.env.now();
        // Filter all the neurons that are ready to disburse.
        let neuron_id_and_disbursements: Vec<(NeuronId, DisburseMaturityInProgress)> = self
            .proto
            .neurons
            .values()
            .filter_map(|neuron| {
                let id = match neuron.id.as_ref() {
                    Some(id) => id,
                    None => {
                        log!(
                            ERROR,
                            "NeuronId is not set for neuron. This should never happen. \
                             Cannot disburse."
                        );
                        return None;
                    }
                };
                // The first entry is the oldest one, check whether it can be completed.
                let first_disbursement = neuron.disburse_maturity_in_progress.first()?;
                let finalize_disbursement_timestamp_seconds =
                    match first_disbursement.finalize_disbursement_timestamp_seconds {
                        Some(finalize_disbursement_timestamp_seconds) => {
                            finalize_disbursement_timestamp_seconds
                        }
                        None => {
                            log!(
                                ERROR,
                                "Finalize disbursement timestamp is not set. Cannot disburse."
                            );
                            return None;
                        }
                    };
                if now_seconds >= finalize_disbursement_timestamp_seconds {
                    Some((id.clone(), first_disbursement.clone()))
                } else {
                    None
                }
            })
            .collect();

    /// Runs periodic tasks that are not directly triggered by user input.
    pub async fn run_periodic_tasks(&mut self) {
        use ic_cdk::println;

        self.process_proposals();

        // None of the upgrade-related tasks should interleave with one another or themselves, so we acquire a global
        // lock for the duration of their execution. This will return `false` if the lock has already been acquired less
        // than 10 minutes ago by a previous invocation of `run_periodic_tasks`, in which case we skip the
        // upgrade-related tasks.
        if self.acquire_upgrade_periodic_task_lock() {
            // We only want to check the upgrade status if we are currently executing an upgrade.
            if self.should_check_upgrade_status() {
                self.check_upgrade_status().await;
            }

            if self.should_refresh_cached_upgrade_steps() {
                match self.try_temporarily_lock_refresh_cached_upgrade_steps() {
                    Err(err) => {
                        log!(ERROR, "{}", err);
                    }
                    Ok(deployed_version) => {
                        self.refresh_cached_upgrade_steps(deployed_version).await;
                    }
                }
            }

            self.initiate_upgrade_if_sns_behind_target_version().await;

            self.release_upgrade_periodic_task_lock();
        }

        let should_distribute_rewards = self.should_distribute_rewards();

        // Getting the total governance token supply from the ledger is expensive enough
        // that we don't want to do it on every call to `run_periodic_tasks`. So
        // we only fetch it when it's needed, which is when rewards should be
        // distributed
        if should_distribute_rewards {
            match self.ledger.total_supply().await {
                Ok(supply) => {
                    // Distribute rewards
                    self.distribute_rewards(supply);
                }
                Err(e) => log!(
                    ERROR,
                    "Error when getting total governance token supply: {}",
                    GovernanceError::from(e)
                ),
            }
        }

        if self.should_update_maturity_modulation() {
            self.update_maturity_modulation().await;
        }

        self.maybe_finalize_disburse_maturity().await;

        self.maybe_move_staked_maturity();

        self.compute_cached_metrics().await;

        self.maybe_gc();
    }

// Once this amount of instructions is used by the
// Governance::prune_some_following, it stops, saves where it is, schedules more
// pruning later, and returns.
//
// Why this value seems to make sense:
//
// I think we can conservatively estimate that it takes 2e6 instructions to pull
// a neuron from stable memory. If we assume 200e3 neurons are in stable memory,
// then 400e9 instructions are needed to read all neurons in stable memory.
// 400e9 instructions / 50e6 instructions per batch = 8e3 batches. If we process
// 1 batch every 10 s (see PRUNE_FOLLOWING_INTERVAL), then it would take less
// than 23 hours to complete a full pass.
//
// This comes to 1.08 full passes per day. If each full pass uses 400e9
// instructions, then we use 432e9 instructions per day doing
// prune_some_following. If we assume 1 terainstruction costs 1 XDR,
// prune_some_following uses less than half an XDR per day.
const MAX_PRUNE_SOME_FOLLOWING_INSTRUCTIONS: u64 = 50_000_000;

pub(super) struct PruneFollowingTask {
    governance: &'static LocalKey<RefCell<Governance>>,
    begin: Bound<NeuronId>,
}

impl PruneFollowingTask {
    pub fn new(governance: &'static LocalKey<RefCell<Governance>>) -> Self {
        Self {
            governance,
            begin: Bound::Unbounded,
        }
    }
}

impl RecurringSyncTask for PruneFollowingTask {
    fn execute(self) -> (Duration, Self) {
        let new_begin = self.governance.with_borrow_mut(|governance| {
            let carry_on = || !is_message_over_threshold(MAX_PRUNE_SOME_FOLLOWING_INSTRUCTIONS);
            governance.prune_some_following(self.begin, carry_on)
        });

        (
            PRUNE_FOLLOWING_INTERVAL,
            Self {
                governance: self.governance,
                begin: new_begin,
            },
        )
    }
