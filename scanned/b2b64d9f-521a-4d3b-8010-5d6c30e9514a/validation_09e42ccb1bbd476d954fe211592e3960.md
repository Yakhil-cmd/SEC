### Title
Missing Canister-ID Validation Accepts Anonymous/Management Principal in SNS Swap Init — (File: `rs/sns/swap/src/types.rs`)

### Summary
The `validate_canister_id` function in `rs/sns/swap/src/types.rs` only checks that a string is a syntactically valid `PrincipalId`, but never verifies it is a legitimate canister principal (i.e., not the anonymous principal `2vxsx-fae` or the management canister `aaaaa-aa`). This function gates all five critical canister-ID fields in `Init::validate()`. A parallel gap exists in `rs/sns/governance/src/proposal.rs`, where the same-named helper uses `CanisterId::unchecked_from_principal` with no further checks when registering a `GenericNervousSystemFunction`.

### Finding Description

**Root cause 1 — SNS Swap `Init::validate()`**

`validate_canister_id` in `rs/sns/swap/src/types.rs` (lines 37–44) parses the input as a `PrincipalId` and returns `Ok(())` unconditionally on success:

```rust
pub fn validate_canister_id(p: &str) -> Result<(), String> {
    let _pp = PrincipalId::from_str(p).map_err(|x| { ... })?;
    Ok(())
}
``` [1](#0-0) 

This function is called for every critical canister-ID field in `Init::validate()`:

```rust
validate_canister_id(&self.nns_governance_canister_id)?;
validate_canister_id(&self.sns_governance_canister_id)?;
validate_canister_id(&self.sns_ledger_canister_id)?;
validate_canister_id(&self.icp_ledger_canister_id)?;
validate_canister_id(&self.sns_root_canister_id)?;
``` [2](#0-1) 

The anonymous principal (`2vxsx-fae`) and the management canister (`aaaaa-aa`) both parse successfully as `PrincipalId`, so both pass this check. The downstream accessor `principal_string_to_canister_id` also uses `CanisterId::unchecked_from_principal`, performing no further validation: [3](#0-2) 

The `nns_governance_canister_id` field is the sole authorization gate for who may call `open` on the swap. The `Swap::new` constructor panics only if `init.validate()` returns `Err`, so an `Init` carrying the anonymous principal for `nns_governance_canister_id` is accepted and stored without error: [4](#0-3) 

The integration-test corpus confirms this: both `rs/sns/integration_tests/src/swap.rs` and `rs/sns/integration_tests/src/timers.rs` install the swap canister with `Principal::anonymous()` for every canister-ID field and the canister initialises successfully: [5](#0-4) 

**Root cause 2 — SNS Governance `GenericNervousSystemFunction` registration**

`validate_canister_id` in `rs/sns/governance/src/proposal.rs` (lines 1240–1252) wraps any non-`None` `PrincipalId` with `CanisterId::unchecked_from_principal`, which the type's own documentation explicitly labels as performing no validation: [6](#0-5) 

This is the only check applied to `target_canister_id` and `validator_canister_id` when a `NervousSystemFunction` is converted to `ValidGenericNervousSystemFunction`. The management canister (`CanisterId::ic_00()`) is not present in the `DISALLOWED_TARGET_CANISTER_IDS` set used by `perform_add_generic_nervous_system_function`, so it passes registration. Any SNS neuron holder can submit such a proposal. [7](#0-6) 

### Impact Explanation

**Swap DoS / authorization bypass:** If `nns_governance_canister_id` is set to the management canister (`aaaaa-aa`), no real entity can ever call `open` on the swap. Any SNS tokens pre-funded into the swap canister become permanently locked, mirroring the "deploy function always reverts" scenario in the original report. If set to the anonymous principal, any unauthenticated ingress sender can call `open`, bypassing the NNS-governance-only authorization gate.

**Governance function misdirection:** A `GenericNervousSystemFunction` whose `target_canister_id` is the management canister passes SNS governance validation and is stored. On execution, the SNS governance canister issues an inter-canister call to `aaaaa-aa` with the proposal payload. If the governance canister happens to control the target of the embedded management-canister call, the call may succeed and produce unintended system-level effects (e.g., stopping or reinstalling a canister).

### Likelihood Explanation

The normal SNS deployment path (via SNS-W) populates canister IDs from actually-deployed canisters, so the swap-init vector requires either a bug in SNS-W or a direct deployment of the swap canister. The governance vector is more directly reachable: any SNS neuron holder can submit an `AddGenericNervousSystemFunction` proposal with the management canister as the target, and the current validation will not reject it.

### Recommendation

- Replace `validate_canister_id` in `rs/sns/swap/src/types.rs` with a check that additionally rejects the anonymous principal and the management canister (zero-length principal). Use `CanisterId::try_from_principal_id` rather than `unchecked_from_principal` to enforce the opaque-10-byte invariant.
- Replace `CanisterId::unchecked_from_principal` in `rs/sns/governance/src/proposal.rs`'s `validate_canister_id` with `CanisterId::try_from_principal_id`, and add the management canister to `DISALLOWED_TARGET_CANISTER_IDS`.
- Audit all other call sites of `unchecked_from_principal` that receive externally-supplied principal strings for the same class of missing validation.

### Proof of Concept

The following already-committed test code demonstrates that the SNS Swap canister accepts `Principal::anonymous()` for every canister-ID field without error, confirming the validation gap:

```rust
// rs/sns/integration_tests/src/swap.rs (lines 14-20)
let args = Encode!(&Init {
    nns_governance_canister_id: Principal::anonymous().to_string(),
    sns_governance_canister_id: Principal::anonymous().to_string(),
    sns_ledger_canister_id:     Principal::anonymous().to_string(),
    icp_ledger_canister_id:     Principal::anonymous().to_string(),
    sns_root_canister_id:       Principal::anonymous().to_string(),
    ...
``` [8](#0-7) 

The canister installs successfully (line 45–47), proving `Init::validate()` does not reject the anonymous principal for any of these fields. An operator who accidentally (or maliciously) supplies the management canister string `"aaaaa-aa"` for `nns_governance_canister_id` would produce an equally valid `Init`, permanently preventing the swap from being opened through NNS governance.

fn validate_canister_id(
    field_name: &str,
    canister_id: &Option<PrincipalId>,
    defects: &mut Vec<String>,
) -> Option<CanisterId> {
    match canister_id {
        None => {
            defects.push(format!("{field_name} field was not populated."));
            None
        }
        Some(canister_id) => Some(CanisterId::unchecked_from_principal(*canister_id)),
    }
}
