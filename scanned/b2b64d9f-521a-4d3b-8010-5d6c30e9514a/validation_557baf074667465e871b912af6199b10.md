### Title
V128 Cross-Group Spillover Check Skipped in `stable_dirty_pages_from_bytemap` — (`rs/embedders/src/wasmtime_embedder.rs`)

### Summary

A logic flaw in the bytemap fast-path optimization inside `stable_dirty_pages_from_bytemap` causes the V128 cross-page spillover check to be silently skipped when a written page sits at the last byte (index 15) of a u128 group and the immediately following u128 group is entirely zero. The result is that a physically-dirty stable memory page is omitted from the checkpoint dirty-page set, producing state divergence across replicas.

---

### Finding Description

`stable_dirty_pages_from_bytemap` splits the stable-memory bytemap into three slices via `align_to::<u128>()` — a byte-level `prefix`, a u128-aligned `middle`, and a byte-level `suffix` — and iterates over them to build the dirty-page list. [1](#0-0) 

For the `middle` slice, a fast-path skips the inner per-byte loop when the entire 16-byte group is zero: [2](#0-1) 

The inner helper `handle_bytemap_entry` maintains a boolean `previous_page_marked_written`. When a page is marked written (`written & 0x1 == 1`), it sets the flag to `true`. When the next page is **not** marked written, it uses the flag to perform a V128 spillover check — reading the first 15 bytes of that page and comparing them to the pre-execution snapshot: [3](#0-2) 

The invariant is: **every page immediately following a written page must be passed through `handle_bytemap_entry`**, because wasmtime's internal `memory.copy` implementation may use unaligned V128 stores that physically write up to 15 bytes past the end of the marked write range.

**The bug:** when `*group == 0` the `if` body is skipped entirely. Neither `previous_page_marked_written` is reset to `false`, nor is `handle_bytemap_entry` called for any page in the group — including the very first page (byte index 0 of the group), which is the direct spillover candidate from the last page of the preceding group (byte index 15).

Concrete broken state machine:

| Step | Event | `previous_page_marked_written` |
|------|-------|-------------------------------|
| Process group A, byte 15 | page written | `true` |
| Process group B (`*group == 0`) | **entire group skipped** | still `true` (not reset) |
| Process group C, byte 0 | spillover check fires on page 32 | reset to `false` |

Page 16 (group B byte 0, the true spillover target) is **never checked**. Page 32 (group C byte 0, 16 pages away) is **incorrectly checked** instead.

---

### Impact Explanation

A dirty stable memory page is silently excluded from the `stable_memory_dirty_pages` list returned by `run()`: [4](#0-3) 

This list drives checkpoint writes. A missing page means the checkpoint records the old page content. On replay or after a replica restart, the canister's stable memory diverges from the true post-execution state. This constitutes **state divergence** — different replicas may hold different stable memory contents for the same canister, breaking the replicated state machine guarantee.

---

### Likelihood Explanation

The conditions are fully under canister-developer control:

1. **Write to the last bytes of a stable memory page at group boundary.** Call `stable_write` or `stable64_write` with a destination ending within the last 15 bytes of a page whose bytemap index is 15 mod 16 (i.e., the last byte of a u128 group). The instrumented write marks exactly that page in the bytemap.
2. **Leave the next 16 pages unwritten.** Ensure pages N+1 through N+16 have never been written; their bytemap group is all zeros.
3. **Access page N+1.** Any read of page N+1 (e.g., a `stable_read`) causes the SIGSEGV handler to map it, so `tracker.is_accessed(index)` returns `true` — satisfying the guard inside `handle_bytemap_entry`.
4. **Trigger the V128 spillover.** Wasmtime's `memory.copy` implementation uses SIMD stores internally; a copy ending at the last bytes of page N physically writes into page N+1. The code's own comment confirms this is the expected threat model.

All four steps are achievable from a normal canister update call with no privileged access.

---

### Recommendation

When `*group == 0` is detected in the middle loop, the code must still handle the cross-group spillover case before skipping. Specifically:

- If `previous_page_marked_written` is `true` entering a zero group, call `handle_bytemap_entry` for **at least the first page** of that group (byte index 0) to perform the spillover check and reset the flag.
- Only after that may the remaining 15 pages of the group be skipped.

A minimal fix at line 1304:

```rust
for group in middle {
    if *group != 0 {
        for (group_index, written) in group.to_ne_bytes().iter().enumerate() {
            handle_bytemap_entry(
                &mut previous_page_marked_written,
                &mut result,
                page_index + group_index,
                heap_memory,
                &tracker,
                *written,
            )?;
        }
    } else {
        // Even if the whole group is zero, the first page may be a V128
        // spillover target from the last page of the previous group.
        if previous_page_marked_written {
            handle_bytemap_entry(
                &mut previous_page_marked_written,
                &mut result,
                page_index,
                heap_memory,
                &tracker,
                0u8,
            )?;
        }
        // previous_page_marked_written is now false; remaining 15 pages safe to skip.
    }
    page_index += size_of::<u128>();
}
```

### Proof of Concept

```
1. Deploy a canister with wasm-native stable memory.
2. In an update call:
   a. Grow stable memory to at least 33 pages.
   b. Call stable64_write(dst = PAGE_SIZE * 15 + PAGE_SIZE - 15, src = ..., len = 14)
      → writes 14 bytes ending at the last byte of page 15 (bytemap byte index 15 of group 0).
      → bytemap group 0 byte 15 = 1; group 1 (pages 16–31) = all zeros.
   c. Call stable64_read(dst = ..., src = PAGE_SIZE * 16, len = 1)
      → accesses page 16, causing tracker.is_accessed(16) = true.
3. Wasmtime's memory.copy for step (b) uses a V128 store that physically writes
   up to 15 bytes into page 16.
4. After execution, stable_dirty_pages_from_bytemap skips group 1 entirely
   (all zeros), never checking page 16 for spillover.
5. The checkpoint omits page 16. On replay, page 16 has its pre-execution content,
   diverging from the true post-execution state.
6. Assert: reference implementation (byte-by-byte scan) includes page 16 in dirty set;
   production code does not. Fuzz all combinations of written/unwritten pages at
   u128 group boundaries to confirm the discrepancy is systematic.
```
