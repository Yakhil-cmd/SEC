### Title
Unbounded Loop Over All Managed Token IDs in `post_upgrade` Can Exhaust Instruction Limit, Bricking the Orchestrator - (File: `rs/ethereum/ledger-suite-orchestrator/src/lifecycle/mod.rs`)

---

### Summary

The `post_upgrade` function in the Ledger Suite Orchestrator canister contains a synchronous, unbounded loop that iterates over every managed token ID and calls `schedule_now` for each one. Because `post_upgrade` is a synchronous canister hook with no `await` points, the entire loop must complete within a single message execution's instruction budget. As the number of managed ERC-20 tokens grows, this loop will eventually exhaust the IC's per-message instruction limit, causing every future NNS upgrade proposal that includes a ledger suite upgrade to trap and fail. The canister would become permanently unupgradeable via that path.

---

### Finding Description

In `post_upgrade`, when the `UpgradeArg` includes new wasm hashes (i.e., `valid_upgrade_args.upgrade_ledger_suite()` is `true`), the following synchronous loop executes:

```rust
for token_id in
    read_state(|s| s.all_managed_tokens_ids_iter().collect::<Vec<_>>())
{
    schedule_now(
        Task::UpgradeLedgerSuite(
            valid_upgrade_args.clone().into_task(token_id),
        ),
        &IC_CANISTER_RUNTIME,
    );
}
``` [1](#0-0) 

Per iteration, this performs:
1. `valid_upgrade_args.clone()` — clones the `UpgradeOrchestratorArgs` struct (containing up to three `WasmHash` values).
2. `.into_task(token_id)` — constructs an `UpgradeLedgerSuite` with a `Vec<UpgradeLedgerSuiteSubtask>` of up to 4 entries.
3. `schedule_now(...)` → `schedule_after(...)` → `TASKS.with(|t| t.borrow_mut().schedule_at(...))` — performs BTreeMap insertions and calls `runtime.global_timer_set(ts)` (a system call). [2](#0-1) 

The collection iterated is `all_managed_tokens_ids_iter()`, which returns every token in `managed_canisters` — both ERC-20 tokens and pre-existing ledger suites — with no upper bound. [3](#0-2) 

There is no cap on the number of managed tokens in the state definition: [4](#0-3) 

The same pattern also appears in `maybe_top_up`, which collects all managed principals (ledger + index + all archive canisters across all tokens) into a `BTreeSet` and fans out `future::join_all()` over all of them in a single async step, with unbounded synchronous pre- and post-await work: [5](#0-4) 

And in `discover_archives`, which similarly fans out inter-canister calls to all ledger canisters at once: [6](#0-5) 

---

### Impact Explanation

The IC enforces a per-message instruction limit (currently ~40 billion instructions for update calls and `post_upgrade` hooks). `post_upgrade` is a synchronous function — it has no `await` points, so the entire body must complete within one message execution. If the loop over managed token IDs consumes too many instructions, the `post_upgrade` call traps.

When `post_upgrade` traps, the canister upgrade is rolled back. This means:
- Any NNS upgrade proposal for the orchestrator that includes a ledger suite upgrade (`ledger_compressed_wasm_hash`, `index_compressed_wasm_hash`, or `archive_compressed_wasm_hash` set) will permanently fail once the token count is large enough.
- The orchestrator cannot receive security patches or feature updates via the standard NNS upgrade path.
- All ckERC20 tokens (ckUSDC, ckUSDT, etc.) managed by the orchestrator would be frozen at the old wasm version.

---

### Likelihood Explanation

The orchestrator is designed to grow: every new ERC-20 token added via an NNS proposal increases the loop size by one. The `manage_ledger_suites` upgrade path also adds pre-existing suites. The current production orchestrator already manages multiple ckERC20 tokens, and the number is expected to grow. The vulnerability is triggered by the natural, legitimate growth of the system — no adversarial action is required beyond submitting standard NNS `AddErc20Arg` proposals, which any NNS neuron holder can do. The exact instruction threshold depends on the IC runtime cost of BTreeMap operations and system calls per iteration, but the risk increases monotonically with token count.

---

### Recommendation

1. **`post_upgrade` loop**: Do not schedule all upgrade tasks synchronously in `post_upgrade`. Instead, schedule a single `Task::UpgradeAllLedgerSuites` task that the timer processes iteratively — one token per timer invocation — using the existing task queue. This is consistent with how the orchestrator already handles other long-running work.

2. **`maybe_top_up`**: Chunk the `managed_principals` set and process a bounded number of canisters per timer invocation, carrying a cursor in state between invocations.

3. **`discover_archives`**: Apply the same chunking approach — process a bounded number of ledgers per timer invocation.

4. Add a documented maximum on the number of managed tokens (analogous to the original report's recommendation to add a comment confirming the expected bound), or enforce it in `record_new_erc20_token` / `record_manage_other_canisters`.

---

### Proof of Concept

**Setup**: Deploy the orchestrator and add N ERC-20 tokens via repeated `AddErc20Arg` upgrade proposals (each is a standard NNS proposal executable by any neuron holder with sufficient voting power).

**Trigger**: Submit an NNS upgrade proposal for the orchestrator with `ledger_compressed_wasm_hash` set to a new wasm hash.

**Execution path**:
1. NNS executes the upgrade proposal → calls `post_upgrade(Some(UpgradeArg { ledger_compressed_wasm_hash: Some(...), ... }))`.
2. `valid_upgrade_args.upgrade_ledger_suite()` returns `true`.
3. The loop at lines 62–71 iterates N times, each time cloning `UpgradeOrchestratorArgs`, constructing an `UpgradeLedgerSuite` with up to 4 subtasks, and performing BTreeMap insertions + `global_timer_set` system calls.
4. At sufficiently large N, the instruction counter exceeds the per-message limit.
5. The IC traps the `post_upgrade` call; the upgrade is rolled back.
6. The orchestrator remains on the old wasm; the upgrade task queue is never populated; no managed ledger suite is upgraded. [7](#0-6)

pub fn post_upgrade(upgrade_arg: Option<UpgradeArg>) {
    if let Some(arg) = upgrade_arg {
        if let Some(git_commit_hash) = &arg.git_commit_hash {
            let git_commit_hash =
                GitCommitHash::from_str(git_commit_hash).expect("ERROR: invalid git commit hash");
            let ledger_suite_version = mutate_wasm_store(|s| {
                record_icrc1_ledger_suite_wasms(s, ic_cdk::api::time(), git_commit_hash)
            })
            .expect("BUG: failed to record icrc1 ledger suite wasms during upgrade");
            mutate_state(|s| s.init_ledger_suite_version(ledger_suite_version));
        }
        if let Some(manage_installed_canisters) = arg.manage_ledger_suites.clone() {
            for managed_canisters in manage_installed_canisters {
                let canisters =
                    read_state(|s| InstalledLedgerSuite::validate(s, managed_canisters))
                        .expect("ERROR: invalid manage installed canisters");
                mutate_state(|s| s.record_manage_other_canisters(canisters.clone()));
                log!(
                    INFO,
                    "[post_upgrade]: recorded manage installed canisters: {:?}",
                    canisters
                );
            }
        }
        match read_wasm_store(|w| UpgradeOrchestratorArgs::validate_upgrade_arg(w, arg.clone())) {
            Ok(valid_upgrade_args) => {
                if valid_upgrade_args.upgrade_ledger_suite() {
                    let current_ledger_suite_version =
                        read_state(|s| s.ledger_suite_version().cloned())
                            .expect("BUG: missing ledger suite version");
                    mutate_state(|s| {
                        s.update_ledger_suite_version(
                            valid_upgrade_args
                                .clone()
                                .new_ledger_suite_version(current_ledger_suite_version),
                        )
                    });
                    for token_id in
                        read_state(|s| s.all_managed_tokens_ids_iter().collect::<Vec<_>>())
                    {
                        schedule_now(
                            Task::UpgradeLedgerSuite(
                                valid_upgrade_args.clone().into_task(token_id),
                            ),
                            &IC_CANISTER_RUNTIME,
                        );
                    }
                }
            }
            Err(e) => {
                ic_cdk::trap(format!(
                    "[post_upgrade]: ERROR: invalid arguments to upgrade {arg:?}: {e:?}"
                ));
            }
        }
        if let Some(update) = arg.cycles_management {
            mutate_state(|s| update.apply(s.cycles_management_mut()));
        }
    }
    read_state(|s| s.validate_config().expect("ERROR: invalid state"));
    setup_tasks_and_timers()
}

pub fn schedule_after<R: CanisterRuntime>(delay: Duration, work: Task, runtime: &R) {
    log!(DEBUG, "Scheduling task {:?} after {:?}", work, delay);
    let now_nanos = runtime.time();
    let execute_at_ns = now_nanos.saturating_add(delay.as_secs().saturating_mul(SEC_NANOS));

    let execution_time = TASKS.with(|t| t.borrow_mut().schedule_at(execute_at_ns, work));
    set_global_timer(execution_time, runtime);
}

/// Schedules a task for immediate execution.
pub fn schedule_now<R: CanisterRuntime>(work: Task, runtime: &R) {
    schedule_after(Duration::from_secs(0), work, runtime)
}

async fn maybe_top_up<R: CanisterRuntime>(runtime: &R) -> Result<(), TaskError> {
    let managed_principals: BTreeSet<_> =
        read_state(|s| s.all_managed_principals().cloned().collect());
    if managed_principals.is_empty() {
        log!(INFO, "[maybe_top_up]: No managed canisters to top-up");
        return Ok(());
    }
    let cycles_management = read_state(|s| s.cycles_management().clone());
    let minimum_orchestrator_cycles =
        cycles_to_u128(cycles_management.minimum_orchestrator_cycles());
    let minimum_monitored_canister_cycles =
        cycles_to_u128(cycles_management.minimum_monitored_canister_cycles());
    let top_up_amount = cycles_to_u128(cycles_management.cycles_top_up_increment.clone());
    log!(
        INFO,
        "[maybe_top_up]: Managed canisters {}. \
        Cycles management: {cycles_management:?}. \
    Required amount of cycles for orchestrator to be able to top-up: {minimum_orchestrator_cycles}. \
    Monitored canister minimum target cycles balance {minimum_monitored_canister_cycles}",
        display_iter(&managed_principals)
    );

    let mut orchestrator_cycle_balance = match runtime.canister_cycles(runtime.id()).await {
        Ok(balance) => balance,
        Err(e) => {
            log!(
                INFO,
                "[maybe_top_up] failed to get orchestrator status, with error: {:?}",
                e
            );
            return Err(TaskError::CanisterStatusError(e));
        }
    };
    if orchestrator_cycle_balance < minimum_orchestrator_cycles {
        return Err(TaskError::InsufficientCyclesToTopUp {
            required: minimum_orchestrator_cycles,
            available: orchestrator_cycle_balance,
        });
    }

    let results = future::join_all(
        managed_principals
            .iter()
            .map(|p| runtime.canister_cycles(*p)),
    )
    .await;

async fn discover_archives<R: CanisterRuntime, F: Fn(&TokenId) -> bool>(
    selector: F,
    runtime: &R,
) -> Result<(), DiscoverArchivesError> {
    let ledgers: BTreeMap<_, _> = read_state(|s| {
        s.all_managed_canisters_iter()
            .filter(|(token, _)| selector(token))
            .filter_map(|(token_id, canisters)| {
                canisters
                    .ledger_canister_id()
                    .cloned()
                    .map(|ledger_id| (token_id, ledger_id))
            })
            .collect()
    });
    if ledgers.is_empty() {
        return Ok(());
    }
    log!(
        INFO,
        "[discover_archives]: discovering archives for {:?}",
        ledgers
    );
    let results = future::join_all(
        ledgers
            .values()
            .map(|p| call_ledger_icrc3_get_archives(*p, runtime)),
    )
    .await;
