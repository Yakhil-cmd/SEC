### Title
Unprivileged caller can trap `get_block_pb` on any archive node with non-zero offset via `checked_sub().unwrap()` panic — (`rs/ledger_suite/icp/archive/src/main.rs`)

### Summary
The `get_block_pb` query endpoint in the ICP archive canister calls `get_block()`, which performs `block_height.checked_sub(block_height_offset()).unwrap()` without any prior bounds check. When a caller supplies a `block_height` strictly less than the node's `block_height_offset`, `checked_sub` returns `None` and `.unwrap()` traps the query execution. Because `get_block_pb` has no caller restriction, any anonymous principal can trigger this trap.

### Finding Description
`get_block` at line 255 of `rs/ledger_suite/icp/archive/src/main.rs` performs:

```rust
let adjusted_height = block_height.checked_sub(block_height_offset()).unwrap();
```

There is no guard before this line. Every archive node beyond the first is initialized with a non-zero `block_height_offset` (e.g., the second archive node gets offset = capacity of the first). Sending any `block_height` in `[0, offset)` causes an unconditional trap. All three sibling endpoints (`get_blocks_pb`, `get_blocks`, `get_encoded_blocks`) perform explicit range validation before the subtraction; `get_block_pb` does not.

### Impact Explanation
Any call to `get_block_pb` with a height below the node's offset receives a trap response instead of a graceful `BlockRes(None)` or a structured error. Clients such as Rosetta, the index canister, or the CMC that rely on this endpoint and do not pre-filter by archive node range will receive traps for those heights. The canister's state is unaffected (query traps do not mutate state), but the endpoint is rendered unreliable for any caller that does not already know the exact offset of each archive node.

### Likelihood Explanation
The precondition is trivially met on mainnet: every ICP ledger archive node after the first has a non-zero `block_height_offset`. The attacker input is a single protobuf-encoded `u64` sent as an anonymous query — no cycles, no authentication, no prior state required. The exploit is deterministic and locally reproducible with a state-machine test.

### Recommendation
Replace the bare `.unwrap()` with an explicit bounds check, mirroring the pattern already used in `read_encoded_blocks`:

```rust
fn get_block(block_height: BlockIndex) -> BlockRes {
    let offset = block_height_offset();
    let adjusted_height = match block_height.checked_sub(offset) {
        Some(h) => h,
        None => return BlockRes(None),
    };
    BlockRes(get_block_stable(adjusted_height).map(Ok))
}
```

This matches the behavior of `get_blocks_pb` and the ICRC1 archive's `get_transaction`.

### Proof of Concept
```
1. Install archive canister with block_height_offset = 500.
2. Append at least one block (so the canister is live).
3. As anonymous principal, call get_block_pb with height = 499 (protobuf-encoded u64).
4. Observe: canister traps instead of returning BlockRes(None).
5. Call get_block_pb with height = 500 — returns the block normally.
6. Fuzz all u64 values in [0, 499]: every call traps.
7. Fuzz all u64 values in [500, 500+blocks_len): no trap.
```
