### Title
Stale Swap-Era `sns_tokens_per_icp` Used as Permanent Price Anchor in SNS Treasury Valuation - (File: rs/sns/governance/token_valuation/src/lib.rs)

### Summary

`IcpsPerSnsTokenClient::fetch_icps_per_sns_token` in the SNS governance token valuation library derives the current ICP-per-SNS-token price by anchoring to the `sns_tokens_per_icp` field returned by the swap canister's `get_derived_state` query, then adjusting only for token supply inflation. This `sns_tokens_per_icp` value is computed once at swap finalization and never updated afterward. The result is a permanently frozen, swap-era price that diverges from the real market price over time, causing the SNS treasury valuation used to enforce 7-day transfer limits to be systematically incorrect.

### Finding Description

`IcpsPerSnsTokenClient::fetch_icps_per_sns_token` in `rs/sns/governance/token_valuation/src/lib.rs` fetches three values concurrently:

1. `get_derived_state` from the swap canister — specifically `sns_tokens_per_icp`
2. Initial SNS token supply (from ledger transaction history)
3. Current SNS token supply (`icrc1_total_supply`)

It then computes:

```
current_icps_per_sns_token = (1 / initial_sns_tokens_per_icp) / (current_supply / initial_supply)
```

The critical flaw is in step 1. The `sns_tokens_per_icp` field in `DerivedState` is computed by `Swap::derived_state()` in `rs/sns/swap/src/swap.rs`:

```rust
let sns_tokens_per_icp = i2d(tokens_available_for_swap)
    .checked_div(i2d(participant_total_icp_e8s))
    .and_then(|d| d.to_f32())
    .unwrap_or(0.0);
```

This is `sns_token_e8s / total_icp_raised` — the clearing price from the swap. Once the swap reaches `COMMITTED` or `ABORTED`, the `buyers` map and `participant_total_icp_e8s` are frozen. The `sns_tokens_per_icp` value returned by `get_derived_state` is therefore permanently fixed at the swap's clearing price and never reflects post-swap market conditions.

The valuation library treats this frozen swap-era ratio as the current market price, adjusting only for supply inflation. This is analogous to the LP oracle bug in the report: instead of using a live price derived from actual current market conditions, it uses a stale cached value (the swap clearing price) that diverges from reality over time.

This valuation is used in `treasury_valuation_if_proposal_amount_is_small_enough_or_err` in `rs/sns/governance/src/proposal.rs` to enforce the 7-day treasury transfer limit (`ProposalsAmountTotalUpperBound`). An incorrect valuation directly affects whether treasury transfer proposals are accepted or rejected.

### Impact Explanation

**Governance accounting bug / cycles-resource accounting analog:**

- If the SNS token's market price rises significantly above the swap clearing price (e.g., 10× appreciation), the treasury valuation is understated. The `ProposalsAmountTotalUpperBound` will classify the treasury as "small" when it is actually "large," allowing proportionally larger transfers than the protocol intends. An SNS governance attacker who can pass proposals can drain more treasury funds than the 7-day limit is designed to permit.
- If the SNS token's market price falls significantly below the swap clearing price, the treasury is overstated as "large," causing the limit to be more restrictive than intended — blocking legitimate treasury operations.
- The `MIN_XDRS_PER_ICP` floor in `ProposalsAmountTotalUpperBound` partially mitigates the downside case but does not bound the upside divergence.
- The `sns_tokens_per_icp` field is stored as `f32` in `DerivedState` (losing precision) and then cast to `f64` in `GetDerivedStateResponse`, compounding the inaccuracy.

**Impact: Medium** — Incorrect treasury transfer limits; can allow excess treasury extraction or block legitimate transfers. Does not break consensus or allow arbitrary fund theft, but undermines the economic safety guarantees of the SNS governance treasury protection mechanism.

### Likelihood Explanation

**Likelihood: Medium** — Any SNS whose token price has moved materially since its swap (which is the normal case for any active SNS over months or years) will have a permanently diverged valuation. This is not a rare edge case; it is the default state for all mature SNS instances. Any SNS governance participant submitting a treasury transfer proposal triggers the valuation path. No special attacker capability is required beyond the ability to submit governance proposals (which is available to any neuron holder).

### Recommendation

Replace the swap-era `sns_tokens_per_icp` anchor with a live market price source. Options include:
1. Query the Exchange Rate Canister (XRC) directly for the SNS token / ICP pair if listed.
2. Use an on-chain DEX price (e.g., from an SNS-controlled liquidity pool) by reading actual pool balances and computing the ratio directly — analogous to the report's recommendation to use `balances / totalSupply` rather than a cached virtual price.
3. At minimum, document that `IcpsPerSnsTokenClient` produces a swap-era estimate and add a staleness bound check.

### Proof of Concept

**Entry path:** Any neuron holder submits a `TransferSnsTreasuryFunds` or `MintSnsTokens` proposal to SNS governance.

**Vulnerable call chain:**

1. SNS governance `proposal.rs` → `treasury_valuation_if_proposal_amount_is_small_enough_or_err` [1](#0-0) 

2. → `assess_treasury_balance` → `try_get_sns_token_balance_valuation` [2](#0-1) 

3. → `IcpsPerSnsTokenClient::fetch_icps_per_sns_token` fetches `get_derived_state` from the swap canister and reads `sns_tokens_per_icp` [3](#0-2) 

4. The swap canister's `derived_state()` computes `sns_tokens_per_icp` as `tokens_available_for_swap / participant_total_icp_e8s` — a value frozen at swap finalization [4](#0-3) 

5. The frozen ratio is used as the price anchor; only supply inflation adjusts it [5](#0-4) 

6. The resulting `Valuation` is fed into `ProposalsAmountTotalUpperBound::in_tokens` to determine the 7-day transfer cap [6](#0-5) 

**Concrete scenario:** An SNS launches with a swap clearing price of 100 SNS/ICP. One year later, the token appreciates to 10 SNS/ICP (10× price increase). The valuation library still computes the price as ~100 SNS/ICP (adjusted only for minting inflation), understating the treasury value by ~10×. The treasury is classified as "small" (under 100,000 XDR) when it is actually "large" (over 1,200,000 XDR), allowing the SNS to execute treasury transfers at the "small" fraction rate (up to 100% of treasury) rather than the "large" cap (300,000 XDR absolute maximum). This allows governance to extract far more value than the protocol intends in a single 7-day window.

    async fn fetch_icps_per_sns_token(&self) -> Result<Decimal, ValuationError> {
        // (Concurrently) fetch the various pieces that we need to sythensize the result:
        let (get_derived_state_result, initial_supply_e8s_result, current_supply_result) = join!(
            // 1. SNS token price from swap.
            call::<_, MyRuntime>(self.swap_canister_id, GetDerivedStateRequest {}),
            // 2. Initial SNS token supply.
            initial_supply_e8s::<MyRuntime>(
                self.sns_token_ledger_canister_id,
                InitialSupplyOptions::new()
            ),
            // 3. Current SNS token supply.
            MyRuntime::call_with_cleanup::<_, (Nat,)>(
                self.sns_token_ledger_canister_id,
                "icrc1_total_supply",
                ()
            ),
        );
        // (Factors 2 and 3 tell us how much inflation there has been. For
        // example, if the amount of tokens has doubled since the beginning,
        // then the current ICPs per SNS token should be half of what it was at
        // the time of the swap.)

        // Unwrap (intermediate) results.
        let get_derived_state_response = get_derived_state_result.map_err(|err| {
            ValuationError::new_external(format!(
                "Unable to obtain SNS token price at the time of the SNS initialization swap: {err:?}",
            ))
        })?;
        let initial_supply_e8s = initial_supply_e8s_result.map_err(|err| {
            ValuationError::new_external(format!(
                "Unable to determine the initial supply of SNS tokens: {err:?}",
            ))
        })?;
        let (current_supply_e8s,) = current_supply_result.map_err(|err| {
            ValuationError::new_external(format!(
                "Unable to obtain the current supply of SNS tokens: {err:?}",
            ))
        })?;

        // Read the relevant fields.

        // Here, a floating point field is used. This is ok, because we are just
        // using this to come up with a valuation, which isn't an exact science.
        let initial_sns_tokens_per_icp: f64 = get_derived_state_response
            .sns_tokens_per_icp
            .ok_or_else(|| {
                ValuationError::new_mismatch(format!(
                    "Response from swap ({}) get_derived_state call did not \
                     contain sns_tokens_per_icp: {:#?}",
                    self.swap_canister_id, get_derived_state_response,
                ))
            })?;

    pub fn derived_state(&self) -> DerivedState {
        let participant_total_icp_e8s = self.current_total_participation_e8s();
        let direct_participant_count = Some(self.buyers.len() as u64);
        let cf_participant_count = Some(self.cf_participants.len() as u64);
        let cf_neuron_count = Some(self.cf_neuron_count());
        let tokens_available_for_swap = match self.sns_token_e8s() {
            Ok(tokens) => tokens,
            Err(err) => {
                log!(ERROR, "{}", err);
                0
            }
        };
        let sns_tokens_per_icp = i2d(tokens_available_for_swap)
            .checked_div(i2d(participant_total_icp_e8s))
            .and_then(|d| d.to_f32())
            .unwrap_or(0.0);
        let direct_participation_icp_e8s = Some(self.current_direct_participation_e8s());
        let neurons_fund_participation_icp_e8s =
            Some(self.current_neurons_fund_participation_e8s());
        DerivedState {
            buyer_total_icp_e8s: participant_total_icp_e8s,
            direct_participant_count,
            cf_participant_count,
            cf_neuron_count,
            sns_tokens_per_icp,
            direct_participation_icp_e8s,
            neurons_fund_participation_icp_e8s,
        }
    }

            icps_per_token,
            xdrs_per_icp,

                // Overflow should not be possible, since fraction is supposed to be at most 1.0.
                .ok_or_else(|| {
                    ProposalsAmountTotalLimitError::new_arithmetic(format!(
                        "Unable to perform {balance_tokens} * {fraction}.",
                    ))
                })?,

                let xdrs_per_token = xdrs_per_icp.checked_mul(icps_per_token).ok_or_else(|| {
