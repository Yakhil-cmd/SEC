### Title
Silent `AmountTooSmall` Skip in `TransferableAmount::transfer_helper` Permanently Locks Buyer ICP in SNS Swap Canister - (File: `rs/sns/swap/src/types.rs`)

### Summary
`TransferableAmount::transfer_helper` silently returns `TransferResult::AmountTooSmall` when `amount <= fee` without setting any transfer timestamps. When this path is hit during `sweep_icp`, the buyer's ICP is permanently locked in the swap canister's subaccount because `error_refund_icp` unconditionally blocks refunds whenever `transfer_success_timestamp_seconds == 0`, with no alternative recovery mechanism.

### Finding Description

`transfer_helper` in `rs/sns/swap/src/types.rs` performs the following check:

```rust
let amount = Tokens::from_e8s(self.amount_e8s);
if amount <= fee {
    // Skip: amount too small...
    return TransferResult::AmountTooSmall;
}
``` [1](#0-0) 

When `AmountTooSmall` is returned, neither `transfer_start_timestamp_seconds` nor `transfer_success_timestamp_seconds` is set — both remain `0`. The function returns without performing any ledger call and without recording any shortfall.

In `sweep_icp` (`rs/sns/swap/src/swap.rs`), this result is handled as:

```rust
TransferResult::AmountTooSmall => {
    sweep_result.invalid += 1;
}
``` [2](#0-1) 

The buyer state is only updated on success (`result.is_success()`), so `transfer_success_timestamp_seconds` remains `0` for the affected buyer. [3](#0-2) 

The recovery path `error_refund_icp` then permanently blocks the buyer:

```rust
if let Some(transfer) = &buyer_state.icp
    && transfer.transfer_success_timestamp_seconds == 0
{
    return ErrorRefundIcpResponse::new_precondition_error(format!(
        "ICP cannot be refunded as principal {} has {} ICP (e8s) in escrow",
        ...
    ));
}
``` [4](#0-3) 

The guard assumes `transfer_success_timestamp_seconds == 0` means the normal sweep mechanism is still pending. It does not account for the case where `sweep_icp` has permanently failed for this buyer via `AmountTooSmall`. The buyer's ICP sits in the swap canister's subaccount indefinitely with no on-chain recovery path.

The same silent-skip pattern applies in `sweep_sns` for SNS token recipes: [5](#0-4) 

### Impact Explanation

A buyer whose `BuyerState.icp.amount_e8s <= DEFAULT_TRANSFER_FEE` at sweep time loses their entire ICP contribution permanently. The shortfall is not reflected in any accounting field — `amount_transferred_e8s` remains `Some(0)`, `transfer_fee_paid_e8s` remains `Some(0)`, and the ICP balance in the subaccount is never moved. The `sweep_result.invalid` counter is incremented but carries no remediation logic. The buyer cannot use `error_refund_icp` (blocked by the `transfer_success_timestamp_seconds == 0` guard) and there is no `error_refund_sns` equivalent for the SNS token case.

### Likelihood Explanation

The code comment acknowledges the scenario: *"AmountToSmall should never happen as the amount contributed is checked in `refresh_buyer_tokens`. In the case of a bug due to programmer error, increment the invalid field. This will require a manual intervention via an upgrade to correct."* [6](#0-5) 

The guard in `refresh_buyer_tokens` enforces `min_participant_icp_e8s`, which is an SNS-configurable parameter. If an SNS is initialized with `min_participant_icp_e8s` at or below `DEFAULT_TRANSFER_FEE` (10,000 e8s), or if a future governance upgrade changes the fee constant, the `AmountTooSmall` path becomes reachable for real buyers. Additionally, the `sweep_sns` path uses `init.transaction_fee_e8s_or_panic()` as the fee, which is SNS-specific and could be set to a value that makes small neuron recipe allocations fall below the threshold. The likelihood is low under correct parameterization but non-zero under adversarial or misconfigured SNS initialization.

### Recommendation

1. When `AmountTooSmall` is returned in `sweep_icp` or `sweep_sns`, set `transfer_success_timestamp_seconds` to a non-zero sentinel value so that `error_refund_icp` does not permanently block the affected buyer.
2. Alternatively, validate during SNS initialization that `min_participant_icp_e8s > DEFAULT_TRANSFER_FEE` and that the minimum SNS neuron recipe allocation exceeds `transaction_fee_e8s`, making the `AmountTooSmall` path unreachable by construction.
3. Add an explicit recovery path (analogous to `error_refund_icp`) for buyers whose sweep was permanently skipped as `invalid`.

### Proof of Concept

1. Deploy an SNS swap with `min_participant_icp_e8s = 5_000` (below `DEFAULT_TRANSFER_FEE = 10_000`).
2. Buyer calls `refresh_buyer_token_e8s` after transferring 5,000 e8s to their subaccount; the call succeeds and records `BuyerState { icp: TransferableAmount { amount_e8s: 5_000, transfer_success_timestamp_seconds: 0, ... } }`.
3. Swap reaches `Committed` or `Aborted`; `sweep_icp` is called.
4. `transfer_helper` evaluates `5_000 <= 10_000` → returns `AmountTooSmall`; `sweep_result.invalid += 1`; buyer state unchanged.
5. Buyer calls `error_refund_icp`; the guard `transfer_success_timestamp_seconds == 0` fires → `"ICP cannot be refunded as principal … has … ICP (e8s) in escrow"`.
6. Buyer's 5,000 e8s remain locked in the swap canister's subaccount with no on-chain recovery path. [7](#0-6) [8](#0-7)

    pub(crate) async fn transfer_helper(
        &mut self,
        now_fn: fn(bool) -> u64,
        fee: Tokens,
        subaccount: Option<Subaccount>,
        dst: &Account,
        ledger: &dyn ICRC1Ledger,
    ) -> TransferResult {
        let amount = Tokens::from_e8s(self.amount_e8s);
        if amount <= fee {
            // Skip: amount too small...
            return TransferResult::AmountTooSmall;
        }
        if self.transfer_start_timestamp_seconds > 0 {
            // Operation in progress...
            return TransferResult::AlreadyStarted;
        }
        self.transfer_start_timestamp_seconds = now_fn(false);

        // The ICRC1Ledger Trait converts any errors to Err(NervousSystemError).
        // No panics should occur when issuing this transfer.
        let result = ledger
            .transfer_funds(
                amount.get_e8s().saturating_sub(fee.get_e8s()),
                fee.get_e8s(),
                subaccount,
                *dst,
                0,
            )
            .await;
        if self.transfer_start_timestamp_seconds == 0 {
            log!(
                ERROR,
                "Token disburse logic error: expected transfer start time",
            );
        }
        match result {
            Ok(h) => {
                self.transfer_success_timestamp_seconds = now_fn(true);
                log!(
                    INFO,
                    "Transferred {} from subaccount {:?} to {} at height {} in Ledger Canister {}",
                    amount,
                    subaccount,
                    dst,
                    h,
                    ledger.canister_id()
                );
                TransferResult::Success(h)
            }
            Err(e) => {
                self.transfer_start_timestamp_seconds = 0;
                self.transfer_success_timestamp_seconds = 0;
                log!(
                    ERROR,
                    "Failed to transfer {} from subaccount {:#?}: {}",
                    amount,
                    subaccount,
                    e
                );
                TransferResult::Failure(e.to_string())
            }
        }
    }

    pub async fn error_refund_icp(
        &self,
        self_canister_id: CanisterId,
        request: &ErrorRefundIcpRequest,
        icp_ledger: &dyn ICRC1Ledger,
    ) -> ErrorRefundIcpResponse {
        // Fail if the request is premature.
        if !(self.lifecycle() == Lifecycle::Aborted || self.lifecycle() == Lifecycle::Committed) {
            return ErrorRefundIcpResponse::new_precondition_error(
                "Error refunds can only be performed when the swap is ABORTED or COMMITTED",
            );
        }

        // Unpack request.
        let source_principal_id = match request {
            ErrorRefundIcpRequest {
                source_principal_id: Some(source_principal_id),
            } => source_principal_id,
            _ => {
                return ErrorRefundIcpResponse::new_invalid_request_error(format!(
                    "Invalid request. Must have source_principal_id. Request:\n{request:#?}",
                ));
            }
        };

        if let Some(buyer_state) = self.buyers.get(&source_principal_id.to_string()) {
            if let Some(transfer) = &buyer_state.icp
                && transfer.transfer_success_timestamp_seconds == 0
            {
                // This buyer has ICP not yet disbursed using the normal mechanism.
                return ErrorRefundIcpResponse::new_precondition_error(format!(
                    "ICP cannot be refunded as principal {} has {} ICP (e8s) in escrow",
                    source_principal_id,
                    buyer_state.amount_icp_e8s()
                ));
            }
            // This buyer has participated in the swap, but all ICP
            // has already been disbursed, either back to the buyer
            // (aborted) or to the SNS Governance canister
            // (committed). Any ICP in this buyer's subaccount must
            // belong to the buyer.
        } else {
            // This buyer is not known to the swap canister. Any
            // balance in a subaccount belongs to the buyer.
        }

        let source_subaccount = principal_to_subaccount(source_principal_id);

        // Figure out how much to send back to source_principal_id based on
        // what's left in the subaccount.
        let account_balance_result = icp_ledger
            .account_balance(Account {
                owner: self_canister_id.into(),
                subaccount: Some(source_subaccount),
            })
            .await;
        let balance_e8s = match account_balance_result {
            Ok(balance) => balance.get_e8s(),
            Err(err) => {
                return ErrorRefundIcpResponse::new_external_error(format!(
                    "Unable to get the balance for the subaccount of {source_principal_id}: {err:?}",
                ));
            }
        };

        // Make transfer.
        let amount_e8s = balance_e8s.saturating_sub(DEFAULT_TRANSFER_FEE.get_e8s());
        let dst = Account {
            owner: source_principal_id.0,
            subaccount: None,
        };
        let transfer_result = icp_ledger
            .transfer_funds(
                amount_e8s,
                DEFAULT_TRANSFER_FEE.get_e8s(),
                Some(source_subaccount),
                dst,
                0, // memo
            )
            .await;
