### Title
Unbounded `StateSyncId.hash` Allocation in `state_sync_chunk_handler` Enables Per-Request Memory Amplification — (`rs/p2p/state_sync_manager/src/routes/chunk.rs`)

---

### Summary

A Byzantine subnet peer can craft a `StateSyncChunkRequest` protobuf with an arbitrarily large `hash` field (up to ~128 MiB, the transport ceiling) and cause the serving replica to heap-allocate the full hash bytes before the request is rejected. No field-level size guard exists between transport receipt and `CryptoHash` construction. With up to 1,000 concurrent QUIC streams per peer, this enables sustained memory pressure on a single replica.

---

### Finding Description

The handler `state_sync_chunk_handler` receives a raw `Bytes` payload and immediately decodes it as a `pb::StateSyncChunkRequest`:

```rust
let pb::StateSyncChunkRequest { id, chunk_id } =
    pb::StateSyncChunkRequest::decode(payload).map_err(|_| StatusCode::BAD_REQUEST)?;
let artifact_id: StateSyncArtifactId = id.map(From::from).ok_or(StatusCode::BAD_REQUEST)?;
``` [1](#0-0) 

The `From<p2p_pb::StateSyncId> for StateSyncArtifactId` conversion performs no size check on `id.hash`:

```rust
impl From<p2p_pb::StateSyncId> for StateSyncArtifactId {
    fn from(id: p2p_pb::StateSyncId) -> Self {
        Self {
            height: Height::from(id.height),
            hash: CryptoHash(id.hash),   // id.hash is an unbounded Vec<u8>
        }
    }
}
``` [2](#0-1) 

The only upstream size guard is the transport-layer `read_to_end` limit:

```rust
let request_bytes = recv_stream
    .read_to_end(MAX_MESSAGE_SIZE_BYTES)   // 128 MiB
    .await
``` [3](#0-2) 

where `MAX_MESSAGE_SIZE_BYTES = 128 * 1024 * 1024` — intentionally large to accommodate consensus summary blocks. [4](#0-3) 

The `StateSyncId.hash` proto field is an untyped `bytes` with no declared max length:

```proto
message StateSyncId {
  uint64 height = 1;
  bytes hash = 2;   // no size constraint
}
``` [5](#0-4) 

A 64 MiB `hash` field fits comfortably within the 128 MiB transport ceiling, so `prost::Message::decode` succeeds and the full `Vec<u8>` is heap-allocated before `state_sync.chunk()` returns `None` and the request is discarded.

---

### Impact Explanation

The QUIC transport allows up to 1,000 concurrent bidirectional streams per peer connection: [6](#0-5) 

The connection-level receive window is 200 MB and the per-stream window is 4 MB: [7](#0-6) 

The per-stream 4 MB window limits the instantaneous in-flight bytes per stream, but `read_to_end` accumulates all arriving bytes into a growing heap buffer; as the receiver reads and sends window updates, the sender can push the full 64 MiB through. The 200 MB connection window bounds the instantaneous total across all streams, but as streams complete and new ones open, the attacker can cycle through 1,000 streams continuously. The net effect is sustained heap pressure well above what a legitimate `StateSyncChunkRequest` (which should be ~40 bytes: a 32-byte hash + height + chunk_id) would ever require.

---

### Likelihood Explanation

The attacker must be a valid subnet peer (TLS-authenticated). This is a realistic Byzantine node scenario — a single compromised or malicious replica below the fault threshold. No governance majority, key compromise, or external infrastructure attack is required. The exploit is mechanically trivial: craft a protobuf with a large `bytes` field and send it repeatedly.

---

### Recommendation

Add an explicit size check on the decoded `hash` field before constructing `StateSyncArtifactId`. A legitimate `CryptoHash` is always 32 bytes (SHA-256). Reject any request where `id.hash.len() != 32` with `StatusCode::BAD_REQUEST` immediately after decode, before any heap allocation of the hash bytes propagates further. Alternatively, add a `RequestBodyLimit` middleware layer scoped specifically to the `/state-sync/chunk` route with a tight bound (e.g., 256 bytes).

---

### Proof of Concept

```rust
// Attacker-side: craft a StateSyncChunkRequest with a 64 MiB hash
let malicious_request = pb::StateSyncChunkRequest {
    id: Some(pb::StateSyncId {
        height: 1,
        hash: vec![0u8; 64 * 1024 * 1024],  // 64 MiB
    }),
    chunk_id: 0,
};
let mut buf = BytesMut::with_capacity(malicious_request.encoded_len());
malicious_request.encode(&mut buf).unwrap();
// Send buf as the body of POST /state-sync/chunk via QUIC transport
// Expected (buggy) behavior: handler allocates 64 MiB, returns NO_CONTENT
// Expected (fixed) behavior: handler returns BAD_REQUEST before allocation
```

The handler will decode successfully, construct `StateSyncArtifactId { hash: CryptoHash(vec![0u8; 64*1024*1024]) }`, call `state_sync.chunk()` which returns `None`, and respond `NO_CONTENT` — having allocated and then immediately discarded 64 MiB per request. [8](#0-7)

pub(crate) async fn state_sync_chunk_handler<T: 'static>(
    State(state): State<Arc<StateSyncChunkHandler<T>>>,
    payload: Bytes,
) -> Result<Bytes, StatusCode> {
    // Parse payload
    let pb::StateSyncChunkRequest { id, chunk_id } =
        pb::StateSyncChunkRequest::decode(payload).map_err(|_| StatusCode::BAD_REQUEST)?;
    let artifact_id: StateSyncArtifactId = id.map(From::from).ok_or(StatusCode::BAD_REQUEST)?;
    let chunk_id = ChunkId::from(chunk_id);

    let jh =
        tokio::task::spawn_blocking(
            move || match state.state_sync.chunk(&artifact_id, chunk_id) {
                Some(data) => {
                    let pb_chunk = pb::StateSyncChunkResponse { data: data.take() };
                    let mut raw = BytesMut::with_capacity(pb_chunk.encoded_len());
                    pb_chunk.encode(&mut raw).expect("Allocated enough memory");
                    let raw = raw.freeze();

                    let compressed = zstd::bulk::compress(&raw, zstd::DEFAULT_COMPRESSION_LEVEL)
                        .expect("Compression failed");
                    state
                        .metrics
                        .compression_ratio
                        .observe(raw.len() as f64 / compressed.len() as f64);
                    Ok(compressed)
                }
                None => Err(StatusCode::NO_CONTENT),
            },
        );
    let data = jh.await.map_err(|_| StatusCode::INTERNAL_SERVER_ERROR)??;

    Ok(data.into())
}
