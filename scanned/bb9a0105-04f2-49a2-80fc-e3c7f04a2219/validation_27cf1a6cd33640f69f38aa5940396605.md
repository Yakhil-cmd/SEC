### Title
Cache Key Collision Between `QueryV2` and `QueryV3` Allows Unsigned Response to Be Served for Signed-Response Endpoint — (`rs/boundary_node/ic_boundary/src/routes.rs`, `src/http/middleware/cache.rs`)

---

### Summary

The custom `Hash` and `PartialEq` implementations for `RequestContext` deliberately omit `request_type`. Combined with the fact that `BypasserIC` allows both `QueryV2` and `QueryV3` through the cache (both satisfy `is_query()`), a `QueryV3` request with the same `(canister_id, sender, method_name, ingress_expiry, arg)` as a previously cached `QueryV2` request will receive a cache hit and be served the `QueryV2` response body — which carries no node signature — in place of the signed envelope that `QueryV3` mandates.

---

### Finding Description

**Root cause — `RequestContext::Hash` and `RequestContext::PartialEq` omit `request_type`:** [1](#0-0) [2](#0-1) 

Both implementations hash/compare only `canister_id`, `sender`, `method_name`, `ingress_expiry`, and `arg`. `request_type` is never consulted. Two `RequestContext` values that differ only in `request_type` (e.g., `QueryV2` vs `QueryV3`) are therefore identical as cache keys.

**`BypasserIC` does not distinguish `QueryV2` from `QueryV3`:** [3](#0-2) [4](#0-3) 

`is_query()` returns `true` for `QueryV2`, `QueryV3`, and `QuerySubnetV3`. Neither variant is bypassed; both are eligible for caching and cache lookup under the same key.

**`KeyExtractorContext::extract` uses `Arc<RequestContext>` directly as the key:** [5](#0-4) 

The key is the `RequestContext` itself, so the collision is direct and unconditional.

**Both endpoints exist in production routing:** [6](#0-5) 

`/api/v2/canister/{id}/query` and `/api/v3/canister/{id}/query` are separate routes, but their `RequestContext` values hash and compare identically when all payload fields match.

---

### Impact Explanation

`/api/v3/canister/{id}/query` is the IC HTTP API endpoint that returns a **signed query response** — the replica attaches a node signature so the caller can verify authenticity without trusting the boundary node. `/api/v2/canister/{id}/query` returns an **unsigned** response.

When the cache serves a stored `QueryV2` response body to a `QueryV3` caller:

- The caller receives a CBOR envelope with no `signatures` field.
- Any client that skips or weakly implements signature verification accepts the unsigned payload as if it were certified.
- The boundary node has silently downgraded the security guarantee from "node-signed" to "trust the boundary node."

The attacker does not need to control the cached content — they only need to ensure a `QueryV2` request for the target `(canister_id, method_name, arg, sender, ingress_expiry)` tuple is in the cache before the victim issues a `QueryV3` request with the same parameters. This is trivially achievable by any unprivileged API client.

---

### Likelihood Explanation

- No privileged access required; any public API caller can prime the cache with a `QueryV2` request.
- The collision is deterministic and 100% reproducible for matching parameter tuples.
- `ingress_expiry` is the only time-varying field, but an attacker who controls both requests (or races against a known expiry window) can align it trivially.
- The bug is present in the current production code path with no existing guard.

---

### Recommendation

Include `request_type` in both `Hash` and `PartialEq` for `RequestContext`:

```rust
// In Hash impl
self.request_type.hash(state);

// In PartialEq impl
self.request_type == other.request_type && ...
```

Alternatively, add a separate `BypassReasonIC` variant that bypasses caching for `QueryV3` entirely (since its signed responses are per-node and not safely shareable across requests anyway).

---

### Proof of Concept

```rust
use std::hash::{Hash, DefaultHasher, Hasher};

fn hash_of(ctx: &RequestContext) -> u64 {
    let mut h = DefaultHasher::new();
    ctx.hash(&mut h);
    h.finish()
}

let ctx_v2 = RequestContext {
    request_type: RequestType::QueryV2,
    canister_id: Some(principal),
    sender: Some(anon),
    method_name: Some("foo".into()),
    ingress_expiry: Some(1000),
    arg: Some(vec![1,2,3]),
    ..Default::default()
};

let ctx_v3 = RequestContext { request_type: RequestType::QueryV3, ..ctx_v2.clone() };

assert_eq!(hash_of(&ctx_v2), hash_of(&ctx_v3)); // passes today
assert_eq!(ctx_v2, ctx_v3);                      // passes today

// Drive cache_middleware:
// 1. POST /api/v2/canister/{id}/query  → CacheStatus::Miss  (response stored, no signature)
// 2. POST /api/v3/canister/{id}/query  → CacheStatus::Hit   (unsigned v2 body returned)
// Assert: response body is identical and contains no `signatures` field.
```

impl Hash for RequestContext {
    fn hash<H: Hasher>(&self, state: &mut H) {
        self.canister_id.hash(state);
        self.sender.hash(state);
