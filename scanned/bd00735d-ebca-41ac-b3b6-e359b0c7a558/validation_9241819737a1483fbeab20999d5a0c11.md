### Title
Unescaped Memo Content Enables Markdown Injection into ICRC-21 Consent Message — (`packages/icrc-ledger-types/src/icrc21/responses.rs`)

---

### Summary

The `add_memo` function in `responses.rs` interpolates raw, user-controlled memo bytes (decoded as UTF-8) directly into a Markdown-formatted `GenericDisplayMessage` string without escaping backticks or newlines. An unprivileged caller can craft a `TransferArg` whose memo breaks out of the intended inline code span and injects arbitrary Markdown fields — including fake `**Amount:**` or `**To:**` entries — into the consent message shown to the user by a wallet or hardware device.

---

### Finding Description

In `packages/icrc-ledger-types/src/icrc21/responses.rs`, `add_memo` constructs the Markdown fragment:

```rust
message.push_str(&format!("\n\n**Memo:**\n`{memo_str}`"));
``` [1](#0-0) 

`memo_str` is the raw UTF-8 decoding of the caller-supplied memo bytes, with no sanitization:

```rust
let memo_str = match std::str::from_utf8(memo.as_slice()) {
    Ok(valid_str) => valid_str.to_string(),
    Err(_) => hex::encode(memo.as_slice()),
};
``` [2](#0-1) 

The hex-encoded fallback is safe (only `[0-9a-f]`), but the UTF-8 path is not. If `memo_str` begins with a backtick character, the resulting string is:

```
\n\n**Memo:**\n``\n\n**Amount:** `999999 ICP`
```

Under CommonMark rules, the two leading backticks (`` `` ``) open a double-backtick code span. Because no matching `` `` `` follows, they are rendered as literal backtick characters. The subsequent `\n\n**Amount:**` is then parsed as a normal Markdown bold field, and `` `999999 ICP` `` becomes a code span — producing a visually convincing fake field.

The call path from an unprivileged ingress message is direct:

1. Caller submits `icrc21_canister_call_consent_message` with `TransferArg { memo: Some(Memo(b"`\n\n**Amount:** `999999 ICP")), ... }`. [3](#0-2) 

2. `build_icrc21_consent_info` decodes the arg and passes the memo bytes as `GenericMemo::Icrc1Memo(memo.0)`. [4](#0-3) 

3. `build()` calls `message.add_memo(memo)` as the last step, appending the unescaped string to the already-complete message. [5](#0-4) 

4. The `GenericDisplayMessage` returned to the wallet contains the injected Markdown. [1](#0-0) 

---

### Impact Explanation

The `GenericDisplayMessage` variant is a Markdown string explicitly intended to be rendered by wallets and hardware signing devices. A crafted memo can append fake `**Amount:**`, `**To:**`, or other fields after the real ones. On a hardware wallet with a small scrollable display, a user reviewing the last visible field may be misled into believing the injected value is the authoritative one, and approve a transaction with a different real amount or recipient than displayed.

---

### Likelihood Explanation

- No privilege is required; any caller can invoke `icrc21_canister_call_consent_message`.
- The ICRC-1 memo field is a free-form byte blob up to 32 bytes; crafting a UTF-8 payload with a leading backtick and embedded newlines is trivial.
- The exploit is deterministic and locally testable with a unit test.
- The only precondition is that the wallet renders `GenericDisplayMessage` as Markdown, which is the intended and specified behavior of the ICRC-21 standard.

---

### Recommendation

Escape all Markdown control characters in `memo_str` before interpolation. At minimum, replace backtick (`` ` ``) and newline (`\n`, `\r`) characters:

```rust
let safe_memo_str = memo_str
    .replace('`', "\\`")
    .replace('\n', " ")
    .replace('\r', " ");
message.push_str(&format!("\n\n**Memo:**\n`{safe_memo_str}`"));
```

Alternatively, hex-encode the memo unconditionally for the `GenericDisplayMessage` path, reserving UTF-8 display only for the `FieldsDisplayMessage` path where the value is a structured `Value::Text` field and not embedded in a Markdown string.

---

### Proof of Concept

```rust
// memo bytes: backtick + newlines + fake bold field
let memo_bytes = b"`\n\n**Amount:** `999999 ICP".to_vec();
let transfer_arg = TransferArg {
    memo: Some(Memo(memo_bytes)),
    amount: Nat::from(1u64),
    to: real_receiver,
    fee: None,
    from_subaccount: None,
    created_at_time: None,
};
let request = ConsentMessageRequest {
    method: "icrc1_transfer".to_string(),
    arg: Encode!(&transfer_arg).unwrap(),
    user_preferences: ConsentMessageSpec {
        metadata: ConsentMessageMetadata { language: "en".to_string(), utc_offset_minutes: None },
        device_spec: Some(DisplayMessageType::GenericDisplay),
    },
};
let info = build_icrc21_consent_info(...).unwrap();
if let ConsentMessage::GenericDisplayMessage(msg) = info.consent_message {
    // The message contains a fake "**Amount:** `999999 ICP`" section
    // injected after the real amount field.
    assert!(msg.contains("**Amount:** `999999 ICP`")); // passes — injection confirmed
}
```

fn icrc21_canister_call_consent_message(
    consent_msg_request: ConsentMessageRequest,
) -> Result<ConsentInfo, Icrc21Error> {
    let caller_principal = caller();
    let ledger_fee = Nat::from(LEDGER.read().unwrap().transfer_fee.get_e8s());
    let token_symbol = LEDGER.read().unwrap().token_symbol.clone();
    let token_name = LEDGER.read().unwrap().token_name.clone();
    let decimals = ic_ledger_core::tokens::DECIMAL_PLACES as u8;

    if consent_msg_request.method == "transfer" {
        let TransferArgs {
            memo,
            amount,
            fee,
            from_subaccount,
            to,
            created_at_time: _,
        } = Decode!(&consent_msg_request.arg, TransferArgs).map_err(|e| {
            Icrc21Error::UnsupportedCanisterCall(ErrorInfo {
                description: format!("Failed to decode TransferArgs: {e}"),
            })
        })?;
        icrc21_check_fee(&Some(Nat::from(fee)), &ledger_fee)?;
        let from = if caller() == Principal::anonymous() {
            AccountOrId::AccountIdAddress(None)
        } else {
            let account = Account {
                owner: caller(),
                subaccount: from_subaccount.map(|sa| sa.0),
            };
            AccountOrId::AccountIdAddress(Some(AccountIdentifier::from(account).to_hex()))
        };
        let receiver = AccountIdentifier::from_slice(&to).map_err(|e| {
            Icrc21Error::UnsupportedCanisterCall(ErrorInfo {
                description: format!("Failed to parse receiver account id: {e}"),
            })
        })?;
        let args = GenericTransferArgs {
            from,
            receiver: AccountOrId::AccountIdAddress(Some(receiver.to_hex())),
            amount: Nat::from(amount.get_e8s()),
            memo: Some(GenericMemo::IntMemo(memo.0)),
        };
        build_icrc21_consent_info(
            consent_msg_request,
            caller_principal,
            ledger_fee,
            token_symbol,
            token_name,
            decimals,
            Some(args),
        )
