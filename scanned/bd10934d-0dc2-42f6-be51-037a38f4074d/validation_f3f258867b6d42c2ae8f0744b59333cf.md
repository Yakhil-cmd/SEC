### Title
`voting_power_refreshed_timestamp_seconds` Inactivity Penalty Bypassed via Neuron Split - (File: `rs/nns/governance/src/governance.rs`)

### Summary
The NNS governance canister enforces an inactivity penalty: neurons that have not refreshed their voting power for more than 6 months experience a linear reduction in `deciding_voting_power`, reaching zero after 7 months. However, `split_neuron` creates the child neuron with `voting_power_refreshed_timestamp_seconds` set to the current time (`created_timestamp_seconds`), regardless of the parent's staleness. This allows any neuron holder whose deciding voting power has been reduced or zeroed out to recover full deciding voting power on the split portion without performing any governance action.

### Finding Description
In `split_neuron()`, the child neuron is constructed via `NeuronBuilder::new()` with `created_timestamp_seconds` (i.e., `now()`) as the creation time. The `NeuronBuilder` default-initializes `voting_power_refreshed_timestamp_seconds` to `created_timestamp_seconds`: [1](#0-0) 

The child neuron construction in `split_neuron` copies many parent fields (`hot_keys`, `followees`, `kyc_verified`, `auto_stake_maturity`, `not_for_profit`, `joined_community_fund_timestamp_seconds`, `neuron_type`) but does **not** copy `voting_power_refreshed_timestamp_seconds`: [2](#0-1) 

The existing test suite explicitly asserts this behavior — the child's `voting_power_refreshed_timestamp_seconds` is always `now()` and always greater than the parent's: [3](#0-2) 

The `deciding_voting_power` is computed from `voting_power_refreshed_timestamp_seconds` via `VotingPowerEconomics.deciding_voting_power_adjustment_factor`: [4](#0-3) 

A neuron that has not refreshed for more than `start_reducing_voting_power_after_seconds` (default 6 months) + `clear_following_after_seconds` (default 1 month) has `deciding_voting_power = 0`. The split child always starts with a fresh timestamp, giving it full `deciding_voting_power` proportional to its stake.

### Impact Explanation
A neuron holder whose neuron has become fully stale (deciding VP = 0 due to 7+ months of inactivity) can call `ManageNeuron::Split` to create a child neuron. The child receives `voting_power_refreshed_timestamp_seconds = now()`, granting it full deciding voting power on its stake. The attacker recovers governance influence that the protocol explicitly intended to reduce to zero, without performing any legitimate governance action (voting, setting following, or calling `refresh_voting_power`). This undermines the inactivity-penalty invariant of NNS governance.

### Likelihood Explanation
Any unprivileged NNS governance user who controls a neuron with a stale `voting_power_refreshed_timestamp_seconds` can trigger this. The `ManageNeuron::Split` call is a standard, permissionless operation available to any neuron controller. No privileged access, key compromise, or subnet-majority is required. The only precondition is owning a neuron that has not refreshed in 6+ months, which is a realistic scenario for long-term passive holders.

### Recommendation
When constructing the child neuron in `split_neuron`, inherit the parent's `voting_power_refreshed_timestamp_seconds` rather than defaulting to `created_timestamp_seconds`. Add a `with_voting_power_refreshed_timestamp_seconds` call (the builder method already exists, currently gated to `#[cfg(test)]`) using the parent's value: [5](#0-4) 

This ensures the child neuron inherits the parent's staleness state, preventing the bypass. The same fix should be evaluated for `disburse_to_neuron`, which also creates a child with a fresh timestamp. [6](#0-5) 

### Proof of Concept
1. Neuron A has 1000 ICP staked and `voting_power_refreshed_timestamp_seconds` set 7+ months ago → `deciding_voting_power = 0`.
2. Attacker calls `ManageNeuron { command: Split { amount_e8s: 999_990_000, memo: None } }` on Neuron A.
3. Child Neuron B is created with `voting_power_refreshed_timestamp_seconds = now()` and ~999.9 ICP stake.
4. When the next proposal is created, Neuron B receives a ballot with full `deciding_voting_power` ≈ 999.9 ICP worth of voting power.
5. Neuron A retains 0 deciding VP (still stale), but Neuron B has full deciding VP.
6. Net result: the attacker recovered ~999.9 ICP worth of deciding voting power from a neuron that was supposed to have zero influence, without performing any governance action.
