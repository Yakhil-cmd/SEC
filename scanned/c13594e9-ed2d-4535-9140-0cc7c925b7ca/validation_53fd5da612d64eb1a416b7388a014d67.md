### Title
SNS Governance `PreInitializationSwap` Mode Permanently Blocks Token Withdrawal After Failed Swap - (File: `rs/sns/governance/src/types.rs`)

### Summary
When an SNS decentralization swap fails (aborts), the SNS governance canister remains permanently in `PreInitializationSwap` mode. This mode explicitly blocks `Disburse`, `DisburseMaturity`, `Configure` (including `StartDissolving`), `Split`, and `MergeMaturity` commands. Developer neuron holders whose SNS tokens were staked at SNS initialization cannot withdraw their staked tokens or disburse accumulated maturity after a failed swap, because the governance mode is never transitioned to `Normal` on the abort path.

### Finding Description

The SNS governance canister defines a `Mode` enum with `Normal` and `PreInitializationSwap` values. [1](#0-0) 

When an SNS is created, governance is placed in `PreInitializationSwap` mode. The function `allows_manage_neuron_command_or_err` enforces this restriction: [2](#0-1) 

The exhaustive list of blocked commands in `PreInitializationSwap` mode is confirmed in tests: [3](#0-2) 

`Disburse`, `DisburseMaturity`, `Configure` (which includes `StartDissolving`), `Split`, and `MergeMaturity` are all blocked. The only path that transitions governance to `Normal` mode is a **successful** swap finalization via `set_sns_governance_to_normal_mode`, called at the end of `finalize_inner` in the committed branch: [4](#0-3) 

In the **aborted** branch, `finalize_inner` returns early after restoring dapp controllers, and `set_sns_governance_to_normal_mode` is never called: [5](#0-4) 

This is explicitly documented in the integration test spec: [6](#0-5) 

The test confirms: `{ !DissolveSnsNeuron.is_enabled() } FinalizeUnSuccessfully { !DissolveSnsNeuron.is_enabled() }` — dissolving and disbursing remain permanently blocked after an aborted swap.

Developer neurons are created at SNS initialization with staked SNS tokens. After a failed swap, these neuron holders have no on-chain path to:
- Start dissolving their neurons
- Disburse their staked tokens
- Disburse accumulated maturity

There is no `emergencyWithdraw`-equivalent endpoint. Exiting `PreInitializationSwap` mode requires an NNS-level canister upgrade of the SNS governance.

### Impact Explanation

Developer neuron holders are permanently locked out of their staked SNS tokens and any accumulated maturity after a failed swap. The SNS governance canister has no self-service mechanism to exit `PreInitializationSwap` mode on the abort path. The SNS tokens themselves may have limited market value post-abort, but the governance maturity disbursement path (`DisburseMaturity`) is also blocked, and the principle that users must always be able to withdraw their principal is violated. Any SNS whose swap aborts leaves its developer neurons in a permanently frozen state without NNS intervention.

### Likelihood Explanation

SNS swaps abort whenever the minimum ICP participation threshold is not reached before the swap deadline. This is a realistic and documented lifecycle outcome (`LIFECYCLE_ABORTED`). The abort path is exercised in production and in integration tests. Every aborted SNS swap triggers this condition for all developer neuron holders.

### Recommendation

In `finalize_inner`, call `set_sns_governance_to_normal_mode` (or a new `Aborted` mode that permits `Disburse` and `DisburseMaturity` while still blocking governance-altering operations) on the abort path, before returning. Alternatively, add an `emergency_disburse` endpoint callable only when the swap is in `Aborted` lifecycle state, analogous to the `emergencyWithdraw` recommended in the original report.

### Proof of Concept

1. An SNS is created; SNS governance enters `PreInitializationSwap` mode. Developer neurons are minted with staked SNS tokens.
2. The swap opens but fails to reach `min_icp_e8s` before the deadline; `try_abort` transitions the swap to `Lifecycle::Aborted`.
3. `finalize_swap` is called. `finalize_inner` executes `sweep_icp` (refunds ICP to buyers), `settle_neurons_fund_participation`, then `restore_dapp_controllers_for_finalize`, and **returns early** — `set_sns_governance_to_normal_mode` is never invoked.
4. A developer neuron holder calls `manage_neuron` with `Command::Disburse`. `allows_manage_neuron_command_or_err` is called with `Mode::PreInitializationSwap`, returns `Err(PreconditionFailed, "Because governance is currently in PreInitializationSwap mode…")`.
5. The developer neuron holder has no further recourse; their staked SNS tokens and maturity are permanently inaccessible without an NNS-level upgrade. [7](#0-6) [8](#0-7)

    pub fn allows_manage_neuron_command_or_err(
        &self,
        command: &manage_neuron::Command,
        caller_is_swap_canister: bool,
    ) -> Result<(), GovernanceError> {
        use governance::Mode;
        match self {
            Mode::Unspecified => panic!("Governance's mode is not specified."),
            Mode::Normal => Ok(()),
            Mode::PreInitializationSwap => {
                Self::manage_neuron_command_is_allowed_in_pre_initialization_swap_or_err(
                    command,
                    caller_is_swap_canister,
                )
            }
        }
    }

    fn manage_neuron_command_is_allowed_in_pre_initialization_swap_or_err(
        command: &manage_neuron::Command,
        caller_is_swap_canister: bool,
    ) -> Result<(), GovernanceError> {
        use manage_neuron::Command as C;
        let ok = match command {
            C::Follow(_)
            | C::MakeProposal(_)
            | C::RegisterVote(_)
            | C::AddNeuronPermissions(_)
            | C::RemoveNeuronPermissions(_) => true,

            C::ClaimOrRefresh(_) => caller_is_swap_canister,

            _ => false,
        };

        if ok {
            return Ok(());
        }

        Err(GovernanceError::new_with_message(
            ErrorType::PreconditionFailed,
            format!(
                "Because governance is currently in PreInitializationSwap mode, \
                 manage_neuron commands of this type are not allowed \
                 (caller_is_swap_canister={caller_is_swap_canister}). command: {command:#?}",
            ),
        ))
    }

        &mut self,
        now_fn: fn(bool) -> u64,
        environment: &mut impl CanisterEnvironment,

        let mut finalize_swap_response = FinalizeSwapResponse::default();

        if let Err(e) = self.can_finalize() {
            finalize_swap_response.set_error_message(e);
            return finalize_swap_response;
        }

        // Transfer the ICP tokens from the Swap canister.
        finalize_swap_response
            .set_sweep_icp_result(self.sweep_icp(now_fn, environment.icp_ledger()).await);
        if finalize_swap_response.has_error_message() {
            return finalize_swap_response;
        }

        // Settle the Neurons' Fund participation in the token swap.
        finalize_swap_response.set_settle_neurons_fund_participation_result(
            self.settle_neurons_fund_participation(environment.nns_governance_mut())
                .await,
        );
        if finalize_swap_response.has_error_message() {
            return finalize_swap_response;
        }

        if self.should_restore_dapp_control() {
            // Restore controllers of dapp canisters to their original
            // owners (i.e. self.init.fallback_controller_principal_ids).

                self.restore_dapp_controllers_for_finalize(environment.sns_root_mut())
                    .await,
            );

            // In the case of returning control of the dapp(s) to the fallback
            // controllers, finalize() need not do any more work, so always return
            // and end execution.
            return finalize_swap_response;
