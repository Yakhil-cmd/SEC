### Title
Byzantine Subnet Node Can Permanently Freeze a Flexible HTTP Callback via Non-Candid Response Body — (`rs/https_outcalls/consensus/src/payload_builder.rs`)

---

### Summary

A single Byzantine subnet node can craft a `FlexibleCanisterHttpResponses` payload with a cryptographically valid but non-Candid-encoded response body. The payload passes `validate_canister_http_payload_impl` (which never Candid-decodes response bodies), gets finalized, and then `into_messages` silently drops the `ConsensusResponse` for that callback. Because `parse_past_payload_ids` records the callback_id as "delivered," the consensus layer never generates a timeout for it either. The canister's callback is permanently frozen with no response and no timeout.

---

### Finding Description

**Step 1 — Validation does not check Candid encoding.**

`validate_flexible_response_with_proof` in `utils.rs` checks:
- Callback ID consistency
- Signer is in the committee
- Registry version
- `crypto_hash(&response_with_proof.response)` matches the hash in the proof
- Content size and `is_reject` flag [1](#0-0) 

It does **not** check whether `CanisterHttpResponseContent::Success(data)` is valid Candid-encoded `CanisterHttpResponsePayload`. A Byzantine node can create a response with arbitrary bytes in `data`, compute the correct `crypto_hash` over that response, sign the metadata, and the proof will pass all validation checks.

**Step 2 — `into_messages` silently drops the callback on Candid failure.**

`flexible_ok_responses_into_consensus_response` Candid-decodes each response body: [2](#0-1) 

If any entry fails `Decode!(&data, CanisterHttpResponsePayload)`, the entire group returns `None`. The `into_messages` caller uses `.flatten()` on the `Option`, so the `ConsensusResponse` for that `callback_id` is silently omitted from the returned vec. [3](#0-2) 

The existing test `flexible_ok_responses_into_messages_decode_failure_is_skipped` explicitly confirms `responses.len() == 0` when a body is invalid Candid: [4](#0-3) 

**Step 3 — The "timeout fallback" claim in the comment is incorrect.**

The code comment at lines 992–996 states:

> "eventually a timeout will gracefully end the outstanding callback" [5](#0-4) 

This is wrong. `parse_past_payload_ids` extracts the `callback_id` from any finalized `FlexibleResponses` message and adds it to `delivered_ids`: [6](#0-5) 

In `get_canister_http_payload_impl`, the very first check for every open context is:

```rust
if delivered_ids.contains(callback_id) {
    continue;
}
``` [7](#0-6) 

The timeout check at line 234 is **never reached** for this callback_id. No timeout is ever proposed or finalized.

**Step 4 — Execution layer state is also stuck.**

`retrieve_context` removes a context from `canister_http_request_contexts` only when a `ConsensusResponse` is processed: [8](#0-7) 

Since no `ConsensusResponse` was delivered (step 2), the context remains in `canister_http_request_contexts` indefinitely. The consensus layer skips it (step 3). The execution layer never resolves it. The callback slot is permanently leaked.

---

### Impact Explanation

The canister's open callback for the targeted `callback_id` is permanently frozen:
- No response is ever delivered to the canister.
- No timeout is ever generated.
- The `CanisterHttpRequestContext` remains in `canister_http_request_contexts` forever, leaking the callback slot.
- The canister cannot make further HTTP outcalls if it exhausts its callback slots.
- Cycles attached to the request are never refunded.

---

### Likelihood Explanation

The attack requires a single Byzantine subnet node (within the `f < n/3` fault tolerance threshold), which is an explicitly modeled threat in the IC protocol. The node must:
1. Sign a response with non-Candid bytes in the body (trivial — just put arbitrary bytes in `CanisterHttpResponseContent::Success`).
2. Be the block proposer when a Flexible request with `min_responses = 1` is pending (happens in rotation; with `min_responses = 1` a single Byzantine node suffices).

No threshold corruption, no governance majority, no key compromise is required. The attack is local-testable as demonstrated by the existing test.

---

### Recommendation

1. **In `validate_canister_http_payload_impl`**, add a Candid-decode check for each `CanisterHttpResponseContent::Success(data)` in `flexible_responses`, rejecting the payload if any body fails `Decode!(&data, CanisterHttpResponsePayload)`.
2. **Remove or correct the misleading comment** at lines 992–996 — the timeout fallback does not work because `delivered_ids` suppresses it.
3. **Alternatively**, if Candid-decoding in validation is too expensive, change `into_messages` to emit a `SysTransient` reject `ConsensusResponse` on decode failure instead of returning `None`, so the callback is always resolved.

pub(crate) fn validate_flexible_response_with_proof(
    response_with_proof: &FlexibleCanisterHttpResponseWithProof,
    callback_id: CallbackId,
    flex_committee: &BTreeSet<NodeId>,
    seen_signers: &mut HashSet<NodeId>,
    consensus_registry_version: RegistryVersion,
    per_replica_allowance: Cycles,
) -> Result<(), InvalidCanisterHttpPayloadReason> {
    if response_with_proof.response.id != callback_id {
        return Err(
            InvalidCanisterHttpPayloadReason::FlexibleCallbackIdMismatch {
                callback_id,
                mismatched_id: response_with_proof.response.id,
            },
        );
    }

    validate_response_share(
        &response_with_proof.proof,
        callback_id,
        flex_committee,
        seen_signers,
        consensus_registry_version,
        per_replica_allowance,
    )?;

    let calculated_hash = crypto_hash(&response_with_proof.response);
    if &calculated_hash != response_with_proof.proof.content.content_hash() {
        return Err(InvalidCanisterHttpPayloadReason::ContentHashMismatch {
            metadata_hash: response_with_proof.proof.content.content_hash().clone(),
            calculated_hash,
        });
    }

    let calculated_size = response_with_proof.response.content.count_bytes() as u32;
    if calculated_size != response_with_proof.proof.content.content_size() {
        return Err(InvalidCanisterHttpPayloadReason::ContentSizeMismatch {
            metadata_size: response_with_proof.proof.content.content_size(),
            calculated_size,
        });
    }

    let calculated_is_reject = response_with_proof.response.content.is_reject();
    if calculated_is_reject != response_with_proof.proof.content.is_reject() {
        return Err(InvalidCanisterHttpPayloadReason::IsRejectMismatch {
            metadata_is_reject: response_with_proof.proof.content.is_reject(),
            calculated_is_reject,
        });
    }

    Ok(())

fn flexible_ok_responses_into_messages_decode_failure_is_skipped() {
    let callback_id = CallbackId::from(42);

    let valid_data = Encode!(&CanisterHttpResponsePayload {
        status: 200,
        headers: vec![],
        body: vec![],
    })
    .unwrap();

    let invalid_entry = flexible_response(42, 1, b"this is invalid candid");

    let payload = flexible_payload(vec![FlexibleCanisterHttpResponses {
        callback_id,

    let (responses, stats) = CanisterHttpPayloadBuilderImpl::into_messages(&bytes);

    assert_eq!(responses.len(), 0);
    assert_eq!(stats.flexible_ok_responses, 0);
    assert_eq!(stats.flexible_ok_responses_candid_failures, 1);
}

fn get_id_from_message(message: CanisterHttpResponseMessage) -> Option<u64> {
    match message.message_type {
        Some(MessageType::Response(response)) => response.response.map(|response| response.id),
        // NOTE: We simply use the id from the first metadata share
        // All metadata shares have the same id, otherwise they would not have been included as a past payload

            .shares
            .first()
            .and_then(|share| share.metadata.as_ref().map(|md| md.id)),
        Some(MessageType::FlexibleResponses(flex_responses)) => Some(flex_responses.callback_id),
        Some(MessageType::FlexibleError(flex_error)) => Some(flex_error.callback_id),
        Some(MessageType::Timeout(id)) => Some(id),
        None => None,
    }
