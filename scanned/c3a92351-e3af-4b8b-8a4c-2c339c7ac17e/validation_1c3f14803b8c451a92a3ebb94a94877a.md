### Title
Irrecoverable Maturity Loss When Ledger Mint Fails and Neuron State Cannot Be Restored — (`rs/nns/governance/src/governance/disburse_maturity.rs`)

---

### Summary

In `try_finalize_maturity_disbursement`, the NNS governance canister pops a pending maturity disbursement from the neuron's queue **before** issuing the ledger mint call. If the mint fails and the subsequent attempt to push the disbursement back to the front of the queue also fails, the disbursement record is permanently destroyed, the maturity is never minted to the user, and the neuron is permanently locked. There is no recovery path.

---

### Finding Description

`try_finalize_maturity_disbursement` executes the following sequence:

**Step 1 — Pop the disbursement (state committed before the await):** [1](#0-0) 

The disbursement is removed from `maturity_disbursements_in_progress` and this mutation is committed to canister state before the cross-canister call is issued.

**Step 2 — Issue the ledger mint (cross-canister await):** [2](#0-1) 

**Step 3 — On mint failure, attempt to push the disbursement back:** [3](#0-2) 

**Step 4 — If the push-back also fails, retain the neuron lock permanently:** [4](#0-3) 

The comment at line 665–667 explicitly acknowledges: *"The best we can do at this point is to retain the neuron lock."* The maturity has already been deducted from `maturity_e8s_equivalent` at disbursement initiation time, the disbursement record is gone from the queue, no ICP was minted, and the neuron is now permanently locked.

The TLA+ formal model explicitly encodes this non-deterministic outcome as the `skip` branch (the "either…or" construct), confirming it is a known reachable state: [5](#0-4) 

The `skip` branch maps directly to the `neuron_lock.retain()` path in the Rust implementation.

---

### Impact Explanation

- **Maturity is permanently destroyed**: the neuron's `maturity_e8s_equivalent` was reduced at `DisburseMaturity` initiation time; the disbursement record is gone; no ICP is ever minted to the user. The tokens are lost with no on-chain evidence of what happened.
- **Neuron is permanently locked**: `neuron_lock.retain()` keeps the in-flight command in `in_flight_commands` indefinitely. No further `manage_neuron` operations (dissolve, vote, stake, etc.) can be performed on that neuron.
- **No retry path exists**: unlike the normal failure path (where the disbursement is pushed back and retried on the next timer tick), this path leaves no disbursement record to retry.

---

### Likelihood Explanation

The double-failure requires:
1. The ICP ledger returns an error (e.g., `TemporarilyUnavailable`) — this is a realistic transient condition.
2. `with_neuron_mut` on the push-back call fails — this requires the neuron to be absent from the neuron store between the pop and the push-back.

Between the pop (committed before the `await`) and the push-back (after the `await` returns), the canister processes the ledger response in a new execution round. Any concurrent upgrade, neuron-store migration, or state-corruption bug that removes the neuron from the store during that window triggers the double-failure. The TLA+ model's explicit `skip` branch indicates the DFINITY team considers this reachable. Likelihood is **low but non-zero**, and the impact when triggered is **irreversible**.

---

### Recommendation

1. **Do not pop before the mint.** Restructure `try_finalize_maturity_disbursement` to keep the disbursement in the queue until the mint succeeds, mirroring the SNS governance approach where `disburse_maturity_in_progress.remove(0)` is called only after a confirmed successful transfer. [6](#0-5) 

2. **Add a quarantine list for unrestorable disbursements.** If the push-back fails, record the disbursement in a dedicated `quarantined_disbursements` map (analogous to `quarantine_withdrawal_reimbursement` in ckBTC) so that governance upgrade can recover the funds manually. [7](#0-6) 

3. **Emit an observable event** when the push-back fails, so the loss is detectable on-chain rather than only in canister logs.

---

### Proof of Concept

1. Neuron controller sends `manage_neuron { DisburseMaturity { percentage_to_disburse: 100, … } }` (unprivileged ingress). Maturity is deducted from `maturity_e8s_equivalent` and a `MaturityDisbursement` record is appended to `maturity_disbursements_in_progress`. [8](#0-7) 

2. After `DISBURSEMENT_DELAY_SECONDS` (7 days), the governance timer fires `finalize_maturity_disbursement` → `try_finalize_maturity_disbursement`.

3. The disbursement is popped from the queue (state committed to stable memory). [9](#0-8) 

4. The ICP ledger returns `TemporarilyUnavailable` (e.g., ledger is mid-upgrade). `mint_result` is `Err`.

5. `with_neuron_mut` for the push-back fails (e.g., neuron store is in an inconsistent state after a concurrent canister upgrade between the two execution rounds). `reverse_neuron_result` is `Err`.

6. `neuron_lock.retain()` is called. The neuron is permanently locked. The maturity is permanently lost. The user has no recourse. [4](#0-3)

```rust
/// Initiates the maturity disbursement process for a neuron.
pub fn initiate_maturity_disbursement(
    neuron_store: &mut NeuronStore,
    caller: &PrincipalId,
    id: &NeuronId,
    disburse_maturity: &DisburseMaturity,
    now_seconds: u64,
) -> Result<u64, InitiateMaturityDisbursementError> {
    let DisburseMaturity {
        percentage_to_disburse,
        to_account,
        to_account_identifier,
    } = disburse_maturity;

    if *percentage_to_disburse == 0 || *percentage_to_disburse > 100 {
        return Err(InitiateMaturityDisbursementError::InvalidPercentage);
    }

    let destination = Destination::try_new(to_account, to_account_identifier, *caller)
        .map_err(|reason| InitiateMaturityDisbursementError::InvalidDestination { reason })?;

    let timestamp_of_disbursement_seconds = now_seconds;
    let finalize_disbursement_timestamp_seconds = now_seconds + DISBURSEMENT_DELAY_SECONDS;

    let (
        is_neuron_spawning,
        is_neuron_controlled_by_caller,
        num_disbursements,
        maturity_e8s_equivalent,
    ) = neuron_store
        .with_neuron(id, |neuron| {
            let is_neuron_spawning = neuron.state(now_seconds) == NeuronState::Spawning;
            let is_neuron_controlled_by_caller = neuron.is_controlled_by(caller);
            let num_disbursements = neuron.maturity_disbursements_in_progress().len();
            let maturity_e8s_equivalent = neuron.maturity_e8s_equivalent;
            (
                is_neuron_spawning,
                is_neuron_controlled_by_caller,
                num_disbursements,
                maturity_e8s_equivalent,
            )
        })
        .map_err(|_| InitiateMaturityDisbursementError::NeuronNotFound)?;

    let disbursement_maturity_e8s =
        percentage_of_maturity(maturity_e8s_equivalent, *percentage_to_disburse)?;
    if disbursement_maturity_e8s < MINIMUM_DISBURSEMENT_E8S {
        return Err(InitiateMaturityDisbursementError::DisbursementTooSmall {
            disbursement_maturity_e8s,
            minimum_disbursement_e8s: MINIMUM_DISBURSEMENT_E8S,
        });
    }

    if is_neuron_spawning {
        return Err(InitiateMaturityDisbursementError::NeuronSpawning);
    }
    if !is_neuron_controlled_by_caller {
        return Err(InitiateMaturityDisbursementError::CallerIsNotNeuronController);
    }
    if num_disbursements >= MAX_NUM_DISBURSEMENTS {
        return Err(InitiateMaturityDisbursementError::TooManyDisbursements);
    }

    let disbursement_in_progress = MaturityDisbursement {
        destination: Some(destination),
        amount_e8s: disbursement_maturity_e8s,
        timestamp_of_disbursement_seconds,
        finalize_disbursement_timestamp_seconds,
    };

    neuron_store
        .with_neuron_mut(id, |neuron| {
            neuron.add_maturity_disbursement_in_progress(disbursement_in_progress);
            neuron.maturity_e8s_equivalent = neuron
                .maturity_e8s_equivalent
                .saturating_sub(disbursement_maturity_e8s);
        })
        .map_err(|_| InitiateMaturityDisbursementError::Unknown {
            reason: "Failed to update neuron even though it was found before".to_string(),
        })?;

    Ok(disbursement_maturity_e8s)
```

```rust
    pub fn schedule_withdrawal_reimbursement(
        &mut self,
        ledger_burn_index: LedgerBurnIndex,
        reimburse_deposit_task: ReimburseWithdrawalTask,
    ) {
        self.pending_retrieve_btc_requests
            .retain(|req| req.block_index != ledger_burn_index);

        if let Some(tx_status) = self.requests_in_flight.get(&ledger_burn_index) {
            panic!(
                "BUG: Cannot reimburse withdrawal request {ledger_burn_index} since there is a transaction for that withdrawal with status: {tx_status:?}"
            )
        }

        for submitted_tx in self
            .submitted_transactions
            .iter()
            .chain(self.stuck_transactions.iter())
        {
            if submitted_tx
                .requests
                .iter_block_index()
                .any(|block_index| block_index == ledger_burn_index)
            {
                panic!(
                    "BUG: Cannot reimburse withdrawal request {ledger_burn_index} since there is a submitted transaction for that withdrawal: {submitted_tx:?}"
                );
            }
        }
        self.pending_withdrawal_reimbursements
```
