### Title
Unbounded JoinSet Task Spawn + AlreadyInserted Unwanted Return via Crafted StrippedBlockProposal — (`rs/p2p/artifact_downloader/src/fetch_stripped_artifact/assembler.rs`)

### Summary

A Byzantine peer below the fault threshold can push a `StrippedBlockProposal` with an arbitrarily large or duplicate-filled `ingress_messages` / `stripped_idkg_dealings` list. Because `assemble_message` spawns one uncapped `JoinSet` task per entry and treats `AlreadyInserted` as a fatal assembly error returning `AssembleResult::Unwanted`, the attacker can (a) exhaust local task/memory resources and (b) permanently prevent the victim replica from assembling a valid block for that height if it is the sole advertiser.

### Finding Description

**Entrypoint — pushed artifact accepted without content validation**

`FetchArtifact::download_artifact` (in `rs/p2p/artifact_downloader/src/fetch_artifact/download.rs`) returns a pushed artifact as-is:

```rust
Some((artifact, peer_id)) => AssembleResult::Done {
    message: artifact,
    peer_id,
},
``` [1](#0-0) 

The comment even acknowledges no content check is performed. The artifact ID for `MaybeStrippedConsensusMessage` is derived solely from `unstripped_consensus_message_id`:

```rust
MaybeStrippedConsensusMessage::StrippedBlockProposal(stripped) => {
    stripped.unstripped_consensus_message_id.clone()
}
``` [2](#0-1) 

The `ingress_messages` and `stripped_idkg_dealings` lists are **not part of the ID** and are never validated against the actual block proposal in the consensus pool.

**No count limit during deserialization**

`TryFrom<pb::StrippedBlockProposal> for StrippedBlockProposal` collects both lists without any length bound:

```rust
stripped_ingress_payload: StrippedIngressPayload {
    ingress_messages: value
        .ingress_messages
        .into_iter()
        .map(SignedIngressId::try_from)
        .collect::<Result<Vec<_>, _>>()?,
},
``` [3](#0-2) 

**Uncapped JoinSet spawn loop**

`assemble_message` iterates over every entry returned by `missing_stripped_messages()` and spawns one task per entry with no cap:

```rust
for stripped_message_id in stripped_message_ids {
    join_set.spawn(get_or_fetch(
        stripped_message_id,
        ...
    ));
}
``` [4](#0-3) 

`missing_stripped_messages()` returns all entries where the slot is `None`, including duplicates if the Byzantine peer injected them: [5](#0-4) 

**AlreadyInserted terminates assembly as Unwanted**

When a duplicate ID's second task completes, `try_insert` finds the slot already filled and returns `AlreadyInserted`. The caller treats any `InsertionError` as fatal:

```rust
if let Err(err) = assembler.try_insert_stripped_message(message) {
    warn!(self.log, "Failed to insert stripped message of type {}: {}. This is a bug.", ...);
    return AssembleResult::Unwanted;
}
``` [6](#0-5)

### Impact Explanation

**Resource exhaustion (Issue 1):** N tasks are spawned immediately before the bouncer loop starts. Each `get_or_fetch` task holds clones of `Arc<dyn Transport>`, `Arc<RwLock<Pool>>`, `ReplicaLogger`, and `Arc<Metrics>`, plus a pending async future. With N = transport-message-size / per-entry-size (potentially thousands), this causes a spike in Tokio task count and heap allocation on the victim replica for the duration of the assembly window.

**Silent block discard (Issue 2):** With even a single duplicate entry, the second task to complete triggers `AlreadyInserted → AssembleResult::Unwanted`. If the Byzantine peer is the sole advertiser of the `StrippedBlockProposal` for that height (e.g., it raced the legitimate proposer), the victim replica permanently fails to assemble the block for that height from that artifact, stalling its local progress until the bouncer expires or another peer re-advertises.

### Likelihood Explanation

- A Byzantine peer below the fault threshold can observe the `unstripped_consensus_message_id` of any in-flight block proposal (it is public in the P2P advertisement).
- It can craft a `StrippedBlockProposal` proto with the correct ID but with 10,000 duplicate `StrippedIngressMessage` entries and push it to a target replica before the legitimate proposer does.
- No privileged access, key material, or majority corruption is required.
- The attack is local-testable: construct a `pb::StrippedBlockProposal` with repeated entries, call `assemble_message`, and observe JoinSet size and the `Unwanted` return.

### Recommendation

1. **Deduplicate entries in `BlockProposalAssembler::new`**: Use a `BTreeSet` or deduplicate the `ingress_messages` and `stripped_dealings` vecs before building the assembler state, so duplicate IDs from a peer never produce duplicate tasks.
2. **Enforce a count cap at deserialization**: In `TryFrom<pb::StrippedBlockProposal>`, reject any proto where `ingress_messages.len()` exceeds the protocol maximum (e.g., `MAX_INGRESS_MESSAGES_PER_BLOCK`) or `stripped_idkg_dealings.len()` exceeds the maximum dealings per block.
3. **Treat `AlreadyInserted` as non-fatal or as a peer-misbehavior signal** rather than immediately returning `Unwanted`, to avoid silently discarding a legitimately assembled block.

```rust

    pruned_block_proposal_proto: pb::BlockProposal::default(),
    unstripped_consensus_message_id: /* valid ID from observed advert */,
    stripped_ingress_payload: StrippedIngressPayload {
        ingress_messages: duplicate_ids,
    },
    stripped_idkg_dealings: StrippedIDkgDealings::default(),

let assembler = BlockProposalAssembler::new(stripped);

    fn try_insert(
        &mut self,
        signed_ingress_id: SignedIngressId,
        ingress_message: SignedIngress,
    ) -> Result<(), InsertionError> {
        // We can have at most 1000 elements in the vector, so it should be reasonably fast to do a
        // linear scan here.
        let (_, ingress) = self
            .ingress_messages
            .iter_mut()
            .find(|(id, _maybe_ingress)| *id == signed_ingress_id)
            .ok_or(InsertionError::NotNeeded)?;

        if ingress.is_some() {
            Err(InsertionError::AlreadyInserted)
        } else {
            *ingress = Some(ingress_message);
            Ok(())
        }

    pub(crate) fn missing_stripped_messages(&self) -> Vec<StrippedMessageId> {
        let ingress_messages = PayloadAssembler::<SignedIngress>::missing_artifacts(self)
            .map(StrippedMessageId::Ingress);
        let idkg_dealings = PayloadAssembler::<SignedIDkgDealing>::missing_artifacts(self)
            .map(|(node_index, dealing_id)| StrippedMessageId::IDkgDealing(dealing_id, node_index));

        ingress_messages.chain(idkg_dealings).collect()
    }
