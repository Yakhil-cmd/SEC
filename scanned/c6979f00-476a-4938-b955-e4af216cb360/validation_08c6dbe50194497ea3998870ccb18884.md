### Title
Residual ICRC-2 Allowance Not Revoked After Treasury Manager Deposit — (`rs/sns/governance/src/extensions.rs`)

### Summary

In `execute_treasury_manager_deposit`, SNS Governance calls `approve_treasury_manager` to grant the treasury manager canister an ICRC-2 allowance over both the SNS token treasury subaccount and the ICP treasury subaccount, then calls `deposit` on the treasury manager. After `deposit` returns, no step revokes the remaining allowance. The allowance persists for up to one hour, giving the treasury manager canister a window to call `icrc2_transfer_from` again and drain additional tokens from the SNS treasury beyond what the governance proposal authorized.

### Finding Description

`execute_treasury_manager_deposit` in `rs/sns/governance/src/extensions.rs` follows this sequence:

1. Call `approve_treasury_manager(extension_canister_id, treasury_allocation_sns_e8s, treasury_allocation_icp_e8s)` — this issues `icrc2_approve` on both the SNS ledger and the ICP ledger, granting the treasury manager canister an allowance equal to the full governance-approved amounts, expiring one hour from now.
2. Call `deposit` on the treasury manager canister.
3. Return `Ok(())`. [1](#0-0) 

`approve_treasury_manager` sets `expected_allowance: None`, which means the ledger blindly overwrites any prior allowance. The comment acknowledges this prevents double-spending from a *previous* approval, but says nothing about the residual allowance that remains after the current deposit completes. [2](#0-1) 

The treasury manager canister is not required to consume the full approved amount during `deposit`. After `deposit` returns, whatever portion of `treasury_allocation_sns_e8s` and `treasury_allocation_icp_e8s` was not consumed remains as a live ICRC-2 allowance on both ledgers for up to one hour. The treasury manager canister can call `icrc2_transfer_from` on either ledger at any point within that window to pull additional tokens from the SNS treasury subaccounts.

The ICRC-2 `use_allowance` logic correctly decrements the allowance on each `transfer_from`, but only down to whatever was actually consumed — it does not zero out the allowance automatically after the deposit operation ends. [3](#0-2) 

A more severe variant: the treasury manager canister can return a successful response from `deposit` while having called `icrc2_transfer_from` for zero or a small amount, then drain the full approved allowance in a subsequent call within the one-hour window.

### Impact Explanation

The treasury manager canister can transfer tokens from the SNS treasury (both SNS tokens and ICP) beyond the amount authorized by the governance proposal, up to the full approved amount, during the one-hour residual allowance window. This is a **ledger conservation bug**: the SNS treasury loses more tokens than the governance-approved deposit amount without any additional governance action.

### Likelihood Explanation

The treasury manager is registered via a governance proposal, but once registered it is an independent canister. If the treasury manager canister has a bug, is later upgraded to a malicious version, or is controlled by an adversary who obtained a governance majority at registration time, the residual allowance provides a direct, no-additional-governance-required path to drain additional treasury funds. The one-hour window is long enough for an automated exploit. The entry path is a direct `icrc2_transfer_from` call from the treasury manager canister to the SNS or ICP ledger — no privileged access beyond the already-granted allowance is required.

### Recommendation

After the `deposit` call returns (whether successfully or with an error), SNS Governance should revoke the remaining allowance on both ledgers by calling `icrc2_approve` with `amount = 0` for the treasury manager canister. This mirrors the mitigation recommended in M-07: approve on-demand and reset to zero after each operation.

```rust
// After deposit call in execute_treasury_manager_deposit:
// Revoke residual SNS token allowance
governance.ledger.icrc2_approve(
    to, 0, None, fee, sns_treasury_subaccount, None
).await?;
// Revoke residual ICP allowance
governance.nns_ledger.icrc2_approve(
    to, 0, None, icp_fee, icp_treasury_subaccount, None
).await?;
```

### Proof of Concept

1. A governance proposal registers a treasury manager canister and executes a deposit of `X` SNS tokens and `Y` ICP.
2. `execute_treasury_manager_deposit` calls `approve_treasury_manager(canister, X, Y)` — both ledgers now have allowances of `X` and `Y` for the treasury manager, expiring in one hour.
3. `deposit` is called on the treasury manager. The treasury manager calls `icrc2_transfer_from` for only `X/2` SNS tokens and `Y/2` ICP, then returns success.
4. `execute_treasury_manager_deposit` returns `Ok(())`. No allowance revocation occurs.
5. Within the next hour, the treasury manager calls `icrc2_transfer_from` on the SNS ledger for the remaining `X/2` SNS tokens and on the ICP ledger for the remaining `Y/2` ICP — both succeed because the allowances are still live.
6. The SNS treasury has lost `X` SNS tokens and `Y` ICP, double the governance-authorized deposit amount. [4](#0-3) [2](#0-1)

    async fn approve_treasury_manager(
        &self,
        treasury_manager_canister_id: CanisterId,
        sns_amount_e8s: u64,
        icp_amount_e8s: u64,
    ) -> Result<(), GovernanceError> {
        let to = Account {
            owner: treasury_manager_canister_id.get().0,
            subaccount: None,
        };

        let expiry_time_sec = self.env.now().saturating_add(ONE_HOUR_SECONDS);
        let expiry_time_nsec = expiry_time_sec.saturating_mul(NANO_SECONDS_PER_SECOND);

        // If expected_allowance is None, the ledger *blindly* overwrites any existing
        // allowance (even if non-zero). Therefore, there is no risk of double spending.

        self.ledger
            .icrc2_approve(
                to,
                sns_amount_e8s,
                Some(expiry_time_nsec),
                self.transaction_fee_e8s_or_panic(),
                self.sns_treasury_subaccount(),
                None,
            )
            .await
            .map(|_| ())
            .map_err(|e| {
                GovernanceError::new_with_message(
                    ErrorType::External,
                    format!("Error making SNS Token treasury transfer: {e}"),
                )
            })?;

        self.nns_ledger
            .icrc2_approve(
                to,
                icp_amount_e8s,
                Some(expiry_time_nsec),

/// Execute a treasury manager deposit operation
async fn execute_treasury_manager_deposit(
    governance: &Governance,
    extension_canister_id: CanisterId,
    arg: ValidatedDepositOperationArg,
) -> Result<(), GovernanceError> {
    let ValidatedDepositOperationArg {
        treasury_allocation_sns_e8s,
        treasury_allocation_icp_e8s,
        original,
    } = arg;

    let context = governance.treasury_manager_deposit_context().await?;
    let arg_blob =
        construct_treasury_manager_deposit_payload(context, original).map_err(|err| {
            GovernanceError::new_with_message(
                ErrorType::PreconditionFailed,
                format!("Failed to construct treasury manager deposit payload: {err}"),
            )
        })?;

    // 1. Transfer funds from treasury to treasury manager
    governance
        .approve_treasury_manager(
            extension_canister_id,
            treasury_allocation_sns_e8s,
            treasury_allocation_icp_e8s,
        )
        .await?;

    // 2. Call deposit on treasury manager
    let balances = governance
        .env
        .call_canister(extension_canister_id, "deposit", arg_blob)
        .await
        .map_err(|(code, err)| {
            GovernanceError::new_with_message(
                ErrorType::External,
                format!(
                    "Canister method call {extension_canister_id}.deposit failed with code {code:?}: {err}"
                ),
            )
        })
        .and_then(|blob| {
            Decode!(&blob, sns_treasury_manager::TreasuryManagerResult).map_err(|err| {
                GovernanceError::new_with_message(
                    ErrorType::External,
                    format!("Error decoding TreasuryManager.deposit response: {err:?}"),
                )
            })
        })?
        .map_err(|err| {
            GovernanceError::new_with_message(
                ErrorType::External,
                format!("TreasuryManager.deposit failed: {err:?}"),
            )
        })?;

    log!(
        INFO,
        "TreasuryManager.deposit succeeded with response: {:?}",
        balances
    );

    Ok(())

    /// Consumes amount from the spender's allowance for the account.
    /// Returns an error if the allowance would go negative.
    pub fn use_allowance(
        &mut self,
        account: &AD::AccountId,
        spender: &AD::AccountId,
        amount: AD::Tokens,
        now: TimeStamp,
    ) -> Result<AD::Tokens, InsufficientAllowance<AD::Tokens>> {
        self.with_postconditions_check(|table| {
            let key = (account.clone(), spender.clone());

            match table.allowances_data.get_allowance(&key) {
                None => Err(InsufficientAllowance(AD::Tokens::zero())),
                Some(old_allowance) => {
                    if old_allowance.expires_at.unwrap_or_else(remote_future) <= now {
                        Err(InsufficientAllowance(AD::Tokens::zero()))
                    } else {
                        if old_allowance.amount < amount {
                            return Err(InsufficientAllowance(old_allowance.amount));
                        }
                        let mut new_allowance = old_allowance.clone();
                        new_allowance.amount = old_allowance
                            .amount
                            .checked_sub(&amount)
                            .expect("Underflow when using allowance");
                        let rest = new_allowance.amount.clone();
                        if rest.is_zero() {
                            if let Some(expires_at) = old_allowance.expires_at {
                                table.allowances_data.remove_expiry(expires_at, key.clone());
                            }
                            table.allowances_data.remove_allowance(&key);
                        } else {
                            table.allowances_data.set_allowance(key, new_allowance);
                        }
                        Ok(rest)
                    }
                }
            }
        })
    }
