### Title
`already_tried_to_auto_finalize` Set Irrevocably Before Finalization Completes, Permanently Blocking Auto-Finalization Retry on Partial Failure - (`File: rs/sns/swap/src/swap.rs`)

### Summary
In the SNS Swap canister, `try_auto_finalize` sets `self.already_tried_to_auto_finalize = Some(true)` **before** calling `self.finalize(...)`. If `finalize` fails partway through (e.g., ICP sweep fails, Neurons' Fund settlement fails, SNS token sweep fails), the flag remains `true` and the auto-finalization path is permanently blocked. The swap is left in a partially-finalized state with no automatic recovery path, analogous to the original report's "state not cleared after a failed claim."

### Finding Description

In `rs/sns/swap/src/swap.rs`, `try_auto_finalize` unconditionally sets the "already tried" flag before attempting finalization:

```rust
self.already_tried_to_auto_finalize = Some(true);  // set BEFORE finalize
let auto_finalize_swap_response = self.finalize(now_fn, environment).await;
```

The `finalize` function itself is a multi-step pipeline (`sweep_icp` → `settle_neurons_fund_participation` → `create_sns_neuron_recipes` → `sweep_sns` → `claim_swap_neurons` → `set_mode`). Any step failure causes `finalize` to return early with an `error_message` set, but the `already_tried_to_auto_finalize` flag is **not reset** on failure.

Subsequently, `can_auto_finalize` checks this flag and permanently rejects further auto-finalization attempts:

```rust
if self.already_tried_to_auto_finalize.unwrap_or(true) {
    return Err(format!(
        "self.already_tried_to_auto_finalize is {:?}, indicating that an attempt has already been made..."
    ));
}
```

The `run_periodic_tasks` function, which drives the heartbeat-based auto-finalization, also gates on `can_auto_finalize()`. Once the flag is set, periodic tasks will never retry finalization automatically.

Additionally, `requires_periodic_tasks` returns `false` once the lifecycle is terminal AND `already_tried_to_auto_finalize` is `true`, causing the timer to be disabled entirely — even if finalization was only partially completed.

### Impact Explanation

When auto-finalization partially fails (e.g., ICP ledger temporarily unavailable during `sweep_icp`, or NNS governance temporarily unavailable during `settle_neurons_fund_participation`):

1. `already_tried_to_auto_finalize` is permanently `true`.
2. The periodic task timer is disabled (`requires_periodic_tasks` returns `false`).
3. Buyers whose ICP was not swept remain with funds locked in the Swap canister subaccounts.
4. Neurons' Fund maturity reserved for this SNS remains unreturned to NNS neurons.
5. SNS tokens remain undistributed.

Manual intervention via the public `finalize_swap` endpoint is still possible, but the automatic recovery path is permanently broken. For swaps where the SNS team or users are not actively monitoring, this results in permanently stuck funds. The `error_refund_icp` path is also blocked until finalization completes, since it requires the swap to be in `Aborted` or `Committed` state with transfers already done.

### Likelihood Explanation

The SNS Swap canister makes multiple cross-canister calls during finalization (ICP ledger, NNS governance, SNS ledger, SNS governance, SNS root). Any transient failure in any of these — which is realistic given IC's asynchronous messaging model and the possibility of canister unavailability — triggers this condition. The ICP ledger or NNS governance being temporarily busy or returning an error is a realistic scenario on mainnet.

### Recommendation

Reset `already_tried_to_auto_finalize` back to `Some(false)` if `finalize` returns a response with `error_message` set, so that the periodic task can retry. Alternatively, set the flag only after a successful finalization. The log message already acknowledges this: *"Will not automatically attempt again even if this fails"* — this is the design flaw.

    pub async fn finalize(
        &mut self,
        now_fn: fn(bool) -> u64,
        environment: &mut impl CanisterEnvironment,
    ) -> FinalizeSwapResponse {
        // Acquire the lock or return a FinalizeSwapResponse with an error message.
        if let Err(error_message) = self.lock_finalize_swap() {
            return FinalizeSwapResponse::with_error(error_message);
        }

        // The lock is now acquired and asynchronous calls to finalize are blocked.
        // Perform all subactions.
        let finalize_swap_response = self.finalize_inner(now_fn, environment).await;

        if finalize_swap_response.has_error_message() {
            log!(
                ERROR,
                "The swap did not finalize successfully. \n\
                finalize_swap_response: {finalize_swap_response:?}"
            );
        } else {
            log!(
                INFO,
                "The swap finalized successfully. \n\
                finalize_swap_response: {finalize_swap_response:?}"
            );
        }

        // Release the lock. Note, if there is a panic, the lock will
        // not be released. In that case, the Swap canister will need
        // to be upgraded to release the lock.
        self.unlock_finalize_swap();

        finalize_swap_response
    }
