### Title
Unconditional TLS Certificate Verification Bypass in `HttpClient` Used by Registry Replicator - (File: `rs/canister_client/src/http_client.rs`)

---

### Summary

The `HttpClient` in `rs/canister_client/src/http_client.rs` unconditionally accepts any TLS server certificate via the `DangerAcceptInvalidCerts` verifier and additionally disables SNI. This client is used in production by `RegistryCanister` (via `Agent::new`) to poll NNS nodes for registry updates. A network-level attacker positioned on the path between a node and NNS nodes can impersonate an NNS node, intercept the TLS session, and stall or block registry update delivery, causing a Denial of Service against the registry replication mechanism.

---

### Finding Description

`HttpClient::new_with_config` constructs a Rustls `ClientConfig` using the `.dangerous()` builder and installs `DangerAcceptInvalidCerts` as the `ServerCertVerifier`. This verifier unconditionally returns `Ok(ServerCertVerified::assertion())` for every certificate presented by a server, without checking identity, chain of trust, or validity period. SNI is also explicitly disabled (`rustls_config.enable_sni = false`). [1](#0-0) [2](#0-1) 

This `HttpClient` is the default client created by `Agent::new`, which is used by `RegistryCanister::new_with_agent_transformer` to construct agents for each NNS URL: [3](#0-2) 

`RegistryCanister` is instantiated by `RegistryReplicator::initialize_local_store` and by `InternalState::poll` during ongoing operation, both of which are production code paths executed by every IC node: [4](#0-3) 

The codebase itself acknowledges this risk in the test file: [5](#0-4) 

---

### Impact Explanation

An attacker who can position themselves on the network path between a node and NNS nodes (e.g., via a compromised router or ISP-level interception) can:

1. **Impersonate an NNS node**: Present any self-signed certificate; it will be accepted unconditionally.
2. **Intercept or drop registry update responses**: The registry replicator will receive no valid updates, causing the node's local registry store to become stale.
3. **Stall initial bootstrap**: During `initialize_local_store`, the node loops indefinitely until it receives valid registry data. A MITM can prevent this, permanently blocking the node from joining the network.
4. **Block ongoing registry updates**: Nodes that cannot receive registry updates cannot learn about subnet membership changes, replica version upgrades, or key rotations.

**Mitigating factor**: The *content* of registry responses is cryptographically verified against the NNS threshold public key, so the attacker cannot inject false registry data. The impact is therefore limited to availability (DoS) rather than integrity.

---

### Likelihood Explanation

The attack requires network-level access on the path between a node and NNS nodes. IC nodes communicate over the public internet using known IPv6 addresses, making them reachable from transit networks. The absence of certificate verification and SNI makes interception straightforward once network access is obtained. The risk is elevated compared to systems that use proper certificate pinning or registry-based TLS verification (as used in the P2P and inter-node layers via `TlsVerifier`). [6](#0-5) 

---

### Recommendation

- **Short term**: Replace `DangerAcceptInvalidCerts` in `HttpClient::new_with_config` with a verifier that pins the expected NNS node certificates (available from the registry) or uses standard CA-based verification. Re-enable SNI.
- **Long term**: Audit all uses of `Agent::new` and `HttpClient::new` in production code paths to ensure they do not rely on the insecure default. Consider enforcing registry-based TLS certificate verification (analogous to `TlsVerifier` used in the boundary node) for all node-to-node and node-to-NNS communication.

---

### Proof of Concept

1. A node's `RegistryReplicator` calls `RegistryCanister::new(nns_urls)`, which creates `Agent::new(url, Sender::Anonymous)` → `HttpClient::new()` → `DangerAcceptInvalidCerts`.
2. An attacker intercepts the TCP connection to an NNS node IP and presents a self-signed certificate with an arbitrary CN.
3. `DangerAcceptInvalidCerts::verify_server_cert` returns `Ok(ServerCertVerified::assertion())` unconditionally.
4. The attacker now controls the TLS session and can drop all registry responses.
5. `initialize_local_store` loops indefinitely; the node never bootstraps. Or, during `start_polling`, the node's registry version freezes, preventing it from receiving upgrade instructions or subnet membership changes. [7](#0-6) [8](#0-7)

    // Initializes the local registry store by repeatedly polling the registry canister
    // until no more changes are available.
    async fn initialize_local_store(
        logger: &ReplicaLogger,
        local_store: Arc<dyn LocalStore>,
        nns_urls: Vec<Url>,
        nns_pub_key: Option<ThresholdSigPublicKey>,
    ) {
        // If the local registry store is not empty, exit.
        if !local_store
            .get_changelog_since_version(ZERO_REGISTRY_VERSION)
            .expect("Could not read registry local store.")
            .is_empty()
        {
            info!(
                logger,
                "Local registry store is not empty, skipping initialization."
            );
            return;
        }

        let nns_pub_key =
            nns_pub_key.expect("Registry Local Store is empty and no NNS Public Key is provided.");
        let mut registry_version = ZERO_REGISTRY_VERSION;
        let mut timeout = 1;

        let registry_canister = RegistryCanister::new(nns_urls);

        // Fill the local registry store by polling the registry canister until we get no
        // more changes.
        loop {
            match write_certified_changes_to_local_store(
                &registry_canister,
                &nns_pub_key,
                local_store.as_ref(),
                registry_version,
            )
            .await
            {
                Ok(SuccessfulPoll {
                    last_stored_version,
                    last_available_version,
                    certified_time: _,
                }) => {
                    if last_stored_version == registry_version {
                        // The last stored version is the same as the requested version, which
                        // means we fetched the latest version.
                        break;
                    }

                    info!(
                        logger,
                        "Stored registry versions up to: {}/{}",
                        last_stored_version,
                        last_available_version
                    );
                    registry_version = last_stored_version;
                    timeout = 1;
                }
                Err(e) => {
                    warn!(
                        logger,
                        "Couldn't fetch registry updates (retry in {}s): {:?}", timeout, e
                    );
                    tokio::time::sleep(Duration::from_secs(timeout)).await;
                    timeout *= 2;
                    timeout = timeout.min(60); // limit the timeout by a minute max
                }
            }
        }

        info!(
            logger,
            "Finished local store initialization at registry version: {}", registry_version,
        );
    }
