### Title
Stale Maturity Modulation Applied to Disbursements Without Freshness Check - (`rs/nns/governance/src/governance/disburse_maturity.rs`)

### Summary

NNS Governance finalizes maturity disbursements using a cached `maturity_modulation.current_value_permyriad` that is derived from XRC price data, but never checks how old that value is before applying it. The `updated_at_days_since_epoch` field exists in the struct but is not consulted at disbursement time. If the Exchange Rate Canister (XRC) is unavailable for an extended period, the frozen modulation value — which can be up to ±5% — is silently applied to all pending disbursements, causing over- or under-minting of ICP relative to current market conditions.

### Finding Description

The NNS Governance canister runs a daily timer task (`UpdateIcpXdrRateRelatedData`) that fetches ICP/XDR rates from the XRC canister and stores them in `icp_price_history`. From this history it computes `maturity_modulation.current_value_permyriad` and stores it in `HeapGovernanceData`. [1](#0-0) 

When XRC calls fail, `update_maturity_modulation` explicitly preserves the prior value unchanged and does not update `updated_at_days_since_epoch`: [2](#0-1) 

When a maturity disbursement is finalized (triggered by the recurring timer task), `try_finalize_maturity_disbursement` reads `current_value_permyriad` directly with no check on `updated_at_days_since_epoch`: [3](#0-2) 

The value is then passed directly to `next_maturity_disbursement_to_finalize` and applied via `apply_maturity_modulation`: [4](#0-3) 

The same pattern exists in SNS Governance, where `effective_maturity_modulation_basis_points()` reads `current_basis_points` without checking `updated_at_timestamp_seconds`: [5](#0-4) 

And applies it unconditionally in `maybe_finalize_disburse_maturity`: [6](#0-5) 

### Impact Explanation

If the XRC canister is unavailable for an extended period (days to weeks), the maturity modulation value becomes stale. All pending disbursements that mature during this window are finalized using the frozen modulation. The modulation range is bounded to `[-500, +500]` permyriad (±5%): [7](#0-6) 

A 5% error on large neuron disbursements represents a real ICP minting discrepancy. If ICP price has moved significantly while XRC was offline, the governance canister either over-mints (positive stale modulation during a price drop) or under-mints (negative stale modulation during a price rise) ICP to neuron holders. This is a ledger conservation issue: the minted amount does not reflect the current market reality that the modulation mechanism is designed to track.

### Likelihood Explanation

The XRC canister is a system canister that itself relies on HTTPS outcalls to external exchanges. It can be temporarily unavailable due to subnet upgrades, rate limiting, or exchange API failures. The NNS Governance timer retries failed fetches but explicitly preserves the stale value indefinitely: [8](#0-7) 

Any neuron holder with a pending disbursement (initiated via the standard `DisburseMaturity` call) is affected automatically when the disbursement timer fires. No special privilege is required; the finalization is triggered by the governance canister's own recurring task.

### Recommendation

Before applying `current_value_permyriad` in `try_finalize_maturity_disbursement`, check `updated_at_days_since_epoch` against `current_day`. If the modulation is older than a defined threshold (e.g., 2–3 days), either:
1. Block finalization and retry later (analogous to the GMX recommendation of not executing orders while the sequencer is offline), or
2. Fall back to a neutral modulation of 0 permyriad.

The same fix should be applied to SNS Governance's `effective_maturity_modulation_basis_points()`, which should check `updated_at_timestamp_seconds` against `now` before returning `current_basis_points`.

### Proof of Concept

1. NNS Governance has a pending maturity disbursement ready to finalize (7-day delay elapsed).
2. XRC canister becomes unavailable; `fetch_and_validate_rate` returns `None` for several consecutive days.
3. `update_maturity_modulation` is skipped each day; `current_value_permyriad` remains frozen at, say, `+400` permyriad (ICP was high when XRC last responded).
4. ICP price drops 20% during the outage.
5. The disbursement finalization timer fires; `try_finalize_maturity_disbursement` reads `current_value_permyriad = Some(400)` with no staleness check.
6. `apply_maturity_modulation` mints `original_maturity * 1.04` ICP — 4% more than the current market-aligned modulation would produce — directly to the neuron holder's account via the ICP ledger. [9](#0-8) [10](#0-9)

    fn effective_maturity_modulation_basis_points(&self) -> Result<i32, GovernanceError> {
        let maturity_modulation_disabled = self
            .parameters
            .as_ref()
            .map(|nervous_system_parameters| {
                nervous_system_parameters
                    .maturity_modulation_disabled
                    .unwrap_or_default()
            })
            .unwrap_or_default();

        if maturity_modulation_disabled {
            return Ok(0);
        }

        self.maturity_modulation
            .as_ref()
            .and_then(|maturity_modulation| maturity_modulation.current_basis_points)
            .ok_or_else(|| {
                GovernanceError::new_with_message(
                    ErrorType::Unavailable,
                    "Maturity modulation not known. Retrying later might work. \
                     If this persists, there is probably a problem with retrieving \
                     the maturity modulation value from the Cycles Minting Canister.",
                )
            })
    }
