### Title
Single Authorized Caller Has Unilateral Control Over CloudEngine Subnet Lifecycle Without Governance Approval - (File: rs/engine_controller/canister/canister.rs)

### Summary
The `engine_controller` canister gates all critical subnet-lifecycle operations — `create_engine`, `delete_engine`, `update_subnet`, and `deploy_guestos_to_all_subnet_nodes` — behind a single hardcoded principal (`DEFAULT_AUTHORIZED_CALLER`). This single principal can create and delete CloudEngine subnets, deploy GuestOS to all subnet nodes, and halt/unhalt subnets without any NNS governance proposal or multisig approval. This is a direct analog to the TangleswapFactory centralization risk: a single owner-equivalent entity can unilaterally change critical protocol state.

### Finding Description

The engine controller canister (`rs/engine_controller/canister/canister.rs`) hard-codes a single authorized principal:

```rust
const DEFAULT_AUTHORIZED_CALLER: &str =
    "bct5z-vccu4-6q4t2-3lb6l-wm43p-ulppt-o5sqq-w6het-rthdz-qp4yn-fqe";
``` [1](#0-0) 

All update methods check only this single principal:

```rust
fn ensure_authorized() -> Result<Principal, String> {
    let caller = msg_caller();
    let expected = AUTHORIZED_CALLER.with(|c| *c.borrow());
    if caller != expected {
        return Err(format!("Caller {caller} is not authorized to call this canister"));
    }
    Ok(caller)
}
``` [2](#0-1) 

This single principal can invoke:

1. **`create_engine`** — calls `registry.create_subnet`, adding a new subnet to the IC routing table and subnet list.
2. **`delete_engine`** — calls `registry.delete_subnet`, removing a CloudEngine subnet from the IC.
3. **`update_subnet`** — calls `registry.update_subnet` to halt/unhalt a subnet or replace its admin list.
4. **`deploy_guestos_to_all_subnet_nodes`** — calls `registry.deploy_guestos_to_all_subnet_nodes`, changing the replica software version running on all nodes of a CloudEngine subnet. [3](#0-2) 

The registry canister accepts these calls from the engine controller as a co-equal to NNS governance:

```rust
fn check_caller_is_governance_or_engine_controller_and_log(method_name: &str) {
    let caller = dfn_core::api::caller();
    assert!(
        caller == GOVERNANCE_CANISTER_ID.into() || caller == ENGINE_CONTROLLER_CANISTER_ID.into(),
        ...
    );
}
``` [4](#0-3) 

The `create_subnet`, `delete_subnet`, `update_subnet`, and `deploy_guestos_to_all_subnet_nodes` registry endpoints are all reachable via the engine controller without an NNS proposal: [5](#0-4) [6](#0-5) 

The `AUTHORIZED_CALLER` is re-evaluated on every `post_upgrade`, meaning whoever controls the canister upgrade can silently rotate the authorized principal: [7](#0-6) 

### Impact Explanation

The authorized principal can, without any NNS governance vote:

- **Create CloudEngine subnets**: New subnets are added to the IC routing table and subnet list, affecting canister routing across the entire network.
- **Delete CloudEngine subnets**: Subnets and all canisters hosted on them are removed from the IC.
- **Deploy arbitrary GuestOS versions**: The replica software running on all nodes of a CloudEngine subnet can be changed to any blessed version, bypassing the normal NNS proposal process.
- **Halt subnets**: CloudEngine subnets can be halted (stopping all message processing) or unhalted unilaterally.
- **Replace subnet admin lists**: The `subnet_admins` list of a CloudEngine subnet can be replaced entirely, locking out legitimate admins. [8](#0-7) 

The `normalize_subnet_admins` function also ensures the authorized caller is always injected into the subnet admin list, permanently cementing their control over every CloudEngine subnet: [9](#0-8) 

### Likelihood Explanation

The authorized caller is a single self-authenticating principal. If its private key is compromised, or if the entity controlling it acts maliciously or negligently, all CloudEngine subnet lifecycle operations can be performed without any governance check or time-lock. The design explicitly documents this as a single-principal gate with no multisig requirement:

> "Only a single, hard-coded authorized principal may invoke its methods." [10](#0-9) 

Additionally, the `post_upgrade` hook re-reads the authorized caller from the upgrade argument, meaning a canister upgrade with `None` silently resets to the hardcoded default — a behavior confirmed by the test suite: [11](#0-10) 

### Recommendation

1. **Require NNS governance proposals** for all subnet lifecycle operations (create, delete, deploy GuestOS) performed via the engine controller, consistent with how all other subnet mutations are gated.
2. **If a fast-path is required**, replace the single authorized principal with a multisig wallet or a threshold-signature scheme, analogous to the TangleswapFactory resolution.
3. **Remove the `normalize_subnet_admins` self-injection** or gate it behind governance, as it permanently embeds the authorized caller into every CloudEngine subnet's admin list without user consent.
4. **Persist the authorized caller in stable memory** rather than re-deriving it from upgrade arguments on every `post_upgrade`, to prevent silent resets to the hardcoded default.

### Proof of Concept

1. The principal `bct5z-vccu4-6q4t2-3lb6l-wm43p-ulppt-o5sqq-w6het-rthdz-qp4yn-fqe` sends an ingress update call to the engine controller canister (`si2b5-pyaaa-aaaaa-aaaja-cai`) invoking `create_engine` with 4 node IDs and a blessed replica version.
2. `ensure_authorized()` passes because the caller matches `AUTHORIZED_CALLER`.
3. The engine controller calls `registry.create_subnet` with `subnet_type: CloudEngine`, adding a new subnet to the IC routing table — no NNS proposal, no voting period, no time-lock.
4. The same principal calls `deploy_guestos_to_all_subnet_nodes` targeting the new subnet, deploying a different GuestOS version to all its nodes without any governance approval.
5. The same principal calls `update_subnet` with `is_halted: Some(true)`, halting the subnet and stopping all canister message processing on it — again with no governance check. [12](#0-11) [13](#0-12)

#[update]
async fn create_engine(args: CreateEngineArgs) -> Result<NewSubnet, String> {
    let caller = ensure_authorized()?;

    // Validate node list.
    if args.node_ids.len() < REQUIRED_NODE_COUNT {
        return Err(format!(
            "Expected at least {REQUIRED_NODE_COUNT} node ids, got {}",
            args.node_ids.len()
        ));
    }
    let mut seen: HashSet<Principal> = HashSet::new();
    for n in &args.node_ids {
        if !seen.insert(*n) {
            return Err(format!("Duplicate node id supplied: {n}"));
        }
    }

    // Make sure the caller is part of the subnet admins.
    let mut subnet_admins: Vec<PrincipalId> =
        args.subnet_admins.into_iter().map(PrincipalId).collect();
    let caller_pid = PrincipalId(caller);
    if !subnet_admins.contains(&caller_pid) {
        subnet_admins.push(caller_pid);
    }

    let node_ids: Vec<NodeId> = args
        .node_ids
        .into_iter()
        .map(|p| NodeId::from(PrincipalId(p)))
        .collect();

    let initial_dkg_subnet_id = INITIAL_DKG_SUBNET_ID.with(|c| *c.borrow());

    let payload = CreateSubnetPayload {
        node_ids,
        subnet_admins: Some(subnet_admins),
        replica_version_id: args.replica_version_id,
        subnet_type: SubnetType::CloudEngine,
        initial_dkg_subnet_id: Some(initial_dkg_subnet_id),
        dkg_interval_length: 499,
        dkg_dealings_per_block: 1,
        initial_notary_delay_millis: 300,
        max_block_payload_size: 4 * 1024 * 1024, // 4 MiB
        max_ingress_bytes_per_message: 2 * 1024 * 1024, // 2 MiB
        max_ingress_messages_per_block: 1000,
        unit_delay_millis: 1000,
        canister_cycles_cost_schedule: Some(CanisterCyclesCostSchedule::Free),
        features: SubnetFeatures {
            canister_sandboxing: false,
            http_requests: true,
            sev_enabled: Some(false),
        },
        ..Default::default()
    };

    let response: Result<NewSubnet, String> =
        Call::unbounded_wait(REGISTRY_CANISTER_ID.into(), "create_subnet")
            .with_arg(payload)
            .await
            .map_err(|e| format!("registry.create_subnet call failed: {e:?}"))?
            .candid()
            .map_err(|e| format!("Failed to decode registry response: {e}"))?;

    response
}
