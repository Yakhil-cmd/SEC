### Title
Predictable Genesis/Recovery Random Beacon Enables Block Maker Rank Pre-computation at Height 1 - (File: `rs/consensus/utils/src/membership.rs`)

---

### Summary

The IC consensus protocol uses the **previous** height's `RandomBeacon` to determine block maker rankings and notarization committee membership for the current height. The genesis (and recovery) `CatchUpPackage` random beacon is constructed with a fully deterministic, publicly known empty signature (`CombinedThresholdSig(vec![])`), making the block maker ranking and committee membership at height 1 (and at the first height after any subnet recovery) completely predictable before the subnet produces a single block. This is an acknowledged design trade-off in the code, but it creates a window where a node operator can pre-compute their role and selectively withhold participation or prepare a crafted block.

---

### Finding Description

`get_shuffled_nodes()` in `rs/consensus/utils/src/membership.rs` derives all block maker rankings and committee memberships by seeding a CSPRNG from the **previous** height's `RandomBeacon`:

```rust
let mut rng = Csprng::from_random_beacon_and_purpose(previous_beacon, purpose);
node_ids.shuffle(&mut rng);
```

The code itself acknowledges the consequence:

> *"One consequence of this design, is that the randomness derived from the genesis random beacon at height 1 is predictable, because the beacon from the genesis height is preconstructed and known in advance."*

The genesis beacon (height 0) is constructed in `make_registry_cup_from_cup_contents()` with a zero-byte signature and zero-byte parent hash:

```rust
let random_beacon = Signed {
    content: RandomBeaconContent {
        version: replica_version,
        height: cup_height,
        parent: Id::from(CryptoHash(Vec::new())),   // empty
    },
    signature: ThresholdSignature {
        signer: low_dkg_id,
        signature: CombinedThresholdSigOf::new(CombinedThresholdSig(vec![])), // empty
    },
};
```

Because `Csprng::from_random_beacon_and_purpose` hashes the beacon deterministically, and the beacon's content and signature are all-zeros/empty, the CSPRNG seed for height-1 block maker ranking and committee sampling is fully computable by any external observer before the subnet starts.

The same construction is used for recovery CUPs, so the predictability window recurs after every subnet recovery event.

---

### Impact Explanation

Any party who knows the subnet's node IDs (publicly available in the registry) can:

1. **Pre-compute the rank-0 block maker at height 1** (and after recovery). The rank-0 node knows in advance it will be the preferred block proposer and can prepare a carefully crafted block payload before the subnet starts, enabling front-running of the first batch of messages.
2. **Pre-compute the notarization committee at height 1**. An adversary can target those specific nodes with network-level interference (e.g., flooding) to delay notarization of the first block, degrading liveness at subnet launch or after recovery.
3. **Selectively withhold a block proposal**. A rank-0 node that finds its pre-computed role undesirable (e.g., it would be blamed for a slow start) can simply not propose, forcing the protocol to fall back to rank-1, rank-2, etc., delaying the first finalized block.

The impact is bounded to a single height window per genesis/recovery event, but subnet launches and recoveries are high-value moments where liveness and fairness matter most.

---

### Likelihood Explanation

- The node IDs of a subnet are publicly readable from the registry before the subnet starts.
- The genesis beacon bytes are deterministic and derivable from the `CatchUpPackageContents` stored in the registry.
- No special access is required beyond being a node operator (a protocol peer below the consensus fault threshold) or an external observer who can read the registry.
- The computation is trivial: hash the genesis beacon, derive the CSPRNG seed, shuffle the sorted node list.

---

### Recommendation

1. **Incorporate unpredictable entropy into the genesis/recovery beacon.** For example, include a hash of the DKG transcript's combined public key or a VRF output from the NNS into the genesis beacon's `parent` field, making it non-zero and subnet-specific in a way that cannot be predicted before the DKG completes.
2. **Alternatively**, accept the current design trade-off but document it explicitly as a known limitation in security-relevant documentation, and ensure the first DKG epoch is kept short so the predictability window closes quickly.

---

### Proof of Concept

1. Read the `CatchUpPackageContents` for a new subnet from the registry. The genesis beacon has `parent = CryptoHash(Vec::new())` and `signature = CombinedThresholdSig(vec![])`.
2. Construct the `RandomBeacon` struct as done in `make_registry_cup_from_cup_contents`.
3. Call `Csprng::from_random_beacon_and_purpose(&genesis_beacon, &RandomnessPurpose::BlockmakerRanking)`.
4. Sort the subnet's node IDs (from the registry) and shuffle with the CSPRNG.
5. The first `get_faults_tolerated(n) + 1` nodes in the shuffled list are the elected block makers at height 1, with the first being rank-0.

This is exactly the computation performed by `get_block_maker_rank()` at runtime, confirming that the rank-0 block maker at height 1 is fully predictable before the subnet produces any block.

```rust
    /// Return a shuffled list of the node IDs at a given height using the given
    /// previous beacon.
    // Here we assert that the given random beacon is from the previous height.
    // Note, if we'd wait until the random beacon is available for the current
    // height, we'd sequentialize the making of the random beacon and block
    // proposals. One consequence of this design, is that the randomness
    // derived from the genesis random beacon at height 1 is predictable,
    // because the beacon from the genesis height is preconstructed and known in
    // advance
    pub fn get_shuffled_nodes(
        &self,
        height: Height,
        previous_beacon: &RandomBeacon,
        purpose: &RandomnessPurpose,
    ) -> Result<Vec<NodeId>, MembershipError> {
        assert_eq!(height, previous_beacon.height().increment());
        let mut node_ids = self.get_nodes(height)?;
        // To achieve a deterministic shuffling, we sort the ids first, to not rely on
        // any ordering by the registry. We assume all node_ids are unique, so
        // `sort_unstable` is effectively the same as `sort` but slightly more
        // efficient.
        node_ids.sort_unstable();
        let mut rng = Csprng::from_random_beacon_and_purpose(previous_beacon, purpose);
        node_ids.shuffle(&mut rng);
        Ok(node_ids)
    }
```

```rust
    fn get_block_maker_rank_from_shuffled_nodes(
        node_id: &NodeId,
        shuffled_nodes: &[NodeId],
    ) -> Result<Option<Rank>, MembershipError> {
        let index = match shuffled_nodes.iter().position(|id| id == node_id) {
            Some(index) => index,
            None => return Err(MembershipError::NodeNotFound(*node_id)),
        };

        // We only elect f+1 nodes as block makers, which is the minimum amount that
        // still guarantees at least one honest block maker is elected.
        if index <= get_faults_tolerated(shuffled_nodes.len()) {
            Ok(Some(Rank(index as u64)))
        } else {
            Ok(None)
        }
    }
```

```rust
impl Csprng {
    /// Creates a CSPRNG from the given random beacon for the given purpose.
    pub fn from_random_beacon_and_purpose(
        random_beacon: &RandomBeacon,
        purpose: &RandomnessPurpose,
    ) -> Self {
        let randomness = randomness_from_crypto_hashable(random_beacon);
        Self::from_randomness_and_purpose(&randomness, purpose)
    }

    /// Creates a CSPRNG from the Randomness value for the given purpose.
    pub fn from_randomness_and_purpose(seed: &Randomness, purpose: &RandomnessPurpose) -> Self {
        let seed = Seed::from_bytes(&seed.get());
        let seed_for_purpose = seed.derive(&purpose.domain_separator());
        Csprng::from_seed(seed_for_purpose)
    }

    /// Creates a CSPRNG from the given seed.
    fn from_seed(seed: ic_crypto_internal_seed::Seed) -> Self {
        Csprng {
            rng: seed.into_rng(),
        }
    }
}
```
