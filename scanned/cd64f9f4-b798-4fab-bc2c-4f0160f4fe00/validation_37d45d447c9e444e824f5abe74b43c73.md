### Title
Hardcoded 5% Worst-Case Modulation in `spawn_neuron` Understates True -10% Mission 70 Floor, Allowing Sub-Minimum-Stake Neuron Creation - (File: rs/nns/governance/src/governance.rs)

---

### Summary

`spawn_neuron` in NNS Governance validates that a spawned neuron will meet the minimum stake requirement by assuming the worst-case maturity modulation is **-5%** (500 basis points). However, the Mission 70 maturity modulation system now allows modulation as low as **-10%** (1,000 basis points). Any neuron controller can call `manage_neuron::Spawn` with maturity in the gap between these two thresholds, creating a child neuron that, when finalized by `maybe_spawn_neurons`, receives less ICP than `neuron_minimum_stake_e8s` — violating the protocol's minimum-stake invariant.

---

### Finding Description

In `spawn_neuron`, the pre-flight check for minimum stake uses a hardcoded 5% worst-case:

```rust
// rs/nns/governance/src/governance.rs, line 2666
let least_possible_stake = (maturity_to_spawn as f64 * (1_f64 - 0.05)) as u64;

if least_possible_stake < economics.neuron_minimum_stake_e8s {
    return Err(GovernanceError::new_with_message(
        ErrorType::InsufficientFunds,
        "There isn't enough maturity to spawn a new neuron due to worst case maturity modulation.",
    ));
}
``` [1](#0-0) 

However, the Mission 70 maturity modulation system defines a minimum of **-10%** (-1,000 permyriad):

```rust
// rs/nns/governance/src/timer_tasks/update_icp_xdr_rate_related_data.rs
pub(crate) const MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70: i64 = -1_000;
pub(crate) const MATURITY_MODULATION_MAX_PERMYRIAD_MISSION_70: i64 = 200;
``` [2](#0-1) 

The valid range used in `maybe_spawn_neurons` to accept or reject the modulation before minting is:

```rust
const VALID_MATURITY_MODULATION_BASIS_POINTS_RANGE: RangeInclusive<i32> =
    MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 as i32
        ..=MATURITY_MODULATION_MAX_PERMYRIAD_MISSION_70 as i32;
``` [3](#0-2) 

When `maybe_spawn_neurons` runs (via timer), it reads the current `maturity_modulation.current_value_permyriad` and applies it to mint ICP:

```rust
let neuron_stake: u64 = match apply_maturity_modulation(
    original_maturity,
    maturity_modulation,
) { ... }
``` [4](#0-3) 

The `apply_maturity_modulation` function computes `amount * (1 + modulation / 10_000)`: [5](#0-4) 

**The gap:** If `maturity_to_spawn` satisfies `maturity_to_spawn × 0.95 ≥ neuron_minimum_stake_e8s` (passes the 5% check) but `maturity_to_spawn × 0.90 < neuron_minimum_stake_e8s` (fails the true 10% worst case), the child neuron is created in spawning state. When `maybe_spawn_neurons` later fires with modulation at -1,000 permyriad (-10%), the minted ICP falls below `neuron_minimum_stake_e8s`.

**Concrete example** (with `neuron_minimum_stake_e8s = 100_000_000` e8s = 1 ICP):
- Maturity of `105_300_000` e8s passes the 5% check: `105_300_000 × 0.95 = 100_035_000 ≥ 100_000_000` ✓
- At -10% modulation: `105_300_000 × 0.90 = 94_770_000 < 100_000_000` — sub-minimum stake neuron is minted.

---

### Impact Explanation

A neuron controller (unprivileged ingress sender) can deliberately or inadvertently create a spawned neuron whose finalized ICP stake is below `neuron_minimum_stake_e8s`. This violates the protocol's minimum-stake invariant that all governance neurons are expected to satisfy. Neurons with sub-minimum stake may behave incorrectly in downstream governance operations (voting power calculations, dissolve operations, etc.) that assume the invariant holds. The maturity is irrevocably consumed from the parent neuron at spawn time, so the user suffers a permanent loss relative to expectations.

---

### Likelihood Explanation

The Mission 70 maturity modulation range of -10% to +2% is already deployed. The modulation moves at most 30 permyriad per day (`MATURITY_MODULATION_DAILY_SPEED_LIMIT_PERMYRIAD`), so reaching -1,000 permyriad takes sustained ICP price decline over ~33 days — a realistic market scenario. Any neuron controller whose maturity falls in the 5%–10% gap relative to `neuron_minimum_stake_e8s` is affected without any special action required. [6](#0-5) 

---

### Recommendation

Replace the hardcoded `0.05` in `spawn_neuron` with the actual Mission 70 minimum modulation constant, expressed as a fraction:

```rust
// Use the true worst-case: MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 = -1_000 permyriad = -10%
let worst_case_fraction = 1_f64
    + (MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 as f64 / 10_000_f64);
let least_possible_stake = (maturity_to_spawn as f64 * worst_case_fraction) as u64;
```

Alternatively, use integer arithmetic via `apply_maturity_modulation` with `MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 as i32` to avoid floating-point imprecision, consistent with how `initiate_maturity_disbursement` in NNS governance already validates the worst-case for disbursements. [7](#0-6) 

---

### Proof of Concept

1. Obtain a neuron with `maturity_e8s_equivalent = M` where:
   - `M × 0.95 ≥ neuron_minimum_stake_e8s` (passes the 5% check)
   - `M × 0.90 < neuron_minimum_stake_e8s` (fails the true 10% worst case)
   - E.g., `neuron_minimum_stake_e8s = 100_000_000`, `M = 105_300_000`

2. Call `manage_neuron` with `Command::Spawn { percentage_to_spawn: Some(100), ... }` as the neuron controller (unprivileged ingress).

3. `spawn_neuron` passes the check at line 2666 (`105_300_000 × 0.95 = 100_035_000 ≥ 100_000_000`) and creates the child neuron in spawning state with `maturity_e8s_equivalent = 105_300_000`.

4. Wait for the ICP price to decline such that `maturity_modulation.current_value_permyriad` reaches -1,000 (within the accepted `VALID_MATURITY_MODULATION_BASIS_POINTS_RANGE`).

5. `maybe_spawn_neurons` fires via timer, reads modulation = -1,000, and mints `105_300_000 × (1 + (-1000/10000)) = 105_300_000 × 0.90 = 94_770_000` e8s — below `neuron_minimum_stake_e8s = 100_000_000`.

6. The child neuron now exists on-chain with a stake below the protocol minimum, violating the invariant. [8](#0-7) [4](#0-3)

```rust
pub fn apply_maturity_modulation(
    amount_maturity_e8s: u64,
    maturity_modulation_basis_points: i32,
) -> Result<u64, String> {
    let amount_e8s = u128::from(amount_maturity_e8s);

    let adjusted_maturity_modulation_basis_points = saturating_add_or_subtract_u128_i32(
        BASIS_POINTS_PER_UNITY,
        maturity_modulation_basis_points,
    );

    let modulated_amount_e8s: u128 = amount_e8s
        .checked_mul(adjusted_maturity_modulation_basis_points)
        .ok_or_else(|| "Underflow or overflow when calculating maturity modulation".to_string())?
        .checked_div(BASIS_POINTS_PER_UNITY)
        .ok_or_else(|| "Underflow or overflow when calculating maturity modulation".to_string())?;

    u64::try_from(modulated_amount_e8s).map_err(|err| err.to_string())
}
