### Title
Unbounded Iteration Over All Subnets in `do_remove_nodes_from_subnet` Causes Instruction-Limit Exhaustion on the NNS Registry Canister - (File: rs/registry/canister/src/mutations/do_remove_nodes_from_subnet.rs)

---

### Summary

`do_remove_nodes_from_subnet` in the NNS registry canister iterates over **every subnet** in the registry to find and remove the specified nodes. There is no upper bound on the number of nodes in the payload, and no upper bound on the number of subnets iterated. As the IC network grows (more subnets, more nodes per subnet), a governance-approved proposal carrying a large `node_ids` list can cause the registry canister's update call to exhaust the system-subnet instruction limit (50 billion instructions), causing the proposal execution to trap and permanently fail.

---

### Finding Description

`do_remove_nodes_from_subnet` iterates over the entire subnet list and, for each subnet, scans its membership list against every node ID in the payload:

```
get_subnet_ids_from_subnet_list(self.get_subnet_list_record())
    .into_iter()
    .map(|subnet_id| (subnet_id, self.get_subnet_or_panic(subnet_id)))
    .filter_map(|(subnet_id, mut subnet)| {
        let initial_len = subnet.membership.len();
        subnet.membership.retain(|node_id_vec| {
            !payload
                .node_ids
                .iter()
                .map(|node_to_remove| node_to_remove.get().to_vec())
                .any(|x| &x == node_id_vec)
        });
        ...
    })
    .collect();
``` [1](#0-0) 

The computational complexity is **O(S × M × N)** where S = number of subnets, M = average subnet membership size, N = number of node IDs in the payload. None of these three dimensions are bounded at the call site.

After collecting mutations, `maybe_apply_mutation_internal` calls `check_global_state_invariants`, which itself iterates over all nodes and all subnets again to run crypto-key uniqueness checks, endpoint checks, subnet membership checks, etc.: [2](#0-1) 

The crypto invariant check alone is documented to cost ~2M instructions per node just for parsing (full validation is ~400M per node and is explicitly skipped to avoid hitting the instruction limit): [3](#0-2) 

The system subnet instruction limit is 50 billion instructions (`50 * B`): [4](#0-3) 

The `RemoveNodesFromSubnetPayload` accepts an unbounded `Vec<NodeId>`: [5](#0-4) 

The canister entry point enforces only that the caller is the governance canister — no payload size validation occurs: [6](#0-5) 

---

### Impact Explanation

If a governance proposal is submitted with a `RemoveNodesFromSubnetPayload` containing a large number of node IDs (e.g., hundreds), the registry canister's update call will iterate over all subnets × all nodes in the payload × all subnet members, then run the full invariant suite over the entire registry snapshot. This can exhaust the 50B instruction limit on the NNS system subnet, causing the proposal execution to trap. The proposal is then permanently marked as failed, and the intended node removal does not occur. This is a **cycles/resource accounting bug** that causes **governance liveness failure**: legitimate NNS proposals cannot be executed, blocking network topology changes.

---

### Likelihood Explanation

The IC mainnet currently has hundreds of subnets and thousands of nodes. As the network grows, even a moderately sized `node_ids` list (e.g., 50–100 nodes) in a single `RemoveNodesFromSubnetPayload` proposal could trigger this. The governance canister does not validate payload size before submitting to the registry. Any NNS neuron with sufficient voting power can submit such a proposal. The condition is reachable without any privileged access beyond normal NNS governance participation.

---

### Recommendation

1. Add a hard upper bound on `payload.node_ids.len()` at the start of `do_remove_nodes_from_subnet`, rejecting payloads that exceed a safe threshold (e.g., 100 nodes).
2. Restructure the inner loop to avoid the O(S × M × N) cross-product: build a `HashSet` of node IDs to remove before iterating subnets, reducing the inner scan to O(1) per membership entry.
3. Consider splitting large removals across multiple proposals.

---

### Proof of Concept

**Attacker-controlled entry path:**
1. An NNS neuron (no special privilege beyond voting power) submits a `RemoveNodesFromSubnetPayload` with `node_ids` containing, e.g., 500 valid unassigned node IDs.
2. The proposal passes governance voting.
3. Governance calls `remove_nodes_from_subnet` on the registry canister.
4. `do_remove_nodes_from_subnet` iterates over all S subnets, and for each subnet iterates over all M members, checking against all 500 node IDs — O(S × M × 500) operations.
5. `maybe_apply_mutation_internal` then calls `check_global_state_invariants`, which runs `check_node_crypto_keys_exist_and_are_unique` (O(N_total_nodes) at ~2M instructions/node), `check_endpoint_invariants` (O(N_total_nodes)), `check_subnet_invariants` (O(S × M)), etc.
6. With ~1,000 nodes and ~50 subnets on mainnet, the combined instruction count exceeds 50B, the registry canister traps, and the proposal execution fails permanently. [7](#0-6) [8](#0-7) [9](#0-8) [4](#0-3)

    pub fn do_remove_nodes_from_subnet(&mut self, payload: RemoveNodesFromSubnetPayload) {
        println!("{LOG_PREFIX}do_remove_nodes_from_subnet started: {payload:?}");

        let mutations = get_subnet_ids_from_subnet_list(self.get_subnet_list_record())
            .into_iter()
            .map(|subnet_id| (subnet_id, self.get_subnet_or_panic(subnet_id)))
            .filter_map(|(subnet_id, mut subnet)| {
                let initial_len = subnet.membership.len();
                subnet.membership.retain(|node_id_vec| {
                    !payload
                        .node_ids
                        .iter()
                        .map(|node_to_remove| node_to_remove.get().to_vec())
                        .any(|x| &x == node_id_vec)
                });

                if initial_len != subnet.membership.len() {
                    Some(update(
                        make_subnet_record_key(subnet_id).as_bytes(),
                        subnet.encode_to_vec(),
                    ))
                } else {
                    None
                }
            })
            .collect();

        // Check invariants before applying mutations
        self.maybe_apply_mutation_internal(mutations);

        println!("{LOG_PREFIX}do_remove_nodes_from_subnet finished: {payload:?}");
    }

pub fn find_subnet_for_node(
    registry: &Registry,
    node_id: NodeId,
    subnet_list_record: &SubnetListRecord,

    subnet_list_record
        .subnets
        .iter()
