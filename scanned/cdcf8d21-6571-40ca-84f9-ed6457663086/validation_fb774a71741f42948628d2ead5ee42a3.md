### Title
Migration Canister Allowlist Bypass via Default `None` State — (`rs/migration_canister/src/canister_state.rs`)

### Summary

The `caller_allowed` function in the migration canister treats `ALLOWLIST = None` as "allow all callers." `ALLOWLIST` is a heap-allocated `thread_local` variable that resets to `None` on every canister upgrade. The `post_upgrade` hook re-applies the allowlist from upgrade arguments, but the `allowlist` field is `Option<Vec<Principal>>` and defaults to `null` in Candid. Any upgrade that omits or passes `null` for the allowlist silently resets the access control to the fully-open state, allowing any unprivileged caller to invoke `migrate_canister`.

### Finding Description

**Root cause — `caller_allowed` treats `None` as "allow all":** [1](#0-0) 

```rust
pub fn caller_allowed(id: &Principal) -> bool {
    ALLOWLIST.with_borrow(|allowlist| match allowlist {
        Some(allowlist) => allowlist.contains(id),
        None => true,   // ← default/unset state grants access to everyone
    })
}
```

`ALLOWLIST` is declared as a heap `thread_local` initialized to `None`: [2](#0-1) 

Heap memory is wiped on every canister upgrade before `post_upgrade` runs, so `ALLOWLIST` always reverts to `None` first. The `post_upgrade` hook then calls `set_allowlist(args.allowlist)`: [3](#0-2) 

The upgrade argument type is: [4](#0-3) 

In Candid, `opt vec principal` defaults to `null` when omitted. If the upgrade is performed without explicitly re-supplying the allowlist, `set_allowlist(None)` is called, leaving `ALLOWLIST = None`. From that point on, `caller_allowed` returns `true` for every caller.

**Enforcement site — the check that is bypassed:** [5](#0-4) 

```rust
let caller = msg_caller();
// For soft rollout purposes
if !caller_allowed(&caller) {
    return Err(Some(ValidationError::MigrationsDisabled(Reserved)));
}
```

With `ALLOWLIST = None`, this guard is permanently disabled and any caller proceeds to `validate_request` and `insert_request`.

**Analogy to the external report:**
The Vultisig bug used `_whitelistIndex[to] > _allowedWhitelistIndex` — the default value `0` for an unregistered address always passed the comparison. Here, the default value `None` for an unset allowlist always passes `caller_allowed`. Both are "Invalid Validation" bugs where the unregistered/default state grants access instead of denying it.

### Impact Explanation

Any unprivileged ingress sender or canister caller can invoke `migrate_canister` after any upgrade of the migration canister that does not explicitly re-supply the allowlist. The `migrate_canister` endpoint triggers cross-subnet canister migration operations: it validates and enqueues a `RequestState::Accepted` entry that the canister's background timers then act on, causing the IC registry to be updated and canisters to be moved between subnets. This constitutes unauthorized state mutation of production canisters and the IC registry.

The `DISABLED` flag (which is the only other gate) is stored in stable memory and survives upgrades, but it defaults to `false` (enabled), so it provides no protection in the default deployment. [6](#0-5) 

### Likelihood Explanation

The Candid interface declares the upgrade argument as `opt vec principal`: [7](#0-6) 

Omitting an `opt` field in a Candid call is the natural default. Any operator who upgrades the canister for a routine fix without re-reading the deployment runbook will silently open the endpoint. The test suite itself uses `allowlist: None` as the default in `Settings::default()`: [8](#0-7) 

This confirms that `None` is the common/default deployment mode, making accidental resets highly likely.

### Recommendation

Invert the semantics of `None`: treat an absent allowlist as "deny all" rather than "allow all." If open access is needed for initial deployment, require an explicit opt-in (e.g., a dedicated `AllowAll` variant) rather than relying on the absence of configuration:

```rust
pub fn caller_allowed(id: &Principal) -> bool {
    ALLOWLIST.with_borrow(|allowlist| match allowlist {
        Some(allowlist) => allowlist.contains(id),
        None => false,  // unset allowlist → deny all (safe default)
    })
}
```

Alternatively, persist the allowlist in stable memory (like `DISABLED`) so it survives upgrades without requiring re-supply in upgrade arguments.

### Proof of Concept

1. Deploy the migration canister with a restrictive allowlist:
   ```
   install migration-canister with allowlist = Some([trusted_principal])
   ```
2. Upgrade the canister without providing the allowlist (Candid default `null`):
   ```
   upgrade migration-canister with allowlist = null
   ```
3. After upgrade, `ALLOWLIST` is `None`; `caller_allowed` returns `true` for any caller.
4. Call `migrate_canister` from any unprivileged principal:
   ```
   migrate_canister({ migrated_canister_id = X; replaced_canister_id = Y })
   ```
5. The call bypasses the allowlist check and proceeds to enqueue a migration request, triggering unauthorized cross-subnet canister migration. [9](#0-8) [10](#0-9)

```rust
#[update]
async fn migrate_canister(args: MigrateCanisterArgs) -> Result<(), Option<ValidationError>> {
    if migrations_disabled() {
        return Err(Some(ValidationError::MigrationsDisabled(Reserved)));
    }
    // Prevent too many interleaved validations.
    let Ok(_guard) = ValidationGuard::new() else {
        return Err(Some(ValidationError::RateLimited(Reserved)));
    };
    if rate_limited() {
        return Err(Some(ValidationError::RateLimited(Reserved)));
    }
    let caller = msg_caller();
    // For soft rollout purposes
    if !caller_allowed(&caller) {
        return Err(Some(ValidationError::MigrationsDisabled(Reserved)));
    }
    match validate_request(args.migrated_canister_id, args.replaced_canister_id, caller).await {
        Err(e) => {
            println!("Failed to validate request {}: {}", args, e);
            return Err(Some(e));
        }
        Ok((request, _guards)) => {
            // Need to check the rate limit again
            if rate_limited() {
                return Err(Some(ValidationError::RateLimited(Reserved)));
            }
            println!("Accepted request {}", request);
            insert_request(RequestState::Accepted { request });
        }
    }
    Ok(())
}
```
