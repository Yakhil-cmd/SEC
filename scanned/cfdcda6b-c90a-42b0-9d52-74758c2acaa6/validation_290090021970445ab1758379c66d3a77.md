### Title
Attacker-Controlled `unsigned_transaction` Causes `panic!` in `handle_change_auto_stake_maturity`, Crashing the Rosetta Node — (`rs/rosetta-api/icp/src/ledger_client/handle_change_auto_stake_maturity.rs`)

---

### Summary

The `handle_change_auto_stake_maturity` function contains an unconditional `panic!` wildcard arm that fires whenever the governance canister returns any `ManageNeuronResponse::Command` variant other than `Configure` or `Error`. Because the `unsigned_transaction` field in `/construction/combine` is fully attacker-controlled and is never validated for consistency between its `RequestType` tag and its Candid-encoded argument, an unprivileged attacker who owns any ICP neuron can craft a transaction that causes the governance canister to reply with `Command::Disburse(...)` (or any other non-Configure variant), which is then dispatched to `handle_change_auto_stake_maturity`, triggering the panic and crashing the Rosetta node process.

---

### Finding Description

**The panic:**

`handle_change_auto_stake_maturity` matches only two arms and panics on everything else:

```rust
match &response.command {
    Some(Command::Configure(_)) => Ok(Ok(None)),
    Some(Command::Error(err)) => Ok(Err(...)),
    _ => panic!(
        "Unexpected change auto staking maturity result: {:?}",
        response.command
    ),
}
``` [1](#0-0) 

**The attacker-controlled input:**

`UnsignedTransaction` is a CBOR+hex string supplied by the caller in the `/construction/combine` request body. It contains a `Vec<(RequestType, HttpCanisterUpdate)>` — the `RequestType` tag and the raw Candid argument are both part of this attacker-supplied blob:

```rust
pub struct UnsignedTransaction {
    pub updates: Vec<(RequestType, HttpCanisterUpdate)>,
    pub ingress_expiries: Vec<u64>,
}
``` [2](#0-1) 

`construction_combine` deserializes this string directly from the request with no cross-validation between the `RequestType` and the Candid argument:

```rust
let unsigned_transaction = UnsignedTransaction::from_str(&msg.unsigned_transaction)
    .map_err(|e| { ... })?;
for (request_type, update) in unsigned_transaction.updates { ... }
``` [3](#0-2) 

**The dispatch path:**

After submission, `handle_reply` dispatches solely on the `RequestType` tag from the (attacker-supplied) `unsigned_transaction`, not on the actual Candid argument that was sent to the canister:

```rust
RequestType::ChangeAutoStakeMaturity { .. } => handle_change_auto_stake_maturity(bytes),
``` [4](#0-3) 

**The governance canister response:**

The NNS governance canister's `manage_neuron` handler returns `Command::Disburse(DisburseResponse { transfer_block_height })` on a successful disburse:

```rust
Some(Command::Disburse(d)) => self
    .disburse_neuron(&id, caller, d)
    .await
    .map(ManageNeuronResponse::disburse_response),
``` [5](#0-4) 

```rust
pub fn disburse_response(transfer_block_height: u64) -> Self {
    ManageNeuronResponse {
        command: Some(manage_neuron_response::Command::Disburse(...)),
    }
}
``` [6](#0-5) 

---

### Impact Explanation

When the panic fires, the Rosetta node process aborts. All in-flight `/construction/submit` calls for concurrent users are lost. Any exchange or custodian running the ICP Rosetta node loses the ability to submit or track ICP transactions until the process is manually restarted. The panic is not caught — Rust panics in the default configuration unwind and terminate the thread/process.

---

### Likelihood Explanation

The preconditions are low-barrier:
1. The attacker needs any ICP neuron (requires staking a small amount of ICP — publicly available).
2. The attacker must craft a CBOR-encoded `UnsignedTransaction` with `RequestType::ChangeAutoStakeMaturity` but with a `Disburse` Candid argument targeting their own neuron. This requires moderate technical skill but no privileged access.
3. The attacker submits via the public Rosetta HTTP API (`/construction/combine`, `/construction/submit`).

No admin keys, governance majority, or network-level attacks are required.

---

### Recommendation

Replace the `panic!` wildcard arm with a graceful error return:

```rust
_ => Err(format!(
    "Unexpected change auto staking maturity result: {:?}",
    response.command
)),
```

Apply the same fix to all sibling handlers that contain identical `panic!` wildcard arms (e.g., `handle_disburse`, `handle_spawn`, etc.). [7](#0-6) 

Additionally, add validation in `construction_combine` or `construction_submit` to verify that the Candid-encoded argument in the `HttpCanisterUpdate` is consistent with the declared `RequestType`, so a mismatched payload is rejected before it reaches the governance canister.

---

### Proof of Concept

```
1. Attacker owns a dissolved NNS neuron (neuron_id = X, controller = attacker_principal).

2. Craft a malicious UnsignedTransaction:
   updates = [(RequestType::ChangeAutoStakeMaturity { neuron_index: 0 },
               HttpCanisterUpdate {
                 canister_id: governance_canister_id,
                 method_name: "manage_neuron",
                 arg: candid::encode_one(ManageNeuron {
                   id: Some(NeuronId { id: X }),
                   command: Some(Command::Disburse(Disburse {
                     amount: None,
                     to_account: Some(attacker_account),
                   })),
                   neuron_id_or_subaccount: None,
                 }).unwrap(),
                 sender: attacker_principal_bytes,
                 ingress_expiry: <valid>,
                 nonce: None,
               })]
   ingress_expiries = [<valid>]
   
   Serialize to CBOR+hex → malicious_unsigned_tx

3. POST /construction/combine
   { unsigned_transaction: malicious_unsigned_tx,
     signatures: [<valid sig over update.id()>] }

4. POST /construction/submit with the returned signed_transaction.

5. Governance canister processes Disburse, returns
   ManageNeuronResponse { command: Some(Command::Disburse(...)) }.

6. handle_reply dispatches to handle_change_auto_stake_maturity(bytes).

7. match arm `_` fires → panic! → Rosetta node process terminates.
```
