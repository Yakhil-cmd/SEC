### Title
SNS Swap Permanent Abort via ICP Cap Saturation Without Sufficient Participation - (File: `rs/sns/swap/src/swap.rs`)

### Summary
An unprivileged ICP holder can permanently abort any open SNS decentralization swap by contributing enough ICP to reach `max_direct_participation_icp_e8s` using fewer accounts than `min_participants`. The swap immediately and irreversibly transitions to `LIFECYCLE_ABORTED`. The attacker recovers their full ICP principal (minus ledger transfer fees) via `sweep_icp`, making the net cost of the attack negligible.

### Finding Description

The SNS swap canister's `can_abort` function in `rs/sns/swap/src/swap.rs` encodes the following abort condition:

```rust
pub fn can_abort(&self, now_seconds: u64) -> bool {
    if self.lifecycle() != Lifecycle::Open {
        return false;
    }
    (self.swap_due(now_seconds) || self.icp_target_progress().is_reached_or_exceeded())
        && !self.sufficient_participation()
}
``` [1](#0-0) 

The abort fires immediately — before the swap's due date — whenever `icp_target_reached && !sufficient_participation`. This is confirmed by the in-tree test:

```
// swap should abort because we have reached the max icp but
// have not reached the minimum number of participants
assert!(result);
assert_eq!(swap.lifecycle, Lifecycle::Aborted as i32);
``` [2](#0-1) 

The lifecycle diagram in the protocol specification makes the abort path explicit:

```
not sufficient_participation
&& (swap_due || icp_target_reached)
→ ABORTED
``` [3](#0-2) 

**Attack mechanics:**

An attacker who holds enough ICP can:

1. Send ICP to their subaccount of the swap canister.
2. Call `refresh_buyer_token_e8s` to register the contribution, pushing `direct_participation_icp_e8s` to `max_direct_participation_icp_e8s`.
3. On the next heartbeat, `try_abort` fires automatically and succeeds because `icp_target_reached && !sufficient_participation` (the attacker used fewer accounts than `min_participants`).
4. The swap is permanently set to `LIFECYCLE_ABORTED`.
5. The attacker calls `sweep_icp` (or waits for it to be called automatically during finalization) and recovers their full ICP principal minus two ledger transfer fees.

The condition `max_participant_icp_e8s * (min_participants − 1) ≥ max_direct_participation_icp_e8s` must hold for the attacker to reach the cap with fewer accounts than `min_participants`. This is satisfied whenever a single participant's cap equals or approaches the total cap — a common configuration (e.g., `min_participants = 2`, `max_participant_icp_e8s = max_direct_participation_icp_e8s`).

The `Params` message confirms that `max_participant_icp_e8s` can be set equal to `max_icp_e8s`:

> "Can effectively be disabled by setting it to `max_icp_e8s`." [4](#0-3) 

The `IcpTargetProgress` enum and `is_reached_or_exceeded` confirm that reaching the cap is a binary, immediately-evaluated condition with no grace period: [5](#0-4) 

### Impact Explanation

The SNS decentralization swap is permanently aborted. Once `LIFECYCLE_ABORTED` is set it cannot be reversed:

```rust
// Cannot open when aborted.
assert!(!swap.can_open(END_TIMESTAMP_SECONDS + 1));
assert!(!swap.try_open(END_TIMESTAMP_SECONDS + 1));
``` [6](#0-5) 

Consequences:
- The SNS project fails to decentralize; governance tokens are never distributed to the public.
- Legitimate participants who had already contributed ICP receive refunds but lose the opportunity to participate.
- The SNS dapp canisters are returned to `fallback_controller_principal_ids`, effectively handing control back to the original developers and defeating the purpose of the swap.
- The NNS Neurons' Fund, if participating, is notified of an `Aborted` outcome and its maturity deductions are reversed — but the SNS launch is dead.

### Likelihood Explanation

**Moderate-to-high.** The attacker's net cost is only two ICP ledger transfer fees (≈ 0.0002 ICP each). The capital requirement (`max_direct_participation_icp_e8s`) is temporary and fully recovered. Any entity with sufficient ICP liquidity — including a competitor of the SNS project, a short-seller of the SNS token, or a protocol adversary — can execute this attack. No privileged access, governance majority, or cryptographic capability is required. The entry point (`refresh_buyer_token_e8s`) is a standard public update call available to any principal.

The attack is most trivially executable when `max_participant_icp_e8s = max_direct_participation_icp_e8s` (a single account suffices), but it generalizes to any configuration where `max_participant_icp_e8s × (min_participants − 1) ≥ max_direct_participation_icp_e8s`.

### Recommendation

1. **Remove early abort on ICP cap saturation.** Only allow abort at or after `swap_due_timestamp_seconds`. The early-commit path (when sufficient participation is met) can remain; only the early-abort path is dangerous.
2. **Alternatively, add a minimum participation window.** Require that the swap has been open for at least some minimum duration before an early abort is permitted, giving legitimate participants time to join.
3. **Cap per-participant contribution relative to `min_participants`.** Enforce `max_participant_icp_e8s ≤ max_direct_participation_icp_e8s / min_participants` at swap initialization so that reaching the cap always requires at least `min_participants` distinct accounts (guaranteeing sufficient participation).

### Proof of Concept

**Setup:** SNS swap is open with `min_participants = 2`, `max_participant_icp_e8s = max_direct_participation_icp_e8s = 500_000 ICP`, `swap_due_timestamp_seconds = T + 7 days`.

**Step 1:** Attacker transfers 500,000 ICP to their subaccount of the swap canister via the ICP ledger.

**Step 2:** Attacker calls `refresh_buyer_token_e8s` (public update call, no authorization required). The swap records `direct_participation_icp_e8s = 500_000 ICP = max_direct_participation_icp_e8s`. `icp_target_progress()` returns `Reached(500_000 ICP)`.

**Step 3:** On the next heartbeat (seconds later), the swap canister calls `try_abort(now)`. `can_abort` evaluates:
- `icp_target_progress().is_reached_or_exceeded()` → `true`
- `sufficient_participation()` → `false` (only 1 participant, need 2)
- Result: `true` → swap transitions to `LIFECYCLE_ABORTED`.

**Step 4:** `finalize` is called (automatically or manually). `sweep_icp` transfers 500,000 ICP (minus two transfer fees ≈ 0.0002 ICP each) back to the attacker's principal.

**Net attacker cost:** ≈ 0.0004 ICP in transfer fees.
**Net impact:** SNS swap permanently aborted; project cannot re-launch the same swap. [7](#0-6) [8](#0-7)
