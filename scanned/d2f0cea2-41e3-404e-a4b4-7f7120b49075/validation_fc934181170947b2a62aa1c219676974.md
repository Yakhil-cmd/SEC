### Title
Missing Anonymous Principal Validation in SNS Swap `Init` Canister ID Fields - (File: rs/sns/swap/src/types.rs)

### Summary
The `validate_canister_id` helper called by `Init::validate()` in the SNS Swap canister only checks that a string is a syntactically valid `PrincipalId`. It never rejects the IC anonymous principal (`2vxsx-fae` / `Principal::anonymous()`), which is the direct IC analog of Ethereum's `address(0)`. All five critical canister-ID fields in `Init` (`nns_governance_canister_id`, `sns_governance_canister_id`, `sns_ledger_canister_id`, `icp_ledger_canister_id`, `sns_root_canister_id`) can be set to the anonymous principal and will pass on-chain validation without error.

### Finding Description

`Init::validate()` calls `validate_canister_id` for each of the five canister-ID string fields: [1](#0-0) 

`validate_canister_id` only parses the string into a `PrincipalId`; it never checks whether the result is the anonymous principal: [2](#0-1) 

The anonymous principal (`"2vxsx-fae"`) is a syntactically valid `PrincipalId`, so passing it for any of those fields silently returns `Ok(())`. The SNS Swap constructor panics only if `init.validate()` returns an error, making this the sole programmatic guard: [3](#0-2) 

If the anonymous principal passes validation, the swap is deployed in a permanently broken or exploitable state.

Similarly, `validate_principal` (used for `fallback_controller_principal_ids`) is defined in the same file and follows the same pattern — it is not shown to reject the anonymous principal either. [4](#0-3) 

### Impact Explanation

**Scenario A — `nns_governance_canister_id` = anonymous principal:**
The swap's `open` guard checks `caller == nns_governance_or_panic()`. If that field is the anonymous principal, any unauthenticated ingress message (which carries the anonymous principal as its caller) satisfies the check. An attacker can open the swap without NNS governance approval, bypassing the intended access control entirely.

**Scenario B — `icp_ledger_canister_id` = anonymous principal:**
All ICP transfer inter-canister calls inside the swap target the anonymous principal, which is not a canister. Every participant's ICP deposit becomes permanently unrecoverable — a ledger conservation bug.

**Scenario C — `sns_root_canister_id` = anonymous principal:**
The finalization step that hands control of SNS canisters to the root canister silently fails, leaving the SNS in an uncontrolled state.

**Scenario D — anonymous principal in `fallback_controller_principal_ids`:**
In a failed swap, the anonymous principal is set as a controller of the SNS canisters. Because any unauthenticated ingress call carries the anonymous principal, any user could exercise controller-level management calls on those canisters.

### Likelihood Explanation

The SNS launch flow is initiated by a developer submitting an NNS proposal containing the raw `Init` struct. The `validate_canister_id` function is the only programmatic guard on these fields. The init args are Candid-encoded and not always human-readable in governance UIs. If neuron holders approve a proposal without carefully inspecting the raw canister-ID fields, a swap with the anonymous principal as a canister ID will be deployed. The likelihood is **low-to-medium**: the NNS governance process provides a social check, but the missing programmatic validation means a single inattentive governance vote is sufficient to trigger the impact.

### Recommendation

Add an explicit anonymous-principal rejection in `validate_canister_id`:

```rust
pub fn validate_canister_id(p: &str) -> Result<(), String> {
    let pp = PrincipalId::from_str(p).map_err(|x| {
        format!("Couldn't validate CanisterId. String \"{p}\" could not be converted to PrincipalId: {x}")
    })?;
    if pp == PrincipalId::new_anonymous() {
        return Err(format!("CanisterId \"{p}\" must not be the anonymous principal"));
    }
    Ok(())
}
```

Apply the same check inside `validate_principal` for `fallback_controller_principal_ids`. The ckETH minter already demonstrates the correct pattern for this check: [5](#0-4) 

### Proof of Concept

1. Craft an SNS `Init` payload where `nns_governance_canister_id = "2vxsx-fae"` (the anonymous principal text encoding).
2. Call `Init::validate()` — it returns `Ok(())` because `validate_canister_id` only checks syntactic validity.
3. The SNS Swap canister is deployed with `nns_governance_canister_id` pointing to the anonymous principal.
4. Any unauthenticated ingress call to the swap's `open` endpoint carries `Principal::anonymous()` as the caller, which equals `nns_governance_or_panic()`, satisfying the access check and allowing anyone to open the swap without NNS governance approval. [2](#0-1) [1](#0-0)

    pub fn validate(&self) -> Result<(), String> {
        validate_canister_id(&self.nns_governance_canister_id)?;
        validate_canister_id(&self.sns_governance_canister_id)?;
        validate_canister_id(&self.sns_ledger_canister_id)?;
        validate_canister_id(&self.icp_ledger_canister_id)?;
        validate_canister_id(&self.sns_root_canister_id)?;

        if self.fallback_controller_principal_ids.is_empty() {
            return Err("at least one fallback controller required".to_string());
        }
        for fc in &self.fallback_controller_principal_ids {
            validate_principal(fc)?;
        }

        if self.transaction_fee_e8s.is_none() {
            // The value itself is not checked; only that it is supplied. Needs to
            // match the value in SNS ledger though.
            return Err("transaction_fee_e8s is required.".to_string());
        }

        if self.neuron_minimum_stake_e8s.is_none() {
            // As with transaction_fee_e8s, the value itself is not checked; only
            // that it is supplied. Needs to match the value in SNS governance
            // though.
            return Err("neuron_minimum_stake_e8s is required.".to_string());
        }

        self.validate_swap_init_for_one_proposal_flow()?;

        if self.should_auto_finalize.is_none() {
            return Err("should_auto_finalize is required.".to_string());
        }

        Ok(())
    }
