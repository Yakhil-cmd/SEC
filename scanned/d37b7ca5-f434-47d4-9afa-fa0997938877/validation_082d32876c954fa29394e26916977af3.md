### Title
Insufficient Init Parameter Validation in SNS Swap `Init::validate()` Allows Zero-Value `transaction_fee_e8s` and `neuron_minimum_stake_e8s` - (`rs/sns/swap/src/types.rs`)

---

### Summary
The SNS Swap canister's `Init::validate()` function explicitly skips value-range validation for `transaction_fee_e8s` and `neuron_minimum_stake_e8s`, only checking that they are `Some(...)`. Supplying `Some(0)` for either field passes canister-level validation, bypassing the invariant that `neuron_minimum_stake_e8s > transaction_fee_e8s`. This can cause the swap to open with economically broken parameters and cause neuron-creation to fail silently during finalization, permanently locking participant ICP.

---

### Finding Description

In `rs/sns/swap/src/types.rs`, the `Init::validate()` function contains the following code:

```rust
if self.transaction_fee_e8s.is_none() {
    // The value itself is not checked; only that it is supplied. Needs to
    // match the value in SNS ledger though.
    return Err("transaction_fee_e8s is required.".to_string());
}

if self.neuron_minimum_stake_e8s.is_none() {
    // As with transaction_fee_e8s, the value itself is not checked; only
    // that it is supplied. Needs to match the value in SNS governance
    // though.
    return Err("neuron_minimum_stake_e8s is required.".to_string());
}
```

The comments are self-incriminating: **"The value itself is not checked."** `Some(0)` satisfies both guards. The proto definition reinforces the risk:

> "Whether the values match is not checked. If they don't match things will break." [1](#0-0) 

The downstream `Params::validate()` computes the minimum-stake adequacy check as:

```rust
let min_participant_sns_e8s = self.min_participant_icp_e8s as u128
    * self.sns_token_e8s as u128
    / self.max_icp_e8s as u128;

let min_participant_icp_e8s_big_enough = min_participant_sns_e8s
    >= neuron_basket_count * (neuron_minimum_stake_e8s + transaction_fee_e8s) as u128;
``` [2](#0-1) 

When both fields are `0`, the right-hand side collapses to `0`, making the inequality trivially true for any `min_participant_icp_e8s`. The guard that is supposed to prevent economically unsound swaps is completely neutralised.

The `canister_init` entry point passes the `Init` payload directly to `Swap::new()` without enforcing `validate()` as a precondition:

```rust
#[init]
fn canister_init(init_payload: Init) {
    let swap = Swap::new(init_payload);
    ...
}
``` [3](#0-2) 

The proto field definitions confirm these are immutable after deployment: [4](#0-3)

### Impact Explanation

**Scenario A – `transaction_fee_e8s = Some(0)`:**
During swap finalization, the canister creates SNS neurons by transferring tokens via the SNS ledger. The actual ledger charges a non-zero fee. The swap canister's internal accounting uses `transaction_fee_e8s = 0`, so it believes each neuron transfer costs nothing. When the real ledger deducts its fee, the computed token amounts fall below `neuron_minimum_stake_e8s`, causing neuron-creation calls to fail. Participant ICP is already committed; the swap cannot be re-opened. Participants lose their ICP with no SNS neurons minted in return.

**Scenario B – `neuron_minimum_stake_e8s = Some(0)`:**
The adequacy check in `Params::validate()` becomes `min_participant_sns_e8s >= 0`, always true. A swap can be opened where each participant receives dust-level SNS neurons (below the governance minimum), rendering those neurons ungovernable and the tokens economically worthless.

**Scenario C – Both zero:**
Both effects compound. The swap opens, collects ICP, and finalization fails entirely, permanently locking all participant ICP in the swap canister with no recourse.

---

### Likelihood Explanation

The normal SNS deployment path goes through `SnsInitPayload::validate_participation_constraints()` in `rs/sns/init/src/lib.rs`, which does enforce `neuron_minimum_stake_e8s > transaction_fee_e8s`: [5](#0-4) 

However, the SNS Swap canister's `Init::validate()` is the **canister's own trust boundary**. Any deployment path that bypasses the SNS-WASM orchestrator (e.g., direct canister installation by a developer, a future alternative deployment tool, or a governance proposal that installs a swap canister with a hand-crafted `Init`) reaches `canister_init` with no value-range enforcement. The canister itself provides no defence. The explicit code comment acknowledging the omission increases the likelihood that this gap will be exploited or triggered by misconfiguration.

---

### Recommendation

Add explicit lower-bound checks inside `Init::validate()`:

```rust
if let Some(fee) = self.transaction_fee_e8s {
    // fee == 0 is technically valid for some ledgers, but
    // neuron_minimum_stake must still exceed it.
} else {
    return Err("transaction_fee_e8s is required.".to_string());
}

if let Some(min_stake) = self.neuron_minimum_stake_e8s {
    let fee = self.transaction_fee_e8s.unwrap_or(0);
    if min_stake == 0 {
        return Err("neuron_minimum_stake_e8s must be > 0".to_string());
    }
    if min_stake <= fee {
        return Err(format!(
            "neuron_minimum_stake_e8s ({min_stake}) must be greater than \
             transaction_fee_e8s ({fee})"
        ));
    }
} else {
    return Err("neuron_minimum_stake_e8s is required.".to_string());
}
```

This mirrors the validation already present in `SnsInitPayload::validate_participation_constraints()` and closes the gap at the canister boundary.

### Proof of Concept

1. Deploy an SNS Swap canister directly (outside the SNS-WASM flow) with:
   ```
   Init {
       transaction_fee_e8s: Some(0),
       neuron_minimum_stake_e8s: Some(0),
       // ... all other required fields valid
   }
   ```
2. Call `init.validate()` — it returns `Ok(())` because both fields are `Some(...)`.
3. Open the swap with any `Params`; `Params::validate()` passes because `neuron_basket_count * (0 + 0) = 0 <= min_participant_sns_e8s`.
4. Participants call `new_sale_ticket` and `refresh_buyer_tokens`, committing ICP.
5. Call `finalize_swap`. The canister attempts to transfer SNS tokens to create neurons. The actual SNS ledger deducts its real fee; the computed per-neuron amount underflows or falls below the ledger minimum. Neuron-creation calls fail. Participant ICP remains locked in the swap canister with no recovery path. [6](#0-5) [7](#0-6)
