### Title
Unpermissioned `ClaimOrRefresh` Neuron Stake Refresh Dilutes Victim's Age Bonus — (`rs/nns/governance/src/governance.rs`, `rs/sns/governance/src/governance.rs`)

---

### Summary

Both NNS and SNS Governance canisters allow **any unprivileged caller** to trigger a neuron stake refresh (`ClaimOrRefresh`) on behalf of any neuron owner. When an attacker first transfers tokens to the victim's neuron subaccount and then calls `manage_neuron` with `ClaimOrRefresh`, the governance canister calls `update_stake_adjust_age` / `update_stake`, which dilutes the victim's `aging_since_timestamp_seconds`. This permanently reduces the victim's age-based voting power bonus without the victim's consent, and the attack can be repeated indefinitely.

---

### Finding Description

The NNS Governance `manage_neuron_internal` function handles `ClaimOrRefresh` commands before any authorization check, and explicitly permits any caller to refresh any neuron: [1](#0-0) 

The `By::NeuronIdOrSubaccount` path calls `refresh_neuron_by_id_or_subaccount`, which resolves the neuron from its public ID and calls `refresh_neuron` with no caller authorization: [2](#0-1) 

Inside `refresh_neuron`, when the ledger balance exceeds the cached stake (i.e., someone deposited tokens to the neuron's subaccount), `update_stake_adjust_age` is called: [3](#0-2) 

`update_stake_adjust_age` computes a weighted-average age: `new_age = (old_age × old_stake) / new_stake`. This permanently moves `aging_since_timestamp_seconds` forward, reducing the neuron's effective age: [4](#0-3) 

The same pattern exists in SNS Governance. `refresh_neuron` is callable by any caller (no authorization check), and calls `update_stake` which applies the same age-dilution formula: [5](#0-4) [6](#0-5) 

The NNS test suite explicitly documents this as intentional behavior ("Anyone can do it"): [7](#0-6) 

---

### Impact Explanation

A neuron's age bonus contributes up to **25% additional voting power** in NNS (up to `max_age_bonus_percentage` in SNS, configurable up to 100%). An attacker who:

1. Queries the victim's neuron subaccount (public on-chain data),
2. Transfers tokens to that subaccount (the attacker's tokens are permanently added to the victim's neuron stake),
3. Calls `manage_neuron` with `ClaimOrRefresh { by: NeuronIdOrSubaccount }` targeting the victim's neuron,

causes `update_stake_adjust_age` to dilute the victim's age. For example, a neuron with 10 ICP staked and 4 years of age (maximum 25% age bonus) that receives an additional 10 ICP deposit will have its effective age halved to 2 years (~12.5% bonus), permanently reducing its voting power. The attack can be repeated to asymptotically drive the age bonus toward zero.

The impact is governance manipulation: targeted reduction of specific neurons' voting power, which can shift the outcome of NNS or SNS proposals. The victim's stake increases (which partially compensates), but the age bonus loss is permanent and irreversible without waiting years to re-accumulate age.

---

### Likelihood Explanation

**Medium.** The attack requires the attacker to spend real ICP/SNS tokens (which are permanently added to the victim's neuron). However, for a well-resourced attacker targeting a high-value neuron (e.g., a whale neuron with years of accumulated age that controls a decisive vote), the cost may be acceptable. The neuron subaccount is deterministic and publicly derivable. No privileged access is required — any ingress sender can execute this.

---

### Recommendation

Restrict `ClaimOrRefresh` refresh operations so that only the neuron's controller or a hotkey can trigger a refresh that modifies the neuron's age. Alternatively, when a refresh is triggered by a third party (not the controller/hotkey), skip the `update_stake_adjust_age` call and only update `cached_neuron_stake_e8s` without adjusting the age. This preserves the legitimate use case (anyone can top up a neuron's stake) while preventing age dilution attacks.

---

    pub fn update_stake_adjust_age(&mut self, updated_stake_e8s: u64, now: u64) {
        // If the updated stake is less than the original stake, preserve the
        // age and distribute it over the new amount. This should not happen
        // in practice, so this code exists merely as a defensive fallback.
        //
        // TODO(NNS1-954) Consider whether update_stake_adjust_age (and other
        // similar methods) should use a neurons effective stake rather than
        // the cached stake.
        if updated_stake_e8s < self.cached_neuron_stake_e8s {
            println!(
                "{}Reducing neuron {:?} stake via update_stake_adjust_age: {} -> {}",
                LOG_PREFIX,
                self.id(),
                self.cached_neuron_stake_e8s,
                updated_stake_e8s
            );
            self.cached_neuron_stake_e8s = updated_stake_e8s;
        } else {
            // If one looks at "stake * age" as describing an area, the goal
            // at this point is to increase the stake while keeping the area
            // constant. This means decreasing the age in proportion to the
            // additional stake, which is the purpose of combine_aged_stakes.
            let (new_stake_e8s, new_age_seconds) = combine_aged_stakes(
                self.cached_neuron_stake_e8s,
                self.age_seconds(now),
                updated_stake_e8s.saturating_sub(self.cached_neuron_stake_e8s),
                0,
            );
            // A consequence of the math above is that the 'new_stake_e8s' is
            // always the same as the 'updated_stake_e8s'. We use
            // 'combine_aged_stakes' here to make sure the age is
            // appropriately pro-rated to accommodate the new stake.
            assert!(new_stake_e8s == updated_stake_e8s);
            self.cached_neuron_stake_e8s = new_stake_e8s;

            let new_aging_since_timestamp_seconds = now.saturating_sub(new_age_seconds);
            let new_disolved_dissolve_state_and_age = self
                .dissolve_state_and_age()
                .adjust_age(new_aging_since_timestamp_seconds);
            self.set_dissolve_state_and_age(new_disolved_dissolve_state_and_age);
        }

    async fn refresh_neuron(&mut self, nid: &NeuronId) -> Result<(), GovernanceError> {
        let now = self.env.now();
        let subaccount = nid.subaccount()?;
        let account = self.neuron_account_id(subaccount);

        // First ensure that the neuron was not created via an NNS Neurons' Fund participation in the
        // decentralization swap
        {
            let neuron = self.get_neuron_result(nid)?;

            if neuron.is_neurons_fund_controlled() {
                return Err(GovernanceError::new_with_message(
                    ErrorType::PreconditionFailed,
                    "Cannot refresh an SNS Neuron controlled by the Neurons' Fund",
                ));
            }
        }

        // Get the balance of the neuron from the ledger canister.
        let balance = self.ledger.account_balance(account).await?;

        let min_stake = self
            .nervous_system_parameters_or_panic()
            .neuron_minimum_stake_e8s
            .expect("NervousSystemParameters must have neuron_minimum_stake_e8s");
        if balance.get_e8s() < min_stake {
            return Err(GovernanceError::new_with_message(
                ErrorType::InsufficientFunds,
                format!(
                    "Account does not have enough funds to refresh a neuron. \
                        Please make sure that account has at least {:?} e8s (was {:?} e8s)",
                    min_stake,
                    balance.get_e8s()
                ),
            ));
        }
        let neuron = self.get_neuron_result_mut(nid)?;
        match neuron.cached_neuron_stake_e8s.cmp(&balance.get_e8s()) {
            Ordering::Greater => {
                log!(
                    ERROR,
                    "ERROR. Neuron cached stake was inconsistent.\
                     Neuron account: {} has less e8s: {} than the cached neuron stake: {}.\
                     Stake adjusted.",
                    account,
                    balance.get_e8s(),
                    neuron.cached_neuron_stake_e8s
                );
                neuron.update_stake(balance.get_e8s(), now);
            }
            Ordering::Less => {
                neuron.update_stake(balance.get_e8s(), now);
            }
            // If the stake is the same as the account balance,
            // just return the neuron id (this way this method
            // also serves the purpose of allowing to discover the
            // neuron id based on the memo and the controller).
            Ordering::Equal => (),
        };

        Ok(())
    }

    pub fn update_stake(&mut self, new_stake_e8s: u64, now: u64) {
        // If this neuron has an age and its stake is being increased, adjust the
        // neuron's age
        if self.aging_since_timestamp_seconds < now && self.cached_neuron_stake_e8s <= new_stake_e8s
        {
            let old_stake = self.cached_neuron_stake_e8s as u128;
            let old_age = now.saturating_sub(self.aging_since_timestamp_seconds) as u128;
            let new_age = (old_age * old_stake) / (new_stake_e8s as u128);

            // new_age * new_stake = old_age * old_stake -
            // (old_stake * old_age) % new_stake. That is, age is
            // adjusted in proportion to the stake, but due to the
            // discrete nature of u64 numbers, some resolution is
            // lost due to the division above. This means the age
            // bonus is derived from a constant times age times
            // stake, minus up to new_stake - 1 each time the
            // neuron is refreshed. Only if old_age * old_stake is
            // a multiple of new_stake does the age remain
            // constant after the refresh operation. However, in
            // the end, the most that can be lost due to rounding
            // from the actual age, is always less 1 second, so
            // this is not a problem.
            self.aging_since_timestamp_seconds = now.saturating_sub(new_age as u64);
            // Note that if new_stake == old_stake, then
            // new_age == old_age, and
            // now - new_age =
            // now-(now-neuron.aging_since_timestamp_seconds)
            // = neuron.aging_since_timestamp_seconds.
        }

        self.cached_neuron_stake_e8s = new_stake_e8s;
    }
