### Title
SNS Swap `finalize_inner` Permanently Halts for All Participants When Any Single Buyer Has an Invalid or Failing ICP Transfer - (File: `rs/sns/swap/src/swap.rs`)

---

### Summary

The SNS Swap canister's `finalize_inner` function implements a sequential pipeline that aborts entirely when `sweep_icp` reports any `invalid` or `failure` result for even a single buyer. Because `invalid` is explicitly defined as a permanent, non-retryable state, a single buyer whose ICP transfer is permanently unprocessable causes all other buyers to be permanently denied their SNS tokens, prevents SNS governance from entering normal mode, and (in the ABORTED lifecycle) prevents dapp controllers from being restored to fallback controllers.

---

### Finding Description

`finalize_inner` in `rs/sns/swap/src/swap.rs` runs a strict sequential pipeline: [1](#0-0) 

```
sweep_icp → settle_neurons_fund → create_sns_neuron_recipes → sweep_sns → claim_swap_neurons → set_mode
```

Each step is gated by `has_error_message()`. The error is set inside `set_sweep_icp_result`: [2](#0-1) 

The trigger condition is `is_successful_sweep()`: [3](#0-2) 

```rust
fn is_successful_sweep(&self) -> bool {
    *failure == 0 && *invalid == 0 && *global_failures == 0
}
```

Inside `sweep_icp`, the loop processes every buyer individually and continues on per-entry errors — but the aggregate `SweepResult` is what gates the pipeline: [4](#0-3) 

`TransferResult::AmountTooSmall` (triggered when `amount_e8s <= fee_e8s`) increments `invalid`: [5](#0-4) 

`TransferResult::Failure` (any ledger error) increments `failure`: [6](#0-5) 

The proto definition of `SweepResult` explicitly states that `invalid` means the item **will never succeed** without manual intervention: [7](#0-6) 

The test `test_finalization_halts_when_sweep_icp_fails` confirms the halt behavior is intentional and complete — all downstream fields (`sweep_sns_result`, `claim_neuron_result`, `set_mode_call_result`) are `None` when `sweep_icp` has any `failure` or `invalid`: [8](#0-7) 

---

### Impact Explanation

**COMMITTED lifecycle (primary impact):**
- `sweep_sns` never executes → all buyers permanently lose their SNS token allocation
- `claim_swap_neurons` never executes → no SNS neurons are ever created
- `set_sns_governance_to_normal_mode` never executes → SNS governance remains in pre-initialization mode, unable to accept proposals or operate normally

**ABORTED lifecycle:**
- `restore_dapp_controllers_for_finalize` never executes → dapp canisters remain under SNS root control instead of being returned to fallback controllers [9](#0-8) 

The `invalid` case is permanent by protocol definition. Without a canister upgrade to manually remove or fix the problematic buyer entry, finalization is irreversibly blocked.

---

### Likelihood Explanation

**Realistic trigger — `AmountTooSmall` (invalid, permanent):** `transfer_helper` returns `AmountTooSmall` when `amount_e8s <= DEFAULT_TRANSFER_FEE`. The code comment says this "should never happen in production" because `refresh_buyer_tokens` enforces `min_participant_icp_e8s`. However, if an SNS deployment sets `min_participant_icp_e8s` equal to `DEFAULT_TRANSFER_FEE` (10,000 e8s), a buyer participating with exactly that amount satisfies `amount <= fee` and permanently triggers `invalid`. An unprivileged user can call `refresh_buyer_tokens` with the minimum allowed amount and create this state. [5](#0-4) 

**Realistic trigger — ledger `Failure` (transient but potentially persistent):** Any ICP ledger error during `sweep_icp` increments `failure` and halts finalization. While retryable, a persistent ledger rejection for a specific subaccount (e.g., due to a canister-level error) creates an indefinite stall affecting all other participants. [10](#0-9) 

The attacker entry path is `refresh_buyer_tokens` (unprivileged ingress), which registers a buyer in the `buyers` map. Any buyer whose subsequent ICP transfer is unprocessable during finalization becomes the blocking entry.

---

### Recommendation

1. **Separate `invalid` from pipeline-halting logic.** Buyers with permanently invalid states should be logged and skipped without setting an error message that halts the pipeline. Only `global_failures` (state corruption) should halt finalization.

2. **Adopt a pull pattern for token distribution.** Instead of pushing SNS tokens to all buyers in `sweep_sns`, allow buyers to claim their tokens individually. This isolates per-buyer failures.

3. **Enforce `min_participant_icp_e8s > DEFAULT_TRANSFER_FEE` at SNS initialization.** Add a validation check that `min_participant_icp_e8s > DEFAULT_TRANSFER_FEE` so that `AmountTooSmall` is structurally impossible.

4. **Separate `invalid` buyers into their own retry-exempt list** (analogous to the Across V3 recommendation to separate problematic addresses into their own refund leaf), allowing the rest of finalization to proceed.

---

### Proof of Concept

1. An SNS is deployed with `min_participant_icp_e8s = DEFAULT_TRANSFER_FEE` (10,000 e8s).
2. An attacker calls `refresh_buyer_tokens` with exactly 10,000 e8s, creating a `BuyerState` with `amount_e8s = 10_000`.
3. The swap reaches `Lifecycle::Committed` or `Lifecycle::Aborted`.
4. Anyone calls `finalize_swap`.
5. `sweep_icp` processes the attacker's entry: `transfer_helper` evaluates `amount (10_000) <= fee (10_000)` → `TransferResult::AmountTooSmall` → `sweep_result.invalid += 1`.
6. `is_successful_sweep()` returns `false` → `set_sweep_icp_result` sets `error_message`.
7. `finalize_inner` returns at line 1559–1561.
8. `sweep_sns`, `claim_swap_neurons`, `set_sns_governance_to_normal_mode` are never called.
9. All legitimate buyers receive neither their SNS tokens nor their ICP refunds. SNS governance is permanently stuck in pre-initialization mode. No canister upgrade can fix this without manual state surgery. [11](#0-10) [12](#0-11)

/// Tests that if transferring does not complete fully, finalize will halt finalization
#[tokio::test]
async fn test_finalization_halts_when_sweep_icp_fails() {
    // Step 1: Prepare the world

    // Setup the necessary buyers for the test
    let mut swap = Swap {
        lifecycle: Committed as i32,
        init: Some(init()),
        params: Some(params()),
        buyers: btreemap! {
            // This Buyer is `Invalid` because the amount committed is less than the
            // DEFAULT_TRANSFER_FEE of the ICP Ledger. This should never be possible
            // in production, but sweep_icp must handle this case.
            i2principal_id_string(1000) => BuyerState {
                icp: Some(TransferableAmount {
                    amount_e8s: DEFAULT_TRANSFER_FEE.get_e8s() - 1,
                    ..Default::default()
                }),
                has_created_neuron_recipes: Some(false),
            },
            // This buyer's state is valid, but a mock call to the ledger will fail the transfer,
            // which should result in a failure field increment.
            i2principal_id_string(1003) => BuyerState {
                icp: Some(TransferableAmount {
                    amount_e8s: 10 * E8,
                    ..Default::default()
                }),
                has_created_neuron_recipes: Some(false),
            },
        },
        ..Default::default()
    };

    let mut clients = CanisterClients {
        icp_ledger: SpyLedger::new(vec![
            // This mocked reply should produce a successful transfer in SweepResult
            LedgerReply::TransferFunds(Err(NervousSystemError::new_with_message(
                "Error when transferring funds",
            ))),
        ]),
        ..spy_clients()
    };

    // Step 2: Call sweep_icp
    let result = swap.finalize(now_fn, &mut clients).await;

    assert_eq!(
        result.sweep_icp_result,
        Some(SweepResult {
            success: 0,
            skipped: 0,
            failure: 1, // Single failed transfer
            invalid: 1, // Single invalid buyer
            global_failures: 0,
        })
    );

    assert_eq!(
        result.error_message,
        Some(String::from(
            "Transferring ICP did not complete fully, some transfers were invalid or failed. Halting swap finalization"
        ))
    );

    // Assert that all other fields are set to None because finalization was halted.
    assert!(result.settle_neurons_fund_participation_result.is_none());
    assert!(result.set_dapp_controllers_call_result.is_none());
    assert!(result.create_sns_neuron_recipes_result.is_none());
    assert!(result.sweep_sns_result.is_none());
    assert!(result.set_mode_call_result.is_none());
    assert!(result.claim_neuron_result.is_none());
}
