### Title
Stale ICP/XDR Conversion Rate Used in Cycle Minting Without Freshness Check - (`rs/nns/cmc/src/main.rs`)

### Summary
The Cycles Minting Canister (CMC) converts ICP to cycles in `notify_top_up`, `notify_create_canister`, and `notify_mint_cycles` by calling `tokens_to_cycles()`, which reads `state.icp_xdr_conversion_rate` without verifying the age of the stored rate. When the automatic exchange-rate-update mechanism is disabled (e.g., after a diverged-rate governance proposal), the rate can become arbitrarily stale. Any unprivileged user can then call these public endpoints and receive more cycles per ICP than the current market rate warrants.

### Finding Description

`tokens_to_cycles()` reads `state.icp_xdr_conversion_rate` and checks only that the field is `Some`; it never compares `rate.timestamp_seconds` against the current canister time: [1](#0-0) 

The three public update endpoints that trigger this path are: [2](#0-1) [3](#0-2) [4](#0-3) 

The rate is normally refreshed every `REFRESH_RATE_INTERVAL_SECONDS` (5 minutes) via the heartbeat: [5](#0-4) 

However, the `UpdateExchangeRateState` can be set to `Disabled` by a governance proposal carrying a `DivergedRate` reason: [6](#0-5) 

Once disabled, the heartbeat path returns early and no automatic rate update occurs: [7](#0-6) 

The rate stored in `state.icp_xdr_conversion_rate` is then frozen indefinitely until a new governance proposal re-enables updates or directly sets a new rate. During this window, `tokens_to_cycles()` silently uses the frozen value with no staleness guard.

### Impact Explanation

An unprivileged user who observes that the CMC's stored rate is stale and that the current ICP market price has risen above the frozen rate can:

1. Transfer ICP to the CMC subaccount (a normal ledger operation).
2. Call `notify_top_up`, `notify_create_canister`, or `notify_mint_cycles`.
3. Receive cycles computed at the old (lower) XDR/ICP rate, yielding more cycles per ICP than the current market price justifies.

This is a **cycles conservation / ledger conservation bug**: the protocol mints cycles in excess of what the current ICP/XDR market rate entitles the user to, effectively subsidising the attacker at the expense of the protocol's economic model. [8](#0-7) 

### Likelihood Explanation

The `DivergedRate` governance path is an explicitly designed operational mode. Historically, rate divergences have occurred on mainnet. During the disabled period — which can last hours to days depending on governance response time — ICP price can move materially. The attacker needs only a valid ICP ledger block and knowledge that the CMC rate is stale, both of which are publicly observable on-chain.

### Recommendation

In `tokens_to_cycles()`, compare `rate.timestamp_seconds` against `now_seconds()` and reject (or warn and abort) if the rate is older than a defined maximum staleness threshold (e.g., 24 hours). This mirrors the pattern already used in NNS Governance's `should_refresh_xdr_rate()`: [9](#0-8) 

### Proof of Concept

1. Submit a governance proposal with `DivergedRate` reason to disable automatic CMC rate updates.
2. Wait for ICP market price to rise above the frozen `xdr_permyriad_per_icp` value stored in CMC state.
3. Transfer N ICP to `CMC_PRINCIPAL / subaccount(caller)` with `MEMO_TOP_UP_CANISTER`.
4. Call `notify_top_up { block_index, canister_id }`.
5. CMC calls `tokens_to_cycles(N ICP)` → reads the stale (lower) rate → mints more cycles than the current market rate warrants.
6. Repeat until governance re-enables the exchange rate canister. [10](#0-9) [11](#0-10)

fn tokens_to_cycles(amount: Tokens) -> Result<Cycles, NotifyError> {
    with_state(|state| {
        let xdr_permyriad_per_icp = state
            .icp_xdr_conversion_rate
            .as_ref()
            .map(|rate| rate.xdr_permyriad_per_icp);
        match xdr_permyriad_per_icp {
            Some(xdr_permyriad_per_icp) => Ok(TokensToCycles {
                xdr_permyriad_per_icp,
                cycles_per_xdr: state.cycles_per_xdr,
            }
            .to_cycles(amount)),
            None => {
                let error_message =
                    "No conversion rate found in CMC, notification aborted".to_string();
                print(&error_message);
                Err(NotifyError::Other {
                    error_code: NotifyErrorCode::Internal as u64,
                    error_message,
                })
            }
        }
    })

    fn should_refresh_xdr_rate(&self) -> bool {
        let xdr_conversion_rate = &self.heap_data.xdr_conversion_rate;

        let now_seconds = self.env.now();

        let seconds_since_last_conversion_rate_refresh =
            now_seconds.saturating_sub(xdr_conversion_rate.timestamp_seconds);

        // Return `true` if more than 1 day has passed since the last `xdr_conversion_rate` was
        // updated. This assumes that `xdr_conversion_rate.timestamp_seconds` is rounded down to
        // the nearest day's beginning.
        seconds_since_last_conversion_rate_refresh > ONE_DAY_SECONDS
    }
