### Title
`spawn_neuron` Lower Bound Uses Stale 5% Worst-Case Modulation While Mission 70 Allows −10% — (File: `rs/nns/governance/src/governance.rs`)

---

### Summary

The `spawn_neuron` function in NNS Governance validates that the maturity to spawn is sufficient by applying a hardcoded **5% worst-case** maturity modulation. Since Proposal 141779, however, the actual neuron-spawning path (`maybe_spawn_neurons`) reads the **Mission 70** maturity modulation, whose lower bound is **−10%** (`MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 = -1_000`). The pre-condition check was never updated to match the new range. A neuron controller can therefore spawn a child neuron whose final minted stake, after actual modulation is applied, falls below `neuron_minimum_stake_e8s`.

---

### Finding Description

**Root cause — insufficient lower bound in `spawn_neuron`:**

```rust
// rs/nns/governance/src/governance.rs:2664-2673
let least_possible_stake = (maturity_to_spawn as f64 * (1_f64 - 0.05)) as u64;

if least_possible_stake < economics.neuron_minimum_stake_e8s {
    return Err(GovernanceError::new_with_message(
        ErrorType::InsufficientFunds,
        "There isn't enough maturity to spawn a new neuron due to worst case maturity modulation.",
    ));
}
```

The factor `(1_f64 - 0.05)` encodes the old CMC-based worst case of −5% (`MIN_MATURITY_MODULATION_PERMYRIAD = -500`).

**Actual spawning path — uses Mission 70 modulation (up to −10%):**

```rust
// rs/nns/governance/src/governance.rs:6427-6435
let maturity_modulation = match self
    .heap_data
    .maturity_modulation          // ← Mission 70 value, range [-1000, +200] permyriad
    .as_ref()
    .and_then(|m| m.current_value_permyriad)
{
    None => return,
    Some(value) => value,
};
```

The Mission 70 bounds are declared in `rs/nns/governance/src/timer_tasks/update_icp_xdr_rate_related_data.rs`:

```
MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 = -1_000   // −10 %
MATURITY_MODULATION_MAX_PERMYRIAD_MISSION_70 =    200   // +2 %
```

The CHANGELOG entry for Proposal 141779 explicitly states:

> *"Neuron spawning and maturity disbursement finalization now read the locally computed Mission 70 maturity modulation … instead of the CMC-polled `cached_daily_maturity_modulation_basis_points`."*

The pre-condition guard in `spawn_neuron` was not updated alongside this change, leaving a 5-percentage-point gap between the bound that is checked and the bound that is actually applied.

**Secondary instance — `initiate_maturity_disbursement`:**

The constant `MINIMUM_DISBURSEMENT_E8S` is documented as:

> *"A neuron can only disburse an amount of maturity that results in minting at least this many ICP (in e8) **assuming the worst case maturity modulation**."*

Yet the guard at `rs/nns/governance/src/governance/disburse_maturity.rs:293` compares the **raw** maturity directly against `MINIMUM_DISBURSEMENT_E8S = E8`, without applying any modulation. After −10% Mission 70 modulation, a disbursement of exactly `E8` maturity would mint only `0.9 × E8`, violating the stated invariant.

---

### Impact Explanation

For `spawn_neuron`: a neuron controller can supply maturity `M` satisfying

```
M × 0.95 ≥ neuron_minimum_stake_e8s   (passes the guard)
M × 0.90 < neuron_minimum_stake_e8s   (actual worst case)
```

When `maybe_spawn_neurons` fires 7 days later with Mission 70 modulation at −10%, the minted stake is `M × 0.90 < neuron_minimum_stake_e8s`. The resulting neuron violates the protocol invariant that every neuron holds at least `neuron_minimum_stake_e8s`. Such a sub-minimum neuron can still exist in the neuron store, participate in governance operations, and be subject to further management commands that assume the minimum-stake invariant holds.

For `initiate_maturity_disbursement`: a disbursement of exactly `MINIMUM_DISBURSEMENT_E8S` maturity will mint less than `MINIMUM_DISBURSEMENT_E8S` ICP when modulation is negative, contradicting the documented guarantee.

---

### Likelihood Explanation

The Mission 70 modulation moves at most 30 permyriad per day (`MATURITY_MODULATION_DAILY_SPEED_LIMIT_PERMYRIAD = 30`), so reaching −10% from 0 takes roughly 33 days of sustained ICP price decline. Once the modulation is in the −5% to −10% window, any neuron controller with maturity in the vulnerable range `[neuron_minimum_stake_e8s / 0.95, neuron_minimum_stake_e8s / 0.90)` can trigger the issue by calling `spawn_neuron`. The call is permissionless (any neuron controller via `manage_neuron`), and the 7-day spawning delay means the attacker only needs the modulation to remain below −5% for one week after the call.

---

### Recommendation

Replace the hardcoded `0.05` factor in `spawn_neuron` with the Mission 70 lower bound:

```rust
// Use apply_maturity_modulation with the Mission 70 floor
let least_possible_stake = apply_maturity_modulation(
    maturity_to_spawn,
    MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 as i32,
).unwrap_or(0);

if least_possible_stake < economics.neuron_minimum_stake_e8s {
    return Err(...);
}
```

For `initiate_maturity_disbursement`, apply the worst-case modulation before comparing against `MINIMUM_DISBURSEMENT_E8S`:

```rust
let worst_case_minted = apply_maturity_modulation(
    disbursement_maturity_e8s,
    MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 as i32,
).unwrap_or(0);
if worst_case_minted < MINIMUM_DISBURSEMENT_E8S {
    return Err(InitiateMaturityDisbursementError::DisbursementTooSmall { ... });
}
```

### Proof of Concept

1. Observe that `neuron_minimum_stake_e8s = 100_000_000` (1 ICP).
2. Wait (or observe) until Mission 70 modulation reaches −10% (e.g., after a sustained ICP price drop).
3. Call `spawn_neuron` with `percentage_to_spawn = 100` on a neuron whose `maturity_e8s_equivalent = 105_263_158` (just above the 5% threshold: `105_263_158 × 0.95 = 100_000_000`).
4. The guard at line 2666–2668 passes: `least_possible_stake = 100_000_000 ≥ 100_000_000`.
5. Seven days later, `maybe_spawn_neurons` fires, reads Mission 70 modulation = −1_000 permyriad (−10%), and calls `apply_maturity_modulation(105_263_158, -1000)` → `94_736_842`.
6. The spawned neuron is minted with `cached_neuron_stake_e8s = 94_736_842 < 100_000_000 = neuron_minimum_stake_e8s`, violating the minimum-stake invariant.
