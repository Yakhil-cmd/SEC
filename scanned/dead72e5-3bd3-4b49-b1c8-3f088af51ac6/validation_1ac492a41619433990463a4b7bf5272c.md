### Title
Missing `Split` guard in `validate_manage_neuron_proposal` allows followees to split a non-not-for-profit neuron via governance proposal â€” (`rs/nns/governance/src/governance.rs`)

### Summary
`validate_manage_neuron_proposal` enforces that only not-for-profit neurons may be targeted by `Disburse` and `DisburseToNeuron` commands in a ManageNeuron proposal, but omits the same guard for `Command::Split`, contrary to the stated invariant in the code comment.

### Finding Description
In `validate_manage_neuron_proposal`, the block at lines 4688â€“4704 checks `is_managed_neuron_not_for_profit` and returns `NotAuthorized` for `Disburse` and `DisburseToNeuron`. `Split` is not listed and falls through `_ => {}`. [1](#0-0) 

When an adopted ManageNeuron proposal is executed, `perform_action` retrieves the managed neuron's controller and calls `manage_neuron` with that controller as the caller: [2](#0-1) 

This routes to `split_neuron`, whose only authorization check is `is_controlled_by(caller)`: [3](#0-2) 

Since `caller` is the managed neuron's own controller (injected by `perform_action`), this check always passes, and the split executes unconditionally.

### Impact Explanation
A registered followee of a normal (non-not-for-profit) neuron on `Topic::NeuronManagement` can submit a `ManageNeuron { command: Split { amount_e8s } }` proposal. If it passes (requires a majority of the target neuron's NeuronManagement followees), the target neuron's stake is reduced and a new child neuron is created. The child neuron's controller remains the victim's controller (no fund theft), but the victim's voting power is reduced and an unwanted neuron is created with an attacker-chosen `memo` field. The victim must take corrective action to merge or dissolve the child neuron.

### Likelihood Explanation
The attacker must be a registered followee of the victim on `Topic::NeuronManagement` â€” a semi-privileged position that the victim explicitly granted. If the attacker is the sole followee or controls a majority, they can unilaterally execute the split. This is a realistic scenario for neurons that delegate management to a team of followees.

### Recommendation
Add `Command::Split(_)` to the `not_for_profit` guard in `validate_manage_neuron_proposal`, matching the stated invariant in the comment:

```rust
if !is_managed_neuron_not_for_profit {
    match command {
        Command::Disburse(_) => { /* existing error */ }
        Command::DisburseToNeuron(_) => { /* existing error */ }
        Command::Split(_) => {
            return Err(GovernanceError::new_with_message(
                ErrorType::NotAuthorized,
                "Cannot issue a split command through a proposal",
            ));
        }
        _ => {}
    }
}
```
