### Title
O(N²) Instruction Exhaustion in `compute_linear_scaling_coefficients` During SNS Swap Creation — (`File: rs/nns/governance/src/neurons_fund.rs`)

### Summary

`NeuronsFundParticipation::compute_constraints()` calls `compute_linear_scaling_coefficients()`, which the code itself documents as **O(N²) worst-case** with respect to the number of Neurons' Fund neurons. This function executes synchronously during SNS swap proposal finalization. As the Neurons' Fund grows, this computation can exhaust the IC instruction limit for a single message execution, causing the NNS governance canister to trap and permanently blocking SNS swap creation.

### Finding Description

In `rs/nns/governance/src/neurons_fund.rs`, the function `compute_constraints()` is explicitly annotated:

> "The worst-case complexity of this function is O(N^2), where N is the number of neurons in the Neurons' Fund." [1](#0-0) 

This complexity originates inside `compute_linear_scaling_coefficients()`. The function first builds O(N) merged `steps` from eligibility and capping intervals (one step per neuron threshold crossing). Then, for each of those O(N) steps, it:

1. Calls `HalfOpenInterval::find()` **twice** — once over `eligibility_intervals` and once over `capping_intervals`, each of which is a Vec of O(N) elements. If `find` is a linear scan, this is O(N) per step. [2](#0-1) 

2. Iterates over `eligible.neurons` (up to N elements) and for each calls `capped.neurons.contains(neuron)`: [3](#0-2) 

The `neurons` field in each interval is collected from a `Vec` of tuples (line 1460), and `.contains()` on a `Vec` is O(N), making the inner body O(N) per step and the total **O(N²)** — matching the documented complexity. [4](#0-3) 

`compute_constraints()` is invoked from `governance.rs` during SNS swap proposal execution — a synchronous, instruction-metered code path on the NNS governance canister. [5](#0-4) 

### Impact Explanation

The IC instruction limit for a single update-call message execution is **5 billion instructions** (20 billion for system tasks). If the Neurons' Fund grows to thousands of opted-in neurons, the O(N²) computation in `compute_linear_scaling_coefficients` will exceed this limit. The governance canister will trap mid-execution, causing:

- **SNS swap creation to permanently fail** for any proposal that triggers `compute_constraints()`.
- **Governance canister state corruption risk** if the trap occurs after partial state mutation.
- A **denial-of-service on the SNS creation pathway** for the entire network, since all SNS proposals go through the same NNS governance canister.

### Likelihood Explanation

Any NNS neuron holder can opt their neuron into the Neurons' Fund via a standard governance call — this is an unprivileged, permissionless action. As the NNS ecosystem grows and more neuron holders opt in, the Neurons' Fund size N increases organically. No adversarial coordination is required; normal protocol growth is sufficient to trigger the condition. The developers themselves acknowledge the O(N²) complexity but dismiss it as acceptable "in practice" — a judgment that becomes incorrect as N scales. [6](#0-5) 

### Recommendation

1. **Replace the inner `Vec`-based `neurons` field** in `NeuronParticipationInterval` with a `BTreeSet` or `HashMap` to reduce `contains()` from O(N) to O(log N) or O(1), lowering overall complexity to O(N² log N) or O(N log N).
2. **Replace `HalfOpenInterval::find` linear scan** with a binary search over the sorted intervals vector, reducing each lookup from O(N) to O(log N).
3. **Add an enforced upper bound** on the number of Neurons' Fund neurons that can participate in a single SNS swap computation, analogous to the per-cell participant limit recommended in the Revolver report.
4. **Add an instruction-count guard** before calling `compute_constraints()` and split the computation across multiple heartbeat/timer ticks if N exceeds a safe threshold.

### Proof of Concept

**Setup:** Register N NNS neurons (e.g., N = 5,000) and opt each into the Neurons' Fund via `manage_neuron` → `JoinCommunityFund`. This is permissionless for any neuron holder.

**Trigger:** Submit an SNS swap proposal. During finalization, the NNS governance canister calls:

```
NeuronsFundParticipation::compute_constraints()
  → compute_linear_scaling_coefficients()
    → for each of O(N) steps:
        HalfOpenInterval::find(&eligibility_intervals, ...)  // O(N) linear scan
        HalfOpenInterval::find(&capping_intervals, ...)      // O(N) linear scan
        eligible.neurons.iter().filter_map(|n| {
            capped.neurons.contains(n)                       // O(N) Vec scan
        })
```

**Result:** Total instruction count ≈ C × N² where C is the per-iteration cost. At N = 5,000 and C ≈ 200 instructions per inner iteration, this yields ~5 × 10⁹ instructions — at or beyond the single-message instruction limit — causing the governance canister to trap and the SNS swap creation to fail. [7](#0-6)

    fn compute_linear_scaling_coefficients(&self) -> Result<Vec<LinearScalingCoefficient>, String> {
        let min_participant_icp =
            rescale_to_icp(self.swap_participation_limits.min_participant_icp_e8s)?;
        let max_participant_icp =
            rescale_to_icp(self.swap_participation_limits.max_participant_icp_e8s)?;
        let eligibility_intervals = self
            .compute_neuron_partition_intervals(min_participant_icp)
            .map_err(|err| format!("Error while computing eligibility intervals: {err}"))?;
        let capping_intervals = self
            .compute_neuron_partition_intervals(max_participant_icp)
            .map_err(|err| format!("Error while computing capping intervals: {err}"))?;
        // Merge all steps into a single vector, removing duplicates (a duplicate step occurs if
        // a neuron becomes eligible exactly exactly when another neuron becomes capped).
        // First, merge the steps from `eligibility_intervals` and `capping_intervals` and sort them.
        let steps: BTreeSet<u64> = eligibility_intervals
            .iter()
            .map(|interval| interval.from_direct_participation_icp_e8s)
            .chain(
                capping_intervals
                    .iter()
                    .map(|interval| interval.from_direct_participation_icp_e8s),
            )
            .collect();
        // Second, pre-compute the upper bounds for the intervals defined by the merged steps.
        let upper_bounds: Vec<u64> = steps
            .iter()
            .skip(1) // 2nd element is the upper bound of the 1st, etc.
            .chain(std::iter::once(&u64::MAX)) // ensure the iterator has the right legnth
            .cloned()
            .collect();
        // Finally, form the ultimate linear scaling coefficient intervals.
        let slope_denominator = Some(self.total_maturity_equivalent_icp_e8s);
        let linear_scaling_coefficients = steps
            .into_iter()
            .zip(upper_bounds)
            .map(
                |(from_direct_participation_icp_e8s, to_direct_participation_icp_e8s)| {
                    // This search should not fail unless there is a bug, as
                    // `from_direct_participation_icp_e8s` comes either from `eligibility_intervals`
                    // or `capping_intervals`, which we merged to create `steps`.
                    let eligible = HalfOpenInterval::find(
                        &eligibility_intervals,
                        from_direct_participation_icp_e8s,
                    )
                    .ok_or_else(|| {
                        format!(
                            "Cannot find the set of eligible neurons for \
                        direct_participation_icp_e8s in [{from_direct_participation_icp_e8s}, {to_direct_participation_icp_e8s})"
                        )
                    })?;
                    let capped = HalfOpenInterval::find(
                        &capping_intervals,
                        from_direct_participation_icp_e8s,
                    )
                    .ok_or_else(|| {
                        format!(
                            "Cannot find the set of capped neurons for \
                        direct_participation_icp_e8s in [{from_direct_participation_icp_e8s}, {to_direct_participation_icp_e8s})"
                        )
                    })?;
                    let intercept_icp_e8s = Some(
                        (capped.neurons.len() as u64)
                            .saturating_mul(self.swap_participation_limits.max_participant_icp_e8s),
                    );
                    let slope_numerator = Some(
                        eligible
                            .neurons
                            .iter()
                            .filter_map(|neuron| {
                                if capped.neurons.contains(neuron) {
                                    None
                                } else {
                                    Some(neuron.1) // maturity_equivalent_icp_e8s
                                }
                            })
                            .fold(0_u64, |sum, maturity_equivalent_icp_e8s| {
                                sum.saturating_add(maturity_equivalent_icp_e8s)
                            }),
                    );
                    Ok(LinearScalingCoefficient {
                        from_direct_participation_icp_e8s: Some(from_direct_participation_icp_e8s),
                        to_direct_participation_icp_e8s: Some(to_direct_participation_icp_e8s),
                        slope_numerator,
                        slope_denominator,
                        intercept_icp_e8s,
                    })
                },
            )
            .collect::<Result<Vec<_>, String>>()?;
        Ok(linear_scaling_coefficients)
    }
