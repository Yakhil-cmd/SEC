### Title
SNS Governance Remains in `PreInitializationSwap` Mode After Aborted Swap, Permanently Blocking Neuron Disbursal - (`rs/sns/governance/src/governance.rs`, `rs/sns/governance/src/types.rs`)

### Summary

When an SNS decentralization swap is aborted, the SNS Governance canister remains permanently in `PreInitializationSwap` mode. In this mode, `Disburse`, `DisburseMaturity`, `Split`, `Configure`, and `MergeMaturity` neuron commands are unconditionally blocked. The only entity that can transition governance out of this mode is the Swap canister (via `set_mode`), but the Swap canister only calls `set_mode` on the **committed** path — not the aborted path. This creates a state where SNS neuron holders whose swap was aborted cannot disburse their staked tokens, analogous to the Reserve Protocol bug where users could not redeem rTokens when the basket was disabled and the protocol was paused.

### Finding Description

The SNS Governance canister has two modes: `Normal` and `PreInitializationSwap`. When an SNS is created and a decentralization swap begins, governance is placed in `PreInitializationSwap` mode. In this mode, `allows_manage_neuron_command_or_err` blocks all commands except `Follow`, `MakeProposal`, `RegisterVote`, `AddNeuronPermissions`, `RemoveNeuronPermissions`, and (only for the swap canister) `ClaimOrRefresh`. [1](#0-0) 

This means `Disburse`, `DisburseMaturity`, `Split`, `Configure` (start/stop dissolving), and `MergeMaturity` are all blocked for any caller, including neuron owners. [2](#0-1) 

The only way to exit `PreInitializationSwap` mode is via `set_mode`, which is restricted exclusively to the Swap canister: [3](#0-2) 

The Swap canister calls `set_sns_governance_to_normal_mode` only on the **committed** finalization path: [4](#0-3) 

On the **aborted** path, `finalize_inner` returns early after `restore_dapp_controllers_for_finalize` without ever calling `set_mode`: [5](#0-4) 

This is confirmed by integration tests that explicitly assert governance remains in `PreInitializationSwap` mode after an aborted swap: [6](#0-5) 

And that `start_dissolving` (a `Configure` command) is blocked after an aborted swap: [7](#0-6) 

### Impact Explanation

SNS neuron holders who staked tokens before a decentralization swap that subsequently aborts are permanently unable to:
- Start dissolving their neurons (`Configure`)
- Disburse dissolved neurons (`Disburse`)
- Disburse maturity (`DisburseMaturity`)
- Split neurons (`Split`)

Their staked SNS tokens are permanently locked in the governance canister with no user-accessible path to recover them. The only recovery path requires an NNS governance proposal to upgrade the SNS Governance canister — a privileged action that users cannot trigger themselves. This is a permanent, irreversible lock of user funds contingent on governance action, matching the severity class of the reference bug.

### Likelihood Explanation

This is a **confirmed, deterministic** state that occurs every time an SNS decentralization swap is aborted. The integration test suite explicitly asserts this behavior as expected. Any SNS whose swap fails to reach minimum participation will leave all pre-existing neuron holders (developer neurons, airdrop neurons) permanently unable to manage their neurons without an NNS upgrade proposal. This is reachable by any unprivileged user who holds an SNS neuron and whose SNS swap was aborted — no special access is required to be affected.

### Recommendation

In `finalize_inner` in `rs/sns/swap/src/swap.rs`, on the aborted path (inside `should_restore_dapp_control()`), call `set_sns_governance_to_normal_mode` before returning, so that governance mode is reset to `Normal` regardless of whether the swap committed or aborted:

```rust
if self.should_restore_dapp_control() {
    finalize_swap_response.set_set_dapp_controllers_result(
        self.restore_dapp_controllers_for_finalize(environment.sns_root_mut()).await,
    );
    // Add: reset governance mode to Normal on abort path too
    finalize_swap_response.set_set_mode_call_result(
        Self::set_sns_governance_to_normal_mode(environment.sns_governance_mut()).await,
    );
    return finalize_swap_response;
}
```

Alternatively, `set_mode` in `rs/sns/governance/src/governance.rs` could be made callable by anyone when the swap is in a terminal state, analogous to the Reserve Protocol fix.

### Proof of Concept

1. An SNS is created; governance enters `PreInitializationSwap` mode.
2. A decentralization swap opens and closes without reaching `min_participants` → swap transitions to `Aborted`.
3. `finalize` is called (or auto-finalized). The aborted path calls `restore_dapp_controllers_for_finalize` and returns **without** calling `set_sns_governance_to_normal_mode`.
4. Governance mode remains `PreInitializationSwap`.
5. A developer neuron holder calls `manage_neuron` with `Command::Configure(StartDissolving)` or `Command::Disburse(...)`.
6. `allows_manage_neuron_command_or_err` returns `Err(PreconditionFailed, "Because governance is currently in PreInitializationSwap mode, manage_neuron commands of this type are not allowed")`.
7. The neuron holder's tokens are permanently locked with no user-accessible recovery path.

This is confirmed by the existing test assertion at: [8](#0-7) 

which documents `{ governance::Mode::PreInitializationSwap } FinalizeUnSuccessfully { governance::Mode::PreInitializationSwap }` and `{ !DissolveSnsNeuron.is_enabled() } FinalizeUnSuccessfully { !DissolveSnsNeuron.is_enabled() }` as the current (broken) invariants.
