### Title
`spawn_neuron` Uses Stale -5% Worst-Case Modulation Instead of Mission 70 -10% Bound — (File: `rs/nns/governance/src/governance.rs`)

### Summary

The `spawn_neuron` function in NNS Governance validates whether a neuron has sufficient maturity to spawn by applying a hardcoded **-5%** worst-case modulation. Under Mission 70, the actual minimum maturity modulation is **-10%** (`MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 = -1_000` permyriad). This stale constant allows neurons with maturity in the range `[neuron_minimum_stake_e8s / 0.90, neuron_minimum_stake_e8s / 0.95)` to pass the spawn eligibility check, but when `maybe_spawn_neurons` later applies the real Mission 70 modulation (which can reach -10%), the minted stake falls below `neuron_minimum_stake_e8s`, creating an under-minimum neuron.

### Finding Description

In `spawn_neuron`, the pre-spawn eligibility check is:

```rust
// rs/nns/governance/src/governance.rs:2664-2673
let least_possible_stake = (maturity_to_spawn as f64 * (1_f64 - 0.05)) as u64;

if least_possible_stake < economics.neuron_minimum_stake_e8s {
    return Err(GovernanceError::new_with_message(
        ErrorType::InsufficientFunds,
        "There isn't enough maturity to spawn a new neuron due to worst case maturity modulation.",
    ));
}
```

The literal `0.05` corresponds to the **old** ±5% modulation range (`MIN_MATURITY_MODULATION_PERMYRIAD = -500`). However, Mission 70 introduced a new, wider range:

```rust
// rs/nns/governance/src/timer_tasks/update_icp_xdr_rate_related_data.rs:47-50
/// Lower bound for Mission 70 maturity modulation: -10% = -1000 permyriad.
pub(crate) const MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70: i64 = -1_000;

/// Upper bound for Mission 70 maturity modulation: +2% = 200 permyriad.
pub(crate) const MATURITY_MODULATION_MAX_PERMYRIAD_MISSION_70: i64 = 200;
```

The `VALID_MATURITY_MODULATION_BASIS_POINTS_RANGE` used in `maybe_spawn_neurons` is already updated to the Mission 70 bounds:

```rust
// rs/nns/governance/src/governance.rs:276-278
const VALID_MATURITY_MODULATION_BASIS_POINTS_RANGE: RangeInclusive<i32> =
    MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 as i32
        ..=MATURITY_MODULATION_MAX_PERMYRIAD_MISSION_70 as i32;
```

And `maybe_spawn_neurons` reads the live Mission 70 modulation from `heap_data.maturity_modulation`:

```rust
// rs/nns/governance/src/governance.rs:6427-6435
let maturity_modulation = match self
    .heap_data
    .maturity_modulation
    .as_ref()
    .and_then(|m| m.current_value_permyriad)
{
    None => return,
    Some(value) => value,
};
```

And applies it directly:

```rust
// rs/nns/governance/src/governance.rs:6484-6487
let neuron_stake: u64 = match apply_maturity_modulation(
    original_maturity,
    maturity_modulation,
) {
```

So `spawn_neuron` guards with -5% but `maybe_spawn_neurons` can apply up to -10%, breaking the invariant that spawned neurons always meet `neuron_minimum_stake_e8s`.

### Impact Explanation

A neuron owner whose maturity M satisfies:

```
neuron_minimum_stake_e8s / 0.95 > M >= neuron_minimum_stake_e8s / 0.90
```

can successfully call `spawn_neuron` (the -5% check passes), creating a child neuron in spawning state. When `maybe_spawn_neurons` fires with a Mission 70 modulation of -10%, the minted stake is `M * 0.90 < neuron_minimum_stake_e8s`. The resulting neuron holds ICP below the protocol minimum, violating the ledger conservation invariant that all neurons meet the minimum stake requirement. This is a **ledger conservation bug** — incorrect ICP accounting due to using the wrong worst-case value.

**Impact: Medium** — Creates neurons with below-minimum stake, which can break governance operations (voting, dissolving, merging) that enforce the minimum stake invariant.

### Likelihood Explanation

**Likelihood: Medium** — Requires Mission 70 to be active (it is, per the changelog dated 2026-05-17) and the daily maturity modulation to be at or near -10%. The modulation is computed from real ICP/XDR price history and can legitimately reach -10% during sustained ICP price declines. Any neuron owner with maturity in the vulnerable range can trigger this by calling `manage_neuron` with a `Spawn` command — no privileged access required.

### Recommendation

Replace the hardcoded `0.05` in `spawn_neuron` with the Mission 70 constant:

```rust
// Before (stale):
let least_possible_stake = (maturity_to_spawn as f64 * (1_f64 - 0.05)) as u64;

// After (correct):
use ic_nervous_system_governance::maturity_modulation::apply_maturity_modulation;
use crate::timer_tasks::update_icp_xdr_rate_related_data::MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70;

let least_possible_stake = apply_maturity_modulation(
    maturity_to_spawn,
    MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 as i32,
).unwrap_or(0);
```

This mirrors the approach already used in SNS governance's `disburse_maturity`, which correctly applies `MIN_MATURITY_MODULATION_PERMYRIAD` to compute the worst-case amount before checking against the transaction fee.

use std::cmp::Ordering;

/// The maturity modulation range in basis points.
pub const MIN_MATURITY_MODULATION_PERMYRIAD: i32 = -500;
pub const MAX_MATURITY_MODULATION_PERMYRIAD: i32 = 500;

pub const BASIS_POINTS_PER_UNITY: u128 = 10_000;

/// Modulate amount_e8s. That is, multiply by 1 + X where
/// X = maturity_modulation_basis_points / 10_000.
pub fn apply_maturity_modulation(
    amount_maturity_e8s: u64,
    maturity_modulation_basis_points: i32,
) -> Result<u64, String> {
    let amount_e8s = u128::from(amount_maturity_e8s);

    let adjusted_maturity_modulation_basis_points = saturating_add_or_subtract_u128_i32(
        BASIS_POINTS_PER_UNITY,
        maturity_modulation_basis_points,
    );

    let modulated_amount_e8s: u128 = amount_e8s
        .checked_mul(adjusted_maturity_modulation_basis_points)
        .ok_or_else(|| "Underflow or overflow when calculating maturity modulation".to_string())?
        .checked_div(BASIS_POINTS_PER_UNITY)
        .ok_or_else(|| "Underflow or overflow when calculating maturity modulation".to_string())?;

    u64::try_from(modulated_amount_e8s).map_err(|err| err.to_string())
}
