### Title
Delegation Certificate Timestamp Not Validated in `verify_delegation_certificate` — (File: `rs/certification/src/lib.rs`)

### Summary
The `verify_delegation_certificate` function in `rs/certification/src/lib.rs` parses the `time` field from the NNS delegation certificate's state tree but explicitly skips any staleness check against the current time. This is a direct analog to the external report's finding: just as the Automata PCCS contract accepts CRLs without checking their `nextUpdate` expiry, the IC's delegation certificate verifier accepts delegation certificates without checking whether their certified timestamp is recent.

### Finding Description

Inside `verify_delegation_certificate`, the struct used to deserialize the delegation certificate's state tree is:

```rust
struct SubnetCertificateData {
    #[allow(unused)] // currently delegation timestamps are not checked
    time: Leb128EncodedU64,
    subnet: BTreeMap<SubnetId, SubnetView>,
    canister_ranges: Option<BTreeMap<SubnetId, TreeCanisterRanges>>,
}
``` [1](#0-0) 

The `time` field is required to be present (deserialization fails if it is absent), but its value is never compared against any current-time bound. After deserialization, the function proceeds directly to canister-range and public-key checks, returning the subnet's public key without any staleness gate. [2](#0-1) 

This function is the single verification path used by:

1. **The NNS delegation manager** (`rs/http_endpoints/nns_delegation_manager/src/nns_delegation_manager.rs`) when it validates a freshly fetched delegation from the NNS before caching it. [3](#0-2) 

2. **All `read_state` response verifiers** via `verify_certificate_internal` → `verify_delegation_certificate`, used by both the replica's public HTTP endpoint and external clients (e.g., `ic-agent`, wallets, dApps). [4](#0-3) 

3. **Canister signature verification** via `verify_certified_data_with_cache_for_canister_sig`, which calls the same delegation verification path. [5](#0-4) 

The external `packages/ic-signature-verification` crate's `verify_delegation` function also omits any time check on the delegation certificate. [6](#0-5) 

### Impact Explanation

A delegation certificate embeds the application subnet's current threshold public key. If the subnet rotates its threshold key (e.g., after a security incident), the NNS issues a new delegation certificate containing the new key. Because the `time` field is never validated:

- A node serving a **stale delegation certificate** (one whose certified time predates the key rotation) will have that certificate accepted by any verifier calling `verify_delegation_certificate`.
- If an attacker has obtained the old subnet threshold key (requiring compromise of a threshold-majority of the old subnet nodes), they can pair the stale delegation certificate with a forged state tree signed under the old key. Clients and canister-signature verifiers would accept this as a valid certified state response, enabling **state certification forgery**.
- Even without key compromise, a malicious node can serve a stale delegation to cause clients to attempt verification against an outdated public key, resulting in a **denial-of-service** for `read_state` and canister-signature consumers.

### Likelihood Explanation

The NNS delegation manager refreshes the delegation every 5 minutes in production: [7](#0-6) 

This limits the practical window for stale-delegation attacks in the normal case. However, the full state-forgery impact requires threshold-majority key compromise, which is a high bar. The denial-of-service path (serving a stale delegation after a key rotation) is reachable by a single malicious node without any key material. The code comment `// currently delegation timestamps are not checked` confirms this is a known, unmitigated gap.

### Recommendation

- **Short term**: In `verify_delegation_certificate`, compare `subnet_state.time` against a caller-supplied `current_time` parameter and reject delegations whose certified time is older than a configurable maximum staleness bound (e.g., matching the `DELEGATION_UPDATE_INTERVAL` of 5 minutes).
- **Long term**: Propagate a `current_time` argument through all callers of `verify_delegation_certificate` (including `verify_certificate`, `verify_certified_data`, and the external `ic-signature-verification` crate) so that staleness enforcement is uniform across all verification paths.

### Proof of Concept

1. Obtain a valid NNS delegation certificate `D_old` for application subnet `S` at time `T_old`, containing subnet public key `PK_old`.
2. Trigger or wait for a subnet key rotation: the NNS issues `D_new` with `PK_new`.
3. A node on subnet `S` continues to serve `D_old` in `read_state` responses.
4. Any client calling `verify_delegation_certificate(&D_old, &subnet_id, &root_pk, ...)` will succeed — the `time` field `T_old` is parsed but never compared to the current time.
5. The client proceeds to verify the state tree signature against `PK_old`. If the node also serves a state tree signed under `PK_old` (possible only with the old threshold key), the client accepts a forged certified state. [8](#0-7)

fn verify_certificate_internal(
    certificate: &[u8],
    canister_id: &CanisterId,
    root_pk: &ThresholdSigPublicKey,
    use_signature_cache: bool,
) -> Result<(Certificate, Option<DelegationSubnetInfo>), CertificateValidationError> {
    let certificate: Certificate = parse_certificate(certificate)?;
    let (key, delegation_info) = if let Some(delegation) = &certificate.delegation {
        let subnet_id = PrincipalId::try_from(&*delegation.subnet_id)
            .map(SubnetId::from)

                CertificateValidationError::DeserError(format!(
                    "failed to parse delegation subnet id: {err}"
                ))
            })?;

            &delegation.certificate,
            &subnet_id,
            root_pk,
            Some(canister_id),
            use_signature_cache,
        )?;
        (key, Some(info))
    } else {
        (*root_pk, None)
    };

    verify_certificate_signature(&certificate, &key, use_signature_cache)?;
    Ok((certificate, delegation_info))

fn verify_delegation(
    delegation: &Delegation,
    signing_canister_id: Principal,
    root_public_key: &[u8],
) -> Result<Vec<u8>, String> {
    let cert: Certificate = parse_certificate_cbor(&delegation.certificate)
        .map_err(|e| format!("invalid delegation certificate: {e}"))?;

    // disallow nested delegations
    if cert.delegation.is_some() {
        return Err("multiple delegations not allowed".to_string());
    }

    check_bls_signature(&cert, root_public_key)?;

    // Try new structure first: /canister_ranges/<subnet_id>/<range_key>
    let canister_ranges_path = ["canister_ranges".as_bytes(), delegation.subnet_id.as_ref()];
    let canister_in_ranges = match cert.tree.lookup_subtree(&canister_ranges_path) {
        SubtreeLookupResult::Found(subnet_tree) => {
            match find_leaf_for_principal(subnet_tree.as_ref(), signing_canister_id.as_slice()) {
                Some(range_data) => {
                    let subnet_ranges: Vec<(Principal, Principal)> =
                        serde_cbor::from_slice(range_data)
                            .map_err(|e| format!("invalid canister range: {e}"))?;
                    principal_is_within_ranges(&signing_canister_id, &subnet_ranges)
                }
                None => false,
            }
        }
        SubtreeLookupResult::Absent | SubtreeLookupResult::Unknown => {
            let old_canister_ranges_path = [
                "subnet".as_bytes(),
                delegation.subnet_id.as_ref(),
                "canister_ranges".as_bytes(),
            ];
            match cert.tree.lookup_path(&old_canister_ranges_path) {
                LookupResult::Found(old_range_data) => {
                    let canister_ranges: Vec<(Principal, Principal)> =
                        serde_cbor::from_slice(old_range_data)
                            .map_err(|e| format!("invalid canister range: {e}"))?;
                    principal_is_within_ranges(&signing_canister_id, &canister_ranges)
                }
                _ => {
                    return Err("canister_ranges-entry not found".to_string());
                }
            }
        }
    };

    if !canister_in_ranges {
        return Err("signing canister id not in canister_ranges".to_string());
    }

    // lookup the public key delegated to
    let public_key_path = [
        "subnet".as_bytes(),
        delegation.subnet_id.as_ref(),
        "public_key".as_bytes(),
    ];
    let LookupResult::Found(subnet_public_key_der) = cert.tree.lookup_path(&public_key_path) else {
        return Err("subnet public key not found".to_string());
    };
    extract_raw_root_pk_from_der(subnet_public_key_der)
}
