### Title
Missing User-Supplied Deadline in `withdraw_eth` and `withdraw_erc20` Chain-Fusion Withdrawal Functions - (File: `rs/ethereum/cketh/minter/src/main.rs`)

### Summary
The `withdraw_eth` and `withdraw_erc20` functions in the ckETH minter canister accept no user-supplied deadline parameter. The user's ckETH/ckERC20 tokens are burned immediately and irrevocably via `transfer_from` (with `created_at_time: None`, explicitly disabling deduplication), but the actual Ethereum transaction is only queued and processed asynchronously. There is no mechanism for the user to bound the time window within which the Ethereum transaction must be submitted, mirroring the exact pattern flagged in the external report.

### Finding Description
`WithdrawalArg` (used by `withdraw_eth`) and `WithdrawErc20Arg` (used by `withdraw_erc20`) contain no `deadline` field:

```rust
// rs/ethereum/cketh/minter/src/main.rs
pub struct WithdrawalArg {
    pub amount: ...,
    pub recipient: String,
    pub from_subaccount: Option<Subaccount>,
    // no deadline
}

pub struct WithdrawErc20Arg {
    pub amount: Nat,
    pub ckerc20_ledger_id: Principal,
    pub recipient: String,
    pub from_cketh_subaccount: Option<Subaccount>,
    pub from_ckerc20_subaccount: Option<Subaccount>,
    // no deadline
}
```

Inside `withdraw_eth`, the burn is executed immediately and the request is queued:

```rust
let now = ic_cdk::api::time();
// burn happens here synchronously
match client.burn_from(Account { owner: caller, subaccount: from_subaccount }, amount, ...).await {
    Ok(ledger_burn_index) => {
        let withdrawal_request = EthWithdrawalRequest {
            ...
            created_at: Some(now),   // only records creation time, no deadline
        };
        mutate_state(|s| { process_event(s, EventType::AcceptedEthWithdrawalRequest(...)); });
    }
}
```

The `burn_from` call in `LedgerClient` explicitly sets `created_at_time: None` to disable ICRC-1 deduplication:

```rust
// rs/ethereum/cketh/minter/src/ledger_client.rs
created_at_time: None, // We don't set this field to disable transaction deduplication
```

Once the burn succeeds, the withdrawal is enqueued in the minter's internal state and processed by a background task. The user has no way to express "execute this only if the Ethereum transaction is submitted before timestamp T."

The same pattern applies to `withdraw_erc20`: both the ckETH gas-fee burn and the ckERC20 principal burn happen immediately, but the Ethereum transaction is deferred. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4) 

### Impact Explanation
Once `withdraw_eth` or `withdraw_erc20` is called:

1. The user's ckETH/ckERC20 tokens are **immediately and irrevocably burned**.
2. The withdrawal request is placed in the minter's pending queue.
3. The minter processes the queue asynchronously; under congestion the delay is unbounded from the user's perspective.
4. The user **cannot cancel** a queued withdrawal.
5. The gas-fee estimate (`erc20_tx_fee`) is captured at call time, but the Ethereum transaction may be submitted much later under very different gas-price or token-price conditions.
6. If the Ethereum transaction ultimately fails (e.g., gas underpriced after a spike), the user receives a reimbursement **minus a penalty fee** (`CKETH_LEDGER_TRANSACTION_FEE`), not a full refund.

A user who intended to withdraw at a specific market price has no on-chain guarantee that the transaction will be submitted within any bounded window. This is the direct IC analog of the `withdraw()` function in `Enigma.sol` having no deadline: the position may no longer be profitable by the time the transaction is executed. [6](#0-5) [7](#0-6) 

### Likelihood Explanation
Any unprivileged principal can call `withdraw_eth` or `withdraw_erc20` directly via ingress. The minter's pending-request queue can grow up to `MAX_CONCURRENT_PENDING_REQUESTS`; under sustained load, newly accepted requests sit at the back of the queue for an indeterminate period. No special privilege, key compromise, or governance majority is required to trigger the scenario — the user's own call is the entry point. [8](#0-7) 

### Recommendation

1. Add an optional `deadline: Option<u64>` (nanoseconds since Unix epoch) to `WithdrawalArg` and `WithdrawErc20Arg`.
2. At the start of `withdraw_eth` / `withdraw_erc20`, before any burn, check:
   ```rust
   if let Some(dl) = args.deadline {
       if ic_cdk::api::time() > dl {
           return Err(WithdrawalError::DeadlineExpired);
       }
   }
   ```
3. Optionally, store the deadline in `EthWithdrawalRequest` / `Erc20WithdrawalRequest` and have the background processing task skip (and reimburse) requests whose deadline has passed before the Ethereum transaction is submitted. [9](#0-8) 

### Proof of Concept

1. ETH spot price is $3 000. User calls `withdraw_eth` for 10 ckETH, intending to receive 10 ETH at current prices.
2. The ckETH burn succeeds immediately; `EthWithdrawalRequest` is enqueued.
3. The minter queue is congested (e.g., 500 pending requests ahead of this one).
4. Over the next several hours the ETH price drops to $2 000.
5. The minter eventually submits the Ethereum transaction; the user receives 10 ETH now worth $20 000 instead of the $30 000 they expected.
6. The user had no way to express "cancel this withdrawal if it is not submitted within 30 minutes."

The same scenario applies to `withdraw_erc20` where both ckETH (gas) and ckERC20 tokens are burned at call time but the ERC-20 transfer on Ethereum is deferred. [10](#0-9)

async fn withdraw_eth(
    WithdrawalArg {
        amount,
        recipient,
        from_subaccount,
    }: WithdrawalArg,
) -> Result<RetrieveEthRequest, WithdrawalError> {
    let caller = validate_caller_not_anonymous();
    let _guard = retrieve_withdraw_guard(caller).unwrap_or_else(|e| {
        ic_cdk::trap(format!(
            "Failed retrieving guard for principal {caller}: {e:?}"
        ))
    });

    let destination = validate_address_as_destination(&recipient).map_err(|e| match e {
        AddressValidationError::Invalid { .. } | AddressValidationError::NotSupported(_) => {
            ic_cdk::trap(e.to_string())
        }
        AddressValidationError::Blocked(address) => WithdrawalError::RecipientAddressBlocked {
            address: address.to_string(),
        },
    })?;

    let amount = Wei::try_from(amount).expect("failed to convert Nat to u256");

    let minimum_withdrawal_amount = read_state(|s| s.cketh_minimum_withdrawal_amount);
    if amount < minimum_withdrawal_amount {
        return Err(WithdrawalError::AmountTooLow {
            min_withdrawal_amount: minimum_withdrawal_amount.into(),
        });
    }

    let client = read_state(LedgerClient::cketh_ledger_from_state);
    let now = ic_cdk::api::time();
    log!(INFO, "[withdraw]: burning {:?}", amount);
    match client
        .burn_from(
            Account {
                owner: caller,
                subaccount: from_subaccount,
            },
            amount,
            BurnMemo::Convert {
                to_address: destination,
            },
        )
        .await
    {
        Ok(ledger_burn_index) => {
            let withdrawal_request = EthWithdrawalRequest {
                withdrawal_amount: amount,
                destination,
                ledger_burn_index,
                from: caller,
                from_subaccount: from_subaccount.and_then(LedgerSubaccount::from_bytes),
                created_at: Some(now),
            };

            log!(
                INFO,
                "[withdraw]: queuing withdrawal request {:?}",
                withdrawal_request,
            );

            mutate_state(|s| {
                process_event(
                    s,
                    EventType::AcceptedEthWithdrawalRequest(withdrawal_request.clone()),
                );
            });
            Ok(RetrieveEthRequest::from(withdrawal_request))
        }
        Err(e) => Err(WithdrawalError::from(e)),
    }
}

                        destination,
                        cketh_ledger_burn_index,
                        ckerc20_ledger_id: ckerc20_token.ckerc20_ledger_id,

                        erc20_contract_address: ckerc20_token.erc20_contract_address,
                        from: caller,
                        from_subaccount: from_ckerc20_subaccount
                            .and_then(LedgerSubaccount::from_bytes),
                        created_at: now,
                    };
                    log!(
                        INFO,
                        "[withdraw_erc20]: queuing withdrawal request {:?}",
                        withdrawal_request
                    );
                    mutate_state(|s| {
                        process_event(

                            EventType::AcceptedErc20WithdrawalRequest(withdrawal_request.clone()),
                        );
                    });
                    Ok(RetrieveErc20Request::from(withdrawal_request))
                }
                Err(ckerc20_burn_error) => {
                    let reimbursed_amount = match &ckerc20_burn_error {
                        LedgerBurnError::TemporarilyUnavailable { .. } => erc20_tx_fee, //don't penalize user in case of an error outside of their control
                        LedgerBurnError::InsufficientFunds { .. }
                        | LedgerBurnError::AmountTooLow { .. }
                        | LedgerBurnError::InsufficientAllowance { .. } => erc20_tx_fee
                            .checked_sub(CKETH_LEDGER_TRANSACTION_FEE)
                            .unwrap_or(Wei::ZERO),
                    };
                    if reimbursed_amount > Wei::ZERO {
                        let reimbursement_request = ReimbursementRequest {
                            ledger_burn_index: cketh_ledger_burn_index,
                            reimbursed_amount: reimbursed_amount.change_units(),
                            to: cketh_account.owner,
                            to_subaccount: cketh_account
                                .subaccount
                                .and_then(LedgerSubaccount::from_bytes),
                            transaction_hash: None,
                        };
                        mutate_state(|s| {
                            process_event(
                                s,
                                EventType::FailedErc20WithdrawalRequest(reimbursement_request),
                            );
                        });
                    }
                    Err(WithdrawErc20Error::CkErc20LedgerError {
                        cketh_block_index: Nat::from(cketh_ledger_burn_index.get()),
                        error: ckerc20_burn_error.into(),
                    })
                }
            }
        }
        Err(cketh_burn_error) => Err(WithdrawErc20Error::CkEthLedgerError {
            error: cketh_burn_error.into(),
        }),
    }
}

type WithdrawErc20Arg = record {
    // Amount of tokens to withdraw.
    // The amount is in the smallest unit of the token, e.g.,

    amount : nat;

    // Ethereum address to withdraw to.
    recipient : text;

    from_cketh_subaccount : opt Subaccount;
