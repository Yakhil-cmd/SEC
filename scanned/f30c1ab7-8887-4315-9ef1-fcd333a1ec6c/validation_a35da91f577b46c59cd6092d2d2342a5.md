### Title
Missing Caller Authorization on `swap_node_in_subnet_directly` and `migrate_node_operator_directly` Registry Update Methods - (File: `rs/registry/canister/canister/canister.rs`)

### Summary
Two state-modifying update methods on the Registry canister — `swap_node_in_subnet_directly` and `migrate_node_operator_directly` — are exposed publicly with **no caller authorization check**. Any unprivileged ingress sender or canister can invoke them directly, bypassing the NNS governance proposal process that is required for all equivalent privileged registry mutations.

### Finding Description
The Registry canister exposes a consistent pattern for privileged mutations: every state-modifying update method calls a guard function (`check_caller_is_governance_and_log`, `check_caller_is_governance_or_engine_controller_and_log`, etc.) before executing the mutation and recertifying the registry. [1](#0-0) 

For example, `update_node_operator_config`, `remove_node_operators`, `set_firewall_config`, and dozens of other methods all enforce this pattern: [2](#0-1) [3](#0-2) 

However, two methods are completely missing this guard:

**`swap_node_in_subnet_directly`** (lines 831–841):
```rust
#[unsafe(export_name = "canister_update swap_node_in_subnet_directly")]
fn swap_node_in_subnet_directly() {
    over(candid_one, |payload: SwapNodeInSubnetDirectlyPayload| {
        swap_node_in_subnet_directly_(payload)
    });
}

fn swap_node_in_subnet_directly_(payload: SwapNodeInSubnetDirectlyPayload) {
    registry_mut().do_swap_node_in_subnet_directly(payload);
    recertify_registry();
}
``` [4](#0-3) 

**`migrate_node_operator_directly`** (lines 844–854):
```rust
#[unsafe(export_name = "canister_update migrate_node_operator_directly")]
fn migrate_node_operator_directly() {
    over(candid_one, |payload: MigrateNodeOperatorPayload| {
        migrate_node_operator_directly_(payload)
    });
}

fn migrate_node_operator_directly_(payload: MigrateNodeOperatorPayload) {
    registry_mut().do_migrate_node_operator_directly(payload);
    recertify_registry();
}
``` [5](#0-4) 

Neither function calls any authorization guard. Both directly mutate the registry and call `recertify_registry()`, making the unauthorized state change part of the certified registry state visible to all clients.

Note that `update_node_operator_config_directly` carries a comment "This method can be called by anyone" but has an internal business-logic check (caller must be the node provider of the record). The two methods above have **no internal check of any kind**. [6](#0-5) 

Both methods are listed in the public Candid interface: [7](#0-6) [8](#0-7) 

### Impact Explanation
The Registry canister is the authoritative configuration database for the entire IC network — it stores subnet membership, node assignments, node operator records, firewall rules, elected replica versions, and more. All replicas poll it to determine network topology and operational parameters.

- **`swap_node_in_subnet_directly`**: An attacker can arbitrarily reassign nodes between subnets in the registry. This corrupts the subnet membership record that replicas use to form consensus committees. Swapping a node out of a subnet mid-operation can cause that subnet to lose a member below its fault-tolerance threshold, potentially stalling consensus or enabling a targeted degradation of a specific subnet.
- **`migrate_node_operator_directly`**: An attacker can migrate node operator records, reassigning node operator identities and their associated allowances, disrupting node onboarding and reward accounting.

Both mutations are immediately recertified, meaning the corrupted state is signed by the subnet and propagated to all registry clients as authoritative truth.

### Likelihood Explanation
The Registry canister is a well-known NNS system canister with a public Candid interface. Both methods appear in the `.did` file and are callable by any principal — including anonymous users — via standard IC ingress. No special privileges, keys, or social engineering are required. The attacker only needs to know a valid `node_id` and `subnet_id` (both of which are publicly observable from the registry itself via query calls). Likelihood is **High**.

### Recommendation
- **Short term:** Add `check_caller_is_governance_and_log("swap_node_in_subnet_directly")` and `check_caller_is_governance_and_log("migrate_node_operator_directly")` as the first statement in each respective outer handler function, matching the pattern used by all other privileged registry mutations.
- **Long term:** Add integration tests (analogous to `test_non_governance_users_cannot_update_node_operator_config` and `test_the_anonymous_user_cannot_remove_node_operators`) that assert anonymous and non-governance callers are rejected for these two methods. [9](#0-8) [10](#0-9) 

### Proof of Concept
Any unprivileged principal can submit the following ingress call to the Registry canister (`rwlgt-iiaaa-aaaaa-aaaaa-cai`) on mainnet:

```
dfx canister --network ic call rwlgt-iiaaa-aaaaa-aaaaa-cai swap_node_in_subnet_directly \
  '(record { node_id = principal "<victim_node_id>"; subnet_id = principal "<target_subnet_id>" })'
```

No governance proposal is required. The call will execute `do_swap_node_in_subnet_directly`, mutate the registry state, and `recertify_registry()` will sign the new state — making the unauthorized topology change part of the certified IC state.

            node_allowance: 5,

        let registry = set_up_registry_canister(
            &runtime,

                .push_init_mutate_request(invariant_compliant_mutation_as_atomic_req(0))
                .push_init_mutate_request(RegistryAtomicMutateRequest {
                    mutations: vec![RegistryMutation {
                        mutation_type: registry_mutation::Type::Insert as i32,
                        key: node_operator_key.as_bytes().to_vec(),
                        value: node_operator_record.encode_to_vec(),
                    }],
                    preconditions: vec![],
                })
                .build(),
        )
        .await;

        let payload = UpdateNodeOperatorConfigPayload {
            node_operator_id: Some(*TEST_NEURON_1_OWNER_PRINCIPAL),
            node_allowance: Some(10),
            dc_id: None,
            rewardable_nodes: btreemap! {},
            node_provider_id: Some(*TEST_NEURON_2_OWNER_PRINCIPAL),
            ipv6: None,
            set_ipv6_to_none: None,
            max_rewardable_nodes: None,
        };

        // The anonymous end-user tries to update a node operator, bypassing
        // governance. This should be rejected.
        let response: Result<(), String> = registry
            .update_("update_node_operator_config", candid_one, payload.clone())
            .await;

        assert_matches!(response,
                Err(s) if s.contains("is not authorized to call this method: update_node_operator_config"));

        // The attacker canister tries to update the node operator, pretending to be the
        // governance canister. This should have no effect.
        assert!(
            !forward_call_via_universal_canister(
                &attacker_canister,
                &registry,
                "update_node_operator_config",
                Encode!(&payload).unwrap()
            )
            .await
        );

        // .. And there should therefore be no change to the node operator record
        assert_eq!(
            get_value_or_panic::<NodeOperatorRecord>(&registry, node_operator_key.as_bytes()).await,
            node_operator_record
        );

        Ok(())
    });

#[test]
fn test_the_anonymous_user_cannot_remove_node_operators() {
    state_machine_test_on_nns_subnet(|runtime| async move {
        let mut registry = set_up_registry_canister(
            &runtime,
            RegistryCanisterInitPayloadBuilder::new()
                .push_init_mutate_request(invariant_compliant_mutation_as_atomic_req(0))
                .build(),
        )
        .await;

        let payload = RemoveNodeOperatorsPayload::new(vec![]);

        // The anonymous end-user tries to remove node operators, bypassing
        // the Governance canister. This should be rejected.
        let response: Result<(), String> = registry
            .update_("remove_node_operators", candid_one, payload.clone())
            .await;

        assert_matches!(
            response,
            Err(s) if s.contains("is not authorized to call this method: remove_node_operators")
        );

        // Go through an upgrade cycle, and verify that it still works the same
        registry.upgrade_to_self_binary(vec![]).await.unwrap();
        let response: Result<(), String> = registry
            .update_("remove_node_operators", candid_one, payload.clone())
            .await;

        assert_matches!(
            response,
            Err(s) if s.contains("is not authorized to call this method: remove_node_operators")
        );

        Ok(())
    });
}
