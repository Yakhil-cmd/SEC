### Title
`NeuronInfo.stake_e8s` Returns Understated Value — Staked Maturity Omitted from Public Neuron Query - (File: rs/nns/governance/src/neuron/types.rs)

### Summary

The `get_neuron_info()` function in the NNS Governance canister populates the publicly queryable `NeuronInfo.stake_e8s` field using `minted_stake_e8s()`, which computes only `cached_neuron_stake_e8s - neuron_fees_e8s`. The canister's own DID interface specification explicitly documents that `stake_e8s` must equal `cached_neuron_stake_e8s - neuron_fees_e8s + staked_maturity_e8s_equivalent`. The `staked_maturity_e8s_equivalent` component is silently dropped, causing every neuron that has staked maturity to report an understated stake to any caller of `get_neuron_info` or `list_neurons`.

### Finding Description

In `rs/nns/governance/src/neuron/types.rs`, the `get_neuron_info()` method constructs a `NeuronInfo` struct for public consumption:

```rust
NeuronInfo {
    ...
    stake_e8s: self.minted_stake_e8s(),   // ← BUG: omits staked_maturity_e8s_equivalent
    ...
    staked_maturity_e8s_equivalent: self.staked_maturity_e8s_equivalent,
}
``` [1](#0-0) 

`minted_stake_e8s()` is defined as:

```rust
pub fn minted_stake_e8s(&self) -> u64 {
    self.cached_neuron_stake_e8s
        .saturating_sub(self.neuron_fees_e8s)
}
``` [2](#0-1) 

The correct full-stake function, `stake_e8s()`, includes staked maturity:

```rust
pub fn stake_e8s(&self) -> u64 {
    neuron_stake_e8s(
        self.cached_neuron_stake_e8s,
        self.neuron_fees_e8s,
        self.staked_maturity_e8s_equivalent,   // ← this is missing from NeuronInfo
    )
}
``` [3](#0-2) 

The canister's own public DID specification explicitly documents the intended semantics of `stake_e8s`:

```
// The amount of ICP (and staked maturity) locked in this neuron.
// This is the foundation of the neuron's voting power.
// cached_neuron_stake_e8s - neuron_fees_e8s + staked_maturity_e8s_equivalent
stake_e8s : nat64;
``` [4](#0-3) 

The discrepancy is confirmed by the existing test for `get_neuron_info`, which asserts `stake_e8s: 10 * E8` for a neuron with `cached_neuron_stake_e8s = 10 * E8` and `staked_maturity_e8s_equivalent = 2 * E8` — the 2 ICP of staked maturity is silently absent from the returned value. [5](#0-4) 

The `get_neuron_info_by_id_or_subaccount` endpoint that calls this function requires no authorization:

```rust
pub fn get_neuron_info_by_id_or_subaccount(
    &self,
    find_by: &NeuronIdOrSubaccount,
    requester: PrincipalId,
) -> Result<NeuronInfo, GovernanceError> {
    self.with_neuron_by_neuron_id_or_subaccount(find_by, |neuron| {
        neuron.get_neuron_info(...)
    })
}
``` [6](#0-5) 

### Impact Explanation

Any unprivileged caller — including dashboards, wallets, governance tooling, and other canisters — that queries `get_neuron_info` or `list_neurons` and reads `NeuronInfo.stake_e8s` receives a value that is understated by the full amount of `staked_maturity_e8s_equivalent`. For neurons that have used `StakeMaturity` (a common operation), the reported stake can be materially lower than the actual stake that drives voting power. This creates a systematic discrepancy between the publicly reported stake and the stake actually used in all governance computations, misleading users and any system that relies on the field as documented. The internal voting power calculation itself uses the correct `stake_e8s()` (including staked maturity), so governance decisions are not directly corrupted, but the public accounting view is wrong.

**Impact: 3** — Misleading/understated public value; no direct fund loss, but violates the documented interface contract and misleads all consumers of the public neuron query API.

### Likelihood Explanation

**Likelihood: 5** — The bug is triggered by any query to `get_neuron_info` or `list_neurons` for any neuron that has ever called `StakeMaturity`. Staking maturity is a standard, widely-used neuron operation. The discrepancy is permanent and unconditional — no special timing or attacker action is required. Any unprivileged ingress query caller can observe it.

### Recommendation

Replace `self.minted_stake_e8s()` with `self.stake_e8s()` on line 950 of `rs/nns/governance/src/neuron/types.rs` so that `NeuronInfo.stake_e8s` matches the formula documented in the DID interface: `cached_neuron_stake_e8s - neuron_fees_e8s + staked_maturity_e8s_equivalent`. Update the corresponding test assertion to expect `12 * E8` (10 ICP stake + 2 ICP staked maturity) instead of `10 * E8`.

### Proof of Concept

1. Create a neuron with `cached_neuron_stake_e8s = 10 * E8` and `neuron_fees_e8s = 0`.
2. Call `StakeMaturity` to move `2 * E8` of maturity into `staked_maturity_e8s_equivalent`.
3. Call `get_neuron_info` (no authorization required) on the NNS Governance canister.
4. Observe that `NeuronInfo.stake_e8s = 10 * E8` — the 2 ICP of staked maturity is absent.
5. Observe that `NeuronInfo.staked_maturity_e8s_equivalent = 2 * E8` is returned separately.
6. The DID specification states `stake_e8s` should equal `10 * E8 + 2 * E8 = 12 * E8`, but the returned value is `10 * E8` — understated by `2 * E8`.
7. Confirm that the neuron's actual voting power is computed from `12 * E8` (correct), while the public `stake_e8s` field reports `10 * E8` (incorrect), creating a persistent discrepancy between the documented interface and the implementation. [7](#0-6)

fn neuron_stake_e8s(
    cached_neuron_stake_e8s: u64,
    neuron_fees_e8s: u64,
    staked_maturity_e8s_equivalent: Option<u64>,
