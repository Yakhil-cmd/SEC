### Title
`spawn_neuron` Minimum-Stake Guard Uses Hardcoded 5% Worst-Case While Mission 70 Modulation Reaches -10% — (`rs/nns/governance/src/governance.rs`)

---

### Summary

`spawn_neuron` validates that the child neuron will have at least `neuron_minimum_stake_e8s` ICP after maturity modulation by computing `least_possible_stake = maturity_to_spawn * 0.95` (hardcoded 5% reduction). However, the Mission 70 maturity modulation lower bound is **-10%** (`MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 = -1_000`). When `maybe_spawn_neurons` later applies the actual modulation, the minted stake can fall below the minimum, violating the protocol invariant that every neuron must hold at least `neuron_minimum_stake_e8s`.

---

### Finding Description

**Guard in `spawn_neuron`** uses a hardcoded 5% worst-case:

```rust
// rs/nns/governance/src/governance.rs:2664-2668
let least_possible_stake = (maturity_to_spawn as f64 * (1_f64 - 0.05)) as u64;
if least_possible_stake < economics.neuron_minimum_stake_e8s {
    return Err(GovernanceError::new_with_message(
        ErrorType::InsufficientFunds,
        "There isn't enough maturity to spawn a new neuron due to worst case maturity modulation.",
    ));
}
``` [1](#0-0) 

**Mission 70 modulation bounds** are asymmetric and wider than ±5%:

```rust
// rs/nns/governance/src/timer_tasks/update_icp_xdr_rate_related_data.rs:47-50
pub(crate) const MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70: i64 = -1_000; // -10%
pub(crate) const MATURITY_MODULATION_MAX_PERMYRIAD_MISSION_70: i64 = 200;    // +2%
``` [2](#0-1) 

**`maybe_spawn_neurons`** reads the live Mission 70 modulation and applies it, bounded by `VALID_MATURITY_MODULATION_BASIS_POINTS_RANGE` which uses the Mission 70 constants:

```rust
// rs/nns/governance/src/governance.rs:276-278
const VALID_MATURITY_MODULATION_BASIS_POINTS_RANGE: RangeInclusive<i32> =
    MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 as i32
        ..=MATURITY_MODULATION_MAX_PERMYRIAD_MISSION_70 as i32;
``` [3](#0-2) 

The actual minting in `maybe_spawn_neurons` applies `apply_maturity_modulation(original_maturity, maturity_modulation)` where `maturity_modulation` can be as low as -1000 permyriad (-10%): [4](#0-3) 

The `apply_maturity_modulation` function computes `amount * (10_000 + basis_points) / 10_000`: [5](#0-4) 

**The gap:** The guard in `spawn_neuron` permits spawning when `maturity_to_spawn * 0.95 >= neuron_minimum_stake_e8s`, but the actual minting applies up to a 10% reduction. Any `maturity_to_spawn` in the range `[neuron_minimum_stake_e8s / 0.95, neuron_minimum_stake_e8s / 0.90)` passes the guard but produces a sub-minimum stake.

---

### Impact Explanation

A spawned neuron receives a `cached_neuron_stake_e8s` below `neuron_minimum_stake_e8s` (default 1 ICP = 100,000,000 e8s). This violates the core protocol invariant. The neuron is created on the ledger with a real ICP balance below the minimum, and subsequent operations that enforce the minimum stake (e.g., split, merge, disburse-to-neuron) will behave inconsistently. The difference between the expected stake (5% reduction) and the actual minted stake (up to 10% reduction) is permanently lost — the maturity is consumed but the ICP is not minted to the expected level, constituting an under-distribution to the neuron owner analogous to the skim-unaware coupon calculation in the reference report.

---

### Likelihood Explanation

The Mission 70 modulation reaches -10% when the 7-day ICP price average is approximately 10% below the 365-day average — a realistic market condition during sustained ICP price declines. The modulation moves at most 30 permyriad per day (`MATURITY_MODULATION_DAILY_SPEED_LIMIT_PERMYRIAD = 30`), so it can reach -1000 after ~33 consecutive days of downward movement. [6](#0-5) 

Any unprivileged NNS neuron controller can trigger this by calling `spawn_neuron` with a `percentage_to_spawn` that places `maturity_to_spawn` in the vulnerable range. No privileged access is required.

---

### Recommendation

Replace the hardcoded `0.05` in `spawn_neuron` with the actual Mission 70 lower bound:

```rust
// Use the actual worst-case modulation instead of hardcoded 5%
let worst_case_reduction =
    (-MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70) as f64 / 10_000.0; // 0.10
let least_possible_stake =
    (maturity_to_spawn as f64 * (1_f64 - worst_case_reduction)) as u64;
```

Alternatively, use `apply_maturity_modulation(maturity_to_spawn, MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 as i32)` directly, mirroring the pattern already used in SNS `disburse_maturity` and NNS `initiate_maturity_disbursement`. [7](#0-6) 

---

### Proof of Concept

Assume `neuron_minimum_stake_e8s = 100_000_000` (1 ICP) and Mission 70 modulation is at -1000 permyriad (-10%):

1. Neuron has `maturity_e8s_equivalent = 120_000_000` (1.2 ICP).
2. Controller calls `spawn_neuron` with `percentage_to_spawn = 89` → `maturity_to_spawn = 106_800_000`.
3. Guard: `106_800_000 * 0.95 = 101_460_000 > 100_000_000` → **passes**.
4. `maybe_spawn_neurons` applies -10%: `106_800_000 * 0.90 = 96_120_000`.
5. Child neuron is minted with `cached_neuron_stake_e8s = 96_120_000` — **below the 100,000,000 minimum**.
6. Parent neuron's maturity is permanently reduced by 106,800,000 e8s; the child receives only 96,120,000 e8s on the ledger. The 10,680,000 e8s difference is neither minted nor returned — it is silently lost, directly analogous to the unaccounted skim fee in the reference report.

```rust
pub fn apply_maturity_modulation(
    amount_maturity_e8s: u64,
    maturity_modulation_basis_points: i32,
) -> Result<u64, String> {
    let amount_e8s = u128::from(amount_maturity_e8s);

    let adjusted_maturity_modulation_basis_points = saturating_add_or_subtract_u128_i32(
        BASIS_POINTS_PER_UNITY,
        maturity_modulation_basis_points,
    );

    let modulated_amount_e8s: u128 = amount_e8s
        .checked_mul(adjusted_maturity_modulation_basis_points)
        .ok_or_else(|| "Underflow or overflow when calculating maturity modulation".to_string())?
        .checked_div(BASIS_POINTS_PER_UNITY)
        .ok_or_else(|| "Underflow or overflow when calculating maturity modulation".to_string())?;

    u64::try_from(modulated_amount_e8s).map_err(|err| err.to_string())
}
