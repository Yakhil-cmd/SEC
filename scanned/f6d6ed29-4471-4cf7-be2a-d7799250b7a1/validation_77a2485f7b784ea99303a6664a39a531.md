### Title
Unbounded Iteration Over All Active Neurons Causes Instruction-Limit Exhaustion in NNS Governance Proposal Execution — (`File: rs/nns/governance/src/neuron_store.rs`)

---

### Summary

`list_active_neurons_fund_neurons` in the NNS Governance canister iterates over the entire active-neuron store (up to 500,000 neurons) in a single synchronous pass with no batch limit. This function is called during `draw_maturity_from_neurons_fund`, which executes as part of a `CreateServiceNervousSystem` proposal. As the neuron population approaches the recently raised cap of 500,000, the instruction cost of this unbounded scan can exceed the per-message instruction limit, causing the proposal execution to trap and permanently fail.

---

### Finding Description

`list_active_neurons_fund_neurons` calls `with_active_neurons_iter_sections(.., sections)` with an unbounded range (`..`) and collects every result into a `Vec` in one shot:

```rust
// rs/nns/governance/src/neuron_store.rs  lines 557-575
pub fn list_active_neurons_fund_neurons(&self) -> Vec<NeuronsFundNeuron> {
    let now = self.now();
    self.with_active_neurons_iter_sections(
        |iter| {
            iter.filter(|neuron| Self::is_active_neurons_fund_neuron(neuron, now))
                .map(|neuron| NeuronsFundNeuron { ... })
                .collect()          // ← no take() / no limit

        NeuronSections { hot_keys: true, ..NeuronSections::NONE },

}

Linear projection: `2,250,175 × (500,000 / 100) ≈ 11.25 billion instructions`. The standard IC per-message instruction limit for update calls is **5 billion instructions**. Even if NNS governance is granted a higher limit, the 25-billion budget assumed by the benchmark is not guaranteed by the protocol, and the projection already exceeds the standard limit by more than 2×.

The same unbounded pattern exists in `list_ready_to_spawn_neuron_ids` (no `take` limit), while the analogous `list_neurons_ready_to_unstake_maturity` was explicitly fixed with a `max_num_neurons` cap — confirming the team is aware that unbounded scans over the full neuron set can exceed instruction limits:

```rust

    &self, now_seconds: u64, max_num_neurons: usize,  // ← bounded

``` [7](#0-6) 

The CHANGELOG explicitly records this class of fix:

> "Unstaking maturity task has a limit of 100 neurons per message, which prevents it from exceeding instruction limit." [8](#0-7) 

---

### Impact Explanation

When a `CreateServiceNervousSystem` proposal executes and the active-neuron count is near 500,000, `draw_maturity_from_neurons_fund` → `list_active_neurons_fund_neurons` will consume more instructions than the per-message limit allows. The execution traps, the proposal is marked as failed, and the SNS launch is permanently blocked for that proposal. No funds are lost, but the governance action is rendered unexecutable — a denial-of-service against the SNS creation pathway. The same trap would affect any future code path that calls `list_ready_to_spawn_neuron_ids` at scale.

---

### Likelihood Explanation

The neuron cap was just raised to 500,000. NNS neuron creation is rate-limited but continuous; the network is on a trajectory toward that cap. Any `CreateServiceNervousSystem` proposal submitted once the active-neuron count is high enough will reliably fail. No special attacker capability is required beyond participating in NNS governance (submitting or voting on a proposal), which is open to any ICP holder. The instruction-limit breach is deterministic and reproducible once the neuron count crosses the threshold.

---

### Recommendation

Apply the same bounded-batch pattern already used for `list_neurons_ready_to_unstake_maturity`:

1. Add a `max_neurons` parameter to `list_active_neurons_fund_neurons` and insert `.take(max_neurons)` before `.collect()`.
2. If more than `max_neurons` Neurons' Fund members exist, either process them in multiple timer-driven batches or reject the proposal with a clear error before the unbounded scan begins.
3. Apply the same fix to `list_ready_to_spawn_neuron_ids`, which has the identical unbounded-collect pattern.
4. Add an integration test that verifies `draw_maturity_from_neurons_fund` completes within the instruction budget when the neuron store is at `MAX_NUMBER_OF_NEURONS`.

---

### Proof of Concept

1. Populate the NNS with neurons approaching `MAX_NUMBER_OF_NEURONS` (500,000), with a significant fraction joined to the Neurons' Fund.
2. Submit and pass a `CreateServiceNervousSystem` proposal.
3. Observe that proposal execution traps with `CanisterInstructionLimitExceeded` (or equivalent) because `list_active_neurons_fund_neurons` exhausts the per-message instruction budget before returning.
4. The proposal transitions to a failed state and cannot be retried without a canister upgrade that fixes the unbounded scan.

        &mut self,
        proposal_id: &ProposalId,
        create_service_nervous_system: &CreateServiceNervousSystem,
    ) -> Result<(NeuronsFundSnapshot, NeuronsFundParticipationConstraints), GovernanceError> {
        let swap_participation_limits = create_service_nervous_system
            .swap_parameters
            .as_ref()
            .ok_or_else(|| {
                "CreateServiceNervousSystem.swap_parameters is not specified.".to_string()
            })?;
        let swap_participation_limits =
            SwapParticipationLimits::try_from_swap_parameters(swap_participation_limits)?;
        let neurons_fund_participation_limits =
            self.try_derive_neurons_fund_participation_limits()?;
        let neurons_fund = self.neuron_store.list_active_neurons_fund_neurons();
        let initial_neurons_fund_participation = PolynomialNeuronsFundParticipation::new(
            neurons_fund_participation_limits,
            swap_participation_limits,
            neurons_fund,
        )?;
        // Check that the maximum number of Neurons' Fund participants is not too high. Otherwise,
        // the SNS may be unable to distribute SNS tokens to all participants after the swap.
        {
            let maximum_neurons_fund_participants = initial_neurons_fund_participation
                .snapshot()
                .neurons()
                .len() as u64;
            if maximum_neurons_fund_participants > MAX_NEURONS_FUND_PARTICIPANTS {
                return Err(GovernanceError::new_with_message(
                    ErrorType::InvalidProposal,
                    format!(
                        "The maximum number of Neurons' Fund participants ({maximum_neurons_fund_participants}) must not exceed \
                        MAX_NEURONS_FUND_PARTICIPANTS ({MAX_NEURONS_FUND_PARTICIPANTS}).",
                    ),
                ));
            };
        }
        let constraints = initial_neurons_fund_participation.compute_constraints()?;
        let initial_neurons_fund_participation_snapshot =
            initial_neurons_fund_participation.snapshot_cloned();
        // First check if the ProposalData is available (and error out if not); then actually draw
        // the funds from the Neurons' Fund. This way, we do not need to issue refunds in case
        // of an error.
        if self.get_proposal_data(*proposal_id).is_none() {
            return Err(GovernanceError::new_with_message(
                ErrorType::InvalidProposal,
                format!("ProposalData must be present for proposal {proposal_id:?}."),
            ));
        }
        self.neuron_store
            .draw_maturity_from_neurons_fund(&initial_neurons_fund_participation_snapshot)?;
        let initial_neurons_fund_participation = Some(initial_neurons_fund_participation.into());
        let neurons_fund_data = NeuronsFundData {
            initial_neurons_fund_participation,
            // These two fields will be known after `settle_neurons_fund_participation` is called.
            final_neurons_fund_participation: None,
            neurons_fund_refunds: None,
        };
        // Unwrapping is safe due to the `self.get_proposal_data().is_none()` check above.
        let proposal_data = self.mut_proposal_data(*proposal_id).unwrap();
        proposal_data.neurons_fund_data = Some(neurons_fund_data);
        Ok((initial_neurons_fund_participation_snapshot, constraints))
    }
