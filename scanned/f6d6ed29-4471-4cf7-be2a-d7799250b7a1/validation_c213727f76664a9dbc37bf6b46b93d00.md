### Title
Single Authorized Caller Possesses Unilateral Power to Halt or Delete CloudEngine Subnets - (`rs/engine_controller/canister/canister.rs`)

### Summary

The engine controller canister grants a single hardcoded principal (`DEFAULT_AUTHORIZED_CALLER`) unilateral, ungoverned power to halt or delete any CloudEngine subnet. No NNS governance vote, timelock, or multi-sig is required. Users who deploy canisters on CloudEngine subnets — and hold state or funds there — have no on-chain protection against this single point of failure.

### Finding Description

The engine controller canister (`rs/engine_controller/canister/canister.rs`) is the sole gateway for creating, deleting, halting, and upgrading CloudEngine subnets. All of its privileged endpoints are guarded by a single check:

```rust
fn ensure_authorized() -> Result<Principal, String> {
    let caller = msg_caller();
    let expected = AUTHORIZED_CALLER.with(|c| *c.borrow());
    if caller != expected {
        return Err(format!(
            "Caller {caller} is not authorized to call this canister"
        ));
    }
    Ok(caller)
}
``` [1](#0-0) 

`AUTHORIZED_CALLER` is initialized from a single hardcoded string constant:

```rust
const DEFAULT_AUTHORIZED_CALLER: &str =
    "bct5z-vccu4-6q4t2-3lb6l-wm43p-ulppt-o5sqq-w6het-rthdz-qp4yn-fqe";
``` [2](#0-1) 

This single principal controls three destructive operations with no additional safeguards:

**1. Halt any CloudEngine subnet** — `update_subnet` with `is_halted = Some(true)`: [3](#0-2) 

The registry enforces that `is_halted` is an allowed field for the engine controller: [4](#0-3) 

**2. Delete any CloudEngine subnet** — `delete_engine`: [5](#0-4) 

**3. Deploy new GuestOS to all subnet nodes** — `deploy_guestos_to_all_subnet_nodes`: [6](#0-5) 

The `normalize_subnet_admins` function further ensures the authorized caller is always a subnet admin, permanently cementing its control: [7](#0-6) 

The authorized caller is re-applied on every `post_upgrade`, meaning even a canister upgrade resets to the hardcoded default if no override is supplied: [8](#0-7) 

### Impact Explanation

A CloudEngine subnet is a real IC subnet type. Canisters deployed on it hold user state, cycles, and potentially token balances or other value. The single authorized caller can:

- **Freeze all user canisters** on a CloudEngine subnet by setting `is_halted = true`. Halted subnets stop processing all ingress and inter-canister messages, making all user funds and state inaccessible indefinitely.
- **Destroy all user canisters** by calling `delete_engine`, which removes the subnet from the registry entirely, causing all canister state to be lost.
- **Deploy arbitrary GuestOS** to all subnet nodes via `deploy_guestos_to_all_subnet_nodes`, potentially replacing the replica with a malicious version that exfiltrates canister state.

None of these operations require a governance proposal, a timelock, or any second factor. The impact is total loss of availability and potentially total loss of user funds/state for all canisters on the targeted subnet.

### Likelihood Explanation

The attack requires only that the single authorized caller's private key be compromised — via phishing, malware, supply-chain attack, or insider threat — or that the authorized caller acts maliciously. This is the same threat model as the external report (owner key compromise). The design is inherently dangerous because:

- A single self-authenticating principal (user key) controls infrastructure-level operations affecting all users of CloudEngine subnets.
- There is no on-chain check that the authorized caller is itself governed by NNS or any multi-party mechanism.
- The `normalize_subnet_admins` function ensures the authorized caller can never be removed from the admin list by other admins, making the privilege irrevocable without a canister upgrade.

### Recommendation

1. **Require NNS governance proposals** for halting or deleting CloudEngine subnets, removing the single-key bypass entirely.
2. **Add a timelock** (e.g., 24–48 hours) for destructive operations (`delete_engine`, `is_halted = true`) so users can observe and react before the operation takes effect.
3. **Replace the single authorized caller** with a multi-sig canister or a governance-controlled canister as the sole caller, eliminating the single point of key compromise.
4. **Remove `normalize_subnet_admins`** or make it governance-controlled, so the authorized caller cannot permanently entrench itself.

### Proof of Concept

1. Attacker compromises the private key of `bct5z-vccu4-6q4t2-3lb6l-wm43p-ulppt-o5sqq-w6het-rthdz-qp4yn-fqe`.
2. Attacker sends an ingress update call to the engine controller canister's `update_subnet` endpoint with `is_halted = Some(true)` targeting any active CloudEngine subnet.
3. `ensure_authorized()` passes because the caller matches `AUTHORIZED_CALLER`.
4. `ensure_only_allowed_fields_set()` passes because `is_halted` is an explicitly allowed field.
5. The registry's `do_update_subnet` sets `is_halted = true` on the subnet record.
6. All nodes on the subnet stop processing messages. All user canisters are frozen. All user funds and state are inaccessible.
7. Alternatively, the attacker calls `delete_engine` to permanently destroy the subnet and all canister state on it.

No governance vote, no timelock, no second factor — a single ingress message from the compromised key is sufficient.

#[update]
async fn update_subnet(payload: UpdateSubnetPayload) -> Result<(), String> {
    ensure_authorized()?;
    ensure_only_allowed_fields_set(&payload)?;

    // Normalize `subnet_admins` so the super admin is always present.
    // The caller may omit the field entirely (no change requested), but if
    // they do supply one, we treat it as the source of truth and add the
    // super admin if missing.
    #[allow(unused_mut)]
    let mut payload = payload;
    if let Some(admins) = payload.subnet_admins {
        payload.subnet_admins = Some(normalize_subnet_admins(admins));
    }

    Call::unbounded_wait(REGISTRY_CANISTER_ID.into(), "update_subnet")
        .with_arg(payload)
        .await
        .map_err(|e| format!("registry.update_subnet call failed: {e:?}"))?
        .candid::<()>()
        .map_err(|e| format!("Failed to decode registry response: {e}"))?;

    Ok(())
}
