### Title
Unbounded Day-Range Loop in `get_node_providers_rewards` Exhausts Per-Message Instruction Limit — (File: `rs/node_rewards/canister/src/canister/mod.rs`)

---

### Summary

The `get_node_providers_rewards` update method in the Node Rewards Canister iterates over every calendar day between a caller-supplied `from_day` and `to_day`, performing a full per-day rewards calculation on each iteration. Although a `reset_instructions` self-call is inserted between iterations, the validation logic imposes **no upper bound on the number of days** in the requested range. A caller can supply a range spanning years, causing the canister to loop over thousands of days, each of which performs a full scan of all node operators and subnets. This is the direct IC analog of the `massUpdateRewards` / `removeIdFromList` unbounded-loop DoS described in the external report.

---

### Finding Description

`get_node_providers_rewards` in `rs/node_rewards/canister/src/canister/mod.rs` accepts a `GetNodeProvidersRewardsRequest` containing `from_day` and `to_day` fields. The only validation performed by `validate_reward_period` is:

1. `last_day_synced >= to_date`
2. `from_date <= to_date`
3. `to_date < today`

There is **no check on the number of days** between `from_date` and `to_date`. [1](#0-0) 

The function then loops over every day in the range: [2](#0-1) 

Each iteration calls `calculate_rewards_for_date`, which in turn calls `get_rewardable_nodes_per_provider` — a function that bulk-fetches **all** node records, all node operator records, and all data center records from the registry, then iterates over every node: [3](#0-2) 

The `reset_instructions` self-call between iterations is a `bounded_wait` call that resets the per-message instruction counter. However, this mechanism is only compiled in for `target_arch = "wasm32"` and, critically, it does **not** prevent the total number of loop iterations from being arbitrarily large — it only prevents a single iteration from exceeding the per-message limit. With a range of, say, 3 years (≈1095 days), the canister will make 1095 self-calls, each doing a full registry scan. This can exhaust cycles, block the canister's message queue, and prevent legitimate callers (including the NNS Governance canister) from receiving responses.

The `GetNodeProvidersRewardsRequest` type has no server-side day-count cap: [4](#0-3) 

The endpoint is exposed as a public update call in the canister's Candid interface: [5](#0-4) 

---

### Impact Explanation

- **Cycles exhaustion / canister DoS**: A single malicious or misconfigured caller can submit a request with `from_day = 2020-01-01` and `to_day = yesterday`, causing ~1900+ day iterations. Each iteration performs a full registry scan (all nodes, all operators, all data centers) and a self-call. This drains the canister's cycles budget rapidly.
- **Message queue saturation**: The 1900+ `reset_instructions` self-calls flood the canister's own input queue, delaying or blocking all other incoming messages, including the NNS Governance canister's legitimate `get_node_providers_rewards` call used to compute monthly node provider rewards.
- **Reward minting disruption**: If the Node Rewards Canister is unresponsive, `mint_monthly_node_provider_rewards` in NNS Governance will fail, preventing node providers from receiving their monthly ICP rewards. [6](#0-5) 

---

### Likelihood Explanation

The endpoint is publicly callable by any principal — no authentication or authorization is required. The `validate_reward_period` function only checks date ordering and sync status, not range size. The canister has been live since at least Proposal 138288 (2025-08-30). Any unprivileged ingress sender can craft a request with a multi-year date range and submit it directly to the Node Rewards Canister. [7](#0-6) 

---

### Recommendation

Add a maximum day-range limit in `validate_reward_period`. For example, cap the range at 35 days (matching the internal telemetry task's usage):

```rust
const MAX_REWARD_PERIOD_DAYS: i64 = 35;
let days = (to_date - from_date).num_days();
if days > MAX_REWARD_PERIOD_DAYS {
    return Err(format!(
        "Reward period too large: {} days (max {})",
        days, MAX_REWARD_PERIOD_DAYS
    ));
}
``` [1](#0-0) 

Additionally, consider adding caller authentication so only the NNS Governance canister can invoke `get_node_providers_rewards`.

---

### Proof of Concept

An attacker sends the following ingress update call to the Node Rewards Canister:

```
get_node_providers_rewards(
  record {
    from_day = record { year = 2020; month = 1; day = 1 };
    to_day   = record { year = 2025; month = 6; day = 20 };
    algorithm_version = null;
  }
)
```

`validate_reward_period` passes (assuming `last_day_synced >= 2025-06-20` and `to_date < today`). [1](#0-0) 

The loop at line 325 then iterates over ≈1997 days. Each iteration:
1. Calls `calculate_rewards_for_date` → `get_rewardable_nodes_per_provider` → bulk-fetches all nodes, operators, data centers from the registry stable BTreeMap.
2. Issues a `bounded_wait` self-call to `reset_instructions`, enqueuing a new message into the canister's own queue. [8](#0-7) 

This produces ~1997 queued self-calls, saturating the canister's message queue and draining its cycles. The NNS Governance canister's subsequent call to `get_node_providers_rewards` (used for monthly reward minting) will either time out or be rejected. [9](#0-8)

    fn validate_reward_period(
        &self,
        from_date: NaiveDate,
        to_date: NaiveDate,
    ) -> Result<(), String> {
        let last_day_synced = self
            .get_last_day_synced()
            .ok_or("Metrics and registry are not synced up")?;

        if last_day_synced < to_date {
            return Err("Metrics and registry are not synced up to to_date".to_string());
        }

        if from_date > to_date {
            return Err("from_date must be before to_date".to_string());
        }

        let today =
            DateTime::from_timestamp_nanos(current_time().as_nanos_since_unix_epoch() as i64)
                .date_naive();

        if to_date >= today {
            return Err("to_date must be earlier than today".to_string());
        }

        Ok(())

    pub fn get_rewardable_nodes_per_provider(
        &self,
        date: &NaiveDate,
        provider_filter: Option<&PrincipalId>,
    ) -> Result<BTreeMap<PrincipalId, Vec<RewardableNode>>, RegistryClientError> {
        let mut rewardable_nodes_per_provider: BTreeMap<_, Vec<RewardableNode>> = BTreeMap::new();
        let registry_version = self
            .version_for_timestamp_nanoseconds(last_unix_timestamp_nanoseconds(date))
            .unwrap();

        // Bulk-fetch all data upfront instead of per-node individual lookups.
        let nodes = self.nodes_in_version(registry_version)?;
        let all_operators = self.all_node_operators(registry_version)?;
        let all_data_centers = self.all_data_centers(registry_version)?;

        for (node_id, node_record) in nodes {
            let node_operator_id: PrincipalId = node_record
                .node_operator_id
                .try_into()
                .expect("Failed to parse PrincipalId from node operator ID");

            let Some(node_operator_record) = all_operators.get(&node_operator_id) else {
                ic_cdk::println!("Node {} has no NodeOperatorRecord: skipping", node_id);
                continue;
            };

            let Some(data_center_record) = all_data_centers.get(&node_operator_record.dc_id) else {
                ic_cdk::println!(
                    "Node {} has NodeOperator but no DataCenterRecord for dc_id {}: skipping",
                    node_id,
                    node_operator_record.dc_id
                );
                continue;
            };

            let node_provider_id: PrincipalId = node_operator_record
                .node_provider_principal_id
                .clone()
                .try_into()
                .expect("Failed to parse PrincipalId");

            if let Some(provider_filter) = provider_filter
                && &node_provider_id != provider_filter
            {
                continue;
            }
            let Some(some_reward_type) = node_record.node_reward_type else {
                // If the node does not have a node_reward_type, we skip it.
                continue;
            };

            let node_reward_type =
                NodeRewardType::try_from(some_reward_type).expect("Invalid node_reward_type value");

            // type4.5 nodes are rewarded as type1.1 nodes.
            let node_reward_type = if node_reward_type == NodeRewardType::Type4dot5 {
                NodeRewardType::Type1dot1
            } else {
                node_reward_type
            };

            rewardable_nodes_per_provider
                .entry(node_provider_id)
                .or_default()
                .push(RewardableNode {
                    node_id,
                    node_reward_type,
                    dc_id: node_operator_record.dc_id.clone(),
                    region: data_center_record.region.clone(),
                });
        }
        Ok(rewardable_nodes_per_provider)

    async fn mint_monthly_node_provider_rewards(&mut self) -> Result<(), GovernanceError> {
        // Return immediately if another call is already in progress.
        thread_local! {
            static LOCK: RefCell<Option<u64>> = const { RefCell::new(None) };
        }
        let release_on_drop = ic_nervous_system_lock::acquire(&LOCK, self.env.now());
        if let Err(earlier_call_start_timestamp) = release_on_drop {
            // Log, but not too frequently (at most once every 5 minutes).
            thread_local! {
                static LAST_LOGGED_UNAVAILABLE_TIMESTAMP_SECONDS: RefCell<u64> = const { RefCell::new(0) };
            }
            let time_since_logged_seconds = LAST_LOGGED_UNAVAILABLE_TIMESTAMP_SECONDS
                .with(|t| self.env.now().saturating_sub(*t.borrow()));
            if time_since_logged_seconds > 5 * 60 {
                println!(
                    "{}Another mint_monthly_node_provider_rewards call (started at \
                     {} seconds since the UNIX epoch) is already in progress.",
                    LOG_PREFIX, earlier_call_start_timestamp,
                );
                LAST_LOGGED_UNAVAILABLE_TIMESTAMP_SECONDS.with(|t| {
                    *t.borrow_mut() = self.env.now();
                });
            }

            return Ok(());
        }

        let monthly_node_provider_rewards = if are_performance_based_rewards_enabled() {
            self.get_node_providers_rewards().await?
        } else {
            self.get_monthly_node_provider_rewards().await?
        };

        let _ = self
            .reward_node_providers(&monthly_node_provider_rewards.rewards)
            .await;
        self.update_most_recent_monthly_node_provider_rewards(monthly_node_provider_rewards);

        // Commit the minting status by making a canister call.
        let _unused_canister_status_response = self
            .env
            .call_canister_method(
                GOVERNANCE_CANISTER_ID,
                "get_build_metadata",
                Encode!().unwrap_or_default(),

            .await;
