### Title
Byzantine Block Maker Can Trigger K-Fold RPC Amplification via Duplicate `SignedIngressId` Entries in Crafted `StrippedBlockProposal` — (`rs/p2p/artifact_downloader/src/fetch_stripped_artifact/assembler.rs`)

---

### Summary

When `HASHES_IN_BLOCKS_ENABLED = true`, a Byzantine block maker below the fault threshold can craft a `StrippedBlockProposal` protobuf with K duplicate `StrippedIngressMessage` entries for the same ingress message. Because neither the deserialization path nor `BlockProposalAssembler::new` / `missing_stripped_messages` deduplicate the `Vec<SignedIngressId>`, the assembler spawns K concurrent `get_or_fetch` tasks for the same message, each potentially issuing an independent RPC to a random peer.

---

### Finding Description

**`HASHES_IN_BLOCKS_ENABLED` is unconditionally `true`:** [1](#0-0) 

**Deserialization of `StrippedBlockProposal` does not deduplicate `ingress_messages`:**

The `TryFrom<pb::StrippedBlockProposal>` implementation collects the repeated protobuf field directly into a `Vec<SignedIngressId>` with no uniqueness check: [2](#0-1) 

**`BlockProposalAssembler::new` maps the Vec directly — no deduplication:** [3](#0-2) 

**`missing_stripped_messages` returns all K None-slots — no deduplication:** [4](#0-3) 

**`assemble_message` spawns one `get_or_fetch` task per returned ID — K tasks for K duplicates:** [5](#0-4) 

**`get_or_fetch` issues an RPC if the message is not in the local pool:** [6](#0-5) 

**The `StrippedIngressPayload` type is a plain `Vec` — no set semantics:** [7](#0-6) 

**The `StrippedBlockProposal` carries no signature over the `stripped_ingress_payload` field.** The block maker's signature covers only the full `BlockProposal` content hash. The `pruned_block_proposal_proto` and `unstripped_consensus_message_id` are signed, but the `ingress_messages` list in the stripped wrapper is unsigned and freely modifiable by the sender.

**Post-spawn failure does not prevent the amplification.** After all K tasks are spawned and run concurrently, `try_insert` uses `find` (linear scan, returns first match): [8](#0-7) 

The first task fills slot 0; the second task finds slot 0 already `Some` and returns `AlreadyInserted`, causing `Unwanted` to be returned. But by this point all K RPCs have already been issued concurrently.

---

### Impact Explanation

- Each receiving replica spawns K concurrent Tokio tasks and issues up to K independent RPCs to random peers for the same ingress message.
- With `max_ingress_messages_per_block ≈ 1000`, a single crafted proposal causes up to 1000× RPC amplification per receiving replica per round the Byzantine node is selected as block maker.
- The Byzantine node is selected as block maker with probability proportional to their rank in each round — this is a regular, predictable event, not a one-time occurrence.
- The assembled block is ultimately rejected (`Unwanted`), so there is no consensus safety impact, but the resource exhaustion (goroutine/task spawning, network bandwidth, peer RPC handler load) is real and repeatable.

---

### Likelihood Explanation

- Requires only a single Byzantine replica acting as block maker for any round — well within the "below fault threshold" attacker model.
- No cryptographic forgery required: the `stripped_ingress_payload` is not signed.
- The crafted protobuf is trivially constructable by repeating the same `StrippedIngressMessage` entry K times.
- `HASHES_IN_BLOCKS_ENABLED = true` is the live production configuration.

---

### Recommendation

Deduplicate `SignedIngressId` entries immediately upon deserialization or at the start of `BlockProposalAssembler::new`. A `HashSet` or sorted-dedup pass over `stripped_ingress_payload.ingress_messages` before constructing the `ingress_messages` Vec eliminates the amplification:

```rust
// In BlockProposalAssembler::new or TryFrom<pb::StrippedBlockProposal>
let mut seen = std::collections::HashSet::new();
let ingress_messages = stripped_block_proposal
    .stripped_ingress_payload
    .ingress_messages
    .iter()
    .filter(|id| seen.insert(id.clone()))  // deduplicate
    .map(|id| (id.clone(), None))
    .collect();
```

Alternatively, reject the `StrippedBlockProposal` during deserialization if any duplicate `SignedIngressId` is detected, treating it as a malformed artifact.

---

### Proof of Concept

```rust
// Construct a StrippedBlockProposal with K=100 duplicate entries
let (ingress, ingress_id) = fake_ingress_message_with_arg_size("dup", 64);
let duplicate_ids = vec![ingress_id.clone(); 100];
let stripped = StrippedBlockProposal {
    pruned_block_proposal_proto: pb::BlockProposal::default(),
    stripped_ingress_payload: StrippedIngressPayload {
        ingress_messages: duplicate_ids,
    },
    unstripped_consensus_message_id: /* valid block id */,
    stripped_idkg_dealings: StrippedIDkgDealings::default(),
};

let assembler = BlockProposalAssembler::new(stripped);
let missing = assembler.missing_stripped_messages();
assert_eq!(missing.len(), 100); // 100 duplicate tasks will be spawned
// Each spawns a get_or_fetch task → up to 100 concurrent RPCs for the same message
```
