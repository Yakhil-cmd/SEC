### Title
NNS Governance `Spawning` Pseudo-State Allows `Configure` to Mutate Dissolve State During Spawning - (File: rs/nns/governance/src/governance.rs)

### Summary
The NNS Governance neuron state machine implements `Spawning` as a pseudo-state via the `spawn_at_timestamp_seconds` flag rather than as a proper state in the dissolve-state machine. The `configure_neuron` function does not guard against the `Spawning` state, allowing a neuron controller to mutate the underlying dissolve state of a spawning neuron. After spawning completes and the flag is cleared, the neuron emerges in an unexpected dissolve state, violating the intended lifecycle invariants.

### Finding Description
`NeuronState::Spawning` is not a real variant of the dissolve-state machine. It is derived at runtime by checking whether `spawn_at_timestamp_seconds.is_some()`:

```rust
pub fn state(&self, now_seconds: u64) -> NeuronState {
    if self.spawn_at_timestamp_seconds.is_some() {
        return NeuronState::Spawning;
    }
    self.dissolve_state_and_age().current_state(now_seconds)
}
``` [1](#0-0) 

The underlying `dissolve_state_and_age` of a spawning neuron is `DissolvingOrDissolved { when_dissolved_timestamp_seconds }` — a real, mutable field. The `configure_neuron` function in NNS Governance acquires a lock and immediately calls `neuron.configure()` without ever checking whether the neuron is in `Spawning` state:

```rust
fn configure_neuron(&mut self, id: &NeuronId, caller: &PrincipalId, c: &manage_neuron::Configure)
    -> Result<(), GovernanceError>
{
    let now_seconds = self.env.now();
    let lock_command = NeuronInFlightCommand { ... };
    let _lock = self.lock_neuron_for_command(id.id, lock_command)?;
    self.with_neuron_mut(id, |neuron| neuron.configure(caller, now_seconds, c))??;
    Ok(())
}
``` [2](#0-1) 

By contrast, every other mutating operation (`split_neuron`, `stake_maturity_of_neuron`, `disburse_to_neuron`, `disburse_maturity`, `merge_neurons`) explicitly rejects the `Spawning` state:

```rust
if parent_neuron.state(self.env.now()) == NeuronState::Spawning {
    return Err(GovernanceError::new_with_message(
        ErrorType::PreconditionFailed,
        "Can't perform operation on neuron: Neuron is spawning.",
    ));
}
``` [3](#0-2) 

`configure_neuron` is the only state-mutating path that omits this guard. The `Configure` command supports operations including `StopDissolving`, `IncreaseDissolveDelay`, and `SetDissolveTimestamp`, all of which directly rewrite `dissolve_state_and_age`. [4](#0-3) 

A spawning neuron is created with `dissolve_state_and_age = DissolvingOrDissolved { when_dissolved_timestamp_seconds: dissolve_and_spawn_at_timestamp_seconds }` and `spawn_at_timestamp_seconds = Some(dissolve_and_spawn_at_timestamp_seconds)`: [5](#0-4) 

When `maybe_spawn_neurons` finalises the neuron it clears `spawn_at_timestamp_seconds` and sets `cached_neuron_stake_e8s`: [6](#0-5) 

If `Configure::StopDissolving` was called in the window between neuron creation and finalisation, the underlying `dissolve_state_and_age` is rewritten to `NotDissolving { dissolve_delay_seconds, aging_since_timestamp_seconds }`. After the flag is cleared the neuron surfaces as `NotDissolving` instead of the expected `Dissolved`/`Dissolving` state.

### Impact Explanation
An unprivileged ingress sender who controls a spawning neuron can call `manage_neuron` with a `Configure` command during the spawning window. Depending on the sub-operation chosen:

- **`StopDissolving`**: converts the neuron from `DissolvingOrDissolved` to `NotDissolving`, so after spawning the neuron is locked with a non-zero dissolve delay instead of being immediately liquid. The owner cannot disburse the freshly minted ICP without waiting out the delay.
- **`IncreaseDissolveDelay` / `SetDissolveTimestamp`**: extends the dissolve delay arbitrarily, again preventing immediate disbursement.
- **`JoinCommunityFund`**: enrols the spawning neuron in the Neurons' Fund, committing its future stake to matched-funding obligations before the stake even exists.

The governance state machine invariant — that a `Spawning` neuron is inert until finalised — is violated. The impact is a governance authorization bug: state transitions that should be blocked by the `Spawning` pseudo-state are reachable via `Configure`.

### Likelihood Explanation
The spawning window is bounded by the `neuron_spawn_dissolve_delay_seconds` parameter (typically 7 days). During this entire period the neuron is visible via `get_full_neuron` and the controller can submit `manage_neuron` calls. No special privilege beyond neuron ownership is required. The attack is deterministic and requires only a single ingress message.

### Recommendation
Add an explicit `Spawning` guard at the top of `configure_neuron` in `rs/nns/governance/src/governance.rs`, consistent with every other mutating operation:

```rust
fn configure_neuron(&mut self, id: &NeuronId, caller: &PrincipalId, c: &manage_neuron::Configure)
    -> Result<(), GovernanceError>
{
    let now_seconds = self.env.now();
    let neuron_state = self.with_neuron(id, |n| n.state(now_seconds))?;
    if neuron_state == NeuronState::Spawning {
        return Err(GovernanceError::new_with_message(
            ErrorType::PreconditionFailed,
            "Can't perform operation on neuron: Neuron is spawning.",
        ));
    }
    // ... existing lock + configure logic
}
```

More broadly, the `Spawning` pseudo-state should be refactored into a first-class variant of `DissolveStateAndAge` so that the compiler enforces exhaustive state checks rather than relying on per-function ad-hoc guards. [7](#0-6) 

### Proof of Concept
1. Controller calls `manage_neuron` → `Spawn` on a neuron with sufficient maturity. A child neuron is created in `Spawning` state with `spawn_at_timestamp_seconds = T_future`.
2. Before `maybe_spawn_neurons` runs (up to 7 days), the controller calls `manage_neuron` → `Configure { operation: StopDissolving {} }` on the child neuron ID.
3. `configure_neuron` acquires the lock (succeeds because `maybe_spawn_neurons` has not locked this neuron yet) and calls `neuron.configure()`, which rewrites `dissolve_state_and_age` to `NotDissolving { dissolve_delay_seconds: T_future - now, aging_since_timestamp_seconds: now }`.
4. `maybe_spawn_neurons` later clears `spawn_at_timestamp_seconds` and mints the ICP stake.
5. `neuron.state(now)` now returns `NotDissolving` instead of `Dissolved`. The controller cannot disburse the minted ICP without waiting out the dissolve delay, and the neuron accrues age — neither of which was intended by the spawning mechanism. [2](#0-1) [8](#0-7) [9](#0-8)
