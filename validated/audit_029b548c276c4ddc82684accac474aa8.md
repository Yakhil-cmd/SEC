### Title
EVM JSON-RPC DNS Rebinding Protection Hardcoded to Disabled — (`evmrpc/server.go`)

### Summary

Sei's EVM JSON-RPC HTTP server hardcodes `Vhosts: []string{"*"}` when calling `EnableRPC`, permanently disabling the `virtualHostHandler` that was explicitly written to prevent DNS rebinding attacks. Combined with a default `CORSOrigins: "*"` and a listen address of `0.0.0.0`, any web page can make unauthenticated cross-origin requests to the EVM RPC on behalf of a user whose browser can reach the node.

### Finding Description

`NewEVMHTTPServer` in `evmrpc/server.go` calls `httpServer.EnableRPC` with a hardcoded `Vhosts: []string{"*"}`: [1](#0-0) 

`EnableRPC` passes this directly to `NewHTTPHandlerStack`, which wraps the handler with `newVHostHandler`: [2](#0-1) 

Inside `virtualHostHandler.ServeHTTP`, the wildcard check short-circuits all hostname validation: [3](#0-2) 

The code comment on the handler explicitly states its purpose is DNS rebinding prevention: [4](#0-3) 

The protection is therefore dead code in production. Additionally, the default config sets both CORS and WS origins to `"*"`: [5](#0-4) 

The server binds to `LocalAddress = "0.0.0.0"`: [6](#0-5) 

And the EVM HTTP server is **enabled by default**: [7](#0-6) 

### Impact Explanation

An attacker who tricks a node operator (or any user whose browser can reach the node) into visiting a malicious page can:

1. **Read all EVM state** — account balances, nonces, contract storage — via `eth_call`, `eth_getBalance`, etc.
2. **Submit pre-signed transactions** via `eth_sendRawTransaction` if the attacker already holds signed payloads.
3. **Exhaust node resources** via repeated `debug_traceTransaction` / `debug_traceBlockByNumber` calls (computationally expensive), degrading RPC availability.
4. **Enumerate node internals** via `rpc_modules`, `net_*`, etc.

Because the service binds to `0.0.0.0`, it is already publicly reachable without DNS rebinding; however, the hardcoded wildcard vhost means even operators who firewall the port to localhost cannot rely on the vhost mechanism for protection. The wildcard CORS additionally allows any browser origin to read responses, completing the cross-origin attack chain.

This maps to **Medium** under the Sei bounty scope: unauthenticated RPC accessible to unprivileged attackers, with potential for resource DoS degrading RPC availability.

### Likelihood Explanation

- EVM HTTP is **on by default** (`HTTPEnabled: true`, port 8545).
- `Vhosts: ["*"]` is **hardcoded** — no operator configuration can override it.
- `CORSOrigins: "*"` is the **default** — operators must actively restrict it.
- The attack requires only that a node operator (or any user with browser access to the node) visits a malicious page, which is a realistic social-engineering scenario.

### Recommendation

1. **Remove the hardcoded `Vhosts: []string{"*"}`** in `NewEVMHTTPServer`. Instead, expose a `vhosts` field in `evmrpc/config/config.go` (e.g., `Vhosts string`) and default it to `"localhost"` — mirroring go-ethereum's own default.
2. **Change the default `CORSOrigins`** from `"*"` to an empty string (disabled) or `"localhost"`, requiring operators to explicitly opt in to cross-origin access.
3. **Change the default listen address** from `0.0.0.0` to `127.0.0.1` for the EVM RPC, consistent with how `sei-tendermint`'s `DefaultRPCConfig` already defaults to `tcp://127.0.0.1:26657`. [7](#0-6) 

### Proof of Concept

```
# 1. Node starts with default config — EVM RPC on 0.0.0.0:8545, CORS=*, vhosts=*

# 2. Attacker hosts a page with:
fetch("http://<node-ip>:8545", {
  method: "POST",
  headers: {"Content-Type": "application/json"},
  body: JSON.stringify({jsonrpc:"2.0",method:"eth_blockNumber",params:[],id:1})
}).then(r => r.json()).then(console.log)

# 3. Browser sends the request; CORS wildcard allows the response to be read.
# 4. For DNS rebinding: attacker sets evil.com → node-IP after TTL expiry;
#    browser sends Host: evil.com — virtualHostHandler passes it because vhosts=["*"].
# 5. Attacker reads chain state or floods debug_trace* to degrade RPC.
```
