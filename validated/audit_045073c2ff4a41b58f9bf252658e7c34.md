### Title
SNS `NervousSystemParameters` Bonus Percentages Validated Individually But Combined Multiplicative Effect Is Unbounded — (`rs/sns/governance/src/types.rs`)

### Summary

`NervousSystemParameters` contains two voting-power bonus parameters — `max_dissolve_delay_bonus_percentage` and `max_age_bonus_percentage` — that are each validated against individual ceilings (900 and 400 respectively). No check is performed on their combined multiplicative effect. Because the SNS voting-power formula applies both bonuses sequentially (multiplicatively, not additively), the combined maximum multiplier is `(1 + 9) × (1 + 4) = 50×`. An SNS initialized with both parameters at their ceilings allows a token holder controlling neurons with maximum dissolve delay and maximum age to hold 50× the voting power of an equivalent stake with no bonuses, enabling a ~2% token minority to achieve 100% effective voting power.

### Finding Description

`NervousSystemParameters::validate()` calls `validate_max_dissolve_delay_bonus_percentage` and `validate_max_age_bonus_percentage` as independent checks: [1](#0-0) 

Each individual validator enforces only its own ceiling: [2](#0-1) [3](#0-2) 

The ceilings are: [4](#0-3) 

The voting-power formula in `Neuron::voting_power` applies the two bonuses multiplicatively:

```
d_stake  = stake  × (1 + dissolve_delay_bonus_percentage/100)   // up to 10×
ad_stake = d_stake × (1 + age_bonus_percentage/100)             // up to 5×
``` [5](#0-4) 

At the allowed ceilings the combined multiplier is `10 × 5 = 50×`. The same uncombined validation is duplicated in `SnsInitPayload` validation: [6](#0-5) 

### Impact Explanation

An SNS initialized with `max_dissolve_delay_bonus_percentage = 900` and `max_age_bonus_percentage = 400` (both individually valid) produces a 50× voting-power multiplier for neurons at maximum dissolve delay and maximum age. A founding team holding 2% of the token supply in such neurons would command 100% of effective voting power, bypassing the SNS governance authorization model entirely. All subsequent governance proposals — including treasury transfers, canister upgrades, and parameter changes — would be under unilateral control of that minority.

### Likelihood Explanation

SNS initialization parameters are submitted by the SNS creator as an `SnsInitPayload`. Both bonus fields pass individual validation and the NNS governance proposal that deploys the SNS has no mechanism to detect the combined 50× effect. The SNS creator can set these values intentionally or inadvertently. After deployment, the same parameters can be raised further via `ManageNervousSystemParameters` proposals if the founding team already holds disproportionate voting power.

### Recommendation

Add a combined validation step in `NervousSystemParameters::validate()` (and the corresponding `SnsInitPayload` validation) that checks the product of the two bonus multipliers against a safe aggregate ceiling, for example:

```rust
let combined_max_multiplier =
    (100 + max_dissolve_delay_bonus_percentage)
        .saturating_mul(100 + max_age_bonus_percentage);
// e.g. reject if combined_max_multiplier > 100 * 100 * SOME_SAFE_CEILING
```

Alternatively, lower the individual ceilings so that their product cannot produce a governance-distorting multiplier (e.g., cap the combined multiplier at 10×, matching NNS defaults of 100% dissolve-delay bonus and 25% age bonus giving `2 × 1.25 = 2.5×`).

### Proof of Concept

1. Deploy an SNS with `SnsInitPayload { max_dissolve_delay_bonus_percentage: Some(900), max_age_bonus_percentage: Some(400), .. }`. Both fields pass `validate_post_execution` individually.
2. The founding team stakes 2% of total supply in neurons locked at `max_dissolve_delay_seconds` with age ≥ `max_neuron_age_for_age_bonus`.
3. Call `compute_ballots_for_new_proposal`. The founding team's neurons receive `stake × 10 × 5 = stake × 50` voting power.
4. With 2% of supply × 50× multiplier = 100% of effective voting power, every proposal passes or fails at the founding team's discretion, regardless of the remaining 98% of token holders. [7](#0-6) [4](#0-3) [6](#0-5)

    pub fn voting_power(
        &self,
        now_seconds: u64,
        max_dissolve_delay_seconds: u64,
        max_neuron_age_for_age_bonus: u64,
        max_dissolve_delay_bonus_percentage: u64,
        max_age_bonus_percentage: u64,
    ) -> u64 {
        // We compute the stake adjustments in u128.
        let stake = self.voting_power_stake_e8s() as u128;
        // Dissolve delay is capped to max_dissolve_delay_seconds, but we cap it
        // again here to make sure, e.g., if this changes in the future.
        let d = std::cmp::min(
            self.dissolve_delay_seconds(now_seconds),
            max_dissolve_delay_seconds,
        ) as u128;
        // 'd_stake' is the stake with bonus for dissolve delay.
        let d_stake = stake
            + if max_dissolve_delay_seconds > 0 {
                (stake * d * max_dissolve_delay_bonus_percentage as u128)
                    / (100 * max_dissolve_delay_seconds as u128)
            } else {
                0
            };
        // Sanity check.
        assert!(d_stake <= stake + (stake * (max_dissolve_delay_bonus_percentage as u128) / 100));
        // The voting power is also a function of the age of the
        // neuron, giving a bonus of up to max_age_bonus_percentage at max_neuron_age_for_age_bonus.
        let a = std::cmp::min(self.age_seconds(now_seconds), max_neuron_age_for_age_bonus) as u128;
        let ad_stake = d_stake
            + if max_neuron_age_for_age_bonus > 0 {
                (d_stake * a * max_age_bonus_percentage as u128)
                    / (100 * max_neuron_age_for_age_bonus as u128)
            } else {
                0
            };
        // Final stake 'ad_stake' has is not more than max_age_bonus_percentage above 'd_stake'.
        assert!(ad_stake <= d_stake + (d_stake * (max_age_bonus_percentage as u128) / 100));

        // Convert the multiplier to u128. The voting_power_percentage_multiplier represents
        // a percent and will always be within the range 0 to 100.
        let v = self.voting_power_percentage_multiplier as u128;

        // Apply the multiplier to 'ad_stake' and divide by 100 to have the same effect as
        // multiplying by a percent.
        let vad_stake = ad_stake
            .checked_mul(v)
            .expect("Overflow detected when calculating voting power")
            .checked_div(100)
            .expect("Underflow detected when calculating voting power");

        // The final voting power is the stake adjusted by both age,
        // dissolve delay, and voting power multiplier. If the stake is is greater than
        // u64::MAX divided by 2.5, the voting power may actually not
        // fit in a u64.
        std::cmp::min(vad_stake, u64::MAX as u128) as u64
    }
