Audit Report

## Title
NNS `spawn_neuron` With `new_controller` Unconditionally Inherits Parent Hot Keys, Granting Previous Owner Unauthorized Governance Access - (`rs/nns/governance/src/governance.rs`)

## Summary
When `spawn_neuron` is called with a `new_controller` that differs from the parent's controller, the child neuron is built with the new controller but unconditionally copies the parent's full hot key list. The previous owner's hot keys remain authorized to vote, follow, and join/leave the Neuron Fund on the child neuron without the new controller's knowledge or consent.

## Finding Description
In `spawn_neuron` (`rs/nns/governance/src/governance.rs`), `child_controller` is correctly set to `spawn.new_controller` when provided (L2651–2655), but the `NeuronBuilder` call at L2704–2718 unconditionally calls `.with_hot_keys(parent_neuron.hot_keys.clone())` regardless of whether `child_controller` differs from the parent's controller. There is no conditional clearing of hot keys on ownership transfer.

The inherited hot keys pass the `is_hotkey_or_controller` check in `neuron/types.rs` (L254–255), which is used by:
- `is_authorized_to_vote` (L243–244) — allows voting on NNS proposals on behalf of the child neuron
- `is_authorized_to_configure_or_err` for `JoinCommunityFund`/`LeaveCommunityFund` (L780–781) — allows enrolling the child neuron in the Neuron Fund

Exploit path:
1. Alice creates a neuron and adds a hot key (e.g., hardware wallet).
2. Alice calls `manage_neuron` with `Command::Spawn(Spawn { new_controller: Some(Bob), ... })`.
3. Governance creates a child neuron with `controller = Bob` but `hot_keys = [Alice's hardware wallet]`.
4. Alice's hardware wallet calls `manage_neuron` on the child neuron with `JoinCommunityFund`.
5. `is_hotkey_or_controller` passes for Alice's key; Bob's neuron is enrolled in the Neuron Fund without Bob's consent.
6. Bob's neuron's maturity is now subject to SNS swap participation, reducing its value.

No existing guard checks whether `child_controller != parent_neuron.controller()` before copying hot keys.

## Impact Explanation
This is **High** severity: unauthorized access to a neuron's governance assets. Alice's retained hot keys can vote on NNS proposals on behalf of Bob's neuron, manipulate following rules, and — most concretely — enroll Bob's neuron in the Neuron Fund, causing Bob's maturity to be committed to SNS swaps without consent. This constitutes unauthorized access to neuron governance assets and potential maturity loss, matching the High bounty impact: *"Unauthorized access to neurons, governance assets, wallets, identities, ledgers, or canister-controlled funds."*

## Likelihood Explanation
Reachable by any unprivileged neuron controller. The `Spawn` command with `new_controller` is a documented, production-accessible NNS operation. Alice requires no special privileges beyond having previously owned the parent neuron and having added a hot key. The attack is free and repeatable for Alice after any spawn-with-new-controller transfer.

## Recommendation
When `spawn_neuron` is called with a `new_controller` that differs from the parent's controller, initialize the child neuron with an empty hot key list:

```rust
let hot_keys = if child_controller == parent_neuron.controller() {
    parent_neuron.hot_keys.clone()
} else {
    vec![]
};
// ...
.with_hot_keys(hot_keys)
```

This mirrors the correct behavior expected for any ownership-transfer operation and allows the new controller to add their own trusted keys.

## Proof of Concept
1. Deploy a local replica or PocketIC test environment.
2. Alice creates a neuron and calls `AddHotKey` to add a second principal as a hot key.
3. Alice accumulates maturity on the neuron.
4. Alice calls `manage_neuron` with `Command::Spawn(Spawn { new_controller: Some(Bob), percentage_to_spawn: Some(100), nonce: Some(42) })`.
5. Inspect the child neuron: assert `controller == Bob` and `hot_keys == [Alice's hot key]`.
6. From Alice's hot key principal, call `manage_neuron` on the child neuron with `Command::Configure(Configure { operation: Some(JoinCommunityFund(JoinCommunityFund {})) })`.
7. Assert the call succeeds and `child_neuron.joined_community_fund_timestamp_seconds` is set — confirming Bob's neuron was enrolled in the Neuron Fund without Bob's action.
