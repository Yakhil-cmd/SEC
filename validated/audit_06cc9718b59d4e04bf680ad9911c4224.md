### Title
Missing Zero-Address Validation in `ExitToEthereum` Precompile Allows Permanent Loss of User Funds - (File: engine-precompiles/src/native.rs)

### Summary
The `ExitToEthereum` precompile in Aurora Engine does not validate that the caller-supplied Ethereum `recipient_address` is non-zero before creating a NEAR withdrawal promise. Any EVM user can trigger a withdrawal of ETH or ERC-20 bridge tokens to `address(0)` on Ethereum, permanently destroying the funds with no recovery path.

### Finding Description
The `ExitToEthereum` precompile (`0xb0bd02f6a392af548bdf1cfaee5dfa0eefcc8eab`) handles two withdrawal flows. In both, the recipient Ethereum address is parsed directly from caller-controlled input with no zero-address guard.

**Flag `0x00` — ETH (base token) exit:**

```rust
let recipient_address: Address = input
    .try_into()
    .map_err(|_| ExitError::Other(Cow::from("ERR_INVALID_RECIPIENT_ADDRESS")))?;
// No check: recipient_address != Address::zero()
serialize_fn(recipient_address, context.apparent_value)?
``` [1](#0-0) 

The only validation is that the slice is exactly 20 bytes. Twenty zero bytes pass this check and produce `Address::zero()`.

**Flag `0x01` — ERC-20 exit:**

```rust
let recipient_address = Address::try_from_slice(input)
    .map_err(|_| ExitError::Other(Cow::from("ERR_WRONG_ADDRESS")))?;
// No check: recipient_address != Address::zero()
``` [2](#0-1) 

Again, twenty zero bytes are accepted as a valid recipient.

In both cases the precompile proceeds to build a `withdraw` NEAR promise targeting the ETH connector with the zero address as the Ethereum recipient:

```rust
let withdraw_promise = PromiseCreateArgs {
    target_account_id: nep141_address,
    method: "withdraw".to_string(),
    args: serialized_args,   // contains recipient = 0x000...000
    attached_balance: Yocto::new(1),
    attached_gas: costs::WITHDRAWAL_GAS,
};
``` [3](#0-2) 

For the ERC-20 path, the burn happens in `EvmErc20.withdrawToEthereum` / `EvmErc20V2.withdrawToEthereum` **before** the precompile is called, and the assembly return value is not checked:

```solidity
function withdrawToEthereum(address recipient, uint256 amount) external override {
    _burn(_msgSender(), amount);          // tokens destroyed first
    ...
    assembly {
        let res := call(gas(), 0xb0bd02f6..., 0, add(input, 32), input_size, 0, 32)
        // res is never checked
    }
}
``` [4](#0-3) [5](#0-4) 

### Impact Explanation
**Permanent freezing of funds (Critical).**

- **ERC-20 path**: The caller's ERC-20 mirror tokens are burned on Aurora. The ETH connector then processes a withdrawal to `address(0)` on Ethereum, sending the corresponding ETH to the unrecoverable zero address. Even if the ETH connector rejects the zero-address withdrawal, the ERC-20 tokens are already burned on Aurora and the `error_refund` callback path (gated behind a compile-time feature flag) may not be active in all deployments, leaving the user with no tokens on either chain.
- **ETH base token path**: The caller's ETH balance on Aurora is debited and a withdrawal promise to `address(0)` on Ethereum is created. The ETH is permanently lost.

In both cases the user's funds are irreversibly destroyed with no protocol-level recovery mechanism.

### Likelihood Explanation
**Low-Medium.** A user must supply `address(0)` as the recipient. This can occur:
1. By user mistake (e.g., uninitialized variable, copy-paste error).
2. Via a malicious or buggy intermediary contract that passes `address(0)` without validating the recipient before calling `withdrawToEthereum`.
3. Via a front-end or SDK bug that fails to populate the recipient field.

The precompile is publicly callable by any EVM account, and the `EvmErc20`/`EvmErc20V2` contracts expose `withdrawToEthereum` as an unrestricted external function.

### Recommendation
Add an explicit zero-address guard in `ExitToEthereum::run` for both the `0x00` and `0x01` branches, immediately after parsing `recipient_address`:

```rust
if recipient_address == Address::zero() {
    return Err(ExitError::Other(Cow::from("ERR_ZERO_RECIPIENT_ADDRESS")));
}
```

Additionally, consider adding the same guard in `EvmErc20.withdrawToEthereum` and `EvmErc20V2.withdrawToEthereum` at the Solidity level:

```solidity
require(recipient != address(0), "ERR_ZERO_RECIPIENT");
```

### Proof of Concept
**ERC-20 path:**
1. Deploy or use an existing `EvmErc20` / `EvmErc20V2` mirror token on Aurora.
2. Hold a non-zero balance of that token.
3. Call `withdrawToEthereum(address(0), amount)`.
4. Observe: `_burn` destroys `amount` tokens from the caller's balance on Aurora; the `ExitToEthereum` precompile creates a NEAR promise to `withdraw` with recipient `0x0000000000000000000000000000000000000000`; the ETH connector processes the withdrawal to the zero address on Ethereum. Funds are permanently lost.

**ETH base token path:**
1. Craft calldata: `\x00` + `\x00` × 20 (flag byte + 20-byte zero address).
2. Send an EVM transaction calling the `ExitToEthereum` precompile at `0xb0bd02f6a392af548bdf1cfaee5dfa0eefcc8eab` with `value > 0` and the above calldata.
3. Observe: the precompile accepts the input, debits the caller's ETH balance on Aurora, and creates a withdrawal promise to `address(0)` on Ethereum. ETH is permanently lost. [6](#0-5) [4](#0-3) [5](#0-4)

impl<I: IO> Precompile for ExitToEthereum<I> {
    fn required_gas(_input: &[u8]) -> Result<EthGas, ExitError> {
        Ok(costs::EXIT_TO_ETHEREUM_GAS)
    }

    #[allow(clippy::too_many_lines)]
    fn run(
        &self,
        input: &[u8],
        target_gas: Option<EthGas>,
        context: &Context,
        is_static: bool,
    ) -> EvmPrecompileResult {
        // ETH (Base token) transfer input format (min size 21 bytes)
        //  - flag (1 byte)
        //  - eth_recipient (20 bytes)
        // ERC-20 transfer input format: max 53 bytes
        //  - flag (1 byte)
        //  - amount (32 bytes)
        //  - eth_recipient (20 bytes)
        validate_input_size(input, 21, 53)?;

        let required_gas = Self::required_gas(input)?;

        if let Some(target_gas) = target_gas
            && required_gas > target_gas
        {
            return Err(ExitError::OutOfGas);
        }

        // It's not allowed to call exit precompiles in static mode
        if is_static {
            return Err(ExitError::Other(Cow::from("ERR_INVALID_IN_STATIC")));
        } else if context.address != exit_to_ethereum::ADDRESS.raw() {
            return Err(ExitError::Other(Cow::from("ERR_INVALID_IN_DELEGATE")));
        }

        // The first byte of the input is a flag, selecting the behavior to be triggered:
        //  0x00 -> ETH (Base token) token transfer
        //  0x01 -> ERC-20 transfer
        let mut input = input;
        let flag = input[0];
        input = &input[1..];

        let (nep141_address, serialized_args, exit_event) = match flag {
            0x0 => {
                // ETH (base) transfer
                //
                // Input slice format:
                //  eth_recipient (20 bytes) - the address of recipient which will receive ETH on Ethereum
                let recipient_address: Address = input
                    .try_into()
                    .map_err(|_| ExitError::Other(Cow::from("ERR_INVALID_RECIPIENT_ADDRESS")))?;
                let serialize_fn = match get_withdraw_serialize_type(&self.io)? {
                    WithdrawSerializeType::Json => json_args,
                    WithdrawSerializeType::Borsh => borsh_args,
                };
                let eth_connector_account_id = self.get_eth_connector_contract_account()?;

                (
                    eth_connector_account_id,
                    // There is no way to inject json, given the encoding of both arguments
                    // as decimal and hexadecimal respectively.
                    serialize_fn(recipient_address, context.apparent_value)?,
                    events::ExitToEth {
                        sender: Address::new(context.caller),
                        erc20_address: events::ETH_ADDRESS,
                        dest: recipient_address,
                        amount: context.apparent_value,
                    },
                )
            }
            0x1 => {
                // ERC-20 transfer
                //
                // This precompile branch is expected to be called from the ERC20 withdraw function
                // (or burn function with some flag provided that this is expected to be withdrawn)
                //
                // Input slice format:
                //  amount (U256 big-endian bytes) - the amount that was burned
                //  eth_recipient (20 bytes) - the address of recipient which will receive ETH on Ethereum

                if context.apparent_value != U256::from(0) {
                    return Err(ExitError::Other(Cow::from(
                        "ERR_ETH_ATTACHED_FOR_ERC20_EXIT",
                    )));
                }

                let erc20_address = context.caller;
                let nep141_address = get_nep141_from_erc20(erc20_address.as_bytes(), &self.io)?;
                let amount = parse_amount(&input[..32])?;

                input = &input[32..];

                if input.len() == 20 {
                    // Parse ethereum address in hex
                    let mut buffer = [0; 40];
                    hex::encode_to_slice(input, &mut buffer).unwrap();
                    let recipient_in_hex = str::from_utf8(&buffer).map_err(|_| {
                        ExitError::Other(Cow::from("ERR_INVALID_RECIPIENT_ADDRESS"))
                    })?;
                    // unwrap cannot fail since we checked the length already
                    let recipient_address = Address::try_from_slice(input)
                        .map_err(|_| ExitError::Other(Cow::from("ERR_WRONG_ADDRESS")))?;

                    (
                        nep141_address,
                        // There is no way to inject json, given the encoding of both arguments
                        // as decimal and hexadecimal respectively.
                        format!(
                            r#"{{"amount": "{}", "recipient": "{}"}}"#,
                            amount.as_u128(),
                            recipient_in_hex
                        )
                        .into_bytes(),
                        events::ExitToEth {
                            sender: Address::new(erc20_address),
                            erc20_address: Address::new(erc20_address),
                            dest: recipient_address,
                            amount,
                        },
                    )
                } else {
                    return Err(ExitError::Other(Cow::from("ERR_INVALID_RECIPIENT_ADDRESS")));
                }
            }
            _ => {
                return Err(ExitError::Other(Cow::from(
                    "ERR_INVALID_RECEIVER_ACCOUNT_ID",
                )));
            }
        };

        let withdraw_promise = PromiseCreateArgs {
            target_account_id: nep141_address,
            method: "withdraw".to_string(),
            args: serialized_args,
            attached_balance: Yocto::new(1),
            attached_gas: costs::WITHDRAWAL_GAS,
        };

        let promise = borsh::to_vec(&PromiseArgs::Create(withdraw_promise)).unwrap();
        let promise_log = Log {
            address: exit_to_ethereum::ADDRESS.raw(),
            topics: Vec::new(),
            data: promise,
        };
        let ethabi::RawLog { topics, data } = exit_event.encode();
        let exit_event_log = Log {
            address: exit_to_ethereum::ADDRESS.raw(),
            topics: topics.into_iter().map(|h| H256::from(h.0)).collect(),
            data,
        };

        Ok(PrecompileOutput {
            logs: vec![promise_log, exit_event_log],
            cost: required_gas,
            output: Vec::new(),
        })
    }
