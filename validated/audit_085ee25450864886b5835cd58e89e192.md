### Title
Unchecked Precompile Call Return Value in `withdrawToNear`/`withdrawToEthereum` Causes Permanent Token Loss - (File: `etc/eth-contracts/contracts/EvmErc20.sol`, `etc/eth-contracts/contracts/EvmErc20V2.sol`)

---

### Summary

Both `EvmErc20.sol` and `EvmErc20V2.sol` burn ERC-20 tokens from the caller **before** invoking the Aurora exit precompile via inline assembly, and **never check the return value** of that assembly `call`. If the precompile call fails for any reason (invalid NEAR recipient, amount overflow, out-of-gas, etc.), the tokens are permanently destroyed with no corresponding NEP-141 transfer to NEAR. The user suffers an irreversible total loss of the bridged assets.

---

### Finding Description

In `EvmErc20.sol`, the `withdrawToNear` function:

```solidity
function withdrawToNear(bytes memory recipient, uint256 amount) external override {
    _burn(_msgSender(), amount);                          // ← tokens destroyed here

    bytes32 amount_b = bytes32(amount);
    bytes memory input = abi.encodePacked("\x01", amount_b, recipient);
    uint input_size = 1 + 32 + recipient.length;

    assembly {
        let res := call(gas(), 0xe9217bc70b7ed1f598ddd3199e80b093fa71124f, 0, add(input, 32), input_size, 0, 32)
        // res is NEVER checked — no require(res != 0)
    }
}
``` [1](#0-0) 

The identical pattern exists in `EvmErc20V2.sol`: [2](#0-1) 

And in `withdrawToEthereum` in both contracts: [3](#0-2) 

The `ExitToNear` precompile (`engine-precompiles/src/native.rs`) can return `Err(ExitError::Other(...))` for multiple reasons:

- `parse_recipient` fails when the recipient bytes are not a valid NEAR account ID.
- `parse_amount` fails when `amount > u128::MAX`.
- Input size validation fails when `recipient.length > MAX_INPUT_SIZE - 33`.
- `get_nep141_from_erc20` fails if the ERC-20 → NEP-141 mapping is absent. [4](#0-3) 

When the precompile returns an error, the EVM `call` opcode returns `0` to the caller. In standard EVM semantics, a failed sub-call does **not** automatically revert the caller. Because the Solidity code never inspects `res`, execution continues normally after the assembly block, and the function returns without reverting. The `_burn` that already executed is irreversible.

The `error_refund` callback mechanism in the engine only activates when the precompile **succeeds** in creating a NEAR promise that subsequently fails on the NEAR side. It provides no protection when the precompile itself fails at the EVM level and no promise is ever scheduled. [5](#0-4) 

---

### Impact Explanation

**Critical — Permanent freezing/loss of funds.**

When `withdrawToNear` or `withdrawToEthereum` is called with any input that causes the precompile to fail:

1. The caller's ERC-20 balance is permanently reduced by `amount` (burned).
2. No NEAR promise is created; no NEP-141 tokens are transferred to the recipient.
3. The transaction succeeds from the EVM's perspective (no revert).
4. The user has lost their bridged assets with no recovery path short of a contract upgrade.

The NEP-141 tokens remain locked in the Aurora Engine contract on NEAR with no mechanism to release them to the original owner.

---

### Likelihood Explanation

**Medium.**

Any unprivileged EVM user holding bridged ERC-20 tokens can trigger this by:

- Supplying a `recipient` byte string that is not a valid NEAR account ID (e.g., empty, too long > 64 bytes, or containing invalid characters). NEAR account ID validation is strict and easy to violate accidentally.
- Supplying `amount > type(uint128).max` (the precompile rejects amounts exceeding `u128::MAX` even though `uint256` allows larger values). [6](#0-5) 

No special privileges are required. The call path is: user → `EvmErc20.withdrawToNear` → `_burn` (succeeds) → assembly `call` to precompile (fails silently).

---

### Recommendation

1. **Check the precompile call return value and revert on failure.** Replace the unchecked assembly block with:

```solidity
assembly {
    let res := call(gas(), 0xe9217bc70b7ed1f598ddd3199e80b093fa71124f, 0, add(input, 32), input_size, 0, 32)
    if iszero(res) { revert(0, 0) }
}
```

This ensures the `_burn` is atomically rolled back if the precompile call fails.

2. **Apply the same fix to `withdrawToEthereum`** in both `EvmErc20.sol` and `EvmErc20V2.sol`.

3. **Alternatively**, move the `_burn` call to after the precompile call succeeds, or use a two-step pattern (lock → confirm → burn).

---

### Proof of Concept

1. Deploy an `EvmErc20` contract (as Aurora does for any bridged NEP-141 token).
2. Mint tokens to address `Alice` via the admin `mint` function.
3. Alice calls `withdrawToNear(bytes(""), amount)` — an empty recipient is not a valid NEAR account ID.
4. Inside `withdrawToNear`:
   - `_burn(Alice, amount)` executes successfully; Alice's balance drops to 0.
   - The assembly `call` to the ExitToNear precompile is made with an empty recipient.
   - The precompile's `parse_recipient` returns `Err(ExitError::Other("ERR_INVALID_RECEIVER_ACCOUNT_ID"))`.
   - The EVM `call` opcode returns `res = 0`.
   - The assembly block exits; `res` is never read.
   - The function returns normally.
5. Alice's ERC-20 balance is 0. No NEAR promise was created. No NEP-141 tokens were transferred. Alice's funds are permanently lost. [1](#0-0) [7](#0-6)

fn parse_amount(input: &[u8]) -> Result<U256, ExitError> {
    let amount = U256::from_big_endian(input);

    if amount > U256::from(u128::MAX) {
        return Err(ExitError::Other(Cow::from("ERR_INVALID_AMOUNT")));
    }

    Ok(amount)
}

fn parse_recipient(recipient: &[u8]) -> Result<Recipient<'_>, ExitError> {
    let recipient = str::from_utf8(recipient)
        .map_err(|_| ExitError::Other(Cow::from("ERR_INVALID_RECEIVER_ACCOUNT_ID")))?;
    let (receiver_account_id, message) = recipient.split_once(':').map_or_else(
        || (recipient, None),
        |(recipient, msg)| {
            if msg == UNWRAP_WNEAR_MSG {
                (recipient, Some(Message::UnwrapWnear))
            } else {
                (recipient, Some(Message::Omni(msg)))
            }
        },
    );

    Ok(Recipient {
        receiver_account_id: receiver_account_id
            .parse()
            .map_err(|_| ExitError::Other(Cow::from("ERR_INVALID_RECEIVER_ACCOUNT_ID")))?,
        message,
    })
}

pub fn exit_to_near_precompile_callback<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<Option<SubmitResult>, ContractError> {
    with_hashchain(io, env, function_name!(), |io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;
        env.assert_private_call()?;

        // This function should only be called as the callback of
        // exactly one promise.
        if handler.promise_results_count() != 1 {
            return Err(errors::ERR_PROMISE_COUNT.into());
        }

        let args: ExitToNearPrecompileCallbackArgs = io.read_input_borsh()?;

        let maybe_result = if let Some(PromiseResult::Successful(_)) = handler.promise_result(0) {
            if let Some(args) = args.transfer_near {
                let action = PromiseAction::Transfer {
                    amount: Yocto::new(args.amount),
                };
                let promise = PromiseBatchAction {
                    target_account_id: args.target_account_id,
                    actions: vec![action],
                };

                // Safety: this call is safe because it comes from the exit to near precompile, not users.
                // The call is to transfer the unwrapped wNEAR tokens.
                let promise_id = handler.promise_create_batch(&promise);
                handler.promise_return(promise_id);
            }

            None
        } else if let Some(args) = args.refund {
            // Exit call failed; need to refund tokens
            let refund_result = engine::refund_on_error(io, env, state, &args, handler)?;

            if !refund_result.status.is_ok() {
                return Err(errors::ERR_REFUND_FAILURE.into());
            }

            Some(refund_result)
        } else {
            None
        };

        Ok(maybe_result)
    })
}
