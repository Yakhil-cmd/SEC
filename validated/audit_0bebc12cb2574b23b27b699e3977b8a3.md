### Title
`new` re-initialization guard bypassed on corrupted engine state — (`engine/src/contract_methods/admin.rs`)

### Summary
`new` uses `state::get_state(&io).is_ok()` to block re-initialization. When the STATE storage key holds bytes that fail Borsh deserialization (e.g., after a buggy `state_migration`), `get_state` returns `Err(DeserializationFailed)`, `is_ok()` is `false`, the guard is skipped, and an attacker can overwrite the engine owner with attacker-controlled `NewCallArgs`.

### Finding Description
`state::get_state` distinguishes three outcomes: `Ok(state)` (key present, valid bytes), `Err(NotFound)` (key absent), and `Err(DeserializationFailed)` (key present, bytes corrupt). The guard in `new` only blocks execution on `Ok`; both error variants allow execution to continue. The `NotFound` case is the intended "uninitialized" path. The `DeserializationFailed` case — which arises when a state migration writes a format incompatible with `BorshableEngineState` and `BorshableEngineStateV1` — is incorrectly treated as equivalent to "not initialized," allowing `new` to overwrite the existing (corrupted) state.

### Impact Explanation
After `new` completes with attacker-supplied `owner_id`, the attacker passes `require_owner_only` in `attach_full_access_key`, `upgrade`, `set_owner`, and `stage_upgrade`. `attach_full_access_key` adds a full-access key to the engine account, giving the attacker unrestricted control over all funds held by the engine contract. This is direct theft of all user funds at rest.

### Likelihood Explanation
The precondition — corrupted STATE bytes — requires a prior buggy `state_migration`. This is not an everyday event, but it is a realistic production risk: any upgrade that writes a new state schema (as demonstrated by the in-repo `state-migration-test`) leaves the STATE key in a format unreadable by the old Borsh paths. The window between a failed/incompatible migration and legitimate recovery is the attack surface. No special privileges are needed by the attacker; calling `new` is a public contract method.

### Recommendation
Replace the `is_ok()` check with an explicit presence check on the raw storage key. Re-initialization must be blocked whenever the STATE key exists in storage, regardless of whether its contents are deserializable:

```rust
// In new():
// Instead of:
if state::get_state(&io).is_ok() {
    return Err(b"ERR_ALREADY_INITIALIZED".into());
}

// Use:
if io.read_storage(&bytes_to_key(KeyPrefix::Config, STATE_KEY)).is_some() {
    return Err(b"ERR_ALREADY_INITIALIZED".into());
}
```

This ensures that any bytes at the STATE key — valid, legacy, or corrupted — block re-initialization.

pub fn upgrade<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<(), ContractError> {
    let state = state::get_state(&io)?;
    require_running(&state)?;
    require_owner_only(&state, &env.predecessor_account_id())?;

    let input = io.read_input().to_vec();
    let (code, state_migration_gas) = match UpgradeParams::try_from_slice(&input) {

    };

    let target_account_id = env.current_account_id();
    let batch = PromiseBatchAction {
        target_account_id,

            PromiseAction::FunctionCall {
                name: "state_migration".to_string(),
                args: vec![],
                attached_yocto: ZERO_YOCTO,
                gas: state_migration_gas,
            },
        ],
    };
