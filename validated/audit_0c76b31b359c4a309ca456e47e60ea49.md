Audit Report

## Title
Unbounded assemble task spawning via `SLOT_TABLE_NO_LIMIT` allows a single Byzantine peer to exhaust replica memory — (`rs/p2p/consensus_manager/src/receiver.rs`, `rs/replica/setup_ic_network/src/lib.rs`)

## Summary
`SLOT_TABLE_NO_LIMIT = usize::MAX` is the production `slot_limit` for consensus, certification, DKG, IDKG, and HTTPS outcalls artifact types. The per-peer slot table guard in `handle_slot_update_receive` is effectively disabled for these types, allowing a single Byzantine subnet peer to insert an unbounded number of slot entries and spawn an unbounded number of Tokio assemble tasks, exhausting replica memory and crashing the process.

## Finding Description
In `rs/replica/setup_ic_network/src/lib.rs` lines 74–75, two constants are defined:

```rust
const SLOT_TABLE_LIMIT_INGRESS: usize = 50_000;
const SLOT_TABLE_NO_LIMIT: usize = usize::MAX;
```

`SLOT_TABLE_LIMIT_INGRESS` is passed only for ingress (line 257). All other artifact types — consensus (lines 237, 246), certifier (line 268), DKG (line 279), IDKG (line 291), and HTTPS outcalls (line 303) — receive `SLOT_TABLE_NO_LIMIT`.

In `rs/p2p/consensus_manager/src/receiver.rs`, `handle_slot_update_receive` (line 373) guards new slot insertions with:

```rust
Entry::Vacant(empty_slot) if peer_slot_table_len < self.slot_limit => { ... }
```

With `slot_limit = usize::MAX`, this condition is always true: no process can hold `usize::MAX` HashMap entries before OOM kills it. The fallthrough `Entry::Vacant(_)` branch (lines 381–390) that drops the update and logs a warning is never reached for non-ingress types.

When `to_add` is true and the artifact ID is not already in `active_assembles` (line 394), a new task is unconditionally spawned into `artifact_processor_tasks` (a `JoinSet`, lines 407–419) and a new entry is inserted into `active_assembles` (line 405). Both `slot_table` (`HashMap<NodeId, HashMap<SlotNumber, SlotEntry>>`, line 187) and `active_assembles` (`HashMap<WireArtifact::Id, watch::Sender<PeerCounter>>`, line 188) are unbounded for non-ingress types.

A Byzantine peer sends N slot updates each with a distinct `slot_number` and a distinct artifact ID: each passes the guard, each inserts into the peer's slot table, and each spawns a new `process_slot_update` task — all without bound.

## Impact Explanation
This is a **High** severity finding matching the allowed impact: *"Application/platform-level DoS, crash, consensus blocking, certified-state disruption, or subnet availability impact not based on raw volumetric DDoS."*

Each spawned task holds a `watch::Receiver<PeerCounter>`, a cloned `Assembler`, a cloned `Sender`, and may initiate network fetch operations. At scale (e.g., 10^6 tasks), heap memory and Tokio task scheduler capacity are exhausted, crashing the replica process. A crashed replica halts local consensus participation; if enough replicas are affected, subnet finalization stalls.

## Likelihood Explanation
The attacker is a single compromised subnet node — a realistic Byzantine adversary below the consensus fault threshold (f < n/3). QUIC/TLS authenticates the sender's `NodeId` but imposes no rate limit or per-peer slot count limit on slot update messages for non-ingress artifact types. The attack requires only sending crafted P2P slot update messages, which is within the capability of any subnet peer. The attack is repeatable and requires no victim mistakes.

## Recommendation
Apply a bounded `slot_limit` for all artifact types, not just ingress. The appropriate bound should reflect the maximum number of in-flight artifacts a legitimate peer would ever advertise simultaneously for each type (e.g., bounded by consensus round depth, DKG epoch size, IDKG transcript count, etc.). The existing `SLOT_TABLE_LIMIT_INGRESS = 50_000` pattern should be replicated with type-appropriate constants for consensus, DKG, IDKG, certification, and HTTPS outcalls, replacing all uses of `SLOT_TABLE_NO_LIMIT` in `rs/replica/setup_ic_network/src/lib.rs`.

## Proof of Concept
State-machine test (no network required):

```rust
let (mut mgr, _channels) = ReceiverManagerBuilder::new()
    .with_slot_limit(usize::MAX)
    .build();
let cancellation = CancellationToken::new();
for i in 0..1_000_000u64 {
    mgr.handle_slot_update_receive(
        SlotUpdate {
            slot_number: SlotNumber::from(i),
            commit_id: CommitId::from(i),
            update: Update::Id(i), // distinct artifact ID per slot
        },
        NODE_1,
        ConnId::from(1),
        cancellation.clone(),
    );
}
assert_eq!(mgr.active_assembles.len(), 1_000_000);
assert_eq!(mgr.artifact_processor_tasks.len(), 1_000_000);
