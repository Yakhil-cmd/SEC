Audit Report

## Title
SNS Swap Canister Has No SNS Token Recovery Path on Aborted Swap — (File: rs/sns/swap/src/swap.rs)

## Summary
When an SNS decentralization swap is aborted, `finalize_inner` refunds ICP to buyers and restores dapp controllers, then returns early without sweeping the pre-loaded SNS token allocation back to any treasury. The SNS tokens remain in the swap canister's ledger account indefinitely, with no callable method on the swap canister capable of moving them out in the `Aborted` lifecycle. The protocol lifecycle diagram explicitly shows `ABORTED → <DELETED>` as the terminal path, meaning eventual deletion of the swap canister would render the entire SNS token swap allocation permanently inaccessible.

## Finding Description
In `finalize_inner` (`rs/sns/swap/src/swap.rs`, L1544–1624), the aborted path executes:
1. `sweep_icp` — refunds ICP to buyers
2. `settle_neurons_fund_participation` — refunds Neurons' Fund maturity
3. `restore_dapp_controllers_for_finalize` — restores dapp control

`should_restore_dapp_control()` returns `true` exactly when `lifecycle() == Lifecycle::Aborted`, so the function unconditionally returns at L1583 without performing any SNS token transfer.

`sweep_sns` (L2165–2178) contains an explicit guard that halts execution if the lifecycle is not `Committed`, making it impossible to call for SNS token recovery in the aborted state. No alternative SNS token transfer method exists on the swap canister for the aborted lifecycle.

The integration test at `rs/nervous_system/integration_tests/tests/sns_lifecycle.rs` L1336–1342 explicitly asserts `swap_canister_balance_sns_e8s == swap_distribution_sns_e8s` after abort — confirming the full allocation remains in the swap canister with no expectation of recovery.

The protocol documentation (`rs/sns/swap/proto/ic_sns_swap/pb/v1/swap.proto`, L149–150) states "The swap canister can be deleted when all tokens registered with the swap canister have been disbursed to their rightful owners," yet the aborted path leaves SNS tokens undisbursed, creating a direct contradiction with the `ABORTED → <DELETED>` terminal lifecycle path.

## Impact Explanation
The entire SNS token swap allocation — which can represent millions of governance tokens — is permanently stranded in the swap canister's ledger account. The swap canister is the sole principal authorized to transfer from its SNS ledger account. After finalization in the aborted state, no callable method on the swap canister can authorize such a transfer. Once the swap canister is deleted (the documented terminal state), the tokens are permanently inaccessible on the SNS ledger. This constitutes a significant SNS framework funds-loss impact with concrete harm to the SNS project and its token holders. This matches the allowed High impact: "Significant... SNS... security impact with concrete user or protocol harm."

## Likelihood Explanation
No attacker action is required. Any SNS swap that fails to reach `min_participants` or `min_icp_e8s` before the deadline transitions to `Aborted` via normal protocol operation. This is a documented, realistic lifecycle path. New or low-visibility SNS projects are particularly susceptible. The trigger is automatic and unprivileged.

## Recommendation
In `finalize_inner`, within the `should_restore_dapp_control()` branch (the aborted path), add a step before returning that transfers the remaining SNS token balance from the swap canister's default account back to the SNS governance treasury account. This mirrors the existing `sweep_icp` pattern. The transfer should be idempotent (guarded by a flag analogous to the `transfer_start_timestamp_seconds`/`transfer_success_timestamp_seconds` pattern used elsewhere) so that retried `finalize` calls do not double-transfer. `sweep_sns` should also be updated to permit `Lifecycle::Aborted` for this return-to-treasury operation, or a separate `sweep_sns_back` method should be introduced.

## Proof of Concept
1. Deploy an SNS with `swap_distribution_sns_e8s = N` tokens pre-loaded into the swap canister.
2. Open the swap; ensure `min_participants` is not reached before the deadline.
3. `try_abort` transitions the swap to `Lifecycle::Aborted`.
4. Call `finalize`. `finalize_inner` executes `sweep_icp`, `settle_neurons_fund_participation`, `restore_dapp_controllers_for_finalize`, then returns at L1583.
5. Query the SNS ledger: `icrc1_balance_of(swap_canister_account)` returns `N` (unchanged). This is directly confirmed by the existing integration test assertion at `rs/nervous_system/integration_tests/tests/sns_lifecycle.rs` L1338.
6. Attempt any SNS token transfer from the swap canister: no callable method exists that authorizes such a transfer in the `Aborted` lifecycle.
7. Delete the swap canister: the `N` SNS tokens are permanently inaccessible.

A deterministic reproduction is already embedded in the existing integration test suite — the assertion at L1338 passes precisely because the tokens are never returned.
