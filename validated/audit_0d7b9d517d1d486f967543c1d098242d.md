### Title
SNS Governance Voting Reward Sniping via Instant Neuron Creation — (File: rs/sns/governance/src/governance.rs)

---

### Summary

The SNS governance canister's `compute_ballots_for_new_proposal` function snapshots voting power at the moment a proposal is submitted, with no spike-detection guard. An unprivileged user can stake a large neuron immediately before a proposal is created, have it vote automatically via following, and receive a share of the reward purse proportional to their freshly-minted stake — diluting rewards for long-term stakers. The NNS governance canister has an explicit countermeasure for exactly this attack; the SNS canister does not.

---

### Finding Description

**Root cause — SNS `compute_ballots_for_new_proposal`**

`rs/sns/governance/src/governance.rs` line 5226 iterates over every neuron in `self.proto.neurons` at `now_seconds` and records each neuron's current voting power directly into the proposal's ballot map:

```rust
for (k, v) in self.proto.neurons.iter() {
    if v.dissolve_delay_seconds(now_seconds) < min_dissolve_delay_for_vote {
        continue;
    }
    let voting_power = v.voting_power(now_seconds, ...);
    total_power += voting_power as u128;
    electoral_roll.insert(k.clone(), Ballot { vote: Vote::Unspecified, voting_power, ... });
}
``` [1](#0-0) 

There is no check for a sudden increase in total voting power. A neuron created one second before `make_proposal` is called is treated identically to a neuron that has been staked for years.

**Reward distribution uses those ballot weights directly**

When `distribute_rewards` runs, it sums `ballot.voting_power` for every neuron that voted and divides the reward purse proportionally:

```rust
let reward_shares = i2d(ballot.voting_power);
*neuron_id_to_reward_shares.entry(neuron_id)... += reward_shares;
...
let neuron_reward_e8s = rewards_purse_e8s * (neuron_reward_shares / total_reward_shares);
``` [2](#0-1) 

A brand-new neuron with a large stake and a following rule set to auto-vote will accumulate `reward_shares` equal to `stake × dissolve_delay_bonus`, with zero age penalty (age = 0 → age bonus = 0, but the base stake still dominates).

**Contrast with NNS governance — the fix that SNS is missing**

NNS governance's `compute_ballots_for_standard_proposal` explicitly detects voting-power spikes by comparing the current total potential voting power against a rolling window of up to seven daily snapshots stored in `VOTING_POWER_SNAPSHOTS`. If the current total exceeds 1.5× the minimum in that window, the function falls back to the most recent pre-spike snapshot:

```rust
let maybe_previous_ballots_if_voting_power_spike_detected = VOTING_POWER_SNAPSHOTS
    .with_borrow(|snapshots| {
        snapshots.previous_ballots_if_voting_power_spike_detected(
            current_voting_power_snapshot.total_potential_voting_power(),
            now_seconds,
        )
    });
``` [3](#0-2) 

The spike threshold is defined as:

```rust
const MULTIPLIER_THRESHOLD_FOR_VOTING_POWER_SPIKE: f64 = 1.5;
``` [4](#0-3) 

SNS governance has no equivalent `VotingPowerSnapshots` structure, no periodic snapshot task, and no fallback path. [5](#0-4) 

---

### Impact Explanation

An attacker who controls a large amount of SNS tokens can:

1. Monitor the SNS governance canister for a pending `make_proposal` call (proposals are public ingress messages visible before execution).
2. Call `manage_neuron { command: Stake / ClaimOrRefresh }` to create a neuron with a large stake and the minimum required dissolve delay, immediately before the proposal is submitted.
3. Set a following rule so the neuron votes automatically on the proposal type.
4. After the reward round closes, receive a share of `rewards_purse_e8s` proportional to their freshly-created voting power.

Long-term stakers receive a diluted share of the reward purse despite having committed capital for far longer. The attacker can dissolve the neuron after the reward round, recovering the principal minus the dissolve-delay lock period. If the reward yield for a single round exceeds the opportunity cost of the lock period, the attack is profitable. This directly mirrors the BellumNursery scenario: User B stakes 900 tokens just before distribution and receives 90% of rewards despite User A having staked 100 tokens for 30 days.

---

### Likelihood Explanation

- **Attacker-controlled entry path**: Any unprivileged principal can call `manage_neuron` (stake) and `make_proposal` (or wait for another proposer) on any SNS governance canister. No privileged role is required.
- **Observable trigger**: Proposal submission is an ingress message; an attacker running a boundary node or monitoring the IC mempool can front-run it.
- **Economic rationality**: SNS reward rates can be significant (initial rates up to 10%+ per year). A single reward round can represent a meaningful fraction of annual yield. The attack is profitable whenever `reward_per_round × attacker_share > opportunity_cost_of_lock`.
- **Partial mitigations that do not eliminate the risk**: The age bonus (`max_age_bonus_percentage`, typically 25%) reduces but does not eliminate the attacker's share. If an SNS sets `max_age_bonus_percentage = 0` (the default in some configurations), there is zero age-based protection.

---

### Recommendation

Port the NNS voting-power spike detection to SNS governance:

1. Add a periodic `SnapshotVotingPowerTask` to SNS governance (analogous to `rs/nns/governance/src/timer_tasks/snapshot_voting_power.rs`) that records total potential voting power at regular intervals (e.g., daily).
2. In `compute_ballots_for_new_proposal`, compare the current total voting power against the rolling minimum. If it exceeds `1.5×` the minimum, use the most recent pre-spike snapshot for ballot creation.
3. Store the snapshot timestamp in `ProposalData` (as NNS does via `previous_ballots_timestamp_seconds`) for auditability.

    fn compute_ballots_for_new_proposal(&self) -> Result<(u64, BTreeMap<String, Ballot>), String> {
        let now_seconds = self.env.now();

        let nervous_system_parameters = self.nervous_system_parameters_or_panic();

        // Voting power bonus parameters.
        let max_dissolve_delay = nervous_system_parameters
            .max_dissolve_delay_seconds
            .expect("NervousSystemParameters must have max_dissolve_delay_seconds");

        let max_age_bonus = nervous_system_parameters
            .max_neuron_age_for_age_bonus
            .expect("NervousSystemParameters must have max_neuron_age_for_age_bonus");

        let max_dissolve_delay_bonus_percentage = nervous_system_parameters
            .max_dissolve_delay_bonus_percentage
            .expect("NervousSystemParameters must have max_dissolve_delay_bonus_percentage");

        let max_age_bonus_percentage = nervous_system_parameters
            .max_age_bonus_percentage
            .expect("NervousSystemParameters must have max_age_bonus_percentage");

        let min_dissolve_delay_for_vote = nervous_system_parameters
            .neuron_minimum_dissolve_delay_to_vote_seconds
            .expect("NervousSystemParameters must have min_dissolve_delay_for_vote");

        let mut electoral_roll = BTreeMap::<String, Ballot>::new();
        let mut total_power: u128 = 0;

        for (k, v) in self.proto.neurons.iter() {
            // If this neuron is eligible to vote, record its
            // voting power at the time of proposal creation (now).
            if v.dissolve_delay_seconds(now_seconds) < min_dissolve_delay_for_vote {
                // Not eligible due to dissolve delay.
                continue;
            }

            let voting_power = v.voting_power(
                now_seconds,
                max_dissolve_delay,
                max_age_bonus,
                max_dissolve_delay_bonus_percentage,
                max_age_bonus_percentage,
            );

            total_power += voting_power as u128;
            electoral_roll.insert(
                k.clone(),
                Ballot {
                    vote: Vote::Unspecified as i32,
                    voting_power,
                    cast_timestamp_seconds: 0,
                },
            );
        }

        if total_power >= (u64::MAX as u128) {
            // The way the neurons are configured, the total voting
            // power on this proposal would overflow a u64!
            return Err("Voting power overflow.".to_string());
        }
        if electoral_roll.is_empty() {
            // Cannot make a proposal with no eligible voters.  This
            // is a precaution that shouldn't happen as we check that
            // the voter is allowed to vote.
            return Err("No eligible voters.".to_string());
        }

        Ok((total_power as u64, electoral_roll))
    }

    fn compute_ballots_for_standard_proposal(
        &self,
        now_seconds: u64,
    ) -> Result<
        (
            HashMap<u64, Ballot>,
            u64,         /*potential_voting_power*/
            Option<u64>, /*previous_ballots_timestamp_seconds*/
        ),
        GovernanceError,
    > {
        let current_voting_power_snapshot = self
            .neuron_store
            .compute_voting_power_snapshot_for_standard_proposal(
                self.voting_power_economics(),
                now_seconds,
            )?;

        // Check if there is a voting power spike. If there is, then the return value here
        // will be `Some(...)`.
        let maybe_previous_ballots_if_voting_power_spike_detected = VOTING_POWER_SNAPSHOTS
            .with_borrow(|snapshots| {
                snapshots.previous_ballots_if_voting_power_spike_detected(
                    current_voting_power_snapshot.total_potential_voting_power(),
                    now_seconds,
                )
            });

        let (voting_power_snapshot, previous_ballots_timestamp_seconds) =
            match maybe_previous_ballots_if_voting_power_spike_detected {
                // This is the extraordinary case - we have a voting power spike, and we
                // need to use the previous snapshot.
                Some((previous_snapshot_timestamp, previous_snapshot)) => {
                    (previous_snapshot, Some(previous_snapshot_timestamp))
                }
                // This is the normal case - we have no voting power spike, so we use the
                // current snapshot.
                None => (current_voting_power_snapshot, None),
            };

        let (ballots, total_potential_voting_power) =
            voting_power_snapshot.create_ballots_and_total_potential_voting_power();
        Ok((
            ballots,
            total_potential_voting_power,
            previous_ballots_timestamp_seconds,
        ))
    }
