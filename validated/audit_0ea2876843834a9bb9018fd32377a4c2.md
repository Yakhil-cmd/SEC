### Title
NNS Governance `Follow` Command Allows Duplicate Followees, Skewing Majority Vote Calculation - (File: `rs/nns/governance/src/governance.rs`)

---

### Summary

The NNS Governance canister's `follow` function accepts a `followees` list without any de-duplication check. The `would_follow_ballots` function then iterates over this raw list and counts each occurrence of a followee's vote separately, inflating both the vote tally and the denominator. A neuron controller or hotkey can exploit this to make their neuron's automatic vote reflect a skewed majority that does not correspond to the true one-neuron-one-vote intent of the follow mechanism.

---

### Finding Description

The `follow` function in `rs/nns/governance/src/governance.rs` stores the caller-supplied `follow_request.followees` directly into the neuron's followee map without deduplication: [1](#0-0) 

The stored list is later consumed verbatim by `would_follow_ballots` in `rs/nns/governance/src/neuron/types.rs`, which iterates over every element and increments `yes` or `no` for each occurrence: [2](#0-1) 

The majority threshold is computed against `followees.len()` — the raw (possibly inflated) length — not the count of unique followees: [3](#0-2) 

An integration test explicitly documents that this is accepted behavior: [4](#0-3) 

The `follow` function checks only the total list length against `MAX_FOLLOWEES_PER_TOPIC`, not the count of unique neurons: [5](#0-4) 

---

### Impact Explanation

Consider a neuron N whose controller sets followees to `[A, A, A, B]` (neuron A listed three times, neuron B once). When A votes Yes and B votes No:

- `yes = 3`, `no = 1`, `followees.len() = 4`
- `3 * 2 = 6 > 4` → N automatically votes **Yes**

Without duplicates (`[A, B]`):

- `yes = 1`, `no = 1`, `followees.len() = 2`
- `1 * 2 = 2 > 2` is false; `1 * 2 = 2 >= 2` → N automatically votes **No**

The controller has reversed the outcome of the automatic vote by inserting duplicates. For neurons with large voting power (e.g., neurons holding significant ICP stake), this can materially affect NNS proposal outcomes. The follow mechanism is intended to implement a fair one-neuron-one-vote majority among followees; duplicates break this invariant without any on-chain enforcement.

---

### Likelihood Explanation

The attack requires only that the neuron's controller (or a registered hotkey) submit a `ManageNeuron { Follow { followees: [A, A, A, B] } }` ingress message. No privileged access, no threshold corruption, and no social engineering is needed. The operation succeeds silently, as confirmed by the existing integration test. Any neuron owner who wishes to bias their neuron's automatic vote — while appearing to follow a set of trusted neurons — can do so at any time.

---

### Recommendation

Add a de-duplication step inside `follow` before storing the followees list, analogous to the check already present in `do_add_api_boundary_nodes`: [6](#0-5) 

Concretely, after the length check in `follow`, reject or deduplicate the list:

```rust
let unique_followees: HashSet<_> = follow_request.followees.iter().collect();
if unique_followees.len() != follow_request.followees.len() {
    return Err(GovernanceError::new_with_message(
        ErrorType::InvalidCommand,
        "Duplicate followees are not allowed.",
    ));
}
```

The `would_follow_ballots` function should also be hardened to deduplicate the stored list before counting, as a defense-in-depth measure.

---

### Proof of Concept

**Entry path:** Any unprivileged ingress sender who controls (or holds a hotkey for) an NNS neuron.

**Steps:**

1. Obtain controller or hotkey access to neuron N.
2. Submit an ingress `ManageNeuron` message to the NNS Governance canister with:
   ```
   Follow { topic: <any topic>, followees: [A, A, A, B] }
   ```
3. The `follow` function stores `[A, A, A, B]` without complaint.
4. When a proposal on that topic is created and neuron A votes Yes while neuron B votes No, `would_follow_ballots` computes `yes=3, no=1, len=4` → N votes **Yes** automatically.
5. Without duplicates the true majority would be a tie (1 Yes, 1 No out of 2), which defaults to **No** per the `no * 2 >= len` rule.

The skew is proportional to the number of duplicates inserted and can be tuned to override any minority-vs-majority split among the unique followees. [7](#0-6) [8](#0-7)

    fn follow(
        &mut self,
        id: &NeuronId,
        caller: &PrincipalId,
        follow_request: &manage_neuron::Follow,
    ) -> Result<(), GovernanceError> {
        // Find the neuron to modify.
        let (is_neuron_controlled_by_caller, is_caller_authorized_to_vote) =
            self.with_neuron(id, |neuron| {
                (
                    neuron.is_controlled_by(caller),
                    neuron.is_authorized_to_vote(caller),
                )
            })?;

        // Only the controller, or a proposal (which passes the controller as the
        // caller), can change the followees for the ManageNeuron topic.
        if follow_request.topic() == Topic::NeuronManagement && !is_neuron_controlled_by_caller {
            return Err(GovernanceError::new_with_message(
                ErrorType::NotAuthorized,
                "Caller is not authorized to manage following of neuron for the ManageNeuron topic.",
            ));
        } else {
            // Check that the caller is authorized, i.e., either the
            // controller or a registered hot key.
            if !is_caller_authorized_to_vote {
                return Err(GovernanceError::new_with_message(
                    ErrorType::NotAuthorized,
                    "Caller is not authorized to manage following of neuron.",
                ));
            }
        }

        // Check that the list of followees is not too
        // long. Allowing neurons to follow too many neurons
        // allows a memory exhaustion attack on the neurons
        // canister.
        if follow_request.followees.len() > MAX_FOLLOWEES_PER_TOPIC {
            return Err(GovernanceError::new_with_message(
                ErrorType::InvalidCommand,
                "Too many followees.",
            ));
        }

        // Validate topic exists
        let topic = Topic::try_from(follow_request.topic).map_err(|_| {
            GovernanceError::new_with_message(
                ErrorType::InvalidCommand,
                format!("Not a known topic number. Follow:\n{follow_request:#?}"),
            )
        })?;

        let now_seconds = self.env.now();
        let new_neuron_followees = self.with_neuron(id, |neuron| {
            modify_followees(
                &self.neuron_store,
                neuron,
                &neuron.followees,
                topic as i32,
                Followees {
                    followees: follow_request.followees.clone(),
                },
            )
        })??;

        self.with_neuron_mut(id, |neuron| {
            neuron.followees = new_neuron_followees;

            // Changing followees may change the voting power, so refresh it.
            neuron.refresh_voting_power(now_seconds);
        })?;

        Ok(())
    }

    pub(crate) fn would_follow_ballots(
        &self,
        topic: Topic,
        ballots: &HashMap<u64, Ballot>,
    ) -> Vote {
        // Compute the list of followees for this topic. If no
        // following is specified for the topic, use the followees
        // from the 'Unspecified' topic.
        if let Some(followees) = self
            .followees
            .get(&(topic as i32))
            .or_else(|| self.followees.get(&(Topic::Unspecified as i32)))
            // extract plain vector from 'Followees' proto
            .map(|x| &x.followees)
        {
            // If, for some reason, a list of followees is specified
            // but empty (this is not normal), don't vote 'no', as
            // would be the natural result of the algorithm below, but
            // instead don't cast a vote.
            if followees.is_empty() {
                return Vote::Unspecified;
            }
            let mut yes: usize = 0;
            let mut no: usize = 0;
            for f in followees.iter() {
                if let Some(f_vote) = ballots.get(&f.id) {
                    if f_vote.vote == (Vote::Yes as i32) {
                        yes = yes.saturating_add(1);
                    } else if f_vote.vote == (Vote::No as i32) {
                        no = no.saturating_add(1);
                    }
                }
            }
            if yes.saturating_mul(2_usize) > followees.len() {
                return Vote::Yes;
            }
            if no.saturating_mul(2_usize) >= followees.len() {
                return Vote::No;
            }
        }
        // No followees specified.
        Vote::Unspecified
    }
