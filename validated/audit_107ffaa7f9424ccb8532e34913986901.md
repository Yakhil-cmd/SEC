Audit Report

## Title
Stale 5% Worst-Case Maturity Modulation Guard in `spawn_neuron` Allows Sub-Minimum-Stake Neurons Under Mission 70 — (`File: rs/nns/governance/src/governance.rs`)

## Summary
The `spawn_neuron` function gates spawning with a hardcoded 5% worst-case modulation bound (`1_f64 - 0.05`), but after the Mission 70 upgrade the actual worst-case modulation applied at finalization time is −10% (`MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 = -1_000` permyriad). When the live modulation is between −5% and −10%, the guard passes but the spawned neuron is minted with `cached_neuron_stake_e8s < neuron_minimum_stake_e8s`, permanently violating the minimum-stake invariant. The parent neuron's maturity is irreversibly deducted at spawn initiation time.

## Finding Description

**Root cause — hardcoded 5% guard in `spawn_neuron`:** [1](#0-0) 

The constant `0.05` was correct when the modulation range was `[−500, +500]` permyriad. Mission 70 extended the lower bound to −1 000 permyriad (−10%): [2](#0-1) 

The CHANGELOG confirms that spawning now reads the locally computed Mission 70 modulation value instead of the CMC-polled value: [3](#0-2) 

**Spawn finalization reads the Mission 70 value:** `maybe_spawn_neurons` reads `maturity_modulation` from `heap_data.maturity_modulation.current_value_permyriad`, which is the Mission 70 value bounded by `[MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70, MATURITY_MODULATION_MAX_PERMYRIAD_MISSION_70]`: [4](#0-3) 

This modulation value (potentially −1 000 permyriad) is then passed directly to `apply_maturity_modulation`: [5](#0-4) 

`apply_maturity_modulation` computes `amount × (10_000 + modulation) / 10_000`, so at −1 000 permyriad the minted stake is `maturity × 0.90`, not `maturity × 0.95`: [6](#0-5) 

**The gap:** The guard at spawn initiation uses 5% worst case; the actual worst case at finalization is 10%. Any maturity `M` satisfying `M × 0.95 ≥ neuron_minimum_stake_e8s` but `M × 0.90 < neuron_minimum_stake_e8s` passes the guard and produces a sub-minimum neuron.

## Impact Explanation

This is a significant NNS governance impact with concrete user harm. The spawned neuron has `cached_neuron_stake_e8s < neuron_minimum_stake_e8s`, permanently violating the protocol invariant. The affected neuron cannot be split (both halves must exceed minimum stake), has degraded operational capabilities, and the parent neuron's maturity is irreversibly zeroed at spawn initiation time — the user cannot recover the maturity even though the resulting neuron is below the minimum. This constitutes concrete, irreversible harm to NNS neuron holders and a permanent governance state invariant violation. Severity: **High** ($2,000–$10,000) — significant NNS governance impact with concrete user harm, requiring specific but realistic market conditions.

## Likelihood Explanation

The condition is reachable by any unprivileged NNS neuron holder with sufficient maturity. No special role or privilege is required — `spawn_neuron` is callable via standard ingress. The −10% modulation is reached when the 7-day ICP price average falls roughly 40% below the 365-day average (sensitivity = 0.25), which is plausible during bear markets. The vulnerable maturity window for `neuron_minimum_stake_e8s = 100_000_000 e8s` (1 ICP) is approximately `105_263_158–111_111_111 e8s`. The attack is repeatable across any number of neurons during the vulnerable market condition.

## Recommendation

Replace the hardcoded `0.05` with a value derived from `MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70`:

```rust
use crate::timer_tasks::update_icp_xdr_rate_related_data::MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70;

// Use the actual Mission 70 worst-case modulation bound instead of the stale 5%.
let least_possible_stake = apply_maturity_modulation(
    maturity_to_spawn,
    MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 as i32,
)
.unwrap_or(0);

if least_possible_stake < economics.neuron_minimum_stake_e8s {
    return Err(GovernanceError::new_with_message(
        ErrorType::InsufficientFunds,
        "There isn't enough maturity to spawn a new neuron due to worst case maturity modulation.",
    ));
}
```

This uses the same integer arithmetic as the rest of the codebase and automatically tracks future changes to the Mission 70 bound.

## Proof of Concept

**Setup:** `neuron_minimum_stake_e8s = 100_000_000` (1 ICP); Mission 70 maturity modulation = −1 000 permyriad (−10%).

**Step 1:** Call `spawn_neuron` with `percentage_to_spawn = 100` on a neuron with `maturity_e8s_equivalent = 106_000_000`.

Guard check: `106_000_000 × 0.95 = 100_700_000 ≥ 100_000_000` → **passes**.

**Step 2:** After `neuron_spawn_dissolve_delay_seconds`, `maybe_spawn_neurons` fires and calls `apply_maturity_modulation(106_000_000, -1000)`:

`106_000_000 × (10_000 − 1_000) / 10_000 = 106_000_000 × 9_000 / 10_000 = 95_400_000`

**Result:** `cached_neuron_stake_e8s = 95_400_000 < neuron_minimum_stake_e8s = 100_000_000`. The spawned neuron permanently violates the minimum-stake invariant. The parent neuron's maturity has been zeroed and cannot be recovered.

A deterministic unit test can reproduce this by: (1) setting `maturity_modulation.current_value_permyriad = Some(-1000)`, (2) creating a neuron with `maturity_e8s_equivalent = 106_000_000` in spawning state, (3) calling `maybe_spawn_neurons`, and (4) asserting `cached_neuron_stake_e8s < neuron_minimum_stake_e8s`.

```rust
pub fn apply_maturity_modulation(
    amount_maturity_e8s: u64,
    maturity_modulation_basis_points: i32,
) -> Result<u64, String> {
    let amount_e8s = u128::from(amount_maturity_e8s);

    let adjusted_maturity_modulation_basis_points = saturating_add_or_subtract_u128_i32(
        BASIS_POINTS_PER_UNITY,
        maturity_modulation_basis_points,
    );

    let modulated_amount_e8s: u128 = amount_e8s
        .checked_mul(adjusted_maturity_modulation_basis_points)
        .ok_or_else(|| "Underflow or overflow when calculating maturity modulation".to_string())?
        .checked_div(BASIS_POINTS_PER_UNITY)
        .ok_or_else(|| "Underflow or overflow when calculating maturity modulation".to_string())?;

    u64::try_from(modulated_amount_e8s).map_err(|err| err.to_string())
}
```
