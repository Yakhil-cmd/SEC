### Title
Hardcoded `WITHDRAWAL_GAS` in `ExitToEthereum` Precompile with No Failure Callback Causes Permanent Fund Loss — (`engine-precompiles/src/native.rs`)

---

### Summary

The `ExitToEthereum` precompile schedules a NEAR `withdraw` promise with a hardcoded gas budget (`WITHDRAWAL_GAS = 100 TGas`) and no failure callback. If the promise fails for any reason — including gas exhaustion due to NEAR protocol gas cost changes — the user's ETH or ERC-20 tokens are permanently burned inside the Aurora EVM with no refund path. This is the direct structural analog of using Solidity's `transfer()` (fixed 2300 gas, no error handling) instead of `call()` (forwarding all gas, return-value checked).

---

### Finding Description

In `engine-precompiles/src/native.rs`, the `ExitToEthereum` precompile's `run()` method constructs a NEAR promise to call `withdraw` on the ETH connector contract:

```rust
// TODO(#332): Determine the correct amount of gas
pub(super) const WITHDRAWAL_GAS: NearGas = NearGas::new(100_000_000_000_000);
``` [1](#0-0) 

The promise is created as a bare `PromiseArgs::Create` — no callback is attached:

```rust
let withdraw_promise = PromiseCreateArgs {
    target_account_id: nep141_address,
    method: "withdraw".to_string(),
    args: serialized_args,
    attached_balance: Yocto::new(1),
    attached_gas: costs::WITHDRAWAL_GAS,
};

let promise = borsh::to_vec(&PromiseArgs::Create(withdraw_promise)).unwrap();
``` [2](#0-1) 

Contrast this with `ExitToNear`, which wraps its promise in a `PromiseArgs::Callback` that invokes `exit_to_near_precompile_callback` on failure (when the `error_refund` feature is enabled): [3](#0-2) 

`ExitToEthereum` has no equivalent callback. The TODO comment on `WITHDRAWAL_GAS` explicitly acknowledges the correct gas amount has not been determined: [1](#0-0) 

Similarly, the EVM gas costs charged to the caller for both exit precompiles are hardcoded to zero, with the same unresolved TODO:

```rust
// TODO(#483): Determine the correct amount of gas
pub(super) const EXIT_TO_NEAR_GAS: EthGas = EthGas::new(0);
// TODO(#483): Determine the correct amount of gas
pub(super) const EXIT_TO_ETHEREUM_GAS: EthGas = EthGas::new(0);
``` [4](#0-3) 

The correct pattern — forwarding remaining gas dynamically — is already implemented in `connector.rs` for other promise-based operations via `calculate_attached_gas`: [5](#0-4) 

But the exit precompiles do not use this pattern; they use static constants instead.

---

### Impact Explanation

When a user calls the `ExitToEthereum` precompile:

1. Their ETH or ERC-20 tokens are **burned inside the Aurora EVM** (balance deducted from EVM state).
2. A NEAR promise is scheduled to call `withdraw` on the ETH connector with exactly `WITHDRAWAL_GAS` (100 TGas).
3. If that promise fails — due to gas exhaustion from NEAR protocol gas cost changes, connector contract complexity increases, or any other runtime failure — **there is no callback and no refund path**.
4. The user's funds are permanently lost: burned in the EVM, never delivered to Ethereum.

Impact: **Critical — permanent freezing/loss of user funds.**

---

### Likelihood Explanation

- The TODO comment on `WITHDRAWAL_GAS` is an explicit acknowledgment by the developers that the correct gas amount is unknown and unvalidated.
- NEAR Protocol has historically adjusted gas costs (e.g., storage and function call costs). A future repricing could push the connector's `withdraw` execution above 100 TGas.
- Any user initiating an ETH or ERC-20 exit to Ethereum is an unprivileged EVM caller who can trigger this path.
- The `ExitToNear` path has a conditional refund mechanism (`error_refund` feature), but `ExitToEthereum` has **none at all**, making it structurally more fragile.

---

### Recommendation

1. **Add a failure callback** to the `ExitToEthereum` promise (analogous to `exit_to_near_precompile_callback`) that refunds the user's tokens if the `withdraw` promise fails.
2. **Replace the hardcoded `WITHDRAWAL_GAS`** with a dynamic calculation that forwards remaining gas (following the `calculate_attached_gas` pattern already used in `connector.rs`).
3. Resolve the `TODO(#332)` and `TODO(#483)` items with proper gas benchmarking before production use.

---

### Proof of Concept

1. User calls the `ExitToEthereum` precompile with ETH value attached.
2. The precompile burns the ETH from the EVM balance and schedules:
   ```
   PromiseArgs::Create(PromiseCreateArgs {
       method: "withdraw",
       attached_gas: 100_000_000_000_000,  // hardcoded, no callback
       ...
   })
   ```
3. If NEAR gas costs increase such that the ETH connector's `withdraw` requires >100 TGas, the promise fails silently.
4. No `exit_to_near_precompile_callback`-equivalent exists for `ExitToEthereum`.
5. The user's ETH is gone from the EVM and never arrives on Ethereum — permanent loss. [6](#0-5) [7](#0-6)

#[named]
pub fn exit_to_near_precompile_callback<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<Option<SubmitResult>, ContractError> {
    with_hashchain(io, env, function_name!(), |io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;
        env.assert_private_call()?;

        // This function should only be called as the callback of
        // exactly one promise.
        if handler.promise_results_count() != 1 {
            return Err(errors::ERR_PROMISE_COUNT.into());
        }

        let args: ExitToNearPrecompileCallbackArgs = io.read_input_borsh()?;

        let maybe_result = if let Some(PromiseResult::Successful(_)) = handler.promise_result(0) {
            if let Some(args) = args.transfer_near {
                let action = PromiseAction::Transfer {
                    amount: Yocto::new(args.amount),
                };
                let promise = PromiseBatchAction {
                    target_account_id: args.target_account_id,
                    actions: vec![action],
                };

                // Safety: this call is safe because it comes from the exit to near precompile, not users.
                // The call is to transfer the unwrapped wNEAR tokens.
                let promise_id = handler.promise_create_batch(&promise);
                handler.promise_return(promise_id);
            }

            None
        } else if let Some(args) = args.refund {
            // Exit call failed; need to refund tokens
            let refund_result = engine::refund_on_error(io, env, state, &args, handler)?;

            if !refund_result.status.is_ok() {
                return Err(errors::ERR_REFUND_FAILURE.into());
            }

            Some(refund_result)
        } else {
            None
        };

        Ok(maybe_result)
    })
}

fn calculate_attached_gas<E: Env>(env: &E) -> NearGas {
    let required_gas = env.used_gas().saturating_add(GAS_FOR_PROMISE_CREATION);

    if required_gas >= env.prepaid_gas() {
        NearGas::new(0)
    } else {
        env.prepaid_gas() - required_gas
    }
}
