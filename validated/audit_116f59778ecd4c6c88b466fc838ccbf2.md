### Title
Hardcoded Fixed NEAR Gas in `ExitToEthereum` Precompile With No Failure Callback Enables Permanent ETH Freeze — (`engine-precompiles/src/native.rs`)

---

### Summary

The `ExitToEthereum` precompile burns a user's ETH from the EVM state and then schedules a NEAR cross-contract call to the ETH connector's `withdraw` method using a hardcoded gas constant (`WITHDRAWAL_GAS = 100 TGas`). No failure callback is attached. If the NEAR promise fails for any reason (gas cost changes after a NEAR protocol upgrade, connector contract failure, etc.), the ETH is permanently destroyed on the Aurora side and never released on Ethereum — an exact structural analog to the Solidity `transfer()` fixed-stipend freeze.

---

### Finding Description

In `engine-precompiles/src/native.rs`, the `costs` module defines a hardcoded gas constant for the ETH withdrawal cross-contract call:

```rust
// TODO(#332): Determine the correct amount of gas
pub(super) const WITHDRAWAL_GAS: NearGas = NearGas::new(100_000_000_000_000);
``` [1](#0-0) 

The `ExitToEthereum::run()` function uses this constant to construct a `PromiseArgs::Create` — a fire-and-forget promise with no callback:

```rust
let withdraw_promise = PromiseCreateArgs {
    target_account_id: nep141_address,
    method: "withdraw".to_string(),
    args: serialized_args,
    attached_balance: Yocto::new(1),
    attached_gas: costs::WITHDRAWAL_GAS,
};

let promise = borsh::to_vec(&PromiseArgs::Create(withdraw_promise)).unwrap();
``` [2](#0-1) 

Contrast this with `ExitToNear`, which conditionally wraps the promise in a `PromiseArgs::Callback` that invokes `exit_to_near_precompile_callback` — a handler that can refund the user on failure (guarded by the `error_refund` feature): [3](#0-2) 

`ExitToEthereum` has no equivalent. The promise is always `PromiseArgs::Create`, meaning there is no code path that can recover funds if the `withdraw` call fails.

The callback handler in `connector.rs` confirms the refund logic exists only for the `ExitToNear` path: [4](#0-3) 

In NEAR Protocol, the EVM state change (ETH burn) is committed in the current transaction receipt. The scheduled promise executes in a separate, later receipt. If that receipt fails, the EVM state change is **not** automatically reverted. This is precisely why the `error_refund` feature was introduced for `ExitToNear` — but `ExitToEthereum` was never given the same protection.

The TODO comment on `WITHDRAWAL_GAS` itself acknowledges the value has not been rigorously determined: [5](#0-4) 

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

When a user calls the `ExitToEthereum` precompile:
1. Their ETH is burned from the Aurora EVM state (irreversible within that receipt).
2. A NEAR promise is scheduled to call `withdraw` on the ETH connector with exactly `WITHDRAWAL_GAS` (100 TGas).
3. If the promise fails — due to insufficient gas after a NEAR protocol gas repricing, a paused or upgraded connector contract, or any other runtime failure — the ETH is gone from Aurora and never minted/released on Ethereum.

There is no callback, no refund path, and no recovery mechanism. The funds are permanently frozen.

---

### Likelihood Explanation

**Medium.**

- NEAR Protocol has repriced operations before and may do so again (direct analog to the Ethereum Istanbul hardfork that broke `transfer()`).
- The `WITHDRAWAL_GAS` value carries an explicit `TODO` acknowledging it was not carefully validated.
- The ETH connector is an external contract; any failure in its `withdraw` method (upgrade, pause, bug) silently destroys user funds with no recourse.
- The entry path is a standard, unprivileged user operation: any EVM user bridging ETH back to Ethereum triggers this code. [6](#0-5) 

---

### Recommendation

1. **Add a failure callback to `ExitToEthereum`** analogous to the `error_refund` mechanism in `ExitToNear`. If the `withdraw` promise fails, the callback should re-credit the ETH to the sender's EVM balance.
2. **Replace the hardcoded `WITHDRAWAL_GAS`** with a dynamically computed value derived from the remaining prepaid gas (as `calculate_attached_gas` does in `connector.rs`), or at minimum validate it experimentally and remove the TODO.
3. **Unify the refund pattern**: both exit precompiles should use `PromiseArgs::Callback` so that any downstream NEAR failure is observable and recoverable. [7](#0-6) 

---

### Proof of Concept

1. User holds 1 ETH on Aurora and calls the `ExitToEthereum` precompile (`0xb0bd02f6a392af548bdf1cfaee5dfa0eefcc8eab`) with a valid Ethereum recipient address.
2. The EVM burns 1 ETH from the user's balance. This state change is committed.
3. A NEAR promise is scheduled: `withdraw` on the ETH connector with `WITHDRAWAL_GAS = 100 TGas`.
4. Suppose NEAR reprices a storage operation used inside `withdraw` (as Ethereum repriced `SLOAD` in Istanbul), making the call require >100 TGas. The promise fails.
5. No callback fires. The user's 1 ETH is gone from Aurora and never appears on Ethereum.
6. The same outcome occurs if the ETH connector contract is paused or returns an error for any reason. [8](#0-7)

pub fn exit_to_near_precompile_callback<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<Option<SubmitResult>, ContractError> {
    with_hashchain(io, env, function_name!(), |io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;
        env.assert_private_call()?;

        // This function should only be called as the callback of
        // exactly one promise.
        if handler.promise_results_count() != 1 {
            return Err(errors::ERR_PROMISE_COUNT.into());
        }

        let args: ExitToNearPrecompileCallbackArgs = io.read_input_borsh()?;

        let maybe_result = if let Some(PromiseResult::Successful(_)) = handler.promise_result(0) {
            if let Some(args) = args.transfer_near {
                let action = PromiseAction::Transfer {
                    amount: Yocto::new(args.amount),
                };
                let promise = PromiseBatchAction {
                    target_account_id: args.target_account_id,
                    actions: vec![action],
                };

                // Safety: this call is safe because it comes from the exit to near precompile, not users.
                // The call is to transfer the unwrapped wNEAR tokens.
                let promise_id = handler.promise_create_batch(&promise);
                handler.promise_return(promise_id);
            }

            None
        } else if let Some(args) = args.refund {
            // Exit call failed; need to refund tokens
            let refund_result = engine::refund_on_error(io, env, state, &args, handler)?;

            if !refund_result.status.is_ok() {
                return Err(errors::ERR_REFUND_FAILURE.into());
            }

            Some(refund_result)
        } else {
            None
        };

        Ok(maybe_result)
    })
}

fn calculate_attached_gas<E: Env>(env: &E) -> NearGas {
    let required_gas = env.used_gas().saturating_add(GAS_FOR_PROMISE_CREATION);

    if required_gas >= env.prepaid_gas() {
        NearGas::new(0)
    } else {
        env.prepaid_gas() - required_gas
    }
