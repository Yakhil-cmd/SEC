### Title
Missing Anonymous Principal Validation in `node_provider_id` Parameter Allows Node Operator Record Takeover - (File: rs/registry/canister/src/mutations/do_update_node_operator_config_directly.rs)

### Summary
In `do_update_node_operator_config_directly_`, the `node_provider_id` parameter from the caller-supplied payload is written directly into `NodeOperatorRecord.node_provider_principal_id` without checking whether it equals the anonymous principal (`Principal::anonymous()`). If a node provider sets this field to the anonymous principal, any subsequent anonymous ingress sender passes the authorization check and can redirect the node operator record — including its node reward attribution — to an arbitrary principal.

### Finding Description
The function `do_update_node_operator_config_directly_` in `rs/registry/canister/src/mutations/do_update_node_operator_config_directly.rs` is the permissionless (no governance gate) path for a node provider to update their own `NodeOperatorRecord`. Its sole authorization guard is:

```rust
// line 59-65
if caller != PrincipalId::try_from(&node_operator_record.node_provider_principal_id).unwrap() {
    return Err(format!(
        "Caller {caller} not equal to the node_provider_princpal_id for this record."
    ));
}
```

After passing this check, the new provider principal is written unconditionally:

```rust
// line 83
node_operator_record.node_provider_principal_id = node_provider_id.to_vec();
```

The only other validation on `node_provider_id` is that it must not equal `node_operator_id` (line 77). There is no check that `node_provider_id` is not the anonymous principal. [1](#0-0) 

The canister entry point explicitly states this method is open to any caller:

```rust
// line 811
// This method can be called by anyone
``` [2](#0-1) 

### Impact Explanation
If a node provider (accidentally or through a UI/tooling bug) submits `node_provider_id = Principal::anonymous()`:

1. `node_provider_principal_id` in the `NodeOperatorRecord` is set to the anonymous principal's byte encoding.
2. Any subsequent anonymous ingress message to `update_node_operator_config_directly` passes the authorization check at line 59–65, because `dfn_core::api::caller()` returns the anonymous principal for unsigned ingress.
3. The anonymous attacker can then set `node_provider_id` to any principal they control, transferring ownership of the node operator record.
4. Node reward payments (XDR) computed by `get_node_providers_monthly_xdr_rewards` are attributed to `node_provider_principal_id`, so the attacker redirects all future node rewards to themselves. [3](#0-2) [4](#0-3) 

### Likelihood Explanation
The `update_node_operator_config_directly` endpoint is open to any ingress sender (no governance gate, no caller allowlist). A node provider only needs to submit `node_provider_id = anonymous` once — through a misconfigured script, a default-initialized Candid struct, or a UI bug — to permanently expose the record. The anonymous principal is a valid `PrincipalId` that Candid encoding accepts without error, so no special crafting is required. Once the record is poisoned, the takeover step is a single unsigned ingress call. [5](#0-4) 

### Recommendation
Add an explicit check that `node_provider_id` is not the anonymous principal before line 83, analogous to the existing `node_provider_id == node_operator_id` guard:

```rust
if node_provider_id.is_anonymous() {
    return Err("The Node Provider ID cannot be the anonymous principal".to_string());
}
```

Apply the same guard to `do_update_node_operator_config` (governance path) and `do_add_node_operator` for defense-in-depth, even though those paths require a governance majority. [1](#0-0) [6](#0-5) 

### Proof of Concept

**Step 1 — Poison the record (legitimate node provider, one call):**
```
update_node_operator_config_directly(
  node_operator_id  = <victim_operator_principal>,
  node_provider_id  = "2vxsx-fae"   // anonymous principal
)
// caller = current node_provider_principal_id  → passes auth check
// Result: NodeOperatorRecord.node_provider_principal_id = anonymous
```

**Step 2 — Takeover (attacker, anonymous ingress):**
```
update_node_operator_config_directly(
  node_operator_id  = <victim_operator_principal>,
  node_provider_id  = <attacker_principal>
)
// caller = anonymous == node_provider_principal_id → passes auth check
// Result: NodeOperatorRecord.node_provider_principal_id = attacker_principal
// All future node rewards flow to attacker_principal
``` [7](#0-6) [8](#0-7)

    fn do_update_node_operator_config_directly_(
        &mut self,
        payload: UpdateNodeOperatorConfigDirectlyPayload,
        caller: PrincipalId,
        now: SystemTime,
    ) -> Result<(), String> {
        println!("{LOG_PREFIX}do_update_node_operator_config_directly: {payload:?}");

        // 1. Look up the record of the requested target NodeOperatorRecord.
        let node_operator_id = payload
            .node_operator_id
            .ok_or("No Node Operator specified in the payload".to_string())?;

        let node_operator_record_key = make_node_operator_record_key(node_operator_id).into_bytes();
        let node_operator_record_vec = &self
            .get(&node_operator_record_key, self.latest_version())
            .ok_or(format!(
                "Node Operator record with ID {node_operator_id} not found in the registry."
            ))?
            .value;

        let mut node_operator_record =
            NodeOperatorRecord::decode(node_operator_record_vec.as_slice())
                .map_err(|e| format!("{e:?}"))?;

        // 2. Make sure that the caller is authorized to make the requested changes to node_operator_record.
        if caller
            != PrincipalId::try_from(&node_operator_record.node_provider_principal_id).unwrap()
        {
            return Err(format!(
                "Caller {caller} not equal to the node_provider_princpal_id for this record."
            ));
        }

        // 3. Check Rate Limits
        let current_node_provider = caller;
        let reservation =
            self.try_reserve_capacity_for_node_provider_operation(now, current_node_provider, 1)?;

        // 4. Check that the Node Provider is not being set with the same ID as the Node Operator
        let node_provider_id = payload
            .node_provider_id
            .ok_or("No Node Provider specified in the payload".to_string())?;

        if node_provider_id == node_operator_id {
            return Err(format!(
                "The Node Operator ID cannot be the same as the Node Provider ID: {node_operator_id}"
            ));
        }

        node_operator_record.node_provider_principal_id = node_provider_id.to_vec();

        // 5. Set and execute the mutation
        let mutations = vec![RegistryMutation {
            mutation_type: registry_mutation::Type::Update as i32,
            key: node_operator_record_key,
            value: node_operator_record.encode_to_vec(),
        }];

        // Check invariants before applying mutations
        self.maybe_apply_mutation_internal(mutations);

        if let Err(e) = self.commit_used_capacity_for_node_provider_operation(now, reservation) {
            println!("{LOG_PREFIX}Error committing Rate Limit usage: {e}");
        }

        Ok(())
    }
