### Title
`key_manager` Not Reset on Owner Transfer Allows Previous Owner to Bypass Pause Control — (`engine/src/contract_methods/admin.rs`)

---

### Summary

When `set_owner` transfers engine ownership to a new NEAR account, the `key_manager` field in `EngineState` is not cleared. A malicious outgoing owner can set themselves as `key_manager` immediately before calling `set_owner`, then retain the ability to call `start_hashchain` — which unpauses the engine — even after the new owner has paused it for emergency reasons.

---

### Finding Description

`set_owner` only updates `state.owner_id` and writes the state back. It does not touch `state.key_manager`:

```rust
// engine/src/contract_methods/admin.rs, lines 104-121
pub fn set_owner<I: IO + Copy, E: Env>(io: I, env: &E) -> Result<(), ContractError> {
    with_hashchain(io, env, function_name!(), |mut io| {
        let mut state = state::get_state(&io)?;
        require_running(&state)?;
        require_owner_only(&state, &env.predecessor_account_id())?;
        let args: SetOwnerArgs = io.read_input_borsh()?;
        if state.owner_id == args.new_owner {
            return Err(errors::ERR_SAME_OWNER.into());
        }
        state.owner_id = args.new_owner;   // only owner_id is updated
        state::set_state(&mut io, &state)?;
        Ok(())
    })
}
``` [1](#0-0) 

`EngineState` holds `key_manager: Option<AccountId>` alongside `owner_id`: [2](#0-1) 

`set_key_manager` is owner-only and writes `key_manager` into the same state struct: [3](#0-2) 

`start_hashchain` is gated by `require_key_manager_only` (not `require_owner_only`) and unpauses the engine by setting `state.is_paused = false`: [4](#0-3) 

`require_key_manager_only` checks only `state.key_manager`, not `state.owner_id`: [5](#0-4) 

`add_relayer_key` is also gated solely by `require_key_manager_only`, allowing the stale key_manager to add NEAR function-call access keys (scoped to `call,submit,submit_with_args`) to the engine contract: [6](#0-5) 

---

### Impact Explanation

**High — Temporary freezing of funds / auth bypass enabling fund theft.**

The new owner's primary emergency lever is `pause_contract`. If the new owner pauses the engine to halt an ongoing exploit, the stale `key_manager` (controlled by the previous owner) can immediately call `start_hashchain` to set `is_paused = false`, nullifying the pause. With the engine unpaused, the ongoing exploit can continue draining user funds. The new owner has no way to prevent this until they call `set_key_manager` to overwrite the stale entry — but they may not know the stale key_manager exists, since there is no event or enumerable record of the previous value.

---

### Likelihood Explanation

**Medium.** The attack requires the outgoing owner to act maliciously in the window before calling `set_owner`. This is realistic in any scenario where ownership is transferred to a new party (governance handoff, protocol acquisition, key rotation after compromise). The outgoing owner simply calls `set_key_manager` with their own account, then calls `set_owner`. The new owner receives no signal that a key_manager was pre-set.

---

### Recommendation

In `set_owner`, clear `state.key_manager` before writing the updated state:

```rust
state.owner_id = args.new_owner;
state.key_manager = None;   // reset stale privileged role
state::set_state(&mut io, &state)?;
``` [7](#0-6) 

Alternatively, require the new owner to explicitly re-set the key_manager after accepting ownership, or emit a log entry whenever `key_manager` is set so the new owner can audit the state on takeover.

---

### Proof of Concept

1. **Attacker (outgoing owner)** calls `set_key_manager({"key_manager": "attacker.near"})` — sets themselves as key_manager.
2. **Attacker** calls `set_owner({"new_owner": "victim.near"})` — transfers ownership. `key_manager` remains `"attacker.near"` in storage.
3. **New owner** discovers an exploit draining EVM funds and calls `pause_contract()` — sets `is_paused = true`.
4. **Attacker** calls `start_hashchain(...)` from `attacker.near` — passes `require_key_manager_only` because `state.key_manager == "attacker.near"`, passes `require_paused` because the contract is paused, and sets `is_paused = false`.
5. The engine is unpaused; the exploit continues draining user funds. [4](#0-3) [8](#0-7)

        state::set_state(&mut io, &state)?;

        Ok(())

pub fn add_relayer_key<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<(), ContractError> {
    let state = state::get_state(&io)?;

    require_running(&state)?;
    require_key_manager_only(&state, &env.predecessor_account_id())?;

    let input = io.read_input().to_vec();
    let public_key = serde_json::from_slice::<RelayerKeyArgs>(&input)
        .map(|args| args.public_key)
        .map_err(|_| errors::ERR_JSON_DESERIALIZE)?;
    let allowance = Yocto::new(env.attached_deposit());
    aurora_engine_sdk::log!("attached key allowance: {allowance}");

    if allowance.as_u128() < 100 {
        // TODO: Clarify the minimum amount if check is needed then change error type
        return Err(errors::ERR_NOT_ALLOWED.into());
    }

    let current_account_id = env.current_account_id();
    let promise = PromiseBatchAction {
        target_account_id: current_account_id.clone(),
        actions: vec![
            PromiseAction::AddFunctionCallKey {
                public_key,
                allowance,
                nonce: 0, // not actually used - depends on block height
                receiver_id: current_account_id,
                function_names: "call,submit,submit_with_args".into(),
            },
            PromiseAction::FunctionCall {
                name: "store_relayer_key_callback".to_string(),
                args: input,
                attached_yocto: ZERO_YOCTO,
                gas: GAS_FOR_STORING_RELAYER_KEY,
            },
        ],
    };

    let promise_id = handler.promise_create_batch(&promise);
    handler.promise_return(promise_id);

    Ok(())
}

#[named]
pub fn start_hashchain<I: IO + Copy, E: Env>(mut io: I, env: &E) -> Result<(), ContractError> {
    let mut state = state::get_state(&io)?;
    require_paused(&state)?;
    require_key_manager_only(&state, &env.predecessor_account_id())?;

    let input = io.read_input().to_vec();
    let args = StartHashchainArgs::try_from_slice(&input).map_err(|_| errors::ERR_SERIALIZE)?;
    let block_height = env.block_height();

    // Starting hashchain must be for an earlier block
    if block_height < args.block_height {
        return Err(errors::ERR_ARGS.into());
    }

    let mut hashchain = Hashchain::new(
        state.chain_id,
        env.current_account_id(),
        args.block_height + 1,
        args.block_hashchain,
    );

    if hashchain.get_current_block_height() < block_height {
        hashchain.move_to_block(block_height)?;
    }

    hashchain.add_block_tx(
        block_height,
        function_name!(),
        &input,
        &[],
        &Bloom::default(),
    )?;
    crate::hashchain::save_hashchain(&mut io, &hashchain)?;

    state.is_paused = false;
    state::set_state(&mut io, &state)?;

    Ok(())

fn require_key_manager_only(
    state: &state::EngineState,
    predecessor_account_id: &AccountId,
) -> Result<(), ContractError> {
    let key_manager = state
        .key_manager
        .as_ref()
        .ok_or(errors::ERR_KEY_MANAGER_IS_NOT_SET)?;
    if key_manager != predecessor_account_id {
        return Err(errors::ERR_NOT_ALLOWED.into());
    }
    Ok(())
}
