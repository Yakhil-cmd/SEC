### Title
Unguarded `new` Initialization Allows Any Caller to Seize Engine Ownership Before Deployer — (`engine/src/contract_methods/admin.rs`)

---

### Summary

The Aurora Engine's `new` initialization function carries no caller restriction. Any NEAR account can invoke it in the window between contract deployment and the deployer's own `new` call, setting an attacker-controlled `owner_id`. Because the workspace deployment flow issues deployment and initialization as two separate transactions, this window is real and exploitable on-chain.

---

### Finding Description

`new` in `engine/src/contract_methods/admin.rs` guards against double-initialization with a single check:

```rust
if state::get_state(&io).is_ok() {
    return Err(b"ERR_ALREADY_INITIALIZED".into());
}
``` [1](#0-0) 

There is no check on `env.predecessor_account_id()`. Any NEAR account that calls `new` before the deployer does will have its supplied `owner_id`, `chain_id`, and `upgrade_delay_blocks` committed to storage as the canonical engine state.

The public WASM export is unconditional:

```rust
pub extern "C" fn new() {
    let io = Runtime;
    let env = Runtime;
    contract_methods::admin::new(io, &env)
        .map_err(ContractError::msg)
        .sdk_unwrap();
}
``` [2](#0-1) 

The workspace deployment helper — which mirrors how the engine is deployed in practice — issues deployment and initialization as two separate awaited transactions:

```rust
let contract = account.deploy(&self.code...).await?;
// ...
engine.new(self.chain_id, self.owner_id, self.upgrade_delay_blocks)
    .transact()
    .await?;
``` [3](#0-2) 

Between the `deploy` receipt and the `new` receipt, any NEAR account can submit a `new` call with attacker-controlled arguments. The deployer's subsequent `new` call returns `ERR_ALREADY_INITIALIZED` and is silently discarded.

All `EngineState` variants produced by `NewCallArgs` set `is_paused: false`: [4](#0-3) 

So the engine immediately accepts user transactions under the attacker's ownership, with no paused guard to buy recovery time.

---

### Impact Explanation

Once the attacker's `owner_id` is stored, they hold every privileged capability gated by `require_owner_only`:

- `upgrade` / `stage_upgrade` / `deploy_upgrade` — deploy arbitrary replacement bytecode.
- `set_owner` — transfer ownership to any account.
- `pause_contract` / `resume_contract` — control liveness.
- `set_eth_connector_contract_account` — redirect the ETH connector to a malicious contract, intercepting all bridge deposits. [5](#0-4) [6](#0-5) 

The attacker can wait for the engine to accumulate bridged ETH balances and then drain them via a malicious upgrade or a redirected connector. This constitutes **direct theft of user funds** — Critical impact.

---

### Likelihood Explanation

The attack requires only:

1. Watching the NEAR blockchain for a `DeployContract` action targeting the Aurora Engine account.
2. Submitting a `new` call in the next block before the deployer's `new` transaction is included.

No privileged access, no key compromise, and no social engineering is required. The workspace deployment helper — the reference implementation for deploying the engine — performs deployment and initialization in two separate transactions, confirming the window exists in the intended deployment flow. [7](#0-6) 

---

### Recommendation

1. **Atomic deployment**: Deploy and call `new` in a single NEAR batch transaction so no inter-transaction window exists.
2. **Caller restriction in `new`**: Add a check that `env.predecessor_account_id() == env.current_account_id()` (i.e., only a self-call or the contract account itself may initialize), mirroring the pattern used in `set_eth_connector_contract_account`: [8](#0-7) 

3. **Start paused**: Initialize `is_paused: true` in `EngineState` so that even if a race occurs, no user-facing operations (deposits, EVM calls) can execute until the legitimate owner explicitly unpauses. [9](#0-8) 

---

### Proof of Concept

```
Block N:
  Deployer → aurora: DeployContract(engine_wasm)

Block N+1 (attacker races):
  Attacker → aurora: new({
      chain_id: <any>,
      owner_id: "attacker.near",
      upgrade_delay_blocks: 0,
      key_manager: "attacker.near"
  })
  // Succeeds: state is empty, no caller check.

Block N+1 or N+2:
  Deployer → aurora: new({owner_id: "aurora.near", ...})
  // Fails: ERR_ALREADY_INITIALIZED

Attacker now calls:
  aurora.upgrade(malicious_wasm)   // owner check passes
  // Malicious code drains all bridged ETH balances on next user interaction.
``` [10](#0-9)

#[named]
pub fn new<I: IO + Copy, E: Env>(mut io: I, env: &E) -> Result<(), ContractError> {
    if state::get_state(&io).is_ok() {
        return Err(b"ERR_ALREADY_INITIALIZED".into());
    }

    let input = io.read_input().to_vec();
    let args = NewCallArgs::deserialize(&input).map_err(|_| errors::ERR_BORSH_DESERIALIZE)?;

    let initial_hashchain = args.initial_hashchain();
    let state: EngineState = args.into();

    if let Some(block_hashchain) = initial_hashchain {
        let block_height = env.block_height();
        let mut hashchain = Hashchain::new(
            state.chain_id,
            env.current_account_id(),
            block_height,
            block_hashchain,
        );

        hashchain.add_block_tx(
            block_height,
            function_name!(),
            &input,
            &[],
            &Bloom::default(),
        )?;
        crate::hashchain::save_hashchain(&mut io, &hashchain)?;
    }

    state::set_state(&mut io, &state)?;
    Ok(())
}

#[named]
pub fn stage_upgrade<I: IO + Copy, E: Env>(io: I, env: &E) -> Result<(), ContractError> {
    with_hashchain(io, env, function_name!(), |mut io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;
        let delay_block_height = env.block_height() + state.upgrade_delay_blocks;
        require_owner_only(&state, &env.predecessor_account_id())?;
        io.read_input_and_store(&storage::bytes_to_key(KeyPrefix::Config, CODE_KEY));
        io.write_storage(
            &storage::bytes_to_key(KeyPrefix::Config, CODE_STAGE_KEY),
            &delay_block_height.to_le_bytes(),
        );
        Ok(())
    })
}
