### Title
Blob Fees (EIP-4844) Charged from Senders but Never Distributed to Operator тАФ (`basic_bootloader/src/bootloader/transaction_flow/zk/mod.rs`, `basic_bootloader/src/bootloader/transaction_flow/ethereum/mod.rs`)

---

### Summary

When EIP-4844 is enabled, both the ZK and Ethereum transaction flows charge the sender a blob fee (`blob_base_fee * num_blobs * GAS_PER_BLOB`) as part of `fee_to_prepay`, but `refund_and_commit_fee` only distributes execution-gas fees to the operator/coinbase. The blob fee portion is permanently removed from the sender's balance without being credited to any recipient тАФ it is silently destroyed.

---

### Finding Description

**Fee collection (validation phase):**

In `validation_impl.rs` for the ZK flow, `fee_to_prepay` is computed as the sum of execution gas fees and blob fees:

```rust
let fee_to_prepay = gas_fee_amount
    .checked_add(fee_for_blob_gas)
    .ok_or(internal_error!("gfa+ffbg"))?;
``` [1](#0-0) 

The same pattern exists in the Ethereum flow:

```rust
let total_fee = fee_amount_execution_gas
    .checked_add(fee_for_blob_gas)
    .ok_or(internal_error!("transaction fee + blob gas"))?;
// fee_to_prepay: total_fee
``` [2](#0-1) 

Both flows deduct the full `fee_to_prepay` (including blob fee) from the sender's balance in `precharge_fee`.

**Fee distribution (post-execution phase):**

In `refund_and_commit_fee` for the ZK flow, only execution gas is handled:

1. **Refund to sender:** `gas_price * (tx_gas_limit - gas_used)` тАФ execution gas only
2. **Payment to operator:** `gas_price_for_operator * gas_used` тАФ execution gas only
3. **Blob fee:** never refunded, never paid to coinbase [3](#0-2) 

The same omission exists in the Ethereum flow's `refund_and_commit_fee`, which only pays `priority_fee_per_gas * gas_used` to the coinbase and refunds unused execution gas тАФ blob fees are absent: [4](#0-3) 

The `blob_gas_used` field is tracked in `ZkTxResult` for reporting purposes only тАФ it is never used to compute a payment to the coinbase: [5](#0-4) 

---

### Impact Explanation

Every EIP-4844 transaction causes `blob_base_fee * num_blobs * GAS_PER_BLOB` tokens to be permanently removed from the sender's balance without being credited to the operator or any other address. The operator loses all blob fee revenue. The tokens are silently destroyed, creating an unaccounted reduction in the L2 base token supply. This is a direct funds-loss path for both the sender (who pays but receives no service credit) and the operator (who processes blobs but receives no compensation for them).

---

### Likelihood Explanation

EIP-4844 is enabled via the `eip-4844` feature flag, which is active in the `for_tests` and `eth_runner` configurations: [6](#0-5) 

It is not currently enabled in the `production` feature set. However, the bug exists in production source files and will manifest immediately if EIP-4844 is activated for production. Any unprivileged user submitting an EIP-4844 transaction triggers the path.

---

### Recommendation

In `refund_and_commit_fee` for both the ZK and Ethereum transaction flows, add a step to credit the blob fee to the coinbase after execution. The blob fee is fixed (not refundable, per EIP-4844 spec), so the full `fee_for_blob_gas` should be transferred to the operator:

```rust
// After paying execution gas to operator:
if context.blob_gas_used > 0 {
    let blob_fee = system.get_blob_base_fee_per_gas()
        .checked_mul(U256::from(context.blob_gas_used))
        .ok_or(internal_error!("blob_fee_overflow"))?;
    // credit blob_fee to coinbase
    system.io.update_account_nominal_token_balance(
        ExecutionEnvironmentType::NoEE,
        &mut inf_resources,
        &coinbase,
        &blob_fee,
        false,
        Config::SIMULATION,
    )?;
}
```

The `TxContextForPreAndPostProcessing` struct already has no `blob_gas_used` field тАФ one must be added to carry the blob gas amount from validation through to the post-execution distribution step. [7](#0-6) 

---

### Proof of Concept

1. Enable the `eip-4844` feature (as done in `for_tests`/`eth_runner`).
2. Submit an EIP-4844 transaction with `num_blobs = 1`, `blob_base_fee = 100`, `GAS_PER_BLOB = 131072`.
3. Observe that the sender's balance decreases by `100 * 131072 = 13,107,200` tokens for the blob fee.
4. Observe that the coinbase balance increases only by `gas_price_for_operator * gas_used` тАФ the blob fee component is absent.
5. Verify that `sender_balance_decrease - execution_gas_fee - execution_gas_refund = 13,107,200` tokens are unaccounted for in any recipient's balance. [8](#0-7)
