### Title
Missing Minimum Gas Price Enforcement Allows Zero-Fee EVM Execution Bypassing Silo `fixed_gas` - (File: engine/src/engine.rs)

### Summary

`charge_gas()` in `engine/src/engine.rs` computes `effective_gas_price` entirely from the user-supplied `max_fee_per_gas` field. Because `block_base_fee_per_gas()` is hardcoded to return `U256::zero()`, any caller who sets `max_fee_per_gas = 0` drives `effective_gas_price` to zero. In Silo mode the `fixed_gas` guard is present but does not prevent this: the early-return bypass is skipped (because `fixed_gas.is_some()`), yet the subsequent multiplication `fixed_gas * 0 = 0` still yields a zero `prepaid_amount`. The sender pays nothing, and the relayer earns nothing, despite the Silo operator having configured a mandatory per-transaction fee.

### Finding Description

`block_base_fee_per_gas()` is hardcoded to `U256::zero()` with no override path: [1](#0-0) 

`charge_gas()` contains an early-return that skips all fee logic when `max_fee_per_gas == 0 && fixed_gas.is_none() && block_base_fee_per_gas == 0`. In non-Silo mode this fires immediately. In Silo mode (`fixed_gas.is_some()`) the early return is not taken, but the subsequent computation still produces a zero result: [2](#0-1) 

Because `block_base_fee_per_gas` is always zero:

```
priority_fee_per_gas = min(max_priority_fee_per_gas, max_fee_per_gas - 0)
                     = min(0, 0) = 0          // when max_fee_per_gas = 0
effective_gas_price  = 0 + 0 = 0
prepaid_amount       = fixed_gas * 0 = 0
```

`get_balance(sender).checked_sub(0)` always succeeds regardless of the sender's balance, so no ETH is deducted. `refund_unused_gas()` then short-circuits on `effective_gas_price.is_zero()` and credits nothing to the relayer: [3](#0-2) 

The Silo `fixed_gas` parameter is stored and retrieved as a mandatory per-transaction gas amount: [4](#0-3) [5](#0-4) 

The `submit` and `submit_with_args` entry points pass `fixed_gas` directly into `charge_gas` with no minimum-price guard: [6](#0-5) 

### Impact Explanation

In Silo mode the operator deploys `fixed_gas` precisely to guarantee that every transaction pays a minimum fee to the relayer. A whitelisted sender who sets `max_fee_per_gas = 0` (and `max_priority_fee_per_gas = 0`) executes arbitrary EVM bytecode at zero ETH cost. The relayer, who pays real NEAR gas to process the call, receives no ETH compensation. This constitutes **theft of unclaimed yield**: relayer fees that the Silo configuration is designed to guarantee are silently denied on every such transaction.

### Likelihood Explanation

The condition is trivially reachable. Any account that passes the Silo whitelist check can craft a legacy or EIP-1559 transaction with `gas_price = 0` / `max_fee_per_gas = 0`. No special state, timing, or privilege beyond whitelist membership is required. The whitelist is an access-control gate, not a fee-enforcement gate, so whitelisted-but-adversarial accounts (or users who simply want free execution) can exploit this continuously.

### Recommendation

Add a minimum gas price check inside `charge_gas()` when `fixed_gas` is present (Silo mode). For example:

```rust
if fixed_gas.is_some() {
    let min_price = /* operator-configured minimum, or at least 1 */;
    if transaction.max_fee_per_gas < min_price {
        return Err(GasPaymentError::MaxFeePerGasLessThanBaseFee);
    }
}
```

Alternatively, store a `min_gas_price` alongside `fixed_gas` in `SiloParamsArgs` and enforce it at the start of `charge_gas()`.

### Proof of Concept

// effective_gas_price = 0  =>  prepaid_amount = 0

The relayer's ETH balance remains at zero after processing, matching the pattern shown in `test_relayer_balance_after_transfer` where the expected reward is `FIXED_GAS * ONE_GAS_PRICE` — a value that collapses to zero when `gas_price = 0`. [7](#0-6)

pub fn submit<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<SubmitResult, ContractError> {
    with_logs_hashchain(io, env, function_name!(), |mut io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;
        let tx_data = io.read_input().to_vec();
        let current_account_id = env.current_account_id();
        let relayer_address = predecessor_address(&env.predecessor_account_id());
        let args = SubmitArgs {
            tx_data,
            ..Default::default()
        };
        let result = engine::submit(
            io,
            env,
            &args,
            state,
            current_account_id,
            relayer_address,
            handler,
        )?;
        let result_bytes = borsh::to_vec(&result).map_err(|_| errors::ERR_SERIALIZE)?;
        io.return_output(&result_bytes);

        Ok(result)
    })
}
