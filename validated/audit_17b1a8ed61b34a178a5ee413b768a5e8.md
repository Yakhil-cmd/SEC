### Title
`RandomSeed` Precompile Exposes Per-Transaction Entropy to Calling Contracts, Enabling Atomic Brute-Force of Any Randomness-Dependent Protocol - (File: `engine-precompiles/src/random.rs`)

### Summary

The Aurora-native `RandomSeed` precompile at address `0xc104f4840573bed437190daf5d2898c2bdf928ac` returns the full 32-byte per-transaction random seed to any EVM contract that calls it. Because the seed is fixed for the entire duration of a single EVM transaction and costs **0 gas** to read, a malicious EVM contract can atomically read the seed, simulate the outcome of any randomness-dependent protocol in the same transaction, and revert if the outcome is unfavorable — effectively brute-forcing the randomness at negligible cost.

### Finding Description

The `RandomSeed` precompile is documented as a "per-block entropy source" but in practice delivers a **per-transaction** random value (derived from `sha256(action_hash || block_random_seed)`). This value is fixed for the entire EVM execution of a given transaction.

**Root cause 1 — seed is fully readable by any contract:** [1](#0-0) 

The `run()` method unconditionally returns `self.random_seed.as_bytes().to_vec()` to any caller. There is no restriction on who can call this precompile.

**Root cause 2 — gas cost is zero:** [2](#0-1) 

`RANDOM_BYTES_GAS = EthGas::new(0)` — the precompile costs nothing to call. The `TODO(#483)` comment acknowledges this is unresolved.

**Root cause 3 — `block_randomness` (EIP-4399 `PREVRANDAO` opcode) also exposes the same seed:** [3](#0-2) 

`block_randomness()` returns `self.env.random_seed()`, meaning the Solidity `block.prevrandao` opcode also exposes the same value to any contract.

**How the seed is constructed (fixed per-transaction):** [4](#0-3) 

The seed is `sha256(action_hash || block_random_value)` — deterministic and immutable for the lifetime of the transaction.

**The seed is injected once at EVM startup:** [5](#0-4) 

It is set in `env.random_seed` before EVM execution begins and never changes during execution.

### Impact Explanation

Any EVM protocol deployed on Aurora that uses the `RandomSeed` precompile (or `block.prevrandao`) for randomness — lotteries, NFT minting, random reward distribution, random selection from a pool — is immediately exploitable. An attacker deploys a contract that:

1. Calls `0xc104f4840573bed437190daf5d2898c2bdf928ac` (or reads `block.prevrandao`) to obtain the transaction's random seed.
2. Derives the expected outcome of the target protocol using the same seed.
3. If the outcome is unfavorable, calls `revert()` — undoing all state changes at the cost of only the base transaction gas.
4. If favorable, proceeds — guaranteeing the attacker always wins.

This is direct theft of user funds from any randomness-dependent protocol on Aurora. Impact: **Critical**.

### Likelihood Explanation

**High.** The attack requires no special privileges — any EVM contract can call the precompile. The 0-gas cost means the attacker pays only base transaction overhead per attempt. Any protocol on Aurora that uses this precompile for randomness is immediately and trivially exploitable by any unprivileged user.

### Recommendation

1. **Remove direct seed exposure from the precompile.** Instead of returning the raw seed, the precompile should return a value derived from both the seed and a caller-specific nonce (e.g., `keccak(seed || caller_address || call_count)`), so the value cannot be used to predict outcomes of other contracts.
2. **Charge a meaningful gas cost.** Resolve `TODO(#483)` and set `RANDOM_BYTES_GAS` to a non-zero value to raise the cost of brute-force attempts.
3. **Document the limitation clearly.** Warn protocol developers that the `RandomSeed` precompile and `block.prevrandao` are readable within the same transaction and must not be used as the sole source of randomness for high-value outcomes.
4. **Consider a commit-reveal scheme** at the protocol level for any randomness-dependent operation.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

interface ITargetProtocol {
    // e.g., a lottery or NFT mint that uses RandomSeed internally
    function participate() external payable returns (uint256 prize);
}

contract BruteForcer {
    address constant RANDOM_SEED_PRECOMPILE = 0xc104f4840573bed437190daf5d2898c2bdf928ac;

    function attack(address target) external payable {
        // 1. Read the fixed per-transaction random seed
        bytes32 seed;
        assembly {
            let ok := call(gas(), RANDOM_SEED_PRECOMPILE, 0, 0, 0, seed, 32)
        }

        // 2. Simulate the target protocol's outcome using the same seed
        //    (attacker reverse-engineers the target's randomness formula)
        uint256 predictedOutcome = uint256(keccak256(abi.encode(seed, target)));

        // 3. Only proceed if the outcome is favorable
        require(predictedOutcome % 100 < 10, "Unfavorable outcome, revert");

        // 4. Call the target — guaranteed to win
        ITargetProtocol(target).participate{value: msg.value}();
    }
}
```

The attacker calls `attack()` repeatedly (each failed attempt reverts, costing only base gas). Because the `RandomSeed` precompile costs 0 gas and the seed is fixed for the transaction, the attacker can always determine the outcome before committing state, achieving a guaranteed win rate against any randomness-dependent protocol on Aurora.         _is_static: bool,

        utils::validate_no_value_attached_to_precompile(context.apparent_value)?;
        let cost = Self::required_gas(input)?;
        if let Some(target_gas) = target_gas

            return Err(ExitError::OutOfGas);
        }

            cost,

        ))
    }

fn compute_random_seed(action_hash: &H256, block_random_value: &H256) -> H256 {
    const BYTES_LEN: usize = 32 + 32;
    let mut bytes: Vec<u8> = Vec::with_capacity(BYTES_LEN);
    bytes.extend_from_slice(action_hash.as_bytes());
    bytes.extend_from_slice(block_random_value.as_bytes());
    aurora_engine_sdk::sha256(&bytes)
}
