### Title
`swap_node_in_subnet_directly` Lacks Canister-Level Access Control and Defaults to Allow-All Caller Policy — (File: `rs/registry/canister/canister/canister.rs`)

---

### Summary

The registry canister exposes `swap_node_in_subnet_directly` as a public update endpoint with no canister-level caller check. Its internal caller whitelist (`NODE_SWAPPING_CALLERS_POLICY`) and subnet whitelist (`NODE_SWAPPING_SUBNETS_POLICY`) both default to `AccessList::allow_all()`, and the global feature flag (`IS_NODE_SWAPPING_ENABLED`) defaults to `true`. Any registered node operator — without governance approval — can therefore swap nodes in any subnet, including the NNS subnet, by calling this endpoint directly.

---

### Finding Description

The canister entry point `swap_node_in_subnet_directly` in `rs/registry/canister/canister/canister.rs` performs no caller identity check before dispatching to the inner logic:

```rust
#[unsafe(export_name = "canister_update swap_node_in_subnet_directly")]
fn swap_node_in_subnet_directly() {
    over(candid_one, |payload: SwapNodeInSubnetDirectlyPayload| {
        swap_node_in_subnet_directly_(payload)
    });
}
``` [1](#0-0) 

Compare this to the adjacent `update_node_operator_config`, which gates on `check_caller_is_governance_and_log`, or `remove_node_operators`, which does the same: [2](#0-1) 

The internal implementation in `do_swap_node_in_subnet_directly.rs` does check a caller whitelist via `swapping_enabled_for_caller`, but that whitelist is backed by the thread-local `NODE_SWAPPING_CALLERS_POLICY`, which is initialized to `AccessList::allow_all()`: [3](#0-2) 

`AccessList::allow_all()` is implemented as a `DenyOnly` set with an empty deny-list, meaning every principal passes the check: [4](#0-3) 

The global feature flag also defaults to `true`: [5](#0-4) 

The `RegistryCanisterInitPayload` fields `swapping_whitelisted_callers` and `swapping_enabled_subnets` are `Option<Vec<...>>`. The init payload comments explicitly state these fields "shouldn't be provided when deploying to mainnet," meaning the static `allow_all()` defaults remain active on mainnet: [6](#0-5) 

The only ownership check inside `swap_nodes_inner` is that the caller must be the `node_operator_id` recorded for both the old and new nodes: [7](#0-6) 

This check is insufficient as a governance gate: it only confirms the caller owns the nodes, not that governance has approved the subnet membership change.

The function ultimately writes a new `SubnetRecord` to the registry via `maybe_apply_mutation_internal`, directly altering subnet membership: [8](#0-7) 

---

### Impact Explanation

Any registered node operator who holds a node already assigned to a subnet can, without any governance proposal, replace that node with another node they own. This applies to **all subnets**, including the NNS subnet. A malicious node operator could:

- Silently rotate a well-behaved node out of the NNS subnet and replace it with a node running modified software, without triggering the normal governance-approved node replacement flow.
- Repeatedly rotate nodes across subnets at will, bypassing the rate-limit window by using multiple node operator identities.
- Disrupt subnet consensus stability during the rotation window.

The registry is the authoritative source of subnet membership for all replicas; a forged membership record propagates to all nodes via certified state.

**Vulnerability class:** Governance authorization bug — privileged registry mutation callable without governance approval.

---

### Likelihood Explanation

Node operators are a non-trivial set of principals on the IC mainnet. Any one of them with a node in a sensitive subnet (e.g., NNS) can exploit this immediately, with no special tooling beyond a standard canister update call. The feature flag defaults to enabled and the caller policy defaults to allow-all, so no precondition needs to be satisfied beyond holding a registered node operator identity. Likelihood is **medium-high**.

---

### Recommendation

1. **Change the default of `NODE_SWAPPING_CALLERS_POLICY` from `AccessList::allow_all()` to `AccessList::deny_all()`** in `rs/registry/canister/src/flags.rs`. This ensures that unless a caller is explicitly whitelisted at init time, the endpoint rejects them.

2. **Change the default of `IS_NODE_SWAPPING_ENABLED` from `true` to `false`** so the feature is opt-in rather than opt-out.

3. **Add a canister-level access control check** at the `swap_node_in_subnet_directly` entry point (analogous to `check_caller_is_governance_and_log` used by `update_node_operator_config`) or require a governance proposal for swaps on sensitive subnets (e.g., NNS, SNS).

4. Consider converting the single `FOUR_POOL_GAUGE_ADDRESS`-style global flag into a per-subnet mapping (already partially done via `NODE_SWAPPING_SUBNETS_POLICY`) and ensure that mapping defaults to deny-all.

---

### Proof of Concept

An attacker who is a registered node operator with principal `ATTACKER_OP` owning `NODE_A` (currently in the NNS subnet) and `NODE_B` (unassigned) calls:

```
dfx canister --network ic call <registry_canister_id> swap_node_in_subnet_directly \
  '(record { old_node_id = opt principal "NODE_A"; new_node_id = opt principal "NODE_B" })'
```

Because:
- `IS_NODE_SWAPPING_ENABLED` = `true` (static default) → feature enabled check passes
- `NODE_SWAPPING_CALLERS_POLICY` = `allow_all()` (static default) → caller whitelist check passes
- `NODE_SWAPPING_SUBNETS_POLICY` = `allow_all()` (static default) → subnet whitelist check passes
- `ATTACKER_OP` is the `node_operator_id` for both `NODE_A` and `NODE_B` → ownership check passes

The registry writes a new `SubnetRecord` replacing `NODE_A` with `NODE_B` in the NNS subnet membership, with no governance proposal ever submitted. [9](#0-8) [10](#0-9)

    pub fn do_swap_node_in_subnet_directly(&mut self, payload: SwapNodeInSubnetDirectlyPayload) {
        self.swap_nodes_inner(payload, dfn_core::api::caller(), now_system_time())
            .unwrap_or_else(|e| panic!("{e}"));
    }

    /// Top level function for the swapping feature which has all inputs.
    fn swap_nodes_inner(
        &mut self,
        payload: SwapNodeInSubnetDirectlyPayload,
        caller: PrincipalId,
        now: SystemTime,
    ) -> Result<(), SwapError> {
        // Check if the feature is enabled on the network.
        if !is_node_swapping_enabled() {
            return Err(SwapError::FeatureDisabled);
        }

        // Check if the payload is valid by itself.
        payload.validate()?;
        let (old_node_id, new_node_id) =
            (payload.old_node_id.unwrap(), payload.new_node_id.unwrap());

        //Check if the feature is allowed on the target subnet and for the caller
        Self::swapping_enabled_for_caller(caller)?;
        let subnet_id = self.find_subnet_for_old_node(old_node_id)?;
        Self::swapping_allowed_on_subnet(subnet_id)?;

        let reservation =
            SWAP_LIMITER.with_borrow_mut(|limiter| limiter.try_reserve(caller, subnet_id, now))?;

        self.validate_node_swap(old_node_id, new_node_id, caller, subnet_id)?;
        self.swap_nodes_in_subnet(subnet_id, old_node_id, new_node_id)?;

        SWAP_LIMITER.with_borrow_mut(|limiter| limiter.commit(reservation, now));
        Ok(())
    }
