### Title
`ExitToEthereum` Precompile Emits Fire-and-Forget `withdraw` Promise With No Error Callback, Permanently Freezing Burned Tokens on Failure - (File: engine-precompiles/src/native.rs)

---

### Summary

The `ExitToEthereum` precompile in Aurora Engine burns ERC-20 tokens (or deducts ETH) in the EVM as part of a bridge-exit flow, then schedules a NEAR promise to call `withdraw` on the ETH connector. Unlike the `ExitToNear` precompile — which conditionally attaches an `exit_to_near_precompile_callback` to handle failures and refund the caller — `ExitToEthereum` always emits a bare `PromiseArgs::Create` with no error callback. If the downstream `withdraw` promise fails for any reason (e.g., connector paused, gas exhaustion in the connector, or any revert), the EVM-side burn is already committed and irrecoverable, permanently freezing the user's funds.

---

### Finding Description

In `ExitToEthereum::run`, after constructing the `withdraw_promise`, the code unconditionally wraps it in `PromiseArgs::Create`:

```rust
// engine-precompiles/src/native.rs  ~line 977
let withdraw_promise = PromiseCreateArgs {
    target_account_id: nep141_address,
    method: "withdraw".to_string(),
    args: serialized_args,
    attached_balance: Yocto::new(1),
    attached_gas: costs::WITHDRAWAL_GAS,
};

let promise = borsh::to_vec(&PromiseArgs::Create(withdraw_promise)).unwrap();
```

`PromiseArgs::Create` is a fire-and-forget promise. There is no `PromiseArgs::Callback` variant attached, so the Aurora Engine contract never receives the outcome of the `withdraw` call.
