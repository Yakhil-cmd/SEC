### Title
Unprotected `new` Initialization Allows Front-Running to Seize Engine Ownership - (File: engine/src/contract_methods/admin.rs)

---

### Summary

The `new` function that initializes the Aurora Engine's on-chain state carries no caller access control. Any NEAR account can invoke it before the legitimate deployer does, supplying attacker-controlled arguments — including a malicious `owner_id`. Because the owner controls contract upgrades, a successful front-run yields full administrative control over the engine and all bridged funds it holds.

---

### Finding Description

`admin::new` is the sole initialization entry point for the Aurora Engine contract state:

```rust
pub fn new<I: IO + Copy, E: Env>(mut io: I, env: &E) -> Result<(), ContractError> {
    if state::get_state(&io).is_ok() {
        return Err(b"ERR_ALREADY_INITIALIZED".into());
    }

    let input = io.read_input().to_vec();
    let args = NewCallArgs::deserialize(&input).map_err(|_| errors::ERR_BORSH_DESERIALIZE)?;
    ...
    state::set_state(&mut io, &state)?;
    Ok(())
}
``` [1](#0-0) 

The only guard is the already-initialized check on line 57. There is no check that `env.predecessor_account_id()` equals `env.current_account_id()` or any other privileged account. The function is exposed as a public NEAR contract method callable by any account.

The `EngineState` written by this call includes `owner_id`, `chain_id`, and `upgrade_delay_blocks`. The `owner_id` field is the sole authority checked by every subsequent privileged operation: [2](#0-1) 

If an attacker supplies `owner_id = attacker.near` and `upgrade_delay_blocks = 0`, they immediately satisfy the preconditions for `stage_upgrade` + `upgrade`: [3](#0-2) [4](#0-3) 

Both functions gate on `require_owner_only`, which the attacker now passes. The `upgrade` function deploys arbitrary WASM to the engine account, giving the attacker full code-level control.

The test harness confirms that `new` is called as a **separate transaction** after contract deployment, creating the exploitable window: [5](#0-4) 

---

### Impact Explanation

**Critical — Direct theft of all user funds.**

Once the attacker is `owner_id` they can:

1. Call `stage_upgrade` with `upgrade_delay_blocks = 0` (which they set during `new`) to stage malicious WASM immediately.
2. Call `upgrade`, which deploys the malicious WASM to the engine account via a NEAR batch promise.
3. The malicious contract can drain all ETH balances tracked in the EVM state and all NEP-141 tokens held by the bridge connector.

All bridged ETH and NEP-141 tokens are at risk. The legitimate deployer cannot recover ownership because `set_owner` itself requires `require_owner_only`, and the attacker already holds that role. [6](#0-5) 

---

### Likelihood Explanation

**Medium.**

The attack requires the deployer to submit the `new` call as a transaction separate from the `DeployContract` action — i.e., not using a NEAR batch transaction that atomically combines deployment and initialization. The test infrastructure and documented deployment flow do exactly this (separate `call("new")` transaction). An attacker monitoring the NEAR RPC for a freshly deployed Aurora Engine account can submit a competing `new` call in the same or next block. NEAR block times are approximately one second, making this a realistic race condition for any deployment that does not use an atomic batch.

---

### Recommendation

**Short term:**
- Add a caller check inside `new`: assert `env.predecessor_account_id() == env.current_account_id()`. This restricts initialization to self-calls, which can only originate from a NEAR batch that includes the `DeployContract` action.
- Alternatively, add an explicit check that the predecessor is the account that deployed the contract (e.g., the root/parent account).

**Long term:**
- Combine `DeployContract` and the `new` function call into a single NEAR batch transaction in all deployment scripts and documentation. This eliminates the race window entirely, analogous to moving price initialization into the Solidity constructor.
- Document and test that any upgrade path that re-deploys the contract also re-initializes atomically.

---

### Proof of Concept

1. Attacker watches the NEAR network for a new Aurora Engine contract deployment (account with no state yet).
2. Attacker immediately submits a call to `new` with:
   - `owner_id = attacker.near`
   - `upgrade_delay_blocks = 0`
   - Any valid `chain_id`
3. If the attacker's transaction is included before the legitimate deployer's `new` call, the engine state is set with the attacker as owner.
4. The legitimate deployer's `new` call fails with `ERR_ALREADY_INITIALIZED`.
5. Attacker calls `stage_upgrade` with malicious WASM (e.g., a contract that transfers all token balances to the attacker).
6. Attacker calls `upgrade`; the malicious WASM is deployed to the engine account.
7. Attacker invokes the drain function in the malicious contract, stealing all bridged ETH and NEP-141 tokens held by the engine.

#[named]
pub fn new<I: IO + Copy, E: Env>(mut io: I, env: &E) -> Result<(), ContractError> {
    if state::get_state(&io).is_ok() {
        return Err(b"ERR_ALREADY_INITIALIZED".into());
    }

    let input = io.read_input().to_vec();
    let args = NewCallArgs::deserialize(&input).map_err(|_| errors::ERR_BORSH_DESERIALIZE)?;

    let initial_hashchain = args.initial_hashchain();
    let state: EngineState = args.into();

    if let Some(block_hashchain) = initial_hashchain {
        let block_height = env.block_height();
        let mut hashchain = Hashchain::new(
            state.chain_id,
            env.current_account_id(),
            block_height,
            block_hashchain,
        );

        hashchain.add_block_tx(
            block_height,
            function_name!(),
            &input,
            &[],
            &Bloom::default(),
        )?;
        crate::hashchain::save_hashchain(&mut io, &hashchain)?;
    }

    state::set_state(&mut io, &state)?;
    Ok(())
}

pub fn set_owner<I: IO + Copy, E: Env>(io: I, env: &E) -> Result<(), ContractError> {
    with_hashchain(io, env, function_name!(), |mut io| {
        let mut state = state::get_state(&io)?;

        require_running(&state)?;
        require_owner_only(&state, &env.predecessor_account_id())?;

        let args: SetOwnerArgs = io.read_input_borsh()?;
        if state.owner_id == args.new_owner {
            return Err(errors::ERR_SAME_OWNER.into());
        }

        state.owner_id = args.new_owner;
        state::set_state(&mut io, &state)?;

        Ok(())
    })
}

pub fn upgrade<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<(), ContractError> {
    let state = state::get_state(&io)?;
    require_running(&state)?;
    require_owner_only(&state, &env.predecessor_account_id())?;

    let input = io.read_input().to_vec();
    let (code, state_migration_gas) = match UpgradeParams::try_from_slice(&input) {
        Ok(args) => (
            args.code,
            args.state_migration_gas
                .map_or(GAS_FOR_STATE_MIGRATION, NearGas::new),
        ),
        Err(_) => (input, GAS_FOR_STATE_MIGRATION), // Backward compatibility
    };

    let target_account_id = env.current_account_id();
    let batch = PromiseBatchAction {
        target_account_id,
        actions: vec![
            PromiseAction::DeployContract { code },
            PromiseAction::FunctionCall {
                name: "state_migration".to_string(),
                args: vec![],
                attached_yocto: ZERO_YOCTO,
                gas: state_migration_gas,
            },
        ],
    };
    let promise_id = handler.promise_create_batch(&batch);

    handler.promise_return(promise_id);

    Ok(())
}

pub fn attach_full_access_key<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<(), ContractError> {
    let state = state::get_state(&io)?;

    require_running(&state)?;
    require_owner_only(&state, &env.predecessor_account_id())?;

    let public_key = serde_json::from_slice::<FullAccessKeyArgs>(&io.read_input().to_vec())
        .map(|args| args.public_key)
        .map_err(|_| errors::ERR_JSON_DESERIALIZE)?;
    let current_account_id = env.current_account_id();
    let action = PromiseAction::AddFullAccessKey {
        public_key,
        nonce: 0, // not actually used - depends on block height
    };
    let promise = PromiseBatchAction {
        target_account_id: current_account_id,
        actions: vec![action],
    };
    // SAFETY: This action is dangerous because it adds a new full access key (FAK) to the Engine account.
    // However, it is safe to do so here because of the `require_owner_only` check above; only the
    // (trusted) owner account can add a new FAK.
    let promise_id = handler.promise_create_batch(&promise);

    handler.promise_return(promise_id);

    Ok(())
}
