### Title
`require_running` check in `exit_to_near_precompile_callback` permanently freezes user funds when engine is paused between `ExitToNear` initiation and refund callback - (`engine/src/contract_methods/connector.rs`)

---

### Summary

The `exit_to_near_precompile_callback` function enforces `require_running` before executing the refund path. If the engine owner pauses the engine between the moment a user's `ExitToNear` EVM call burns their tokens and the moment the NEP-141 transfer callback fires (in the failure branch), the refund is never executed and the user's EVM funds are permanently destroyed.

---

### Finding Description

The `ExitToNear` precompile executes a two-step NEAR promise chain:

1. **Initiation (EVM execution):** The user's EVM tokens (ETH or ERC-20) are burned from their Aurora balance. A `PromiseArgs::Callback` log is emitted, which `filter_promises_from_logs` converts into a NEAR promise: a NEP-141 `ft_transfer` call with `exit_to_near_precompile_callback` attached as its callback.

2. **Callback (separate NEAR receipt):** `exit_to_near_precompile_callback` is invoked. If the NEP-141 transfer succeeded, nothing further is needed. If it **failed**, the callback is supposed to call `engine::refund_on_error` to re-mint the burned tokens back to the user.

The vulnerability is that `exit_to_near_precompile_callback` unconditionally calls `require_running` **before** reaching the refund branch:

```rust
// engine/src/contract_methods/connector.rs
pub fn exit_to_near_precompile_callback<...>(...) -> Result<(), ContractError> {
    with_hashchain(io, env, function_name!(), |io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;          // ← blocks everything if paused
        env.assert_private_call()?;
        ...
        } else if let Some(args) = args.refund {
            // Exit call failed; need to refund tokens
            let refund_result = engine::refund_on_error(io, env, state, &args, handler)?;
``` [1](#0-0) 

If the engine is paused at callback time, `require_running` returns an error, the function exits early, and `refund_on_error` is never reached. The NEAR runtime marks the callback receipt as failed and does not retry it. The user's EVM tokens, already burned at step 1, are gone forever.

The `require_running` guard is appropriate for user-initiated calls, but it is incorrect for a **self-callback** that exists solely to undo a state change that has already been committed. The analogous pattern in the reference report is reading the latest `requiredQuorum` at processing time instead of the quorum that was in effect at initiation time.

---

### Impact Explanation

**Critical – Permanent freezing / destruction of user funds.**

The user's EVM balance is debited (tokens burned) atomically during the EVM execution phase. If the NEP-141 transfer subsequently fails and the engine is paused when the callback fires, the refund branch is silently skipped. The user loses their tokens with no recovery path: the NEAR receipt is finalized as failed, there is no retry mechanism, and the burned EVM tokens cannot be re-minted without a separate privileged call.

---

### Likelihood Explanation

**Medium.**

- Engine pausing is a routine, legitimate admin operation (e.g., emergency security response, upgrade preparation). It is not a compromise or malicious act.
- NEP-141 transfer failures are realistic: the recipient NEAR account may lack a storage deposit for the token, the NEP-141 contract itself may be paused, or the recipient account may not exist.
- The window between the EVM execution receipt and the callback receipt spans at least one NEAR block, giving the admin time to pause the engine in between.
- Any user who calls `ExitToNear` while the engine is about to be paused is at risk, with no way to predict or prevent it.

---

### Recommendation

Remove `require_running` from `exit_to_near_precompile_callback`, or restructure the function so that the refund branch executes unconditionally regardless of the engine's pause state. The callback is a private self-call (`env.assert_private_call()`) that only performs a compensating state change; it does not expose any user-controlled execution surface that the pause flag is meant to guard against.

### Proof of Concept

1. User submits an EVM transaction calling the `ExitToNear` precompile to exit 1 ETH to a NEAR recipient account that has **no storage deposit** for the bridged ETH NEP-141 token.
2. Aurora Engine burns 1 ETH from the user's EVM balance and schedules a NEP-141 `ft_transfer` with `exit_to_near_precompile_callback` as the callback.
3. The engine owner calls `pause_contract` (a legitimate maintenance action).
4. The NEP-141 `ft_transfer` receipt executes and **fails** (storage deposit missing).
5. The NEAR runtime fires `exit_to_near_precompile_callback`.
6. `require_running` returns `Err` because the engine is paused; the function returns early.
7. `refund_on_error` is never called; the user's 1 ETH is permanently destroyed. [3](#0-2) [4](#0-3)

pub fn exit_to_near_precompile_callback<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<Option<SubmitResult>, ContractError> {
    with_hashchain(io, env, function_name!(), |io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;
        env.assert_private_call()?;

        // This function should only be called as the callback of
        // exactly one promise.
        if handler.promise_results_count() != 1 {
            return Err(errors::ERR_PROMISE_COUNT.into());
        }

        let args: ExitToNearPrecompileCallbackArgs = io.read_input_borsh()?;

        let maybe_result = if let Some(PromiseResult::Successful(_)) = handler.promise_result(0) {
            if let Some(args) = args.transfer_near {
                let action = PromiseAction::Transfer {
                    amount: Yocto::new(args.amount),
                };
                let promise = PromiseBatchAction {
                    target_account_id: args.target_account_id,
                    actions: vec![action],
                };

                // Safety: this call is safe because it comes from the exit to near precompile, not users.
                // The call is to transfer the unwrapped wNEAR tokens.
                let promise_id = handler.promise_create_batch(&promise);
                handler.promise_return(promise_id);
            }

            None
        } else if let Some(args) = args.refund {
            // Exit call failed; need to refund tokens
            let refund_result = engine::refund_on_error(io, env, state, &args, handler)?;

            if !refund_result.status.is_ok() {
                return Err(errors::ERR_REFUND_FAILURE.into());
            }

            Some(refund_result)
        } else {
            None
        };

        Ok(maybe_result)
    })
}
