Audit Report

## Title
Ed25519 Small-Order Public Key Accepted in Ingress Verification Enables Signature Forgery for a Fixed Derived Principal — (`rs/crypto/standalone-sig-verifier/src/lib.rs`, `packages/ic-ed25519/src/lib.rs`)

## Summary
`verify_basic_sig_by_public_key` calls `PublicKey::deserialize_raw` (which explicitly omits torsion/subgroup checks) followed by `verify_signature`, which uses a cofactor-based ZIP215 check. For any public key of order dividing 8 (e.g., `[0u8;32]`) with forged signature `(R=identity, S=0)`, the cofactor check `(recomputed_r − R)·8 = identity` is algebraically satisfied for every possible message. An attacker can derive the self-authenticating principal for such a key and submit ingress messages that pass all validation as that principal.

## Finding Description

**Root cause — missing torsion check in the ingress verification path:**

`PublicKey::deserialize_raw` explicitly documents that it does not check subgroup membership: [1](#0-0) 

`verify_basic_sig_by_public_key` calls `deserialize_raw` then `verify_signature` with no intervening torsion check: [2](#0-1) 

**Why `(R=identity, S=0)` passes `verify_signature` for any small-order key:**

`verify_signature` computes `recomputed_r = S·B + k·(−A)` and checks `(recomputed_r − R)·8 = identity`: [3](#0-2) 

With `S=0`, `R=identity`, and `A` of order dividing 8:
- `recomputed_r = 0·B + k·(−A) = k·(−A)`
- `(k·(−A) − identity)·8 = k·(8·(−A)) − identity·8 = k·identity − identity = identity − identity = identity ✓`

This holds for **any** message (any `k`).

**`(R=identity, S=0)` is accepted by `Signature::from_slice`:**

`CompressedEdwardsY([1,0,...,0])` decompresses to the identity point; `Scalar::from_canonical_bytes([0u8;32])` succeeds since 0 is a valid canonical scalar. No rejection of either value occurs: [4](#0-3) 

**`[0u8;32]` is confirmed to be a small-order point (order 8) by an existing test:** [5](#0-4) 

**The ingress path never calls `verify_public_key`:**

`validate_user_id_and_signature` → `validate_signature` → `validate_signature_plain` → `validator.verify_basic_sig_by_public_key`. No step calls `verify_public_key` (which does call `is_torsion_free()`): [6](#0-5) [7](#0-6) 

The torsion check exists in `verify_public_key` and in node key validation, but is absent from the ingress signature verification path: [8](#0-7) [9](#0-8) 

## Impact Explanation
An attacker can forge valid ingress signatures for the specific self-authenticating principal derived from any small-order Ed25519 key (e.g., `SHA224(DER([0u8;32])) || 0x02`). This constitutes unauthorized access to that principal's identity on the IC. The impact is constrained: only the one fixed, publicly computable principal is affected — arbitrary user impersonation is not possible. This maps to **Medium** severity: a concrete authentication bypass requiring no privileges, but limited to a single deterministic principal whose practical permissions depend on canister-level grants.

## Likelihood Explanation
No preconditions, no privileged access, no secret key material required. The attack is fully constructible offline. The forged principal is deterministic and publicly known, so practical damage is limited unless a canister or governance entity has explicitly granted permissions to that principal. Repeatability is unlimited.

## Recommendation
In `verify_basic_sig_by_public_key` (or in `PublicKey::deserialize_raw` when used for signature verification contexts), add a subgroup check after deserialization:

```rust
if !pk.is_torsion_free() {
    return Err(CryptoError::MalformedPublicKey {
        algorithm: AlgorithmId::Ed25519,
        key_bytes: Some(pk_bytes.to_vec()),
        internal_error: "Public key has torsion component".to_string(),
    });
}
```

This mirrors the existing guard in `verify_public_key` and `ValidNodeSigningPublicKey`.

## Proof of Concept

```rust
// 1. Small-order public key (order 8), accepted by deserialize_raw
let pk_raw = [0u8; 32];

// 2. DER-encode and derive self-authenticating principal
let pk_der = /* standard Ed25519 DER prefix */ + pk_raw;
let principal = PrincipalId::new_self_authenticating(&pk_der);

// 3. Forged signature: R = identity encoding, S = 0
let mut forged_sig = [0u8; 64];
forged_sig[0] = 1; // CompressedEdwardsY identity = [1, 0, ..., 0]

// 4. Verify directly:
// verify_basic_sig_by_public_key(Ed25519, any_msg, &forged_sig, &pk_raw) → Ok(())

// 5. Submit ingress message with sender=principal, signer_pubkey=pk_der,
//    signature=forged_sig → passes all ingress validation
```

A unit test in `rs/crypto/standalone-sig-verifier/src/lib.rs` can confirm step 4 directly. A PocketIC integration test can confirm step 5 end-to-end.
