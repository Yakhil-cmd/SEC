Audit Report

## Title
Unvalidated Pool Unbounded Growth via Injected `CatchUpPackageShare` at Unfinalized Height тАФ (`rs/consensus/src/consensus/purger.rs`)

## Summary
A Byzantine P2P peer below the fault threshold can permanently suppress unvalidated-pool purging on an honest replica by continuously injecting `CatchUpPackageShare` artifacts at heights that have no finalized block. The validator leaves such shares in the unvalidated pool indefinitely (returning `None`), the purger's skip-guard fires on every round, and `prev_expected_batch_height` is never updated. The unvalidated pool grows without bound until the replica is OOM-killed.

## Finding Description

**Step 1 тАФ Purger skip-guard is triggered by a single injected share.**

`purge_unvalidated_pool_by_expected_batch_height` at `rs/consensus/src/consensus/purger.rs` lines 193тАУ218 skips purging when both predicates hold simultaneously:

```
below_range_max(catch_up_height, range)  тЖТ  catch_up_height < range.max
above_range_min(expected_batch_height, range)  тЖТ  expected_batch_height > range.min
``` [1](#0-0) 

A single share at height H where `catch_up_height < H < expected_batch_height` satisfies both predicates (since `range.min = range.max = H`), causing an immediate `return` before `*prev_expected_batch_height = expected_batch_height` is reached at line 219. Because `prev_expected_batch_height` is never updated, the outer guard `*prev_expected_batch_height < expected_batch_height` remains true on every subsequent round, and the skip fires again each time the attacker injects a fresh share at the new boundary height. [2](#0-1) 

**Step 2 тАФ `check_integrity()` is trivially satisfiable.**

The only pre-signature integrity check for a `CatchUpPackageShare` verifies that the embedded random beacon's hash field equals `crypto_hash(random_beacon)`: [3](#0-2) 

An attacker can choose any random beacon value and compute the matching hash. No subnet membership, no threshold key, and no valid block reference is required to pass this check.

**Step 3 тАФ Validator leaves shares at unfinalized heights in the pool forever.**

`validate_catch_up_package_shares` calls `check_integrity()` first (line 1622), then calls `validate_catch_up_share_content`. That function immediately returns `Err(ValidationFailure::FinalizedBlockNotFound(height))` when no finalized block exists at the share's height: [4](#0-3) 

The `ValidationFailed` arm returns `None` тАФ no `HandleInvalid`, no `RemoveFromUnvalidated`: [5](#0-4) 

The signature is never verified in this code path. The share remains in the unvalidated pool indefinitely because a finalized block at a fabricated height never appears.

**Step 4 тАФ No unvalidated pool size bounds exist.**

`bounds.rs` defines and enforces `validated_pool_within_bounds`, which checks only the **validated** pool: [6](#0-5) 

There is no equivalent bound on the unvalidated pool. The `check_integrity` call is not present at the P2P/artifact-pool insertion layer (confirmed absent in `rs/artifact_pool/` and `rs/p2p/`), so artifacts enter the unvalidated pool freely before any validation occurs.

## Impact Explanation

The unvalidated pool is explicitly documented as in-memory only and volatile: [7](#0-6) 

Unbounded accumulation of injected shares causes heap exhaustion and OOM-kill of the replica process. Applied simultaneously to multiple replicas (still below the `f` fault threshold), this constitutes a subnet-level availability impact. This matches the **High** impact category: *Application/platform-level DoS, crash, consensus blocking, certified-state disruption, or subnet availability impact not based on raw volumetric DDoS.*

## Likelihood Explanation

The attacker requires only:
- Standard P2P connectivity to the target replica (normal subnet peer relationship, below the fault threshold)
- The ability to craft a `CatchUpPackageShare` with a self-consistent random beacon hash (trivial arithmetic, no cryptographic secret)
- Knowledge of the current `catch_up_height` (publicly observable from the pool)

No threshold key, no subnet majority, and no admin access are required. The attack is repeatable every consensus round and can be directed at multiple replicas simultaneously.

## Recommendation

1. **Emit `RemoveFromUnvalidated` when `FinalizedBlockNotFound` persists beyond a timeout.** The existing `unvalidated_for_too_long` path only logs a warning; it should also return `Some(ChangeAction::HandleInvalid(...))` or `Some(ChangeAction::RemoveFromUnvalidated(...))` to evict the share.
2. **Bound the unvalidated pool per artifact type**, analogous to the existing `validated_pool_within_bounds` check, to cap worst-case memory impact regardless of injection rate.
3. **Move the purge-skip guard to fire only after the validator has had a chance to remove invalid artifacts**, or add a staleness threshold after which the guard no longer defers purging for shares that have never progressed to validation.

## Proof of Concept

```
Setup:
  catch_up_height = 10
  expected_batch_height = 12
  finalized_height = 11

Attacker action (each consensus round):
  Craft CatchUpPackageShare at height 11:
    - random_beacon = RandomBeacon::fake(height=11)
    - content.random_beacon.hash = crypto_hash(&random_beacon)  // passes check_integrity()
    - signature = ThresholdSignatureShare::fake(attacker_node_id) // never reached
  Inject via P2P into target replica's unvalidated pool.

Purger round:
  unvalidated_catch_up_share_range = Some(HeightRange { min: 11, max: 11 })
  below_range_max(10, range) = 10 < 11 = true
  above_range_min(12, range) = 12 > 11 = true
  тЖТ early return; prev_expected_batch_height unchanged

Validator round:
  validate_catch_up_share_content(height=11):
    get_finalized_block(11) = None
    тЖТ Err(ValidationFailure::FinalizedBlockNotFound(11))
  тЖТ None (share stays in pool)

Next round: expected_batch_height = 13, catch_up_height = 11
  Attacker injects share at height 12.
  Same bypass fires. Pool grows by one share per round.
```

A deterministic integration test can be written using `ic_consensus_mocks::Dependencies`: advance the pool normally, inject a `CatchUpPackageShare` at `catch_up_height + 1` without a corresponding finalized block, invoke the purger and validator repeatedly, and assert that `unvalidated().catch_up_package_share().size()` grows monotonically while `PurgeUnvalidatedBelow` is never emitted.

pub fn validated_pool_within_bounds(

    replica_config: &ReplicaConfig,

        || actual_counts.cups > bounds.cups
