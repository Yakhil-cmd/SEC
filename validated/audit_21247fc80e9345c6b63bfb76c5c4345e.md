Audit Report

## Title
Fee Accounting Cleared Unconditionally When Burn Is Skipped in `disburse_neuron` - (File: rs/nns/governance/src/governance.rs)

## Summary
In NNS Governance's `disburse_neuron`, `neuron_fees_e8s` is zeroed and `cached_neuron_stake_e8s` is reduced unconditionally after the conditional fee-burn block, even when the burn is skipped because `fees_amount_e8s <= transaction_fee_e8s`. This creates a permanent ledger/governance state divergence: the fee tokens remain in the neuron's subaccount but governance no longer tracks them, meaning they are never burned and the ICP supply is silently inflated by the unburned fee amount.

## Finding Description
In `rs/nns/governance/src/governance.rs` at L2046–2076, the fee burn is gated:

```rust
if fees_amount_e8s > transaction_fee_e8s {
    self.ledger.transfer_funds(fees_amount_e8s, 0, ...).await?;
}
// Unconditional state update follows:
self.with_neuron_mut(id, |neuron| {
    if neuron.cached_neuron_stake_e8s > fees_amount_e8s {
        neuron.cached_neuron_stake_e8s -= fees_amount_e8s;
    } else {
        neuron.cached_neuron_stake_e8s = 0;
    }
    neuron.neuron_fees_e8s = 0;  // always zeroed
})
```

When `fees_amount_e8s <= transaction_fee_e8s`, the ledger burn is skipped entirely, but governance still subtracts `fees_amount_e8s` from `cached_neuron_stake_e8s` and sets `neuron_fees_e8s = 0`. Transfer 2 (the disburse) then sends `neuron_minted_stake_e8s - transaction_fee_e8s` from the subaccount, which was computed as `cached_neuron_stake_e8s - fees_amount_e8s`. The net result: the subaccount retains `fees_amount_e8s` e8s that governance believes were burned, but were not. Governance's `cached_neuron_stake_e8s` reaches 0 after Transfer 2's state update, while the ledger subaccount holds `fees_amount_e8s` residual tokens.

The SNS governance implementation explicitly avoids this: the state update for `cached_neuron_stake_e8s` and `neuron_fees_e8s` is inside the `if max_burnable_fee > transaction_fee_e8s` block at `rs/sns/governance/src/governance.rs` L1181–1209, with an explicit comment: *"We only update the cached_neuron_stake_e8s and neuron_fees_e8s if we actually burn fees, otherwise this leads to ledger and governance getting out of sync."* [1](#0-0) [2](#0-1) 

## Impact Explanation
The concrete impact is a governance conservation violation: fee tokens that should be burned on the ICP ledger are not burned, causing the actual ICP supply to exceed what NNS governance accounts for by up to `transaction_fee_e8s - 1` (at most 9,999 e8s ≈ 0.0001 ICP) per affected disburse call. The residual tokens are stranded in the neuron's subaccount — governance no longer tracks them (fees zeroed, stake reduced to 0), so no future governance operation will burn or transfer them. A `claim_or_refresh_neuron` call can detect the ledger surplus and re-credit `cached_neuron_stake_e8s`, but the recovered amount (≤ 9,999 e8s) is below the NNS minimum neuron stake, limiting practical re-exploitation. The primary impact is a repeatable, small-scale ICP supply inflation and NNS ledger/governance state divergence — a concrete NNS governance protocol harm. This maps to **Medium** severity: moderate user-funds/protocol impact, easily triggered, but bounded per call to a small amount.

## Likelihood Explanation
Any NNS neuron controller can trigger this by calling `disburse_neuron` on a dissolved neuron that has `0 < neuron_fees_e8s <= transaction_fee_e8s`. This arises naturally when a neuron submitted one rejected proposal and the configured `reject_cost_e8s` is at or below 10,000 e8s. No privileged access is required; the caller is an ordinary ingress sender (the neuron controller). The condition is realistic and repeatable across many neurons.

## Recommendation
Mirror the SNS governance fix: move the `cached_neuron_stake_e8s` and `neuron_fees_e8s` state updates inside the `if fees_amount_e8s > transaction_fee_e8s` block, so governance state is only modified when the burn actually occurs. When the burn is skipped, neither field should be modified.

```rust
if fees_amount_e8s > transaction_fee_e8s {
    self.ledger.transfer_funds(fees_amount_e8s, 0, ...).await?;
    self.with_neuron_mut(id, |neuron| {
        neuron.cached_neuron_stake_e8s =
            neuron.cached_neuron_stake_e8s.saturating_sub(fees_amount_e8s);
        neuron.neuron_fees_e8s = neuron.neuron_fees_e8s.saturating_sub(fees_amount_e8s);
    })?;
}
// Do NOT unconditionally zero neuron_fees_e8s here
```

## Proof of Concept
1. Create a dissolved NNS neuron with `cached_neuron_stake_e8s = 100_000` and `neuron_fees_e8s = 5_000` (below `transaction_fee_e8s = 10_000`).
2. Call `disburse_neuron` with no explicit amount.
3. `fees_amount_e8s (5_000) > transaction_fee_e8s (10_000)` is false → no burn ledger call.
4. Governance unconditionally sets `cached_neuron_stake_e8s = 95_000`, `neuron_fees_e8s = 0`.
5. Transfer 2 sends `85_000` e8s (95,000 − 10,000 tx fee) from subaccount; governance reduces `cached_neuron_stake_e8s` by `85_000 + 10_000 = 95_000` → reaches 0.
6. Ledger subaccount: `100_000 − 85_000 − 10_000 = 5_000` e8s remaining (never burned).
7. Governance: `cached_neuron_stake_e8s = 0`, `neuron_fees_e8s = 0` — believes fees were burned.
8. The 5,000 e8s are permanently stranded and untracked; the ICP supply is 5,000 e8s higher than governance accounts for.
9. A unit test using a mock ledger can assert that after this disburse, the mock ledger's burn call was never invoked but governance's `neuron_fees_e8s` is 0, confirming the divergence. [3](#0-2) [4](#0-3)
