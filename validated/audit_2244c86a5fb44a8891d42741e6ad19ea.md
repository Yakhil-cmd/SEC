### Title
BLS12-381 G2 MSM Precompile Accepts Unbounded Input That Exceeds NEAR Gas Limit, Making Transactions Permanently Unexecutable - (File: `engine-precompiles/src/bls12_381/g2_msm.rs`)

### Summary

The `BlsG2Msm` precompile accepts arbitrarily large inputs (any multiple of 288 bytes) and charges EVM gas correctly per EIP-2537, but the actual NEAR gas consumed by the underlying BLS12-381 multi-scalar multiplication loop grows linearly with the number of pairs `k` and can exceed NEAR's hard 200 Tgas per-transaction limit. The codebase itself acknowledges this in the test suite. Any transaction invoking this precompile with a sufficiently large input will be permanently unexecutable regardless of how much EVM gas the caller provides.

### Finding Description

`BlsG2Msm::run` in `engine-precompiles/src/bls12_381/g2_msm.rs` validates only that the input length is a non-zero multiple of `G2_MUL_INPUT_LENGTH` (288 bytes) and that the caller has enough EVM gas per the EIP-2537 formula. It then calls `Self::execute(input)`, which delegates to `bls12_381::g2_msm(input)`. [1](#0-0) 

The EVM gas formula in `msm_required_gas` caps the discount at table index 127 (for `k ≥ 128`) and grows linearly as `k × 524 × 22500 / 1000 = k × 11790` EVM gas: [2](#0-1) 

The underlying execution in `g2_msm` (standalone path) iterates over all `k` pairs in a `for i in 0..k` loop, performing expensive subgroup checks and field-element extractions per iteration, then calls the multi-exponentiation: [3](#0-2) 

The NEAR gas consumed by this loop and the final `mult` call grows linearly with `k` and is entirely independent of the EVM gas charged. The test suite explicitly acknowledges the resulting NEAR gas limit breach: [4](#0-3) 

The comment `// Maximum input size for BlsG2Msm before NEAR gas limits are exceeded` at `BLS_G2_MSM_MAX_INPUT_SIZE = 41_000` bytes (≈142 pairs) confirms the production precompile has no corresponding guard.

### Impact Explanation

NEAR enforces a hard 200 Tgas limit per transaction. A caller who submits a transaction invoking `BlsG2Msm` with input larger than ~41 KB will have their EVM gas check pass (EIP-2537 formula is satisfied), but the NEAR runtime will abort the transaction with a gas-limit error before the EVM result is committed. Any EVM contract whose only path to release locked funds requires a large G2 MSM call becomes permanently inaccessible — the transaction cannot be split across blocks and no amount of EVM gas increase helps, because the bottleneck is NEAR gas, not EVM gas. This constitutes **temporary (and in practice permanent for that call path) freezing of funds**.

### Likelihood Explanation

The precompile is reachable by any EVM user or contract on Aurora. No special privilege is required. An attacker or a legitimate user of a ZK-proof-verifying contract can trivially craft calldata with `k > 142` G2 pairs (input length > 40,896 bytes, a multiple of 288). The EVM gas check passes, the NEAR gas limit is exceeded, and the transaction is irrecoverably rejected. The test suite already demonstrates this threshold is reachable in practice.

### Recommendation

Add an explicit upper bound on the number of G2 pairs accepted by `BlsG2Msm::run` before the gas check, mirroring the empirically determined limit from the test suite:

```rust
const MAX_G2_MSM_PAIRS: usize = 142; // ~41 KB, stays within 200 Tgas

let k = input_len / G2_MUL_INPUT_LENGTH;
if k > MAX_G2_MSM_PAIRS {
    return Err(ExitError::Other(Borrowed("ERR_BLS_G2MSM_INPUT_TOO_LARGE")));
}
```

Alternatively, calibrate the EVM gas cost to accurately reflect the NEAR gas consumed per pair so that the EVM gas limit naturally prevents NEAR gas exhaustion.

### Proof of Concept

1. Deploy any EVM contract on Aurora that calls the `BlsG2Msm` precompile at address `0x000…0E` with `k = 200` valid G2 pairs (57,600 bytes of input).
2. The EVM gas required is `200 × 11790 = 2,358,000` — well within any reasonable EVM gas limit.
3. The transaction is submitted with sufficient EVM gas.
4. The NEAR runtime aborts the transaction with `GasLimit` before the EVM result is written, as confirmed by the test-suite constant `BLS_G2_MSM_MAX_INPUT_SIZE = 41_000` (≈142 pairs).
5. Any funds locked behind this call path cannot be recovered via this code path. [5](#0-4) [6](#0-5)

impl Precompile for BlsG2Msm {
    fn required_gas(input: &[u8]) -> Result<EthGas, ExitError>
    where
        Self: Sized,
    {
        let k = input.len() / G2_MUL_INPUT_LENGTH;
        Ok(EthGas::new(msm_required_gas(
            k,
            &DISCOUNT_TABLE,
            BASE_GAS_FEE,
        )?))
    }

    /// Implements EIP-2537 G2MSM precompile.
    /// G2 multi-scalar-multiplication call expects `288*k` bytes as an input that is interpreted
    /// as byte concatenation of `k` slices each of them being a byte concatenation
    /// of encoding of G2 point (`256` bytes) and encoding of a scalar value (`32`
    /// bytes).
    /// Output is an encoding of multi-scalar-multiplication operation result - single G2
    /// point (`256` bytes).
    /// See also: <https://eips.ethereum.org/EIPS/eip-2537#abi-for-g2-multiexponentiation>
    fn run(
        &self,
        input: &[u8],
        target_gas: Option<EthGas>,
        _context: &Context,
        _is_static: bool,
    ) -> EvmPrecompileResult {
        let input_len = input.len();
        if input_len == 0 || !input_len.is_multiple_of(G2_MUL_INPUT_LENGTH) {
            return Err(ExitError::Other(Borrowed("ERR_BLS_G2MSM_INPUT_LEN")));
        }

        let cost = Self::required_gas(input)?;
        if let Some(target_gas) = target_gas
            && cost > target_gas
        {
            return Err(ExitError::OutOfGas);
        }

        let output = Self::execute(input)?;
        Ok(PrecompileOutput::without_logs(cost, output))
    }

/// Amount used to calculate the multi-scalar-multiplication discount.
const MSM_MULTIPLIER: u64 = 1000;

/// Implements the gas schedule for G1/G2 Multiscalar-multiplication assuming 30
/// MGas/second, see also: <https://eips.ethereum.org/EIPS/eip-2537#g1g2-multiexponentiation>
fn msm_required_gas(
    k: usize,
    discount_table: &[u16],
    multiplication_cost: u64,
) -> Result<u64, aurora_evm::ExitError> {
    if k == 0 {
        return Ok(0);
    }

    let index = core::cmp::min(k - 1, discount_table.len() - 1);
    let discount = u64::from(discount_table[index]);

    let k = u64::try_from(k).map_err(crate::utils::err_usize_conv)?;
    Ok((k * discount * multiplication_cost) / MSM_MULTIPLIER)

pub fn g1_msm(input: &[u8]) -> Result<Vec<u8>, Bls12381Error> {
    let k = input.len() / G1_MUL_INPUT_LENGTH;
    let mut g1_points: Vec<blst_p1> = Vec::with_capacity(k);
    let mut scalars: Vec<u8> = Vec::with_capacity(k * SCALAR_LENGTH);
    for i in 0..k {
        let slice = &input[i * G1_MUL_INPUT_LENGTH..i * G1_MUL_INPUT_LENGTH + G1_INPUT_ITEM_LENGTH];

        // BLST batch API for p1_affines blows up when you pass it a point at infinity, so we must
        // filter points at infinity (and their corresponding scalars) from the input.
        if slice.iter().all(|i| *i == 0) {
            continue;
        }

        // NB: Scalar multiplications, MSMs and pairings MUST perform a subgroup check.
        //
        // So we set the subgroup_check flag to `true`
        let p0_aff = &g1::extract_g1_input(slice, true)?;

        let mut p0 = blst_p1::default();
        // SAFETY: p0 and p0_aff are blst values.
        unsafe { blst_p1_from_affine(&raw mut p0, p0_aff) };
        g1_points.push(p0);

        scalars.extend_from_slice(
            &utils::extract_scalar_input(
                &input[i * G1_MUL_INPUT_LENGTH + G1_INPUT_ITEM_LENGTH
                    ..i * G1_MUL_INPUT_LENGTH + G1_INPUT_ITEM_LENGTH + SCALAR_LENGTH],
            )?
            .b,
        );
    }

    // return infinity point if all points are infinity
    if g1_points.is_empty() {
        return Ok([0; 128].into());
    }

    let points = p1_affines::from(&g1_points);
    let multiexp = points.mult(&scalars, NBITS);

    let mut multiexp_aff = blst_p1_affine::default();
    // SAFETY: multiexp_aff and multiexp are blst values.
    unsafe { blst_p1_to_affine(&raw mut multiexp_aff, &raw const multiexp) };

    Ok(g1::encode_g1_point(&raw const multiexp_aff))
