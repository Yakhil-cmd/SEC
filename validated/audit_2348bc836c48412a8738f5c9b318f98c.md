Audit Report

## Title
Unbounded Neuron Scan and Unguarded Lock in SNS Governance `maybe_finalize_disburse_maturity` Enables Permanent Disbursement Task DoS - (File: `rs/sns/governance/src/governance.rs`)

## Summary

`maybe_finalize_disburse_maturity` iterates over every neuron in the canister's in-memory store before its first `await` point, with no per-invocation cap. Critically, the reentrancy lock `is_finalizing_disburse_maturity` is set to `Some(true)` before the scan and is only cleared at the very end of the function; if the heartbeat message traps (e.g., due to instruction-limit exhaustion from a large neuron population or a large disbursement loop), the lock is never cleared, permanently disabling all future maturity disbursements for every user of that SNS instance. The NNS governance team has already patched the identical class of bug in analogous tasks.

## Finding Description

**Unbounded pre-`await` scan:** At lines 4938–4975 of `rs/sns/governance/src/governance.rs`, the function collects every neuron with a ready disbursement by iterating `self.proto.neurons.values()` — the full `BTreeMap<String, Neuron>` — with no limit. This entire collection executes in a single uninterrupted instruction-counting segment before the first `await`. [1](#0-0) 

**Lock set before scan, cleared only at end:** The reentrancy guard is set at line 4935 and only cleared at line 5082. Any trap between these two points permanently sets the guard. [2](#0-1) [3](#0-2) 

**Unbounded disbursement loop:** After the scan, the function issues one ledger `transfer_funds` call per ready neuron with no per-invocation cap, occupying many consecutive heartbeat rounds when the ready-disbursement set is large. [4](#0-3) 

**Unconditional heartbeat dispatch:** `maybe_finalize_disburse_maturity` is called on every heartbeat with no guard other than the lock that can be permanently set. [5](#0-4) 

**`can_finalize_disburse_maturity` check:** The early-return guard at line 4922 reads `is_finalizing_disburse_maturity`; once permanently set to `Some(true)` by a trap, every subsequent heartbeat invocation returns immediately, freezing all pending disbursements. [6](#0-5) 

**NNS governance has already fixed the analogous issue:** The NNS CHANGELOG explicitly records that the unstaking-maturity task was bounded to 100 neurons per message to prevent instruction-limit exhaustion, and the NNS `disburse_maturity.rs` now processes exactly one neuron per timer invocation via `try_finalize_maturity_disbursement`. No equivalent bound exists in the SNS path. [7](#0-6) [8](#0-7) 

## Impact Explanation

If the SNS neuron population is large enough that the pre-`await` scan or the disbursement loop causes the heartbeat message to trap, `is_finalizing_disburse_maturity` is left permanently set to `Some(true)`. Every subsequent heartbeat invocation of `run_periodic_tasks` hits the early-return at line 4922 and skips the disbursement task entirely. All users who have initiated maturity disbursements on that SNS instance will never receive their tokens until the canister is upgraded with a patched implementation. This is a concrete, application-level DoS on SNS governance disbursement functionality — matching the **High ($2,000–$10,000) Application/platform-level DoS** impact category.

## Likelihood Explanation

Any principal can create SNS neurons by staking the minimum required SNS token amount. The `max_number_of_neurons` parameter caps the total, but at large SNS deployments with many token holders, the neuron count can reach tens of thousands organically. An attacker holding a modest amount of SNS tokens can accelerate this by creating the maximum number of neurons each with a pending disbursement, maximizing per-iteration cost. The secondary lock-out (permanent `is_finalizing_disburse_maturity = Some(true)`) can also be triggered by any other panic path within the function, not only instruction-limit exhaustion.

## Recommendation

1. **Batch the scan with a per-invocation cap**: process at most N neurons per heartbeat (analogous to the NNS `UnstakeMaturityOfDissolvedNeuronsTask` limit of 100 neurons per message), persisting a cursor in canister state so successive heartbeats make forward progress.
2. **Clear the lock on trap**: use a RAII guard (drop impl sets `is_finalizing_disburse_maturity = None`) so the lock is always released even if the function panics or traps.
3. **Cap the disbursement loop**: limit the number of ledger calls issued per heartbeat invocation to bound total round-time impact.
4. **Consider adopting the NNS architecture**: process exactly one neuron per timer invocation (as in `rs/nns/governance/src/governance/disburse_maturity.rs`) to eliminate both the unbounded scan and the lock-permanence risk.

## Proof of Concept

A deterministic integration test can reproduce the lock-out:

1. Deploy an SNS governance canister in a PocketIC environment.
2. Create N neurons (e.g., 10,000) each with a `disburse_maturity_in_progress` entry whose `finalize_disbursement_timestamp_seconds` is in the past.
3. Trigger `run_periodic_tasks` (heartbeat). If N is large enough to exhaust the instruction budget, the message traps.
4. Observe that `proto.is_finalizing_disburse_maturity` is now `Some(true)`.
5. Trigger `run_periodic_tasks` again; observe that `maybe_finalize_disburse_maturity` returns immediately at the `can_finalize_disburse_maturity()` check and no disbursements are processed — confirming permanent lock-out.

Even without reaching the instruction limit, step 4 can be forced by injecting a panic in a test harness after line 4935 to confirm the lock is not cleared on trap.

async fn try_finalize_maturity_disbursement(
    governance: &'static LocalKey<RefCell<Governance>>,
) -> Result<(), FinalizeMaturityDisbursementError> {
    let (maturity_disbursement_finalization, now_seconds) = governance.with_borrow(|governance| {
        let now_seconds = governance.env.now();
        let maturity_modulation = governance
            .heap_data
            .maturity_modulation
            .as_ref()
            .and_then(|m| m.current_value_permyriad);
        let maturity_disbursement_finalization = next_maturity_disbursement_to_finalize(
            &governance.neuron_store,
            &governance.heap_data.in_flight_commands,
            maturity_modulation,
            now_seconds,
        );
        (maturity_disbursement_finalization, now_seconds)
    });
