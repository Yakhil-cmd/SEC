Audit Report

## Title
Unbounded `active_assembles` / `artifact_processor_tasks` Growth via `SLOT_TABLE_NO_LIMIT` Enables OOM Crash — (`rs/p2p/consensus_manager/src/receiver.rs`)

## Summary
Five of six P2P artifact channels (consensus, certifier, DKG, IDKG, HTTPS-outcalls) are configured with `slot_limit = usize::MAX`, rendering the per-peer slot-table guard permanently ineffective. A single Byzantine subnet peer can flood any honest replica with slot updates carrying distinct slot numbers and artifact IDs, causing unbounded growth of `active_assembles` and `artifact_processor_tasks` until the replica OOMs and crashes. Because the Byzantine peer is connected to all honest replicas simultaneously, it can crash all of them in parallel, halting the subnet.

## Finding Description
`SLOT_TABLE_NO_LIMIT = usize::MAX` is defined at [1](#0-0)  and passed to `abortable_broadcast_channel` for consensus, certifier, DKG, IDKG, and HTTPS-outcalls channels. [2](#0-1)  Only ingress uses the finite `SLOT_TABLE_LIMIT_INGRESS = 50_000`. [3](#0-2) 

The guard in `handle_slot_update_receive` is: [4](#0-3)  With `self.slot_limit = usize::MAX`, the condition `peer_slot_table_len < usize::MAX` is always true in practice, so every new `(peer, slot_number)` pair is inserted unconditionally.

For each new slot entry whose artifact ID is not already tracked, a `watch::Sender<PeerCounter>` is inserted into `active_assembles` and a new Tokio task is spawned into `artifact_processor_tasks`: [5](#0-4) 

Tasks only terminate when the advertising peer's counter reaches zero (i.e., the peer overwrites or deletes the slot) or the cancellation token fires: [6](#0-5)  Cleanup of `active_assembles` only occurs after a task joins and the peer counter is empty: [7](#0-6)  A Byzantine peer that never overwrites its slots keeps every spawned task alive indefinitely.

## Impact Explanation
A single Byzantine subnet node, connected via authenticated QUIC/TLS to all honest replicas, can simultaneously flood all of them with N distinct `(slot_number, artifact_id)` pairs per channel. Each honest replica accumulates N entries in `active_assembles` and N live Tokio tasks in `artifact_processor_tasks` with no bound. At sufficient N, all honest replicas OOM-crash in parallel, halting the subnet entirely. This matches the **High** impact class: *Application/platform-level DoS, crash, consensus blocking, or subnet availability impact not based on raw volumetric DDoS.*

## Likelihood Explanation
The attacker must be an authenticated subnet peer (QUIC/TLS mutual auth). A single Byzantine node — within the `f = ⌊(n−1)/3⌋` fault threshold — suffices. No threshold corruption, key compromise, or external dependency is required. The attack is fully local to the P2P layer, trivially scriptable, and repeatable. The Byzantine node is already connected to all honest replicas, so the attack is symmetric across the entire subnet.

## Recommendation
Apply finite per-peer slot limits to all artifact channels, not just ingress. The existing `SLOT_TABLE_LIMIT_INGRESS = 50_000` pattern is correct. Analogous constants should be derived from protocol-level bounds on concurrent advertisements per channel (e.g., bounded by DKG interval length, certification height window, IDKG transcript count) and substituted for `SLOT_TABLE_NO_LIMIT` in [2](#0-1) 

## Proof of Concept
The existing `ReceiverManagerBuilder` test harness with `with_slot_limit` (already used in `slot_table_limit_exceeded` at [8](#0-7) ) directly supports the PoC. Construct a receiver with `slot_limit = usize::MAX`, send N iterations of `handle_slot_update_receive` with distinct `slot_number` and `Update::Id(i)` values from a single peer, and assert `active_assembles.len() == N` and `artifact_processor_tasks.len() == N`. Both assertions pass because no guard fires. At large N (e.g., 100,000), heap usage grows proportionally and the replica OOMs.

        );
        mgr.handle_slot_update_receive(

                slot_number: SlotNumber::from(2),
                commit_id: CommitId::from(2),
                update: Update::Id(1),
            },
            NODE_1,
            ConnId::from(1),
            cancellation.clone(),
