## Title
`AccountState::transfer()` Always Returns `Ok(())` on Partial Neuron-Transfer Failure, Permanently Locking GTC Funds - (File: rs/nns/gtc/src/lib.rs)

## Summary
`AccountState::transfer()` unconditionally removes each neuron ID from `self.neuron_ids` and always returns `Ok(())` regardless of whether individual `transfer_gtc_neuron` calls succeeded or failed. Both callers (`donate_account` and `forward_whitelisted_unclaimed_accounts`) interpret this `Ok(())` as full success and permanently set `has_donated = true` or `has_forwarded = true`. Once set, these flags block any future retry via early-return guards, leaving any neuron whose on-chain transfer failed permanently stranded in the governance canister under GTC control with no recovery path and its ICP stake permanently inaccessible.

## Finding Description
In `rs/nns/gtc/src/lib.rs`, `AccountState::transfer()` (lines 188–209) iterates over cloned neuron IDs and for each one:

1. Calls `GovernanceCanister::transfer_gtc_neuron(neuron_id, custodian_neuron_id).await` and captures the result.
2. **Unconditionally** removes the neuron ID from `self.neuron_ids` via `self.neuron_ids.retain(|id| id != &neuron_id)` (line 192) — before inspecting the result.
3. On `Err`, pushes to `self.failed_transferred_neurons` (a log field only) with no effect on control flow.
4. **Always returns `Ok(())`** at line 209, even if every individual transfer failed.

`donate_account` (lines 89–90) uses `?` to propagate errors from `transfer()`, but since `transfer()` never returns `Err`, `account.has_donated = true` is unconditionally committed. `forward_whitelisted_unclaimed_accounts` (lines 118–119) matches `Ok(_)` to set `account.has_forwarded = true`, which is always reached. The re-entry guards at lines 175–183 then permanently block any future call to `transfer()` for that account. The neuron ID has already been removed from `self.neuron_ids`, so the failed neuron is not claimable, not forwardable, and not donatable — it is orphaned in governance under GTC control with no owner and no mechanism to transfer it again.

`GovernanceCanister::transfer_gtc_neuron` (lines 240–262) itself can fail: it calls `self.ledger.transfer_funds(...)` which propagates ledger errors, and it can fail if the donor neuron's cached stake is less than the transaction fee.

## Impact Explanation
Any transient or permanent failure of `transfer_gtc_neuron` (ledger unavailability, message-queue backpressure, ledger upgrade in progress, insufficient cached stake) causes the following irreversible state: the neuron ID is removed from `self.neuron_ids`, `has_forwarded` or `has_donated` is set to `true`, and the ICP staked in that neuron is permanently inaccessible — a ledger conservation violation. This constitutes permanent loss of NNS governance assets (staked ICP neurons) with no recovery path, matching the **High** impact class: unauthorized/permanent loss of governance assets and canister-controlled funds where exploitation requires a transient ledger error as a constraint.

## Likelihood Explanation
`forward_whitelisted_unclaimed_accounts` is callable by **any unprivileged ingress sender** after 188 days from IC genesis (line 27 of `lib.rs`; this time gate has long since passed on mainnet). No special role or key is required. A transient ledger error during the sweep — which can occur due to subnet load, message-queue backpressure, or a ledger upgrade in progress — is sufficient to trigger the permanent lock. The caller receives `Ok(())` and has no way to distinguish a fully successful sweep from a partially or fully failed one.

## Recommendation
`AccountState::transfer()` should not remove a neuron ID from `self.neuron_ids` on failure, and should return `Err` if any neuron transfer fails so that callers can avoid setting `has_forwarded`/`has_donated` to `true`. Concretely: (1) only call `self.neuron_ids.retain(...)` inside the `Ok(_)` arm of the match, and (2) after the loop, return `Err(...)` if `self.failed_transferred_neurons` is non-empty. Callers should only set `has_donated`/`has_forwarded` to `true` when `transfer()` returns `Ok(())` with an empty failure list, preserving retryability for partial failures.
