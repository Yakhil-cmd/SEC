### Title
Unprotected `new()` Initialization Allows Any Caller to Seize Engine Ownership — (File: `engine/src/contract_methods/admin.rs`)

---

### Summary
The Aurora Engine's `new` initialization function performs no caller authentication. Any NEAR account that calls it before the legitimate deployer can supply an arbitrary `owner_id`, permanently seizing privileged control of the engine. Because the function guards against re-entry only by checking whether state already exists, the first caller wins unconditionally.

---

### Finding Description
`new` is exported as a public WASM entry point with no access control:

```rust
// engine/src/lib.rs
#[unsafe(no_mangle)]
pub extern "C" fn new() {
    let io = Runtime;
    let env = Runtime;
    contract_methods::admin::new(io, &env)
        .map_err(ContractError::msg)
        .sdk_unwrap();
}
``` [1](#0-0) 

The implementation only checks whether state already exists; it never verifies `env.predecessor_account_id()`:

```rust
pub fn new<I: IO + Copy, E: Env>(mut io: I, env: &E) -> Result<(), ContractError> {
    if state::get_state(&io).is_ok() {
        return Err(b"ERR_ALREADY_INITIALIZED".into());
    }
    let input = io.read_input().to_vec();
    let args = NewCallArgs::deserialize(&input)...;
    ...
    state::set_state(&mut io, &state)?;
    Ok(())
}
``` [2](#0-1) 

The `owner_id` field is taken directly from the caller-supplied `NewCallArgs`. Once written, all subsequent calls to `new` are rejected with `ERR_ALREADY_INITIALIZED`, so the legitimate deployer cannot recover.

The workspace deployment helper confirms that contract deployment and initialization are issued as **two separate transactions**, creating an exploitable window:

```rust
let contract = account.deploy(&self.code...).await?;
// ← attacker window here ←
engine.new(self.chain_id, self.owner_id, self.upgrade_delay_blocks).transact().await?;
``` [3](#0-2) 

---

### Impact Explanation
`owner_id` is the single privileged role that gates every sensitive operation:

| Function | Effect |
|---|---|
| `upgrade` | Deploy arbitrary new contract code |
| `set_eth_connector_contract_account` | Redirect the ETH connector to a malicious contract |
| `pause_contract` / `resume_contract` | Freeze or unfreeze all user funds |
| `attach_full_access_key` | Add a full-access key to the engine account |
| `stage_upgrade` | Stage malicious code for deployment |
| `set_key_manager` | Control relayer key management | [4](#0-3) [5](#0-4) 

An attacker who wins the `new` race can redirect `set_eth_connector_contract_account` to a malicious contract that mints unbacked tokens or blocks withdrawals — direct theft and permanent freeze of all bridged ETH held by users. [6](#0-5) 

---

### Likelihood Explanation
The attack requires:
1. Observing the Aurora Engine deployment transaction on-chain (public information).
2. Submitting a `new` call with attacker-controlled `owner_id` before the deployer's initialization transaction lands.
3. Having that transaction ordered before the deployer's in the same or next block.

NEAR's public mempool and block-ordering model make this feasible for any attacker monitoring the chain. The workspace code demonstrates the two-transaction pattern is the default deployment path. No special privileges, leaked keys, or social engineering are required — only a race on a public network.

---

### Recommendation
Add a predecessor check to `new` so only the contract account itself (or a designated deployer) can initialize it:

```rust
pub fn new<I: IO + Copy, E: Env>(mut io: I, env: &E) -> Result<(), ContractError> {
    if state::get_state(&io).is_ok() {
        return Err(b"ERR_ALREADY_INITIALIZED".into());
    }
    // Add: only the contract account may initialize itself
    if env.predecessor_account_id() != env.current_account_id() {
        return Err(b"ERR_NOT_ALLOWED".into());
    }
    ...
}
```

Alternatively, bundle the `DeployContract` and `FunctionCall(new)` actions into a single atomic NEAR batch transaction so no external call can be interleaved.

---

### Proof of Concept

1. Attacker watches the NEAR blockchain for a `DeployContract` action targeting the `aurora` account.
2. In the same block (or the next), attacker submits:
   ```
   aurora.new({ chain_id: ..., owner_id: "attacker.near", upgrade_delay_blocks: 0 })
   ```
3. Attacker's transaction is ordered before the legitimate deployer's `new` call.
4. Deployer's `new` call reverts with `ERR_ALREADY_INITIALIZED`.
5. Attacker calls `set_eth_connector_contract_account` to point the engine at a malicious connector, draining all bridged ETH from users who subsequently bridge or withdraw. [7](#0-6) [1](#0-0)

pub fn new<I: IO + Copy, E: Env>(mut io: I, env: &E) -> Result<(), ContractError> {
    if state::get_state(&io).is_ok() {
        return Err(b"ERR_ALREADY_INITIALIZED".into());
    }

    let input = io.read_input().to_vec();
    let args = NewCallArgs::deserialize(&input).map_err(|_| errors::ERR_BORSH_DESERIALIZE)?;

    let initial_hashchain = args.initial_hashchain();
    let state: EngineState = args.into();

    if let Some(block_hashchain) = initial_hashchain {
        let block_height = env.block_height();
        let mut hashchain = Hashchain::new(
            state.chain_id,
            env.current_account_id(),
            block_height,
            block_hashchain,
        );

        hashchain.add_block_tx(
            block_height,
            function_name!(),
            &input,
            &[],
            &Bloom::default(),
        )?;
        crate::hashchain::save_hashchain(&mut io, &hashchain)?;
    }

    state::set_state(&mut io, &state)?;
    Ok(())
}

pub fn upgrade<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<(), ContractError> {
    let state = state::get_state(&io)?;
    require_running(&state)?;
    require_owner_only(&state, &env.predecessor_account_id())?;

    let input = io.read_input().to_vec();
    let (code, state_migration_gas) = match UpgradeParams::try_from_slice(&input) {
        Ok(args) => (
            args.code,
            args.state_migration_gas
                .map_or(GAS_FOR_STATE_MIGRATION, NearGas::new),
        ),
        Err(_) => (input, GAS_FOR_STATE_MIGRATION), // Backward compatibility
    };

    let target_account_id = env.current_account_id();
    let batch = PromiseBatchAction {
        target_account_id,
        actions: vec![
            PromiseAction::DeployContract { code },
            PromiseAction::FunctionCall {
                name: "state_migration".to_string(),
                args: vec![],
                attached_yocto: ZERO_YOCTO,
                gas: state_migration_gas,
            },
        ],
    };
    let promise_id = handler.promise_create_batch(&batch);

    handler.promise_return(promise_id);

    Ok(())
}
