Audit Report

## Title
Missing Neurons' Fund Membership Guard in `initiate_maturity_disbursement` Enables Double ICP Minting — (`rs/nns/governance/src/governance/disburse_maturity.rs`)

## Summary

`initiate_maturity_disbursement` reduces a Neurons' Fund neuron's `maturity_e8s_equivalent` and queues a delayed ICP mint without checking `is_a_neurons_fund_member()`. Because the same maturity is snapshotted for SNS swap participation, an attacker can cause the ICP Ledger to mint ICP twice from the same maturity: once to the SNS treasury at swap settlement, and once to the attacker's account when the queued disbursement finalizes 7 days later. The result is unbacked ICP inflation absorbed by the protocol.

## Finding Description

**Root cause — missing guard in `initiate_maturity_disbursement`:**

`initiate_maturity_disbursement` ( [1](#0-0) ) inspects the neuron for four conditions — spawning state, caller authorization, disbursement count, and minimum amount — but never calls `is_a_neurons_fund_member()`. [2](#0-1) 

By contrast, `merge_neurons` explicitly extracts `source_is_not_in_neurons_fund` and returns `MergeNeuronsError::SourceNeuronInNeuronsFund` when the flag is false. [3](#0-2) 

The predicate itself is straightforward: [4](#0-3) 

**Maturity disbursement stores the amount at initiation, not at finalization:**

`initiate_maturity_disbursement` immediately subtracts `disbursement_maturity_e8s` from `maturity_e8s_equivalent` and stores the original amount in `MaturityDisbursement.amount_e8s`. [5](#0-4) 

Seven days later, `try_finalize_maturity_disbursement` mints `amount_to_mint_e8s` (derived from the stored `amount_e8s`) directly to the ledger, with no re-check of the neuron's current maturity. [6](#0-5) 

**Silent failure in `apply_neurons_fund_snapshot`:**

When the SNS swap commits, `apply_neurons_fund_snapshot` with `NeuronsFundAction::DrawMaturity` attempts `checked_sub` on each neuron's current maturity. If the maturity was already reduced to 0 by the prior disbursement, `checked_sub` returns `None`, an error string is pushed to `neurons_fund_action_error`, and the loop continues. [7](#0-6) 

The ICP minting to the SNS treasury is computed from the snapshot amounts and is independent of whether this maturity draw succeeds. The error is returned from `apply_neurons_fund_snapshot` but does not reverse the already-executed ledger mint to the SNS treasury. [8](#0-7) 

**Exploit flow:**
1. Attacker opts a neuron into the Neurons' Fund (`joined_community_fund_timestamp_seconds > 0`).
2. A `CreateServiceNervousSystem` proposal executes; governance snapshots the neuron's maturity (e.g., 100 ICP).
3. Attacker calls `ManageNeuron::DisburseMaturity { percentage_to_disburse: 100 }`. All checks pass; `maturity_e8s_equivalent` is set to 0; a 100 ICP disbursement is queued for T+7 days.
4. The SNS swap commits. Governance mints 100 ICP to the SNS treasury from the snapshot. `apply_neurons_fund_snapshot(DrawMaturity)` attempts `0u64.checked_sub(100 ICP)` → underflow → error collected, neuron maturity stays 0, settlement proceeds.
5. T+7 days: `try_finalize_maturity_disbursement` mints 100 ICP to the attacker's account.

**Result:** 200 ICP minted from 100 ICP of maturity. The SNS treasury is unharmed; the protocol absorbs 100 ICP of unbacked inflation.

## Impact Explanation

This is illegal ICP minting — a direct violation of the ICP supply conservation invariant. The attacker receives ICP that was never backed by a corresponding maturity burn. The SNS treasury is not harmed, but the protocol absorbs the inflation. At scale (a Neurons' Fund neuron with large maturity, or multiple neurons), this constitutes significant unbacked ICP issuance. This matches the Critical/High allowed impact: *"Theft, permanent loss, illegal minting, or protocol insolvency involving exorbitant ICP/Cycles or in-scope chain-key/ledger assets"* and *"Unauthorized access to neurons, governance assets, wallets, identities, ledgers, or canister-controlled funds."* Severity is High at minimum; Critical if the attacker controls neurons with maturity exceeding the $1M threshold.

## Likelihood Explanation

All preconditions are achievable by any ICP holder without special privileges: opting into the Neurons' Fund is a standard neuron operation; `CreateServiceNervousSystem` proposals pass routinely on mainnet; the window between snapshot and swap settlement is days to weeks, giving ample time to call `disburse_maturity`. The `disburse_maturity` endpoint is publicly callable by any neuron controller. No governance majority, subnet majority, or leaked key is required.

## Recommendation

Add a Neurons' Fund membership guard to `initiate_maturity_disbursement`, mirroring the existing guard in `merge_neurons`:

```rust
// Inside the with_neuron closure in initiate_maturity_disbursement:
let is_neurons_fund_member = neuron.is_a_neurons_fund_member();
// ...
if is_neurons_fund_member {
    return Err(InitiateMaturityDisbursementError::NeuronIsNeuronsFundMember);
}
```

The same guard should be applied to `spawn_neuron` in `rs/nns/governance/src/governance.rs`, which also reduces `maturity_e8s_equivalent` without checking Neurons' Fund membership. [9](#0-8) 

## Proof of Concept

A deterministic integration test using PocketIC or the NNS integration test harness:

1. Create an NNS neuron N with `maturity_e8s_equivalent = 100_000_000` (100 ICP). Set `joined_community_fund_timestamp_seconds` to a nonzero value.
2. Execute a `CreateServiceNervousSystem` proposal. Assert that `list_active_neurons_fund_neurons` includes N with 100 ICP maturity.
3. Call `ManageNeuron::DisburseMaturity { percentage_to_disburse: 100, to_account: attacker_account }` on N. Assert success and that `maturity_e8s_equivalent` is now 0.
4. Simulate swap commitment: call `settle_neurons_fund_participation(Committed { ... })` with the snapshot from step 2. Assert that 100 ICP is minted to the SNS treasury.
5. Advance time by 8 days. Run the maturity disbursement finalization task. Assert that 100 ICP is minted to `attacker_account`.
6. **Invariant check**: total ICP minted = 200 ICP; total maturity burned = 100 ICP. Assert this violates conservation. [10](#0-9)
