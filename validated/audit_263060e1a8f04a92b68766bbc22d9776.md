### Title
Global Neuron-Creation Rate-Limiter Key Enables Governance DoS for All Principals - (File: `rs/nns/governance/src/governance.rs`)

---

### Summary

The NNS Governance canister rate-limits neuron-creation operations (`claim_neuron`, `split_neuron`, `disburse_to_neuron`, `create_neuron`) using a single, hardcoded global key `"ADD_NEURON"`. Because every caller shares the same capacity bucket, any sufficiently funded unprivileged principal can exhaust the global budget and prevent all other users from creating neurons for up to ~20 hours.

---

### Finding Description

`NEURON_RATE_LIMITER_KEY` is defined as the constant string `"ADD_NEURON"` and is passed verbatim as the `key` argument to every `rate_limiter.try_reserve(...)` call across all four neuron-creation paths. [1](#0-0) 

The `RateLimiter<K, S>` implementation stores one `CapacityUsageRecord` per key. When the same key is used for every caller, all callers draw from a single shared bucket. [2](#0-1) 

The bucket is configured with a spike ceiling of `MAX_NEURON_CREATION_SPIKE = 300` and a refill rate of `MAX_SUSTAINED_NEURONS_PER_HOUR = 15` (one slot every 240 seconds). [3](#0-2) 

All four creation paths use the identical key:

- `split_neuron` [4](#0-3) 
- `disburse_to_neuron` [5](#0-4) 
- `claim_neuron` [6](#0-5) 
- `create_neuron` [7](#0-6) 

This is structurally identical to the `flashProof` anti-pattern: a protection mechanism that uses a coarse-grained identifier (one global bucket) instead of a fine-grained one (per-caller/per-principal bucket), making the protection exploitable. The Registry canister's own rate limiters demonstrate the correct pattern — they key by `node_provider_id` and `node_operator_id` so each principal has an independent bucket. [8](#0-7) 

---

### Impact Explanation

An attacker who exhausts the 300-slot global spike budget causes every subsequent `claim_neuron`, `split_neuron`, `disburse_to_neuron`, or `create_neuron` call from **any** principal to fail with:

> "Reached maximum number of neurons that can be created in this hour. Please wait and try again later." [9](#0-8) 

Full recovery requires 300 × 240 s ≈ 20 hours of idle time. During this window, no user can stake a new neuron, split an existing one, or disburse to a new neuron — directly blocking governance participation (voting power acquisition) for the entire NNS.

---

### Likelihood Explanation

The attack is reachable by any unprivileged ingress sender. The cost is bounded by the minimum neuron stake multiplied by 300 (for `claim_neuron`) or by holding 300 existing neurons (for `split_neuron`). No privileged role, admin key, or subnet-majority corruption is required. The attacker recovers their ICP after the attack (neurons can be dissolved and disbursed), making the net cost only transaction fees and opportunity cost.

---

### Recommendation

Replace the single global key with a per-caller key, e.g. `format!("ADD_NEURON_{caller}")`, so each principal has an independent rate-limit bucket. This mirrors the pattern already used in the Registry canister and eliminates the cross-caller exhaustion vector while preserving per-principal abuse prevention.

---

### Proof of Concept

1. Attacker holds ≥ 300 ICP (or ≥ 300 existing neurons with sufficient stake to split).
2. Attacker sends 300 rapid `ManageNeuron::ClaimOrRefresh` (or `Split`) ingress messages to the Governance canister.
3. Each call succeeds and commits one unit of the global `"ADD_NEURON"` capacity bucket.
4. After 300 commits the bucket is exhausted (`capacity_used == max_capacity`).
5. Any subsequent `claim_neuron`, `split_neuron`, `disburse_to_neuron`, or `create_neuron` call from **any** principal returns `ErrorType::Unavailable` — confirmed by the existing test: [10](#0-9) 

6. The global budget refills at 1 slot per 240 seconds; full recovery takes ≈ 20 hours.

```rust
    pub fn try_reserve(
        &mut self,
        now: SystemTime,
        key: K,
        requested_capacity: u64,
    ) -> Result<Reservation<K>, RateLimiterError> {
        // validate reservation for actual capacity.
        if requested_capacity < 1 {
            return Err(RateLimiterError::InvalidArguments(
                "To make a rate-limit reservation, requested_capacity must be at least 1"
                    .to_string(),
            ));
        }

        let reservations = self.reservations.lock().unwrap();
        // validate that system can handle more reservations
        let used_reservations: u64 = reservations
            .len()
            .try_into()
            .expect("usize should always safely convert to u64");

        if used_reservations >= self.config.max_reservations {
            return Err(RateLimiterError::MaxReservationsReached);
        }

        // Drop this borrow so we can borrow mutably to get usage
        drop(reservations);

        if requested_capacity <= self.get_available_capacity(key.clone(), now) {
            let mut reservations = self.reservations.lock().unwrap();
            // Only allocate global index on successful reservation
            let index = self.next_index;
            self.next_index += 1;

            let reservation_data = ReservationData {
                now,
                capacity: requested_capacity,
            };
            reservations.insert((key.clone(), index), reservation_data);

            let reservation = Reservation {
                key: key.clone(),
                index,
                reservations_map: Arc::downgrade(&self.reservations),
            };

            Ok(reservation)
        } else {
            Err(RateLimiterError::NotEnoughCapacity)
        }
```
