### Title
Zero-Price Bypass of Silo Fixed-Gas Charging Allows Free Transaction Execution — (`engine/src/engine.rs`)

---

### Summary

In silo mode, when `fixed_gas` is `Some(F)` and `block_base_fee_per_gas` is zero (Aurora's default), an attacker can submit a transaction with `max_fee_per_gas = 0`. The early-return guard in `charge_gas` is not taken (because `fixed_gas.is_some()`), but the subsequent arithmetic still produces `effective_gas_price = 0` and `prepaid_amount = 0`. The sender pays nothing, and `refund_unused_gas` immediately returns without crediting the relayer. The silo operator's `fixed_gas` invariant is silently nullified.

---

### Finding Description

`charge_gas` contains an early-return shortcut designed for the zero-fee case:

```rust
if transaction.max_fee_per_gas.is_zero()
    && fixed_gas.is_none()
    && block_base_fee_per_gas.is_zero()
{
    return Ok(GasPaymentResult::default());
}
``` [1](#0-0) 

The guard correctly skips gas charging when there is no gas price and no fixed gas. However, when `fixed_gas = Some(F)`, the condition is `false` and execution continues — but the downstream arithmetic still collapses to zero:

```rust
let priority_fee_per_gas = transaction
    .max_priority_fee_per_gas                          // 0
    .min(transaction.max_fee_per_gas - block_base_fee_per_gas); // min(0, 0-0) = 0
let effective_gas_price = priority_fee_per_gas + block_base_fee_per_gas; // 0 + 0 = 0

let prepaid_amount = fixed_gas
    .map_or(transaction.gas_limit, EthGas::as_u256)   // F
    .checked_mul(effective_gas_price)                  // F * 0 = 0
    .map(Wei::new)
    .ok_or(GasPaymentError::EthAmountOverflow)?;
``` [2](#0-1) 

`prepaid_amount = 0` means the sender's balance is not reduced. Then in `refund_unused_gas`:

```rust
if gas_result.effective_gas_price.is_zero() {
    return Ok(());
}
``` [3](#0-2) 

The relayer reward path is never reached. The relayer receives nothing.

The only upstream guard that could block this is the base-fee check:

```rust
if transaction.max_fee_per_gas < block_base_fee_per_gas {
    return Err(GasPaymentError::MaxFeePerGasLessThanBaseFee);
}
``` [4](#0-3) 

When `block_base_fee_per_gas = 0` (Aurora's standard deployment), `0 < 0` is false, so no error is raised. There is no additional check that enforces `effective_gas_price > 0` when `fixed_gas` is set.

---

### Impact Explanation

The silo operator deploys with `fixed_gas = Some(F)` specifically to guarantee that every transaction pays `F * effective_gas_price` to the relayer. With this bypass, any user who sets `max_fee_per_gas = 0` executes transactions at zero cost. The relayer's expected yield — the entire economic model of the silo — is stolen for every such transaction. This matches **High: Theft of unclaimed yield**.

---

### Likelihood Explanation

Aurora's `block_base_fee_per_gas` is zero by default (Aurora does not implement EIP-1559 base-fee burning). Any user who knows the silo is in fixed-gas mode can trivially craft a legacy or EIP-1559 transaction with `gas_price = 0` / `max_fee_per_gas = 0`. No special privilege is required — this is a standard EVM transaction field. The attacker gains free execution; the relayer loses all yield from those transactions.

---

### Recommendation

After computing `effective_gas_price`, add an explicit guard when `fixed_gas` is set:

```rust
if fixed_gas.is_some() && effective_gas_price.is_zero() {
    return Err(GasPaymentError::MaxFeePerGasLessThanBaseFee);
}
```

This enforces the invariant that silo transactions must always carry a non-zero gas price, consistent with the operator's intent when configuring `fixed_gas`. [5](#0-4) 

---

### Proof of Concept

The following mirrors the structure of the existing unit tests in `engine/src/engine.rs`:

```rust
#[test]
fn test_silo_fixed_gas_zero_price_bypass() {
    let origin = Address::zero();
    let current_account_id = AccountId::default();
    let env = Fixed::default(); // block_base_fee_per_gas = 0
    let storage = RefCell::new(Storage::default());
    let mut io = StoragePointer(&storage);

    // Give sender a non-zero balance
    add_balance(&mut io, &origin, Wei::new_u64(1_000_000_000)).unwrap();

    let mut engine: Engine<_, _> =
        Engine::new_with_state(EngineState::default(), origin, current_account_id, io, &env);

    let transaction = NormalizedEthTransaction {
        address: Address::default(),
        chain_id: None,
        nonce: U256::default(),
        gas_limit: 2_000_000.into(),
        max_priority_fee_per_gas: U256::zero(), // attacker sets 0
        max_fee_per_gas: U256::zero(),           // attacker sets 0
        to: None,
        value: Wei::default(),
        data: vec![],
        access_list: vec![],
        authorization_list: vec![],
    };

    let fixed_gas = Some(EthGas::new(1_000_000)); // silo operator set fixed gas

    let result = engine
        .charge_gas(&origin, &transaction, None, fixed_gas)
        .unwrap();

    // Both are zero — sender paid nothing, relayer will get nothing
    assert_eq!(result.prepaid_amount, Wei::zero());
    assert_eq!(result.effective_gas_price, U256::zero());
}

    pub fn charge_gas(
        &mut self,
        sender: &Address,
        transaction: &NormalizedEthTransaction,
        max_gas_price: Option<U256>,
        fixed_gas: Option<EthGas>,
    ) -> Result<GasPaymentResult, GasPaymentError> {
        let block_base_fee_per_gas = self.block_base_fee_per_gas();
        if transaction.max_fee_per_gas.is_zero()
            && fixed_gas.is_none()
            && block_base_fee_per_gas.is_zero()
        {
            return Ok(GasPaymentResult::default());
        }

        if transaction.max_fee_per_gas < block_base_fee_per_gas {
            return Err(GasPaymentError::MaxFeePerGasLessThanBaseFee);
        }

        let priority_fee_per_gas = transaction
            .max_priority_fee_per_gas
            .min(transaction.max_fee_per_gas - block_base_fee_per_gas);
        let priority_fee_per_gas = max_gas_price.map_or(priority_fee_per_gas, |price| {
            price.min(priority_fee_per_gas)
        });
        let effective_gas_price = priority_fee_per_gas + block_base_fee_per_gas;
        // First, we try to use `fixed_gas`. At this point we already know that the `fixed_gas` is
        // less than the `gas_limit`. It allows avoiding refunding unused gas to the sender later.
        let prepaid_amount = fixed_gas
            .map_or(transaction.gas_limit, EthGas::as_u256)
            .checked_mul(effective_gas_price)
            .map(Wei::new)
            .ok_or(GasPaymentError::EthAmountOverflow)?;

        let new_balance = get_balance(&self.io, sender)
            .checked_sub(prepaid_amount)
            .ok_or(GasPaymentError::OutOfFund)?;

        set_balance(&mut self.io, sender, &new_balance);

        self.gas_price = effective_gas_price;

        Ok(GasPaymentResult {
            prepaid_amount,
            effective_gas_price,
            priority_fee_per_gas,
        })
    }
