## Title
SNS Governance Voting Rewards Permanently Lost When Proposals Settle With Zero Total Voting Power - (File: `rs/sns/governance/src/governance.rs`)

## Summary

In `distribute_rewards` (`rs/sns/governance/src/governance.rs`), when one or more proposals reach `ReadyToSettle` but every ballot carries `Vote::Unspecified`, `total_reward_shares` is zero, no maturity is distributed, yet the proposals are still written into `settled_proposals` of the new `RewardEvent`. Because `rewards_rolled_over()` gates on `settled_proposals.is_empty()`, the entire `rewards_purse_e8s` ã including any previously accumulated rollover ã is silently discarded and never recoverable.

## Finding Description

**Rewards purse construction** (`governance.rs` L5854ã5875): `rewards_purse_e8s` is seeded with `self.latest_reward_event().e8s_equivalent_to_be_rolled_over()` plus newly minted rewards for every elapsed round.

**Voting-share tally** (`governance.rs` L5892ã5934): Only `Yes`/`No` votes pass `eligible_for_rewards()`; `Unspecified` ballots hit `continue`. If every neuron abstains, `neuron_id_to_reward_shares` is empty and `total_reward_shares == dec!(0)`.

**Zero-share branch** (`governance.rs` L5946ã5952): The function logs a warning and falls through without adding any maturity; `distributed_e8s_equivalent` remains `0`.

**Proposals settled unconditionally** (`governance.rs` L6013ã6081): Regardless of the zero-share branch, the loop over `considered_proposals` runs: each proposal gets `reward_event_end_timestamp_seconds` and `reward_event_round` stamped, and `p.ballots.clear()` is called.

**RewardEvent written with non-empty `settled_proposals`** (`governance.rs` L6083ã6092): The new event records `settled_proposals: considered_proposals` (non-empty), `distributed_e8s_equivalent: 0`, and `total_available_e8s_equivalent: Some(rewards_purse_e8s)`.

**Rollover gate** (`types.rs` L2054ã2067): `e8s_equivalent_to_be_rolled_over()` returns `total_available_e8s_equivalent` only when `rewards_rolled_over()` is `true`. `rewards_rolled_over()` is defined as `self.settled_proposals.is_empty()`. Because `settled_proposals` is non-empty, it returns `false`, so `e8s_equivalent_to_be_rolled_over()` returns `0`.

**Next round**: `rewards_purse_e8s` is seeded from `0`, and the entire prior purse ã including all accumulated rollover ã is gone with no recovery path.

## Impact Explanation

SNS governance is explicitly in scope. The lost amount equals the full `rewards_purse_e8s` at the time of the triggering round, which compounds with every prior rollover round. Maturity is the yield mechanism for SNS token holders; permanent destruction of the maturity purse is a direct, irreversible financial loss for all SNS participants. This matches the allowed High impact: *"Significant SNS... security impact with concrete user or protocol harm."*

## Likelihood Explanation

The trigger requires only that at least one proposal reaches `ReadyToSettle` in a round where no neuron casts `Yes` or `No`. This occurs naturally in early-stage SNS deployments with low neuron participation (neurons following a lead neuron that never votes, or simply inactive). No privileged access is required: any governance participant can submit a proposal via the public `manage_neuron` endpoint, and `distribute_rewards` is called automatically by `run_periodic_tasks`. The condition is repeatable ã each such round destroys that round's purse independently.

## Recommendation

When `total_reward_shares == dec!(0)` but `considered_proposals` is non-empty, the rewards purse must be carried forward rather than discarded. Two equivalent fixes:

1. **Broaden the rollover predicate**: Change `rewards_rolled_over()` in `rs/sns/governance/src/types.rs` to also return `true` when `distributed_e8s_equivalent == 0`, regardless of `settled_proposals`.

2. **Treat the event as a pure rollover**: When `total_reward_shares == dec!(0)`, write the `RewardEvent` with `settled_proposals: vec![]` (keeping proposals logically settled via their stamped fields) so the existing `settled_proposals.is_empty()` gate passes and the purse rolls over automatically.

Either approach must be applied symmetrically to the NNS governance rollover logic in `rs/nns/governance/src/reward/calculation.rs`.

## Proof of Concept

1. Deploy an SNS with `voting_rewards_parameters` configured.
2. Create one proposal via `manage_neuron`; let it reach `ReadyToSettle` (voting window expires) with all neurons holding `Vote::Unspecified`.
3. Advance the clock past one reward round boundary so `should_distribute_rewards()` returns `true`.
4. Call `distribute_rewards` (triggered by `run_periodic_tasks`).
5. Assert: `latest_reward_event().distributed_e8s_equivalent == 0` and `latest_reward_event().settled_proposals` is non-empty.
6. Advance the clock past a second reward round boundary; call `distribute_rewards` again with a neuron that votes `Yes`.
7. Assert: the second round's `rewards_purse_e8s` does **not** include the first round's `total_available_e8s_equivalent` ã confirming permanent loss.
