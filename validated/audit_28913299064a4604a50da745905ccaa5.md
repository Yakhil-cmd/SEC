Audit Report

## Title
Silent Discard of Neurons' Fund Maturity Refund Failure Causes Permanent Maturity Loss When Neuron Is Disbursed During Active SNS Swap - (File: `rs/nns/governance/src/governance.rs`)

## Summary
When a `CreateServiceNervousSystem` proposal executes, maturity is drawn from Neurons' Fund neurons and reserved for the SNS swap. If a participating neuron is disbursed during the swap window, the subsequent `refund_maturity_to_neurons_fund` call fails silently because the neuron no longer exists in the store. The error is explicitly discarded via `let _ = ...` at governance.rs L7351–7359, and the reserved-but-unused maturity is permanently destroyed with no compensation to the neuron owner.

## Finding Description
**Step 1 – Maturity draw:** `draw_maturity_from_neurons_fund` at governance.rs L7416–7417 reduces `maturity_e8s_equivalent` on each participating NF neuron and propagates errors with `?`, so this step cannot silently fail.

**Step 2 – Neuron disbursement:** SNS swaps run for days to weeks. During this window, an NF neuron owner can call `disburse`, which removes the neuron from the stable neuron store. No guard checks whether the neuron has reserved maturity in an active swap. The neuron's maturity was already reduced in Step 1, so the owner receives their stake but the drawn maturity is now unattached to any live neuron.

**Step 3 – Silent refund failure:** At swap settlement, `settle_neurons_fund_participation` computes the refund and calls `refund_maturity_to_neurons_fund`. This calls `apply_neurons_fund_snapshot` (neurons_fund.rs L1914–1945), which iterates the snapshot and calls `neuron_store.modify_neuron_maturity` for each neuron. `modify_neuron_maturity` (neuron_store.rs L818–833) calls `with_main_part_mut` on the stable neuron store; if the neuron was disbursed, it no longer exists and a `NeuronStoreError` is returned. The error is collected into `neurons_fund_action_error` and returned as `Err(String)`. Back in `settle_neurons_fund_participation`, this error is explicitly discarded:

```rust
// governance.rs L7348–7359
let _ = neuron_store
    .refund_maturity_to_neurons_fund(&refund)
    .map_err(|err| {
        println!(
            "{}ERROR while trying to refund Neurons' Fund: {}. \
            Total refund amount: {} ICP e8s.",
            LOG_PREFIX, err, total_refund_amount_icp_e8s,
        );
    });
```

The developer comment confirms this is intentional: *"we opt for providing data to the SNS Swap canister … merely logging the error for human inspection."* The maturity is never returned to any account or neuron.

## Impact Explanation
This is a ledger conservation bug. Maturity drawn from a Neurons' Fund neuron that is subsequently disbursed before swap settlement is permanently destroyed. Maturity has direct monetary value — it can be spawned into ICP tokens. The loss equals the difference between the reserved (maximum) participation amount and the actual participation amount for the affected neuron. For large NF neurons this can be substantial. The lost maturity is not credited anywhere; it simply disappears from the system's accounting. This matches the allowed High impact: *"Significant NNS, SNS… security impact with concrete user or protocol harm."*

## Likelihood Explanation
- SNS swaps run for days to weeks, providing a wide disbursement window.
- Disbursing a neuron is a standard, unprivileged ingress call available to any neuron controller.
- No protocol-level lock prevents disbursement while NF maturity is reserved.
- The scenario requires no privileged access, social engineering, or threshold corruption.
- The neuron owner may be unaware that disbursing during a swap forfeits their reserved maturity, making accidental triggering plausible.

## Recommendation
1. **Propagate the refund error** instead of discarding it. If `refund_maturity_to_neurons_fund` fails, surface the failure so a governance hot-fix can restore the maturity before it is permanently lost.
2. **Guard disbursement while NF maturity is reserved.** Before allowing a neuron to be fully disbursed, check whether it appears in any active `initial_neurons_fund_participation` snapshot and reject the disbursement (or require NF participation to be settled first).
3. **Alternatively**, record the failed refund amount in a dedicated recovery ledger so that a subsequent NNS proposal can manually restore the maturity to the neuron owner's account.

## Proof of Concept
1. NF neuron N has `maturity_e8s_equivalent = 1_000_000_000`.
2. A `CreateServiceNervousSystem` proposal executes; `draw_maturity_from_neurons_fund` reduces N's maturity to `800_000_000` (reserving `200_000_000` e8s).
3. The SNS swap opens. During the swap window, N's controller calls `disburse` on N. N is removed from the neuron store.
4. The swap commits with only `50_000_000` e8s of actual NF participation from N. The refund snapshot contains `150_000_000` e8s owed back to N.
5. `settle_neurons_fund_participation` calls `refund_maturity_to_neurons_fund`. `modify_neuron_maturity` returns `NeuronStoreError` (neuron not found). The error is logged and discarded via `let _ = ...` at governance.rs L7351.
6. `150_000_000` e8s of maturity is permanently destroyed. The neuron owner receives no compensation.

A deterministic integration test using PocketIC can reproduce this by: (a) creating an NF neuron, (b) executing a CSNS proposal to draw maturity, (c) disbursing the neuron, (d) calling `settle_neurons_fund_participation` with a partial participation result, and (e) asserting that total system maturity has decreased by the refund amount.
