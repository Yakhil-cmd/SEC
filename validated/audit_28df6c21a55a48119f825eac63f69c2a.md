### Title
Owner Can Permanently Renounce Ownership via Empty `AccountId` Bypass in `set_owner`, Permanently Freezing Paused-Contract Funds — (`engine/src/contract_methods/admin.rs`, `engine-types/src/account_id.rs`)

---

### Summary

The `BorshDeserialize` implementation for `AccountId` contains a backward-compatibility escape hatch that silently accepts an empty string and returns `AccountId::default()` (an empty `AccountId`) without triggering validation. The `set_owner` function does not guard against an empty `new_owner`. As a result, the current owner can call `set_owner` with a Borsh-encoded empty string, setting `owner_id` to an empty `AccountId` that no valid NEAR account can ever match. This permanently locks every function guarded by `require_owner_only`, including `resume_contract`. If the contract is paused at the time of (or after) this call, all user funds are permanently frozen with no recovery path.

---

### Finding Description

**Root cause — `engine-types/src/account_id.rs` lines 89–101:**

`BorshDeserialize` for `AccountId` has a special branch:

```rust
if account.is_empty() {
    return Ok(Self::default());   // bypasses validate()
}
```

`AccountId::default()` produces an empty `Box<str>`, which is explicitly rejected by `AccountId::validate()` (`TooShort`, minimum length = 2). The bypass exists for "backward compatibility" but is reachable at runtime through any Borsh-deserialized `AccountId` field.

**Propagation — `engine/src/contract_methods/admin.rs` lines 103–121:**

```rust
pub fn set_owner<I: IO + Copy, E: Env>(io: I, env: &E) -> Result<(), ContractError> {
    with_hashchain(io, env, function_name!(), |mut io| {
        let mut state = state::get_state(&io)?;
        require_running(&state)?;
        require_owner_only(&state, &env.predecessor_account_id())?;

        let args: SetOwnerArgs = io.read_input_borsh()?;   // ← empty string accepted here
        if state.owner_id == args.new_owner {
            return Err(errors::ERR_SAME_OWNER.into());     // passes if current owner ≠ ""
        }

        state.owner_id = args.new_owner;                   // ← owner_id set to ""
        state::set_state(&mut io, &state)?;
        Ok(())
    })
}
```

`SetOwnerArgs { new_owner: AccountId }` is Borsh-deserialized from the call input. If the caller supplies a 4-byte zero-length Borsh string (`[0x00, 0x00, 0x00, 0x00]`), `new_owner` becomes `AccountId::default()` (empty). The same-owner guard passes (current owner is non-empty), and `owner_id` is written as empty to persistent state.

**Lock-out — `engine/src/contract_methods/mod.rs` lines 79–87:**

```rust
pub fn require_owner_only(
    state: &state::EngineState,
    predecessor_account_id: &AccountId,
) -> Result<(), ContractError> {
    if &state.owner_id != predecessor_account_id {
        return Err(errors::ERR_NOT_ALLOWED.into());
    }
    Ok(())
}
```

With `owner_id = ""`, no NEAR transaction can ever supply a matching `predecessor_account_id` (NEAR enforces minimum 2-character account IDs at the protocol level). Every subsequent call to `require_owner_only` returns `ERR_NOT_ALLOWED` unconditionally.

**Functions permanently locked:**
- `set_owner` — cannot restore a valid owner
- `upgrade` / `stage_upgrade` — no contract upgrades possible
- `pause_contract` / `resume_contract` — **critical**: a paused contract can never be resumed
- `resume_precompiles`
- `set_eth_connector_contract_account`
- `mirror_erc20_token`

There is no recovery path: `new()` rejects re-initialization (`ERR_ALREADY_INITIALIZED`), and `upgrade` is itself owner-gated.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

If the engine contract is paused (via `pause_contract`) at any point before or after ownership is renounced, `resume_contract` is permanently inaccessible. All user ETH and ERC-20 token balances held in the Aurora EVM are frozen with no mechanism for recovery. The `is_paused` flag in `EngineState` can never be cleared, blocking every `submit`/`call` path that checks `require_running`.

Even without a prior pause, the permanent loss of `upgrade` capability means any future critical vulnerability in the engine cannot be patched, creating a latent path to fund loss.

---

### Likelihood Explanation

**Low-Medium.** The action requires the current owner to call `set_owner` with a Borsh-encoded empty string. This can occur:

1. **Accidentally** — an operator scripting a `set_owner` call passes an empty string for `new_owner` (e.g., a misconfigured deployment script or CLI tool).
2. **Intentionally** — the owner deliberately renounces control, analogous to calling `renounceOwnership()` in Solidity.

The backward-compatibility escape hatch in `BorshDeserialize` makes this reachable without any special knowledge; the caller only needs to supply four zero bytes as the Borsh-encoded string length.

---

### Recommendation

1. **Remove or restrict the empty-string bypass** in `BorshDeserialize for AccountId`. If backward compatibility is required for reading old state, scope the bypass to state deserialization only, not to user-supplied input deserialization.

2. **Add an explicit non-empty check in `set_owner`**:
   ```rust
   if args.new_owner.as_bytes().is_empty() {
       return Err(b"ERR_INVALID_OWNER".into());
   }
   ```

3. **Consider a two-step ownership transfer** (propose + accept) so that ownership cannot be transferred to an unreachable account in a single transaction.

### Proof of Concept

The owner constructs a NEAR function call to `set_owner` on the Aurora Engine contract with the following Borsh-encoded input (representing `SetOwnerArgs { new_owner: "" }`):

```
Input bytes: 00 00 00 00
             ^^^^^^^^^^^
             Borsh u32 length = 0 → empty string
```

Step-by-step:

1. Current owner calls `set_owner` with input `[0x00, 0x00, 0x00, 0x00]`.
2. `io.read_input_borsh::<SetOwnerArgs>()` deserializes `new_owner` via `BorshDeserialize for AccountId`.
3. The empty-string branch fires: `return Ok(Self::default())` — `new_owner = AccountId("")`.
4. `state.owner_id == args.new_owner` → `"aurora" != ""` → guard passes.
5. `state.owner_id` is written as `""` to storage.
6. Any subsequent call to `pause_contract`, `resume_contract`, `upgrade`, or `set_owner` hits `require_owner_only`, which compares `"" != <any valid NEAR account>` and returns `ERR_NOT_ALLOWED`.
7. If the contract is now paused (or was already paused), `resume_contract` is permanently inaccessible → all user funds are permanently frozen. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4)

impl BorshDeserialize for AccountId {
    fn deserialize_reader<R: io::Read>(reader: &mut R) -> io::Result<Self> {
        let account: String = BorshDeserialize::deserialize_reader(reader)?;

        // It's for saving backward compatibility.
        if account.is_empty() {
            return Ok(Self::default());
        }

        Self::try_from(account)
            .map_err(|e| io::Error::new(io::ErrorKind::InvalidData, e.to_string()))
    }
}
