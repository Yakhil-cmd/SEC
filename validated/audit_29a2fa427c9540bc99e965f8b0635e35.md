Audit Report

## Title
Stale 5% Worst-Case Modulation Constant in `spawn_neuron` Pre-Check Allows Spawning Neurons Below Minimum Stake After Mission 70 Expanded Negative Bound to −10% — (File: `rs/nns/governance/src/governance.rs`)

## Summary
The `spawn_neuron` pre-flight check uses a hardcoded `0.05` (5%) worst-case maturity modulation factor to validate that a spawned neuron will meet `neuron_minimum_stake_e8s`. Mission 70 expanded the negative modulation bound to −10% (`MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 = -1_000`), and the CHANGELOG confirms that neuron spawning now reads this Mission 70 value instead of the old CMC-polled value. The pre-check was never updated, so any neuron holder can spawn a neuron whose final ICP stake, after the actual −10% modulation is applied, falls below the governance-enforced minimum stake.

## Finding Description
In `rs/nns/governance/src/governance.rs` at line 2666, the pre-check computes:

```rust
let least_possible_stake = (maturity_to_spawn as f64 * (1_f64 - 0.05)) as u64;
```

This uses a 5% worst-case factor. The check passes if `least_possible_stake >= neuron_minimum_stake_e8s`.

In `rs/nns/governance/src/timer_tasks/update_icp_xdr_rate_related_data.rs` at lines 47–50, the Mission 70 bounds are defined:

```rust
pub(crate) const MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70: i64 = -1_000; // -10%
pub(crate) const MATURITY_MODULATION_MAX_PERMYRIAD_MISSION_70: i64 = 200;    // +2%
```

In `maybe_spawn_neurons` (governance.rs lines 6427–6435), the actual modulation applied is read from `self.heap_data.maturity_modulation.current_value_permyriad` — the Mission 70 value — and the sanity check at line 6438 explicitly allows values down to `MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70` (−1000 permyriad = −10%). The CHANGELOG (lines 20–22) confirms this switchover is live in production.

The `apply_maturity_modulation` call at line 6484 then applies the actual modulation. With `maturity_modulation = -1000`, the multiplier is `(10_000 − 1_000) / 10_000 = 0.90`.

**Invariant violation**: For any `maturity_to_spawn` in the range `[neuron_minimum_stake_e8s / 0.95, neuron_minimum_stake_e8s / 0.90)`:
- Pre-check: `maturity × 0.95 ≥ neuron_minimum_stake_e8s` → **passes**
- Actual stake: `maturity × 0.90 < neuron_minimum_stake_e8s` → **below minimum**

With `neuron_minimum_stake_e8s = 100_000_000` (1 ICP), the minimum maturity to pass the 5% check is ≈105,263,158 e8s, but after −10% modulation the resulting stake is ≈94,736,842 e8s — about 5,263,158 e8s short of the minimum.

The parent neuron's maturity is zeroed out and the child neuron's `cached_neuron_stake_e8s` is set to the below-minimum value (lines 6515–6517) before the ledger mint, with no rollback path.

## Impact Explanation
This is a concrete NNS governance impact: any unprivileged neuron holder can trigger the creation of a spawned neuron whose ICP stake is below `neuron_minimum_stake_e8s`, violating the invariant the pre-check explicitly guarantees. The parent neuron's maturity is permanently consumed. The spawned neuron may be unable to participate in governance (voting, proposing) if the protocol enforces minimum stake at those call sites. The error message ("There isn't enough maturity to spawn a new neuron due to worst case maturity modulation") explicitly promises this outcome is prevented, but the guarantee is false. This constitutes significant NNS impact with concrete user and protocol harm, qualifying as **High** severity.

## Likelihood Explanation
Mission 70 is confirmed live in production (CHANGELOG proposal 141779, 2026-05-17). The modulation formula can reach −10% whenever the 7-day ICP price is sufficiently below the 365-day average. The exploit requires no privileged role — any neuron holder can call `manage_neuron { Spawn { ... } }` with a maturity value in the vulnerable range. The vulnerable window is narrow (between the 5% and 10% thresholds) but deterministically reachable when modulation is near its minimum.

## Recommendation
Replace the undocumented literal `0.05` with a constant derived from `MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70`:

```rust
// Worst-case modulation under Mission 70 is -10% = -1000 permyriad.
const WORST_CASE_MODULATION_FRACTION: f64 =
    (-MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 as f64) / 10_000.0; // 0.10

let least_possible_stake =
    (maturity_to_spawn as f64 * (1.0 - WORST_CASE_MODULATION_FRACTION)) as u64;
```

Add a unit test asserting that `least_possible_stake` computed with the worst-case constant, when passed through `apply_maturity_modulation` at `MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70`, always yields a value ≥ `neuron_minimum_stake_e8s`.

## Proof of Concept
1. Hold a neuron with `maturity_e8s_equivalent = M` where `neuron_minimum_stake_e8s / 0.95 ≤ M < neuron_minimum_stake_e8s / 0.90` (e.g., `M = 105_263_158` with minimum stake = 1 ICP).
2. Call `manage_neuron { command: Spawn { percentage_to_spawn: 100, ... } }`.
3. Pre-check at line 2666: `105_263_158 × 0.95 = 100_000_000 ≥ neuron_minimum_stake_e8s` → passes.
4. Child neuron is created with `maturity_e8s_equivalent = 105_263_158`.
5. When `maybe_spawn_neurons` fires with `maturity_modulation = -1000` (Mission 70 minimum), `apply_maturity_modulation(105_263_158, -1000)` returns `105_263_158 × 9_000 / 10_000 = 94_736_842`.
6. `cached_neuron_stake_e8s` is set to `94_736_842 < 100_000_000 = neuron_minimum_stake_e8s`.
7. Parent maturity is zeroed; spawned neuron has below-minimum stake. Invariant violated.

A deterministic unit test can be written against `spawn_neuron` + `maybe_spawn_neurons` in the existing governance test harness, setting `maturity_modulation` to `MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70` and asserting the resulting `cached_neuron_stake_e8s` is below `neuron_minimum_stake_e8s` with the current code.
