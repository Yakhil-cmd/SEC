Audit Report

## Title
SNS Neuron Stake Permanently Locked When All Permissions Are Removed - (File: `rs/sns/governance/src/governance.rs`)

## Summary
The `remove_neuron_permissions` function in the SNS Governance canister imposes no guard preventing the removal of the last permission entry from a neuron. Once `neuron.permissions` is empty, every sensitive operation — including `Disburse` — fails the `check_authorized` call unconditionally, permanently trapping staked SNS tokens in the neuron's ledger subaccount with no recovery path. The code itself documents this danger but provides no enforcement.

## Finding Description
`remove_neuron_permissions` (governance.rs L4659–4716) validates only that the caller holds `ManagePrincipals` and that the listed permission types exist on the target principal; it performs no post-removal check on the resulting state of `neuron.permissions`. [1](#0-0) 

The underlying `remove_permissions_for_principal` (neuron.rs L784–786) calls `swap_remove` on the last `NeuronPermission` entry when all permission types for a principal are cleared, leaving `neuron.permissions` as an empty `Vec`. [2](#0-1) 

`disburse_neuron` (governance.rs L1127) calls `neuron.check_authorized(caller, NeuronPermissionType::Disburse)` before any other logic. [3](#0-2) 

`is_authorized` (neuron.rs L130–139) iterates `self.permissions` looking for the caller's principal. With an empty list the loop finds nothing and returns `false`, causing `check_authorized` to return `NotAuthorized`. [4](#0-3) 

The integration test `test_neuron_remove_all_permissions_of_self` (integration_tests/src/neuron.rs L2213–2264) explicitly exercises this path and asserts `neuron.permissions.len() == 0`, confirming the behavior is reachable in a real SNS deployment. [5](#0-4) 

The proto comment for `RemoveNeuronPermissions` acknowledges the danger without enforcing a guard. [6](#0-5) 

**Malicious co-principal attack path (primary concern):**
1. Alice stakes SNS tokens; her neuron is created with `ManagePrincipals` (and optionally `Disburse`) via `neuron_claimer_permissions`.
2. Alice grants `ManagePrincipals` to Bob (e.g., a hot key or multisig).
3. Bob calls `manage_neuron → RemoveNeuronPermissions` targeting Alice's principal with all permission types. `check_principal_authorized_to_change_permissions` passes because Bob holds `ManagePrincipals`.
4. `remove_permissions_for_principal` removes Alice's `NeuronPermission` entry; `neuron.permissions` now contains only Bob's entry.
5. Bob repeats the call targeting his own principal. `neuron.permissions` is now empty.
6. Alice's neuron dissolves. She calls `Disburse`; `is_authorized` returns `false`; the call returns `NotAuthorized`. Tokens are permanently locked.

## Impact Explanation
Staked SNS governance tokens held in a neuron's ledger subaccount become permanently inaccessible. The neuron record persists in governance state but no principal can invoke `Disburse`, `DisburseMaturity`, `Split`, `Configure`, or any other neuron management command. The tokens are effectively burned without a burn event. This constitutes a **permanent, irreversible loss of user funds** in an in-scope SNS canister, matching the allowed High impact: *"Unauthorized access to neurons, governance assets, wallets, identities, ledgers, or canister-controlled funds where exploitation requires meaningful per-target work or other constraints."*

## Likelihood Explanation
The malicious co-principal path requires only that the victim previously granted `ManagePrincipals` to the attacker — a common pattern for hot keys and multisig setups in SNS deployments. Once that precondition is met, the attack is two `manage_neuron` calls with no cost, no timing constraint, and no reversibility. The `DEFAULT_NEURON_CLAIMER_PERMISSIONS` grants `ManagePrincipals` by default, and many SNS deployments configure `neuron_claimer_permissions` to include all permissions (as the integration test demonstrates), making `Disburse` strippable in the same operation. [7](#0-6) 

## Recommendation
Add a post-removal invariant check inside `remove_neuron_permissions` (or `remove_permissions_for_principal`) that rejects any operation that would leave the neuron with zero principals holding `Disburse` permission. Concretely: after computing the would-be resulting permissions, verify that at least one `NeuronPermission` entry across all principals still contains `NeuronPermissionType::Disburse`; if not, return a descriptive `GovernanceError`. Alternatively, implement a governance-level recovery mechanism allowing the SNS root or governance canister to forcibly disburse a permission-less neuron to its original staking subaccount.

## Proof of Concept
The existing integration test `test_neuron_remove_all_permissions_of_self` (neuron.rs L2213–2264) already demonstrates the self-inflicted variant end-to-end and asserts `neuron.permissions.len() == 0`. To prove the malicious co-principal variant:

1. Deploy a local SNS with `neuron_claimer_permissions = NeuronPermissionType::all()`.
2. Alice stakes and claims a neuron; assert `neuron.permissions.len() == 1` with all permissions.
3. Alice calls `AddNeuronPermissions` granting Bob `ManagePrincipals`.
4. Bob calls `RemoveNeuronPermissions` targeting Alice's principal with `NeuronPermissionType::all()`.
5. Bob calls `RemoveNeuronPermissions` targeting his own principal with `[ManagePrincipals]`.
6. Assert `neuron.permissions.len() == 0`.
7. Advance time past the dissolve delay; Alice calls `Disburse`.
8. Assert the response is `GovernanceError { error_type: NotAuthorized }`. [8](#0-7)

fn test_neuron_remove_all_permissions_of_self() {
    local_test_on_sns_subnet(|runtime| async move {
        let user = Sender::from_keypair(&TEST_USER1_KEYPAIR);
        let account_identifier = Account {
            owner: user.get_principal_id().0,
            subaccount: None,
        };
        let alloc = Tokens::from_tokens(1000).unwrap();

        let system_params = NervousSystemParameters {
            neuron_claimer_permissions: Some(NeuronPermissionList {
                permissions: NeuronPermissionType::all(),
            }),
            ..NervousSystemParameters::with_default_values()
        };

        let sns_init_payload = SnsTestsInitPayloadBuilder::new()
            .with_ledger_account(account_identifier, alloc)
            .with_nervous_system_parameters(system_params)
            .build();

        let sns_canisters = SnsCanisters::set_up(&runtime, sns_init_payload).await;

        let neuron_id = sns_canisters.stake_and_claim_neuron(&user, None).await;
        let neuron = sns_canisters.get_neuron(&neuron_id).await;
        let subaccount = neuron.subaccount().expect("Error creating the subaccount");

        // Assert that the Claimer has been granted all permissions
        assert_eq!(neuron.permissions.len(), 1);
        let mut neuron_permission =
            get_neuron_permission_from_neuron(&neuron, &user.get_principal_id());
        // .sort() emits () and needs to be called outside of the assert!
        neuron_permission.permission_type.sort_unstable();
        assert_eq!(
            neuron_permission.permission_type,
            NeuronPermissionType::all(),
        );

        sns_canisters
            .remove_neuron_permissions_or_panic(
                &user,
                &subaccount,
                &user.get_principal_id(),
                NeuronPermissionType::all(),
            )
            .await;

        let neuron = sns_canisters.get_neuron(&neuron_id).await;
        assert_eq!(neuron.permissions.len(), 0);

        Ok(())
    });
