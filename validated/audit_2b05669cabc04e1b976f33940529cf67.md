Audit Report

## Title
NNS `spawn_neuron` With `new_controller` Unconditionally Inherits Parent Hot Keys, Granting Previous Owner Unauthorized Governance Access - (File: `rs/nns/governance/src/governance.rs`)

## Summary
When `spawn_neuron` is called with a `new_controller` that differs from the parent's controller, the child neuron is created with the new principal as controller but unconditionally copies the parent's full hot key list. The previous owner's hot keys remain authorized to vote, follow, and join/leave the Neuron Fund on the child neuron without the new controller's knowledge or consent.

## Finding Description
In `spawn_neuron`, the `child_controller` is resolved from `spawn.new_controller` if provided, otherwise falls back to the parent's controller:

```rust
// L2651-2655
let child_controller = if let Some(child_controller) = &spawn.new_controller {
    *child_controller
} else {
    parent_neuron.controller()
};
```

Despite `child_controller` potentially being a completely different principal, the child neuron is built at L2704-2718 with an unconditional `.with_hot_keys(parent_neuron.hot_keys.clone())` — no check is made whether `child_controller == parent_neuron.controller()`. There is no guard anywhere in `spawn_neuron` that clears or filters hot keys on controller change.

The inherited hot keys pass `is_hotkey_or_controller` checks (L254-255), granting them:
- `is_authorized_to_vote` (L243-244): vote on NNS proposals on behalf of the child neuron
- `is_authorized_to_configure_or_err` for `JoinCommunityFund`/`LeaveCommunityFund` (L780-781): enroll the child neuron in the Neuron Fund, committing its maturity to SNS swaps

No notification or disclosure mechanism exists to inform the new controller that inherited hot keys are present.

## Impact Explanation
This matches the High ($2,000–$10,000) bounty impact: **Unauthorized access to neurons and governance assets**. Alice's hot keys retain the ability to (1) cast NNS governance votes on behalf of Bob's neuron, skewing governance outcomes without consent, and (2) join Bob's neuron to the Neuron Fund, causing Bob's maturity to be committed to SNS swap participation and potentially reduced — without Bob's approval. Bob can remove the hot keys only if he discovers they exist, which requires active inspection of the neuron state.

## Likelihood Explanation
Reachable by any unprivileged NNS neuron controller. `Spawn` with `new_controller` is a documented, production-accessible governance operation. A realistic scenario is Alice transferring a maturity-bearing neuron to Bob via spawn. No special privileges, admin keys, or majority corruption are required. The attack is repeatable and requires no victim mistake beyond accepting the spawned neuron.

## Recommendation
When `spawn_neuron` is called with a `new_controller` that differs from the parent's controller, initialize the child neuron with an empty hot key list:

```rust
let hot_keys = if child_controller == parent_neuron.controller() {
    parent_neuron.hot_keys.clone()
} else {
    vec![]
};
// ...
.with_hot_keys(hot_keys)
```

This change should be applied at the `NeuronBuilder` call in `spawn_neuron` (L2714 in `rs/nns/governance/src/governance.rs`).

## Proof of Concept
1. Alice creates an NNS neuron and adds a hot key `H` via `AddHotKey`.
2. Alice's neuron accumulates maturity.
3. Alice calls `manage_neuron` with `Command::Spawn(Spawn { new_controller: Some(Bob), percentage_to_spawn: Some(100), nonce: Some(42) })`.
4. Governance creates a child neuron with `controller = Bob` and `hot_keys = [H]` (L2707, L2714).
5. `H` calls `manage_neuron` on the child neuron with `Command::Configure(Configure { operation: Some(JoinCommunityFund(...)) })`.
6. `is_authorized_to_configure_or_err` passes at L2781 because `H` is in `hot_keys`.
7. Bob's neuron is enrolled in the Neuron Fund and its maturity is subject to SNS swap participation without Bob's consent.

A deterministic integration test can be written using PocketIC: create a parent neuron with a hot key, spawn with `new_controller = Bob`, then assert the child neuron's hot key list is non-empty and that the hot key principal can successfully call `JoinCommunityFund` on the child neuron.
