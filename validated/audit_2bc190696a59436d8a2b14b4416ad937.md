### Title
Secret Scalar Key Material Exposed in Plaintext via `EccScalar::Debug` Implementation - (File: `rs/crypto/internal/crypto_lib/threshold_sig/canister_threshold_sig/src/utils/group.rs`)

---

### Summary
The `EccScalar` type — the internal representation of secret cryptographic scalars used in threshold ECDSA/Schnorr signing and MEGa encryption — implements `Debug` by printing the full hex-encoded secret scalar value. This is directly analogous to the mnemonic phrase being stored/exposed in plaintext: any code path that formats an `EccScalar` with `{:?}` (common in Rust error handling) will emit the raw secret key material into logs or error strings, where it persists in memory and may be written to disk.

---

### Finding Description

In `rs/crypto/internal/crypto_lib/threshold_sig/canister_threshold_sig/src/utils/group.rs`, the `Debug` implementation for `EccScalar` serializes and hex-encodes the full secret scalar:

```rust
impl fmt::Debug for EccScalar {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        write!(
            f,
            "{}(0x{})",
            self.curve_type(),
            hex::encode(self.serialize())   // ← full 32-byte secret emitted
        )
    }
}
``` [1](#0-0) 

This is in direct contradiction to every other secret-bearing type in the codebase, all of which explicitly redact:

- `MEGaPrivateKey::Debug` → `"MEGaPrivateKey({curve}) - REDACTED"` [2](#0-1) 
- `SecretKeyBytes::Debug` (multi-sig) → `"REDACTED"` [3](#0-2) 
- `TlsEd25519SecretKeyDerBytes::Debug` → `"REDACTED"` [4](#0-3) 
- `CommitmentOpening::Debug` → `"CommitmentOpening::{name}({curve}(REDACTED))"` [5](#0-4) 
- `SecretShares::Debug` → `"SecretShares::{name}{curve} - REDACTED"` [6](#0-5) 
- `CspSecretKey::Debug` (test-only) → all variants print `"REDACTED"` [7](#0-6) 

`EccScalar` is the raw secret scalar underlying all of these types. It is the direct inner value of `MEGaPrivateKey` [8](#0-7)  and the payload of `CommitmentOpening::Simple` / `CommitmentOpening::Pedersen` [9](#0-8) , which are the threshold signing key shares stored in the production `InMemorySecretKeyStore` (the `canister_secret_key_store` of `ProdLocalCspVault`). [10](#0-9) 

The `InMemorySecretKeyStore` holds these `CspSecretKey::IDkgCommitmentOpening` values in a plain `HashMap` in the replica process memory for the lifetime of the process, with no disk persistence. [11](#0-10) 

The `secrets_containers` crate itself acknowledges the limitation: zeroization is best-effort and "any copies that exist, say during creation of the secret or during serialization/deserialization, may need to be cleared manually." [12](#0-11)  When `EccScalar::Debug` is invoked, `self.serialize()` produces a fresh `Vec<u8>` containing the raw secret bytes, which is then passed to `hex::encode` producing a `String` — both heap allocations that are **not** zeroized and persist in memory until the allocator reclaims them.

Concrete reachable trigger: the IDKG load-transcript error path formats the full result with `{:?}`:

```rust
Err(e @ IDkgComputeSecretSharesInternalError::UnableToReconstruct(_)) => {
    Err(IDkgLoadTranscriptError::InvalidArguments {
        internal_error: format!("{compute_secret_shares_result:?}"),  // ← {:?} on Result<CommitmentOpening, …>
    })
}
``` [13](#0-12) 

Although `CommitmentOpening` itself has a custom redacting Debug, the `EccScalar` Debug is called whenever `EccScalar` appears in any struct that derives `Debug` without a custom implementation — a pattern that already exists in `ThresholdEcdsaCombinedSigInternal` (r, s scalars) and `MEGaCiphertextSingle`/`MEGaCiphertextPair` (ciphertext scalars). Any future struct holding a secret `EccScalar` with `#[derive(Debug)]` will silently leak the secret.

---

### Impact Explanation

If a threshold signing key share (`EccScalar` inside `CommitmentOpening`) is emitted via `{:?}` formatting into a log line or error string, the raw 32-byte secret scalar is written as a hex string into the replica's log output and into heap-allocated `String`/`Vec<u8>` buffers that are not zeroized. An attacker who can read replica logs (e.g., a compromised log aggregation pipeline, a misconfigured log export, or a co-located malicious node operator below the consensus fault threshold) can recover individual nodes' threshold key shares. Collecting shares from `t` nodes allows reconstruction of the subnet's master threshold signing key, enabling forgery of threshold ECDSA/Schnorr signatures on behalf of the subnet.

Impact: **High** — full threshold key compromise if shares from a threshold number of nodes are obtained.

---

### Likelihood Explanation

Likelihood: **Low-to-Medium**. The immediate wrappers (`CommitmentOpening`, `SecretShares`, `MEGaPrivateKey`) all have custom redacting Debug implementations, so existing error paths are largely protected. However, the root cause is present and reachable: (1) the `EccScalar` Debug impl unconditionally serializes the secret; (2) the type is widely used and any new struct with `#[derive(Debug)]` containing a secret `EccScalar` will silently regress; (3) log aggregation or crash-dump exposure of replica logs is a realistic operational scenario.

---

### Recommendation

Replace the `EccScalar::Debug` implementation to redact the secret value, consistent with every other secret-bearing type in the codebase:

```rust
impl fmt::Debug for EccScalar {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        write!(f, "EccScalar({}) - REDACTED", self.curve_type())
    }
}
```

Additionally, audit all structs that `#[derive(Debug)]` and contain `EccScalar` fields to ensure none hold secret key material without a custom redacting Debug implementation.

        };

        let curve = match self {

                } else {
                    format!("{c1:?}")
                }
            }
        };

        write!(f, "CommitmentOpening::{name}({curve}(REDACTED))")
    }
}

    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        let name = match self {
            Self::Random => "Random",
            Self::RandomUnmasked => "RandomUnmasked",
            Self::ReshareOfUnmasked(_) => "ReshareOfUnmasked",
            Self::ReshareOfMasked(_, _) => "ReshareOfMasked",
            Self::UnmaskedTimesMasked(_, _) => "UnmaskedTimesMasked",
        };

        fn format_curve_id(curves: &[EccCurveType]) -> String {
            if curves.is_empty() {
                return "".to_string();
            }

            if curves.windows(2).all(|w| w[0] == w[1]) {
                format!("({:?})", curves[0])
            } else {
                format!(" unsupported curve combination {curves:?}")
            }
        }

        let curve = match self {
            Self::Random => "".to_string(),
            Self::RandomUnmasked => "".to_string(),
            Self::ReshareOfUnmasked(s) => format_curve_id(&[s.curve_type()]),
            Self::ReshareOfMasked(s1, s2) => format_curve_id(&[s1.curve_type(), s2.curve_type()]),
            Self::UnmaskedTimesMasked(s1, (s2, s3)) => {
                format_curve_id(&[s1.curve_type(), s2.curve_type(), s3.curve_type()])
            }
        };

        let redacted = if curve.is_empty() { "" } else { " - REDACTED" };
        write!(f, "SecretShares::{name}{curve}{redacted}")
    }
