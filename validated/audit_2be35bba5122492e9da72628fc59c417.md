### Title
Missing Caller-Controlled Minimum Cycles Output in CMC `notify_top_up` / `notify_mint_cycles` - (File: rs/nns/cmc/src/main.rs)

### Summary

The Cycles Minting Canister (CMC) converts ICP to cycles using a stored `icp_xdr_conversion_rate` at the moment `notify_top_up` or `notify_mint_cycles` is executed. Neither endpoint accepts a caller-supplied minimum cycles floor. Because the ICP transfer to the CMC subaccount is irrevocable before the notify call is made, a rate update that occurs in the window between the transfer and the notify call silently reduces the cycles the caller receives, with no recourse.

### Finding Description

The two-step ICP-to-cycles flow is:

1. Caller queries `get_icp_xdr_conversion_rate` (query call, uncertified or certified).
2. Caller transfers ICP to the CMC subaccount on the ICP ledger (irreversible).
3. Caller calls `notify_top_up` or `notify_mint_cycles`.

At step 3, `tokens_to_cycles` reads the current `icp_xdr_conversion_rate` from CMC state and computes the cycles amount with no reference to what the caller observed at step 1: [1](#0-0) 

Neither `NotifyTopUpArg` nor `NotifyMintCyclesArg` carries a `minimum_cycles` field: [2](#0-1) [3](#0-2) 

The structs in Rust confirm the same absence: [4](#0-3) [5](#0-4) 

`process_top_up` and `process_mint_cycles` both call `tokens_to_cycles` and immediately proceed to deposit or mint with no minimum check: [6](#0-5) [7](#0-6) 

The rate is updated by NNS governance via `set_icp_xdr_conversion_rate`, which is restricted to the governance canister but executes on a regular cadence driven by the Exchange Rate Canister (XRC): [8](#0-7) 

### Impact Explanation

A caller who observed a rate of R XDR/ICP and transferred N ICP expects at least `N * R * cycles_per_xdr` cycles. If the governance canister updates the rate downward between the ICP transfer (step 2) and the notify call (step 3), the caller receives fewer cycles than expected with no ability to abort. The ICP is burned regardless. The shortfall is proportional to the rate drop and the ICP amount. For large top-ups (e.g., the subnet rental path, which has a 500 PT/month limit), the absolute cycle loss can be significant. [9](#0-8) 

### Likelihood Explanation

The XDR/ICP rate is updated by NNS governance on a regular schedule from the Exchange Rate Canister. The rate can move materially between the time a user queries it and the time their notify call is executed, especially during periods of ICP price volatility. The two-step protocol (transfer then notify) is mandatory and creates an unavoidable window. Any unprivileged ingress sender can trigger this path. No special access is required. [10](#0-9) 

### Recommendation

- **Short term**: Add an optional `minimum_cycles: opt nat` field to `NotifyTopUpArg` and `NotifyMintCyclesArg`. In `process_top_up` and `process_mint_cycles`, after computing `cycles = tokens_to_cycles(amount)`, check `cycles >= minimum_cycles` and refund the ICP if the check fails.
- **Long term**: Expose the minimum-output guard as a standard field across all ICP-to-resource conversion endpoints so callers can always express their rate assumptions explicitly, consistent with the best practice used by large DeFi protocols.

### Proof of Concept

1. Alice queries `get_icp_xdr_conversion_rate` and observes 50,000 XDR/ICP (permyriad).
2. Alice transfers 10 ICP to the CMC subaccount for her canister, expecting `10 * 50000/10000 * 1e12 = 50T` cycles.
3. Before Alice calls `notify_top_up`, NNS governance updates the rate to 40,000 XDR/ICP.
4. Alice calls `notify_top_up`. `tokens_to_cycles` reads the new rate and mints only `10 * 40000/10000 * 1e12 = 40T` cycles.
5. Alice's canister receives 40T cycles instead of the expected 50T. The 10 ICP is burned. Alice has no recourse and received no warning. [1](#0-0) [5](#0-4)

fn set_icp_xdr_conversion_rate(
    proposed_conversion_rate: UpdateIcpXdrConversionRatePayload,
) -> Result<(), String> {
    let caller = caller();

    assert_eq!(
        caller,
        GOVERNANCE_CANISTER_ID.into(),
        "{} is not authorized to call this method: {}",
        caller,
        "set_icp_xdr_conversion_rate"
    );

    let env = CanisterEnvironment;
    let rate = IcpXdrConversionRate::from(&proposed_conversion_rate);
    let rate_timestamp_seconds = rate.timestamp_seconds;
    let result = do_set_icp_xdr_conversion_rate(&STATE, &env, rate);
    if result.is_ok() && with_state(|state| state.exchange_rate_canister_id.is_some()) {
        exchange_rate_canister::set_update_exchange_rate_state(
            &STATE,
            &proposed_conversion_rate.reason,
            rate_timestamp_seconds,
        );
    }

    result
}

fn tokens_to_cycles(amount: Tokens) -> Result<Cycles, NotifyError> {
    with_state(|state| {
        let xdr_permyriad_per_icp = state
            .icp_xdr_conversion_rate
            .as_ref()
            .map(|rate| rate.xdr_permyriad_per_icp);
        match xdr_permyriad_per_icp {
            Some(xdr_permyriad_per_icp) => Ok(TokensToCycles {
                xdr_permyriad_per_icp,
                cycles_per_xdr: state.cycles_per_xdr,
            }
            .to_cycles(amount)),
            None => {
                let error_message =
                    "No conversion rate found in CMC, notification aborted".to_string();
                print(&error_message);
                Err(NotifyError::Other {
                    error_code: NotifyErrorCode::Internal as u64,
                    error_message,
                })
            }
        }
    })
}

async fn process_mint_cycles(
    to_account: Account,
    amount: Tokens,
    deposit_memo: Option<Vec<u8>>,
    from: AccountIdentifier,
    sub: Subaccount,
) -> NotifyMintCyclesResult {
    let cycles = tokens_to_cycles(amount)?;
    match do_mint_cycles(to_account, cycles, deposit_memo).await {
        Ok(deposit_result) => {
            burn_and_log(sub, amount).await;
            Ok(NotifyMintCyclesSuccess {
                block_index: deposit_result.block_index,
                minted: cycles.into(),
                balance: deposit_result.balance,
            })
        }
        Err(err) => {
            let refund_block = refund_icp(sub, from, amount, MINT_CYCLES_REFUND_FEE).await?;
            Err(NotifyError::Refunded {
                reason: err,
                block_index: refund_block,
            })
        }
    }
}

async fn process_top_up(
    canister_id: CanisterId,
    from: AccountIdentifier,
    amount: Tokens,
    limiter_to_use: CyclesMintingLimiterSelector,
) -> Result<Cycles, NotifyError> {
    let cycles = tokens_to_cycles(amount)?;

    let sub = Subaccount::from(&canister_id);

    print(format!(
        "Topping up canister {canister_id} by {cycles} cycles."
    ));

    match deposit_cycles(canister_id, cycles, true, limiter_to_use).await {
        Ok(()) => {
            burn_and_log(sub, amount).await;
            Ok(cycles)
        }
        Err(err) => {
            let refund_block = refund_icp(sub, from, amount, TOP_UP_CANISTER_REFUND_FEE).await?;
            Err(NotifyError::Refunded {
                reason: err.to_string(),
                block_index: refund_block,
            })
        }
    }
}
