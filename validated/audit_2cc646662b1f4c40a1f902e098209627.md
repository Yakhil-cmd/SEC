### Title
`resume_precompiles` Blocked by `require_running` When Contract Is Paused, Preventing Independent Precompile Resumption - (File: `engine/src/contract_methods/admin.rs`)

---

### Summary

The `resume_precompiles` function enforces a `require_running` check, meaning it can only be called when the contract is **not** paused. Aurora Engine has two independent pause mechanisms — the global contract pause (`state.is_paused`) and per-precompile pause flags (`PrecompileFlags`). When both are active simultaneously, the `require_running` guard inside `resume_precompiles` makes it impossible to resume the exit precompiles without first resuming the entire contract. This forces the admin into a binary choice: keep all user funds frozen, or expose the full contract to resume precompile access.

---

### Finding Description

`resume_precompiles` in `engine/src/contract_methods/admin.rs` reads the engine state and immediately calls `require_running`, which returns `ERR_PAUSED` if `state.is_paused == true`:

```rust
pub fn resume_precompiles<I: IO + Copy, E: Env>(io: I, env: &E) -> Result<(), ContractError> {
    with_hashchain(io, env, function_name!(), |io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;   // ← blocks when contract is paused
        ...
        pauser.resume_precompiles(flags);
        Ok(())
    })
}
``` [1](#0-0) 

`require_running` is defined as:

```rust
pub fn require_running(state: &state::EngineState) -> Result<(), ContractError> {
    if state.is_paused {
        return Err(errors::ERR_PAUSED.into());
    }
    Ok(())
}
``` [2](#0-1) 

The contract pause state is set by `pause_contract`:

```rust
pub fn pause_contract<I: IO + Copy, E: Env>(io: I, env: &E) -> Result<(), ContractError> {
    with_hashchain(io, env, function_name!(), |mut io| {
        let mut state = state::get_state(&io)?;
        require_owner_only(&state, &env.predecessor_account_id())?;
        require_running(&state)?;
        state.is_paused = true;
        state::set_state(&mut io, &state)?;
        Ok(())
    })
}
``` [3](#0-2) 

The precompile pause flags are a **separate** storage key (`PAUSE_FLAGS`) managed independently: [4](#0-3) 

These two mechanisms are orthogonal — one can be active without the other. However, `resume_precompiles` conflates them by requiring the contract to be running before allowing precompile flag changes.

---

### Impact Explanation

The exit precompiles `ExitToNear` and `ExitToEthereum` are the **only** mechanisms for users to withdraw ETH and ERC-20 tokens from Aurora to NEAR or Ethereum. When these precompiles are paused, user funds are frozen on Aurora. If the contract is also paused (a realistic emergency scenario), the admin cannot resume the precompiles independently. The admin is forced to:

1. Call `resume_contract` (unpauses the entire contract, potentially re-exposing whatever triggered the pause), then
2. Call `resume_precompiles`.

During this window, user funds remain frozen. The admin cannot selectively restore withdrawal capability while keeping other contract operations blocked. This constitutes a **temporary freezing of funds** for all Aurora users whose assets depend on the exit precompiles.

---

### Likelihood Explanation

This requires both the contract and exit precompiles to be paused simultaneously. This is a realistic operational scenario: an admin may pause precompiles in response to a bridge exploit, then separately pause the contract for an emergency upgrade. The two pause actions are independent and can be triggered by different authorized parties (`pause_precompiles` uses an `EngineAuthorizer` ACL, while `pause_contract` requires the owner). The combination is not hypothetical — the test utility `init_hashchain` explicitly documents the pattern of pausing the contract while other state is active. [5](#0-4) 

---

### Recommendation

Remove the `require_running` guard from `resume_precompiles`. Resuming individual precompile flags is a recovery action that should be callable regardless of the global contract pause state. The analogous `pause_precompiles` function also carries `require_running`, which should similarly be reconsidered — an admin should be able to adjust precompile pause flags even when the contract is paused. At minimum, `resume_precompiles` must not be gated on the contract running state, since it is the mechanism to restore user withdrawal access.

---

### Proof of Concept

1. Authorized account calls `pause_precompiles` with `EXIT_TO_NEAR | EXIT_TO_ETHEREUM` mask → precompile flags set, user withdrawals blocked.
2. Owner calls `pause_contract` → `state.is_paused = true`.
3. Users attempt to exit Aurora via `ExitToNear`/`ExitToEthereum` precompiles → blocked by precompile pause flags.
4. Owner calls `resume_precompiles` to restore withdrawal access → **fails with `ERR_PAUSED`** because `require_running` rejects the call.
5. Owner is forced to call `resume_contract` first (fully unpausing the contract), then call `resume_precompiles`.
6. Between steps 4 and 5, user funds remain frozen and the admin has no path to selectively restore exit functionality while keeping the contract paused.

impl<I: IO> EnginePrecompilesPauser<I> {
    /// Key for storing [`PrecompileFlags`].
    const PAUSE_FLAGS_KEY: &'static [u8; 11] = b"PAUSE_FLAGS";

    /// Creates new [`EnginePrecompilesPauser`] instance that reads from and writes into storage accessed using `io`.
    pub const fn from_io(io: I) -> Self {
        Self { io }
    }

    fn read_flags_from_storage(&self) -> PrecompileFlags {
        self.io
            .read_storage(&Self::storage_key())
            .map_or_else(PrecompileFlags::empty, |bytes| {
                const U32_SIZE: usize = size_of::<u32>();
                assert_eq!(bytes.len(), U32_SIZE, "PrecompileFlags value is corrupted");
                let mut buffer = [0u8; U32_SIZE];
                bytes.copy_to_slice(&mut buffer);
                PrecompileFlags::from_bits_truncate(u32::from_le_bytes(buffer))
            })
    }

    fn write_flags_into_storage(&mut self, pause_flags: PrecompileFlags) {
        self.io
            .write_storage(&Self::storage_key(), &pause_flags.bits().to_le_bytes());
    }

    fn storage_key() -> Vec<u8> {
        bytes_to_key(KeyPrefix::Config, Self::PAUSE_FLAGS_KEY)
    }
