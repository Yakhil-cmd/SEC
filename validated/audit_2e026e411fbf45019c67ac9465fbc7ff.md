### Title
Silo Whitelist Bypass via `deploy_code` Entrypoint — (`File: engine/src/contract_methods/evm_transactions.rs`)

### Summary

In Silo mode, the `Admin` and `EvmAdmin` whitelists are intended to restrict EVM code deployment to authorized accounts. The `submit` entrypoint enforces this via `assert_access`, but the `deploy_code` entrypoint performs no whitelist check at all. Any NEAR account can bypass the Silo deployment whitelist by calling `deploy_code` directly.

### Finding Description

The Aurora Engine exposes two paths for deploying EVM bytecode:

1. **`submit` / `submit_with_args`** — accepts a signed Ethereum transaction. When the transaction has no `to` field (contract creation), `engine::submit_with_alt_modexp` calls `assert_access` at line 1052 of `engine/src/engine.rs`, which routes to `silo::is_allow_deploy`. This checks both `WhitelistKind::Admin` (NEAR account) and `WhitelistKind::EvmAdmin` (EVM address).

2. **`deploy_code`** — a NEAR-native entrypoint that deploys EVM bytecode directly. Its implementation in `engine/src/contract_methods/evm_transactions.rs` only calls `require_running` and then immediately invokes `engine.deploy_code_with_input`. There is no call to `assert_access`, `is_allow_deploy`, or any whitelist check.

The `deploy_code` WASM export in `engine/src/lib.rs` is callable by any NEAR account with no access restriction beyond the contract not being paused.

### Impact Explanation

In a Silo deployment where the operator has enabled the `Admin` and `EvmAdmin` whitelists to restrict code deployment to trusted parties, any unprivileged NEAR account can call `deploy_code` to deploy arbitrary EVM bytecode. This completely nullifies the Silo deployment whitelist. Deployed contracts can interact with existing EVM state, potentially draining funds from other contracts or disrupting the permissioned environment the operator intended to enforce.

**Impact: High — whitelist bypass enabling unauthorized EVM code deployment in Silo mode.**

### Likelihood Explanation

The `deploy_code` entrypoint is a standard exported WASM function callable by any NEAR account. No special privileges, leaked keys, or social engineering are required. Any actor aware of the Aurora Engine ABI can call it directly. Likelihood is high whenever a Silo operator has enabled the `Admin`/`EvmAdmin` whitelists.

### Recommendation

Add the Silo whitelist check to `deploy_code` analogous to what `assert_access` does for contract-creation transactions in `submit`. Specifically, call `silo::is_allow_deploy` with the predecessor account ID and the derived EVM address before invoking `engine.deploy_code_with_input`.

```rust
fn assert_access<I: IO + Copy, E: Env>(
    io: &I,
    env: &E,
    transaction: &NormalizedEthTransaction,
) -> Result<(), EngineError> {
    let allowed = if transaction.to.is_some() {
        silo::is_allow_submit(io, &env.predecessor_account_id(), &transaction.address)
    } else {
        silo::is_allow_deploy(io, &env.predecessor_account_id(), &transaction.address)
    };

    if !allowed {
        return Err(EngineError {
            kind: EngineErrorKind::NotAllowed,
            gas_used: 0,
        });
    }

    Ok(())
}
```

```rust
fn is_account_allowed_deploy<I: IO + Copy>(io: &I, account_id: &AccountId) -> bool {
    let list = Whitelist::init(io, WhitelistKind::Admin);
    !list.is_enabled() || list.is_exist(account_id)
}

fn is_address_allowed_deploy<I: IO + Copy>(io: &I, address: &Address) -> bool {
```

```rust
pub fn deploy_code<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<SubmitResult, ContractError> {
    with_logs_hashchain(io, env, function_name!(), |mut io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;
        let input = io.read_input().to_vec();
        let current_account_id = env.current_account_id();
        let mut engine: Engine<_, E, AuroraModExp> = Engine::new_with_state(
            state,
            predecessor_address(&env.predecessor_account_id()),
            current_account_id,
            io,
            env,
        );
        let result = engine.deploy_code_with_input(input, None, handler)?;
        let result_bytes = borsh::to_vec(&result).map_err(|_| errors::ERR_SERIALIZE)?;
        io.return_output(&result_bytes);
        Ok(result)
    })
```
