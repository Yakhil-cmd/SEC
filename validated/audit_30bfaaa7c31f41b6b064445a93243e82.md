### Title
Silo Whitelist Bypass via `call` and `deploy_code` Entry Points — (File: `engine/src/contract_methods/evm_transactions.rs`)

### Summary
The `submit` and `submit_with_args` NEAR entry points enforce silo whitelist access control through `assert_access`, but the `call` and `deploy_code` entry points omit this check entirely. Any unprivileged NEAR account can invoke `call` or `deploy_code` to interact with EVM contracts or deploy EVM code in Silo mode, circumventing the operator-configured whitelist that is the primary access control mechanism for Silo deployments.

### Finding Description
In Silo mode, Aurora Engine enforces access control via four whitelists (`Admin`, `EvmAdmin`, `Account`, `Address`). The `submit` and `submit_with_args` entry points call the free function `engine::submit`, which internally invokes `assert_access`:

```rust
fn assert_access<I: IO + Copy, E: Env>(
    io: &I,
    env: &E,
    transaction: &NormalizedEthTransaction,
) -> Result<(), EngineError> {
    let allowed = if transaction.to.is_some() {
        silo::is_allow_submit(io, &env.predecessor_account_id(), &transaction.address)
    } else {
        silo::is_allow_deploy(io, &env.predecessor_account_id(), &transaction.address)
    };
    if !allowed {
        return Err(EngineError { kind: EngineErrorKind::NotAllowed, gas_used: 0 });
    }
    Ok(())
}
``` [1](#0-0) 

`is_allow_submit` checks both the `Account` and `Address` whitelists; `is_allow_deploy` checks both the `Admin` and `EvmAdmin` whitelists. [2](#0-1) 

However, the `call` entry point calls `engine.call_with_args` with **no silo whitelist check**:

```rust
pub fn call<I: IO + Copy, E: Env, H: PromiseHandler>(...) -> Result<SubmitResult, ContractError> {
    with_logs_hashchain(io, env, function_name!(), |mut io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;          // ← only check: contract is running
        ...
        let result = engine.call_with_args(args, handler)?;
        ...
    })
}
``` [3](#0-2) 

Similarly, the `deploy_code` entry point calls `engine.deploy_code_with_input` with **no silo whitelist check**:

```rust
pub fn deploy_code<I: IO + Copy, E: Env, H: PromiseHandler>(...) -> Result<SubmitResult, ContractError> {
    with_logs_hashchain(io, env, function_name!(), |mut io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;          // ← only check: contract is running
        ...
        let result = engine.deploy_code_with_input(input, None, handler)?;
        ...

``` [4](#0-3) 

`deploy_code_with_input` itself also contains no call to `assert_access`:

```rust
pub fn deploy_code_with_input<P: PromiseHandler>(
    &mut self, input: Vec<u8>, address: Option<Address>, handler: &mut P,
) -> EngineResult<SubmitResult> {
    let origin = Address::new(self.origin());
    let value = Wei::zero();
    self.deploy_code(origin, value, input, address, u64::MAX, Vec::new(), handler)
}
``` [5](#0-4) 

The silo tests confirm that whitelist enforcement is only exercised through `submit_transaction` (the `submit` path), not through `call` or `deploy_code`. [6](#0-5) 

**Exploit path:**
1. Silo operator enables whitelists (`Admin`, `EvmAdmin`, `Account`, `Address`) and does not add the attacker's NEAR account or EVM address.
2. Attacker calls the `call` NEAR entry point directly with calldata targeting any EVM contract. The predecessor account ID is mapped to an EVM origin address via `predecessor_address`.
3. The call executes against the EVM contract with no whitelist rejection — the attacker can invoke any EVM function as their own EVM address.
4. Alternatively, the attacker calls `deploy_code` to deploy arbitrary EVM bytecode, bypassing the `Admin`/`EvmAdmin` whitelist entirely.

### Impact Explanation
In Silo mode, the whitelist is the sole mechanism preventing unauthorized EVM access. Bypassing it allows non-whitelisted NEAR accounts to:

- **Call any EVM contract** as their derived EVM address. If EVM contracts hold ETH or ERC-20 balances accessible to the caller's address (e.g., tokens previously bridged to that address via `ft_on_transfer`, which has no whitelist check on the recipient in `receive_base_tokens`), the attacker can transfer those funds out.
- **Deploy arbitrary EVM code**, circumventing the `Admin`/`EvmAdmin` deployment restriction.

The operator's security model is broken: the whitelist restriction is enforced on `submit`/`submit_with_args` but not on `call`/`deploy_code`, creating an inconsistent and exploitable gap. Impact is **High** — unauthorized fund access and contract deployment in Silo deployments. [7](#0-6) 

### Likelihood Explanation
The `call` and `deploy_code` NEAR entry points are public, callable by any NEAR account with no special privileges. The bypass requires no cryptographic material, no governance capture, and no admin keys — only the ability to send a NEAR transaction to the Aurora Engine contract. Likelihood is **High**. [8](#0-7) 

### Recommendation
Add silo whitelist checks to both `call` and `deploy_code` before dispatching to the EVM, mirroring the `assert_access` pattern used in `engine::submit`. Specifically:

- In `call`: check `silo::is_allow_submit(io, &predecessor_account_id, &derived_evm_address)` and return `ERR_NOT_ALLOWED` if the check fails.
- In `deploy_code`: check `silo::is_allow_deploy(io, &predecessor_account_id, &derived_evm_address)` and return `ERR_NOT_ALLOWED` if the check fails. [9](#0-8) 

### Proof of Concept
```
1. Deploy Aurora Engine in Silo mode.
2. Enable all four whitelists (Admin, EvmAdmin, Account, Address).
3. Do NOT add attacker's NEAR account or EVM address to any whitelist.
4. Attempt `submit` with attacker's key → rejected with ERR_NOT_ALLOWED (whitelist enforced).
5. Attempt `call` with attacker's key targeting any EVM contract → succeeds (no whitelist check).
6. Attempt `deploy_code` with attacker's key → succeeds (no whitelist check).
7. Confirm the silo whitelist is bypassed for steps 5 and 6.
```

pub fn deploy_code<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<SubmitResult, ContractError> {
    with_logs_hashchain(io, env, function_name!(), |mut io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;
        let input = io.read_input().to_vec();
        let current_account_id = env.current_account_id();
        let mut engine: Engine<_, E, AuroraModExp> = Engine::new_with_state(
            state,
            predecessor_address(&env.predecessor_account_id()),
            current_account_id,
            io,
            env,
        );
        let result = engine.deploy_code_with_input(input, None, handler)?;
        let result_bytes = borsh::to_vec(&result).map_err(|_| errors::ERR_SERIALIZE)?;
        io.return_output(&result_bytes);
        Ok(result)
    })
}

pub fn call<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<SubmitResult, ContractError> {
    with_logs_hashchain(io, env, function_name!(), |mut io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;
        let bytes = io.read_input().to_vec();
        let args = CallArgs::deserialize(&bytes).ok_or(errors::ERR_BORSH_DESERIALIZE)?;
        let current_account_id = env.current_account_id();
        let predecessor_account_id = env.predecessor_account_id();

        let mut engine: Engine<_, E, AuroraModExp> = Engine::new_with_state(
            state,
            predecessor_address(&predecessor_account_id),
            current_account_id,
            io,
            env,
        );
        let result = engine.call_with_args(args, handler)?;
        let result_bytes = borsh::to_vec(&result).map_err(|_| errors::ERR_SERIALIZE)?;
        io.return_output(&result_bytes);
        Ok(result)
    })
}

    set_silo_params(&mut runner, Some(SILO_PARAMS_ARGS));
    enable_all_whitelists(&mut runner);

    validate_address_balance_and_nonce(&runner, sender, INITIAL_BALANCE, INITIAL_NONCE.into())
        .unwrap();
    validate_address_balance_and_nonce(&runner, receiver, ZERO_BALANCE, INITIAL_NONCE.into())
        .unwrap();

    // perform transfer
    let err = runner
        .submit_transaction(&signer.secret_key, transaction.clone())
        .unwrap_err();
    assert_eq!(err.kind, EngineErrorKind::NotAllowed);

    // validate post-state
    validate_address_balance_and_nonce(&runner, sender, INITIAL_BALANCE, INITIAL_NONCE.into())
        .unwrap();
    validate_address_balance_and_nonce(&runner, receiver, ZERO_BALANCE, INITIAL_NONCE.into())
        .unwrap();

    // Add caller and signer to whitelists.
    add_account_to_whitelist(&mut runner, caller);
    add_address_to_whitelist(&mut runner, sender);

    // perform transfer
    let result = runner
        .submit_transaction(&signer.secret_key, transaction)
        .unwrap();
