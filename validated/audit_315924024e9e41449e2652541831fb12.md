Looking at the Aurora Engine codebase, I need to find an analog to the "state variable set on process initiation, not reset on process completion" vulnerability class. Let me trace the upgrade state management.

**Key finding:** `upgrade()` does not clear the staged upgrade state (`CODE_KEY` / `CODE_STAGE_KEY`) set by `stage_upgrade()`.

In `engine/src/contract_methods/admin.rs`, `stage_upgrade()` writes two storage entries:
- `CODE_KEY` — the staged code bytes
- `CODE_STAGE_KEY` — the delay block height

In `engine/src/lib.rs`, `deploy_upgrade()` reads `CODE_KEY`, deploys it, and removes `CODE_STAGE_KEY`. But `upgrade()` (the direct upgrade path) deploys code via a promise batch and **never touches `CODE_KEY` or `CODE_STAGE_KEY`**.

Since `deploy_upgrade()` is callable by anyone (per CHANGES.md: "It is now possible for anyone to call the contract method `deploy_upgrade`"), the staged state left behind by `upgrade()` is exploitable.

---

### Title
`upgrade()` Does Not Clear Staged Upgrade State, Allowing Anyone to Deploy Old Code via `deploy_upgrade()` — (File: engine/src/contract_methods/admin.rs, engine/src/lib.rs)

### Summary
The `upgrade()` function deploys new contract code directly but does not clear the `CODE_KEY` and `CODE_STAGE_KEY` storage entries written by a prior `stage_upgrade()` call. Because `deploy_upgrade()` is permissionless, any external account can invoke it after the delay period to deploy the old staged code, overwriting whatever the owner deployed via `upgrade()`.

### Finding Description
Aurora Engine exposes two independent upgrade paths:

**Path 1 — Staged:** `stage_upgrade()` → `deploy_upgrade()`

`stage_upgrade()` writes:
```
CODE_KEY      ← staged code bytes
CODE_STAGE_KEY ← env.block_height() + state.upgrade_delay_blocks
```
`deploy_upgrade()` reads `CODE_KEY`, self-deploys it, and removes `CODE_STAGE_KEY`. `CODE_KEY` itself is **never removed**.

**Path 2 — Direct:** `upgrade()` (owner-only)

`upgrade()` constructs a `PromiseBatchAction` with `DeployContract` + `state_migration` and returns it. It **does not touch `CODE_KEY` or `CODE_STAGE_KEY`**.

Attack scenario:
1. Owner calls `stage_upgrade()` with code A → `CODE_KEY = code_A`, `CODE_STAGE_KEY = block N`
2. A critical vulnerability is found in code A; owner calls `upgrade()` with hotfix code B → code B is deployed, but `CODE_KEY = code_A` and `CODE_STAGE_KEY = block N` remain in storage
3. After block N, any unprivileged account calls `deploy_upgrade()` → code A (the vulnerable version) is deployed, reverting the hotfix

The root cause is that `upgrade()` has no awareness of, and performs no cleanup of, the staged upgrade state.

### Impact Explanation
An attacker can force deployment of old contract code after the owner has already applied a security fix via `upgrade()`. In the worst case — where the staged code contains a vulnerability that the direct upgrade was meant to patch — this enables the attacker to exploit that vulnerability to steal user funds or permanently freeze them. Impact: **High to Critical** (fund theft / permanent freeze), depending on the content of the staged code.

### Likelihood Explanation
**Medium.** The scenario requires: (a) the owner to have previously called `stage_upgrade()`, and (b) the owner to subsequently use `upgrade()` directly (e.g., for an emergency hotfix). Emergency hotfix workflows are a realistic operational pattern, especially when the staged code is discovered to be flawed after staging. The permissionless nature of `deploy_upgrade()` means no attacker capability beyond a standard NEAR account is required.

### Recommendation
`upgrade()` should clear any pending staged upgrade state before deploying new code:
```rust
// In upgrade(), before deploying:
io.remove_storage(&storage::bytes_to_key(KeyPrefix::Config, CODE_KEY));
io.remove_storage(&storage::bytes_to_key(KeyPrefix::Config, CODE_STAGE_KEY));
```
Alternatively, `deploy_upgrade()` should also remove `CODE_KEY` after deployment to prevent stale code from persisting.

### Proof of Concept
1. Owner calls `stage_upgrade(code_A)` → `CODE_KEY = code_A`, `CODE_STAGE_KEY = block 100`
2. Vulnerability discovered in `code_A`; owner calls `upgrade(code_B)` (hotfix) → `code_B` deployed on-chain; `CODE_KEY` and `CODE_STAGE_KEY` unchanged
3. Block 101 arrives; attacker calls `deploy_upgrade()` (permissionless)
4. `internal_get_upgrade_index()` reads `CODE_STAGE_KEY = 100`; `block_height (101) > 100` → check passes
5. `Runtime::self_deploy(CODE_KEY)` deploys `code_A`, reverting the hotfix
6. `CODE_STAGE_KEY` is removed; `code_A` is now live

```rust
pub fn stage_upgrade<I: IO + Copy, E: Env>(io: I, env: &E) -> Result<(), ContractError> {
    with_hashchain(io, env, function_name!(), |mut io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;
        let delay_block_height = env.block_height() + state.upgrade_delay_blocks;
        require_owner_only(&state, &env.predecessor_account_id())?;
        io.read_input_and_store(&storage::bytes_to_key(KeyPrefix::Config, CODE_KEY));
        io.write_storage(
            &storage::bytes_to_key(KeyPrefix::Config, CODE_STAGE_KEY),
            &delay_block_height.to_le_bytes(),
        );
        Ok(())
    })
}
```

```rust
pub fn upgrade<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<(), ContractError> {
    let state = state::get_state(&io)?;
    require_running(&state)?;
    require_owner_only(&state, &env.predecessor_account_id())?;

    let input = io.read_input().to_vec();
    let (code, state_migration_gas) = match UpgradeParams::try_from_slice(&input) {
        Ok(args) => (
            args.code,
            args.state_migration_gas
                .map_or(GAS_FOR_STATE_MIGRATION, NearGas::new),
        ),
        Err(_) => (input, GAS_FOR_STATE_MIGRATION), // Backward compatibility
    };

    let target_account_id = env.current_account_id();
    let batch = PromiseBatchAction {
        target_account_id,
        actions: vec![
            PromiseAction::DeployContract { code },
            PromiseAction::FunctionCall {
                name: "state_migration".to_string(),
                args: vec![],
                attached_yocto: ZERO_YOCTO,
                gas: state_migration_gas,
            },
        ],
    };
    let promise_id = handler.promise_create_batch(&batch);

    handler.promise_return(promise_id);

    Ok(())
```

```rust
    /// Deploy staged upgrade.
    #[unsafe(no_mangle)]
    pub extern "C" fn deploy_upgrade() {
        // This function is intentionally not implemented in `contract_methods`
        // because it only makes sense in the context of the NEAR runtime.
        let mut io = Runtime;
        let state = state::get_state(&io).sdk_unwrap();
        require_running(&state)
            .map_err(ContractError::msg)
            .sdk_unwrap();
        let index = internal_get_upgrade_index();
        if io.block_height() <= index {
            sdk::panic_utf8(errors::ERR_NOT_ALLOWED_TOO_EARLY);
        }
        Runtime::self_deploy(&bytes_to_key(KeyPrefix::Config, CODE_KEY));
        io.remove_storage(&bytes_to_key(KeyPrefix::Config, CODE_STAGE_KEY));
    }
```
