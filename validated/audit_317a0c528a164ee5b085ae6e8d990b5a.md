## Title
Stale Hardcoded 5% Worst-Case Slippage in `spawn_neuron` Allows Minting Below Minimum Stake Under Mission 70 Modulation Range - (File: `rs/nns/governance/src/governance.rs`)

## Summary
`spawn_neuron` guards against worst-case maturity modulation using a hardcoded `0.05` literal (−5%), but Mission 70 extended the valid modulation range to −10%. The `maybe_spawn_neurons` timer enforces the Mission 70 range at runtime and applies the real modulation with no minimum-output floor. Any maturity value that passes the stale 5% pre-flight check but falls below `neuron_minimum_stake_e8s` after −10% modulation will produce a permanently sub-minimum child neuron, with the parent's maturity already zeroed.

## Finding Description
In `spawn_neuron`, the pre-flight check at line 2666 of `rs/nns/governance/src/governance.rs` computes:

```rust
let least_possible_stake = (maturity_to_spawn as f64 * (1_f64 - 0.05)) as u64;
```

This hardcodes the old ±5% modulation range defined by `MIN_MATURITY_MODULATION_PERMYRIAD = -500` in `rs/nervous_system/governance/src/maturity_modulation/mod.rs` (line 4).

Mission 70 introduced a wider range in `rs/nns/governance/src/timer_tasks/update_icp_xdr_rate_related_data.rs`:
- `MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 = -1_000` (−10%) at line 47
- `MATURITY_MODULATION_MAX_PERMYRIAD_MISSION_70 = 200` (+2%) at line 50

`maybe_spawn_neurons` enforces this Mission 70 range via `VALID_MATURITY_MODULATION_BASIS_POINTS_RANGE` at lines 276–278 of `governance.rs`, rejecting any modulation outside `[-1000, 200]`. It then applies the real modulation at lines 6484–6488 and immediately zeros the parent's maturity and sets `cached_neuron_stake_e8s = neuron_stake` at lines 6515–6517, with no post-modulation minimum stake floor check anywhere in the path (lines 6530–6545 proceed directly to ledger minting). The ledger transfer failure path at lines 6554–6558 does restore maturity on ledger error, but a successful transfer with sub-minimum `neuron_stake` is irreversible.

The gap: `spawn_neuron` permits maturity where `maturity × 0.95 ≥ neuron_minimum_stake_e8s`, but `maybe_spawn_neurons` can apply −10%, producing `maturity × 0.90 < neuron_minimum_stake_e8s`. The SNS governance already uses the correct pattern (`worst_case_maturity_modulation` via `apply_maturity_modulation` at lines 1654–1655 of `rs/sns/governance/src/governance.rs`), but NNS `spawn_neuron` was never updated to match.

## Impact Explanation
A child neuron is permanently recorded with `cached_neuron_stake_e8s < neuron_minimum_stake_e8s`. The parent neuron's maturity is irreversibly zeroed before the ledger mint. This violates the NNS governance invariant that every neuron holds at least the minimum stake, constituting concrete user harm (permanent maturity loss, sub-minimum neuron) and a governance accounting integrity failure. This matches the allowed High impact: **Significant NNS governance security impact with concrete user or protocol harm**.

## Likelihood Explanation
Medium. Two conditions must hold simultaneously: (1) Mission 70 is active — confirmed deployed on mainnet by the presence of `MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70` in production governance constants; (2) the daily modulation value falls in the −5% to −10% band. The `compute_maturity_modulation_permyriad` function (lines 130–196 of `update_icp_xdr_rate_related_data.rs`) can reach −10% when the 7-day ICP price average is sufficiently below the 365-day reference. No privileged access is required — any neuron controller with sufficient maturity can call `spawn_neuron` and trigger this during a period of ICP price decline.

## Recommendation
Replace the hardcoded `0.05` literal in `spawn_neuron` with a computation using the actual enforced Mission 70 minimum:

```rust
use crate::timer_tasks::update_icp_xdr_rate_related_data::MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70;

let worst_case_stake = apply_maturity_modulation(
    maturity_to_spawn,
    MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 as i32,
).map_err(|e| GovernanceError::new_with_message(ErrorType::InsufficientFunds, e))?;

if worst_case_stake < economics.neuron_minimum_stake_e8s {
    return Err(GovernanceError::new_with_message(
        ErrorType::InsufficientFunds,
        "There isn't enough maturity to spawn a new neuron due to worst case maturity modulation.",
    ));
}
```

This mirrors the SNS governance pattern at lines 1654–1655 of `rs/sns/governance/src/governance.rs` and ensures the guard stays in sync with the enforced runtime range.

## Proof of Concept
**Setup**: Mission 70 active. `neuron_minimum_stake_e8s = 100_000_000` (1 ICP). Daily modulation = −800 permyriad (−8%), within the valid Mission 70 range `[-1000, 200]`.

**Step 1** — Controller calls `spawn_neuron` with `maturity_to_spawn = 106_000_000` e8s:
```
least_possible_stake = 106_000_000 × (1 − 0.05) = 100_700_000
100_700_000 ≥ 100_000_000  →  check PASSES  (line 2668)
```
Child neuron enters spawning state with `maturity_e8s_equivalent = 106_000_000`.

**Step 2** — Timer fires `maybe_spawn_neurons`. Modulation = −800 permyriad:
```
neuron_stake = apply_maturity_modulation(106_000_000, -800)
             = 106_000_000 × (10_000 − 800) / 10_000
             = 106_000_000 × 0.92
             = 97_520_000
```
`97_520_000 < 100_000_000 = neuron_minimum_stake_e8s`

Parent maturity zeroed at line 6515. Ledger mints 97,520,000 e8s into child subaccount at lines 6531–6540. Child neuron permanently holds sub-minimum stake. No recourse.
