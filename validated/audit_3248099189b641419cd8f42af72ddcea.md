### Title
Subnet-Wide HTTP Outcall Slot Pool Exhaustion by a Single Canister — (File: `rs/execution_environment/src/execution_environment.rs`)

---

### Summary
The Internet Computer enforces a single subnet-wide cap of 3,000 concurrent canister HTTP outcall contexts (`MAX_CANISTER_HTTP_REQUESTS_IN_FLIGHT`). The enforcement check in `try_add_http_context_to_replicated_state` counts only the **total** number of in-flight contexts with no per-canister sub-limit. A single malicious canister can therefore monopolize all 3,000 slots, causing every other canister on the same subnet to receive `CanisterRejectedMessage` errors when attempting HTTP outcalls for the full duration of the attack.

---

### Finding Description

The global limit is defined as a constant: [1](#0-0) 

It is enforced in `try_add_http_context_to_replicated_state` with a single aggregate count check: [2](#0-1) 

The check reads `canister_http_request_contexts.len()` — the total across **all** canisters on the subnet — and compares it to `max_canister_http_requests_in_flight`. There is no secondary check that limits how many of those 3,000 slots any individual canister may hold. A canister that issues 3,000 concurrent `ic00::HttpRequest` calls (each with a 60-second timeout defined by `CANISTER_HTTP_TIMEOUT_INTERVAL`) fills the entire pool: [3](#0-2) 

Once the pool is full, every subsequent `HttpRequest` from any other canister on the subnet is rejected with `CanisterRejectedMessage`. The attacker can continuously re-fill the pool as slots expire, sustaining the denial indefinitely.

On subnets configured with `CanisterCyclesCostSchedule::Free` (rented subnets), the `charged_fee.real()` is zero, so the payment guard at line 2172 is trivially satisfied with zero cycles: [4](#0-3) [5](#0-4) 

Even on normal-cost subnets, a well-funded canister can sustain the attack economically.

---

### Impact Explanation

All canisters on the targeted subnet that attempt to call `ic00::HttpRequest` while the pool is saturated receive a `CanisterRejectedMessage` rejection. This denies the HTTP outcalls feature to the entire subnet for as long as the attacker maintains the pool at capacity. The 60-second timeout (`CANISTER_HTTP_TIMEOUT_INTERVAL`) is the only natural drain; the attacker can re-issue requests before slots expire. On free-cost-schedule subnets the attack costs nothing. The impact is a complete, sustained DoS of the canister HTTP outcalls feature for all co-tenants of the subnet.

---

### Likelihood Explanation

The attacker entry path requires only the ability to deploy a canister on a subnet with HTTP outcalls enabled — an unprivileged action available to any developer. No special keys, governance majority, or threshold corruption is required. The attack is mechanically simple: call `ic00::HttpRequest` in a tight loop from a canister update method. The 3,000-slot pool can be saturated within a single consensus round given the scheduler's throughput. Likelihood is high for targeted subnets.

---

### Recommendation

Introduce a per-canister sub-limit on the number of in-flight HTTP outcall contexts. Before inserting a new context, count how many existing entries in `canister_http_request_contexts` belong to `request.sender` and reject if that count exceeds a per-canister cap (e.g., `MAX_CANISTER_HTTP_REQUESTS_IN_FLIGHT / expected_active_canisters`). This mirrors the fairness model already applied to ingress message selection (round-robin per-canister quota in `rs/ingress_manager/src/ingress_selector.rs`). [6](#0-5) 

---

### Proof of Concept

1. Deploy a canister `attacker` on any application subnet with HTTP outcalls enabled.
2. In an update method, call `ic00::HttpRequest` in a loop 3,000 times with `max_response_bytes = Some(1)` (minimum fee) and `is_replicated = Some(false)` (non-replicated, cheapest path), each targeting a slow or non-existent URL to keep slots occupied for the full 60-second timeout.
3. Observe that `state.metadata.subnet_call_context_manager.canister_http_request_contexts.len()` reaches 3,000.
4. From any other canister on the same subnet, call `ic00::HttpRequest`. The call is immediately rejected with `"max number (3000) of http requests in-flight reached."` as confirmed by the existing test: [7](#0-6) 

5. Repeat step 2 before the 60-second timeout expires to sustain the denial indefinitely. On a free-cost-schedule subnet, step 2 requires zero cycles.

#[test]
fn http_request_bound_holds() {
    let own_subnet = subnet_test_id(1);
    let caller_canister = canister_test_id(10);
    let mut test = ExecutionTestBuilder::new()
        .with_own_subnet_id(own_subnet)
        .with_caller(own_subnet, caller_canister)
        // set number of max in-flight calls to 10
        .with_max_canister_http_requests_in_flight(10)
        .build();
    test.state_mut().metadata.own_subnet_features.http_requests = true;

    // Create payload of the request.
    let url = "https://".to_string();
    let response_size_limit = 1000_u64;
    let transform_method_name = "transform".to_string();
    let transform_context = vec![0, 1, 2];
    let args = CanisterHttpRequestArgs {
        url: url.clone(),
        max_response_bytes: Some(response_size_limit),
        headers: BoundedHttpHeaders::new(vec![]),
        body: None,
        method: HttpMethod::GET,
        transform: Some(TransformContext {
            function: TransformFunc(candid::Func {
                principal: caller_canister.get().0,
                method: transform_method_name.clone(),
            }),
            context: transform_context.clone(),
        }),
        is_replicated: None,
        pricing_version: None,
    };

    // Create request to HTTP_REQUEST method.
    let payload = args.clone().encode();
    test.inject_call_to_ic00(Method::HttpRequest, payload, Cycles::new(1_000_000_000));
    test.execute_all();
    // Check that the SubnetCallContextManager contains the request.
    let canister_http_request_contexts = &test
        .state()
        .metadata
        .subnet_call_context_manager
        .canister_http_request_contexts;
    assert_eq!(canister_http_request_contexts.len(), 1);

    // Now we try to inject more than the maximum number of requests we allow to be in-flight

    for _ in 0..15 {
        let payload = args.clone().encode();
        test.inject_call_to_ic00(Method::HttpRequest, payload, Cycles::new(1_000_000_000));
    }

    test.execute_all();
    let canister_http_request_contexts = &test
        .state()
        .metadata
        .subnet_call_context_manager
        .canister_http_request_contexts;

    // Check that the SubnetCallContextManager contains the maximum number but not more
    assert_eq!(canister_http_request_contexts.len(), 10);
}
