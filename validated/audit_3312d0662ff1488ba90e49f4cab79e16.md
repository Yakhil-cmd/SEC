Audit Report

## Title
`spawn_neuron` Hardcoded 5% Worst-Case Guard Mismatches Mission 70's -10% Minimum Modulation, Producing Sub-Minimum-Stake Neurons — (`rs/nns/governance/src/governance.rs`)

## Summary

`spawn_neuron` gates spawning on a hardcoded 5% worst-case modulation check, but the Mission 70 maturity modulation floor is -10% (`MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 = -1_000`). Any neuron whose maturity satisfies the 5% guard but not the 10% guard passes `spawn_neuron`, waits 7 days, and has `maybe_spawn_neurons` mint a child neuron with `cached_neuron_stake_e8s < neuron_minimum_stake_e8s`, violating the protocol invariant.

## Finding Description

**Root cause — hardcoded guard in `spawn_neuron`** (`rs/nns/governance/src/governance.rs`, line 2666):
```rust
let least_possible_stake = (maturity_to_spawn as f64 * (1_f64 - 0.05)) as u64;
if least_possible_stake < economics.neuron_minimum_stake_e8s {
    return Err(...)
}
```
This assumes the worst-case modulation is -5% (500 permyriad), matching the legacy `MIN_MATURITY_MODULATION_PERMYRIAD = -500` in `rs/nervous_system/governance/src/maturity_modulation/mod.rs`.

**Mission 70 minimum is -10%** (`rs/nns/governance/src/timer_tasks/update_icp_xdr_rate_related_data.rs`, line 47):
```rust
pub(crate) const MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70: i64 = -1_000;
```

**`maybe_spawn_neurons` accepts -10% modulation** (`rs/nns/governance/src/governance.rs`, lines 276–278 and 6438–6447):
```rust
const VALID_MATURITY_MODULATION_BASIS_POINTS_RANGE: RangeInclusive<i32> =
    MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 as i32
        ..=MATURITY_MODULATION_MAX_PERMYRIAD_MISSION_70 as i32;
```
The sanity check in `maybe_spawn_neurons` only rejects values outside `[-1000, 200]`, so `-1000` passes.

**No secondary minimum-stake check before minting** (`rs/nns/governance/src/governance.rs`, lines 6484–6517):
```rust
let neuron_stake: u64 = match apply_maturity_modulation(original_maturity, maturity_modulation) {
    Ok(neuron_stake) => neuron_stake,
    ...
};
// ...
neuron.cached_neuron_stake_e8s = neuron_stake;  // set unconditionally
```
`apply_maturity_modulation` applies the modulation arithmetically with no floor at `neuron_minimum_stake_e8s`. The result is written directly to `cached_neuron_stake_e8s` and minted via ledger transfer without any post-modulation minimum-stake validation.

**Exploit flow:**
1. Neuron owner calls `spawn_neuron` with maturity in the band `[ceil(min_stake / 0.95), ceil(min_stake / 0.90) - 1]`.
2. `spawn_neuron`'s 5% guard passes.
3. After the 7-day dissolve delay, `maybe_spawn_neurons` fires as a timer task.
4. With `maturity_modulation = -1000` permyriad, `apply_maturity_modulation` produces a stake below `neuron_minimum_stake_e8s`.
5. The sub-minimum-stake value is written to `cached_neuron_stake_e8s` and minted unconditionally.

## Impact Explanation

The protocol invariant `cached_neuron_stake_e8s >= neuron_minimum_stake_e8s` is violated for every spawned neuron in the affected maturity band whenever Mission 70 modulation is at its floor. This constitutes significant NNS governance impact with concrete user harm: neuron owners permanently lose ICP value (maturity is zeroed out and the minted stake is below the minimum), and the resulting sub-minimum-stake neurons may behave unexpectedly in downstream governance logic that assumes the invariant holds. This matches the **High** impact class: "Significant NNS security impact with concrete user or protocol harm."

## Likelihood Explanation

The precondition (Mission 70 modulation at -1000 permyriad) is a protocol-level state driven by ICP price relative to its 365-day average; it is not attacker-controlled but is a reachable protocol state. When it occurs, the exploit is automatic and unprivileged: any neuron owner with maturity in the affected band who calls `spawn_neuron` will have their spawn processed by the timer task with no further checks. No special privileges, social engineering, or external compromise is required.

## Recommendation

Replace the hardcoded `0.05` in `spawn_neuron` with the actual deployed minimum:
```rust
use crate::timer_tasks::MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70;

let worst_case_factor =
    1.0 + (MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 as f64 / 10_000.0);
let least_possible_stake = (maturity_to_spawn as f64 * worst_case_factor) as u64;
```
Additionally, add a post-modulation minimum-stake guard inside `maybe_spawn_neurons` before minting, and on failure return the maturity to the parent neuron rather than minting a sub-minimum-stake child.

## Proof of Concept

```
neuron_minimum_stake_e8s = 100_000_000 (1 ICP)
maturity_to_spawn        = 105_263_159

spawn_neuron guard (5%):
  least_possible_stake = floor(105_263_159 * 0.95) = 100_000_001 >= 100_000_000  ✓ PASSES

maybe_spawn_neurons with maturity_modulation = -1000 permyriad:
  apply_maturity_modulation(105_263_159, -1000)
  = 105_263_159 * (10_000 - 1_000) / 10_000
  = 105_263_159 * 9_000 / 10_000
  = 94_736_843  <  100_000_000  ✗ INVARIANT VIOLATED
```

A deterministic integration test or PocketIC test can reproduce this by: (1) setting `maturity_modulation_permyriad` to `-1000` in governance state, (2) calling `spawn_neuron` with the maturity value above, (3) advancing time past the dissolve delay, (4) triggering `maybe_spawn_neurons`, and (5) asserting `cached_neuron_stake_e8s < neuron_minimum_stake_e8s`.
