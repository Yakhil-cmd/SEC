### Title
`require_running` Guard in `exit_to_near_precompile_callback` Permanently Freezes In-Flight Exit Funds When Contract Is Paused - (File: `engine/src/contract_methods/connector.rs`)

### Summary

The `exit_to_near_precompile_callback` function in `engine/src/contract_methods/connector.rs` calls `require_running` at the very top of its body. This function is the NEAR async callback that finalises an ExitToNear operation: it either completes the NEAR transfer to the user or, if the prior promise failed, executes the refund path that re-mints the burned ERC-20/ETH tokens. If the Aurora Engine contract is paused while this callback is in-flight, `require_running` panics the callback before either branch executes, permanently destroying the user's funds with no recovery path.

The same structural defect exists in `withdraw_wnear_to_router` (`engine/src/contract_methods/xcc.rs`), which is the self-callback that moves wNEAR to the XCC router during cross-contract calls.

---

### Finding Description

**Root cause — `exit_to_near_precompile_callback`:** [1](#0-0) 

```rust

    with_hashchain(io, env, function_name!(), |io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;          // ← blocks the entire callback
        env.assert_private_call()?;
        ...
        let maybe_result = if let Some(PromiseResult::Successful(_)) = handler.promise_result(0) {

            // wNEAR transfer FAILED → refund burned tokens
            let refund_result = engine::refund_on_error(io, env, state, &args, handler)?;
            ...
        };
    })
}
```

`require_running` is defined as: [2](#0-1) 

If `state.is_paused == true`, it returns `Err(ERR_PAUSED)`, which propagates through `with_hashchain` and causes `sdk_unwrap()` in `lib.rs` to panic the NEAR receipt.

The ExitToNear flow is:
1. User calls the ExitToNear precompile from EVM — ERC-20 tokens (or ETH) are **burned** on Aurora.
2. Aurora schedules a promise to transfer wNEAR to the user's NEAR account.
3. `exit_to_near_precompile_callback` is attached as the callback.

If the contract is paused between steps 1–3 and the callback receipt arrives:
- `require_running` panics the callback.
- The **refund branch** (`engine::refund_on_error`) never executes.
- The burned tokens are gone with no on-chain recovery path.

**Secondary root cause — `withdraw_wnear_to_router`:** [3](#0-2) 

This is the self-callback in the XCC flow. The XCC precompile deducts wNEAR from the user's EVM balance before this callback runs. If the contract is paused when the callback arrives, `require_running` panics it, the wNEAR transfer to the router never occurs, and the user's wNEAR is stuck in the engine with no retry mechanism.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

For `exit_to_near_precompile_callback`: ERC-20 tokens or ETH are burned on the Aurora side before the callback is ever invoked. If the callback panics, the refund path that would re-mint those tokens is skipped. The user loses their assets permanently.

For `withdraw_wnear_to_router`: wNEAR is deducted from the user's EVM balance before the callback runs. A panic leaves the wNEAR stranded inside the engine contract with no user-accessible recovery.

---

### Likelihood Explanation

**Medium.** The contract owner can call `pause_contract` at any time (e.g., emergency response, upgrade preparation). NEAR's async execution model means that callbacks from already-scheduled promises will still arrive after the pause. Any user who had an ExitToNear or XCC operation in-flight at the moment of the pause is affected. No attacker action is required beyond the normal user flow; the pause alone is sufficient to trigger the loss.

---

### Recommendation

Remove `require_running` from all async callback functions that are responsible for completing or refunding in-flight operations:

- `exit_to_near_precompile_callback` — the refund branch must always be reachable.
- `withdraw_wnear_to_router` — the wNEAR transfer must always complete.
- `deploy_erc20_token_callback` and `mirror_erc20_token_callback` — lower severity but same structural issue.

For callbacks that must remain pause-aware, move the `require_running` check to after the refund/completion logic, or skip it entirely for callbacks that only finalise already-committed state changes.

---

### Proof of Concept

1. User calls the ExitToNear precompile from an EVM contract, burning 1000 USDC on Aurora. Aurora schedules a wNEAR transfer promise with `exit_to_near_precompile_callback` attached.
2. Before the callback receipt is processed, the contract owner calls `pause_contract`.
3. The NEAR runtime delivers the callback receipt. `exit_to_near_precompile_callback` is invoked.
4. `require_running` reads `state.is_paused == true` and returns `Err(ERR_PAUSED)`.
5. `sdk_unwrap()` panics the receipt.
6. The `refund_on_error` branch is never reached.

pub fn exit_to_near_precompile_callback<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<Option<SubmitResult>, ContractError> {
    with_hashchain(io, env, function_name!(), |io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;
        env.assert_private_call()?;

        // This function should only be called as the callback of
        // exactly one promise.
        if handler.promise_results_count() != 1 {
            return Err(errors::ERR_PROMISE_COUNT.into());
        }

        let args: ExitToNearPrecompileCallbackArgs = io.read_input_borsh()?;

        let maybe_result = if let Some(PromiseResult::Successful(_)) = handler.promise_result(0) {
            if let Some(args) = args.transfer_near {
                let action = PromiseAction::Transfer {
                    amount: Yocto::new(args.amount),
                };
                let promise = PromiseBatchAction {
                    target_account_id: args.target_account_id,
                    actions: vec![action],
                };

                // Safety: this call is safe because it comes from the exit to near precompile, not users.
                // The call is to transfer the unwrapped wNEAR tokens.
                let promise_id = handler.promise_create_batch(&promise);
                handler.promise_return(promise_id);
            }

            None
        } else if let Some(args) = args.refund {
            // Exit call failed; need to refund tokens
            let refund_result = engine::refund_on_error(io, env, state, &args, handler)?;

            if !refund_result.status.is_ok() {
                return Err(errors::ERR_REFUND_FAILURE.into());
            }

            Some(refund_result)
        } else {
            None
        };

        Ok(maybe_result)
    })
}

pub fn withdraw_wnear_to_router<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<SubmitResult, ContractError> {
    with_logs_hashchain(io, env, function_name!(), |io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;
        env.assert_private_call()?;
        if matches!(handler.promise_result_check(), Some(false)) {
            return Err(b"ERR_CALLBACK_OF_FAILED_PROMISE".into());
        }
        let args: WithdrawWnearToRouterArgs = io.read_input_borsh()?;
        let current_account_id = env.current_account_id();
        let recipient = AccountId::try_from(format!(
            "{}.{}",
            args.target.encode(),
            current_account_id.as_ref()
        ))?;
        let wnear_address = aurora_engine_precompiles::xcc::state::get_wnear_address(&io);
        let mut engine: Engine<_, E, AuroraModExp> = Engine::new_with_state(
            state,
            predecessor_address(&current_account_id),
            current_account_id,
            io,
            env,
        );
        let (result, ids) = xcc::withdraw_wnear_to_router(
            &recipient,
            args.amount,
            wnear_address,
            &mut engine,
            handler,
        )?;
        if !result.status.is_ok() {
            return Err(b"ERR_WITHDRAW_FAILED".into());
        }
        let id = ids.last().ok_or(b"ERR_NO_PROMISE_CREATED")?;
        handler.promise_return(*id);
        Ok(result)
    })
}

pub fn pause_contract<I: IO + Copy, E: Env>(io: I, env: &E) -> Result<(), ContractError> {
    with_hashchain(io, env, function_name!(), |mut io| {
        let mut state = state::get_state(&io)?;
        require_owner_only(&state, &env.predecessor_account_id())?;
        require_running(&state)?;
        state.is_paused = true;
        state::set_state(&mut io, &state)?;
        Ok(())
    })
