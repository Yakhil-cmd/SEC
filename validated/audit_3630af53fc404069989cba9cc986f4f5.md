### Title
BLS12-381 G1MSM/G2MSM Precompiles Charge Zero Gas for Non-Empty Invalid Inputs — (`File: basic_bootloader/src/bootloader/block_flow/ethereum/hooks/eip_2537/msm.rs`)

---

### Summary

The `compute_cost` helper used by both `Bls12381G1MSMPrecompile` and `Bls12381G2MSMPrecompile` returns `0` whenever `num_pairs == 0`. Because gas is charged **before** input-length validation, a caller who supplies a non-empty input that is shorter than one full pair encoding (1–159 bytes for G1MSM, 1–287 bytes for G2MSM) causes the precompile to charge **zero gas** and then immediately fail with `InvalidInputSize`. This is the direct structural analog of the reported Badger-Citadel bug: a value is only set inside a conditional branch; when the branch is not taken the value stays at its default (0), and that 0 propagates to a downstream operation that should never receive it.

---

### Finding Description

`compute_cost` is defined as:

```rust
fn compute_cost(
    input_len: usize,
    pair_len: usize,
    per_pair_cost: u64,
    discounts_table: &[u16; 128],
) -> u64 {
    let num_pairs = input_len / pair_len;
    if num_pairs == 0 {
        return 0;          // ← base case missing: non-empty but sub-pair input
    }
    let discount = if num_pairs > 128 {
        discounts_table[127]
    } else {
        discounts_table[num_pairs - 1]
    };
    (per_pair_cost * num_pairs as u64) * (discount as u64) / (DISCOUNT_DENOMINATOR as u64)
}
``` [1](#0-0) 

Both `Bls12381G1MSMPrecompile::execute` and `Bls12381G2MSMPrecompile::execute` follow the same pattern:

1. Reject empty input (`input.len() == 0` → error). ✓
2. Call `compute_cost` and charge the result — **before** validating that the length is a multiple of `pair_len`.
3. Only then check `!input.len().is_multiple_of(pair_len)` → `InvalidInputSize`. [2](#0-1) [3](#0-2) 

For any input whose byte-length is in `[1, G1_MSM_PAIR_LEN - 1]` (i.e., 1–159 bytes) for G1MSM, or `[1, G2_MSM_PAIR_LEN - 1]` (1–287 bytes) for G2MSM:

- `num_pairs = input_len / pair_len = 0`
- `compute_cost` returns `0`
- `cost_ergs = Ergs(0 * ERGS_PER_GAS) = Ergs(0)`
- `resources.charge(Ergs(0))` succeeds trivially
- The precompile then returns `InvalidInputSize`

The caller's gas budget is reduced by exactly **0** for the precompile work, yet the precompile has been invoked and has performed its input-rejection logic.

---

### Impact Explanation

**Resource accounting bug / EVM semantic mismatch.** Per EIP-2537, a non-empty input that is not a multiple of the pair encoding length is an error, but the gas cost for that error path is not zero — the caller should still pay for the attempted invocation. In ZKsync OS the precompile charges nothing. Depending on how the EVM interpreter handles the returned `SubsystemError` from a precompile (whether it burns all forwarded gas on failure, as standard EVM requires, or only burns what the precompile internally charged), an attacker can:

- **Best case for attacker (non-standard handling):** call G1MSM or G2MSM with 1–159 / 1–287 bytes of garbage input and pay zero gas per invocation, enabling free spam of precompile calls.
- **Worst case for attacker (standard EVM handling):** the CALL instruction still burns all forwarded gas on precompile failure, so the direct gas-theft is mitigated; however the internal accounting diverges from what the prover expects, creating a forward/proving divergence risk where the sequencer and prover disagree on resource consumption for this code path.

Either way the invariant "every non-trivial precompile invocation costs at least some gas" is violated, which is the same class of invariant broken in the Badger-Citadel report.

---

### Likelihood Explanation

Any unprivileged EVM transaction can call address `0x000...000b` (G1MSM) or `0x000...000c` (G2MSM) via `CALL`/`STATICCALL` with a crafted input length. No special role or privileged access is required. The triggering condition (input length in `[1, pair_len - 1]`) is trivially constructable by any caller.

---

### Recommendation

Move the input-length validation **before** the gas charge, or ensure a minimum non-zero gas cost is charged for any non-empty input regardless of whether it forms a complete pair. For example:

```rust
// Validate length before charging
if !input.len().is_multiple_of(G1_MSM_PAIR_LEN) {
    return Err(interface_error!(Bls12PrecompileInterfaceError::InvalidInputSize));
}
let cost = compute_cost(input.len(), G1_MSM_PAIR_LEN, ...);
resources.charge(...)?;
```

Alternatively, `compute_cost` should return a minimum base cost (e.g., `per_pair_cost`) when `num_pairs == 0` but `input_len > 0`, mirroring the fix pattern from the Badger-Citadel report (assign the base value unconditionally, then conditionally apply the multiplier).

---

### Proof of Concept

1. Deploy a contract that calls `STATICCALL` to address `0x000...000b` (BLS12-381 G1MSM) with 100 bytes of arbitrary input and a gas limit of 100,000.
2. Observe that the precompile returns failure but the gas consumed by the precompile itself is 0 (only the CALL overhead is charged).
3. Repeat with 1 byte, 50 bytes, 159 bytes — all produce the same 0-gas-charged-by-precompile result.
4. Compare with a valid 160-byte input: the precompile charges `floor(1 × 12000 × 1000 / 1000) = 12000` gas before returning success.
