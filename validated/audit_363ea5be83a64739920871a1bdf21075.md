### Title
Maturity Permanently Lost When Account Conversion Fails After Disbursement Queue Pop in `try_finalize_maturity_disbursement` - (File: rs/nns/governance/src/governance/disburse_maturity.rs)

### Summary

In `try_finalize_maturity_disbursement`, the neuron's maturity disbursement entry is removed from the queue (Step 2) **before** the ledger minting operation is confirmed. A failure in the intermediate `try_into_account_identifier()` conversion (Step 3) causes an early return via `?` without restoring the popped disbursement, permanently destroying the user's maturity.

### Finding Description

The function `try_finalize_maturity_disbursement` executes in three sequential steps:

**Step 2** — pops the disbursement from the neuron's queue before any ledger interaction:

```rust
let Ok(Some(maturity_disbursement_in_progress)) = governance.with_borrow_mut(|governance| {
    governance.with_neuron_mut(&neuron_id, |neuron| {
        neuron.pop_maturity_disbursement_in_progress()
    })
}) else { ... };
``` [1](#0-0) 

**Step 3** — converts the destination to an `AccountIdentifier`. If this conversion fails, the `?` operator returns early with `AccountConversionFailure`:

```rust
let account_identifier = destination
    .try_into_account_identifier()
    .map_err(|reason| FinalizeMaturityDisbursementError::AccountConversionFailure { reason })?;
``` [2](#0-1) 

The reversal logic — `push_front_maturity_disbursement_in_progress` — is only invoked after `mint_icp_with_ledger()` fails, not after `AccountConversionFailure`: [3](#0-2) 

When `AccountConversionFailure` fires, `maturity_disbursement_in_progress` is a local variable that goes out of scope and is dropped. The neuron lock is released (no `retain()` call), so the next timer invocation finds no disbursement in the queue and silently skips. The maturity is neither in the neuron's `maturity_e8s_equivalent` (deducted at initiation) nor minted to the destination.

The initiation path deducts maturity unconditionally: [4](#0-3) 

The TLA formal model (`Disburse_Maturity_Timer.tla`) does not model the `AccountConversionFailure` path — it only models transfer-level failures — leaving this code path outside the verified safety envelope: [5](#0-4) 

A secondary, acknowledged variant exists: if `mint_icp_with_ledger()` fails AND `push_front_maturity_disbursement_in_progress` also fails, the code calls `neuron_lock.retain()` and returns `FailToRestoreMaturityDisbursement`, permanently locking the neuron and losing the maturity. The TLA model explicitly models this non-deterministic outcome as a valid (if undesirable) state: [6](#0-5) 

### Impact Explanation

**Ledger conservation bug.** A neuron holder's maturity is deducted from `maturity_e8s_equivalent` at `DisburseMaturity` call time and placed in `maturity_disbursements_in_progress`. If finalization fails at the account conversion step, the maturity is permanently destroyed: it is not in the neuron's balance, not minted to the destination, and not re-queued for retry. The ICP total supply is unaffected (no mint occurred), but the individual user suffers a permanent, unrecoverable loss of their earned maturity.

### Likelihood Explanation

The `AccountConversionFailure` path fires when `Destination::try_into_account_identifier()` fails on a destination that was accepted at initiation time. The `MaturityDisbursement` proto supports both an ICRC1 `Account` field and a legacy `AccountIdentifier` field as the destination: [7](#0-6) 

If the initiation path (`Destination::try_new`) does not pre-validate that the stored destination is convertible to `AccountIdentifier` at finalization time (e.g., due to a future proto schema change, data migration, or an ICRC1 account variant that cannot round-trip), the conversion will fail at finalization and trigger the unguarded early return. Any NNS neuron holder who calls `DisburseMaturity` is exposed to this path. The trigger is a governance canister periodic task, not a direct user call, so the user has no ability to retry or intervene once the disbursement is popped.

### Recommendation

Move the `try_into_account_identifier()` conversion to **before** `pop_maturity_disbursement_in_progress()`, so that any conversion failure is detected while the disbursement is still safely in the queue. Alternatively, add a reversal call (`push_front_maturity_disbursement_in_progress`) in the `AccountConversionFailure` error arm, mirroring the pattern already used for `FailToMintIcp`. The TLA model should also be extended to cover the pre-transfer failure path.

### Proof of Concept

1. A neuron holder calls `DisburseMaturity` on an NNS neuron. `maturity_e8s_equivalent` is decremented and a `MaturityDisbursement` entry is added to `maturity_disbursements_in_progress`.
2. After the 7-day delay, the governance timer calls `finalize_maturity_disbursement` → `try_finalize_maturity_disbursement`.
3. Step 2 calls `pop_maturity_disbursement_in_progress()`, removing the entry from the queue.
4. Step 3 calls `destination.try_into_account_identifier()`. If this returns `Err`, the `?` operator returns `Err(AccountConversionFailure)` immediately.
5. `maturity_disbursement_in_progress` is dropped. The neuron lock is released normally.
6. The next timer invocation calls `next_maturity_disbursement_to_finalize`, finds no disbursement in the queue, and returns `Ok(None)`.
7. The maturity is permanently lost: not in the neuron, not minted, not re-queued.
