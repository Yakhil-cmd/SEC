Audit Report

## Title
Unbounded `CANISTER_DISPLAY_NAMES` HashMap and Prometheus Label Growth via Attacker-Controlled `network_identifier.network` Causes OOM DoS — (`rs/rosetta-api/common/rosetta_core/src/metrics.rs`)

## Summary
The `RosettaMetricsMiddleware` reads `network_identifier.network` from every incoming POST body without any length or format validation, and unconditionally inserts the attacker-supplied string into the process-global `CANISTER_DISPLAY_NAMES` HashMap and into the Prometheus default registry as a new label set. Both data structures have no size cap and no eviction. An unauthenticated attacker can exhaust the ICRC-1 Rosetta process heap by sending requests with distinct strings in that field, causing an OOM-kill and complete denial of service of the Rosetta node.

## Finding Description
`CANISTER_DISPLAY_NAMES` is declared as a process-global `lazy_static` with no bound: [1](#0-0) 

`RosettaMetrics::new` unconditionally inserts the caller-supplied `canister_id` and `token_display_name` into the map on every invocation: [2](#0-1) 

`RosettaMetricsMiddleware::call` invokes `RosettaMetrics::new` for every request that carries a `network_identifier.network` field. `get_display_name_from_canister_id` returns the ID itself when it is not already registered, so the attacker-supplied string becomes both key and value: [3](#0-2) 

`extract_canister_id` performs no length or character validation — it accepts any JSON string of arbitrary length: [4](#0-3) 

The body is read with `usize::MAX` as the limit, meaning there is no effective request body size cap in the ICRC-1 Axum server: [5](#0-4) 

Every unique `token_display_name` also creates a new Prometheus time-series label set in the default registry (which has no eviction) via `inc_api_status_count` and `observe_request_duration`: [6](#0-5) [7](#0-6) 

The metrics layer is applied to all routes, and the server binds to `0.0.0.0` with no authentication: [8](#0-7) 

There are no existing guards: no entry count check before `map.insert`, no principal-format validation on the extracted string, and no body size limit in the Axum server path.

## Impact Explanation
This is a High-severity application/platform-level DoS. The ICRC-1 Rosetta node is an in-scope financial integration component. Exhausting its heap causes the OS OOM-killer to terminate the process, making the Rosetta API completely unavailable — users cannot query balances, submit transactions, or interact with ICRC-1 ledgers through this node. The impact is concrete and non-hypothetical: heap growth is proportional to the number of unique strings sent, and both the HashMap and the Prometheus registry retain entries permanently for the lifetime of the process.

## Likelihood Explanation
The attack requires only unauthenticated HTTP POST access to any Rosetta endpoint (e.g., `/network/status`). No credentials, no privileged role, and no protocol-level access are needed. The default bind address is `0.0.0.0:8080`, making the port reachable from any network-adjacent host. The attack is trivially scriptable with standard HTTP tooling and requires no special knowledge of the IC protocol.

## Recommendation
1. **Cap `CANISTER_DISPLAY_NAMES`**: enforce a maximum entry count (e.g., 1,000) and silently drop insertions beyond that limit.
2. **Validate the extracted canister ID**: accept only strings matching the canonical IC principal text format (e.g., `[a-z0-9]{5}(-[a-z0-9]{5}){4}`) before inserting into the map or using as a Prometheus label.
3. **Do not construct new `RosettaMetrics` instances per request for unknown IDs**: the middleware should fall back to `default_metrics` when the extracted ID is not already registered, rather than calling `RosettaMetrics::new` with an unrecognised attacker-supplied string.
4. **Apply a request body size limit** in the ICRC-1 Axum server (replace `usize::MAX` with a reasonable cap such as 4 MB, consistent with the ICP Actix server).

## Proof of Concept
```python
import requests, string, random

TARGET = "http://<rosetta-host>:8080/network/status"

for i in range(100_000):
    unique_id = ''.join(random.choices(string.ascii_lowercase, k=64)) + str(i)
    requests.post(TARGET, json={
        "network_identifier": {"blockchain": "Internet Computer", "network": unique_id},
        "metadata": {}
    })
    if i % 1000 == 0:
        print(f"Sent {i} requests")
# Monitor RSS of the rosetta process: grows proportionally to i
```

```rust
for i in 0..N {

}
```
