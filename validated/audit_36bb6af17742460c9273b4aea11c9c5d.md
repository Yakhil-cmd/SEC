Audit Report

## Title
`ManageLedgerParameters` Proposal Action Not Restricted During `PreInitializationSwap` Mode - (File: `rs/sns/governance/src/types.rs`)

## Summary
`functions_disallowed_in_pre_initialization_swap()` in `rs/sns/governance/src/types.rs` enumerates six blocked proposal actions during `PreInitializationSwap` mode but omits `ManageLedgerParameters` (Id = 13). Because developer neurons hold 100% of voting power before any swap participant receives tokens, a malicious SNS developer can unilaterally pass a `ManageLedgerParameters` proposal mid-swap to raise the SNS ledger's `transfer_fee`, causing every SNS token transfer during finalization to be charged the inflated fee and silently reducing participant allocations.

## Finding Description
`functions_disallowed_in_pre_initialization_swap()` at `rs/sns/governance/src/types.rs` lines 253–262 lists exactly six blocked actions:

```rust
pub fn functions_disallowed_in_pre_initialization_swap() -> Vec<NervousSystemFunction> {
    vec![
        NervousSystemFunction::manage_nervous_system_parameters(),
        NervousSystemFunction::transfer_sns_treasury_funds(),
        NervousSystemFunction::mint_sns_tokens(),
        NervousSystemFunction::upgrade_sns_controlled_canister(),
        NervousSystemFunction::register_dapp_canisters(),
        NervousSystemFunction::deregister_dapp_canisters(),
    ]
}
```

`ManageLedgerParameters` is absent. The gating function `proposal_action_is_allowed_in_pre_initialization_swap_or_err` (lines 264–298) performs a simple membership check and returns `Ok(())` for every action not present in that list — including `ManageLedgerParameters`. There is no secondary lifecycle guard in `perform_manage_ledger_parameters` (called at governance.rs lines 2213–2216). The `PreInitializationSwap` mode persists from SNS creation until the swap canister calls `set_mode(Normal)` after successful finalization (governance.rs lines 785–801). During this entire window, developer neurons hold 100% of voting power, enabling unilateral proposal adoption.

## Impact Explanation
This is a **High** severity finding matching: *"Significant SNS security impact with concrete user or protocol harm."* Swap participants who have committed ICP cannot withdraw once the swap reaches `LIFECYCLE_COMMITTED`. A developer raising `transfer_fee` to an arbitrarily large value before finalization causes every SNS token distribution transfer to be charged the inflated fee, directly reducing the net tokens credited to each participant. The developer's genesis allocation is unaffected. This is a concrete, quantifiable loss of funds for swap participants enabled by an incomplete security denylist in a protection mechanism that exists specifically to guard against developer abuse during the swap window.

## Likelihood Explanation
The attack requires only that the attacker hold developer neurons — a condition that is true by definition for any SNS creator. No external collusion, special capability, or victim mistake is required. The attack window spans the entire `PreInitializationSwap` period. The `ManageLedgerParameters` action is a standard ingress call path (`manage_neuron` → `MakeProposal`) available on all mainnet SNS instances. Developer neurons hold 100% of voting power during this window, so the proposal self-adopts immediately.

## Recommendation
Add `NervousSystemFunction::manage_ledger_parameters()` to the `functions_disallowed_in_pre_initialization_swap()` vector in `rs/sns/governance/src/types.rs`:

```rust
pub fn functions_disallowed_in_pre_initialization_swap() -> Vec<NervousSystemFunction> {
    vec![
        NervousSystemFunction::manage_nervous_system_parameters(),
        NervousSystemFunction::transfer_sns_treasury_funds(),
        NervousSystemFunction::mint_sns_tokens(),
        NervousSystemFunction::upgrade_sns_controlled_canister(),
        NervousSystemFunction::register_dapp_canisters(),
        NervousSystemFunction::deregister_dapp_canisters(),
        NervousSystemFunction::manage_ledger_parameters(), // ADD THIS
    ]
}
```

Additionally audit `AdvanceSnsTargetVersion` (Id = 15) and `UpgradeSnsToNextVersion` (Id = 7) for the same omission.

## Proof of Concept
1. Deploy an SNS via NNS `CreateServiceNervousSystem`; governance enters `PreInitializationSwap` mode.
2. Open the swap (`LIFECYCLE_OPEN`); participants transfer ICP.
3. Developer calls `manage_neuron` → `MakeProposal` with `Action::ManageLedgerParameters { transfer_fee: Some(100_000_000) }`.
4. Developer's neuron self-adopts (100% voting power); `perform_manage_ledger_parameters` executes with no lifecycle guard, updating the SNS ledger fee.
5. Swap reaches `LIFECYCLE_COMMITTED`; `finalize` distributes SNS tokens — each basket-neuron transfer is charged 100 SNS tokens, silently reducing every participant's allocation.
6. Developer's genesis neurons are unaffected; participants receive materially fewer tokens than swap parameters implied.

A deterministic integration test can be written using PocketIC: deploy SNS, open swap, submit and adopt `ManageLedgerParameters` as developer neuron, commit swap, assert participant neuron balances are less than expected by the inflated fee amount.
