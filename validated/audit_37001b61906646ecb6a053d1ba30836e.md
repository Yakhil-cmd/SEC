### Title
Integer Division Truncation in XCC EVM Gas Cost Calculation Allows Free NEAR Gas Attachment - (`engine-precompiles/src/xcc.rs`)

### Summary

The XCC precompile charges EVM gas for NEAR gas attached to cross-contract call promises using integer division. Because the divisor (`CROSS_CONTRACT_CALL_NEAR_GAS = 175_000_000`) is large, any `attached_gas` value below 175,000,000 NEAR gas truncates to zero EVM gas charged. An unprivileged EVM user can craft XCC calls that attach up to 174,999,999 NEAR gas per promise at zero additional EVM gas cost, systematically under-paying for the NEAR computation they consume.

### Finding Description

In `engine-precompiles/src/xcc.rs` line 174, the EVM gas cost for the NEAR gas attached to a promise is computed as:

```rust
cost += EthGas::new(promise.attached_gas.as_u64() / costs::CROSS_CONTRACT_CALL_NEAR_GAS);
``` [1](#0-0) 

The constant `CROSS_CONTRACT_CALL_NEAR_GAS` is defined as `175_000_000`: [2](#0-1) 

This constant represents the ratio `NEAR Gas / EVM Gas`. Integer division in Rust truncates toward zero. Therefore:

```
attached_gas = 174_999_999
EVM cost = 174_999_999 / 175_000_000 = 0
```

Any `attached_gas` value in the range `[1, 174_999_999]` produces an EVM gas cost of **zero** for the NEAR gas component. The user still pays the base cost (`CROSS_CONTRACT_CALL_BASE = 343_650` EVM gas) and the per-byte cost, but pays nothing for attaching up to ~0.175 TGas of NEAR gas to the promise. [3](#0-2) 

The `attached_gas` field is fully attacker-controlled: it is deserialized directly from the calldata passed to the XCC precompile: [4](#0-3) 

### Impact Explanation

The NEAR gas attached to the promise is consumed from the NEAR transaction's gas budget, which is paid by the relayer in NEAR tokens. The EVM gas fee collected from the user is supposed to compensate the relayer for this cost. When the EVM gas charge truncates to zero, the relayer subsidizes the user's NEAR gas consumption without receiving compensation. A user who repeatedly submits XCC calls with `attached_gas` just below the divisor threshold (e.g., `174_999_999`) extracts NEAR gas computation at no EVM cost, constituting systematic theft of relayer fee income — **High: Theft of unclaimed yield**.

### Likelihood Explanation

The attack requires only the ability to submit an EVM transaction calling the XCC precompile with a crafted `attached_gas` value. No privileged access, governance capture, or external oracle is needed. The XCC precompile is a standard production interface reachable by any EVM user. The threshold (174,999,999 NEAR gas ≈ 0.175 TGas) is large enough to be practically useful for real cross-contract calls, making exploitation straightforward and repeatable.

### Recommendation

Multiply before dividing to preserve precision, or use ceiling division so that any non-zero NEAR gas attachment always costs at least 1 EVM gas:

```rust
// Ceiling division: charges at least 1 EVM gas for any non-zero attached_gas
let near_gas = promise.attached_gas.as_u64();
let near_gas_cost = near_gas
    .checked_add(costs::CROSS_CONTRACT_CALL_NEAR_GAS - 1)
    .unwrap_or(u64::MAX)
    / costs::CROSS_CONTRACT_CALL_NEAR_GAS;
cost += EthGas::new(near_gas_cost);
```

This mirrors the fix recommended in the reference report: increase precision so that rounding cannot cause a non-trivial quantity to appear as zero.

### Proof of Concept

1. Construct a `CrossContractCallArgs::Eager` payload with `attached_gas = NearGas::new(174_999_999)`.
2. Submit it as an EVM transaction to the XCC precompile address (`0x516cded1d16af10cad47d6d49128e2eb7d27b372`).
3. Observe that `cost` after line 174 equals `CROSS_CONTRACT_CALL_BASE + CROSS_CONTRACT_CALL_BYTE * input_len` — the NEAR gas component contributes **0** EVM gas.
4. Compare with `attached_gas = NearGas::new(175_000_000)`, which correctly adds 1 EVM gas.
5. Repeat the call in a loop: each iteration attaches 174,999,999 NEAR gas (~0.175 TGas) at zero marginal EVM gas cost, while the relayer bears the full NEAR gas expense. [1](#0-0)
