### Title
Predictable `PREVRANDAO` derived solely from `block.timestamp` enables randomness manipulation in EVM contracts — (`x/evm/keeper/keeper.go`, `giga/deps/xevm/keeper/keeper.go`)

---

### Summary

Sei's EVM implementation sets the `PREVRANDAO` opcode value (`vm.BlockContext.Random`) to `keccak256(block.timestamp.MarshalBinary())`. Because this value is a deterministic, one-input hash of the block timestamp — a value that is publicly known the moment a block is proposed — any EVM contract on Sei that uses `block.prevrandao` as a source of randomness is trivially exploitable by an unprivileged attacker.

---

### Finding Description

In `GetVMBlockContext`, both in the main EVM keeper and the giga EVM keeper, the `Random` field of the EVM `BlockContext` is constructed as:

```go
// Use hash of block timestamp as info for PREVRANDAO
r, err := ctx.BlockHeader().Time.MarshalBinary()
rh := crypto.Keccak256Hash(r)
// ...
Random: &rh,
``` [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) 

The `PREVRANDAO` opcode was introduced post-Merge on Ethereum to replace `DIFFICULTY` as a source of randomness, backed by the beacon chain's RANDAO which is cryptographically unpredictable. On Sei, there is no beacon chain, so the chain substitutes `keccak256(block.timestamp)` instead. This is a single-input, fully deterministic function: once the block timestamp is known (which happens the moment the block is proposed and propagated), the `PREVRANDAO` value for that block is computable by anyone.

The same `block.timestamp` is also exposed directly as `Time` in the same `BlockContext`:

```go
Time: uint64(ctx.BlockHeader().Time.Unix()),
``` [5](#0-4) 

So `block.prevrandao == keccak256(abi.encode(block.timestamp))` (modulo binary encoding), meaning the two values are trivially correlated and neither provides independent entropy.

---

### Impact Explanation

Any EVM smart contract deployed on Sei that uses `block.prevrandao` as a randomness source (lotteries, NFT trait rolls, random winner selection, etc.) is vulnerable. An attacker can:

1. **Predict the outcome** before submitting their transaction: once the block is proposed and the timestamp is visible, compute `keccak256(timestamp.MarshalBinary())` off-chain and determine the result before the block is finalized.
2. **Selectively participate**: only submit a transaction (e.g., enter a lottery, mint an NFT) when the precomputed `PREVRANDAO` produces a favorable outcome.
3. **Validator-level manipulation**: a block proposer can try multiple valid timestamps (within the consensus-allowed drift window) to find one that produces a desired `PREVRANDAO` value, then propose that block.

The root cause is in sei-chain itself — the chain is the necessary vulnerable step, as it is the entity that defines what `PREVRANDAO` returns. Contracts that correctly use `block.prevrandao` per the EIP-4399 specification would be secure on Ethereum but are insecure on Sei due to this implementation.

Fund loss from a single exploited lottery or NFT mint can easily exceed $5k, placing this at **Critical** severity.

---

### Likelihood Explanation

- The `PREVRANDAO` opcode is widely documented as the recommended randomness source for EVM contracts post-Merge. Developers porting Ethereum contracts to Sei will naturally use it.
- The attack requires no special privileges: any user can observe the block timestamp and compute the hash before their transaction is included.
- The computation is trivial (a single `keccak256` call).
- The vulnerability is present in both the main EVM keeper and the giga EVM keeper, covering all EVM execution paths on Sei.

---

### Recommendation

Replace the timestamp-only derivation with a value that incorporates unpredictable entropy. Options include:

1. **Use the block hash of the previous block** (or a combination of previous block hash + timestamp + block height), which is not known to the proposer when constructing the current block.
2. **Incorporate the Tendermint/CometBFT commit hash or last-commit hash** from the block header, which aggregates validator signatures and is not manipulable by a single party.
3. **Document clearly** that `block.prevrandao` on Sei does not provide beacon-chain-grade randomness, so contract developers are not misled by the opcode name.

---

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

interface ILottery {
    function enter() external payable;
    function draw() external;
    function winner() external view returns (address);
}

contract SeiPrevrandaoExploit {
    // Attacker precomputes block.prevrandao off-chain:
    //   r, _ = time.Now().MarshalBinary()   // approximate next block timestamp
    //   prevrandao = keccak256(r)
    // Then checks if prevrandao % numParticipants == attackerIndex
    // If yes, submits enter() in that block; otherwise waits.

    function predictPrevrandao(uint256 blockTimestamp) external pure returns (bytes32) {
        // Mirrors: ctx.BlockHeader().Time.MarshalBinary() -> crypto.Keccak256Hash(r)
        // (simplified; actual Go time.MarshalBinary encoding adds version prefix)
        return keccak256(abi.encodePacked(blockTimestamp));
    }
}
```

An attacker monitors the mempool for the next block's proposed timestamp, computes `keccak256(timestamp.MarshalBinary())`, checks whether the resulting `PREVRANDAO` value would make them the winner of a target lottery contract, and only submits their entry transaction when the answer is yes. func (k *Keeper) GetVMBlockContext(ctx sdk.Context, gp core.GasPool) (*vm.BlockContext, error) {
	if k.EthBlockTestConfig.Enabled {
		return k.getBlockTestBlockCtx(ctx)
	}
	if k.EthReplayConfig.Enabled {
		return k.getReplayBlockCtx(ctx)
	}
	coinbase, err := k.GetFeeCollectorAddress(ctx)
	if err != nil {
		return nil, err
	}

	// Use hash of block timestamp as info for PREVRANDAO
	r, err := ctx.BlockHeader().Time.MarshalBinary()
	if err != nil {
		return nil, err
	}
	rh := crypto.Keccak256Hash(r)

	txfer := func(db vm.StateDB, sender, recipient common.Address, amount *uint256.Int) {
		if IsPayablePrecompile(&recipient) {
			state.TransferWithoutEvents(db, sender, recipient, amount)
		} else {
			core.Transfer(db, sender, recipient, amount)
		}
	}
	var baseFee *big.Int
	if ctx.ChainID() == Pacific1ChainID && ctx.BlockHeight() < 114945913 {
		baseFee = k.GetBaseFeePerGas(ctx).TruncateInt().BigInt()
	} else {
		baseFee = k.GetNextBaseFeePerGas(ctx).TruncateInt().BigInt()
	}

	return &vm.BlockContext{
		CanTransfer: core.CanTransfer,
		Transfer:    txfer,
		GetHash:     k.GetHashFn(ctx),
		Coinbase:    coinbase,
		GasLimit: func() uint64 {
			if ctx.ConsensusParams() != nil && ctx.ConsensusParams().Block != nil {
				return uint64(ctx.ConsensusParams().Block.MaxGas) //nolint:gosec
			}
			return DefaultBlockGasLimit
		}(),
		BlockNumber: big.NewInt(ctx.BlockHeight()),
		Time:        uint64(ctx.BlockHeader().Time.Unix()), //nolint:gosec
		Difficulty:  utils.Big0,                            // only needed for PoW
		BaseFee:     baseFee,
		BlobBaseFee: utils.Big1, // Cancun not enabled
		Random:      &rh,
	}, nil
}
