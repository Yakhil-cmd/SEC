### Title
Stale Maturity Modulation Applied to Disbursements Without Freshness Validation - (File: `rs/sns/governance/src/governance.rs`)

### Summary
The SNS governance canister caches the maturity modulation value fetched from the Cycles Minting Canister (CMC) in `proto.maturity_modulation`, which stores both `current_basis_points` and `updated_at_timestamp_seconds`. The function `effective_maturity_modulation_basis_points()` returns the cached value without ever checking whether `updated_at_timestamp_seconds` is within a reasonable timeframe. If the CMC is persistently unavailable, the stale modulation is silently applied to all maturity disbursements indefinitely, directly affecting the ICP amount minted to users.

### Finding Description

`effective_maturity_modulation_basis_points()` in `rs/sns/governance/src/governance.rs` reads `self.maturity_modulation.current_basis_points` and returns it as long as it is `Some`, with no check on `updated_at_timestamp_seconds`:

```rust
self.maturity_modulation
    .as_ref()
    .and_then(|maturity_modulation| maturity_modulation.current_basis_points)
    .ok_or_else(|| { ... })
``` [1](#0-0) 

The refresh path is `update_maturity_modulation()`, which calls `self.cmc.neuron_maturity_modulation().await` and silently returns on any error, leaving the cached value unchanged:

```rust
let Ok(maturity_modulation) = maturity_modulation else {
    return;
};
``` [2](#0-1) 

`should_update_maturity_modulation()` only checks whether the age of the stored value exceeds `ONE_DAY_SECONDS`, but if the CMC call fails, the timestamp is never updated and the stale value persists across all future disbursement rounds: [3](#0-2) 

The stale value is then consumed by `maybe_finalize_disburse_maturity()`, which applies it to every pending disbursement via `apply_maturity_modulation()`: [4](#0-3) 

The `MaturityModulation` proto explicitly carries `updated_at_timestamp_seconds` for exactly this purpose, but it is never consulted at the consumption site: [5](#0-4) 

### Impact Explanation

`apply_maturity_modulation()` scales the disbursed ICP amount by `1 + current_basis_points / 10_000`. The allowed range is −500 to +500 basis points (−5 % to +5 %). If the cached value is stale and the real ICP/XDR price has moved significantly, every neuron owner who finalizes a maturity disbursement receives a systematically incorrect ICP amount — either overpaid (protocol loss) or underpaid (user loss) — for as long as the CMC remains unreachable. Because disbursements are irreversible ledger mints, the error cannot be corrected after the fact. [6](#0-5) 

### Likelihood Explanation

The CMC is a core NNS canister and is generally available, but transient unavailability (upgrade windows, subnet issues) is a known operational reality. When `update_maturity_modulation()` fails, it returns silently without advancing `updated_at_timestamp_seconds`, so the next periodic-task invocation will again attempt a refresh but, if the CMC is still down, will again fail silently. There is no upper bound on how stale the cached value can become; a multi-day CMC outage would leave every SNS canister applying a potentially weeks-old modulation to all disbursements that mature during that window.

### Recommendation

Add a maximum-age guard inside `effective_maturity_modulation_basis_points()`. If `updated_at_timestamp_seconds` is absent or older than a configurable threshold (e.g., 3 days), return `Err` with an `Unavailable` error rather than silently applying the stale value. This mirrors the fix applied in the referenced TWAP oracle report: validate freshness at the point of consumption, not only at the point of ingestion.

```rust
fn effective_maturity_modulation_basis_points(&self) -> Result<i32, GovernanceError> {
    // ... existing disabled check ...
    let mm = self.maturity_modulation.as_ref()
        .ok_or_else(|| GovernanceError::new_with_message(...))?;

    // NEW: staleness guard
    let age = now_seconds.saturating_sub(
        mm.updated_at_timestamp_seconds.unwrap_or(0));
    if age > MAX_MATURITY_MODULATION_AGE_SECONDS {
        return Err(GovernanceError::new_with_message(
            ErrorType::Unavailable,
            "Maturity modulation is stale; retrying later."));
    }

    mm.current_basis_points.ok_or_else(|| ...)
}
```

### Proof of Concept

1. An SNS governance canister successfully fetches maturity modulation from CMC at time T and stores `current_basis_points = 300` with `updated_at_timestamp_seconds = T`.
2. The CMC becomes unavailable (e.g., subnet upgrade). Every subsequent call to `update_maturity_modulation()` returns early at the `let Ok(...) else { return; }` branch.
3. `should_update_maturity_modulation()` continues returning `true` (age ≥ `ONE_DAY_SECONDS`), but the refresh always fails silently.
4. At time T + N days, a neuron owner's disbursement matures. `maybe_finalize_disburse_maturity()` calls `effective_maturity_modulation_basis_points()`, which returns the N-day-old value `300` with no staleness check.
5. `apply_maturity_modulation()` mints `amount * 1.03` ICP to the user, even though the real ICP/XDR price may have dropped significantly, causing a protocol-level overpayment (or underpayment if the stale value is negative). [7](#0-6) [8](#0-7)

pub fn apply_maturity_modulation(
    amount_maturity_e8s: u64,
    maturity_modulation_basis_points: i32,
) -> Result<u64, String> {
    let amount_e8s = u128::from(amount_maturity_e8s);

    let adjusted_maturity_modulation_basis_points = saturating_add_or_subtract_u128_i32(
        BASIS_POINTS_PER_UNITY,
        maturity_modulation_basis_points,
    );

    let modulated_amount_e8s: u128 = amount_e8s
        .checked_mul(adjusted_maturity_modulation_basis_points)
        .ok_or_else(|| "Underflow or overflow when calculating maturity modulation".to_string())?
        .checked_div(BASIS_POINTS_PER_UNITY)
        .ok_or_else(|| "Underflow or overflow when calculating maturity modulation".to_string())?;

    u64::try_from(modulated_amount_e8s).map_err(|err| err.to_string())
}
