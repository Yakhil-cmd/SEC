Audit Report

## Title
SNS Tokens Permanently Locked in Swap Canister After Aborted Swap — (`rs/sns/swap/src/swap.rs`, `rs/sns/swap/canister/canister.rs`)

## Summary
When an SNS swap reaches the `ABORTED` lifecycle state, `finalize_inner` executes `sweep_icp` to refund ICP to buyers and then returns early after restoring dapp controllers, never invoking `sweep_sns` or any equivalent SNS token recovery step. No function on the swap canister can transfer SNS tokens back to the SNS treasury after this point. The SNS tokens pre-loaded into the swap canister for the swap distribution are permanently irrecoverable without an NNS-level canister upgrade.

## Finding Description

**Root cause — early return in ABORTED path.** `should_restore_dapp_control()` returns `true` exactly when `lifecycle() == Lifecycle::Aborted`: [1](#0-0) 

Inside `finalize_inner`, this condition triggers an early return after restoring dapp controllers, before `sweep_sns` is ever reached: [2](#0-1) 

`sweep_sns` itself enforces a hard guard that rejects any call when the lifecycle is not `COMMITTED`: [3](#0-2) 

**No recovery endpoint exists.** The canister exposes `error_refund_icp` for ICP sent in error, but no analogous function for SNS tokens: [4](#0-3) 

A grep across all of `rs/sns/swap/` for `error_refund_sns`, `sweep_sns_on_abort`, `refund_sns`, and `return_sns` returns zero matches.

**Integration test confirms and documents the stuck state.** The lifecycle integration test explicitly asserts that the full `swap_distribution_sns_e8s` balance remains in the swap canister after an abort: [5](#0-4) 

**Protocol documentation contradicts the implementation.** The proto comment states: *"The `swap` canister can be deleted when all tokens registered with the `swap` canister have been disbursed to their rightful owners."* This implies SNS tokens should be returned, but the code provides no mechanism to do so: [6](#0-5) 

**Exploit path.** The `finalize_swap` endpoint carries no caller check: [7](#0-6) 

Any principal can call it once the swap is in `ABORTED` state, completing finalization and making the stuck state permanent. Even without an external caller, the auto-finalization path (`run_periodic_tasks` → `can_auto_finalize`) will trigger the same code path automatically.

## Impact Explanation

Every SNS swap that fails to meet its minimum participation threshold aborts. The SNS tokens allocated for the swap distribution (`swap_distribution_sns_e8s`) remain permanently locked in the swap canister's SNS ledger account. They cannot be redistributed, burned, or returned to the SNS governance treasury without an NNS-level canister upgrade — a governance action that is not guaranteed and may not occur. This constitutes a permanent, concrete loss of SNS token assets for every aborted SNS launch, matching the allowed impact: **Significant SNS security impact with concrete user or protocol harm (High, $2,000–$10,000)**. For large SNS launches where the swap distribution represents millions of dollars in token value, the impact could reach the Critical threshold.

## Likelihood Explanation

Swap abortion is a normal, expected outcome for any SNS that fails to reach `min_participants` or `min_icp_e8s` before the deadline. No attacker action is required to reach the `ABORTED` state — it occurs automatically via the heartbeat. Once aborted, finalization (which locks the stuck state) is triggered either automatically (if `should_auto_finalize` is set) or by any unprivileged caller. The preconditions require no special privileges, no tokens, and no prior relationship with the SNS. Every aborted SNS swap is affected.

## Recommendation

1. Add a `sweep_sns_on_abort` step inside `finalize_inner` for the `ABORTED` branch, immediately before or after `restore_dapp_controllers_for_finalize`, that transfers all remaining SNS tokens from the swap canister's default account back to the SNS governance treasury subaccount (mirroring the `sweep_icp` refund logic).
2. Remove or relax the `Lifecycle::Committed`-only guard in `sweep_sns`, or add a separate `return_sns_to_treasury` function callable in the `ABORTED` state.
3. Add a post-finalization invariant check asserting that the swap canister's SNS token balance is zero before considering finalization complete, to catch regressions.

## Proof of Concept

1. Deploy an SNS with `swap_distribution_sns_e8s = N` tokens loaded into the swap canister (PENDING → OPEN transition).
2. Allow the swap to expire without reaching `min_participants` or `min_icp_e8s`. The heartbeat calls `try_abort`, setting `lifecycle = ABORTED`.
3. Call `finalize_swap({})` from any principal (or wait for auto-finalization).
4. `finalize_inner` executes: `sweep_icp` refunds all ICP; `settle_neurons_fund_participation` notifies NNS; `should_restore_dapp_control()` returns `true`; `restore_dapp_controllers_for_finalize` runs; function returns early.
5. `sweep_sns` is never called. Query the SNS ledger: `icrc1_balance_of(swap_canister_id, None)` returns `N` (the full `swap_distribution_sns_e8s`).
6. Attempt any recovery: no `error_refund_sns`, no `sweep_sns` (blocked by `Lifecycle::Committed` guard), no admin endpoint. The `N` SNS tokens are permanently locked.
7. This is directly reproducible using the existing PocketIC integration test framework in `rs/nervous_system/integration_tests/tests/sns_lifecycle.rs` — the `SwapFinalizationStatus::Aborted` branch already asserts this exact stuck state at line 1338.
