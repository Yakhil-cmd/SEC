### Title
Missing Subgroup Check in `Bls12381G2AdditionPrecompile` Allows Non-Subgroup G2 Points - (`basic_bootloader/src/bootloader/block_flow/ethereum/hooks/eip_2537/addition.rs`)

---

### Summary

`Bls12381G2AdditionPrecompile::execute` calls `parse_g2`, which only verifies that input points lie on the curve E'(Fq2) but never checks r-order subgroup membership. A purpose-built `parse_g2_with_subgroup_check` helper exists and is correctly used by G2MSM and Pairing, but is absent from G2ADD. This directly violates EIP-2537, which mandates rejection of any input point not in the prime-order subgroup.

---

### Finding Description

`Bls12381G2AdditionPrecompile::execute` parses both input points with `parse_g2`: [1](#0-0) 

`parse_g2` delegates to `parse_g2_bytes`, which constructs the point with `new_unchecked` and only calls `is_on_curve()`: [2](#0-1) 

The subgroup check is never invoked. By contrast, `parse_g2_with_subgroup_check` — which calls `is_in_correct_subgroup_assuming_on_curve()` — exists in the same module: [3](#0-2) 

And is correctly used by both G2MSM and Pairing: [4](#0-3) [5](#0-4) 

The `PointNotInSubgroup` error variant is defined in the subsystem but never reachable from G2ADD: [6](#0-5) 

BLS12-381's G2 has cofactor `h ≈ 2^383`, so the curve E'(Fq2) contains a large number of points outside the r-order subgroup. `is_in_correct_subgroup_assuming_on_curve` uses the efficient endomorphism-based check from §4 of [2021/1130](https://eprint.iacr.org/2021/1130): [7](#0-6) 

---

### Impact Explanation

An unprivileged caller sends a transaction calling `BLS12_G2ADD` with a point `P` on E'(Fq2) but of order `h*r` (not in G2). The precompile returns `Ok(())` and writes a result that is also outside the r-torsion subgroup. A ZK proof is generated for this state transition. Any downstream consumer (e.g., a smart contract performing BLS signature verification by chaining G2ADD outputs into a pairing check) that assumes G2ADD outputs are in G2 will operate on invalid group elements. The proving system accepts the execution, making the invalid state transition indistinguishable from a valid one — this is the "invalid provable execution" impact class.

---

### Likelihood Explanation

The exploit requires only a standard EVM call to address `BLS12_G2ADD` with 512 bytes of crafted calldata. No privilege, key, or governance access is needed. Constructing a point on E'(Fq2) outside G2 is straightforward: take any generator of E'(Fq2) and multiply by `r` to get a cofactor-order point, or use a known test vector. The call costs only 600 gas.

---

### Recommendation

Replace both `parse_g2` calls in `Bls12381G2AdditionPrecompile::execute` with `parse_g2_with_subgroup_check`, mirroring the pattern already used in G2MSM and Pairing:

```rust
// addition.rs, inside Bls12381G2AdditionPrecompile::execute
let p0 = parse_g2_with_subgroup_check(input[0..G2_SERIALIZATION_LEN].try_into().unwrap())?;
let p1 = parse_g2_with_subgroup_check(
    input[G2_SERIALIZATION_LEN..(2 * G2_SERIALIZATION_LEN)].try_into().unwrap(),
)?;
```

Apply the same fix to `Bls12381G1AdditionPrecompile::execute` for G1ADD (same pattern, `parse_g1` → `parse_g1_with_subgroup_check`), as G1 has cofactor 1 but consistency with the spec is still required.

---

### Proof of Concept

1. Find a point `P` on E'(Fq2) with `P.is_on_curve() == true` but `P.is_in_correct_subgroup_assuming_on_curve() == false`. This can be done by taking the standard G2 generator and multiplying by the cofactor inverse to get a point of order `h*r`, then verifying subgroup membership fails.
2. Encode `P` as 256 bytes per EIP-2537 (uncompressed, big-endian Fq2 coordinates, 16-byte zero padding per field element).
3. Encode the G2 identity point (512 zero bytes) as the second argument.
4. Call `BLS12_G2ADD` with the 512-byte concatenation.
5. Observe: the call returns success and outputs 256 bytes encoding `P`.
6. Verify: `output_point.is_on_curve() == true`, `output_point.is_in_correct_subgroup_assuming_on_curve() == false`.
7. A proof is generated for this execution — the circuit accepts a state transition that EIP-2537 requires to be rejected.

pub fn parse_g2_bytes(input: &[u8; G2_LEN]) -> Option<(G2Affine, bool)> {
    if input.iter().all(|el| *el == 0) {
        return Some((G2Affine::identity(), false));
    }
    let x = parse_fq2_bytes(input[0..128].try_into().ok()?)?;
    let y = parse_fq2_bytes(input[128..256].try_into().ok()?)?;
    let point = G2Affine::new_unchecked(x, y);

    if !point.is_on_curve() {
        return None;
    }

    Some((point, true))

fn parse_g2_with_subgroup_check(
    input: &[u8; G2_SERIALIZATION_LEN],
) -> Result<G2Affine, Bls12PrecompileSubsystemError> {
    let point = parse_g2(input)?;
    if point.is_zero() || point.is_in_correct_subgroup_assuming_on_curve() {
        Ok(point)
    } else {
        Err(interface_error!(
            Bls12PrecompileInterfaceError::PointNotInSubgroup
        ))
    }
}

    fn is_in_correct_subgroup_assuming_on_curve(point: &G2Affine) -> bool {
        // Algorithm from Section 4 of https://eprint.iacr.org/2021/1130.
        //
        // Checks that [p]P = [X]P

        let mut x_times_point = point.mul_bigint(crate::bls12_381::curves::Config::X);
        if crate::bls12_381::curves::Config::X_IS_NEGATIVE {
            x_times_point = -x_times_point;
        }

        let p_times_point = p_power_endomorphism(point);

        x_times_point.eq(&p_times_point)
    }
