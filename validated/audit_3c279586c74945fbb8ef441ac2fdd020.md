Audit Report

## Title
Unprivileged Caller Can Dilute Neuron Age Bonus via `ClaimOrRefresh` Stake Refresh - (`File: rs/nns/governance/src/governance.rs`)

## Summary

`refresh_neuron` in NNS governance (and its SNS counterpart) performs no authorization check on the caller. Any principal can transfer a small amount of ICP to a victim's neuron subaccount and then call `manage_neuron` with `ClaimOrRefresh { by: NeuronIdOrSubaccount }` to force `update_stake_adjust_age` to execute, permanently diluting the neuron's `aging_since_timestamp_seconds` and reducing its age bonus on voting power and voting rewards. The attack is repeatable at a cost of approximately one ICP transfer fee per iteration.

## Finding Description

**Call path (NNS):**

`manage_neuron` → `manage_neuron_internal` dispatches `Command::ClaimOrRefresh` with `By::NeuronIdOrSubaccount` directly to `refresh_neuron_by_id_or_subaccount` without passing or checking the caller: [1](#0-0) 

`refresh_neuron_by_id_or_subaccount` resolves the neuron and calls `refresh_neuron` — no caller parameter is accepted or checked: [2](#0-1) 

`refresh_neuron` reads the ledger balance and, when `balance > cached_stake` (`Ordering::Less` branch), unconditionally calls `update_stake_adjust_age`: [3](#0-2) 

`update_stake_adjust_age` computes a weighted-average age via `combine_aged_stakes`, reducing `aging_since_timestamp_seconds` proportionally to the new stake added with zero age: [4](#0-3) 

**SNS analog:** `claim_or_refresh_neuron` in SNS governance dispatches `By::NeuronId` to `refresh_neuron` with no authorization check: [5](#0-4) 

SNS `refresh_neuron` also has no caller check before calling `neuron.update_stake`: [6](#0-5) 

**Existing guards are insufficient:** The only checks in `refresh_neuron` are a neuron lock (prevents concurrent calls, not unauthorized calls) and a minimum balance check (easily satisfied since the victim's neuron already has sufficient stake). No controller/hotkey check exists anywhere in this path.

**Attack steps:**
1. Attacker transfers 1 e8s (plus the 10,000 e8s transfer fee) to Alice's neuron subaccount on the ICP ledger.
2. Attacker calls `manage_neuron` as any principal with `ClaimOrRefresh { by: NeuronIdOrSubaccount }` targeting Alice's neuron ID.
3. `refresh_neuron` reads the new balance (now `cached_stake + 1 e8s`), hits `Ordering::Less`, calls `update_stake_adjust_age`.
4. Alice's `aging_since_timestamp_seconds` is advanced toward `now`, permanently reducing her age.
5. Attacker repeats, each time paying only ~0.0001 ICP.

## Impact Explanation

The age bonus in NNS is up to 25% at 4 years (`MAX_NEURON_AGE_FOR_AGE_BONUS = 4 * ONE_YEAR_SECONDS`): [7](#0-6) 

Voting power is `stake * dissolve_delay_bonus * age_bonus_multiplier`: [8](#0-7) 

Reducing the age bonus directly reduces both voting power and the proportional share of voting rewards. For a neuron with 1000 ICP and 4 years of age, each 1 ICP attack reduces the age by ~1 day. Repeated attacks can prevent the victim from ever accumulating maximum age bonus. This constitutes a significant NNS/SNS governance security impact with concrete, measurable, and irreversible harm to targeted neuron holders' voting rewards.

**Severity: High** — Significant NNS/SNS governance security impact with concrete user harm (permanent reduction in voting rewards), requiring no special privileges and minimal cost.

## Likelihood Explanation

The `ClaimOrRefresh` endpoint is publicly callable by any principal with no authorization check, as confirmed by the code and by the test comment "anyone can do it": [9](#0-8) 

The neuron subaccount is deterministically computable from the controller's principal ID and memo (both public for known neurons). The per-attack cost is ~0.0001 ICP (transfer fee) plus 1 e8s. The attack is trivially scriptable and repeatable indefinitely.

## Recommendation

1. **Add an authorization check to `refresh_neuron` (NNS):** Before calling `update_stake_adjust_age` in the `Ordering::Less` branch, verify that the caller is the neuron's controller or a registered hotkey. The `Ordering::Equal` branch (neuron ID discovery) can remain permissionless since it has no state-changing effect.

2. **Apply the same fix to SNS `refresh_neuron`** in `rs/sns/governance/src/governance.rs`: require the caller to hold `NeuronPermissionType::ManagePrincipals` or equivalent before allowing a stake refresh that modifies `aging_since_timestamp_seconds`.

3. **Alternatively**, separate the "discover neuron ID" use case (permissionless, `Ordering::Equal`) from the "refresh stake" use case (requires authorization, `Ordering::Less`), and only gate the latter.

pub(crate) fn age_bonus_multiplier(age_seconds: u64) -> Decimal {
    let age_seconds = Decimal::from(age_seconds.clamp(0, MAX_NEURON_AGE_FOR_AGE_BONUS));

    // t is (clamped) age in units of max age, so its value is from 0.0 to 1.0
    let t = age_seconds / Decimal::from(MAX_NEURON_AGE_FOR_AGE_BONUS);

    // 0.25 * t + 1
    t / Decimal::from(4) + Decimal::from(1)
}

fn refresh_neuron_by_id_or_subaccount(
    owner: PrincipalId,
    caller: PrincipalId,
    refresh_by: RefreshBy,
) {
    let stake = Tokens::from_tokens(10_u64).unwrap();
    let memo = Memo(1234_u64);
    let (mut driver, mut gov, nid, subaccount) = governance_with_staked_neuron(
        INITIAL_NEURON_DISSOLVE_DELAY,
        stake.get_e8s(),
        0,
        owner,
        memo.0,
    );

    let neuron = gov.neuron_store.with_neuron(&nid, |n| n.clone()).unwrap();
    assert_eq!(neuron.cached_neuron_stake_e8s, stake.get_e8s());

    driver.add_funds_to_account(
        AccountIdentifier::new(GOVERNANCE_CANISTER_ID.get(), Some(subaccount)),
        stake.get_e8s(),
    );

    // stake shouldn't have changed.
    let neuron = gov
        .neuron_store
        .with_neuron(&nid, |n| n.clone())
        .unwrap()
        .clone();
    assert_eq!(neuron.cached_neuron_stake_e8s, stake.get_e8s());

    let neuron_id_or_subaccount = match refresh_by {
        RefreshBy::NeuronId => NeuronIdOrSubaccount::NeuronId(neuron.id()),
        RefreshBy::Subaccount => NeuronIdOrSubaccount::Subaccount(subaccount.into()),
    };

    let manage_neuron_response = gov
        .manage_neuron(
            &caller,
            &ManageNeuron {
                neuron_id_or_subaccount: Some(neuron_id_or_subaccount),
                id: None,
                command: Some(Command::ClaimOrRefresh(ClaimOrRefresh {
                    by: Some(By::NeuronIdOrSubaccount(Empty {})),
                })),
            },
        )
        .now_or_never()
        .unwrap();

    let nid = match manage_neuron_response.command.unwrap() {
        CommandResponse::ClaimOrRefresh(response) => response.refreshed_neuron_id,
        CommandResponse::Error(error) => panic!("Error claiming neuron: {error:?}"),
        _ => panic!("Invalid response."),
    };

    assert!(nid.is_some());
    let nid = nid.unwrap();
    let neuron = gov.neuron_store.with_neuron(&nid, |n| n.clone()).unwrap();
    assert_eq!(neuron.controller(), owner);
    assert_eq!(neuron.cached_neuron_stake_e8s, stake.get_e8s() * 2);
