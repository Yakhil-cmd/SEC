Audit Report

## Title
Unprivileged User Can Exhaust SNS Governance `max_number_of_neurons` Limit, Blocking Legitimate Stakers and SNS Swap Neuron Distribution - (File: `rs/sns/governance/src/governance.rs`)

## Summary

SNS Governance enforces a hard cap (`max_number_of_neurons`, default 200,000) on total neurons with no rate limiter or per-principal neuron count limit on `claim_neuron`. An attacker holding sufficient SNS tokens can fill the entire neuron table by repeatedly calling `manage_neuron::ClaimOrRefresh` with minimum-stake neurons, then disburse all tokens to recover capital (paying only transaction fees). This permanently blocks all new staking and also blocks the `claim_swap_neurons` path used during SNS decentralization swaps, which returns `MemoryExhausted` for every new swap participant.

## Finding Description

`claim_neuron` in SNS Governance ( [1](#0-0) ) calls `self.add_neuron(neuron.clone())?` before making the async ledger balance check. `add_neuron` internally calls `check_neuron_population_can_grow` ( [2](#0-1) ), which enforces a hard ceiling: if `(self.proto.neurons.len() as u64) + 1 > max_number_of_neurons`, it returns `PreconditionFailed`. The default cap is 200,000 neurons. ( [3](#0-2) )

The minimum stake floor only requires `neuron_minimum_stake_e8s > transaction_fee_e8s` ( [4](#0-3) ), meaning the per-neuron capital cost can be as low as 1 e8s above the fee. With default values (1 SNS token per neuron), filling 200,000 slots costs 200,000 tokens — fully recoverable by disbursing all neurons, with only transaction fees as permanent loss.

Unlike NNS Governance, which gates `claim_neuron` with an explicit `self.rate_limiter.try_reserve(...)` call ( [5](#0-4) ), SNS Governance has no rate limiter, no per-principal neuron count limit, and no minimum dissolve delay before disbursement. A grep for `rate_limit`, `rate_limiter`, or `InMemoryRateLimiter` in `rs/sns/governance/src/governance.rs` returns zero matches.

The `claim_swap_neurons` path is also affected: when `add_neuron` fails due to the cap, it records `ClaimedSwapNeuronStatus::MemoryExhausted` for every swap participant neuron recipe. ( [6](#0-5) ) This means an attacker can pre-fill the neuron table before an SNS swap concludes, causing all swap participants to receive `MemoryExhausted` status and fail to receive their neurons.

## Impact Explanation

This is a **High** severity finding matching the allowed impact: *"Application/platform-level DoS... or significant SNS security impact with concrete user or protocol harm."* Once the cap is reached, no new neurons can be created by any user. All legitimate staking is blocked. More critically, the `claim_swap_neurons` path used during SNS decentralization swaps is also blocked, meaning an attacker can sabotage an entire SNS launch by preventing swap participants from receiving their neurons. The attacker retains full ownership of all 200,000 neurons and can vote to block any governance proposal to raise `max_number_of_neurons`, potentially making the DoS permanent.

## Likelihood Explanation

The attack is reachable by any unprivileged user who can acquire SNS tokens. The capital cost is `max_number_of_neurons × neuron_minimum_stake_e8s` tokens, which are fully recoverable. The only permanent cost is transaction fees (200,000 × `transaction_fee_e8s`). There is no rate limiting, no per-principal neuron count limit, and no minimum lock-up period enforced before disbursement. The existing integration test `test_claim_neuron_fails_when_max_number_of_neurons_is_reached` ( [7](#0-6) ) directly confirms the behavior. For SNS instances with low `neuron_minimum_stake_e8s`, the attack is essentially free.

## Recommendation

1. **Add a per-principal neuron count limit** in `add_neuron` or `claim_neuron` to prevent a single principal from claiming a disproportionate share of the neuron table.
2. **Add a rate limiter** on `claim_neuron` in SNS Governance, analogous to the `InMemoryRateLimiter` used in NNS Governance. ( [8](#0-7) )
3. **Enforce a meaningful minimum stake floor** that makes bulk neuron creation economically prohibitive (e.g., a fixed minimum independent of `transaction_fee_e8s`).
4. **Require a minimum dissolve delay** before a neuron can be disbursed, increasing the capital lock-up cost of the attack.

## Proof of Concept

1. Deploy an SNS with `max_number_of_neurons = 200_000` and `neuron_minimum_stake_e8s = 100_000_000` (1 token).
2. Attacker acquires 200,000 SNS tokens.
3. Attacker calls `manage_neuron` with `ClaimOrRefresh { by: MemoAndController { memo: i, controller: None } }` for `i` in `0..200_000`, each time having pre-transferred exactly 1 token to the corresponding subaccount.
4. All 200,000 neuron slots are occupied by the attacker.
5. Any subsequent `manage_neuron::ClaimOrRefresh` by a legitimate user returns `PreconditionFailed: "Cannot add neuron. Max number of neurons reached."` — confirmed by the existing test. ( [7](#0-6) )
6. Attacker calls `disburse` on all neurons, recovering ~200,000 tokens minus transaction fees.
7. The SNS governance is DoS'd; recovery requires a governance proposal to raise `max_number_of_neurons`, which the attacker (holding 200,000 neurons) can block by voting against it.

    /// Checks whether new neurons can be added or whether the maximum number of neurons,
    /// as defined in the nervous system parameters, has already been reached.
    fn check_neuron_population_can_grow(&self) -> Result<(), GovernanceError> {
        let max_number_of_neurons = self
            .nervous_system_parameters_or_panic()
            .max_number_of_neurons
            .expect("NervousSystemParameters must have max_number_of_neurons");

        if (self.proto.neurons.len() as u64) + 1 > max_number_of_neurons {
            return Err(GovernanceError::new_with_message(
                ErrorType::PreconditionFailed,
                "Cannot add neuron. Max number of neurons reached.",
            ));
        }

        Ok(())
    }

fn test_claim_neuron_fails_when_max_number_of_neurons_is_reached() {
    local_test_on_sns_subnet(|runtime| async move {
        // Set up an SNS with a ledger account for two users
        let user1 = UserInfo::new(Sender::from_keypair(&TEST_USER1_KEYPAIR));
        let user2 = UserInfo::new(Sender::from_keypair(&TEST_USER2_KEYPAIR));
        let alloc = Tokens::from_tokens(1000).unwrap();

        let sys_params = NervousSystemParameters {
            neuron_claimer_permissions: Some(NeuronPermissionList {
                permissions: NeuronPermissionType::all(),
            }),
            max_number_of_neurons: Some(1),
            ..NervousSystemParameters::with_default_values()
        };

        let sns_init_payload = SnsTestsInitPayloadBuilder::new()
            .with_ledger_account(user1.sender.get_principal_id().0.into(), alloc)
            .with_ledger_account(user2.sender.get_principal_id().0.into(), alloc)
            .with_nervous_system_parameters(sys_params.clone())
            .build();

        let sns_canisters = SnsCanisters::set_up(&runtime, sns_init_payload).await;

        // Successfully STAKE and CLAIM user1's neuron, reaching the configured maximum number of
        // neurons allowed in the system
        sns_canisters
            .stake_and_claim_neuron(&user1.sender, Some(ONE_YEAR_SECONDS as u32))
            .await;

        // Only STAKE for user2.
        sns_canisters
            .stake_neuron_account(&user2.sender, &user2.subaccount, 1)
            .await;

        let manage_neuron_command = ManageNeuron {
            subaccount: user2.subaccount.to_vec(),
            command: Some(Command::ClaimOrRefresh(ClaimOrRefresh {
                by: Some(By::MemoAndController(MemoAndController {
                    memo: NONCE,
                    controller: None,
                })),
            })),
        };

        // Try claiming the Neuron for user2 (via memo and controller). This should fail due to
        // the max_number_of_neurons being reached.
        let response: ManageNeuronResponse = sns_canisters
            .governance
            .update_from_sender(
                "manage_neuron",
                candid_one,
                manage_neuron_command.clone(),
                &user2.sender,
            )
            .await
            .expect("Error calling the manage_neuron api.");

        // assert that the error_type is PreconditionFailed.
        let error = match response.command.unwrap() {
            CommandResponse::Error(error) => error,
            CommandResponse::ClaimOrRefresh(_) => {
                panic!(
                    "User should not have been able to claim a neuron due to reaching max_number_of_neurons"
                )
            }
            _ => panic!("Unexpected command response when claiming neuron."),
        };
        assert_eq!(error.error_type, ErrorType::PreconditionFailed as i32);
