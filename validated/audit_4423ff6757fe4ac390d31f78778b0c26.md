### Title
Subnet Delegation Certificate Timestamp Is Never Validated Against Current Time - (File: `rs/certification/src/lib.rs`)

### Summary

The `verify_delegation_certificate` function in `rs/certification/src/lib.rs` parses the `time` field from a subnet delegation certificate's state tree but explicitly skips any staleness check on it. This is the IC analog of the BufferRouter `closeAnytime` timestamp issue: a signed artifact carries a timestamp that is extracted but never compared against the current time, allowing arbitrarily old delegation certificates to be accepted.

### Finding Description

In `rs/certification/src/lib.rs`, `verify_delegation_certificate` deserializes the delegation certificate's state tree into a `SubnetCertificateData` struct:

```rust
struct SubnetCertificateData {
    #[allow(unused)] // currently delegation timestamps are not checked
    time: Leb128EncodedU64,
    subnet: BTreeMap<SubnetId, SubnetView>,
    canister_ranges: Option<BTreeMap<SubnetId, TreeCanisterRanges>>,
}
```

The `#[allow(unused)]` annotation and the comment `// currently delegation timestamps are not checked` confirm that the `time` field is parsed from the certificate tree but is never compared against any current time or maximum-age bound. [1](#0-0) 

The function verifies the BLS threshold signature over the certificate tree and checks canister ID ranges, but returns successfully without ever asserting that `subnet_state.time` is recent. [2](#0-1) 

This function is called from `verify_certificate_internal`, which is the root of all canister-signature and certified-data verification on the IC: [3](#0-2) 

The canister-signature verification path (`rs/crypto/iccsa/src/api.rs`) calls `verify_certified_data_with_cache_for_canister_sig`, which in turn calls `verify_certificate_internal` → `verify_delegation_certificate`: [4](#0-3) 

The ingress-message validator calls this path when verifying canister signatures used as the `sender_pubkey` for ingress authentication, and also when verifying `sender_info` canister signatures: [5](#0-4) 

### Impact Explanation

A subnet delegation certificate is issued by the NNS root subnet and certifies the public key of an application subnet. It is embedded inside every canister signature. Because the delegation timestamp is never checked, a canister signature whose embedded delegation certificate is arbitrarily old (months or years) will still pass verification, as long as the BLS signature over the certificate tree is cryptographically valid.

Concretely:

1. **Stale canister-signature acceptance for ingress authentication**: A user whose canister-signature public key was valid at some past time (e.g., Internet Identity session key) can present a canister signature whose embedded delegation certificate is from that past time. The replica accepts it even if the subnet's key has since been rotated or the subnet has been decommissioned, because the delegation timestamp is never checked against the current consensus time.

2. **Stale `sender_info` canister signatures**: The `validate_sender_info` path in `rs/validator/src/ingress_validation.rs` verifies a canister signature over the `info` blob. The same unchecked delegation timestamp applies, meaning a `sender_info` canister signature produced with an old delegation certificate is accepted indefinitely. [6](#0-5) 

### Likelihood Explanation

The attack is reachable by any unprivileged ingress sender. No privileged access is required. The attacker only needs to have obtained a valid canister signature (e.g., from Internet Identity) at any point in the past and can reuse it with the old delegation certificate indefinitely. The `#[allow(unused)]` comment confirms this is a known, intentional omission rather than an oversight, but it remains a real gap: the IC specification requires that delegation certificates be fresh, and the code explicitly skips that check.

### Recommendation

In `verify_delegation_certificate`, compare `subnet_state.time` against a caller-supplied `current_time` parameter (or a maximum-age bound such as the consensus time minus a configurable staleness window). Reject delegation certificates whose embedded timestamp is older than the allowed maximum age. This mirrors the fix pattern used for ingress expiry (`validate_ingress_expiry` in `rs/validator/src/ingress_validation.rs`) and sender delegation expiry (`validate_sender_delegation_expiry`). [7](#0-6) [8](#0-7) 

### Proof of Concept

1. Obtain a valid canister signature from Internet Identity (or any canister) at time T₀. The signature embeds a subnet delegation certificate with `time = T₀`.
2. Wait until time T₁ >> T₀ (e.g., months later).
3. Submit an ingress message to any replica using the old canister signature (with the old delegation certificate from T₀) as `sender_pubkey`.
4. The replica calls `verify_canister_sig` → `verify_certified_data_with_cache_for_canister_sig` → `verify_certificate_internal` → `verify_delegation_certificate`.
5. `verify_delegation_certificate` parses `time = T₀` from the delegation certificate tree but, due to the `#[allow(unused)]` suppression, never compares it against the current consensus time T₁.
6. The BLS signature over the certificate tree is still cryptographically valid (the NNS root key has not changed), so the function returns `Ok(...)`.
7. The ingress message is accepted as if it were freshly authenticated.

fn verify_certificate_internal(
    certificate: &[u8],
    canister_id: &CanisterId,
    root_pk: &ThresholdSigPublicKey,
    use_signature_cache: bool,
) -> Result<(Certificate, Option<DelegationSubnetInfo>), CertificateValidationError> {
    let certificate: Certificate = parse_certificate(certificate)?;
    let (key, delegation_info) = if let Some(delegation) = &certificate.delegation {
        let subnet_id = PrincipalId::try_from(&*delegation.subnet_id)
            .map(SubnetId::from)
            .map_err(|err| {
                CertificateValidationError::DeserError(format!(
                    "failed to parse delegation subnet id: {err}"
                ))
            })?;
        let (key, info) = verify_delegation_certificate(
            &delegation.certificate,
            &subnet_id,
            root_pk,
            Some(canister_id),
            use_signature_cache,
        )?;
        (key, Some(info))
    } else {
        (*root_pk, None)
    };

    verify_certificate_signature(&certificate, &key, use_signature_cache)?;
    Ok((certificate, delegation_info))
}

                public_key_bytes: pk.0.clone(),

    };

    verify_sender_info_canister_sig(
        sender_info,
        sender_pubkey,
        ingress_signature_verifier,
        root_of_trust_provider,
    )
}

// Check if ingress_expiry is within a proper range with respect to the given
// time, i.e., it is not expired yet and is not too far in the future.
fn validate_ingress_expiry<C: HttpRequestContent>(
    request: &HttpRequest<C>,
    current_time: Time,
) -> Result<(), RequestValidationError> {
    let ingress_expiry = request.ingress_expiry();
    let provided_expiry = Time::from_nanos_since_unix_epoch(ingress_expiry);
    let min_allowed_expiry = current_time;
    // We need to account for time drift and be more forgiving at rejecting ingress
    // messages due to their expiry being too far in the future.
    // If this logic changes, then the migration canister in `//rs/migration_canister`
    // must be updated, too.
    let max_expiry_diff = MAX_INGRESS_TTL
        .checked_add(PERMITTED_DRIFT_AT_VALIDATOR)
        .ok_or_else(|| {
            InvalidRequestExpiry(format!(
                "Addition of MAX_INGRESS_TTL {MAX_INGRESS_TTL:?} with \
                PERMITTED_DRIFT_AT_VALIDATOR {PERMITTED_DRIFT_AT_VALIDATOR:?} overflows",
            ))
        })?;
    let max_allowed_expiry = min_allowed_expiry
        .checked_add(max_expiry_diff)
        .ok_or_else(|| {
            InvalidRequestExpiry(format!(
                "Addition of min_allowed_expiry {min_allowed_expiry:?} \
                with max_expiry_diff {max_expiry_diff:?} overflows",
            ))
        })?;
    if !(min_allowed_expiry <= provided_expiry && provided_expiry <= max_allowed_expiry) {
        let msg = format!(
            "Specified ingress_expiry not within expected range: \
             Minimum allowed expiry: {min_allowed_expiry}, \
             Maximum allowed expiry: {max_allowed_expiry}, \
             Provided expiry:        {provided_expiry}"
        );
        return Err(InvalidRequestExpiry(msg));
    }
    Ok(())
}

// Check if any of the sender delegation has expired with respect to the
// `current_time`, and return an error if so.
fn validate_sender_delegation_expiry(
    sender_delegation: &Option<Vec<SignedDelegation>>,
    current_time: Time,
) -> Result<(), RequestValidationError> {
    if let Some(delegations) = &sender_delegation {
        for delegation in delegations.iter() {
            let expiry = delegation.delegation().expiration();
            if delegation.delegation().expiration() < current_time {
                return Err(InvalidDelegationExpiry(format!(
                    "Specified sender delegation has expired:\n\
                     Provided expiry:    {expiry}\n\
                     Local replica time: {current_time}",
                )));
            }
        }
    }
    Ok(())
}
