### Title
NNS Governance `initiate_maturity_disbursement` Minimum-Output Check Ignores Worst-Case Maturity Modulation, Allowing Sub-Minimum ICP Minting - (File: rs/nns/governance/src/governance/disburse_maturity.rs)

---

### Summary

The NNS governance canister's `initiate_maturity_disbursement` function enforces a minimum disbursement floor (`MINIMUM_DISBURSEMENT_E8S = 1 ICP`) against the **pre-modulation** maturity amount. Because the actual ICP minting occurs 7 days later with live maturity modulation applied (range: −5 % to +5 %), a disbursement that passes the gate check can produce a minted amount **below** `MINIMUM_DISBURSEMENT_E8S`. The code comment explicitly states the limit is intended to guarantee at least that many ICP "assuming the worst case maturity modulation," but the implementation never applies worst-case modulation to the check. The analogous `spawn_neuron` path in the same canister correctly applies the 5 % haircut before comparing against the minimum stake.

---

### Finding Description

`initiate_maturity_disbursement` in `rs/nns/governance/src/governance/disburse_maturity.rs` computes the raw maturity percentage and then checks:

```rust
// MINIMUM_DISBURSEMENT_E8S = E8 = 100_000_000
if disbursement_maturity_e8s < MINIMUM_DISBURSEMENT_E8S {
    return Err(InitiateMaturityDisbursementError::DisbursementTooSmall { … });
}
``` [1](#0-0) 

The constant is documented as:

> "The minimum amount of ICP that need to be minted when disbursing maturity. A neuron can only disburse an amount of maturity that results in minting at least this many ICP (in e8) **assuming the worst case maturity modulation**." [2](#0-1) 

However, the check is performed on the **pre-modulation** value. Seven days later, `next_maturity_disbursement_to_finalize` applies the live modulation:

```rust
let maturity_to_disburse_after_modulation_e8s = apply_maturity_modulation(
    original_maturity_e8s_equivalent,
    maturity_modulation_basis_points,
)?;
``` [3](#0-2) 

`apply_maturity_modulation` with `MIN_MATURITY_MODULATION_PERMYRIAD = −500` (−5 %) reduces `E8` to `95_000_000` e8s — 5 % below the stated minimum. [4](#0-3) 

By contrast, `spawn_neuron` in the same canister **does** apply the worst-case haircut before the gate check:

```rust
let least_possible_stake = (maturity_to_spawn as f64 * (1_f64 - 0.05)) as u64;
if least_possible_stake < economics.neuron_minimum_stake_e8s {
    return Err(GovernanceError::new_with_message(
        ErrorType::InsufficientFunds,
        "There isn't enough maturity to spawn a new neuron due to worst case maturity modulation.",
    ));
}
``` [5](#0-4) 

The SNS governance's `disburse_maturity` also performs the correct worst-case check:

```rust
let worst_case_maturity_modulation =
    apply_maturity_modulation(maturity_to_deduct, MIN_MATURITY_MODULATION_PERMYRIAD)…;
if worst_case_maturity_modulation < transaction_fee_e8s {
    return Err(…);
}
``` [6](#0-5) 

The NNS governance path is the only one that omits this worst-case application.

---

### Impact Explanation

A neuron controller who initiates a disbursement of exactly `MINIMUM_DISBURSEMENT_E8S` (1 ICP worth of maturity) will have that maturity **irrevocably deducted** from the neuron at initiation time:

```rust
neuron.maturity_e8s_equivalent = neuron
    .maturity_e8s_equivalent
    .saturating_sub(disbursement_maturity_e8s);
``` [7](#0-6) 

Seven days later, if maturity modulation is at its minimum (−5 %), the ledger minting call uses `amount_to_mint_e8s = 95_000_000` — 5 % less than the minimum the user was led to believe they were guaranteed. The user cannot cancel or reverse the disbursement after initiation. The maturity is gone and the minted ICP is below the stated floor.

---

### Likelihood Explanation

- Any NNS neuron controller with maturity in the range `[E8, E8/0.95)` (i.e., between 1.00 ICP and ~1.053 ICP) can trigger this condition.
- Maturity modulation goes negative whenever the ICP/XDR price declines over the prior 28-day window — a routine market event.
- The entry path is a standard `manage_neuron` ingress call with `DisburseMaturity { percentage_to_disburse: 100 }` from any unprivileged neuron controller.
- No special privileges, governance majority, or threshold corruption are required.

---

### Recommendation

Apply worst-case modulation before the minimum check, mirroring `spawn_neuron` and SNS `disburse_maturity`:

```rust
let worst_case_disbursement_e8s = apply_maturity_modulation(
    disbursement_maturity_e8s,
    MIN_MATURITY_MODULATION_PERMYRIAD,
).map_err(|_| InitiateMaturityDisbursementError::Unknown { … })?;

if worst_case_disbursement_e8s < MINIMUM_DISBURSEMENT_E8S {
    return Err(InitiateMaturityDisbursementError::DisbursementTooSmall {
        disbursement_maturity_e8s,
        minimum_disbursement_e8s: MINIMUM_DISBURSEMENT_E8S,
    });
}
```

This ensures the stated invariant — "at least `MINIMUM_DISBURSEMENT_E8S` minted assuming worst-case modulation" — is actually enforced.

---

### Proof of Concept

1. Neuron has `maturity_e8s_equivalent = 100_000_000` (exactly 1 ICP).
2. Controller calls `manage_neuron` → `DisburseMaturity { percentage_to_disburse: 100, … }`.
3. `initiate_maturity_disbursement` computes `disbursement_maturity_e8s = 100_000_000`.
4. Check `100_000_000 < 100_000_000` → **false** → disbursement is accepted.
5. Maturity is deducted from the neuron immediately.
6. Seven days later, `finalize_maturity_disbursement` runs with `maturity_modulation_basis_points = −500`.
7. `apply_maturity_modulation(100_000_000, −500)` → `95_000_000` e8s minted to the user.
8. User receives **0.95 ICP** despite the code comment guaranteeing at least **1 ICP** under worst-case modulation. [8](#0-7) [9](#0-8)

pub fn apply_maturity_modulation(
    amount_maturity_e8s: u64,
    maturity_modulation_basis_points: i32,
) -> Result<u64, String> {
    let amount_e8s = u128::from(amount_maturity_e8s);

    let adjusted_maturity_modulation_basis_points = saturating_add_or_subtract_u128_i32(
        BASIS_POINTS_PER_UNITY,
        maturity_modulation_basis_points,
    );

    let modulated_amount_e8s: u128 = amount_e8s
        .checked_mul(adjusted_maturity_modulation_basis_points)
        .ok_or_else(|| "Underflow or overflow when calculating maturity modulation".to_string())?
        .checked_div(BASIS_POINTS_PER_UNITY)
        .ok_or_else(|| "Underflow or overflow when calculating maturity modulation".to_string())?;

    u64::try_from(modulated_amount_e8s).map_err(|err| err.to_string())
