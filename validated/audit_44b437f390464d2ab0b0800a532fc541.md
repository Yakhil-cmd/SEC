### Title
SNS Swap `refresh_buyer_tokens` Accepts SNS Governance Canister as Buyer, Enabling ICP Accounting Confusion and SNS Token Dilution - (File: rs/sns/swap/canister/canister.rs, rs/sns/swap/src/swap.rs)

### Summary
The `refresh_buyer_tokens` endpoint in the SNS swap canister accepts any arbitrary principal as the `buyer` field, including the SNS governance canister (the ICP recipient / "seller" equivalent). There is no check that `buyer != sns_governance_canister_id`. An unprivileged caller can register the SNS governance canister as a direct participant by sending ICP to its subaccount on the swap canister and calling `refresh_buyer_tokens` with `buyer = sns_governance_canister_id`. This creates a role-collision analogous to the original report: the same principal acts as both the ICP destination (seller) and a registered buyer, corrupting the SNS token distribution and inflating the governance canister's voting power.

### Finding Description

In `rs/sns/swap/canister/canister.rs`, the `refresh_buyer_tokens` update handler parses the `buyer` field from the request without any identity validation:

```rust
let p: PrincipalId = if arg.buyer.is_empty() {
    caller_principal_id()
} else {
    PrincipalId::from_str(&arg.buyer).unwrap()   // any principal accepted
};
``` [1](#0-0) 

This `p` is passed directly to `refresh_buyer_token_e8s` in `rs/sns/swap/src/swap.rs`. That function checks the ICP balance of the buyer's subaccount on the swap canister and, if sufficient, inserts the principal into the `self.buyers` map:

```rust
self.buyers
    .entry(buyer.to_string())
    .or_insert_with(|| BuyerState::new(0))
    .set_amount_icp_e8s(new_balance_e8s);
self.update_total_participation_amounts();
``` [2](#0-1) 

There is no guard anywhere in `refresh_buyer_token_e8s` that rejects `buyer == sns_governance_canister_id`, `buyer == nns_governance_canister_id`, or `buyer == this_canister`. The SNS governance canister is the ICP treasury destination on a committed swap — it is the structural "seller" of SNS tokens. [3](#0-2) 

During `sweep_icp` on a COMMITTED swap, the destination for every buyer's ICP is hardcoded to the SNS governance canister:

```rust
let dst = if lifecycle == Lifecycle::Committed {
    Account {
        owner: sns_governance.get().0,
        subaccount: None,
    }
} else { ... };
``` [4](#0-3) 

If `principal == sns_governance`, the transfer source is `Account { owner: swap_canister, subaccount: Some(principal_to_subaccount(&sns_governance)) }` and the destination is `Account { owner: sns_governance, subaccount: None }`. This is a valid ICP ledger transfer (different accounts), so it succeeds and counts as a `success` in `SweepResult`. The SNS governance canister's ICP contribution is then used to compute its proportional share of SNS tokens in `sweep_sns` and to create neurons for it in `claim_swap_neurons`. [5](#0-4) 

### Impact Explanation

**Ledger conservation / SNS token dilution:** The SNS governance canister is credited with a buyer contribution equal to the ICP the attacker deposited. During `sweep_sns`, SNS tokens are distributed proportionally across all buyers including the governance canister. Legitimate buyers receive fewer SNS tokens than they are entitled to — a direct financial loss proportional to the attacker's ICP stake.

**Governance authorization:** During `claim_swap_neurons`, the SNS governance canister has neuron baskets created on its behalf. These neurons carry voting power within the SNS, inflating the governance canister's influence beyond what was intended by the SNS design. Because the SNS governance canister is itself the target of governance proposals, this creates a circular self-reinforcing voting weight.

**Premature swap commitment:** The attacker's ICP inflates `direct_participation_icp_e8s`. If the attacker's deposit pushes the total past `max_direct_participation_icp_e8s`, the swap commits immediately, potentially before the intended participation window closes. [6](#0-5) 

### Likelihood Explanation

The attack requires the attacker to send real ICP to `Account { owner: swap_canister, subaccount: Some(principal_to_subaccount(&sns_governance_canister_id)) }` on the ICP ledger, then call `refresh_buyer_tokens` with `buyer = sns_governance_canister_id`. Both steps are permissionless and reachable by any unprivileged ingress sender during the `Open` lifecycle. The attacker loses the ICP (it is swept to the SNS governance canister's default account), making this a griefing/dilution attack rather than a theft. Likelihood is **low** for large-scale attacks but **medium** for targeted dilution of a specific SNS launch.

### Recommendation

Add identity-conflict guards at the top of `refresh_buyer_token_e8s` (before the async ICP ledger call) to reject reserved system principals:

```rust
let init = self.init_or_panic();
if buyer == PrincipalId::from_str(&init.sns_governance_canister_id).unwrap()
    || buyer == PrincipalId::from_str(&init.nns_governance_canister_id).unwrap()
    || buyer == this_canister.get()
{
    return Err(format!(
        "Principal {} is a system canister and cannot participate as a buyer.",
        buyer
    ));
}
```

Analogously, `updateBuyer` and `updateSeller` in the original report were recommended to check `_buyer != seller` and `_buyer != buyer`; the IC fix mirrors that pattern by ensuring the buyer principal does not coincide with any role that already holds a privileged position in the swap.

### Proof of Concept

1. Attacker reads `sns_governance_canister_id` from `swap.get_state()`.
2. Attacker transfers `min_participant_icp_e8s` ICP to `Account { owner: SWAP_CANISTER_ID, subaccount: Some(principal_to_subaccount(&sns_governance_canister_id)) }` on the ICP ledger.
3. Attacker calls `refresh_buyer_tokens({ buyer: sns_governance_canister_id.to_text(), confirmation_text: None })` on the swap canister. The call succeeds because the subaccount balance meets the minimum.
4. `self.buyers` now contains an entry for `sns_governance_canister_id` with `amount_icp_e8s = min_participant_icp_e8s`. `direct_participation_icp_e8s` is incremented.
5. When the swap commits and `finalize_swap` runs:
   - `sweep_icp`: ICP is transferred from `Account { owner: swap, subaccount: principal_to_subaccount(sns_governance) }` → `Account { owner: sns_governance, subaccount: None }`. Transfer succeeds; `sweep_result.success += 1`.
   - `sweep_sns`: SNS tokens are distributed to all buyers including `sns_governance_canister_id`, diluting legitimate buyers.
   - `claim_swap_neurons`: Neuron baskets are created for `sns_governance_canister_id`, granting it unintended voting weight. [7](#0-6)

    pub async fn sweep_icp(
        &mut self,
        now_fn: fn(bool) -> u64,
        icp_ledger: &dyn ICRC1Ledger,
    ) -> SweepResult {
        let lifecycle: Lifecycle = self.lifecycle();

        let init = match self.init_and_validate() {
            Ok(init) => init,
            Err(error_message) => {
                log!(
                    ERROR,
                    "Halting sweep_icp(). State is missing or corrupted: {:?}",
                    error_message
                );
                return SweepResult::new_with_global_failures(1);
            }
        };

        // The following methods are safe to call since we validated Init in the above block
        let sns_governance = init.sns_governance_or_panic();

        let mut sweep_result = SweepResult::default();

        for (principal_str, buyer_state) in self.buyers.iter_mut() {
            // principal_str should always be parseable as a PrincipalId as that is enforced
            // in `refresh_buyer_tokens`. In the case of a bug due to programmer error, increment
            // the invalid field. This will require a manual intervention via an upgrade to correct
            let principal = match string_to_principal(principal_str) {
                Some(p) => p,
                None => {
                    sweep_result.invalid += 1;
                    continue;
                }
            };

            let subaccount = principal_to_subaccount(&principal);
            let dst = if lifecycle == Lifecycle::Committed {
                // This Account should be given a name, such as SNS ICP Treasury...
                Account {
                    owner: sns_governance.get().0,
                    subaccount: None,
                }
            } else {
                Account {
                    owner: principal.0,
                    subaccount: None,
                }
            };

            let icp_transferable_amount = match buyer_state.icp.as_mut() {
                Some(transferable_amount) => transferable_amount,
                // BuyerState.icp should always be present as it is set in `refresh_buyer_tokens`.
                // In the case of a bug due to programmer error, increment the invalid field.
                // This will require a manual intervention via an upgrade to correct
                None => {
                    log!(
                        ERROR,
                        "PrincipalId {} has corrupted BuyerState: {:?}",
                        principal,
                        buyer_state
                    );
                    sweep_result.invalid += 1;
                    continue;
                }
            };

            let result = icp_transferable_amount
                .transfer_helper(
                    now_fn,
                    DEFAULT_TRANSFER_FEE,
                    Some(subaccount),
                    &dst,
                    icp_ledger,
                )
                .await;
            match result {
                // AmountToSmall should never happen as the amount contributed is checked in
                // `refresh_buyer_tokens`. In the case of a bug due to programmer error,
                // increment the invalid field. This will require a manual intervention
                // via an upgrade to correct
                TransferResult::AmountTooSmall => {
                    sweep_result.invalid += 1;
                }
                TransferResult::AlreadyStarted => {
                    sweep_result.skipped += 1;
                }
                TransferResult::Success(_) => {
                    sweep_result.success += 1;
                }
                TransferResult::Failure(_) => {
                    sweep_result.failure += 1;
                }
            }

            // Update the buyer state to indicate funds that have been successfully committed or refunded.
            if result.is_success() {
                // Record transfer fee
                icp_transferable_amount.transfer_fee_paid_e8s =
                    Some(DEFAULT_TRANSFER_FEE.get_e8s());
                // Record the amount minus transfer fee that was refunded or committed.
                let amount_transferred_e8s =
                    Some(icp_transferable_amount.amount_e8s - DEFAULT_TRANSFER_FEE.get_e8s());
                icp_transferable_amount.amount_transferred_e8s = amount_transferred_e8s;
            }
        }

        sweep_result
    }
