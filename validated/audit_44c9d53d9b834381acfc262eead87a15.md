Audit Report

## Title
`DogecoinFeeEstimator::estimate_nth_fee` Returns Raw Fee Percentile Without Minimum Floor, Enabling Permanently Stuck Withdrawals - (File: `rs/dogecoin/ckdoge/minter/src/fees/mod.rs`)

## Summary

`DogecoinFeeEstimator::estimate_nth_fee` returns the raw fee percentile from the Dogecoin canister with no minimum fee-rate floor. The identical omission in the ckBTC minter caused real withdrawal transactions to become permanently stuck on 2025-06-21 and required an emergency governance upgrade to patch. The ckDOGE minter shares the same `FeeEstimator` trait and transaction-building pipeline but never received the analogous fix. If the Dogecoin canister returns an anomalously low fee percentile, the minter will broadcast a dust-fee transaction that miners will never include, and the resubmission path calls the same estimator with the same data, so it cannot self-heal.

## Finding Description

`DogecoinFeeEstimator::estimate_nth_fee` at `rs/dogecoin/ckdoge/minter/src/fees/mod.rs` lines 53–66 returns `Some(fee_percentiles[nth])` unconditionally for Mainnet:

```rust
Network::Mainnet => {
    if fee_percentiles.len() < 100 || nth >= 100 {
        return None;
    }
    Some(fee_percentiles[nth])   // ← raw value, no floor
}
``` [1](#0-0) 

`BitcoinFeeEstimator::estimate_nth_fee` at `rs/bitcoin/ckbtc/minter/src/fees/mod.rs` line 112 applies a minimum floor after the same pattern:

```rust
median_fee.map(|f| f.max(self.minimum_fee_per_vbyte()))
``` [2](#0-1) 

where `minimum_fee_per_vbyte()` returns `1_500` millisat/byte for Mainnet and `1_000` for Testnet: [3](#0-2) 

The ckBTC upgrade proposal `rs/bitcoin/ckbtc/mainnet/minter_upgrade_2025_06_27.md` documents the real-world incident and the stop-gap fix introduced in PR #5742: [4](#0-3) 

The ckDOGE minter uses `DogecoinFeeEstimator` as its `CanisterRuntime::Estimator` and the same shared `ic_ckbtc_minter` transaction-building pipeline (`ic_ckbtc_minter::queries::estimate_withdrawal_fee`): [5](#0-4) [6](#0-5) 

The resubmission path calls `estimate_nth_fee` again via the same estimator, so a stuck transaction cannot be replaced with a higher-fee one if the underlying fee data remains anomalously low.

## Impact Explanation

Any ckDOGE withdrawal can produce a Dogecoin transaction with a near-zero fee rate if the Dogecoin canister returns anomalously low fee percentiles. Such transactions will not be mined. Because the minter's resubmission logic calls the same `estimate_nth_fee` with the same bad data, it cannot self-heal. All affected withdrawal requests become permanently stuck, freezing user DOGE funds in the minter. This is a **High** impact: significant ck-token security impact with concrete, demonstrated user harm, matching the allowed impact class "Significant Chain Fusion, ck-token, ledger … security impact with concrete user or protocol harm."

## Likelihood Explanation

This failure mode has **already occurred** in the sister ckBTC minter on 2025-06-21 and required an emergency governance upgrade. No attacker action is required — it is a passive failure triggered by the Dogecoin canister returning an anomalously low fee percentile, which is a realistic, already-observed event class. The ckDOGE minter uses the same infrastructure and is exposed to the same failure mode. Likelihood is high.

## Recommendation

Add a `minimum_fee_per_byte` method to `DogecoinFeeEstimator` and apply it in `estimate_nth_fee`, mirroring the fix applied to `BitcoinFeeEstimator`:

```rust
const fn minimum_fee_per_byte(&self) -> FeeRate {
    match &self.network {
        Network::Mainnet => FeeRate::from_millis_per_byte(/* appropriate Dogecoin floor, e.g. 100_000 */),
        Network::Regtest => FeeRate::from_millis_per_byte(0),
    }
}

fn estimate_nth_fee(&self, fee_percentiles: &[FeeRate], nth: usize) -> Option<FeeRate> {
    // ...
    let fee = match &self.network {
        Network::Mainnet => { /* existing bounds check */ Some(fee_percentiles[nth]) }
        Network::Regtest => Some(DEFAULT_REGTEST_FEE),
    };
    fee.map(|f| f.max(self.minimum_fee_per_byte()))
}
```

The appropriate floor value should be determined based on Dogecoin's typical minimum relay fee (Dogecoin's default `minRelayTxFee` is 1 DOGE/kB = 1,000,000 koinu/kB = 1,000 millikoinu/byte).

}
