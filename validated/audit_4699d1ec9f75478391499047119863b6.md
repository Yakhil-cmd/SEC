### Title
Deprecated `transfer()` in `WSEI.withdraw()` Permanently Freezes Funds for Contract Callers — (`contracts/src/WSEI.sol`)

---

### Summary

`WSEI.sol` is a production contract (its compiled artifacts are embedded into the chain binary). Its `withdraw()` function uses `payable(msg.sender).transfer(wad)`, which forwards only 2300 gas. Any smart contract whose `receive()` or `fallback()` consumes more than 2300 gas will have its withdrawal permanently revert, freezing the deposited native SEI inside WSEI forever.

---

### Finding Description

`WSEI.withdraw()` sends native SEI back to the caller using Solidity's deprecated `.transfer()`: [1](#0-0) 

The 2300-gas stipend imposed by `.transfer()` is insufficient for any non-trivial contract recipient — multisigs (Gnosis Safe), proxy wallets, DeFi vaults, or any contract with storage writes in its `receive()` hook. The call reverts silently from the callee's perspective, and because the balance deduction at line 29 is already committed before the transfer at line 30, the entire transaction reverts, leaving the user's balance intact but the withdrawal path permanently broken for that caller type.

The contract is not a test artifact — its ABI and bytecode are compiled and embedded into the chain binary: [2](#0-1) 

---

### Impact Explanation

Any smart contract that:
1. Calls `WSEI.deposit()` (or sends ETH directly via `fallback`/`receive`), and
2. Later calls `WSEI.withdraw()`

…will have the withdrawal revert if its own `receive()`/`fallback()` uses >2300 gas. The deposited SEI is not lost from the contract's accounting (`balanceOf` is decremented then re-incremented on revert), but the user is permanently unable to unwrap via the standard path. DeFi protocols, multisigs, and proxy wallets integrating WSEI on Sei are all affected. Depending on the value locked, this meets the **Medium** (fund freeze < $5k) or **Critical** (≥ $5k) threshold.

---

### Likelihood Explanation

- The path is fully unprivileged: any address can call `deposit()` then `withdraw()`.
- Gnosis Safe and proxy-based wallets are extremely common on EVM chains; their `receive()` hooks routinely exceed 2300 gas.
- No special conditions, governance, or operator keys are required.

---

### Recommendation

Replace `.transfer()` with a low-level `.call{}` and check the return value:

```solidity
function withdraw(uint wad) public {
    require(balanceOf[msg.sender] >= wad);
    balanceOf[msg.sender] -= wad;
    (bool success, ) = payable(msg.sender).call{value: wad}("");
    require(success, "WSEI: ETH transfer failed");
    emit Withdrawal(msg.sender, wad);
}
```

This pattern is standard for WETH9 replacements and removes the gas-stipend restriction entirely.

---

### Proof of Concept

1. Deploy a contract `Vault` whose `receive()` writes to storage (>2300 gas).
2. `Vault` calls `WSEI.deposit{value: 1 ether}()`.
3. `Vault` calls `WSEI.withdraw(1 ether)`.
4. The inner `payable(msg.sender).transfer(1 ether)` forwards only 2300 gas to `Vault.receive()`, which runs out of gas and reverts.
5. The entire `withdraw` transaction reverts; `Vault`'s WSEI balance remains, but every future `withdraw` call will also revert — funds are frozen.
