### Title
EIP-7623 Floor Gas Not Enforced in Refund Calculation, Allowing Underpayment of Gas Fees — (`engine/src/engine.rs`)

---

### Summary

`submit_with_alt_modexp` validates that `gas_limit >= max(intrinsic_gas, floor_gas)` per EIP-7623, but `refund_unused_gas` computes the refund using only `gas_used` — never `floor_gas`. When `floor_gas > gas_used`, the sender receives a larger refund than EIP-7623 permits, underpaying gas fees and depriving the relayer of their entitled priority-fee yield.

---

### Finding Description

**Step 1 — Validation (correct):**

In `submit_with_alt_modexp`, both `intrinsic_gas` and `floor_gas` are computed and the transaction is rejected unless `gas_limit >= max(intrinsic_gas, floor_gas)`:

```rust
let intrinsic_gas = transaction.intrinsic_gas(CONFIG)...;
let floor_gas     = transaction.floor_gas(CONFIG)...;

if transaction.gas_limit < core::cmp::max(intrinsic_gas, floor_gas).into() {
    return Err(EngineErrorKind::IntrinsicGasNotMet.into());
}
``` [1](#0-0) 

**Step 2 — Charging (correct):**

`charge_gas` deducts `gas_limit * effective_gas_price` from the sender upfront:

```rust
let prepaid_amount = fixed_gas
    .map_or(transaction.gas_limit, EthGas::as_u256)
    .checked_mul(effective_gas_price)
    .map(Wei::new)...;
``` [2](#0-1) 

**Step 3 — Refund (broken):**

`refund_unused_gas` computes the refund using only `gas_used`, with no reference to `floor_gas`:

```rust
let gas_to_wei = |price: U256| {
    fixed_gas
        .map_or_else(|| gas_used.into(), EthGas::as_u256)
        .checked_mul(price)...
};
let spent_amount  = gas_to_wei(gas_result.effective_gas_price)?;
let reward_amount = gas_to_wei(gas_result.priority_fee_per_gas)?;
let refund = gas_result.prepaid_amount.checked_sub(spent_amount)...;
``` [3](#0-2) 

`floor_gas` is computed in `submit_with_alt_modexp` but is **never passed** to `refund_unused_gas`. The function signature confirms it has no `floor_gas` parameter:

```rust
pub fn refund_unused_gas<I: IO>(
    io: &mut I,
    sender: &Address,
    gas_used: u64,
    gas_result: &GasPaymentResult,
    relayer: &Address,
    fixed_gas: Option<EthGas>,
) -> Result<(), GasPaymentError>
``` [4](#0-3) 

**The mismatch:** EIP-7623 requires that when `floor_gas > gas_used`, the effective gas charged must be `floor_gas`, not `gas_used`. The current implementation enforces `floor_gas` only as a minimum gas-limit requirement (Step 1), but not as a minimum gas-cost requirement (Step 3). The result:

| Quantity | Should be (EIP-7623) | Actual |
|---|---|---|
| `spent_amount` | `max(gas_used, floor_gas) * effective_gas_price` | `gas_used * effective_gas_price` |
| `reward_amount` | `max(gas_used, floor_gas) * priority_fee_per_gas` | `gas_used * priority_fee_per_gas` |
| Sender refund | `(gas_limit - floor_gas) * effective_gas_price` | `(gas_limit - gas_used) * effective_gas_price` |

**Root cause in `floor_gas` calculation:** For any calldata with zero bytes, `floor_gas > intrinsic_gas` always holds because `total_cost_floor_per_token = 10 > gas_transaction_zero_data = 4`:

```rust
let tokens_in_calldata = num_non_zero_bytes.checked_mul(4)
    .and_then(|gas| gas.checked_add(num_zero_bytes))...;
tokens_in_calldata
    .checked_mul(config.total_cost_floor_per_token)
    .and_then(|gas| gas.checked_add(base_gas))
``` [5](#0-4) 

---

### Impact Explanation

The relayer is entitled to `floor_gas * priority_fee_per_gas` but receives only `gas_used * priority_fee_per_gas`. The difference `(floor_gas - gas_used) * priority_fee_per_gas` is instead refunded to the sender. This constitutes **theft of unclaimed yield** (relayer fees) — High severity per the allowed impact scope.

---

### Likelihood Explanation

High. The Prague hard fork is active (confirmed by `has_floor_gas = true` in the Prague config and the recent addition in CHANGES.md
