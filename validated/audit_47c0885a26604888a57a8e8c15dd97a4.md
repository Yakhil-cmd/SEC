Audit Report

## Title
Accumulated SNS Voting Rewards Claimable in Full by First Voter After Empty-Proposal Rounds - (`rs/sns/governance/src/governance.rs`)

## Summary

The SNS `distribute_rewards` function accrues a rewards purse each round proportional to the **total token supply**, regardless of whether any neurons are staked. When no proposals settle in a round, the entire purse rolls over unconditionally. An unprivileged attacker who stakes the first neuron after any period of empty rounds can submit a single proposal, cast the only vote, and receive the entire accumulated purse as maturity — far exceeding their stake cost.

## Finding Description

**Reward accrual (no staked-supply guard):**
In `distribute_rewards`, the purse is computed as `rate × round_duration × total_supply` for every elapsed round, starting from the previously rolled-over amount: [1](#0-0) 

The `supply` variable is the full circulating supply fetched from the ledger — there is no check that any tokens are actually staked.

**Rollover path:**
When `considered_proposals` is empty (no proposals ready to settle), `total_reward_shares == 0`, so the `else` branch that distributes maturity is skipped entirely: [2](#0-1) 

The resulting `RewardEvent` is written with `settled_proposals: considered_proposals` (an empty vec): [3](#0-2) 

`rewards_rolled_over()` then returns `true` because `settled_proposals` is empty: [4](#0-3) 

And `e8s_equivalent_to_be_rolled_over()` returns the full `total_available_e8s_equivalent` back into the next round's starting purse: [5](#0-4) 

**Exploit flow:**
1. SNS runs for N rounds with no settled proposals (e.g., during the gap between SNS genesis and swap completion, or after all neurons dissolve). Each round the purse grows by `rate × duration × total_supply` and rolls over in full.
2. Attacker stakes one neuron at `neuron_minimum_stake_e8s`.
3. Attacker submits a `Motion` proposal (cost: `proposal_reject_cost_e8s`). Since the attacker's neuron is the only neuron, it holds 100% of voting power.
4. Proposal settles; `considered_proposals` is now non-empty. `total_reward_shares` equals the attacker's voting power alone.
5. `distribute_rewards` runs: `neuron_reward_e8s = rewards_purse_e8s × (attacker_shares / total_shares) = rewards_purse_e8s × 1.0`.
6. Attacker calls `DisburseMaturity`, converting the entire accumulated purse to tokens.

No existing guard checks whether `total_staked_e8s == 0` before accruing or rolling over rewards.

## Impact Explanation

This matches **High** — "Significant SNS security impact with concrete user or protocol harm." The attacker mints SNS tokens as maturity far exceeding their stake, diluting all existing token holders. For a realistic SNS with a 30-day no-proposal gap, a 5% annual reward rate, and a $10M total supply, the claimable purse is approximately $41,000 — entirely captured by one attacker at a cost of one minimum-stake neuron plus one proposal fee. The harm is concrete: illegitimate inflation accrues to a single unprivileged actor at the expense of all other holders.

## Likelihood Explanation

The attack is triggerable by any unprivileged principal. Realistic preconditions include: (1) the gap between SNS genesis and swap completion, during which no neurons exist in governance yet but `genesis_timestamp_seconds` has already been set and rewards begin accruing; (2) all initial neurons dissolving with no re-staking; (3) a prolonged proposal drought on a small SNS. No privileged access, social engineering, or external oracle compromise is required. The attacker needs only the public `manage_neuron` endpoint.

## Recommendation

Add a guard in `distribute_rewards` before computing the purse to skip accrual (or discard rather than roll over) when no neurons are staked:

```rust
let total_staked_e8s: u64 = self.proto.neurons.values()
    .map(|n| n.cached_neuron_stake_e8s)
    .sum();
if total_staked_e8s == 0 {
    // Advance the reward event round counter without rolling over the purse.
    return;
}
```

Alternatively, change `rewards_rolled_over` to also return `false` (i.e., discard the purse) when `total_staked_e8s == 0`, so that rewards only accumulate during periods when there is actual staked supply to distribute to.

## Proof of Concept

**Deterministic integration test plan (using PocketIC or the SNS test framework):**

1. Deploy an SNS with `initial_reward_rate_basis_points = 500`, `round_duration_seconds = 86400`, no initial neurons, and a fixed total supply of 1,000,000,000 e8s.
2. Advance the test clock by 30 days, calling `run_periodic_tasks` each simulated day. Assert that `latest_reward_event().distributed_e8s_equivalent == 0` and `latest_reward_event().total_available_e8s_equivalent > 0` after each round.
3. Stake one neuron with `neuron_minimum_stake_e8s` tokens and set dissolve delay to `neuron_minimum_dissolve_delay_to_vote_seconds`.
4. Submit a `Motion` proposal. Advance the clock past the voting period so the proposal settles.
5. Advance the clock by one more `round_duration_seconds` and call `run_periodic_tasks`.
6. Assert that `neuron.maturity_e8s_equivalent ≈ 30 × (5%/365) × 1,000,000,000 e8s` — the entire 30-day accumulated purse — and that `distributed_e8s_equivalent` in the new `RewardEvent` equals this amount.
7. Call `DisburseMaturity` and confirm the attacker receives tokens far exceeding their original stake.
