Audit Report

## Title
Unpermissioned `ClaimOrRefresh` Allows Any Caller to Permanently Dilute Victim Neuron's Age Bonus — (`rs/nns/governance/src/governance.rs`, `rs/sns/governance/src/governance.rs`)

## Summary
Both NNS and SNS Governance canisters allow any unprivileged caller to trigger a neuron stake refresh (`ClaimOrRefresh` via `By::NeuronIdOrSubaccount`) on any neuron without authorization. By first transferring tokens to the victim's neuron subaccount and then calling `manage_neuron`, an attacker causes `update_stake_adjust_age` / `update_stake` to permanently dilute the victim's `aging_since_timestamp_seconds`, reducing the age-based voting power bonus. The attack is repeatable and the age loss is irreversible without waiting years to re-accumulate.

## Finding Description
In NNS Governance, `manage_neuron_internal` handles `ClaimOrRefresh` before any authorization check and returns early, bypassing all subsequent caller validation: [1](#0-0) 

The `By::NeuronIdOrSubaccount` path calls `refresh_neuron_by_id_or_subaccount`, which resolves the neuron from its public ID and delegates to `refresh_neuron` with no caller identity check: [2](#0-1) 

Inside `refresh_neuron`, when the ledger balance exceeds the cached stake (i.e., an attacker deposited tokens), `update_stake_adjust_age` is called unconditionally: [3](#0-2) 

`update_stake_adjust_age` uses `combine_aged_stakes` to compute a weighted-average age, permanently advancing `aging_since_timestamp_seconds` via `adjust_age`: [4](#0-3) 

`adjust_age` only modifies `NotDissolving` neurons — exactly the neurons that accumulate age bonuses: [5](#0-4) 

The identical pattern exists in SNS Governance: `refresh_neuron` has no caller authorization and calls `update_stake` which applies the same dilution formula (`new_age = (old_age * old_stake) / new_stake`): [6](#0-5) [7](#0-6) 

The NNS test suite explicitly documents this as intentional behavior: [8](#0-7) 

Existing guards are insufficient: the only checks in `refresh_neuron` are a minimum-stake balance check and a neuron lock for concurrency — neither restricts the caller's identity.

## Impact Explanation
This matches the **High** bounty impact: *"Significant NNS, SNS security impact with concrete user or protocol harm."* A neuron's age bonus contributes up to 25% additional voting power in NNS (up to `max_age_bonus_percentage` in SNS, configurable up to 100%). An attacker who matches a victim's existing stake halves the effective age (e.g., 4 years → 2 years, 25% bonus → ~12.5%). The attack can be repeated to asymptotically drive the age bonus toward zero. This enables targeted governance manipulation: selectively reducing the voting power of specific neurons to shift the outcome of NNS or SNS proposals. The age loss is permanent and irreversible without waiting years to re-accumulate.

## Likelihood Explanation
**Medium.** The attack requires the attacker to permanently spend real ICP/SNS tokens (which are irrecoverably added to the victim's neuron). The cost scales with the victim's stake — halving the age requires matching the existing stake. However, no privileged access is required; any ingress sender can execute this. The victim's neuron subaccount is deterministically derivable from public on-chain data. For a well-resourced attacker targeting a decisive whale neuron, the cost may be acceptable. The attack is repeatable, with each iteration requiring progressively more tokens as the victim's stake grows.

## Recommendation
Restrict `ClaimOrRefresh` refresh operations via `By::NeuronIdOrSubaccount` so that only the neuron's controller or an authorized hotkey can trigger a refresh that modifies the neuron's age. The minimal fix is: when `refresh_neuron` / `refresh_neuron_by_id_or_subaccount` is called, pass the `caller` principal and verify it matches the neuron's controller or a hotkey before calling `update_stake_adjust_age` / `update_stake`. Alternatively, when the caller is not the controller/hotkey, skip the age adjustment and only update `cached_neuron_stake_e8s` to reflect the new balance — preserving the legitimate use case (anyone can top up a neuron's stake) while preventing age dilution attacks.

    pub fn update_stake_adjust_age(&mut self, updated_stake_e8s: u64, now: u64) {
        // If the updated stake is less than the original stake, preserve the
        // age and distribute it over the new amount. This should not happen
        // in practice, so this code exists merely as a defensive fallback.
        //
        // TODO(NNS1-954) Consider whether update_stake_adjust_age (and other
        // similar methods) should use a neurons effective stake rather than
        // the cached stake.
        if updated_stake_e8s < self.cached_neuron_stake_e8s {
            println!(
                "{}Reducing neuron {:?} stake via update_stake_adjust_age: {} -> {}",
                LOG_PREFIX,
                self.id(),
                self.cached_neuron_stake_e8s,
                updated_stake_e8s
            );
            self.cached_neuron_stake_e8s = updated_stake_e8s;
        } else {
            // If one looks at "stake * age" as describing an area, the goal
            // at this point is to increase the stake while keeping the area
            // constant. This means decreasing the age in proportion to the
            // additional stake, which is the purpose of combine_aged_stakes.
            let (new_stake_e8s, new_age_seconds) = combine_aged_stakes(
                self.cached_neuron_stake_e8s,
                self.age_seconds(now),
                updated_stake_e8s.saturating_sub(self.cached_neuron_stake_e8s),
                0,
            );
            // A consequence of the math above is that the 'new_stake_e8s' is
            // always the same as the 'updated_stake_e8s'. We use
            // 'combine_aged_stakes' here to make sure the age is
            // appropriately pro-rated to accommodate the new stake.
            assert!(new_stake_e8s == updated_stake_e8s);
            self.cached_neuron_stake_e8s = new_stake_e8s;

            let new_aging_since_timestamp_seconds = now.saturating_sub(new_age_seconds);
            let new_disolved_dissolve_state_and_age = self
                .dissolve_state_and_age()
                .adjust_age(new_aging_since_timestamp_seconds);
            self.set_dissolve_state_and_age(new_disolved_dissolve_state_and_age);
        }

    async fn refresh_neuron(&mut self, nid: &NeuronId) -> Result<(), GovernanceError> {
        let now = self.env.now();
        let subaccount = nid.subaccount()?;
        let account = self.neuron_account_id(subaccount);

        // First ensure that the neuron was not created via an NNS Neurons' Fund participation in the
        // decentralization swap
        {
            let neuron = self.get_neuron_result(nid)?;

            if neuron.is_neurons_fund_controlled() {
                return Err(GovernanceError::new_with_message(
                    ErrorType::PreconditionFailed,
                    "Cannot refresh an SNS Neuron controlled by the Neurons' Fund",
                ));
            }
        }

        // Get the balance of the neuron from the ledger canister.
        let balance = self.ledger.account_balance(account).await?;

        let min_stake = self
            .nervous_system_parameters_or_panic()
            .neuron_minimum_stake_e8s
            .expect("NervousSystemParameters must have neuron_minimum_stake_e8s");
        if balance.get_e8s() < min_stake {
            return Err(GovernanceError::new_with_message(
                ErrorType::InsufficientFunds,
                format!(
                    "Account does not have enough funds to refresh a neuron. \
                        Please make sure that account has at least {:?} e8s (was {:?} e8s)",
                    min_stake,
                    balance.get_e8s()
                ),
            ));
        }
        let neuron = self.get_neuron_result_mut(nid)?;
        match neuron.cached_neuron_stake_e8s.cmp(&balance.get_e8s()) {
            Ordering::Greater => {
                log!(
                    ERROR,
                    "ERROR. Neuron cached stake was inconsistent.\
                     Neuron account: {} has less e8s: {} than the cached neuron stake: {}.\
                     Stake adjusted.",
                    account,
                    balance.get_e8s(),
                    neuron.cached_neuron_stake_e8s
                );
                neuron.update_stake(balance.get_e8s(), now);
            }
            Ordering::Less => {
                neuron.update_stake(balance.get_e8s(), now);
            }
            // If the stake is the same as the account balance,
            // just return the neuron id (this way this method
            // also serves the purpose of allowing to discover the
            // neuron id based on the memo and the controller).
            Ordering::Equal => (),
        };

        Ok(())
    }

    pub fn update_stake(&mut self, new_stake_e8s: u64, now: u64) {
        // If this neuron has an age and its stake is being increased, adjust the
        // neuron's age
        if self.aging_since_timestamp_seconds < now && self.cached_neuron_stake_e8s <= new_stake_e8s
        {
            let old_stake = self.cached_neuron_stake_e8s as u128;
            let old_age = now.saturating_sub(self.aging_since_timestamp_seconds) as u128;
            let new_age = (old_age * old_stake) / (new_stake_e8s as u128);

            // new_age * new_stake = old_age * old_stake -
            // (old_stake * old_age) % new_stake. That is, age is
            // adjusted in proportion to the stake, but due to the
            // discrete nature of u64 numbers, some resolution is
            // lost due to the division above. This means the age
            // bonus is derived from a constant times age times
            // stake, minus up to new_stake - 1 each time the
            // neuron is refreshed. Only if old_age * old_stake is
            // a multiple of new_stake does the age remain
            // constant after the refresh operation. However, in
            // the end, the most that can be lost due to rounding
            // from the actual age, is always less 1 second, so
            // this is not a problem.
            self.aging_since_timestamp_seconds = now.saturating_sub(new_age as u64);
            // Note that if new_stake == old_stake, then
            // new_age == old_age, and
            // now - new_age =
            // now-(now-neuron.aging_since_timestamp_seconds)
            // = neuron.aging_since_timestamp_seconds.
        }

        self.cached_neuron_stake_e8s = new_stake_e8s;
    }
