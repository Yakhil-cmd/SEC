### Title
Truncating Division in `native_per_pubdata` Calculation Silently Zeros Pubdata Cost, Enabling Free Pubdata Generation — (`File: basic_bootloader/src/bootloader/transaction_flow/zk/validation_impl.rs`)

---

### Summary

In `validate_and_compute_fee_for_transaction`, the `native_per_pubdata` rate is computed using truncating integer division (`wrapping_div`). When `pubdata_price < native_price`, the result silently rounds to zero. This makes all pubdata completely free in native resources for the duration of that block, allowing any unprivileged L2 transaction sender to generate maximum pubdata without paying the native cost the operator must cover on L1.

---

### Finding Description

In `validation_impl.rs` at line 142, `native_per_pubdata` is computed as:

```rust
let native_per_pubdata = u256_try_to_u64(&pubdata_price.wrapping_div(native_price))
    .ok_or(TxError::Validation(InvalidTransaction::PubdataPriceTooHigh))?;
``` [1](#0-0) 

`wrapping_div` performs truncating (floor) integer division. When `pubdata_price < native_price`, the result is `0`. There is no guard against this zero result.

Contrast this with `native_per_gas`, computed on line 135 using `div_ceil`:

```rust
u256_try_to_u64(&gas_price.div_ceil(native_price))
``` [2](#0-1) 

`div_ceil` ensures that any non-zero `gas_price` yields at least `native_per_gas = 1`. No equivalent protection exists for `native_per_pubdata`.

When `native_per_pubdata = 0`, every downstream pubdata charge becomes zero:

**1. Intrinsic pubdata overhead (in `create_resources_for_tx`):**
```rust
let intrinsic_pubdata_overhead = native_per_pubdata_byte.saturating_mul(intrinsic_pubdata);
// = 0 * intrinsic_pubdata = 0
``` [3](#0-2) 

**2. Execution pubdata charge (in `get_resources_to_charge_for_pubdata`):**
```rust
let native = current_pubdata_spent.checked_mul(native_per_pubdata)...;
// = current_pubdata_spent * 0 = 0
``` [4](#0-3) 

**3. Post-execution pubdata check (in `check_enough_resources_for_pubdata`):**
```rust
let enough = resources.has_enough(&resources_for_pubdata);
// resources_for_pubdata = 0, so always enough
``` [5](#0-4) 

The same truncating division also appears in the public API helper `validate_l2_tx_intrinsic_native_resources`:
```rust
let native_per_pubdata = u256_try_to_u64(&pubdata_price.wrapping_div(native_price)).ok_or(())?;
``` [6](#0-5) 

---

### Impact Explanation

When `pubdata_price < native_price` (a valid and realistic block configuration — e.g., `pubdata_price = 500`, `native_price = 1000`), `native_per_pubdata` silently becomes `0`. Every L2 transaction in that block can generate pubdata up to the block's `pubdata_limit` at zero native cost. The operator must still publish all state diffs to L1 and pay Ethereum data availability fees, but receives no native resource compensation from the transactions that caused the pubdata. This is a direct operator fund-loss path: the operator subsidizes L1 data costs for attacker-generated pubdata without any on-chain compensation.

The `native_per_gas` path is protected by `div_ceil` (ensuring at least 1 when gas_price > 0), but `native_per_pubdata` has no equivalent protection, creating an asymmetric and exploitable rounding gap.

---

### Likelihood Explanation

The condition `pubdata_price < native_price` is realistic in production. For example, if the operator sets `native_price` to reflect RISC-V cycle cost (a large number) and `pubdata_price` to reflect current L1 calldata cost (which can be smaller), the truncation silently zeros pubdata fees. The attacker only needs to observe the block context (both values are public) and submit transactions that maximize SSTORE writes or L1 messages. No privileged access, leaked keys, or oracle manipulation is required.

---

### Recommendation

Replace `wrapping_div` with `div_ceil` for the `native_per_pubdata` calculation, consistent with how `native_per_gas` is computed:

```rust
// Before (truncates to 0 when pubdata_price < native_price):
let native_per_pubdata = u256_try_to_u64(&pubdata_price.wrapping_div(native_price))
    .ok_or(TxError::Validation(InvalidTransaction::PubdataPriceTooHigh))?;

// After (rounds up, ensuring non-zero when pubdata_price > 0):
let native_per_pubdata = u256_try_to_u64(&pubdata_price.div_ceil(native_price))
    .ok_or(TxError::Validation(InvalidTransaction::PubdataPriceTooHigh))?;
```

Apply the same fix to `api/src/helpers.rs` line 427. This mirrors the existing protection on `native_per_gas` and ensures that any non-zero `pubdata_price` always results in at least 1 native unit charged per pubdata byte.

### Proof of Concept

**Setup:**
- Block context: `native_price = 1000`, `pubdata_price = 999`, `eip1559_basefee = 1`
- Attacker submits an L2 EIP-1559 transaction with `max_fee_per_gas = 1000`, `gas_limit = 10_000_000`
- Transaction body: a loop of 100 SSTORE writes to distinct slots (generating ~3,200 bytes of pubdata)

**Trace through the code:**

1. `gas_price = base_fee + priority_fee = 1000`
2. `native_per_gas = ceil(1000 / 1000) = 1` ✓ (correctly non-zero)
3. `native_per_pubdata = 999 / 1000 = 0` ✗ (truncated to zero)
4. `intrinsic_pubdata_overhead = 0 * L2_TX_INTRINSIC_PUBDATA = 0`
5. Transaction executes, generates 3,200 bytes of pubdata
6. `get_resources_to_charge_for_pubdata(system, 0, ...)` → charges `3200 * 0 = 0` native
7. `check_enough_resources_for_pubdata` → always passes (0 ≤ any remaining native)
8. Transaction succeeds; operator publishes 3,200 bytes to L1 and pays ETH data fees; attacker paid zero native for pubdata

**Expected behavior:** `native_per_pubdata` should be `ceil(999/1000) = 1`, charging 3,200 native for the pubdata.

**Actual behavior:** `native_per_pubdata = 0`; pubdata is free; operator absorbs the L1 cost. [1](#0-0) [7](#0-6)

```rust
pub fn get_resources_to_charge_for_pubdata<S: EthereumLikeTypes>(
    system: &mut System<S>,
    native_per_pubdata: u64,
    base_pubdata: Option<u64>,
) -> Result<(u64, S::Resources), SystemError> {
    let current_pubdata_spent = system
        .net_pubdata_used()?
        .saturating_sub(base_pubdata.unwrap_or(0));
    let native = current_pubdata_spent
        .checked_mul(native_per_pubdata)
        .ok_or(out_of_native_resources!())?;
    let native = <S::Resources as zk_ee::system::Resources>::Native::from_computational(native);
    Ok((current_pubdata_spent, S::Resources::from_native(native)))
}
