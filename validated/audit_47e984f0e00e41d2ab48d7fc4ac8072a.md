### Title
Truncating Division in `native_per_pubdata` Calculation Allows Zero-Cost Pubdata Publication - (File: `basic_bootloader/src/bootloader/transaction_flow/zk/validation_impl.rs`)

### Summary

The `native_per_pubdata` resource rate is computed using truncating (floor) integer division. When `pubdata_price < native_price`, the result truncates to zero, making all pubdata publication free of native resource cost for every transaction in that block. This is a direct analog to the reported truncation-causes-zero-fee pattern.

### Finding Description

In `validate_and_compute_fee_for_transaction`, the per-pubdata-byte native resource rate is derived as:

```rust
// basic_bootloader/src/bootloader/transaction_flow/zk/validation_impl.rs:142
let native_per_pubdata = u256_try_to_u64(&pubdata_price.wrapping_div(native_price))
    .ok_or(TxError::Validation(InvalidTransaction::PubdataPriceTooHigh))?;
```

`wrapping_div` performs **floor (truncating) integer division**. When `pubdata_price < native_price` (e.g., `pubdata_price = 500`, `native_price = 1000`), the result is `0`. [1](#0-0) 

This contrasts directly with `native_per_gas`, which uses **ceiling division** to avoid the same truncation-to-zero problem:

```rust
// basic_bootloader/src/bootloader/transaction_flow/zk/validation_impl.rs:135
u256_try_to_u64(&gas_price.div_ceil(native_price))
``` [2](#0-1) 

The same truncating pattern is replicated in `api/src/helpers.rs`:

```rust
// api/src/helpers.rs:427
let native_per_pubdata = u256_try_to_u64(&pubdata_price.wrapping_div(native_price)).ok_or(())?;
``` [3](#0-2) 

When `native_per_pubdata = 0`, the downstream effects cascade through the entire resource accounting pipeline:

**1. Intrinsic pubdata overhead becomes zero** in `create_resources_for_tx`:
```rust
let intrinsic_pubdata_overhead = native_per_pubdata_byte.saturating_mul(intrinsic_pubdata);
// = 0 * intrinsic_pubdata = 0
``` [4](#0-3) 

**2. Post-execution pubdata charge becomes zero** in `get_resources_to_charge_for_pubdata`:
```rust
let native = current_pubdata_spent
    .checked_mul(native_per_pubdata)  // = pubdata_bytes * 0 = 0
    .ok_or(out_of_native_resources!())?;
``` [5](#0-4) 

**3. The pubdata sufficiency check always passes** in `check_enough_resources_for_pubdata`, since zero native resources are required regardless of how much pubdata the transaction writes. [6](#0-5) 

The same zero-cost pubdata path applies to L2 ZK transactions processed through `validate_and_compute_fee_for_transaction`. [7](#0-6) 

### Impact Explanation

When `pubdata_price < native_price`, every L2 transaction in the block can write up to the block's `pubdata_limit` worth of state diffs to L1 without consuming any native resource budget for pubdata. The operator bears the real L1 data-availability cost while the transaction sender pays nothing for it. This breaks the economic invariant that pubdata publication is bounded by the sender's native resource budget, enabling:

- **Unbounded L1 data spam**: Attackers submit storage-write-heavy transactions that generate maximum pubdata, all at zero native cost.
- **Operator loss**: The operator must pay L1 calldata/blob costs for the published state diffs without being compensated through the native resource mechanism.
- **Block stuffing**: Pubdata limit (`pubdata_limit`) becomes the only constraint, which is a block-wide shared resource, allowing a single attacker to exhaust it for all other users.

### Likelihood Explanation

The condition `pubdata_price < native_price` is realistic. Both values are operator-set block-level parameters from the oracle. If `native_price` is denominated in a finer unit than `pubdata_price` (e.g., `native_price = 10_000`, `pubdata_price = 5_000`), the truncation silently produces zero. An attacker monitoring block metadata can detect this condition and immediately exploit it by submitting pubdata-heavy transactions. No privileged access is required — any EOA can submit such a transaction. [8](#0-7) 

### Recommendation

Replace the truncating `wrapping_div` with ceiling division (`div_ceil`) for `native_per_pubdata`, consistent with how `native_per_gas` is computed:

```rust
// Before (truncates to 0 when pubdata_price < native_price):
let native_per_pubdata = u256_try_to_u64(&pubdata_price.wrapping_div(native_price))
    .ok_or(TxError::Validation(InvalidTransaction::PubdataPriceTooHigh))?;

// After (rounds up, ensuring non-zero cost when pubdata_price > 0):
let native_per_pubdata = u256_try_to_u64(&pubdata_price.div_ceil(native_price))
    .ok_or(TxError::Validation(InvalidTransaction::PubdataPriceTooHigh))?;
```

Apply the same fix in `api/src/helpers.rs` line 427. Additionally, consider adding an explicit check that rejects or warns when `native_per_pubdata == 0` but `pubdata_price != 0`, to surface misconfigured block contexts early.

### Proof of Concept

1. Operator sets block context: `pubdata_price = 500`, `native_price = 1000`.
2. `native_per_pubdata = 500.wrapping_div(1000) = 0`.
3. Attacker submits a transaction that writes 20 new storage slots (each slot = 64 bytes pubdata = 1280 bytes total).
4. In `get_resources_to_charge_for_pubdata`: `native = 1280 * 0 = 0`.
5. `check_enough_resources_for_pubdata` returns `enough = true` with zero native charged.
6. Transaction succeeds; 1280 bytes of state diffs are published to L1 at zero native cost to the attacker.
7. Operator pays L1 blob/calldata fees for the published data with no compensation. [9](#0-8)

```rust
pub fn get_resources_to_charge_for_pubdata<S: EthereumLikeTypes>(
    system: &mut System<S>,
    native_per_pubdata: u64,
    base_pubdata: Option<u64>,
) -> Result<(u64, S::Resources), SystemError> {
    let current_pubdata_spent = system
        .net_pubdata_used()?
        .saturating_sub(base_pubdata.unwrap_or(0));
    let native = current_pubdata_spent
        .checked_mul(native_per_pubdata)
        .ok_or(out_of_native_resources!())?;
    let native = <S::Resources as zk_ee::system::Resources>::Native::from_computational(native);
    Ok((current_pubdata_spent, S::Resources::from_native(native)))
}
```

pub fn check_enough_resources_for_pubdata<S: EthereumLikeTypes>(
    system: &mut System<S>,
    native_per_pubdata: u64,
    resources: &S::Resources,
    base_pubdata: Option<u64>,
) -> Result<(bool, S::Resources, u64), SystemError> {
    let (pubdata_used, resources_for_pubdata) =
        get_resources_to_charge_for_pubdata(system, native_per_pubdata, base_pubdata)?;
    system_log!(system, "Checking gas for pubdata, resources_for_pubdata: {resources_for_pubdata:?}, resources: {resources:?}\n");
    let enough = resources.has_enough(&resources_for_pubdata);
    Ok((enough, resources_for_pubdata, pubdata_used))
