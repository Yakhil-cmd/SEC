Audit Report

## Title
Prometheus Label Cardinality Explosion via Unsanitized HTTP Path and Request Body — (`rs/rosetta-api/common/rosetta_core/src/metrics.rs`)

## Summary
`RosettaMetricsMiddleware` records the raw HTTP request path and the attacker-supplied `network_identifier.network` JSON field as Prometheus label values with no sanitization, normalization, or cardinality cap. Any unauthenticated HTTP client can exhaust heap memory by sending requests with unique paths and unique network identifiers, causing unbounded growth of the `rosetta_http_request_duration_seconds` HistogramVec and `rosetta_api_status_total` IntCounterVec in the default Prometheus registry, ultimately OOM-killing the Rosetta process.

## Finding Description

**Vector 1 — `endpoint` label (raw HTTP path):**

At line 326 of `metrics.rs`, the raw URI path is captured before dispatch:
```rust
let path = req.uri().path().to_owned();
``` [1](#0-0) 

This value is passed verbatim to `observe_request_duration` for every response, including 404s:
```rust
metrics.observe_request_duration(&path, &method, &status, duration);
``` [2](#0-1) 

The `metrics_layer` is applied to the entire router in `main.rs`, meaning all requests — including those to unregistered paths that Axum returns 404 for — pass through the middleware and have their raw path recorded as a label: [3](#0-2) 

**Vector 2 — `token_display_name` label (attacker-controlled body field):**

`extract_canister_id` reads `network_identifier.network` from the JSON body with no validation: [4](#0-3) 

`get_display_name_from_canister_id` falls back to returning the raw attacker-supplied string when the value is not in the registered map: [5](#0-4) 

This raw value is then used as the `token_display_name` label in `RosettaMetrics::new`, and also inserted into `CANISTER_DISPLAY_NAMES`, permanently growing the map: [6](#0-5) 

**Combined effect:**

`rosetta_http_request_duration_seconds` has four labels `[token_display_name, endpoint, method, status]`: [7](#0-6) 

Each unique `(token_display_name, endpoint, method, status)` tuple causes Prometheus to allocate a new histogram object with ~15 bucket counters. With N unique paths × M unique network identifiers, the registry grows to O(N×M) histogram objects. There is no cardinality cap, no label allowlist, and no rate limiting anywhere in the middleware.

## Impact Explanation

OOM termination of the Rosetta process disrupts all in-flight ICRC-1 token operations (ckBTC, ckETH, ckERC20 withdrawals/deposits) until the process is manually restarted. This constitutes a **High** severity application-level DoS with concrete user and protocol harm against an in-scope financial integration component (Rosetta API). It matches: *"High ($2,000–$10,000). Application/platform-level DoS, crash... not based on raw volumetric DDoS"* and *"Significant... Rosetta... security impact with concrete user or protocol harm."*

## Likelihood Explanation

The Rosetta API is publicly reachable by design. The attack requires only standard HTTP POST requests with crafted JSON bodies and unique URL paths — no authentication, no special tooling beyond `curl` or a simple script. Prometheus never evicts label combinations, so the heap growth is monotonic and persistent across the process lifetime. Even a moderate request rate (e.g., 10K requests/minute with unique paths) can exhaust memory.

## Recommendation

1. **Normalize the `endpoint` label** to a fixed allowlist of known route strings (e.g., map any unrecognized path to a sentinel like `"unknown"` before recording it as a label).
2. **Validate `token_display_name`** against the set of registered canister IDs/display names at startup; replace unknown values with `"unknown"` instead of using the raw attacker-supplied string.
3. **Add a cardinality guard** — reject `with_label_values` calls when the number of existing label combinations exceeds a configured threshold (e.g., 100).

## Proof of Concept

```python
import requests, threading

BASE = "http://<rosetta-host>:<port>"

def flood(i):
    requests.post(
        f"{BASE}/block/{i}",
        json={"network_identifier": {"network": f"fake-canister-{i}"},
              "block_identifier": {"index": 0}},
        timeout=5
    )

threads = [threading.Thread(target=flood, args=(i,)) for i in range(100_000)]
for t in threads: t.start()
for t in threads: t.join()
```

After execution, `GET /metrics` returns a `rosetta_http_request_duration_seconds` family with ~100K distinct label sets (each with ~15 histogram buckets). Process RSS grows proportionally; repeating the flood triggers OOM. The attack is reproducible locally by running the ICRC-1 Rosetta binary and executing the script against it.
