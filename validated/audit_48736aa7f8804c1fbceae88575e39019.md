### Title
Missing Zero-Address Check for `authority` Recovered from EIP-7702 Authorization Signature - (`File: basic_bootloader/src/bootloader/transaction/authorization_list.rs`)

---

### Summary

In the EIP-7702 authorization list processing, after `ecrecover` is called to derive the `authority` address from the authorization signature, there is no check that the recovered address is non-zero. An attacker can craft a degenerate signature that causes `ecrecover` to return the zero address (`0x0000...0000`), and the code will proceed to call `set_delegation` on the zero address — permanently writing delegation state to address zero and bumping its nonce.

---

### Finding Description

In `validate_and_apply_delegation` inside `basic_bootloader/src/bootloader/transaction/authorization_list.rs`, after recovering the `authority` from the signature:

```rust
let Some(authority) = resources
    .with_infinite_ergs(|inf_ergs| recover_authority(system, inf_ergs, auth_sig_data, &msg))?
else {
    return Ok(false);
};
```

The `recover_authority` function returns `None` only when `ecrecover_output.is_empty()` (i.e., the precompile returned no bytes). However, the EVM `ecrecover` precompile can return a 32-byte output whose last 20 bytes are all zero — the zero address — which is a valid non-empty output. In that case `recover_authority` returns `Some(B160::ZERO)`.

The code then proceeds without any zero-address guard:

```rust
// 4. Read authority account
let account_properties = resources.with_infinite_ergs(|inf_ergs| {
    system.io.read_account_properties(
        ExecutionEnvironmentType::NoEE,
        inf_ergs,
        &authority,   // <-- could be B160::ZERO
        ...
    )
})?;
// 5. Check authority is not a contract
if account_properties.is_contract() { return Ok(false); }
// 6. Check nonce
if account_properties.nonce.0 != auth_nonce { return Ok(false); }
...
// 8. Set code for authority
resources.with_infinite_ergs(|inf_ergs| {
    system.io.set_delegation(inf_ergs, &authority, &delegation_address)  // authority == B160::ZERO
})?;
// 9. Bump nonce
system.io.increment_nonce(..., &authority, 1)  // authority == B160::ZERO
``` [1](#0-0) [2](#0-1) 

The `recover_authority` function in the same file only guards against an empty output slice, not against a zero-valued address:

```rust
if ecrecover_output.is_empty() {
    Ok(None)
} else {
    Ok(Some(
        B160::try_from_be_slice(&ecrecover_output.build()[12..])...
    ))
}
``` [3](#0-2) 

The EIP-7702 specification (EIP-7702, step 3) explicitly requires: "If the authority is the zero address, return false." This check is absent in ZKsync OS.

---

### Impact Explanation

An attacker submits an EIP-7702 type-4 transaction with a crafted authorization entry whose `(y_parity, r, s)` signature causes `ecrecover` to return the zero address. Because address zero has nonce 0 and no bytecode, checks 5 and 6 pass. The bootloader then:

1. Calls `set_delegation(inf_ergs, &B160::ZERO, &delegation_address)` — writing a `0xef0100 || <attacker_address>` delegation marker into the zero address's account state.
2. Calls `increment_nonce(..., &B160::ZERO, 1)` — bumping the nonce of address zero.

This corrupts the canonical state of address zero. Because address zero is the burn/null address used by many EVM contracts as a sentinel (e.g., `transfer(address(0), ...)` checks, `ecrecover` failure detection), poisoning its state can cause downstream EVM contracts that rely on address-zero being empty/non-delegated to behave incorrectly. It also constitutes a state-transition bug: the ZKsync OS state root will diverge from what a correct EIP-7702 implementation would produce, creating a forward/proving divergence.

---

### Likelihood Explanation

The attack requires only a valid EIP-7702 type-4 transaction (unprivileged, any EOA can submit one). Crafting a signature that makes `ecrecover` return the zero address is a known cryptographic technique (it requires finding a specific `(r, s)` pair for the target message, which is computationally hard in general but the zero address is a special case that may be reachable via degenerate inputs). Even if direct crafting is hard, the missing check is a spec deviation that is reachable in principle and represents a correctness gap that must be closed per EIP-7702.

---

### Recommendation

Add an explicit zero-address check for the recovered `authority` immediately after `recover_authority` returns, before any state-mutating operations:

```rust
let Some(authority) = resources
    .with_infinite_ergs(|inf_ergs| recover_authority(system, inf_ergs, auth_sig_data, &msg))?
else {
    return Ok(false);
};

// EIP-7702 spec: if authority is the zero address, skip this entry
if authority == B160::ZERO {
    return Ok(false);
}
``` [1](#0-0)

### Proof of Concept

1. Construct an EIP-7702 type-4 transaction with a single authorization entry `[chain_id=0, address=<any>, nonce=0, y_parity, r, s]` where `(y_parity, r, s)` is chosen so that `ecrecover(msg, y_parity+27, r, s)` returns the zero address (or use a degenerate all-zero signature that the precompile maps to zero output — noting the `is_empty()` guard only catches the no-output case, not the zero-address output case).
2. Submit the transaction from any funded EOA.
3. Observe that `validate_and_apply_delegation` passes checks 5 (zero address is not a contract) and 6 (nonce of zero address is 0, matching `auth_nonce=0`).
4. `set_delegation` is called on `B160::ZERO` with the attacker-chosen `delegation_address`, writing `0xef0100 || delegation_address` as the bytecode of address zero.
5. `increment_nonce` is called on `B160::ZERO`, setting its nonce to 1.
6. The state root now reflects a non-empty, delegated address zero — a state that no correct EIP-7702 implementation would produce. [4](#0-3) [5](#0-4)

fn validate_and_apply_delegation<S: EthereumLikeTypes>(
    system: &mut System<S>,
    resources: &mut S::Resources,
    auth_chain_id: &U256,
    auth_nonce: u64,
    delegation_address: &[u8; 20],
    auth_sig_data: (u8, &[u8], &[u8]),
    hasher: &mut crypto::sha3::Keccak256,
) -> Result<bool, TxError>
where
    S::IO: IOSubsystemExt,
{
    let chain_id = system.get_chain_id();

    // 0. Pre-charge intrinsic gas
    resources.charge(&S::Resources::from_ergs_and_native(
        Ergs(evm_interpreter::gas_constants::NEWACCOUNT * ERGS_PER_GAS),
        <<S::Resources as Resources>::Native as zk_ee::system::Computational>::from_computational(
            crate::bootloader::constants::PER_AUTH_NATIVE_COMPUTATIONAL_OVERHEAD,
        ),
    ))?;

    // 1. Check chain id
    if !auth_chain_id.is_zero() && auth_chain_id != &U256::from(chain_id) {
        return Ok(false);
    }
    // 2. Check for nonce overflow
    if auth_nonce == u64::MAX {
        return Ok(false);
    }
    // 3. Signature
    // EIP-2 check
    let (_, _, auth_s) = auth_sig_data;
    let s = U256::try_from_be_slice(auth_s)
        .ok_or::<TxError>(InvalidTransaction::InvalidStructure.into())?;
    if s > crypto::secp256k1::SECP256K1N_HALF_U256 {
        return Ok(false);
    }
    let msg = resources.with_infinite_ergs(|inf_ergs| {
        compute_auth_message_signed_hash::<S>(
            inf_ergs,
            auth_chain_id,
            auth_nonce,
            delegation_address,
            hasher,
        )
    })?;
    let Some(authority) = resources
        .with_infinite_ergs(|inf_ergs| recover_authority(system, inf_ergs, auth_sig_data, &msg))?
    else {
        return Ok(false);
    };

    // 4. Read authority account
    // Gas already charged in intrinsic
    let account_properties = resources.with_infinite_ergs(|inf_ergs| {
        system.io.read_account_properties(
            ExecutionEnvironmentType::NoEE,
            inf_ergs,
            &authority,
            AccountDataRequest::empty()
                .with_nonce()
                .with_nominal_token_balance()
                .with_is_delegated()
                .with_has_bytecode(),
        )
    })?;
    // 5. Check authority is not a contract
    if account_properties.is_contract() {
        return Ok(false);
    }
    // 6. Check nonce
    if account_properties.nonce.0 != auth_nonce {
        return Ok(false);
    }
    // 7. Add refund if authority is not empty.
    let is_empty = account_properties.nonce.0 == 0
        && account_properties.has_bytecode() == false
        && account_properties.nominal_token_balance.0.is_zero();

    if !is_empty {
        let ergs = Ergs(
            (evm_interpreter::gas_constants::NEWACCOUNT
                - evm_interpreter::gas_constants::PER_AUTH_BASE_COST)
                * ERGS_PER_GAS,
        );
        system
            .io
            .add_to_refund_counter(S::Resources::from_ergs(ergs))?
    }

    let delegation_address = B160::from_be_bytes(*delegation_address);
    system_log!(
        system,
        "Will delegate address 0x{:040x} -> 0x{:040x}\n",
        authority.as_uint(),
        delegation_address.as_uint()
    );

    // 8. Set code for authority, system function
    //    will handle the two cases (unsetting).
    resources.with_infinite_ergs(|inf_ergs| {
        system
            .io
            .set_delegation(inf_ergs, &authority, &delegation_address)
    })?;
    // 9.Bump nonce
    resources
        .with_infinite_ergs(|inf_ergs| {
            system
                .io
                .increment_nonce(ExecutionEnvironmentType::NoEE, inf_ergs, &authority, 1)
        })
        .map_err(|e| -> BootloaderSubsystemError {
            match e {
                SubsystemError::LeafUsage(InterfaceError(NonceError::NonceOverflow, _)) => {
                    internal_error!("Cannot overflow, already checked").into()
                }
                _ => wrap_error!(e),
            }
        })?;
    Ok(true)
}

fn recover_authority<S: EthereumLikeTypes>(
    system: &mut System<S>,
    resources: &mut S::Resources,
    auth_sig_data: (u8, &[u8], &[u8]),
    msg: &[u8; 32],
) -> Result<Option<B160>, TxError> {
    use zk_ee::system::SystemFunctions;
    let mut ecrecover_input = [0u8; 128];
    let (parity, r, s) = auth_sig_data;
    if parity > 1 {
        return Ok(None);
    }
    ecrecover_input[0..32].copy_from_slice(msg);
    ecrecover_input[63] = parity + 27;
    ecrecover_input[64..96][(32 - r.len())..].copy_from_slice(r);
    ecrecover_input[96..128][(32 - s.len())..].copy_from_slice(s);
    let mut ecrecover_output = ArrayBuilder::default();
    // Recover is counted in intrinsic gas
    resources
        .with_infinite_ergs(|inf_ergs| {
            S::SystemFunctions::secp256k1_ec_recover(
                ecrecover_input.as_slice(),
                &mut ecrecover_output,
                inf_ergs,
                system.get_allocator(),
            )
        })
        .map_err(SystemError::from)?;
    if ecrecover_output.is_empty() {
        Ok(None)
    } else {
        Ok(Some(
            B160::try_from_be_slice(&ecrecover_output.build()[12..])
                .ok_or(internal_error!("Invalid ecrecover return value"))?,
        ))
    }
