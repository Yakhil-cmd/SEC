Audit Report

## Title
Deferred Async Initialization Leaves `API_BOUNDARY_NODE_PRINCIPALS` Empty, Causing Denial of Salt Access During Every Install/Upgrade Window - (`File: rs/boundary_node/salt_sharing/canister/canister.rs`)

## Summary
The `salt_sharing` canister's `#[init]` and `#[post_upgrade]` handlers defer all initialization — including populating the `API_BOUNDARY_NODE_PRINCIPALS` access-control set — to an async timer chain. Because `API_BOUNDARY_NODE_PRINCIPALS` is a non-stable `thread_local!` `HashSet` that starts empty and is reset on every upgrade, there is a guaranteed window of at least `registry_polling_interval_secs` (300 seconds on mainnet) after every install and upgrade during which all callers, including legitimate API boundary nodes, are rejected with `Unauthorized` or `message_inspection_failed`. The salt value itself survives upgrades in stable memory, so the rejection is not due to missing salt but solely due to the empty principals set.

## Finding Description
**Root cause — deferred async init chain:**

`#[init]` schedules `init_async` via a zero-delay timer rather than executing synchronously: [1](#0-0) 

`#[post_upgrade]` calls `init()` verbatim, applying the same deferred pattern: [2](#0-1) 

`init_async` first awaits `try_regenerate_salt()` (an async call to the management canister `raw_rand`), then sets up a periodic interval timer. The first firing of `poll_api_boundary_nodes` is deferred by the full `registry_polling_interval_secs` period: [3](#0-2) 

`poll_api_boundary_nodes` is the sole writer to `API_BOUNDARY_NODE_PRINCIPALS`: [4](#0-3) 

**The storage variable starts empty and is not in stable memory:** [5](#0-4) 

`SALT` is a `StableBTreeMap` and survives upgrades, but `API_BOUNDARY_NODE_PRINCIPALS` is a plain `thread_local!` heap variable that is reset to an empty `HashSet` on every upgrade.

**Access control gates on this empty set:**

`inspect_message` rejects all update-mode callers when the set is empty: [6](#0-5) 

`get_salt` returns `Unauthorized` for every caller when the set is empty: [7](#0-6) 

`is_api_boundary_node_principal` reads only the in-memory set: [8](#0-7) 

**Vulnerable window timeline:**
1. `#[init]`/`#[post_upgrade]` returns → canister is live.
2. Zero-delay timer fires next round → `init_async` starts.
3. `init_async` awaits `raw_rand` (multiple rounds).
4. `set_timer_interval(period, ...)` is registered — first `poll_api_boundary_nodes` fires only after `period` = 300 s.
5. Only after step 4's first interval fires and the registry call returns does `API_BOUNDARY_NODE_PRINCIPALS` get populated.

The integration test explicitly confirms the deferred initialization behavior: [9](#0-8) 

The mainnet deployment proposal confirms `registry_polling_interval_secs = 300`: [10](#0-9) 

## Impact Explanation
During the initialization window on every install and every upgrade, all API boundary nodes receive `GetSaltError::Unauthorized` from `get_salt` queries and `message_inspection_failed` from update-mode calls. On upgrades, the salt value is already present in stable memory (`SALT` is a `StableBTreeMap`), so `is_salt_init()` returns `true`, but `API_BOUNDARY_NODE_PRINCIPALS` is wiped. Boundary nodes receive `Unauthorized` — not `SaltNotInitialized` — giving no signal to retry with a different strategy. On mainnet, this window is at least 5 minutes per upgrade. The salt is used by API boundary nodes to consistently anonymize sensitive user data (IP addresses); denial of the salt during the upgrade window causes failed or inconsistent anonymization across the entire boundary node fleet. This matches the allowed impact: **High — Significant boundary/API infrastructure security impact with concrete user or protocol harm**.

## Likelihood Explanation
The condition is triggered deterministically on every canister install and every upgrade — no attacker action is required. The mainnet deployment proposal confirms `registry_polling_interval_secs = 300`, making the window at least 5 minutes. Any NNS governance-approved canister upgrade (a routine operational event) triggers the window. No special privileges, victim mistakes, or external compromise are required beyond the normal upgrade lifecycle.

## Recommendation
1. **Persist `API_BOUNDARY_NODE_PRINCIPALS` in stable memory** (e.g., as a `StableBTreeMap` or serialized in a stable cell) so it survives upgrades and is immediately available after `post_upgrade`, eliminating the window for the upgrade case.
2. **Schedule an immediate `poll_api_boundary_nodes` call** with `set_timer(Duration::ZERO, ...)` in addition to the periodic interval, so the set is populated in the first round after init rather than after `registry_polling_interval_secs`.
3. **Return `SaltNotInitialized` (or a distinct `Initializing` error) instead of `Unauthorized`** when `API_BOUNDARY_NODE_PRINCIPALS` is empty but the canister is still initializing, so callers can distinguish a transient state from a permanent access denial.

## Proof of Concept
1. Deploy the salt-sharing canister with `registry_polling_interval_secs = 300`.
2. Immediately (within the first 300 seconds) call `get_salt` as a query from a principal that is a registered API boundary node.
3. Observe `GetSaltError::Unauthorized` — even though the caller is a legitimate API boundary node and the salt may already be in stable memory.
4. Upgrade the canister. Repeat step 2 immediately after upgrade. Observe the same `Unauthorized` response for up to 5 minutes, despite the salt being present in stable memory from before the upgrade.

This is directly reproducible using the existing PocketIC integration test harness in `rs/boundary_node/salt_sharing/integration_tests/tests/salt_sharing_canister_tests.rs` by asserting that a known API boundary node principal receives `Unauthorized` immediately after install/upgrade (before `tick_n_times` is called).

fn get_salt() -> GetSaltResponse {
    let caller_id = caller();
    if is_api_boundary_node_principal(&caller_id) {
        let stored_salt = SALT
            .with(|cell| cell.borrow().get(&()))
            .ok_or(GetSaltError::SaltNotInitialized)?;

        return Ok(SaltResponse {
            salt: stored_salt.salt,
            salt_id: stored_salt.salt_id,
        });
    }
    Err(GetSaltError::Unauthorized)

        && let Err(err) = try_regenerate_salt().await
    {

    // Start salt generation schedule based on the argument.

    }

    set_timer_interval(period, async || poll_api_boundary_nodes().await);
}
