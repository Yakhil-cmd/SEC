### Title
Unbounded `EndBlocker` Iteration via Governance Proposal Flooding — (`sei-cosmos/x/gov/abci.go`)

---

### Summary

There is no cap on the number of active governance proposals in Sei's governance module. An attacker with sufficient tokens can submit an unbounded number of proposals that all expire in the same block, forcing `EndBlocker` to call the expensive `Tally()` function for each one in a single block execution. Since rejected proposals refund the deposit, the attacker recovers their capital after the attack, making the net cost only transaction fees and temporary token lockup.

---

### Finding Description

`SubmitProposalWithExpedite` imposes no limit on the number of concurrent proposals: [1](#0-0) 

There is no `MaxActiveProposals` parameter anywhere in the governance params: [2](#0-1) 

`EndBlocker` iterates over **all** proposals whose voting period has expired in the current block with no iteration cap: [3](#0-2) 

For each such proposal, `Tally()` is called. `Tally()` performs three nested iterations:
1. All bonded validators (`IterateBondedValidatorsByPower`)
2. All votes on the proposal (`IterateVotes`)
3. For each voter, all their delegations (`IterateDelegations`) [4](#0-3) 

If N proposals all expire in the same block, the EndBlocker executes O(N × V × D) work (N proposals, V validators, D delegations per voter) with no bound.

Additionally, the same unbounded pattern exists for the inactive queue — all proposals submitted in the same block expire at the same `DepositEndTime`: [5](#0-4) 

---

### Impact Explanation

Sei targets ~400ms block times. If the `EndBlocker` processes hundreds of proposals simultaneously — each requiring full validator-set iteration — block processing time can exceed the block time budget, causing:

- Block production delays exceeding 2.5s (Medium per scope)
- Potential proposer timeouts (≥10 min freeze if the attack is sustained across multiple voting cycles)

The inactive-queue variant (no MinDeposit required to reach voting period) is cheaper to execute since only a positive deposit is needed; those deposits are burned on expiry, but the attacker can minimize loss by depositing the minimum positive amount.

The active-queue variant (full MinDeposit) is nearly free in net cost: rejected proposals (which fail quorum since the attacker does not vote) trigger `RefundDeposits`, returning all locked tokens: [6](#0-5) 

---

### Likelihood Explanation

- **Unprivileged path**: any account can call `MsgSubmitProposal` / `submitProposal` precompile.
- **Capital requirement**: `DefaultMinDepositTokens = 10,000,000 usei` (10 SEI) per proposal. Submitting 500 proposals requires 5,000 SEI locked for 2 days, fully refunded afterward.
- **Timing**: all proposals submitted in the same block share the same `VotingEndTime`, so they all expire together.
- **Repeatability**: the attacker can repeat the cycle every 2 days with the same capital. [7](#0-6) 

---

### Recommendation

1. **Add a `MaxActiveProposals` governance parameter** and enforce it in `SubmitProposalWithExpedite` — reject new proposals when the active count is at the limit.
2. **Cap `EndBlocker` iterations per block** — process at most K expired proposals per block and carry the remainder to the next block.
3. **Increase `MinDeposit`** to raise the capital cost of flooding, or introduce a per-address proposal rate limit.

---

### Proof of Concept

```
Block B (same block):
  attacker → MsgSubmitProposal(deposit = MinDeposit) × N   // all proposals submitted
  → each proposal enters VotingPeriod immediately (deposit meets MinDeposit)
  → all proposals share VotingEndTime = B.time + VotingPeriod (2 days)

Block B + VotingPeriod:
  EndBlocker fires
  → IterateActiveProposalsQueue iterates N proposals (no cap)
  → for each proposal: Tally() called
      → IterateBondedValidatorsByPower (all validators)
      → IterateVotes (0 votes, but validator loop still runs)
  → N × O(V) work in a single block

After EndBlocker:
  → all N proposals rejected (no quorum)
  → RefundDeposits called for each → attacker recovers all MinDeposit tokens
  → net attacker cost: only tx fees
``` [8](#0-7) [3](#0-2)

func (keeper Keeper) SubmitProposalWithExpedite(ctx sdk.Context, content types.Content, isExpedited bool) (types.Proposal, error) {
	if !keeper.router.HasRoute(content.ProposalRoute()) {
		return types.Proposal{}, sdkerrors.Wrap(types.ErrNoProposalHandlerExists, content.ProposalRoute())
	}
	// Ensure that the parameter exists
	if content.ProposalType() == proposal.ProposalTypeChange {
		paramProposal, ok := content.(*proposal.ParameterChangeProposal)
		if !ok {
			return types.Proposal{}, sdkerrors.Wrap(types.ErrInvalidProposalContent, "proposal content is not a ParameterChangeProposal")
		}

		// Validate each parameter change exists
		for _, change := range paramProposal.Changes {
			subspace, ok := keeper.paramsKeeper.GetSubspace(change.Subspace)
			if !ok {
				return types.Proposal{}, sdkerrors.Wrapf(types.ErrInvalidProposalContent, "parameter %s/%s does not exist", change.Subspace, change.Key)
			}
			validKey := subspace.Has(ctx, []byte(change.Key))
			if !validKey {
				return types.Proposal{}, sdkerrors.Wrapf(types.ErrInvalidProposalContent, "parameter %s not found in subspace %s", change.Key, change.Subspace)
			}
		}
	}

	proposalID, err := keeper.GetProposalID(ctx)
	if err != nil {
		return types.Proposal{}, err
	}

	submitTime := ctx.BlockHeader().Time
	depositPeriod := keeper.GetDepositParams(ctx).MaxDepositPeriod

	proposal, err := types.NewProposal(content, proposalID, submitTime, submitTime.Add(depositPeriod), isExpedited)
	if err != nil {
		return types.Proposal{}, err
	}

	keeper.SetProposal(ctx, proposal)
	keeper.InsertInactiveProposalQueue(ctx, proposalID, proposal.DepositEndTime)
	keeper.SetProposalID(ctx, proposalID+1)

	// called right after a proposal is submitted
	keeper.AfterProposalSubmission(ctx, proposalID)

	ctx.EventManager().EmitEvent(
		sdk.NewEvent(
			types.EventTypeSubmitProposal,
			sdk.NewAttribute(types.AttributeKeyProposalID, fmt.Sprintf("%d", proposalID)),
		),
	)

	return proposal, nil
}

func (keeper Keeper) Tally(ctx sdk.Context, proposal types.Proposal) (passes bool, burnDeposits bool, tallyResults types.TallyResult) {
	results := make(map[types.VoteOption]sdk.Dec)
	results[types.OptionYes] = sdk.ZeroDec()
	results[types.OptionAbstain] = sdk.ZeroDec()
	results[types.OptionNo] = sdk.ZeroDec()
	results[types.OptionNoWithVeto] = sdk.ZeroDec()

	totalVotingPower := sdk.ZeroDec()
	currValidators := make(map[string]types.ValidatorGovInfo)

	// fetch all the bonded validators, insert them into currValidators
	keeper.sk.IterateBondedValidatorsByPower(ctx, func(index int64, validator stakingtypes.ValidatorI) (stop bool) {
		currValidators[validator.GetOperator().String()] = types.NewValidatorGovInfo(
			validator.GetOperator(),
			validator.GetBondedTokens(),
			validator.GetDelegatorShares(),
			sdk.ZeroDec(),
			types.WeightedVoteOptions{},
		)

		return false
	})

	keeper.IterateVotes(ctx, proposal.ProposalId, func(vote types.Vote) bool {
		// if validator, just record it in the map
		voter := sdk.MustAccAddressFromBech32(vote.Voter)

		valAddrStr := sdk.ValAddress(voter.Bytes()).String()
		if val, ok := currValidators[valAddrStr]; ok {
			val.Vote = vote.Options
			currValidators[valAddrStr] = val
		}

		// iterate over all delegations from voter, deduct from any delegated-to validators
		keeper.sk.IterateDelegations(ctx, voter, func(index int64, delegation stakingtypes.DelegationI) (stop bool) {
			valAddrStr := delegation.GetValidatorAddr().String()

			if val, ok := currValidators[valAddrStr]; ok {
				// There is no need to handle the special case that validator address equal to voter address.
				// Because voter's voting power will tally again even if there will deduct voter's voting power from validator.
				val.DelegatorDeductions = val.DelegatorDeductions.Add(delegation.GetShares())
				currValidators[valAddrStr] = val

				// delegation shares * bonded / total shares
				votingPower := delegation.GetShares().MulInt(val.BondedTokens).Quo(val.DelegatorShares)

				for _, option := range vote.Options {
					subPower := votingPower.Mul(option.Weight)
					results[option.Option] = results[option.Option].Add(subPower)
				}
				totalVotingPower = totalVotingPower.Add(votingPower)
			}

			return false
		})

		keeper.deleteVote(ctx, vote.ProposalId, voter)
		return false
	})
