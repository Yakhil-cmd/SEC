### Title
Missing Upper-Bound Validation on `vesting_period_seconds` in SNS `NeuronDistribution` - (File: rs/sns/init/src/distributions.rs)

### Summary
The `validate_neurons` function in `rs/sns/init/src/distributions.rs` validates `dissolve_delay_seconds` against `max_dissolve_delay_seconds`, but performs **no validation whatsoever on `vesting_period_seconds`** in each `NeuronDistribution`. An SNS creator can supply an astronomically large `vesting_period_seconds` (e.g., `u64::MAX - created_timestamp_seconds`) that permanently locks developer neurons in the vesting state, causing a permanent DoS on those neurons' management operations and misleading swap participants about the actual lock-up commitment.

### Finding Description

`NeuronDistribution.vesting_period_seconds` is an `optional uint64` field with no enforced upper bound anywhere in the validation pipeline.

The `validate_neurons` function checks:
- Missing controllers
- Duplicate neurons
- Max developer distribution count
- Memo range
- At least one voting neuron
- `dissolve_delay_seconds <= max_dissolve_delay_seconds`

But it never checks `vesting_period_seconds`: [1](#0-0) 

The field is defined as unbounded: [2](#0-1) 

The `is_vesting` check in SNS governance uses `created_timestamp_seconds + vesting_period_seconds > now_seconds`. With a value such as `u64::MAX - created_timestamp_seconds`, the sum approaches `u64::MAX`, which will always exceed any realistic `now_seconds`, permanently locking the neuron in vesting state: [3](#0-2) 

A vesting neuron cannot call `StartDissolving`, `Disburse`, `Split`, `MergeMaturity`, or `DisburseMaturity`: [4](#0-3) 

The `validate_neurons` function is called from `FractionalDeveloperVotingPower::validate`, which is the sole validation gate for the initial token distribution: [5](#0-4) 

By contrast, `dissolve_delay_seconds` is properly bounded: [6](#0-5) 

No analogous bound exists for `vesting_period_seconds`.

### Impact Explanation

An SNS creator who sets `vesting_period_seconds` to a near-`u64::MAX` value for developer neurons causes:

1. **Permanent DoS on developer neuron management**: The neurons can never exit the vesting state. `StartDissolving`, `Disburse`, `Split`, and maturity operations are all blocked for the lifetime of the SNS.
2. **Misleading swap participants**: Token buyers rely on the vesting period as a commitment signal. An effectively infinite vesting period is indistinguishable from a legitimate multi-decade commitment at the UI level, but the SNS creator may have intended to deceive (e.g., by setting a value that overflows to 0 in a future code path, or by exploiting the permanent lock to claim governance control).
3. **Governance power concentration**: Permanently vesting developer neurons retain their `voting_power_percentage_multiplier` and can still vote, giving the developer permanent outsized governance influence with no ability for the community to dilute it through normal neuron lifecycle operations.

### Likelihood Explanation

The entry path is a canister developer/SNS creator submitting a `CreateServiceNervousSystem` NNS proposal. The NNS governance voters review the proposal payload but the `vesting_period_seconds` field is not surfaced prominently in any UI or validation warning. The field is `optional` and its absence is the common case (most test fixtures use `None`), so reviewers are unlikely to scrutinize an unusually large value. The likelihood is **medium**: it requires an SNS creator with malicious or mistaken intent, but the NNS approval process provides no automated guard.

### Recommendation

Add a maximum bound check on `vesting_period_seconds` inside `validate_neurons`, analogous to the existing check on `dissolve_delay_seconds`. A reasonable ceiling is `max_dissolve_delay_seconds` (the SNS-configured maximum, e.g., 8 years by default) or a protocol-wide constant. Example addition to `validate_neurons`:

```rust
for neuron_distribution in &developer_distribution.developer_neurons {
    if let Some(vesting_period) = neuron_distribution.vesting_period_seconds {
        if vesting_period > *max_dissolve_delay_seconds {
            return Err(format!(
                "Error: vesting_period_seconds ({}) exceeds max_dissolve_delay_seconds ({})",
                vesting_period, max_dissolve_delay_seconds
            ));
        }
    }
}
```

### Proof of Concept

1. Craft a `SnsInitPayload` with a `NeuronDistribution` where `vesting_period_seconds = Some(u64::MAX - 1_700_000_000)` (approximately `u64::MAX - current_unix_time`).
2. Call `FractionalDeveloperVotingPower::validate(&nervous_system_parameters)` — it returns `Ok(())` with no error.
3. Deploy the SNS via `CreateServiceNervousSystem` NNS proposal.
4. After SNS genesis, the developer neuron has `created_timestamp_seconds ≈ 1_700_000_000`. The vesting check evaluates `1_700_000_000 + (u64::MAX - 1_700_000_000) = u64::MAX > now_seconds` — always true.
5. Any attempt by the developer to call `StartDissolving`, `Disburse`, `Split`, or `DisburseMaturity` on the neuron is rejected with `"Neuron ... is vesting and cannot call ..."`.
6. The neuron remains permanently locked in vesting state for the lifetime of the SNS. [1](#0-0) [7](#0-6)

    /// Validate the NeuronDistributions in the developer bucket.
    fn validate_neurons(
        &self,
        developer_distribution: &DeveloperDistribution,
        nervous_system_parameters: &NervousSystemParameters,
    ) -> Result<(), String> {
        let neuron_minimum_dissolve_delay_to_vote_seconds = nervous_system_parameters
            .neuron_minimum_dissolve_delay_to_vote_seconds
            .as_ref()
            .expect("Expected NervousSystemParameters.neuron_minimum_dissolve_delay_to_vote_seconds to be set");

        let max_dissolve_delay_seconds = nervous_system_parameters
            .max_dissolve_delay_seconds
            .as_ref()
            .expect("Expected NervousSystemParameters.max_dissolve_delay_seconds to be set");

        let missing_developer_principals_count = developer_distribution
            .developer_neurons
            .iter()
            .filter(|neuron_distribution| neuron_distribution.controller.is_none())
            .count();

        if missing_developer_principals_count != 0 {
            return Err(format!(
                "Error: {missing_developer_principals_count} developer_neurons are missing controllers"
            ));
        }

        let deduped_dev_neurons = developer_distribution
            .developer_neurons
            .iter()
            .map(|neuron_distribution| {
                (
                    (neuron_distribution.controller, neuron_distribution.memo),
                    neuron_distribution.stake_e8s,
                )
            })
            .collect::<BTreeMap<_, _>>();

        if deduped_dev_neurons.len() != developer_distribution.developer_neurons.len() {
            return Err(
                "Error: Neurons with the same controller and memo found in developer_neurons"
                    .to_string(),
            );
        }

        if deduped_dev_neurons.len() > MAX_DEVELOPER_DISTRIBUTION_COUNT {
            return Err(format!(
                "Error: The number of developer neurons must be less than {}. Current count is {}",
                MAX_DEVELOPER_DISTRIBUTION_COUNT,
                deduped_dev_neurons.len(),
            ));
        }

        for (controller, memo) in deduped_dev_neurons.keys() {
            if NEURON_BASKET_MEMO_RANGE_START <= *memo && *memo <= SALE_NEURON_MEMO_RANGE_END {
                return Err(format!(
                    "Error: Developer neuron with controller {} cannot have a memo in the range {} to {}",
                    controller.unwrap(),
                    NEURON_BASKET_MEMO_RANGE_START,
                    SALE_NEURON_MEMO_RANGE_END
                ));
            }
        }

        let configured_at_least_one_voting_neuron = developer_distribution
            .developer_neurons
            .iter()
            .any(|neuron_distribution| {
                neuron_distribution.dissolve_delay_seconds
                    >= *neuron_minimum_dissolve_delay_to_vote_seconds
            });

        if !configured_at_least_one_voting_neuron {
            return Err(format!(
                "Error: There needs to be at least one voting-eligible neuron configured. To be \
                 eligible to vote, a neuron must have dissolve_delay_seconds of at least {neuron_minimum_dissolve_delay_to_vote_seconds}"
            ));
        }

        let misconfigured_dissolve_delay_principals: Vec<PrincipalId> = developer_distribution
            .developer_neurons
            .iter()
            .filter(|neuron_distribution| {
                neuron_distribution.dissolve_delay_seconds > *max_dissolve_delay_seconds
            })
            .map(|neuron_distribution| neuron_distribution.controller.unwrap())
            .collect();

        if !misconfigured_dissolve_delay_principals.is_empty() {
            return Err(format!(
                "Error: The following PrincipalIds have a dissolve_delay_seconds configured greater than \
                 the allowed max_dissolve_delay_seconds ({max_dissolve_delay_seconds}): {misconfigured_dissolve_delay_principals:?}"
            ));
        }

        Ok(())
    }

  //
  // A neuron that is vesting is non-dissolving and cannot start dissolving until the vesting duration has elapsed.

  // project.
