Audit Report

## Title
Markdown Injection via Unsanitized UTF-8 Memo in ICRC-21 `GenericDisplayMessage` — (`packages/icrc-ledger-types/src/icrc21/responses.rs`)

## Summary
The `add_memo` function in `responses.rs` inserts a user-controlled memo byte buffer directly into a markdown format string without sanitizing backtick characters. When the memo is valid UTF-8 and contains a backtick, it prematurely closes the surrounding inline code span, allowing arbitrary markdown to render outside the code fence in the `GenericDisplayMessage` consent message. Because ICRC-21 consent messages are the primary human-readable security boundary shown to users before signing a transaction, this allows a malicious dApp to display a fabricated (lower) amount while the actual transaction encodes a much larger one.

## Finding Description
In `packages/icrc-ledger-types/src/icrc21/responses.rs`, the `add_memo` function at line 288 performs only a UTF-8 validity check on the raw memo bytes:

```rust
let memo_str = match std::str::from_utf8(memo.as_slice()) {
    Ok(valid_str) => valid_str.to_string(),
    Err(_) => hex::encode(memo.as_slice()),
};
``` [1](#0-0) 

If valid UTF-8, the string is interpolated verbatim into a markdown inline code span at line 294:

```rust
message.push_str(&format!("\n\n**Memo:**\n`{memo_str}`"));
``` [2](#0-1) 

No backtick escaping or markdown sanitization is applied. A memo containing a backtick (e.g., the UTF-8 bytes for `` x`\n\n**Amount:** `1 ICP ``) closes the inline code span early and injects arbitrary markdown fields into the rendered consent message.

The full call chain is reachable by any unprivileged principal:

1. `icrc21_canister_call_consent_message` — public `#[update]` endpoint, no authentication beyond a valid caller identity. [3](#0-2) 
2. For `icrc1_transfer`, the memo from the decoded `TransferArg` is passed as `GenericMemo::Icrc1Memo(memo.0)` directly to `display_message_builder.with_memo(...)`. [4](#0-3) 
3. `ConsentMessageBuilder::build` calls `message.add_memo(memo)`, which reaches the unsanitized format string. [5](#0-4) 

The `FieldsDisplayMessage` variant is not affected because the memo is stored as a structured `Value::Text` field rather than rendered as markdown. [6](#0-5) 

## Impact Explanation
The ICRC-21 consent message is the primary security mechanism that wallets use to show users a human-readable description of a transaction before signing. A malicious dApp can craft an `icrc1_transfer` with a large amount (e.g., 1000 ICP) and embed a memo that injects a fake `**Amount:** \`1 ICP\`` field into the rendered markdown. A user who reads the consent message sees a fabricated lower amount and approves a transaction that transfers far more than intended. This constitutes **significant ledger security impact with concrete user financial harm**, mapping to the **High** bounty tier ($2,000–$10,000): unauthorized ledger fund transfer enabled by a broken security control in the ICP/ICRC ledger stack.

## Likelihood Explanation
The endpoint is public with no privilege requirement beyond a valid caller identity. The memo field accepts up to 32 bytes of arbitrary bytes by default, which is sufficient to inject convincing fake fields. The attack requires only constructing a `ConsentMessageRequest` with a crafted `TransferArg` memo — no key material, no governance majority, no node control. Any wallet that renders `GenericDisplayMessage` as markdown (the intended use per the ICRC-21 spec) is directly exposed. The attack is repeatable and low-cost.

## Recommendation
Escape backtick characters in `memo_str` before interpolating into the markdown format string:

```rust
let safe_memo = memo_str.replace('`', "\\`");
message.push_str(&format!("\n\n**Memo:**\n`{safe_memo}`"));
```

Alternatively, unconditionally hex-encode all memo values in the `GenericDisplayMessage` path to eliminate the UTF-8 display path entirely, removing any possibility of markdown injection regardless of memo content.

## Proof of Concept
Construct a `TransferArg` with `memo` set to the UTF-8 bytes of:
```
x`\n\n**Amount:** `1 ICP
```
(24 bytes — within the default 32-byte limit). Call `icrc21_canister_call_consent_message` with `method = "icrc1_transfer"` and this arg (with `amount` set to e.g. 100_000_000_000 e8s = 1000 ICP). The returned `GenericDisplayMessage` will contain injected markdown rendering a fake `**Amount:** \`1 ICP\`` field visually indistinguishable from the real one. A unit test in `packages/icrc-ledger-types/src/icrc21/` can assert that `build_icrc21_consent_info` with this memo produces a `GenericDisplayMessage` whose string does not contain unescaped backtick sequences outside the memo's code span.
