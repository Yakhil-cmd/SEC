Audit Report

## Title
NNS Neurons' Fund Members Can Exit Before Proposal Execution to Avoid Maturity Draw - (File: rs/nns/governance/src/governance.rs)

## Summary

When a `CreateServiceNervousSystem` proposal is adopted and executed, `draw_maturity_from_neurons_fund` reads the live Neurons' Fund membership list at execution time rather than a snapshot taken at proposal submission. Any neuron controller can call `LeaveCommunityFund` during the voting period (days to weeks), causing their neuron to be excluded from the maturity draw at execution, then immediately rejoin after the swap settles. This allows a neuron owner to avoid the maturity lock-up entirely while remaining in the fund for all future swaps.

## Finding Description

**Root cause — live membership read at execution time.**

`do_create_service_nervous_system` calls `draw_maturity_from_neurons_fund` at proposal execution time: [1](#0-0) 

Inside `draw_maturity_from_neurons_fund`, the live membership list is fetched via `list_active_neurons_fund_neurons()`: [2](#0-1) 

`list_active_neurons_fund_neurons` iterates over neurons whose `joined_community_fund_timestamp_seconds` is non-zero at the moment of the call: [3](#0-2) 

Membership is determined solely by the `joined_community_fund_timestamp_seconds` field: [4](#0-3) 

**The exit path — `leave_community_fund` has no timing restriction.**

`leave_community_fund` simply clears the timestamp field with no guard against pending proposals: [5](#0-4) 

**The re-entry path — `join_community_fund` has no cooldown.**

After the swap settles, the neuron owner can immediately rejoin: [6](#0-5) 

Both operations are exposed as standard unprivileged `manage_neuron` `Configure` operations: [7](#0-6) 

**Attack flow:**
1. Neuron N holds large maturity and is a Neurons' Fund member.
2. A `CreateServiceNervousSystem` proposal P enters the voting period.
3. N's controller calls `LeaveCommunityFund` during the voting period.
4. P is adopted; `do_create_service_nervous_system` executes.
5. `list_active_neurons_fund_neurons()` does not include N; N's maturity is not drawn.
6. The swap runs to completion (committed or aborted).
7. N's controller calls `JoinCommunityFund`; N is back in the fund for future swaps.

No existing check prevents this. The `draw_maturity_from_neurons_fund` function has no mechanism to detect or block neurons that left after proposal submission: [8](#0-7) 

## Impact Explanation

**High — Significant NNS/SNS security impact with concrete user and protocol harm.**

The `NeuronsFundParticipationConstraints` computed at execution time and sent to the SNS swap canister reflect a smaller pool than was present when the proposal was submitted. This:
- Reduces the matched ICP available to the SNS treasury, potentially causing the swap to fall short of its minimum or deliver fewer SNS neurons than expected.
- Shifts the effective participation curve for all remaining Neurons' Fund neurons without their consent, as the polynomial is computed over the reduced total maturity.
- Allows the exiting neuron to avoid permanent maturity conversion (if the swap commits) or even temporary lock-up (if the swap aborts), with zero cost.

This fits: *High ($2,000–$10,000) — Significant NNS, SNS, or infrastructure security impact with concrete user or protocol harm.*

## Likelihood Explanation

**Medium likelihood.**

- NNS proposals have voting periods of days to weeks, providing ample time to observe a pending `CreateServiceNervousSystem` proposal via the public governance query API.
- The two required calls (`LeaveCommunityFund`, later `JoinCommunityFund`) are standard, unprivileged operations with no fees or delays.
- The attack is most profitable for large-maturity neurons whose proportional contribution to matched funding is significant.
- The attack is repeatable for every future SNS swap.

## Recommendation

Take an immutable snapshot of Neurons' Fund membership and maturity at **proposal submission time** (inside `make_proposal` or the initial proposal validation), store it in `ProposalData`, and use that frozen snapshot in `draw_maturity_from_neurons_fund` instead of calling `list_active_neurons_fund_neurons()` at execution time. This mirrors the approach already used for ballots (voting power is snapshotted when a proposal is created, not re-read at settlement).

Alternatively, introduce a lock that prevents `LeaveCommunityFund` while any `CreateServiceNervousSystem` proposal is in the `Open` or `Adopted` lifecycle state, analogous to how dissolve-delay changes are restricted during certain neuron states.

## Proof of Concept

```
# Step 1 – Neuron N (large maturity) is in the Neurons' Fund.
manage_neuron(N, Configure { JoinCommunityFund {} })   # already done

# Step 2 – Attacker observes a CreateServiceNervousSystem proposal P
#           enter the voting period (publicly visible via governance query API).

# Step 3 – Attacker exits the fund before P is adopted.
manage_neuron(N, Configure { LeaveCommunityFund {} })
# joined_community_fund_timestamp_seconds is now None for N.

# Step 4 – P is adopted; do_create_service_nervous_system executes.
#           list_active_neurons_fund_neurons() does NOT include N.
#           draw_maturity_from_neurons_fund skips N entirely.
#           SNS swap opens with reduced matched-funding commitment.

# Step 5 – Swap reaches terminal state (Committed or Aborted).
#           settle_neurons_fund_participation is called; N is unaffected.

# Step 6 – Attacker rejoins immediately.
manage_neuron(N, Configure { JoinCommunityFund {} })
# N is back in the fund, maturity intact, ready for the next swap.
```

A deterministic integration test can reproduce this by: (1) setting up a governance instance with a large-maturity Neurons' Fund neuron, (2) inserting a `CreateServiceNervousSystem` `ProposalData`, (3) calling `leave_community_fund` on the neuron, (4) calling `draw_maturity_from_neurons_fund`, and (5) asserting that the returned `NeuronsFundSnapshot` does not include the exited neuron and that its `maturity_e8s_equivalent` is unchanged.

    fn draw_maturity_from_neurons_fund(
        &mut self,
        proposal_id: &ProposalId,
        create_service_nervous_system: &CreateServiceNervousSystem,
    ) -> Result<(NeuronsFundSnapshot, NeuronsFundParticipationConstraints), GovernanceError> {
        let swap_participation_limits = create_service_nervous_system
            .swap_parameters
            .as_ref()
            .ok_or_else(|| {
                "CreateServiceNervousSystem.swap_parameters is not specified.".to_string()
            })?;
        let swap_participation_limits =
            SwapParticipationLimits::try_from_swap_parameters(swap_participation_limits)?;
        let neurons_fund_participation_limits =
            self.try_derive_neurons_fund_participation_limits()?;
        let neurons_fund = self.neuron_store.list_active_neurons_fund_neurons();
        let initial_neurons_fund_participation = PolynomialNeuronsFundParticipation::new(
            neurons_fund_participation_limits,
            swap_participation_limits,
            neurons_fund,
        )?;
        // Check that the maximum number of Neurons' Fund participants is not too high. Otherwise,
        // the SNS may be unable to distribute SNS tokens to all participants after the swap.
        {
            let maximum_neurons_fund_participants = initial_neurons_fund_participation
                .snapshot()
                .neurons()
                .len() as u64;
            if maximum_neurons_fund_participants > MAX_NEURONS_FUND_PARTICIPANTS {
                return Err(GovernanceError::new_with_message(
                    ErrorType::InvalidProposal,
                    format!(
                        "The maximum number of Neurons' Fund participants ({maximum_neurons_fund_participants}) must not exceed \
                        MAX_NEURONS_FUND_PARTICIPANTS ({MAX_NEURONS_FUND_PARTICIPANTS}).",
                    ),
                ));
            };
        }
        let constraints = initial_neurons_fund_participation.compute_constraints()?;
        let initial_neurons_fund_participation_snapshot =
            initial_neurons_fund_participation.snapshot_cloned();
        // First check if the ProposalData is available (and error out if not); then actually draw
        // the funds from the Neurons' Fund. This way, we do not need to issue refunds in case
        // of an error.
        if self.get_proposal_data(*proposal_id).is_none() {
            return Err(GovernanceError::new_with_message(
                ErrorType::InvalidProposal,
                format!("ProposalData must be present for proposal {proposal_id:?}."),
            ));
        }
        self.neuron_store
            .draw_maturity_from_neurons_fund(&initial_neurons_fund_participation_snapshot)?;
        let initial_neurons_fund_participation = Some(initial_neurons_fund_participation.into());
        let neurons_fund_data = NeuronsFundData {
            initial_neurons_fund_participation,
            // These two fields will be known after `settle_neurons_fund_participation` is called.
            final_neurons_fund_participation: None,
            neurons_fund_refunds: None,
        };
        // Unwrapping is safe due to the `self.get_proposal_data().is_none()` check above.
        let proposal_data = self.mut_proposal_data(*proposal_id).unwrap();
        proposal_data.neurons_fund_data = Some(neurons_fund_data);
        Ok((initial_neurons_fund_participation_snapshot, constraints))
    }

    /// Join the Internet Computer's Neurons' Fund. If this neuron is
    /// already a member of the Neurons' Fund, an error is returned.
    fn join_community_fund(&mut self, now_seconds: u64) -> Result<(), GovernanceError> {
        if self.joined_community_fund_timestamp_seconds.unwrap_or(0) == 0 {
            self.joined_community_fund_timestamp_seconds = Some(now_seconds);
            Ok(())
        } else {
            // Already joined...
            Err(GovernanceError::new(ErrorType::AlreadyJoinedCommunityFund))
        }
