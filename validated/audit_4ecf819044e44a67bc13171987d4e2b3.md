Audit Report

## Title
`spawn_neuron` Pre-Check Uses Stale 5% Worst-Case Modulation While Mission 70 Allows −10% — (`rs/nns/governance/src/governance.rs`)

## Summary
`spawn_neuron` guards against spawning a neuron whose eventual ICP stake would fall below `neuron_minimum_stake_e8s` using a hardcoded 5% worst-case maturity modulation. The Mission 70 algorithm, now live in production, has a lower bound of −10% (−1,000 permyriad). The pre-check therefore under-estimates the worst case by a factor of two, allowing neurons to be spawned whose `cached_neuron_stake_e8s` will be below the minimum once the actual modulation is applied seven days later.

## Finding Description
In `spawn_neuron`, the minimum-stake pre-check at line 2666 of `rs/nns/governance/src/governance.rs` is:

```rust
let least_possible_stake = (maturity_to_spawn as f64 * (1_f64 - 0.05)) as u64;
```

This hardcodes a 5% worst-case reduction. The Mission 70 algorithm, implemented in `rs/nns/governance/src/timer_tasks/update_icp_xdr_rate_related_data.rs`, defines the actual production range at lines 47–50:

```rust
pub(crate) const MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70: i64 = -1_000;  // −10%
pub(crate) const MATURITY_MODULATION_MAX_PERMYRIAD_MISSION_70: i64 =    200;  // +2%
```

`compute_maturity_modulation_permyriad` clamps its output to this range (lines 193–196), so −1,000 permyriad is a reachable production value. `maybe_spawn_neurons` reads `heap_data.maturity_modulation.current_value_permyriad` (lines 6427–6435) and passes it directly to `apply_maturity_modulation` (line 6484–6487), which multiplies maturity by `(10_000 + modulation) / 10_000`. At −1,000 permyriad this is `9_000 / 10_000 = 0.90`. There is no post-spawn check against `neuron_minimum_stake_e8s`; `cached_neuron_stake_e8s` is set unconditionally to the modulated value at line 6517.

The gap between the pre-check threshold (÷0.95) and the actual worst-case threshold (÷0.90) creates a window of maturity values that pass the pre-check but produce a below-minimum stake. For `neuron_minimum_stake_e8s = 100_000_000 e8s`, any `maturity_to_spawn` in `(111_111_111, 105_263_158]` passes the pre-check yet yields a stake below 100_000_000 at −10% modulation.

## Impact Explanation
Any neuron controller with sufficient maturity can create a child neuron whose `cached_neuron_stake_e8s` is permanently below `neuron_minimum_stake_e8s`. This violates the core NNS governance invariant that all neurons hold at least the minimum stake, corrupts voting-power accounting for the affected neuron, and represents a concrete, demonstrable NNS protocol harm. This matches the allowed High impact: "Significant NNS security impact with concrete user or protocol harm."

## Likelihood Explanation
The Mission 70 modulation is live and its range is [−10%, +2%]. The bug triggers whenever the daily modulation is between −5% and −10%, which is a realistic scenario during sustained ICP price declines (the algorithm moves at most 30 permyriad per day, so reaching −500 permyriad takes roughly 17 days of consecutive decline). The entry path is a standard unprivileged `manage_neuron` → `Spawn` call requiring no special permissions beyond controlling a neuron with sufficient maturity. The condition is repeatable as long as modulation stays in the vulnerable range.

## Recommendation
Replace the hardcoded `0.05` with the actual Mission 70 worst-case permyriad constant, using integer arithmetic consistent with `apply_maturity_modulation`:

```rust
// 10_000 + MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 = 10_000 + (-1_000) = 9_000
let least_possible_stake = maturity_to_spawn
    .saturating_mul(9_000)
    / 10_000;
```

This mirrors the exact arithmetic in `apply_maturity_modulation` and eliminates floating-point imprecision. The constant `9_000` should be derived from `MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70` to stay in sync with future range changes.

## Proof of Concept
1. Deploy NNS governance with Mission 70 maturity modulation active at −1,000 permyriad (`current_value_permyriad = -1000`).
2. Create neuron A with `maturity_e8s_equivalent = 105_263_158` (just above `100_000_000 / 0.95 = 105_263_157`).
3. Call `manage_neuron` → `Spawn { percentage_to_spawn: 100, new_controller: ..., nonce: ... }`.
4. `spawn_neuron` computes `least_possible_stake = (105_263_158_f64 * 0.95) as u64 = 100_000_000` — equals `neuron_minimum_stake_e8s`, so the check passes.
5. Child neuron is created in spawning state with `maturity_e8s_equivalent = 105_263_158` and `spawn_at_timestamp_seconds = now + 7 days`.
6. After 7 days, `maybe_spawn_neurons` calls `apply_maturity_modulation(105_263_158, -1_000)` → `105_263_158 * 9_000 / 10_000 = 94_736_842`.
7. Child neuron's `cached_neuron_stake_e8s` is set to `94_736_842`, which is below `neuron_minimum_stake_e8s = 100_000_000`.
8. Assert `child_neuron.cached_neuron_stake_e8s < neuron_minimum_stake_e8s` — invariant violated.

This can be reproduced as a deterministic PocketIC or unit integration test by setting `current_value_permyriad = -1000` in the governance heap state and advancing the clock by 7 days.
