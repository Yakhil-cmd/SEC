Audit Report

## Title
Unbounded `read_state` Response Materialization Without Size Cap — (`rs/http_endpoints/public/src/read_state.rs`)

## Summary
The `/api/v{2,3}/canister/.../read_state` and `/api/v{2,3}/subnet/.../read_state` endpoints allow any unauthenticated caller to request the entire `["subnet"]` or `["api_boundary_nodes"]` subtrees of the state tree. The replica fully materializes the resulting `MixedHashTree` in memory and then CBOR-serializes it — a second full allocation — with no response size cap at any point in the pipeline. Under concurrent load this can exhaust the replica's heap and cause a denial of service.

## Finding Description

**Step 1 — Path validation enforces count and depth only, not response size.**

`validate_paths_width_and_depth()` in `rs/validator/src/ingress_validation.rs` (lines 178–194) enforces at most `MAXIMUM_NUMBER_OF_PATHS = 1,000` paths and at most `MAXIMUM_NUMBER_OF_LABELS_PER_PATH = 127` labels per path. No check on the byte size of the response those paths would produce is performed anywhere in the validation pipeline. [1](#0-0) 

**Step 2 — `["subnet"]` and `["api_boundary_nodes"]` are unconditionally allowed for any caller, including anonymous.**

In `verify_paths()` (`rs/http_endpoints/public/src/read_state.rs`, lines 461–474), the match arms for `[b"api_boundary_nodes"]` and `[b"subnet"]` only record a metric and return no error. There is no identity check, no authentication requirement, and no restriction on the scope of the subtree returned. [2](#0-1) 

Anonymous senders are explicitly permitted: in `ingress_validation.rs` (lines 164–168), `validate_ingress_expiry` is skipped for anonymous senders, and `validate_user_id_and_signature` (lines 853–857) returns `Ok(CanisterIdSet::all())` for them with no signature required. [3](#0-2) 

**Step 3 — Full in-memory materialization with no size gate.**

`get_certificate_and_create_response()` (`rs/http_endpoints/public/src/read_state.rs`, lines 355–404):
1. Calls `read_certified_state_with_exclusion()`, which materializes the full `MixedHashTree` via `materialize_partial` + `hash_tree.witness::<MixedHashTree>` (confirmed in `rs/state_manager/src/lib.rs`, lines 3883–3888).
2. Immediately serializes the entire `Certificate` (containing the tree) to CBOR via `into_cbor()` — a second full allocation.
3. Returns the result with **no size check at any point**. [4](#0-3) [5](#0-4) 

**Step 4 — Concurrency amplification.**

The handler runs inside `tokio::task::spawn_blocking`. The `max_read_state_concurrent_requests` concurrency limiter is the only throttle, but it bounds only the number of concurrent requests, not per-request memory. With `N` concurrent requests each materializing a multi-MB subtree, the replica allocates `O(N × subtree_size)` bytes. [6](#0-5) 

## Impact Explanation

An unauthenticated attacker sending concurrent `read_state` requests with path `["subnet"]` forces the replica to materialize and CBOR-serialize the full subnet subtree (all subnets × all nodes × public keys + canister ranges) for each request. On a production IC subnet with dozens of subnets and hundreds of nodes, this subtree can be several MB. With the concurrency limiter set to, e.g., 100 concurrent requests, the replica can be forced to hold hundreds of MB of additional heap allocations simultaneously, leading to OOM termination and denial of service for all honest users of that subnet.

This matches the in-scope bounty impact: **Application/platform-level DoS, crash, or subnet availability impact not based on raw volumetric DDoS** — **High ($2,000–$10,000)**.

## Likelihood Explanation

The `/read_state` endpoint is publicly reachable by any unauthenticated HTTP client. No authentication, cycles payment, or privileged role is required. The attack requires only standard HTTP tooling and knowledge of the IC API spec. The state tree grows naturally as the subnet adds subnets, nodes, and boundary nodes, increasing the amplification factor over time. The attack is repeatable and requires no victim interaction.

## Recommendation

1. **Enforce a response size cap.** After `read_certified_state_with_exclusion` returns the `MixedHashTree`, measure its serialized size (or an upper-bound estimate) and reject with HTTP 413 if it exceeds a configurable limit (e.g., 2 MB).
2. **Restrict broad subtree paths.** Require at least one non-root label for paths like `["subnet"]` and `["api_boundary_nodes"]` so callers cannot request the entire subtree in a single path.
3. **Cap total label data per request.** In `verify_paths`, accumulate the byte length of all label values across all paths and reject requests whose aggregate label size exceeds a threshold, providing an early pre-materialization gate.

## Proof of Concept

```python
import cbor2, requests, time, threading

envelope = {
    "content": {
        "request_type": "read_state",
        "sender": bytes(29),          # anonymous principal
        "ingress_expiry": int(time.time() * 1e9) + 300_000_000_000,
        "paths": [["subnet"]],        # entire subnet subtree
    }
}
body = cbor2.dumps(envelope)

REPLICA = "https://<boundary-node>/api/v2/canister/<any-canister-id>/read_state"

def flood():
    while True:
        requests.post(REPLICA, data=body,
                      headers={"Content-Type": "application/cbor"})

threads = [threading.Thread(target=flood) for _ in range=50)]
for t in threads: t.start()
```

Each request forces the replica to materialize and CBOR-serialize the full subnet subtree. With 50 concurrent threads and a multi-MB subtree, the replica's heap grows unboundedly until OOM termination. A safe local reproduction can be constructed as a PocketIC integration test that populates a state tree with many subnets/nodes and measures heap growth under concurrent `read_state` requests for `["subnet"]`.

fn validate_paths_width_and_depth(paths: &[Path]) -> Result<(), RequestValidationError> {
    if paths.len() > MAXIMUM_NUMBER_OF_PATHS {
        return Err(TooManyPaths {
            maximum: MAXIMUM_NUMBER_OF_PATHS,
            length: paths.len(),
        });
    }
    for path in paths {
        if path.len() > MAXIMUM_NUMBER_OF_LABELS_PER_PATH {
            return Err(PathTooLong {
                maximum: MAXIMUM_NUMBER_OF_LABELS_PER_PATH,
                length: path.len(),
            });
        }
    }
    Ok(())
}
