### Title
Unbounded Allowance Table Storage Bloat via Permanent ICRC-2 Approvals Without Expiry - (`rs/ledger_suite/common/ledger_core/src/approvals.rs`)

### Summary

The ICRC-2 `approve` endpoint on both the ICP ledger and ICRC-1 ledger allows any unprivileged caller to create permanent (no-expiry) allowance entries in stable storage at the cost of a single transfer fee per entry. Because the `prune()` function only removes entries that have an `expires_at` timestamp, allowances created without expiry are never automatically cleaned up. There is no cap on the total number of allowances stored. An attacker with a token balance can generate unlimited distinct spender principals (free) and call `icrc2_approve` repeatedly, bloating `ALLOWANCES_MEMORY` indefinitely.

---

### Finding Description

The `AllowanceTable::approve` function inserts a new entry into `ALLOWANCES_MEMORY` (a `StableBTreeMap` in stable storage) whenever `amount > 0`. When `expires_at` is `None`, no entry is added to the expiration queue:

```rust
if let Some(expires_at) = expires_at {
    table.allowances_data.insert_expiry(expires_at, key.clone());
}
table.allowances_data.set_allowance(
    key,
    Allowance { amount: amount.clone(), expires_at, arrived_at: now },
);
``` [1](#0-0) 

The `prune()` function iterates only over the expiration queue. If the queue is empty (all allowances are permanent), it returns immediately without removing anything:

```rust
match table.allowances_data.first_expiry() {
    ...
    None => { return pruned; }
}
``` [2](#0-1) 

The stable-memory allowance store for the ICRC-1 ledger (`StableAllowancesData`) and the ICP ledger (`StableAllowancesData`) both back `ALLOWANCES_MEMORY` with an unbounded `StableBTreeMap`: [3](#0-2) [4](#0-3) 

The only read-side limit is `MAX_TAKE_ALLOWANCES = 500`, which caps query results but does not cap storage: [5](#0-4) 

There is no check in `approve()` or `icrc2_approve_not_async()` for the total number of allowances stored globally or per account: [6](#0-5) 

---

### Impact Explanation

An attacker who controls a token balance can:

1. Generate N distinct spender principals (free — just generate key pairs off-chain).
2. Call `icrc2_approve(spender_i, amount=1, expires_at=None)` N times.
3. Each call costs exactly one transfer fee and inserts one permanent entry into `ALLOWANCES_MEMORY`.
4. These entries are never automatically removed.

The `ALLOWANCES_MEMORY` stable BTreeMap grows without bound. Each entry is approximately 100–200 bytes (two account IDs + amount + optional expiry). Filling the canister's stable memory degrades ledger performance and, at the extreme, causes the ledger canister to trap on memory allocation, halting all token transfers, approvals, and governance staking operations that depend on the ledger.

The ICP ledger does expose a `remove_approval` endpoint, but this only allows the *approver* to remove their own allowances — it does not help against an attacker who intentionally leaves entries in place. [7](#0-6) 

---

### Likelihood Explanation

- **Entry point**: `icrc2_approve` is a public ingress update call on both the ICP ledger and all ICRC-1 ledger canisters. Any principal with a nonzero token balance can call it.
- **Cost per entry**: One transfer fee (10,000 e8s ≈ $0.001 for ICP; 10 satoshis ≈ $0.006 for ckBTC at current prices).
- **Spender accounts**: Generating a new principal requires only an off-chain key generation — no on-chain cost.
- **No rate limiting**: There is no per-account or global cap on the number of allowance entries.
- **Persistence**: Entries without expiry survive indefinitely across canister upgrades (stable memory).

A motivated attacker can sustain a slow, low-cost bloat campaign that accumulates over months without triggering any alarm.

---

### Recommendation

1. **Require `expires_at`**: Reject `icrc2_approve` calls that do not supply an `expires_at` timestamp, or enforce a maximum allowance lifetime.
2. **Cap allowances per account**: Enforce a maximum number of active allowances per `(from_account)` in `approve()`, rejecting new entries once the cap is reached.
3. **Minimum allowance amount**: Reject approvals below a configurable dust threshold (analogous to `MinimumPoolLiquidity` in the Omnipool).
4. **Charge a storage deposit**: Require a refundable cycles deposit when creating a permanent allowance, returned when the allowance is revoked.

---

### Proof of Concept

```rust
// Attacker controls `attacker_account` with balance > N * fee.
// Generates N distinct spender principals off-chain.
for i in 0..N {
    let spender = Account {
        owner: generate_fresh_principal(i), // free, off-chain
        subaccount: None,
    };
    icrc2_approve(ApproveArgs {
        from_subaccount: None,
        spender,
        amount: Nat::from(1_u64),   // minimum nonzero amount
        expected_allowance: None,
        expires_at: None,           // permanent — never pruned
        fee: None,
        memo: None,
        created_at_time: None,
    }).await.unwrap();
    // Cost: one transfer fee. Effect: one permanent entry in ALLOWANCES_MEMORY.
}
// After N calls: N permanent entries in stable storage.
// prune() cannot remove any of them.
assert_eq!(ledger.approvals().get_num_approvals(), N);
```

The `prune()` call that runs on each `apply_transaction` finds an empty expiration queue and returns 0 pruned, leaving all N entries intact: [8](#0-7)

    /// Prunes allowances that are expired, removes at most `limit` allowances.
    pub fn prune(&mut self, now: TimeStamp, limit: usize) -> usize {
        self.with_postconditions_check(|table| {
            let mut pruned = 0;
            for _ in 0..limit {
                match table.allowances_data.first_expiry() {
                    Some((ts, _key)) => {
                        if ts > now {
                            return pruned;
                        }
                    }
                    None => {
                        return pruned;
                    }
                }
                if let Some((_, (account, spender))) = table.allowances_data.pop_first_expiry() {
                    let key = (account, spender);
                    if let Some(allowance) = table.allowances_data.get_allowance(&key)
                        && allowance.expires_at.unwrap_or_else(remote_future) <= now
                    {
                        table.allowances_data.remove_allowance(&key);
                        pruned += 1;
                    }
                }
            }
            pruned
        })
    }

fn icrc2_approve_not_async(caller: Principal, arg: ApproveArgs) -> Result<u64, ApproveError> {
    let block_idx = Access::with_ledger_mut(|ledger| {
        let now = TimeStamp::from_nanos_since_unix_epoch(ic_cdk::api::time());

        let from_account = Account {
            owner: caller,
            subaccount: arg.from_subaccount,
        };
        if from_account.owner == arg.spender.owner {
            ic_cdk::trap("self approval is not allowed")
        }
        if &from_account == ledger.minting_account() {
            ic_cdk::trap("the minting account cannot delegate mints")
        }
        match arg.memo.as_ref() {
            Some(memo) if memo.0.len() > ledger.max_memo_length() as usize => {
                ic_cdk::trap("the memo field is too large")
            }
            _ => {}
        };
        let amount = Tokens::try_from(arg.amount).unwrap_or_else(|_| Tokens::max_value());
        let expected_allowance = match arg.expected_allowance {
            Some(n) => match Tokens::try_from(n) {
                Ok(n) => Some(n),
                Err(_) => {
                    let current_allowance = ledger
                        .approvals()
                        .allowance(&from_account, &arg.spender, now)
                        .amount;
                    return Err(ApproveError::AllowanceChanged {
                        current_allowance: current_allowance.into(),
                    });
                }
            },
            None => None,
        };

        let expected_fee_tokens = ledger.transfer_fee();
        let expected_fee: Nat = expected_fee_tokens.into();
        if arg.fee.is_some() && arg.fee.as_ref() != Some(&expected_fee) {
            return Err(ApproveError::BadFee { expected_fee });
        }

        let tx = Transaction {
            operation: Operation::Approve {
                from: from_account,
                spender: arg.spender,
                amount,
                expected_allowance,
                expires_at: arg.expires_at,
                fee: arg.fee.map(|_| expected_fee_tokens),
            },
            created_at_time: arg.created_at_time,
            memo: arg.memo,
        };

        let (block_idx, _) = apply_transaction(ledger, tx, now, expected_fee_tokens)
            .map_err(convert_transfer_error)
            .map_err(|err| {
                let err: ApproveError = match err.try_into() {
                    Ok(err) => err,
                    Err(err) => ic_cdk::trap(&err),
                };
                err
            })?;
        Ok(block_idx)
    })?;

    update_total_volume(Tokens::zero(), true);

    Ok(block_idx)
}
