Audit Report

## Title
Unbounded `BouncerValue::Wants` Window in `CertifierBouncer::new_bouncer` Enables Unvalidated Pool Flooding by a Byzantine Peer — (`rs/consensus/certification/src/certifier.rs`)

## Summary
`CertifierBouncer::new_bouncer` returns `BouncerValue::Wants` for every `CertificationMessageId` at any height ≥ `cup_height` that is not already certified, with no upper-bound guard. Combined with `SLOT_TABLE_NO_LIMIT` (`usize::MAX`) for the certification channel and the absence of any size cap on the unvalidated pool, a single Byzantine subnet peer can flood the unvalidated certification pool with syntactically valid but cryptographically invalid artifacts at arbitrarily large heights, causing unbounded memory growth and eventual OOM crash on all honest replicas of the subnet.

## Finding Description

**Root cause — unbounded bouncer window:**
`CertifierBouncer::new_bouncer` at `rs/consensus/certification/src/certifier.rs` L86–95 returns `Wants` for every height ≥ `cup_height` that is not in `certified_heights`, with no upper-bound check:

```rust
Box::new(move |id| {
    let height = id.height;
    if height < cup_height || certified_heights.contains(&height) {
        BouncerValue::Unwanted
    } else {
        BouncerValue::Wants   // no upper-bound guard
    }
})
```

By contrast, the consensus bouncer (`rs/consensus/src/consensus/priority.rs` L56–59) caps acceptance at `next_cup_height + ACCEPTABLE_NOTARIZATION_CUP_GAP` and uses `LOOK_AHEAD = 10` for per-type windows, and the IDkg bouncer (`rs/consensus/idkg/src/lib.rs` L563–567) uses `LOOK_AHEAD` for every message type. The certification bouncer has no equivalent guard.

**No slot-table limit for certification:**
The certification channel is registered with `SLOT_TABLE_NO_LIMIT = usize::MAX` at `rs/replica/setup_ic_network/src/lib.rs` L268. The slot-table enforcement in `rs/p2p/consensus_manager/src/receiver.rs` L373 only drops new entries when `peer_slot_table_len >= slot_limit`; with `usize::MAX` that check never fires. A Byzantine peer may therefore occupy an unlimited number of per-peer slots simultaneously, each spawning a new `process_slot_update` task and a new entry in `active_assembles` (receiver.rs L405–419).

**Unvalidated pool has no size limit:**
`CertificationPoolImpl::insert` at `rs/artifact_pool/src/certification_pool.rs` L259–272 unconditionally inserts into an in-memory `BTreeMap` with no capacity check.

**Certifier only validates heights the state manager requests:**
`CertifierImpl::on_state_change` at `rs/consensus/certification/src/certifier.rs` L214–244 calls `validate` only over `state_heights_to_aggregate_and_validate`, derived from `list_state_hashes_to_certify` / `list_state_heights_to_certify`. Artifacts at heights far above the current execution tip are never validated and therefore never removed from the unvalidated pool.

**Purger only removes artifacts *below* `cup_height − MINIMUM_CHAIN_LENGTH`:**
`get_purge_height` at `rs/consensus/certification/src/certifier.rs` L463–474 issues `RemoveAllBelow(cup_height − MINIMUM_CHAIN_LENGTH)`. Artifacts at heights far *above* `cup_height` are never touched.

**Download loop retries indefinitely:**
`download_artifact` at `rs/p2p/artifact_downloader/src/fetch_artifact/download.rs` L219 sets `with_max_elapsed_time(None)`. Tasks for future-height adverts never time out as long as the bouncer returns `Wants`.

**Exploit flow:**
1. Byzantine peer sends slot updates with N distinct slot numbers, each advertising a `CertificationShare` at height H >> `cup_height`.
2. Bouncer returns `Wants` for every one; N download tasks are spawned and N entries are added to `active_assembles`.
3. The peer constructs syntactically valid `CertificationShare` protobuf messages at those heights using its own legitimate signing key (no cryptographic forgery required), ensuring `message.id() == id` passes.
4. Each artifact is delivered to the unvalidated pool via `UnvalidatedArtifactMutation::Insert`.
5. The certifier never validates or purges these artifacts because the state manager never requests those heights.
6. The unvalidated `BTreeMap` and `HeightIndex` structures grow without bound → OOM → replica crash.
7. Because the Byzantine node can send slot updates to all honest replicas simultaneously, the entire subnet can be taken down.

## Impact Explanation
A single Byzantine subnet node (below the f < n/3 fault threshold) can crash all honest replicas on the subnet via OOM, halting the subnet entirely. This is a concrete subnet-availability impact not based on raw volumetric DDoS — it exploits a missing application-level bound in the certification bouncer and pool. This matches the **High ($2,000–$10,000)** impact: *"Application/platform-level DoS, crash, consensus blocking, certified-state disruption, or subnet availability impact not based on raw volumetric DDoS."*

## Likelihood Explanation
- Requires one compromised subnet node below the fault threshold — an explicitly in-scope threat model.
- No threshold-majority corruption needed; a single node suffices.
- The attack is mechanically simple: send many slot-update messages with distinct slot numbers and respond to fetch requests with crafted but syntactically valid artifacts.
- No cryptographic forgery is required; the attacker uses its own legitimate signing key.
- The attack is repeatable and can be sustained indefinitely.

## Recommendation
1. **Add an upper-bound guard to `CertifierBouncer::new_bouncer`** analogous to the consensus and IDkg bouncers. Return `MaybeWantsLater` for heights beyond `cup_height + LOOK_AHEAD` (or a certification-specific constant).
2. **Set a finite `SLOT_TABLE_LIMIT_CERTIFICATION`** (similar to `SLOT_TABLE_LIMIT_INGRESS = 50_000`) for the certification channel in `rs/replica/setup_ic_network/src/lib.rs`.
3. **Add a size cap to the unvalidated certification pool** in `CertificationPoolImpl::insert` and drop insertions that exceed it.
4. **Periodically validate and discard unvalidated artifacts at heights far above the current execution tip**, rather than waiting for the state manager to request them.

## Proof of Concept

```rust
// Confirms the unbounded Wants window in new_bouncer
let cup_height = Height::from(0);
let certified_heights: HashSet<Height> = HashSet::new();
let bouncer = move |id: &CertificationMessageId| {
    let h = id.height;
    if h < cup_height || certified_heights.contains(&h) {
        BouncerValue::Unwanted
    } else {
        BouncerValue::Wants
    }
};
for h in [1u64, 1_000, 1_000_000, u64::MAX - 1] {
    let id = CertificationMessageId {
        height: Height::from(h),
        hash: CertificationMessageHash::CertificationShare(
            CryptoHashOf::from(CryptoHash(vec![]))
        ),
    };
    assert_eq!(bouncer(&id), BouncerValue::Wants);
}
// All assertions pass: no upper bound exists.
```
