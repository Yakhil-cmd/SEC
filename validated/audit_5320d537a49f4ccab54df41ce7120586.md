### Title
Unbounded Double Iteration Over Storage Cache in `apply_storage_diffs_pubdata` Causes Unmetered Native Resource Exhaustion - (`basic_system/src/system_implementation/flat_storage_model/mod.rs`)

---

### Summary

`apply_storage_diffs_pubdata` in `FlatTreeWithAccountsUnderHashesStorageModel` iterates the entire block-level storage cache **twice** — once to count diffs and once to write them — without charging any native resources for the first pass. An attacker can bloat the cache by writing to many distinct storage slots across transactions, making block finalization consume unbounded unmetered native resources and potentially rendering the block unprovable.

---

### Finding Description

In `apply_storage_diffs_pubdata` (called at block finalization), the diff count is computed by consuming a full iterator over the storage cache:

```rust
// basic_system/src/system_implementation/flat_storage_model/mod.rs:571-574
let encoded_state_diffs_count =
    (self.storage_cache.net_diffs_iter().count() as u32).to_be_bytes();
pubdata_dst.write(&encoded_state_diffs_count);
result_keeper.pubdata(&encoded_state_diffs_count);
```

`net_diffs_iter()` calls `iter_as_storage_types()` which calls `self.0.cache.iter()` — a full scan of every unique `WarmStorageKey` entry accumulated across the entire block:

```rust
// basic_system/src/system_implementation/flat_storage_model/storage_cache.rs:258-263
pub fn net_diffs_iter(&self) -> impl Iterator<...> {
    self.iter_as_storage_types()
        .filter(|(_, v)| v.current_value != v.initial_value)
}
```

Immediately after, the function performs a **second full scan** via `apply_to_all_updated_elements` to actually write the pubdata:

```rust
// basic_system/src/system_implementation/flat_storage_model/mod.rs:576-624
self.storage_cache
    .0
    .cache
    .apply_to_all_updated_elements::<_, ()>(|l, r, k| { ... })
    .expect("must compute pubdata");
```

The count could trivially be maintained as a running counter incremented on each write and decremented on rollback, eliminating the first pass entirely. The same pattern exists in `calculate_pubdata_used_by_tx` in `storage_cache.rs` (lines 265–308) and `account_cache.rs` (lines 433–473), which iterate `iter_altered_since_commit()` on every transaction boundary to recompute pubdata costs from scratch.

---

### Impact Explanation

`apply_storage_diffs_pubdata` executes at block finalization, **outside** of per-transaction EVM gas metering. The native resources consumed by the redundant first-pass iteration are never charged to any transaction. Because the storage cache accumulates one entry per unique `(address, slot)` pair touched across the entire block, an attacker who fills a block with transactions that each write to many distinct storage slots can make the first-pass iteration arbitrarily expensive relative to what was charged. In the RISC-V proving environment, native resource budgets are finite and fixed per block. Exhausting them during unmetered finalization work causes the block to be **unprovable**, stalling the chain.

---

### Likelihood Explanation

Every EVM `SSTORE` to a new slot costs 20,000 gas. With a 30 M gas block limit, a single transaction can touch ~1,500 unique slots; a full block of such transactions can push the cache into the tens of thousands of entries. The attacker needs only to submit valid, gas-paying transactions — no privileged access, no oracle manipulation, no governance majority. The attack is fully permissionless and economically feasible because the gas cost of the writes is bounded while the unmetered proving cost of the double iteration grows linearly with cache size.

---

### Recommendation

Replace the `net_diffs_iter().count()` call with a maintained counter:

- Increment the counter in the write path of `NewStorageWithAccountPropertiesUnderHash` whenever a slot transitions from `initial == current` to `initial != current`.
- Decrement it on rollback when a slot reverts to its initial value.
- Expose the counter directly in `apply_storage_diffs_pubdata` instead of calling `.count()`.

Apply the same fix to `calculate_pubdata_used_by_tx` in both `storage_cache.rs` and `account_cache.rs`: maintain a running pubdata counter updated incrementally on each write/rollback rather than re-scanning `iter_altered_since_commit()` on every transaction boundary.

---

### Proof of Concept

1. Craft a contract that writes to N distinct storage slots (e.g., `for i in 0..N { sstore(i, 1) }`).
2. Fill a block with transactions calling this contract, maximizing N subject to the block gas limit.
3. At block finalization, `apply_storage_diffs_pubdata` is called. The first-pass `net_diffs_iter().count()` iterates all N·(txs) unique cache entries with zero native resource charge.
4. The second-pass `apply_to_all_updated_elements` iterates again.
5. Total unmetered native work scales as O(2 × total unique slots in block). With a sufficiently large block, this exceeds the prover's native resource budget, making the block unprovable.

    pub fn calculate_pubdata_used_by_tx(&self) -> u32 {
        let mut visited_elements = BTreeSet::new_in(self.0.alloc.clone());

        let mut pubdata_used = 0u32;
        for element_history in self.0.cache.iter_altered_since_commit() {
            // Elements are sorted chronologically

            let element_key = element_history.key();

            // we publish preimages for account details, so no need to publish hash
            if element_key.address == ACCOUNT_PROPERTIES_STORAGE_ADDRESS {
                continue;
            }

            // Skip if already calculated pubdata for this element
            if visited_elements.contains(element_key) {
                continue;
            }
            visited_elements.insert(element_key);

            let current_value = element_history.current().value();
            let initial_value = element_history.initial().value();
            let at_tx_start_value = element_history.committed().value();

            // If the current value is resetting to the initial one,
            // we don't consider this diff in the pubdata charging.
            // This change will be optimized away, so it's actually reducing
            // pubdata.
            if current_value == initial_value {
                continue;
            }

            if at_tx_start_value != current_value {
                // TODO(EVM-1074): use tree index instead of key for repeated writes
                pubdata_used += 32; // key
                pubdata_used += ValueDiffCompressionStrategy::optimal_compression_length(
                    at_tx_start_value,
                    current_value,
                ) as u32;
            }
        }

        pubdata_used
    }

    pub fn calculate_pubdata_used_by_tx(&self) -> u32 {
        let mut visited_elements = BTreeSet::new_in(self.alloc.clone());

        let mut pubdata_used = 0u32;
        for element_history in self.cache.iter_altered_since_commit() {
            // Elements are sorted chronologically

            let element_key = element_history.key();

            // Skip if already calculated pubdata for this element
            if visited_elements.contains(element_key) {
                continue;
            }
            visited_elements.insert(element_key);

            let current = element_history.current();
            let initial = element_history.initial();
            let at_tx_start = element_history.committed();

            // If the current value is resetting to the initial one,
            // we don't consider this diff in the pubdata charging.
            // This change will be optimized away, so it's actually reducing
            // pubdata.
            if current.value() == initial.value() && !current.metadata().not_compress_balance {
                continue;
            }

            if current.value() != at_tx_start.value() || current.metadata().not_compress_balance {
                pubdata_used += 32; // key
                pubdata_used += AccountProperties::diff_compression_length(
                    at_tx_start.value(),
                    current.value(),
                    current.metadata().not_publish_bytecode,
                    current.metadata().not_compress_balance,
                )
                .unwrap();
            }
        }

        pubdata_used
    }
