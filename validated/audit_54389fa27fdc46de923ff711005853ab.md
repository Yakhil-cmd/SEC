### Title
Signature forgery for arbitrary public key via digest = n causing z = 0 in P256 verify — (`crypto/src/secp256r1/verify.rs`, `scalar/scalar64.rs`)

### Summary
`verify` uses `reduce_be_bytes` for the digest, which silently maps the curve order n to 0. When z = 0, u1 = 0, and `ecmult` skips the public-key scalar entirely, computing only u2·G. An attacker can craft (r, s) such that x_coord(u2·G) = r, making verification return `Ok(true)` for **any** supplied public key.

### Finding Description
`Scalar::reduce_be_bytes` accepts any 32-byte value and reduces it mod n. When the input equals n exactly, the result is 0. In `verify`, this value becomes z, and u1 = z · s⁻¹ = 0. Inside `ecmult`, the branch `if !na.is_zero() && !a.is_infinity()` is false, so the public key Jacobian is replaced with `Wnaf::default()` / `OddMultiplesTable::default()` and never contributes to the output. The final point is solely u2·G = (r·s⁻¹)·G. By choosing k freely and setting r = x_coord(k·G), s = r·k⁻¹, the attacker makes this x-coordinate equal r, passing the final `r == recovered` check regardless of what (x, y) was supplied. [7](#0-6) 

### Impact Explanation
Any smart contract or account using the p256_verify precompile for authentication (e.g., P256-based smart accounts per EIP-7951) can be bypassed. The attacker submits a transaction whose calldata triggers the precompile with digest = n and a crafted (r, s, x, y). The precompile returns `U256::ONE` (success), and the contract accepts the forged authentication. Because ZKsync OS proves forward execution, this forged acceptance is also provable — the ZK proof will attest to a valid signature that never existed. [8](#0-7) 

### Likelihood Explanation
The attack requires only knowledge of the curve order n (public) and basic elliptic-curve arithmetic. No privileged access, leaked keys, or external oracle cooperation is needed. Any unprivileged caller can invoke the precompile directly.

### Recommendation
Add an explicit zero-check on z after `reduce_be_bytes` and return `Ok(false)` (or `Err`) immediately:

```rust
let z = Scalar::reduce_be_bytes(digest);
if z.is_zero() {
    return Ok(false);
}
```

Alternatively, use `from_be_bytes` for the digest as well, rejecting inputs ≥ n. The standard ECDSA spec (FIPS 186-4, SEC 1) requires z to be derived from a hash and to be treated as an integer; a zero z is always invalid. [9](#0-8)
