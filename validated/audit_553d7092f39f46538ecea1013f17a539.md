### Title
Missing Ordering Invariant Between `contribution_threshold_xdr` and `one_third_participation_milestone_xdr` Allows Inverted Neurons' Fund Matching Function Parameters - (`File: rs/nns/governance/src/network_economics.rs`)

---

### Summary

`NeuronsFundMatchedFundingCurveCoefficients::validate()` enforces only one of the two required ordering constraints between the three Neurons' Fund participation thresholds. The missing check allows a `ManageNetworkEconomics` governance proposal to set `contribution_threshold_xdr >= one_third_participation_milestone_xdr`, which passes on-chain validation but produces a structurally broken `PolynomialMatchingFunction` — directly analogous to M-01's liquidation factor < collateral factor inversion.

---

### Finding Description

`NeuronsFundMatchedFundingCurveCoefficients` holds three thresholds that define the shape of the Neurons' Fund matched-funding curve:

```
contribution_threshold_xdr  <  one_third_participation_milestone_xdr  <  full_participation_milestone_xdr
```

These map 1-to-1 to `t_1`, `t_2`, `t_3` in `PolynomialMatchingFunctionPersistentData`:

```rust
// rs/nervous_system/neurons_fund/src/lib.rs:862-868
let persistent_data = PolynomialMatchingFunctionPersistentData {
    t_1: neurons_fund_participation_limits.contribution_threshold_icp,
    t_2: neurons_fund_participation_limits.one_third_participation_milestone_icp,
    t_3: neurons_fund_participation_limits.full_participation_milestone_icp,
    t_4: dec!(2.0) * cap,
    cap,
};
```

The `validate()` function in `rs/nns/governance/src/network_economics.rs` checks only the second relationship:

```rust
// rs/nns/governance/src/network_economics.rs:243-252
// later milestones must be > earlier ones
if let (Ok(one_third_participation_milestone_xdr), Ok(full_participation_milestone_xdr)) = (
    one_third_participation_milestone_xdr,
    full_participation_milestone_xdr,
) && one_third_participation_milestone_xdr >= full_participation_milestone_xdr
{
    defects.push(format!(...));
}
```

There is **no check** that `contribution_threshold_xdr < one_third_participation_milestone_xdr`. The docstring for `validate()` confirms this omission — it lists only three invariants, and the missing ordering is not among them:

```
///     3. one_third_participation_milestone_xdr <
///        full_participation_milestone_xdr.
``` [1](#0-0) 

When `contribution_threshold_xdr >= one_third_participation_milestone_xdr`, the ICP-converted values produce `t_1 >= t_2`. The `PolynomialMatchingFunction::apply()` function then has a dead branch:

```rust
// rs/nervous_system/neurons_fund/src/lib.rs:896-907
let res = if x < t1 {
    Decimal::ZERO
} else if t1 <= x && x < t2 {   // DEAD: impossible when t1 >= t2
    self.cache.f_1.apply(x)?
} else if t2 <= x && x < t3 {
    self.cache.f_2.apply(x)?
} ...
``` [2](#0-1) 

Additionally, `F1Cache::new(t1, t2, cap)` computes polynomial coefficients using denominators of the form `(t2 - t1)^3`:

```rust
// rs/nervous_system/neurons_fund/src/lib.rs:493
let a_denom = BinomialFormula::new("a_denom", 3, vec![1, -3, 3, -1], t1, t2)?.eval()?;
let a = a_num
    .checked_div(a_denom)
    .ok_or("division by zero while computing a in f_1.".to_string())?;
``` [3](#0-2) 

- If `t_1 == t_2`: denominator is zero → `F1Cache::new` returns `Err` → `PolynomialMatchingFunction::new` returns `Err` → `CreateServiceNervousSystem` proposal execution fails, blocking all future SNS launches.
- If `t_1 > t_2`: denominator is negative → polynomial coefficients are computed with wrong signs → the matching function is non-monotonic, producing incorrect (potentially inflated or deflated) Neurons' Fund participation amounts for every SNS swap.

The conversion path from governance storage to the polynomial function is:

```
ManageNetworkEconomics proposal
  → NetworkEconomics::apply_changes_and_validate()   [rs/nns/governance/src/network_economics.rs:35-42]
  → NeuronsFundMatchedFundingCurveCoefficients::validate()  [missing check]
  → stored in heap_data.economics
  → try_derive_neurons_fund_participation_limits()    [rs/nns/governance/src/neurons_fund.rs:242-325]
  → PolynomialMatchingFunction::new()                [rs/nervous_system/neurons_fund/src/lib.rs:832-875]
  → used in every CreateServiceNervousSystem execution
``` [4](#0-3) 

---

### Impact Explanation

**Governance/ledger conservation bug.** If `contribution_threshold_xdr >= one_third_participation_milestone_xdr` is stored:

1. **Equal case (`t_1 == t_2`):** Every subsequent `CreateServiceNervousSystem` proposal execution fails at `PolynomialMatchingFunction::new()`, permanently blocking all new SNS launches until a corrective governance proposal is passed.
2. **Inverted case (`t_1 > t_2`):** The matching function silently produces incorrect Neurons' Fund participation amounts — the `f_1` segment (the ramp-up region) is skipped entirely, causing the fund to contribute at `f_2` rates for direct participation amounts that should be in the `f_1` range. This misallocates ICP maturity from Neurons' Fund neurons, harming both SNS projects (wrong funding) and NF neuron holders (incorrect maturity reservation). [5](#0-4) 

---

### Likelihood Explanation

The entry path is a `ManageNetworkEconomics` NNS governance proposal. Such proposals are submitted and voted on by NNS neuron holders. The incomplete validation means:

- A well-intentioned but mistaken proposal (e.g., setting `contribution_threshold_xdr` to a large value to "raise the bar" for NF participation) could accidentally invert the ordering.
- The proposal passes the on-chain validation gate (`apply_changes_and_validate`) without error, giving no signal that the parameters are broken.
- The defect only manifests later, at `CreateServiceNervousSystem` execution time, making diagnosis harder.

The existing test `test_manage_network_economics_revalidate_at_execution_time` (in `rs/nns/governance/tests/governance.rs`) demonstrates that the system already handles the case where two proposals interact to produce an invalid `maximum_icp_xdr_rate < minimum_icp_xdr_rate` — confirming that this class of ordering bug is a known concern, yet the analogous check for the curve coefficients is absent. [6](#0-5)

### Recommendation

Add the missing ordering check inside `NeuronsFundMatchedFundingCurveCoefficients::validate()` in `rs/nns/governance/src/network_economics.rs`, immediately after the existing `one_third < full` check:

```rust
// After line 252, add:
if let (Ok(contribution_threshold_xdr), Ok(one_third_participation_milestone_xdr)) = (
    try_convert_decimal(contribution_threshold_xdr),
    try_convert_decimal(one_third_participation_milestone_xdr),
) && contribution_threshold_xdr >= one_third_participation_milestone_xdr
{
    defects.push(format!(
        "contribution_threshold_xdr ({contribution_threshold_xdr}) must be less than \
         one_third_participation_milestone_xdr ({one_third_participation_milestone_xdr}).",
    ));
}
```

Also update the docstring to list this as invariant 4. [7](#0-6)

### Proof of Concept

1. Submit a `ManageNetworkEconomics` proposal with:
   ```
   neurons_fund_economics.neurons_fund_matched_funding_curve_coefficients = {
     contribution_threshold_xdr:          "300_000.0"   // > one_third milestone
     one_third_participation_milestone_xdr: "225_000.0"
     full_participation_milestone_xdr:      "375_000.0"
   }
   ```
2. The proposal passes `NeuronsFundMatchedFundingCurveCoefficients::validate()` — only `one_third (225k) < full (375k)` is checked; `contribution (300k) > one_third (225k)` is not caught.
3. The new economics are stored. `t_1 = 300_000 / icp_xdr_rate`, `t_2 = 225_000 / icp_xdr_rate` → `t_1 > t_2`.
4. The next `CreateServiceNervousSystem` proposal calls `PolynomialMatchingFunction::new(...)`, which calls `F1Cache::new(t_1, t_2, cap)`. The denominator `(t_2 - t_1)^3 < 0` produces inverted polynomial coefficients, and `apply()` skips the `f_1` branch entirely for all direct participation amounts in `[t_1, t_2)` — a range that is now empty — causing the Neurons' Fund to compute participation amounts using `f_2` coefficients from the very start of the ramp, misallocating ICP maturity. [8](#0-7) [9](#0-8)

impl NeuronsFundMatchedFundingCurveCoefficients {
    /// This verifies the following:
    ///
    ///     1. All fields are set.
    ///
    ///     2. All (Decimal) values are valid for their type (i.e. can be
    ///        converted to rust_decimal::Decimal).
    ///
    ///     3. one_third_participation_milestone_xdr <
    ///        full_participation_milestone_xdr.
    ///
    /// If Err is returned, it will be a nonempty Vec of defects.
    fn validate(&self) -> Result<(), Vec<String>> {
        let Self {
            contribution_threshold_xdr,
            one_third_participation_milestone_xdr,
            full_participation_milestone_xdr,
        } = self;

        let mut defects = vec![];

        // Everything must be set.
        if contribution_threshold_xdr.is_none() {
            defects.push("contribution_threshold_xdr must be set.".to_string());
        }
        if one_third_participation_milestone_xdr.is_none() {
            defects.push("one_third_participation_milestone_xdr must be set.".to_string());
        }
        if full_participation_milestone_xdr.is_none() {
            defects.push("full_participation_milestone_xdr must be set.".to_string());
        }

        // All values must be valid (per their type).
        fn try_convert_decimal(
            original: &Option<DecimalProto>,
        ) -> Result<Decimal, /* human_readable */ &str> {
            const DEFAULT_DECIMAL: DecimalProto = DecimalProto {
                human_readable: None,
            };

            let human_readable: &str = original
                .as_ref()
                .unwrap_or(&DEFAULT_DECIMAL)
                .human_readable
                .as_deref()
                .unwrap_or("");

            Decimal::try_from(human_readable).map_err(|_ignore| human_readable)
        }

        let _contribution_threshold_xdr = try_convert_decimal(contribution_threshold_xdr)
            .inspect_err(|original| {
                defects.push(format!(
                    "contribution_threshold_xdr ({original}) is not a Decimal."
                ));
            });
        let one_third_participation_milestone_xdr =
            try_convert_decimal(one_third_participation_milestone_xdr).inspect_err(|original| {
                defects.push(format!(
                    "one_third_participation_milestone_xdr ({original}) is not a Decimal."
                ));
            });
        let full_participation_milestone_xdr =
            try_convert_decimal(full_participation_milestone_xdr).inspect_err(|original| {
                defects.push(format!(
                    "full_participation_milestone_xdr ({original}) is not a Decimal."
                ));
            });

        // later milestones must be > earlier ones
        if let (Ok(one_third_participation_milestone_xdr), Ok(full_participation_milestone_xdr)) = (
            one_third_participation_milestone_xdr,
            full_participation_milestone_xdr,
        ) && one_third_participation_milestone_xdr >= full_participation_milestone_xdr
        {
            defects.push(format!(
                    "one_third_participation_milestone_xdr ({one_third_participation_milestone_xdr}) must be less than full_participation_milestone_xdr ({full_participation_milestone_xdr}).",
                ));
        }

        if !defects.is_empty() {
            return Err(defects);
        }

        Ok(())
    }
}

impl PolynomialMatchingFunctionCache {
    fn from_persistent_data(
        data: &PolynomialMatchingFunctionPersistentData,
    ) -> Result<Self, String> {
        let f_1 = F1Cache::new(data.t_1, data.t_2, data.cap)
            .map_err(|e| format!("Error while computing cached data for f_1: {e}"))?;
        let f_2 = F2Cache::new(data.t_2, data.t_3, data.cap)
            .map_err(|e| format!("Error while computing cached data for f_2: {e}"))?;
        let f_3 = if data.t_4 > Decimal::ZERO {
            F3Cache::new(data.t_3, data.t_4, data.cap)
                .map_err(|e| format!("Error while computing cached data for f_3: {e}"))?
        } else {
            // Setting all polynomial coefficients to `1.0` to avoid dealing with `0^0`; at the same
            // time, `cap == 0.0` makes `F3Cache::apply` always return `0.0`, respecting
            // the semantics of `f_3` for `t4 == 0`.
            F3Cache {
                t3: data.t_3,
                t4: data.t_4,
                a: Decimal::ONE,
                b: Decimal::ONE,
                c: Decimal::ONE,
                d: Decimal::ONE,
                e: Decimal::ONE,
                cap: Decimal::ZERO,
            }
        };
        Ok(Self { f_1, f_2, f_3 })
    }

impl Governance {
    fn try_derive_neurons_fund_participation_limits_impl(
        neurons_fund_economics: &NeuronsFundEconomicsPb,
        icp_xdr_rate: Decimal,
    ) -> Result<NeuronsFundParticipationLimits, String> {
        let NeuronsFundEconomics {
            max_theoretical_neurons_fund_participation_amount_xdr,
            contribution_threshold_xdr,
            one_third_participation_milestone_xdr,
            full_participation_milestone_xdr,
            minimum_icp_xdr_rate,
            maximum_icp_xdr_rate,
        } = NeuronsFundEconomics::try_from(neurons_fund_economics)?;

        if icp_xdr_rate <= minimum_icp_xdr_rate {
            println!(
                "{}WARNING: icp_xdr_rate ({}) is being clamped at the lower bound ({}).",
                governance::LOG_PREFIX,
                icp_xdr_rate,
                minimum_icp_xdr_rate,
            );
        }
        if icp_xdr_rate >= maximum_icp_xdr_rate {
            println!(
                "{}WARNING: icp_xdr_rate ({}) is being clamped at the upper bound ({}).",
                governance::LOG_PREFIX,
                icp_xdr_rate,
                maximum_icp_xdr_rate,
            );
        }
        let icp_xdr_rate = icp_xdr_rate.clamp(minimum_icp_xdr_rate, maximum_icp_xdr_rate);

        if icp_xdr_rate.is_zero() {
            // We don't expect this to ever happen in practice.
            return Err("icp_xdr_rate must be greater than zero.".to_string());
        }
        let xdr_icp_rate = icp_xdr_rate.inv();

        let convert_xdr_to_icp = |amount_xdr: Decimal| -> Result<Decimal, String> {
            amount_xdr.checked_mul(xdr_icp_rate).ok_or_else(|| {
                format!(
                    "Cannot convert {amount_xdr} XDR to ICP due to a Decimal overflow. xdr_icp_rate = {xdr_icp_rate}.",
                )
            })
        };

        let max_theoretical_neurons_fund_participation_amount_icp =
            convert_xdr_to_icp(max_theoretical_neurons_fund_participation_amount_xdr)?;

        let contribution_threshold_icp = convert_xdr_to_icp(contribution_threshold_xdr)?;

        let one_third_participation_milestone_icp =
            convert_xdr_to_icp(one_third_participation_milestone_xdr)?;

        let full_participation_milestone_icp =
            convert_xdr_to_icp(full_participation_milestone_xdr)?;

        Ok(NeuronsFundParticipationLimits {
            max_theoretical_neurons_fund_participation_amount_icp,
            contribution_threshold_icp,
            one_third_participation_milestone_icp,
            full_participation_milestone_icp,
        })
    }
