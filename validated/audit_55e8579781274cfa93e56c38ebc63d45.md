### Title
Inactive Dissolved Neuron with Staked Maturity Permanently Skipped by Unstake Timer Task — (`rs/nns/governance/src/neuron_store.rs`)

### Summary
`list_neurons_ready_to_unstake_maturity` uses `with_active_neurons_iter_sections`, which filters neurons via `!n.is_inactive(now)`. The `is_inactive` condition 2 checks `is_funded()`, which covers only `cached_neuron_stake_e8s` and `maturity_e8s_equivalent` (unstaked maturity), explicitly excluding `staked_maturity_e8s_equivalent`. A dissolved neuron with only staked maturity (no ICP stake, no unstaked maturity), dissolved more than 2 weeks ago, is classified as inactive and permanently excluded from the timer task that would unstake it.

### Finding Description
The `is_inactive` function at `rs/nns/governance/src/neuron/types.rs:1060` returns `true` for a neuron satisfying all four conditions:
1. Not seed/ECT neuron
2. `is_funded()` returns false — only checks stake and unstaked maturity, **not** `staked_maturity_e8s_equivalent`
3. Dissolved more than 2 weeks ago (`now - 2 * 7 * ONE_DAY_SECONDS`)
4. Not a Neurons' Fund member

Such a neuron is filtered out by `with_active_neurons_iter_sections` (line 529: `.filter(|n| !n.is_inactive(now))`), so `list_neurons_ready_to_unstake_maturity` never sees it, and `unstake_maturity_of_dissolved_neurons` never processes it. The staked maturity is permanently locked.

This state is reachable by any neuron controller:
1. Stake maturity (toggle `stake_maturity = true`)
2. Start dissolving; wait for dissolve delay to expire
3. Disburse the ICP stake (sets `cached_neuron_stake_e8s = 0`)
4. Wait 2 weeks (neuron now classified as inactive)

After step 4, the timer task will never unstake the maturity.

### Impact Explanation
`staked_maturity_e8s_equivalent` represents ICP-equivalent value that should be converted to liquid maturity upon dissolution. Permanent lock means the neuron controller loses access to this value indefinitely — it cannot be disbursed, spawned, or recovered through any user-facing operation or the timer task. The only recovery would be a governance upgrade that fixes the bug and retroactively processes affected neurons.

### Likelihood Explanation
The state is reachable through normal user operations (stake maturity, dissolve, disburse, wait). No privileged access is required. Any neuron controller can trigger this. The bug is silent — no error is logged, no alert is raised, and the timer task silently skips the neuron every 60 seconds. [7](#0-6) 

### Recommendation
Fix `is_inactive` condition 2 to include `staked_maturity_e8s_equivalent` in the funded check:

```rust
// In is_funded() or directly in is_inactive condition 2:
if self.is_funded() || self.staked_maturity_e8s_equivalent.unwrap_or(0) > 0 {
    return false;
}
```

Alternatively, add a separate dedicated iterator in `unstake_maturity_of_dissolved_neurons` that scans **all** neurons (not just active ones), similar to how `first_neuron_id` / `range_neurons_sections` is used for full-range grooming tasks.

            return false;

                return false;

            Some(ok) => ok,
