### Title
Undercharged EVM Gas for XCC Calls with wNEAR Withdrawal — Missing `WITHDRAW_GAS` in Cost Calculation - (File: engine-precompiles/src/xcc.rs)

---

### Summary

The `CrossContractCall` precompile charges EVM gas only for the router's `execute` call (`ROUTER_EXEC_BASE + call_gas`), but omits the NEAR gas cost of the mandatory `withdraw_wnear_to_router` step (`WITHDRAW_GAS = 40 TGas`) that is unconditionally scheduled whenever `required_near > 0`. This creates a ~37% undercharge on every XCC call that requires wNEAR, causing relayers to subsidize a significant portion of the actual NEAR gas cost. An unprivileged EVM user can exploit this to drain the relay service's NEAR balance, eventually causing XCC calls to stop executing and temporarily freezing user funds.

---

### Finding Description

The EVM gas cost charged by the XCC precompile is computed in `run_with_handle`:

```
cost = CROSS_CONTRACT_CALL_BASE
     + CROSS_CONTRACT_CALL_BYTE * input_len
     + promise.attached_gas / CROSS_CONTRACT_CALL_NEAR_GAS
```

where `promise.attached_gas = ROUTER_EXEC_BASE + callback_count * ROUTER_EXEC_PER_CALLBACK + call_gas`. [1](#0-0) [2](#0-1) [3](#0-2) 

However, in `handle_precompile_promise` (called after the EVM transaction completes), whenever `required_near != ZERO_YOCTO`, an additional NEAR promise is unconditionally created:

```rust
let withdraw_call = PromiseCreateArgs {
    target_account_id: current_account_id.clone(),
    method: "withdraw_wnear_to_router".into(),
    attached_gas: WITHDRAW_GAS,   // 40 TGas — never charged to user
};
``` [4](#0-3) [5](#0-4) 

`WITHDRAW_GAS = 40 TGas` is **never included** in `promise.attached_gas` and therefore **never reflected in the EVM gas charged to the user**.

`required_near > 0` in two common cases:
1. **First-time XCC user** — always, because `required_near = attached_near + STORAGE_AMOUNT` (STORAGE_AMOUNT = 2 NEAR).
2. **Any XCC call with attached NEAR** — whenever the user's promise carries `attached_near > 0`. [6](#0-5) 

For first-time users, additional uncharged NEAR gas also includes `INITIALIZE_GAS` (15 TGas), `VERSION_UPDATE_GAS` (5 TGas), and `REFUND_GAS` (5 TGas): [7](#0-6) 

**Quantified undercharge for an existing user with `attached_near > 0`:**

| Cost component | NEAR gas | EVM gas equivalent |
|---|---|---|
| `ROUTER_EXEC_BASE` (charged) | 7 TGas | 40,000 |
| `WITHDRAW_GAS` (**not charged**) | 40 TGas | **228,571** |

The uncharged portion is ≈ 228,571 EVM gas vs. the charged ≈ 383,650 EVM gas — a **~37% undercharge** on the NEAR gas actually consumed by the relayer.

The conversion ratio `CROSS_CONTRACT_CALL_NEAR_GAS = 175_000_000` is derived from a historical gas report and is hardcoded as a fixed constant, with no mechanism to adjust for the variable overhead of the withdrawal step: [8](#0-7) 

---

### Impact Explanation

Every XCC call with `required_near > 0` causes the relayer to pay ~40 TGas of NEAR gas that is not compensated by the EVM gas fees collected from the user. An attacker who repeatedly submits XCC calls with `attached_near > 0` forces the relay service to absorb this cost on every call. If the relay service's NEAR balance is exhausted, it stops submitting NEAR transactions for XCC calls. At that point, users who have already had their EVM transaction execute (and their EVM state updated) will find their NEAR cross-contract calls never dispatched — a **temporary freezing of funds** (High impact).

---

### Likelihood Explanation

The condition `required_near > 0` is triggered by any XCC call that attaches NEAR to the cross-contract call, which is the normal use case for XCC (e.g., paying for storage deposits on target contracts). First-time XCC users always trigger it. The attack requires no special privileges — any EVM address can call the XCC precompile at `0x516cded1d16af10cad47d6d49128e2eb7d27b372`. The attacker's only cost is the (undercharged) EVM gas and the wNEAR they attach, while the relay service absorbs the 40 TGas overhead per call. [9](#0-8) 

---

### Recommendation

Include the `WITHDRAW_GAS` cost in the EVM gas charged when `required_near > 0`. Since `required_near` is known at precompile execution time, the cost can be conditionally added:

```rust
if required_near != ZERO_YOCTO {
    cost += EthGas::new(WITHDRAW_GAS.as_u64() / costs::CROSS_CONTRACT_CALL_NEAR_GAS);
    check_cost(cost)?;
}
```

Similarly, for first-time users, add `INITIALIZE_GAS + VERSION_UPDATE_GAS + REFUND_GAS` to the charged cost. Consider also periodically re-calibrating `CROSS_CONTRACT_CALL_NEAR_GAS` as NEAR protocol gas costs evolve. [10](#0-9) 

---

### Proof of Concept

1. Attacker holds wNEAR and calls the XCC precompile with:
   - `CrossContractCallArgs::Eager(PromiseArgs::Create(promise))` where `promise.attached_balance > 0`
   - This sets `required_near = attached_near > 0`
2. EVM gas charged to attacker: `CROSS_CONTRACT_CALL_BASE + ROUTER_EXEC_BASE / CROSS_CONTRACT_CALL_NEAR_GAS ≈ 343,650 + 40,000 = 383,650` EVM gas.
3. Relayer submits the NEAR transaction; `handle_precompile_promise` schedules `withdraw_wnear_to_router` with `WITHDRAW_GAS = 40 TGas`.
4. Relayer's actual NEAR gas: `7 TGas (ROUTER_EXEC_BASE) + 40 TGas (WITHDRAW_GAS) = 47 TGas` → EVM equivalent: `268,571` EVM gas.
5. **Undercharge per call: 228,571 EVM gas worth of NEAR gas paid by relayer, not by attacker.**
6. Attacker repeats across many calls; relay service NEAR balance depletes; XCC calls cease executing; user funds are temporarily frozen. [11](#0-10) [12](#0-11)

pub mod costs {
    use crate::prelude::types::{EthGas, NearGas};

    /// Base EVM gas cost for calling this precompile.
    /// Value obtained from the following methodology:
    /// 1. Estimate the cost of calling this precompile in terms of NEAR gas.
    ///    This is done by calling the precompile with inputs of different lengths
    ///    and performing a linear regression to obtain a function
    ///    `NEAR_gas = CROSS_CONTRACT_CALL_BASE + (input_length) * (CROSS_CONTRACT_CALL_BYTE)`.
    /// 2. Convert the NEAR gas cost into an EVM gas cost using the conversion ratio below
    ///    (`CROSS_CONTRACT_CALL_NEAR_GAS`).
    ///
    /// This process is done in the `test_xcc_eth_gas_cost` test in
    /// `engine-tests/src/tests/xcc.rs`.
    pub const CROSS_CONTRACT_CALL_BASE: EthGas = EthGas::new(343_650);
    /// Additional EVM gas cost per bytes of input given.
    /// See `CROSS_CONTRACT_CALL_BASE` for estimation methodology.
    pub const CROSS_CONTRACT_CALL_BYTE: EthGas = EthGas::new(4);
    /// EVM gas cost per NEAR gas attached to the created promise.
    /// This value is derived from the gas report `https://hackmd.io/@birchmd/Sy4piXQ29`
    /// The units on this quantity are `NEAR Gas / EVM Gas`.
    /// The report gives a value `0.175 T(NEAR_gas) / k(EVM_gas)`. To convert the units to
    /// `NEAR Gas / EVM Gas`, we simply multiply `0.175 * 10^12 / 10^3 = 175 * 10^6`.
    pub const CROSS_CONTRACT_CALL_NEAR_GAS: u64 = 175_000_000;

    pub const ROUTER_EXEC_BASE: NearGas = NearGas::new(7_000_000_000_000);
    pub const ROUTER_EXEC_PER_CALLBACK: NearGas = NearGas::new(12_000_000_000_000);
    pub const ROUTER_SCHEDULE: NearGas = NearGas::new(5_000_000_000_000);
}
