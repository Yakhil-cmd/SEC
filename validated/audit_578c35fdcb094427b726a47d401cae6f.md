### Title
Authorized Accounts Can Permanently Keep Exit Precompiles Paused, Freezing User Bridge Funds - (File: `engine/src/contract_methods/admin.rs`)

---

### Summary

The `pause_precompiles` function is callable by any account in the `EngineAuthorizer` ACL — a role separate from and less trusted than the owner. Unlike `pause_contract`, which enforces a state machine (`require_running` prevents double-pause), `pause_precompiles` has no such guard and can be called an unlimited number of times. Meanwhile, `resume_precompiles` is owner-only. A malicious or compromised authorized account can repeatedly re-pause the `ExitToNear` and `ExitToEthereum` precompiles immediately after the owner resumes them, permanently blocking all bridge withdrawals and freezing user funds inside Aurora.

---

### Finding Description

**Asymmetric access control between pause and resume:**

`pause_precompiles` uses the `EngineAuthorizer` ACL:

```rust
// engine/src/contract_methods/admin.rs:226-240
pub fn pause_precompiles<I: IO + Copy, E: Env>(io: I, env: &E) -> Result<(), ContractError> {
    with_hashchain(io, env, function_name!(), |io| {
        require_running(&state::get_state(&io)?)?;
        let authorizer: EngineAuthorizer = engine::get_authorizer(&io);
        if !authorizer.is_authorized(&env.predecessor_account_id()) {
            return Err(b"ERR_UNAUTHORIZED".into());
        }
        let args: PausePrecompilesCallArgs = io.read_input_borsh()?;
        let flags = PrecompileFlags::from_bits_truncate(args.paused_mask);
        let mut pauser = EnginePrecompilesPauser::from_io(io);
        pauser.pause_precompiles(flags);
        Ok(())
    })
}
```

`resume_precompiles` is owner-only:

```rust
// engine/src/contract_methods/admin.rs:209-223
pub fn resume_precompiles<I: IO + Copy, E: Env>(io: I, env: &E) -> Result<(), ContractError> {
    with_hashchain(io, env, function_name!(), |io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;
        let predecessor_account_id = env.predecessor_account_id();
        require_owner_only(&state, &predecessor_account_id)?;
        ...
    })
}
```

**No state machine guard on `pause_precompiles`:**

`pause_contract` enforces `require_running`, preventing a double-pause. `pause_precompiles` has no equivalent guard — it simply ORs flags into storage unconditionally:

```rust
// engine/src/pausables.rs:163-167
fn pause_precompiles(&mut self, precompiles_to_pause: PrecompileFlags) {
    let mut pause_flags = self.read_flags_from_storage();
    pause_flags.insert(precompiles_to_pause);
    self.write_flags_into_storage(pause_flags);
}
```

**Precompile pause state is never reset on upgrade:**

`deploy_upgrade` only removes the `CODE_STAGE_KEY` and self-deploys; it does not clear `PAUSE_FLAGS`:

```rust
// engine/src/lib.rs:183-184
Runtime::self_deploy(&bytes_to_key(KeyPrefix::Config, CODE_KEY));
io.remove_storage(&bytes_to_key(KeyPrefix::Config, CODE_STAGE_KEY));
```

`state_migration` is currently a no-op:

```rust
// engine/src/lib.rs:192-194
pub extern "C" fn state_migration() {
    // TODO: currently we don't have migrations
}
```

**The only pausable precompiles are the bridge exit paths:**

```rust
// engine/src/pausables.rs:13-16
pub struct PrecompileFlags: u32 {
    const EXIT_TO_NEAR     = 0b01;
    const EXIT_TO_ETHEREUM = 0b10;
}
```

`ExitToNear` and `ExitToEthereum` are the sole mechanisms by which EVM users withdraw assets from Aurora back to NEAR or Ethereum.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

If an authorized account in the `EngineAuthorizer` ACL is malicious or compromised, it can execute the following loop indefinitely:

1. Call `pause_precompiles(EXIT_TO_NEAR | EXIT_TO_ETHEREUM)`.
2. Owner calls `resume_precompiles`.
3. Authorized account immediately calls `pause_precompiles` again.

Because there is no per-upgrade-cycle limit, no cooldown, and no state machine guard, the authorized account wins any race against the owner. All EVM users holding ETH, ERC-20 tokens, or NEP-141 mirror tokens inside Aurora are unable to exit. Their funds are frozen inside the Aurora EVM with no recourse, since the only exit precompiles are blocked.

Additionally, because `deploy_upgrade` and `state_migration` do not clear `PAUSE_FLAGS`, even a successful contract upgrade does not automatically restore bridge functionality.

---

### Likelihood Explanation

**Medium.** The `EngineAuthorizer` ACL is a distinct, lower-trust role from the owner — it is specifically designed to allow non-owner accounts to pause precompiles as an emergency measure. This is directly analogous to the Security Council role in the referenced report. A compromised or malicious account in this ACL (e.g., a hot-wallet key used for emergency response) can execute this attack without any owner-level access. The attack requires no special on-chain conditions, no gas tricks, and is executable in a single transaction per cycle.

---

### Recommendation

1. **Add a per-upgrade-cycle pause limit.** Track a `pause_epoch` counter that increments on each `deploy_upgrade` or `upgrade`. Allow `pause_precompiles` to succeed at most once per epoch for each authorized account, analogous to the `UnfrozenUntilUpgrade` state proposed in the referenced report.

2. **Reset `PAUSE_FLAGS` on upgrade.** In `deploy_upgrade` (or `state_migration`), clear the precompile pause flags so that a successful upgrade automatically restores bridge functionality.

3. **Alternatively, make `pause_precompiles` owner-only**, and restrict the `EngineAuthorizer` ACL to a read-only advisory role, removing the asymmetry between pause and resume.

---

### Proof of Concept

```
// Attacker controls `authorized.near`, which is in the EngineAuthorizer ACL.

Step 1: authorized.near → pause_precompiles({ paused_mask: 0b11 })
        // EXIT_TO_NEAR and EXIT_TO_ETHEREUM are now paused.
        // All user bridge withdrawals fail with ERR_PAUSED.

Step 2: owner.near → resume_precompiles({ paused_mask: 0b11 })
        // Precompiles resumed.

Step 3: authorized.near → pause_precompiles({ paused_mask: 0b11 })
        // No state machine check prevents this. Precompiles paused again.

Step 4: Repeat steps 2-3 indefinitely.
        // Owner cannot win the race; authorized.near can front-run every resume.
        // User funds are permanently frozen inside Aurora EVM.
        // deploy_upgrade does not clear PAUSE_FLAGS, so even an upgrade
        // does not restore bridge functionality.
```

    /// Creates new [`EngineAuthorizer`] and grants permission to pause precompiles for all given `accounts`.
    pub fn from_accounts(accounts: impl Iterator<Item = AccountId>) -> Self {
        Self {
            acl: accounts.collect(),
        }
    }
}
