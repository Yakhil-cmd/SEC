### Title
Unbounded BFS Loop in SNS Governance `cast_vote_and_cascade_follow` Exceeds Instruction Limit - (File: rs/sns/governance/src/governance.rs)

---

### Summary

The SNS Governance canister's `cast_vote_and_cascade_follow` function contains an unbounded synchronous BFS loop over the neuron follower graph with no instruction-limit guard. Any unprivileged neuron holder can trigger this loop via `register_vote`, causing the update call to trap with `CanisterInstructionLimitExceeded` when the follower graph is sufficiently large. The NNS Governance canister received a fix for the same class of bug (a state-machine with soft/hard instruction limits), but the SNS Governance canister did not.

---

### Finding Description

`Governance::cast_vote_and_cascade_follow` in SNS Governance is a **synchronous, non-async** static method: [1](#0-0) 

It performs BFS over the follower graph using a `while !induction_votes.is_empty()` loop: [2](#0-1) 

There is **no instruction counter check** anywhere inside this loop. The loop terminates only when no more neurons can be swayed — bounded solely by the total number of neurons and their following relationships.

This function is called synchronously from `register_vote`, which is an ingress-reachable update method: [3](#0-2) 

By contrast, the NNS Governance canister received an explicit fix for this same class of bug. It now uses a `ProposalVotingStateMachine` with a `SOFT_VOTING_INSTRUCTIONS_LIMIT` (1 billion) and `HARD_VOTING_INSTRUCTIONS_LIMIT` (750 billion), and can yield via `noop_self_call_if_over_instructions`: [4](#0-3) [5](#0-4) 

The SNS Governance canister has no equivalent protection.

**Secondary unbounded loop — `distribute_rewards`:**

The SNS `distribute_rewards` function also contains nested loops over all settled proposals and all their ballots (one ballot per neuron at proposal creation time), followed by a loop over all rewarded neurons, with no instruction limit check: [6](#0-5) [7](#0-6) 

The NNS Governance equivalent was refactored to distribute rewards asynchronously in batches with an instruction limit guard (`DISTRIBUTION_MESSAGE_LIMIT`): [8](#0-7) 

---

### Impact Explanation

When `cast_vote_and_cascade_follow` is triggered on a proposal whose follower graph spans a large number of neurons, the IC execution environment traps the update call with `CanisterInstructionLimitExceeded` (currently ~40 billion instructions per update message). Because the function is synchronous and stateless with respect to progress, **no partial progress is saved** — the vote cascade cannot be resumed. This means:

- Voting on any proposal with a sufficiently large follower graph becomes permanently impossible for the duration of that proposal.
- If the SNS has grown to a large number of neurons with chained following, **all voting** on non-critical proposals (which use catch-all following) can be blocked.
- `distribute_rewards` trapping in the timer causes reward distribution to stall indefinitely for that SNS.

---

### Likelihood Explanation

SNS neurons are created by any user who stakes SNS tokens — there is no privileged gate. The `NervousSystemParameters::MAX_NUMBER_OF_NEURONS_CEILING` is a large value: [9](#0-8) 

Following relationships are set freely by neuron holders. A chain of N neurons each following the previous one causes the BFS to process N tiers sequentially. At ~120K instructions per neuron (the figure cited in NNS Governance comments for maturity operations), a chain of ~330,000 neurons would exhaust 40 billion instructions. Even at smaller scales (tens of thousands of neurons), the loop can exceed the limit. SNS token distributions routinely create thousands of neurons via swap baskets, making this a realistic scenario without any deliberate attack.

---

### Recommendation

1. **Port the NNS Governance state-machine pattern to SNS Governance.** Replace the synchronous `cast_vote_and_cascade_follow` with an async, resumable `ProposalVotingStateMachine` that checks `is_message_over_threshold` after each BFS tier and stores intermediate state in stable memory, mirroring: [10](#0-9) 

2. **Add an instruction-limit guard to `distribute_rewards`.** Refactor SNS `distribute_rewards` to distribute rewards in batches across multiple timer firings, as NNS Governance does via `RewardsDistributionStateMachine`: [8](#0-7) 

3. **Document and enforce a maximum follower-chain depth** or a maximum number of neurons per SNS that keeps the worst-case BFS within the instruction budget.

---

### Proof of Concept

1. Deploy an SNS with SNS token distribution to N participants (e.g., N = 50,000 via swap).
2. Each participant's neuron sets its following on a non-critical topic to the previous participant's neuron, forming a chain of length N.
3. The first neuron in the chain submits and votes `Yes` on a Motion proposal.
4. `register_vote` is called, which invokes `cast_vote_and_cascade_follow`.
5. The BFS loop at line 3749 processes all N tiers synchronously with no instruction check.
6. The IC execution environment traps the update call with `CanisterInstructionLimitExceeded`.
7. The vote is not recorded; the cascade is not saved; the proposal cannot be voted on by any neuron whose ballot would be filled via following.

with the call site at: [11](#0-10)

            // Process the current tier in the BFS.
            for (current_neuron_id, current_new_vote) in &induction_votes {
                let current_ballot = match ballots.get_mut(current_neuron_id) {
                    Some(b) => b,
                    None => {
                        // neuron_id has no (blank) ballot, which means they
                        // were not eligible when the proposal was first
                        // created. This is fairly unusual, but does not
                        // indicate a bug (therefore, no log).
                        continue;

                };

                // Only fill in "blank" ballots. I.e. those with vote ==
                // Unspecified. This check could just as well be done before
                // current_neuron_id is added to induction_votes.
                if current_ballot.vote != (Vote::Unspecified as i32) {
                    continue;
                }

                // Fill in current_ballot.
                assert_ne!(*current_new_vote, Vote::Unspecified);
                current_ballot.vote = *current_new_vote as i32;
                current_ballot.cast_timestamp_seconds = now_seconds;

                // Take note of the followers of current_neuron_id, and add them
                // to the next "tier" in the BFS.

                if let Some(new_follower_neuron_ids) = topic_followers
                    .and_then(|topic_followers| topic_followers.get(current_neuron_id))
                {
                    for follower_neuron_id in new_follower_neuron_ids {
                        follower_neuron_ids.insert(follower_neuron_id.clone());
                    }
                }

                if let Some(new_follower_neuron_ids) =
                    neuron_id_to_follower_neuron_ids.get(current_neuron_id)
                {
                    for follower_neuron_id in new_follower_neuron_ids {
                        follower_neuron_ids.insert(follower_neuron_id.clone());
                    }
                }
            }

            // Prepare for the next iteration of the (outer most) loop by
            // constructing the next BFS tier (from follower_neuron_ids).
            induction_votes.clear();
            for follower_neuron_id in follower_neuron_ids {
                let Some(follower_neuron) = neurons.get(&follower_neuron_id.to_string()) else {
                    // This is a highly suspicious, because currently, we do not
                    // delete neurons, which means that we have an invalid NeuronId
                    // floating around in the system, which indicates that we have a
                    // bug. For now, we deal with that by logging, and pretending like
                    // we did not see follower_neuron_id.
                    log!(

                        "Missing neuron {} while trying to record (and cascade) \

                        follower_neuron_id,
                        proposal_id,
                    );
                    continue;
                };

                let follower_vote = follower_neuron.vote_from_ballots_following(
                    function_id,
                    topic,
                    ballots,
                    proposal_id,
                );

                if follower_vote != Vote::Unspecified {
                    // follower_neuron would be swayed by its followees!
                    //
                    // This is the other (earlier) point at which we could
                    // consider whether a neuron is already locked in, and that
                    // no recursion is needed.
                    induction_votes.insert(follower_neuron_id.to_string(), follower_vote);
                }
            }

    fn continue_processing(
        &mut self,
        neuron_store: &mut NeuronStore,
        ballots: &mut HashMap<u64, Ballot>,
        is_over_instructions_limit: fn() -> bool,
    ) {
        let voting_finished = self.is_voting_finished();

        if !voting_finished {
            while let Some(neuron_id) = self.neurons_to_check_followers.pop_first() {
                self.add_followers_to_check(neuron_store, neuron_id, self.topic);

                // Before we check the next one, see if we're over the limit.
                if is_over_instructions_limit() {
                    return;
                }
            }

            // Memory optimization, will not cause tests to fail if removed
            retain_neurons_with_castable_ballots(&mut self.followers_to_check, ballots);

            while let Some(follower) = self.followers_to_check.pop_first() {
                let vote = match neuron_store
                    .neuron_would_follow_ballots(follower, self.topic, ballots)
                {
                    Ok(vote) => vote,
                    Err(e) => {
                        // This is a bad inconsistency, but there is
                        // nothing that can be done about it at this
                        // place.  We somehow have followers recorded that don't exist.
                        eprintln!(
                            "error in cast_vote_and_cascade_follow when gathering induction votes: {:?}",
                            e
                        );
                        Vote::Unspecified
                    }
                };
                // Casting vote immediately might affect other follower votes, which makes
                // voting resolution take fewer iterations.
                // Vote::Unspecified is ignored by cast_vote.
                self.cast_vote(ballots, follower, vote);

                if is_over_instructions_limit() {
                    return;
                }
            }
        } else {
            while let Some((neuron_id, vote)) = self.recent_neuron_ballots_to_record.pop_first() {
                match neuron_store.record_neuron_vote(neuron_id, self.topic, self.proposal_id, vote)
                {
                    Ok(_) => {}
                    Err(e) => {
                        // This is a bad inconsistency, but there is
                        // nothing that can be done about it at this
                        // place.  We somehow have followers recorded that don't exist.
                        eprintln!(
                            "error in cast_vote_and_cascade_follow when gathering induction votes: {:?}",
                            e
                        );
                    }
                };

                if is_over_instructions_limit() {
                    return;
                }
            }
        }
    }
