Audit Report

## Title
Unprivileged Caller Can DoS Victim's `update_balance` Calls via Arbitrary `owner` Parameter - (File: rs/bitcoin/ckbtc/minter/src/updates/update_balance.rs)

## Summary
The `update_balance` endpoint accepts an `owner: Option<Principal>` parameter that any caller can set to an arbitrary principal. The function acquires a per-account concurrency guard keyed on the supplied owner before making async inter-canister calls. An unprivileged attacker can continuously call `update_balance` with a victim's principal as `owner`, keeping the victim's account locked in `update_balance_accounts` and causing the victim's own `update_balance` calls to always return `AlreadyProcessing`.

## Finding Description
`update_balance` in `rs/bitcoin/ckbtc/minter/src/updates/update_balance.rs` constructs the target account from the caller-supplied `args.owner` with no authorization check on who may supply a foreign principal:

```rust
let caller_account = Account {
    owner: args.owner.unwrap_or(caller),
    subaccount: args.subaccount,
};
let _guard = balance_update_guard(caller_account)?;
``` [1](#0-0) 

The only guard against misuse is a trap preventing the minter's own canister ID from being used as owner:

```rust
if args.owner.unwrap_or(caller) == runtime.id() {
    ic_cdk::trap("cannot update minter's balance");
}
``` [2](#0-1)
