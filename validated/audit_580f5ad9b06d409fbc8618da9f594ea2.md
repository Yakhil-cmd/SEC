Audit Report

## Title
`created_timestamp_seconds` Not Initialized in Genesis Developer Neurons Causes Immediate Vesting Bypass - (File: rs/sns/init/src/distributions.rs)

## Summary
`FractionalDeveloperVotingPower::create_neuron` constructs genesis developer neurons using `..Default::default()`, leaving `created_timestamp_seconds` at `0`. The `is_vesting` check computes `0 + vesting_period_seconds >= now`, which is always `false` against any real Unix timestamp, silently nullifying the entire vesting schedule for every genesis developer neuron from the moment of SNS deployment.

## Finding Description
In `rs/sns/init/src/distributions.rs`, `create_neuron` (L119–162) builds the neuron struct with `vesting_period_seconds` correctly copied from `NeuronDistribution` but never assigns `created_timestamp_seconds`, leaving it at the `u64` default of `0`:

```rust
Ok(Neuron {
    ...
    vesting_period_seconds,
    ..Default::default()   // created_timestamp_seconds = 0
})
``` [1](#0-0) 

`get_initial_neurons` (L51–81) calls `create_neuron` with no timestamp parameter, so there is no path by which `created_timestamp_seconds` can receive a non-zero value in the genesis flow. [2](#0-1) 

The vesting guard in `rs/sns/governance/src/neuron.rs` evaluates:

```rust
self.created_timestamp_seconds + vesting_period_seconds >= now
``` [3](#0-2) 

With `created_timestamp_seconds = 0` and any realistic `vesting_period_seconds` (e.g., 3 years ≈ 94,608,000 s), the expression `94,608,000 >= ~1,720,000,000` is `false`, so `is_vesting` returns `false` for every genesis developer neuron.

`check_command_is_valid_if_neuron_is_vesting` in `rs/sns/governance/src/governance.rs` short-circuits immediately when `is_vesting` returns `false`:

```rust
if !neuron.is_vesting(self.env.now()) {
    return Ok(());
}
``` [4](#0-3) 

This allows `StartDissolving`, `Disburse`, `Split`, `IncreaseDissolveDelay`, and `SetDissolveTimestamp` to execute without restriction. [5](#0-4) 

By contrast, both the `claim_or_refresh_neuron` path and `claim_swap_neurons` explicitly set `created_timestamp_seconds: now`: [6](#0-5) [7](#0-6) 

The genesis developer neuron path is the sole omission.

## Impact Explanation
A developer controlling a genesis neuron configured with `vesting_period_seconds` can immediately call `StartDissolving` after SNS launch, wait out only the dissolve delay, then `Disburse` all staked SNS tokens — bypassing years of the published vesting schedule. Token holders who participated in the decentralization swap based on the advertised developer vesting commitments receive no protection. Developer allocations in SNS launches routinely represent millions of dollars in token value. This matches the High bounty impact: **unauthorized access to governance assets / canister-controlled funds with concrete user harm**, and potentially Critical if the developer allocation exceeds $1M.

## Likelihood Explanation
Every SNS that sets `vesting_period_seconds` on developer neurons is affected from genesis. Exploitation requires only that the developer call the standard `manage_neuron` ingress endpoint on their own neuron — no privileged access, no governance majority, no key compromise. The neuron appears correctly configured (the `vesting_period_seconds` field is non-zero), so the bypass is silent and undetectable from on-chain state inspection alone.

## Recommendation
In `rs/sns/init/src/distributions.rs`, pass the SNS genesis timestamp into `create_neuron` and assign it explicitly:

```rust
fn create_neuron(
    &self,
    neuron_distribution: &NeuronDistribution,
    voting_power_percentage_multiplier: u64,
    parameters: &NervousSystemParameters,
    now: u64,   // <-- add genesis timestamp
) -> Result<Neuron, String> {
    ...
    Ok(Neuron {
        ...
        vesting_period_seconds,
        created_timestamp_seconds: now,       // <-- add this
        aging_since_timestamp_seconds: now,   // <-- add this
        ..Default::default()
    })
}
```

`get_initial_neurons` must also accept and forward `now`. This mirrors the pattern already used in `claim_swap_neurons` and `claim_or_refresh_neuron`.

## Proof of Concept
1. Deploy an SNS with a developer neuron configured with `vesting_period: 3 years`, `dissolve_delay: 1 year`.
2. Immediately after genesis, call `manage_neuron` → `Configure` → `StartDissolving` on the developer neuron.
3. Observe the call succeeds — no `"Neuron X is vesting and cannot call StartDissolving"` error is returned.
4. Confirm by reading the neuron: `created_timestamp_seconds = 0`, `vesting_period_seconds = 94608000`; evaluating `is_vesting` with current time returns `false`.
5. After 1 year (dissolve delay), call `Disburse` and receive all staked SNS tokens as liquid funds, 2 years ahead of the published schedule.

A deterministic unit test can reproduce this without mainnet access: construct a `Neuron` with `created_timestamp_seconds: 0` and `vesting_period_seconds: Some(94_608_000)`, call `is_vesting(1_720_000_000)`, and assert it incorrectly returns `false`.

    pub fn get_initial_neurons(
        &self,
        parameters: &NervousSystemParameters,
    ) -> Result<BTreeMap<String, Neuron>, String> {
        let developer_neurons = &self.developer_distribution()?.developer_neurons;

        let swap = self.swap_distribution()?;

        // Multiplying this way will give the developer_voting_power_percentage_multiplier

        let developer_voting_power_percentage_multiplier = ((swap.initial_swap_amount_e8s as u128)
            * 100)
            .checked_div(swap.total_e8s as u128)
            .expect(
                "Underflow detected when calculating developer voting power percentage multiplier",
            ) as u64;

        let mut initial_neurons = btreemap! {};

        for developer_neuron_distribution in developer_neurons {
            let neuron = self.create_neuron(
                developer_neuron_distribution,
                developer_voting_power_percentage_multiplier,
                parameters,
            )?;

            initial_neurons.insert(neuron.id.as_ref().unwrap().to_string(), neuron);
        }

        Ok(initial_neurons)
    }
