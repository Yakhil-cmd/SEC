Audit Report

## Title
Gzip Footer Manipulation Exhausts Round Instruction Budget via Uncapped `compilation_cost` in Error Path — (`rs/execution_environment/src/hypervisor.rs`, `rs/embedders/src/wasm_utils/decoding.rs`)

## Summary

`wasm_encoding_and_size` reads the RFC-1952 gzip footer's ISIZE field as a raw `u32` cast to `usize` with no cap. `Hypervisor::create_execution_state` multiplies this attacker-controlled value by 6,000 to produce `compilation_cost`, then deducts the full inflated amount from `round_limits.instructions` in the `Err` branch when `wasm_executor.create_execution_state` rejects the module as too large. A single crafted `install_code` call can drive `round_limits.instructions` deeply negative, starving all other canisters on the subnet of execution for that round.

## Finding Description

**Step 1 — Uncapped attacker-controlled size**

`wasm_encoding_and_size` reads the last 4 bytes of the gzip stream as a raw `u32` and casts directly to `usize` with no upper bound:

```rust
let uncompressed_size = u32::from_le_bytes(isize_bytes) as usize;
return Ok((WasmEncoding::Gzip, uncompressed_size));
```

The function's own doc-comment explicitly warns this value is untrusted. [1](#0-0) 

**Step 2 — Uncapped size drives `compilation_cost`**

`Hypervisor::create_execution_state` calls `decoded_wasm_size`, which delegates to `wasm_encoding_and_size`. Since the crafted gzip returns `Ok(4_294_967_295)`, the `if let Err` guard at L161 is **not** triggered. The code proceeds to compute:

```rust
let compilation_cost = self.cost_to_compile_wasm_instruction * wasm_size as u64;
// = 6_000 × 4_294_967_295 = 25_769_803_770_000
```

No overflow (fits in `u64`), no cap applied. [2](#0-1) 

**Step 3 — `decode_wasm` rejects the module inside `wasm_executor`**

`wasm_executor.create_execution_state` calls `decode_wasm(wasm_max_size, ...)`. Since `4_294_967_295 > 100 MiB`, `decode_wasm` returns `Err(ModuleTooLarge)` before any decompression occurs. [3](#0-2) [4](#0-3) 

**Step 4 — `Err` branch deducts the inflated cost from `round_limits`**

Back in `Hypervisor::create_execution_state`, the `Err` arm uses the pre-computed, attacker-inflated `compilation_cost`:

```rust
Err(err) => {
    let total_cost = self.create_execution_state_base_cost + compilation_cost;
    round_limits.instructions -= as_round_instructions(total_cost);
    (total_cost, Err(err))
}
```

Subtracting ~25.7 trillion from a typical round budget of ~7 billion drives `round_limits.instructions` deeply negative. The scheduler treats a non-positive value as "round exhausted" and stops executing further messages. [5](#0-4) 

**Step 5 — No upstream guard**

`execute_install` calls `create_execution_state` directly after assembling the module, with no pre-check on the decoded size. [6](#0-5) 

The default cost constant is confirmed at 6,000 instructions per Wasm instruction. [7](#0-6) 

## Impact Explanation

Every execution round in which the attacker submits this crafted `install_code` message has its instruction budget exhausted before any other canister message runs. All other canisters on the same subnet are starved of execution for that round. The attack is repeatable every round at the cost of one failing `install_code` call, constituting a sustained subnet-wide availability impact. This matches the **High** bounty impact: *"Application/platform-level DoS … or subnet availability impact not based on raw volumetric DDoS."* Given the trivial execution requirements (no privileges, ~20-byte payload, repeatable), it may also qualify as **Critical** under *"Easy to perform without special privileges, low cost, and severe global impact."*

## Likelihood Explanation

Requires only: a canister ID (create one with minimal cycles), a ~20-byte crafted gzip blob with `ISIZE = 0xFFFFFFFF` in the footer, and a standard `install_code` ingress call. No privileged access, no key material, no governance majority, no victim interaction. Locally testable and reproducible. Repeatable every round.

## Recommendation

Cap `wasm_size` to `wasm_max_size` before computing `compilation_cost` in `Hypervisor::create_execution_state`, since `decoded_wasm_size` is explicitly documented as untrusted for gzip:

```rust
let wasm_size = wasm_size.min(self.wasm_max_size.get() as usize);
let compilation_cost = self.cost_to_compile_wasm_instruction * wasm_size as u64;
```

Alternatively, move the `decode_wasm` size check to occur before `compilation_cost` is computed, so an oversized footer causes an early `Err` return that charges only the base cost.

## Proof of Concept

```python
# Minimal valid gzip: 10-byte header + 2-byte empty deflate + 8-byte footer
# Footer: CRC32=0x00000000, ISIZE=0xFFFFFFFF
payload  = b'\x1f\x8b\x08\x00\x00\x00\x00\x00\x00\x03'  # gzip header
payload += b'\x03\x00'                                    # empty deflate block
payload += b'\x00\x00\x00\x00'                           # CRC32
payload += b'\xff\xff\xff\xff'                           # ISIZE = u32::MAX

# Submit via install_code to any canister you control.
# Expected: install_code fails with ModuleTooLarge.
# Observed: round_limits.instructions drops by ~25.7 trillion.
# All other canister messages in the same round are skipped.
```

A deterministic integration test can confirm this by:
1. Constructing the above blob.
2. Calling `install_code` on a test canister.
3. Asserting that `round_limits.instructions` after the call is negative (or that a second canister's message in the same round was not executed).

```rust
/// # Warning
///
/// If the Wasm is gzipped, then the returned size cannot be trusted. It would
/// come from the gzip footer which could have been manipulated.
fn wasm_encoding_and_size(
    module_bytes: &[u8],
) -> Result<(WasmEncoding, usize), WasmValidationError> {
    // \0asm is WebAssembly module magic bytes.
    // https://webassembly.github.io/spec/core/binary/modules.html#binary-module
    if module_bytes.starts_with(b"\x00asm") {
        return Ok((WasmEncoding::Wasm, module_bytes.len()));
    }

    // 1f 8b is GZIP magic number, 08 is DEFLATE algorithm.
    // https://datatracker.ietf.org/doc/html/rfc1952.html#page-6
    if module_bytes.starts_with(b"\x1f\x8b\x08") {
        // There should be at least an 8-byte header and an 8-byte footer.
        if module_bytes.len() < 16 {
            return Err(WasmValidationError::DecodingError(
                "invalid Wasm module: gzip stream is too short".to_string(),
            ));
        }

        // Get the uncompressed size from the footer.
        // The size is in the last 4 bytes in little-endian encoding.
        // https://datatracker.ietf.org/doc/html/rfc1952.html#page-5
        let mut isize_bytes = [0_u8; 4];
        // We checked the size in advance so it's safe to access the last 4 bytes.
        isize_bytes.copy_from_slice(&module_bytes[module_bytes.len() - 4..module_bytes.len()]);
        let uncompressed_size = u32::from_le_bytes(isize_bytes) as usize;
        return Ok((WasmEncoding::Gzip, uncompressed_size));
```
