Audit Report

## Title
`StableNeuronStore::delete` Omits `maturity_disbursements_map` Cleanup, Enabling Permanent DoS of Maturity Disbursement Finalization Timer — (`rs/nns/governance/src/storage/neurons.rs`)

## Summary

`StableNeuronStore::delete` clears every auxiliary map except `maturity_disbursements_map` and does not remove the corresponding `MaturityDisbursementIndex` entry. Because `disburse_neuron` has no guard against pending maturity disbursements, an attacker can initiate `DisburseMaturity`, then dissolve and disburse the neuron's stake, leaving orphaned index entries. The finalization timer then permanently loops on `NeuronNotFound`, blocking all users' maturity disbursement finalizations.

## Finding Description

**Root cause — `StableNeuronStore::delete` omits `maturity_disbursements_map` cleanup:**

`rs/nns/governance/src/storage/neurons.rs` lines 411–431 show that `delete` clears `hot_keys_map`, `recent_ballots_map`, followees, `known_neuron_data_map`, and `transfer_map`, but never calls `update_repeated_field` on `maturity_disbursements_map`. The `MaturityDisbursementIndex` entry (in `stable_neuron_indexes`) is similarly not removed.

**Enabling condition — `disburse_neuron` has no guard for pending maturity disbursements:**

`rs/nns/governance/src/governance.rs` lines 1970–1995 show the only precondition checks are: caller is controller, neuron state is `Dissolved`, and KYC verified. There is no check for `maturity_disbursements_in_progress.is_empty()`. A neuron with pending maturity disbursements can be fully deleted via `Disburse`.

**Finalization timer loops permanently on orphaned entries:**

`rs/nns/governance/src/governance/disburse_maturity.rs` lines 462–475 show `next_maturity_disbursement_to_finalize` reads from `MaturityDisbursementIndex` (which still contains the orphaned entry), then calls `neuron_store.with_neuron(&neuron_id, ...)`. For a deleted neuron this returns `NeuronStoreError::NeuronNotFound`, mapped to `FinalizeMaturityDisbursementError::NeuronNotFound`. Lines 547–553 show `finalize_maturity_disbursement` returns `RETRY_INTERVAL` on any error. The index is never self-healed on the error path, so the next invocation finds the same orphaned entry and fails again, indefinitely.

## Impact Explanation

This is a **High** severity finding matching the allowed impact: *"Application/platform-level DoS … not based on raw volumetric DDoS."* The finalization timer processes one disbursement per invocation. A single orphaned entry causes it to retry indefinitely on `NeuronNotFound`, starving all legitimate pending maturity disbursements for all NNS neuron holders. The attacker also permanently loses their own maturity (self-inflicted), but the primary harm is the global DoS of the finalization timer for all users.

## Likelihood Explanation

The attack requires only: (1) owning a neuron with maturity ≥ `MINIMUM_DISBURSEMENT_E8S` (1 ICP), (2) calling `DisburseMaturity` (standard ingress), (3) dissolving the neuron and calling `Disburse` (standard ingress, requires dissolved state). No privileged access, key compromise, or governance majority is required. Any unprivileged neuron owner can trigger this. It is repeatable and cheap.

## Recommendation

1. **Fix `StableNeuronStore::delete`**: Add the missing cleanup in `rs/nns/governance/src/storage/neurons.rs` after line 428:
   ```rust
   update_repeated_field(neuron_id, vec![], &mut self.maturity_disbursements_map);
   ```
   Also remove the corresponding `MaturityDisbursementIndex` entry for the deleted neuron.

2. **Guard `disburse_neuron` against pending maturity disbursements**: In `rs/nns/governance/src/governance.rs`, before allowing stake disbursal, check `neuron.maturity_disbursements_in_progress.is_empty()` and return a `PreconditionFailed` error if not.

3. **Make the finalization timer resilient**: In `rs/nns/governance/src/governance/disburse_maturity.rs`, if `NeuronNotFound` is returned for an index entry, remove the stale entry from the index rather than returning `RETRY_INTERVAL` and retrying indefinitely.

## Proof of Concept

State-machine test sequence:
1. Create a neuron with maturity ≥ `MINIMUM_DISBURSEMENT_E8S` and a dissolve delay.
2. Call `DisburseMaturity` — verify `maturity_disbursements_in_progress` has one entry and `MaturityDisbursementIndex` has one entry.
3. Start dissolving; advance time past the dissolve delay.
4. Call `Disburse` (stake) — neuron is deleted.
5. Assert `maturity_disbursements_map` still contains the orphaned `(neuron_id, 0)` entry (bug confirmed at storage layer).
6. Advance time past `finalize_disbursement_timestamp_seconds`.
7. Run the finalization timer — assert it returns `RETRY_INTERVAL` (error path) and that no ICP was minted.
8. Run the timer again — assert it still returns `RETRY_INTERVAL` (infinite loop confirmed).
9. Assert `MaturityDisbursementIndex` still contains the orphaned entry (no self-healing).
