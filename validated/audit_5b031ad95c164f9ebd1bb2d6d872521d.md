### Title
Unchecked Precompile Call Return Value in `EvmErc20` Withdrawal Functions Causes Permanent Token Loss - (File: `etc/eth-contracts/contracts/EvmErc20.sol`, `etc/eth-contracts/contracts/EvmErc20V2.sol`)

### Summary

`EvmErc20.sol` and `EvmErc20V2.sol` implement custom withdrawal logic (`withdrawToNear`, `withdrawToEthereum`) that burns ERC-20 tokens via `_burn` **before** invoking the Aurora exit precompile via a low-level assembly `call`. The return value of that `call` is captured in a local variable `res` but is **never checked**. If the precompile call fails for any reason — including the precompile being paused, an out-of-gas condition, or a malformed input — the ERC-20 tokens are permanently destroyed with no corresponding release of NEP-141 tokens on NEAR.

This is the direct analog of the Holdefi M09 finding: instead of using a well-audited, battle-tested pattern (e.g., reverting on failed low-level calls, or using OpenZeppelin's `SafeERC20`-style checks), the project uses a custom implementation that silently swallows call failures.

### Finding Description

In `EvmErc20.sol`, both `withdrawToNear` and `withdrawToEthereum` follow this pattern:

```solidity
function withdrawToNear(bytes memory recipient, uint256 amount) external override {
    _burn(_msgSender(), amount);          // tokens destroyed here

    bytes32 amount_b = bytes32(amount);
    bytes memory input = abi.encodePacked("\x01", amount_b, recipient);
    uint input_size = 1 + 32 + recipient.length;

    assembly {
        let res := call(gas(), 0xe9217bc70b7ed1f598ddd3199e80b093fa71124f, 0, add(input, 32), input_size, 0, 32)
        // res is NEVER checked
    }
}

And identically in `EvmErc20V2.sol`: [3](#0-2) [4](#0-3) 

The precompile at `0xe9217bc70b7ed1f598ddd3199e80b093fa71124f` (`ExitToNear`) and `0xb0bd02f6a392af548bdf1cfaee5dfa0eefcc8eab` (`ExitToEthereum`) can fail and return `0` in several realistic scenarios:

1. **Precompile is paused**: The engine exposes `pause_precompiles` callable by an authorized account. When the `ExitToNear` or `ExitToEthereum` precompile is paused, the precompile returns `ExitError`, causing the low-level `call` to return `0`. [5](#0-4) 

2. **Out-of-gas**: The `call(gas(), ...)` forwards all remaining gas. If the precompile consumes more gas than available, the call fails silently.

3. **Input validation failure inside the precompile**: The `ExitToNear` precompile validates input size and format; any mismatch causes it to return an error. [6](#0-5) 

In all these cases, `_burn` has already executed and the ERC-20 tokens are gone. The contract does not revert, does not emit a failure event, and does not refund the user.

The root cause is the custom `AdminControlled`-based contract architecture (not using OpenZeppelin's `Ownable` or `SafeERC20` patterns) combined with the absence of a return-value check on the critical exit call. [7](#0-6) 

### Impact Explanation

**Critical — Permanent freezing/destruction of user funds.**

When the precompile call fails silently:
- The user's ERC-20 tokens (representing bridged NEP-141 assets) are burned and gone from the EVM state.
- No corresponding `ft_transfer` or `near_withdraw` is scheduled on NEAR.
- The NEP-141 tokens remain locked in the Aurora engine contract with no mechanism to recover them.
- The user has no recourse: the transaction succeeds from the EVM's perspective (no revert), so there is no automatic rollback.

This constitutes permanent loss of bridged user funds.

### Likelihood Explanation

**Medium.** The precompile pause mechanism is a legitimate operational feature used during upgrades or incident response. Any user who calls `withdrawToNear` or `withdrawToEthereum` during a pause window loses their tokens permanently. The window need not be long — a single block is sufficient. Additionally, out-of-gas scenarios are possible if the ERC-20 contract is called with a tight gas limit. The entry path requires no special privileges: any holder of a bridged ERC-20 token can trigger this.

### Recommendation

1. **Check the return value** of the low-level `call` and revert if it returns `0`:
   ```solidity
   assembly {
       let res := call(gas(), 0xe9217bc70b7ed1f598ddd3199e80b093fa71124f, 0, add(input, 32), input_size, 0, 32)
       if iszero(res) { revert(0, 0) }
   }
   ```
2. **Invert the operation order**: check that the precompile call will succeed (or use a try-pattern) before burning tokens, or use a two-step commit-reveal pattern.
3. **Adopt OpenZeppelin's `Ownable`** in place of the custom `AdminControlled.sol` to benefit from audited, maintained access-control logic, and follow established patterns for safe low-level calls.

### Proof of Concept

1. Deploy `EvmErc20` for a NEP-141 token on Aurora.
2. Mint tokens to `user`.
3. Owner calls `pause_precompiles` with the `ExitToNear` flag set.
4. `user` calls `withdrawToNear(recipient, amount)`.
5. `_burn(user, amount)` executes — user's ERC-20 balance drops to zero.
6. The assembly `call` to the paused precompile returns `0` (failure).
7. The function returns normally (no revert).
8. `user` has zero ERC-20 tokens and zero NEP-141 tokens. Funds are permanently lost.

#[named]
pub fn pause_precompiles<I: IO + Copy, E: Env>(io: I, env: &E) -> Result<(), ContractError> {
    with_hashchain(io, env, function_name!(), |io| {
        require_running(&state::get_state(&io)?)?;
        let authorizer: EngineAuthorizer = engine::get_authorizer(&io);

        if !authorizer.is_authorized(&env.predecessor_account_id()) {
            return Err(b"ERR_UNAUTHORIZED".into());
        }

        let args: PausePrecompilesCallArgs = io.read_input_borsh()?;
        let flags = PrecompileFlags::from_bits_truncate(args.paused_mask);
        let mut pauser = EnginePrecompilesPauser::from_io(io);
        pauser.pause_precompiles(flags);
        Ok(())
    })
}

pragma solidity ^0.8.0;


contract AdminControlled {
    // slither-disable-next-line immutable-states
    address public admin;
    uint public paused;

    constructor(address _admin, uint flags) {
        // slither-disable-next-line missing-zero-check
        admin = _admin;

        // Add the possibility to set pause flags on the initialization
        paused = flags;
    }

    modifier onlyAdmin {
        require(msg.sender == admin);
        _;
    }

    modifier pausable(uint flag) {
        require((paused & flag) == 0 || msg.sender == admin);
        _;
    }

    function adminPause(uint flags) public onlyAdmin {
        paused = flags;
    }

    function adminSstore(uint key, uint value) public onlyAdmin {
        assembly {
            sstore(key, value)
        }
    }

    function adminSendEth(address payable destination, uint amount) public onlyAdmin {
        // slither-disable-next-line missing-zero-check
        destination.transfer(amount);
    }

    function adminReceiveEth() public payable onlyAdmin {}

    function adminDelegatecall(address target, bytes memory data) public payable onlyAdmin returns (bytes memory) {
        // slither-disable-next-line controlled-delegatecall,low-level-calls,missing-zero-check
        (bool success, bytes memory rdata) = target.delegatecall(data);
        require(success);
        return rdata;
    }
}
