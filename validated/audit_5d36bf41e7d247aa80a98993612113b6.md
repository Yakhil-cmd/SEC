### Title
Stale 5% Worst-Case Guard in `spawn_neuron` Allows Sub-Minimum-Stake Neurons Under Mission 70's -10% Modulation — (`rs/nns/governance/src/governance.rs`)

---

### Summary

`spawn_neuron` gates the spawn on a hardcoded 5% worst-case modulation check, but Mission 70 extended the actual minimum modulation to -10%. A neuron owner whose maturity satisfies the 5% check but not the 10% check can successfully call `spawn_neuron`, wait 7 days, and have `maybe_spawn_neurons` mint a child neuron whose `cached_neuron_stake_e8s` is below `neuron_minimum_stake_e8s`.

---

### Finding Description

**The guard in `spawn_neuron`** (line 2666) computes:

```rust
let least_possible_stake = (maturity_to_spawn as f64 * (1_f64 - 0.05)) as u64;
if least_possible_stake < economics.neuron_minimum_stake_e8s { ... }
```

This hardcodes 5% as the worst-case modulation. [1](#0-0) 

**The old modulation range** was ±500 permyriad (±5%), defined in: [2](#0-1) 

**Mission 70 extended the minimum to -1000 permyriad (-10%)**:

```rust
pub(crate) const MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70: i64 = -1_000;
``` [3](#0-2) 

The `apply_maturity_modulation` function applies the modulation as `amount * (10_000 + basis_points) / 10_000`, so at -1000 permyriad it multiplies by 0.90: [4](#0-3) 

The `compute_maturity_modulation_permyriad` function clamps the result to `[MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70, MATURITY_MODULATION_MAX_PERMYRIAD_MISSION_70]`, meaning -10% is a reachable, protocol-enforced steady state: [5](#0-4) 

**The gap**: `spawn_neuron` checks `maturity * 0.95 >= min_stake`, but `maybe_spawn_neurons` can apply up to -10%, minting only `maturity * 0.90`. Any neuron with maturity `M` satisfying:

```
M * 0.95 >= min_stake   AND   M * 0.90 < min_stake
```

passes the spawn gate but produces a sub-minimum-stake child neuron.

---

### Impact Explanation

Every spawning neuron in this maturity band — when modulation is at -1000 permyriad — produces a child neuron with `cached_neuron_stake_e8s < neuron_minimum_stake_e8s`. This violates the minimum-stake invariant that governance enforces everywhere else. Sub-minimum-stake neurons are an anomalous state: they cannot be dissolved for the expected minimum ICP, and the aggregate minted ICP across all such spawns is less than the maturity burned, breaking ledger conservation expectations.

---

### Likelihood Explanation

- The attacker is an unprivileged neuron owner — no special role required.
- The entrypoint is the standard `manage_neuron` ingress call (`spawn_neuron`).
- The condition requires modulation to be at or near -1000 permyriad, which is a reachable protocol state (ICP price significantly below its 365-day average for an extended period).
- The maturity band is narrow but non-empty; with thousands of neurons, some will fall in it.
- The 7-day delay between spawn and mint means the modulation at mint time may differ from spawn time, but the worst case is fully reachable.

---

### Recommendation

Replace the hardcoded `0.05` in `spawn_neuron` with the actual Mission 70 minimum modulation constant:

```rust
// Instead of:
let least_possible_stake = (maturity_to_spawn as f64 * (1_f64 - 0.05)) as u64;

// Use:
use crate::timer_tasks::update_icp_xdr_rate_related_data::MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70;
let worst_case_factor = (10_000 + MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70) as u64;
let least_possible_stake = maturity_to_spawn
    .saturating_mul(worst_case_factor)
    / 10_000;
```

This eliminates the floating-point conversion and keeps the guard in sync with the actual modulation bounds.

---

### Proof of Concept

Let `min_stake = 100_000_000` (1 ICP in e8s). Choose `maturity = 106_000_000` (1.06 ICP).

- Spawn check: `106_000_000 * 0.95 = 100_700_000 >= 100_000_000` → **passes**
- Actual mint at -1000 permyriad: `apply_maturity_modulation(106_000_000, -1000)` = `106_000_000 * 9_000 / 10_000 = 95_400_000`
- `95_400_000 < 100_000_000` → **child neuron is below minimum stake**

A state-machine test setting modulation to -1000 permyriad and spawning a neuron with this maturity will confirm `child_neuron.cached_neuron_stake_e8s < neuron_minimum_stake_e8s` after `maybe_spawn_neurons` executes.

pub fn apply_maturity_modulation(
    amount_maturity_e8s: u64,
    maturity_modulation_basis_points: i32,
) -> Result<u64, String> {
    let amount_e8s = u128::from(amount_maturity_e8s);

    let adjusted_maturity_modulation_basis_points = saturating_add_or_subtract_u128_i32(
        BASIS_POINTS_PER_UNITY,
        maturity_modulation_basis_points,
    );

    let modulated_amount_e8s: u128 = amount_e8s
        .checked_mul(adjusted_maturity_modulation_basis_points)
        .ok_or_else(|| "Underflow or overflow when calculating maturity modulation".to_string())?
        .checked_div(BASIS_POINTS_PER_UNITY)
        .ok_or_else(|| "Underflow or overflow when calculating maturity modulation".to_string())?;

    u64::try_from(modulated_amount_e8s).map_err(|err| err.to_string())
```
