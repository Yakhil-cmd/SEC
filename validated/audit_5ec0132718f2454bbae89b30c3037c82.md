### Title
Any Unprivileged Caller Can Trigger Persistent State Changes for Arbitrary Accounts via `update_balance` — (`rs/bitcoin/ckbtc/minter/src/updates/update_balance.rs`)

---

### Summary

The ckBTC minter's `update_balance` endpoint accepts an optional `owner` parameter with no check that the caller equals the owner. Any unprivileged ingress sender can invoke `update_balance` targeting any victim account, acquiring the per-account lock, removing the victim's `finalized_utxos` protective state, and quarantining the victim's UTXOs under the victim's account key — state the victim cannot immediately undo. This is the IC analog of M-7: a third party can set persistent minter state for another user's account without that user's consent.

---

### Finding Description

**Root cause — missing caller/owner authorization check**

In `rs/bitcoin/ckbtc/minter/src/updates/update_balance.rs`, the `update_balance` function resolves the target account from the caller-supplied `args.owner` field with no check that `caller == owner`:

```rust
let caller_account = Account {
    owner: args.owner.unwrap_or(caller),   // ← any principal accepted
    subaccount: args.subaccount,
};
let _guard = balance_update_guard(caller_account)?;   // per-account lock
```

The only guard is that `owner` must not equal the minter's own principal ID (line 149). No check enforces `args.owner == caller`.

**State mutations performed on the victim's account**

1. **Per-account lock acquired** (line 168): `balance_update_guard(caller_account)` inserts `caller_account` into `update_balance_accounts`. While the attacker's async call is in flight (spanning multiple inter-canister calls to the Bitcoin network, the Bitcoin checker, and the ledger), any legitimate `update_balance` call from the real owner returns `AlreadyProcessing`.

2. **`finalized_utxos` removed** (line 213):
   ```rust
   state::mutate_state(|s| s.finalized_utxos.remove(&caller_account));
   ```
   `finalized_utxos` is a protective map populated when a transaction is finalized concurrently with an `update_balance` call; it prevents already-spent UTXOs from being re-processed. Removing it for the victim's account during a concurrent finalization can corrupt the victim's UTXO accounting.

3. **UTXOs quarantined under victim's account** (lines 311–316): If the Bitcoin checker returns `Failed` for any UTXO belonging to the victim's address, `quarantine_utxo` is called with `caller_account` (the victim's account):
   ```rust
   UtxoCheckStatus::Tainted => {
       mutate_state(|s| {
           state::audit::quarantine_utxo(s, utxo.clone(), caller_account, now, runtime)
       });
   ```
   This inserts the UTXO into `suspended_utxos` keyed by the victim's account with `SuspendedReason::Quarantined`. The UTXO cannot be re-evaluated for at least one full day (the `DAY` cooldown in `processable_utxos_for_account`).

**Exploit flow**

1. Attacker observes that victim `V` has a UTXO at their ckBTC deposit address.
2. Attacker calls `update_balance({ owner: Some(V), subaccount: ... })` from any principal.
3. Minter acquires the per-account lock for `V`'s account; `V`'s own concurrent calls return `AlreadyProcessing`.
4. Minter calls the Bitcoin checker for `V`'s UTXO. If the checker returns `Failed` (tainted), the UTXO is quarantined under `V`'s account for ≥1 day; the check fee is deducted from the UTXO value.
5. `finalized_utxos.remove(&V_account)` is executed unconditionally, removing any concurrent-finalization protection for `V`.
6. Attacker can repeat immediately after the call completes, sustaining the DoS.

---

### Impact Explanation

- **Temporary but repeatable DoS**: The per-account lock blocks the victim's own `update_balance` calls for the duration of each attacker call. The attacker can chain calls to sustain the block.
- **Forced UTXO quarantine**: The attacker can cause the victim's UTXOs to be quarantined for ≥1 day, delaying ckBTC minting. The check fee (deducted from UTXO value) is consumed on each check, causing direct financial loss if UTXOs are tainted.
- **`finalized_utxos` corruption**: Removing the victim's `finalized_utxos` entry during a concurrent transaction finalization can cause already-spent UTXOs to be re-evaluated, potentially triggering duplicate-outpoint detection and logging a critical-bug metric.

---

### Likelihood Explanation

The attack requires no privileged access. Any ingress sender with a valid identity can call `update_balance` with an arbitrary `owner` principal. The ckBTC minter is a public, production canister. The attack is cheap (only cycles for the call) and requires no knowledge beyond the victim's principal ID, which is public.

---

### Recommendation

Add an authorization check that the caller must equal the resolved owner, or explicitly document and enforce that only the owner (or a whitelisted proxy) may trigger balance updates for a given account:

```rust
let resolved_owner = args.owner.unwrap_or(caller);
if resolved_owner != caller {
    // Only allow if caller is an explicitly whitelisted proxy
    return Err(UpdateBalanceError::TemporarilyUnavailable(
        "Caller is not authorized to update balance for another account".to_string()
    ));
}
let caller_account = Account { owner: resolved_owner, subaccount: args.subaccount };
```

### Proof of Concept

```
Attacker principal: A (any non-anonymous principal)
Victim principal:   V (any ckBTC user with a pending UTXO)

1. A calls update_balance({ owner: Some(V), subaccount: None })
   → minter acquires lock for Account{V, None}
   → V's own update_balance call returns AlreadyProcessing

2. Bitcoin checker returns Failed for V's UTXO
   → quarantine_utxo(state, utxo, Account{V,None}, now, runtime) is called
   → V's UTXO enters suspended_utxos[Account{V,None}] = Quarantined
   → V cannot mint ckBTC for ≥1 day

3. finalized_utxos.remove(&Account{V,None}) executes unconditionally
   → protective state for V's concurrent finalization is erased

4. A repeats step 1 immediately after completion → sustained DoS
```

```rust
    ///
    /// The returned UTXOs include:
    /// * new UTXOs that are not known to the minter,
    /// * UTXOs that were previously ignored and that can be re-evaluated,
    /// * UTXOs that were previously quarantined and that can be re-evaluated.
    pub fn processable_utxos_for_account<I: IntoIterator<Item = Utxo>>(
        &self,
        all_utxos_for_account: I,
        account: &Account,
        now: &Timestamp,
    ) -> (ProcessableUtxos, Vec<SuspendedUtxo>) {
        const DAY: std::time::Duration = std::time::Duration::from_secs(24 * 60 * 60);

        let is_known = |utxo: &Utxo| {
            self.utxos_state_addresses
                .get(account)
                .map(|utxos| utxos.contains(utxo))
                .unwrap_or(false)
                || self
                    .finalized_utxos
                    .get(account)
                    .map(|utxos| utxos.contains(utxo))
                    .unwrap_or(false)
        };
        let mut processable_utxos = ProcessableUtxos::default();
        let mut suspended_utxos = vec![];

        for utxo in all_utxos_for_account.into_iter() {
            match self.suspended_utxos.contains_utxo(&utxo, account) {
                (Some(last_time_checked), Some(reason)) => {
                    match now.checked_duration_since(*last_time_checked) {
                        Some(elapsed) if elapsed >= DAY => {
                            processable_utxos.insert_once_suspended_utxo(utxo, reason);
                        }
                        _ => suspended_utxos.push(SuspendedUtxo {
                            utxo,
                            reason: *reason,
                            earliest_retry: last_time_checked
                                .saturating_add(DAY)
                                .as_nanos_since_unix_epoch(),
                        }),
                    }
                }
                (None, Some(reason)) => {
                    processable_utxos.insert_once_suspended_utxo(utxo, reason);
                }
                (_, None) => {
                    if !is_known(&utxo) {
                        processable_utxos.insert_once_new_utxo(utxo);
                    }
                }
            }
        }

        (processable_utxos, suspended_utxos)
    }
```
