### Title
`key_manager` Can Unpause Engine Without Owner Authorization via `start_hashchain` — (`engine/src/contract_methods/admin.rs`)

### Summary
`start_hashchain` unconditionally sets `state.is_paused = false` after passing only `require_paused` and `require_key_manager_only`. This allows the `key_manager` account to resume the engine without owner approval, violating the invariant enforced by `resume_contract` that only the owner can unpause.

### Finding Description
`pause_contract` and `resume_contract` both enforce `require_owner_only`. [5](#0-4) 

`start_hashchain` bypasses this invariant: it gates on `require_key_manager_only` (a separate, non-owner role) and then writes `state.is_paused = false` unconditionally at line 460. [6](#0-5) 

The `key_manager` is a distinct account from the owner — it is set by the owner while the engine is running, and its intended purpose is relayer key management (`add_relayer_key`, `remove_relayer_key`), not contract lifecycle control. [7](#0-6) 

### Impact Explanation
If the owner pauses the contract (e.g., in response to an active exploit or bridge incident to freeze fund movements), the `key_manager` can immediately call `start_hashchain` with valid `StartHashchainArgs` to resume the engine. This lifts the owner's intended freeze without owner approval, allowing EVM transactions, deposits, withdrawals, and bridge operations to proceed — exactly the fund movements the owner intended to block.

**Impact: High — Temporary freezing of funds** (the freeze is lifted by an unauthorized party).

### Likelihood Explanation
Preconditions are realistic in production:
1. A `key_manager` is set (standard operational setup for relayer infrastructure).
2. The owner pauses the contract (incident response).
3. The `key_manager` calls `start_hashchain` — a publicly callable NEAR contract method — with any valid `StartHashchainArgs` (block height ≤ current block, arbitrary `block_hashchain` bytes).

No key compromise or social engineering is required. The `key_manager` uses its legitimately granted access to call a function that has an unintended side effect.

### Recommendation
Remove the `state.is_paused = false` assignment from `start_hashchain`, or replace it with `require_owner_only` if unpausing is intentional. The hashchain initialization does not logically require unpausing the engine — these are orthogonal concerns. If the design intent is that starting the hashchain should resume the engine, the authorization check must be `require_owner_only`, consistent with `resume_contract`.

### Proof of Concept
```
1. Deploy engine; owner = "aurora"
2. owner calls set_key_manager(key_manager = "relayer.near")
3. owner calls pause_contract()          → state.is_paused = true
4. "relayer.near" calls start_hashchain(
       block_height = current_block - 1,
       block_hashchain = [0u8; 32]
   )
   → require_paused passes (is_paused == true)
   → require_key_manager_only passes ("relayer.near" == state.key_manager)
   → state.is_paused = false is written
5. Assert state.is_paused == false       → engine is running again
6. Any user can now call submit/call/deposit/withdraw
``` [8](#0-7)

#[named]
pub fn start_hashchain<I: IO + Copy, E: Env>(mut io: I, env: &E) -> Result<(), ContractError> {
    let mut state = state::get_state(&io)?;
    require_paused(&state)?;
    require_key_manager_only(&state, &env.predecessor_account_id())?;

    let input = io.read_input().to_vec();
    let args = StartHashchainArgs::try_from_slice(&input).map_err(|_| errors::ERR_SERIALIZE)?;
    let block_height = env.block_height();

    // Starting hashchain must be for an earlier block
    if block_height < args.block_height {
        return Err(errors::ERR_ARGS.into());
    }

    let mut hashchain = Hashchain::new(
        state.chain_id,
        env.current_account_id(),
        args.block_height + 1,
        args.block_hashchain,
    );

    if hashchain.get_current_block_height() < block_height {
        hashchain.move_to_block(block_height)?;
    }

    hashchain.add_block_tx(
        block_height,
        function_name!(),
        &input,
        &[],
        &Bloom::default(),
    )?;
    crate::hashchain::save_hashchain(&mut io, &hashchain)?;

    state.is_paused = false;
    state::set_state(&mut io, &state)?;

    Ok(())

fn require_key_manager_only(
    state: &state::EngineState,
    predecessor_account_id: &AccountId,
) -> Result<(), ContractError> {
    let key_manager = state
        .key_manager
        .as_ref()
        .ok_or(errors::ERR_KEY_MANAGER_IS_NOT_SET)?;
    if key_manager != predecessor_account_id {
        return Err(errors::ERR_NOT_ALLOWED.into());
    }
    Ok(())
}
