### Title
EVM Refund Applied Before `delta_gas` Baseline Causes User Overcharge in `compute_gas_refund` - (File: `basic_bootloader/src/bootloader/transaction_flow/refund_calculation.rs`)

### Summary
In `compute_gas_refund`, the EVM storage refund (`evm_refund`) is subtracted from `gas_used` **before** it is used as the baseline for the `delta_gas` calculation. This causes `delta_gas` to be computed against an artificially lowered denominator, making it larger than intended and partially or fully negating the EVM refund. Users who both clear storage slots and have significant native resource consumption are overcharged gas.

### Finding Description
The `compute_gas_refund` function in `basic_bootloader/src/bootloader/transaction_flow/refund_calculation.rs` implements ZKsync OS's double-resource accounting reconciliation. The intended logic (documented in `docs/double_resource_accounting.md`) is:

```
deltaGas := (nativeUsed / nativePerGas) - gasUsed
```

where `gasUsed` is the raw EVM gas consumed by execution. If `deltaGas > 0`, extra gas is charged to ensure native resource costs are covered.

The code executes in this order:

```rust
// Step 1: compute raw gas_used from ergs
let mut gas_used = gas_limit
    .checked_sub(resources.ergs().0.div_floor(ERGS_PER_GAS))...;

// Step 2: compute and APPLY evm_refund — gas_used is now REDUCED
let evm_refund = { ... core::cmp::min(full_refund_gas, max_refund) };
gas_used -= evm_refund;                          // ← gas_used mutated here

let mut gas_used = core::cmp::max(gas_used, minimal_gas_used);

// Step 3: compute native_used
let native_used = full_native_limit.saturating_sub(resources.native().remaining().as_u64());

// Step 4: delta_gas uses the ALREADY-REDUCED gas_used as baseline
let delta_gas = (native_used / native_per_gas) as i64 - (gas_used as i64);
if delta_gas > 0 {
    gas_used += delta_gas as u64;   // ← overcharges because baseline was too low
}
// TODO: return delta_gas to gas_used?   ← developer uncertainty noted
``` [1](#0-0) 

Because `gas_used` is decremented by `evm_refund` before the `delta_gas` baseline is read, the threshold for triggering a positive `delta_gas` is lower than intended. When native consumption falls in the range `(gas_used_after_refund * native_per_gas, gas_used_before_refund * native_per_gas)`, `delta_gas` becomes positive and extra gas is charged — even though the raw gas consumption already covered the native cost. The EVM refund is partially or fully cancelled.

The documentation explicitly states the baseline should be the raw gas used: [2](#0-1) 

The `// TODO: return delta_gas to gas_used?` comment at line 80 further indicates developer uncertainty about this calculation.

### Impact Explanation
Any user whose transaction both:
1. Clears storage slots (generating an EVM SSTORE refund, capped at 1/5 of gas used per EIP-3529), and
2. Has native resource consumption that falls in the "gap" created by the premature refund subtraction

…will receive a smaller gas refund than they are entitled to. The shortfall is `min(evm_refund, delta_gas_excess) * gas_price` tokens, paid to the operator instead of returned to the sender. This is a direct, per-transaction financial loss for users.

Concrete example with `gas_limit = 100,000`, `native_per_gas = 100`, `native_remaining = 1,500,000`:
- Raw `gas_used = 100,000`; `evm_refund = 20,000`; post-refund `gas_used = 80,000`
- `native_used / native_per_gas = 85,000`
- Current code: `delta_gas = 85,000 − 80,000 = +5,000` → final `gas_used = 85,000`, refund = 15,000
- Correct code: `delta_gas = 85,000 − 100,000 = −15,000` → no extra charge, final `gas_used = 80,000`, refund = 20,000
- User loses `5,000 × gas_price` tokens per transaction.

### Likelihood Explanation
The condition is reachable by any unprivileged L2 transaction sender. DeFi operations that clear storage (removing liquidity, closing positions, deleting mappings) routinely generate EVM refunds. The `delta_gas` path is active in production (it is gated by `#[cfg(not(feature = "unlimited_native"))]`, and `unlimited_native` is a test-only feature). [3](#0-2) 

### Recommendation
Move the `evm_refund` subtraction to **after** the `delta_gas` calculation, so the baseline reflects raw EVM gas consumption:

```rust
// 1. Compute raw gas_used
let mut gas_used = gas_limit - remaining_ergs / ERGS_PER_GAS;

// 2. Compute native_used and delta_gas against raw gas_used
let native_used = full_native_limit.saturating_sub(resources.native().remaining().as_u64());
let delta_gas = (native_used / native_per_gas) as i64 - (gas_used as i64);
if delta_gas > 0 {
    gas_used += delta_gas as u64;
}

// 3. THEN apply EVM refund
gas_used -= evm_refund;
gas_used = core::cmp::max(gas_used, minimal_gas_used);
```

This matches the documented formula `deltaGas := (nativeUsed / nativePerGas) − gasUsed` where `gasUsed` is the raw pre-refund value.

### Proof of Concept
1. Deploy a contract that writes to N storage slots (e.g., N = 5 SSTORE writes to non-zero values).
2. Send a transaction that clears all N slots (SSTORE to zero), generating `evm_refund = min(N × 4800, gas_used / 5)`.
3. Ensure the transaction has enough native consumption that `native_used / native_per_gas` falls between `gas_used_after_refund` and `gas_used_before_refund`.
4. Observe that the actual gas refund received is less than `(gas_limit − raw_gas_used) × gas_price`, with the shortfall equal to `delta_gas × gas_price` tokens transferred to the operator instead of the sender.

The entry path is: unprivileged EOA → `validate_and_compute_fee_for_transaction` → `create_frame_and_execute_transaction_payload` → `before_refund` → `compute_gas_refund`. [4](#0-3) [5](#0-4)

pub(crate) fn compute_gas_refund<S: EthereumLikeTypes>(
    system: &mut System<S>,
    to_charge_for_pubdata: S::Resources,
    gas_limit: u64,
    minimal_gas_used: u64,
    native_per_gas: u64,
    resources: &mut S::Resources,
) -> Result<RefundInfo, InternalError> {
    // Already checked
    resources.charge_unchecked(&to_charge_for_pubdata);

    let mut gas_used = gas_limit
        .checked_sub(resources.ergs().0.div_floor(ERGS_PER_GAS))
        .ok_or(internal_error!("gas remaining > gas limit"))?;
    resources.exhaust_ergs();

    system_log!(system, "Gas used before refund calculations: {gas_used}\n");

    // Following EIP-3529, refunds are capped to 1/5 of the gas used
    let evm_refund = {
        let full_refund_ergs = system.io.get_refund_counter().ergs();
        let full_refund_gas = full_refund_ergs.0.div_floor(ERGS_PER_GAS);
        let max_refund = gas_used / 5;
        core::cmp::min(full_refund_gas, max_refund)
    };

    system_log!(system, "Gas refund from refund counters = {evm_refund}\n");

    gas_used -= evm_refund;

    system_log!(
        system,
        "Minimal gas used from validation = {minimal_gas_used}\n"
    );

    #[allow(unused_mut)]
    let mut gas_used = core::cmp::max(gas_used, minimal_gas_used);

    // Note: for zero gas price, we use "unlimited native"
    let full_native_limit = if cfg!(feature = "unlimited_native") || native_per_gas == 0 {
        u64::MAX - 1
    } else {
        gas_limit.saturating_mul(native_per_gas)
    };
    let native_used = full_native_limit.saturating_sub(resources.native().remaining().as_u64());

    #[cfg(not(feature = "unlimited_native"))]
    {
        // Adjust gas_used with difference with used native
        let delta_gas = if native_per_gas == 0 {
            0
        } else {
            (native_used / native_per_gas) as i64 - (gas_used as i64)
        };

        if delta_gas > 0 {
            // In this case, the native resource consumption is more than the
            // gas consumption accounted for. Consume extra gas.
            gas_used += delta_gas as u64;
        }
        // TODO: return delta_gas to gas_used?
    }

    let total_gas_refund = gas_limit - gas_used;
    system_log!(system, "Refund after accounting for unused gas, refund counters and native cost: {total_gas_refund}\n");
    require_internal!(
        total_gas_refund <= gas_limit,
        "Gas refund greater than gas limit",
        system
    )?;
    let refund_info = RefundInfo {
        gas_used,
        evm_refund,
        native_used,
    };
    system_log!(system, "Final gas used: {gas_used}\n");
    Ok(refund_info)
}

    fn before_refund<'a, Config: BasicBootloaderExecutionConfig>(
        system: &mut System<S>,
        transaction: &Transaction<<S as SystemTypes>::Allocator>,
        context: &mut Self::TransactionContext,
        _result: &ExecutionResult<'a, <S as SystemTypes>::IOTypes>,
        pubdata_info: Self::ExecutionBodyExtraData,
        _tracer: &mut impl Tracer<S>,
    ) -> Result<(), BootloaderSubsystemError> {
        use evm_interpreter::ERGS_PER_GAS;

        // Just used for computing native used
        context.resources_before_refund = context.resources.main_resources.clone();

        // Now we can actually reclaim resources withheld for pubdata
        context
            .resources
            .main_resources
            .reclaim_withheld(context.resources.withheld.take());

        system_log!(
            system,
            "Have {:?} resources available before refund, and need to cover {:?} pubdata\n",
            &context.resources.main_resources,
            &pubdata_info
        );

        let intrinsic_pubdata = calculate_l2_tx_intrinsic_pubdata(
            context.authorization_list_num,
            transaction.is_service(),
        );

        // Pubdata for validation has been charged already,
        // we charge for the rest now.
        let (total_pubdata_used, to_charge_for_pubdata) = match pubdata_info {
            Some(CachedPubdataInfo {
                pubdata_used,
                to_charge_for_pubdata,
            }) => (pubdata_used + intrinsic_pubdata, to_charge_for_pubdata),
            None => {
                let (execution_pubdata_spent, to_charge_for_pubdata) =
                    get_resources_to_charge_for_pubdata(
                        system,
                        context.native_per_pubdata,
                        Some(context.validation_pubdata),
                    )?;
                (
                    execution_pubdata_spent + intrinsic_pubdata,
                    to_charge_for_pubdata,
                )
            }
        };
        let min_gas_used = context.minimal_ergs_to_charge.0 / ERGS_PER_GAS;
        let refund_info = compute_gas_refund(
            system,
            to_charge_for_pubdata,
            transaction.gas_limit(),
            min_gas_used,
            context.native_per_gas,
            &mut context.resources.main_resources,
        )?;
        debug_assert_eq!(context.gas_used, 0);
        context.gas_used = refund_info.gas_used;
        context.gas_refunded = refund_info.evm_refund;
        context.total_pubdata = total_pubdata_used;
        context.native_used = refund_info.native_used;

        Ok(())
    }
