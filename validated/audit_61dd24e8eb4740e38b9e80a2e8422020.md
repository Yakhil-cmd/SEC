### Title
SNS Governance Computes Proposal Ballots from Live Neuron State Without Historical Snapshot Protection, Enabling Voting Power Inflation - (File: `rs/sns/governance/src/governance.rs`)

### Summary
The SNS governance canister computes voting power for proposal ballots entirely from the current live neuron state at proposal-creation time, with no historical snapshot or spike-detection mechanism. An unprivileged token holder can stake tokens immediately before submitting a proposal to inflate their ballot voting power, then vote to pass a malicious proposal.

### Finding Description
NNS governance implements `VotingPowerSnapshots` — a daily-snapshot store that detects when current total voting power exceeds 1.5× the historical minimum and falls back to a prior snapshot for ballot creation. This prevents a sudden stake increase from granting outsized influence on any single proposal.

SNS governance has no equivalent mechanism. `compute_ballots_for_new_proposal` in `rs/sns/governance/src/governance.rs` iterates directly over `self.proto.neurons` and calls `v.voting_power(now_seconds, ...)` for each neuron, writing the result straight into the ballot map: [1](#0-0) 

The voting power used is the neuron's **current** `cached_neuron_stake_e8s` at the moment `make_proposal` is called: [2](#0-1) 

By contrast, NNS governance's `compute_ballots_for_standard_proposal` checks `VOTING_POWER_SNAPSHOTS` and substitutes a historical snapshot when a spike is detected: [3](#0-2) 

The NNS snapshot task runs daily and explicitly guards against recording a new snapshot when the latest is already a spike: [4](#0-3) 

SNS has no analogous task or guard anywhere under `rs/sns/`:



### Impact Explanation
An attacker who controls a large block of SNS tokens can:
1. Stake them into a neuron with sufficient dissolve delay just before calling `make_proposal`.
2. `compute_ballots_for_new_proposal` captures the inflated `voting_power` into the ballot at proposal creation.
3. The proposer's neuron automatically votes Yes at proposal creation time.
4. If the inflated stake represents a majority (or enough to meet `minimum_yes_proportion_of_total`), the proposal passes immediately or with minimal additional votes.
5. A malicious proposal can drain the SNS treasury (`TransferSnsTreasuryFunds`), upgrade a canister with malicious WASM (`UpgradeSnsControlledCanister`), or alter governance parameters.

The ballot voting power is frozen at creation time, so the attacker does not need to maintain the stake after the proposal is submitted.

### Likelihood Explanation
Any SNS token holder with sufficient liquid tokens can execute this without any privileged role, admin key, or social engineering. The only barrier is acquiring enough tokens to reach the `minimum_yes_proportion_of_total` threshold (default 3% of total for normal proposals, higher for critical ones). For SNS DAOs with low total staked supply or concentrated token distribution this threshold is easily reachable.

### Recommendation
Port the NNS `VotingPowerSnapshots` spike-detection mechanism to SNS governance, or implement an equivalent guard:
- Add a periodic timer task that records daily snapshots of total voting power per neuron.
- In `compute_ballots_for_new_proposal`, compare the current total voting power against the historical minimum; if it exceeds the spike threshold (e.g., 1.5×), use the snapshot with the minimum total instead of the live state.
- Until the snapshot store is populated (first N days of an SNS), apply a conservative fallback (e.g., reject proposals whose total voting power exceeds a configurable ceiling).

### Proof of Concept

**Attacker-controlled entry path**: unprivileged SNS governance user → `manage_neuron` (stake + claim) → `make_proposal` → `compute_ballots_for_new_proposal`.

**Step-by-step**:

1. Attacker holds a large amount of SNS tokens on the ledger.
2. Attacker calls `manage_neuron::ClaimOrRefresh` to stake them into a neuron with dissolve delay ≥ `neuron_minimum_dissolve_delay_to_vote_seconds`.
3. Attacker immediately calls `make_proposal` with a `TransferSnsTreasuryFunds` action targeting their own account.
4. Inside `make_proposal` → `compute_ballots_for_new_proposal` (line 5263):
   ```rust

   electoral_roll.insert(k.clone(), Ballot { vote: Vote::Unspecified, voting_power, ... });
   ```
   The attacker's neuron receives a ballot with the full inflated voting power.
5. The proposer's neuron auto-votes Yes (standard SNS behavior).
6. If the attacker's stake ≥ `minimum_yes_proportion_of_total` × total staked supply, the proposal is decided immediately in favor.
7. The treasury transfer executes, draining SNS funds to the attacker.

    /// Computes the total potential voting power of the governance canister and ballots.
    fn compute_ballots_for_new_proposal(&self) -> Result<(u64, BTreeMap<String, Ballot>), String> {
        let now_seconds = self.env.now();

        let nervous_system_parameters = self.nervous_system_parameters_or_panic();

        // Voting power bonus parameters.
        let max_dissolve_delay = nervous_system_parameters
            .max_dissolve_delay_seconds
            .expect("NervousSystemParameters must have max_dissolve_delay_seconds");

        let max_age_bonus = nervous_system_parameters
            .max_neuron_age_for_age_bonus
            .expect("NervousSystemParameters must have max_neuron_age_for_age_bonus");

        let max_dissolve_delay_bonus_percentage = nervous_system_parameters
            .max_dissolve_delay_bonus_percentage
            .expect("NervousSystemParameters must have max_dissolve_delay_bonus_percentage");

        let max_age_bonus_percentage = nervous_system_parameters
            .max_age_bonus_percentage
            .expect("NervousSystemParameters must have max_age_bonus_percentage");

        let min_dissolve_delay_for_vote = nervous_system_parameters
            .neuron_minimum_dissolve_delay_to_vote_seconds
            .expect("NervousSystemParameters must have min_dissolve_delay_for_vote");

        let mut electoral_roll = BTreeMap::<String, Ballot>::new();
        let mut total_power: u128 = 0;

        for (k, v) in self.proto.neurons.iter() {
            // If this neuron is eligible to vote, record its
            // voting power at the time of proposal creation (now).
            if v.dissolve_delay_seconds(now_seconds) < min_dissolve_delay_for_vote {
                // Not eligible due to dissolve delay.
                continue;
            }

            let voting_power = v.voting_power(
                now_seconds,
                max_dissolve_delay,
                max_age_bonus,
                max_dissolve_delay_bonus_percentage,
                max_age_bonus_percentage,
            );

            total_power += voting_power as u128;
            electoral_roll.insert(
                k.clone(),
                Ballot {
                    vote: Vote::Unspecified as i32,
                    voting_power,
                    cast_timestamp_seconds: 0,
                },
            );
        }

        if total_power >= (u64::MAX as u128) {
            // The way the neurons are configured, the total voting
            // power on this proposal would overflow a u64!
            return Err("Voting power overflow.".to_string());
        }
        if electoral_roll.is_empty() {
            // Cannot make a proposal with no eligible voters.  This
            // is a precaution that shouldn't happen as we check that
            // the voter is allowed to vote.
            return Err("No eligible voters.".to_string());
        }

        Ok((total_power as u64, electoral_roll))
    }

/// A task to snapshot the voting power every day, so that the snapshot can be used to disable
/// early adoption of proposals if such proposals have unusually high voting power.
pub(super) struct SnapshotVotingPowerTask {
    governance: &'static LocalKey<RefCell<Governance>>,
    snapshots: &'static LocalKey<RefCell<VotingPowerSnapshots>>,
}

const VOTING_POWER_SNAPSHOT_INTERVAL: Duration = Duration::from_secs(60 * 60 * 24);

impl SnapshotVotingPowerTask {
    pub fn new(
        governance: &'static LocalKey<RefCell<Governance>>,
        snapshots: &'static LocalKey<RefCell<VotingPowerSnapshots>>,
    ) -> Self {
        Self {
            governance,
            snapshots,
        }
    }
}

impl RecurringSyncTask for SnapshotVotingPowerTask {
    fn execute(self) -> (Duration, Self) {
        let now_seconds = self
            .governance
            .with_borrow(|governance| governance.env.now());
        if self
            .snapshots
            .with_borrow(|snapshots| snapshots.is_latest_snapshot_a_spike(now_seconds))
        {
            return (VOTING_POWER_SNAPSHOT_INTERVAL, self);
        }

        let voting_power_snapshot = self.governance.with_borrow_mut(|governance| {
            let voting_power_economics = governance.voting_power_economics();
            governance
                .neuron_store
                .compute_voting_power_snapshot_for_standard_proposal(
                    voting_power_economics,
                    now_seconds,
                )
                .expect("Voting power snapshot failed")
        });

        self.snapshots.with_borrow_mut(|snapshots| {
            snapshots.record_voting_power_snapshot(now_seconds, voting_power_snapshot);
        });

        (VOTING_POWER_SNAPSHOT_INTERVAL, self)
