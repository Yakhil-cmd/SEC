### Title
Missing Privileged Controller-Restoration Escape Hatch in Migration Canister — (`rs/migration_canister/src/privileged.rs`)

### Summary
The migration canister sets itself (`canister_self()`) as the exclusive controller of both the migrated and replaced user canisters during the `process_accepted` step. While an automatic timer-driven recovery mechanism exists, the privileged interface exposes only `enable_api` and `disable_api`. There is no privileged `force_restore_controllers` method callable by the canister's controllers or the NNS governance canister. If the automatic recovery fails permanently (e.g., due to a logic bug introduced by an upgrade, or a persistent xnet failure), the user's canisters remain permanently locked under the migration canister's exclusive control with no runtime escape hatch.

### Finding Description

In `rs/migration_canister/src/processing.rs`, `process_accepted` calls `set_exclusive_controller` for both the migrated and replaced canisters:

```rust
// Set controller of migrated canister
let res = set_exclusive_controller(request.migrated_canister).await ...
// Set controller of replaced canister
set_exclusive_controller(request.replaced_canister).await ...
``` [1](#0-0) 

`set_exclusive_controller` passes `canister_self()` as the sole controller:

```rust
settings: CanisterSettings {
    controllers: Some(vec![canister_self()]),
},
``` [2](#0-1) 

The automatic recovery in `controller_recovery.rs` is the only mechanism to restore original controllers on failure. It runs via timers and retries indefinitely, but there is no manually-triggerable privileged override.

The entire privileged interface is:

```rust
#[update]
fn enable_api() -> Result<(), Option<MigrationCanisterError>> { ... }

#[update]
fn disable_api() -> Result<(), Option<MigrationCanisterError>> { ... }
``` [3](#0-2) 

The public Candid interface confirms no `force_restore_controllers` or equivalent method exists:

```
service : (MigrationCanisterInitArgs) -> {
  disable_api : () -> (MigrationCanisterResult);
  enable_api  : () -> (MigrationCanisterResult);
  migrate_canister : (MigrateCanisterArgs) -> (ValidationResult);
  migration_status : (MigrateCanisterArgs) -> (opt MigrationStatus) query;
}
``` [4](#0-3) 

Additionally, the `ALLOWLIST` is stored in non-stable heap memory (`thread_local! { static ALLOWLIST: RefCell<Option<Vec<Principal>>> }`), meaning it is reset on every upgrade to whatever is passed in `post_upgrade` args. If upgrade args omit the allowlist, it defaults to `None` (all callers permitted), silently widening access. [5](#0-4) [6](#0-5) 

### Impact Explanation

If the automatic controller recovery fails permanently — due to a logic bug introduced in an upgrade, a persistent xnet failure, or a corrupted `RequestState` in stable memory — the user's canisters remain permanently locked under the migration canister's exclusive control. The original controllers (the user's principals) are no longer listed as controllers and cannot call `update_settings` to reclaim control. The NNS governance canister and the migration canister's own controllers have no runtime method to force-restore the original controllers without first deploying a new version of the migration canister that adds such a method — a process that itself requires a governance proposal and introduces additional risk.

### Likelihood Explanation

The entry path is reachable by any unprivileged ingress sender who is a controller of both the migrated and replaced canisters. The automatic recovery is designed to be robust, but the absence of a manual override means any persistent failure in the recovery path (e.g., a regression introduced by a canister upgrade, a stable-memory deserialization failure of `RequestState`, or a permanent xnet routing issue) results in permanent user canister lockout. The likelihood of the recovery failing permanently is low but non-zero, and the impact when it occurs is severe.

### Recommendation

Add a privileged `force_restore_controllers(migrated_canister: Principal, replaced_canister: Principal, original_controllers: Vec<Principal>)` method to `rs/migration_canister/src/privileged.rs`, callable only by the canister's controllers or the NNS governance canister. This method should directly call `set_controllers` to restore the specified controllers, bypassing the state machine. This is the IC analog of adding a `setOwner` function to `FathomProxyWalletOwner`.

Additionally, move `ALLOWLIST` to stable memory (using `ic_stable_structures`) so that it survives upgrades without requiring explicit re-specification in upgrade args, preventing accidental allowlist resets.

### Proof of Concept

1. User (controller of both canisters) calls `migrate_canister`.
2. `process_accepted` calls `set_exclusive_controller` for both canisters — user is no longer a controller.
3. A bug in the recovery logic (e.g., introduced by a canister upgrade that changes `RequestState` serialization) causes `process_all_failed` to panic or skip the recovery step permanently.
4. The user's canisters are now exclusively controlled by the migration canister.
5. The user calls `update_settings` on the management canister to reclaim control — this is rejected because the user is no longer a controller.
6. The NNS governance canister calls `enable_api` / `disable_api` — neither restores the user's controllers.
7. No runtime escape hatch exists; the user's canisters are permanently locked until a new migration canister version is deployed via NNS governance proposal. [7](#0-6) [2](#0-1) [8](#0-7)

/// Accepts an `Accepted` request, returns `ControllersChanged` on success.
/// This function is an exception in that it tries to make _two_ effectful calls.
pub async fn process_accepted(
    request: RequestState,
) -> ProcessingResult<RequestState, RequestState> {
    let RequestState::Accepted { request } = request else {
        println!("Error: list_by Accepted returned bad variant");
        return ProcessingResult::NoProgress;
    };

    // Set controller of migrated canister
    let res = set_exclusive_controller(request.migrated_canister)
        .await
        .map_success(|_| RequestState::ControllersChanged {
            request: request.clone(),
        })
        .map_failure(|reason| RequestState::Failed {
            request: request.clone(),
            recovery_state: RecoveryState::new(),
            reason,
        });
    if !res.is_success() {
        return res;
    }

    // Set controller of replaced canister
    set_exclusive_controller(request.replaced_canister)
        .await
        .map_success(|_| RequestState::ControllersChanged {
            request: request.clone(),
        })
        .map_failure(|reason| RequestState::Failed {
            request,
            recovery_state: RecoveryState::new(),
            reason,
        })
}

pub async fn set_exclusive_controller(canister_id: Principal) -> ProcessingResult<(), String> {
    let args = UpdateSettingsArgs {
        canister_id,
        settings: CanisterSettings {
            controllers: Some(vec![canister_self()]),
        },
    };
    match Call::bounded_wait(Principal::management_canister(), "update_settings")
        .with_arg(args)
        .await
    {
        Ok(_) => ProcessingResult::Success(()),
        Err(e) => {
            println!("Call `update_settings` for {} failed: {:?}", canister_id, e);
            ProcessingResult::FatalFailure(format!(
                "Failed to set the migration canister as the exclusive controller of canister {canister_id}: {e}",
            ))
        }
    }
}

//! This module contains APIs that only the controllers may call.  
//!
//!

use candid::{CandidType, Principal, Reserved};
use ic_cdk::{api::msg_caller, update};
use serde::Deserialize;

use crate::canister_state::privileged::set_disabled_flag;

const GOVERNANCE_CANISTER_ID: &str = "rrkah-fqaaa-aaaaa-aaaaq-cai";

/// Only controllers and the governance canister are allowed to call privileged endpoints.
fn check_caller() -> Result<(), Option<MigrationCanisterError>> {
    let is_controller = ic_cdk::api::is_controller(&msg_caller());
    match is_controller || (msg_caller() == Principal::from_text(GOVERNANCE_CANISTER_ID).unwrap()) {
        true => Ok(()),
        false => Err(Some(MigrationCanisterError::CallerNotAuthorized(Reserved))),
    }
}

#[derive(Clone, Debug, CandidType, Deserialize)]
pub(crate) enum MigrationCanisterError {
    CallerNotAuthorized(Reserved),
}

#[update]
fn enable_api() -> Result<(), Option<MigrationCanisterError>> {
    check_caller()?;
    set_disabled_flag(false);
    Ok(())
}

#[update]
fn disable_api() -> Result<(), Option<MigrationCanisterError>> {
    check_caller()?;
    set_disabled_flag(true);
    Ok(())
}
