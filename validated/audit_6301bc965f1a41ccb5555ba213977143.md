### Title
Unbounded Recursive Deserialization of `NearPromise::Then` Enables Stack Overflow / Wasm Trap in the XCC Precompile - (File: `engine-types/src/parameters/promise.rs`)

### Summary

The `BorshDeserialize` implementation for `NearPromise` is hand-written and directly recursive for the `Then` variant (tag `0x01`). An unprivileged EVM user can craft a `CrossContractCallArgs` payload containing an arbitrarily deeply nested `NearPromise::Then` chain and submit it to the XCC precompile (`0x516cded1d16af10cad47d6d49128e2eb7d27b372`). Deserializing this input causes unbounded Rust call-stack growth with no depth limit, which in a wasm32 environment causes a deterministic wasm stack-overflow trap, panicking the Aurora Engine NEAR contract and permanently preventing the transaction from completing. Because the EVM gas check is performed only on raw byte length (not nesting depth), an attacker can construct a deeply nested structure in a small number of bytes and bypass the gas-proportional cost check.

### Finding Description

**Root cause — `engine-types/src/parameters/promise.rs`, lines 190–196:**

```rust
0x01 => {
    let base = Self::deserialize_reader(reader)?;   // ← unbounded recursion
    let callback = SimpleNearPromise::deserialize_reader(reader)?;
    Ok(Self::Then {
        base: Box::new(base),
        callback,
    })
}
```

Each `Then` node is encoded as a single byte `0x01` followed by the serialized inner `NearPromise`. A chain of depth N is encoded as `N` bytes of `0x01` followed by a single leaf `0x00` + leaf payload. Deserializing this calls `deserialize_reader` recursively N times, consuming N stack frames with no bound.

**Attacker-controlled entry path:**

1. An unprivileged EVM user sends an EVM transaction to the XCC precompile address `cross_contract_call::ADDRESS` (`0x516cded1...`).
2. The transaction `data` field is a Borsh-encoded `CrossContractCallArgs::Eager(PromiseArgs::Recursive(NearPromise::Then{...}))` with an arbitrarily deep `Then` chain.
3. In `engine-precompiles/src/xcc.rs` line 113–115, the gas check is `CROSS_CONTRACT_CALL_BASE + CROSS_CONTRACT_CALL_BYTE * input_len`. A 10 000-deep `Then` chain is only ~10 001 bytes (one `0x01` byte per level plus a small leaf), so the EVM gas cost is `343_650 + 4 * 10_001 ≈ 383_654` EVM gas — easily affordable.
4. At line 137, `CrossContractCallArgs::try_from_slice(input)` is called, which triggers `NearPromise::deserialize_reader` recursively to depth N, exhausting the wasm call stack.
5. The wasm runtime traps, causing the NEAR transaction to fail with an unrecoverable error.

**Why the gas check does not protect against this:**

The EVM gas cost is proportional to `input.len()` (4 gas/byte), but nesting depth is proportional to the number of `0x01` bytes, which is exactly 1 byte per level. A 300 000-deep chain costs only 300 000 bytes × 4 = 1 200 000 EVM gas — well within a normal gas limit — but causes 300 000 recursive stack frames.

**Both execution paths are affected:**

- `CrossContractCallArgs::Eager` — deserialization happens in the EVM execution itself.
- `CrossContractCallArgs::Delayed` — deserialization happens when the router's `execute_scheduled` is called, which also calls `recursive_promise_create` (a mutually recursive function in `etc/xcc-router/src/lib.rs` lines 242–281) on the already-deserialized `NearPromise`, doubling the stack pressure.

### Impact Explanation

**Severity: High — Temporary Freezing of Funds / Denial of Service**

- Any EVM user can submit a crafted XCC transaction that traps the Aurora Engine wasm contract. The NEAR transaction fails, but the attacker's EVM account state is not rolled back in a way that prevents re-submission.
- If the attacker targets the `Delayed` path, the malicious `PromiseArgs` is stored in the XCC router's `scheduled_promises` map. When any party later calls `execute_scheduled`, the router itself traps. Any NEAR tokens attached to that scheduled promise are locked in the router sub-account until the router is upgraded — constituting a temporary (potentially long-duration) freeze of funds.
- Repeated submissions can cause sustained denial of service against the Aurora Engine NEAR contract, preventing legitimate users from executing XCC calls.

### Likelihood Explanation

**High.** The XCC precompile is a publicly documented, production feature reachable by any EVM user with no special privileges. The exploit requires only crafting a Borsh-encoded byte string with repeated `0x01` bytes, which is trivial. The EVM gas cost is low and proportional only to byte length, not nesting depth. No admin keys, governance, or external oracle is required.

### Recommendation

1. **Enforce a maximum nesting depth** during `NearPromise::BorshDeserialize`. Add a depth counter parameter (or a thread-local/stack-based counter) and return an error if depth exceeds a safe bound (e.g., 64).
2. **Alternatively, convert the recursive deserialization to an iterative one** using an explicit stack, eliminating call-stack growth entirely.
3. **Add a depth/count check in the XCC precompile** before deserialization: reject inputs whose `promise_count()` exceeds a protocol-defined maximum.
4. **Apply the same fix to `recursive_promise_create`** in `etc/xcc-router/src/lib.rs`, which is also recursive over `NearPromise::Then`.

### Proof of Concept

Craft the following Borsh payload and submit it as the `data` of an EVM transaction to `cross_contract_call::ADDRESS`:

```
// CrossContractCallArgs::Eager = variant 0
// PromiseArgs::Recursive = variant 2
// NearPromise::Then chain of depth D: D bytes of 0x01, then a leaf
//
// Borsh encoding:
[0x00]                          // CrossContractCallArgs::Eager
[0x02]                          // PromiseArgs::Recursive
[0x01, 0x01, 0x01, ... (D times)]  // NearPromise::Then nested D levels
[0x00]                          // NearPromise::Simple leaf
[... SimpleNearPromise::Create with minimal valid PromiseCreateArgs ...]
```

With D = 50 000, the payload is ~50 010 bytes. EVM gas cost ≈ `343_650 + 4 × 50_010 = 543_690` EVM gas. Deserializing this calls `NearPromise::deserialize_reader` 50 000 times recursively, exhausting the wasm stack and trapping the Aurora Engine contract. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)

impl BorshDeserialize for NearPromise {
    fn deserialize_reader<R: io::Read>(reader: &mut R) -> io::Result<Self> {
        let variant_byte = {
            let mut buf = [0u8; 1];
            reader.read_exact(&mut buf)?;
            buf[0]
        };
        match variant_byte {
            0x00 => {
                let inner = SimpleNearPromise::deserialize_reader(reader)?;
                Ok(Self::Simple(inner))
            }
            0x01 => {
                let base = Self::deserialize_reader(reader)?;
                let callback = SimpleNearPromise::deserialize_reader(reader)?;
                Ok(Self::Then {
                    base: Box::new(base),
                    callback,
                })
            }
            0x02 => {
                let promises: Vec<Self> = Vec::deserialize_reader(reader)?;
                Ok(Self::And(promises))
            }
            _ => Err(io::Error::new(
                io::ErrorKind::InvalidInput,
                "Invalid variant byte for NearPromise",
            )),
        }
    }
