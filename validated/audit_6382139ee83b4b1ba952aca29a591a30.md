### Title
Incorrect Hardcoded Worst-Case Maturity Modulation Constant in `spawn_neuron` Allows Under-Minimum-Stake Neuron Creation — (File: `rs/nns/governance/src/governance.rs`)

---

### Summary

The `spawn_neuron` function in NNS Governance uses a hardcoded worst-case maturity modulation of **5%** (`1_f64 - 0.05`) to pre-validate whether a spawning neuron will meet the minimum stake requirement. However, the Mission 70 system — which now governs actual maturity modulation applied at spawn time — enforces a lower bound of **−10%** (`MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 = -1_000`). This mismatch means the pre-check is too permissive: a neuron owner can initiate a spawn that passes the 5% guard but, when the modulation is at −10%, the resulting spawned neuron will hold less ICP than `neuron_minimum_stake_e8s`.

---

### Finding Description

In `rs/nns/governance/src/governance.rs`, `spawn_neuron` performs a pre-flight check:

```rust
let least_possible_stake = (maturity_to_spawn as f64 * (1_f64 - 0.05)) as u64;

if least_possible_stake < economics.neuron_minimum_stake_e8s {
    return Err(GovernanceError::new_with_message(
        ErrorType::InsufficientFunds,
        "There isn't enough maturity to spawn a new neuron due to worst case maturity modulation.",
    ));
}
``` [1](#0-0) 

The constant `0.05` (5%) was correct under the old CMC-based system, where the modulation range was `[−500, +500]` permyriad (±5%), defined in:

```rust
pub const MIN_MATURITY_MODULATION_PERMYRIAD: i32 = -500;
pub const MAX_MATURITY_MODULATION_PERMYRIAD: i32 = 500;
``` [2](#0-1) 

However, the Mission 70 system — now the authoritative source for NNS Governance maturity modulation — defines a **different, asymmetric range**:

```rust
/// Lower bound for Mission 70 maturity modulation: -10% = -1000 permyriad.
pub(crate) const MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70: i64 = -1_000;

/// Upper bound for Mission 70 maturity modulation: +2% = 200 permyriad.
pub(crate) const MATURITY_MODULATION_MAX_PERMYRIAD_MISSION_70: i64 = 200;
``` [3](#0-2) 

The `VALID_MATURITY_MODULATION_BASIS_POINTS_RANGE` used in `maybe_spawn_neurons` already reflects the Mission 70 bounds `[−1000, +200]`:

```rust
const VALID_MATURITY_MODULATION_BASIS_POINTS_RANGE: RangeInclusive<i32> =
    MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 as i32
        ..=MATURITY_MODULATION_MAX_PERMYRIAD_MISSION_70 as i32;
``` [4](#0-3) 

And `maybe_spawn_neurons` applies the Mission 70 modulation via `apply_maturity_modulation`: [5](#0-4) 

The `apply_maturity_modulation` function itself enforces no bounds — it accepts any `i32`: [6](#0-5) 

The CHANGELOG confirms the switchover to Mission 70 is live: [7](#0-6) 

---

### Impact Explanation

For `neuron_minimum_stake_e8s = 1 ICP = 1e8 e8s`:

- The spawn pre-check passes when `maturity_to_spawn * 0.95 ≥ 1e8`, i.e., `maturity_to_spawn ≥ ~1.053e8`.
- The actual worst-case Mission 70 modulation is −10%, so the spawned stake = `maturity_to_spawn * 0.90`.
- For `maturity_to_spawn` in `[1.053e8, 1.111e8)`, the pre-check passes but the spawned neuron receives **less than 1 ICP** when modulation is at −10%.

A spawned neuron below `neuron_minimum_stake_e8s` is a governance accounting violation: such a neuron may be unable to vote, earn rewards, or participate normally in NNS governance, while ICP has already been minted to it. This is a ledger conservation / governance correctness bug reachable by any unprivileged NNS neuron owner.

---

### Likelihood Explanation

The Mission 70 modulation reaches −10% when the 7-day ICP price average is significantly below the 365-day average. This is a realistic market condition (e.g., a sustained bear market). The daily speed limit of 30 permyriad/day means the modulation can reach −1000 after ~33 days of sustained price decline. Any neuron owner with maturity in the vulnerable range `[1.053e8, 1.111e8)` e8s who spawns during such a period will trigger the bug. The entry path requires no privilege — `spawn_neuron` is callable by any neuron controller via `manage_neuron`.

---

### Recommendation

Replace the hardcoded `0.05` in `spawn_neuron` with the Mission 70 worst-case constant:

```rust
// Before (incorrect — uses old CMC worst-case of 5%):
let least_possible_stake = (maturity_to_spawn as f64 * (1_f64 - 0.05)) as u64;

// After (correct — uses Mission 70 worst-case of 10%):
use crate::timer_tasks::update_icp_xdr_rate_related_data::MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70;
let worst_case_factor = 1.0 + (MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 as f64 / 10_000.0);
let least_possible_stake = (maturity_to_spawn as f64 * worst_case_factor) as u64;
```

Also update the stale constants `MIN_MATURITY_MODULATION_PERMYRIAD = -500` and `MAX_MATURITY_MODULATION_PERMYRIAD = 500` in `rs/nervous_system/governance/src/maturity_modulation/mod.rs` to reflect the Mission 70 bounds, or remove them to avoid confusion. Add a unit test that verifies `spawn_neuron` rejects maturity amounts that would fall below `neuron_minimum_stake_e8s` at −10% modulation.

---

### Proof of Concept

1. NNS neuron owner holds a neuron with `maturity_e8s_equivalent = 1_060_000_000` (≈1.06 ICP, just above the 5% threshold for `neuron_minimum_stake_e8s = 1e8`).
2. Owner calls `manage_neuron` → `Spawn { percentage_to_spawn: 100, ... }`.
3. `spawn_neuron` computes `least_possible_stake = 1_060_000_000 * 0.95 = 1_007_000_000 ≥ 1e8` → check passes, spawning neuron is created.
4. Mission 70 maturity modulation is at −1000 permyriad (−10%) due to sustained ICP price decline.
5. `maybe_spawn_neurons` fires, passes the `VALID_MATURITY_MODULATION_BASIS_POINTS_RANGE` check (−1000 is within `[−1000, 200]`), and calls `apply_maturity_modulation(1_060_000_000, -1000)`.
6. Spawned neuron receives `1_060_000_000 * (1 − 0.10) = 954_000_000 e8s` — **below** `neuron_minimum_stake_e8s = 1e8 = 1_000_000_000 e8s`.
7. A sub-minimum-stake neuron now exists in NNS governance state with minted ICP, violating the protocol invariant.

pub fn apply_maturity_modulation(
    amount_maturity_e8s: u64,
    maturity_modulation_basis_points: i32,
) -> Result<u64, String> {
    let amount_e8s = u128::from(amount_maturity_e8s);

    let adjusted_maturity_modulation_basis_points = saturating_add_or_subtract_u128_i32(
        BASIS_POINTS_PER_UNITY,
        maturity_modulation_basis_points,
    );

    let modulated_amount_e8s: u128 = amount_e8s
        .checked_mul(adjusted_maturity_modulation_basis_points)
        .ok_or_else(|| "Underflow or overflow when calculating maturity modulation".to_string())?
        .checked_div(BASIS_POINTS_PER_UNITY)
        .ok_or_else(|| "Underflow or overflow when calculating maturity modulation".to_string())?;

    u64::try_from(modulated_amount_e8s).map_err(|err| err.to_string())
}
