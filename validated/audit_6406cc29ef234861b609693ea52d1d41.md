### Title
Silent No-Op on `pause_precompiles` with Unknown-Only Bitmask — (`engine/src/contract_methods/admin.rs`)

### Summary
`pause_precompiles` converts the caller-supplied `u32` mask via `PrecompileFlags::from_bits_truncate`, which silently discards all bits that do not correspond to a defined flag. When the supplied mask has no known bits set (e.g. `0xFFFFFFFC`, `0x4`, `0xFFFFFFF8`), the result is `PrecompileFlags::empty()`, the storage write is a no-op, and the function returns `Ok(())`. The caller receives no indication of failure.

### Finding Description
`PrecompileFlags` currently defines only two flags (bits 0 and 1). [4](#0-3) 

In `pause_precompiles`, the raw mask is decoded with `from_bits_truncate`: [2](#0-1) 

`from_bits_truncate` is documented by the `bitflags` crate to silently drop any bits that do not correspond to a named constant. There is no post-truncation check that the resulting `PrecompileFlags` value is non-empty. The `PausedPrecompilesManager::pause_precompiles` implementation then calls `pause_flags.insert(precompiles_to_pause)`, which is a no-op when `precompiles_to_pause` is empty: [3](#0-2) 

The same silent-truncation pattern is present in `resume_precompiles`: [5](#0-4) 

### Impact Explanation
An authorized caller (e.g. a security monitoring contract or automated guardian) that passes a mask whose set bits all fall outside the two currently-defined flags will believe the pause succeeded — the transaction commits, no error is returned, and the on-chain state is unchanged. During an emergency where `ExitToNear` and `ExitToEthereum` must be halted, the precompiles remain active, allowing users to continue bridging funds out. This directly maps to **High — Temporary freezing of funds** (the freeze is not achieved when intended).

### Likelihood Explanation
Likelihood is **low-to-medium**. Realistic triggering scenarios include:

- A guardian contract hard-coded with `0xFFFFFFFF` or a "pause all" sentinel that happens to have bits 0–1 clear.
- A caller targeting a precompile added in a future upgrade (e.g. bit 2 = `0x4`) before the engine is actually upgraded, resulting in a mask with no currently-known bits.
- An off-chain script that computes the mask from an outdated ABI or documentation.

In all cases the caller is authorized and acts in good faith; the silent failure is invisible without an explicit follow-up `paused_precompiles` query.

### Recommendation
After truncation, validate that the resulting flags are non-empty when the raw input was non-zero:

```rust
let flags = PrecompileFlags::from_bits_truncate(args.paused_mask);
if args.paused_mask != 0 && flags.is_empty() {
    return Err(b"ERR_UNKNOWN_PRECOMPILE_FLAGS".into());
}
```

Alternatively, use `PrecompileFlags::from_bits(args.paused_mask).ok_or(...)` to reject any mask containing unknown bits outright. Apply the same fix to `resume_precompiles`.

### Proof of Concept
Unit test (no external dependencies, runs on unmodified code):

```rust
#[test]
fn test_pause_with_unknown_only_mask_is_silent_noop() {
    use std::cell::RefCell;
    use aurora_engine_test_doubles::io::{Storage, StoragePointer};

    let storage = RefCell::new(Storage::default());
    let io = StoragePointer(&storage);
    let mut pauser = EnginePrecompilesPauser::from_io(io);

    // 0xFFFFFFFC has bits 0 and 1 clear — both known flags are absent
    let raw_mask: u32 = 0xFFFF_FFFC;

    // Demonstrates the silent truncation
    assert!(flags.is_empty(), "from_bits_truncate silently dropped all bits");

    pauser.pause_precompiles(flags);

    // Neither precompile is paused despite the non-zero input mask
    assert!(!pauser.is_paused(PrecompileFlags::EXIT_TO_NEAR));
    assert!(!pauser.is_paused(PrecompileFlags::EXIT_TO_ETHEREUM));
}
```
