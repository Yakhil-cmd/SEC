Audit Report

## Title
SNS Swap Canister Has No Mechanism to Return SNS Tokens to Treasury on Aborted Swap — (File: rs/sns/swap/src/swap.rs)

## Summary
When an SNS decentralization swap aborts, `finalize_inner` refunds ICP to buyers, settles Neurons' Fund maturity, and restores dapp controllers, then returns early without sweeping the pre-loaded SNS token allocation back to the SNS governance treasury. The SNS tokens remain in the swap canister's ledger account indefinitely. The integration test suite explicitly asserts this as the observed post-abort state. Because the documented terminal lifecycle path is `ABORTED → <DELETED>`, deletion of the swap canister without a prior recovery step would permanently destroy the entire SNS token swap allocation.

## Finding Description
`finalize_inner` in `rs/sns/swap/src/swap.rs` (L1544–1584) dispatches on lifecycle via `should_restore_dapp_control()`, which returns `true` exactly when `self.lifecycle() == Lifecycle::Aborted` (L1348–1350). On that branch it calls `restore_dapp_controllers_for_finalize` and then unconditionally returns (L1583). No SNS ledger transfer is issued.

The `sweep_sns` function (L2165–2178) is guarded by an explicit lifecycle check that rejects any call where the lifecycle is not `Committed`, so it cannot be used as a recovery path in the aborted state.

The integration test at `rs/nervous_system/integration_tests/tests/sns_lifecycle.rs` (L1336–1342) asserts:
```rust
if swap_finalization_status == SwapFinalizationStatus::Aborted {
    assert_eq!(swap_canister_balance_sns_e8s, swap_distribution_sns_e8s);
}
```
confirming the full allocation remains in the swap canister after finalization.

The lifecycle diagram in `rs/sns/swap/proto/ic_sns_swap/pb/v1/swap.proto` (L56–66) shows `ABORTED → <DELETED>` as the terminal path. The protocol documentation at L149–150 states the swap canister "can be deleted when all tokens registered with the swap canister have been disbursed to their rightful owners" — a precondition that is never satisfied for the SNS token allocation in the aborted path, since no disbursement mechanism exists.

The only recovery path is an NNS governance proposal to upgrade the swap canister with a post-upgrade hook that performs the transfer. This is not automatic, not documented as a required step, and not enforced by the protocol.

## Impact Explanation
This is a significant SNS framework accounting gap. The entire `swap_distribution_sns_e8s` allocation — which can represent millions of governance tokens — is stranded in the swap canister's ledger account with no automatic recovery. If the swap canister is deleted following the documented terminal lifecycle path without a prior NNS upgrade to recover the tokens, those tokens are permanently inaccessible on the SNS ledger. This constitutes a concrete, protocol-level loss of SNS governance assets. This matches the High impact class: "Significant SNS or infrastructure security impact with concrete user or protocol harm."

## Likelihood Explanation
Any SNS swap that fails to reach `min_participants` or `min_icp_e8s` before the deadline transitions to `Aborted` via normal protocol operation — no attacker action is required. New or low-visibility SNS projects are particularly susceptible. The risk of permanent loss materializes if the swap canister is subsequently deleted without a recovery upgrade, which is the intended terminal state per the lifecycle diagram.

## Recommendation
In `finalize_inner`, when `should_restore_dapp_control()` is true (i.e., lifecycle is `Aborted`), add a step before returning that transfers the remaining SNS token balance from the swap canister's account back to the SNS governance treasury account. This mirrors the existing `sweep_icp` pattern. The transfer should be idempotent (guarded by a flag analogous to other sweep operations) so that retried `finalize` calls do not double-transfer. Alternatively, add a dedicated `sweep_sns_to_treasury` method callable only in the `Aborted` state, and invoke it as part of the aborted finalization path.

## Proof of Concept
1. Deploy an SNS with `swap_distribution_sns_e8s = N`.
2. Open the swap; do not reach `min_participants` before the deadline.
3. `try_abort` transitions lifecycle to `Aborted`.
4. Call `finalize`. `finalize_inner` executes `sweep_icp`, `settle_neurons_fund_participation`, `restore_dapp_controllers_for_finalize`, then returns at L1583.
5. Query `icrc1_balance_of(swap_canister_account)` on the SNS ledger — returns `N` (unchanged), as asserted by the integration test at L1336–1342.
6. No callable method on the swap canister in its current code will authorize a transfer of those tokens.
7. Submit an NNS proposal to delete the swap canister. Upon deletion, the `N` SNS tokens are permanently inaccessible.
