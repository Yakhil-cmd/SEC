Audit Report

## Title
Unprivileged Caller Can Permanently Dilute Any Neuron's Age Bonus via Unauthorized `refresh_neuron` — (`rs/nns/governance/src/governance.rs`)

## Summary

The NNS Governance canister's `manage_neuron_internal` function processes `ClaimOrRefresh` commands before any authorization check, allowing any unprivileged caller to trigger `refresh_neuron` against any neuron by ID or subaccount. When an attacker sends ICP to a victim's neuron subaccount and triggers a refresh, `update_stake_adjust_age` is called, permanently reducing the neuron's `aging_since_timestamp_seconds` via a weighted-average dilution. This allows systematic, permissionless reduction of a victim's age bonus (up to 25% voting power multiplier) and proportional staking rewards in NNS governance.

## Finding Description

In `rs/nns/governance/src/governance.rs`, `manage_neuron_internal` handles `ClaimOrRefresh` before any neuron ownership or hotkey check is performed:

```rust
// L6104-6142
if let Some(Command::ClaimOrRefresh(claim_or_refresh)) = &mgmt.command {
    return match &claim_or_refresh.by {
        Some(By::NeuronIdOrSubaccount(_)) => {
            self.refresh_neuron_by_id_or_subaccount(id, claim_or_refresh).await
        }
        Some(By::MemoAndController(memo_and_controller)) => {
            self.claim_or_refresh_neuron_by_memo_and_controller(...)
        }
        ...
    };
}
```

`refresh_neuron_by_id_or_subaccount` (L5875–5896) resolves the neuron and calls `refresh_neuron` with no caller identity check. Inside `refresh_neuron` (L5936–5958), when `balance > cached_stake` (which the attacker arranges by sending ICP to the neuron's subaccount), `update_stake_adjust_age` is called unconditionally.

`update_stake_adjust_age` in `rs/nns/governance/src/neuron/types.rs` (L1021–1038) computes:

```rust
let (new_stake_e8s, new_age_seconds) = combine_aged_stakes(
    self.cached_neuron_stake_e8s,
    self.age_seconds(now),
    updated_stake_e8s.saturating_sub(self.cached_neuron_stake_e8s),
    0,  // new ICP has age 0
);
```

This permanently reduces `aging_since_timestamp_seconds` proportionally to the added stake. The test suite at `rs/nns/governance/tests/governance.rs` L4663–4666 explicitly documents this as the current behavior:

```
// - Refresh with memo
//   - Anyone can do it
// - Refresh with subaccount
//   - Anyone can do it
```

The `By::MemoAndController` path (L5852–5871) also allows any caller to specify an arbitrary `controller` principal, targeting any neuron whose subaccount is derivable from a known principal and memo. The SNS governance `refresh_neuron` at `rs/sns/governance/src/governance.rs` L4237–4298 has the same structure and absence of authorization.

No existing guard prevents a third party from triggering age dilution on a neuron they do not control.

## Impact Explanation

A neuron's age bonus contributes up to **25% additional voting power** and proportionally higher ICP staking rewards, accruing over a maximum of 4 years. The `aging_since_timestamp_seconds` field is the sole determinant of this bonus. Each attacker-triggered refresh with a small ICP top-up permanently dilutes this value:

- Victim: 10 ICP staked, 4 years old → full 25% age bonus
- Attacker sends 1 ICP, triggers refresh → new age ≈ 3.64 years, bonus ≈ 22.7%
- After ~40 iterations (40 ICP spent): 50 ICP staked, age ≈ 0.8 years, bonus ≈ 5%

The victim's ICP stake increases (attacker's ICP is locked in victim's neuron), but their relative governance influence and annual reward share are permanently reduced. For large neuron holders (e.g., 100M ICP staked for 4+ years), the age bonus represents substantial governance influence. This constitutes significant NNS governance security impact with concrete, measurable harm to neuron holders.

This maps to the **High ($2,000–$10,000)** impact class: "Significant NNS security impact with concrete user or protocol harm."

## Likelihood Explanation

The attack is reachable by any unprivileged ingress sender. Neuron IDs and subaccounts are public (derivable from principal + memo, or queryable via `get_neuron_ids`). The attacker must spend ICP per dilution step (currently minimum 1 ICP on mainnet), but this ICP is locked in the victim's neuron rather than destroyed, making the attack economically rational against high-value governance targets where reducing a competitor's voting power is worth the cost. The attack requires no coordination, no special role, and is indefinitely repeatable.

## Recommendation

Add an authorization check to the `ClaimOrRefresh` refresh path (distinct from the claim path, which legitimately needs to be open). For `By::NeuronIdOrSubaccount` and `By::MemoAndController` when the neuron already exists, verify that the caller is the neuron's controller or a registered hot key before calling `refresh_neuron`. The claim path (neuron does not yet exist) can remain open, as the subaccount is derived from the controller's principal and an attacker cannot claim a neuron they do not control.

Alternatively, modify `update_stake_adjust_age` so that age dilution only occurs when the caller is the neuron's controller or hot key, while still allowing balance synchronization for anyone (without the age side-effect) — preserving the utility of open refresh for balance discovery.

Apply the same fix to the SNS governance `refresh_neuron` path.

## Proof of Concept

1. Victim `V` has neuron `N` with 10 ICP staked, 4 years old, dissolve delay 8 years (not dissolving). Full 25% age bonus applies.
2. Attacker `A` (any principal) queries `get_neuron_ids` or derives `N`'s subaccount from known principal + memo.
3. `A` transfers 1 ICP to `N`'s subaccount on the ICP ledger. Ledger balance = 11 ICP; cached stake = 10 ICP.
4. `A` calls `manage_neuron` on NNS governance:
   ```
   ManageNeuron {
     neuron_id_or_subaccount: Some(NeuronId(N.id)),
     command: Some(ClaimOrRefresh { by: Some(NeuronIdOrSubaccount(())) })
   }
   ```
5. `refresh_neuron` is called. Balance (11 ICP) > cached stake (10 ICP) → `update_stake_adjust_age(11e8, now)`. New age = (10 × 4 years) / 11 ≈ 3.64 years. Age bonus reduced from 25% to ~22.7%.
6. Repeat steps 3–5. Each iteration costs `A` 1 ICP (locked in `V`'s neuron) and reduces `V`'s age bonus further.

A deterministic unit test can be written against the existing `fake::FakeDriver` + `Governance` test harness (as used throughout `rs/nns/governance/tests/governance.rs`) with a third-party caller, a pre-aged neuron, and ledger balance manipulation to confirm `aging_since_timestamp_seconds` is reduced after each third-party refresh.     async fn refresh_neuron(&mut self, nid: &NeuronId) -> Result<(), GovernanceError> {
        let now = self.env.now();
        let subaccount = nid.subaccount()?;
        let account = self.neuron_account_id(subaccount);

        // First ensure that the neuron was not created via an NNS Neurons' Fund participation in the
        // decentralization swap
        {
            let neuron = self.get_neuron_result(nid)?;

            if neuron.is_neurons_fund_controlled() {
                return Err(GovernanceError::new_with_message(
                    ErrorType::PreconditionFailed,
                    "Cannot refresh an SNS Neuron controlled by the Neurons' Fund",
                ));
            }
        }

        // Get the balance of the neuron from the ledger canister.
        let balance = self.ledger.account_balance(account).await?;

        let min_stake = self
            .nervous_system_parameters_or_panic()
            .neuron_minimum_stake_e8s
            .expect("NervousSystemParameters must have neuron_minimum_stake_e8s");
        if balance.get_e8s() < min_stake {
            return Err(GovernanceError::new_with_message(
                ErrorType::InsufficientFunds,
                format!(
                    "Account does not have enough funds to refresh a neuron. \
                        Please make sure that account has at least {:?} e8s (was {:?} e8s)",
                    min_stake,
                    balance.get_e8s()
                ),
            ));
        }
        let neuron = self.get_neuron_result_mut(nid)?;
        match neuron.cached_neuron_stake_e8s.cmp(&balance.get_e8s()) {
            Ordering::Greater => {
                log!(
                    ERROR,
                    "ERROR. Neuron cached stake was inconsistent.\
                     Neuron account: {} has less e8s: {} than the cached neuron stake: {}.\
                     Stake adjusted.",
                    account,
                    balance.get_e8s(),
                    neuron.cached_neuron_stake_e8s
                );
                neuron.update_stake(balance.get_e8s(), now);
            }
            Ordering::Less => {
                neuron.update_stake(balance.get_e8s(), now);
            }
            // If the stake is the same as the account balance,
            // just return the neuron id (this way this method
            // also serves the purpose of allowing to discover the
            // neuron id based on the memo and the controller).
            Ordering::Equal => (),
        };

        Ok(())
    }
