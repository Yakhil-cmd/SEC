Audit Report

## Title
Missing Execution-Time Capacity Check Allows Exceeding `MAX_NUMBER_OF_GENERIC_NERVOUS_SYSTEM_FUNCTIONS` - (File: rs/sns/governance/src/governance.rs)

## Summary
`perform_add_generic_nervous_system_function` in `rs/sns/governance/src/governance.rs` does not re-check the `MAX_NUMBER_OF_GENERIC_NERVOUS_SYSTEM_FUNCTIONS` cap at execution time. Two concurrently adopted `AddGenericNervousSystemFunction` proposals with distinct IDs can both execute successfully when the registry is at `MAX - 1`, pushing it to `MAX + 1` and permanently violating the intended invariant.

## Finding Description
The submission-time validator `validate_and_render_add_generic_nervous_system_function` in `rs/sns/governance/src/proposal.rs` (L1395–1397) checks:

```rust
if existing_functions.len() >= MAX_NUMBER_OF_GENERIC_NERVOUS_SYSTEM_FUNCTIONS {
    return Err("Reached maximum number of allowed GenericNervousSystemFunctions".to_string());
}
``` [1](#0-0) 

This check is evaluated against the state snapshot at proposal-submission time. The execution function `perform_add_generic_nervous_system_function` (L2247–2299) performs only a duplicate-ID guard via `is_registered_function_id` and canister-target checks, with no capacity re-check before inserting:

```rust
self.proto
    .id_to_nervous_system_functions
    .insert(id, nervous_system_function);
``` [2](#0-1) 

When two proposals with different IDs (e.g., 1000 and 1001) are both submitted while `len == MAX - 1`, both pass submission-time validation. When both are adopted and executed sequentially, the first brings the count to `MAX`; the second passes the duplicate-ID check (1001 ≠ 1000) and inserts unconditionally, bringing the count to `MAX + 1`. By contrast, `perform_manage_nervous_system_parameters` explicitly re-validates at execution time and carries a comment acknowledging this exact concurrent-proposal risk: [3](#0-2) 

## Impact Explanation
The SNS governance canister ends up with more registered `GenericNervousSystemFunction` entries than `MAX_NUMBER_OF_GENERIC_NERVOUS_SYSTEM_FUNCTIONS` permits. This is a persistent, corrupted governance state that violates an explicitly enforced invariant, can inflate canister heap memory beyond the intended bound, and may cause downstream logic that assumes the cap is respected to behave incorrectly. The corrupted state persists until a corrective governance proposal is executed. This matches **High ($2,000–$10,000): Significant SNS security impact with concrete user or protocol harm**.

## Likelihood Explanation
No special privileges or key compromise are required. Any SNS neuron holder with sufficient voting power can submit `AddGenericNervousSystemFunction` proposals. Two independent neuron holders each submit a proposal with a distinct function ID when the registry is near the cap — a realistic operational state for active SNSes. Both proposals pass through normal governance voting. Both execute sequentially in the same or consecutive heartbeat rounds. The duplicate-ID guard does not prevent this because the two proposals use different IDs.

## Recommendation
Add the capacity check inside `perform_add_generic_nervous_system_function`, mirroring the pattern in `perform_manage_nervous_system_parameters`:

```rust
let current_count = self.proto.id_to_nervous_system_functions
    .values()
    .filter(|f| !f.is_deletion_marker())
    .count();
if current_count >= MAX_NUMBER_OF_GENERIC_NERVOUS_SYSTEM_FUNCTIONS {
    return Err(GovernanceError::new_with_message(
        ErrorType::PreconditionFailed,
        "Reached maximum number of allowed GenericNervousSystemFunctions",
    ));
}
```

This re-check should be placed immediately before the `insert` call at L2295–2297 in `rs/sns/governance/src/governance.rs`. [4](#0-3) 

## Proof of Concept
1. Deploy an SNS with `MAX_NUMBER_OF_GENERIC_NERVOUS_SYSTEM_FUNCTIONS - 1` registered generic functions.
2. Submit `AddGenericNervousSystemFunction` proposal A with `id = 1000`. Submission-time validation passes (`len == MAX - 1 < MAX`).
3. Submit `AddGenericNervousSystemFunction` proposal B with `id = 1001`. Submission-time validation passes (same snapshot).
4. Both proposals gather sufficient votes and are adopted.
5. Proposal A executes: `perform_add_generic_nervous_system_function` inserts id 1000. Registry count = `MAX`.
6. Proposal B executes: `is_registered_function_id(1001, ...)` returns false (1001 ≠ 1000), no capacity check fires, id 1001 is inserted. Registry count = `MAX + 1`.
7. Assert `id_to_nervous_system_functions.len() > MAX_NUMBER_OF_GENERIC_NERVOUS_SYSTEM_FUNCTIONS` — invariant violated with no error returned.

This is reproducible as a deterministic integration test or PocketIC test against the SNS governance canister. [5](#0-4)

    fn perform_add_generic_nervous_system_function(
        &mut self,
        nervous_system_function: NervousSystemFunction,
    ) -> Result<(), GovernanceError> {
        let id = nervous_system_function.id;

        if nervous_system_function.is_native() {
            return Err(GovernanceError::new_with_message(
                ErrorType::PreconditionFailed,
                "Can only add NervousSystemFunction's of \
                                                          GenericNervousSystemFunction function_type",
            ));
        }

        if is_registered_function_id(id, &self.proto.id_to_nervous_system_functions) {
            return Err(GovernanceError::new_with_message(
                ErrorType::PreconditionFailed,
                format!(
                    "Failed to add NervousSystemFunction. \
                             There is/was already a NervousSystemFunction with id: {id}"
                ),
            ));
        }

        // This validates that it is well-formed, but not the canister targets.
        match ValidGenericNervousSystemFunction::try_from(&nervous_system_function) {
            Ok(valid_function) => {
                let reserved_canisters = self.reserved_canister_targets();
                let target_canister_id = valid_function.target_canister_id;
                let validator_canister_id = valid_function.validator_canister_id;

                if reserved_canisters.contains(&target_canister_id)
                    || reserved_canisters.contains(&validator_canister_id)
                {
                    return Err(GovernanceError::new_with_message(
                        ErrorType::PreconditionFailed,
                        "Cannot add generic nervous system functions that targets sns core canisters, the NNS ledger, or ic00",
                    ));
                }
            }
            Err(msg) => {
                return Err(GovernanceError::new_with_message(
                    ErrorType::PreconditionFailed,
                    msg,
                ));
            }
        }

        self.proto
            .id_to_nervous_system_functions
            .insert(id, nervous_system_function);
        Ok(())
    }
```
