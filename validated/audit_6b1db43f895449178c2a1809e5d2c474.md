Audit Report

## Title
Unbounded Body Buffering via `to_bytes(body, usize::MAX)` in `RosettaMetricsMiddleware` Enables Unauthenticated OOM DoS — (`rs/rosetta-api/common/rosetta_core/src/metrics.rs`)

## Summary

`RosettaMetricsMiddleware::call` unconditionally passes every non-`/metrics` HTTP request body to `extract_canister_id`, which calls `to_bytes(body, usize::MAX)` with no upper bound. Because the ICRC1 Rosetta Axum router applies no `RequestBodyLimitLayer` before this middleware, a single unauthenticated TCP connection streaming an arbitrarily large chunked body will cause unbounded heap allocation, OOM-killing the Rosetta process and halting all ledger sync and API operations.

## Finding Description

In `rs/rosetta-api/common/rosetta_core/src/metrics.rs`, the helper `extract_canister_id` reads the entire request body into memory with no size cap:

```rust
// line 389
let bytes = to_bytes(body, usize::MAX).await?;
```

This is invoked unconditionally for every request that is not `/metrics` inside `RosettaMetricsMiddleware::call` (lines 336–339):

```rust
let (canister_id, reconstructed_body) = match extract_canister_id(body).await {
    Ok((canister_id, body_bytes)) => (canister_id, Body::from(body_bytes)),
    Err(_) => (None, Body::empty()),
};
```

The ICRC1 Rosetta router in `rs/rosetta-api/icrc1/src/main.rs` (lines 361–391) applies `metrics_layer` as the innermost Tower layer wrapping all routes, with no `RequestBodyLimitLayer` or `DefaultBodyLimit` anywhere in the stack:

```rust
.layer(metrics_layer)
.layer(add_request_span())
.layer(RequestIdLayer)
```

By contrast, the IC HTTP endpoint in `rs/http_endpoints/public/src/lib.rs` (lines 688–691) explicitly guards against this:

```rust
.layer(DefaultBodyLimit::disable())
.layer(RequestBodyLimitLayer::new(config.max_request_size_bytes as usize))
```

No equivalent guard exists in the Rosetta ICRC1 stack. The body is fully buffered before any routing, authentication, or JSON parsing occurs, so the attacker does not need a valid JSON payload or any credentials.

## Impact Explanation

This matches **High ($2,000–$10,000): Significant Rosetta/ledger infrastructure security impact with concrete user or protocol harm**, and also **High: Application/platform-level DoS, crash, or subnet availability impact not based on raw volumetric DDoS**. A single TCP connection with a streaming chunked body (no `Content-Length` required) exhausts process RSS, triggering an OOM kill. The Rosetta sync thread, withdrawal processor, and all in-flight ckBTC/ckETH/ICRC1 operations served by that instance stall until the process is manually restarted. The attack is repeatable immediately after restart.

## Likelihood Explanation

The Rosetta HTTP port is the exchange-facing API, typically bound to `0.0.0.0:<port>` and publicly reachable. No credentials, valid JSON, prior state, or special privileges are required. A single TCP connection with chunked transfer encoding suffices. The attack is deterministic and repeatable with no per-attempt cost beyond a TCP connection.

## Recommendation

Replace `usize::MAX` with a bounded constant in `extract_canister_id` in `rs/rosetta-api/common/rosetta_core/src/metrics.rs`:

```rust
const MAX_BODY_SIZE: usize = 10 * 1024 * 1024; // 10 MB
let bytes = to_bytes(body, MAX_BODY_SIZE).await
    .map_err(|_| /* return 413 Payload Too Large */)?;
```

Alternatively (or additionally), add `RequestBodyLimitLayer::new(MAX_BODY_SIZE)` to the Axum router in `rs/rosetta-api/icrc1/src/main.rs` before `metrics_layer` is applied, mirroring the pattern in `rs/http_endpoints/public/src/lib.rs`.

## Proof of Concept

```bash
# Stream a 2 GB body to the /block endpoint; no Content-Length header needed
python3 -c "
import socket, time
s = socket.create_connection(('rosetta-host', 8082))
s.send(b'POST /block HTTP/1.1\r\nHost: rosetta-host\r\nTransfer-Encoding: chunked\r\nContent-Type: application/json\r\n\r\n')
chunk = b'%x\r\n%s\r\n' % (65536, b'A' * 65536)
while True:
    s.send(chunk)
    time.sleep(0.001)
"
# Monitor: watch -n1 'ps aux | grep rosetta'
# Expected: process RSS grows unboundedly → OOM kill → sync thread dies
```

A deterministic integration test can be written by spawning a local Rosetta instance and issuing a single POST to `/block` with a body reader that yields bytes indefinitely, asserting the server returns HTTP 413 (after the fix) rather than crashing.
