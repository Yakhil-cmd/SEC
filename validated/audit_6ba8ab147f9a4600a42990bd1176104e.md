### Title
Missing Genesis Moratorium Guard on `donate_account` Allows Irreversible Neuron Donation Before Moratorium Ends — (`File: rs/nns/gtc/src/lib.rs`)

### Summary

The Genesis Token Canister (GTC) enforces a 3-day post-genesis moratorium on `claim_neurons` via `assert_claim_neurons_can_be_called`, but the sibling function `donate_account` carries no equivalent guard. Any GTC account owner can call `donate_account` via ingress immediately after genesis, permanently and irreversibly transferring their neurons to the custodian neuron before the moratorium window closes. Because `claim_neurons` subsequently rejects accounts that have already donated (`has_donated == true`), the moratorium's protective window is bypassed for the donation path.

---

### Finding Description

`rs/nns/gtc/src/lib.rs` defines two time-gated constants:

```
SECONDS_UNTIL_CLAIM_NEURONS_CAN_BE_CALLED = 3 * 86400          // 3 days
SECONDS_UNTIL_FORWARD_WHITELISTED_UNCLAIMED_ACCOUNTS_CAN_BE_CALLED = 188 * 86400  // 188 days
```

`claim_neurons` enforces the 3-day guard at its very first line:

```rust
self.assert_claim_neurons_can_be_called()?;
```

`forward_whitelisted_unclaimed_accounts` enforces the 188-day guard at its first line:

```rust
self.assert_forward_whitelisted_unclaimed_accounts_can_be_called()?;
```

`donate_account` has **no such guard**:

```rust
pub async fn donate_account(
    &mut self,
    caller: &PrincipalId,
    public_key_hex: String,
) -> Result<(), String> {
    let public_key = decode_hex_public_key(&public_key_hex)?;
    validate_public_key_against_caller(&public_key, caller)?;
    // ← no assert_claim_neurons_can_be_called() here
    let custodian_neuron_id = self.donate_account_recipient_neuron_id;
    let address = public_key_to_gtc_address(&public_key);
    let account = self.get_account_mut(&address)?;
    account.authenticated_principal_id = Some(*caller);
    account.transfer(custodian_neuron_id).await?;
    account.has_donated = true;
    Ok(())
}
```

After a successful donation, `claim_neurons` permanently rejects the account:

```rust
if account.has_donated {
    return Err("Account has previously donated its funds".to_string());
}
```

The integration test `test_gtc_at_genesis` explicitly verifies that `claim_neurons` and `forward_whitelisted_unclaimed_accounts` fail during the moratorium, but it never asserts that `donate_account` also fails during the moratorium — confirming the guard is absent and untested.

---

### Impact Explanation

`donate_account` calls `AccountState::transfer`, which iterates over all neuron IDs and calls `GovernanceCanister::transfer_gtc_neuron` for each one, moving every neuron's controller from the GTC to the custodian neuron. This is irreversible: once `has_donated = true` is set, the account can never claim its neurons. A GTC account owner who calls `donate_account` during the 3-day moratorium permanently forfeits their ability to claim their neurons, bypassing the moratorium's protective intent. The moratorium was designed to give the IC time to stabilize and give users a guaranteed window to claim; the missing guard on `donate_account` undermines this guarantee for the donation path.

---

### Likelihood Explanation

The entry path is a standard ingress update call to the publicly exposed `canister_update donate_account` endpoint on the GTC canister. The caller must supply their own valid secp256k1 public key (verified against `ic_cdk::api::msg_caller`), so only the legitimate account owner can trigger this. The likelihood of accidental or socially-engineered premature donation during the 3-day window is low but non-zero, and the consequence is permanent and irreversible loss of the ability to claim neurons.

---

### Recommendation

Add the same moratorium guard to `donate_account` that is already present in `claim_neurons`:

```rust
pub async fn donate_account(
    &mut self,
    caller: &PrincipalId,
    public_key_hex: String,
) -> Result<(), String> {
    self.assert_claim_neurons_can_be_called()?;   // ← add this
    // ... rest of function unchanged
}
```

Additionally, add a test case to `test_gtc_at_genesis` asserting that `donate_account` also returns an error during the moratorium period.

/// At Genesis, calls to `claim_neurons` and `forward_all_unclaimed_accounts`
/// should fail, as they both depend on a certain amount of time passing before
/// they are able to be called.
#[test]
pub fn test_gtc_at_genesis() {
    let state_machine = state_machine_builder_for_nns_tests().build();

    let mut nns_init_payload_builder = NnsInitPayloadsBuilder::new();
    add_test_gtc_neurons(&mut nns_init_payload_builder);

    // Set the Genesis Moratorium to start now
    nns_init_payload_builder
        .genesis_token
        .genesis_timestamp_seconds = SystemTime::now().elapsed().unwrap().as_secs();

    let nns_init_payload = nns_init_payload_builder.build();

    setup_nns_canisters(&state_machine, nns_init_payload);

    let sign_cmd = move |msg: &[u8]| Ok(TEST_IDENTITY_1.sign(msg));
    let sender = Sender::ExternalHsm {
        pub_key: TEST_IDENTITY_1.public_key().serialize_der(),
        sign: Arc::new(sign_cmd),
    };

    // Assert `claim_neurons` fails during the moratorium
    let claim_neurons_response: Result<Vec<NeuronId>, String> =
        claim_neurons(&state_machine, sender.get_principal_id(), &TEST_IDENTITY_1);
    assert!(claim_neurons_response.is_err());

    // Assert that `TEST_IDENTITY_1` did not claim their neurons
    let account_has_claimed_neurons_response: Result<AccountState, String> =
        get_gtc_account(&state_machine, &TEST_IDENTITY_1);
    assert!(!account_has_claimed_neurons_response.unwrap().has_claimed);

    // Assert that `forward_all_unclaimed_accounts` fails
    let forward_all_unclaimed_accounts_response: Result<(), String> =
        forward_whitelisted_unclaimed_accounts(&state_machine, &sender);
    assert!(forward_all_unclaimed_accounts_response.is_err());
}
