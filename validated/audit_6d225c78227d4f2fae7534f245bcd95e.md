### Title
`native_per_pubdata` Rounds to Zero via Floor Division, Enabling Free Pubdata Generation — (`basic_bootloader/src/bootloader/transaction_flow/zk/validation_impl.rs`)

### Summary

When the operator-set `pubdata_price` is less than `native_price`, the `native_per_pubdata` calculation in L2 transaction validation truncates to zero via floor division. This causes the entire pubdata native-resource charge to collapse to zero, allowing any transaction sender to generate arbitrary amounts of pubdata (state diffs) without paying the corresponding native (proving) resource cost.

### Finding Description

In `validate_and_compute_fee_for_transaction`, the per-pubdata-byte native cost is computed as:

```rust
// basic_bootloader/src/bootloader/transaction_flow/zk/validation_impl.rs, line 142
let native_per_pubdata = u256_try_to_u64(&pubdata_price.wrapping_div(native_price))
    .ok_or(TxError::Validation(InvalidTransaction::PubdataPriceTooHigh))?;
```

`wrapping_div` is unsigned floor division. When `pubdata_price < native_price` (e.g., `pubdata_price = 2`, `native_price = 100`, as used in the existing test suite), the result is `0`. [1](#0-0) 

This zero propagates through every downstream pubdata charge:

**1. Intrinsic pubdata overhead in `create_resources_for_tx`:**
```rust
let intrinsic_pubdata_overhead = native_per_pubdata_byte.saturating_mul(intrinsic_pubdata);
// = 0 * intrinsic_pubdata = 0  → no native deducted for intrinsic pubdata
``` [2](#0-1) 

**2. Execution pubdata charge in `get_resources_to_charge_for_pubdata`:**
```rust
let native = current_pubdata_spent
    .checked_mul(native_per_pubdata)   // = anything * 0 = 0
    .ok_or(out_of_native_resources!())?;
``` [3](#0-2) 

**3. Post-execution pubdata sufficiency check in `check_enough_resources_for_pubdata`:**
```rust
let enough = resources.has_enough(&resources_for_pubdata);
// resources_for_pubdata = 0 → always true, regardless of pubdata generated
``` [4](#0-3) 

**Contrast with `native_per_gas`**, which correctly uses ceiling division to guarantee a non-zero result:
```rust
u256_try_to_u64(&gas_price.div_ceil(native_price))
``` [5](#0-4) 

The asymmetry is the root cause: `native_per_gas` uses `div_ceil` (never zero when `gas_price > 0`), while `native_per_pubdata` uses `wrapping_div` (zero whenever `pubdata_price < native_price`).

### Impact Explanation

When `native_per_pubdata == 0`:

- The native resource pool is never charged for pubdata, regardless of how many storage slots are written.
- A user is limited only by EVM gas (ergs), not by native resources, for pubdata generation.
- The operator must prove all generated state diffs at full RISC-V cycle cost, but collected zero native-resource compensation for that pubdata.
- A single transaction can fill a block's pubdata budget (bounded only by block gas limit) at zero native cost, forcing the operator to absorb the full proving overhead.
- Repeated across many transactions, this constitutes a sustained resource drain on the proving infrastructure.

### Likelihood Explanation

The condition `pubdata_price < native_price` is a realistic and tested operator configuration. The existing test suite explicitly uses `pubdata_price = U256::from(2)` with `native_price = U256::from(100)`: [6](#0-5) 

No on-chain or protocol-level constraint prevents the operator from deploying with `pubdata_price < native_price`. Any unprivileged transaction sender can exploit this condition the moment it holds, with no special setup beyond submitting a storage-write-heavy transaction.

### Recommendation

Replace `wrapping_div` with `div_ceil` for `native_per_pubdata`, mirroring the pattern already used for `native_per_gas`:

```rust
// Before (floor division — can produce 0):
let native_per_pubdata = u256_try_to_u64(&pubdata_price.wrapping_div(native_price))
    .ok_or(TxError::Validation(InvalidTransaction::PubdataPriceTooHigh))?;

// After (ceiling division — guarantees ≥ 1 when pubdata_price > 0):
let native_per_pubdata = u256_try_to_u64(&pubdata_price.div_ceil(native_price))
    .ok_or(TxError::Validation(InvalidTransaction::PubdataPriceTooHigh))?;
```

This ensures that even when `pubdata_price < native_price`, each byte of pubdata costs at least 1 native unit, preventing the rounding-to-zero bypass.

### Proof of Concept

1. Operator deploys a block with `pubdata_price = 1`, `native_price = 2` (a plausible low-pubdata-cost configuration).
2. `native_per_pubdata = 1 / 2 = 0` (floor division).
3. Attacker submits a transaction calling a contract that writes to 100 distinct storage slots (each write generates ~66 bytes of pubdata).
4. `get_resources_to_charge_for_pubdata` computes `native = 6600 * 0 = 0`.
5. `check_enough_resources_for_pubdata` returns `enough = true` unconditionally.
6. The transaction succeeds; the attacker pays only EVM gas for the `SSTORE` opcodes, zero native resources for ~6600 bytes of pubdata.
7. The operator must prove those 6600 bytes of state diffs at full RISC-V cycle cost with no native-resource compensation.
8. Repeating across many transactions exhausts the operator's proving budget without corresponding user payment.
