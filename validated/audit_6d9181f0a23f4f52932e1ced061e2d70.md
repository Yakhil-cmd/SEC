Audit Report

## Title
Hardcoded -5% Worst-Case Modulation in `spawn_neuron` Understates Mission 70's -10% Floor, Allowing Underfunded Spawned Neurons — (File: `rs/nns/governance/src/governance.rs`)

## Summary

The `spawn_neuron` function guards against spawning neurons below `neuron_minimum_stake_e8s` using a hardcoded -5% worst-case modulation factor. After Mission 70 (Proposal 141779), the actual execution path in `maybe_spawn_neurons` accepts and applies modulation values down to -10% (`MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 = -1_000`). A neuron controller can pass the spawn guard with maturity in the window `[minimum_stake / 0.95, minimum_stake / 0.90)` and have the spawned neuron minted with a stake below the protocol minimum.

## Finding Description

In `spawn_neuron`, the pre-spawn guard at `rs/nns/governance/src/governance.rs:2664-2673` computes:

```rust
let least_possible_stake = (maturity_to_spawn as f64 * (1_f64 - 0.05)) as u64;
if least_possible_stake < economics.neuron_minimum_stake_e8s { ... }
```

The literal `0.05` encodes the old CMC-based worst case of -500 bps (-5%).

After Mission 70, `VALID_MATURITY_MODULATION_BASIS_POINTS_RANGE` at `rs/nns/governance/src/governance.rs:276-278` is defined as:

```rust
const VALID_MATURITY_MODULATION_BASIS_POINTS_RANGE: RangeInclusive<i32> =
    MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 as i32   // -1_000
        ..=MATURITY_MODULATION_MAX_PERMYRIAD_MISSION_70 as i32; // 200
```

`MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 = -1_000` is defined at `rs/nns/governance/src/timer_tasks/update_icp_xdr_rate_related_data.rs:47`.

The execution path in `maybe_spawn_neurons` at `rs/nns/governance/src/governance.rs:6484-6488` calls `apply_maturity_modulation(original_maturity, maturity_modulation)` where `maturity_modulation` is validated against the Mission 70 range (accepting -10%). The `apply_maturity_modulation` function at `rs/nervous_system/governance/src/maturity_modulation/mod.rs:11-29` computes `amount * (10_000 + bps) / 10_000`, so at -1_000 bps it returns 90% of the input.

The guard and the execution path are now mismatched: the guard uses -5% but execution can apply -10%.

## Impact Explanation

This is a **Medium** impact governance invariant violation. The protocol invariant "every spawned neuron has at least `neuron_minimum_stake_e8s` ICP" is broken. The underfunded neuron cannot be split and its stake is below the floor assumed by governance economics. No ICP is lost from total supply (minting still occurs), but the governance state becomes inconsistent with its own `NetworkEconomics` parameters. This matches the allowed Medium impact: "moderate user-funds/security impact" under strict target conditions.

## Likelihood Explanation

**Low likelihood** with three required conditions:
1. Mission 70 maturity modulation must be in `(-1_000, -500]` bps (between -10% and -5%), requiring a sustained ICP price decline relative to the 365-day average.
2. A neuron controller must call `ManageNeuron::Spawn` with maturity in the narrow window `[minimum_stake / 0.95, minimum_stake / 0.90)`.
3. The spawning timer must fire while modulation remains in that range.

The window is narrow and market-condition-dependent, but reachable by any unprivileged neuron controller without special access.

## Recommendation

Replace the hardcoded `0.05` literal with the canonical Mission 70 minimum, mirroring the approach already used in SNS governance's `disburse_maturity` at `rs/sns/governance/src/governance.rs:1654-1655`:

```rust
use crate::timer_tasks::update_icp_xdr_rate_related_data::MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70;

let worst_case_bps = MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 as i32;
let least_possible_stake = apply_maturity_modulation(maturity_to_spawn, worst_case_bps)
    .unwrap_or(0);

if least_possible_stake < economics.neuron_minimum_stake_e8s {
    return Err(GovernanceError::new_with_message(
        ErrorType::InsufficientFunds,
        "There isn't enough maturity to spawn a new neuron due to worst case maturity modulation.",
    ));
}
```

## Proof of Concept

**Setup**: `neuron_minimum_stake_e8s = 100_000_000` (1 ICP default).

1. Controller calls `ManageNeuron::Spawn` with `maturity_to_spawn = 106_000_000`.
2. Guard: `106_000_000 * 0.95 = 100_700_000 ≥ 100_000_000` → spawn **allowed**.
3. `maybe_spawn_neurons` fires with `maturity_modulation = -1_000` (Mission 70 minimum).
4. `apply_maturity_modulation(106_000_000, -1_000)` = `106_000_000 * 9_000 / 10_000 = 95_400_000`.
5. Spawned neuron receives **95,400,000 e8s** — below `neuron_minimum_stake_e8s` of 100,000,000 e8s.

A deterministic integration test can reproduce this by: (a) setting `neuron_minimum_stake_e8s = 100_000_000`, (b) injecting a Mission 70 modulation of `-1_000`, (c) calling `spawn_neuron` with `maturity_to_spawn = 106_000_000`, (d) advancing the timer to trigger `maybe_spawn_neurons`, and (e) asserting the spawned neuron's cached stake is below the minimum.
