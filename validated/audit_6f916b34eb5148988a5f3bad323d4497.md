### Title
Zero-Gas Promise Silently Swallows `storage_deposit` Attached NEAR Deposit — (`engine/src/contract_methods/connector.rs`)

---

### Summary

`calculate_attached_gas` can return `NearGas::new(0)` when a caller provides a `prepaid_gas` value that leaves fewer than `GAS_FOR_PROMISE_CREATION` (2 TGas) remaining after the function body executes. `return_promise` unconditionally forwards this zero-gas value to the cross-contract call to `engine_storage_deposit`, causing that promise to fail. Because no failure callback is registered, the attached NEAR deposit is refunded to the Aurora engine contract rather than to the user, and the user's storage balance is never updated.

---

### Finding Description

`calculate_attached_gas` is defined as: [1](#0-0) 

```rust
// TODO: Return `Result` with an error about lacking of gas instead.
fn calculate_attached_gas<E: Env>(env: &E) -> NearGas {
    let required_gas = env.used_gas().saturating_add(GAS_FOR_PROMISE_CREATION);
    if required_gas >= env.prepaid_gas() {
        NearGas::new(0)
    } else {
        env.prepaid_gas() - required_gas
    }
}
```

`GAS_FOR_PROMISE_CREATION` is 2 TGas: [2](#0-1) 

The return value is used directly as `attached_gas` in `return_promise`, with no guard against zero: [3](#0-2) 

`storage_deposit` forwards the caller's full attached deposit to this promise: [4](#0-3) 

There is no `promise_then` callback registered anywhere in `return_promise` to handle a failed promise and refund the deposit to the original caller. [5](#0-4) 

---

### Impact Explanation

When `calculate_attached_gas` returns 0:

1. NEAR schedules the `engine_storage_deposit` cross-contract call with 0 gas.
2. The callee immediately fails (no gas to execute).
3. NEAR Protocol refunds the attached deposit to the **calling contract** (Aurora engine), not to the original user.
4. Aurora engine has no callback to relay this refund back to the user.
5. The user's storage balance on the connector is never updated.
6. The attached NEAR deposit is effectively frozen inside Aurora engine's account balance, inaccessible to the user.

For `storage_unregister` and `storage_withdraw`, the same zero-gas path exists, but both functions enforce `assert_one_yocto()`, so only 1 yocto NEAR is at risk per call — negligible financial impact. The critical case is `storage_deposit`, where the user can attach a meaningful NEAR deposit (e.g., the NEP-145 minimum storage balance). [6](#0-5) 

---

### Likelihood Explanation

The attack requires the caller to supply a `prepaid_gas` value in the window `[function_body_gas, function_body_gas + 2 TGas)`. This window is 2 TGas wide. The gas cost of the `storage_deposit` function body is deterministic for a given contract state (it performs two storage reads and one JSON serialization). An attacker can:

1. Call `storage_deposit` with max gas and observe `used_gas` at completion.
2. Subtract the gas consumed by the promise creation itself.
3. Re-submit with `prepaid_gas` set to `function_body_gas + 1 TGas` — inside the vulnerable window.

The developers themselves acknowledge this is a bug via the `// TODO: Return Result with an error about lacking of gas instead.` comment directly above `calculate_attached_gas`. [7](#0-6) 

---

### Recommendation

1. **Fix `calculate_attached_gas`**: Return `Result<NearGas, ContractError>` and propagate the error up through `return_promise` and all callers. This is exactly what the existing TODO comment prescribes.
2. **Add a failure callback**: In `return_promise`, register a `promise_then` callback that detects promise failure and refunds the attached deposit to the original predecessor. This is the correct defense-in-depth even if the gas check is added.

---

### Proof of Concept

```rust
// Unit test using Fixed env
use aurora_engine_sdk::env::Fixed;
use aurora_engine_types::types::NearGas;

// Measure function_body_gas empirically (e.g., 5 TGas for storage_deposit body).
// Set prepaid_gas = function_body_gas + 1 TGas  (inside the 2 TGas window).
let env = Fixed {
    prepaid_gas: NearGas::new(6_000_000_000_000),  // 6 TGas
    used_gas:    NearGas::new(5_000_000_000_000),  // 5 TGas consumed so far
    attached_deposit: 1_250_000_000_000_000_000_000, // 1.25 mNEAR
    ..Default::default()
};

// calculate_attached_gas:
//   required_gas = 5 TGas + 2 TGas = 7 TGas
//   7 TGas >= 6 TGas  =>  returns NearGas::new(0)
let gas = calculate_attached_gas(&env);
assert_eq!(gas, NearGas::new(0));

// Promise is created with 0 gas and 1.25 mNEAR attached.
// Promise fails on the connector side.
// Deposit is refunded to Aurora engine, NOT to the user.
// User's storage balance remains unregistered.
```
