Audit Report

## Title
Unchecked `u64` Multiplication Overflow in `merge_maturity` Causes Silent Maturity Destruction - (File: `rs/sns/governance/src/governance.rs`)

## Summary
The SNS governance `merge_maturity` function at line 1474–1475 performs an unchecked `u64` multiplication of `maturity_e8s_equivalent` by `percentage_to_merge`. In Rust release builds (the default for canister compilation), integer overflow wraps silently. No `overflow-checks = true` setting exists anywhere in the workspace `Cargo.toml` or crate-level `Cargo.toml`. When a neuron's maturity exceeds `u64::MAX / percentage_to_merge`, the multiplication wraps to a small value, the existing guard is bypassed, and the bulk of the maturity is permanently destroyed — neither credited to stake nor retained as maturity.

## Finding Description
At `rs/sns/governance/src/governance.rs` lines 1474–1475:

```rust
let mut maturity_to_merge =
    (neuron.maturity_e8s_equivalent * merge_maturity.percentage_to_merge as u64) / 100;
```

Both operands are `u64`. The guard at lines 1479–1481 only catches the case where overflow produces a value *larger* than `maturity_e8s_equivalent`; when overflow wraps to a *smaller* value (the typical case), the guard is bypassed entirely. The subsequent operations at lines 1515–1521 then deduct and credit only the small wrapped value, permanently destroying the difference. The NNS governance equivalent in `rs/nns/governance/src/governance/disburse_maturity.rs` lines 229–231 correctly widens to `u128` before multiplying, preventing this class of overflow entirely. ## Impact Explanation
A neuron holder with `maturity_e8s_equivalent` above the overflow threshold who calls `merge_maturity` will have the bulk of their maturity permanently destroyed. The maturity field is reduced by only the small wrapped value, and `cached_neuron_stake_e8s` is increased by only that same small value. The difference — potentially hundreds of millions of e8s — is neither in maturity nor in stake. This is a permanent, irreversible loss of user funds and a ledger conservation violation within the SNS framework, matching the allowed impact: **Significant SNS security impact with concrete user or protocol harm** (High, $2,000–$10,000), or Critical if the maturity lost exceeds $1M in value for high-supply SNS deployments. [2](#0-1) 

## Likelihood Explanation
The overflow threshold for `percentage_to_merge = 100` is `u64::MAX / 100 ≈ 1.84 × 10^17 e8s` (≈ 1.84 billion tokens at 8 decimals). SNS deployments with large total supplies can have neurons that accumulate maturity above this threshold over time. The call requires only `NeuronPermissionType::MergeMaturity` on the caller's own neuron — an unprivileged, standard user operation. No special access, governance majority, or external compromise is required. The operation is irreversible once executed. [4](#0-3) 

## Recommendation
Replace the unchecked `u64` multiplication with a `u128`-widened calculation, matching the NNS pattern already used in `percentage_of_maturity`:

```rust
let maturity_to_merge = u64::try_from(
    (neuron.maturity_e8s_equivalent as u128)
        .checked_mul(merge_maturity.percentage_to_merge as u128)
        .expect("overflow computing maturity_to_merge")
        / 100_u128,
).expect("maturity_to_merge exceeds u64::MAX");
``` [5](#0-4) 

## Proof of Concept
Set `maturity_e8s_equivalent = 2_000_000_000_00000000u64` (2×10^17 e8s) and `percentage_to_merge = 100`.

1. `2e17 * 100 = 2e19`. `u64::MAX ≈ 1.844e19`. Wraps to `2e19 - 2^64 ≈ 1.553e18`.
2. `maturity_to_merge = 1.553e18 / 100 = 1.553e16`.
3. Guard: `1.553e16 > 2e17`? No — bypassed.
4. `maturity_e8s_equivalent` reduced by `1.553e16` (not `2e17`).
5. `cached_neuron_stake_e8s` increased by `1.553e16` (not `2e17`).
6. Net permanent loss: ≈ `1.845e17 e8s`.

A unit test can reproduce this by constructing a `Neuron` with the above maturity value, calling `merge_maturity` with `percentage_to_merge = 100` in a mock governance environment, and asserting that `merged_maturity_e8s == 2e17` — which will fail, demonstrating the bug. [6](#0-5)

    pub async fn merge_maturity(
        &mut self,
        id: &NeuronId,
        caller: &PrincipalId,
        merge_maturity: &manage_neuron::MergeMaturity,
    ) -> Result<MergeMaturityResponse, GovernanceError> {
        let now = self.env.now();

        let neuron = self.get_neuron_result(id)?.clone();

        neuron.check_authorized(caller, NeuronPermissionType::MergeMaturity)?;

        if merge_maturity.percentage_to_merge > 100 || merge_maturity.percentage_to_merge == 0 {
            return Err(GovernanceError::new_with_message(
                ErrorType::PreconditionFailed,
                "The percentage of maturity to merge must be a value between 1 and 100 (inclusive).",
            ));
        }

        let transaction_fee_e8s = self.transaction_fee_e8s_or_panic();

        let mut maturity_to_merge =
            (neuron.maturity_e8s_equivalent * merge_maturity.percentage_to_merge as u64) / 100;

        // Converting u64 to f64 can cause the u64 to be "rounded up", so we
        // need to account for this possibility.
        if maturity_to_merge > neuron.maturity_e8s_equivalent {
            maturity_to_merge = neuron.maturity_e8s_equivalent;
        }

        if maturity_to_merge <= transaction_fee_e8s {
            return Err(GovernanceError::new_with_message(
                ErrorType::PreconditionFailed,
                format!(
                    "Tried to merge {maturity_to_merge} e8s, but can't merge an amount less than the transaction fee of {transaction_fee_e8s} e8s"
                ),
            ));
        }

        let nid = neuron.id.as_ref().expect("Neurons must have an id");
        let subaccount = neuron.subaccount()?;

        // Do the transfer, this is a minting transfer, from the governance canister's
        // (which is also the minting canister) main account into the neuron's
        // subaccount.
        #[rustfmt::skip]
        let _block_height: u64 = self
            .ledger
            .transfer_funds(
                maturity_to_merge,
                0, // Minting transfer don't pay a fee
                None, // This is a minting transfer, no 'from' account is needed
                self.neuron_account_id(subaccount), // The account of the neuron on the ledger
                self.env.insecure_random_u64(), // Random memo(nonce) for the ledger's transaction
            )
            .await?;

        // Adjust the maturity, stake, and age of the neuron
        let neuron = self
            .get_neuron_result_mut(nid)
            .expect("Expected the neuron to exist");

        neuron.maturity_e8s_equivalent = neuron
            .maturity_e8s_equivalent
            .saturating_sub(maturity_to_merge);
        let new_stake = neuron
            .cached_neuron_stake_e8s
            .saturating_add(maturity_to_merge);
        neuron.update_stake(new_stake, now);
        let new_stake_e8s = neuron.cached_neuron_stake_e8s;

        Ok(MergeMaturityResponse {
            merged_maturity_e8s: maturity_to_merge,
            new_stake_e8s,
        })
    }

fn percentage_of_maturity(
    total_maturity_e8s: u64,
    percentage_to_disburse: u32,
) -> Result<u64, InitiateMaturityDisbursementError> {
    (total_maturity_e8s as u128)
        .checked_mul(percentage_to_disburse as u128)
        .and_then(|result| result.checked_div(100))
        .and_then(|result| {
            // This should be impossible as long as `percentage_to_disburse` is between 0 and 100.
            if result > u64::MAX as u128 {
                None
            } else {
                Some(result as u64)
            }
        })
        .ok_or_else(|| InitiateMaturityDisbursementError::Unknown {
            reason: format!(
                "Failed to calculate percentage of maturity: {percentage_to_disburse}% of {total_maturity_e8s} e8s"
            ),
        })
}
