### Title
Silo Mode Fixed Gas Fee Fully Bypassed via Zero `max_fee_per_gas` - (`engine/src/engine.rs`)

### Summary

In Silo mode, the fixed gas fee is computed as `fixed_gas * effective_gas_price`. Because `effective_gas_price` is derived entirely from user-controlled transaction fields (`max_fee_per_gas`, `max_priority_fee_per_gas`) and Aurora's `block_base_fee_per_gas` is always zero, any whitelisted user can set `max_fee_per_gas = 0` to make `effective_gas_price = 0`, resulting in `prepaid_amount = 0`. The Silo operator collects no fee revenue despite having configured a mandatory fixed gas cost.

### Finding Description

The `charge_gas` function in `engine/src/engine.rs` contains an early-return guard:

```rust
if transaction.max_fee_per_gas.is_zero()
    && fixed_gas.is_none()
    && block_base_fee_per_gas.is_zero()
{
    return Ok(GasPaymentResult::default());
}
``` [1](#0-0) 

This guard correctly skips fee charging when there is no Silo fixed gas and no base fee. However, when Silo mode is active (`fixed_gas = Some(...)`), the guard is not triggered and execution falls through to:

```rust
let effective_gas_price = priority_fee_per_gas + block_base_fee_per_gas;
let prepaid_amount = fixed_gas
    .map_or(transaction.gas_limit, EthGas::as_u256)
    .checked_mul(effective_gas_price)
    .map(Wei::new)
    .ok_or(GasPaymentError::EthAmountOverflow)?;
``` [2](#0-1) 

Since Aurora has no EIP-1559 base fee market, `block_base_fee_per_gas` is always zero. The `priority_fee_per_gas` is computed as:

```rust
let priority_fee_per_gas = transaction
    .max_priority_fee_per_gas
    .min(transaction.max_fee_per_gas - block_base_fee_per_gas);
``` [3](#0-2) 

When a user sets `max_fee_per_gas = 0`, `priority_fee_per_gas = min(anything, 0) = 0`, so `effective_gas_price = 0`, and therefore `prepaid_amount = fixed_gas * 0 = 0`. The user pays nothing.

The `submit_with_alt_modexp` function retrieves `fixed_gas` from storage and passes it to `charge_gas`, but performs no validation that `max_fee_per_gas > 0` when Silo mode is active:

```rust
let fixed_gas = silo::get_fixed_gas(&io);
// ...
if fixed_gas.is_some_and(|gas| gas.as_u256() > transaction.gas_limit) {
    return Err(EngineErrorKind::FixedGasOverflow.into());
}
``` [4](#0-3) 

The only gas-related validation is that `fixed_gas <= gas_limit`. There is no floor on `max_fee_per_gas`. The `refund_unused_gas` function also short-circuits on `effective_gas_price == 0`, so the relayer receives nothing either:

```rust
if gas_result.effective_gas_price.is_zero() {
    return Ok(());
}
``` [5](#0-4) 

The Silo fee design, as shown in tests, expects `fee = fixed_gas * gas_price` to be deducted from the sender and credited to the relayer:

```rust
// INITIAL_BALANCE - TRANSFER_AMOUNT - FIXED_GAS * ONE_GAS_PRICE
validate_address_balance_and_nonce(&runner, sender, ...).unwrap();
// relayer receives FIXED_GAS * ONE_GAS_PRICE
validate_address_balance_and_nonce(&runner, relayer, FIXED_GAS * ONE_GAS_PRICE, ...).unwrap();
``` [6](#0-5) 

### Impact Explanation

**High — Theft of unclaimed yield.** In Silo mode, the fixed gas fee is the mechanism by which relayers are compensated for submitting transactions on behalf of users. By setting `max_fee_per_gas = 0`, any whitelisted user can execute an unlimited number of transactions — including failed ones — without paying any fee. The relayer address receives zero reward. The Silo operator loses all fee revenue that the fixed gas mechanism was designed to collect.

### Likelihood Explanation

**High.** The exploit requires only that the attacker be a whitelisted address in the Silo (or that whitelists are disabled). The attacker simply crafts a standard EIP-1559 transaction with `max_fee_per_gas = 0` and `max_priority_fee_per_gas = 0`. No special timing, no admin access, no external dependencies. The transaction is valid and accepted by the engine. The bypass is repeatable on every transaction.

### Recommendation

When `fixed_gas` is `Some(...)`, enforce a minimum `effective_gas_price > 0` before computing `prepaid_amount`. Concretely, in `charge_gas`, add a check after the existing early-return guard:

```rust
if fixed_gas.is_some() && effective_gas_price.is_zero() {
    return Err(GasPaymentError::MaxFeePerGasLessThanBaseFee);
}
```

Alternatively, decouple the fixed gas fee from the user-supplied gas price entirely: when `fixed_gas` is set, charge a flat amount (e.g., `fixed_gas * minimum_required_price`) that is not reducible to zero by the user.

### Proof of Concept

1. Silo operator enables Silo mode with `fixed_gas = 10^18` and `gas_price = 1` (as in the test constants).
2. Attacker (whitelisted) constructs a Legacy or EIP-1559 transaction with `max_fee_per_gas = 0`, `max_priority_fee_per_gas = 0`, `gas_limit >= fixed_gas`.
3. `submit` is called. `fixed_gas = Some(10^18)` is loaded from storage.
4. The `FixedGasOverflow` check passes because `gas_limit >= fixed_gas`.
5. `charge_gas` is called: `effective_gas_price = min(0, 0 - 0) + 0 = 0`.
6. `prepaid_amount = 10^18 * 0 = 0`. Sender balance is unchanged.
7. Transaction executes. `refund_unused_gas` returns early because `effective_gas_price == 0`.
8. Relayer receives zero. Attacker paid zero fees.
9. Repeat indefinitely — the Silo collects no revenue. [7](#0-6) [4](#0-3) [8](#0-7)

```rust
    pub fn charge_gas(
        &mut self,
        sender: &Address,
        transaction: &NormalizedEthTransaction,
        max_gas_price: Option<U256>,
        fixed_gas: Option<EthGas>,
    ) -> Result<GasPaymentResult, GasPaymentError> {
        let block_base_fee_per_gas = self.block_base_fee_per_gas();
        if transaction.max_fee_per_gas.is_zero()
            && fixed_gas.is_none()
            && block_base_fee_per_gas.is_zero()
        {
            return Ok(GasPaymentResult::default());
        }

        if transaction.max_fee_per_gas < block_base_fee_per_gas {
            return Err(GasPaymentError::MaxFeePerGasLessThanBaseFee);
        }

        let priority_fee_per_gas = transaction
            .max_priority_fee_per_gas
            .min(transaction.max_fee_per_gas - block_base_fee_per_gas);
        let priority_fee_per_gas = max_gas_price.map_or(priority_fee_per_gas, |price| {
            price.min(priority_fee_per_gas)
        });
        let effective_gas_price = priority_fee_per_gas + block_base_fee_per_gas;
        // First, we try to use `fixed_gas`. At this point we already know that the `fixed_gas` is
        // less than the `gas_limit`. It allows avoiding refunding unused gas to the sender later.
        let prepaid_amount = fixed_gas
            .map_or(transaction.gas_limit, EthGas::as_u256)
            .checked_mul(effective_gas_price)
            .map(Wei::new)
            .ok_or(GasPaymentError::EthAmountOverflow)?;

        let new_balance = get_balance(&self.io, sender)
            .checked_sub(prepaid_amount)
            .ok_or(GasPaymentError::OutOfFund)?;

        set_balance(&mut self.io, sender, &new_balance);

        self.gas_price = effective_gas_price;

        Ok(GasPaymentResult {
            prepaid_amount,
            effective_gas_price,
            priority_fee_per_gas,
        })
    }
