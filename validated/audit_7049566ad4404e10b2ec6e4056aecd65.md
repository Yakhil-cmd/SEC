### Title
Intrinsic pubdata overcharged for all EIP-7702 authorization entries regardless of delegation outcome - (File: `basic_bootloader/src/bootloader/transaction_flow/gas_helpers.rs`)

### Summary
`calculate_l2_tx_intrinsic_pubdata` charges `L2_TX_INTRINSIC_PUBDATA_PER_AUTHORIZATION` (80 bytes) for **every** authorization entry in an EIP-7702 transaction, using the raw count of entries. When a delegation fails, no state diff is written to L1, yet the user is still charged the full 80-byte pubdata cost per failed entry — an asymmetry between what is charged and what is actually published.

### Finding Description
In `gas_helpers.rs` lines 302–314, `calculate_l2_tx_intrinsic_pubdata` multiplies `authorization_list_num` (the total count of authorization entries, set at `validation_impl.rs` lines 158–166) by `L2_TX_INTRINSIC_PUBDATA_PER_AUTHORIZATION` (80 bytes) unconditionally:

```rust
intrinsic_pubdata = intrinsic_pubdata.saturating_add(
    authorization_list_num.saturating_mul(L2_TX_INTRINSIC_PUBDATA_PER_AUTHORIZATION),
);
```

The constant `L2_TX_INTRINSIC_PUBDATA_PER_AUTHORIZATION` is explicitly defined as the pubdata for a **successful** delegation — account key, metadata, nonce, balance, code length, artifacts length, padded bytecode, and observable length — totalling 80 bytes. It is not a processing overhead; it is the state-diff cost of a committed delegation.

In `validate_and_apply_delegation` (`authorization_list.rs`), a delegation silently fails and returns `Ok(false)` — writing **no state** — under any of these conditions:

- Wrong chain ID (line 108)
- Nonce overflow (`auth_nonce == u64::MAX`, line 112)
- High `s` value (line 120)
- `ecrecover` returns no result (lines 132–136)
- Authority is a contract (line 153)
- Authority nonce mismatch (line 157)

None of these failure paths produce a state diff, yet the intrinsic pubdata charged upfront in `create_resources_for_tx` (lines 351–359) already deducted `N × 80 × native_per_pubdata` from the user's native resource budget. In `before_refund` (mod.rs lines 401–412), `intrinsic_pubdata` is added to `total_pubdata_used` using the same full count, with no adjustment for failed entries:

```rust
let intrinsic_pubdata = calculate_l2_tx_intrinsic_pubdata(
    context.authorization_list_num,   // total entries, not successful ones
    transaction.is_service(),
);
// ...
(pubdata_used + intrinsic_pubdata, to_charge_for_pubdata)
```

There is no refund path for failed authorizations.
