### Title
EIP-7623 Floor Gas Premium Is Fully Refundable — Sender Recovers Non-Refundable Fee — (`engine/src/engine.rs`)

### Summary

`refund_unused_gas` computes the sender's refund as `prepaid_amount − gas_used × effective_gas_price`, where `gas_used` is the actual EVM execution gas. It has no knowledge of `floor_gas`. When `floor_gas > intrinsic_gas` and `gas_limit = floor_gas`, the sender is charged `floor_gas × gas_price` upfront but receives back `(floor_gas − gas_used) × gas_price` as a refund, fully recovering the floor-gas premium that EIP-7623 was designed to make non-refundable.

### Finding Description

`submit_with_alt_modexp` in `engine/src/engine.rs` computes both `intrinsic_gas` and `floor_gas`, enforces `gas_limit ≥ max(intrinsic_gas, floor_gas)`, and then calls `charge_gas` which deducts `gas_limit × effective_gas_price` from the sender. [1](#0-0) 

`charge_gas` uses `transaction.gas_limit` (not `floor_gas`) as the prepaid amount: [2](#0-1) 

After EVM execution, `refund_unused_gas` is called with `gas_used` (actual execution gas) but **without `floor_gas`**: [3](#0-2) 

Inside `refund_unused_gas`, the refund is computed purely as `prepaid_amount − gas_used × effective_gas_price`: [4](#0-3) 

The function signature confirms `floor_gas` is never threaded in: [5](#0-4) 

The floor_gas formula for 1000 non-zero bytes is confirmed by the existing test: [6](#0-5) 

### Impact Explanation

With 1000 non-zero bytes, `floor_gas = 61000` and `intrinsic_gas = 37000`. Setting `gas_limit = 61000`:

- Sender is charged `61000 × P` upfront
- EVM executes a simple call, `gas_used = 21000`
- Refund = `(61000 − 21000) × P = 40000 × P` → returned to sender
- Relayer receives only `21000 × priority_fee_per_gas`

The `40000 × P` floor-gas premium — which EIP-7623 mandates be non-refundable to compensate for calldata bandwidth cost — is fully recovered by the sender. This is **theft of unclaimed yield**: fees that should be retained by the protocol/relayer are instead returned to the attacker.

### Likelihood Explanation

- Requires only a valid signed legacy transaction with crafted calldata (many non-zero bytes) and `gas_limit = floor_gas`
- No special privileges, no admin access, no external dependencies
- Works on any Prague-config Aurora deployment (`CONFIG = &Config::prague()` is hardcoded at `engine/src/engine.rs:433`)
- Repeatable on every transaction; the attacker can systematically drain the floor-gas yield on every submission [7](#0-6) 

### Recommendation

Pass `floor_gas` into `refund_unused_gas` and use `max(gas_used, floor_gas)` as the minimum non-refundable gas when computing `spent_amount`. Concretely:

```rust
// In refund_unused_gas, replace:
let effective_gas_used = gas_used.into();
// with:
let effective_gas_used = U256::from(gas_used).max(U256::from(floor_gas));
```

This ensures the sender can never recover more than `(gas_limit − floor_gas) × gas_price` regardless of actual execution cost, which is the invariant EIP-7623 requires.

### Proof of Concept

```rust

// floor_gas  = 21000 + 4000*10 = 61000
// intrinsic  = 21000 + 16*1000 = 37000
// gas_limit  = 61000  (passes max(37000,61000) check)
// gas_price  = 1 wei

// charge_gas: prepaid = 61000 * 1 = 61000 wei
// EVM executes simple CALL to EOA: gas_used = 21000
// refund_unused_gas: refund = 61000 - 21000 = 40000 wei → sender
// relayer gets: 21000 * 1 = 21000 wei

// Assert: sender_balance_after == sender_balance_before - 21000 wei
//         (recovered 40000 wei of the 61000 wei floor-gas charge)
// Assert: relayer_balance_after == 21000 wei
//         (not 61000 wei as floor_gas intended)
```
