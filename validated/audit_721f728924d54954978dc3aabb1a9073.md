### Title
Single Canister Can Exhaust Subnet-Wide Canister HTTP Outcall Pool, DOSing All Other Canisters - (`rs/execution_environment/src/execution_environment.rs`)

### Summary
The subnet-wide canister HTTP outcall in-flight request pool (`canister_http_request_contexts`) is bounded by `MAX_CANISTER_HTTP_REQUESTS_IN_FLIGHT = 3000` but has **no per-canister quota**. A single malicious canister with sufficient cycles can fill all 3000 slots, preventing every other canister on the same subnet from making HTTP outcalls for as long as the attack is sustained.

### Finding Description

In `try_add_http_context_to_replicated_state`, the only guard before accepting a new HTTP outcall is a subnet-wide count check:

```rust
if state
    .metadata
    .subnet_call_context_manager
    .canister_http_request_contexts
    .len()
    >= self.config.max_canister_http_requests_in_flight
{
    return Err(UserError::new(
        ErrorCode::CanisterRejectedMessage,
        format!(
            "max number ({}) of http requests in-flight reached.",
            self.config.max_canister_http_requests_in_flight
        ),
    ));
}
``` [1](#0-0) 

The limit is defined as:

```rust
pub const MAX_CANISTER_HTTP_REQUESTS_IN_FLIGHT: usize = 3000;
``` [2](#0-1) 

There is no per-canister sub-limit, no per-sender quota, and no fairness mechanism. The check is purely on the total length of `canister_http_request_contexts` across the entire subnet. [3](#0-2) 

Requests time out and are removed from the pool after `CANISTER_HTTP_TIMEOUT_INTERVAL = 60 seconds`: [4](#0-3) 

An attacker who continuously re-submits 3000 requests every 60 seconds holds the pool full indefinitely.

### Impact Explanation

All canisters on the targeted subnet that call `ic00::HttpRequest` receive `CanisterRejectedMessage: "max number (3000) of http requests in-flight reached."` for the duration of the attack. Any protocol or application relying on canister HTTP outcalls (oracle feeds, cross-chain bridges, HTTPS API integrations) is completely blocked. This is a permanent, sustained availability failure of the canister HTTP feature for the entire subnet, directly analogous to the BribeRewarder DOS where a bounded per-pool slot list was filled with worthless entries.

### Likelihood Explanation

Any canister deployed on the target subnet can execute this attack. The only cost is cycles for the base HTTP request fee multiplied by 3000 slots, renewed every 60 seconds. Under pay-as-you-go pricing, only the non-refundable base fee is consumed per request. For a motivated attacker targeting a high-value subnet (e.g., one running DeFi protocols dependent on price oracles via HTTP outcalls), the economic cost is bounded and predictable. No privileged access, governance majority, or threshold corruption is required.

### Recommendation

Introduce a per-canister limit on the number of simultaneously in-flight HTTP outcall requests. For example, track the count of in-flight requests per `sender` canister ID within `canister_http_request_contexts` and reject new requests from a canister that has already reached its individual quota (e.g., `MAX_CANISTER_HTTP_REQUESTS_IN_FLIGHT / expected_active_canisters`). This mirrors the fix applied to the BribeRewarder vulnerability (a whitelist/per-actor cap), and is consistent with the existing `canister_guaranteed_callback_quota` pattern already used for general inter-canister callbacks. [5](#0-4) 

### Proof of Concept

1. Deploy a malicious canister `attacker` on any application subnet with HTTP outcalls enabled.
2. In a loop, have `attacker` call `ic00::HttpRequest` with a URL that will not respond quickly (e.g., a slow or non-existent endpoint), attaching the minimum required cycles.
3. After 3000 such calls are in flight, any other canister on the same subnet calling `ic00::HttpRequest` receives `CanisterRejectedMessage: "max number (3000) of http requests in-flight reached."`.
4. After 60 seconds the attacker's requests time out; the attacker immediately re-submits 3000 new requests, maintaining the DOS.
5. The existing test `http_request_bound_holds` in `rs/execution_environment/src/execution_environment/tests.rs` already demonstrates that the subnet-wide cap is enforced with no per-canister distinction — a single caller fills the entire pool. [6](#0-5)

#[test]
fn http_request_bound_holds() {
    let own_subnet = subnet_test_id(1);
    let caller_canister = canister_test_id(10);
    let mut test = ExecutionTestBuilder::new()
        .with_own_subnet_id(own_subnet)
        .with_caller(own_subnet, caller_canister)
        // set number of max in-flight calls to 10
        .with_max_canister_http_requests_in_flight(10)
        .build();
    test.state_mut().metadata.own_subnet_features.http_requests = true;

    // Create payload of the request.
    let url = "https://".to_string();
    let response_size_limit = 1000_u64;
    let transform_method_name = "transform".to_string();
    let transform_context = vec![0, 1, 2];
    let args = CanisterHttpRequestArgs {
        url: url.clone(),
        max_response_bytes: Some(response_size_limit),
        headers: BoundedHttpHeaders::new(vec![]),
        body: None,
        method: HttpMethod::GET,
        transform: Some(TransformContext {
            function: TransformFunc(candid::Func {
                principal: caller_canister.get().0,
                method: transform_method_name.clone(),
            }),
            context: transform_context.clone(),
        }),
        is_replicated: None,
        pricing_version: None,
    };

    // Create request to HTTP_REQUEST method.
    let payload = args.clone().encode();
    test.inject_call_to_ic00(Method::HttpRequest, payload, Cycles::new(1_000_000_000));
    test.execute_all();
    // Check that the SubnetCallContextManager contains the request.
    let canister_http_request_contexts = &test
        .state()
        .metadata
        .subnet_call_context_manager
        .canister_http_request_contexts;
    assert_eq!(canister_http_request_contexts.len(), 1);

    // Now we try to inject more than the maximum number of requests we allow to be in-flight

    for _ in 0..15 {
        let payload = args.clone().encode();
        test.inject_call_to_ic00(Method::HttpRequest, payload, Cycles::new(1_000_000_000));
    }

    test.execute_all();
    let canister_http_request_contexts = &test
        .state()
        .metadata
        .subnet_call_context_manager
        .canister_http_request_contexts;

    // Check that the SubnetCallContextManager contains the maximum number but not more
    assert_eq!(canister_http_request_contexts.len(), 10);
}
