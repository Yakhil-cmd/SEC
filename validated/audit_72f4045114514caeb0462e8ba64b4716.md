Audit Report

## Title
Stale 5% Worst-Case Modulation Constant in `spawn_neuron` Allows Below-Minimum-Stake Neuron Creation - (File: `rs/nns/governance/src/governance.rs`)

## Summary
`spawn_neuron` gates neuron spawning with a hardcoded 5% worst-case maturity modulation floor, but `maybe_spawn_neurons` now applies the Mission 70 modulation whose lower bound is −10% (−1,000 permyriad). Any neuron controller whose maturity falls in the gap between the two floors can pass the pre-check and have a child neuron minted with `cached_neuron_stake_e8s` below `neuron_minimum_stake_e8s`, permanently violating the NNS protocol invariant that every neuron holds at least the minimum stake.

## Finding Description
In `rs/nns/governance/src/governance.rs` at line 2666, `spawn_neuron` computes:

```rust
let least_possible_stake = (maturity_to_spawn as f64 * (1_f64 - 0.05)) as u64;
```

This hardcodes 5% as the worst-case modulation. The guard at line 2668 rejects the call only when this value is below `economics.neuron_minimum_stake_e8s`.

After the Mission 70 change (CHANGELOG proposal 141779), `maybe_spawn_neurons` reads `heap_data.maturity_modulation.current_value_permyriad` and validates it against `VALID_MATURITY_MODULATION_BASIS_POINTS_RANGE`, whose lower bound is `MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 = -1_000` (−10%), defined in `rs/nns/governance/src/timer_tasks/update_icp_xdr_rate_related_data.rs` at line 47. The actual stake minted is computed via `apply_maturity_modulation(original_maturity, maturity_modulation)` at line 6484, which at −1,000 permyriad yields `amount * 9_000 / 10_000 = 90%` of maturity.

The pre-check and execution path are therefore mismatched: the pre-check permits maturity `M` where `M * 0.95 >= neuron_minimum_stake_e8s`, but the execution path can produce a stake as low as `M * 0.90`. No post-spawn correction exists.

## Impact Explanation
A neuron controller (unprivileged ingress sender) can permanently create a child neuron whose `cached_neuron_stake_e8s` is below `neuron_minimum_stake_e8s`. With `neuron_minimum_stake_e8s = 100_000_000 e8s` (1 ICP), any maturity `M` in `[105_263_158, 111_111_111)` e8s passes the pre-check but produces a neuron with stake in `[94_736_842, 100_000_000)` e8s — below the protocol minimum. This is a concrete NNS protocol invariant violation with permanent on-chain effect, qualifying as a significant NNS security impact with concrete protocol harm (High, $2,000–$10,000).

## Likelihood Explanation
The Mission 70 modulation moves at most 30 permyriad/day. Reaching −500 permyriad (the old floor) takes approximately 17 days from zero; reaching −1,000 permyriad takes approximately 33 days. Once below −500 permyriad, any neuron controller with maturity in the ~5.5% vulnerable window can trigger the bug with a standard `manage_neuron` → `Spawn` ingress call. No privileged access is required. The condition is realistic for active NNS participants and repeatable for as long as the modulation remains below −500 permyriad.

## Recommendation
Replace the hardcoded `0.05` with a call to `apply_maturity_modulation` using the actual Mission 70 worst-case constant, mirroring the pattern already used in SNS `disburse_maturity`:

```rust
use ic_nervous_system_governance::maturity_modulation::apply_maturity_modulation;
// MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 = -1_000 permyriad = -10%
let least_possible_stake = apply_maturity_modulation(
    maturity_to_spawn,
    MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 as i32,
).unwrap_or(0);
```

This ensures the pre-check is always at least as conservative as the execution path.

## Proof of Concept
1. Wait for Mission 70 maturity modulation to reach −600 permyriad (achievable after ~20 days of sustained ICP price decline).
2. As a neuron controller with `maturity_e8s_equivalent = 106_000_000` e8s:
   - Pre-check: `106_000_000 * 0.95 = 100_700_000 >= 100_000_000` → **passes** (line 2668).
3. Call `manage_neuron` → `Spawn { percentage_to_spawn: 100, ... }`. Child neuron enters spawning state with `maturity_e8s_equivalent = 106_000_000`.
4. When `maybe_spawn_neurons` fires with modulation = −600 permyriad:
   - `apply_maturity_modulation(106_000_000, -600)` = `106_000_000 * 9_400 / 10_000 = 99_640_000`.
5. Child neuron is minted with `cached_neuron_stake_e8s = 99_640_000 < 100_000_000`, permanently violating the minimum-stake invariant.

A deterministic integration test can reproduce this by setting `heap_data.maturity_modulation.current_value_permyriad = -600`, creating a neuron with `maturity_e8s_equivalent = 106_000_000`, calling `spawn_neuron`, advancing time past the spawn delay, and asserting that `maybe_spawn_neurons` mints a neuron with stake below `neuron_minimum_stake_e8s`.
