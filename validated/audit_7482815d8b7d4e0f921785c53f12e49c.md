### Title
SNS `split_neuron` Does Not Proportionally Reduce Parent Maturity When Stake Decreases - (`rs/sns/governance/src/governance.rs`)

### Summary

The SNS `split_neuron` function reduces the parent neuron's `cached_neuron_stake_e8s` but does not proportionally reduce `maturity_e8s_equivalent` or `staked_maturity_e8s_equivalent`. The parent retains 100% of its accumulated maturity regardless of how much stake is split away. This is the direct analog of the Beanstalk bug: a value that should decrease proportionally with the underlying asset (stake) does not.

### Finding Description

In `rs/sns/governance/src/governance.rs`, `split_neuron` creates the child neuron with zero maturity and zero staked maturity, and only adjusts `cached_neuron_stake_e8s` on both sides:

```rust
let child_neuron = Neuron {
    ...
    maturity_e8s_equivalent: 0,          // line 1373
    staked_maturity_e8s_equivalent: None, // line 1377
    ...
};
```

After the ledger transfer succeeds:

```rust
parent_neuron.cached_neuron_stake_e8s -= split.amount_e8s;  // line 1429
child_neuron.cached_neuron_stake_e8s = staked_amount;        // line 1435
```

No adjustment is made to `maturity_e8s_equivalent` or `staked_maturity_e8s_equivalent` of the parent. [1](#0-0) [2](#0-1) 

This is confirmed by the unit test, which explicitly asserts the parent keeps all maturity after a split:

```rust
assert_eq!(parent_neuron.maturity_e8s_equivalent, maturity_e8s); // parent keeps ALL
assert_eq!(child_neuron.maturity_e8s_equivalent, 0);             // child gets 0
``` [3](#0-2) 

Contrast this with the NNS `split_neuron`, which uses `calculate_split_neuron_effect` to proportionally transfer maturity, staked maturity, and the eight-year-gang bonus base to the child: [4](#0-3) [5](#0-4) 

The NNS correctly computes `transfer_maturity_e8s = source_maturity * split_amount / source_stake` and deducts it from the parent. The SNS does not perform this step at all.

The SNS neuron's voting power stake is `cached_neuron_stake_e8s - neuron_fees_e8s + staked_maturity_e8s_equivalent`: [6](#0-5) 

And maturity can be staked via `StakeMaturity` to directly increase `staked_maturity_e8s_equivalent`, which feeds into voting power: [7](#0-6) 

### Impact Explanation

An attacker can:

1. Stake a large amount of SNS tokens and accumulate significant `maturity_e8s_equivalent` through voting rewards.
2. Call `split_neuron` to move nearly all of the stake to a child neuron (parent retains only the minimum stake, child gets the bulk).
3. The parent retains 100% of the accumulated maturity.
4. Call `StakeMaturity` on the parent to convert all maturity into `staked_maturity_e8s_equivalent`, which directly boosts voting power.
5. Disburse the child neuron's stake to recover the tokens.

**Net result:** The attacker holds disproportionate SNS voting power backed by only a small locked stake, while having recovered most of their capital. This inflates their governance influence and voting reward share at the expense of honest participants. The attack is repeatable.

### Likelihood Explanation

Any SNS neuron holder with `NeuronPermissionType::Split` (the default) can execute this. No privileged access is required. The entry path is a standard `manage_neuron` ingress call to the SNS governance canister. The attacker only needs to have accumulated maturity, which happens naturally through normal participation. [8](#0-7) 

### Recommendation

Apply the same proportional maturity transfer logic used by NNS. After the ledger transfer succeeds, compute the fraction of maturity to transfer as `maturity * split_amount / parent_stake_before_split` and deduct it from the parent, crediting it to the child. The NNS helper `calculate_split_neuron_effect` can serve as a direct template.

```rust
// After successful transfer:
let transfer_maturity = (parent_maturity as u128 * split.amount_e8s as u128)
    / parent_stake_before_split as u128;
parent_neuron.maturity_e8s_equivalent -= transfer_maturity as u64;
child_neuron.maturity_e8s_equivalent = transfer_maturity as u64;

// Similarly for staked_maturity_e8s_equivalent
```

### Proof of Concept

1. Attacker stakes 1,000,000 SNS tokens. Over time, accumulates `maturity_e8s_equivalent = 500,000`.
2. Calls `split_neuron` with `amount_e8s = 900,000` (leaving minimum stake in parent).
3. Parent state: `cached_neuron_stake_e8s = 100,000`, `maturity_e8s_equivalent = 500,000` (unchanged).
4. Calls `StakeMaturity { percentage_to_stake: 100 }` on parent.
5. Parent state: `cached_neuron_stake_e8s = 100,000`, `staked_maturity_e8s_equivalent = 500,000`, voting power stake = 600,000.
6. Calls `Disburse` on child neuron, recovering ~900,000 tokens.
7. Net: Attacker has 600,000 units of voting power from only 100,000 tokens of locked stake, with 900,000 tokens liquid — a 6x leverage on governance influence.

    split_amount_e8s: u64,
    source_neuron_stake_e8s: u64,
    source_neuron_maturity_e8s: u64,
    source_neuron_staked_maturity_e8s: u64,
    source_neuron_eight_year_gang_bonus_base_e8s: u64,
) -> SplitNeuronEffect {
    let transfer_maturity_e8s: u64 = (source_neuron_maturity_e8s as u128)
        .checked_mul(split_amount_e8s as u128)
        .expect("Two u64s can't overflow when multiplied")
        .checked_div(source_neuron_stake_e8s as u128)
        .expect("The input source_neuron_stake_e8s should be greater than zero")
        .try_into()
        .expect("The result should be smaller than source_neuron_maturity_e8s which should fit into u64");

    let transfer_staked_maturity_e8s: u64 = (source_neuron_staked_maturity_e8s as u128)
        .checked_mul(split_amount_e8s as u128)
        .expect("Two u64s can't overflow when multiplied")
        .checked_div(source_neuron_stake_e8s as u128)
        .expect("The input source_neuron_stake_e8s should be greater than zero")
        .try_into()
        .expect("The result should be smaller than source_neuron_staked_maturity_e8s which should fit into u64");
    let transfer_eight_year_gang_bonus_base_e8s = (source_neuron_eight_year_gang_bonus_base_e8s
        as u128)
        .checked_mul(split_amount_e8s as u128)
        .expect("Two u64s can't overflow when multiplied")
        .checked_div(source_neuron_stake_e8s as u128)
        .expect("The input source_neuron_stake_e8s should be greater than zero")
        .try_into()
        .expect("The result should be smaller than source_neuron_eight_year_gang_bonus_base_e8s which should fit into u64");

    SplitNeuronEffect {
        transfer_maturity_e8s,
        transfer_staked_maturity_e8s,
        transfer_eight_year_gang_bonus_base_e8s,
