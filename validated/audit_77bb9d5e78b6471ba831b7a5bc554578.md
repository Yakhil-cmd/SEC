### Title
Permissionless `ClaimOrRefresh` Allows Any Caller to Permanently Dilute Victim Neuron's Age Bonus - (File: `rs/nns/governance/src/governance.rs`)

### Summary

Any unprivileged ingress sender can permanently reduce the `aging_since_timestamp_seconds` of any NNS neuron by depositing a small amount of ICP into the victim's neuron subaccount and then calling `manage_neuron` with `ClaimOrRefresh`. The `refresh_neuron` function performs no authorization check, allowing an attacker to irreversibly dilute a victim's neuron age and thus their voting power bonus and voting rewards.

### Finding Description

The NNS Governance canister exposes `manage_neuron` with a `ClaimOrRefresh` command that accepts either `By::MemoAndController` (specifying any controller principal) or `By::NeuronIdOrSubaccount` (specifying any neuron ID directly). Both paths ultimately call `refresh_neuron`, which contains **no authorization check**:

```rust

    &mut self,
    nid: NeuronId,
    subaccount: Subaccount,
    claim_or_refresh: &ClaimOrRefresh,
) -> Result<NeuronId, GovernanceError> {
    // No caller identity check here
    let balance = self.ledger.account_balance(account).await?;
    ...
    self.with_neuron_mut(&nid, |neuron| {
        match neuron.cached_neuron_stake_e8s.cmp(&balance.get_e8s()) {
            Ordering::Less => {
                neuron.update_stake_adjust_age(balance.get_e8s(), now);
            }
            ...
        };
    })?;

When the ledger balance exceeds the cached stake (i.e., someone deposited ICP into the neuron's subaccount), `update_stake_adjust_age` is called. This function computes a weighted-average age, where the newly added stake has age 0, permanently diluting the neuron's `aging_since_timestamp_seconds`:

```rust
let (new_stake_e8s, new_age_seconds) = combine_aged_stakes(
    self.cached_neuron_stake_e8s,
    self.age_seconds(now),
    updated_stake_e8s.saturating_sub(self.cached_neuron_stake_e8s),
    0,  // new stake has age 0 — dilutes existing age
);
let new_aging_since_timestamp_seconds = now.saturating_sub(new_age_seconds);
``` [2](#0-1)

### Impact Explanation

An attacker can permanently reduce any NNS neuron's age bonus (up to 1.25x) and thus its voting power and voting rewards. The dilution is irreversible without a governance-level intervention (as seen in the `ResetAging`/`RestoreAging` audit events that were required historically for similar age corruption issues). [7](#0-6) 

For a neuron with stake `S` aged `A` years, depositing `δ` ICP reduces the effective age to `A * S / (S + δ)`. For small neurons (e.g., 1 ICP staked), depositing 1 ICP halves the age bonus in a single transaction. The attack is repeatable, driving the age toward zero over time. The victim suffers reduced voting power and reduced voting rewards with no recourse.

### Likelihood Explanation

The attack entry path is a standard ingress call to the publicly exposed `manage_neuron` endpoint on the NNS Governance canister. No privileged role is required. The attacker only needs:
1. The victim's neuron ID (publicly queryable via `get_neuron_info`)
2. Enough ICP to deposit into the victim's neuron subaccount (minimum 1 e8 = 0.00000001 ICP, though the ICP ledger transfer fee of 10,000 e8s is the practical floor)

The deposited ICP remains in the victim's neuron (increasing their stake), so the attacker does not recover it, but the cost is proportional to the desired damage and is very low for small neurons.

### Recommendation

Add an authorization check in `refresh_neuron` (or in `refresh_neuron_by_id_or_subaccount`) to verify that the caller is the neuron's controller or a registered hot key before allowing `update_stake_adjust_age` to modify `aging_since_timestamp_seconds`. Alternatively, decouple the age adjustment from the permissionless stake-sync path: allow anyone to notify governance of a balance increase (updating `cached_neuron_stake_e8s`) but restrict the age recalculation to authorized callers only.

### Proof of Concept

1. Victim holds neuron `N` with 100 ICP staked for 4 years (`aging_since_timestamp_seconds = now - 4*365*86400`), giving a 1.25x age bonus.
2. Attacker transfers 1 ICP to the victim's neuron subaccount on the ICP ledger (cost: 1 ICP + 10,000 e8s fee).
3. Attacker calls `manage_neuron` as any principal:
   ```
   ManageNeuron {
     neuron_id_or_subaccount: Some(NeuronIdOrSubaccount::NeuronId(N)),
     command: Some(Command::ClaimOrRefresh(ClaimOrRefresh {
       by: Some(By::NeuronIdOrSubaccount(Empty {}))
     }))
   }
   ```
4. `refresh_neuron` reads ledger balance = 101 ICP > cached 100 ICP, calls `update_stake_adjust_age(101 ICP, now)`.
5. New age = `4 years * 100 / 101 ≈ 3.96 years`. The `aging_since_timestamp_seconds` is permanently advanced.
6. Attacker repeats steps 2–5 to continuously dilute the age toward zero, eliminating the victim's age bonus entirely. [8](#0-7) [9](#0-8)

    async fn refresh_neuron(
        &mut self,
        nid: NeuronId,
        subaccount: Subaccount,
        claim_or_refresh: &ClaimOrRefresh,
    ) -> Result<NeuronId, GovernanceError> {
        let account = neuron_subaccount(subaccount);
        // We need to lock the neuron to make sure it doesn't undergo
        // concurrent changes while we're checking the balance and
        // refreshing the stake.
        let now = self.env.now();
        let _neuron_lock = self.lock_neuron_for_command(
            nid.id,
            NeuronInFlightCommand {
                timestamp: now,
                command: Some(InFlightCommand::ClaimOrRefreshNeuron(
                    claim_or_refresh.clone(),
                )),
            },
        )?;

        // Get the balance of the neuron from the ledger canister.
        tla_log_locals! { neuron_id: nid.id };
        let balance = self.ledger.account_balance(account).await?;
        let min_stake = self.economics().neuron_minimum_stake_e8s;
        if balance.get_e8s() < min_stake {
            return Err(GovernanceError::new_with_message(
                ErrorType::InsufficientFunds,
                format!(
                    "Account does not have enough funds to refresh a neuron. \
                     Please make sure that account has at least {:?} e8s (was {:?} e8s)",
                    min_stake,
                    balance.get_e8s()
                ),
            ));
        }
        self.with_neuron_mut(&nid, |neuron| {
            match neuron.cached_neuron_stake_e8s.cmp(&balance.get_e8s()) {
                Ordering::Greater => {
                    println!(
                        "{}ERROR. Neuron cached stake was inconsistent.\
                     Neuron account: {} has less e8s: {} than the cached neuron stake: {}.\
                     Stake adjusted.",
                        LOG_PREFIX,
                        account,
                        balance.get_e8s(),
                        neuron.cached_neuron_stake_e8s
                    );
                    neuron.update_stake_adjust_age(balance.get_e8s(), now);
                }
                Ordering::Less => {
                    neuron.update_stake_adjust_age(balance.get_e8s(), now);
                }
                // If the stake is the same as the account balance,
                // just return the neuron id (this way this method
                // also serves the purpose of allowing to discover the
                // neuron id based on the memo and the controller).
                Ordering::Equal => (),
            };
        })?;

        Ok(nid)
    }

    pub fn update_stake_adjust_age(&mut self, updated_stake_e8s: u64, now: u64) {
        // If the updated stake is less than the original stake, preserve the
        // age and distribute it over the new amount. This should not happen
        // in practice, so this code exists merely as a defensive fallback.
        //
        // TODO(NNS1-954) Consider whether update_stake_adjust_age (and other
        // similar methods) should use a neurons effective stake rather than
        // the cached stake.
        if updated_stake_e8s < self.cached_neuron_stake_e8s {
            println!(
                "{}Reducing neuron {:?} stake via update_stake_adjust_age: {} -> {}",
                LOG_PREFIX,
                self.id(),
                self.cached_neuron_stake_e8s,
                updated_stake_e8s
            );
            self.cached_neuron_stake_e8s = updated_stake_e8s;
        } else {
            // If one looks at "stake * age" as describing an area, the goal
            // at this point is to increase the stake while keeping the area
            // constant. This means decreasing the age in proportion to the
            // additional stake, which is the purpose of combine_aged_stakes.
            let (new_stake_e8s, new_age_seconds) = combine_aged_stakes(
                self.cached_neuron_stake_e8s,
                self.age_seconds(now),
                updated_stake_e8s.saturating_sub(self.cached_neuron_stake_e8s),
                0,
            );
            // A consequence of the math above is that the 'new_stake_e8s' is
            // always the same as the 'updated_stake_e8s'. We use
            // 'combine_aged_stakes' here to make sure the age is
            // appropriately pro-rated to accommodate the new stake.
            assert!(new_stake_e8s == updated_stake_e8s);
            self.cached_neuron_stake_e8s = new_stake_e8s;

            let new_aging_since_timestamp_seconds = now.saturating_sub(new_age_seconds);
            let new_disolved_dissolve_state_and_age = self
                .dissolve_state_and_age()
                .adjust_age(new_aging_since_timestamp_seconds);
            self.set_dissolve_state_and_age(new_disolved_dissolve_state_and_age);
        }
    }

pub(crate) fn age_bonus_multiplier(age_seconds: u64) -> Decimal {
    let age_seconds = Decimal::from(age_seconds.clamp(0, MAX_NEURON_AGE_FOR_AGE_BONUS));

    // t is (clamped) age in units of max age, so its value is from 0.0 to 1.0
    let t = age_seconds / Decimal::from(MAX_NEURON_AGE_FOR_AGE_BONUS);

    // 0.25 * t + 1
    t / Decimal::from(4) + Decimal::from(1)
}
