### Title
`ExitToNear` Omni Path Attributes Token Positions to Aurora Engine Instead of User, Causing Permanent Fund Freeze — (`engine-precompiles/src/native.rs`)

---

### Summary

When an EVM user exits ERC-20 (or base ETH) tokens to a NEAR DeFi protocol via the `ExitToNear` precompile using the Omni message path, the resulting `ft_transfer_call` is issued from the Aurora engine account (`aurora`). The target NEAR contract's `ft_on_transfer` callback receives `sender_id = aurora`, not the original EVM user's identity. Any NEAR protocol that uses `sender_id` to attribute positions (collateral, debt, LP shares, etc.) will permanently lock the user's funds under the engine's account, with no user-accessible recovery path.

---

### Finding Description

The `ExitToNear` precompile handles ERC-20 token exits in `exit_erc20_token_to_near`. When the user appends a `:` message (the Omni path), the function constructs an `ft_transfer_call` promise targeting the user-specified NEAR protocol: [1](#0-0) 

This promise is created by the Aurora engine contract itself. Under NEAR's cross-contract call semantics, the `predecessor_account_id` (i.e., `sender_id`) seen by the target contract's `ft_on_transfer` callback is `aurora` — the engine account — not the originating EVM user.

The promise is then assembled and emitted as a log: [2](#0-1) 

The same structural issue exists for the base ETH token Omni path: [3](#0-2) 

The XCC system correctly avoids this by deploying a **per-user** router sub-account (`{evm_address}.aurora`) so that each NEAR cross-contract call carries the user's unique identity as `predecessor_account_id`: [4](#0-3) 

The `ExitToNear` Omni path has no equivalent per-user identity mechanism.

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

1. The user's ERC-20 tokens are **burned** inside the EVM (irreversible).
2. The corresponding NEP-141 tokens are transferred to the target NEAR protocol via `ft_transfer_call`.
3. The target protocol's `ft_on_transfer` records the deposit under `sender_id = aurora`.
4. The user has no access to the `aurora` account on the target protocol and cannot withdraw, claim, or liquidate the position.
5. The Aurora engine admin would need to manually intervene on the target protocol — this is not guaranteed and is outside the user's control.

Any NEAR DeFi protocol that tracks user positions by `sender_id` (the standard pattern in NEP-141 `ft_on_transfer`) is affected. This is not a hypothetical edge case; it is the standard behavior of NEAR fungible token protocols.

---

### Likelihood Explanation

**High.** The Omni path is explicitly designed for interacting with NEAR DeFi protocols (the `:msg` suffix is the documented mechanism for this). Any user who attempts to deposit ERC-20 tokens into a NEAR lending, AMM, or staking protocol via the `ExitToNear` precompile Omni path will trigger this bug. No special attacker knowledge is required — a normal user following the intended workflow is sufficient to permanently lose funds.

---

### Recommendation

1. **Document the invariant break**: Clearly state that `sender_id` in `ft_on_transfer` will always be `aurora`, not the user, and that protocols relying on `sender_id` for position attribution are incompatible with this path.
2. **Encode user identity in `msg`**: Require or recommend that the Omni `msg` field encodes the originating EVM address so that target protocols can attribute positions correctly (requires protocol-side support).
3. **Per-user exit routing**: Mirror the XCC router pattern — route `ft_transfer_call` through a per-user sub-account so that `predecessor_account_id` carries the user's identity, not the engine's.

---

### Proof of Concept

**Entry path** (unprivileged EVM user):

1. User holds ERC-20 tokens on Aurora (e.g., a NEP-141-backed token).
2. User calls the `ExitToNear` precompile at `0xe9217bc70b7ed1f598ddd3199e80b093fa71124f` with:
   - `flag = 0x01` (ERC-20 path)
   - `amount = N`
   - `receiver_account_id = some-near-defi.near`
   - `message = ":deposit"` (any non-`unwrap` colon-prefixed message triggers Omni)
3. The precompile burns `N` ERC-20 tokens from the user's EVM balance.
4. Aurora engine issues `ft_transfer_call` to the NEP-141 contract with `receiver_id = some-near-defi.near` and `msg = "deposit"`.
5. `some-near-defi.near`'s `ft_on_transfer(sender_id: "aurora", amount: N, msg: "deposit")` is called.
6. The protocol records a deposit of `N` tokens under account `aurora`.
7. The user's ERC-20 tokens are gone. The NEP-141 tokens are locked in `some-near-defi.near` under `aurora`'s account. The user has no access. [5](#0-4) [6](#0-5)

impl<I: IO> Precompile for ExitToNear<I> {
    fn required_gas(_input: &[u8]) -> Result<EthGas, ExitError> {
        Ok(costs::EXIT_TO_NEAR_GAS)
    }

    #[allow(clippy::too_many_lines)]
    fn run(
        &self,
        input: &[u8],
        target_gas: Option<EthGas>,
        context: &Context,
        is_static: bool,
    ) -> EvmPrecompileResult {
        // ETH (base) transfer input format: (85 bytes)
        //  - flag (1 byte)
        //  - refund_address (20 bytes), present if the feature "error_refund" is enabled
        //  - recipient_account_id (max MAX_INPUT_SIZE - 20 - 1 bytes)
        // ERC-20 transfer input format: (124 bytes)
        //  - flag (1 byte)
        //  - refund_address (20 bytes), present if the feature "error_refund" is enabled.
        //  - amount (32 bytes)
        //  - recipient_account_id (max MAX_INPUT_SIZE - 1 - (20) - 32 bytes)
        //  - `:unwrap` suffix in a case of wNEAR (7 bytes)
        let required_gas = Self::required_gas(input)?;

        if let Some(target_gas) = target_gas
            && required_gas > target_gas
        {
            return Err(ExitError::OutOfGas);
        }

        // It's not allowed to call exit precompiles in static mode
        if is_static {
            return Err(ExitError::Other(Cow::from("ERR_INVALID_IN_STATIC")));
        } else if context.address != exit_to_near::ADDRESS.raw() {
            return Err(ExitError::Other(Cow::from("ERR_INVALID_IN_DELEGATE")));
        }

        let exit_to_near_params = ExitToNearParams::try_from(input)?;

        let (nep141_address, args, exit_event, method, transfer_near_args) =
            match exit_to_near_params {
                // ETH(base) token transfer
                //
                // Input slice format:
                //  recipient_account_id (bytes) - the NEAR recipient account which will receive
                //  NEP-141 (base) tokens, or also can contain the `:unwrap` suffix in case of
                //  withdrawing wNEAR, or another message of JSON in case of OMNI, or address of
                //  receiver in case of transfer tokens to another engine contract.
                ExitToNearParams::BaseToken(ref exit_params) => {
                    let eth_connector_account_id = self.get_eth_connector_contract_account()?;
                    exit_base_token_to_near(eth_connector_account_id, context, exit_params)?
                }
                // ERC-20 token transfer
                //
                // This precompile branch is expected to be called from the ERC-20 burn function.
                //
                // Input slice format:
                //  amount (U256 big-endian bytes) - the amount that was burned
                //  recipient_account_id (bytes) - the NEAR recipient account which will receive
                //  NEP-141 tokens, or also can contain the `:unwrap` suffix in case of withdrawing
                //  wNEAR, or another message of JSON in case of OMNI, or address of receiver in case
                //  of transfer tokens to another engine contract.
                ExitToNearParams::Erc20TokenParams(ref exit_params) => {
                    exit_erc20_token_to_near(context, exit_params, &self.io)?
                }
            };

        let callback_args = ExitToNearPrecompileCallbackArgs {
            #[cfg(feature = "error_refund")]
            refund: refund_call_args(&exit_to_near_params, &exit_event),
            #[cfg(not(feature = "error_refund"))]
            refund: None,
            transfer_near: transfer_near_args,
        };
        let attached_gas = if method == "ft_transfer_call" {
            costs::FT_TRANSFER_CALL_GAS
        } else {
            costs::FT_TRANSFER_GAS
        };

        let transfer_promise = PromiseCreateArgs {
            target_account_id: nep141_address,
            method,
            args: args.into_bytes(),
            attached_balance: Yocto::new(1),
            attached_gas,
        };

        let promise = if callback_args == ExitToNearPrecompileCallbackArgs::default() {
            PromiseArgs::Create(transfer_promise)
        } else {
            PromiseArgs::Callback(PromiseWithCallbackArgs {
                base: transfer_promise,
                callback: PromiseCreateArgs {
                    target_account_id: self.current_account_id.clone(),
                    method: "exit_to_near_precompile_callback".to_string(),
                    args: borsh::to_vec(&callback_args).unwrap(),
                    attached_balance: Yocto::new(0),
                    attached_gas: costs::EXIT_TO_NEAR_CALLBACK_GAS,
                },
            })
        };
        let promise_log = Log {
            address: exit_to_near::ADDRESS.raw(),
            topics: Vec::new(),
            data: borsh::to_vec(&promise).unwrap(),
        };
        let ethabi::RawLog { topics, data } = exit_event.encode();
        let exit_event_log = Log {
            address: exit_to_near::ADDRESS.raw(),
            topics: topics.into_iter().map(|h| H256::from(h.0)).collect(),
            data,
        };

        Ok(PrecompileOutput {
            logs: vec![promise_log, exit_event_log],
            cost: required_gas,
            output: Vec::new(),
        })
    }

fn exit_erc20_token_to_near<I: IO>(
    context: &Context,
    exit_params: &Erc20TokenParams,
    io: &I,
) -> Result<
    (
        AccountId,
        String,
        events::ExitToNear,
        String,
        Option<TransferNearArgs>,
    ),
    ExitError,
> {
    // In case of withdrawing ERC-20 tokens, the `apparent_value` should be zero. In opposite way
    // the funds will be locked in the address of the precompile without any possibility
    // to withdraw them in the future. So, in case if the `apparent_value` is not zero, the error
    // will be returned to prevent that.
    if context.apparent_value != U256::zero() {
        return Err(ExitError::Other(Cow::from(
            "ERR_ETH_ATTACHED_FOR_ERC20_EXIT",
        )));
    }

    let erc20_address = context.caller; // because ERC-20 contract calls the precompile.
    let nep141_account_id = get_nep141_from_erc20(erc20_address.as_bytes(), io)?;

    let (nep141_account_id, args, method, transfer_near_args, event) = match exit_params.message {
        // wNEAR address should be set via the `factory_set_wnear_address` transaction first.
        Some(Message::UnwrapWnear) if erc20_address == get_wnear_address(io).raw() =>
        // The flow is following here:
        // 1. We call `near_withdraw` on wNEAR account id on `aurora` behalf.
        // In such way we unwrap wNEAR to NEAR.
        // 2. After that, we call callback `exit_to_near_precompile_callback` on the `aurora`
        // in which make transfer of unwrapped NEAR to the `target_account_id`.
        {
            (
                nep141_account_id,
                format!(r#"{{"amount":"{}"}}"#, exit_params.amount.as_u128()),
                "near_withdraw",
                Some(TransferNearArgs {
                    target_account_id: exit_params.receiver_account_id.clone(),
                    amount: exit_params.amount.as_u128(),
                }),
                events::ExitToNear::Legacy(ExitToNearLegacy {
                    sender: Address::new(erc20_address),
                    erc20_address: Address::new(erc20_address),
                    dest: exit_params.receiver_account_id.to_string(),
                    amount: exit_params.amount,
                }),
            )
        }
        // In this flow, we're just forwarding the `msg` to the `ft_transfer_call` transaction.
        Some(Message::Omni(msg)) => (
            nep141_account_id,
            ft_transfer_call_args(&exit_params.receiver_account_id, exit_params.amount, msg)?,
            "ft_transfer_call",
            None,
            events::ExitToNear::Omni(ExitToNearOmni {
                sender: Address::new(erc20_address),
                erc20_address: Address::new(erc20_address),
                dest: exit_params.receiver_account_id.to_string(),
                amount: exit_params.amount,
                msg: msg.to_string(),
            }),
        ),
        // The legacy flow. Just withdraw the tokens to the NEAR account id.
        // P.S. We use underscore here instead of `None` to handle the case when a user
        // could add the `unwrap` suffix for non wNEAR ERC-20 token by mistake.
        _ => {
            // There is no way to inject json, given the encoding of both arguments
            // as decimal and valid account id respectively.
            (
                nep141_account_id,
                format!(
                    r#"{{"receiver_id":"{}","amount":"{}"}}"#,
                    exit_params.receiver_account_id,
                    exit_params.amount.as_u128()
                ),
                "ft_transfer",
                None,
                events::ExitToNear::Legacy(ExitToNearLegacy {
                    sender: Address::new(erc20_address),
                    erc20_address: Address::new(erc20_address),
                    dest: exit_params.receiver_account_id.to_string(),
                    amount: exit_params.amount,
                }),
            )
        }
    };

    Ok((
        nep141_account_id,
        args,
        event,
        method.to_string(),
        transfer_near_args,
    ))
}

fn create_target_account_id(
    sender: H160,
    engine_account_id: &str,
) -> Result<AccountId, PrecompileFailure> {
    let mut buffer = [0; 40];
    hex::encode_to_slice(sender.as_bytes(), &mut buffer)
        .map_err(|_| revert_with_message(consts::ERR_XCC_ACCOUNT_ID))?;
    let sender_in_hex =
        str::from_utf8(&buffer).map_err(|_| revert_with_message(consts::ERR_XCC_ACCOUNT_ID))?;

    AccountId::try_from(format!("{sender_in_hex}.{engine_account_id}"))
        .map_err(|_| revert_with_message(consts::ERR_XCC_ACCOUNT_ID))
}
