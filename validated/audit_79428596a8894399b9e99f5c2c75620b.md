### Title
EIP-7702 Authorization List Processing Performs Expensive Computation and State Mutations Before Balance Check — (`File: basic_bootloader/src/bootloader/transaction_flow/ethereum/validation_impl.rs`, `basic_bootloader/src/bootloader/transaction_flow/zk/validation_impl.rs`)

### Summary

In both the Ethereum STF path and the ZK path, the EIP-7702 authorization list is fully processed — including per-entry `ecrecover`, Keccak hashing, cold account reads, delegation writes, and nonce increments — before the sender balance check is performed. There is no upper bound on the number of authorization entries. An attacker with zero balance can submit EIP-7702 type-4 transactions with arbitrarily large authorization lists, forcing the sequencer to perform unbounded expensive computation and state mutations before the balance check rejects the transaction.

### Finding Description

**Root cause — ordering in the Ethereum STF path:**

In `basic_bootloader/src/bootloader/transaction_flow/ethereum/validation_impl.rs`, `create_resources_for_tx` (line 26–60) computes intrinsic gas from `TX_INTRINSIC_GAS + calldata_tokens` only — it accepts no `authorization_list_num` parameter and does not include per-authorization `NEWACCOUNT` gas in the intrinsic overhead. [1](#0-0) 

The authorization list is then processed at line 347–353 using `tx_resources.main_resources` (bounded only by the gas limit, not by the sender's balance), and the balance check does not occur until lines 392–399. [2](#0-1) 

**Root cause — ordering in the ZK path:**

In `basic_bootloader/src/bootloader/transaction_flow/zk/validation_impl.rs`, the authorization list is processed at lines 421–436 using `intrinsic_resources.with_infinite_ergs(...)` — meaning ergs are unlimited for this step. The balance check does not occur until lines 438–451. [3](#0-2) 

**Per-entry cost of `validate_and_apply_delegation`:**

Each authorization entry triggers: a Keccak hash (`compute_auth_message_signed_hash`), a full `secp256k1_ec_recover`, a cold account read (`read_account_properties`), and — on success — a `set_delegation` write and `increment_nonce` write. [4](#0-3) 

**No maximum authorization list size:**

`AuthorizationList` is `HomList<AuthorizationEntry, true>`. `HomList::decode_list_from` iterates through all entries to count them with no upper-bound check. The only guard is rejection of an empty list (`count == Some(0)`). [5](#0-4) [6](#0-5) 

**The design comment acknowledges the risk but assumes non-mutating operations:**

The ZK path comment at line 233–235 states: *"either 'from' is valid … or we will perform non-mutating operations without any payment."* Authorization list processing violates this assumption — it performs state-mutating operations (delegation writes, nonce increments) before the balance check. [7](#0-6) 

### Impact Explanation

An attacker with zero (or insufficient) balance submits an EIP-7702 transaction with N authorization entries and a gas limit large enough to pass the intrinsic gas check. The system performs N ecrecovers, N Keccak hashes, N cold account reads, and potentially N delegation writes and nonce increments before discovering the sender cannot pay. The IO frame rollback undoes the state mutations, but the computation is irrecoverable. In the ZK path, the auth list processing runs with `FORMAL_INFINITE` resources, so it cannot be cut short by resource exhaustion. This is a resource accounting bug: expensive cryptographic and storage operations are performed without any guarantee of payment, directly analogous to the external report's unauthenticated challenge-response DoS vector. The sequencer's validation pipeline and, if such transactions reach block inclusion, the prover's workload are both affected.

### Likelihood Explanation

EIP-7702 is a standard transaction type accepted by any compliant node. No privileged access is required — any unprivileged sender can craft a type-4 transaction with a large authorization list and zero balance. The attacker only needs to broadcast valid RLP-encoded transactions. The sequencer mempool processes these transactions through the full validation path (including auth list processing) before the balance check rejects them, making repeated submission a low-cost, high-impact DoS vector.

### Recommendation

1. Move the balance check (and nonce check) **before** the authorization list processing in both `ethereum/validation_impl.rs` and `zk/validation_impl.rs`.
2. Add an explicit maximum authorization list size (e.g., `MAX_AUTH_LIST_ENTRIES = 255` or tied to block gas limit) enforced during parsing in `EIP7702Tx::decode_list_body`.
3. In `create_resources_for_tx` (Ethereum path), include `authorization_list_num * NEWACCOUNT` in the intrinsic gas calculation, consistent with the ZK path's `calculate_tx_intrinsic_gas`.

### Proof of Concept

1. Craft an EIP-7702 type-4 transaction:
   - `from`: any address with zero balance
   - `gas_limit`: `N * NEWACCOUNT_GAS + TX_INTRINSIC_GAS` (sufficient to pass intrinsic check)
   - `authorization_list`: N entries, each with a valid-length but garbage signature (so ecrecover returns `None` and the entry is skipped after the ecrecover call)
2. Submit to the sequencer/node.
3. The system will execute N Keccak hashes and N `secp256k1_ec_recover` calls before reaching the balance check at line 392 (Ethereum path) or line 444 (ZK path).
4. The balance check fails with `LackOfFundForMaxFee`.
5. Repeat with a new transaction (nonce is rolled back by the IO frame).
6. Each submission forces O(N) ecrecover + Keccak work with zero cost to the attacker. [8](#0-7) [9](#0-8) [10](#0-9)

fn create_resources_for_tx<S: EthereumLikeTypes>(
    gas_limit: u64,
    is_deployment: bool,
    calldata_len: u64,
    calldata_tokens: u64,
) -> Result<ResourcesForEthereumTx<S>, TxError> {
    let mut intrinsic_overhead = TX_INTRINSIC_GAS;
    if is_deployment {
        if calldata_len > MAX_INITCODE_SIZE as u64 {
            return Err(TxError::Validation(CreateInitCodeSizeLimit));
        }
        intrinsic_overhead = intrinsic_overhead.saturating_add(DEPLOYMENT_TX_EXTRA_INTRINSIC_GAS);
        let initcode_gas_cost =
            evm_interpreter::gas_constants::INITCODE_WORD_COST * calldata_len.div_ceil(32);
        intrinsic_overhead = intrinsic_overhead.saturating_add(initcode_gas_cost);
    }
    intrinsic_overhead =
        intrinsic_overhead.saturating_add(calldata_tokens.saturating_mul(CALLDATA_TOKEN_GAS_COST));

    if intrinsic_overhead > gas_limit {
        Err(TxError::Validation(
            InvalidTransaction::OutOfGasDuringValidation,
        ))
    } else {
        let gas_limit_for_tx = gas_limit - intrinsic_overhead;
        let ergs = gas_limit_for_tx.saturating_mul(ERGS_PER_GAS); // we checked at the very start that gas_limit * ERGS_PER_GAS doesn't overflow
        let native_limit =
            <<S as zk_ee::system::SystemTypes>::Resources as Resources>::Native::from_computational(
                u64::MAX,
            );
        let main_resources = S::Resources::from_ergs_and_native(Ergs(ergs), native_limit);

        Ok(ResourcesForEthereumTx { main_resources })
    }
}

pub fn parse_authorization_list_and_apply_delegations<S: EthereumLikeTypes>(
    system: &mut System<S>,
    resources: &mut S::Resources,
    auth_list: AuthorizationList<'_>,
) -> Result<(), TxError>
where
    S::IO: IOSubsystemExt,
{
    use crate::bootloader::transaction::rlp_encoded::AuthorizationEntry;
    let mut hasher = crypto::sha3::Keccak256::new();

    for entry in auth_list.iter() {
        let AuthorizationEntry {
            chain_id,
            address,
            nonce,
            y_parity,
            r,
            s,
        } = entry;
        let success = validate_and_apply_delegation(
            system,
            resources,
            &chain_id,
            nonce,
            address,
            (y_parity, r, s),
            &mut hasher,
        )?;
        system_log!(system, "Delegation success: {success}\n");

        if !success {}
    }
    Ok(())
}

fn validate_and_apply_delegation<S: EthereumLikeTypes>(
    system: &mut System<S>,
    resources: &mut S::Resources,
    auth_chain_id: &U256,
    auth_nonce: u64,
    delegation_address: &[u8; 20],
    auth_sig_data: (u8, &[u8], &[u8]),
    hasher: &mut crypto::sha3::Keccak256,
) -> Result<bool, TxError>
where
    S::IO: IOSubsystemExt,
{
    let chain_id = system.get_chain_id();

    // 0. Pre-charge intrinsic gas
    resources.charge(&S::Resources::from_ergs_and_native(
        Ergs(evm_interpreter::gas_constants::NEWACCOUNT * ERGS_PER_GAS),
        <<S::Resources as Resources>::Native as zk_ee::system::Computational>::from_computational(
            crate::bootloader::constants::PER_AUTH_NATIVE_COMPUTATIONAL_OVERHEAD,
        ),
    ))?;

    // 1. Check chain id
    if !auth_chain_id.is_zero() && auth_chain_id != &U256::from(chain_id) {
        return Ok(false);
    }
    // 2. Check for nonce overflow
    if auth_nonce == u64::MAX {
        return Ok(false);
    }
    // 3. Signature
    // EIP-2 check
    let (_, _, auth_s) = auth_sig_data;
    let s = U256::try_from_be_slice(auth_s)
        .ok_or::<TxError>(InvalidTransaction::InvalidStructure.into())?;
    if s > crypto::secp256k1::SECP256K1N_HALF_U256 {
        return Ok(false);
    }
    let msg = resources.with_infinite_ergs(|inf_ergs| {
        compute_auth_message_signed_hash::<S>(
            inf_ergs,
            auth_chain_id,
            auth_nonce,
            delegation_address,
            hasher,
        )
    })?;
    let Some(authority) = resources
        .with_infinite_ergs(|inf_ergs| recover_authority(system, inf_ergs, auth_sig_data, &msg))?
    else {
        return Ok(false);
    };

    // 4. Read authority account
    // Gas already charged in intrinsic
    let account_properties = resources.with_infinite_ergs(|inf_ergs| {
        system.io.read_account_properties(
            ExecutionEnvironmentType::NoEE,
            inf_ergs,
            &authority,
            AccountDataRequest::empty()
                .with_nonce()
                .with_nominal_token_balance()
                .with_is_delegated()
                .with_has_bytecode(),
        )
    })?;
    // 5. Check authority is not a contract
    if account_properties.is_contract() {
        return Ok(false);
    }
    // 6. Check nonce
    if account_properties.nonce.0 != auth_nonce {
        return Ok(false);
    }
    // 7. Add refund if authority is not empty.
    let is_empty = account_properties.nonce.0 == 0
        && account_properties.has_bytecode() == false
        && account_properties.nominal_token_balance.0.is_zero();

    if !is_empty {
        let ergs = Ergs(
            (evm_interpreter::gas_constants::NEWACCOUNT
                - evm_interpreter::gas_constants::PER_AUTH_BASE_COST)
                * ERGS_PER_GAS,
        );
        system
            .io
            .add_to_refund_counter(S::Resources::from_ergs(ergs))?
    }

    let delegation_address = B160::from_be_bytes(*delegation_address);
    system_log!(
        system,
        "Will delegate address 0x{:040x} -> 0x{:040x}\n",
        authority.as_uint(),
        delegation_address.as_uint()
    );

    // 8. Set code for authority, system function
    //    will handle the two cases (unsetting).
    resources.with_infinite_ergs(|inf_ergs| {
        system
            .io
            .set_delegation(inf_ergs, &authority, &delegation_address)
    })?;
    // 9.Bump nonce
    resources
        .with_infinite_ergs(|inf_ergs| {
            system
                .io
                .increment_nonce(ExecutionEnvironmentType::NoEE, inf_ergs, &authority, 1)
        })
        .map_err(|e| -> BootloaderSubsystemError {
            match e {
                SubsystemError::LeafUsage(InterfaceError(NonceError::NonceOverflow, _)) => {
                    internal_error!("Cannot overflow, already checked").into()
                }
                _ => wrap_error!(e),
            }
        })?;
    Ok(true)
}

impl<'a, T: RlpItemDecode<'a>, const VALIDATE: bool> HomList<'a, T, VALIDATE> {
    pub fn decode_list_from(r: &mut Rlp<'a>) -> Result<Self, InvalidTransaction> {
        let inner = r.list()?;
        let all = inner.remaining();

        let count = if VALIDATE {
            let mut chk = Rlp::new(all);
            let mut c = 0;
            while !chk.is_empty() {
                T::decode_from_item(&mut chk)?;
                c += 1;
            }
            Some(c)
        } else {
            None
        };

        Ok(Self {
            payload: all,
            count,
            _marker: PhantomData,
        })
    }
