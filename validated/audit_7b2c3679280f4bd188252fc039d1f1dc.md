Audit Report

## Title
`spawn_neuron` Pre-flight Guard Uses Hardcoded 5% Worst-Case Modulation While Mission 70 Settlement Can Apply −10% - (File: `rs/nns/governance/src/governance.rs`)

## Summary
`Governance::spawn_neuron` validates the spawned neuron's minimum stake using a hardcoded 5% worst-case modulation constant, but `maybe_spawn_neurons` settles using the Mission 70 modulation system whose lower bound is −10%. Any neuron holder can initiate a spawn whose pre-flight check passes but whose 7-day-later settlement mints a stake below `neuron_minimum_stake_e8s`, violating the NNS neuron-stake invariant without any privileged access.

## Finding Description

**Phase 1 – `spawn_neuron` pre-flight check (line 2666):**

```rust
let least_possible_stake = (maturity_to_spawn as f64 * (1_f64 - 0.05)) as u64;
``` [1](#0-0) 

The constant `0.05` was correct when the only modulation source was the CMC, whose range is `[MIN_MATURITY_MODULATION_PERMYRIAD, MAX_MATURITY_MODULATION_PERMYRIAD]` = `[−500, +500]` permyriad = ±5%, defined in `rs/nervous_system/governance/src/maturity_modulation/mod.rs` lines 4–5. [2](#0-1) 

**Phase 2 – `maybe_spawn_neurons` settlement (line 6484):**

Settlement reads `heap_data.maturity_modulation.current_value_permyriad` and validates it against `VALID_MATURITY_MODULATION_BASIS_POINTS_RANGE`, which is bounded by the Mission 70 constants: [3](#0-2) 

The Mission 70 lower bound is −1000 permyriad (−10%): [4](#0-3) 

The actual minting call applies whatever value is in `heap_data.maturity_modulation`: [5](#0-4) 

**The gap:** For any maturity value M satisfying `M × 0.95 ≥ neuron_minimum_stake_e8s` and `M × 0.90 < neuron_minimum_stake_e8s`, the pre-flight check at line 2666 passes, the parent neuron's maturity is irrevocably transferred to the child neuron in `Spawning` state, and 7 days later `maybe_spawn_neurons` mints a stake below `neuron_minimum_stake_e8s`.

## Impact Explanation

This is a **High** severity finding. Any unprivileged NNS neuron holder can trigger it once Mission 70 modulation reaches −10%. The result is a neuron permanently existing in the NNS with sub-minimum stake, violating the protocol invariant that every neuron holds at least `neuron_minimum_stake_e8s`. The parent neuron's maturity is permanently consumed in the process. This constitutes a significant NNS governance security impact with concrete protocol harm — the invariant violation is not self-correcting and affects the integrity of the neuron store.

## Likelihood Explanation

Mission 70 modulation starts at 0 and is speed-limited to 30 permyriad per day (`MATURITY_MODULATION_DAILY_SPEED_LIMIT_PERMYRIAD`). Reaching −10% requires roughly 33 consecutive days of ICP price decline relative to the 365-day average — a realistic bear-market scenario. Once modulation is at or near −1000 permyriad, the exploit window is open to any neuron holder with maturity in the range `[neuron_minimum_stake_e8s / 0.95, neuron_minimum_stake_e8s / 0.90)`. No special privileges are required; the attack is repeatable by any qualifying neuron holder.

## Recommendation

Replace the hardcoded `0.05` constant in `spawn_neuron` with the actual Mission 70 lower bound so the pre-flight check is consistent with what `maybe_spawn_neurons` will apply:

```rust
use ic_nervous_system_governance::maturity_modulation::apply_maturity_modulation;
let least_possible_stake = apply_maturity_modulation(
    maturity_to_spawn,
    MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 as i32,
).unwrap_or(0);

if least_possible_stake < economics.neuron_minimum_stake_e8s {
    return Err(GovernanceError::new_with_message(
        ErrorType::InsufficientFunds,
        "There isn't enough maturity to spawn a new neuron due to worst case maturity modulation.",
    ));
}
```

This ensures the guard uses the same worst-case bound (−10%) that `maybe_spawn_neurons` validates against via `VALID_MATURITY_MODULATION_BASIS_POINTS_RANGE`.

## Proof of Concept

1. Set `heap_data.maturity_modulation.current_value_permyriad = Some(-1000)` in a test environment (or wait 33+ days of price decline on mainnet).
2. Let `min_stake = economics.neuron_minimum_stake_e8s`. Choose `M` such that `M × 0.95 ≥ min_stake` and `M × 0.90 < min_stake` (e.g., `M = min_stake * 100 / 92`).
3. Call `spawn_neuron` with `maturity_to_spawn = M`. The check at line 2666 computes `M × 0.95 ≥ min_stake` → passes.
4. The child neuron enters `Spawning` state; the parent's maturity is permanently reduced by M.
5. After `neuron_spawn_dissolve_delay_seconds` (7 days), `maybe_spawn_neurons` fires, reads `maturity_modulation = −1000`, calls `apply_maturity_modulation(M, −1000)` → minted stake = `M × 0.90 < min_stake`.
6. Assert the resulting neuron's `cached_neuron_stake_e8s < neuron_minimum_stake_e8s` — invariant violated.
