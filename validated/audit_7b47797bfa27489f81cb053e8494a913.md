### Title
SNS Governance `run_periodic_tasks()` Mints Tokens in `PreInitializationSwap` Mode Without Mode Guard - (File: rs/sns/governance/src/governance.rs)

### Summary
`SNS Governance.run_periodic_tasks()` unconditionally executes token-minting operations — `distribute_rewards()` and `maybe_finalize_disburse_maturity()` — regardless of the current governance `mode`. The `PreInitializationSwap` mode explicitly exists to "ensure the integrity of the initial token swap," and it blocks user-initiated minting proposals (`mint_sns_tokens`, `transfer_sns_treasury_funds`). However, the periodic task that fires on every timer tick bypasses this restriction entirely, minting tokens into neuron accounts during the swap period.

### Finding Description
`SNS Governance` has two operational modes defined in `governance.proto`:

- `MODE_NORMAL = 1` — all operations allowed
- `MODE_PRE_INITIALIZATION_SWAP = 2` — various operations restricted to protect swap integrity [1](#0-0) 

The mode enforcement is applied to user-initiated calls via `allows_manage_neuron_command_or_err` and `allows_proposal_action_or_err`, which block minting-related actions in `PreInitializationSwap` mode: [2](#0-1) 

However, `run_periodic_tasks()` — invoked on every timer tick — performs no mode check before executing:

1. **`distribute_rewards(supply)`** — mints new governance tokens to neuron holders as voting rewards (calls `self.ledger.total_supply().await` then mints)
2. **`maybe_finalize_disburse_maturity()`** — calls `self.ledger.transfer_funds()` to mint tokens to neuron owners [3](#0-2) 

Neither `should_distribute_rewards()` nor `maybe_finalize_disburse_maturity()` contains any check against `self.proto.mode`: [4](#0-3) [5](#0-4) 

The canister entry point for the timer simply calls `run_periodic_tasks()` with no mode guard: [6](#0-5) 

### Impact Explanation
During `PreInitializationSwap` mode, the SNS token supply is supposed to be stable to preserve the integrity of the decentralization swap. The swap canister uses the token supply and neuron distribution to calculate swap parameters and token prices. Unconditional reward distribution and maturity disbursement during this period:

1. **Inflates the token supply** before the swap completes, altering the effective price per token for swap participants.
2. **Violates the integrity guarantee** of `PreInitializationSwap` mode — the same guarantee that blocks `mint_sns_tokens` and `transfer_sns_treasury_funds` proposals.
3. **Alters neuron stake balances** (via `maybe_move_staked_maturity()`, also called unconditionally) before the swap, affecting voting power calculations.

The impact is a **governance authorization bypass** leading to unauthorized token minting during a protected operational mode, directly undermining the economic integrity of the SNS token swap.

### Likelihood Explanation
The timer fires automatically on every heartbeat. If a reward round duration has elapsed since the last reward event (checked by `should_distribute_rewards()`), rewards are distributed unconditionally. Any SNS that has been in `PreInitializationSwap` mode for longer than its configured `round_duration_seconds` will trigger this path with certainty — no attacker action is required. The condition is met in normal operation for any SNS whose swap preparation takes longer than one reward round.

### Recommendation
Add a mode check at the top of `run_periodic_tasks()` (or individually in `distribute_rewards()` and `maybe_finalize_disburse_maturity()`) to skip token-minting operations when `self.proto.mode != Mode::Normal`. This mirrors the fix described in the reference report: silently skip the financial operation if the system is not in the expected operational state.

```rust
pub async fn run_periodic_tasks(&mut self) {
    let mode = governance::Mode::try_from(self.proto.mode).unwrap_or(governance::Mode::Unspecified);

    self.process_proposals();
    // ... upgrade tasks ...

    if mode == governance::Mode::Normal {
        if self.should_distribute_rewards() { ... }
        if self.should_update_maturity_modulation() { ... }
        self.maybe_finalize_disburse_maturity().await;
        self.maybe_move_staked_maturity();
    }

    self.compute_cached_metrics().await;
    self.maybe_gc();
}
```

### Proof of Concept
1. Deploy an SNS via the SNS-W canister. Governance enters `PreInitializationSwap` mode.
2. Configure `voting_rewards_parameters.round_duration_seconds` to a short value (e.g., 1 day).
3. Wait for the swap preparation period to exceed one reward round duration.
4. Observe that `run_periodic_tasks()` fires via the timer and calls `distribute_rewards()`, minting new governance tokens to neuron holders.
5. Confirm via `get_latest_reward_event` that a reward event was recorded with `distributed_e8s_equivalent > 0` while `get_mode` still returns `PreInitializationSwap`.
6. The token supply has increased during the protected mode, violating the swap integrity guarantee. [7](#0-6) [8](#0-7)

    /// Runs periodic tasks that are not directly triggered by user input.
    pub async fn run_periodic_tasks(&mut self) {
        use ic_cdk::println;

        self.process_proposals();

        // None of the upgrade-related tasks should interleave with one another or themselves, so we acquire a global
        // lock for the duration of their execution. This will return `false` if the lock has already been acquired less
        // than 10 minutes ago by a previous invocation of `run_periodic_tasks`, in which case we skip the
        // upgrade-related tasks.
        if self.acquire_upgrade_periodic_task_lock() {
            // We only want to check the upgrade status if we are currently executing an upgrade.
            if self.should_check_upgrade_status() {
                self.check_upgrade_status().await;
            }

            if self.should_refresh_cached_upgrade_steps() {
                match self.try_temporarily_lock_refresh_cached_upgrade_steps() {
                    Err(err) => {
                        log!(ERROR, "{}", err);
                    }
                    Ok(deployed_version) => {
                        self.refresh_cached_upgrade_steps(deployed_version).await;
                    }
                }
            }

            self.initiate_upgrade_if_sns_behind_target_version().await;

            self.release_upgrade_periodic_task_lock();
        }

        let should_distribute_rewards = self.should_distribute_rewards();

        // Getting the total governance token supply from the ledger is expensive enough
        // that we don't want to do it on every call to `run_periodic_tasks`. So
        // we only fetch it when it's needed, which is when rewards should be
        // distributed
        if should_distribute_rewards {
            match self.ledger.total_supply().await {
                Ok(supply) => {
                    // Distribute rewards
                    self.distribute_rewards(supply);
                }
                Err(e) => log!(
                    ERROR,
                    "Error when getting total governance token supply: {}",
                    GovernanceError::from(e)
                ),
            }
        }

        if self.should_update_maturity_modulation() {
            self.update_maturity_modulation().await;
        }

        self.maybe_finalize_disburse_maturity().await;

        self.maybe_move_staked_maturity();

        self.compute_cached_metrics().await;

        self.maybe_gc();
    }
