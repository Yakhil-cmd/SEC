### Title
Unprotected `new()` Initializer Allows Any Caller to Seize Engine Ownership — (`File: engine/src/contract_methods/admin.rs`)

### Summary
The Aurora Engine's `new()` initializer enforces a one-time-only guard (state must not exist) but performs **no check on who the caller is**. Any NEAR account that calls `new()` before the legitimate deployer can set an arbitrary `owner_id`, gaining full administrative control over the engine, including the ability to deploy arbitrary contract code and add full-access keys to the Aurora Engine NEAR account.

### Finding Description
The public WASM export `new()` in `engine/src/lib.rs` delegates to `contract_methods::admin::new()`:

```rust
// engine/src/lib.rs:77-83
pub extern "C" fn new() {
    let io = Runtime;
    let env = Runtime;
    contract_methods::admin::new(io, &env)
        .map_err(ContractError::msg)
        .sdk_unwrap();
}
```

The implementation in `engine/src/contract_methods/admin.rs` only checks whether state already exists:

```rust
// engine/src/contract_methods/admin.rs:56-88
pub fn new<I: IO + Copy, E: Env>(mut io: I, env: &E) -> Result<(), ContractError> {
    if state::get_state(&io).is_ok() {
        return Err(b"ERR_ALREADY_INITIALIZED".into());
    }
    // ← no env.predecessor_account_id() check
    let args = NewCallArgs::deserialize(&input)...;
    let state: EngineState = args.into();   // owner_id comes from attacker-supplied args
    state::set_state(&mut io, &state)?;
    Ok(())
}
```

The `env` parameter is passed in but is only used for `env.block_height()` inside the optional hashchain path. There is no `require_owner_only`, no `env.predecessor_account_id()` comparison, and no `env.assert_private_call()`. Any NEAR account can call `new()` and supply an arbitrary `owner_id` in the Borsh-encoded input.

The `owner_id` stored in `EngineState` is the sole gating identity for every privileged operation:

- `upgrade()` / `stage_upgrade()` — deploy arbitrary WASM to the contract account [1](#0-0) 
- `attach_full_access_key()` — add a full-access key to the Aurora Engine NEAR account [2](#0-1) 
- `set_eth_connector_contract_account()` — redirect the ETH connector to an attacker-controlled contract
- `factory_update()` — replace XCC router bytecode for all sub-accounts [3](#0-2) 

All of these gates call `require_owner_only(&state, &env.predecessor_account_id())`, which compares against the `owner_id` written during `new()`. [4](#0-3) 

### Impact Explanation
**Critical — Direct theft of all user funds and permanent freezing.**

An attacker who wins the initialization race becomes the engine owner. They can immediately call `upgrade()` to deploy a malicious WASM contract that:
- Drains all ETH and ERC-20 tokens bridged through Aurora (held in the ETH connector and NEP-141 contracts whose addresses are controlled by the engine owner).
- Alternatively, `attach_full_access_key()` adds a full-access key to the `aurora` NEAR account, giving the attacker unrestricted control over the account balance and all sub-accounts.

### Likelihood Explanation
**Medium.** In NEAR Protocol, contract deployment and initialization are separate transactions unless the deployer explicitly batches them. The `engine-workspace` test harness shows the two steps are issued as separate `transact()` calls:

```rust
// engine-workspace/src/lib.rs:107-125
let contract = account.deploy(&self.code...).await?;
// ...
engine.new(self.chain_id, self.owner_id, self.upgrade_delay_blocks)
    .transact()
    .await
``` [5](#0-4) 

Between the `deploy` receipt and the `new` receipt there is a window — at minimum one block — during which any NEAR account can submit a competing `new()` call. An attacker monitoring the NEAR RPC for a fresh `aurora` contract deployment can front-run initialization with a standard function-call transaction.

### Recommendation
Add a caller restriction to `new()`. The canonical NEAR pattern is to assert that the predecessor is the contract account itself (i.e., the deployer who holds the account's access key):

```rust
pub fn new<I: IO + Copy, E: Env>(mut io: I, env: &E) -> Result<(), ContractError> {
    if state::get_state(&io).is_ok() {
        return Err(b"ERR_ALREADY_INITIALIZED".into());
    }
    // Ensure only the contract account itself (deployer) can initialize.
    if env.predecessor_account_id() != env.current_account_id() {
        return Err(b"ERR_NOT_ALLOWED".into());
    }
    // ... rest unchanged
}
```

Alternatively, batch the `DeployContract` and `FunctionCall(new)` actions into a single NEAR promise batch so no external transaction can be interleaved.

### Proof of Concept
1. Attacker watches NEAR RPC for a transaction deploying WASM to the `aurora` account.
2. In the next block (before the legitimate `new()` call), attacker submits:
   ```
   near call aurora new '{"chain_id": [...], "owner_id": "attacker.near", "upgrade_delay_blocks": 0}' --accountId attacker.near
   ```
   (Borsh-encoded equivalent via RPC.)
3. `state::get_state(&io).is_ok()` returns `Err` (state absent) → guard passes.
4. `state::set_state` writes `owner_id = "attacker.near"`. [6](#0-5) 
5. Legitimate deployer's `new()` call now returns `ERR_ALREADY_INITIALIZED`.
6. Attacker calls `upgrade()` with malicious WASM → arbitrary code executes under the `aurora` account → all bridged funds are drained. [7](#0-6)

pub fn new<I: IO + Copy, E: Env>(mut io: I, env: &E) -> Result<(), ContractError> {
    if state::get_state(&io).is_ok() {
        return Err(b"ERR_ALREADY_INITIALIZED".into());
    }

    let input = io.read_input().to_vec();
    let args = NewCallArgs::deserialize(&input).map_err(|_| errors::ERR_BORSH_DESERIALIZE)?;

    let initial_hashchain = args.initial_hashchain();
    let state: EngineState = args.into();

    if let Some(block_hashchain) = initial_hashchain {
        let block_height = env.block_height();
        let mut hashchain = Hashchain::new(
            state.chain_id,
            env.current_account_id(),
            block_height,
            block_hashchain,
        );

        hashchain.add_block_tx(
            block_height,
            function_name!(),
            &input,
            &[],
            &Bloom::default(),
        )?;
        crate::hashchain::save_hashchain(&mut io, &hashchain)?;
    }

    state::set_state(&mut io, &state)?;
    Ok(())
}

pub fn upgrade<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<(), ContractError> {
    let state = state::get_state(&io)?;
    require_running(&state)?;
    require_owner_only(&state, &env.predecessor_account_id())?;

    let input = io.read_input().to_vec();
    let (code, state_migration_gas) = match UpgradeParams::try_from_slice(&input) {
        Ok(args) => (
            args.code,
            args.state_migration_gas
                .map_or(GAS_FOR_STATE_MIGRATION, NearGas::new),
        ),
        Err(_) => (input, GAS_FOR_STATE_MIGRATION), // Backward compatibility
    };

    let target_account_id = env.current_account_id();
    let batch = PromiseBatchAction {
        target_account_id,
        actions: vec![
            PromiseAction::DeployContract { code },
            PromiseAction::FunctionCall {
                name: "state_migration".to_string(),
                args: vec![],
                attached_yocto: ZERO_YOCTO,
                gas: state_migration_gas,
            },
        ],
    };
    let promise_id = handler.promise_create_batch(&batch);

    handler.promise_return(promise_id);

    Ok(())
}

pub fn attach_full_access_key<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<(), ContractError> {
    let state = state::get_state(&io)?;

    require_running(&state)?;
    require_owner_only(&state, &env.predecessor_account_id())?;

    let public_key = serde_json::from_slice::<FullAccessKeyArgs>(&io.read_input().to_vec())
        .map(|args| args.public_key)
        .map_err(|_| errors::ERR_JSON_DESERIALIZE)?;
    let current_account_id = env.current_account_id();
    let action = PromiseAction::AddFullAccessKey {
        public_key,
        nonce: 0, // not actually used - depends on block height
    };
    let promise = PromiseBatchAction {
        target_account_id: current_account_id,
        actions: vec![action],
    };
    // SAFETY: This action is dangerous because it adds a new full access key (FAK) to the Engine account.
    // However, it is safe to do so here because of the `require_owner_only` check above; only the
    // (trusted) owner account can add a new FAK.
    let promise_id = handler.promise_create_batch(&promise);

    handler.promise_return(promise_id);

    Ok(())
```

pub fn factory_update<I: IO + Copy, E: Env>(io: I, env: &E) -> Result<(), ContractError> {
    with_hashchain(io, env, function_name!(), |mut io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;
        require_owner_only(&state, &env.predecessor_account_id())?;
        let bytes = io.read_input().to_vec();
        let router_bytecode = xcc::RouterCode::new(bytes);
        xcc::update_router_code(&mut io, &router_bytecode);
        Ok(())
    })
