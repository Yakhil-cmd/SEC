### Title
Anonymous `read_state` Requests for Large Subtree Paths Trigger Unbounded Merkle Witness Generation Without Authentication or Cost Accounting - (File: `rs/http_endpoints/public/src/read_state.rs`)

### Summary
The IC replica's `read_state` endpoint allows anonymous (unauthenticated, no expiry check) requests for entire large subtrees such as `/subnet` and `/api_boundary_nodes`. Each such request triggers expensive `materialize_partial` + `hash_tree.witness` Merkle proof generation proportional to the full size of the requested subtree. There is no per-request cost accounting or replica-level rate limiting for these paths, creating a resource-exhaustion DoS vector against any publicly reachable replica node.

### Finding Description

The `read_state` handler in `rs/http_endpoints/public/src/read_state.rs` processes requests through two stages:

**Stage 1 – Validation (cheap):** `verify_paths` checks that each requested path is in the allowed set. The paths `["subnet"]` and `["api_boundary_nodes"]` are unconditionally allowed with no restriction: [1](#0-0) [2](#0-1) 

**Stage 2 – Proof generation (expensive):** `get_certificate_and_create_response` calls `sparse_labeled_tree_from_paths` and then `read_certified_state_with_exclusion`, which materializes the partial lazy tree and generates a `MixedHashTree` Merkle witness: [3](#0-2) 

The underlying `CertifiedStateSnapshotImpl::read_certified_state_with_exclusion` performs `materialize_partial` + `hash_tree.witness` — work proportional to the size of the requested subtree: [4](#0-3) 

**Anonymous requests bypass expiry validation entirely.** In `rs/validator/src/ingress_validation.rs`, the `ReadState` verifier skips `validate_ingress_expiry` when the sender is anonymous: [5](#0-4) 

This is confirmed by the explicit test `should_not_error_when_anonymous_read_state_request_expired` in `rs/validator/ingress_message/tests/validate_request.rs`: [6](#0-5) 

The `["subnet"]` path requests the entire subnet subtree — on mainnet this includes public keys, node public keys, canister ranges, and metrics for all ~30+ subnets. The `["api_boundary_nodes"]` path similarly returns all API boundary node entries. Both are valid paths accepted by `verify_paths` with no additional restriction.

The path count/depth limits (1,000 paths × 127 labels) are enforced: [7](#0-6) 

But these limits do not bound the *size of the subtree* returned per path. A single path `["subnet"]` is within limits yet triggers the most expensive possible witness generation.

### Impact Explanation

An attacker sends a continuous stream of anonymous `read_state` requests with path `["subnet"]` (or `["api_boundary_nodes"]`) to any publicly reachable replica node. Each request:
1. Passes validation (anonymous, single valid path, no expiry check)
2. Triggers full subnet-subtree materialization and Merkle witness generation on the replica's blocking thread pool

The concurrency limit (`max_read_state_concurrent_requests`) bounds simultaneous in-flight requests but does not prevent sustained queue saturation. With the queue full, legitimate `read_state` requests (e.g., polling for ingress status) receive `429 Too Many Requests`, degrading user-facing functionality. The `read_state` endpoint is a mandatory part of the IC protocol and cannot be disabled. [8](#0-7) 

### Likelihood Explanation

The attack requires no credentials, no on-chain resources, and no privileged access. Any internet-connected party can send anonymous CBOR-encoded `read_state` requests directly to a replica's public HTTPS endpoint. The path `["subnet"]` is a single-element path that is trivially constructable. Boundary-node rate limiting is optional and not present on direct node access (e.g., `api/v3/subnet/.../read_state` is accessed directly per the test infrastructure). The attack is low-cost for the attacker and high-cost for the replica.

### Recommendation

1. **Add a per-IP or per-connection rate limit at the replica level** for `read_state` requests, independent of boundary-node rate limiting.
2. **Restrict or cost-account subtree paths**: paths like `["subnet"]` and `["api_boundary_nodes"]` that request entire large subtrees should either be disallowed in favor of specific sub-paths, or subject to a stricter concurrency limit separate from leaf-path requests.
3. **Require authentication (non-anonymous sender) for subtree paths**: since subtree paths are primarily used by infrastructure tooling rather than end users, requiring a valid signature would eliminate the zero-cost anonymous amplification vector.

### Proof of Concept

```python
import cbor2, requests, time

# Construct anonymous read_state request for the entire /subnet subtree
envelope = {
    "content": {
        "request_type": "read_state",
        "sender": bytes([4]),          # anonymous principal
        "paths": [[b"subnet"]],        # entire subnet subtree
        "ingress_expiry": 0,           # ignored for anonymous sender
    }
    # no sender_pubkey, no sender_sig
}

node_url = "https://<replica-node>/api/v2/subnet/<subnet-id>/read_state"
body = cbor2.dumps(envelope)

# Flood with concurrent requests to saturate read_state queue
while True:
    requests.post(node_url, data=body,
                  headers={"Content-Type": "application/cbor"})
    time.sleep(0.01)
```

fn get_certificate_and_create_response(
    mut paths: Vec<Path>,
    delegation_from_nns: Option<CertificateDelegation>,
    certified_state_reader: &dyn CertifiedStateSnapshot<State = ReplicatedState>,
    deprecated_canister_ranges_filter: DeprecatedCanisterRangesFilter,
) -> axum::response::Response {
    // Create labeled tree. This may be an expensive operation and by
    // creating the labeled tree after verifying the paths we know that
    // the depth is max 4.
    // Always add "time" to the paths even if not explicitly requested.
    paths.push(Path::from(Label::from("time")));
    let labeled_tree = match sparse_labeled_tree_from_paths(&paths) {
        Ok(tree) => tree,
        Err(TooLongPathError) => {
            let status = StatusCode::BAD_REQUEST;
            let text = "Failed to parse requested paths: path is too long.".to_string();
            return (status, text).into_response();
        }
    };

    let exclusion_rule = match deprecated_canister_ranges_filter {
        DeprecatedCanisterRangesFilter::KeepAll => None,
        DeprecatedCanisterRangesFilter::KeepOnlyNNS(nns_subnet_id) => {
            let deprecated_canister_ranges_except_the_nns_subnet_id_pattern = vec![
                MatchPattern::Inclusive(Label::from("subnet")),
                MatchPattern::Exclusive(Label::from(nns_subnet_id.get_ref())),
                MatchPattern::Inclusive(Label::from("canister_ranges")),
            ];

            Some(deprecated_canister_ranges_except_the_nns_subnet_id_pattern)
        }
    };

    let Some((tree, certification)) = certified_state_reader
        .read_certified_state_with_exclusion(&labeled_tree, exclusion_rule.as_ref())
    else {
        return make_service_unavailable_response();
    };

    let signature = certification.signed.signature.signature.get().0;

    Cbor(HttpReadStateResponse {
        certificate: Blob(into_cbor(&Certificate {
            tree,
            signature: Blob(signature),
            delegation: delegation_from_nns,
        })),
    })
    .into_response()

    fn read_certified_state_with_exclusion(
        &self,
        paths: &LabeledTree<()>,
        exclusion: Option<&MatchPatternPath>,
    ) -> Option<(MixedHashTree, Certification)> {
        let _timer = self.read_certified_state_duration_histogram.start_timer();

        let mixed_hash_tree = {
            let lazy_tree = replicated_state_as_lazy_tree(self.get_state(), self.get_height());
            let partial_tree = materialize_partial(&lazy_tree, paths, exclusion.map(|v| &v[..]));
            self.hash_tree.witness::<MixedHashTree>(&partial_tree)
        }
        .ok()?;

        debug_assert_eq!(
            crypto_hash_of_partial_state(&mixed_hash_tree.digest()),
            self.certification.signed.content.hash,
            "produced invalid hash tree {:?} for paths {:?}, full hash tree: {:?}",
            mixed_hash_tree,
            paths,
            self.hash_tree
        );

        Some((mixed_hash_tree, self.certification.clone()))
    }

fn validate_paths_width_and_depth(paths: &[Path]) -> Result<(), RequestValidationError> {
    if paths.len() > MAXIMUM_NUMBER_OF_PATHS {
        return Err(TooManyPaths {
            maximum: MAXIMUM_NUMBER_OF_PATHS,
            length: paths.len(),
        });
    }
    for path in paths {
        if path.len() > MAXIMUM_NUMBER_OF_LABELS_PER_PATH {
            return Err(PathTooLong {
                maximum: MAXIMUM_NUMBER_OF_LABELS_PER_PATH,
                length: path.len(),
            });
        }
    }
    Ok(())
}
