### Title
`SenderInfoContent` Domain Separator Lacks Target-Canister Binding, Enabling Cross-Canister Replay of `sender_info` Signatures — (`rs/types/types/src/crypto/sign.rs`, `rs/types/types/src/messages/http.rs`)

---

### Summary

The `SenderInfoContent` signable type uses a fixed, global domain separator `"ic-sender-info"` with no binding to the target canister ID. A canister signature over `SenderInfoContent(info)` produced by a signing canister (e.g., Internet Identity) for one target canister is cryptographically valid when replayed verbatim to any other canister. This is the direct IC analog of the ERC-7739 `verifyingContract` bug: a shared signing entity's attestation is not scoped to the individual recipient, defeating the purpose of per-canister isolation.

---

### Finding Description

`SenderInfoContent` is the signable wrapper used when a signing canister (e.g., Internet Identity) attests to a user's `info` blob via a canister signature. Its `SignatureDomain` implementation returns a domain derived solely from the static string `"ic-sender-info"`:

```rust
impl<'a> SignatureDomain for SenderInfoContent<'a> {
    fn domain(&self) -> Vec<u8> {
        domain_with_prepended_length("ic-sender-info")
    }
}
