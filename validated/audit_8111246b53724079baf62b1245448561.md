Audit Report

## Title
Unvalidated `file_offset` in `deserialize_page` Causes Replica Panic via Compromised Sandbox IPC — (`rs/replicated_state/src/page_map/page_allocator/mmap.rs`)

## Summary

`MmapBasedPageAllocatorCore::deserialize_page` contains a hard `assert!` that verifies a full page fits within a mapped chunk. The caller `deserialize_page_delta` performs no alignment or bounds validation on `file_offset` values received from the sandbox IPC channel before invoking `deserialize_page`. A compromised sandbox process can send a `PageDeltaSerialization` with a non-page-aligned `file_offset` (e.g., `file_len - 1`) to satisfy the outer chunk-membership check while failing the inner `assert!`, panicking the replica process with no `catch_unwind` in the call chain.

## Finding Description

In `deserialize_page` (lines 735–743 of `mmap.rs`), the loop finds a chunk where `chunk.offset <= file_offset < chunk.offset + chunk.size`, then immediately asserts the full page fits:

```rust
assert!(
    file_offset + PAGE_SIZE as FileOffset
        <= chunk.offset + chunk.size as FileOffset
);
``` [1](#0-0) 

The outer `if` only checks that the **start** of the page is within the chunk; the `assert!` checks the **end**. With `file_offset = chunk.offset + chunk.size - 1` (e.g., 4095 for a 4096-byte chunk starting at 0):

- Outer condition: `0 <= 4095 < 4096` → **true** (chunk found)
- Inner assert: `4095 + 4096 <= 4096` → `8191 <= 4096` → **PANIC**

The caller `deserialize_page_delta` (lines 291–312) treats the precondition as a comment-only assumption, not an enforced check:

```
// File offsets of all pages are smaller than `file_len`, which means
// that the precondition of `deserialize_page()` is fulfilled after
// the call to `grow_for_deserialization(file_len)`.
``` [2](#0-1) 

No alignment check (`file_offset % PAGE_SIZE == 0`) and no end-of-page bounds check (`file_offset + PAGE_SIZE <= file_len`) are performed anywhere before `deserialize_page` is called.

`grow_for_deserialization` maps a single new chunk from `self.file_len` to `file_len`, so with `file_len=4096` and initial state `self.file_len=0`, the result is exactly one chunk `{offset:0, size:4096}`. [3](#0-2) 

`update_execution_state` calls `deserialize_delta` on both wasm and stable memory page maps with no intervening validation: [4](#0-3) 

`MmapPageSerialization` (containing `file_offset`) is a plain `#[derive(Deserialize, Serialize)]` struct with no validation logic: [5](#0-4) 

There is no `catch_unwind` anywhere in the `rs/canister_sandbox` call chain (confirmed by search), so the `assert!` panic propagates uncaught and terminates the replica process.

The codebase explicitly acknowledges sandbox compromise as a realistic threat and clamps instruction counts defensively: [6](#0-5) 

No equivalent defensive guard exists for `file_offset`.

## Impact Explanation

A compromised sandbox process can crash the entire replica node by sending a single malformed `PageDeltaSerialization`. This terminates the replica process, taking down all canisters hosted on that node. This matches the **High** impact class: *Application/platform-level DoS, crash, or subnet availability impact not based on raw volumetric DDoS* ($2,000–$10,000).

## Likelihood Explanation

The precondition is a compromised sandbox process, which requires a separate exploit (e.g., a Wasm JIT/interpreter vulnerability enabling sandbox escape). This is a non-trivial prerequisite. However, the IC security model explicitly treats sandbox compromise as a realistic threat and defends against it in other places (instruction count clamping). Once the sandbox is compromised, the exploit is trivial: send one IPC message with `file_offset = file_len - 1`. The attack is repeatable and deterministic.

## Recommendation

In `deserialize_page_delta`, validate each `file_offset` before calling `deserialize_page`:

1. Return an error (or log and skip) if `file_offset % PAGE_SIZE != 0`.
2. Return an error (or log and skip) if `file_offset + PAGE_SIZE > file_len`.

These checks should produce a recoverable error rather than a panic, consistent with the defensive clamping of instruction counts. The `assert!` inside `deserialize_page` can be downgraded to `debug_assert!` once input is validated at the IPC boundary.

## Proof of Concept

```rust
// Attacker-controlled PageDeltaSerialization sent over sandbox IPC:
//   file_len = 4096
//   pages = [MmapPageSerialization { page_index: 0, file_offset: 4095, validation: ... }]
//
// In deserialize_page_delta:
//   grow_for_deserialization(4096) → chunk = {offset: 0, size: 4096}
//
// In deserialize_page with file_offset = 4095:
//   chunk.offset(0) <= 4095 < chunk.offset + chunk.size(4096)  → true (chunk found)
//   assert!(4095 + 4096 <= 4096)  →  assert!(8191 <= 4096)  →  PANIC
//   No catch_unwind → replica process terminates
```

A unit test in `rs/replicated_state/src/page_map/page_allocator/mmap.rs` can reproduce this by constructing a `PageDeltaSerialization` with `file_len=4096` and `file_offset=4095` and calling `deserialize_page_delta` directly, observing the panic.

    fn grow_for_deserialization(&mut self, file_len: FileOffset) {
        if file_len == self.file_len {
            return;
        }
        if file_len < self.file_len {
            // This may happen if another thread already called `grow_for_deserialization`
            // while this thread was waiting for the lock. In that case the actual file
            // length is the same or is larger than the saved file length.
            let actual_file_len = unsafe { get_file_length(self.file_descriptor) };
            assert!(
                actual_file_len >= self.file_len,
                "The page allocator file was truncated: actual file_len = {}, new file_len = {}, old file_len = {}",
                actual_file_len,
                file_len,
                self.file_len
            );
            return;
        }
        let mmap_size = (file_len - self.file_len) as usize;
        let mmap_file_offset = self.file_len;
        self.file_len = file_len;

        // The mapping is read/write because freeing of pages uses `madvise()` with
        // `MADV_REMOVE`, which requires writable mapping.
        // SAFETY: The parameters are valid.
        let mmap_ptr = unsafe {
            mmap(
                std::ptr::null_mut(),
                mmap_size,
                ProtFlags::PROT_READ | ProtFlags::PROT_WRITE,
                MapFlags::MAP_SHARED,
                self.file_descriptor,
                mmap_file_offset,
            )
        }
        .unwrap_or_else(|err| {
            panic!(
                "MmapPageAllocator failed to mmap {} bytes to memory file #{} \
                         at offset {} for deserialization: {}",
                mmap_size, self.file_descriptor, mmap_file_offset, err,
            )
        }) as *mut u8;

        self.chunks.push(Chunk {
            ptr: mmap_ptr,
            size: mmap_size,
            offset: mmap_file_offset,
        });
    }
```
