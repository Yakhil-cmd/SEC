### Title
Unchecked Exit-Precompile Return Value in `EvmErc20.withdrawToNear` / `withdrawToEthereum` Causes Permanent ERC-20 Token Freeze - (File: `etc/eth-contracts/contracts/EvmErc20.sol`)

---

### Summary

`EvmErc20.sol` and `EvmErc20V2.sol` burn the caller's ERC-20 tokens **before** calling the `ExitToNear` / `ExitToEthereum` precompile via inline assembly, but **never check the return value** of that call. When the precompile returns failure (0) — which is user-triggerable by supplying an invalid NEAR account ID as `recipient` — the ERC-20 tokens are permanently destroyed with no corresponding NEP-141 transfer ever initiated. This is a direct bridge-accounting analog to the GoodDollar bug: value is consumed on one side of the bridge without the corresponding credit being issued on the other side.
