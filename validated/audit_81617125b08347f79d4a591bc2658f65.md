## Verdict

### Title
Missing EVM Precompile Warm-Up in ZK `tx_loop` Causes Gas Underestimation vs Forward Simulation — (`basic_bootloader/src/bootloader/block_flow/zk/tx_loop.rs`)

### Summary
The ZK proving bootloader (`ZKHeaderStructureTxLoop`) does not warm EVM precompile addresses before transaction execution, while the forward/simulation bootloader (`generic_loop_op` / `EthereumLoopOp`) does. This creates a systematic 2500-gas-per-precompile-call divergence between `eth_estimateGas` (forward mode) and actual ZK proving execution.

### Finding Description
In `ethereum/loop_op.rs` lines 95–109, before every transaction, the forward bootloader iterates `PRECOMPILE_ADDRESSES_LOWS` and calls `touch_account` with `FORMAL_INFINITE` resources on each precompile address, marking them warm. In `zk/tx_loop.rs` lines 58–70, the ZK bootloader only warms the coinbase — the equivalent precompile warm-up block is entirely absent. The comment on line 41 of `ethereum/loop_op.rs` even reads `// TODO: unify with ZK version`, confirming the divergence is known but unresolved.

### Impact Explanation
A user calls `eth_estimateGas` for a transaction that calls ecrecover (or any other precompile). The forward system returns gas estimate X, where the precompile CALL costs 100 gas (warm). The user submits the transaction with `gas_limit = X`. In ZK proving mode, the precompile is cold, costing 2600 gas — 2500 more than estimated. If the margin is tight, the transaction runs out of gas mid-execution, reverts, and the user loses the full `gas_limit * gas_price` in fees for a transaction that simulation guaranteed would succeed.

### Likelihood Explanation
Any contract that calls EVM precompiles (ecrecover for signature verification, sha256, modexp for RSA/ZK, ecPairing for on-chain ZK verification) is affected. These are extremely common patterns. The gas shortfall is deterministic and reproducible: exactly 2500 gas per precompile call that is the first call in the transaction. The ignored test `test_modexp_gas_cost_correct` with the comment "we need to fix gas model for whole pipeline, warm/cold logic for precompiles" confirms the developers are aware the issue exists.

### Recommendation
Add the precompile warm-up loop to `zk/tx_loop.rs` immediately after the coinbase warm-up, mirroring lines 95–109 of `ethereum/loop_op.rs`. The `// TODO: unify with ZK version` comment in `ethereum/loop_op.rs` points directly to this fix. Alternatively, extract the shared warm-up logic into a shared function called by both paths.

### Proof of Concept
1. Deploy a contract with bytecode: `PUSH20 0x0000000000000000000000000000000000000001; PUSH1 0x80; PUSH1 0x00; PUSH1 0x80; PUSH1 0x00; PUSH1 0x80; CALL; STOP` (calls ecrecover).
2. Call `eth_estimateGas` → returns estimate X (ecrecover CALL costs 100 gas, warm).
3. Submit transaction with `gas_limit = X`.
4. In ZK proving mode, ecrecover CALL costs 2600 gas (cold, 2500 more than estimated).
5. Transaction reverts with out-of-gas; user pays `X * gas_price` for a failed transaction.
6. Differential assertion: `assert_eq!(gas_used_forward, gas_used_zk)` fails by ≥ 2500.

/// Check that gas and native used is the same in simulation and actual execution
#[test]
fn test_simulation_gas_and_native_used() {
    let mut tester = TestingFramework::new();
    let wallet = tester.prefunded_random_signer();
    let target_address = common_target_address();

    let tx = TxEip1559 {
        chain_id: 37u64,
        nonce: 0,
        max_fee_per_gas: 1000,
        max_priority_fee_per_gas: 1000,
        gas_limit: 1_000_000,
        to: TxKind::Call(target_address),
        value: U256::from(100),
        input: Default::default(),
        access_list: Default::default(),
    };

    let tx_for_simulation = ZKsyncTxEnvelope::from_eth_tx(tx.clone(), wallet.clone());

    // We use a very low native per gas ratio to force the transaction to require extra gas
    let block_context = BlockContext {
        eip1559_basefee: U256::from(1000),
        native_price: U256::from(500),
        ..Default::default()
    };

    tester = tester.with_block_context(block_context.clone());
    let result_simulation = tester.simulate_block(vec![tx_for_simulation]);
    let tx_result_simulation = result_simulation.tx_results[0]
        .clone()
        .expect("Simulation must succeed");

    let tx = TxEip1559 {
        gas_limit: tx_result_simulation.gas_used,
        ..tx
    };
    let encoded = ZKsyncTxEnvelope::from_eth_tx(tx, wallet);

    tester = tester.with_block_context(block_context);
    let result_normal = tester.execute_block(vec![encoded]);
    let tx_result_normal = result_normal.tx_results[0]
        .clone()
        .expect("Normal execution must succeed");

    assert_eq!(
        tx_result_simulation.gas_used, tx_result_normal.gas_used,
        "Mismatch in gas used"
    );
    assert_eq!(
        tx_result_simulation.native_used, tx_result_normal.native_used,
        "Mismatch in native used"
    );
}
