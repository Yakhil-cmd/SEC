Audit Report

## Title
`spawn_neuron` Pre-Check Uses Stale 5% Worst-Case Modulation While Mission 70 Allows ‚àí10% ‚Äî (File: `rs/nns/governance/src/governance.rs`)

## Summary
`spawn_neuron` guards against spawning a neuron whose eventual ICP stake would fall below `neuron_minimum_stake_e8s` using a hardcoded 5% worst-case modulation factor. The Mission 70 algorithm, now live in production, defines a lower bound of ‚àí10% (‚àí1 000 permyriad). The pre-check therefore under-estimates the worst case by a factor of two, allowing neurons to be spawned whose `cached_neuron_stake_e8s` will be below the minimum once the actual modulation is applied seven days later.

## Finding Description
In `spawn_neuron`, the minimum-stake pre-check is:

```rust
// rs/nns/governance/src/governance.rs  L2664-2673
let least_possible_stake = (maturity_to_spawn as f64 * (1_f64 - 0.05)) as u64;

if least_possible_stake < economics.neuron_minimum_stake_e8s {
    return Err(GovernanceError::new_with_message(
        ErrorType::InsufficientFunds,
        "There isn't enough maturity to spawn a new neuron due to worst case maturity modulation.",
    ));
}
```

The hardcoded `0.05` (5%) was correct when the old CMC-based modulation range was `[‚àí500, +500]` permyriad (¬±5%). The Mission 70 algorithm defines a new range in `update_icp_xdr_rate_related_data.rs`:

```rust
// L47-50
pub(crate) const MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70: i64 = -1_000;  // ‚àí10%
pub(crate) const MATURITY_MODULATION_MAX_PERMYRIAD_MISSION_70: i64 =    200;  // +2%
```

`compute_maturity_modulation_permyriad` clamps its output to this range (L193-196), so ‚àí1 000 permyriad is a fully reachable, valid value. `maybe_spawn_neurons` reads `heap_data.maturity_modulation.current_value_permyriad` (L6427-6435), passes it through the sanity-check range `[MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70, MATURITY_MODULATION_MAX_PERMYRIAD_MISSION_70]` (L6438-6447), and then calls `apply_maturity_modulation(original_maturity, maturity_modulation)` (L6484-6487), which multiplies by `(10_000 + permyriad) / 10_000`. At ‚àí1 000 permyriad this is `9_000 / 10_000 = 0.90`.

The old `MIN_MATURITY_MODULATION_PERMYRIAD: i32 = -500` constant in `rs/nervous_system/governance/src/maturity_modulation/mod.rs` (L4) is never consulted by the pre-check; it is only used by `apply_maturity_modulation`'s caller-side validation in the old CMC path, which is no longer the active path.

**Exploit flow:**
1. Attacker controls a neuron with `maturity_e8s_equivalent` in the range `(neuron_minimum_stake_e8s / 0.90, neuron_minimum_stake_e8s / 0.95]`.
2. Calls `manage_neuron` ‚Üí `Spawn { percentage_to_spawn: 100 }` (unprivileged).
3. `spawn_neuron` computes `least_possible_stake = maturity √ó 0.95 ‚â• neuron_minimum_stake_e8s` ‚Äî check passes.
4. Child neuron is created in spawning state.
5. After 7 days, `maybe_spawn_neurons` applies the actual ‚àí10% modulation: `maturity √ó 0.90 < neuron_minimum_stake_e8s`.
6. `neuron.cached_neuron_stake_e8s` is set to the below-minimum value (L6517), violating the core governance invariant.

## Impact Explanation
The result is a live NNS neuron whose `cached_neuron_stake_e8s` is below `neuron_minimum_stake_e8s`. This violates the core governance invariant that all neurons must hold at least the minimum stake, corrupts voting-power accounting for that neuron, and can cause unexpected behavior in any governance code path that assumes the invariant holds (dissolve, merge, split, stake-refresh). This constitutes a significant NNS governance security impact with concrete protocol harm, matching the High bounty impact class: "Significant NNS governance security impact with concrete user or protocol harm."

## Likelihood Explanation
The Mission 70 modulation is already live and its range is [‚àí10%, +2%]. The daily speed limit is 30 permyriad/day, so reaching ‚àí500 permyriad (‚àí5%) from 0 takes ~17 days of sustained ICP price decline ‚Äî a realistic scenario. Any modulation value strictly below ‚àí500 permyriad triggers the bug. The entry path is a standard unprivileged `manage_neuron` ‚Üí `Spawn` call; the only precondition is owning a neuron with maturity in the vulnerable range, which any long-term NNS participant can accumulate.

## Recommendation
Replace the hardcoded `0.05` with the Mission 70 lower bound using integer arithmetic consistent with `apply_maturity_modulation`:

```rust
// Use the Mission 70 lower bound: -1_000 permyriad ‚Üí multiply by 9_000/10_000
let least_possible_stake = maturity_to_spawn
    .saturating_mul(
        (BASIS_POINTS_PER_UNITY as i64 + MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70)
            as u64,
    )
    / BASIS_POINTS_PER_UNITY as u64;
```

This mirrors the exact arithmetic in `apply_maturity_modulation` and eliminates floating-point imprecision. The constant `MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70` should be imported from `timer_tasks::update_icp_xdr_rate_related_data` so the pre-check automatically tracks any future range changes.

## Proof of Concept
**Setup:** NNS governance running with Mission 70 maturity modulation at ‚àí1 000 permyriad (‚àí10%). `neuron_minimum_stake_e8s = 100_000_000` (1 ICP).

**Steps:**
1. Neuron A has `maturity_e8s_equivalent = 105_263_158` (just above `100_000_000 / 0.95 = 105_263_157`).
2. Controller calls `manage_neuron` ‚Üí `Spawn { percentage_to_spawn: 100 }`.
3. `spawn_neuron` computes `least_possible_stake = (105_263_158 as f64 * 0.95) as u64 = 100_000_000` ‚Äî equals minimum, check passes.
4. Child neuron created in spawning state with `maturity_e8s_equivalent = 105_263_158`.
5. After 7 days, `maybe_spawn_neurons` calls `apply_maturity_modulation(105_263_158, -1_000)` ‚Üí `105_263_158 * 9_000 / 10_000 = 94_736_842`.
6. `neuron.cached_neuron_stake_e8s = 94_736_842 < 100_000_000 = neuron_minimum_stake_e8s`. Invariant violated.

A deterministic integration test or PocketIC test can reproduce this by: (a) setting `heap_data.maturity_modulation.current_value_permyriad = Some(-1_000)`; (b) spawning a neuron with the above maturity; (c) advancing time by 7 days; (d) asserting `cached_neuron_stake_e8s < neuron_minimum_stake_e8s`.
