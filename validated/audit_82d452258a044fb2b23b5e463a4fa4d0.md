### Title
`refund_unused_gas` Ignores EIP-7623 Floor Gas, Allowing Under-Payment of Transaction Fees - (`engine/src/engine.rs`)

### Summary

The `refund_unused_gas` function in `engine/src/engine.rs` does not account for the EIP-7623 floor gas when computing the sender's refund. The transaction validation path correctly enforces `max(intrinsic_gas, floor_gas)` as the minimum gas limit, but the refund path uses only the raw `gas_used` value returned by the EVM executor. When `gas_used < floor_gas`, the sender is refunded more than they should be, effectively paying less than the EIP-7623 minimum.

---

### Finding Description

Aurora Engine recently added EIP-7623 floor gas support (PR #1064, CHANGES.md line 14). The floor gas is computed in `NormalizedEthTransaction::floor_gas` and enforced during transaction validation: [1](#0-0) 

The validation correctly rejects any transaction whose `gas_limit < max(intrinsic_gas, floor_gas)`. However, after EVM execution, the refund is computed in `refund_unused_gas`: [2](#0-1) 

When `fixed_gas` is `None` (the normal non-Silo path), `gas_to_wei` uses `gas_used.into()` — the raw EVM gas consumed — with no reference to `floor_gas`. The `floor_gas` value computed at lines 1073–1075 is never passed into `refund_unused_gas`. [3](#0-2) 

The call site passes `fixed_gas` but not `floor_gas`. The floor gas is computed locally in `submit_with_args` and then discarded after the validation check.

The `floor_gas` function itself always uses `gas_transaction_call` as the base, and is only active when `config.has_floor_gas` is true (Prague hardfork): [4](#0-3) 

---

### Impact Explanation

**High — Theft of unclaimed yield (relayer fees).**

Consider a Prague-hardfork transaction with 1 000 non-zero calldata bytes calling a contract that does minimal computation:

| Quantity | Value |
|---|---|
| `floor_gas` | `1000 × 4 × 10 + 21 000 = 61 000` |
| `intrinsic_gas` | `21 000 + 1000 × 16 = 37 000` |
| `gas_limit` | `70 000` (valid: ≥ 61 000) |
| `gas_used` (EVM) | `37 000` |
| `prepaid_amount` | `70 000 × gas_price` |
| **Correct** `spent_amount` | `61 000 × gas_price` |
| **Actual** `spent_amount` | `37 000 × gas_price` |
| **Excess refund to sender** | `24 000 × gas_price` |

The relayer receives `gas_used × priority_fee` instead of `floor_gas × priority_fee`. The sender recovers `(floor_gas − gas_used) × effective_gas_price` that EIP-7623 requires them to pay. This is a direct, repeatable extraction of relayer yield by any unprivileged EVM user.

---

### Likelihood Explanation

**Medium.** Any unprivileged EVM user can craft a transaction with large calldata (maximising `floor_gas`) targeting a contract that performs minimal computation (keeping `gas_used` low). No special privileges, governance access, or external oracle are required. The Prague hardfork config must be active, which is the current production target per CHANGES.md. [5](#0-4) 

---

### Recommendation

Pass `floor_gas` into `refund_unused_gas` (or compute it inside) and replace the raw `gas_used` with `max(gas_used, floor_gas)` when `fixed_gas` is `None`:

```rust
// In refund_unused_gas, replace:
fixed_gas.map_or_else(|| gas_used.into(), EthGas::as_u256)

// With:
fixed_gas.map_or_else(
    || U256::from(gas_used.max(floor_gas)),
    EthGas::as_u256,
)
```

The `floor_gas` value should be threaded from the call site in `submit_with_args` where it is already computed.

---

### Proof of Concept

1. Deploy a minimal contract on a Prague-hardfork Aurora instance (e.g., a contract that does nothing but `STOP`).
2. Submit a transaction to that contract with 1 000 non-zero calldata bytes, `gas_limit = 70 000`, `gas_price = 1`.
3. Observe `gas_used ≈ 37 000` (intrinsic only).
4. Observe sender balance after execution: `initial − value − 37 000` instead of the correct `initial − value − 61 000`.
5. The sender retains `24 000 wei` that EIP-7623 requires them to pay, at the relayer's expense. [6](#0-5) [1](#0-0) [4](#0-3)

```rust
pub fn refund_unused_gas<I: IO>(
    io: &mut I,
    sender: &Address,
    gas_used: u64,
    gas_result: &GasPaymentResult,
    relayer: &Address,
    fixed_gas: Option<EthGas>,
) -> Result<(), GasPaymentError> {
    if gas_result.effective_gas_price.is_zero() {
        return Ok(());
    }

    let (refund, relayer_reward) = {
        let gas_to_wei = |price: U256| {
            fixed_gas
                .map_or_else(|| gas_used.into(), EthGas::as_u256)
                .checked_mul(price)
                .map(Wei::new)
                .ok_or(GasPaymentError::EthAmountOverflow)
        };

        let spent_amount = gas_to_wei(gas_result.effective_gas_price)?;
        let reward_amount = gas_to_wei(gas_result.priority_fee_per_gas)?;

        let refund = gas_result
            .prepaid_amount
            .checked_sub(spent_amount)
            .ok_or(GasPaymentError::EthAmountOverflow)?;

        (refund, reward_amount)
    };

    if !refund.is_zero() {
        add_balance(io, sender, refund)?;
    }

    if !relayer_reward.is_zero() {
        add_balance(io, relayer, relayer_reward)?;
    }

    Ok(())
}
```

```rust
    pub fn floor_gas(&self, config: &aurora_evm::Config) -> Result<u64, Error> {
        if config.has_floor_gas {
            let num_zero_bytes = u64::try_from(self.data.iter().filter(|b| **b == 0).count())
                .map_err(|_e| Error::IntegerConversion)?;
            let data_len = u64::try_from(self.data.len()).map_err(|_e| Error::IntegerConversion)?;
            let num_non_zero_bytes = data_len
                .checked_sub(num_zero_bytes)
                .ok_or(Error::GasOverflow)?;

            let base_gas = config.gas_transaction_call;
            let tokens_in_calldata = num_non_zero_bytes
                .checked_mul(4)
                .and_then(|gas| gas.checked_add(num_zero_bytes))
                .ok_or(Error::GasOverflow)?;

            tokens_in_calldata
                .checked_mul(config.total_cost_floor_per_token)
                .and_then(|gas| gas.checked_add(base_gas))
                .ok_or(Error::GasOverflow)
        } else {
            Ok(0)
        }
    }
```
