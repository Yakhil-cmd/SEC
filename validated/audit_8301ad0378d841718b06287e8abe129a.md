### Title
SNS Governance `disburse_maturity` Defaults Maturity to Caller's Account Instead of Neuron Owner's Account When No Destination Is Specified - (File: `rs/sns/governance/src/governance.rs`)

---

### Summary

In SNS governance, `disburse_maturity` permits any principal holding `NeuronPermissionType::DisburseMaturity` permission to disburse a neuron's maturity. When the caller omits the optional `to_account` field, the function silently defaults the destination to `caller.0` — the caller's own principal account — rather than the neuron owner's account. This is a direct analog of the FrankenDAO H-4 bug: a delegated caller (non-owner) triggers a state-modifying operation, and the financial effect is applied to the caller instead of the owner.

---

### Finding Description

`disburse_maturity` in `rs/sns/governance/src/governance.rs` first checks that the caller holds `DisburseMaturity` permission on the neuron:

```rust
neuron.check_authorized(caller, NeuronPermissionType::DisburseMaturity)?;
``` [1](#0-0) 

This permission can be granted to any arbitrary principal by the neuron owner via `AddNeuronPermissions`. After the authorization check, when no destination account is provided, the code defaults to the caller's account:

```rust
// If no account was provided, transfer to the caller's account.
let to_account: Account = match disburse_maturity.to_account.as_ref() {
    None => Account {
        owner: caller.0,   // ← caller, not neuron owner
        subaccount: None,
    },
    ...
};
``` [2](#0-1) 

The maturity is then deducted from the neuron and queued for disbursement to that account: [3](#0-2) 

The SNS proto documentation confirms the default is the caller's account, not the owner's:

> "If an account is not specified, the caller's account will be used." [4](#0-3) 

By contrast, the NNS governance `disburse_maturity` restricts the operation to the neuron controller only (`CallerIsNotNeuronController` error), so the caller is always the owner there: [5](#0-4) 

The SNS governance has no such restriction — it allows any principal with `DisburseMaturity` permission to call the function, making the caller ≠ owner scenario reachable.

---

### Impact Explanation

A principal granted `DisburseMaturity` permission (but who is not the neuron owner) can call `manage_neuron` with `DisburseMaturity { percentage_to_disburse: 100, to_account: None }`. The maturity is deducted from the victim's neuron and, after the 7-day delay, minted and transferred to the attacker's account. The neuron owner loses their accumulated maturity rewards with no recourse. This is a direct theft of governance token rewards.

---

### Likelihood Explanation

The neuron owner must have previously granted `DisburseMaturity` permission to the attacker. This is a realistic scenario: neuron owners may grant this permission to a hot key or a third-party service for operational convenience (e.g., to automate disbursals to a specified account), not anticipating that the grantee can exploit the default-to-caller behavior to redirect maturity to themselves. The `NeuronPermissionType` system in SNS is explicitly designed for fine-grained delegation, making such grants common.

---

### Recommendation

When `to_account` is `None`, default to the neuron **controller's** account (the neuron owner), not the caller's account. The caller should be required to explicitly specify a destination if they are not the neuron owner. Alternatively, restrict `disburse_maturity` so that only the neuron controller can invoke it without specifying a destination, mirroring the NNS governance behavior.

---

### Proof of Concept

1. **Setup**: Victim (neuron owner, principal `V`) creates an SNS neuron with 1,000,000 e8s of accumulated maturity. Victim grants `DisburseMaturity` permission to attacker principal `A` (e.g., for a legitimate automation purpose).

2. **Attack**: Attacker `A` calls `manage_neuron` on the SNS governance canister:
   ```
   ManageNeuron {
     subaccount: <victim_neuron_subaccount>,
     command: DisburseMaturity {
       percentage_to_disburse: 100,
       to_account: None,   // ← no destination specified
     }
   }
   ```
   Caller identity: `A`.

3. **Execution**: `disburse_maturity` is called with `caller = A`. Authorization passes because `A` holds `DisburseMaturity` permission. Since `to_account` is `None`, `to_account = Account { owner: A.0, subaccount: None }`. [6](#0-5) 

4. **Result**: After the 7-day disbursement delay, 1,000,000 e8s of SNS tokens are minted and transferred to attacker `A`'s account. Victim `V`'s neuron maturity is reduced to 0. The victim has no way to recover the maturity.
