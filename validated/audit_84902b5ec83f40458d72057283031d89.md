### Title
Staged Upgrade State Not Cleared After Direct `upgrade()` Allows Unprivileged Contract Downgrade — (`engine/src/lib.rs`, `engine/src/contract_methods/admin.rs`)

---

### Summary

The Aurora Engine exposes two distinct upgrade paths: a direct `upgrade()` function and a two-step `stage_upgrade()` + `deploy_upgrade()` flow. When the owner uses `upgrade()` directly after having previously called `stage_upgrade()`, the staged code (`CODE_KEY`) and its timelock (`CODE_STAGE_KEY`) are never cleared. Because `deploy_upgrade()` has no access-control check, any unprivileged caller can subsequently invoke it to force the contract back to the old staged code, overriding the owner's intended upgrade.

---

### Finding Description

**`stage_upgrade()`** stores the new WASM binary under `CODE_KEY` and writes the unlock block height to `CODE_STAGE_KEY`: [1](#0-0) 

**`upgrade()`** deploys code directly via a NEAR promise batch. It performs no storage cleanup of `CODE_KEY` or `CODE_STAGE_KEY`: [2](#0-1) 

**`deploy_upgrade()`** has **no `require_owner_only` guard**. It only checks that the contract is running and that the current block height exceeds the staged index, then deploys whatever is stored at `CODE_KEY`: [3](#0-2) 

The `CODE_STAGE_KEY` is removed after `deploy_upgrade()` succeeds, but `CODE_KEY` is never removed by either `upgrade()` or `deploy_upgrade()`. The critical gap is that `upgrade()` does not remove `CODE_KEY` or `CODE_STAGE_KEY`, leaving the staged state permanently active. [4](#0-3) 

---

### Impact Explanation

**Scenario:**

1. Owner calls `stage_upgrade(code_v2)` → `CODE_KEY = code_v2`, `CODE_STAGE_KEY = block + delay`.
2. Owner discovers a critical bug in `code_v2` and calls `upgrade(code_v3)` directly to deploy a patched version immediately. The contract now runs `code_v3`.
3. `CODE_KEY` still holds `code_v2`; `CODE_STAGE_KEY` still holds the old delay block (which may already have passed).
4. Any unprivileged user calls `deploy_upgrade()`. The contract is redeployed with `code_v2` — the vulnerable version the owner explicitly tried to avoid.

If `code_v2` contains a vulnerability (e.g., a fund-draining bug, a reentrancy, or a broken accounting invariant), the attacker can then exploit it immediately after forcing the downgrade. This maps directly to **Critical: direct theft of user funds** or **Critical: permanent freezing of funds**, depending on the nature of the vulnerability in the staged code.

---

### Likelihood Explanation

The scenario is realistic:

- The owner may stage an upgrade, then discover a flaw and use the emergency `upgrade()` path to deploy a fix, not realizing the staged state persists.
- `deploy_upgrade()` is callable by any account with no restrictions beyond the block-height check.
- An attacker monitoring the chain can detect the presence of `CODE_STAGE_KEY` in storage and call `deploy_upgrade()` as soon as the delay passes (or immediately if it already has). [5](#0-4) 

---

### Recommendation

The `upgrade()` function should clear both `CODE_KEY` and `CODE_STAGE_KEY` from storage before or after deploying the new code, so that any pending staged upgrade is cancelled:

```rust
// Inside upgrade(), before or after creating the promise:
io.remove_storage(&storage::bytes_to_key(KeyPrefix::Config, CODE_KEY));
io.remove_storage(&storage::bytes_to_key(KeyPrefix::Config, CODE_STAGE_KEY));
```

Alternatively, `deploy_upgrade()` should be restricted to the owner, consistent with the access control applied to `stage_upgrade()`. [6](#0-5) 

---

### Proof of Concept

1. Deploy Aurora Engine with `upgrade_delay_blocks = 1`.
2. Owner calls `stage_upgrade(code_v2)` at block N → `CODE_KEY = code_v2`, `CODE_STAGE_KEY = N+1`.
3. Owner calls `upgrade(code_v3)` at block N+1 → contract now runs `code_v3`. `CODE_KEY` and `CODE_STAGE_KEY` are untouched.
4. Attacker (any account) calls `deploy_upgrade()` at block N+2 (> N+1). The check `io.block_height() <= index` passes. `Runtime::self_deploy(&CODE_KEY)` redeploys `code_v2`. The contract now runs the old, potentially vulnerable code. [5](#0-4) [7](#0-6)

pub fn stage_upgrade<I: IO + Copy, E: Env>(io: I, env: &E) -> Result<(), ContractError> {
    with_hashchain(io, env, function_name!(), |mut io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;
        let delay_block_height = env.block_height() + state.upgrade_delay_blocks;
        require_owner_only(&state, &env.predecessor_account_id())?;
        io.read_input_and_store(&storage::bytes_to_key(KeyPrefix::Config, CODE_KEY));
        io.write_storage(
            &storage::bytes_to_key(KeyPrefix::Config, CODE_STAGE_KEY),
            &delay_block_height.to_le_bytes(),
        );
        Ok(())
    })
}

pub fn upgrade<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<(), ContractError> {
    let state = state::get_state(&io)?;
    require_running(&state)?;
    require_owner_only(&state, &env.predecessor_account_id())?;

    let input = io.read_input().to_vec();
    let (code, state_migration_gas) = match UpgradeParams::try_from_slice(&input) {
        Ok(args) => (
            args.code,
            args.state_migration_gas
                .map_or(GAS_FOR_STATE_MIGRATION, NearGas::new),
        ),
        Err(_) => (input, GAS_FOR_STATE_MIGRATION), // Backward compatibility
    };

    let target_account_id = env.current_account_id();
    let batch = PromiseBatchAction {
        target_account_id,
        actions: vec![
            PromiseAction::DeployContract { code },
            PromiseAction::FunctionCall {
                name: "state_migration".to_string(),
                args: vec![],
                attached_yocto: ZERO_YOCTO,
                gas: state_migration_gas,
            },
        ],
    };
    let promise_id = handler.promise_create_batch(&batch);

    handler.promise_return(promise_id);

    Ok(())
}

    /// Deploy staged upgrade.
    #[unsafe(no_mangle)]
    pub extern "C" fn deploy_upgrade() {
        // This function is intentionally not implemented in `contract_methods`
        // because it only makes sense in the context of the NEAR runtime.
        let mut io = Runtime;
        let state = state::get_state(&io).sdk_unwrap();
        require_running(&state)
            .map_err(ContractError::msg)
            .sdk_unwrap();
        let index = internal_get_upgrade_index();
        if io.block_height() <= index {
            sdk::panic_utf8(errors::ERR_NOT_ALLOWED_TOO_EARLY);
        }
        Runtime::self_deploy(&bytes_to_key(KeyPrefix::Config, CODE_KEY));
        io.remove_storage(&bytes_to_key(KeyPrefix::Config, CODE_STAGE_KEY));
    }
