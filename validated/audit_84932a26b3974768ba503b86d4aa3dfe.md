Audit Report

## Title
Missing Proposal-Caller Binding in `settle_neurons_fund_participation` Allows Any SNS Swap to Settle Another SNS's Neurons' Fund Participation - (File: `rs/nns/governance/src/governance.rs`)

## Summary
`NNS Governance.settle_neurons_fund_participation` verifies only that the caller is *a* valid SNS swap canister, never that it is the swap canister *associated with the supplied `nns_proposal_id`*. A legitimate but malicious SNS swap canister can call this function with a victim SNS's proposal ID and an attacker-controlled `sns_governance_canister_id`, causing NNS Governance to mint and send Neurons' Fund ICP to the attacker's canister and permanently locking the victim SNS's settlement state via the idempotency path.

## Finding Description

**Root cause — authorization check is proposal-agnostic.**

`settle_neurons_fund_participation` performs two authorization steps. First it validates the request fields via `ValidatedSettleNeuronsFundParticipationRequest::try_from`, which only checks that required fields are present and well-formed — it does not bind `sns_governance_canister_id` to the proposal's actual SNS instance. [1](#0-0) 

Second, it checks that the caller is *any* deployed SNS swap canister via `is_canister_id_valid_swap_canister_id`. The developer comment at line 7018 explicitly acknowledges the gap: *"Note that a Swap could settle each other's participation."* [2](#0-1) 

`is_canister_id_valid_swap_canister_id` uses `.any()` — it confirms the caller is *some* swap canister, not the one that corresponds to `nns_proposal_id`: [3](#0-2) 

**Attacker-controlled mint destination.**

After the authorization check passes, the function extracts `sns_governance_canister_id` directly from the attacker-supplied request and passes it verbatim to `mint_to_sns_governance`. There is no lookup of the proposal's actual SNS governance canister for comparison: [4](#0-3) 

The `Committed` struct in the proto and generated Rust code confirms `sns_governance_canister_id` is a free field with no cross-validation: [5](#0-4) 

**Attacker-controlled participation amount.**

`total_direct_participation_icp_e8s` from the request is used as the sole input to `from_initial_participation`, which computes the final NF participation amount. Setting this to `u64::MAX` maximizes the ICP minted (bounded by the NF maturity reserved for the victim proposal): [6](#0-5) 

**Idempotency permanently locks the victim.**

Once settled, `previously_computed_final_neurons_fund_participation` is set. All subsequent calls for the same proposal ID return the attacker-computed result immediately, preventing the legitimate swap canister from ever re-settling with correct parameters: [7](#0-6) 

The design comment in `swap.proto` acknowledges the trust assumption that all SNS swap canisters are honest because they are deployed by SNS-W and upgraded via NNS proposals — but this assumption does not hold when one SNS project acts adversarially against another: [8](#0-7) 

## Impact Explanation

A malicious SNS swap canister can redirect the full Neurons' Fund ICP reserved for a victim SNS's proposal to the attacker's own governance canister. The stolen amount equals the NF maturity reserved for the victim swap, which for large SNS launches can be substantial. Additionally, the victim SNS's proposal lifecycle is permanently set to `Committed` with attacker-controlled parameters, meaning the victim's `finalize_swap` will create incorrectly sized SNS neurons for all Neurons' Fund participants and cannot be corrected without an NNS hotfix. This constitutes **theft and permanent loss of ICP/governance assets** and **significant SNS framework security impact with concrete user and protocol harm**, matching the High ($2,000–$10,000) impact tier. If the reserved NF participation for the targeted swap exceeds $1M in ICP value, the impact escalates to Critical.

## Likelihood Explanation

The attacker must control a legitimate SNS swap canister, which requires a successful `CreateServiceNervousSystem` NNS proposal — a meaningful but not prohibitive barrier satisfied by any existing SNS project. No additional setup is required beyond having a deployed SNS. The attack is a single inter-canister call. The attack window opens when the victim SNS's swap opens (NF maturity is reserved) and closes when the victim's `finalize_swap` calls `settle_neurons_fund_participation`; this window typically spans days to weeks. The attack executes in one block and is not detectable in advance.

## Recommendation

After confirming the caller is a valid swap canister, additionally assert that the caller is the swap canister *associated with the supplied `nns_proposal_id`*. The `list_deployed_snses` response already contains `nns_proposal_id` per SNS instance. Concretely:

1. In `is_canister_id_valid_swap_canister_id` (or at the call site), change the `.any()` check to find the specific `DeployedSns` whose `nns_proposal_id` matches `request.nns_proposal_id`.
2. Assert `caller == matched_sns.swap_canister_id`.
3. Additionally, validate that `sns_governance_canister_id` in the `Committed` request matches `matched_sns.governance_canister_id` from SNS-W, rather than accepting it as a free caller-supplied field.

## Proof of Concept

1. SNS B has an open swap with `nns_proposal_id = 42`; NF has reserved 100,000 ICP of maturity.
2. Attacker controls SNS A with swap canister `swap_a` and governance canister `gov_a` (legitimately deployed by SNS-W).
3. `swap_a` calls `NNS Governance.settle_neurons_fund_participation` with:
   ```
   nns_proposal_id = 42
   result = Committed {
     sns_governance_canister_id = gov_a,
     total_direct_participation_icp_e8s = u64::MAX,
     total_neurons_fund_participation_icp_e8s = u64::MAX,
   }
   ```
4. NNS Governance confirms `swap_a` is a valid swap canister — it is. No proposal-caller binding check exists.
5. NNS Governance computes maximum NF participation for proposal 42, mints ICP, and sends it to `gov_a`.
6. Proposal 42's lifecycle is set to `Committed`; `previously_computed_final_neurons_fund_participation` is now `Some(attacker_result)`.
7. SNS B's legitimate swap canister later calls `settle_neurons_fund_participation` for proposal 42 and hits the idempotency path at lines 7166–7174, receiving the attacker-computed result. SNS B's NF neurons are created with wrong amounts; the stolen ICP is unrecoverable without an NNS hotfix.

A deterministic integration test can reproduce this by deploying two SNS instances in a PocketIC environment, calling `settle_neurons_fund_participation` from SNS A's swap canister with SNS B's proposal ID, and asserting that ICP is minted to SNS A's governance canister and that SNS B's subsequent settlement call returns the attacker-controlled snapshot.

    for ValidatedSettleNeuronsFundParticipationRequest

    type Error = GovernanceError;

    /// Collect defects of a SettleNeuronsFundParticipationRequest request into Err,
    /// or return validated data in the Ok case.
    fn try_from(request: SettleNeuronsFundParticipationRequest) -> Result<Self, Self::Error> {
        // Validate request.nns_proposal_id
        let validated_proposal_id = {
            if let Some(id) = request.nns_proposal_id {
                Ok(ProposalId { id })
            } else {
                Err(vec!["Request.nns_proposal_id is unspecified.".to_string()])
            }
        };
        let request_str = format!("{:#?}", &request);
        // Validate request.result
        let swap_result = if let Some(result) = request.result {
            SwapResult::try_from(result).map_err(|err| vec![err])
        } else {
            Err(vec![
                "Request.result is unspecified (must be either Committed or Aborted).".to_string(),
            ])
        };
        // Compose the validated results
        match (validated_proposal_id, swap_result) {
            (Ok(nns_proposal_id), Ok(swap_result)) => Ok(Self {
                request_str,
                nns_proposal_id,
                swap_result,
            }),
            (Ok(_), Err(proposal_type_defects)) => Err(proposal_type_defects),
            (Err(proposal_id_defects), Ok(_)) => Err(proposal_id_defects),
            (Err(proposal_id_defects), Err(proposal_type_defects)) => {
                let defects = proposal_id_defects
                    .into_iter()
                    .chain(proposal_type_defects)
                    .collect();
                Err(defects)
            }
        }
            .map_err(|defects| {
                GovernanceError::new_with_message(
                    ErrorType::InvalidCommand,
                    format!(
                        "SettleNeuronsFundParticipation is invalid for the following reason(s):\n  - {}",
                        defects.join("\n  - "),
                    ),
                )
            })
    }
