### Title
Silo Fixed-Gas Requirement Can Permanently Freeze User ETH Balances Below the Gas Threshold - (`engine/src/engine.rs`, `engine/src/contract_methods/silo/mod.rs`)

### Summary
In Aurora's Silo mode, every EVM transaction is charged a mandatory fixed gas amount (`fixed_gas`). Because the only mechanism to withdraw ETH from the Aurora EVM is the `ExitToEthereum` precompile — which is invoked via an EVM transaction — any user whose EVM ETH balance falls below `fixed_gas × gas_price` is permanently unable to withdraw their remaining funds. There is no NEAR-level escape hatch that bypasses the EVM gas requirement for ETH held inside the EVM state.

### Finding Description

Aurora's Silo mode introduces a configurable `fixed_gas` parameter stored and retrieved via `get_fixed_gas` / `set_fixed_gas` in `engine/src/contract_methods/silo/mod.rs`. [1](#0-0) 

This fixed gas amount is applied to every EVM transaction submitted through the engine. The engine enforces it during execution: if the sender's balance cannot cover `fixed_gas × gas_price`, the transaction fails with `OutOfFund` (confirmed by `test_transfer_insufficient_balance_fee` in `engine-tests/src/tests/silo.rs`). [2](#0-1) 

The only way for a user to withdraw ETH from the Aurora EVM to Ethereum is by calling the `ExitToEthereum` precompile (flag `0x0`) in `engine-precompiles/src/native.rs`, which uses `context.apparent_value` as the withdrawal amount. [3](#0-2) 

This precompile call is itself an EVM transaction and is therefore subject to the same fixed-gas deduction. There is no NEAR-level contract method that directly reads and transfers ETH from the EVM state without going through an EVM transaction. The NEAR-level `withdraw` function in `engine/src/contract_methods/connector.rs` operates only on the NEP-141 ETH-on-NEAR balance, not on ETH held inside the EVM. [4](#0-3) 

### Impact Explanation

A user whose EVM ETH balance `B` satisfies `B < fixed_gas × gas_price` cannot execute any EVM transaction. This includes the `ExitToEthereum` and `ExitToNear` precompile calls that are the only paths to recover EVM-held ETH. The ETH is frozen inside the EVM state with no user-accessible recovery path. If the Silo operator does not reduce `fixed_gas` or disable Silo mode, the freeze is permanent.

**Impact: High — Temporary (potentially permanent) freezing of user funds.**

### Likelihood Explanation

This condition arises through ordinary usage:

1. A user bridges ETH to Aurora EVM (via `ft_transfer_call` / `FtOnTransfer`).
2. The user executes several EVM transactions in Silo mode, each consuming `fixed_gas × gas_price` ETH.
3. After enough transactions, the user's remaining balance is less than `fixed_gas × gas_price`.
4. The user attempts to withdraw the remainder via `ExitToEthereum` — the transaction fails with `OutOfFund`.

No special attacker action is required; this is a natural consequence of normal usage. The Silo fixed-gas value is set by the owner, but users have no visibility into how many transactions will exhaust their balance below the threshold. [5](#0-4) 

### Recommendation

Add a special-case exemption in the fixed-gas enforcement logic: if the EVM transaction is a call to the `ExitToEthereum` or `ExitToNear` precompile address and the sender's entire balance is being withdrawn (i.e., `value == balance`), waive or reduce the fixed-gas requirement. This mirrors the M-03 recommendation of allowing users to withdraw their full balance even when it falls below the minimum threshold, and is safe because the withdrawal burns the sender's tokens rather than leaving residual value.

### Proof of Concept

1. Silo mode is enabled with `fixed_gas = G` and `gas_price = P` (so each tx costs `G × P` ETH).
2. User ALICE bridges `N` ETH to her Aurora EVM address, where `N = k × (G × P) + R` and `0 < R < G × P`.
3. ALICE executes `k` EVM transactions, each consuming `G × P` ETH for gas.
4. ALICE's remaining balance is `R` ETH.
5. ALICE calls the `ExitToEthereum` precompile with `value = R`. The engine deducts `G × P` for fixed gas first; since `R < G × P`, the transaction fails with `OutOfFund`.
6. ALICE's `R` ETH is permanently inaccessible. [6](#0-5) [7](#0-6)

/// Return gas amount per transaction.
pub fn get_fixed_gas<I: IO>(io: &I) -> Option<EthGas> {
    let key = fixed_gas_key();
    io.read_storage(&key)
        .and_then(|bytes| bytes.to_value().ok())
}

/// Set an amount of gas per transaction.
pub fn set_fixed_gas<I: IO>(io: &mut I, gas: Option<EthGas>) {
    let key = fixed_gas_key();

    if let Some(gas) = gas {
        io.write_borsh(&key, &gas);
    } else {
        io.remove_storage(&key);
    }
}

#[test]
fn test_transfer_insufficient_balance_fee() {
    const HALF_FIXED_GAS: EthGas = EthGas::new(10u64.pow(18) / 2);

    let (mut runner, mut source_account, receiver) = initialize_transfer();
    let sender = utils::address_from_secret_key(&source_account.secret_key);

    set_silo_params(&mut runner, Some(SILO_PARAMS_ARGS));

    // validate pre-state
    validate_address_balance_and_nonce(&runner, sender, INITIAL_BALANCE, INITIAL_NONCE.into())
        .unwrap();
    validate_address_balance_and_nonce(&runner, receiver, ZERO_BALANCE, INITIAL_NONCE.into())
        .unwrap();

    // We want to leave TRANSFER_AMOUNT + HALF_FIXED_GAS on the balance.
    let amount = INITIAL_BALANCE
        - FIXED_GAS * ONE_GAS_PRICE
        - TRANSFER_AMOUNT
        - HALF_FIXED_GAS * ONE_GAS_PRICE;
    // attempt transfer
    let result = runner
        .submit_with_signer(&mut source_account, |nonce| {
            // try to transfer more than we have
            utils::transfer_with_price(receiver, amount, nonce, ONE_GAS_PRICE.raw())
        })
        .unwrap();
    assert!(matches!(result.status, TransactionStatus::Succeed(_)));

    // validate post-state
    validate_address_balance_and_nonce(
        &runner,
        sender,
        TRANSFER_AMOUNT + HALF_FIXED_GAS * ONE_GAS_PRICE,
        // the nonce is still incremented even though the transfer failed
        (INITIAL_NONCE + 1).into(),
    )
    .unwrap();
    validate_address_balance_and_nonce(&runner, receiver, amount, INITIAL_NONCE.into()).unwrap();

    // attempt transfer
    let result = runner
        .submit_with_signer(&mut source_account, |nonce| {
            // try to transfer more than we have
            utils::transfer_with_price(receiver, TRANSFER_AMOUNT, nonce, ONE_GAS_PRICE.raw())
        })
        .unwrap();
    assert!(matches!(result.status, TransactionStatus::OutOfFund));
}
