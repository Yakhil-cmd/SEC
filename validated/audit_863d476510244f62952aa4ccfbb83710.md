### Title
Missing 7-Day Minting Rate Limit Enforcement for `MintSnsTokens` Proposals at Execution Time - (`File: rs/sns/governance/src/proposal.rs`)

### Summary
The SNS governance canister intentionally disables the 7-day upper-bound rate limit for `MintSnsTokens` proposals. The `recent_amount_total_upper_bound_tokens` implementation for `MintSnsTokens` unconditionally returns `Decimal::MAX` (effectively no limit), bypassing the treasury-valuation-based cap that is correctly enforced for `TransferSnsTreasuryFunds`. This means an SNS governance majority can pass an unlimited number of `MintSnsTokens` proposals in rapid succession, minting an unbounded quantity of SNS tokens to any arbitrary principal, with no on-chain rate-limit check at either proposal submission or execution time.

### Finding Description
The `TokenProposalAction` trait defines `recent_amount_total_upper_bound_tokens` to cap how many tokens can be minted or transferred within a 7-day window. For `TransferSnsTreasuryFunds`, this is correctly wired to `transfer_sns_treasury_funds_7_day_total_upper_bound_tokens`, which enforces a treasury-valuation-based cap (25% of medium treasury, 300,000 XDR cap for large treasury, etc.).

For `MintSnsTokens`, the implementation is explicitly stubbed out with a `TODO(NNS1-2982)` comment:

```rust

### Impact Explanation
An SNS governance majority (which can be a single whale neuron holder, as demonstrated in the integration tests) can submit and pass an unlimited number of `MintSnsTokens` proposals within any 7-day window, minting an arbitrary total quantity of SNS tokens to any principal. This inflates the SNS token supply without bound, diluting all existing token holders. The `MintSnsTokens` action is classified as a `Critical` proposal under `TreasuryAssetManagement`, meaning it requires a higher voting threshold — but once that threshold is met, there is no on-chain cap on the minted amount or frequency. The impact is equivalent to the BridgeEscrow rug-pull: a privileged role (governance majority) can drain value from all token holders by inflating supply to themselves. [6](#0-5) 

### Likelihood Explanation
The entry path is a standard SNS governance proposal submitted via `manage_neuron` by any principal holding sufficient voting power. No special access, leaked keys, or admin privileges beyond a governance majority are required. The `MintSnsTokens` proposal type is a native, publicly documented SNS action. The disabled rate limit is a known, intentional temporary state (tracked as `TODO(NNS1-2982)`), meaning it is present in the current production canister WASM. Any SNS where a single neuron or coordinated group holds majority voting power can exploit this immediately. [7](#0-6) 

### Recommendation
1. **Immediately uncomment** the `mint_sns_tokens_7_day_total_upper_bound_tokens` import and the `recent_amount_total_upper_bound_tokens` implementation for `MintSnsTokens` in `rs/sns/governance/src/proposal.rs`.
2. **Add an execution-time check** in `perform_mint_sns_tokens` analogous to `transfer_sns_treasury_funds_amount_is_small_enough_at_execution_time_or_err` to guard against race conditions between proposal validation and execution.
3. **Remove the `doomed_make_proposal_result.unwrap()` workaround** in the integration test and restore the assertion that the second proposal is rejected.

### Proof of Concept
1. Deploy an SNS where a single principal controls a majority neuron.
2. Submit `MintSnsTokens` proposal #1: mint `u64::MAX / E8` SNS tokens to attacker's principal. Pass and execute it.
3. Immediately submit `MintSnsTokens` proposal #2 with the same amount. Pass and execute it.
4. Repeat indefinitely within the same 7-day window.
5. Each proposal succeeds because `recent_amount_total_upper_bound_tokens` returns `Decimal::MAX`, so the check `proposal_amount_tokens > allowance_remainder_tokens` is never true.

The attacker mints unbounded SNS tokens, diluting all other holders to near-zero ownership and extracting the full economic value of the SNS treasury indirectly through token inflation. [8](#0-7)

            .amount_e8s
