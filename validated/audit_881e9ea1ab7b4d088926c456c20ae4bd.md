Audit Report

## Title
Unchecked Application-Level Return Value in SNS Governance Generic Proposal Execution Marks Failed Proposals as Executed - (File: rs/sns/governance/src/canister_control.rs)

## Summary
`perform_execute_generic_nervous_system_function_call` in the SNS governance canister makes an inter-canister call to a target canister whose documented method signature is `Result<(), String>`. The function correctly handles transport-level failures but unconditionally treats any reply — including an application-level `Err(String)` — as `Ok(())`. This causes any passed `ExecuteGenericNervousSystemFunction` proposal whose target method returns an error to be permanently and irrevocably marked as successfully executed in governance state, even though the intended on-chain action was never performed.

## Finding Description
The proto file documents the required target method signature:

```
// The signature of the method must be equivalent to the following:
// <method_name>(proposal_data: ProposalData) -> Result<(), String>.
optional string target_method_name = 3;
``` [1](#0-0) 

The execution function at lines 302–312 of `canister_control.rs` discards the reply bytes entirely:

```rust
Ok(_reply) => {
    // TODO: Do something with reply. ...
    // For now, any reply is considered a success.
    Ok(())
}
``` [2](#0-1) 

The sibling validate-and-render function in the same file demonstrates the correct pattern is known and already implemented for the validation path:

```rust
Ok(reply) => {
    let result = Decode!(&reply, Result<String,String>);
    ...
}
``` [3](#0-2) 

The `Ok(())` result propagates up through `perform_execute_generic_nervous_system_function` → `perform_action` → `set_proposal_execution_status`: [4](#0-3) [5](#0-4) [6](#0-5) 

`set_proposal_execution_status` then sets `executed_timestamp_seconds` to the current time, permanently recording the proposal as successfully executed with no `failed_timestamp_seconds` set.

## Impact Explanation
This is a **High** severity SNS governance integrity issue. Any SNS using generic nervous system functions with a fallible target method will have its governance state permanently corrupted: proposals are marked executed when the intended action was never performed. Observers, dapps, and downstream canisters querying proposal status via `get_proposal` will see `executed_timestamp_seconds > 0` and `failed_timestamp_seconds == 0`, indicating success. Systems gating further actions on proposal execution status will proceed incorrectly. There is no audit trail of the actual failure, and the only recourse is submitting a new proposal requiring a full voting cycle. This matches the allowed impact: **High — Significant SNS security impact with concrete user or protocol harm**.

## Likelihood Explanation
`ExecuteGenericNervousSystemFunction` is the primary extensibility mechanism for every deployed SNS. The documented and expected interface for target methods is `Result<(), String>`, meaning any SNS whose target canister method can return `Err(String)` — which is the standard interface — is affected. This is reachable by any SNS token holder with sufficient voting power to pass a proposal, which is an unprivileged role by design. The bug is triggered by normal governance operation, not by an adversarial majority attack.

## Recommendation
Decode the reply bytes and propagate application-level errors as governance failures, mirroring the pattern already used in `perform_execute_generic_nervous_system_function_validate_and_render_call`:

```rust
Ok(reply) => {
    match candid::Decode!(&reply, Result<(), String>) {
        Ok(Ok(())) => Ok(()),
        Ok(Err(app_err)) => Err(GovernanceError::new_with_message(
            ErrorType::External,
            format!("Target canister returned application error: {app_err}"),
        )),
        Err(decode_err) => Err(GovernanceError::new_with_message(
            ErrorType::External,
            format!("Failed to decode reply from target canister: {decode_err}"),
        )),
    }
}
``` [2](#0-1) 

## Proof of Concept
1. Register a generic nervous system function whose target method is:
   ```rust
   #[update]
   fn my_action(_payload: Vec<u8>) -> Result<(), String> {
       Err("intentional failure".to_string())
   }
   ```
2. Submit and pass an `ExecuteGenericNervousSystemFunction` proposal targeting this method.
3. After execution, query the proposal via `get_proposal`. Observe `executed_timestamp_seconds > 0` and `failed_timestamp_seconds == 0`.
4. The target canister's action was never performed, but governance permanently records the proposal as successfully executed.
5. A deterministic PocketIC integration test can register such a function, pass the proposal, and assert the incorrect terminal state, confirming the bug without mainnet interaction.
