### Title
NNS Governance `disburse_neuron` Unconditionally Zeros `neuron_fees_e8s` Without Burning, Leaving ICP Permanently Stuck in Neuron Subaccount - (File: `rs/nns/governance/src/governance.rs`)

---

### Summary

In `disburse_neuron` of NNS Governance, when `neuron_fees_e8s ≤ transaction_fee_e8s`, the ledger burn is correctly skipped (the ICP ledger rejects burns below the transaction fee), but the governance state unconditionally sets `neuron_fees_e8s = 0` and reduces `cached_neuron_stake_e8s` by `fees_amount_e8s` as if the burn had occurred. The actual ledger subaccount retains `fees_amount_e8s` ICP tokens with no recovery path — a direct ledger conservation violation analogous to the "last withdrawer donates fees to an empty pool" pattern.

---

### Finding Description

In `rs/nns/governance/src/governance.rs`, the `disburse_neuron` function performs two ledger operations:

1. **Burn neuron management fees** — only if `fees_amount_e8s > transaction_fee_e8s`.
2. **Transfer disbursed stake** to the target account.

The conditional burn guard is correct:

```rust
if fees_amount_e8s > transaction_fee_e8s {
    self.ledger.transfer_funds(
        fees_amount_e8s,
        0, // Burning transfers don't pay a fee.
        Some(neuron_subaccount),
        governance_minting_account(),
        now,
    ).await?;
}
```

However, the governance state update that follows is **unconditional** — it runs regardless of whether the burn happened:

```rust
self.with_neuron_mut(id, |neuron| {
    // Update the stake and the fees to reflect the burning above.
    if neuron.cached_neuron_stake_e8s > fees_amount_e8s {
        neuron.cached_neuron_stake_e8s -= fees_amount_e8s;
    } else {
        neuron.cached_neuron_stake_e8s = 0;
    }
    neuron.neuron_fees_e8s = 0;  // ← always zeroed
})
```

When `fees_amount_e8s ≤ transaction_fee_e8s`:

- The ledger burn is **skipped** — the neuron's subaccount retains `fees_amount_e8s` ICP.
- Governance state is updated **as if the burn occurred**: `neuron_fees_e8s = 0`, `cached_neuron_stake_e8s -= fees_amount_e8s`.
- The disburse transfer (Transfer 2) sends `disburse_amount_e8s ≈ minted_stake - transaction_fee` to the user, draining the subaccount of all stake except the fee residual.

**Post-disburse state:**

| Layer | `neuron_fees_e8s` | `cached_neuron_stake_e8s` | Ledger subaccount balance |
|---|---|---|---|
| Governance | 0 | 0 | — |
| Ledger | — | — | `fees_amount_e8s` (stuck) |

The `fees_amount_e8s` ICP tokens remain in the neuron's governance subaccount permanently. There is no governance operation that can recover them:
- `disburse_neuron` cannot be called again (governance sees 0 stake).
- `claim_or_refresh_neuron_from_account` would re-stake the residual, but the residual (≤ 10,000 e8s) is below the minimum neuron stake, so it would be rejected.
- The governance canister has no "sweep residual" function.

This is the direct IC analog of the "last withdrawer donates fees to an empty pool" pattern: the last operation on the neuron leaves a small token residual in the subaccount that is permanently inaccessible.

**Contrast with SNS Governance:** The SNS `disburse_neuron` in `rs/sns/governance/src/governance.rs` handles this correctly — when `max_burnable_fee ≤ transaction_fee_e8s`, it skips the burn **and** does not update `neuron_fees_e8s`, keeping governance state consistent with the ledger. ---

### Impact Explanation

**Ledger conservation bug.** ICP tokens equal to `neuron_fees_e8s` (up to `transaction_fee_e8s` = 10,000 e8s = 0.0001 ICP) are permanently locked in the neuron's governance subaccount per affected neuron. The governance canister's view of total ICP under management diverges from the actual ledger balances. While the per-neuron amount is small, the pattern is structurally identical to the reported "stuck funds" class: tokens sent to an entity (the neuron subaccount) that has no remaining claim on them. [3](#0-2) 

---

### Likelihood Explanation

**Low with current NNS parameters.** The NNS `reject_cost_e8s` is currently 0.01 ICP = 1,000,000 e8s, which is 100× the transaction fee (10,000 e8s). Under current economics, `neuron_fees_e8s` is always a multiple of 1,000,000 e8s and thus always exceeds the transaction fee threshold, making the buggy branch unreachable in practice.

However, the code path is reachable if:
- The NNS economics are updated via governance proposal to reduce `reject_cost_e8s` to ≤ 10,000 e8s.
- Any future mechanism sets `neuron_fees_e8s` to a sub-threshold value.

The entry path is fully unprivileged: any principal can create a neuron, submit a proposal that gets rejected (accumulating fees), and call `disburse_neuron`. [4](#0-3) 

---

### Recommendation

Mirror the SNS Governance approach: only update `neuron_fees_e8s` and `cached_neuron_stake_e8s` when the burn actually executes. When `fees_amount_e8s ≤ transaction_fee_e8s`, skip both the ledger burn **and** the governance state deduction, preserving accounting consistency. Alternatively, transfer the residual fee amount to the governance treasury account using a standard (non-burn) transfer, which has no minimum amount restriction beyond the transaction fee itself. [5](#0-4) 

---

### Proof of Concept

1. Create an NNS neuron with stake S.
2. Submit a proposal that gets rejected, setting `neuron_fees_e8s = F` where `F ≤ transaction_fee_e8s` (requires NNS economics with small `reject_cost_e8s`, or a future parameter change).
3. Dissolve the neuron and call `disburse_neuron`.
4. Observe:
   - Governance state: `neuron_fees_e8s = 0`, `cached_neuron_stake_e8s = 0`.
   - Ledger subaccount balance: `F` ICP (not burned, not transferred).
   - No governance operation can recover `F` ICP from the subaccount.
5. The `F` ICP tokens are permanently stuck, confirming the conservation violation.

The SNS unit test `test_disburse_neuron_small_fees_not_burned` in `rs/sns/governance/src/governance/disburse_neuron_tests.rs` (lines 540–580) demonstrates the analogous scenario in SNS governance, where the SNS correctly preserves `neuron_fees_e8s = 1_000` after disburse. The NNS governance does not preserve this invariant. [6](#0-5) [7](#0-6)

#[test]
fn test_disburse_neuron_small_fees_not_burned() {
    // Test that disburse_neuron doesn't burn fees that are too small and preserves accounting
    let (mut governance, neuron_id, ledger) =
        setup_disburse_neuron_test(DissolveState::WhenDissolvedTimestampSeconds(0), 1000);

    let disburse = manage_neuron::Disburse {
        amount: None,
        to_account: None,
    };

    // This should succeed but not burn any fees
    let result = governance
        .disburse_neuron(&neuron_id, &A_NEURON_PRINCIPAL_ID, &disburse)
        .now_or_never()
        .unwrap();

    assert_eq!(result, Ok(1)); // Mock ledger returns block height 1

    // Verify that only one transfer was made (no burn), just the disburse transfer
    let transfer_calls = ledger.get_transfer_calls();
    assert_eq!(transfer_calls.len(), 1); // Only one transfer (disburse), no burn

    // Check disburse call
    let disburse_call = &transfer_calls[0];
    assert!(disburse_call.is_transfer());
    // Disburse: (500M stake - 1K fees) - 10K tx_fee = 499,989,000
    disburse_call.assert_amount_and_fee(499_989_000, 10_000);

    // Check that the neuron fees were NOT reduced (preserved for future)
    let updated_neuron = governance
        .proto
        .neurons
        .get(&neuron_id.to_string())
        .unwrap();
    // Fees should remain unchanged since they were too small to burn
    assert_eq!(updated_neuron.neuron_fees_e8s, 1_000);

    // Check cached_neuron_stake_e8s: 500M - 499.989M disbursed - 10K tx_fee = 1K (equals fees)
    assert_eq!(updated_neuron.cached_neuron_stake_e8s, 1_000);
}

impl BurnNeuronFeesOperation {
    /// Burns the neuron fees by calling ledger and changing the neuron. Recoverable errors are
    /// returned while unrecoverable errors cause a panic. A neuron lock should be held before
    /// calling this.
    pub async fn burn_neuron_fees_with_ledger(
        self,
        ledger: &dyn IcpLedger,
        neuron_store: &mut NeuronStore,
        now_seconds: u64,
    ) -> Result<(), GovernanceError> {
        let subaccount = neuron_store.with_neuron(&self.neuron_id, |neuron| neuron.subaccount())?;

        // If the ledger call fails, it's recoverable since nothing else has been changed.
        ledger
            .transfer_funds(
                self.amount_e8s,
                0, // This is a burn, and there is no transaction fee for burning.
                Some(subaccount),
                governance_minting_account(), // A transfer into the minting account is a burn.
                now_seconds,
            )
            .await
            .map_err(|err| {
                GovernanceError::new_with_message(
                    ErrorType::External,
                    format!("Failed to burn fees: {err}"),
                )
            })?;

        // After the ledger call, failing to update the neuron will be non-recoverable. Therefore we
        // panic and therefore retaining the neuron lock.
        neuron_store
            .with_neuron_mut(&self.neuron_id, |neuron| {
                self.apply_to_neuron(neuron);
            })
            .expect("Neuron not found after burning fees");
        Ok(())
    }
