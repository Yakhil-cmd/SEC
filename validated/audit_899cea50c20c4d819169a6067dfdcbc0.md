### Title
EIP-7702 Authorization Frontrunning Enables Persistent DoS of Victim Transactions — (`basic_bootloader/src/bootloader/transaction/authorization_list.rs`)

### Summary

Any unprivileged transaction sender can include a victim's EIP-7702 authorization (extracted from the mempool) in their own type-4 transaction. Because `validate_and_apply_delegation` unconditionally increments the authority's nonce on success, the attacker's transaction bumps the victim's nonce before the victim's transaction is processed. When the victim's transaction is then validated, the nonce check fails and the transaction is rejected. The attacker can repeat this indefinitely, permanently preventing the victim from executing their EIP-7702 delegation.

---

### Finding Description

`parse_authorization_list_and_apply_delegations` iterates over every `AuthorizationEntry` in a type-4 transaction's authorization list and calls `validate_and_apply_delegation` for each one. [1](#0-0) 

`validate_and_apply_delegation` performs the following state-mutating steps unconditionally for any caller:

1. Recovers the `authority` address via `ecrecover` from the embedded signature.
2. Checks `account_properties.nonce.0 != auth_nonce` — returns `false` (silent skip) if mismatched.
3. If the nonce matches, **increments the authority's nonce by 1** at line 196. [2](#0-1) 

There is **no check** that the `authority` recovered from the signature is the same as the transaction sender (`from`). Any transaction sender may embed any valid signed authorization in their authorization list.

In `validation_impl.rs`, the originator's own nonce is incremented first (line 280), then checked against the expected nonce (line 319), and only then is the authorization list processed (line 347–353): [3](#0-2) [4](#0-3) 

**Attack path (authority == transaction sender — the standard EIP-7702 self-delegation pattern):**

1. Victim Alice broadcasts an EIP-7702 type-4 transaction with:
   - Transaction nonce `N` (Alice's current account nonce)
   - Authorization list: one entry signed by Alice with `auth_nonce = N`
2. Attacker Bob observes Alice's pending transaction in the mempool and extracts the raw authorization entry `(chain_id, address, nonce=N, y_parity, r, s)`.
3. Bob crafts his own EIP-7702 transaction embedding Alice's authorization in his authorization list and submits it with a higher gas price.
4. Bob's transaction is processed first:
   - Bob's own nonce is incremented normally.
   - `validate_and_apply_delegation` is called for Alice's authorization: Alice's nonce matches `N`, so Alice's nonce is incremented from `N` to `N+1`.
5. Alice's transaction is processed:
   - Alice's nonce is incremented from `N+1` to `N+2`; `old_nonce = N+1`.
   - Nonce check: `N+1 == N` → **false** → `NonceTooLow` → **transaction rejected**.
6. Alice rebroadcasts with nonce `N+1`; Bob repeats step 3–5 indefinitely.

---

### Impact Explanation

- Alice's EIP-7702 delegation transaction is permanently blocked at zero cost to the attacker beyond gas fees.
- Alice cannot set up her intended code delegation; any protocol flow depending on the delegation (e.g., smart-account initialization, batch execution) is permanently denied.
- The attacker only needs to pay gas for a minimal EIP-7702 transaction (the authorization list intrinsic cost is charged to the attacker, not the victim).
- In the weaker variant (authority ≠ sender), the delegation silently fails at line 157–158, causing Alice's transaction to execute without the intended delegation applied — a silent correctness failure. [5](#0-4) 

---

### Likelihood Explanation

- EIP-7702 transactions are public in the mempool; the authorization signature is fully visible.
- No privileged access is required — any EOA with enough ETH to pay gas can execute this attack.
- The attack is cheap and repeatable: the attacker pays only the intrinsic cost per authorization entry (`PER_AUTH_BASE_COST` gas) per frontrun.
- The most common EIP-7702 use case (a user delegating their own EOA) is exactly the scenario where authority == sender, making this the highest-impact variant.

---

### Recommendation

The EIP-7702 specification does not mandate that `authority == tx.origin`, so a strict equality check would break legitimate multi-authority delegation. Instead, mitigations should focus on making the authorization non-extractable or non-replayable:

1. **Bind the authorization to the transaction hash**: Include the outer transaction hash (or a commitment to the sender) in the signed message, so the authorization is only valid inside the specific transaction it was created for. This is a protocol-level change.
2. **Operator-level mempool protection**: The sequencer/operator should detect and drop transactions that replay another pending transaction's authorization entries, treating them as griefing attempts.
3. **User-level mitigation**: Users should sign authorizations with `auth_nonce = current_nonce + 1` (one ahead of the transaction nonce) so that even if the authorization is replayed, it does not match the nonce at the time the victim's transaction is processed. However, this only shifts the window and does not fully eliminate the attack.

---

### Proof of Concept

```
State: Alice's account nonce = 5

Alice signs authorization:
  auth = sign(keccak(0x05 || rlp([chain_id, delegate_addr, 5])))
  → (y_parity, r, s)

Alice broadcasts EIP-7702 tx:
  { nonce: 5, authorization_list: [(chain_id, delegate_addr, 5, y_parity, r, s)] }

Bob extracts (chain_id, delegate_addr, 5, y_parity, r, s) from mempool.

Bob broadcasts EIP-7702 tx (higher gas price):
  { nonce: Bob_nonce, authorization_list: [(chain_id, delegate_addr, 5, y_parity, r, s)] }

Bob's tx executes:
  validate_and_apply_delegation:
    authority = ecrecover(...) = Alice
    Alice.nonce (5) == auth_nonce (5) → match
    increment_nonce(Alice, 1)  // Alice.nonce = 6
    → returns true

Alice's tx executes:
  increment_nonce(Alice, 1)  // Alice.nonce = 6 → 7, old_nonce = 6
  require!(6 == 5)  // NonceTooLow → REJECTED
```

pub fn parse_authorization_list_and_apply_delegations<S: EthereumLikeTypes>(
    system: &mut System<S>,
    resources: &mut S::Resources,
    auth_list: AuthorizationList<'_>,
) -> Result<(), TxError>
where
    S::IO: IOSubsystemExt,
{
    use crate::bootloader::transaction::rlp_encoded::AuthorizationEntry;
    let mut hasher = crypto::sha3::Keccak256::new();

    for entry in auth_list.iter() {
        let AuthorizationEntry {
            chain_id,
            address,
            nonce,
            y_parity,
            r,
            s,
        } = entry;
        let success = validate_and_apply_delegation(
            system,
            resources,
            &chain_id,
            nonce,
            address,
            (y_parity, r, s),
            &mut hasher,
        )?;
        system_log!(system, "Delegation success: {success}\n");

        if !success {}
    }
    Ok(())
}

fn validate_and_apply_delegation<S: EthereumLikeTypes>(
    system: &mut System<S>,
    resources: &mut S::Resources,
    auth_chain_id: &U256,
    auth_nonce: u64,
    delegation_address: &[u8; 20],
    auth_sig_data: (u8, &[u8], &[u8]),
    hasher: &mut crypto::sha3::Keccak256,
) -> Result<bool, TxError>
where
    S::IO: IOSubsystemExt,
{
    let chain_id = system.get_chain_id();

    // 0. Pre-charge intrinsic gas
    resources.charge(&S::Resources::from_ergs_and_native(
        Ergs(evm_interpreter::gas_constants::NEWACCOUNT * ERGS_PER_GAS),
        <<S::Resources as Resources>::Native as zk_ee::system::Computational>::from_computational(
            crate::bootloader::constants::PER_AUTH_NATIVE_COMPUTATIONAL_OVERHEAD,
        ),
    ))?;

    // 1. Check chain id
    if !auth_chain_id.is_zero() && auth_chain_id != &U256::from(chain_id) {
        return Ok(false);
    }
    // 2. Check for nonce overflow
    if auth_nonce == u64::MAX {
        return Ok(false);
    }
    // 3. Signature
    // EIP-2 check
    let (_, _, auth_s) = auth_sig_data;
    let s = U256::try_from_be_slice(auth_s)
        .ok_or::<TxError>(InvalidTransaction::InvalidStructure.into())?;
    if s > crypto::secp256k1::SECP256K1N_HALF_U256 {
        return Ok(false);
    }
    let msg = resources.with_infinite_ergs(|inf_ergs| {
        compute_auth_message_signed_hash::<S>(
            inf_ergs,
            auth_chain_id,
            auth_nonce,
            delegation_address,
            hasher,
        )
    })?;
    let Some(authority) = resources
        .with_infinite_ergs(|inf_ergs| recover_authority(system, inf_ergs, auth_sig_data, &msg))?
    else {
        return Ok(false);
    };

    // 4. Read authority account
    // Gas already charged in intrinsic
    let account_properties = resources.with_infinite_ergs(|inf_ergs| {
        system.io.read_account_properties(
            ExecutionEnvironmentType::NoEE,
            inf_ergs,
            &authority,
            AccountDataRequest::empty()
                .with_nonce()
                .with_nominal_token_balance()
                .with_is_delegated()
                .with_has_bytecode(),
        )
    })?;
    // 5. Check authority is not a contract
    if account_properties.is_contract() {
        return Ok(false);
    }
    // 6. Check nonce
    if account_properties.nonce.0 != auth_nonce {
        return Ok(false);
    }
    // 7. Add refund if authority is not empty.
    let is_empty = account_properties.nonce.0 == 0
        && account_properties.has_bytecode() == false
        && account_properties.nominal_token_balance.0.is_zero();

    if !is_empty {
        let ergs = Ergs(
            (evm_interpreter::gas_constants::NEWACCOUNT
                - evm_interpreter::gas_constants::PER_AUTH_BASE_COST)
                * ERGS_PER_GAS,
        );
        system
            .io
            .add_to_refund_counter(S::Resources::from_ergs(ergs))?
    }

    let delegation_address = B160::from_be_bytes(*delegation_address);
    system_log!(
        system,
        "Will delegate address 0x{:040x} -> 0x{:040x}\n",
        authority.as_uint(),
        delegation_address.as_uint()
    );

    // 8. Set code for authority, system function
    //    will handle the two cases (unsetting).
    resources.with_infinite_ergs(|inf_ergs| {
        system
            .io
            .set_delegation(inf_ergs, &authority, &delegation_address)
    })?;
    // 9.Bump nonce
    resources
        .with_infinite_ergs(|inf_ergs| {
            system
                .io
                .increment_nonce(ExecutionEnvironmentType::NoEE, inf_ergs, &authority, 1)
        })
        .map_err(|e| -> BootloaderSubsystemError {
            match e {
                SubsystemError::LeafUsage(InterfaceError(NonceError::NonceOverflow, _)) => {
                    internal_error!("Cannot overflow, already checked").into()
                }
                _ => wrap_error!(e),
            }
        })?;
    Ok(true)
