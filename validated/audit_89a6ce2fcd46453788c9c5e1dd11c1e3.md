### Title
Incorrect `refund_address` in `EvmErc20V2.withdrawToNear` When Called Through Intermediate Contracts - (File: `etc/eth-contracts/contracts/EvmErc20V2.sol`)

---

### Summary

When the `error_refund` feature is enabled, `EvmErc20V2.withdrawToNear` encodes `_msgSender()` as the `refund_address` passed to the `exit_to_near` precompile. If the function is called through an intermediate contract (e.g., a DeFi router or aggregator), `_msgSender()` resolves to the intermediate contract's address rather than the original user. If the downstream NEAR `ft_transfer` promise fails, the `exit_to_near_precompile_callback` re-mints the burned ERC-20 tokens to the intermediate contract instead of the original user, permanently locking the user's funds.

---

### Finding Description

`EvmErc20V2.withdrawToNear` is the production ERC-20 contract deployed by Aurora Engine when the `error_refund` feature is compiled in. It encodes the caller's address as the `refund_address` in the calldata sent to the `exit_to_near` precompile:

```solidity
function withdrawToNear(bytes memory recipient, uint256 amount) external override {
    address sender = _msgSender();   // <-- resolves to intermediate contract, not original user
    _burn(sender, amount);

    bytes32 amount_b = bytes32(amount);
    bytes memory input = abi.encodePacked("\x01", sender, amount_b, recipient);
    // ...
    assembly {
        let res := call(gas(), 0xe9217bc70b7ed1f598ddd3199e80b093fa71124f, 0, add(input, 32), input_size, 0, 32)
    }
}
``` [1](#0-0) 

The `exit_to_near` precompile (with `error_refund` enabled) parses bytes 1–20 of the input as the `refund_address`:

```rust
#[cfg(feature = "error_refund")]
fn parse_input(input: &[u8]) -> Result<(Address, &[u8]), ExitError> {
    let mut buffer = [0; 20];
    buffer.copy_from_slice(&input[1..21]);
    let refund_address = Address::from_array(buffer);
    Ok((refund_address, &input[21..]))
}
``` [2](#0-1) 

This `refund_address` is stored in `Erc20TokenParams` and forwarded into `RefundCallArgs` via `refund_call_args`: [3](#0-2) 

When the NEAR `ft_transfer` promise fails, `exit_to_near_precompile_callback` calls `engine::refund_on_error` with these args, which re-mints the burned ERC-20 tokens to `args.recipient_address` — the address that was encoded in the calldata: [4](#0-3) 

The re-mint target is `refund_address` = `_msgSender()` at the time `withdrawToNear` was called. When an intermediate contract calls `withdrawToNear`, `_msgSender()` is the intermediate contract, not the original user. The re-minted tokens land in the intermediate contract, not the user.

The `refund_on_error` function confirms it mints directly to `args.recipient_address`: [5](#0-4) 

The `EvmErc20V2` contract is selected as the production ERC-20 binary precisely when `error_refund` is enabled: [6](#0-5) 

---

### Impact Explanation

When a user interacts with `EvmErc20V2.withdrawToNear` through any intermediate contract (DeFi router, aggregator, wrapper), and the downstream NEAR `ft_transfer` fails:

1. The user's ERC-20 tokens have already been burned from the intermediate contract.
2. The refund re-mints those tokens to the intermediate contract's EVM address.
3. The user receives nothing; their tokens are permanently locked in the intermediate contract unless that contract has an explicit recovery path (which generic routers do not).

This constitutes **permanent freezing of user funds** — matching the Critical/High impact tier.

---

### Likelihood Explanation

The `error_refund` feature exists specifically to handle NEAR-side `ft_transfer` failures, which can occur due to insufficient storage deposits, NEP-141 contract errors, or network conditions. Any user routing an ERC-20 withdrawal through a DeFi protocol, aggregator, or wrapper contract on Aurora triggers this path. This is a realistic and common interaction pattern on EVM-compatible chains.

---

### Recommendation

`EvmErc20V2.withdrawToNear` should accept an explicit `refund_address` parameter from the caller, rather than deriving it from `_msgSender()`. This mirrors the fix recommended in the external report: pass the original user address explicitly so that intermediate contracts can forward the true beneficiary. For example:

```solidity
function withdrawToNear(bytes memory recipient, uint256 amount, address refundTo) external override {
    address sender = _msgSender();
    _burn(sender, amount);
    bytes32 amount_b = bytes32(amount);
    bytes memory input = abi.encodePacked("\x01", refundTo, amount_b, recipient);
    // ...
}
```

Alternatively, the contract can enforce that it is only called directly (not through intermediaries) by checking `msg.sender == tx.origin`, though this is a weaker mitigation.

---

### Proof of Concept

1. Deploy an intermediate router contract `Router` on Aurora that calls `erc20Token.withdrawToNear(recipient, amount)`.
2. User approves `Router` to spend their ERC-20 tokens and calls `Router.doWithdraw(recipient, amount)`.
3. `Router` calls `erc20Token.withdrawToNear(...)`. Inside `withdrawToNear`, `_msgSender()` = `Router` address. Tokens are burned from `Router`. Calldata encodes `Router` address as `refund_address`.
4. The NEAR `ft_transfer` promise fails (e.g., insufficient storage on the NEP-141 contract).
5. `exit_to_near_precompile_callback` fires, detects failure, calls `refund_on_error` with `recipient_address` = `Router` address.
6. ERC-20 tokens are re-minted to `Router`.
7. The user's tokens are permanently locked in `Router` with no recovery path. [1](#0-0) [7](#0-6) [8](#0-7)

pub fn exit_to_near_precompile_callback<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<Option<SubmitResult>, ContractError> {
    with_hashchain(io, env, function_name!(), |io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;
        env.assert_private_call()?;

        // This function should only be called as the callback of
        // exactly one promise.
        if handler.promise_results_count() != 1 {
            return Err(errors::ERR_PROMISE_COUNT.into());
        }

        let args: ExitToNearPrecompileCallbackArgs = io.read_input_borsh()?;

        let maybe_result = if let Some(PromiseResult::Successful(_)) = handler.promise_result(0) {
            if let Some(args) = args.transfer_near {
                let action = PromiseAction::Transfer {
                    amount: Yocto::new(args.amount),
                };
                let promise = PromiseBatchAction {
                    target_account_id: args.target_account_id,
                    actions: vec![action],
                };

                // Safety: this call is safe because it comes from the exit to near precompile, not users.
                // The call is to transfer the unwrapped wNEAR tokens.
                let promise_id = handler.promise_create_batch(&promise);
                handler.promise_return(promise_id);
            }

            None
        } else if let Some(args) = args.refund {
            // Exit call failed; need to refund tokens
            let refund_result = engine::refund_on_error(io, env, state, &args, handler)?;

            if !refund_result.status.is_ok() {
                return Err(errors::ERR_REFUND_FAILURE.into());
            }

            Some(refund_result)
        } else {
            None
        };

        Ok(maybe_result)
    })
