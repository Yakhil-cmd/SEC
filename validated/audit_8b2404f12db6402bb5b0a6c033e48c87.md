### Title
NNS Governance `disburse_neuron` Unconditionally Zeros `neuron_fees_e8s` Without Burning When Fees ≤ Transaction Fee - (`File: rs/nns/governance/src/governance.rs`)

### Summary
In the NNS governance canister, `disburse_neuron` unconditionally zeroes `neuron_fees_e8s` and decrements `cached_neuron_stake_e8s` by the fee amount even when no ledger burn is performed (because `fees_amount_e8s <= transaction_fee_e8s`). This leaves the fee tokens stranded in the neuron's ledger subaccount while governance believes they were burned. The neuron controller can subsequently call `ClaimOrRefresh` to re-sync the stake from the ledger, recovering tokens that should have been permanently burned as a governance penalty.

### Finding Description

In `rs/nns/governance/src/governance.rs`, `disburse_neuron` performs two transfers: first burning `neuron_fees_e8s` (the accumulated reject-cost penalty), then transferring the remaining stake to the user. The burn is conditional — it only fires when `fees_amount_e8s > transaction_fee_e8s`: [1](#0-0) 

However, the governance state update that zeros the fees is **unconditional** — it runs regardless of whether the burn happened: [2](#0-1) 

When `fees_amount_e8s <= transaction_fee_e8s` (i.e., the neuron's accumulated reject-cost penalty is ≤ 10,000 e8s = 0.0001 ICP):

1. No ledger burn is issued.
2. Governance still subtracts `fees_amount_e8s` from `cached_neuron_stake_e8s` and sets `neuron_fees_e8s = 0`.
3. The disburse transfer sends `cached_stake - fees - tx_fee` tokens to the user, deducting `cached_stake - fees` total from the subaccount.
4. After the transfer, the ledger subaccount retains exactly `fees_amount_e8s` tokens.
5. Governance records `cached_neuron_stake_e8s = 0` and `neuron_fees_e8s = 0`.

The `fees_amount_e8s` tokens are now stranded in the neuron's subaccount on the ICP ledger, but governance believes the neuron is empty. The controller can call `ClaimOrRefresh` on the neuron, which reads the actual ledger balance and updates `cached_neuron_stake_e8s` to `fees_amount_e8s`, effectively recovering the unburned penalty tokens.

By contrast, the SNS governance implementation correctly guards the state update inside the same `if` block as the burn, only updating `cached_neuron_stake_e8s` and `neuron_fees_e8s` when the ledger burn actually succeeds: [3](#0-2) 

The NNS `disburse_neuron` lacks this guard.

### Impact Explanation

A neuron controller whose neuron has accumulated `neuron_fees_e8s` in the range `(0, transaction_fee_e8s]` (i.e., 1–10,000 e8s) can disburse the neuron and then call `ClaimOrRefresh` to recover those tokens without them being burned. The governance penalty mechanism — which is intended to permanently destroy tokens from neurons whose proposals were rejected — is bypassed. This is a **ledger conservation bug**: tokens that the protocol intends to burn remain in circulation.

The maximum recoverable amount per exploit is bounded by `transaction_fee_e8s` (10,000 e8s = 0.0001 ICP), but the attack is repeatable across many neurons and requires no special privilege beyond controlling a dissolved neuron with small accumulated fees.

### Likelihood Explanation

Any NNS neuron controller can trigger this path. A neuron accumulates `neuron_fees_e8s` whenever one of its proposals is rejected; the reject cost is configurable but defaults to 1 ICP. However, a neuron could have a small residual fee balance from partial burns (e.g., via `merge_neurons` which uses `BurnNeuronFeesOperation` and may leave a remainder), or from governance parameter changes that reduce the reject cost after fees were already accumulated. The condition `fees_amount_e8s <= transaction_fee_e8s` is reachable in practice.

### Recommendation

Mirror the SNS governance pattern: only update `neuron_fees_e8s` and `cached_neuron_stake_e8s` when the ledger burn is actually performed. Move the state update inside the `if fees_amount_e8s > transaction_fee_e8s` block:

```rust
if fees_amount_e8s > transaction_fee_e8s {
    // ... ledger burn ...
    self.with_neuron_mut(id, |neuron| {
        if neuron.cached_neuron_stake_e8s > fees_amount_e8s {
            neuron.cached_neuron_stake_e8s -= fees_amount_e8s;
        } else {
            neuron.cached_neuron_stake_e8s = 0;
        }
        neuron.neuron_fees_e8s = 0;
    }).expect("...");
}
```

When fees are too small to burn, they should remain recorded in `neuron_fees_e8s` so they are not silently forgiven, consistent with the SNS behavior shown in `test_disburse_neuron_small_fees_not_burned`. [4](#0-3) 

### Proof of Concept

1. Create an NNS neuron and have one of its proposals rejected, accumulating `neuron_fees_e8s = 5_000` (less than `transaction_fee_e8s = 10_000`).
2. Start dissolving the neuron and wait for it to reach the `Dissolved` state.
3. Call `disburse_neuron` with no amount specified. The function:
   - Skips the ledger burn (5,000 ≤ 10,000).
   - Transfers `cached_stake - 5_000 - 10_000` tokens to the controller.
   - Sets `cached_neuron_stake_e8s = 0` and `neuron_fees_e8s = 0`.
   - Leaves 5,000 e8s in the neuron's ledger subaccount.
4. Call `ClaimOrRefresh` on the neuron. The governance canister reads the ledger balance (5,000 e8s) and sets `cached_neuron_stake_e8s = 5_000`.
5. The controller now has a neuron with 5,000 e8s of stake that should have been burned as a governance penalty.

#[test]
fn test_disburse_neuron_small_fees_not_burned() {
    // Test that disburse_neuron doesn't burn fees that are too small and preserves accounting
    let (mut governance, neuron_id, ledger) =
        setup_disburse_neuron_test(DissolveState::WhenDissolvedTimestampSeconds(0), 1000);

    let disburse = manage_neuron::Disburse {
        amount: None,
        to_account: None,
    };

    // This should succeed but not burn any fees
    let result = governance
        .disburse_neuron(&neuron_id, &A_NEURON_PRINCIPAL_ID, &disburse)
        .now_or_never()
        .unwrap();

    assert_eq!(result, Ok(1)); // Mock ledger returns block height 1

    // Verify that only one transfer was made (no burn), just the disburse transfer
    let transfer_calls = ledger.get_transfer_calls();
    assert_eq!(transfer_calls.len(), 1); // Only one transfer (disburse), no burn

    // Check disburse call
    let disburse_call = &transfer_calls[0];
    assert!(disburse_call.is_transfer());
    // Disburse: (500M stake - 1K fees) - 10K tx_fee = 499,989,000
    disburse_call.assert_amount_and_fee(499_989_000, 10_000);

    // Check that the neuron fees were NOT reduced (preserved for future)
    let updated_neuron = governance
        .proto
        .neurons
        .get(&neuron_id.to_string())
        .unwrap();
    // Fees should remain unchanged since they were too small to burn
    assert_eq!(updated_neuron.neuron_fees_e8s, 1_000);

    // Check cached_neuron_stake_e8s: 500M - 499.989M disbursed - 10K tx_fee = 1K (equals fees)
    assert_eq!(updated_neuron.cached_neuron_stake_e8s, 1_000);
}
