Audit Report

## Title
Cross-SNS Authorization Allows Any Valid Swap Canister to Redirect Neurons' Fund ICP Mint to Attacker-Controlled Destination - (File: rs/nns/governance/src/governance.rs)

## Summary
`settle_neurons_fund_participation` authorizes the caller by checking only that it is *any* registered SNS swap canister in SNS-W, not the swap canister specifically associated with the proposal being settled. Simultaneously, the `sns_governance_canister_id` field in a `Committed` request — which determines the ICP mint destination — is taken directly from caller-supplied input with no validation against the proposal's stored SNS governance canister ID. An attacker controlling a legitimate SNS swap canister can call this endpoint for a victim SNS's proposal, supply their own governance canister as the mint destination, steal the Neurons' Fund ICP, and permanently corrupt the victim SNS's settlement state.

## Finding Description

**Authorization checks class, not identity (lines 7018–7038):**

The authorization block converts the caller to a `CanisterId` and calls `is_canister_id_valid_swap_canister_id`, which queries SNS-W's `list_deployed_snses` and checks whether the caller appears as *any* swap canister across all deployed SNS instances:

```rust
// Check authorization. Note that a Swap could settle each other's participation.
let target_canister_id: CanisterId = caller.try_into()...?;
if let Err(err_msg) =
    is_canister_id_valid_swap_canister_id(target_canister_id, &*self.env).await
{ ... }
``` [1](#0-0) 

```rust
let is_swap = list_deployed_snses_response
    .instances
    .iter()
    .any(|sns| sns.swap_canister_id == Some(target_canister_id.into()));
``` [2](#0-1) 

There is no check that the caller is the swap canister *associated with the specific `nns_proposal_id`* in the request. The code comment explicitly acknowledges this: *"Note that a Swap could settle each other's participation."*

**Caller-supplied `sns_governance_canister_id` used as mint destination without validation (lines 7249–7273):**

When the result is `Committed`, the `sns_governance_canister_id` field is taken from the caller's request and passed directly to `mint_to_sns_governance`:

```rust
} else if let SwapResult::Committed {
    sns_governance_canister_id,
    ...
} = request.swap_result
{
    let mint_icp_result = self
        .mint_to_sns_governance(
            &request.nns_proposal_id,
            sns_governance_canister_id,   // ← caller-supplied, unvalidated
            ...
        )
        .await;
``` [3](#0-2) 

Inside `mint_to_sns_governance`, this value is used directly as the ICP transfer destination:

```rust
let destination =
    AccountIdentifier::new(sns_governance_canister_id, /* subaccount = */ None);
let _ = self.ledger.transfer_funds(amount_icp_e8s, 0, None, destination, 0).await...;
``` [4](#0-3) 

The `sns_governance_canister_id` stored in the proposal's `CreateServiceNervousSystem` action is never compared against the caller-supplied value.

**Idempotency check makes corruption permanent (lines 7166–7174):**

Once the function completes successfully, `previously_computed_final_neurons_fund_participation` is set. Any subsequent call returns the previously computed (attacker-manipulated) result immediately, making the corruption irreversible without an NNS upgrade:

```rust
(Some(_), Some(_), Some(previously_computed_final_neurons_fund_participation)) => {
    // Ok case I: Return the priorly computed results (this is an idempotent function).
    return Ok(previously_computed_final_neurons_fund_participation.into_snapshot());
}
``` [5](#0-4) 

**Exploit path:**
1. Attacker creates SNS A through the normal NNS governance process, obtaining a valid `swap_canister_id` registered in SNS-W.
2. Victim SNS B has an active swap with Neurons' Fund participation enabled and `initial_neurons_fund_participation` set.
3. Attacker's Swap A calls `settle_neurons_fund_participation` with `nns_proposal_id` = SNS B's proposal ID and `result = Committed { sns_governance_canister_id: SNS_A_governance, ... }`.
4. Authorization passes: Swap A is a valid swap canister in SNS-W.
5. Proposal check passes: SNS B's proposal is a `CreateServiceNervousSystem`.
6. State machine check passes: first call, lifecycle not yet terminal.
7. NNS Governance mints Neurons' Fund ICP and sends it to SNS A's governance canister (attacker-controlled).
8. SNS B's proposal lifecycle is set to `Committed` (terminal), permanently blocking legitimate settlement.

## Impact Explanation

This is a **High to Critical** impact vulnerability matching "Theft, permanent loss, illegal minting, or protocol insolvency involving ICP/Cycles or in-scope chain-key/ledger assets." Specifically:

- **ICP theft**: Neurons' Fund ICP (potentially millions of ICP depending on the SNS's Matched Funding parameters) is minted and sent to the attacker's governance canister instead of the victim SNS's treasury.
- **Permanent state corruption**: SNS B's proposal lifecycle is set to `Committed` (terminal). The idempotency check makes this irreversible without an NNS upgrade.
- **Neurons' Fund neuron harm**: NNS neurons that contributed maturity to SNS B's Neurons' Fund participation lose their maturity permanently, receiving no SNS neurons in return.

If the Neurons' Fund participation exceeds $1M in value, this reaches Critical severity.

## Likelihood Explanation

**Low-Medium.** The precondition is that the attacker must control a legitimate SNS swap canister, which requires passing an NNS governance vote. This is a meaningful but not insurmountable barrier for a sufficiently staked neuron holder. Once an SNS is created, the attack is a single canister call with no further preconditions. The attack window is the period between SNS B's swap committing and its legitimate `settle_neurons_fund_participation` call being processed — a race condition that the attacker can win by front-running.

## Recommendation

1. **Bind the caller to the proposal**: After fetching the proposal data for `nns_proposal_id`, extract the swap canister ID from the `CreateServiceNervousSystem` action's deployed SNS record and assert `caller == proposal_swap_canister_id`. This eliminates the cross-swap settlement attack entirely.

2. **Validate `sns_governance_canister_id` against proposal data**: When processing a `Committed` result, retrieve the `sns_governance_canister_id` from the proposal's stored SNS deployment record (not from the caller's request) and use that as the mint destination. The caller-supplied value should be ignored or treated as advisory only.

## Proof of Concept

```rust
// Attacker controls SNS A swap canister (canister ID: swap_a)
// Victim is SNS B with proposal_id = 42, governance = gov_b

// Call from swap_a to NNS Governance:
settle_neurons_fund_participation(SettleNeuronsFundParticipationRequest {
    nns_proposal_id: Some(42),  // SNS B's proposal
    result: Some(Result::Committed(Committed {
        sns_governance_canister_id: Some(gov_a),  // SNS A's governance (attacker-controlled)
        total_direct_participation_icp_e8s: Some(u64::MAX),
        total_neurons_fund_participation_icp_e8s: Some(u64::MAX),
    })),
})

// Authorization check (L7018-7038): is swap_a in list_deployed_snses? YES → passes
// Proposal check (L6991-7017): is proposal 42 a CreateServiceNervousSystem? YES → passes
// State machine (L7112-7180): first call, not terminal → proceeds
// mint_to_sns_governance (L7495-7506): mints ICP to gov_a (attacker)
// Proposal 42 lifecycle set to Committed (terminal) → SNS B's legitimate settlement permanently blocked
```

A deterministic integration test using PocketIC can reproduce this by deploying two SNS instances, having SNS A's swap canister call `settle_neurons_fund_participation` with SNS B's proposal ID and SNS A's governance canister as the destination, and asserting that the ICP balance of SNS A's governance canister increases while SNS B's governance canister receives nothing.
