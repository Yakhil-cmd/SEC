Audit Report

## Title
Unbounded `inbound_connecting` JoinSet Allows Byzantine Subnet Peer to Exhaust Replica Resources via QUIC Connection Flood — (`rs/p2p/quic_transport/src/connection_manager.rs`)

## Summary
`handle_inbound_conn_attemp` unconditionally spawns a Tokio task into the unbounded `inbound_connecting` JoinSet for every accepted QUIC `Incoming` with no per-peer cap, no global cap, and no deduplication. A Byzantine subnet peer holding a valid NNS-issued TLS certificate can open arbitrarily many concurrent QUIC connections, accumulating N tasks in the JoinSet — each alive for up to `CONNECT_TIMEOUT` (10 s) — exhausting Tokio task heap, QUIC connection state, and OS file descriptors, stalling or OOM-crashing the replica's event loop and removing it from consensus participation.

## Finding Description
**Root cause — unconditional spawn with no guard:**
`handle_inbound_conn_attemp` (lines 538–588) takes an `Incoming`, wraps a timeout future around it, and calls `self.inbound_connecting.spawn(timeout_conn_fut)` at line 587 with no check on `inbound_connecting.len()`, no per-source-IP/NodeId counter, and no `incoming.refuse()` path.

**`inbound_connecting` is a plain, unbounded `JoinSet`:**
Declared at line 121 as `JoinSet<Result<ConnectionWithPeerId, ConnectionEstablishError>>` and initialized at line 256 as `JoinSet::new()`. There is no capacity bound anywhere in the file, and no `max_concurrent_connections` limit is set on the Quinn `ServerConfig` or `TransportConfig` (lines 217–228).

**Contrast with outbound path:**
`outbound_connecting` is a `JoinMap<NodeId, …>` (line 118), so `spawn_on(peer_id, …)` at line 488 silently deduplicates: at most one outbound task per peer exists at any time. `inbound_connecting` has no equivalent key, so every new `Incoming` adds a new task regardless of how many are already in flight from the same peer.

**TLS does not prevent the flood:**
`handle_topology_change` (lines 395–436) updates the server config to only accept connections from current subnet members (lines 401–411). This rejects outsiders at the TLS layer. A Byzantine peer *is* a subnet member with a valid certificate; its connections complete TLS successfully, so every spawned task runs to completion (or timeout) — none are rejected early.

**`peer_id > node_id` check does not prevent resource consumption:**
The dialer-role check at line 567–572 rejects connections where the remote peer's NodeId is greater than the server's NodeId, but this check executes *inside* the spawned async task, after `incoming.await` completes the TLS handshake. The task is already allocated and the QUIC connection state already held before the check runs. The rejection returns an error but does not prevent the resource from being consumed.

**Event loop drains one completion per iteration:**
The `select!` loop at lines 291–381 handles exactly one branch per iteration. If the attacker opens connections faster than `inbound_connecting.join_next()` is polled, the JoinSet grows without bound.

**`CONNECT_TIMEOUT` = 10 s** (line 82) gives each task a 10-second lifetime, providing a sustained accumulation window.

## Impact Explanation
Each spawned task holds a Tokio task stack allocation, a `quinn::Incoming`/`quinn::Connection` object (QUIC crypto context, stream buffers, endpoint connection-table entry), and a UDP socket receive-buffer entry. With N concurrent connections from one Byzantine peer, the replica accumulates N × (task overhead + QUIC connection state). At sufficient N this exhausts Tokio task heap memory or OS-level resources (file descriptors), stalling or OOM-crashing the replica's event loop. A crashed replica loses its role in consensus; if enough replicas in a subnet are targeted simultaneously, subnet availability and consensus liveness are at risk. This matches the allowed High impact: **application/platform-level DoS, consensus blocking, or subnet availability impact not based on raw volumetric DDoS**.

## Likelihood Explanation
The attacker is a Byzantine subnet peer — explicitly within the IC's threat model (up to `f` Byzantine nodes in a subnet of `3f+1`). The peer holds a valid NNS-issued node TLS certificate. Opening many concurrent QUIC connections from a single process is trivial using the `quinn` client library in a loop. No privileged access beyond subnet membership, no key compromise, and no majority corruption is required. The attack is local-testable, repeatable, and requires no victim interaction.

## Recommendation
1. **Add a global cap** before spawning: check `self.inbound_connecting.len()` against a limit (e.g., `2 × subnet_size`) and call `incoming.refuse()` when the cap is reached.
2. **Add a per-source-IP cap** by tracking in-flight inbound task counts keyed by remote `SocketAddr` (available from `incoming.remote_address()` before accepting) and refusing excess attempts from the same address.
3. **Consider migrating `inbound_connecting` to a `JoinMap`** keyed by source IP or peer identity once extractable pre-handshake, mirroring the deduplication already present in `outbound_connecting`.

## Proof of Concept
```rust
// Byzantine peer opens N concurrent QUIC connections to victim replica
let client_config = make_client_config_with_valid_node_cert(); // valid NNS-issued cert
let victim_addr: SocketAddr = "VICTIM_IP:PORT".parse().unwrap();
let mut handles = Vec::new();
for _ in 0..N {
    let cfg = client_config.clone();
    handles.push(tokio::spawn(async move {
        let endpoint = quinn::Endpoint::client("0.0.0.0:0".parse().unwrap()).unwrap();
        // Connection completes TLS; task enters inbound_connecting and lives for CONNECT_TIMEOUT
        let _ = endpoint.connect_with(cfg, victim_addr, "irrelevant")
                        .unwrap()
                        .await;
    }));
}
futures::future::join_all(handles).await;
// Observable: victim's `connecting_connections` metric grows to N
// Observable: victim RSS grows proportionally to N
// Observable: victim event loop latency degrades as N increases
// Reproducible as a local integration test using PocketIC or a two-node replica setup
```
