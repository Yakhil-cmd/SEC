Audit Report

## Title
Unbounded JoinSet Spawn + AlreadyInserted Fatal Return via Crafted StrippedBlockProposal — (`rs/p2p/artifact_downloader/src/fetch_stripped_artifact/assembler.rs`)

## Summary

A Byzantine peer below the consensus fault threshold can push a `StrippedBlockProposal` containing arbitrarily many duplicate `ingress_messages` entries. Because `BlockProposalAssembler::new` performs no deduplication and `assemble_message` spawns one uncapped `JoinSet` task per entry, the attacker causes a large task/memory spike on the victim replica. Additionally, when the second duplicate task completes, `try_insert` returns `InsertionError::AlreadyInserted`, which the caller unconditionally treats as fatal and returns `AssembleResult::Unwanted`, silently discarding the assembly attempt.

## Finding Description

**Pushed artifact accepted without content validation.**
`FetchArtifact::download_artifact` returns a pushed artifact as-is without validating that its content matches the advertised ID beyond the ID field itself:

```rust
Some((artifact, peer_id)) => AssembleResult::Done {
    message: artifact,
    peer_id,
},
```

The `StrippedConsensusMessageId` is derived solely from `unstripped_consensus_message_id`; the `ingress_messages` and `stripped_idkg_dealings` lists are not part of the ID and are never validated against the consensus pool.

**No length cap or deduplication at deserialization.**
`TryFrom<pb::StrippedBlockProposal> for StrippedBlockProposal` (lines 100–108 of `stripped.rs`) collects `ingress_messages` with no length bound and no deduplication:

```rust
ingress_messages: value
    .ingress_messages
    .into_iter()
    .map(SignedIngressId::try_from)
    .collect::<Result<Vec<_>, _>>()?,
```

**No deduplication in `BlockProposalAssembler::new`.**
`BlockProposalAssembler::new` (lines 572–588 of `assembler.rs`) maps the raw `ingress_messages` Vec directly into `self.ingress_messages` without deduplication. Duplicate `SignedIngressId` entries appear as separate `(id, None)` slots.

**Uncapped JoinSet spawn.**
`assemble_message` (lines 211–226 of `assembler.rs`) calls `assembler.missing_stripped_messages()`, which returns one entry per `None` slot — including all duplicates — and spawns one `JoinSet` task per entry with no cap:

```rust
for stripped_message_id in stripped_message_ids {
    join_set.spawn(get_or_fetch(stripped_message_id, ...));
}
```

**`AlreadyInserted` treated as fatal.**
`try_insert` (lines 435–453) does a linear scan and finds the *first* slot matching `signed_ingress_id`. After the first duplicate task fills that slot, the second duplicate task finds it already filled and returns `Err(InsertionError::AlreadyInserted)`. The caller at lines 262–271 treats any `InsertionError` as a fatal bug and returns `AssembleResult::Unwanted`, aborting the entire assembly:

```rust
if let Err(err) = assembler.try_insert_stripped_message(message) {
    warn!(self.log, "Failed to insert stripped message of type {}: {}. This is a bug.", ...);
    return AssembleResult::Unwanted;
}
```

The `try_assemble` ID-consistency check (lines 640–646) is never reached.

**Exploit flow:**
1. Byzantine peer observes a valid `unstripped_consensus_message_id` from an in-flight block proposal advertisement (public in P2P).
2. Peer crafts a `pb::StrippedBlockProposal` with that ID and N duplicate `StrippedIngressMessage` entries (e.g., N = 10,000).
3. Peer pushes the crafted artifact to the victim replica before the legitimate proposer.
4. Victim's `assemble_message` accepts the push (no content validation), creates a `BlockProposalAssembler` with N duplicate slots, and spawns N `JoinSet` tasks.
5. As soon as two tasks for the same duplicate ID complete, `AlreadyInserted` is returned and `AssembleResult::Unwanted` is propagated, dropping the `JoinSet` and aborting all remaining tasks.
6. The attack is repeatable for every new block proposal height.

## Impact Explanation

**Resource exhaustion:** N tasks are spawned simultaneously before the bouncer loop begins. Each `get_or_fetch` task holds clones of `Arc<dyn Transport>`, `Arc<RwLock<Pool>>`, `ReplicaLogger`, and `Arc<Metrics>`, plus a pending async future. If the referenced ingress messages are absent from the local pool, all N tasks enter the async download path and remain alive until the bouncer refresh (every 3 seconds) or until a duplicate completes. Repeated per-block-proposal attacks cause sustained Tokio task and heap pressure on the victim replica.

**Consensus assembly disruption:** The `AssembleResult::Unwanted` return discards the assembly for the Byzantine peer's advertisement. If the Byzantine peer races the legitimate proposer, the victim must wait for the legitimate proposer's separate advertisement and RPC fetch cycle, introducing a per-block assembly delay. Sustained attacks across multiple heights can degrade replica throughput and subnet liveness.

This matches the allowed High impact: *Application/platform-level DoS, consensus blocking, or subnet availability impact not based on raw volumetric DDoS.*

## Likelihood Explanation

- No privileged access, key material, or majority corruption is required.
- The `unstripped_consensus_message_id` is observable from public P2P advertisements.
- A single Byzantine peer below the fault threshold can execute the attack.
- The attack is repeatable at every block height.
- No victim interaction or social engineering is needed.

## Recommendation

1. **Deduplicate at deserialization:** In `TryFrom<pb::StrippedBlockProposal>`, deduplicate `ingress_messages` (e.g., collect into a `BTreeSet` and reject or deduplicate on duplicate entries) before constructing `StrippedIngressPayload`.
2. **Enforce a count cap at deserialization:** Reject any proto where `ingress_messages.len()` exceeds the protocol maximum (e.g., `MAX_INGRESS_MESSAGES_PER_BLOCK`) or `stripped_idkg_dealings.len()` exceeds the maximum dealings per block.
3. **Treat `AlreadyInserted` as non-fatal:** In the `assemble_message` loop, `AlreadyInserted` for a slot that is already filled by a concurrent duplicate task is not a protocol violation — it should be logged as a peer-misbehavior signal and skipped rather than returning `Unwanted`.
4. **Deduplicate in `BlockProposalAssembler::new`:** As a defense-in-depth measure, deduplicate `ingress_messages` and `stripped_dealings` before building the assembler state, so duplicate IDs from a peer never produce duplicate tasks regardless of upstream validation.

## Proof of Concept

```rust
// Craft a StrippedBlockProposal with 10_000 duplicate ingress IDs
let (ingress, ingress_id) = fake_ingress_message_with_sig("dup", vec![1]);
let duplicate_ids = vec![ingress_id.clone(); 10_000];

// Manually construct StrippedBlockProposal with duplicate entries
// (bypassing normal strip path, simulating a Byzantine push)
let stripped = StrippedBlockProposal {
    pruned_block_proposal_proto: pb::BlockProposal::default(),
    unstripped_consensus_message_id: /* valid ID from observed advert */,
    stripped_ingress_payload: StrippedIngressPayload {
        ingress_messages: duplicate_ids,
    },
    stripped_idkg_dealings: StrippedIDkgDealings::default(),
};

let mut assembler = BlockProposalAssembler::new(stripped);
// Verify 10_000 tasks would be spawned
assert_eq!(assembler.missing_stripped_messages().len(), 10_000);

// Simulate two tasks completing for the same duplicate ID:
assembler.try_insert_stripped_message(
    StrippedMessage::Ingress(ingress_id.clone(), ingress.clone())
).unwrap(); // first: Ok(())

let result = assembler.try_insert_stripped_message(
    StrippedMessage::Ingress(ingress_id.clone(), ingress.clone())
);
assert_eq!(result, Err(InsertionError::AlreadyInserted));
// In assemble_message, this Err causes AssembleResult::Unwanted
```
