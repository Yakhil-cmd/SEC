### Title
Missing Anonymous-Principal Validation in `spawn_neuron` Allows Permanent Locking of ICP Maturity - (File: rs/nns/governance/src/governance.rs)

### Summary
`spawn_neuron` in NNS Governance accepts a caller-supplied `new_controller` principal for the child neuron without checking whether it equals the anonymous principal (`PrincipalId::new_anonymous()`). A neuron controller can deliberately pass the anonymous principal as the child controller, causing the parent neuron's maturity to be moved into a child neuron that no one can ever control. When `maybe_spawn_neurons` later mints ICP to that child neuron's subaccount, the ICP is permanently locked.

### Finding Description

In `rs/nns/governance/src/governance.rs`, `spawn_neuron` resolves the child controller as follows:

```rust
// Validate that if a child neuron controller was provided, it is a valid
// principal.
let child_controller = if let Some(child_controller) = &spawn.new_controller {
    *child_controller
} else {
    parent_neuron.controller()
};
``` [1](#0-0) 

The comment promises validation, but the code performs none. There is no check that `child_controller != PrincipalId::new_anonymous()`. The child neuron is then built with this controller:

```rust
let child_neuron = NeuronBuilder::new(
    child_nid,
    to_subaccount,
    child_controller,
    ...
)
``` [2](#0-1) 

The parent neuron's maturity is immediately deducted and placed in the child neuron's `maturity_e8s_equivalent`. After the spawn delay, `maybe_spawn_neurons` mints ICP to the child neuron's governance subaccount:

```rust
self.ledger
    .transfer_funds(
        neuron_stake,
        0, // Minting transfer don't pay a fee.
        None,
        neuron_subaccount(subaccount),
        now_seconds,
    )
    .await
``` [3](#0-2) 

Because the anonymous principal cannot sign update calls on the IC, no one can ever call `disburse_neuron` on the child neuron. The minted ICP is permanently locked in the governance canister's subaccount.

The same missing check exists in `disburse_to_neuron`:

```rust
let child_controller = disburse_to_neuron.new_controller.ok_or_else(|| {
    GovernanceError::new_with_message(
        ErrorType::InvalidCommand,
        "Must specify a new controller for disburse to neuron.",
    )
})?;
``` [4](#0-3) 

Only `None` is rejected; `Some(anonymous_principal)` passes through and ICP is transferred to a neuron that is permanently uncontrollable.

### Impact Explanation

A neuron controller (unprivileged ingress sender) can permanently destroy their own ICP maturity — or, if they control a neuron on behalf of others (e.g., a custodian), destroy third-party ICP. The minted ICP is recorded on the ledger but is locked in a governance subaccount forever, reducing the effective circulating supply and constituting a ledger conservation bug. The parent neuron's maturity is zeroed out at the time of the spawn call, so the loss is irreversible even before the ICP is minted.

### Likelihood Explanation

Any holder of a dissolved NNS neuron with sufficient maturity can trigger this by submitting a `ManageNeuron { command: Spawn { new_controller: Some(anonymous_principal), ... } }` ingress message. No privileged access, governance majority, or threshold key is required. The call is a standard update call to the NNS governance canister, reachable by any IC user. Accidental triggering is unlikely, but deliberate griefing of one's own neuron (or a custodied neuron) is straightforward.

### Recommendation

Add an explicit anonymous-principal guard immediately after resolving `child_controller` in both `spawn_neuron` and `disburse_to_neuron`:

```rust
if child_controller == PrincipalId::new_anonymous() {
    return Err(GovernanceError::new_with_message(
        ErrorType::InvalidCommand,
        "The controller of the spawned neuron must not be the anonymous principal.",
    ));
}
```

This mirrors the pattern already used in SNS governance for `TransferSnsTreasuryFunds` and `MintSnsTokens`:

```rust
if to_principal == PrincipalId::new_anonymous() {
    defects.push("to_principal must not be anonymous.".to_string());
}
``` [5](#0-4) 

### Proof of Concept

1. Obtain a dissolved NNS neuron with maturity ≥ `neuron_minimum_stake_e8s` (e.g., 100_000_000 e8s).
2. Submit an ingress `manage_neuron` update call to the NNS governance canister:
   ```
   ManageNeuron {
     neuron_id_or_subaccount: NeuronId { id: <your_neuron_id> },
     command: Spawn {
       new_controller: Some(PrincipalId::new_anonymous()),
       nonce: Some(0),
       percentage_to_spawn: Some(100),
     }
   }
   ```
3. The call succeeds. A child neuron is created with `controller = anonymous_principal` and `maturity_e8s_equivalent = <parent_maturity>`. The parent neuron's maturity is now 0.
4. After `neuron_spawn_dissolve_delay_seconds`, `maybe_spawn_neurons` fires and mints ICP to `neuron_subaccount(anonymous_principal_subaccount)` on the ICP ledger.
5. Attempt to call `disburse_neuron` as the anonymous principal — the IC rejects the update call because the anonymous principal cannot authenticate ingress messages. The ICP is permanently locked.
