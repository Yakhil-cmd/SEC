Audit Report

## Title
SNS Governance `disburse_neuron` and `disburse_maturity` Default Destination Uses Caller Instead of Neuron Owner, Enabling Authorized-Delegate Fund Drain - (File: `rs/sns/governance/src/governance.rs`)

## Summary
In SNS Governance, `disburse_neuron` and `disburse_maturity` correctly gate execution on `NeuronPermissionType::Disburse` / `NeuronPermissionType::DisburseMaturity`, but when no explicit `to_account` is supplied, both functions default the transfer destination to the **caller's** account (`Account { owner: caller.0, subaccount: None }`). Because SNS's permission model explicitly allows these permissions to be delegated to principals other than the neuron owner, any such delegate can drain the full stake or maturity to their own ledger account without the owner's consent.

## Finding Description
`disburse_neuron` at `rs/sns/governance/src/governance.rs` L1127 calls `neuron.check_authorized(caller, NeuronPermissionType::Disburse)`, which at `rs/sns/governance/src/neuron.rs` L109 succeeds for **any** principal listed in the neuron's `permissions` with the matching type — not only the neuron's creator or controller. Immediately after, at L1143–1147, the destination is resolved as:

```rust
let to_account = match disburse.to_account.as_ref() {
    None => Account { owner: caller.0, subaccount: None },
    ...
};
```

The identical pattern appears in `disburse_maturity` at L1619–1623. In NNS Governance (`rs/nns/governance/src/governance.rs` L1970–1978), disbursement is gated on `is_controlled_by(caller)`, so `caller` is always the controller and the default destination is safe. SNS deliberately removed that restriction in favour of a richer permission model, but the default-destination logic was never updated to match. The result is a semantic mismatch: the authorization check permits any delegate, while the fallback destination silently redirects funds to that delegate rather than to the neuron owner.

Exploit path:
1. Alice creates an SNS neuron with staked tokens.
2. Alice grants Bob `NeuronPermissionType::Disburse` (a standard hot-wallet / operational delegation).
3. Neuron dissolves.
4. Bob calls `manage_neuron` → `Disburse { amount: None, to_account: None }`.
5. `check_authorized(bob, Disburse)` passes.
6. `to_account` defaults to `Account { owner: bob.0, subaccount: None }`.
7. The SNS ledger transfers the full stake to Bob's account; Alice's neuron is drained.

The same four steps apply to `DisburseMaturity { percentage_to_disburse: 100, to_account: None }`.

## Impact Explanation
A principal holding a delegated `Disburse` or `DisburseMaturity` permission can irreversibly transfer the entire staked balance or accumulated maturity of a neuron to their own account in a single `manage_neuron` call. This constitutes **unauthorized theft of neuron-controlled funds** and maps to the allowed High impact: *"Unauthorized access to neurons, governance assets, wallets, identities, ledgers, or canister-controlled funds where exploitation requires meaningful per-target work or other constraints."* The loss is complete and irreversible for the neuron owner.

## Likelihood Explanation
The attack requires that the neuron owner has previously granted `Disburse` or `DisburseMaturity` to a second principal. This is an explicitly supported and common SNS operational pattern (hot-wallet keys, DAO-controlled operational accounts, third-party integrations). Once the permission exists, the exploit requires only a single `manage_neuron` call with default parameters — no additional privileges, no race conditions, no complex setup. The attack is repeatable for every neuron where such delegation has occurred.

## Recommendation
When `to_account` is `None`, resolve the neuron's primary owner (the principal that holds `ManagePrincipals` or was the original creator) and use `Account { owner: neuron_owner_principal, subaccount: None }` as the fallback destination instead of `caller.0`. Alternatively, require `to_account` to be explicitly provided whenever `caller` is not the sole principal with `ManagePrincipals` on the neuron, returning `InvalidCommand` if it is absent. Apply the same fix to both `disburse_neuron` (L1142–1147) and `disburse_maturity` (L1618–1623) in `rs/sns/governance/src/governance.rs`.

## Proof of Concept
A deterministic PocketIC / integration test can prove this without mainnet interaction:

1. Deploy a local SNS with a ledger and governance canister.
2. As `alice`, stake a neuron and call `AddNeuronPermissions` to grant `bob` `NeuronPermissionType::Disburse`.
3. Advance time past the dissolve delay so the neuron is `Dissolved`.
4. As `bob`, call `manage_neuron` with `Command::Disburse(Disburse { amount: None, to_account: None })`.
5. Assert that `bob`'s ledger balance increased by the neuron's stake (minus fee) and Alice's neuron stake is now zero.

The same test structure applies to `DisburseMaturity` with `percentage_to_disburse: 100` and `to_account: None`.
