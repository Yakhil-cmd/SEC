### Title
CBOR Deserialization Before Rate Limiting in IC Boundary Node Enables Memory Amplification — (File: `rs/boundary_node/ic_boundary/src/http/middleware/process.rs`)

---

### Summary

The IC boundary node (`ic_boundary`) deserializes the full CBOR request body in `preprocess_request` before any per-IP or generic rate limiting is applied. For query and read-state routes, no IP/subnet rate limiting exists at all. An unprivileged attacker can send max-size CBOR payloads that expand significantly in memory during deserialization, with no per-source throttle to prevent concurrent amplification across many connections.

---

### Finding Description

The middleware stack in `setup_router` is assembled as follows:

```
canister_layers = ServiceBuilder::new()
    .layer(validate_request)
    .layer(validate_canister_request)
    .layer(common_layers)          ← contains preprocess_request (CBOR parse)
    .layer(middleware_subnet_lookup)
    .layer(middleware_generic_limiter)   ← rate limiting AFTER parsing
    ...
``` [1](#0-0) 

The comment in the source explicitly states: *"Layers under ServiceBuilder are executed top-down (opposite to that under Router) — 1st layer wraps 2nd layer and so on."* This means `preprocess_request` (step 9 in `common_layers`) executes before `middleware_generic_limiter` (step 5 of `canister_layers`). [2](#0-1) 

Inside `preprocess_request`, the full body is buffered and then deserialized into a Rust struct tree via `serde_cbor::from_slice`:

```rust
let body = buffer_body_to_bytes(body, MAX_REQUEST_BODY_SIZE, Duration::from_secs(60)).await?;
let envelope: ICRequestEnvelope = serde_cbor::from_slice(&body)
    .map_err(|err| ErrorCause::UnableToParseCBOR(err.to_string()))?;
``` [3](#0-2) 

For `http_request` query calls, a **second deserialization** occurs immediately after — the `arg` blob is Candid-decoded into an `HttpRequest` struct:

```rust
let mut req: HttpRequest = Decode!([decoder_config()]; &arg.0, HttpRequest)
``` [4](#0-3) 

The `ICRequestContent` struct contains an `arg: Option<Blob>` field with no size validation beyond the outer body limit, and the TODO comment in the source acknowledges this gap:

```rust

/// we don't process too big forged requests.
/// E.g. the nonce is probably fixed-length etc.
``` [5](#0-4) 

For **query and read-state routes**, there is zero IP/subnet rate limiting — it is only optionally configured for call routes:

```rust
let canister_call_routes = {
    let mut route = Router::new()
        .route(PATH_CALL_V2, ...)
        ...;
    if let Some(rl) = cli.rate_limiting.rate_limit_per_second_per_ip {
        route = RateLimit::try_from(rl).unwrap().add_ip_rate_limiting(route);
    }

``` [6](#0-5) 

Query routes (`canister_query_routes`) and read-state routes (`canister_read_state_routes`) are merged without any IP/subnet rate limiting applied. [7](#0-6)

```rust

``` [8](#0-7) 

---

### Impact Explanation

CBOR deserialization via `serde_cbor::from_slice` into typed Rust structs (containing `Vec<u8>` blobs, `Option<Vec<Vec<Blob>>>` path trees, `String` method names) amplifies wire bytes into heap allocations. A max-size body (e.g., 2 MB) can expand to 4–8× in memory per concurrent request. With no per-IP rate limiting on query/read-state routes and the generic limiter running only after parsing, an attacker from a single IP (or distributed across IPs) can hold many concurrent parse operations in flight simultaneously, exhausting boundary node heap memory and causing OOM or severe latency degradation for all users of that boundary node.

The Rosetta API instance is more severe: the body is read with `usize::MAX` and parsed into a `serde_json::Value` tree with no size guard whatsoever, making it trivially exploitable by any HTTP client.

---

### Likelihood Explanation

The boundary node is publicly reachable by any internet user with no authentication required for query or read-state calls. The attacker-controlled entry path is straightforward: send concurrent max-size CBOR-encoded query requests to `/api/v2/canister/{id}/query` or `/api/v2/canister/{id}/read_state`. No privileged access, no key material, and no consensus-level attack is required. The optional nature of IP rate limiting (disabled by default unless `--rate-limit-per-second-per-ip` is configured) means many deployments may have no per-IP throttle at all.

---

### Recommendation

1. **Move rate limiting before body buffering and deserialization**: Apply IP-based rate limiting (or at minimum the bouncer check) before `buffer_body_to_bytes` and `serde_cbor::from_slice` in `preprocess_request`.
2. **Apply IP/subnet rate limiting to query and read-state routes**, not only call routes.
3. **Fix the Rosetta metrics middleware**: Replace `to_bytes(body, usize::MAX)` with a bounded read (e.g., matching the actix-web `JsonConfig` 4 MB limit), and consider parsing only the first N bytes needed to extract the `network_identifier` field rather than deserializing the full body into a `serde_json::Value`.
4. **Add size validation on `Blob` fields** inside `ICRequestContent` (the TODO at line 30 of `process.rs` acknowledges this gap).

### Proof of Concept

```
# Flood query endpoint with max-size CBOR payloads (no IP rate limit on query routes)
for i in $(seq 1 500); do
  curl -s -X POST https://<boundary-node>/api/v2/canister/<canister-id>/query \
    -H "Content-Type: application/cbor" \
    --data-binary @max_size_cbor_payload.bin &
done
wait
```

Where `max_size_cbor_payload.bin` is a valid CBOR-encoded `ICRequestEnvelope` with a maximally-sized `arg` blob (approaching `MAX_REQUEST_BODY_SIZE`). Each concurrent request triggers `serde_cbor::from_slice` + `Decode!` (for `http_request` method) before any rate limiting check, amplifying memory consumption per request. With 500 concurrent connections, the boundary node process faces sustained heap pressure with no per-IP throttle to stop it. [9](#0-8) [1](#0-0)

```rust
pub async fn preprocess_request(
    Extension(request_type): Extension<RequestType>,
    request: Request,
    next: Next,
) -> Result<impl IntoResponse, ApiError> {
    // Consume body
    let (mut parts, body) = request.into_parts();

    // Early check for the body size to avoid streaming too big requests
    if body.size_hint().exact() > Some(MAX_REQUEST_BODY_SIZE as u64) {
        return Err(ErrorCause::PayloadTooLarge(MAX_REQUEST_BODY_SIZE).into());
    }

    let body = buffer_body_to_bytes(body, MAX_REQUEST_BODY_SIZE, Duration::from_secs(60)).await?;

    // Parse the request body
    let envelope: ICRequestEnvelope = serde_cbor::from_slice(&body)
        .map_err(|err| ErrorCause::UnableToParseCBOR(err.to_string()))?;
    let content = envelope.content;

    // Check if the request is HTTP and try to parse the arg
    let (arg, http_request) = match (&content.method_name, content.arg) {
        (Some(method), Some(arg)) => {
            if request_type.is_query() && method == METHOD_HTTP {
                let mut req: HttpRequest = Decode!([decoder_config()]; &arg.0, HttpRequest)
                    .map_err(|err| {
                        ErrorCause::UnableToParseHTTPArg(format!(
                            "unable to decode arg as HttpRequest: {err}"
                        ))
                    })?;

                // Remove specific headers
                req.headers
                    .retain(|x| !HEADERS_HIDE_HTTP_REQUEST.contains(&(x.0.as_str())));

                // Drop the arg as it's now redundant
                (None, Some(req))
            } else {
                (Some(arg), None)
            }
        }

        (_, arg) => (arg, None),
    };

    // Check if it's a subnet read state request & it's eligible for caching.
    // If it is - insert the paths into extensions.
    if request_type.is_read_state_subnet()
        && let Some(x) = content.paths
        && should_cache_paths(&x)
    {
        parts.extensions.insert(ReadStatePaths::from(x));
    }

    // Construct the context
    let ctx = RequestContext {
        request_type,
        request_size: body.len() as u32,
        sender: Some(content.sender),
        canister_id: content.canister_id,
        method_name: content.method_name,
        ingress_expiry: content.ingress_expiry,
        arg: arg.map(|x| x.0),
        nonce: content.nonce.map(|x| x.0),
        http_request,
    };

    let ctx = Arc::new(ctx);

    // Reconstruct request back from parts
    let mut request = Request::from_parts(parts, Body::from(body));

    // Inject variables into the request
    request.extensions_mut().insert(ctx.clone());

    // Pass request to the next processor
    let mut response = next.run(request).await;

    // Inject context into the response for access by other middleware
    response.extensions_mut().insert(ctx);

    Ok(response)
}
