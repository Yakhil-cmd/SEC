### Title
Unprivileged Contracts Can Inject Fake Interop Roots via Missing Emitter Validation in `interop_root_reporter_event_hook` — (`system_hooks/src/event_hooks/interop_root_reporter.rs`)

---

### Summary

The `interop_root_reporter_event_hook` event hook does not validate the address of the contract that emitted the `InteropRootAdded` event. Any unprivileged EVM contract can emit this event with arbitrary `root`, `chain_id`, and `block_or_batch_number` values, causing fake interop roots to be injected into the system's `interop_root_storage`. These fake roots are then committed into the batch's `interop_roots_rolling_hash` in the public input, corrupting the cross-chain state commitment that the settlement layer relies on.

---

### Finding Description

The hook at `system_hooks/src/event_hooks/interop_root_reporter.rs` is registered as a system event hook and fires whenever any contract emits an event whose first topic matches `INTEROP_ROOT_ADDED_EVENT_SIG` (`keccak256("InteropRootAdded(uint256,uint256,bytes32[])")`). [1](#0-0) 

The hook validates the event data format (offset, length, topic count) but performs **no check on the address of the emitting contract**: [2](#0-1) 

The hook's function signature does not even receive an emitter address:

```rust
pub fn interop_root_reporter_event_hook<S: EthereumLikeTypes>(
    topics: &arrayvec::ArrayVec<...>,
    data: &[u8],
    _caller_ee: u8,
    system: &mut System<S>,
    resources: &mut S::Resources,
) -> Result<(), SystemError>
```

The `HooksStorage::try_intercept_event` dispatch path also carries no emitter address: [3](#0-2) 

The hook unconditionally calls `system.io.add_interop_root(...)` with attacker-controlled `root`, `chain_id`, and `block_or_batch_number`: [4](#0-3) 

`add_interop_root` performs no further validation and pushes the root directly into `interop_root_storage`: [5](#0-4) [6](#0-5) 

At batch finalization, all accumulated interop roots are folded into `interop_roots_rolling_hash`, which becomes part of the `BatchOutput` public input committed to the settlement layer: [7](#0-6) [8](#0-7) 

The intended path for importing interop roots is via operator-controlled service transactions (type `0x7d`) targeting `L2_INTEROP_ROOT_STORAGE_ADDRESS`, which are whitelisted and validated: [9](#0-8) 

The event hook was designed to capture the `InteropRootAdded` event emitted by `L2_INTEROP_ROOT_STORAGE_ADDRESS` after a legitimate service transaction. However, because the hook does not enforce that the emitter is `L2_INTEROP_ROOT_STORAGE_ADDRESS`, any user-deployed contract can bypass the service-transaction whitelist entirely.

---

### Impact Explanation

An attacker can inject arbitrary interop roots into the batch's `interop_roots_rolling_hash`. The settlement layer uses this commitment to verify which cross-chain roots were processed in the batch. A corrupted rolling hash can:

1. Allow the attacker to assert that fake cross-chain roots from arbitrary chains were processed, potentially enabling fraudulent cross-chain asset claims on the settlement layer.
2. Cause the settlement layer to accept a batch whose cross-chain state is inconsistent with the actual state of other chains, breaking the trust assumptions of the interop protocol.

This is a **public funds-loss path** via cross-chain state manipulation.

---

### Likelihood Explanation

**High reachability**: The attack requires only a standard L2 transaction calling a malicious contract. No privileged access, leaked keys, or operator cooperation is needed. Any unprivileged user can deploy a contract and emit the `InteropRootAdded` event with arbitrary parameters.

**Medium overall likelihood**: The attack is straightforward to execute but requires the settlement layer to act on the corrupted `interop_roots_rolling_hash`. The impact materializes when the settlement layer processes cross-chain messages based on the injected fake roots.

---

### Recommendation

Add an emitter address check in `interop_root_reporter_event_hook`. The hook must receive the emitting contract's address and reject events not originating from `L2_INTEROP_ROOT_STORAGE_ADDRESS`:

```rust
// Reject events not emitted by the authorized interop root storage contract
if emitter != L2_INTEROP_ROOT_STORAGE_ADDRESS {
    return Ok(());
}
```

This requires updating `HooksStorage::try_intercept_event` and all call sites to pass the emitter address, analogous to how `call_hooks` receive the `caller` address.

---

### Proof of Concept

1. Attacker deploys `MaliciousContract` with a function:
   ```solidity
   function injectFakeRoot(uint256 chainId, uint256 batchNum, bytes32 fakeRoot) external {
       bytes32[] memory sides = new bytes32[](1);
       sides[0] = fakeRoot;
       emit InteropRootAdded(chainId, batchNum, sides);
   }
   ```
2. Attacker submits a regular L2 transaction calling `injectFakeRoot(victimChainId, victimBatchNum, fakeRoot)`.
3. The EVM interpreter executes the LOG opcode; `HooksStorage::try_intercept_event` fires `interop_root_reporter_event_hook`.
4. The hook parses the event data and calls `system.io.add_interop_root(InteropRoot { root: fakeRoot, block_or_batch_number: victimBatchNum, chain_id: victimChainId })`.
5. At batch finalization, `calculate_interop_roots_rolling_hash` folds the fake root into `interop_roots_rolling_hash`.
6. The batch public input is submitted to the settlement layer with the attacker-controlled rolling hash, enabling fraudulent cross-chain message processing.

pub fn interop_root_reporter_event_hook<S: EthereumLikeTypes>(
    topics: &arrayvec::ArrayVec<<S::IOTypes as SystemIOTypesConfig>::EventKey, MAX_EVENT_TOPICS>,
    data: &[u8],
    _caller_ee: u8,
    system: &mut System<S>,
    resources: &mut S::Resources,
) -> Result<(), SystemError>
where
{
    // First, ensure we're capturing the InteropRootAdded event
    if topics.is_empty() || topics[0].as_u8_array() != INTEROP_ROOT_ADDED_EVENT_SIG {
        return Ok(());
    }
    // Internal error if the data supplied doesn't match the expected value
    if data.len() != 96 {
        return Err(internal_error!("Interop root reporter event hook received bad data").into());
    }

    // Parse data
    let offset: u32 = match U256::from_be_slice(&data[..32]).try_into() {
        Ok(offset) => offset,
        Err(_) => {
            return Err(
                internal_error!("Interop root reporter event hook received bad offset").into(),
            );
        }
    };
    // This event is part of the system, but we check it anyways
    if offset != 32 {
        return Err(internal_error!("Interop root reporter event hook received bad offset").into());
    }

    let len: u32 = match U256::from_be_slice(&data[32..64]).try_into() {
        Ok(offset) => offset,
        Err(_) => {
            return Err(
                internal_error!("Interop root reporter event hook received bad length").into(),
            );
        }
    };
    // It should have exactly one side
    if len != 1 {
        return Err(internal_error!("Interop root reporter event hook received bad length").into());
    }
    // Validate topics length
    if topics.len() != 3 {
        return Err(internal_error!("Interop root reporter event hook received bad topics").into());
    }

    let root = Bytes32::from_array(data[64..96].try_into().unwrap());
    let chain_id = U256::from_be_bytes(topics[1].as_u8_array());
    let block_or_batch_number = U256::from_be_bytes(topics[2].as_u8_array());
    system.io.add_interop_root(
        ExecutionEnvironmentType::NoEE,
        resources,
        InteropRoot {
            root,
            block_or_batch_number,
            chain_id,
        },
    )?;

    Ok(())
}

pub struct HooksStorage<S: SystemTypes, A: Allocator + Clone> {
    call_hooks: BTreeMap<u16, SystemCallHook<S>, A>,
    event_hooks: BTreeMap<u32, SystemEventHook<S>, A>,
}

impl<S: SystemTypes, A: Allocator + Clone> HooksStorage<S, A> {
    ///
    /// Creates empty hooks storage with a given allocator.
    ///
    pub fn new_in(allocator: A) -> Self {
        Self {
            call_hooks: BTreeMap::new_in(allocator.clone()),
            event_hooks: BTreeMap::new_in(allocator),
        }
    }

    ///
    /// Adds a new call hook into a given address.
    /// Fails if there was another hook registered there before.
    ///
    pub fn add_call_hook(
        &mut self,
        for_address_low: u16,
        hook: SystemCallHook<S>,
    ) -> Result<(), InternalError> {
        let existing = self.call_hooks.insert(for_address_low, hook);
        if existing.is_some() {
            return Err(internal_error!("System call hook already registered"));
        }
        Ok(())
    }

    ///
    /// Adds a new event hook into a given address.
    /// Fails if there was another hook registered there before.
    ///
    pub fn add_event_hook(
        &mut self,
        for_address_low: u32,
        hook: SystemEventHook<S>,
    ) -> Result<(), InternalError> {
        let existing = self.event_hooks.insert(for_address_low, hook);
        if existing.is_some() {
            return Err(internal_error!("System event hook already registered"));
        }
        Ok(())
    }

    ///
    /// Intercepts calls to low addresses (< 2^16) and executes hooks
    /// stored under that address. If no hook is stored there, return `Ok(None)`.
    /// Always return unused return_memory.
    ///
    pub fn try_intercept<'a>(
        &mut self,
        address_low: u16,
        request: ExternalCallRequest<S>,
        caller_ee: u8,
        system: &mut System<S>,
        return_memory: &'a mut [MaybeUninit<u8>],
    ) -> Result<(Option<CompletedExecution<'a, S>>, &'a mut [MaybeUninit<u8>]), SystemError> {
        let Some(hook) = self.call_hooks.get(&address_low) else {
            return Ok((None, return_memory));
        };
        let (res, remaining_memory) = hook.0(request, caller_ee, system, return_memory)?;

        Ok((Some(res), remaining_memory))
    }

const SERVICE_DESTINATION_WHITELIST: &[(B160, [u8; 4])] = &[

fn whitelisted(to: B160, data: &[u8]) -> bool {
    let selector: [u8; 4] = match data.get(..4).and_then(|bytes| bytes.try_into().ok()) {
        Some(selector) => selector,
        None => return false,
    };
    SERVICE_DESTINATION_WHITELIST.contains(&(to, selector))
