Audit Report

## Title
`StableNeuronStore::delete` Omits `maturity_disbursements_map` Cleanup, Enabling Permanent DoS of the Maturity Disbursement Finalization Timer — (`rs/nns/governance/src/storage/neurons.rs`)

## Summary

`StableNeuronStore::delete` clears every auxiliary map except `maturity_disbursements_map`, leaving orphaned entries when a neuron is deleted. Because `disburse_neuron` has no guard against pending maturity disbursements, any neuron owner can initiate a `DisburseMaturity`, dissolve their neuron, and call `Disburse` to delete it — leaving a stale `(neuron_id, index)` entry in both `maturity_disbursements_map` and the `MaturityDisbursementIndex`. The finalization timer then permanently loops on `NeuronNotFound` for that entry, blocking all users' maturity disbursement finalizations indefinitely.

## Finding Description

**Root cause — `StableNeuronStore::delete` omits `maturity_disbursements_map`:**

At `rs/nns/governance/src/storage/neurons.rs` lines 411–431, `delete` clears `hot_keys_map`, `recent_ballots_map`, followees, `known_neuron_data_map`, and `transfer_map`, but never calls `update_repeated_field` on `maturity_disbursements_map`. The field is confirmed to exist in the struct builder (lines 34–46) and `NeuronSections` (lines 50–78). The `MaturityDisbursementIndex` (held in `stable_neuron_indexes`, outside `StableNeuronStore`) is also not cleaned up during deletion.

**Enabling condition — `disburse_neuron` has no guard for pending maturity disbursements:**

At `rs/nns/governance/src/governance.rs` lines 1944–1995, the only precondition checks are: caller is controller, neuron state is `Dissolved`, and KYC verified. There is no check for `maturity_disbursements_in_progress`. A neuron with pending maturity disbursements can be fully deleted via `Disburse`.

**Finalization timer loops forever on orphaned entries:**

`next_maturity_disbursement_to_finalize` (`disburse_maturity.rs` lines 451–527) reads from `MaturityDisbursementIndex` via `get_neuron_ids_ready_to_finalize` (lines 82–91 of `maturity_disbursement_index.rs`), which returns the orphaned `neuron_id`. It then calls `neuron_store.with_neuron(&neuron_id, ...)` (line 472), which returns `NeuronStoreError::NeuronNotFound`, mapped to `FinalizeMaturityDisbursementError::NeuronNotFound` (line 475). This propagates as `Err` to `finalize_maturity_disbursement` (lines 544–554), which logs the error and returns `RETRY_INTERVAL`. The timer reschedules, finds the same orphaned entry, and repeats indefinitely. No self-healing occurs because the index entry is never removed on error.

## Impact Explanation

This is a **High** severity application-level DoS. The finalization timer processes one disbursement per invocation. A single orphaned entry causes it to retry indefinitely on `NeuronNotFound`, starving all legitimate pending maturity disbursements for all NNS neuron holders. This matches the allowed impact: *"Application/platform-level DoS … or subnet availability impact not based on raw volumetric DDoS"* and *"Significant NNS … security impact with concrete user or protocol harm."* The attacker sacrifices their own maturity (self-inflicted loss), but the DoS impact on all other users is the primary harm.

## Likelihood Explanation

The attack requires only: (1) owning a neuron with maturity ≥ `MINIMUM_DISBURSEMENT_E8S` (1 ICP), (2) calling `DisburseMaturity` (standard ingress), (3) dissolving the neuron and calling `Disburse` (standard ingress). No privileged access, key compromise, or majority attack is required. Any unprivileged neuron owner can trigger this. It is repeatable and cheap.

## Recommendation

1. **Fix `StableNeuronStore::delete`**: Add the missing cleanup at `rs/nns/governance/src/storage/neurons.rs` after line 428:
   ```rust
   update_repeated_field(neuron_id, vec![], &mut self.maturity_disbursements_map);
   ```
   Also remove the corresponding `MaturityDisbursementIndex` entries for the deleted neuron.

2. **Guard `disburse_neuron` against pending maturity disbursements**: In `rs/nns/governance/src/governance.rs`, before allowing stake disbursal, check `neuron.maturity_disbursements_in_progress.is_empty()` and return a `PreconditionFailed` error if not.

3. **Make the finalization timer resilient**: In `rs/nns/governance/src/governance/disburse_maturity.rs`, if `NeuronNotFound` is returned for an index entry, remove the stale entry from the `MaturityDisbursementIndex` rather than retrying indefinitely.

## Proof of Concept

State-machine test sequence:
1. Create a neuron with maturity ≥ `MINIMUM_DISBURSEMENT_E8S` and a dissolve delay.
2. Call `DisburseMaturity` — verify `maturity_disbursements_in_progress` has one entry and `MaturityDisbursementIndex` has one entry.
3. Start dissolving; advance time past the dissolve delay.
4. Call `Disburse` (stake) — neuron is deleted from `main` map.
5. Assert `maturity_disbursements_map` still contains the orphaned `(neuron_id, 0)` entry (bug confirmed at storage layer).
6. Advance time past `finalize_disbursement_timestamp_seconds`.
7. Run the finalization timer — assert it returns `RETRY_INTERVAL` and no ICP was minted.
8. Run the timer again — assert it still returns `RETRY_INTERVAL` (infinite loop confirmed).
9. Assert `MaturityDisbursementIndex` still contains the orphaned entry (no self-healing).
