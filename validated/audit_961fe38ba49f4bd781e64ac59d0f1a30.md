### Title
Paused Engine Blocks All User Fund Withdrawals and Transfers, Causing Temporary (or Permanent) Freezing of Bridged Assets — (`engine/src/contract_methods/connector.rs`, `engine/src/contract_methods/evm_transactions.rs`)

---

### Summary

When the Aurora Engine contract is paused via `pause_contract`, the `require_running` guard blocks every user-facing mutative operation — including `withdraw`, `ft_transfer`, `ft_transfer_call`, `submit`, and `call`. Users who have bridged ETH to NEAR (held as NEP-141 tokens on the engine) or who hold EVM balances have no alternative path to recover or move their funds during the pause. Because there is no enforced time-limit on the pause, the owner can freeze all user funds indefinitely.

---

### Finding Description

`pause_contract` sets `state.is_paused = true` in the engine state: [1](#0-0) 

The `require_running` helper immediately returns `ERR_PAUSED` whenever this flag is set: [2](#0-1) 

This guard is placed at the entry point of every user-facing mutative function:

- **`withdraw`** — the only path for a NEAR-side user to burn NEP-141 tokens and reclaim ETH on Ethereum: [3](#0-2) 

- **`ft_transfer` / `ft_transfer_call`** — NEP-141 token transfers between NEAR accounts: [4](#0-3) 

- **`submit` / `submit_with_args` / `call` / `deploy_code`** — all EVM transaction submission paths: [5](#0-4) 

- **`exit_to_near_precompile_callback`** — the callback that refunds ERC-20 tokens when an exit-to-NEAR operation fails: [6](#0-5) 

- **`withdraw_wnear_to_router`** — the XCC wNEAR-unwrapping callback: [7](#0-6) 

No alternative withdrawal or transfer path exists for an ordinary user when the engine is paused. The owner is the only account that can call `resume_contract`: [8](#0-7) 

There is no on-chain time-limit or forced-unpause mechanism.

A secondary, higher-severity path exists: a user who initiates an exit-to-NEAR (ERC-20 → NEP-141) has their ERC-20 tokens burned in the EVM state during the originating transaction. The `ft_transfer` on the NEP-141 contract and the `exit_to_near_precompile_callback` execute asynchronously in subsequent NEAR blocks. If the engine is paused before the callback executes and the underlying `ft_transfer` has already failed, `exit_to_near_precompile_callback` returns `ERR_PAUSED` before the refund branch runs, leaving the user with burned ERC-20 tokens and no NEP-141 tokens received — a permanent loss.

---

### Impact Explanation

**High — Temporary freezing of funds** (always, for the duration of any pause): every user who holds bridged ETH as NEP-141 tokens or an EVM balance is unable to withdraw, transfer, or move those assets via any on-chain path while the engine is paused.

**Critical — Permanent freezing of funds** (conditional): users who have an in-flight exit-to-NEAR operation where the NEP-141 `ft_transfer` fails concurrently with a pause lose their ERC-20 tokens with no refund path, because `exit_to_near_precompile_callback` aborts before the `refund_on_error` branch.

---

### Likelihood Explanation

The owner is expected to pause the engine during emergency bug fixes — a legitimate and documented use-case. During any such pause, all users with bridged assets are affected simultaneously. The pause has no enforced expiry. The owner is a single privileged account with no on-chain accountability mechanism for the duration of the pause, creating the same trust assumption highlighted in the reference report.

---

### Recommendation

1. Allow `withdraw` (and optionally `ft_transfer`) to succeed even when the engine is paused, so users can always exit their bridged positions.
2. Alternatively, exempt `exit_to_near_precompile_callback`'s refund branch from the `require_running` check so that in-flight exits can be safely unwound regardless of pause state.
3. Introduce an on-chain maximum pause duration after which the contract automatically resumes, or require a time-locked governance action to extend the pause.

---

### Proof of Concept

1. User calls `deposit` on the ETH side of the bridge; ETH is locked and NEP-141 tokens are minted on the Aurora engine for the user.
2. Owner calls `pause_contract` (e.g., to patch a bug).
3. User calls `withdraw` on the engine contract to burn their NEP-141 tokens and reclaim ETH on Ethereum.
4. `withdraw` executes `require_running(&state::get_state(&io)?)` — state has `is_paused = true` — and returns `ERR_PAUSED`.
5. No alternative withdrawal function exists for the user; funds remain locked for the entire pause duration.
6. (Severe variant) Concurrently, a user whose `exit_to_near_precompile_callback` fires while the engine is paused receives `ERR_PAUSED` before the `refund_on_error` branch, permanently losing the burned ERC-20 tokens.

pub fn pause_contract<I: IO + Copy, E: Env>(io: I, env: &E) -> Result<(), ContractError> {
    with_hashchain(io, env, function_name!(), |mut io| {
        let mut state = state::get_state(&io)?;
        require_owner_only(&state, &env.predecessor_account_id())?;
        require_running(&state)?;
        state.is_paused = true;
        state::set_state(&mut io, &state)?;
        Ok(())
    })

pub fn exit_to_near_precompile_callback<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<Option<SubmitResult>, ContractError> {
    with_hashchain(io, env, function_name!(), |io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;
        env.assert_private_call()?;

        // This function should only be called as the callback of
        // exactly one promise.
        if handler.promise_results_count() != 1 {
            return Err(errors::ERR_PROMISE_COUNT.into());
        }

        let args: ExitToNearPrecompileCallbackArgs = io.read_input_borsh()?;

        let maybe_result = if let Some(PromiseResult::Successful(_)) = handler.promise_result(0) {
            if let Some(args) = args.transfer_near {
                let action = PromiseAction::Transfer {
                    amount: Yocto::new(args.amount),
                };
                let promise = PromiseBatchAction {
                    target_account_id: args.target_account_id,
                    actions: vec![action],
                };

                // Safety: this call is safe because it comes from the exit to near precompile, not users.
                // The call is to transfer the unwrapped wNEAR tokens.
                let promise_id = handler.promise_create_batch(&promise);
                handler.promise_return(promise_id);
            }

            None
        } else if let Some(args) = args.refund {
            // Exit call failed; need to refund tokens
            let refund_result = engine::refund_on_error(io, env, state, &args, handler)?;

            if !refund_result.status.is_ok() {
                return Err(errors::ERR_REFUND_FAILURE.into());
            }

            Some(refund_result)
        } else {
            None
        };

        Ok(maybe_result)
    })
}

pub fn withdraw_wnear_to_router<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<SubmitResult, ContractError> {
    with_logs_hashchain(io, env, function_name!(), |io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;
        env.assert_private_call()?;
        if matches!(handler.promise_result_check(), Some(false)) {
            return Err(b"ERR_CALLBACK_OF_FAILED_PROMISE".into());
        }
        let args: WithdrawWnearToRouterArgs = io.read_input_borsh()?;
        let current_account_id = env.current_account_id();
        let recipient = AccountId::try_from(format!(
            "{}.{}",
            args.target.encode(),
            current_account_id.as_ref()
        ))?;
        let wnear_address = aurora_engine_precompiles::xcc::state::get_wnear_address(&io);
        let mut engine: Engine<_, E, AuroraModExp> = Engine::new_with_state(
            state,
            predecessor_address(&current_account_id),
            current_account_id,
            io,
            env,
        );
        let (result, ids) = xcc::withdraw_wnear_to_router(
            &recipient,
            args.amount,
            wnear_address,
            &mut engine,
            handler,
        )?;
        if !result.status.is_ok() {
            return Err(b"ERR_WITHDRAW_FAILED".into());
        }
        let id = ids.last().ok_or(b"ERR_NO_PROMISE_CREATED")?;
        handler.promise_return(*id);
        Ok(result)
    })
}
