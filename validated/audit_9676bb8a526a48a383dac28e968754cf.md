### Title
Authorized Caller Silently Reverts to Hardcoded Default on `post_upgrade` Without Explicit Override - (File: `rs/engine_controller/canister/canister.rs`)

### Summary
The `engine_controller` canister stores `AUTHORIZED_CALLER` in `thread_local!` (non-stable memory). Every `post_upgrade` call unconditionally resets `AUTHORIZED_CALLER` to either the provided argument or the hardcoded default principal. If the canister was deployed with a custom authorized caller and then upgraded without re-specifying that caller, the authorized caller silently reverts to the hardcoded default — giving the hardcoded default principal access to all privileged functions while locking out the intended authorized caller.

### Finding Description
`AUTHORIZED_CALLER` is declared in `thread_local!` (non-stable, heap memory):

```rust
thread_local! {
    static AUTHORIZED_CALLER: RefCell<Principal> = RefCell::new(default_authorized_caller());
}
``` [1](#0-0) 

The `apply_init_args` function, called by both `init` and `post_upgrade`, always unconditionally overwrites `AUTHORIZED_CALLER`:

```rust
fn apply_init_args(args: Option<EngineControllerInitArgs>) {
    let args = args.unwrap_or_default();
    let authorized = args
        .authorized_caller
        .unwrap_or_else(default_authorized_caller);
    AUTHORIZED_CALLER.with(|c| *c.borrow_mut() = authorized);
    ...
}
``` [2](#0-1) 

Both `init` and `post_upgrade` delegate to this function:

```rust
#[init]
fn init(args: Option<EngineControllerInitArgs>) { apply_init_args(args); }

#[post_upgrade]
fn post_upgrade(args: Option<EngineControllerInitArgs>) { apply_init_args(args); }
``` [3](#0-2) 

Because `AUTHORIZED_CALLER` lives in `thread_local!` (not stable memory), it is **not persisted across upgrades**. Every upgrade must re-specify the authorized caller, or it silently reverts to the hardcoded constant `DEFAULT_AUTHORIZED_CALLER`:

```rust
const DEFAULT_AUTHORIZED_CALLER: &str =
    "bct5z-vccu4-6q4t2-3lb6l-wm43p-ulppt-o5sqq-w6het-rthdz-qp4yn-fqe";
``` [4](#0-3) 

The integration test `init_arg_overrides_authorized_caller_and_survives_upgrade` explicitly demonstrates and asserts this silent revert behavior:

```rust
// Upgrade with no override: default principal becomes authorized again.
pic.upgrade_canister(
    ENGINE_CONTROLLER_CANISTER_ID.into(),
    wasm.bytes(),
    Encode!(&None::<EngineControllerInitArgs>).unwrap(),
    Some(Principal::anonymous()),
).await.expect("upgrade should succeed");

let err = call_create_engine(&pic, custom_caller, &args).await.unwrap_err();
assert!(err.contains("not authorized"), ...);
``` [5](#0-4) 

This is the direct IC analog of the Stader M-10 bug: just as `StaderConfig.initialize` grants the admin role via `_grantRole()` but does not update the `accounts[ADMIN]` mapping that `getAdmin()` reads from (causing `VaultProxy.owner` to be `address(0)`), the `engine_controller`'s `post_upgrade` does not read the previously active `AUTHORIZED_CALLER` from stable memory — it silently discards it and falls back to the hardcoded default whenever the upgrade argument omits `authorized_caller`.

A secondary consequence: `normalize_subnet_admins` injects the current `AUTHORIZED_CALLER` into every `update_subnet` call's admin list:

```rust
fn normalize_subnet_admins(admins: Vec<PrincipalId>) -> Vec<PrincipalId> {
    let super_admin = PrincipalId(AUTHORIZED_CALLER.with(|c| *c.borrow()));
    ...
}
``` [6](#0-5) 

After a silent revert, the hardcoded default is injected into the on-chain subnet admin list, giving it persistent registry-level access to the engine subnet even after the authorized caller is corrected in a subsequent upgrade.

### Impact Explanation
If the canister is deployed with a custom `authorized_caller` (e.g., a governance canister or a dedicated operator principal) and then upgraded without re-specifying that caller, the hardcoded default principal silently regains exclusive access to:
- `create_engine` — creates new CloudEngine subnets
- `delete_engine` — deletes existing subnets
- `update_subnet` — modifies subnet configurations and admin lists
- `deploy_guestos_to_all_subnet_nodes` — rolls out new GuestOS versions [7](#0-6) [8](#0-7) [9](#0-8) [10](#0-9) 

The intended authorized caller is simultaneously locked out of all these operations. Additionally, the hardcoded default is injected into the on-chain subnet admin list via `normalize_subnet_admins`, granting it persistent registry-level access that survives even a subsequent corrective upgrade.

### Likelihood Explanation
Medium. The upgrade process requires controller access to the canister, but it is operationally straightforward to omit `authorized_caller` from the upgrade argument — especially during routine version bumps, automated CI deployments, or when the upgrade is performed by a different operator who is unaware that a custom caller was previously set. The Stader team made an identical argument ("We are handling it as a part of the deployment process"), and the judge maintained Medium severity. The `EngineControllerInitArgs` struct derives `Default` with `authorized_caller: None`, making a silent revert the path of least resistance. [11](#0-10) 

### Recommendation
Persist `AUTHORIZED_CALLER` in stable memory (e.g., using `ic_stable_structures`) so it survives upgrades. The `post_upgrade` function should read the previously stored value and only override it when a new caller is **explicitly** provided in the upgrade argument:

```rust
#[post_upgrade]
fn post_upgrade(args: Option<EngineControllerInitArgs>) {
    let args = args.unwrap_or_default();
    if let Some(caller) = args.authorized_caller {
        AUTHORIZED_CALLER.with(|c| *c.borrow_mut() = caller);
        // persist to stable memory
    }
    // else: leave the previously persisted value intact
}
```

Alternatively, reject `None` upgrade arguments outright so that the caller is always explicitly re-confirmed on every upgrade.

### Proof of Concept
The integration test `init_arg_overrides_authorized_caller_and_survives_upgrade` in `rs/engine_controller/tests/tests.rs` is a self-contained proof of concept:

1. Install the canister with `authorized_caller: Some(custom_caller)`.
2. Confirm `custom_caller` is accepted and the hardcoded default is rejected.
3. Upgrade the canister with `None` args (simulating an operator omitting the field).
4. Confirm `custom_caller` is now rejected and the hardcoded default is accepted again. [12](#0-11)

#[tokio::test]
async fn init_arg_overrides_authorized_caller_and_survives_upgrade() {
    let custom_caller = Principal::self_authenticating(b"custom-caller");
    let pic = PocketIcBuilder::new().with_nns_subnet().build_async().await;
    let nns_subnet_id = pocket_ic_nns_subnet_id(&pic).await;

    // Bootstrap the registry with 4 nodes so create_engine has something to work with.
    let (init_mutate, node_ids) = prepare_nodes(4, INITIAL_MUTATION_ID);
    let mut builder = RegistryCanisterInitPayloadBuilder::new();
    builder.push_init_mutate_request(invariant_compliant_mutation_as_atomic_req(
        4 + INITIAL_MUTATION_ID,
        nns_subnet_id,
    ));
    builder.push_init_mutate_request(init_mutate);
    install_at(
        &pic,
        REGISTRY_CANISTER_ID,
        build_registry_wasm().bytes(),
        Encode!(&builder.build()).unwrap(),
        ROOT_CANISTER_ID.get().0,
    )
    .await;

    // Install with a custom authorized caller.
    let wasm = Project::cargo_bin_maybe_from_env("engine-controller-canister", &[]);
    install_at(
        &pic,
        ENGINE_CONTROLLER_CANISTER_ID,
        wasm.clone().bytes(),
        Encode!(&Some(EngineControllerInitArgs {
            authorized_caller: Some(custom_caller),
            initial_dkg_subnet_id: Some(nns_subnet_id.get().0),
        }))
        .unwrap(),
        Principal::anonymous(),
    )
    .await;

    // The hardcoded default principal must now be rejected.
    let args = CreateEngineArgs {
        node_ids: node_principals(&node_ids),
        subnet_admins: vec![],
        replica_version_id: test_replica_version(),
    };
    let err = call_create_engine(&pic, authorized(), &args)
        .await
        .unwrap_err();
    assert!(err.contains("not authorized"), "unexpected error: {err}");

    // The custom principal must be accepted.
    call_create_engine(&pic, custom_caller, &args)
        .await
        .expect("custom caller should be authorized after init");

    // Upgrade with no override: default principal becomes authorized again.
    pic.upgrade_canister(
        ENGINE_CONTROLLER_CANISTER_ID.into(),
        wasm.bytes(),
        Encode!(&None::<EngineControllerInitArgs>).unwrap(),
        Some(Principal::anonymous()),
    )
    .await
    .expect("upgrade should succeed");

    let err = call_create_engine(&pic, custom_caller, &args)
        .await
        .unwrap_err();
    assert!(
        err.contains("not authorized"),
        "custom caller must be rejected after upgrade with default: {err}"
    );
}
