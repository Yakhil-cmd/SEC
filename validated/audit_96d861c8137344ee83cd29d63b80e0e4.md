Audit Report

## Title
SNS Swap Canister Leaves SNS Token Allocation Unrecoverable on Aborted Swap — (File: rs/sns/swap/src/swap.rs)

## Summary
When an SNS decentralization swap is aborted, `finalize_inner` executes `sweep_icp`, `settle_neurons_fund_participation`, and `restore_dapp_controllers_for_finalize`, then returns early. No corresponding sweep of SNS tokens back to the SNS governance treasury is performed. The full `swap_distribution_sns_e8s` allocation remains locked in the swap canister's ledger account with no callable path to recover it, and the integration test suite explicitly asserts this stranded state as the expected post-abort outcome.

## Finding Description
`should_restore_dapp_control()` returns `true` exactly when `self.lifecycle() == Lifecycle::Aborted` (swap.rs L1348–1350). Inside `finalize_inner` (L1572–1584), this condition triggers the early return after restoring dapp controllers, bypassing all SNS token handling. The only SNS token transfer function, `sweep_sns()`, guards itself with an explicit `Lifecycle::Committed` check (L2170–2178) and returns a global failure for any other lifecycle, so even a hypothetical retry cannot move the tokens. No alternative transfer path (e.g., a `sweep_sns_tokens_back` or treasury-return function) exists anywhere in `rs/sns/swap/src/`. The integration test at `rs/nervous_system/integration_tests/tests/sns_lifecycle.rs` L1336–1342 asserts `swap_canister_balance_sns_e8s == swap_distribution_sns_e8s` after abort, confirming the tokens are fully retained in the swap canister. The lifecycle diagram in `rs/sns/swap/proto/ic_sns_swap/pb/v1/swap.proto` L65 shows `ABORTED → <DELETED>` as the terminal path, and the proto comment at L149–150 states deletion is appropriate only "when all tokens registered with the swap canister have been disbursed to their rightful owners" — a condition that is never satisfied for SNS tokens in the aborted path.

## Impact Explanation
The entire SNS token swap allocation — which can represent millions of governance tokens — is left in the swap canister's ledger account with no protocol-level mechanism for the SNS project, NNS governance, or any other party to recover them through normal canister operation. Recovery requires an NNS-approved upgrade of the swap canister to add a transfer-back function, which is a non-trivial governance action and is not guaranteed. Until such an upgrade is executed, the tokens are inaccessible. This constitutes a significant SNS framework security impact with concrete protocol harm: tokens minted and transferred for a specific purpose are rendered non-functional without serving that purpose and without being returned to any treasury, violating ledger conservation for the SNS token supply.

## Likelihood Explanation
No attacker action is required. Any SNS swap that fails to reach `min_participants` or `min_icp_e8s` before the deadline transitions automatically to `Aborted` via `try_abort()`. This is a documented, realistic lifecycle path. New or low-visibility SNS projects are particularly susceptible. The trigger is purely a function of participation levels, which are outside the control of the SNS project team.

## Recommendation
In `finalize_inner`, within the `should_restore_dapp_control()` branch (the aborted path), add a step before returning that transfers the remaining SNS token balance from the swap canister's default account back to the SNS governance treasury account. This mirrors the existing `sweep_icp` pattern. The transfer should be idempotent (guarded by a flag analogous to the transfer timestamps used in `BuyerState`) so that retried `finalize` calls do not attempt double-transfers. `sweep_sns()` should also be refactored or a new `sweep_sns_to_treasury()` function added that operates on `Lifecycle::Aborted` rather than only `Lifecycle::Committed`.

## Proof of Concept
1. Deploy an SNS with `swap_distribution_sns_e8s = N` loaded into the swap canister.
2. Open the swap; have fewer than `min_participants` participate before the deadline.
3. `try_abort()` transitions the swap to `Lifecycle::Aborted`.
4. Call `finalize`. Observe `finalize_inner` executes `sweep_icp` (ICP refunded), `settle_neurons_fund_participation` (NF maturity refunded), `restore_dapp_controllers_for_finalize` (dapp control restored), then returns at L1583.
5. Query the SNS ledger: `icrc1_balance_of(swap_canister_account)` returns `N` (unchanged), confirmed by the existing integration test assertion at `rs/nervous_system/integration_tests/tests/sns_lifecycle.rs` L1338.
6. Attempt any further `finalize` call: `sweep_sns()` rejects with a global failure because `lifecycle != Committed` (L2170–2178). No other transfer path exists.
7. The SNS governance treasury balance is unchanged from its pre-swap value; the `swap_distribution_sns_e8s` allocation is permanently inaccessible without an NNS-approved canister upgrade. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4) [6](#0-5)
