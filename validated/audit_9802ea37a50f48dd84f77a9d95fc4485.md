### Title
Unauthenticated Enumeration of GTC Account State Including Authenticated Principal IDs - (File: rs/nns/gtc/canister/canister.rs)

### Summary
The `get_account` query method on the Genesis Token Canister (GTC) allows any unprivileged caller to retrieve the complete `AccountState` of any GTC address without authorization. This exposes neuron IDs, ICP balances, claim/donation/forwarding status, and critically the `authenticated_principal_id` field — the IC principal that authenticated as the account owner — creating a publicly queryable mapping from Ethereum-style GTC addresses to IC principals.

### Finding Description
The `get_account_` function accepts an arbitrary address string and returns the full `AccountState` without verifying that the caller owns or is authorized to view that account. [1](#0-0) 

No caller check is performed. The returned `AccountState` struct contains: [2](#0-1) 

The most sensitive field is `authenticated_principal_id`, which is set by `claim_neurons` or `donate_account` when a user proves ownership of a GTC address: [3](#0-2) 

Once set, this field is permanently readable by any caller via `get_account`, creating a durable, unauthenticated mapping from Ethereum-style GTC addresses (which were publicly allocated at genesis) to IC principals.

The `balance_` function has the same pattern — it exposes the ICP balance of any address to any caller: [4](#0-3) 

By contrast, the write-path functions (`claim_neurons`, `donate_account`) correctly enforce ownership by calling `validate_public_key_against_caller`, which verifies the caller's principal matches the public key provided: [5](#0-4) 

The read path has no equivalent guard.

### Impact Explanation
Any unprivileged ingress sender or canister can:

1. **Enumerate all GTC accounts** — by iterating known or guessed Ethereum-style addresses, retrieve the full `AccountState` for each, including `neuron_ids`, `icpts`, and status flags (`has_claimed`, `has_donated`, `has_forwarded`).
2. **De-anonymize account owners** — the `authenticated_principal_id` field links a GTC Ethereum address to an IC principal. Once a user has claimed or donated their neurons, this mapping is permanently exposed to any caller, enabling identity correlation across the Ethereum and IC ecosystems.
3. **Discover unclaimed neuron IDs** — `neuron_ids` lists the Governance canister neuron IDs associated with each account. While these cannot be stolen (the write path is properly guarded), their exposure enables targeted monitoring and social engineering.

The `AccountState` proto comment itself acknowledges the sensitivity of `authenticated_principal_id`: [6](#0-5) 

### Likelihood Explanation
The `get_account` method is a publicly exposed query endpoint on the GTC canister, callable by any IC user or canister without authentication. GTC addresses are a finite, known set (genesis allocation). An attacker can enumerate all accounts in a single sweep. No privileged access, key material, or threshold corruption is required.

### Recommendation
- Add a caller authorization check to `get_account_`: only allow the call if `ic_cdk::api::caller()` matches the `authenticated_principal_id` of the account, or if the account has not yet been claimed (to allow the owner to check their own unclaimed state).
- Alternatively, redact `authenticated_principal_id` and `neuron_ids` from the public response, returning only non-sensitive fields (e.g., `icpts`, `has_claimed`) to unauthenticated callers.
- Apply the same pattern used in `get_full_neuron` in the Governance canister, which explicitly gates full neuron data behind caller authorization: [7](#0-6) 

### Proof of Concept
Any IC principal (including anonymous) can issue the following query call to the GTC canister on mainnet:

```
dfx canister --network ic call genesis-token get_account '("<gtc_ethereum_address>")'
```

The response returns the full `AccountState` including `authenticated_principal_id` (the IC principal of the account owner), `neuron_ids`, and all status flags — with no signature, delegation, or ownership proof required from the caller.

The root cause is the absence of any authorization guard in `get_account_`: [8](#0-7) 

compared to the properly guarded write path: [9](#0-8)

    pub async fn claim_neurons(
        &mut self,
        caller: &PrincipalId,
        public_key_hex: String,
    ) -> Result<Vec<NeuronId>, String> {
        self.assert_claim_neurons_can_be_called()?;

        let public_key = decode_hex_public_key(&public_key_hex)?;
        validate_public_key_against_caller(&public_key, caller)?;

        let address = public_key_to_gtc_address(&public_key);
        let account = self.get_account_mut(&address)?;
        account.authenticated_principal_id = Some(*caller);

        if account.has_donated {
            return Err("Account has previously donated its funds".to_string());
        }

        if account.has_forwarded {
            return Err("Account has previously forwarded its funds".to_string());
        }

        if account.has_claimed {
            return Ok(account.neuron_ids.clone());
        }

        GovernanceCanister::claim_gtc_neurons(caller, account.neuron_ids.clone()).await?;

        account.has_claimed = true;
        Ok(account.neuron_ids.clone())
    }

fn validate_public_key_against_caller(
    public_key: &PublicKey,
    caller: &PrincipalId,
) -> Result<(), String> {
    let public_key_principal = public_key_to_principal(public_key);

    if caller != &public_key_principal {
        Err("Public key does not match caller".to_string())
    } else {
        Ok(())
    }
}

        // efficient since it avoids reading the same data twice. However, if most of the calls do

        // calls come from list_neurons with `include_neurons_readable_by_caller` set to true, where
        // get_full_neuron is only called for the neurons that the caller has access to.
        let neuron_clone = self.with_neuron(&neuron_id, |neuron| neuron.clone())?;

            return Ok(neuron_clone);
        }

        if self.can_principal_vote_on_proposals_that_target_neuron(principal_id, &neuron_clone) {
            Ok(neuron_clone)
        } else {
            Err(NeuronStoreError::not_authorized_to_get_full_neuron(
                principal_id,
                neuron_id,
            ))
        }
