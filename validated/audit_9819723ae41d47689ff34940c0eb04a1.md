### Title
Excessively Permissive `INITIAL_REWARD_RATE_BASIS_POINTS_CEILING` Allows 100% Annual SNS Token Inflation - (File: rs/sns/governance/src/reward.rs)

### Summary
The SNS governance reward parameter `INITIAL_REWARD_RATE_BASIS_POINTS_CEILING` is set to `10_000` (100% annual inflation). The code itself explicitly acknowledges that "highly not sensible" values up to 90% are permitted. An SNS creator can configure this ceiling value at SNS initialization time, causing extreme token supply inflation that dilutes all non-staking token holders and concentrates voting power in early stakers.

### Finding Description
In `rs/sns/governance/src/reward.rs`, the constant `INITIAL_REWARD_RATE_BASIS_POINTS_CEILING` is defined as `10_000`, which corresponds to 100% annual token inflation:

```rust
pub const INITIAL_REWARD_RATE_BASIS_POINTS_CEILING: u64 = 10_000;
```

The validation in `initial_reward_rate_basis_points_defects()` uses an exclusive upper bound (`..Self::INITIAL_REWARD_RATE_BASIS_POINTS_CEILING`), meaning values up to `9_999` (99.99%) are accepted. The code comment explicitly acknowledges the problem:

> "Some 'highly not sensible' values are allowed (e.g. 90% growth rate) simply because the transition between 'sensible' and 'insane' is gradual, not hard."

The same ceiling is enforced in `rs/sns/init/src/lib.rs` via `validate_initial_reward_rate_basis_points`, which checks against `VotingRewardsParameters::INITIAL_REWARD_RATE_BASIS_POINTS_CEILING`. Both the SNS init validation path and the runtime `NervousSystemParameters::validate()` path accept values up to this ceiling.

The NNS uses 10% (`1_000` basis points) as its initial reward rate. The SNS default is 0%, but the ceiling permits 100x the NNS rate.

### Impact Explanation
An SNS launched with `initial_reward_rate_basis_points` near `9_999` (≈100% per year) would:
1. Double the total token supply within approximately one year through voting rewards.
2. Massively dilute all token holders who do not stake and vote, transferring wealth to stakers.
3. Concentrate voting power: early stakers accumulate compounding rewards, increasing their governance share relative to later participants.
4. Undermine the economic integrity of the SNS token, as any token price model assuming controlled inflation is invalidated.

This is a **governance parameter validation bypass** — the IC code accepts a configuration value that is explicitly acknowledged as "highly not sensible" with no hard enforcement of a reasonable bound.

### Likelihood Explanation
An SNS developer (canister developer) submitting a `CreateServiceNervousSystem` NNS proposal can set `initial_reward_rate_basis_points` to any value up to `9_999`. The NNS proposal validation (`validate_initial_reward_rate_basis_points` in `rs/sns/init/src/lib.rs`) accepts this value without error. NNS voters reviewing the proposal may not scrutinize this specific numeric field. Once the SNS is live, the SNS community can also raise this parameter via `ManageNervousSystemParameters` proposals up to the same ceiling. The risk is realistic for any SNS whose founding team retains majority voting power.

### Recommendation
Lower `INITIAL_REWARD_RATE_BASIS_POINTS_CEILING` to a value consistent with reasonable tokenomics — for example, `1_000` (10% per year, matching the NNS initial rate) or at most `2_000` (20%). The code comment acknowledging that "highly not sensible" values are permitted should be treated as a design defect, not a feature. A hard ceiling at a sensible maximum prevents both accidental misconfiguration and intentional economic exploitation.

### Proof of Concept

1. Construct an `SnsInitPayload` with:
   ```
   initial_reward_rate_basis_points = 9_999   // ≈ 100% annual inflation
   final_reward_rate_basis_points   = 9_999
   ```
2. Submit as a `CreateServiceNervousSystem` NNS proposal.
3. `validate_initial_reward_rate_basis_points` in `rs/sns/init/src/lib.rs` passes because `9_999 <= 10_000`.
4. `initial_reward_rate_basis_points_defects` in `rs/sns/governance/src/reward.rs` passes because the range check is `..Self::INITIAL_REWARD_RATE_BASIS_POINTS_CEILING` (exclusive), so `9_999 < 10_000` is valid.
5. The SNS launches. Within one year, the token supply approximately doubles via voting rewards, diluting all non-staking holders.
