### Title
Unrestricted Caller on `finalize_swap` Enables Unauthorized Triggering of Multi-Asset Settlement — (File: `rs/sns/swap/canister/canister.rs`)

---

### Summary

The `finalize_swap` update endpoint on the SNS Swap canister has no caller identity check. Any unprivileged ingress sender or canister can invoke it at will once the swap reaches a terminal lifecycle state (COMMITTED or ABORTED). This is the direct IC analog of the `harvest()` front-running issue: a sensitive, asset-moving function is publicly callable with no access restriction on the caller.

---

### Finding Description

The canister-level handler for `finalize_swap` is:

```rust
/// See Swap.finalize.
#[update]
async fn finalize_swap(_arg: FinalizeSwapRequest) -> FinalizeSwapResponse {
    log!(INFO, "finalize_swap");
    let mut clients = swap()
        .init_or_panic()
        .environment()
        .expect("unable to create canister clients");

    swap_mut().finalize(now_fn, &mut clients).await
}
``` [1](#0-0) 

There is no `check_caller_is_*` guard, no allowlist check, and no `ic_cdk::api::msg_caller()` inspection of any kind before the call is forwarded to `Swap::finalize`. Compare this with every other sensitive NNS/SNS endpoint in the codebase, all of which gate on a specific caller identity before proceeding. [2](#0-1) 

The `Swap::finalize` implementation itself only acquires an in-progress lock and checks the lifecycle state; it performs no caller validation: [3](#0-2) 

When the swap is COMMITTED, `finalize_inner` executes the following asset-moving sub-actions in sequence:

1. `sweep_icp` — transfers all ICP from the Swap canister to SNS Governance (or back to buyers if ABORTED).
2. `settle_neurons_fund_participation` — calls NNS Governance to mint ICP for Neurons' Fund neurons.
3. `sweep_sns` — distributes SNS tokens to participant ledger accounts.
4. `claim_swap_neurons` — calls SNS Governance to claim neurons on behalf of all participants.
5. `set_sns_governance_to_normal_mode` — transitions SNS Governance out of pre-initialization mode.
6. `take_sole_control_of_dapp_controllers` — transfers dapp canister control to SNS Root. [4](#0-3) 

The Candid interface exposes this as a plain public update method with an empty request type: [5](#0-4) 

---

### Impact Explanation

An unprivileged ingress sender (any Internet Computer principal, including the anonymous principal) can call `finalize_swap` at any moment after the swap reaches COMMITTED or ABORTED state, forcing the entire multi-step settlement sequence to execute. Concrete impacts:

- **Forced premature settlement**: The SNS project team or NNS governance cannot delay or sequence finalization relative to other on-chain actions (e.g., coordinating with a dapp upgrade or a governance proposal). Any external party can race them to trigger it.
- **Neurons' Fund ICP minting timing**: `settle_neurons_fund_participation` causes NNS Governance to mint ICP for Neurons' Fund neurons at the exact moment the call is made. Maturity modulation is applied at disbursement time, not at minting time, but the timing of the NNS governance inter-canister call is fully attacker-controlled.
- **Dapp controller transfer**: `take_sole_control_of_dapp_controllers` is triggered by the same call, transferring control of the SNS dapp canisters to SNS Root. An attacker can force this transfer at an arbitrary time.
- **Lock-based DoS**: Because the lock is held across multiple `await` points spanning inter-canister calls, an attacker who calls `finalize_swap` at the moment a transient ledger error occurs can hold the lock open until the call completes, blocking any legitimate retry during that window. [6](#0-5) 

---

### Likelihood Explanation

The attack requires no privileged access, no leaked keys, and no social engineering. Any principal with an Internet Computer identity can submit an ingress update call to the SNS Swap canister's `finalize_swap` endpoint. The swap canister ID is publicly discoverable via the SNS-W registry. Once the swap reaches a terminal state (which is publicly observable via `get_lifecycle`), the attack window opens immediately. Likelihood is **high** for the unauthorized triggering scenario and **medium** for the timing/coordination impact.

---

### Recommendation

Restrict `finalize_swap` to a defined allowlist of authorized callers, mirroring the pattern used throughout the rest of the NNS/SNS codebase. Acceptable callers should be limited to: the SNS Governance canister, the NNS Governance canister, the swap canister itself (for auto-finalization via timer), and optionally the SNS project's fallback controller principals. A guard equivalent to the `onlyBenevolent` fix in the original report should be added at the canister entry point:

```rust
#[update]
async fn finalize_swap(_arg: FinalizeSwapRequest) -> FinalizeSwapResponse {
    let caller = caller_principal_id();
    let init = swap().init_or_panic();
    assert!(
        is_authorized_to_finalize(&caller, init),
        "Caller {} is not authorized to call finalize_swap",
        caller
    );
    // ... existing logic
}
```

The auto-finalization path via `run_periodic_tasks` / `try_auto_finalize` already exists and handles the legitimate self-triggered case. [7](#0-6)

### Proof of Concept

1. Observe the SNS Swap canister ID for any active SNS (publicly listed in SNS-W).
2. Poll `get_lifecycle` (a query call, no authentication required) until the lifecycle transitions to `Committed` (3) or `Aborted` (4).
3. Submit an ingress update call from any principal (including anonymous):

```bash
dfx canister --network ic call <swap_canister_id> finalize_swap '(record {})'
```

4. Observe that `FinalizeSwapResponse` is returned with populated `sweep_icp_result`, `sweep_sns_result`, `claim_neuron_result`, etc., confirming that ICP was swept, SNS tokens were distributed, and neurons were claimed — all triggered by an unauthorized caller before the SNS team or NNS governance had the opportunity to do so.     pub async fn finalize(
        &mut self,
        now_fn: fn(bool) -> u64,
        environment: &mut impl CanisterEnvironment,
    ) -> FinalizeSwapResponse {
        // Acquire the lock or return a FinalizeSwapResponse with an error message.
        if let Err(error_message) = self.lock_finalize_swap() {
            return FinalizeSwapResponse::with_error(error_message);
        }

        // The lock is now acquired and asynchronous calls to finalize are blocked.
        // Perform all subactions.
        let finalize_swap_response = self.finalize_inner(now_fn, environment).await;

        if finalize_swap_response.has_error_message() {
            log!(
                ERROR,
                "The swap did not finalize successfully. \n\
                finalize_swap_response: {finalize_swap_response:?}"
            );
        } else {
            log!(
                INFO,
                "The swap finalized successfully. \n\
                finalize_swap_response: {finalize_swap_response:?}"
            );
        }

        // Release the lock. Note, if there is a panic, the lock will
        // not be released. In that case, the Swap canister will need
        // to be upgraded to release the lock.
        self.unlock_finalize_swap();

        finalize_swap_response
    }

    pub async fn finalize_inner(
        &mut self,
        now_fn: fn(bool) -> u64,
        environment: &mut impl CanisterEnvironment,
    ) -> FinalizeSwapResponse {
        let mut finalize_swap_response = FinalizeSwapResponse::default();

        if let Err(e) = self.can_finalize() {
            finalize_swap_response.set_error_message(e);
            return finalize_swap_response;
        }

        // Transfer the ICP tokens from the Swap canister.
        finalize_swap_response
            .set_sweep_icp_result(self.sweep_icp(now_fn, environment.icp_ledger()).await);
        if finalize_swap_response.has_error_message() {
            return finalize_swap_response;
        }

        // Settle the Neurons' Fund participation in the token swap.
        finalize_swap_response.set_settle_neurons_fund_participation_result(
            self.settle_neurons_fund_participation(environment.nns_governance_mut())
                .await,
        );
        if finalize_swap_response.has_error_message() {
            return finalize_swap_response;
        }

        if self.should_restore_dapp_control() {
            // Restore controllers of dapp canisters to their original
            // owners (i.e. self.init.fallback_controller_principal_ids).
            finalize_swap_response.set_set_dapp_controllers_result(
                self.restore_dapp_controllers_for_finalize(environment.sns_root_mut())
                    .await,
            );

            // In the case of returning control of the dapp(s) to the fallback
            // controllers, finalize() need not do any more work, so always return
            // and end execution.
            return finalize_swap_response;
        }

        // Create the SnsNeuronRecipes based on the contribution of direct and NF participants
        finalize_swap_response
            .set_create_sns_neuron_recipes_result(self.create_sns_neuron_recipes());
        if finalize_swap_response.has_error_message() {
            return finalize_swap_response;
        }

        // Transfer the SNS tokens from the Swap canister.
        finalize_swap_response
            .set_sweep_sns_result(self.sweep_sns(now_fn, environment.sns_ledger()).await);
        if finalize_swap_response.has_error_message() {
            return finalize_swap_response;
        }

        // Once SNS tokens have been distributed to the correct accounts, claim
        // them as neurons on behalf of the Swap participants.
        finalize_swap_response.set_claim_neuron_result(
            self.claim_swap_neurons(environment.sns_governance_mut())
                .await,
        );
        if finalize_swap_response.has_error_message() {
            return finalize_swap_response;
        }

        finalize_swap_response.set_set_mode_call_result(
            Self::set_sns_governance_to_normal_mode(environment.sns_governance_mut()).await,
        );

        // The following step is non-critical, so we'll do it after we set
        // governance to normal mode, but only if there were no errors.
        if !finalize_swap_response.has_error_message() {
            finalize_swap_response.set_set_dapp_controllers_result(
                self.take_sole_control_of_dapp_controllers_for_finalize(environment.sns_root_mut())
                    .await,
            );
        }

        finalize_swap_response
    }
