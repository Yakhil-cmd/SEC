### Title
Zero-Amount Disbursal Executes Ledger Transfer Without Guard, Burning Transaction Fee for No Tokens - (File: rs/sns/governance/src/governance.rs)

### Summary
The `disburse_neuron` function in SNS governance does not validate that `disburse_amount_e8s > 0` before issuing the stake-transfer call to the ledger. When a caller supplies `amount = Some(Amount { e8s: 0 })`, the code computes a zero disbursal amount, skips the fee-subtraction branch, and unconditionally calls `transfer_funds(0, transaction_fee_e8s, …)`. The ICRC1 ledger accepts zero-amount regular transfers, so the call succeeds: the neuron's subaccount is debited only the transaction fee, the neuron's `cached_neuron_stake_e8s` is decremented by `transaction_fee_e8s`, and the caller receives nothing. The same structural gap exists in the NNS governance counterpart.

---

### Finding Description

In `rs/sns/governance/src/governance.rs`, `disburse_neuron` computes the disbursal amount as follows:

```rust
let mut disburse_amount_e8s = disburse
    .amount
    .as_ref()
    .map_or(neuron.stake_e8s(), |a| a.e8s);          // ← user-supplied 0

disburse_amount_e8s = disburse_amount_e8s.min(neuron.stake_e8s());  // still 0

if disburse_amount_e8s > transaction_fee_e8s {        // false when amount == 0
    disburse_amount_e8s -= transaction_fee_e8s
}
// disburse_amount_e8s == 0 here, no guard
``` [1](#0-0) 

Immediately after, without any zero-check, the code issues:

```rust
let block_height = self
    .ledger
    .transfer_funds(
        disburse_amount_e8s,      // 0
        transaction_fee_e8s,      // e.g. 10_000 e8s
        Some(from_subaccount),
        to_account,
        self.env.now(),
    )
    .await?;
``` [2](#0-1) 

On success the neuron's cached stake is decremented:

```rust
let to_deduct = disburse_amount_e8s + transaction_fee_e8s;  // 0 + fee
neuron.cached_neuron_stake_e8s =
    neuron.cached_neuron_stake_e8s.saturating_sub(to_deduct);
``` [3](#0-2) 

The ICRC1 ledger's `execute_transfer_not_async` has no minimum-amount guard for regular (non-burn) transfers — only burns are rejected at zero:

```rust
if Tokens::is_zero(&amount) {
    return Err(CoreTransferError::BadBurn { … });
}
``` [4](#0-3) 

So the zero-amount transfer succeeds, a ledger block is written, and the neuron loses `transaction_fee_e8s` with nothing disbursed.

The identical pattern exists in NNS governance: [5](#0-4) 

---

### Impact Explanation

- **Ledger bloat / spurious block**: A valid but economically meaningless block is appended to the SNS ledger chain.
- **Neuron stake drained by fee**: The neuron's `cached_neuron_stake_e8s` is reduced by `transaction_fee_e8s` (e.g. 10 000 e8s) with zero tokens reaching the destination. Repeated calls drain the neuron's stake one fee at a time.
- **State consistency**: `cached_neuron_stake_e8s` remains consistent with the on-chain balance (the fee is actually deducted), but the operation is semantically incorrect — the user requested a disbursal and received nothing while paying a fee.

---

### Likelihood Explanation

Any principal holding `NeuronPermissionType::Disburse` over a dissolved SNS neuron can trigger this via a standard ingress `manage_neuron` call with `Command::Disburse { amount: Some(Amount { e8s: 0 }), … }`. No privileged access, no governance majority, and no threshold corruption is required. The entry path is fully unprivileged ingress.

---

### Recommendation

Add an explicit guard immediately after computing `disburse_amount_e8s`, before the lock is acquired and any ledger call is made:

```rust
if disburse_amount_e8s == 0 {
    return Err(GovernanceError::new_with_message(
        ErrorType::InvalidCommand,
        "Disbursal amount must be greater than zero after subtracting fees.",
    ));
}
```

Apply the same guard in `rs/nns/governance/src/governance.rs` `disburse_neuron` at the equivalent location. [6](#0-5) 

---

### Proof of Concept

1. Deploy an SNS with a dissolved neuron whose controller is `caller`.
2. Submit an ingress `manage_neuron` message:
   ```
   ManageNeuron {
     subaccount: <neuron_subaccount>,
     command: Some(Command::Disburse(Disburse {
       amount: Some(Amount { e8s: 0 }),
       to_account: None,
     })),
   }
   ```
3. Observe the call returns `Ok(block_height)`.
4. Query the SNS ledger: a new block exists with `amount = 0` and `fee = transaction_fee_e8s`.
5. Query the neuron: `cached_neuron_stake_e8s` has decreased by `transaction_fee_e8s`; the caller's account balance is unchanged.
6. Repeat to drain the neuron's stake one fee at a time.
