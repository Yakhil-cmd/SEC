### Title
`transfer()` 2300-gas Stipend Causes Permanent Fund Freeze in WSEI.withdraw() - (File: `contracts/src/WSEI.sol`)

---

### Summary
`WSEI.sol` is a production Wrapped-SEI contract embedded in the sei-chain EVM module. Its `withdraw()` function uses `payable(msg.sender).transfer(wad)`, which hard-caps the recipient's gas at 2300. Any smart-contract caller whose `receive()` or `fallback()` consumes more than 2300 gas will have every withdrawal revert, permanently freezing the deposited SEI inside the contract.

---

### Finding Description
`WSEI.withdraw()` sends native SEI back to the caller using Solidity's `transfer()`:

```solidity
function withdraw(uint wad) public {
    require(balanceOf[msg.sender] >= wad);
    balanceOf[msg.sender] -= wad;
    payable(msg.sender).transfer(wad);   // ← 2300-gas stipend
    emit Withdrawal(msg.sender, wad);
}
``` [1](#0-0) 

`transfer()` forwards exactly 2300 gas to the recipient. Smart-contract wallets (e.g., multisigs, account-abstraction wallets, proxy contracts) routinely execute storage writes, event emissions, or delegate calls in their `receive()` hooks, all of which exceed 2300 gas. For such callers, `withdraw()` will always revert after the balance has already been decremented — but because the decrement happens before the transfer (CEI is followed), the revert rolls back the decrement too, leaving the balance intact but the ETH permanently unwithdrawable via this path.

The contract is compiled into on-chain artifacts and deployed as part of the sei-chain EVM module: [2](#0-1) [3](#0-2) 

---

### Impact Explanation
Any smart-contract address that calls `WSEI.deposit()` (or sends ETH directly, triggering `receive()`/`fallback()`) and later calls `withdraw()` will have its funds permanently frozen if its `receive()` uses >2300 gas. The amount frozen equals whatever that contract deposited. Depending on the contract and market conditions this can exceed $5k (Critical) or remain below (Medium).

---

### Likelihood Explanation
Smart-contract wallets are the dominant wallet type for DAOs, protocols, and institutional users on EVM chains. EIP-2929 (Berlin hard fork) already raised cold-storage access costs above 2300 gas for a single `SLOAD`, making the 2300-stipend insufficient for virtually any non-trivial `receive()`. The likelihood that a real user hits this is moderate-to-high as Sei EVM adoption grows.

---

### Recommendation
Replace `transfer()` with a low-level `call{}` and check the return value:

```solidity
function withdraw(uint wad) public {
    require(balanceOf[msg.sender] >= wad);
    balanceOf[msg.sender] -= wad;
    (bool success, ) = payable(msg.sender).call{value: wad}("");
    require(success, "WSEI: ETH transfer failed");
    emit Withdrawal(msg.sender, wad);
}
```

The CEI pattern is already in place, so reentrancy risk from switching to `call()` is minimal.

---

### Proof of Concept
1. Deploy a contract `Victim` with a `receive()` that does one `SSTORE` (costs ~20 000 gas post-EIP-2929).
2. From `Victim`, call `WSEI.deposit{value: 1 ether}()`.
3. From `Victim`, call `WSEI.withdraw(1 ether)`.
4. The `transfer()` forwards only 2300 gas → `SSTORE` in `Victim.receive()` reverts → `withdraw()` reverts.
5. `Victim`'s WSEI balance remains 1 ether; the underlying SEI is locked in the WSEI contract with no alternative withdrawal path. [1](#0-0)
