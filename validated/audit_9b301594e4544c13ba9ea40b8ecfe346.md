Audit Report

## Title
Unbounded Duplicate `(NodeIndex, IDkgArtifactId)` Entries in `StrippedBlockProposal` Spawn N Concurrent Fetch Tasks Without Deduplication — (`rs/p2p/artifact_downloader/src/fetch_stripped_artifact/types/stripped.rs`, `assembler.rs`)

## Summary

`TryFrom<pb::StrippedBlockProposal>` collects `stripped_idkg_dealings` into a plain `Vec` with no uniqueness check, and `BlockProposalAssembler::new()` mirrors that `Vec` directly into `signed_dealings`. A Byzantine peer can craft a proto with N identical `(NodeIndex, IDkgArtifactId)` entries, causing `assemble_message` to spawn N concurrent `get_or_fetch` Tokio tasks for the same dealing. Each task that misses the local pool enters an unbounded exponential-backoff RPC loop against honest subnet peers. Assembly then permanently fails with `AssembleResult::Unwanted` on the first `AlreadyInserted` collision, so the victim replica never assembles the block from that peer while the N RPC tasks have already been dispatched.

## Finding Description

**Root cause 1 — no deduplication at deserialization boundary (`stripped.rs` L110–128):**

`TryFrom<pb::StrippedBlockProposal>` iterates `value.stripped_idkg_dealings` and `.collect()`s into a `Vec<(NodeIndex, IDkgArtifactId)>` with no `HashSet` guard or duplicate rejection:

```rust
stripped_idkg_dealings: StrippedIDkgDealings {
    stripped_dealings: value
        .stripped_idkg_dealings
        .into_iter()
        .map(|dealing| { ... Ok((dealing.dealer_index, idkg_artifact_id)) })
        .collect::<Result<Vec<_>, ProxyDecodeError>>()?,
},
``` [1](#0-0) 

**Root cause 2 — `BlockProposalAssembler::new()` mirrors the Vec without deduplication (`assembler.rs` L580–585):**

```rust
signed_dealings: stripped_block_proposal
    .stripped_idkg_dealings
    .stripped_dealings
    .iter()
    .map(|(node_index, dealing_id)| ((*node_index, dealing_id.clone()), None))
    .collect(),
``` [2](#0-1) 

**Root cause 3 — one task spawned per Vec entry, including all N duplicates (`assembler.rs` L211–226):**

`missing_stripped_messages()` returns every `None`-valued entry — all N duplicates — and `assemble_message` spawns one `get_or_fetch` task per entry before the bouncer is consulted: [3](#0-2) 

**Root cause 4 — each task issues live RPC calls in an unbounded retry loop (`download.rs` L301–375):**

`download_stripped_message` uses `.with_max_elapsed_time(None)` — no overall timeout — and loops forever until a peer responds: [4](#0-3) [5](#0-4) 

**Root cause 5 — assembly permanently fails on the second duplicate result (`assembler.rs` L262–271):**

`try_insert` for `SignedIDkgDealing` does a linear scan and finds the first matching entry. After the first task fills it with `Some(dealing)`, the second task's result hits `dealing.is_some()` and returns `InsertionError::AlreadyInserted`, causing `assemble_message` to immediately return `AssembleResult::Unwanted` and drop the `join_set` — but all N tasks were already dispatched and may have already issued RPC calls: [6](#0-5) [7](#0-6) 

**No existing guard is sufficient:** The only validation in `TryFrom` checks that each `IDkgArtifactId` is of the `Dealing` variant (not `DealingSupport`), but performs no cross-entry uniqueness check. There is no size cap on the `stripped_idkg_dealings` repeated field. The bouncer check occurs after the task spawn loop, not before it.

## Impact Explanation

This matches the **High** bounty impact: *"Application/platform-level DoS, crash, consensus blocking, certified-state disruption, or subnet availability impact not based on raw volumetric DDoS."*

A single Byzantine peer can, once per consensus round (~1 s), send a crafted `StrippedBlockProposal` with N=1000 duplicate dealing entries to a victim replica. The victim fires 1000 concurrent `transport.rpc()` calls to honest subnet peers for the same dealing, creating a sustained amplified RPC storm. The victim replica also permanently fails to assemble the block from the Byzantine peer and must re-fetch from another peer, adding latency. If the Byzantine peer targets multiple replicas simultaneously, the RPC amplification is multiplied across the subnet's honest nodes.

## Likelihood Explanation

- Requires only a single Byzantine peer — well below the consensus fault threshold (f < n/3).
- The crafted proto is trivially constructed: repeat the same `pb::StrippedIDkgDealing` entry N times in the `stripped_idkg_dealings` repeated field.
- The Byzantine peer advertises a valid `StrippedConsensusMessageId` (pointing to any real block proposal hash), causing the victim to fetch the stripped artifact from it.
- No privileged access, key material, or majority corruption required.
- Repeatable every consensus round with no cooldown.

## Recommendation

Deduplicate `stripped_idkg_dealings` at the earliest validation boundary in `TryFrom<pb::StrippedBlockProposal>`. The strictest fix is to return a `ProxyDecodeError` on any duplicate, since a well-formed stripped block proposal produced by an honest node will never contain duplicates:

```rust
let mut seen = std::collections::HashSet::new();
stripped_dealings: value
    .stripped_idkg_dealings
    .into_iter()
    .map(|dealing| {
        let idkg_artifact_id: IDkgArtifactId = try_from_option_field(
            dealing.dealing_id,
            "StrippedIDkgDealings::dealing_id",
        )?;
        if !matches!(idkg_artifact_id, IDkgArtifactId::Dealing(_, _)) {
            return Err(ProxyDecodeError::Other(...));
        }
        let key = (dealing.dealer_index, idkg_artifact_id.clone());
        if !seen.insert(key) {
            return Err(ProxyDecodeError::Other(
                "Duplicate (dealer_index, dealing_id) in stripped_idkg_dealings".into()
            ));
        }
        Ok((dealing.dealer_index, idkg_artifact_id))
    })
    .collect::<Result<Vec<_>, ProxyDecodeError>>()?,
```

The same deduplication should be applied to `ingress_messages` in the same `TryFrom` impl for consistency.

## Proof of Concept

Unit test sketch (safe, deterministic, no mainnet interaction):

```rust
#[test]
fn duplicate_stripped_dealings_spawn_multiple_tasks() {
    let dealing_id = fake_idkg_dealing(NODE_1, 1).id();
    // Craft a StrippedBlockProposal with 1000 duplicate dealing entries
    let mut stripped = fake_stripped_block_proposal_with_messages(vec![]);
    for _ in 0..1000 {
        stripped.stripped_idkg_dealings.stripped_dealings
            .push((1u32, dealing_id.clone()));
    }
    let assembler = BlockProposalAssembler::new(stripped);
    let missing = assembler.missing_stripped_messages();
    // Without deduplication, this returns 1000 tasks instead of 1
    assert_eq!(missing.len(), 1,
        "Expected deduplication; got {} tasks spawned", missing.len());
}
```

This test will fail on the current code (returning 1000 instead of 1), directly demonstrating the bug. The `fake_stripped_block_proposal_with_messages` and `fake_idkg_dealing` helpers already exist in the test utilities used by the existing test suite. [8](#0-7)
