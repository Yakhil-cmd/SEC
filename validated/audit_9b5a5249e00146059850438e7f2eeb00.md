Audit Report

## Title
Unbounded `disburse_maturity_in_progress` Vec Allows O(N) Ledger Calls Per Periodic Task — (`rs/sns/governance/src/governance.rs`)

## Summary

The SNS governance `disburse_maturity` function pushes entries to `disburse_maturity_in_progress` without any cap on the vector length. The NNS governance enforces `MAX_NUM_DISBURSEMENTS = 10`; the SNS governance has no equivalent guard. An attacker holding a neuron with `DisburseMaturity` permission and sufficient maturity can enqueue thousands of pending disbursements, causing O(N) inter-canister ledger calls during every `run_periodic_tasks` invocation and effectively DoS-ing SNS governance periodic operations.

## Finding Description

In `rs/sns/governance/src/governance.rs`, the `disburse_maturity` function performs only two substantive checks before pushing a new entry:

1. `percentage_to_disburse` is in `[1, 100]`
2. `worst_case_maturity_modulation >= transaction_fee_e8s`

It then unconditionally pushes to the vec with no length check:

```rust
neuron
    .disburse_maturity_in_progress
    .push(disbursement_in_progress);
``` [1](#0-0) 

There is no `MAX_DISBURSE_MATURITY_IN_PROGRESS` constant and no `disburse_maturity_in_progress.len()` check anywhere in the SNS governance code, confirmed by grep returning zero matches for such patterns in `rs/sns/**`.

By contrast, the NNS governance defines and enforces:

```rust
const MAX_NUM_DISBURSEMENTS: usize = 10;
// ...
if num_disbursements >= MAX_NUM_DISBURSEMENTS {
    return Err(InitiateMaturityDisbursementError::TooManyDisbursements);
}
``` [2](#0-1) [3](#0-2) 

The `maybe_finalize_disburse_maturity` function (called from `run_periodic_tasks`) iterates all entries in `disburse_maturity_in_progress` and issues one ledger transfer call per ready entry. With thousands of entries, this becomes O(N) inter-canister calls per heartbeat.

The natural floor imposed by the transaction fee check: with a starting maturity of `u64::MAX` (~1.8×10¹⁹ e8s) and a fee of 10,000 e8s, each call at 1% deducts geometrically decreasing amounts. The floor is reached after approximately **~3,460 calls** (solving `u64::MAX × 0.99^n × 0.95 ≥ 10,000`), yielding a vec of ~3,460 entries.

## Impact Explanation

With ~3,460 `DisburseMaturityInProgress` entries in a single neuron, every invocation of `run_periodic_tasks` triggers ~3,460 outbound ledger calls. This causes:

1. **Application-level DoS of SNS governance**: The periodic task exhausts its instruction/cycle budget on ledger calls for one attacker neuron, blocking proposal execution, reward distribution, and disbursement finalization for all other users.
2. **Increased serialization cost**: The bloated neuron proto must be deserialized/serialized on every heartbeat.
3. **Potential task timeout**: Other governance operations dependent on `run_periodic_tasks` are starved.

This matches the allowed High impact: *"Application/platform-level DoS, crash, consensus blocking, certified-state disruption, or subnet availability impact not based on raw volumetric DDoS"* and *"Significant SNS security impact with concrete user or protocol harm."*

**Severity: High ($2,000–$10,000)**

## Likelihood Explanation

The attacker must legitimately hold an SNS neuron with `DisburseMaturity` permission and sufficient accumulated maturity. No privileged access, key compromise, or consensus attack is required. The attack is fully executable via the public `manage_neuron` ingress endpoint by any SNS participant who has accumulated enough maturity through normal staking and reward participation. The attack is repeatable and deterministic.

## Recommendation

Add an explicit cap on `disburse_maturity_in_progress.len()` in the SNS `disburse_maturity` function, mirroring the NNS pattern. Insert before the push at line 1696:

```rust
const MAX_DISBURSE_MATURITY_IN_PROGRESS: usize = 10;

if neuron.disburse_maturity_in_progress.len() >= MAX_DISBURSE_MATURITY_IN_PROGRESS {
    return Err(GovernanceError::new_with_message(
        ErrorType::PreconditionFailed,
        "Too many disbursements in progress for this neuron.",
    ));
}
```

Add a corresponding unit test mirroring `test_initiate_maturity_disbursement_too_many_disbursements` from `rs/nns/governance/src/governance/disburse_maturity_tests.rs`. [2](#0-1) 

## Proof of Concept

1. Create an SNS neuron with large `maturity_e8s_equivalent` (e.g., `u64::MAX` in a test environment).
2. Repeatedly call `manage_neuron(DisburseMaturity { percentage_to_disburse: 1, to_account: None })` as the neuron controller.
3. Observe that each call succeeds and `disburse_maturity_in_progress.len()` grows by 1 each time — confirmed by the absence of any length check in `rs/sns/governance/src/governance.rs` lines 1609–1706.
4. After ~3,460 calls, the vec reaches its natural floor (transaction fee check blocks further entries).
5. Advance time past `MATURITY_DISBURSEMENT_DELAY_SECONDS` (7 days) and invoke `run_periodic_tasks`; observe ~3,460 outbound ledger calls issued in a single periodic task execution, exhausting the canister's per-round instruction budget.

The NNS test `test_initiate_maturity_disbursement_too_many_disbursements` in `rs/nns/governance/src/governance/disburse_maturity_tests.rs` confirms the NNS guard works correctly; no equivalent test exists in the SNS governance test suite. [4](#0-3)

    pub fn disburse_maturity(
        &mut self,
        id: &NeuronId,
        caller: &PrincipalId,
        disburse_maturity: &DisburseMaturity,
    ) -> Result<DisburseMaturityResponse, GovernanceError> {
        let neuron = self.get_neuron_result(id)?;
        neuron.check_authorized(caller, NeuronPermissionType::DisburseMaturity)?;

        // If no account was provided, transfer to the caller's account.
        let to_account: Account = match disburse_maturity.to_account.as_ref() {
            None => Account {
                owner: caller.0,
                subaccount: None,
            },
            Some(account) => Account::try_from(account.clone()).map_err(|e| {
                GovernanceError::new_with_message(
                    ErrorType::InvalidCommand,
                    format!("The given account to disburse the maturity to is invalid due to: {e}"),
                )
            })?,
        };
        let to_account_proto: AccountProto = AccountProto::from(to_account);

        if disburse_maturity.percentage_to_disburse > 100
            || disburse_maturity.percentage_to_disburse == 0
        {
            return Err(GovernanceError::new_with_message(
                ErrorType::PreconditionFailed,
                "The percentage of maturity to disburse must be a value between 1 and 100 (inclusive).",
            ));
        }

        // The amount to deduct = the amount in the neuron * request.percentage / 100.
        let maturity_to_deduct = neuron
            .maturity_e8s_equivalent
            .checked_mul(disburse_maturity.percentage_to_disburse as u64)
            .expect("Overflow while processing maturity to disburse.")
            .checked_div(100)
            .expect("Error when processing maturity to disburse.")
            as u128;

        let maturity_to_deduct = maturity_to_deduct as u64;

        let transaction_fee_e8s = self.transaction_fee_e8s_or_panic();
        let worst_case_maturity_modulation =
            apply_maturity_modulation(maturity_to_deduct, MIN_MATURITY_MODULATION_PERMYRIAD)
                // Applying maturity modulation is a safe operation.
                // However, in the case that the method fails to apply the equation, return an
                // error instead of throwing a panic.
                .map_err(|err| {
                    GovernanceError::new_with_message(
                        ErrorType::PreconditionFailed,
                        format!(
                            "Could not calculate worst case maturity modulation \
                            and therefore cannot disburse maturity. Err: {err}"
                        ),
                    )
                })?;

        if worst_case_maturity_modulation < transaction_fee_e8s {
            return Err(GovernanceError::new_with_message(
                ErrorType::PreconditionFailed,
                format!(
                    "If worst case maturity modulation is applied (-5%) then this neuron would \
                     disburse {worst_case_maturity_modulation} e8s, but can't disburse an amount less than the transaction fee \
                     of {transaction_fee_e8s} e8s."
                ),
            ));
        }

        let now_seconds = self.env.now();
        let disbursement_in_progress = DisburseMaturityInProgress {
            amount_e8s: maturity_to_deduct,
            timestamp_of_disbursement_seconds: now_seconds,
            account_to_disburse_to: Some(to_account_proto),
            finalize_disbursement_timestamp_seconds: Some(
                now_seconds + MATURITY_DISBURSEMENT_DELAY_SECONDS,
            ),
        };

        // Re-borrow the neuron mutably to update now that the maturity has been
        // deducted and is waiting until the end of the window to modulate and disburse.
        let neuron = self.get_neuron_result_mut(id)?;
        neuron.maturity_e8s_equivalent = neuron
            .maturity_e8s_equivalent
            .saturating_sub(maturity_to_deduct);
        neuron
            .disburse_maturity_in_progress
            .push(disbursement_in_progress);

        Ok(DisburseMaturityResponse {
            // We still populate this field even though it's deprecated, since we cannot remove
            // required fields yet.
            amount_disbursed_e8s: maturity_to_deduct,
            amount_deducted_e8s: Some(maturity_to_deduct),
        })
    }
