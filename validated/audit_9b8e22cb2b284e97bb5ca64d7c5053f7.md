### Title
ERC-20 Tokens Permanently Burned When Engine Is Paused During `exit_to_near_precompile_callback` — (`engine/src/contract_methods/connector.rs`)

---

### Summary

When a user calls the `exit_to_near` EVM precompile, their ERC-20 tokens are burned immediately and an `ft_transfer` promise is scheduled with `exit_to_near_precompile_callback` as its callback. If the engine is paused (via `pause_contract`) between the promise dispatch and the callback execution, the callback aborts at the `require_running` guard before it can reach the refund branch. The burned ERC-20 tokens are never re-minted, causing permanent loss of user funds.

---

### Finding Description

The `exit_to_near` precompile burns the user's ERC-20 tokens and schedules an `ft_transfer` on the NEP-141 contract. The callback `exit_to_near_precompile_callback` is responsible for two outcomes:

1. **Success path** — if `ft_transfer` succeeded, optionally transfer unwrapped wNEAR.
2. **Refund path** — if `ft_transfer` failed, re-mint the burned ERC-20 tokens via `refund_on_error`.

The callback begins with:

```rust
let state = state::get_state(&io)?;
require_running(&state)?;   // ← aborts here if engine is paused
``` [1](#0-0) 

If the engine is paused after the ERC-20 burn but before the callback executes, `require_running` returns `ERR_PAUSED`, the function panics, and neither the success path nor the refund path is reached. [2](#0-1) 

The refund logic that would re-mint the burned tokens:

```rust
} else if let Some(args) = args.refund {
    let refund_result = engine::refund_on_error(io, env, state, &args, handler)?;
    ...
}
```

is never executed. The ERC-20 tokens are permanently destroyed with no corresponding NEP-141 transfer and no on-chain recovery path.

The engine-level pause is a separate mechanism from the precompile-level pause: [3](#0-2) [4](#0-3) 

Pausing the engine is a legitimate owner action (e.g., for upgrades or emergency response). In NEAR, callbacks execute as separate receipts and cannot be retried once they fail. There is no recovery function analogous to `redistributeLockedAmount` that accounts for tokens in this intermediate burned-but-not-refunded state.

---

### Impact Explanation

**Critical — Permanent freezing/loss of funds.**

When `ft_transfer` fails (e.g., recipient not registered with the NEP-141 contract) and the engine is simultaneously paused, the user's ERC-20 tokens are burned with no refund. The tokens are gone from the EVM state and the NEP-141 balance is unchanged. There is no admin function that can recover tokens from this intermediate state because the burned ERC-20 balance no longer exists anywhere — it was destroyed before the callback ran.

---

### Likelihood Explanation

**Low-Medium.** The engine pause is a legitimate owner-only operation used during planned maintenance or emergency response. The vulnerable window is the NEAR receipt execution gap between the `ft_transfer` receipt and the `exit_to_near_precompile_callback` receipt. Any user whose `ft_transfer` fails (e.g., unregistered recipient) during a pause window loses their tokens permanently. The `ft_transfer` failure case is a documented and expected code path (the refund branch exists precisely for this reason), making the combination realistic.

---

### Recommendation

Move the `require_running` check to after the promise result is inspected, or exempt the refund branch from the running check entirely. The refund path does not perform any state-advancing operation — it only re-mints tokens to correct an error — and should be allowed to execute regardless of the engine's pause state:

```rust
pub fn exit_to_near_precompile_callback<...>(...) {
    with_hashchain(io, env, function_name!(), |io| {
        let state = state::get_state(&io)?;
        env.assert_private_call()?;

        if handler.promise_results_count() != 1 {
            return Err(errors::ERR_PROMISE_COUNT.into());
        }

        let args: ExitToNearPrecompileCallbackArgs = io.read_input_borsh()?;

        if let Some(PromiseResult::Successful(_)) = handler.promise_result(0) {
            require_running(&state)?; // only gate the success/advance path
            // ... transfer NEAR ...
        } else if let Some(args) = args.refund {
            // refund is always allowed, even when paused
            let refund_result = engine::refund_on_error(io, env, state, &args, handler)?;
            ...
        }
        Ok(maybe_result)
    })
}
```

---

### Proof of Concept

1. User calls the `exit_to_near` EVM precompile targeting an unregistered NEP-141 recipient. ERC-20 tokens are burned from the user's EVM balance. An `ft_transfer` promise is dispatched with `exit_to_near_precompile_callback` attached. [5](#0-4) 

2. Before the callback receipt executes, the engine owner calls `pause_contract`. [6](#0-5) 

3. The `ft_transfer` receipt executes and fails (recipient not registered).

4. `exit_to_near_precompile_callback` executes as the callback receipt. `require_running` returns `ERR_PAUSED` and the function panics. [7](#0-6) 

5. The refund branch (`refund_on_error` → re-mint ERC-20) is never reached. [8](#0-7) 

6. The user's ERC-20 tokens are permanently destroyed. The engine's `refund_on_error` function has no way to recover tokens that no longer exist in any account's balance. [9](#0-8)

pub fn exit_to_near_precompile_callback<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<Option<SubmitResult>, ContractError> {
    with_hashchain(io, env, function_name!(), |io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;
        env.assert_private_call()?;

        // This function should only be called as the callback of
        // exactly one promise.
        if handler.promise_results_count() != 1 {
            return Err(errors::ERR_PROMISE_COUNT.into());
        }

        let args: ExitToNearPrecompileCallbackArgs = io.read_input_borsh()?;

        let maybe_result = if let Some(PromiseResult::Successful(_)) = handler.promise_result(0) {
            if let Some(args) = args.transfer_near {
                let action = PromiseAction::Transfer {
                    amount: Yocto::new(args.amount),
                };
                let promise = PromiseBatchAction {
                    target_account_id: args.target_account_id,
                    actions: vec![action],
                };

                // Safety: this call is safe because it comes from the exit to near precompile, not users.
                // The call is to transfer the unwrapped wNEAR tokens.
                let promise_id = handler.promise_create_batch(&promise);
                handler.promise_return(promise_id);
            }

            None
        } else if let Some(args) = args.refund {
            // Exit call failed; need to refund tokens
            let refund_result = engine::refund_on_error(io, env, state, &args, handler)?;

            if !refund_result.status.is_ok() {
                return Err(errors::ERR_REFUND_FAILURE.into());
            }

            Some(refund_result)
        } else {
            None
        };

        Ok(maybe_result)
    })
}

#[named]
pub fn pause_contract<I: IO + Copy, E: Env>(io: I, env: &E) -> Result<(), ContractError> {
    with_hashchain(io, env, function_name!(), |mut io| {
        let mut state = state::get_state(&io)?;
        require_owner_only(&state, &env.predecessor_account_id())?;
        require_running(&state)?;
        state.is_paused = true;
        state::set_state(&mut io, &state)?;
        Ok(())
    })

    if let Some(erc20_address) = args.erc20_address {
        // ERC-20 exit; re-mint burned tokens
        let erc20_admin_address = current_address(&current_account_id);
        let mut engine: Engine<_, _> =
            Engine::new_with_state(state, erc20_admin_address, current_account_id, io, env);

        let refund_address = args.recipient_address;
        let amount = U256::from_big_endian(&args.amount);
        let input = setup_refund_on_error_input(amount, refund_address);

        engine.call(
            &erc20_admin_address,

            u64::MAX,
