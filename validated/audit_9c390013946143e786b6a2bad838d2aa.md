### Title
Hardcoded ICP Transfer Fee in SNS Swap `sweep_icp` Causes Finalization Failure if Ledger Fee Changes - (File: rs/sns/swap/src/swap.rs)

---

### Summary

The `sweep_icp` function in the SNS Swap canister hardcodes the ICP ledger transfer fee as `DEFAULT_TRANSFER_FEE` (10,000 e8s) instead of querying or storing the actual fee. The ICP ledger itself explicitly documents that this constant may not match the deployed fee. If the ICP ledger fee is changed via NNS governance, every call to `sweep_icp` will fail with a `BadFee` error, permanently blocking swap finalization and locking all buyer ICP in the swap canister subaccounts.

---

### Finding Description

In `rs/sns/swap/src/swap.rs`, the `sweep_icp` function transfers ICP from buyer subaccounts to either the SNS governance canister (on commit) or back to buyers (on abort). It passes `DEFAULT_TRANSFER_FEE` as the fee to `transfer_helper`: [1](#0-0) 

After a successful transfer, it also records the fee using the same hardcoded constant: [2](#0-1) 

The `error_refund_icp` emergency path has the same issue: [3](#0-2) 

The constant itself is defined with an explicit warning that it may not match the deployed ledger fee: [4](#0-3) 

By contrast, `sweep_sns` correctly reads the fee dynamically from the `Init` struct: [5](#0-4) 

The ICP ledger enforces the fee strictly — if the submitted fee does not match the ledger's configured `transfer_fee`, it returns `BadFee`: [6](#0-5) 

The `transfer_helper` function propagates this as `TransferResult::Failure`, which increments `sweep_result.failure` and prevents finalization from completing: [7](#0-6) 

---

### Impact Explanation

If the NNS changes the ICP ledger transfer fee (a routine governance action), all in-progress SNS swaps that have not yet finalized will have their `sweep_icp` calls fail for every buyer. The `finalize_swap` flow halts when `sweep_icp` does not complete fully:

- In the **committed** path: buyer ICP cannot be transferred to the SNS governance treasury, blocking SNS neuron creation.
- In the **aborted** path: buyer ICP cannot be refunded, locking funds in the swap canister subaccounts indefinitely.
- The `error_refund_icp` emergency path also fails, removing the fallback recovery mechanism.

The `Init` struct stores no ICP ledger fee field (only `transaction_fee_e8s` for the SNS ledger), so there is no stored correct value to fall back to without a canister upgrade. [8](#0-7) 

---

### Likelihood Explanation

The ICP ledger fee has historically been stable at 10,000 e8s, but the NNS has the authority to change it via governance proposal. The `DEFAULT_TRANSFER_FEE` constant's own documentation acknowledges this risk. Any NNS proposal to adjust the ICP transfer fee — a plausible economic governance action — would immediately break all active SNS swap finalizations. The entry path (`finalize_swap`) is a public, permissionless endpoint callable by any unprivileged ingress sender.

---

### Recommendation

Store the ICP ledger transfer fee in the `Init` struct (analogous to `transaction_fee_e8s` for the SNS ledger), populated at swap initialization time. Use this stored value in `sweep_icp` and `error_refund_icp` instead of `DEFAULT_TRANSFER_FEE`. Alternatively, query the ICP ledger's `transfer_fee` endpoint before each sweep operation.

---

### Proof of Concept

1. An SNS swap is opened and buyers participate, depositing ICP into their subaccounts.
2. The NNS passes a governance proposal changing the ICP ledger transfer fee from 10,000 e8s to, e.g., 20,000 e8s.
3. The swap reaches its due date and `finalize_swap` is called (by any caller).
4. `sweep_icp` calls `transfer_helper` with `DEFAULT_TRANSFER_FEE` = 10,000 e8s.
5. The ICP ledger rejects every transfer with `BadFee { expected_fee: 20_000 }`.
6. All transfers land in `sweep_result.failure`; finalization halts with: `"Transferring ICP did not complete fully, some transfers were invalid or failed. Halting swap finalization"`.
7. Buyer ICP remains locked in swap subaccounts. `error_refund_icp` also fails for the same reason. [9](#0-8) [4](#0-3)

```rust
    pub(crate) async fn transfer_helper(
        &mut self,
        now_fn: fn(bool) -> u64,
        fee: Tokens,
        subaccount: Option<Subaccount>,
        dst: &Account,
        ledger: &dyn ICRC1Ledger,
    ) -> TransferResult {
        let amount = Tokens::from_e8s(self.amount_e8s);
        if amount <= fee {
            // Skip: amount too small...
            return TransferResult::AmountTooSmall;
        }
        if self.transfer_start_timestamp_seconds > 0 {
            // Operation in progress...
            return TransferResult::AlreadyStarted;
        }
        self.transfer_start_timestamp_seconds = now_fn(false);

        // The ICRC1Ledger Trait converts any errors to Err(NervousSystemError).
        // No panics should occur when issuing this transfer.
        let result = ledger
            .transfer_funds(
                amount.get_e8s().saturating_sub(fee.get_e8s()),
                fee.get_e8s(),
                subaccount,
                *dst,
                0,
            )
            .await;
        if self.transfer_start_timestamp_seconds == 0 {
            log!(
                ERROR,
                "Token disburse logic error: expected transfer start time",
            );
        }
        match result {
            Ok(h) => {
                self.transfer_success_timestamp_seconds = now_fn(true);
                log!(
                    INFO,
                    "Transferred {} from subaccount {:?} to {} at height {} in Ledger Canister {}",
                    amount,
                    subaccount,
                    dst,
                    h,
                    ledger.canister_id()
                );
                TransferResult::Success(h)
            }
            Err(e) => {
                self.transfer_start_timestamp_seconds = 0;
                self.transfer_success_timestamp_seconds = 0;
                log!(
                    ERROR,
                    "Failed to transfer {} from subaccount {:#?}: {}",
                    amount,
                    subaccount,
                    e
                );
                TransferResult::Failure(e.to_string())
            }
        }
```
