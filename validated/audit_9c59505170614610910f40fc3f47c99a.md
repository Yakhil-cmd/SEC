### Title
Reentrancy in `donate_account` / `AccountState::transfer` Allows Double Neuron Transfer — (`rs/nns/gtc/src/lib.rs`)

### Summary

`Gtc::donate_account` sets `has_donated = true` **after** the async `account.transfer(...).await` call returns. Because the IC execution model allows a canister to process a second ingress message while suspended at an inter-canister `.await` point, a second concurrent `donate_account` call from the same account owner can enter `AccountState::transfer`, pass all guards (which still see `has_donated = false`), clone the full `neuron_ids` list, and dispatch duplicate `transfer_gtc_neuron` calls to the Governance canister — one per neuron, per concurrent call.

---

### Finding Description

**Entrypoint:** `canister_update donate_account` — callable by any unprivileged principal who owns a GTC account.

**Execution path:**

```
donate_account_() [canister.rs:166]
  → gtc_mut().donate_account(&caller(), hex_pubkey).await [lib.rs:75]
      → account.transfer(custodian_neuron_id).await   [lib.rs:89]  ← SUSPEND POINT
      → account.has_donated = true                    [lib.rs:90]  ← SET AFTER AWAIT
```

Inside `AccountState::transfer`:

```rust
// lib.rs:174-210
pub async fn transfer(&mut self, custodian_neuron_id: Option<NeuronId>) -> Result<(), String> {
    if self.has_claimed   { return Err(...); }
    else if self.has_donated  { return Err(...); }   // ← guard checked at entry only
    else if self.has_forwarded { return Err(...); }
    // ...
    let neuron_ids = self.neuron_ids.clone();         // ← full list cloned here
    for neuron_id in neuron_ids {
        GovernanceCanister::transfer_gtc_neuron(neuron_id, custodian_neuron_id).await; // ← SUSPEND
        self.neuron_ids.retain(|id| id != &neuron_id); // ← retain called AFTER await
        // ...
    }
    Ok(())
}
```

**Reentrancy window — step by step:**

| Step | Call 1 | Call 2 | `has_donated` | `self.neuron_ids` |
|------|--------|--------|---------------|-------------------|
| 1 | enters `transfer()`, guard passes | — | `false` | `[n1,n2,…]` |
| 2 | calls `transfer_gtc_neuron(n1).await` → **SUSPENDS** | — | `false` | `[n1,n2,…]` (retain not yet called) |
| 3 | — | enters `donate_account`, passes `validate_public_key_against_caller` | `false` | `[n1,n2,…]` |
| 4 | — | enters `transfer()`, guard sees `has_donated=false` → **passes** | `false` | `[n1,n2,…]` |
| 5 | — | clones `neuron_ids=[n1,n2,…]`, calls `transfer_gtc_neuron(n1).await` → **SUSPENDS** | `false` | `[n1,n2,…]` |
| 6 | resumes, `retain` removes `n1`, continues to `n2`… | resumes, `retain` is no-op (n1 already gone), continues to `n2`… | `false` | `[n2,…]` |
| 7 | completes all neurons, returns `Ok(())` | completes all neurons, returns `Ok(())` | `false` | `[]` |
| 8 | sets `has_donated = true` | sets `has_donated = true` | `true` | `[]` |

Both calls clone the full `neuron_ids` list before any `retain` has been called, so both dispatch `transfer_gtc_neuron` for every neuron.

The partial mitigation (`self.neuron_ids.retain(...)` after each per-neuron await) only helps if Call 2 enters `transfer()` **after** Call 1 has already processed at least one neuron. If Call 2 enters while Call 1 is suspended at its very first `transfer_gtc_neuron` await — which is the common case — both calls clone the complete list.

---

### Impact Explanation

Each `transfer_gtc_neuron` call to the Governance canister requests that the GTC neuron's controller be changed to the custodian neuron. Whether the Governance canister rejects the second call (because the neuron is already controlled by the custodian) determines the severity:

- **If Governance rejects the duplicate:** The GTC records the second attempt in `failed_transferred_neurons`, `has_donated` is still set, and the account state is permanently corrupted (spurious failure entries). The donor cannot recover their neurons regardless.
- **If Governance processes the duplicate:** The neuron stake is merged into the custodian neuron twice, permanently destroying the donor's ICP with no recourse.

In either case the donor's neurons are irrevocably transferred and the GTC account state is inconsistent.

---

### Likelihood Explanation

- The attacker is the legitimate account owner (no privilege required beyond owning the GTC account).
- Two concurrent ingress messages are trivially submitted via any IC agent/SDK.
- The IC execution model explicitly allows a canister to process a second message while suspended at an inter-canister `.await` — this is the standard reentrancy surface on the IC.
- No timing precision is required: the window is open for the entire duration of the first `transfer_gtc_neuron` inter-canister call (one consensus round or more).

---

### Recommendation

Set `has_donated = true` (and check it) **before** the first `.await` point, following the IC reentrancy-prevention pattern:

```rust
// In donate_account, before calling transfer:
if account.has_donated {
    return Err("Account has already donated its funds".to_string());
}
account.has_donated = true;          // set flag BEFORE await
account.transfer(custodian_neuron_id).await?;
```

Similarly, `AccountState::transfer` should not be the sole guard; the flag must be committed to state before any inter-canister call is dispatched.

---

### Proof of Concept

State-machine test outline:

1. Initialize GTC with one account holding neurons `[n1, n2]`.
2. Submit two concurrent `donate_account` ingress messages from the account owner (same `hex_pubkey`).
3. Advance the state machine so both messages interleave at the `transfer_gtc_neuron` await point.
4. Assert: `transfer_gtc_neuron` was called **more than once** for `n1` (or `n2`).
5. Assert: `account.neuron_ids` is empty and `account.successfully_transferred_neurons.len() > neuron_count` **or** `failed_transferred_neurons` is non-empty — both indicate the invariant was violated.

        account.has_donated = true;

        Ok(())

    pub async fn transfer(&mut self, custodian_neuron_id: Option<NeuronId>) -> Result<(), String> {
        if self.has_claimed {
            return Err("Neurons already claimed".to_string());
        } else if self.has_donated {
            return Err("Account has already donated its funds".to_string());
        } else if self.has_forwarded {
            return Err("Account has already forwarded its funds".to_string());
        } else if custodian_neuron_id.is_none() {
            return Err("No custodian neuron ID is defined".to_string());
        }

        let custodian_neuron_id = custodian_neuron_id.unwrap();
        let neuron_ids = self.neuron_ids.clone();

        for neuron_id in neuron_ids {
            let result =
                GovernanceCanister::transfer_gtc_neuron(neuron_id, custodian_neuron_id).await;

            self.neuron_ids.retain(|id| id != &neuron_id);

            let mut donated_neuron = TransferredNeuron {
                neuron_id: Some(neuron_id),
                timestamp_seconds: now_secs(),
                error: None,
            };

            match result {
                Ok(_) => self.successfully_transferred_neurons.push(donated_neuron),
                Err(e) => {
                    donated_neuron.error = Some(e.to_string());
                    self.failed_transferred_neurons.push(donated_neuron)
                }
            }
        }

        Ok(())
    }
