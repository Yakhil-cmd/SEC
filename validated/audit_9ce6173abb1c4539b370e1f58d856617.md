## Title
Unconditional `neuron_ids` Removal Before Result Check Permanently Orphans ICP Stake on Failed Transfer â€” (`rs/nns/gtc/src/lib.rs`)

## Summary
`AccountState::transfer` removes each `neuron_id` from `self.neuron_ids` unconditionally at line 192, before inspecting the result of `GovernanceCanister::transfer_gtc_neuron`. The function always returns `Ok(())` regardless of per-neuron failures. Callers then set `has_donated = true` or `has_forwarded = true`, permanently sealing the account. Any neuron whose transfer failed transiently is evicted from GTC state with no retry path, while remaining in governance under GTC control â€” its ICP stake permanently inaccessible to any user.

## Finding Description
In `rs/nns/gtc/src/lib.rs`, `AccountState::transfer` (lines 188â€“210):

```rust
for neuron_id in neuron_ids {

        GovernanceCanister::transfer_gtc_neuron(neuron_id, custodian_neuron_id).await;

    self.neuron_ids.retain(|id| id != &neuron_id);   // line 192 â€” unconditional

    match result {
        Ok(_) => self.successfully_transferred_neurons.push(donated_neuron),
        Err(e) => {
            donated_neuron.error = Some(e.to_string());
            self.failed_transferred_neurons.push(donated_neuron)
            // neuron_id is gone from self.neuron_ids; no re-add, no retry
        }
    }
}
Ok(())   // line 209 â€” always Ok

The `retain` at line 192 fires before the `match`, so a failed transfer still evicts the neuron from `self.neuron_ids`. The function then returns `Ok(())` unconditionally.

`donate_account` propagates this with `?` and immediately sets `has_donated = true`: [2](#0-1) 

`forward_whitelisted_unclaimed_accounts` sets `has_forwarded = true` on `Ok(_)`, which is always the case: [3](#0-2) 

The guards at the top of `transfer()` then permanently block re-entry: [4](#0-3) 

On the governance side, `transfer_gtc_neuron` performs the ledger transfer with `?` before calling `remove_neuron`. If the ledger call fails, the donor neuron is **not** deleted â€” it remains in governance under GTC control: [5](#0-4) 

Post-failure state: `neuron_ids` is empty, `has_donated`/`has_forwarded` is `true`, the neuron exists in governance under GTC control, and no GTC method can ever reach it again.

## Impact Explanation
Any neuron whose `transfer_gtc_neuron` call fails transiently is permanently orphaned: it is not claimable, not re-donatable, not re-forwardable, and not controlled by any user. The ICP stake locked in that neuron is permanently inaccessible. This constitutes permanent loss of user funds (ICP stake) in an NNS canister, matching the **High** impact class: *unauthorized access to neurons, governance assets, wallets, identities, ledgers, or canister-controlled funds* â€” specifically permanent loss of GTC-controlled ICP stake with no recovery path.

## Likelihood Explanation
The trigger is a transient inter-canister call failure, which is realistic on the IC: governance canister upgrade in progress, ledger message queue full, cycles exhaustion on GTC mid-loop, or any governance-side precondition failure (e.g., recipient neuron not found, donor stake below transaction fee). `donate_account` is callable by any GTC account owner (unprivileged ingress). `forward_whitelisted_unclaimed_accounts` is callable by **anyone** after 188 days from genesis. No special privileges are required to trigger the vulnerable path â€” only the natural occurrence of a transient failure during execution.

## Recommendation
Move the `retain` call inside the `Ok` branch so the neuron ID is only removed on confirmed success:

```rust
match result {
    Ok(_) => {
        self.neuron_ids.retain(|id| id != &neuron_id);
        self.successfully_transferred_neurons.push(donated_neuron);
    }
    Err(e) => {
        donated_neuron.error = Some(e.to_string());
        self.failed_transferred_neurons.push(donated_neuron);
        // neuron_id remains in self.neuron_ids for retry
    }
}
```

Additionally, `transfer()` should return `Err` if any neuron failed, so callers do not set `has_donated`/`has_forwarded = true` prematurely, preserving the ability to retry.
