### Title
SNS Governance Lacks Voting Power Spike Detection After Bulk Neuron Creation via `claim_swap_neurons` - (`rs/sns/governance/src/governance.rs`)

---

### Summary

SNS Governance's `compute_ballots_for_new_proposal()` computes quorum and ballots using the live current voting power with no historical checkpoint comparison. NNS Governance has an explicit `VotingPowerSnapshots` mechanism that detects sudden spikes in total voting power and falls back to a pre-spike snapshot. SNS Governance has no equivalent. After a swap finalizes and all participant neurons are created in bulk via `claim_swap_neurons()`, a participant holding a dominant share of voting power can immediately submit and pass a treasury-draining proposal before other participants can react.

---

### Finding Description

**NNS Governance (has the protection):**

NNS Governance's `compute_ballots_for_standard_proposal()` checks a rolling history of daily voting power snapshots. If the current total potential voting power exceeds 1.5× the minimum in the snapshot window, it falls back to the pre-spike snapshot for ballot creation, preventing a newly-minted whale from immediately passing proposals. [1](#0-0) 

The snapshots are taken daily by `SnapshotVotingPowerTask`: [2](#0-1) 

The spike detection threshold and staleness window are defined in `VotingPowerSnapshots`: [3](#0-2) 

**SNS Governance (missing the protection):**

SNS Governance's `compute_ballots_for_new_proposal()` simply iterates over `self.proto.neurons` at the current moment with no historical comparison: [4](#0-3) 

There is no `VotingPowerSnapshots`, no `SnapshotVotingPowerTask`, and no spike detection anywhere in the SNS governance codebase.

**The bulk neuron creation trigger:**

When a swap finalizes, `finalize_inner()` calls `claim_swap_neurons()` which creates all participant neurons at once in a single execution: [5](#0-4) 

`claim_swap_neurons()` in SNS Governance adds every neuron immediately to `self.proto.neurons`: [6](#0-5) 

Immediately after, governance is switched to Normal mode, enabling all proposal types including `TransferSnsTreasuryFunds`: [7](#0-6) 

**The quorum acceptance check:**

SNS proposal acceptance uses `tally.total` (sum of ballot voting powers set at proposal creation) as the denominator for quorum: [8](#0-7) 

The default quorum thresholds are 3% of total and 50% of exercised: [9](#0-8) 

---

### Impact Explanation

An attacker who controls a majority of voting power immediately after swap finalization can:

1. Submit a `TransferSnsTreasuryFunds` or `MintSnsTokens` proposal in the first message after governance enters Normal mode.
2. Vote YES with their newly-created neurons.
3. Since they hold >50% of exercised voting power and >3% of total voting power, the proposal is accepted immediately via early decision.
4. The SNS treasury (ICP and SNS tokens accumulated from the swap) is drained.

Other participants cannot react because their neurons were also just created — they have no prior governance history, no established following relationships, and no time to organize a counter-vote before the early-decision threshold is crossed.

---

### Likelihood Explanation

The Neurons' Fund can participate in SNS swaps and may receive a large allocation. A single large direct participant or the Neurons' Fund holding >50% of voting-eligible stake (neurons with dissolve delay ≥ `neuron_minimum_dissolve_delay_to_vote_seconds`) can execute this attack. The swap basket assigns varying dissolve delays to neurons, so some portion of every participant's neurons will be voting-eligible immediately. The attack window opens the moment `set_sns_governance_to_normal_mode` completes, which is a deterministic, publicly observable on-chain event. [10](#0-9) 

---

### Recommendation

Port the NNS `VotingPowerSnapshots` mechanism to SNS Governance:

1. Add a recurring timer task to SNS Governance that snapshots total voting power daily (analogous to `SnapshotVotingPowerTask`).
2. In `compute_ballots_for_new_proposal()`, compare the current total voting power against the snapshot history. If a spike is detected (e.g., >1.5× the minimum in the window), use the pre-spike snapshot for ballot creation.
3. Alternatively, enforce a governance delay (minimum voting period before any proposal can be executed) that is long enough for all participants to claim their neurons and organize, similar to the fix applied to the NounsBuilder protocol. [11](#0-10) 

---

### Proof of Concept

**Setup:** An SNS swap completes. Participant A received 60% of all voting-eligible SNS tokens (e.g., as the Neurons' Fund or a single large direct participant). All neurons are created via `claim_swap_neurons()`.

**Attack sequence (all in the same or adjacent messages after `set_mode(Normal)`):**

1. Participant A calls `manage_neuron` → `MakeProposal` with action `TransferSnsTreasuryFunds` targeting their own ICP account.

2. SNS Governance calls `compute_ballots_for_new_proposal()`:
   - Iterates `self.proto.neurons` — all swap neurons now exist
   - Participant A's neurons contribute 60% of `total_power`
   - Ballot is created with A's voting power = 60% of total

3. Participant A calls `manage_neuron` → `RegisterVote(Yes)` for their neurons.

4. `is_accepted()` evaluates:
   - `yes = 60%`, `total = 100%`
   - `quorum_met`: `60% * 10_000 >= 100% * 300` → `600_000 >= 300` ✓
   - `majority_met`: `60% > 50% of exercised (60%)` → `60/60 = 100% > 50%` ✓
   - Proposal adopted immediately.

5. Treasury funds are transferred to Participant A. [12](#0-11) [13](#0-12)

/// The maximum number of voting power snapshots to keep.
const MAX_VOTING_POWER_SNAPSHOTS: u64 = 7;
/// The multiplier used to define what is a "voting power spike": if the current total voting
/// power is more than this multiplier times the minimum total voting power in the snapshots,
/// then we consider it a spike.
const MULTIPLIER_THRESHOLD_FOR_VOTING_POWER_SPIKE: f64 = 1.5;
/// The maximum staleness of a voting power snapshot. This is usually not needed since
/// the snapshots should be added frequently. However, we do not want to use a snapshot that is too
/// old, in the event of a failure in taking the snapshots.
const MAXIMUM_STALENESS_SECONDS: u64 = ONE_MONTH_SECONDS * 3;

type DefaultMemory = VirtualMemory<DefaultMemoryImpl>;
type TimestampSeconds = u64;

/// A collection of voting power snapshots, each associated with a timestamp. The totals are used to
/// detect whether there is a spike of total voting power, and the voting power per neuron is used
/// to create ballots if a spike is detected. Note that a snapshot is stored in 2 separate
/// `StableBTreeMap` so that the totals can be checked without having to load the entire
/// `NeuronIdToVotingPowerMap` into memory.
pub(crate) struct VotingPowerSnapshots {
    neuron_id_to_voting_power_maps:
        StableBTreeMap<TimestampSeconds, NeuronIdToVotingPowerMap, DefaultMemory>,
    voting_power_totals: StableBTreeMap<TimestampSeconds, VotingPowerTotal, DefaultMemory>,
}

fn insert_and_truncate<Value: Storable>(
    map: &mut StableBTreeMap<TimestampSeconds, Value, DefaultMemory>,
    timestamp_seconds: TimestampSeconds,
    value: Value,
) {
    let existing_value = map.insert(timestamp_seconds, value);

    // Log if we just clobbered an existing entry, because it is a exceedingly unlikely
    // that this would happen in practice.
    if let Some(existing_value) = existing_value {
        eprintln!(
            "{}Somehow the voting power snapshot is taken multiple times at \
	            the same timestamp {}",
            LOG_PREFIX, timestamp_seconds,
        );
    }

    // Drop earlier entries from map.
    while map.len() > MAX_VOTING_POWER_SNAPSHOTS {
        let (first_key, _) = map
            .first_key_value()
            .expect("No first key value even though the length is checked right before.");
        map.remove(&first_key);
    }
}

    /// Computes the total potential voting power of the governance canister and ballots.
    fn compute_ballots_for_new_proposal(&self) -> Result<(u64, BTreeMap<String, Ballot>), String> {
        let now_seconds = self.env.now();

        let nervous_system_parameters = self.nervous_system_parameters_or_panic();

        // Voting power bonus parameters.
        let max_dissolve_delay = nervous_system_parameters
            .max_dissolve_delay_seconds
            .expect("NervousSystemParameters must have max_dissolve_delay_seconds");

        let max_age_bonus = nervous_system_parameters
            .max_neuron_age_for_age_bonus
            .expect("NervousSystemParameters must have max_neuron_age_for_age_bonus");

        let max_dissolve_delay_bonus_percentage = nervous_system_parameters
            .max_dissolve_delay_bonus_percentage
            .expect("NervousSystemParameters must have max_dissolve_delay_bonus_percentage");

        let max_age_bonus_percentage = nervous_system_parameters
            .max_age_bonus_percentage
            .expect("NervousSystemParameters must have max_age_bonus_percentage");

        let min_dissolve_delay_for_vote = nervous_system_parameters
            .neuron_minimum_dissolve_delay_to_vote_seconds
            .expect("NervousSystemParameters must have min_dissolve_delay_for_vote");

        let mut electoral_roll = BTreeMap::<String, Ballot>::new();
        let mut total_power: u128 = 0;

        for (k, v) in self.proto.neurons.iter() {
            // If this neuron is eligible to vote, record its
            // voting power at the time of proposal creation (now).
            if v.dissolve_delay_seconds(now_seconds) < min_dissolve_delay_for_vote {
                // Not eligible due to dissolve delay.
                continue;
            }

            let voting_power = v.voting_power(
                now_seconds,
                max_dissolve_delay,
                max_age_bonus,
                max_dissolve_delay_bonus_percentage,
                max_age_bonus_percentage,
            );

            total_power += voting_power as u128;
            electoral_roll.insert(
                k.clone(),
                Ballot {
                    vote: Vote::Unspecified as i32,
                    voting_power,
                    cast_timestamp_seconds: 0,
                },
            );
        }

        if total_power >= (u64::MAX as u128) {
            // The way the neurons are configured, the total voting
            // power on this proposal would overflow a u64!
            return Err("Voting power overflow.".to_string());
        }
        if electoral_roll.is_empty() {
            // Cannot make a proposal with no eligible voters.  This
            // is a precaution that shouldn't happen as we check that
            // the voter is allowed to vote.
            return Err("No eligible voters.".to_string());
        }

        Ok((total_power as u64, electoral_roll))
