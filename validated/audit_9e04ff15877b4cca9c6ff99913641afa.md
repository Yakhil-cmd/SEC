Audit Report

## Title
Node-Swap Phased-Rollout Caller and Subnet Allowlists Permanently Bypassed in Production — (File: rs/registry/canister/src/flags.rs)

## Summary

`NODE_SWAPPING_CALLERS_POLICY` and `NODE_SWAPPING_SUBNETS_POLICY` are hardcoded to `AccessList::allow_all()` in the production registry canister binary. The only code paths that can restrict these policies are compiled exclusively under `#[cfg(any(test, feature = "test"))]` and are never reachable in the production Wasm. As a result, the phased-rollout caller and subnet gating on `swap_node_in_subnet_directly` is completely non-functional on mainnet, and any registered node operator can swap their own nodes on any subnet without the intended governance-controlled rollout restrictions.

## Finding Description

**Root cause — permanently open policies:**

In `rs/registry/canister/src/flags.rs` (lines 18–20), both policies are initialized as `AccessList::allow_all()`:

```rust
static NODE_SWAPPING_CALLERS_POLICY: RefCell<AccessList<PrincipalId>> =
    RefCell::new(AccessList::allow_all());
static NODE_SWAPPING_SUBNETS_POLICY: RefCell<AccessList<SubnetId>> =
    RefCell::new(AccessList::allow_all());
```

`AccessList::allow_all()` is implemented as `DenyOnly(HashSet::new())` (`rs/nervous_system/access_list/src/lib.rs` lines 126–130), so `is_allowed()` unconditionally returns `true` for every input (lines 160–164).

**Root cause — setters are test-only:**

The only functions that can replace these policies are `test_set_swapping_whitelisted_callers` and `test_set_swapping_enabled_subnets`, both inside `pub mod temporary_overrides` gated by `#[cfg(any(test, feature = "test"))]` (`flags.rs` lines 65–82). These symbols do not exist in the production binary.

**Root cause — `canister_init` applies init-payload fields only under `#[cfg(feature = "test")]`:**

In `rs/registry/canister/canister/canister.rs` (lines 208–237), the block that calls `test_set_swapping_whitelisted_callers` and `test_set_swapping_enabled_subnets` from the init payload is wrapped in `#[cfg(feature = "test")]`. In the production build this entire block is dead code, so even if an operator were to supply `swapping_whitelisted_callers` / `swapping_enabled_subnets` in the init payload, they would be silently ignored.

**Exploit path:**

`swap_node_in_subnet_directly` in `canister.rs` (lines 831–842) has no governance-caller check — it is callable by any ingress sender. Inside `do_swap_node_in_subnet_directly` → `swap_nodes_inner` (`do_swap_node_in_subnet_directly.rs` lines 118–147):

1. `is_node_swapping_enabled()` → `true` (hardcoded default, line 11 of `flags.rs`).
2. `swapping_enabled_for_caller(caller)` calls `is_node_swapping_enabled_for_caller(caller)` (lines 150–156), which reads `NODE_SWAPPING_CALLERS_POLICY` → `allow_all()` → `true`.
3. `swapping_allowed_on_subnet(subnet_id)` calls `is_node_swapping_enabled_on_subnet(subnet_id)` (lines 169–175), which reads `NODE_SWAPPING_SUBNETS_POLICY` → `allow_all()` → `true`.

The only remaining checks are: caller must be the registered node operator of both nodes, the new node must be unassigned, and the subnet must not be halted. The phased-rollout caller and subnet gates are entirely inoperative.

## Impact Explanation

Any registered node operator who owns at least one node currently assigned to a subnet and one unassigned node can call `swap_node_in_subnet_directly` and alter the membership of that subnet — including subnets that were never intended to participate in the phased rollout. This bypasses the governance-controlled rollout gating that was explicitly designed to limit which operators and which subnets could use the feature. Subnet membership changes affect consensus participation and can degrade subnet availability or performance. This matches the allowed ICP bounty impact: **Medium** — meaningful security impact on NNS/registry governance with a significant attacker constraint (must be a registered node operator owning both nodes).

## Likelihood Explanation

The `swap_node_in_subnet_directly` endpoint is live and reachable via standard ingress. No special privilege beyond being a registered node operator with two owned nodes (one assigned, one unassigned) is required. The feature flag `IS_NODE_SWAPPING_ENABLED` defaults to `true`. Rate limits (1 swap per subnet per 4 hours, 1 swap per operator per subnet per 24 hours) constrain frequency but do not prevent exploitation. Any node operator meeting the ownership preconditions can trigger this immediately and repeatably.

## Recommendation

Move the policy-setting logic out of the `#[cfg(feature = "test")]` gate. The most direct fix is to apply `swapping_whitelisted_callers` and `swapping_enabled_subnets` from `RegistryCanisterInitPayload` unconditionally in `canister_init` (not only under `#[cfg(feature = "test")]`), and move `test_set_swapping_whitelisted_callers` / `test_set_swapping_enabled_subnets` out of the `#[cfg(any(test, feature = "test"))]` gate (or add production-accessible equivalents). Alternatively, add a governance-gated update method callable only by the governance canister to set `NODE_SWAPPING_CALLERS_POLICY` and `NODE_SWAPPING_SUBNETS_POLICY` at runtime. The `RegistryCanisterInitPayloadBuilder` methods `whitelist_swapping_feature_caller` and `enable_swapping_feature_for_subnet` already exist and are correct; the missing piece is applying them in the production binary path.

## Proof of Concept

1. Deploy the production registry canister (without `feature = "test"`). Confirm `NODE_SWAPPING_CALLERS_POLICY` and `NODE_SWAPPING_SUBNETS_POLICY` are `allow_all()` — no init-payload override is applied.
2. Register as a node operator with two nodes: `old_node` assigned to any subnet, `new_node` unassigned.
3. Send an ingress update call to `swap_node_in_subnet_directly` with `{ old_node_id: old_node, new_node_id: new_node }`.
4. Observe: `swapping_enabled_for_caller` returns `Ok(())` (policy is `allow_all()`); `swapping_allowed_on_subnet` returns `Ok(())` (policy is `allow_all()`); the swap proceeds on the target subnet regardless of whether that subnet or operator was intended to be part of the phased rollout.
5. Reproducible as a unit test by calling `swap_nodes_inner` directly without invoking `test_set_swapping_whitelisted_callers` or `test_set_swapping_enabled_subnets` — the default state of the policies allows the call through.
