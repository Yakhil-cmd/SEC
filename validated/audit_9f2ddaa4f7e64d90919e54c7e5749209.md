Audit Report

## Title
`spawn_neuron` Minimum-Stake Guard Uses Hardcoded 5% Worst-Case While Mission 70 Modulation Floor Is -10% — (`rs/nns/governance/src/governance.rs`)

## Summary

The pre-spawn guard in `spawn_neuron` computes `least_possible_stake` using a hardcoded 5% discount, but the Mission 70 maturity modulation system enforces a valid range down to -10% (`MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 = -1_000`). Any neuron whose `maturity_to_spawn` falls in the band `[neuron_minimum_stake_e8s / 0.95, neuron_minimum_stake_e8s / 0.90)` will pass the guard but be minted with `cached_neuron_stake_e8s < neuron_minimum_stake_e8s` when `maybe_spawn_neurons` applies a modulation between -5% and -10%, violating the protocol invariant.

## Finding Description

**Guard (governance.rs:2664–2673):**
```rust
let least_possible_stake = (maturity_to_spawn as f64 * (1_f64 - 0.05)) as u64;
if least_possible_stake < economics.neuron_minimum_stake_e8s {
    return Err(...);
}
```
The constant `0.05` encodes a -5% worst-case assumption.

**Actual Mission 70 floor (update_icp_xdr_rate_related_data.rs:47):**
```rust
pub(crate) const MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70: i64 = -1_000; // -10%
```

**Valid range accepted by `maybe_spawn_neurons` (governance.rs:276–278):**
```rust
const VALID_MATURITY_MODULATION_BASIS_POINTS_RANGE: RangeInclusive<i32> =
    MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 as i32
        ..=MATURITY_MODULATION_MAX_PERMYRIAD_MISSION_70 as i32;
```
The sanity check at governance.rs:6437–6447 accepts any value in `[-1000, 200]` as valid and does **not** return early — it only logs and returns on out-of-range values. Values in `[-1000, -500]` pass silently.

**Unconditional application (governance.rs:6484–6502):**
`apply_maturity_modulation(original_maturity, maturity_modulation)` is called with no post-modulation minimum-stake check. The result is written directly to `cached_neuron_stake_e8s` at line 6517.

The exploit path:
1. Controller calls `spawn_neuron` with `maturity_to_spawn` just above the 5% threshold (e.g., `105_263_159` e8s when `neuron_minimum_stake_e8s = 100_000_000`). Guard passes: `floor(105_263_159 × 0.95) = 100_000_001 ≥ 100_000_000`.
2. The neuron enters spawning state with `spawn_at_timestamp_seconds` set 7 days out.
3. Within those 7 days (or later), modulation drifts to -1000 permyriad (-10%), which is within the enforced valid range.
4. `maybe_spawn_neurons` fires: `apply_maturity_modulation(105_263_159, -1000)` = `floor(105_263_159 × 9000 / 10000)` = `94_736_843`.
5. Neuron is minted with `cached_neuron_stake_e8s = 94_736_843 < 100_000_000 = neuron_minimum_stake_e8s`.

No post-spawn minimum-stake enforcement exists anywhere in the `maybe_spawn_neurons` path.

**Note on the cited SNS comparison:** `MIN_MATURITY_MODULATION_PERMYRIAD` in `rs/nervous_system/governance/src/maturity_modulation/mod.rs` is `-500` (-5%), not -10%. The SNS `disburse_maturity` guard therefore also uses -5% as its worst case. The correct constant for the NNS Mission 70 fix is `MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 = -1_000`.

## Impact Explanation

A neuron is created on-chain with `cached_neuron_stake_e8s < neuron_minimum_stake_e8s`, violating the NNS governance protocol invariant. The neuron cannot be split (split enforces the minimum stake check). The controller must dissolve the neuron to recover the ICP, which may require waiting out a dissolve delay. This constitutes concrete user harm within the NNS governance system — a significant NNS security impact with demonstrable user-funds degradation. Severity: **High** ($2,000–$10,000) under "Significant NNS security impact with concrete user or protocol harm."

## Likelihood Explanation

- Reachable by any unprivileged neuron controller with maturity in the vulnerable band — no special role required.
- The trigger condition (modulation between -500 and -1000 permyriad) is within the documented and enforced Mission 70 valid range.
- The speed limit of 30 permyriad/day means reaching -1000 from 0 takes ~33 days of sustained ICP price decline — a realistic market scenario.
- The 7-day spawn window means a controller who spawns during a period of declining ICP price is exposed without any ability to cancel.
- Repeatable: any neuron with maturity in the vulnerable band is affected each time modulation is in `[-1000, -500)`.

## Recommendation

Replace the hardcoded `0.05` with the actual Mission 70 floor constant:

```rust
// Before
let least_possible_stake = (maturity_to_spawn as f64 * (1_f64 - 0.05)) as u64;

// After
let least_possible_stake = apply_maturity_modulation(
    maturity_to_spawn,
    MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 as i32,
).unwrap_or(0);
if least_possible_stake < economics.neuron_minimum_stake_e8s {
    return Err(...);
}
```

`MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70` is `-1_000` (-10%), matching the actual floor enforced by `VALID_MATURITY_MODULATION_BASIS_POINTS_RANGE`. Additionally, consider adding a post-modulation minimum-stake assertion inside `maybe_spawn_neurons` before writing `cached_neuron_stake_e8s`.

## Proof of Concept

```
neuron_minimum_stake_e8s = 100_000_000

// Step 1: spawn_neuron called with maturity just above 5% guard
maturity_to_spawn = 105_263_159
least_possible_stake = floor(105_263_159 * 0.95) = 100_000_001 >= 100_000_000  ✓ passes

// Step 2: ~33 days of ICP price decline, modulation reaches -1000 permyriad
// VALID_MATURITY_MODULATION_BASIS_POINTS_RANGE.contains(-1000) == true → no early return

// Step 3: maybe_spawn_neurons fires
apply_maturity_modulation(105_263_159, -1000)
  = floor(105_263_159 * (10_000 - 1_000) / 10_000)
  = floor(105_263_159 * 9_000 / 10_000)
  = 94_736_843

// Invariant violated:
94_736_843 < 100_000_000  ← cached_neuron_stake_e8s below neuron_minimum_stake_e8s
```

A deterministic unit test can reproduce this by: (1) constructing a governance fixture with `neuron_minimum_stake_e8s = 100_000_000`, (2) calling `spawn_neuron` with `maturity_to_spawn = 105_263_159`, (3) advancing the clock past `spawn_at_timestamp_seconds`, (4) calling `maybe_spawn_neurons` with `maturity_modulation = -1000`, and (5) asserting `cached_neuron_stake_e8s < neuron_minimum_stake_e8s`.
