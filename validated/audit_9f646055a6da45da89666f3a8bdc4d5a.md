### Title
`ExitToEthereum` Precompile Burns EVM Tokens Before NEAR-Side `withdraw` Promise With No Failure Callback, Enabling Permanent Fund Freeze - (File: `engine-precompiles/src/native.rs`)

### Summary

The `ExitToEthereum` precompile (`0xb0bd02f6...`) burns EVM tokens (ETH or ERC-20) atomically during EVM execution, then schedules a `withdraw` NEAR promise to the ETH connector. Unlike the `ExitToNear` precompile, `ExitToEthereum` attaches **no callback** to handle promise failure. If the `withdraw` promise fails for any reason (connector paused, gas exhaustion, connector bug), the EVM-side burn is permanent and irrecoverable, permanently freezing the user's funds.

### Finding Description

The `ExitToEthereum` precompile's `run` method constructs only a bare `PromiseArgs::Create` with no attached callback:

```rust
// engine-precompiles/src/native.rs:977-985
let withdraw_promise = PromiseCreateArgs {
    target_account_id: nep141_address,
    method: "withdraw".to_string(),
    args: serialized_args,
    attached_balance: Yocto::new(1),
    attached_gas: costs::WITHDRAWAL_GAS,
};

let promise = borsh::to_vec(&PromiseArgs::Create(withdraw_promise)).unwrap();
``` [1](#0-0) 

This is structurally different from `ExitToNear`, which (when the `error_refund` feature is enabled) wraps the transfer promise in a `PromiseArgs::Callback` that calls `exit_to_near_precompile_callback` on failure to re-mint burned tokens:

```rust
// engine-precompiles/src/native.rs:470-483
let promise = if callback_args == ExitToNearPrecompileCallbackArgs::default() {
    PromiseArgs::Create(transfer_promise)
} else {
    PromiseArgs::Callback(PromiseWithCallbackArgs {
        base: transfer_promise,
        callback: PromiseCreateArgs {
            method: "exit_to_near_precompile_callback".to_string(),
            ...
        },
    })
};
``` [2](#0-1) 

The `exit_to_near_precompile_callback` handler explicitly re-mints burned ERC-20 tokens or refunds ETH from the precompile address when the NEAR-side promise fails:

```rust
// engine/src/contract_methods/connector.rs:231-239
} else if let Some(args) = args.refund {
    // Exit call failed; need to refund tokens
    let refund_result = engine::refund_on_error(io, env, state, &args, handler)?;
    ...

    None

``` [3](#0-2) 

`refund_on_error` re-mints burned ERC-20 tokens via `ERC20_MINT_SELECTOR` or transfers ETH back from the precompile address: [4](#0-3) 

No equivalent recovery path exists for `ExitToEthereum`. The `EvmErc20.sol` `withdrawToEthereum` function burns tokens before calling the precompile:

```solidity
// etc/eth-contracts/contracts/EvmErc20.sol:65-76
function withdrawToEthereum(address recipient, uint256 amount) external override {
    _burn(_msgSender(), amount);
    // ... calls ExitToEthereum precompile
}
``` [5](#0-4) 

The burn is committed in the EVM execution phase. The `withdraw` promise executes asynchronously in a subsequent NEAR receipt. If that receipt fails, the EVM state (burned tokens) is permanently inconsistent with the NEAR state (no withdrawal recorded), and there is no mechanism to recover the funds.

### Impact Explanation

**Permanent fund freeze (Critical).** For ERC-20 tokens: the `_burn` is irreversible; no re-mint occurs. For ETH: the value is transferred to the `ExitToEthereum` precompile address (`0xb0bd02f6a392af548bdf1cfaee5dfa0eefcc8eab`) during the EVM call; if the `withdraw` promise fails, the ETH is stranded at that address with no admin recovery function. In both cases the user loses their funds permanently with no protocol-side recourse.

### Likelihood Explanation

**Medium.** The ETH connector's `withdraw` function can fail in at least one realistic, non-adversarial scenario: the connector's FT transfer pause mechanism (added in v3.6.3, `CHANGES.md`) can be activated by an admin. If a user's `withdrawToEthereum` EVM execution commits in one NEAR block and the connector is paused before the `withdraw` receipt executes in the next block, the promise fails and the user's tokens are permanently frozen. Gas exhaustion is less likely given the 100 TGas allocation (`WITHDRAWAL_GAS`), but connector-side panics or future upgrades introduce additional failure modes. Any EVM user can trigger this path without special privileges. [6](#0-5) 

### Recommendation

Add a failure callback to `ExitToEthereum` analogous to the `exit_to_near_precompile_callback` pattern already used by `ExitToNear`. The callback should:

1. Check `handler.promise_result(0)` for failure.
2. On failure, call `engine::refund_on_error` with the appropriate `RefundCallArgs` (re-mint ERC-20 tokens to the original sender, or transfer ETH back from the precompile address).

The `ExitToNearPrecompileCallbackArgs` / `RefundCallArgs` types and `refund_on_error` function already exist and handle both the ERC-20 and ETH cases. The fix requires:
- Extending `ExitToEthereum::run` to build a `PromiseArgs::Callback` wrapping the `withdraw` promise with a new `exit_to_ethereum_precompile_callback` entry point.
- Adding a `refund_address` field to the `ExitToEthereum` input format (mirroring the `error_refund` path in `ExitToNear`).
- Registering the new callback in `engine/src/lib.rs`.

### Proof of Concept

1. User holds 1000 units of a bridged ERC-20 token on Aurora.
2. User calls `EvmErc20::withdrawToEthereum(eth_recipient, 1000)`.
3. `_burn(msg.sender, 1000)` executes — EVM balance drops to 0.
4. `ExitToEthereum` precompile schedules `PromiseArgs::Create(withdraw_promise)` targeting the ETH connector.
5. Before the `withdraw` receipt executes, an admin pauses FT transfers on the ETH connector.
6. The `withdraw` receipt fails (connector rejects the call).
7. No callback fires. The EVM burn is permanent. The user has 0 ERC-20 tokens on Aurora and 0 ETH unlocked on Ethereum.
8. Funds are permanently frozen with no recovery path. [7](#0-6) [5](#0-4)

impl<I: IO> Precompile for ExitToEthereum<I> {
    fn required_gas(_input: &[u8]) -> Result<EthGas, ExitError> {
        Ok(costs::EXIT_TO_ETHEREUM_GAS)
    }

    #[allow(clippy::too_many_lines)]
    fn run(
        &self,
        input: &[u8],
        target_gas: Option<EthGas>,
        context: &Context,
        is_static: bool,
    ) -> EvmPrecompileResult {
        // ETH (Base token) transfer input format (min size 21 bytes)
        //  - flag (1 byte)
        //  - eth_recipient (20 bytes)
        // ERC-20 transfer input format: max 53 bytes
        //  - flag (1 byte)
        //  - amount (32 bytes)
        //  - eth_recipient (20 bytes)
        validate_input_size(input, 21, 53)?;

        let required_gas = Self::required_gas(input)?;

        if let Some(target_gas) = target_gas
            && required_gas > target_gas
        {
            return Err(ExitError::OutOfGas);
        }

        // It's not allowed to call exit precompiles in static mode
        if is_static {
            return Err(ExitError::Other(Cow::from("ERR_INVALID_IN_STATIC")));
        } else if context.address != exit_to_ethereum::ADDRESS.raw() {
            return Err(ExitError::Other(Cow::from("ERR_INVALID_IN_DELEGATE")));
        }

        // The first byte of the input is a flag, selecting the behavior to be triggered:
        //  0x00 -> ETH (Base token) token transfer
        //  0x01 -> ERC-20 transfer
        let mut input = input;
        let flag = input[0];
        input = &input[1..];

        let (nep141_address, serialized_args, exit_event) = match flag {
            0x0 => {
                // ETH (base) transfer
                //
                // Input slice format:
                //  eth_recipient (20 bytes) - the address of recipient which will receive ETH on Ethereum
                let recipient_address: Address = input
                    .try_into()
                    .map_err(|_| ExitError::Other(Cow::from("ERR_INVALID_RECIPIENT_ADDRESS")))?;
                let serialize_fn = match get_withdraw_serialize_type(&self.io)? {
                    WithdrawSerializeType::Json => json_args,
                    WithdrawSerializeType::Borsh => borsh_args,
                };
                let eth_connector_account_id = self.get_eth_connector_contract_account()?;

                (
                    eth_connector_account_id,
                    // There is no way to inject json, given the encoding of both arguments
                    // as decimal and hexadecimal respectively.
                    serialize_fn(recipient_address, context.apparent_value)?,
                    events::ExitToEth {
                        sender: Address::new(context.caller),
                        erc20_address: events::ETH_ADDRESS,
                        dest: recipient_address,
                        amount: context.apparent_value,
                    },
                )
            }
            0x1 => {
                // ERC-20 transfer
                //
                // This precompile branch is expected to be called from the ERC20 withdraw function
                // (or burn function with some flag provided that this is expected to be withdrawn)
                //
                // Input slice format:
                //  amount (U256 big-endian bytes) - the amount that was burned
                //  eth_recipient (20 bytes) - the address of recipient which will receive ETH on Ethereum

                if context.apparent_value != U256::from(0) {
                    return Err(ExitError::Other(Cow::from(
                        "ERR_ETH_ATTACHED_FOR_ERC20_EXIT",
                    )));
                }

                let erc20_address = context.caller;
                let nep141_address = get_nep141_from_erc20(erc20_address.as_bytes(), &self.io)?;
                let amount = parse_amount(&input[..32])?;

                input = &input[32..];

                if input.len() == 20 {
                    // Parse ethereum address in hex
                    let mut buffer = [0; 40];
                    hex::encode_to_slice(input, &mut buffer).unwrap();
                    let recipient_in_hex = str::from_utf8(&buffer).map_err(|_| {
                        ExitError::Other(Cow::from("ERR_INVALID_RECIPIENT_ADDRESS"))
                    })?;
                    // unwrap cannot fail since we checked the length already
                    let recipient_address = Address::try_from_slice(input)
                        .map_err(|_| ExitError::Other(Cow::from("ERR_WRONG_ADDRESS")))?;

                    (
                        nep141_address,
                        // There is no way to inject json, given the encoding of both arguments
                        // as decimal and hexadecimal respectively.
                        format!(
                            r#"{{"amount": "{}", "recipient": "{}"}}"#,
                            amount.as_u128(),
                            recipient_in_hex
                        )
                        .into_bytes(),
                        events::ExitToEth {
                            sender: Address::new(erc20_address),
                            erc20_address: Address::new(erc20_address),
                            dest: recipient_address,
                            amount,
                        },
                    )
                } else {
                    return Err(ExitError::Other(Cow::from("ERR_INVALID_RECIPIENT_ADDRESS")));
                }
            }
            _ => {
                return Err(ExitError::Other(Cow::from(
                    "ERR_INVALID_RECEIVER_ACCOUNT_ID",
                )));
            }
        };

        let withdraw_promise = PromiseCreateArgs {
            target_account_id: nep141_address,
            method: "withdraw".to_string(),
            args: serialized_args,
            attached_balance: Yocto::new(1),
            attached_gas: costs::WITHDRAWAL_GAS,
        };

        let promise = borsh::to_vec(&PromiseArgs::Create(withdraw_promise)).unwrap();
        let promise_log = Log {
            address: exit_to_ethereum::ADDRESS.raw(),
            topics: Vec::new(),
            data: promise,
        };
        let ethabi::RawLog { topics, data } = exit_event.encode();
        let exit_event_log = Log {
            address: exit_to_ethereum::ADDRESS.raw(),
            topics: topics.into_iter().map(|h| H256::from(h.0)).collect(),
            data,
        };

        Ok(PrecompileOutput {
            logs: vec![promise_log, exit_event_log],
            cost: required_gas,
            output: Vec::new(),
        })
    }

pub fn refund_on_error<I: IO + Copy, E: Env, P: PromiseHandler>(
    io: I,
    env: &E,
    state: EngineState,
    args: &RefundCallArgs,
    handler: &mut P,
) -> EngineResult<SubmitResult> {
    let current_account_id = env.current_account_id();
    if let Some(erc20_address) = args.erc20_address {
        // ERC-20 exit; re-mint burned tokens
        let erc20_admin_address = current_address(&current_account_id);
        let mut engine: Engine<_, _> =
            Engine::new_with_state(state, erc20_admin_address, current_account_id, io, env);

        let refund_address = args.recipient_address;
        let amount = U256::from_big_endian(&args.amount);
        let input = setup_refund_on_error_input(amount, refund_address);

        engine.call(
            &erc20_admin_address,
            &erc20_address,
            Wei::zero(),
            input,
            u64::MAX,
            Vec::new(),
            Vec::new(),
            handler,
        )
    } else {
        // ETH exit; transfer ETH back from precompile address
        let exit_address = exit_to_near::ADDRESS;
        let mut engine: Engine<_, _> =
            Engine::new_with_state(state, exit_address, current_account_id, io, env);
        let refund_address = args.recipient_address;
        let amount = Wei::new(U256::from_big_endian(&args.amount));
        engine.call(
            &exit_address,
            &refund_address,
            amount,
            Vec::new(),
            u64::MAX,
            vec![
                (exit_address.raw(), Vec::new()),
                (refund_address.raw(), Vec::new()),
            ],
            Vec::new(),
            handler,
        )
    }
