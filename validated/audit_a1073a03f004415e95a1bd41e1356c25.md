### Title
Silo Owner Can Front-Run User Transactions to Inflate `fixed_gas` and Drain Sender ETH Balances - (`engine/src/lib.rs`, `engine/src/engine.rs`)

---

### Summary

In Aurora Engine's Silo Mode, the contract owner can call `set_fixed_gas` (or `set_silo_params`) at any time with no time-lock or delay. A malicious Silo owner can observe a user's pending `submit` transaction in the NEAR mempool, front-run it by inflating the `fixed_gas` value, and cause the user to pay far more in gas fees than they signed for — up to the full `gas_limit * gas_price` of the transaction. The excess fee is transferred to the relayer rather than refunded to the sender.

---

### Finding Description

**Step 1 — Privileged, undelayed fee setter:**

The `set_fixed_gas` entry point in `engine/src/lib.rs` is guarded only by `require_owner_and_running`, which checks that the caller is the stored `owner_id`. There is no time-lock, no minimum delay, and no upper-bound validation on the new value:

```rust
pub extern "C" fn set_fixed_gas() {
    let mut io = Runtime;
    let state = state::get_state(&io).sdk_unwrap();
    require_owner_and_running(&state, &io.predecessor_account_id())
        .map_err(ContractError::msg)
        .sdk_unwrap();

    let args: FixedGasArgs = io.read_input_borsh().sdk_unwrap();
    silo::set_fixed_gas(&mut io, args.fixed_gas);
}
``` [1](#0-0) 

This contrasts with the upgrade mechanism, which enforces `upgrade_delay_blocks` before code changes take effect. No equivalent delay exists for fee changes. [2](#0-1) 

**Step 2 — `fixed_gas` is read fresh at execution time:**

Inside `submit_with_alt_modexp` (called by both `submit` and `submit_with_args`), the `fixed_gas` value is read from storage at the moment the transaction executes — not at signing time:

```rust
let fixed_gas = silo::get_fixed_gas(&io);
``` [3](#0-2) 

**Step 3 — `charge_gas` uses `fixed_gas` to compute the full prepaid amount:**

```rust
let prepaid_amount = fixed_gas
    .map_or(transaction.gas_limit, EthGas::as_u256)
    .checked_mul(effective_gas_price)
    .map(Wei::new)
    .ok_or(GasPaymentError::EthAmountOverflow)?;

let new_balance = get_balance(&self.io, sender)
    .checked_sub(prepaid_amount)
    .ok_or(GasPaymentError::OutOfFund)?;

set_balance(&mut self.io, sender, &new_balance);
``` [4](#0-3) 

**Step 4 — `refund_unused_gas` also uses `fixed_gas`, so no refund is given for the inflated portion:**

When `fixed_gas` is set, `refund_unused_gas` computes the "spent amount" as `fixed_gas * effective_gas_price` (not `gas_used * effective_gas_price`). The excess over actual gas used is paid to the relayer, not refunded to the sender:

```rust
let gas_to_wei = |price: U256| {
    fixed_gas
        .map_or_else(|| gas_used.into(), EthGas::as_u256)
        .checked_mul(price)
        .map(Wei::new)
        .ok_or(GasPaymentError::EthAmountOverflow)
};
``` [5](#0-4) 

**Step 5 — The only guard against an unbounded `fixed_gas` is the transaction's own `gas_limit`:**

```rust
if fixed_gas.is_some_and(|gas| gas.as_u256() > transaction.gas_limit) {
    return Err(EngineErrorKind::FixedGasOverflow.into());
}
``` [6](#0-5) 

This means the owner can set `fixed_gas` to any value up to the user's `gas_limit`, extracting the maximum possible fee from the user.

---

### Impact Explanation

**Impact: High — Theft of user funds (ETH balances held inside the Aurora EVM).**

A user who signs a transaction expecting to pay `original_fixed_gas * gas_price` (e.g., `5,000 * 1 = 5,000 wei`) can instead be charged up to `gas_limit * gas_price` (e.g., `21,000 * 1 = 21,000 wei`). The difference (`16,000 wei`) is transferred to the relayer address, which the Silo owner controls. Repeated across many users or transactions, this constitutes systematic theft of ETH balances.

The `set_silo_params` entry point has the same issue, as it calls `set_fixed_gas` internally: [7](#0-6) 

---

### Likelihood Explanation

**Likelihood: Medium.**

The attack requires the Silo owner to act maliciously. In a Silo deployment, the owner is the Silo operator — a separate entity from the users who submit transactions. Users of a Silo are whitelisted participants who may not have full visibility into or control over the operator's behavior. NEAR's mempool ordering means a NEAR account (the owner) can submit a transaction that executes before a user's pending `submit` call. No special tooling is required beyond a standard NEAR RPC call to `set_fixed_gas`.

---

### Recommendation

1. **Add an upper bound** on `fixed_gas` (e.g., a protocol-defined maximum) to limit the worst-case fee extraction.
2. **Introduce a time-lock** (analogous to `upgrade_delay_blocks`) before a `fixed_gas` increase takes effect, giving users time to observe and react to fee changes.
3. **Emit an event** when `fixed_gas` is changed so users can monitor for unexpected fee increases.
4. **Consider making `fixed_gas` increases subject to a minimum notice period**, while allowing decreases to take effect immediately (decreases cannot harm users).

---

### Proof of Concept

1. Silo is deployed with `fixed_gas = 5_000` and `gas_price = 1`. Users expect to pay `5,000 wei` per transaction.
2. Alice signs and broadcasts a `submit` transaction with `gas_limit = 21_000` and `gas_price = 1`.
3. The malicious Silo owner observes Alice's pending transaction in the NEAR mempool.
4. Owner calls `set_fixed_gas(FixedGasArgs { fixed_gas: Some(EthGas::new(21_000)) })` — this executes before Alice's transaction.
5. Alice's `submit` executes: `charge_gas` computes `prepaid_amount = 21_000 * 1 = 21_000 wei` and deducts it from Alice's balance.
6. `refund_unused_gas` computes `spent_amount = 21_000 * 1 = 21_000 wei` (using `fixed_gas`, not `gas_used`). Refund to Alice = `21_000 - 21_000 = 0`. Relayer reward = `21_000 wei`.
7. Alice paid `21,000 wei` instead of the expected `5,000 wei`. The extra `16,000 wei` went to the relayer controlled by the owner.
8. Owner calls `set_fixed_gas` back to `5_000` to avoid detection.

The test `test_switch_between_fix_gas` in `engine-tests/src/tests/silo.rs` confirms that changing `fixed_gas` mid-session immediately affects subsequent transactions' fee deductions: [8](#0-7)

fn test_switch_between_fix_gas() {
    const TRANSFER: Wei = Wei::new_u64(10_000_000);
    let (mut runner, mut signer, receiver) = initialize_transfer();
    let sender = utils::address_from_secret_key(&signer.secret_key);

    // validate pre-state
    validate_address_balance_and_nonce(&runner, sender, INITIAL_BALANCE, INITIAL_NONCE.into())
        .unwrap();
    validate_address_balance_and_nonce(&runner, receiver, ZERO_BALANCE, INITIAL_NONCE.into())
        .unwrap();

    // Defining gas cost in transaction
    // do transfer
    let result = runner
        .submit_with_signer(&mut signer, |nonce| {
            let mut tx = utils::transfer(receiver, TRANSFER, nonce);
            tx.gas_limit = 30_0000.into();
            tx.gas_price = 1.into();
            tx
        })
        .unwrap();

    // validate post-state
    validate_address_balance_and_nonce(
        &runner,
        sender,
        INITIAL_BALANCE - TRANSFER - EthGas::new(result.gas_used) * ONE_GAS_PRICE,
        (INITIAL_NONCE + 1).into(),
    )
    .unwrap();
    validate_address_balance_and_nonce(&runner, receiver, TRANSFER, 0.into()).unwrap();

    // Set fixed gas
    let fixed_gas = EthGas::new(1_000_000);
    set_silo_params(
        &mut runner,
        Some(SiloParamsArgs {
            fixed_gas,
            erc20_fallback_address: ERC20_FALLBACK_ADDRESS,
        }),
    );
    // Check that fixed gas cost has been set successfully.
    assert_eq!(runner.get_fixed_gas(), Some(fixed_gas));

    let balance_before_transfer = runner.get_balance(sender);
    let result = runner
        .submit_with_signer(&mut signer, |nonce| {
            utils::transfer_with_price(receiver, TRANSFER, nonce, TWO_GAS_PRICE.raw())
        })
        .unwrap();
    assert!(matches!(result.status, TransactionStatus::Succeed(_)));

    let sender_balance = balance_before_transfer - TRANSFER - fixed_gas * TWO_GAS_PRICE;
    let receiver_balance = TRANSFER + TRANSFER;

    // validate post-state
    validate_address_balance_and_nonce(&runner, sender, sender_balance, (INITIAL_NONCE + 2).into())
        .unwrap();
    validate_address_balance_and_nonce(&runner, receiver, receiver_balance, INITIAL_NONCE.into())
        .unwrap();

    // Unset fixed gas cost. Should be used usual gas charge mechanism.
    set_silo_params(&mut runner, None);
    assert_eq!(runner.get_fixed_gas(), None);
    let balance_before_transfer = runner.get_balance(sender);

    // do transfer
    let result = runner
        .submit_with_signer(&mut signer, |nonce| {
            let mut tx = utils::transfer(receiver, TRANSFER, nonce);
            tx.gas_limit = 30_0000.into();
            tx.gas_price = 1.into();
            tx
        })
        .unwrap();

    let sender_balance = balance_before_transfer - TRANSFER - Wei::new_u64(result.gas_used);
    let receiver_balance = TRANSFER + TRANSFER + TRANSFER;

    // validate post-state
    validate_address_balance_and_nonce(&runner, sender, sender_balance, (INITIAL_NONCE + 3).into())
        .unwrap();
    validate_address_balance_and_nonce(&runner, receiver, receiver_balance, INITIAL_NONCE.into())
        .unwrap();
}
