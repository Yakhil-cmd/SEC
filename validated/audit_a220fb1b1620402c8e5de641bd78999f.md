### Title
SNS `ManageLedgerParameters` Transfer Fee Update Does Not Propagate to `neuron_minimum_stake_e8s`, Breaking Governance Invariant - (File: rs/sns/governance/src/governance.rs)

---

### Summary

When an SNS `ManageLedgerParameters` proposal raises the ledger's `transfer_fee` to a value ≥ `neuron_minimum_stake_e8s`, the governance correctly mirrors the new fee into `NervousSystemParameters.transaction_fee_e8s` but never adjusts `neuron_minimum_stake_e8s`. This silently violates the required invariant `neuron_minimum_stake_e8s > transaction_fee_e8s`, after which every subsequent `ManageNervousSystemParameters` proposal fails its own validation check and the SNS governance is unable to update any nervous-system parameter.

---

### Finding Description

`perform_manage_ledger_parameters()` in `rs/sns/governance/src/governance.rs` (lines 3190–3195) propagates the new fee into governance state:

```rust
// update nervous-system-parameters transaction_fee if the fee is changed.
if let Some(nervous_system_parameters) = self.proto.parameters.as_mut()
    && let Some(transfer_fee) = manage_ledger_parameters.transfer_fee
{
    nervous_system_parameters.transaction_fee_e8s = Some(transfer_fee);
}
``` [1](#0-0) 

It does **not** touch `neuron_minimum_stake_e8s`. The `NervousSystemParameters` type enforces a hard invariant in `validate_neuron_minimum_stake_e8s()`:

```rust
if neuron_minimum_stake_e8s <= transaction_fee_e8s {
    Err(...)
}
``` [2](#0-1) 

The proposal-submission validator `validate_and_render_manage_ledger_parameters()` checks only name/symbol/logo format and that at least one field is set; it performs **no** comparison of the proposed fee against `neuron_minimum_stake_e8s`:

```rust
if let Some(transfer_fee) = transfer_fee {
    render += &format!("# Set token transfer fee: {transfer_fee} token-quantums. \n",);
    change = true;
}
``` [3](#0-2) 

The `ManageLedgerParameters` struct itself only carries `transfer_fee`, `token_name`, `token_symbol`, and `token_logo`; there is no field for `neuron_minimum_stake_e8s` and no cross-field constraint is enforced anywhere in the execution path: [4](#0-3) 

The `perform_manage_nervous_system_parameters()` path that would later be used to fix the broken state calls `new_params.validate()`, which re-runs `validate_neuron_minimum_stake_e8s()` and will reject any proposal until the invariant is restored: [5](#0-4) 

---

### Impact Explanation

After a `ManageLedgerParameters` proposal executes with `transfer_fee ≥ neuron_minimum_stake_e8s`:

1. The ICRC-1 ledger's fee is correctly updated.
2. `NervousSystemParameters.transaction_fee_e8s` is updated to the new value.
3. `NervousSystemParameters.neuron_minimum_stake_e8s` is **not** updated.
4. The invariant `neuron_minimum_stake_e8s > transaction_fee_e8s` is permanently violated in the governance canister's persisted state.
5. Every future `ManageNervousSystemParameters` proposal fails at execution time with a validation error, because `validate()` checks the invariant against the current (now broken) parameters.
6. The SNS governance is effectively locked from updating any nervous-system parameter until a manual canister upgrade is performed by the SNS root.

Additionally, governance operations that read `transaction_fee_e8s_or_panic()` (e.g., `split_neuron`) now operate with a fee value that exceeds the minimum stake, making it impossible to split neurons into valid child neurons: [6](#0-5) 

---

### Likelihood Explanation

Any SNS community can submit a `ManageLedgerParameters` proposal. The proposal UI renders only the human-readable fee change; there is no warning that the new fee must remain below `neuron_minimum_stake_e8s`. A community that wants to raise fees to combat spam (a common governance action) can inadvertently trigger this with no indication that the invariant will be broken. The integration test `test_manage_ledger_parameters_change_transfer_fee` confirms the fee-change path is exercised but does not test the cross-parameter constraint: [7](#0-6) 

---

### Recommendation

1. **Add cross-parameter validation at proposal submission time** in `validate_and_render_manage_ledger_parameters()`: if `transfer_fee` is set, reject the proposal if `transfer_fee >= current neuron_minimum_stake_e8s`.
2. **Propagate the constraint at execution time**: in `perform_manage_ledger_parameters()`, after updating `transaction_fee_e8s`, also raise `neuron_minimum_stake_e8s` to at least `transfer_fee + 1` (or reject the upgrade if that would violate other constraints).
3. **Add a post-execution invariant check**: call `self.proto.parameters.as_ref().unwrap().validate()` after updating `transaction_fee_e8s` and trap/return an error if the resulting state is invalid, preventing the governance canister from persisting a broken state.

---

### Proof of Concept

1. SNS is initialized with `transaction_fee_e8s = 10_000` and `neuron_minimum_stake_e8s = 100_000` (the default relationship).
2. A neuron holder submits a `ManageLedgerParameters` proposal with `transfer_fee = 200_000`.
3. The proposal passes the governance vote and `perform_manage_ledger_parameters()` executes.
4. The ledger's fee is set to `200_000`; `transaction_fee_e8s` in governance is set to `200_000`; `neuron_minimum_stake_e8s` remains `100_000`.
5. The invariant `100_000 > 200_000` is now false.
6. A subsequent `ManageNervousSystemParameters` proposal (e.g., to fix `neuron_minimum_stake_e8s`) is submitted. At execution, `perform_manage_nervous_system_parameters()` calls `new_params.validate()`, which calls `validate_neuron_minimum_stake_e8s()`. The check `neuron_minimum_stake_e8s (100_000) <= transaction_fee_e8s (200_000)` is true, so validation fails and the proposal is marked as failed.
7. The SNS governance is now unable to update any nervous-system parameter via the standard proposal path. [8](#0-7) [9](#0-8)
