### Title
NNS Governance `update_node_provider` Hardcodes `caller()` as Node Provider Identity, Blocking Intermediary Canister Calls - (File: rs/nns/governance/canister/canister.rs)

### Summary
The `update_node_provider` endpoint in NNS Governance unconditionally uses `caller()` as the node provider principal ID. The `UpdateNodeProvider` request struct contains no field for specifying the node provider principal — the caller *is* the node provider by design. A node provider whose principal is a canister (e.g., a DAO, multisig, or vesting-style canister) that must route the call through an intermediary canister will have the intermediary's principal used as the lookup key, causing the call to fail with "Node Provider X is not known by the NNS," even though the actual node provider is registered.
