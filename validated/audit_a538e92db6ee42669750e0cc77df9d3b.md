### Title
`ExitToEthereum` ERC-20 Withdrawal Path Hardcodes JSON Serialization, Ignoring `WithdrawSerializeType` Configuration — (File: `engine-precompiles/src/native.rs`)

### Summary
The `ExitToEthereum` precompile's ERC-20 exit path (flag `0x1`) always serializes `withdraw` call arguments as JSON, ignoring the `WithdrawSerializeType` configuration that governs which serialization format the eth-connector expects. The ETH base token path (flag `0x0`) in the same precompile correctly reads this setting and dispatches to either JSON or Borsh. Because the default `WithdrawSerializeType` is `Borsh`, any deployment using the default will cause ERC-20 exit-to-Ethereum calls to send JSON-encoded arguments to a connector expecting Borsh, making the `withdraw` promise fail. Since ERC-20 tokens are burned before the precompile call and no refund path exists for this code branch, the result is permanent loss of user funds.

### Finding Description
In `engine-precompiles/src/native.rs`, the `ExitToEthereum` precompile handles two withdrawal cases.

**ETH base token path (flag `0x0`)** — correctly reads `WithdrawSerializeType` from storage and dispatches: [1](#0-0) 

```rust
let serialize_fn = match get_withdraw_serialize_type(&self.io)? {
    WithdrawSerializeType::Json => json_args,
    WithdrawSerializeType::Borsh => borsh_args,
};
```

**ERC-20 path (flag `0x1`)** — always uses hardcoded JSON, never reads `WithdrawSerializeType`: [2](#0-1) 

```rust
format!(
    r#"{{"amount": "{}", "recipient": "{}"}}"#,
    amount.as_u128(),
    recipient_in_hex
)
```

The `WithdrawSerializeType` enum (`Json` | `Borsh`) was introduced precisely to support different eth-connector implementations. [3](#0-2) 

The `get_withdraw_serialize_type` function returns `Borsh` as the default when the storage key is absent: [4](#0-3) 

The ETH base token path is consistent with this default; the ERC-20 path is not.

The promise that is ultimately created calls `withdraw` on the NEP-141 contract with the JSON-encoded bytes: [5](#0-4) 

### Impact Explanation
The entry point is `EvmErc20.withdrawToEthereum`, which is callable by any token holder: [6](#0-5) 

The Solidity contract burns the caller's ERC-20 tokens (`_burn`) **before** invoking the precompile. The precompile then schedules a NEAR promise to call `withdraw` on the NEP-141 contract with JSON-encoded arguments. When the NEP-141 contract (or eth-connector) is configured for Borsh (the default), it rejects the JSON payload and the promise fails. The EVM transaction has already committed the burn; there is no `error_refund` callback registered for the `ExitToEthereum` ERC-20 branch (unlike `ExitToNear` which has an optional refund feature). The burned tokens are permanently unrecoverable.

**Impact: Critical — Permanent freezing/loss of user funds.**

### Likelihood Explanation
`get_withdraw_serialize_type` returns `Borsh` when the `WithdrawSerializationType` storage key is absent, making Borsh the out-of-the-box default for any fresh deployment. A user holding any bridged ERC-20 token who calls `withdrawToEthereum` on a deployment using the default configuration will trigger this path and permanently lose their tokens. No privileged access or special conditions are required beyond holding ERC-20 tokens.

**Likelihood: High** — the default configuration is the vulnerable one, and the call is a standard user-facing operation.

### Recommendation
Apply the same `WithdrawSerializeType` dispatch to the ERC-20 branch of `ExitToEthereum` that already exists in the ETH base token branch. Read `get_withdraw_serialize_type` and select between a JSON formatter and a Borsh formatter for the `{"amount": ..., "recipient": ...}` payload before constructing the `withdraw` promise.

fn get_withdraw_serialize_type<I: IO>(io: &I) -> Result<WithdrawSerializeType, ExitError> {
    io.read_storage(&construct_contract_key(
        EthConnectorStorageId::WithdrawSerializationType,
    ))
    .map_or(Ok(WithdrawSerializeType::Borsh), |value| {
        value
            .to_value()
            .map_err(|_| ExitError::Other(Cow::Borrowed("ERR_DESERIALIZE")))
    })
}
