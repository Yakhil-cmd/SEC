Audit Report

## Title
SNS Developer Neuron Vesting Bypass via Zero `created_timestamp_seconds` — (File: `rs/sns/init/src/distributions.rs`)

## Summary
The `create_neuron` helper in `rs/sns/init/src/distributions.rs` constructs genesis developer neurons using `..Default::default()`, leaving `created_timestamp_seconds` at its protobuf/Rust default of `0`. The `is_vesting` predicate in `rs/sns/governance/src/neuron.rs` computes `created_timestamp_seconds + vesting_period_seconds >= now`, which evaluates to `false` against any real-world Unix timestamp, causing every developer neuron with a `vesting_period_seconds` to be treated as non-vesting from the moment the SNS launches. The vesting enforcement gate in `rs/sns/governance/src/governance.rs` is therefore never entered, and `StartDissolving`, `Disburse`, and `Split` are all immediately permitted.

## Finding Description
**Root cause — `create_neuron` (`rs/sns/init/src/distributions.rs`, L151–162):**
The neuron struct is constructed with `..Default::default()`. No field assignment for `created_timestamp_seconds` appears anywhere in `rs/sns/init/src/` (confirmed by grep). The field silently inherits `0`.

**Vesting predicate — `is_vesting` (`rs/sns/governance/src/neuron.rs`, L796–801):**
```rust
self.created_timestamp_seconds + vesting_period_seconds >= now
```
With `created_timestamp_seconds = 0` and `vesting_period_seconds = 3 * ONE_YEAR_SECONDS ≈ 94_608_000`, the expression is `94_608_000 >= ~1_720_000_000` → `false`. The neuron is never considered vesting.

**Enforcement gate — `check_command_is_valid_if_neuron_is_vesting` (`rs/sns/governance/src/governance.rs`, L4862–4863):**
```rust
if !neuron.is_vesting(self.env.now()) {
    return Ok(());
}
```
Because `is_vesting` returns `false`, the gate exits immediately and all restricted operations (`StartDissolving`, `Disburse`, `Split`) are permitted without restriction.

**Initialization call chain — `get_initial_neurons` (`rs/sns/init/src/distributions.rs`, L51–81):**
Neurons returned by `create_neuron` are inserted directly into the initial neuron map with no post-processing step that sets `created_timestamp_seconds`. There are zero references to `created_timestamp_seconds` anywhere in `rs/sns/init/src/`.

## Impact Explanation
A developer neuron controller — an unprivileged ingress caller — can immediately call `manage_neuron → Configure → StartDissolving` on their genesis neuron, wait out the dissolve delay, then call `Disburse` to extract all staked SNS tokens. The intended vesting lock (e.g., 3 years) is completely bypassed from SNS launch. This constitutes a significant SNS governance and financial security impact: investors who participated in the decentralization swap based on the published developer vesting schedule face immediate token dumping risk, with concrete and direct harm to SNS token holders. This matches the **High** impact category: *Significant SNS security impact with concrete user or protocol harm*.

## Likelihood Explanation
Every SNS launched via the one-proposal flow that includes developer neurons with a non-zero `vesting_period_seconds` is affected. The exploit requires no special privileges, no governance majority, no key material, and no victim interaction. The developer neuron controller simply calls the public `manage_neuron` endpoint. The bypass is automatic and silent — no error is raised and no on-chain signal distinguishes the bypassed state from a legitimately non-vesting neuron.

## Recommendation
In `create_neuron` (`rs/sns/init/src/distributions.rs`, L151–162), explicitly set `created_timestamp_seconds` to the SNS initialization timestamp. Since `create_neuron` does not currently receive a timestamp parameter, either pass the current time from the caller (`get_initial_neurons`) or accept a `now: u64` argument. Additionally, add a validation guard in the SNS governance initialization path that rejects any neuron whose `vesting_period_seconds` is `Some(_)` but whose `created_timestamp_seconds` is `0`.

## Proof of Concept
1. Submit a `CreateServiceNervousSystem` NNS proposal with a developer neuron configured with `vesting_period_seconds: Some(3 * ONE_YEAR_SECONDS)`.
2. After SNS launch, inspect the stored neuron: `created_timestamp_seconds == 0` (default from `..Default::default()` in `create_neuron`).
3. As the developer neuron controller, call `manage_neuron → Configure → StartDissolving`.
4. `check_command_is_valid_if_neuron_is_vesting` calls `is_vesting(now)`: `0 + 94_608_000 >= 1_720_000_000` → `false` → returns `Ok(())`.
5. The neuron begins dissolving. After the dissolve delay elapses, call `Disburse` to extract all staked tokens. The 3-year vesting lock is fully bypassed.
6. A deterministic integration test or PocketIC test can reproduce this by creating an SNS with a developer neuron having `vesting_period_seconds = Some(X)`, asserting `created_timestamp_seconds == 0` on the stored neuron, and confirming that `StartDissolving` succeeds immediately.
