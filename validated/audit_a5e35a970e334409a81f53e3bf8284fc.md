### Title
Stale In-Memory ACL Allows Decommissioned Boundary Node to Retrieve Salt During Registry Polling Window — (`rs/boundary_node/salt_sharing/canister/helpers.rs`, `canister.rs`)

---

### Summary

The salt-sharing canister maintains its authorized-principal set (`API_BOUNDARY_NODE_PRINCIPALS`) exclusively in heap memory, refreshed only by a periodic timer-driven inter-canister call to the registry. Both the `inspect_message` gate and the `get_salt` body read this same in-memory set. A boundary node whose principal has been removed from the registry retains full salt access until the next `poll_api_boundary_nodes` execution, a window of up to `registry_polling_interval_secs` seconds.

---

### Finding Description

`API_BOUNDARY_NODE_PRINCIPALS` is a `thread_local! RefCell<HashSet<Principal>>` stored in heap memory only: [1](#0-0) 

It is populated exclusively by `poll_api_boundary_nodes`, which is scheduled as a repeating timer with period `registry_polling_interval_secs`: [2](#0-1) 

The full replacement of the set happens only on a successful registry response: [3](#0-2) 

`is_api_boundary_node_principal` reads directly from this set with no freshness check: [4](#0-3) 

Both the pre-consensus `inspect_message` gate and the `get_salt` body call this same function: [5](#0-4) [6](#0-5) 

The `salt_fetcher` on every boundary node calls `get_salt` as an **update** (ingress) call, not a query: [7](#0-6) 

This means the call passes through `inspect_message`. Because both gates consult the same stale set, a decommissioned node whose principal has not yet been evicted from `API_BOUNDARY_NODE_PRINCIPALS` will pass both checks and receive the salt.

**Concrete call sequence:**

```
registry removes node principal
  → API_BOUNDARY_NODE_PRINCIPALS still contains it (timer hasn't fired)
  → decommissioned node sends update call to get_salt
  → inspect_message: is_api_boundary_node_principal → true → accept_message()
  → get_salt body: is_api_boundary_node_principal → true → salt returned
```

There is no out-of-band revocation path; the only mechanism to evict a principal is the next successful `poll_api_boundary_nodes` execution.

---

### Impact Explanation

The salt is the anonymization key used by boundary nodes to hash sensitive user data (e.g., IP addresses in logs and metrics). A decommissioned or compromised boundary node retaining this salt can continue to de-anonymize user traffic for up to `registry_polling_interval_secs` seconds after its removal. In the integration test the interval is 60 s; in the system test it is 5 s; the production value is operator-configured. [8](#0-7) 

---

### Likelihood Explanation

The precondition is that the attacker controls the private key of a principal that was previously a legitimate API boundary node. This is realistic in a node-compromise scenario: the node is discovered to be compromised, removed from the registry, but the attacker continues to use the node's identity during the polling window. The window is bounded but non-zero and deterministic.

---

### Recommendation

1. **Immediate revocation via canister upgrade or admin method**: Add a privileged `remove_api_boundary_node` update method that atomically removes a principal from `API_BOUNDARY_NODE_PRINCIPALS` without waiting for the next poll.
2. **Reduce polling interval**: Lower `registry_polling_interval_secs` to minimize the exposure window.
3. **Salt rotation on decommission**: Trigger an immediate salt rotation when the authorized set shrinks, so any salt obtained during the stale window becomes useless.
4. **Persist the authorized set in stable memory**: Ensure the set survives upgrades and is not reset to empty on canister reinstall, which currently also creates a brief open-access window.

---

```rust
pub async fn poll_api_boundary_nodes() {
    let canister_id = Principal::from(REGISTRY_CANISTER_ID);

    let (call_status, message) = match call::<_, (Result<Vec<ApiBoundaryNodeIdRecord>, String>,)>(
        canister_id,
        REGISTRY_CANISTER_METHOD,
        (&GetApiBoundaryNodeIdsRequest {},),
    )
    .await
    {
        Ok((Ok(api_bn_records),)) => {
            // Set authorized readers of salt.
            let principals: HashSet<_> = api_bn_records.into_iter().filter_map(|n| n.id).collect();
            API_BOUNDARY_NODE_PRINCIPALS.with(|cell| *cell.borrow_mut() = principals);
```
