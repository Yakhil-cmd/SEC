### Title
SNS `ManageNervousSystemParameters` Allows Setting `neuron_minimum_dissolve_delay_to_vote_seconds` to Zero, Enabling Governance Takeover - (File: rs/sns/governance/src/types.rs)

### Summary
The SNS governance canister's `NervousSystemParameters` validation imposes no lower bound on `neuron_minimum_dissolve_delay_to_vote_seconds`. A `ManageNervousSystemParameters` proposal can set this value to zero, permanently removing the time-lock requirement for voting. Once enacted, any party that can transiently acquire a majority of SNS tokens can immediately stake, vote on any subsequent proposal (including `MintSnsTokens`, `TransferSnsTreasuryFunds`, or `UpgradeSnsControlledCanister`), and dissolve the neuron — a flash-loan-style governance takeover with no lock-up cost.

### Finding Description

`NervousSystemParameters::validate_neuron_minimum_dissolve_delay_to_vote_seconds` in `rs/sns/governance/src/types.rs` only checks that the value does not exceed `max_dissolve_delay_seconds`:

```rust
fn validate_neuron_minimum_dissolve_delay_to_vote_seconds(&self) -> Result<(), String> {
    let max_dissolve_delay_seconds = self.validate_max_dissolve_delay_seconds()?;
    let neuron_minimum_dissolve_delay_to_vote_seconds = self
        .neuron_minimum_dissolve_delay_to_vote_seconds
        .ok_or_else(|| { ... })?;

    if neuron_minimum_dissolve_delay_to_vote_seconds > max_dissolve_delay_seconds {
        Err(...)
    } else {
        Ok(())   // ← zero is accepted
    }
}
``` [1](#0-0) 

There is no floor. The NNS governance, by contrast, enforces a hard range:

```rust
pub const NEURON_MINIMUM_DISSOLVE_DELAY_TO_VOTE_SECONDS_BOUNDS: RangeInclusive<u64> =
    (14 * ONE_DAY_SECONDS)..=(6 * ONE_MONTH_SECONDS);
``` [2](#0-1) 

and validates against it:

```rust
if !VotingPowerEconomics::NEURON_MINIMUM_DISSOLVE_DELAY_TO_VOTE_SECONDS_BOUNDS
    .contains(&delay)
{
    defects.push(format!("neuron_minimum_dissolve_delay_to_vote_seconds ({:?}) must be between two weeks and six months.", ...));
}
``` [3](#0-2) 

The SNS has no equivalent guard. A `ManageNervousSystemParameters` proposal with `neuron_minimum_dissolve_delay_to_vote_seconds: Some(0)` passes `validate_and_render_manage_nervous_system_parameters` and, if adopted, is applied by `perform_manage_nervous_system_parameters`:

```rust
fn perform_manage_nervous_system_parameters(
    &mut self,
    proposed_params: NervousSystemParameters,
) -> Result<(), GovernanceError> {
    let new_params = proposed_params.inherit_from(self.nervous_system_parameters_or_panic());
    match new_params.validate() {
        Ok(()) => {
            self.proto.parameters = Some(new_params);
            Ok(())
        }
        ...
    }
}
``` [4](#0-3) 

After execution, the SNS governance permanently accepts neurons with zero dissolve delay as eligible voters.

Similarly, `validate_reject_cost_e8s` imposes no minimum — it only checks the field is `Some`:

```rust
fn validate_reject_cost_e8s(&self) -> Result<u64, String> {
    self.reject_cost_e8s
        .ok_or_else(|| "NervousSystemParameters.reject_cost_e8s must be set".to_string())
}
``` [5](#0-4) 

Setting `reject_cost_e8s = 0` simultaneously removes the economic spam barrier for proposal submission.

### Impact Explanation

After a single adopted `ManageNervousSystemParameters` proposal sets `neuron_minimum_dissolve_delay_to_vote_seconds = 0`:

1. Any party that transiently acquires a majority of SNS tokens can stake them in a neuron with zero dissolve delay, vote to adopt any subsequent proposal, and immediately dissolve the neuron to recover the tokens.
2. Exploitable follow-on proposals include:
   - `MintSnsTokens` — mint arbitrary SNS tokens to an attacker-controlled account. [6](#0-5) 
   - `TransferSnsTreasuryFunds` — drain ICP or SNS token treasury.
   - `UpgradeSnsControlledCanister` — install arbitrary WASM in any dapp canister registered with SNS Root. [7](#0-6) 
   - `AddGenericNervousSystemFunction` — register a new generic function targeting any non-reserved canister. [8](#0-7) 

The state change is permanent and irreversible without a subsequent governance proposal, which itself requires the same weakened governance to pass.

### Likelihood Explanation

The triggering proposal (`ManageNervousSystemParameters` with `neuron_minimum_dissolve_delay_to_vote_seconds = 0`) requires a non-malicious governance majority — token holders who collectively believe that removing the dissolve-delay requirement increases participation or accessibility. This is a realistic bad collective decision: the parameter name does not obviously signal the flash-loan attack surface, and the SNS documentation does not warn about it. The NNS explicitly bounds this value and documents the rationale; the SNS does not. Likelihood is **Low** but non-negligible, consistent with the class of unintended-consequence governance risks described in the reference report.

### Recommendation

Add a floor to `validate_neuron_minimum_dissolve_delay_to_vote_seconds` in `rs/sns/governance/src/types.rs`, mirroring the NNS approach:

```rust
const NEURON_MINIMUM_DISSOLVE_DELAY_TO_VOTE_SECONDS_FLOOR: u64 = ONE_DAY_SECONDS; // or higher

if neuron_minimum_dissolve_delay_to_vote_seconds < Self::NEURON_MINIMUM_DISSOLVE_DELAY_TO_VOTE_SECONDS_FLOOR {
    return Err(format!(
        "neuron_minimum_dissolve_delay_to_vote_seconds must be at least {} seconds",
        Self::NEURON_MINIMUM_DISSOLVE_DELAY_TO_VOTE_SECONDS_FLOOR
    ));
}
```

Similarly, add a minimum floor to `validate_reject_cost_e8s` to preserve the economic spam barrier.

### Proof of Concept

1. Deploy an SNS with default parameters (`neuron_minimum_dissolve_delay_to_vote_seconds = 6 months`).
2. Acquire majority SNS voting power.
3. Submit and pass a `ManageNervousSystemParameters` proposal:
   ```
   ManageNervousSystemParameters {
       neuron_minimum_dissolve_delay_to_vote_seconds: Some(0),
       reject_cost_e8s: Some(0),
       ..Default::default()
   }
   ```
   This passes `validate_and_render_manage_nervous_system_parameters` because `validate_neuron_minimum_dissolve_delay_to_vote_seconds` only checks `value <= max_dissolve_delay_seconds`. [9](#0-8) 
4. After execution, `self.proto.parameters` is updated with the new values. [10](#0-9) 
5. Now acquire tokens transiently (e.g., via a large purchase or loan), stake in a neuron with 0 dissolve delay, vote to adopt a `MintSnsTokens` proposal minting tokens to an attacker-controlled account, dissolve the neuron immediately, and return the borrowed tokens. The SNS treasury or dapp canisters are now under attacker control.

    fn perform_manage_nervous_system_parameters(
        &mut self,
        proposed_params: NervousSystemParameters,
    ) -> Result<(), GovernanceError> {
        // Only set `self.proto.parameters` if "applying" the proposed params to the
        // current params results in valid params
        let new_params = proposed_params.inherit_from(self.nervous_system_parameters_or_panic());

        log!(
            INFO,
            "Setting Governance nervous system params to: {:?}",
            &new_params
        );

        match new_params.validate() {
            Ok(()) => {
                self.proto.parameters = Some(new_params);
                Ok(())
            }

            // Even though proposals are validated when they are first made, this is still
            // possible, because the inner value of a ManageNervousSystemParameters
            // proposal is only valid with respect to the current
            // nervous_system_parameters() at the time when the proposal was first
            // made. If nervous_system_parameters() changed (by another proposal) since
            // the current proposal was first made, the current proposal might have become
            // invalid. Basically, this might occur if there are conflicting (concurrent)
            // proposals, but we expect this to be highly unusual in practice.
            Err(msg) => Err(GovernanceError::new_with_message(
                ErrorType::PreconditionFailed,
                format!(
                    "Failed to perform ManageNervousSystemParameters action, proposed \
                        parameters would lead to invalid NervousSystemParameters: {msg}"
                ),
            )),
        }

/// Validates and renders a proposal with action ManageNervousSystemParameters.
fn validate_and_render_manage_nervous_system_parameters(
    new_parameters: &NervousSystemParameters,
    current_parameters: &NervousSystemParameters,
) -> Result<String, String> {
    if new_parameters == &NervousSystemParameters::default() {
        return Err("NervousSystemParameters: at least one field must be set.".to_string());
    }

    new_parameters.inherit_from(current_parameters).validate()?;

    Ok(format!(
        r"# Proposal to change nervous system parameters:
## Current nervous system parameters:

{:#?}

## New nervous system parameters:

{:#?}",
        &current_parameters, new_parameters
    ))
}
