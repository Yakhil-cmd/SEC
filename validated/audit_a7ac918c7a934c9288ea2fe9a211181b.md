### Title
XCC Precompile NEAR-to-EVM Gas Conversion Truncates Toward Zero, Undercharging Callers - (File: `engine-precompiles/src/xcc.rs`)

### Summary

The `CrossContractCall` precompile converts attached NEAR gas into an EVM gas cost using integer division. Because Rust's `/` operator truncates toward zero, the computed EVM gas cost is always rounded down, meaning callers pay less than the true cost of the NEAR gas they consume. The protocol loses the fractional EVM gas unit on every XCC call, and the loss is accumulative.

### Finding Description

In `engine-precompiles/src/xcc.rs` line 174, after the promise is constructed with its total attached NEAR gas, the precompile converts that NEAR gas amount into an EVM gas surcharge:

```rust
cost += EthGas::new(promise.attached_gas.as_u64() / costs::CROSS_CONTRACT_CALL_NEAR_GAS);
```

`CROSS_CONTRACT_CALL_NEAR_GAS` is `175_000_000` (the NEAR-gas-per-EVM-gas conversion ratio). [1](#0-0) 

Rust's integer division truncates toward zero. Whenever `promise.attached_gas.as_u64()` is not an exact multiple of `175_000_000`, the remainder is silently discarded. The maximum discarded fraction per call is `174_999_999 / 175_000_000` EVM gas units — effectively 1 EVM gas unit lost per call. [2](#0-1) 

The `cost` value is returned inside `PrecompileOutput` and consumed by the EVM executor to deduct ETH from the caller. Undercharging `cost` by 1 EVM gas unit means the caller pays `1 * effective_gas_price` fewer Wei than they should. [3](#0-2) 

The `promise.attached_gas` for an Eager XCC call is `ROUTER_EXEC_BASE + callback_count * ROUTER_EXEC_PER_CALLBACK + user_call_gas`. A caller can craft `user_call_gas` so that the total is `N * 175_000_000 + 174_999_999` for any integer N, maximising the rounding loss on every call. [4](#0-3) 

The analogous correct formula (ceiling division, rounding in favour of the protocol) would be:

```rust
cost += EthGas::new(
    (promise.attached_gas.as_u64() + costs::CROSS_CONTRACT_CALL_NEAR_GAS - 1)
        / costs::CROSS_CONTRACT_CALL_NEAR_GAS
);
```

### Impact Explanation

Every XCC precompile invocation that carries a NEAR gas amount not divisible by `175_000_000` causes the protocol to collect up to 1 EVM gas unit less than it should. At a gas price of 1 Gwei this is 1 Gwei per call; at higher gas prices the per-call loss grows proportionally. Because the XCC precompile is a standard, publicly reachable interface, an attacker can issue a high volume of XCC calls with crafted gas values to maximise the accumulated shortfall. The loss is a form of unclaimed protocol fee revenue — **High: Theft of unclaimed yield**.

### Likelihood Explanation

The XCC precompile is reachable by any unprivileged EVM caller. No special role, key, or governance action is required. The rounding error is deterministic and reproducible on every call where the gas amount is not an exact multiple of the conversion constant. Likelihood is **High**.

### Recommendation

Replace the truncating division with ceiling division so that rounding always favours the protocol:

```rust
// ceiling division: (a + b - 1) / b
let near_gas = promise.attached_gas.as_u64();
let denom = costs::CROSS_CONTRACT_CALL_NEAR_GAS;
cost += EthGas::new((near_gas + denom - 1) / denom);
```

This mirrors the fix recommended in M-3 and ensures the protocol never under-collects gas fees.

### Proof of Concept

Suppose a caller submits an Eager XCC call whose total `attached_gas` resolves to `350_000_174` NEAR gas (i.e., `2 * 175_000_000 + 174`).

Current code:
```
350_000_174 / 175_000_000 = 2   (174 NEAR gas worth of EVM cost discarded)
```

Correct ceiling division:
```
(350_000_174 + 174_999_999) / 175_000_000 = 524_999_999 / 175_000_000 = 2  ... wait
```

More illustrative: `attached_gas = 175_000_001`:
```
Current:  175_000_001 / 175_000_000 = 1  EVM gas charged
Correct:  ceil(175_000_001 / 175_000_000) = 2  EVM gas charged
```

The caller pays for 1 EVM gas unit instead of 2, saving `1 * gas_price` Wei. Repeated across thousands of calls this accumulates into a meaningful protocol shortfall.
