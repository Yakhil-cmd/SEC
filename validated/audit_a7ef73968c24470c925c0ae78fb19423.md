### Title
Cross-Operator `num_removed_same_ip_same_type` Quota Bypass in `do_add_node_` — (`rs/registry/canister/src/mutations/node_management/do_add_node.rs`)

---

### Summary

`do_add_node_` computes `num_removed_same_ip_same_type` by scanning **all** nodes at the target IP regardless of ownership, but computes `num_in_registry_same_type` by filtering only the **caller's** (OP2's) nodes. When OP1 (a different operator sharing the same `dc_id` and `node_provider_principal_id` as OP2) has a node at the same IP with the same reward type, the counter is inflated by OP1's node, causing `num_in_registry_same_type.saturating_sub(num_removed_same_ip_same_type)` to undercount OP2's effective node count. The quota check then passes incorrectly, and `make_remove_or_replace_node_mutations` succeeds (because the same-DC/same-provider fallback allows OP2 to remove OP1's node), resulting in OP2 registering one node beyond their `max_rewardable_nodes` quota.

---

### Finding Description

**Step 1 — `num_removed_same_ip_same_type` is not scoped to the caller.** [1](#0-0) 

`scan_for_nodes_by_ip` returns every node at the target IP regardless of `node_operator_id`. The loop then increments `num_removed_same_ip_same_type` for any such node whose reward type matches — including nodes owned by OP1, not OP2. [2](#0-1) 

**Step 2 — The quota check uses a per-operator numerator but a global subtrahend.** [3](#0-2) 

`get_node_operator_nodes(self, caller_id)` is filtered to OP2's nodes only. But `num_removed_same_ip_same_type` was incremented for OP1's node. With OP2 at quota Q:

```
num_in_registry_same_type = Q        (OP2's nodes only)
num_removed_same_ip_same_type = 1    (OP1's node at same IP)
Q <= Q.saturating_sub(1)  →  Q <= Q-1  →  false  →  quota check PASSES
```

**Step 3 — `make_remove_or_replace_node_mutations` permits cross-operator removal when DC and node provider match.** [4](#0-3) 

When `caller_id (OP2) != node_operator_id (OP1)`, the fallback asserts that both operators share the same `dc_id` and `node_provider_principal_id`. If they do, the removal of OP1's node proceeds unconditionally. The code comment at line 81 says "This will only succeed if the same NO was in control of the original nodes" — this is factually incorrect; the same-DC/same-provider fallback is a second, broader path.

**Net result:** OP1's node is removed, OP2's new node is inserted, and OP2 ends up with Q+1 nodes of type T.

---

### Impact Explanation

A node provider controlling two operators (OP1 and OP2) in the same data center can cause OP2 to register exactly one node beyond its governance-approved `max_rewardable_nodes` quota for a given reward type. The bypass is bounded to +1 per attempt (a second attempt fails because `Q <= Q` is true), but the node provider can cycle OP1 nodes to repeat the bypass. Each cycle sacrifices one of OP1's nodes and gains one extra node for OP2, allowing gradual accumulation beyond quota.

---

### Likelihood Explanation

The preconditions are realistic and common in IC deployments: a single node provider operating multiple operator principals in the same data center is the standard multi-operator deployment pattern. The attack requires no admin keys, no governance majority, and no threshold corruption — only two legitimately registered operator principals under the same node provider.

---

### Recommendation

Scope `num_removed_same_ip_same_type` to only count nodes belonging to `caller_id`:

```rust
for node_with_same_ip in &nodes_with_same_ip {
    // Only count nodes owned by the caller
    let node_operator = get_node_operator_id_for_node(self, *node_with_same_ip)?;
    if node_operator != caller_id {
        continue;
    }
    let node_same_ip_reward_type = get_node_reward_type_for_node(self, *node_with_same_ip)?;
    if Some(node_same_ip_reward_type) == node_reward_type {
        num_removed_same_ip_same_type += 1;
    }
}
```

This makes the subtrahend consistent with the per-operator numerator used in the quota check.

### Proof of Concept

```
Setup:
  NP = node provider principal
  OP1: dc_id="dc1", node_provider_principal_id=NP, max_rewardable_nodes={type1: 1}
  OP2: dc_id="dc1", node_provider_principal_id=NP, max_rewardable_nodes={type1: 1}

Step 1: OP1 registers node N1 at IP=192.0.2.1, type=type1
        → registry: {N1: operator=OP1, ip=192.0.2.1, type=type1}

Step 2: OP2 registers node N2 at IP=192.0.2.2, type=type1
        → registry: {N1: OP1, N2: OP2}
        → OP2 is now at quota (1 node of type1)

Step 3: OP2 calls do_add_node(http_endpoint=192.0.2.1, type=type1)
        → scan_for_nodes_by_ip → [N1]
        → get_node_reward_type_for_node(N1) = type1 == type1 → num_removed_same_ip_same_type = 1
        → get_node_operator_nodes(OP2) filtered type1 → [N2] → num_in_registry_same_type = 1

            OP2 != OP1, but dc_caller==dc_orig and provider_caller==provider_orig → assert passes
            N1 is removed, new node N3 inserted at 192.0.2.1

Result: registry contains N2 (OP2, type1) and N3 (OP2, type1)
        OP2 has 2 nodes of type1, quota is 1 → quota bypassed by 1

pub fn scan_for_nodes_by_ip(registry: &Registry, ip_addr: &str) -> Vec<NodeId> {
    get_key_family::<NodeRecord>(registry, NODE_RECORD_KEY_PREFIX)
        .into_iter()
        .filter_map(|(k, v)| {
            v.http.and_then(|v| {
                (v.ip_addr == ip_addr).then(|| NodeId::from(PrincipalId::from_str(&k).unwrap()))
            })
        })
        .collect()
}

    pub fn make_remove_or_replace_node_mutations(
        &mut self,
        payload: RemoveNodeDirectlyPayload,
        caller_id: PrincipalId,
        new_node_id: Option<NodeId>,
    ) -> Vec<RegistryMutation> {
        // 1. Find the node operator id for this record
        // and abort if the node record is not found
        let node_operator_id = get_node_operator_id_for_node(self, payload.node_id)
            .map_err(|e| format!("{LOG_PREFIX}do_remove_node_directly: Aborting node removal: {e}"))
            .unwrap();

        // 2. Compare the caller_id (node operator) with the node's node operator and, if that fails,
        // fall back to comparing the DC and the node provider ID for the caller and the node.
        // That covers the case when the node provider added a new operator record in the same DC, and
        // is trying to redeploy the nodes under the new operator.
        // Hence, if the DC and the node provider of the caller and the original node operator match,
        // the removal should succeed.
        if caller_id != node_operator_id {
            let node_operator_caller = get_node_operator_record(self, caller_id)
                .map_err(|e| {
                    format!("{LOG_PREFIX}do_remove_node_directly: Aborting node removal: {e}")
                })
                .unwrap();
            let dc_caller = node_operator_caller.dc_id;
            let dc_orig_node_operator = get_node_operator_record(self, node_operator_id)
                .map_err(|e| {
                    format!("{LOG_PREFIX}do_remove_node_directly: Aborting node removal: {e}")
                })
                .unwrap()
                .dc_id;
            assert_eq!(
                dc_caller, dc_orig_node_operator,
                "The DC {dc_caller} of the caller {caller_id}, does not match the DC of the node {dc_orig_node_operator}."
            );
            let node_provider_caller = get_node_provider_id_for_operator_id(self, caller_id)
                .map_err(|e| {
                    format!("{LOG_PREFIX}do_remove_node_directly: Aborting node removal: {e}")
                });
            let node_provider_of_the_node =
                get_node_provider_id_for_operator_id(self, node_operator_id).map_err(|e| {
                    format!("{LOG_PREFIX}do_remove_node_directly: Aborting node removal: {e}")
                });
            assert_eq!(
                node_provider_caller, node_provider_of_the_node,
                "The node provider {:?} of the caller {}, does not match the node provider {:?} of the node {}.",
                node_provider_caller, caller_id, node_provider_of_the_node, payload.node_id
            );
        }
