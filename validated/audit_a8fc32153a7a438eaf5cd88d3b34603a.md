Audit Report

## Title
Unbounded `IngressQueue` + System-Subnet Guard Bypass Enables O(n) Per-Round Expiry Purge DoS on NNS — (`rs/replicated_state/src/canister_state/queues/queue.rs`, `rs/execution_environment/src/scheduler.rs`)

## Summary

The `IngressQueue` carries no capacity field and its `push()` never returns an error. On system subnets (NNS), the only induction-time size guard (`ingress_history_max_messages`) is unconditionally skipped. Free subnet-method ingress messages can therefore accumulate without bound. The scheduler's `purge_expired_ingress_messages`, called at the start of every execution round with no per-round batch cap, iterates every expired message and calls `set_status` once per message — O(n) native-Rust work outside the Wasm instruction-limit sandbox. An unprivileged attacker can flood the NNS ingress queue with free, zero-cost subnet-method messages timed to expire simultaneously, forcing a single round to process an arbitrarily large expiry batch and causing the replica to miss its consensus slot.

## Finding Description

**Step 1 — Unbounded queue, explicitly documented.**
`IngressQueue` is documented as having no upper bound: [1](#0-0) 

`push()` unconditionally appends and never returns an error: [2](#0-1) 

**Step 2 — System subnets bypass the only induction-time guard.**
`ValidSetRuleImpl::enqueue` checks `ingress_history_max_messages` only when `own_subnet_type != SubnetType::System`: [3](#0-2) 

The NNS is `SubnetType::System`, so this guard is never reached. The test suite explicitly verifies this bypass: [4](#0-3) 

The default limit that is bypassed is `INGRESS_HISTORY_MAX_MESSAGES = 2 * 1000 * 300 = 600,000`: [5](#0-4) 

**Step 3 — Free subnet-method ingress is the zero-cost entry path.**
Messages addressed to subnet methods with `IngressInductionCost::Free` are enqueued directly with no cycles charge: [6](#0-5) 

`push_ingress` on `ReplicatedState` calls `subnet_queues.push_ingress(msg)` with no size check: [7](#0-6) 

**Step 4 — O(n) expiry purge runs every round with no per-round cap.**
`purge_expired_ingress_messages` is called unconditionally at the start of every execution round: [8](#0-7) 

The function collects all expired messages via `retain_ingress_messages` (O(n) scan of the entire queue) and then calls `ingress_history_writer.set_status` once per expired message with no bound on batch size: [9](#0-8) 

`retain_messages` itself is O(n) over all messages in the queue: [10](#0-9) 

Each `set_status` call invokes `IngressHistoryState::insert`, which performs a BTreeMap insert (O(log n)) and may trigger `forget_terminal_statuses` if memory capacity is exceeded: [11](#0-10) 

**Step 5 — Existing guards are insufficient.**
The boundary-node rate limiter (`rate_limit_per_second_per_ip`, `rate_limit_per_second_per_subnet`) is optional — configured via CLI flags, not enforced by the protocol: [12](#0-11) 

The nftables firewall allows 2,000 packets/minute burst 1,000 per IP: [13](#0-12) 

With `MAX_INGRESS_MESSAGES_PER_BLOCK = 1000` and `MAX_INGRESS_TTL = 5 minutes`, an attacker using multiple IPs can accumulate tens of thousands of messages before the TTL window closes: [14](#0-13) [15](#0-14) 

## Impact Explanation

`purge_expired_ingress_messages` executes in native Rust, entirely outside the Wasm instruction-limit sandbox. An O(n) expiry purge over a large batch of simultaneously-expiring messages causes the replica to spend excessive wall-clock time in round preparation, causing it to miss its consensus slot. Repeated across TTL windows, this degrades NNS block production throughput and can stall governance proposal execution, canister upgrades, and subnet management operations that depend on timely NNS rounds.

This matches the **High ($2,000–$10,000)** impact class: *"Application/platform-level DoS, crash, consensus blocking, certified-state disruption, or subnet availability impact not based on raw volumetric DDoS."* The attack exploits a specific algorithmic weakness in the scheduler, not raw packet flooding.

## Likelihood Explanation

The attack requires no privileged access, no cycles (free subnet methods on NNS), and no threshold corruption. Any boundary-node user can submit valid signed ingress messages. The `ingress_history_max_messages` bypass for system subnets is an intentional design choice (to avoid DoS-ing system canisters), making it unlikely to be closed as a side-effect of unrelated changes. The boundary-node rate limiter is optional and bypassable with distributed submission from multiple IPs. The 5-minute accumulation window is realistic and repeatable after each TTL expiry.

## Recommendation

1. **Bound the per-round expiry purge.** Process at most `K` expired messages per round in `purge_expired_ingress_messages`, deferring the remainder to subsequent rounds.
2. **Apply an ingress-queue size cap to system subnets.** Even if `ingress_history_max_messages` is intentionally bypassed, enforce a separate per-subnet ingress-queue depth limit at induction time to prevent unbounded accumulation.
3. **Rate-limit free subnet-method ingress per sender at the protocol level.** Enforce a minimum interval between free-method ingress submissions from the same principal, independent of boundary-node configuration.

## Proof of Concept

```
1. Attacker submits N signed ingress messages to the NNS management canister
   (e.g., method `raw_rand` with no arguments, or `canister_status` with a
   valid canister ID), each with expiry_time = now + MAX_INGRESS_TTL (5 min),
   using multiple source IPs to bypass per-IP boundary-node rate limits.

2. ValidSetRuleImpl::enqueue skips the ingress_history_max_messages guard
   (SubnetType::System branch), so all N messages are inducted into the
   IngressQueue, which has no upper bound.

3. After 5 minutes, all N messages expire simultaneously.

4. The next call to execute_round() calls purge_expired_ingress_messages():
     a. subnet_queues_retain_ingress_messages() — O(N) scan of the queue.
     b. Iterates expired_ingress_messages (N entries) and calls set_status()
        for each — O(N log N) BTreeMap operations in IngressHistoryState.

5. The round takes far longer than the consensus slot deadline, causing the
   NNS replica to miss its block-making window.

Reproducible as a unit test: construct a ReplicatedState with SubnetType::System,
push N ingress messages with expiry_time = T, advance state time past T, call
purge_expired_ingress_messages(), and measure wall-clock duration vs. N.
```

    pub(super) fn push(&mut self, msg: Ingress) {
        let msg_size = Self::ingress_size_bytes(&msg);
        let receiver_ingress_queue = self.queues.entry(msg.effective_canister_id).or_default();

        if receiver_ingress_queue.is_empty() {
            self.schedule.push_back(msg.effective_canister_id);
            self.size_bytes += Self::PER_CANISTER_QUEUE_OVERHEAD_BYTES;
        }

        receiver_ingress_queue.push_back(Arc::new(msg));

        self.size_bytes += msg_size;
        debug_assert_eq!(Self::size_bytes(&self.queues), self.size_bytes);

        self.total_ingress_count += 1;
    }

    pub(super) fn retain_messages<F>(&mut self, mut predicate: F) -> Vec<Arc<Ingress>>
    where
        F: FnMut(&Ingress) -> bool,
    {
        let mut filtered_messages = vec![];
        for canister_ingress_queue in self.queues.values_mut() {
            canister_ingress_queue.retain_mut(|item| {
                if predicate(item) {
                    return true;
                }
                // Empty `canister_ingress_queues` and their corresponding schedule entry
                // are pruned below.
                filtered_messages.push(Arc::clone(item));
                self.size_bytes -= Self::ingress_size_bytes(&(*item));
                self.total_ingress_count -= 1;
                false
            });
        }

        self.schedule
            .retain_mut(|canister_id| match self.queues.entry(*canister_id) {
                Entry::Occupied(entry) if entry.get().is_empty() => {
                    entry.remove();
                    self.size_bytes -= Self::PER_CANISTER_QUEUE_OVERHEAD_BYTES;
                    false
                }
                Entry::Occupied(_) => true,
                Entry::Vacant(_) => unreachable!(),
            });

        filtered_messages
    }

    pub fn insert(
        &mut self,
        message_id: MessageId,
        status: IngressStatus,
        time: Time,
        ingress_memory_capacity: NumBytes,
        observe_time_in_terminal_state: impl Fn(u64),
    ) -> Arc<IngressStatus> {
        // Store the associated expiry time for the given message id only for a
        // "terminal" ingress status. This way we are not risking deleting any status
        // for a message that is still not in a terminal status.
        if let IngressStatus::Known { state, .. } = &status
            && state.is_terminal()
        {
            let timeout = time + MAX_INGRESS_TTL;

            // Reset `self.next_terminal_time` in case it is after the current timeout
            // and the entry is completed or failed.
            if self.next_terminal_time > timeout && state.is_terminal_with_payload() {
                self.next_terminal_time = timeout;
            }
            Arc::make_mut(&mut self.pruning_times)
                .entry(timeout)
                .or_default()
                .insert(message_id.clone());
        }
        self.memory_usage += status.payload_bytes();
        let old_status = Arc::make_mut(&mut self.statuses).insert(message_id, Arc::new(status));
        if let Some(old) = &old_status {
            self.memory_usage -= old.payload_bytes();
        }

        if self.memory_usage > ingress_memory_capacity.get() as usize {
            self.forget_terminal_statuses(
                ingress_memory_capacity,
                time,
                observe_time_in_terminal_state,
            );
        }

        debug_assert_eq!(
            Self::compute_memory_usage(&self.statuses),
            self.memory_usage
        );

        old_status.unwrap_or_else(|| IngressStatus::Unknown.into())
    }
