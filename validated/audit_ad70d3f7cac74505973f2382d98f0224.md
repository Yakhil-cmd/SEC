### Title
Irrevocable Maturity Disbursement Time-Lock with No Cancellation Mechanism - (File: `rs/nns/governance/src/governance/disburse_maturity.rs`, `rs/sns/governance/src/governance.rs`)

### Summary
Both NNS and SNS governance implement a maturity disbursement time-lock that, once initiated, cannot be cancelled, redirected, or shortened. Maturity is immediately deducted from the neuron at initiation, and the disbursement is irrevocably committed to a fixed destination for a 7-day window with no escape hatch. If the destination account becomes inaccessible or was set incorrectly, the only remedy is a governance canister upgrade — directly mirroring the TimeLock analog.

### Finding Description

The maturity is immediately subtracted from the neuron and a `MaturityDisbursement` record is written with a fixed `finalize_disbursement_timestamp_seconds`. There is no `cancel_maturity_disbursement` endpoint anywhere in the NNS governance canister. [1](#0-0) [2](#0-1) [3](#0-2) 

**SNS path** — `disburse_maturity` in `rs/sns/governance/src/governance.rs`:

```rust
finalize_disbursement_timestamp_seconds: Some(
    now_seconds + MATURITY_DISBURSEMENT_DELAY_SECONDS,
```

Same pattern: maturity is deducted immediately, destination is locked in, no cancellation exists. [4](#0-3) 

The finalization logic in both NNS (`next_maturity_disbursement_to_finalize`) and SNS (`maybe_finalize_disburse_maturity`) only checks `now_seconds >= finalize_disbursement_timestamp_seconds` — there is no early-exit or cancellation path. [5](#0-4) [6](#0-5) 

### Impact Explanation

Once a neuron controller calls `manage_neuron` → `DisburseMaturity`:

1. The maturity is **immediately and irreversibly deducted** from the neuron.
2. The disbursement is committed to a specific destination account for exactly 7 days (NNS) or `MATURITY_DISBURSEMENT_DELAY_SECONDS` (SNS).
3. There is **no cancellation, no redirection, and no speed-up** available to the neuron controller.

Concrete harm scenarios:
- The controller specified a destination account they subsequently lose access to (key compromise, exchange account closure, etc.). Funds are minted to the inaccessible account after 7 days with no recourse.
- The controller made a typo in the destination subaccount. The disbursement cannot be corrected.
- In the SNS case, a developer neuron with a large maturity balance initiates a disbursement to a deprecated account; the SNS community cannot intervene during the lock window.

The maturity is already gone from the neuron at initiation time — the 7-day window is purely a waiting period with no user-accessible abort path. The only remedy is a governance canister upgrade, exactly as described in the external report for the `TimeLock`. [7](#0-6) 

### Likelihood Explanation

The entry path is a standard unprivileged ingress call (`manage_neuron` with `DisburseMaturity`) available to any neuron controller on NNS or any principal with `DisburseMaturity` permission on an SNS neuron. No privileged access is required. The scenario of a user losing access to a destination account, or making a destination error, is realistic given the irreversibility of the operation and the 7-day window. The NNS governance canister documentation itself notes the 7-day delay is intentional for maturity modulation, but provides no user-facing escape hatch. [8](#0-7) 

### Recommendation

1. Add a `CancelMaturityDisbursement` command to `ManageNeuron` in both NNS and SNS governance that allows the neuron controller to cancel a pending disbursement during the lock window, restoring the maturity to the neuron.
2. Alternatively, allow the destination account to be updated (redirected) during the lock window, so that a controller who loses access to the original destination can redirect to a new account before finalization.
3. At minimum, document clearly and prominently that disbursements are irrevocable once initiated, so users understand the risk before calling `DisburseMaturity`.

### Proof of Concept

1. Neuron controller calls `manage_neuron` → `DisburseMaturity { percentage_to_disburse: 100, to_account: Some(<wrong_account>) }`.
2. `initiate_maturity_disbursement` immediately sets `neuron.maturity_e8s_equivalent -= disbursement_maturity_e8s` and writes `MaturityDisbursement { finalize_disbursement_timestamp_seconds: now + 604800, destination: <wrong_account> }`.
3. Controller realizes the destination is wrong and attempts to cancel — no such endpoint exists in the NNS or SNS governance canister interface.
4. After 7 days, `finalize_maturity_disbursement` fires, mints ICP/SNS tokens to `<wrong_account>`, and removes the disbursement record. Funds are permanently lost.
5. The only remediation path is an NNS/SNS governance proposal to upgrade the canister with custom migration code — identical to the "replace the TimeLock implementation" remedy described in the external report. [9](#0-8) [10](#0-9)

```rust
/// Initiates the maturity disbursement process for a neuron.
pub fn initiate_maturity_disbursement(
    neuron_store: &mut NeuronStore,
    caller: &PrincipalId,
    id: &NeuronId,
    disburse_maturity: &DisburseMaturity,
    now_seconds: u64,
) -> Result<u64, InitiateMaturityDisbursementError> {
    let DisburseMaturity {
        percentage_to_disburse,
        to_account,
        to_account_identifier,
    } = disburse_maturity;

    if *percentage_to_disburse == 0 || *percentage_to_disburse > 100 {
        return Err(InitiateMaturityDisbursementError::InvalidPercentage);
    }

    let destination = Destination::try_new(to_account, to_account_identifier, *caller)
        .map_err(|reason| InitiateMaturityDisbursementError::InvalidDestination { reason })?;

    let timestamp_of_disbursement_seconds = now_seconds;
    let finalize_disbursement_timestamp_seconds = now_seconds + DISBURSEMENT_DELAY_SECONDS;

    let (
        is_neuron_spawning,
        is_neuron_controlled_by_caller,
        num_disbursements,
        maturity_e8s_equivalent,
    ) = neuron_store
        .with_neuron(id, |neuron| {
            let is_neuron_spawning = neuron.state(now_seconds) == NeuronState::Spawning;
            let is_neuron_controlled_by_caller = neuron.is_controlled_by(caller);
            let num_disbursements = neuron.maturity_disbursements_in_progress().len();
            let maturity_e8s_equivalent = neuron.maturity_e8s_equivalent;
            (
                is_neuron_spawning,
                is_neuron_controlled_by_caller,
                num_disbursements,
                maturity_e8s_equivalent,
            )
        })
        .map_err(|_| InitiateMaturityDisbursementError::NeuronNotFound)?;

    let disbursement_maturity_e8s =
        percentage_of_maturity(maturity_e8s_equivalent, *percentage_to_disburse)?;
    if disbursement_maturity_e8s < MINIMUM_DISBURSEMENT_E8S {
        return Err(InitiateMaturityDisbursementError::DisbursementTooSmall {
            disbursement_maturity_e8s,
            minimum_disbursement_e8s: MINIMUM_DISBURSEMENT_E8S,
        });
    }

    if is_neuron_spawning {
        return Err(InitiateMaturityDisbursementError::NeuronSpawning);
    }
    if !is_neuron_controlled_by_caller {
        return Err(InitiateMaturityDisbursementError::CallerIsNotNeuronController);
    }
    if num_disbursements >= MAX_NUM_DISBURSEMENTS {
        return Err(InitiateMaturityDisbursementError::TooManyDisbursements);
    }

    let disbursement_in_progress = MaturityDisbursement {
        destination: Some(destination),
        amount_e8s: disbursement_maturity_e8s,
        timestamp_of_disbursement_seconds,
        finalize_disbursement_timestamp_seconds,
    };

    neuron_store
        .with_neuron_mut(id, |neuron| {
            neuron.add_maturity_disbursement_in_progress(disbursement_in_progress);
            neuron.maturity_e8s_equivalent = neuron
                .maturity_e8s_equivalent
                .saturating_sub(disbursement_maturity_e8s);
        })
        .map_err(|_| InitiateMaturityDisbursementError::Unknown {
            reason: "Failed to update neuron even though it was found before".to_string(),
        })?;

    Ok(disbursement_maturity_e8s)
}
```
