### Title
Missing `transferFrom` Return Value Check in XCC Precompile Allows Free Cross-Contract Calls - (File: `engine-precompiles/src/xcc.rs`)

### Summary

The `CrossContractCall` precompile calls `transferFrom` on the wNEAR ERC-20 contract to collect the required NEAR payment from the user, but only checks the EVM execution status (`ExitReason`) and never decodes or validates the boolean return value of `transferFrom`. A wNEAR token that returns `false` without reverting on a failed transfer would allow the precompile to proceed as if payment was collected, enabling free cross-contract calls at the engine's expense.

### Finding Description

In `engine-precompiles/src/xcc.rs`, `run_with_handle` performs a sub-call to the wNEAR ERC-20 contract's `transferFrom` function to move wNEAR from the caller to the engine's implicit address when `required_near != ZERO_YOCTO`: [1](#0-0) 

The call returns `(exit_reason, return_value)`. The code matches on `exit_reason` and treats `ExitReason::Succeed(_)` as a successful transfer. However, `return_value` — which contains the ABI-encoded `bool` returned by `transferFrom` — is **never decoded or checked**. [2](#0-1) 

The ERC-20 standard permits a compliant token to return `false` on failure without reverting. In that case, `exit_reason` is `Succeed`, but the transfer did not actually occur. The precompile then proceeds to emit the XCC promise log as if payment was collected.

The `transfer_from_args` helper correctly encodes the `transferFrom(from, to, amount)` calldata: [3](#0-2) 

But the boolean result is silently discarded.

### Impact Explanation

If `transferFrom` returns `false` without reverting:

1. The user's wNEAR balance is **not reduced**.
2. The engine's implicit address does **not receive** the wNEAR.
3. The XCC precompile still emits the promise log, causing the engine to schedule the cross-contract call.
4. The engine's own NEAR balance is consumed to fund the call.

This allows an attacker to repeatedly invoke the XCC precompile without paying the required wNEAR, draining the engine's NEAR balance — a **permanent fund freeze** or **direct theft** of the engine's NEAR reserves depending on how the NEAR balance is managed.

### Likelihood Explanation

The wNEAR address is set via `factory_set_wnear_address` and looked up at runtime: [4](#0-3) 

If the registered wNEAR contract is a non-reverting ERC-20 (e.g., a wrapped token that follows the original ERC-20 spec returning `false` on failure rather than reverting), the vulnerability is directly exploitable by any EVM user who calls the XCC precompile without a sufficient allowance or balance. The entry path is fully unprivileged: any EVM transaction can invoke the XCC precompile at `cross_contract_call::ADDRESS`. [5](#0-4) 

### Recommendation

After the `handle.call(...)` returns `ExitReason::Succeed`, decode `return_value` as an ABI-encoded `bool` and revert if it is `false`:

```rust
aurora_evm::ExitReason::Succeed(_) => {
    // Decode the bool return value of transferFrom
    let success = return_value
        .get(31)
        .map(|&b| b == 1)
        .unwrap_or(false);
    if !success {
        return Err(PrecompileFailure::Revert {
            exit_status: aurora_evm::ExitRevert::Reverted,
            output: b"ERR_WNEAR_TRANSFER_FROM_FAILED".to_vec(),
        });
    }
}
```

Alternatively, use a `safeTransferFrom` pattern that treats a `false` return as a revert.

### Proof of Concept

1. Deploy a wNEAR-compatible ERC-20 contract whose `transferFrom` returns `false` (without reverting) when the caller has insufficient allowance.
2. Register it as the wNEAR address via `factory_set_wnear_address`.
3. Call the XCC precompile (`0x516cded1d16af10cad47d6d49128e2eb7d27b372`) with a valid `CrossContractCallArgs::Eager` payload requiring NEAR payment, without granting any allowance to the precompile.
4. Observe: `transferFrom` returns `false`, `exit_reason` is `Succeed`, the precompile emits the XCC promise log, and the cross-contract call is scheduled — without any wNEAR being collected from the caller. [6](#0-5)
