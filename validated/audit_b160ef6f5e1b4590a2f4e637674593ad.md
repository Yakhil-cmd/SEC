### Title
SNS Swap Canister Permanently Locks SNS Tokens on Aborted Swap — (`rs/sns/swap/src/swap.rs`)

---

### Summary

When an SNS decentralization swap is aborted, the `finalize_inner` function in the SNS Swap canister returns early after restoring dapp controllers, without ever calling `sweep_sns`. The SNS tokens pre-loaded into the swap canister's account on the SNS ledger are permanently locked with no recovery mechanism. There is no `error_refund_sns` equivalent in the canister's public API.

---

### Finding Description

The `finalize_inner` function in `rs/sns/swap/src/swap.rs` handles both COMMITTED and ABORTED swap finalization paths. The critical control-flow divergence is:

```
finalize_inner():
  1. sweep_icp()                          ← ICP returned to buyers ✓
  2. settle_neurons_fund_participation()  ← NF maturity refunded ✓
  3. if should_restore_dapp_control():    ← true when Lifecycle::Aborted
       restore_dapp_controllers()
       return ← EARLY RETURN, sweep_sns() never reached
  4. create_sns_neuron_recipes()          ← only on COMMITTED
  5. sweep_sns()                          ← only on COMMITTED
```

`should_restore_dapp_control()` is defined as:

```rust
pub fn should_restore_dapp_control(&self) -> bool {
    self.lifecycle() == Lifecycle::Aborted
}
``` [1](#0-0) 

When the swap is aborted, `finalize_inner` hits the early-return branch at line 1583 and never reaches `sweep_sns`: [2](#0-1) 

`sweep_sns` is only reachable in the committed path: [3](#0-2) 

The canister's public API (`canister.rs`) exposes `error_refund_icp` for ICP recovery but has no analogous `error_refund_sns` or standalone `sweep_sns` endpoint: [4](#0-3) 

The SNS tokens loaded into the swap canister's SNS ledger account before the swap opens (the `swap_distribution_sns_e8s` amount) remain permanently locked. The protocol's own invariant states: *"The swap canister can be deleted when all tokens registered with the swap canister have been disbursed to their rightful owners"* — this condition is never satisfied on abort. [5](#0-4) 

The integration test at `rs/nervous_system/integration_tests/tests/sns_lifecycle.rs` explicitly confirms that on abort, the swap canister retains the full SNS token balance:

```rust
if swap_finalization_status == SwapFinalizationStatus::Aborted {
    // If the swap fails, the SNS swap does not distribute any tokens.
    assert_eq!(swap_canister_balance_sns_e8s, swap_distribution_sns_e8s);
``` [6](#0-5) 

---

### Impact Explanation

**Ledger conservation bug.** All SNS tokens pre-loaded for the swap (potentially millions of dollars worth of governance tokens) are permanently locked in the swap canister's SNS ledger account when the swap is aborted. The SNS governance treasury never receives these tokens back. The SNS governance canister also remains in `PreInitializationSwap` mode (since `set_sns_governance_to_normal_mode` is also skipped), compounding the loss by leaving the entire SNS in a degraded state. The swap canister cannot be properly deleted per protocol invariants. [7](#0-6) 

---

### Likelihood Explanation

SNS swaps failing to meet minimum participation thresholds is a realistic and documented scenario. Any unprivileged user can trigger this outcome simply by not participating. The swap automatically transitions to `Aborted` when the due date passes without `sufficient_participation`. Once aborted, any caller can invoke `finalize_swap` (it is a public, permissionless endpoint), which executes `finalize_inner` and triggers the early return. No privileged access, key compromise, or majority attack is required. [8](#0-7) 

---

### Recommendation

In `finalize_inner`, when the swap is aborted, add a `sweep_sns` call that transfers the SNS tokens back to the SNS governance treasury (or a designated recovery subaccount) **before** restoring dapp controllers and returning. Alternatively, add a standalone public `sweep_sns` endpoint (analogous to `error_refund_icp`) that can be called after finalization to recover stranded SNS tokens from the swap canister's account. [9](#0-8) 

---

### Proof of Concept

The existing integration test already demonstrates the issue without recognizing it as a bug. After an aborted swap finalization:

1. A swap is opened with `swap_distribution_sns_e8s` SNS tokens loaded into the swap canister's SNS ledger account.
2. The swap is aborted (minimum participation not reached).
3. `finalize_swap` is called → `finalize_inner` runs → `sweep_icp` returns ICP to buyers → `should_restore_dapp_control()` returns `true` → early return.
4. Query the SNS ledger balance of the swap canister's default account: it equals `swap_distribution_sns_e8s`.
5. There is no callable method on the swap canister to recover these tokens.
6. The SNS governance treasury balance is unchanged — the tokens are permanently lost.

The test at `rs/nervous_system/integration_tests/tests/sns_lifecycle.rs` line 1336–1342 already asserts step 4 as "expected" behavior, confirming the tokens are retained by the swap canister with no recovery path. [10](#0-9)
