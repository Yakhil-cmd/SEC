Audit Report

## Title
SNS Governance Permanently Stuck in `PreInitializationSwap` Mode After Aborted Swap, Blocking All Neuron Management - (`rs/sns/swap/src/swap.rs`, `rs/sns/governance/src/governance.rs`)

## Summary

When an SNS decentralization swap is aborted, `finalize_inner` in `rs/sns/swap/src/swap.rs` returns early on the `should_restore_dapp_control()` branch without calling `set_sns_governance_to_normal_mode`. Because `set_mode` in SNS Governance is exclusively callable by the Swap canister, governance remains permanently in `PreInitializationSwap` mode. In this mode, `Disburse`, `DisburseMaturity`, `Split`, `Configure`, and `MergeMaturity` commands are unconditionally blocked for all callers, permanently locking SNS neuron holders' staked tokens with no user-accessible recovery path.

## Finding Description

**Root cause — missing `set_mode` call on the aborted path:**

In `rs/sns/swap/src/swap.rs`, `finalize_inner` handles the aborted swap path at lines 1572–1584:

```rust
if self.should_restore_dapp_control() {
    finalize_swap_response.set_set_dapp_controllers_result(
        self.restore_dapp_controllers_for_finalize(environment.sns_root_mut()).await,
    );
    return finalize_swap_response;  // returns WITHOUT calling set_sns_governance_to_normal_mode
}
```

`set_sns_governance_to_normal_mode` is only called on the committed path (lines 1610–1612), never on the aborted path.

**Access control on `set_mode` prevents any other recovery:**

In `rs/sns/governance/src/governance.rs` lines 785–801, `set_mode` panics if the caller is not the Swap canister:

```rust
if !self.is_swap_canister(caller) {
    panic!("Caller must be the swap canister.");
}
```

No other entity — not the neuron owner, not NNS, not the SNS root — can call `set_mode`.

**Commands blocked in `PreInitializationSwap` mode:**

In `rs/sns/governance/src/types.rs` lines 182–211, `manage_neuron_command_is_allowed_in_pre_initialization_swap_or_err` returns `Err(PreconditionFailed)` for all commands except `Follow`, `MakeProposal`, `RegisterVote`, `AddNeuronPermissions`, `RemoveNeuronPermissions`, and (swap canister only) `ClaimOrRefresh`. This unconditionally blocks `Disburse`, `DisburseMaturity`, `Split`, `Configure` (start/stop dissolving), and `MergeMaturity`.

**Confirmed by integration tests:**

`rs/nervous_system/integration_tests/tests/sns_lifecycle.rs` lines 1118–1127 explicitly asserts governance remains in `PreInitializationSwap` mode after an aborted swap. Lines 1268–1289 assert that `start_dissolving` returns `PreconditionFailed` with message containing "PreInitializationSwap" after abort. The test comment at lines 128–135 documents `{ !DissolveSnsNeuron.is_enabled() } FinalizeUnSuccessfully { !DissolveSnsNeuron.is_enabled() }` as the current invariant — confirming this is the actual production behavior.

## Impact Explanation

SNS neuron holders (developer neurons, airdrop neurons) whose swap was aborted cannot start dissolving, disburse dissolved neurons, disburse maturity, or split neurons. Their staked SNS tokens are locked in the governance canister indefinitely. The only recovery path is an NNS governance proposal to upgrade the SNS Governance canister — a privileged action unavailable to ordinary neuron holders. This constitutes a significant SNS security impact with concrete, demonstrable user harm: permanent inaccessibility of staked funds contingent on external governance action. This matches the **High** impact category: "Significant SNS or infrastructure security impact with concrete user or protocol harm."

## Likelihood Explanation

This is a **deterministic, confirmed** outcome for every SNS whose decentralization swap fails to reach `min_participants`. No special privileges are required to be affected — any holder of a developer neuron or airdrop neuron in an SNS whose swap aborts is automatically impacted. The integration test suite explicitly encodes this as expected behavior, confirming it is present in production code. The condition (swap abort) is a normal, foreseeable outcome of the SNS lifecycle.

## Recommendation

In `finalize_inner` in `rs/sns/swap/src/swap.rs`, call `set_sns_governance_to_normal_mode` before returning on the aborted path:

```rust
if self.should_restore_dapp_control() {
    finalize_swap_response.set_set_dapp_controllers_result(
        self.restore_dapp_controllers_for_finalize(environment.sns_root_mut()).await,
    );
    // Reset governance to Normal mode on the aborted path
    finalize_swap_response.set_set_mode_call_result(
        Self::set_sns_governance_to_normal_mode(environment.sns_governance_mut()).await,
    );
    return finalize_swap_response;
}
```

The integration test invariants at `rs/nervous_system/integration_tests/tests/sns_lifecycle.rs` lines 128–135 and 1118–1127 should be updated to assert `governance::Mode::Normal` after an unsuccessful finalization.

## Proof of Concept

1. Deploy an SNS; governance enters `PreInitializationSwap` mode.
2. Open a decentralization swap; close it without reaching `min_participants` → swap state transitions to `Aborted`.
3. Call `finalize` (or allow auto-finalization). `finalize_inner` executes `sweep_icp`, `settle_neurons_fund_participation`, then hits `should_restore_dapp_control() == true`, calls `restore_dapp_controllers_for_finalize`, and returns — **without** calling `set_sns_governance_to_normal_mode`.
4. Call `sns_governance.get_mode()` → returns `PreInitializationSwap`. (Confirmed by test at lines 1118–1127.)
5. Call `manage_neuron` with `Command::Configure(StartDissolving)` as a developer neuron holder → returns `PreconditionFailed: "Because governance is currently in PreInitializationSwap mode, manage_neuron commands of this type are not allowed"`. (Confirmed by test at lines 1268–1289.)
6. Repeat for `Disburse`, `DisburseMaturity`, `Split` — all return the same error.
7. Neuron holder's staked SNS tokens are inaccessible with no user-triggered recovery path.
