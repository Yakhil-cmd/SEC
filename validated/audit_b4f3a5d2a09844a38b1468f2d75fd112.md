### Title
Unprivileged Caller Can Force Deployment of Staged Contract Code, Removing Owner's Ability to Cancel Upgrades Without Pausing the Engine — (`engine/src/lib.rs`)

---

### Summary

The `deploy_upgrade` function in the Aurora Engine contract has **no caller authentication**. Once the owner stages a new contract binary via `stage_upgrade` and the configured delay period elapses, **any unprivileged NEAR account** can call `deploy_upgrade` to deploy that code. The owner has no way to cancel a staged upgrade without pausing the entire engine contract, which temporarily freezes all user funds and operations.

---

### Finding Description

`stage_upgrade` is correctly gated behind `require_owner_only`: [1](#0-0) 

It stores the new binary at `KeyPrefix::Config / CODE_KEY` and records the earliest allowed deployment block at `KeyPrefix::Config / CODE_STAGE_KEY`.

`deploy_upgrade`, however, performs **no predecessor check**: [2](#0-1) 

The only guards are:
1. `require_running` — the contract must not be paused.
2. A block-height comparison against the staged index.

This was explicitly made open to any caller per the project's own changelog: [3](#0-2) 

Because `require_running` is the only gate, the owner's sole recourse to block an unwanted deployment is to call `pause_contract`, which is itself owner-only: [4](#0-3) 

Pausing the contract sets `state.is_paused = true`, which blocks all mutative user operations (EVM transactions, withdrawals, bridge interactions).

---

### Impact Explanation

**Impact: High — Temporary Freezing of Funds**

The attack creates a forced dilemma for the owner after a staging event:

1. Owner calls `stage_upgrade` with new code (legitimate).
2. Owner discovers a bug in the staged binary, or needs to delay the rollout for operational reasons.
3. The upgrade delay elapses.
4. An attacker calls `deploy_upgrade`, deploying the staged code before the owner can replace it.

To prevent step 4, the owner must call `pause_contract` first. While paused, **all user-facing operations are blocked**: EVM `submit`/`call`, bridge `withdraw`, `ft_transfer`, and all connector interactions. This constitutes a temporary freezing of user funds.

If the staged code itself contains a logic error (e.g., a broken withdrawal path or incorrect balance accounting), the forced deployment can escalate to **permanent fund freezing or direct theft**, since the new code is live and immutable until another upgrade cycle completes.

---

### Likelihood Explanation

**Likelihood: Medium**

The precondition is that the owner has staged an upgrade and the delay has elapsed. On a live production system (`aurora` on NEAR mainnet), upgrades are routine operational events. Any NEAR account — including a bot monitoring the chain for `stage_upgrade` transactions — can call `deploy_upgrade` the moment the delay block is reached. The owner has a narrow window to react (pause the contract) before an attacker can race them. The CHANGES.md entry confirms this is the current production behavior.

---

### Recommendation

Add a caller check to `deploy_upgrade` so that only the owner (or a designated account) can trigger it, consistent with how `stage_upgrade`, `upgrade`, `pause_contract`, and all other sensitive admin functions are protected:

```rust
pub extern "C" fn deploy_upgrade() {
    let mut io = Runtime;
    let env = Runtime;
    let state = state::get_state(&io).sdk_unwrap();
    require_running(&state).map_err(ContractError::msg).sdk_unwrap();
    require_owner_only(&state, &env.predecessor_account_id())
        .map_err(ContractError::msg)
        .sdk_unwrap();
    // ... rest of function
}
```

Alternatively, introduce a `cancel_upgrade` function that allows the owner to clear `CODE_KEY` and `CODE_STAGE_KEY` without needing to pause the contract.

---

### Proof of Concept

**Entry path (no special privileges required after staging):**

1. Owner calls `stage_upgrade` with new WASM binary → stored at `CODE_KEY`; delay index written to `CODE_STAGE_KEY`.
2. `upgrade_delay_blocks` blocks pass on NEAR.
3. Attacker (any NEAR account) calls `deploy_upgrade`:
   - `require_running` passes (contract is live).
   - `block_height > index` passes (delay elapsed).
   - `Runtime::self_deploy` executes, replacing the engine contract binary.
   - `CODE_STAGE_KEY` is cleared.
4. Owner discovers the deployment happened without their consent. To have prevented it, they would have had to call `pause_contract` first — freezing all user funds in the interim.

The asymmetry is structural: `stage_upgrade` is owner-only (line 159 of `admin.rs`), but `deploy_upgrade` (lines 171–185 of `lib.rs`) has no equivalent guard, making the two-step upgrade process exploitable by any unprivileged observer. [5](#0-4) [6](#0-5) [7](#0-6)

```rust
#[named]
pub fn stage_upgrade<I: IO + Copy, E: Env>(io: I, env: &E) -> Result<(), ContractError> {
    with_hashchain(io, env, function_name!(), |mut io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;
        let delay_block_height = env.block_height() + state.upgrade_delay_blocks;
        require_owner_only(&state, &env.predecessor_account_id())?;
        io.read_input_and_store(&storage::bytes_to_key(KeyPrefix::Config, CODE_KEY));
        io.write_storage(
            &storage::bytes_to_key(KeyPrefix::Config, CODE_STAGE_KEY),
            &delay_block_height.to_le_bytes(),
        );
        Ok(())
    })
}
```

```rust
pub fn pause_contract<I: IO + Copy, E: Env>(io: I, env: &E) -> Result<(), ContractError> {
    with_hashchain(io, env, function_name!(), |mut io| {
        let mut state = state::get_state(&io)?;
        require_owner_only(&state, &env.predecessor_account_id())?;
        require_running(&state)?;
        state.is_paused = true;
        state::set_state(&mut io, &state)?;
        Ok(())
    })
}
```

```rust
    /// Deploy staged upgrade.
    #[unsafe(no_mangle)]
    pub extern "C" fn deploy_upgrade() {
        // This function is intentionally not implemented in `contract_methods`
        // because it only makes sense in the context of the NEAR runtime.
        let mut io = Runtime;
        let state = state::get_state(&io).sdk_unwrap();
        require_running(&state)
            .map_err(ContractError::msg)
            .sdk_unwrap();
        let index = internal_get_upgrade_index();
        if io.block_height() <= index {
            sdk::panic_utf8(errors::ERR_NOT_ALLOWED_TOO_EARLY);
        }
        Runtime::self_deploy(&bytes_to_key(KeyPrefix::Config, CODE_KEY));
        io.remove_storage(&bytes_to_key(KeyPrefix::Config, CODE_STAGE_KEY));
    }
```

```rust
pub fn require_owner_only(
    state: &state::EngineState,
    predecessor_account_id: &AccountId,
) -> Result<(), ContractError> {
    if &state.owner_id != predecessor_account_id {
        return Err(errors::ERR_NOT_ALLOWED.into());
    }
    Ok(())
}
```
