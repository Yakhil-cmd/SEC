Audit Report

## Title
Byzantine Peer Can Abort Block Proposal Assembly via Duplicate Ingress IDs in `StrippedBlockProposal` — (`rs/p2p/artifact_downloader/src/fetch_stripped_artifact/assembler.rs`)

## Summary
`StrippedBlockProposal::try_from` in `stripped.rs` performs no deduplication on the deserialized `ingress_messages` Vec. A Byzantine peer can craft a `StrippedBlockProposal` protobuf with the same `SignedIngressId` repeated twice. This causes `BlockProposalAssembler` to spawn two concurrent `get_or_fetch` tasks for the same ID; when the second result arrives, `try_insert`'s linear `find` lands on the already-filled first slot and returns `InsertionError::AlreadyInserted`, which `assemble_message` maps directly to `AssembleResult::Unwanted`, aborting assembly of an otherwise valid block proposal.

## Finding Description

**Root cause — no dedup in `StrippedBlockProposal::try_from`** (`stripped.rs` lines 100–107):
```rust
ingress_messages: value
    .ingress_messages
    .into_iter()
    .map(SignedIngressId::try_from)
    .collect::<Result<Vec<_>, _>>()?,
```
A Byzantine peer can include the same `pb::StrippedIngressMessage` twice; deserialization succeeds with no uniqueness check.

**Duplicates propagate into `BlockProposalAssembler::ingress_messages`** (`assembler.rs` lines 574–579):
```rust
ingress_messages: stripped_block_proposal
    .stripped_ingress_payload
    .ingress_messages
    .iter()
    .map(|signed_ingress_id| (signed_ingress_id.clone(), None))
    .collect(),
```
Both duplicate entries become `(id, None)` slots.

**`missing_artifacts` returns both `None` slots** (`assembler.rs` lines 428–433), so `missing_stripped_messages` emits the same `SignedIngressId` twice.

**Two `get_or_fetch` tasks are spawned** (`assembler.rs` lines 214–226) — one per entry in `stripped_message_ids`.

**`try_insert` uses first-match `find`** (`assembler.rs` lines 442–453):
```rust
let (_, ingress) = self
    .ingress_messages
    .iter_mut()
    .find(|(id, _maybe_ingress)| *id == signed_ingress_id)
    .ok_or(InsertionError::NotNeeded)?;

if ingress.is_some() {
    Err(InsertionError::AlreadyInserted)
} else {
    *ingress = Some(ingress_message);
    Ok(())
}
```
After task 1 fills slot 0 with `Some(ingress)`, task 2's `find` again returns slot 0 (now `Some`) and returns `Err(InsertionError::AlreadyInserted)`. Slot 1 (the duplicate, still `None`) is never reached.

**`AlreadyInserted` → `AssembleResult::Unwanted`** (`assembler.rs` lines 262–271):
```rust
if let Err(err) = assembler.try_insert_stripped_message(message) {
    warn!(...);
    return AssembleResult::Unwanted;
}
```
Assembly is aborted before `try_assemble`'s hash-mismatch check is ever reached.

## Impact Explanation
A single Byzantine P2P peer can cause any replica that receives its crafted `StrippedBlockProposal` to return `AssembleResult::Unwanted` for a valid block proposal, aborting that assembly invocation. If the P2P layer does not immediately re-fetch the stripped artifact from an honest peer (or if the Byzantine peer races each retry), the block proposal is suppressed for that assembly cycle, stalling consensus progress. This matches the allowed impact: **High ($2,000–$10,000) — Application/platform-level DoS, consensus blocking, or subnet availability impact not based on raw volumetric DDoS.**

## Likelihood Explanation
The attack requires only a single Byzantine peer in the P2P gossip layer — well below any consensus fault threshold. The crafted protobuf is trivially constructable by repeating one `StrippedIngressMessage` entry. The target `ConsensusMessageId` is publicly advertised. No cryptographic material, admin keys, or privileged access is needed. The attack is repeatable across every block proposal round.

## Recommendation
Add a duplicate-ID check in `StrippedBlockProposal::try_from` (in `stripped.rs`) immediately after collecting the `Vec<SignedIngressId>`:
```rust
let ingress_messages: Vec<SignedIngressId> = value
    .ingress_messages
    .into_iter()
    .map(SignedIngressId::try_from)
    .collect::<Result<Vec<_>, _>>()?;

let unique_count = ingress_messages
    .iter()
    .collect::<std::collections::HashSet<_>>()
    .len();
if unique_count != ingress_messages.len() {
    return Err(ProxyDecodeError::Other(
        "stripped_ingress_payload contains duplicate ingress IDs".into(),
    ));
}
```
As defense-in-depth, also fix `try_insert` to search for the first `None` slot matching the ID rather than the first slot matching the ID regardless of fill state.
