Audit Report

## Title
`inspect_message`-Only Authorization Bypassed via Inter-Canister Calls — (`rs/boundary_node/rate_limits/canister/canister.rs`)

## Summary
The rate-limits canister relies exclusively on `canister_inspect_message` to gate access to its three read-only query methods (`get_config`, `get_rule_by_id`, `get_rules_by_incident_id`). Because the IC execution environment only invokes `inspect_message` for signed ingress messages — never for inter-canister calls — any on-chain canister can call these methods directly, bypassing the gate entirely. The query bodies apply only confidentiality *formatting* (redaction) for `RestrictedRead` callers rather than an outright rejection, so structural metadata of confidential, undisclosed rate-limit rules is returned to the attacker.

## Finding Description
**Root cause — `inspect_message` as sole gate:**
`canister.rs` defines a single `#[inspect_message]` hook that enforces access control for all methods:
- `get_config` (`REPLICATED_QUERY_METHOD`): accepted only for `FullAccess` or `FullRead` callers.
- `get_rule_by_id` and `get_rules_by_incident_id`: fall into the `else` branch and are trapped for *all* ingress callers.
- `add_config` / `disclose_rules`: accepted only for `FullAccess`. [1](#0-0) 

**No in-body authorization in the query methods:**
All three query methods unconditionally construct an `AccessLevelResolver`, resolve the caller's level, and return data. There is no `Err(Unauthorized)` path for `RestrictedRead` callers. [2](#0-1) 

**Confidentiality formatter leaks structural metadata:**
For `RestrictedRead` callers, `ConfigConfidentialityFormatter::format` sets `is_redacted = true` and nulls out `rule_raw`/`description` for non-disclosed rules, but leaves `rule_id` and `incident_id` intact in every rule entry. [3](#0-2) 

`RuleConfidentialityFormatter` similarly preserves `rule_id`, `incident_id`, `added_in_version`, and `removed_in_version` for non-disclosed rules. [4](#0-3) 

This is confirmed by the unit test for `RestrictedRead` access: `rule_id` and `incident_id` are present even for the non-disclosed rule (`disclosed_at: None`). [5](#0-4) 

**IC protocol property — `inspect_message` not called for inter-canister calls:**
`execute_inspect_message` takes a `&SignedIngressContent` argument, making it structurally impossible to invoke for canister-to-canister messages. [6](#0-5) 

**Contrast with update methods — they carry independent authorization:**
`add_config` and `disclose_rules` wrap their inner logic in `WithAuthorization`, which independently checks `AccessLevel::FullAccess` at execution time regardless of how the message arrived. [7](#0-6) [8](#0-7) 

The query methods have no equivalent wrapper.

**Exploit flow:**
1. Attacker deploys a canister on any subnet.
2. Attacker canister issues a composite-query inter-canister call to `get_config` (or `get_rule_by_id` / `get_rules_by_incident_id`) on the rate-limits canister.
3. `inspect_message` is not invoked (IC protocol guarantee).
4. `AccessLevelResolver::get_access_level()` resolves the attacker canister's principal to `RestrictedRead` (it is neither the authorized principal nor an API boundary node).
5. `ConfigGetter::get` returns a `ConfigResponse` with `is_redacted: true` but with all rule entries present — including non-disclosed rules — each carrying `rule_id`, `incident_id`, `added_in_version`, and `removed_in_version`. [9](#0-8) 

## Impact Explanation
The rate-limits canister's confidentiality model requires that undisclosed rules be completely invisible to unauthorized parties. The bypass exposes the existence of every confidential rate-limit rule and its structural metadata (`rule_id`, `incident_id`, version numbers). An attacker can enumerate all active undisclosed rules, track when new rules are added or removed, and correlate rule identifiers with known security incidents — undermining the operational security of the boundary-node rate-limiting system. This constitutes a **significant boundary/API infrastructure security impact with concrete protocol harm**, fitting the High ($2,000–$10,000) impact category.

## Likelihood Explanation
The attack requires no special privileges, no key material, and no subnet-majority corruption. Any canister deployed on any subnet can execute it deterministically and repeatably. The only prerequisite is knowledge of the rate-limits canister ID, which is public. The bypass is a guaranteed IC protocol property, not a speculative edge case.

## Recommendation
Add explicit caller-identity checks inside the query function bodies, mirroring the `WithAuthorization` pattern used for `add_config` and `disclose_rules`. For `RestrictedRead` callers, return an `Unauthorized` error rather than a redacted response. Apply this to all three query methods: `get_config`, `get_rule_by_id`, and `get_rules_by_incident_id`. The `inspect_message` hook should be treated as a best-effort pre-consensus cycle-saving filter, not as the authoritative access-control boundary.

## Proof of Concept
1. Deploy an attacker canister on any application subnet.
2. From the attacker canister, issue a composite-query inter-canister call to the rate-limits canister's `get_config` method (no argument needed for latest version).
3. Observe that `inspect_message` is not invoked (IC protocol guarantee; confirmed by `execute_inspect_message` signature taking `SignedIngressContent`).
4. `get_config` executes; `AccessLevelResolver` resolves the attacker canister's principal to `RestrictedRead`.
5. The returned `ConfigResponse` has `is_redacted: true` but contains all rule entries — including non-disclosed rules — with `rule_id` and `incident_id` populated and `rule_raw`/`description` set to `None`.
6. Repeat with `get_rule_by_id(rule_id)` using a harvested `rule_id` to confirm `added_in_version` and `removed_in_version` are also returned for non-disclosed rules.

A deterministic integration test can be written using PocketIC: install the rate-limits canister, add a config with an undisclosed rule, install an attacker canister, have it issue a composite-query call to `get_config`, and assert that the non-disclosed rule's `rule_id` and `incident_id` appear in the response.

#[inspect_message]
fn inspect_message() {
    // In order for this hook to succeed, accept_message() must be invoked.
    let caller_id: Principal = ic_cdk::api::caller();
    let called_method = ic_cdk::api::call::method_name();

    let (has_full_access, has_full_read_access) = with_canister_state(|state| {
        let authorized_principal = state.get_authorized_principal();
        (
            Some(caller_id) == authorized_principal,
            state.is_api_boundary_node_principal(&caller_id),
        )
    });

    if called_method == REPLICATED_QUERY_METHOD {
        if has_full_access || has_full_read_access {
            ic_cdk::api::call::accept_message();
        } else {
            ic_cdk::api::trap(
                "message_inspection_failed: method call is prohibited in the current context",
            );
        }
    } else if UPDATE_METHODS.contains(&called_method.as_str()) {
        if has_full_access {
            ic_cdk::api::call::accept_message();
        } else {
            ic_cdk::api::trap("message_inspection_failed: unauthorized caller");
        }
    } else {
        // All others calls are rejected
        ic_cdk::api::trap(
            "message_inspection_failed: method call is prohibited in the current context",
        );
    }
}

impl ConfidentialityFormatting for ConfigConfidentialityFormatter {
    type Input = OutputConfig;

    fn format(&self, config: OutputConfig) -> OutputConfig {
        let mut config = config;
        config.is_redacted = true;
        // Redact (hide) fields of non-disclosed rules
        config.rules.iter_mut().for_each(|rule| {
            if rule.disclosed_at.is_none() {
                rule.description = None;
                rule.rule_raw = None;
            }
        });
        config
    }

impl<T: AddsConfig, R: ResolveAccessLevel> AddsConfig for WithAuthorization<T, R> {
    fn add_config(
        &self,
        input_config: rate_limits_api::InputConfig,
        time: Timestamp,
    ) -> Result<(), AddConfigError> {
        // Only privileged users can perform this operation
        if self.access_resolver.get_access_level() == AccessLevel::FullAccess {
            // Perform the inner call only if authorized.
            return self.inner.add_config(input_config, time);
        }
        Err(AddConfigError::Unauthorized)
    }
}

        }
        Err(DiscloseRulesError::Unauthorized)
    }
