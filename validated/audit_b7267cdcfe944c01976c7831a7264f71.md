### Title
Unsanitized Memo Interpolation Enables Markdown Injection into ICRC-21 GenericDisplayMessage — (`packages/icrc-ledger-types/src/icrc21/responses.rs`)

---

### Summary

`ConsentMessage::add_memo` interpolates raw, user-controlled UTF-8 memo bytes directly into a Markdown string without escaping. A backtick character inside the memo closes the surrounding code-span fence, allowing an attacker to inject arbitrary Markdown — including fake `**To:**` fields — into the `GenericDisplayMessage` returned by `icrc21_canister_call_consent_message`. Hardware wallets that render this string as Markdown (as the ICRC-21 spec intends) will display the injected content as legitimate transaction fields.

---

### Finding Description

In `add_memo`, the memo bytes are decoded as UTF-8 (or hex-encoded if invalid), then spliced verbatim into the Markdown string:

```rust
// responses.rs line 294
message.push_str(&format!("\n\n**Memo:**\n`{memo_str}`"));
``` [1](#0-0) 

There is no escaping of backticks, newlines, or any other Markdown control characters in `memo_str`. If `memo_str` contains a backtick, it terminates the opening `` ` `` code-span early. Everything after that backtick is rendered as raw Markdown.

**Concrete injection payload:**

Memo bytes (UTF-8): `` hello`\n\n**To:**\n`attacker_address ``

The string appended to the message becomes:

```
\n\n**Memo:**\n`hello`\n\n**To:**\n`attacker_address`
```

Which renders in Markdown as:

| Rendered element | Source |
|---|---|
| **Memo:** | legitimate label |
| `hello` | code span (closed by injected backtick) |
| **To:** | **injected bold label** |
| `attacker_address` | **injected code span** |

The build order in `lib.rs` places the real `**To:**` field (from `add_account`) before the memo: [2](#0-1) 

So the full rendered message contains **two `**To:**` fields**: the real one (attacker's account, shown first) and the injected one (victim/decoy address, shown last). On a hardware wallet with a small scrolling display, the attacker can craft the memo to make the injected field appear to "correct" or "supersede" the earlier one — e.g., with a memo like:

```
`\n\n**Note:** Address updated.\n\n**To:**\n`legitimate_looking_address
```

The `FieldsDisplayMessage` variant is **not affected** — it stores fields as structured `Vec<(String, Value)>` and never concatenates user content into a Markdown string. [3](#0-2) 

---

### Impact Explanation

Any unprivileged caller can submit a `ConsentMessageRequest` with `method = "icrc1_transfer"` and a crafted `TransferArg` whose `memo` field contains Markdown injection. The ledger canister will return a `ConsentMessage::GenericDisplayMessage` containing the injected content. A hardware wallet (Ledger, Keystone, etc.) that renders this string as Markdown — which is the intended behavior per ICRC-21 — will display the injected `**To:**` field as a legitimate transaction field. A user who approves based on the injected display will authorize a transfer to the attacker's actual `TransferArg.to` address.

---

### Likelihood Explanation

- The entrypoint is fully public: any caller can invoke `icrc21_canister_call_consent_message`.
- The memo field accepts arbitrary bytes up to the ICRC-1 memo size limit (32 bytes is enough for the payload).
- The arg size limit of 500 bytes is easily satisfied.
- `GenericDisplayMessage` is explicitly designed to be rendered as Markdown by wallets.
- No privilege, key, or majority is required.

---

### Recommendation

Escape all Markdown control characters in `memo_str` before interpolation. At minimum, replace backticks with their HTML entity or a safe substitute, and strip or escape newlines:

```rust
let safe_memo = memo_str
    .replace('`', "\\`")
    .replace('\n', " ")
    .replace('\r', "");
message.push_str(&format!("\n\n**Memo:**\n`{safe_memo}`"));
```

Alternatively, use a Markdown-escaping library, or switch the memo display to a `FieldsDisplayMessage` field (which is already injection-safe).

### Proof of Concept

```rust
#[test]
fn memo_injection_creates_fake_to_field() {
    let mut msg = ConsentMessage::GenericDisplayMessage(String::new());
    msg.add_intent(Icrc21Function::Transfer, Some("ICP".to_string()));
    msg.add_account("To", "attacker-principal".to_string());
    // Memo payload closes the code span and injects a fake To: field
    let evil_memo = b"`\n\n**To:**\n`victim-principal".to_vec();
    msg.add_memo(GenericMemo::Icrc1Memo(evil_memo.into()));

    let rendered = match msg {
        ConsentMessage::GenericDisplayMessage(s) => s,
        _ => panic!(),
    };
    // The rendered string now contains a second **To:** field
    let to_count = rendered.matches("**To:**").count();
    assert_eq!(to_count, 1, "Injection succeeded: found {} **To:** fields", to_count);
    // This assertion FAILS with the current code, proving the injection.
}
