### Title
Voting Rewards Permanently Lost When Voting Neurons Are Deleted Before Reward Distribution - (File: rs/nns/governance/src/governance.rs)

### Summary
In both NNS and SNS governance, when a neuron votes on a proposal and is then dissolved and disbursed before the reward distribution period ends, its proportional share of voting rewards is permanently lost. The rewards are neither distributed to any neuron nor rolled over to the next reward round. This is a direct analog to the Velocimeter M-6 bug: rewards computed for a period with no eligible recipients are permanently discarded rather than preserved.

### Finding Description

**NNS Governance (`rs/nns/governance/src/governance.rs`):**

`calculate_voting_rewards` computes `total_available_e8s_equivalent_float` as the full reward purse for the period. It then iterates over voters from proposal ballots and checks whether each neuron still exists in the neuron store:

```rust
for (neuron_id, used_voting_rights) in voters_to_used_voting_right {
    if self.neuron_store.contains(neuron_id) {
        let reward = (used_voting_rights * total_available_e8s_equivalent_float
            / total_voting_rights) as u64;
        reward_distribution.add_reward(neuron_id, reward);
        actually_distributed_e8s_equivalent += reward;
    } else {
        println!(
            "{}Cannot find neuron {}, despite having voted with power {} \
                in the considered reward period. The reward that should have been \
                distributed to this neuron is simply skipped, so the total amount \
                of distributed reward for this period will be lower than the maximum \
                allowed.",
            ...
        );
    }
}
``` [1](#0-0) 

The reward event is then recorded with `settled_proposals` non-empty (the proposals were settled) and `distributed_e8s_equivalent < total_available_e8s_equivalent`. The critical rollover logic is:

```rust
pub(crate) fn rewards_rolled_over(&self) -> bool {
    self.settled_proposals.is_empty()
}

pub(crate) fn e8s_equivalent_to_be_rolled_over(&self) -> u64 {
    if self.rewards_rolled_over() {
        self.total_available_e8s_equivalent
    } else {
        0
    }
}
``` [2](#0-1) [3](#0-2) 

Because `settled_proposals` is non-empty, `rewards_rolled_over()` returns `false`, `e8s_equivalent_to_be_rolled_over()` returns `0`, and the next round's `total_available_e8s_equivalent_float` does not include the undistributed amount. The gap between `total_available_e8s_equivalent` and `distributed_e8s_equivalent` is permanently discarded.

**SNS Governance (`rs/sns/governance/src/governance.rs`):**

The same pattern exists. When `total_reward_shares == dec!(0)` (all voting neurons deleted), the code logs an ERROR and skips distribution entirely:

```rust
if total_reward_shares == dec!(0) {
    log!(
        ERROR,
        "Warning: total_reward_shares is 0. Therefore, we skip increasing \
         neuron maturity. ...",
    );
}
``` [4](#0-3) 

The SNS rollover check is identical in structure:

```rust
pub(crate) fn rewards_rolled_over(&self) -> bool {
    self.settled_proposals.is_empty()
}
pub(crate) fn e8s_equivalent_to_be_rolled_over(&self) -> u64 {
    if self.rewards_rolled_over() {
        self.total_available_e8s_equivalent.unwrap_or_default()
    } else {
        0
    }
}
``` [5](#0-4) 

So in SNS, if proposals are settled but all voting neurons were deleted, the entire `rewards_purse_e8s` is permanently lost.

### Impact Explanation

The undistributed portion of `total_available_e8s_equivalent` — proportional to the voting power of deleted neurons — is permanently discarded. It is not credited to any neuron and not carried forward to the next reward round. Over time, this silently reduces the total maturity distributed across the system. Since maturity is the precursor to minted ICP (via neuron spawning), this represents a permanent reduction in the ICP rewards that neuron holders are entitled to receive.

The NNS test at line 3241–3248 explicitly confirms this: when neuron 999 (with 3× the voting power of neuron 2) is deleted before reward distribution, 75 of 100 available e8s are permanently lost: [6](#0-5) 

### Likelihood Explanation

The trigger is a normal, permissionless operation: any neuron holder can dissolve and disburse their neuron at any time. The window of vulnerability is the entire reward period (at least 1 day for NNS, configurable for SNS). A neuron holder who votes on a proposal and then disburses their neuron within the same reward period will silently cause their reward share to be permanently lost. No privileged access, governance majority, or external dependency is required. The automatic timer-driven reward distribution (`CalculateDistributableRewardsTask`) ensures this path is exercised every reward period without any manual trigger. [7](#0-6) 

### Recommendation

The `rewards_rolled_over()` / `e8s_equivalent_to_be_rolled_over()` logic should be updated to also roll over the undistributed portion when `distributed_e8s_equivalent < total_available_e8s_equivalent`, regardless of whether `settled_proposals` is empty. Concretely, the rollover amount should be `total_available_e8s_equivalent - distributed_e8s_equivalent` rather than `0` whenever rewards were not fully distributed. This mirrors the Velocimeter fix: preserve undistributed rewards for future eligible recipients rather than discarding them.

### Proof of Concept

1. In NNS governance, neuron 999 votes on proposal 1 with 3× the voting power of neuron 2.
2. Neuron 999 is dissolved and disbursed before the reward distribution period ends.
3. `calculate_voting_rewards` is called. `total_voting_rights` is computed from ballot data (neuron 999's ballot still exists in the proposal). `total_available_e8s_equivalent = 100`.
4. During distribution, `self.neuron_store.contains(neuron_999_id)` returns `false`. Neuron 999's reward (75 e8s) is skipped. `actually_distributed_e8s_equivalent = 25`.
5. The reward event is recorded: `settled_proposals = [proposal_1]`, `distributed_e8s_equivalent = 25`, `total_available_e8s_equivalent = 100`.
6. `rewards_rolled_over()` returns `false` (settled_proposals is non-empty). `e8s_equivalent_to_be_rolled_over()` returns `0`.
7. The next reward period starts with `rolling_over_from_previous_reward_event_e8s_equivalent = 0`. The 75 e8s are permanently lost.

```rust
#[async_trait]
impl RecurringAsyncTask for CalculateDistributableRewardsTask {
    async fn execute(self) -> (Duration, Self) {
        let total_supply = self
            .governance
            .with_borrow(|governance| governance.get_ledger())
            .total_supply()
            .await;
        match total_supply {
            Ok(total_supply) => {
                self.governance.with_borrow_mut(|governance| {
                    governance.distribute_voting_rewards_to_neurons(total_supply);
                });
            }
            Err(err) => {
                println!(
                    "{}Error when getting total ICP supply: {}",
                    LOG_PREFIX,
                    GovernanceError::from(err)
                )
            }
        }

        let next_run = self.next_reward_task_from_now();
        (next_run, self)
    }
