### Title
`require_running` in `exit_to_near_precompile_callback` blocks the refund path when the contract is paused, causing permanent loss of in-flight user funds — (File: `engine/src/contract_methods/connector.rs`)

---

### Summary

When the Aurora Engine contract is paused while a user's exit-to-NEAR operation is in-flight, the `exit_to_near_precompile_callback` function fails immediately at the `require_running` guard, blocking **both** the success path and the error-refund path. Because the user's EVM tokens were already burned before the callback fires, and the refund cannot execute, those tokens are permanently destroyed.

---

### Finding Description

The `exit_to_near_precompile_callback` function opens with an unconditional `require_running` check:

```rust
// engine/src/contract_methods/connector.rs  L196-L246
pub fn exit_to_near_precompile_callback<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I, env: &E, handler: &mut H,
) -> Result<Option<SubmitResult>, ContractError> {
    with_hashchain(io, env, function_name!(), |io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;          // ← blocks everything when paused
        env.assert_private_call()?;
        ...
        let maybe_result =
            if let Some(PromiseResult::Successful(_)) = handler.promise_result(0) {
                // success path: forward NEAR tokens
                ...
            } else if let Some(args) = args.refund {
                // refund path: restore burned tokens on error
                let refund_result = engine::refund_on_error(io, env, state, &args, handler)?;
                ...
            };
        Ok(maybe_result)
    })
}
``` [1](#0-0) 

The `require_running` guard is defined as:

```rust
pub fn require_running(state: &state::EngineState) -> Result<(), ContractError> {
    if state.is_paused {
        return Err(errors::ERR_PAUSED.into());
    }
    Ok(())
}
``` [2](#0-1) 

The `ExitToNear` precompile burns the caller's ERC-20 or ETH balance inside the EVM execution, then schedules a NEAR-side NEP-141 transfer promise. `exit_to_near_precompile_callback` is the callback of that promise. If the NEAR-side transfer fails (e.g., insufficient gas, non-existent account), the callback's `else if let Some(args) = args.refund` branch is supposed to call `engine::refund_on_error` to restore the burned tokens. However, because `require_running` fires first, the refund branch is **never reached** when the contract is paused.

`pause_contract` sets `state.is_paused = true` and is callable by the owner at any time:

```rust
pub fn pause_contract<I: IO + Copy, E: Env>(io: I, env: &E) -> Result<(), ContractError> {
    with_hashchain(io, env, function_name!(), |mut io| {
        let mut state = state::get_state(&io)?;
        require_owner_only(&state, &env.predecessor_account_id())?;
        require_running(&state)?;
        state.is_paused = true;
        state::set_state(&mut io, &state)?;
        Ok(())
    })
}
``` [3](#0-2)

---

### Impact Explanation

**Critical — Permanent freezing of funds.**

Any user who:
1. Calls the `ExitToNear` precompile (burning their EVM-side tokens), and
2. Whose NEAR-side transfer promise fails (e.g., gas exhaustion, account not registered), and
3. Whose callback executes while the contract is paused

loses their tokens permanently. The EVM burn is committed; the NEAR transfer never completes; the refund path is blocked by `require_running`; and NEAR's async model does not re-execute failed callbacks. Once the callback receipt is consumed, there is no on-chain mechanism to recover the funds.

---

### Likelihood Explanation

**Low-medium.** The window requires two independent events to coincide:

- A user's NEAR-side transfer promise fails (non-trivial but possible under gas pressure or misconfigured recipient accounts).
- The owner calls `pause_contract` in the interval between the exit precompile execution and the callback receipt execution (a single NEAR block window).

Emergency pauses are the primary intended use of `pause_contract`, and they are most likely to be triggered during periods of high activity or stress — exactly when in-flight exit operations are most common. The combination is realistic in a production incident scenario.

---

### Recommendation

Move the `require_running` guard **below** the refund path, or introduce a separate code path that allows `refund_on_error` to execute unconditionally regardless of the pause state. The refund is a pure internal accounting correction (restoring a balance that was already debited) and carries no security risk that the pause is designed to prevent. Concretely:

```rust
// Allow refund even when paused; only gate the forward-transfer path
let maybe_result =
    if let Some(args) = args.refund {
        // always allow refund regardless of pause state
        let refund_result = engine::refund_on_error(io, env, state, &args, handler)?;
        ...
    } else {
        require_running(&state)?;   // gate only the success/forward path
        ...
    };
```

---

### Proof of Concept

1. Alice calls the `ExitToNear` precompile from an EVM contract, burning 1 ETH worth of wETH ERC-20 tokens. The EVM execution succeeds; the burn is committed to Aurora state.
2. Aurora schedules a NEAR promise to transfer the equivalent NEP-141 tokens to Alice's NEAR account.
3. The NEAR-side transfer fails (e.g., Alice's NEAR account is not registered for the NEP-141 token).
4. Before the callback receipt is processed, the Aurora owner calls `pause_contract` in response to an unrelated security incident.
5. `exit_to_near_precompile_callback` is invoked as the callback. It immediately returns `Err(ERR_PAUSED)` at the `require_running` check.
6. The `refund_on_error` branch is never reached. Alice's 1 ETH of ERC-20 tokens remain burned with no corresponding NEAR-side credit and no refund.
7. Even after the owner calls `resume_contract`, the callback receipt has already been consumed; there is no mechanism to replay it. Alice's funds are permanently lost.

pub fn exit_to_near_precompile_callback<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<Option<SubmitResult>, ContractError> {
    with_hashchain(io, env, function_name!(), |io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;
        env.assert_private_call()?;

        // This function should only be called as the callback of
        // exactly one promise.
        if handler.promise_results_count() != 1 {
            return Err(errors::ERR_PROMISE_COUNT.into());
        }

        let args: ExitToNearPrecompileCallbackArgs = io.read_input_borsh()?;

        let maybe_result = if let Some(PromiseResult::Successful(_)) = handler.promise_result(0) {
            if let Some(args) = args.transfer_near {
                let action = PromiseAction::Transfer {
                    amount: Yocto::new(args.amount),
                };
                let promise = PromiseBatchAction {
                    target_account_id: args.target_account_id,
                    actions: vec![action],
                };

                // Safety: this call is safe because it comes from the exit to near precompile, not users.
                // The call is to transfer the unwrapped wNEAR tokens.
                let promise_id = handler.promise_create_batch(&promise);
                handler.promise_return(promise_id);
            }

            None
        } else if let Some(args) = args.refund {
            // Exit call failed; need to refund tokens
            let refund_result = engine::refund_on_error(io, env, state, &args, handler)?;

            if !refund_result.status.is_ok() {
                return Err(errors::ERR_REFUND_FAILURE.into());
            }

            Some(refund_result)
        } else {
            None
        };

        Ok(maybe_result)
    })
}
