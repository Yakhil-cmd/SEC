Audit Report

## Title
Missing Minimum Fee Floor in `DogecoinFeeEstimator::estimate_nth_fee()` Causes Stuck Withdrawals - (File: rs/dogecoin/ckdoge/minter/src/fees/mod.rs)

## Summary

`DogecoinFeeEstimator::estimate_nth_fee()` returns raw fee percentile values with no minimum floor, unlike `BitcoinFeeEstimator::estimate_nth_fee()` which was patched after a confirmed production incident on 2025-06-21 to enforce a `minimum_fee_per_vbyte()` floor. If the Dogecoin canister reports anomalously low fee percentiles, the ckDOGE minter will build and sign withdrawal transactions with a fee rate below the Dogecoin relay minimum, causing them to never be mined. Because the ckDOGE ledger burn is final at the time of the withdrawal request, affected users permanently lose access to their DOGE until a governance-approved minter upgrade is deployed.

## Finding Description

`BitcoinFeeEstimator::estimate_nth_fee()` applies a hardcoded minimum floor:

```rust
// rs/bitcoin/ckbtc/minter/src/fees/mod.rs lines 99-113
median_fee.map(|f| f.max(self.minimum_fee_per_vbyte()))
```

The `minimum_fee_per_vbyte()` method returns 1,500 millisat/vbyte for Mainnet and 1,000 for Testnet:

```rust
// rs/bitcoin/ckbtc/minter/src/fees/mod.rs lines 80-87
const fn minimum_fee_per_vbyte(&self) -> FeeRate {
    let rate = match &self.network {
        Network::Mainnet => 1_500,
        Network::Testnet => 1_000,
        Network::Regtest => 0,
    };
    FeeRate::from_millis_per_byte(rate)
}
```

This floor was introduced after the 2025-06-21 incident where the ckBTC minter selected 142 millisatoshis/vbyte (actual median ~2,000), documented in `rs/bitcoin/ckbtc/mainnet/minter_upgrade_2025_06_27.md`:

> "A stop-gap solution was introduced in #5742, to ensure that the fee per vbyte computed by the minter is always at least 1.5 sats/vbyte (for Bitcoin Mainnet)."

`DogecoinFeeEstimator::estimate_nth_fee()` has no equivalent protection:

```rust
// rs/dogecoin/ckdoge/minter/src/fees/mod.rs lines 53-66
fn estimate_nth_fee(&self, fee_percentiles: &[FeeRate], nth: usize) -> Option<FeeRate> {
    match &self.network {
        Network::Mainnet => {
            if fee_percentiles.len() < 100 || nth >= 100 {
                return None;
            }
            Some(fee_percentiles[nth])  // raw value, no floor
        }
        Network::Regtest => Some(DEFAULT_REGTEST_FEE),
    }
}
```

The returned `FeeRate` flows directly into the shared `ic_ckbtc_minter` fee estimation infrastructure, which stores it as `last_median_fee_per_vbyte` and uses it to compute transaction fees for all outgoing Dogecoin withdrawal transactions. The `fee_based_minimum_withdrawal_amount` in `DogecoinFeeEstimator` also receives this unclamped value, meaning the minimum withdrawal guard is also computed from a potentially near-zero fee rate.

The ckDOGE minter is confirmed deployed on mainnet at canister `eqltq-xqaaa-aaaar-qb3vq-cai` since 2026-01-30, with multiple subsequent upgrades.

## Impact Explanation

This matches the **High** impact class: "Significant Chain Fusion, ck-token, ledger, Rosetta, boundary/API, XRC, Internet Identity, NNS, SNS, or infrastructure security impact with concrete user or protocol harm."

Any `retrieve_doge_with_approval` request processed during a period of anomalously low reported fee percentiles will produce a signed Dogecoin transaction with a fee rate below the Dogecoin relay minimum (~1 koinu/byte = 1,000 millikoinus/byte). Dogecoin nodes will not relay or mine such transactions. The ckDOGE ledger burn is final; the user's DOGE is permanently inaccessible until a governance upgrade is deployed. The `MIN_RELAY_FEE_RATE_INCREASE` RBF bump of 100,000 millikoinus/byte applied on top of a near-zero stored fee rate may still produce a transaction below the relay threshold, compounding the stuck state.

## Likelihood Explanation

No attacker action is required. The identical failure mode already occurred in production for ckBTC on 2025-06-21 under natural network conditions. The ckDOGE minter reuses the same shared `estimate_fee_per_vbyte` infrastructure from `ic_ckbtc_minter` but does not inherit the fix applied in PR #5742. The Dogecoin fee market is thinner and more volatile than Bitcoin's, making anomalously low fee percentile reports at least as likely. Any ordinary user initiating a withdrawal during such a period triggers the impact with no special privileges required.

## Recommendation

Add a `minimum_fee_per_byte()` method to `DogecoinFeeEstimator` analogous to `BitcoinFeeEstimator::minimum_fee_per_vbyte()`, and apply it as a floor in `estimate_nth_fee()`:

```rust
const fn minimum_fee_per_byte(&self) -> FeeRate {
    let rate = match &self.network {
        Network::Mainnet => 1_000, // 1 koinu/byte = 1,000 millikoinus/byte (Dogecoin relay minimum)
        Network::Regtest => 0,
    };
    FeeRate::from_millis_per_byte(rate)
}

fn estimate_nth_fee(&self, fee_percentiles: &[FeeRate], nth: usize) -> Option<FeeRate> {
    // ...
    Some(fee_percentiles[nth]).map(|f| f.max(self.minimum_fee_per_byte()))
}
```

The minimum value should be calibrated to Dogecoin's relay fee policy (currently 1 koinu/byte).

## Proof of Concept

1. The Dogecoin Bitcoin canister returns 100 fee percentile entries all set to `FeeRate::from_millis_per_byte(50)` (0.05 koinu/byte, below the Dogecoin relay minimum of 1 koinu/byte).
2. `DogecoinFeeEstimator::estimate_nth_fee(&percentiles, 50)` returns `Some(FeeRate(50))` — no floor is applied. [1](#0-0) 
3. The shared `estimate_fee_per_vbyte` infrastructure stores `last_median_fee_per_vbyte = Some(FeeRate(50))`.
4. A user calls `retrieve_doge_with_approval`; the minter builds and signs a Dogecoin transaction using `fee_rate = FeeRate(50)`.
5. The transaction is broadcast but rejected by all Dogecoin relay nodes as below the minimum relay fee.
6. The ckDOGE ledger burn is final; the user's DOGE is permanently inaccessible until a governance upgrade.

Contrast with the fixed ckBTC path: [2](#0-1)
