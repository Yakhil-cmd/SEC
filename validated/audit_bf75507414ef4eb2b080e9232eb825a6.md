Audit Report

## Title
`StableNeuronStore::delete` Fails to Clear `maturity_disbursements_map`, Enabling Permanent DoS of the Maturity Disbursement Finalization Timer — (`rs/nns/governance/src/storage/neurons.rs`)

## Summary

`StableNeuronStore::delete` clears every auxiliary stable map except `maturity_disbursements_map`, leaving orphaned `(neuron_id, index)` entries after a neuron is removed. Because `disburse_neuron` imposes no guard against pending maturity disbursements, an attacker can initiate `DisburseMaturity`, then dissolve and disburse the neuron's stake, producing orphaned entries in both `maturity_disbursements_map` and the `MaturityDisbursementIndex`. The finalization timer then permanently loops on `NeuronNotFound` for the deleted neuron, starving all other users' maturity disbursement finalizations.

## Finding Description

**Root cause — missing cleanup in `StableNeuronStore::delete`:**

`rs/nns/governance/src/storage/neurons.rs` lines 411–431 show that `delete` clears `hot_keys_map`, `recent_ballots_map`, `followees_map`, `known_neuron_data_map`, and `transfer_map`, but never touches `maturity_disbursements_map` (declared at lines 125–126 as `StableBTreeMap<(NeuronId, u64), MaturityDisbursement, Memory>`):

```rust
update_repeated_field(neuron_id, vec![], &mut self.hot_keys_map);
update_repeated_field(neuron_id, vec![], &mut self.recent_ballots_map);
self.update_followees(neuron_id, hashmap![]);
update_singleton_field(neuron_id, None, &mut self.known_neuron_data_map);
update_singleton_field(neuron_id, None, &mut self.transfer_map);
// maturity_disbursements_map: never touched
```

The `MaturityDisbursementIndex` (a separate file, `maturity_disbursement_index.rs`, distinct from `neuron_indexes.rs`) is similarly not cleaned up on neuron deletion, leaving the index entry pointing to a now-deleted neuron.

**Enabling condition — `disburse_neuron` has no guard for pending maturity disbursements:**

`rs/nns/governance/src/governance.rs` lines 1970–1995 show the only precondition checks are: caller is controller, neuron state is `Dissolved`, and KYC verified. There is no check for `maturity_disbursements_in_progress`. A neuron with a pending maturity disbursement can be fully deleted via `Disburse`.

**Finalization timer loops forever on orphaned entries:**

`rs/nns/governance/src/governance/disburse_maturity.rs` lines 462–478 show `next_maturity_disbursement_to_finalize` reads from the `MaturityDisbursementIndex` to find neuron IDs ready to finalize, then calls `neuron_store.with_neuron(&neuron_id, ...)`. For a deleted neuron this returns `NeuronStoreError::NeuronNotFound`, mapped to `FinalizeMaturityDisbursementError::NeuronNotFound`.

Lines 544–554 show that on any error, `finalize_maturity_disbursement` logs the error and returns `RETRY_INTERVAL`. The timer reschedules itself, finds the same orphaned index entry, fails again, and loops indefinitely — permanently blocking all other users' maturity disbursement finalizations.

## Impact Explanation

This is a **High** severity application/platform-level DoS. The maturity disbursement finalization timer processes one disbursement per invocation. A single orphaned `MaturityDisbursementIndex` entry for a deleted neuron causes the timer to retry indefinitely on `NeuronNotFound`, starving all legitimate pending disbursements for all NNS governance users. This matches the allowed impact: *Application/platform-level DoS, crash, consensus blocking, certified-state disruption, or subnet availability impact not based on raw volumetric DDoS* ($2,000–$10,000).

## Likelihood Explanation

The attack requires only standard, unprivileged neuron operations available to any neuron owner:
1. Own a neuron with maturity ≥ `MINIMUM_DISBURSEMENT_E8S`.
2. Call `DisburseMaturity` (standard ingress).
3. Start dissolving; wait for the dissolve delay to pass.
4. Call `Disburse` (standard ingress on a dissolved neuron).

No privileged access, key compromise, governance majority, or social engineering is required. The attack is repeatable and cheap.

## Recommendation

1. **Fix `StableNeuronStore::delete`**: Add the missing cleanup:
   ```rust
   update_repeated_field(neuron_id, vec![], &mut self.maturity_disbursements_map);
   ```
2. **Guard `disburse_neuron` against pending maturity disbursements**: Before allowing stake disbursal (which deletes the neuron), check `neuron.maturity_disbursements_in_progress.is_empty()` and return a `PreconditionFailed` error if not.
3. **Make the finalization timer self-healing**: If `NeuronNotFound` is returned for an index entry, remove the stale entry from the `MaturityDisbursementIndex` rather than retrying indefinitely.

## Proof of Concept

State-machine test sequence:
1. Create a neuron with maturity ≥ `MINIMUM_DISBURSEMENT_E8S` and a dissolve delay.
2. Call `DisburseMaturity` — assert `maturity_disbursements_in_progress` has one entry and `MaturityDisbursementIndex` has one entry.
3. Start dissolving; advance time past the dissolve delay.
4. Call `Disburse` (stake) — neuron is deleted.
5. Assert `maturity_disbursements_map` still contains the orphaned `(neuron_id, 0)` entry (confirms storage-layer bug).
6. Assert `MaturityDisbursementIndex` still contains the orphaned entry (confirms index not cleaned up).
7. Advance time past `finalize_disbursement_timestamp_seconds`.
8. Run the finalization timer — assert it returns `RETRY_INTERVAL` (error path, no ICP minted).
9. Run the timer again — assert it still returns `RETRY_INTERVAL` (infinite loop confirmed).
10. Assert no other pending maturity disbursements for other neurons are processed (DoS confirmed).
