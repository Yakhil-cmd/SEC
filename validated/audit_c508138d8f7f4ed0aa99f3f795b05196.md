Audit Report

## Title
NNS Governance `disburse_neuron` Unconditionally Zeroes `neuron_fees_e8s` Without Burning — Fee Tokens Permanently Stranded in Neuron Subaccount - (File: `rs/nns/governance/src/governance.rs`)

## Summary
In `disburse_neuron`, the ledger burn of `neuron_fees_e8s` is correctly guarded by `fees_amount_e8s > transaction_fee_e8s`, but the governance state update (`neuron_fees_e8s = 0` and `cached_neuron_stake_e8s -= fees_amount_e8s`) executes unconditionally outside that branch. When fees fall at or below the transaction fee threshold, the burn is skipped but governance records it as complete, leaving `fees_amount_e8s` tokens permanently stranded in the neuron's ICP ledger subaccount with no recovery path.

## Finding Description
In `rs/nns/governance/src/governance.rs`, the fee-burn transfer is conditional:

```rust
if fees_amount_e8s > transaction_fee_e8s {
    self.ledger.transfer_funds(fees_amount_e8s, 0, Some(neuron_subaccount),
        governance_minting_account(), now).await?;
}
```

Immediately after, unconditionally:

```rust
self.with_neuron_mut(id, |neuron| {
    if neuron.cached_neuron_stake_e8s > fees_amount_e8s {
        neuron.cached_neuron_stake_e8s -= fees_amount_e8s;
    } else {
        neuron.cached_neuron_stake_e8s = 0;
    }
    neuron.neuron_fees_e8s = 0;   // ← always zeroed
})
```

Transfer 2 then sends `disburse_amount_e8s` (computed as `cached_stake - fees - tx_fee`) to the user, debiting `disburse_amount_e8s + transaction_fee_e8s` from the subaccount. A second `with_neuron_mut` reduces `cached_neuron_stake_e8s` by that same amount, reaching 0. The subaccount retains exactly `fees_amount_e8s` tokens. Governance records `cached_neuron_stake_e8s = 0` and `neuron_fees_e8s = 0`, believing the neuron is fully empty. No governance code path will ever issue another transfer from that subaccount.

The SNS governance canister identified and fixed this exact pattern (proposal 137687, 2025-08-01): the state update is placed inside the burn branch, with an explicit comment noting that updating outside the branch causes ledger/governance desync. The NNS governance has not received the equivalent fix.

## Impact Explanation
For every NNS neuron disbursed while holding `0 < neuron_fees_e8s <= transaction_fee_e8s` (1–10,000 e8s, up to 0.0001 ICP), the fee tokens are permanently locked in the neuron's ICP ledger subaccount. The governance canister controls those subaccounts but has no code path to recover the stranded balance. This is a permanent, irrecoverable loss of user funds caused by a governance accounting bug in in-scope NNS production code. The per-neuron loss is bounded at 0.0001 ICP; the impact class is **Medium** — permanent user-funds loss with constrained per-instance magnitude, triggerable without privilege.

## Likelihood Explanation
The condition `fees_amount_e8s <= transaction_fee_e8s` requires accumulated fees of 1–10,000 e8s. The current NNS `reject_cost_e8s` is 10 ICP (1,000,000,000 e8s), far above the threshold, making this rare under normal operation. However, it is reachable if a neuron accumulated fees during a period when `reject_cost_e8s` was lower, or if partial fee refunds (from accepted follow-up proposals) reduced `neuron_fees_e8s` to a sub-threshold value before disbursement. Any dissolved NNS neuron holder can trigger this path by calling `disburse_neuron` — no privileged access is required.

## Recommendation
Mirror the SNS governance fix: move the `neuron_fees_e8s` and corresponding `cached_neuron_stake_e8s` update inside the `if fees_amount_e8s > transaction_fee_e8s` branch, so governance state is only modified when the ledger burn actually executes:

```rust
if fees_amount_e8s > transaction_fee_e8s {
    self.ledger.transfer_funds(fees_amount_e8s, 0, Some(neuron_subaccount),
        governance_minting_account(), now).await?;

    self.with_neuron_mut(id, |neuron| {
        neuron.cached_neuron_stake_e8s =
            neuron.cached_neuron_stake_e8s.saturating_sub(fees_amount_e8s);
        neuron.neuron_fees_e8s = 0;
    }).expect("Expected the parent neuron to exist");
}
// else: leave neuron_fees_e8s unchanged; the tokens remain in the subaccount
//       and will be disbursed to the user as part of the stake transfer.
```

## Proof of Concept
**Setup:** NNS neuron with `cached_neuron_stake_e8s = 1_000_000`, `neuron_fees_e8s = 5_000`, `transaction_fee_e8s = 10_000`.

**Step 1 — disburse amount computed:** `disburse_amount_e8s = (1_000_000 - 5_000) - 10_000 = 985_000`

**Step 2 — burn skipped** (`5_000 <= 10_000`): no ledger call.

**Step 3 — governance state updated unconditionally:** `cached_neuron_stake_e8s = 995_000`, `neuron_fees_e8s = 0`

**Step 4 — stake transfer to user:** Ledger debits `985_000 + 10_000 = 995_000` from the subaccount. Second `with_neuron_mut` sets `cached_neuron_stake_e8s = 0`.

**Result:**
- ICP ledger subaccount balance: `1_000_000 - 995_000 = 5_000` ← permanently stranded
- Governance: `cached_neuron_stake_e8s = 0`, `neuron_fees_e8s = 0` ← believes neuron is empty
- No governance function can issue a transfer from this subaccount again

A deterministic unit test can reproduce this by constructing a mock ledger, setting `neuron_fees_e8s = 5_000` and `cached_neuron_stake_e8s = 1_000_000`, calling `disburse_neuron`, and asserting that the mock subaccount retains 5,000 e8s while governance records 0.
