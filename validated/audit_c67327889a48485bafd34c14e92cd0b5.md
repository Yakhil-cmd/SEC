### Title
Unhalting a Subnet via `do_update_subnet` Does Not Reset `halt_at_cup_height`, Leaving the Subnet Permanently Halted - (File: `rs/registry/canister/src/mutations/do_update_subnet.rs`)

### Summary

The Registry canister's `do_update_subnet` function allows a governance proposal to set `is_halted = false` (intending to resume a subnet) without checking or resetting the independent `halt_at_cup_height` flag. Because the consensus layer's `should_halt` function gates block production on `halt_at_cup_height` — not on `is_halted` — a subnet with `halt_at_cup_height = true` will re-halt at the next CUP height regardless of the `is_halted` value, silently defeating the governance proposal's intent.

### Finding Description

The `SubnetRecord` protobuf contains two independent halt-related flags:

- `is_halted` — "the subnet will be halted: it will no longer create or execute blocks"
- `halt_at_cup_height` — "the subnet will be halted after reaching the next cup height … this flag is reset automatically when a new CUP proposal is approved. When that happens, the `is_halted` flag is set to `true`" [1](#0-0) 

The consensus layer's `should_halt` function reads **only** `halt_at_cup_height` (via `get_halt_at_cup_height`) to decide whether to stop producing blocks: [2](#0-1) 

The `do_recover_subnet` function correctly handles the `halt_at_cup_height → is_halted` transition atomically: [3](#0-2) 

However, `do_update_subnet` applies `is_halted` and `halt_at_cup_height` as fully independent, optional fields via `merge_subnet_record`: [4](#0-3) 

There is no validation in `do_update_subnet` that enforces: *"if `is_halted` is being set to `false`, then `halt_at_cup_height` must also be `false`."* [5](#0-4) 

**Vulnerable scenario:**

1. NNS governance passes a proposal: `do_update_subnet` with `halt_at_cup_height = Some(true)`. The subnet halts at the next CUP height.
2. Without going through `do_recover_subnet` (which would atomically flip `halt_at_cup_height = false, is_halted = true`), governance passes a second proposal: `do_update_subnet` with `is_halted = Some(false)` and `halt_at_cup_height = None`.
3. `merge_subnet_record` sets `is_halted = false` but leaves `halt_at_cup_height = true` unchanged.
4. Consensus calls `should_halt`, which reads `halt_at_cup_height = true` and returns `Some(true)`, keeping the subnet halted indefinitely. [6](#0-5) 

The same gap exists in `do_set_subnet_operational_level`, which sets `is_halted` based on an operational level enum but never touches `halt_at_cup_height`: [7](#0-6) 

### Impact Explanation

A subnet stuck with `halt_at_cup_height = true` stops producing blocks and delivering batches. All canisters on that subnet become unreachable: ingress messages are rejected, inter-canister calls time out, and chain-key signing requests stall. The subnet is effectively bricked until a further governance proposal explicitly resets `halt_at_cup_height = false`. Because the unhalting proposal appears to succeed (no error is returned), operators may not immediately diagnose the root cause, prolonging the outage.

### Likelihood Explanation

The scenario requires two sequential NNS governance proposals, both of which pass legitimately. The second proposal omits `halt_at_cup_height = None` (the default when the field is not explicitly set), which is the natural behavior of tooling that only sets the field it intends to change. The `ic-admin` CLI constructs `UpdateSubnetPayload` with independent optional fields, making it straightforward to submit a proposal that sets only `is_halted` without touching `halt_at_cup_height`. The `do_recover_subnet` path is not always used (it requires a full DKG re-run), so operators may attempt the simpler `do_update_subnet` path instead.

### Recommendation

In `do_update_subnet` (and `do_set_subnet_operational_level`), add a consistency check: if `is_halted` is being set to `false`, automatically reset `halt_at_cup_height` to `false` as well (or reject the payload if `halt_at_cup_height` is currently `true` and not also being cleared). Mirroring the logic already present in `do_recover_subnet`:

```rust
// In merge_subnet_record or do_update_subnet:
if let Some(false) = is_halted {
    // Clearing is_halted must also clear halt_at_cup_height,
    // otherwise consensus will re-halt the subnet at the next CUP.
    subnet_record.halt_at_cup_height = false;
}
maybe_set!(subnet_record, is_halted);
maybe_set!(subnet_record, halt_at_cup_height);
```

### Proof of Concept

**State before:**
```
subnet_record.is_halted          = false
subnet_record.halt_at_cup_height = true   ← set by a prior proposal
```

**Governance proposal 1 (halt):** `UpdateSubnetPayload { halt_at_cup_height: Some(true), .. }`
→ `merge_subnet_record` sets `halt_at_cup_height = true`. Consensus `should_halt` returns `true`. Subnet halts.

**Governance proposal 2 (intended unhalt):** `UpdateSubnetPayload { is_halted: Some(false), halt_at_cup_height: None, .. }`
→ `merge_subnet_record` sets `is_halted = false` but `halt_at_cup_height` remains `true`.

**Consensus check after proposal 2:**
```rust
// rs/consensus/src/consensus/status.rs:71-83
let should_halt_by_subnet_record = registry_client
    .get_halt_at_cup_height(subnet_id, registry_version) // → Some(true)
    ...;
match (upgrading, should_halt_by_subnet_record) {
    (_, Some(true)) => Some(true),  // ← subnet still halts
    ...
}
```

The subnet remains halted despite the governance proposal intending to resume it. [2](#0-1)

pub(crate) fn should_halt(
    height: Height,
    registry_client: &(impl RegistryClient + ?Sized),
    subnet_id: SubnetId,
    pool: &PoolReader<'_>,
    logger: &ReplicaLogger,
) -> Option<bool> {
    let registry_version = pool.registry_version(height).warn_if_none(
        logger,
        format!("Failed to get the registry version at height {height}"),
    )?;

    let upgrading = lookup_replica_version(registry_client, subnet_id, logger, registry_version)
        .map(|replica_version| replica_version != ReplicaVersion::default())
        .warn_if_none(logger, "Failed to check if the upgrade is pending!");

    let should_halt_by_subnet_record = registry_client
        .get_halt_at_cup_height(subnet_id, registry_version)
        .ok()
        .flatten()
        .warn_if_none(
            logger,
            format!(
                "Failed to check if the registry version at height {height} instructs the subnet to halt!",
            ),
        );

    match (upgrading, should_halt_by_subnet_record) {
        (Some(true), _) | (_, Some(true)) => Some(true),
        (Some(false), Some(false)) => Some(false),
        (_, _) => None,
    }
}

    pub fn do_update_subnet(&mut self, caller: PrincipalId, payload: UpdateSubnetPayload) {
        println!("{LOG_PREFIX}do_update_subnet: caller={caller}, payload={payload:?}");

        let subnet_id = payload.subnet_id;

        // The engine controller canister is only allowed to mutate CloudEngine
        // subnets, and only a small subset of fields. Other authorized callers
        // (governance) can update any subnet and any field.
        if caller == ENGINE_CONTROLLER_CANISTER_ID.get() {
            let subnet_record = self.get_subnet_or_panic(subnet_id);
            assert_eq!(
                subnet_record.subnet_type,
                i32::from(SubnetTypePb::CloudEngine),
                "{LOG_PREFIX}do_update_subnet: engine controller may only update CloudEngine \
                 subnets; subnet {subnet_id} has subnet_type {:?}",
                subnet_record.subnet_type,
            );
            ensure_engine_controller_payload_scope(&payload);
        }

        self.validate_update_payload_chain_key_config(&payload);
        self.validate_update_sev_feature(&payload);

        let new_subnet_record =
            merge_subnet_record(self.get_subnet_or_panic(subnet_id), payload.clone());

        let subnet_record_mutation = upsert(
            make_subnet_record_key(subnet_id).into_bytes(),
            new_subnet_record.encode_to_vec(),
        );

        let mut mutations = vec![subnet_record_mutation];

        if let Some(chain_key_signing_enable) = payload.chain_key_signing_enable {
            mutations.append(
                &mut self.mutations_to_enable_chain_key(subnet_id, &chain_key_signing_enable),
            );
        }

        let chain_key_signing_disable = payload.chain_key_signing_disable;

        if let Some(chain_key_signing_disable) = chain_key_signing_disable {
            mutations.append(
                &mut self
                    .mutations_to_disable_subnet_chain_key(subnet_id, &chain_key_signing_disable),
            );
        }

        // Check invariants before applying mutations
        self.maybe_apply_mutation_internal(mutations);
    }
