Audit Report

## Title
Unprivileged Caller Can Force-Register Any Principal as SNS Swap Buyer via `refresh_buyer_tokens` — (`rs/sns/swap/canister/canister.rs`)

## Summary
The `refresh_buyer_tokens` canister endpoint accepts an arbitrary `buyer` principal in its request with no verification that it matches the caller. An unprivileged attacker can register any victim principal as a swap buyer, which blocks the victim's `error_refund_icp` recovery path and, if the swap commits, permanently converts the victim's accidentally-sent ICP into unwanted SNS neurons without their consent.

## Finding Description
In `rs/sns/swap/canister/canister.rs` at lines 130–134, when `arg.buyer` is non-empty, the parsed principal is used directly as the buyer identity with no assertion that it equals `caller_principal_id()`:

```rust
let p: PrincipalId = if arg.buyer.is_empty() {
    caller_principal_id()
} else {
    PrincipalId::from_str(&arg.buyer).unwrap()  // no caller == buyer check
};
``` [1](#0-0) 

This principal is passed directly to `refresh_buyer_token_e8s`, which queries the ICP ledger balance of `principal_to_subaccount(&buyer)` on the swap canister and, if it meets `min_participant_icp_e8s`, inserts a `BuyerState` entry for the victim: [2](#0-1) [3](#0-2) 

`BuyerState::new(0)` initializes `transfer_success_timestamp_seconds` to `0`: [4](#0-3) 

Once the victim is registered, `error_refund_icp` checks for a `BuyerState` entry with `transfer_success_timestamp_seconds == 0` and returns a precondition error, blocking the victim's self-rescue: [5](#0-4) 

The ticket system (lines 1249–1272) does not prevent this attack: if no open ticket exists for the victim, the code explicitly skips the ticket check entirely ("the payment flow will simply ignore the ticket"): [6](#0-5) 

The `confirmation_text` mitigation is ineffective because `get_init` is an unauthenticated `#[query]` endpoint — any caller can read the confirmation text and supply it: [7](#0-6) 

## Impact Explanation
If the swap **commits**: after `sweep_icp` runs, the victim's ICP is transferred to SNS governance and the victim receives SNS neurons they never consented to acquire. Their ICP is permanently converted into illiquid governance tokens without authorization. This constitutes unauthorized access to and permanent loss of user funds, matching the **High** impact class: "Unauthorized access to neurons, governance assets, wallets, identities, ledgers, or canister-controlled funds." If the amounts involved are large (e.g., a whale accidentally sends a large ICP amount), the impact could reach the Critical threshold for theft/permanent loss.

If the swap **aborts**: the victim's ICP is eventually returned after `sweep_icp`, but the victim is denied the ability to self-rescue via `error_refund_icp` during the window between swap close and `sweep_icp` completion.

## Likelihood Explanation
- The attack requires only that the victim has accidentally sent ICP to their swap subaccount — a realistic user error given the subaccount-based participation model, and the explicit reason `error_refund_icp` exists.
- The attacker needs no special privileges — only the ability to send an ingress message to the swap canister.
- The confirmation text (if set) is publicly readable from `get_init`, providing no real barrier.
- The attack window is the entire OPEN lifecycle of the swap.
- The attack is repeatable for any victim who has accidentally sent ICP to their subaccount.

## Recommendation
In the canister endpoint, enforce that the `buyer` field, if provided, must equal the caller:

```rust
let p: PrincipalId = if arg.buyer.is_empty() {
    caller_principal_id()
} else {
    let specified = PrincipalId::from_str(&arg.buyer).unwrap();
    assert_eq!(specified, caller_principal_id(), "buyer field must match the caller");
    specified
};
``` [8](#0-7) 

Alternatively, remove the `buyer` field from `RefreshBuyerTokensRequest` entirely and always use `caller_principal_id()`.

## Proof of Concept
1. Victim accidentally sends `>= min_participant_icp_e8s` ICP to `swap_subaccount(victim_principal)` on the ICP ledger.
2. Attacker (any principal) reads the confirmation text via `get_init` (if set), then calls:
   ```
   refresh_buyer_tokens(RefreshBuyerTokensRequest {
       buyer: victim_principal.to_string(),
       confirmation_text: <read from get_init>,
   })
   ```
3. The swap canister queries the victim's subaccount balance, finds sufficient ICP, and inserts `BuyerState { icp: TransferableAmount { amount_e8s: X, transfer_success_timestamp_seconds: 0, ... } }` for the victim.
4. Victim calls `error_refund_icp` — blocked by the `transfer_success_timestamp_seconds == 0` guard at swap.rs:1950–1960.
5. Swap closes COMMITTED. `sweep_icp` runs, transferring victim's ICP to SNS governance. Victim receives unwanted SNS neurons; ICP is permanently converted.

A deterministic integration test can be written using PocketIC: initialize a swap canister, have a test principal send ICP to a victim's subaccount, call `refresh_buyer_tokens` from a different (attacker) principal with the victim's ID, verify the victim's `BuyerState` is created, then verify `error_refund_icp` returns a precondition error for the victim.
