### Title
Silo `fixed_gas` enforcement bypassed via `max_fee_per_gas = 0` in EIP-1559 transactions, relayer reward permanently zeroed — (`engine/src/engine.rs`)

### Summary
When a silo has `fixed_gas` configured, any sender can submit an EIP-1559 transaction with `max_fee_per_gas = 0`. Because Aurora's `block_base_fee_per_gas` is always 0, `effective_gas_price` resolves to 0, `prepaid_amount` is 0, no balance is deducted, and `refund_unused_gas` returns early without crediting the relayer. The `fixed_gas` parameter only substitutes the *quantity* of gas in the multiplication; it provides no floor on the *price*. The silo invariant — that every transaction pays `fixed_gas × gas_price` to the relayer — is completely broken.

### Finding Description

**Root cause — `charge_gas`** [1](#0-0) 

The guard that skips gas charging requires `fixed_gas.is_none()`. When `fixed_gas` is `Some`, the guard is bypassed and execution falls through to price computation. However, `effective_gas_price` is derived solely from `max_fee_per_gas` and `block_base_fee_per_gas`: [2](#0-1) 

With `max_fee_per_gas = 0` and `block_base_fee_per_gas = 0` (Aurora constant), `effective_gas_price = 0`. The `fixed_gas` value is then multiplied by this zero price: [3](#0-2) 

Result: `prepaid_amount = 0`, sender balance unchanged.

**Root cause — `refund_unused_gas`** [4](#0-3) 

The function returns immediately when `effective_gas_price` is zero, so the relayer receives no reward regardless of `fixed_gas`.

**Pre-submission validation gap** — the only `fixed_gas`-related check is: [5](#0-4) 

There is no check that `max_fee_per_gas > 0` when `fixed_gas` is set.

### Impact Explanation

Every transaction in silo mode is supposed to cost the sender `fixed_gas × gas_price` and credit that amount to the relayer. [6](#0-5) 

By setting `max_fee_per_gas = 0`, an attacker executes transactions at zero cost and the relayer's expected reward (unclaimed yield) is never credited. The relayer processes real NEAR compute but receives nothing. This can be repeated for every transaction, permanently denying the relayer of all yield from silo operations.

**Impact: High — Theft of unclaimed yield** (relayer reward is the yield the silo model guarantees per transaction).

### Likelihood Explanation

- No special privilege required; any whitelisted sender (or any sender if the whitelist is disabled) can craft this transaction.
- EIP-1559 transactions with `max_fee_per_gas = 0` are structurally valid and pass all existing validation checks.
- Aurora's `block_base_fee_per_gas` is always 0, making this trivially reproducible on any Aurora silo deployment.
- The attacker loses nothing (zero gas cost) and can repeat indefinitely.

### Recommendation

In `charge_gas`, when `fixed_gas.is_some()`, enforce a minimum effective gas price. Concretely, after computing `effective_gas_price`, add:

```rust
if fixed_gas.is_some() && effective_gas_price.is_zero() {
    return Err(GasPaymentError::MaxFeePerGasLessThanBaseFee);
    // or a new error variant: ZeroGasPriceInSiloMode
}
```

Alternatively, reject the transaction earlier in `submit_transaction` when `fixed_gas.is_some() && transaction.max_fee_per_gas.is_zero()`.
