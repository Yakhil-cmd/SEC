### Title
Unbounded Tokio Task Spawning via Oversized `StrippedBlockProposal` in P2P Artifact Assembly — (`rs/p2p/artifact_downloader/src/fetch_stripped_artifact/assembler.rs`)

---

### Summary

`HASHES_IN_BLOCKS_ENABLED` is `true` in production. A single Byzantine peer (below the fault threshold) can push a crafted `StrippedBlockProposal` proto containing an arbitrarily large `ingress_messages` or `stripped_idkg_dealings` repeated field. The receiver's `assemble_message` spawns one unbounded Tokio task per entry with no count guard, exhausting memory and scheduler capacity before the bouncer fires.

---

### Finding Description

**Feature is active in production:**

`HASHES_IN_BLOCKS_ENABLED = true` is a compile-time constant. [1](#0-0) 

This causes `FetchStrippedConsensusArtifact` to be used as the consensus assembler. [2](#0-1) 

**No count limit in deserialization:**

`TryFrom<pb::StrippedBlockProposal>` iterates over `value.ingress_messages` and `value.stripped_idkg_dealings` with zero count validation. The proto fields are unbounded `repeated` fields. [3](#0-2) 

The proto schema confirms no `max_count` constraint: [4](#0-3) 

**Unbounded task spawning:**

After the outer bouncer check passes (on the `StrippedConsensusMessageId`, not the content), `assemble_message` calls `missing_stripped_messages()` which returns all N entries, then spawns one `get_or_fetch` task per entry with no cap: [5](#0-4) 

**The comment is not enforced:**

The comment "We can have at most 1000 elements in the vector" at line 440 is aspirational, not a runtime check: [6](#0-5) 

The actual protocol limit `MAX_INGRESS_MESSAGES_PER_BLOCK = 1000` is enforced only at consensus validation time (after assembly), not at P2P receipt time: [7](#0-6) [8](#0-7) 

**Bouncer fires after tasks are already spawned:**

The bouncer watcher is set up at line 233, *after* all N tasks are spawned at lines 214–226. The `select!` loop at line 237 can only abort tasks that are already running: [9](#0-8) 

The bouncer refresh period is 3 seconds: [10](#0-9) 

**Each spawned task is long-lived:**

`get_or_fetch` checks the local pool (brief read lock), then if not found, calls `download_stripped_message` which loops forever with exponential backoff (5 s to 120 s), never returning until a valid response arrives: [11](#0-10) 

---

### Impact Explanation

A Byzantine peer crafts a `StrippedBlockProposal` proto with N = 10^5 `StrippedIngressMessage` entries (each ~50 bytes; total wire size ~5 MB). The receiver:

1. Passes the outer bouncer check (the `StrippedConsensusMessageId` is just a height + hash, trivially forgeable within the consensus window).
2. Deserializes the proto — no count check.
3. Spawns 10^5 `get_or_fetch` Tokio tasks synchronously.
4. Each task acquires a read lock on the ingress pool, finds nothing, then sleeps for 5 seconds.
5. After ~3 seconds the bouncer fires and drops the `JoinSet`, aborting all tasks.
6. The attacker repeats immediately with a new artifact ID.

At 10^5 tasks × ~8 KB Tokio task overhead ≈ 800 MB memory spike per attack cycle, repeated every 3 seconds. This causes sustained replica memory pressure, Tokio scheduler saturation, and consensus stall on the receiving node. A single below-threshold peer is sufficient.

---

### Likelihood Explanation

- The feature is enabled in production (`HASHES_IN_BLOCKS_ENABLED = true`).
- The attacker only needs a valid P2P connection to the target node (any subnet peer).
- No cryptographic material, governance access, or threshold corruption is required.
- The crafted proto is trivially constructable.
- The attack is repeatable with no rate limiting at the P2P layer for pushed artifacts.

---

### Recommendation

1. **In `TryFrom<pb::StrippedBlockProposal>`**: Reject deserialization if `value.ingress_messages.len() > MAX_INGRESS_MESSAGES_PER_BLOCK` or `value.stripped_idkg_dealings.len() > DKG_DEALINGS_PER_BLOCK`. [3](#0-2) 

2. **In `assemble_message`**: Add a guard before the `for` loop that returns `AssembleResult::Unwanted` if `stripped_message_ids.len()` exceeds the protocol maximum. [5](#0-4) 

---

pub(crate) async fn download_stripped_message<P: Peers>(
    transport: Arc<dyn Transport>,
    stripped_message_id: StrippedMessageId,
    block_proposal_id: ConsensusMessageId,
    log: &ReplicaLogger,
    metrics: &FetchStrippedConsensusArtifactMetrics,
    peer_rx: P,
) -> (StrippedMessage, NodeId) {
    let message_type = StrippedMessageType::from(&stripped_message_id);
    metrics.report_started_stripped_message_download(message_type);
    let mut artifact_download_timeout = ExponentialBackoffBuilder::new()
        .with_initial_interval(MIN_ARTIFACT_RPC_TIMEOUT)
        .with_max_interval(MAX_ARTIFACT_RPC_TIMEOUT)
        .with_max_elapsed_time(None)
        .build();

    let mut rng = SmallRng::from_entropy();

    let request = match &stripped_message_id {
        StrippedMessageId::Ingress(signed_ingress_id) => {
            let request = GetIngressMessageInBlockRequest {
                signed_ingress_id: signed_ingress_id.clone(),
                block_proposal_id,
            };
            let bytes = Bytes::from(pb::GetIngressMessageInBlockRequest::proxy_encode(request));
            Request::builder().uri(INGRESS_URI).body(bytes).unwrap()
        }
        StrippedMessageId::IDkgDealing(dealing_id, node_index) => {
            let request = GetIDkgDealingInBlockRequest {
                node_index: *node_index,
                dealing_id: dealing_id.clone(),
                block_proposal_id,
            };
            let bytes = Bytes::from(pb::GetIDkgDealingInBlockRequest::proxy_encode(request));
            Request::builder()
                .uri(IDKG_DEALING_URI)
                .body(bytes)
                .unwrap()
        }
    };

    loop {
        let next_request_at = Instant::now()
            + artifact_download_timeout
                .next_backoff()
                .unwrap_or(MAX_ARTIFACT_RPC_TIMEOUT);
        if let Some(peer) = { peer_rx.peers().into_iter().choose(&mut rng) } {
            match timeout_at(next_request_at, transport.rpc(&peer, request.clone())).await {
                Ok(Ok(response)) if response.status() == StatusCode::OK => {
                    match parse_response(&stripped_message_id, response.into_body()) {
                        Ok(stripped_message) => {
                            metrics.report_finished_stripped_message_download(message_type);
                            return (stripped_message, peer);
                        }
                        Err(ParseResponseError::MessageIdMismatch) => {
                            metrics.report_download_error(
                                "mismatched_stripped_message_id",
                                message_type,
                            );
                            warn!(
                                log,
                                "Peer {} responded with wrong {} message for advert",
                                peer,
                                message_type.as_str(),
                            );
                        }
                        Err(ParseResponseError::ParsingError(reason)) => {
                            metrics.report_download_error(reason, message_type);
                        }
                    };
                }
                Ok(Ok(_response)) => {
                    metrics.report_download_error("status_not_ok", message_type);
                }
                Ok(Err(_rpc_error)) => {
                    metrics.report_download_error("rpc_error", message_type);
                }
                Err(_timeout) => {
                    metrics.report_download_error("timeout", message_type);
                }
            }
        }

        sleep_until(next_request_at).await;
    }
}
