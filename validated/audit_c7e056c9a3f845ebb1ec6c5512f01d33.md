### Title
Truncating Division for `native_per_pubdata` Undercharges Users for Pubdata — (`basic_bootloader/src/bootloader/transaction_flow/zk/validation_impl.rs`)

---

### Summary

`native_per_pubdata` — the number of native resource units charged per byte of pubdata — is computed with floor (truncating) division (`wrapping_div`), while the analogous `native_per_gas` is computed with ceiling division (`div_ceil`). When `pubdata_price` is not an exact multiple of `native_price`, the truncation causes users to be charged less for pubdata than the operator intended, resulting in a loss for the operator/sequencer.

---

### Finding Description

In `validate_and_compute_fee_for_transaction` (the L2 transaction validation path), two conversion ratios are derived from the block-level pricing parameters:

```rust
// native_per_gas — uses CEILING division
let native_per_gas = u256_try_to_u64(&gas_price.div_ceil(native_price))...

// native_per_pubdata — uses FLOOR (truncating) division
let native_per_pubdata = u256_try_to_u64(&pubdata_price.wrapping_div(native_price))...
```

`native_per_pubdata` is subsequently used in two places:

1. **Intrinsic pubdata overhead deduction** (`create_resources_for_tx`):
   ```rust
   let intrinsic_pubdata_overhead = native_per_pubdata_byte.saturating_mul(intrinsic_pubdata);
   let native_limit = native_limit.checked_sub(intrinsic_pubdata_overhead)...
   ```

2. **Runtime pubdata charging** (`get_resources_to_charge_for_pubdata`):
   ```rust
   let native = current_pubdata_spent.checked_mul(native_per_pubdata)...
   ```

Because `native_per_pubdata` is truncated, every pubdata byte costs up to 1 fewer native unit than the true fractional value. The shortfall propagates into `deltaGas`:

```
deltaGas := (native_used / native_per_gas) - gas_used
```

A lower `native_used` (from undercharged pubdata) produces a lower `deltaGas`, so the user's final `gas_used` — and therefore the ETH fee paid — is lower than it should be.

**Extreme case:** When `pubdata_price < native_price`, floor division yields `native_per_pubdata = 0`, making all pubdata **completely free** regardless of how many bytes the transaction writes. A user can write the maximum allowed pubdata and pay nothing for it.

The same truncating division is mirrored in the public API helper `validate_l2_tx_intrinsic_native_resources` in `api/src/helpers.rs`, where the comment even acknowledges the inconsistency:

```rust
// native_per_gas = ceil(gas_price / native_price)   ← ceiling
// native_per_pubdata = pubdata_price / native_price  ← floor (wrapping_div)
```

---

### Impact Explanation

- **Operator/sequencer revenue loss**: Every L2 transaction that writes pubdata pays less than the operator-configured `pubdata_price` implies. The shortfall per transaction is bounded by `pubdata_bytes_written / native_per_gas` gas units, but in the degenerate case (`pubdata_price < native_price`) pubdata is entirely free.
- **Resource accounting bug**: The native resource model is designed so that the operator is fully compensated for L1 data-availability costs via `pubdata_price`. Truncation breaks this invariant.
- **No user funds at risk**, but the operator systematically under-collects fees for every transaction that generates pubdata.

---

### Likelihood Explanation

- Every L2 transaction that touches storage generates pubdata. The bug fires on every such transaction whenever `pubdata_price % native_price != 0`.
- The extreme case (`pubdata_price < native_price`) is reachable whenever the operator sets a low pubdata price relative to native price — a plausible configuration when L1 DA costs are cheap but proving is expensive.
- No special privileges are required; any unprivileged transaction sender triggers the path.

---

### Recommendation

Replace `wrapping_div` with `div_ceil` for `native_per_pubdata` in both locations, consistent with how `native_per_gas` is computed:

```rust
// basic_bootloader/src/bootloader/transaction_flow/zk/validation_impl.rs  line 142
- let native_per_pubdata = u256_try_to_u64(&pubdata_price.wrapping_div(native_price))
+ let native_per_pubdata = u256_try_to_u64(&pubdata_price.div_ceil(native_price))
      .ok_or(TxError::Validation(InvalidTransaction::PubdataPriceTooHigh))?;

// api/src/helpers.rs  line 427
- let native_per_pubdata = u256_try_to_u64(&pubdata_price.wrapping_div(native_price)).ok_or(())?;
+ let native_per_pubdata = u256_try_to_u64(&pubdata_price.div_ceil(native_price)).ok_or(())?;
```

This ensures `native_per_pubdata >= ceil(pubdata_price / native_price)`, so the operator is never under-compensated for pubdata.

---

### Proof of Concept

**Setup:** operator sets `native_price = 3`, `pubdata_price = 2` (pubdata is cheaper than one native unit).

**Current behaviour (floor):**
```
native_per_pubdata = 2 / 3 = 0   (wrapping_div truncates)
```
A transaction writing 1 000 pubdata bytes is charged `1000 × 0 = 0` native units for pubdata. `deltaGas` contribution from pubdata = 0. The operator receives **zero** compensation for 1 000 bytes of L1 data.

**Correct behaviour (ceiling):**
```
native_per_pubdata = ceil(2 / 3) = 1
```
The same transaction is charged `1000 × 1 = 1000` native units, which translates to `1000 / native_per_gas` extra gas charged to the user, correctly compensating the operator.

**General case:** For any `pubdata_price` not divisible by `native_price`, the per-byte shortfall is `1` native unit. Over a transaction writing `N` pubdata bytes, the operator loses `N / native_per_gas` gas units of fee — small per transaction but systematic across all transactions.
