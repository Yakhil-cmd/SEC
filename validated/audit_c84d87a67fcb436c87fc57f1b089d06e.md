### Title
Zero EVM Gas Cost on `RandomSeed` Precompile Enables NEAR Gas Exhaustion Loop to Permanently Freeze Vault Funds - (`engine-precompiles/src/random.rs`)

---

### Summary

`RANDOM_BYTES_GAS = EthGas::new(0)` is hardcoded with an unresolved TODO. An unprivileged attacker can deploy a Solidity vault that gates withdrawal on a tight loop calling the `RandomSeed` precompile. Because the precompile charges 0 EVM gas, the loop exhausts the NEAR 200 Tgas receipt budget after only ~200 iterations while consuming only ~24,000 EVM gas — far below any EVM gas limit. The NEAR runtime aborts the wasm execution, the receipt reverts, and deposited ETH is permanently unwithdrawable.

---

### Finding Description

**Root cause — confirmed in source:** [1](#0-0) 

```rust
mod costs {
    // TODO(#483): Determine the correct amount of gas
    pub(super) const RANDOM_BYTES_GAS: EthGas = EthGas::new(0);
}
```

`RandomSeed::required_gas` returns this zero unconditionally: [2](#0-1) 

And `RandomSeed::run` charges exactly that zero cost before returning the 32-byte seed: [3](#0-2) 

**The gas accounting gap:**

Aurora's established conversion ratio is `NEAR_GAS_PER_EVM = 175_000_000` (175 M NEAR gas per 1 EVM gas): [4](#0-3) 

The NEAR transaction gas limit is 200 Tgas: [5](#0-4) 

Benchmarks confirm the EVM-to-NEAR ratio is roughly 80–120 M NEAR gas per EVM gas in practice (706 k EVM gas ≈ 83 Tgas; 1.73 M EVM gas ≈ 143 Tgas): [6](#0-5) [7](#0-6) 

This means the NEAR budget is exhausted after consuming only ~1.7–2.5 M EVM gas. Any EVM transaction whose gas limit exceeds that threshold can exhaust NEAR gas before EVM gas — a known architectural tension. The zero precompile cost sharpens this gap dramatically for loops targeting `RandomSeed`.

**Quantified attack path:**

Each loop iteration costs:
- EVM gas: `CALL` opcode (100 gas warm) + loop opcodes (~20 gas) ≈ **120 EVM gas**
- NEAR gas: wasm interpreter overhead for the CALL dispatch + precompile execution ≈ **~1 Tgas per call** (consistent with the benchmark ratios above)

To exhaust 200 Tgas: ~200 iterations × 120 EVM gas = **24,000 EVM gas consumed** while NEAR gas is fully spent. With any EVM gas limit ≥ 30,000 (trivially satisfiable), the EVM never reports `OutOfGas`; the NEAR runtime aborts the wasm execution mid-flight, the receipt reverts, and no state change is committed.

The same TODO(#483) zero-gas pattern also appears in `PrepaidGas`, `PredecessorAccount`, and `CurrentAccount` precompiles: [8](#0-7) [9](#0-8) 

---

### Impact Explanation

An attacker deploys a Solidity vault contract (permissionless) whose `withdraw()` function loops over `RandomSeed` for a fixed iteration count calibrated to exhaust 200 Tgas. Users deposit ETH. Every subsequent `withdraw()` call triggers the loop, exhausts NEAR gas, causes the NEAR receipt to revert, and leaves the ETH locked in the vault with no alternative withdrawal path. Because the vault bytecode is immutable and the loop is deterministic, the freeze is permanent. Impact: **Critical — permanent freezing of funds**.

---

### Likelihood Explanation

The attacker needs only to deploy a contract (no privilege required) and attract deposits (e.g., by presenting the vault as a legitimate yield product). The technical exploit is deterministic and repeatable every block. The calibration of the loop count is straightforward given the public 200 Tgas limit. Likelihood: **Medium** (gated only by the social step of attracting deposits, not by any on-chain guard).

---

### Recommendation

1. **Assign a non-zero EVM gas cost to `RANDOM_BYTES_GAS`** calibrated to the actual NEAR gas cost of the precompile call, using the same methodology applied to `XCC` (`NEAR_gas / CROSS_CONTRACT_CALL_NEAR_GAS`): [10](#0-9) 

2. **Apply the same fix to all other TODO(#483) zero-cost precompiles** (`PrepaidGas`, `PredecessorAccount`, `CurrentAccount`).

3. **Add a regression test** that calls `RandomSeed::run` in a loop of N iterations under a fixed NEAR gas budget and asserts that EVM gas charged equals `N × RANDOM_BYTES_GAS` and that NEAR gas consumed grows proportionally, verifying the invariant holds.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

interface IRandomSeed {
    function randomSeed() external view returns (bytes32);
}

contract FreezeVault {
    address constant RANDOM_SEED = 0xc104f4840573bed437190daf5d2898c2bdf928ac;
    mapping(address => uint256) public balances;

    function deposit() external payable {
        balances[msg.sender] += msg.value;
    }

    function withdraw() external {
        uint256 amount = balances[msg.sender];
        require(amount > 0, "no balance");

        // Loop calibrated to exhaust ~200 Tgas NEAR gas budget
        // while consuming only ~24,000 EVM gas (200 * 120).
        // RANDOM_BYTES_GAS = 0, so EVM never runs out of gas here.
        bytes32 seed;
        for (uint256 i = 0; i < 200; i++) {
            (bool ok, bytes memory data) = RANDOM_SEED.staticcall("");
            require(ok);
            seed = abi.decode(data, (bytes32));
        }
        // Require an impossible seed value — withdrawal always reverts
        // at the NEAR level before reaching here anyway.
        require(seed == bytes32(0), "bad seed");

        balances[msg.sender] = 0;
        payable(msg.sender).transfer(amount);
    }
}
```

**Call sequence:**
1. Attacker deploys `FreezeVault`.
2. Users call `deposit{value: X}()` — ETH is locked.
3. Any call to `withdraw()` triggers the 200-iteration `RandomSeed` loop, exhausting the 200 Tgas NEAR budget after ~24,000 EVM gas.
4. NEAR runtime aborts wasm execution; receipt reverts; ETH remains locked.
5. Repeat for every block — funds are permanently frozen.

**Invariant violated:** Every EVM gas unit consumed by a precompile call must correspond to a bounded NEAR gas cost, so that a transaction within its EVM gas budget cannot silently exhaust NEAR gas and revert the enclosing receipt. `RANDOM_BYTES_GAS = 0` breaks this invariant. [11](#0-10)
