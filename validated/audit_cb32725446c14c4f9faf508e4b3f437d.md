Audit Report

## Title
Spawned NNS Neuron Unconditionally Inherits Parent's Hot Keys When Assigned a New Controller, Granting Previous Owner's Delegates Unauthorized Governance Rights — (`rs/nns/governance/src/governance.rs`)

## Summary

When `spawn_neuron` is called with a `new_controller` that differs from the parent neuron's controller, the child neuron is built with `.with_hot_keys(parent_neuron.hot_keys.clone())` unconditionally, regardless of whether the child controller matches the parent controller. Any principal listed in the parent's `hot_keys` retains the ability to vote and change followees on the child neuron without the new controller's knowledge or consent.

## Finding Description

In `spawn_neuron`, the child controller is resolved at line 2651–2655: if `spawn.new_controller` is provided, it becomes `child_controller`; otherwise the parent's controller is used. [1](#0-0) 

The child neuron is then constructed at lines 2704–2718. The `.with_hot_keys(parent_neuron.hot_keys.clone())` call at line 2714 is unconditional — it does not check whether `child_controller == parent_neuron.controller()`. [2](#0-1) 

By contrast, `disburse_to_neuron` (lines 3012–3021) creates the child neuron with no hot keys at all and resets followees to `default_followees`, demonstrating that the codebase already handles the ownership-transfer concern in at least one path. [3](#0-2) 

`is_authorized_to_vote` (and its delegate `is_hotkey_or_controller`) returns `true` for any principal in `hot_keys`, independently of who the controller is. [4](#0-3) 

The `follow` function enforces only that the caller is the controller **or** an authorized hot key for non-`ManageNeuron` topics, so inherited hot keys pass this check without restriction. [5](#0-4) 

Exploit path:
1. Alice controls neuron N with `hot_keys = [mallory]` and sufficient maturity.
2. Alice calls `Spawn { new_controller: Some(bob), ... }`.
3. `spawn_neuron` creates child neuron C with `controller = bob` and `hot_keys = [mallory]`.
4. Mallory calls `Follow` on C for any non-`NeuronManagement` topic — succeeds because `is_authorized_to_vote` returns `true`.
5. Mallory calls `RegisterVote` on C — succeeds for the same reason.
6. Bob's neuron votes and follows Mallory's neuron without Bob's knowledge.

No existing guard prevents this: there is no post-spawn hot-key reset, no notification to Bob, and no check that hot keys were authorized by the current controller.

## Impact Explanation

This constitutes unauthorized access to a governance asset (NNS neuron) controlled by an unsuspecting principal. Mallory can exercise voting power and redirect followees on Bob's neuron for all topics except `NeuronManagement`, silently influencing NNS proposal outcomes. This matches the **High** bounty impact: *"Unauthorized access to neurons, governance assets … where exploitation requires meaningful per-target work or other constraints."*

## Likelihood Explanation

`spawn_neuron` with `new_controller` is a standard, permissionless NNS governance operation. The attacker (Alice) needs only to control a neuron with maturity and add a colluding hot key before spawning to a victim. Realistic scenarios include protocol-level maturity distribution, vesting custodians, and neuron gifting. The victim (Bob) receives no on-chain notification and must proactively inspect and clean the hot key list — a step most users will not take.

## Recommendation

In `spawn_neuron`, clear the hot key list when the child controller differs from the parent controller, mirroring `disburse_to_neuron`:

```rust
let child_hot_keys = if child_controller == parent_neuron.controller() {
    parent_neuron.hot_keys.clone()
} else {
    vec![]
};
// then: .with_hot_keys(child_hot_keys)
```

Optionally, also reset followees to `default_followees` (as `disburse_to_neuron` does) rather than inheriting the parent's followees, to fully isolate the child neuron's governance configuration from the parent's.

## Proof of Concept

1. Deploy a local NNS replica or use PocketIC.
2. Create neuron N controlled by Alice; add `mallory` as a hot key; accumulate maturity above the minimum stake threshold.
3. Call `manage_neuron` → `Spawn { new_controller: Some(bob), nonce: Some(1), percentage_to_spawn: Some(100) }` as Alice.
4. Retrieve the child neuron C; assert `C.controller == bob` and `C.hot_keys == [mallory]`.
5. As Mallory, call `manage_neuron` on C → `Follow { topic: NetworkEconomics, followees: [mallory_neuron] }` — assert success.
6. As Mallory, call `manage_neuron` on C → `RegisterVote { proposal_id: X, vote: Yes }` — assert success.
7. Confirm Bob never authorized Mallory and was never notified.
