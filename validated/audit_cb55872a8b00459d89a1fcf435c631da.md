Audit Report

## Title
VetKD Key Share Verification Accepts Undeserializable Content, Allowing a Single Byzantine Node to Block Key Derivation — (`rs/crypto/src/vetkd/mod.rs`)

## Summary

`verify_encrypted_key_share_internal` validates only the Ed25519 signature over the raw bytes of `encrypted_key_share` without attempting to deserialize the content. A Byzantine subnet node can submit a `VetKdEncryptedKeyShare` with empty `encrypted_key_share` bytes and a valid signature over those bytes. The share passes verification and enters the validated pool. When `combine_encrypted_key_shares_internal` later processes all validated shares, it calls `EncryptedKeyShare::deserialize` on the empty bytes, which fails immediately, causing the entire combination to fail and the key derivation request to be silently dropped.

## Finding Description

**Gap in `verify_encrypted_key_share_internal`:**
The function at `rs/crypto/src/vetkd/mod.rs` lines 292–301 calls only `BasicSigVerifierInternal::verify_basic_sig` over `key_share.encrypted_key_share`. There is no attempt to deserialize or structurally validate the content. A valid Ed25519 signature over empty bytes passes this check. [1](#0-0) 

**Deserialization failure in `combine_encrypted_key_shares_internal`:**
All shares are deserialized via `.collect::<Result<_, _>>()?`. If any single share fails deserialization, the entire combination fails immediately with `InvalidArgumentEncryptedKeyShare` — before `combine_all` or the `combine_valid_shares` fallback is ever reached. [2](#0-1) 

**`EncryptedKeyShare::deserialize` rejects wrong-length input:**
Empty bytes (length 0) fail the `val.len() != Self::BYTES` check immediately. [3](#0-2) 

**The combiner includes ALL validated shares, not just threshold-many:**
`shares.vetkd_shares.get(callback_id)` returns the full `BTreeMap<NodeId, VetKdEncryptedKeyShare>` collected from all validated pool entries. There is no pre-filtering to exclude the Byzantine node's share. [4](#0-3) 

**The combination failure is silently dropped:**
`InvalidArgumentEncryptedKeyShare` is not matched by `is_unsatisfied_reconstruction_threshold`, so it falls into the `Err(err)` branch: a warning is logged and `None` is returned — no agreement is produced for that request. [5](#0-4) [6](#0-5) 

**The share enters the validated pool via the signer:**
`VetKdProtocol::verify_encrypted_key_share` is the only gate before a share is added to the validated pool as `IDkgMessage::VetKdKeyShare`. Since it only checks the signature, the Byzantine share passes. [7](#0-6) 

## Impact Explanation

A single Byzantine subnet node (below the BFT fault threshold) can permanently block any VetKD key derivation request for the duration of the DKG interval by submitting one signed-but-empty share per request. The request is silently dropped from every block payload until it expires. The node can repeat this for every new request, effectively denying the VetKD service to all users of the subnet. This matches the allowed ICP bounty impact: **High ($2,000–$10,000) — Application/platform-level DoS, subnet availability impact not based on raw volumetric DDoS.**

## Likelihood Explanation

Any subnet node with a valid registered Ed25519 key can execute this. No threshold corruption, governance majority, or privileged access is required. The attack is deterministic and repeatable. The only natural limit is the DKG interval expiry, but the attacker can re-attack each new request indefinitely.

## Recommendation

Add deserialization validation inside `verify_encrypted_key_share_internal` in `rs/crypto/src/vetkd/mod.rs`. After the signature check passes, attempt `EncryptedKeyShare::deserialize(&key_share.encrypted_key_share.0)` and return a `VetKdKeyShareVerificationError` (e.g., a new `InvalidEncryptedKeyShareContent` variant, or reuse `VerificationError`) if deserialization fails. This closes the gap between what is verified and what is later consumed by the combiner. [8](#0-7) 

## Proof of Concept

1. Byzantine node holds a valid registered Ed25519 key pair `(sk, pk)`.
2. Construct `VetKdEncryptedKeyShare { encrypted_key_share: VetKdEncryptedKeyShareContent(vec![]), node_signature: Ed25519Sign(sk, b"") }`.
3. Broadcast this as a `VetKdKeyShare` P2P artifact for any active VetKD request.
4. Other nodes call `verify_encrypted_key_share` → signature over empty bytes is valid → `Ok(())` → share enters validated pool.
5. Combiner calls `combine_encrypted_key_shares` with all validated shares including the Byzantine one.
6. `EncryptedKeyShare::deserialize(&[])` fails → `collect::<Result<_, _>>()?` returns `Err(InvalidArgumentEncryptedKeyShare)`.
7. `combine_shares` returns `Err(CombineSharesError::VetKd(InvalidArgumentEncryptedKeyShare))` → not matched by `is_unsatisfied_reconstruction_threshold` → logged as warning, `None` returned → no agreement produced → request is dropped.

A deterministic integration test can be written using `TempCryptoComponent` to construct a malformed share, inject it into the validated pool, and assert that `get_chain_key_payload_impl` returns an empty payload despite sufficient honest shares being present.

fn verify_encrypted_key_share_internal<S: CspSigner>(
    lockable_threshold_sig_data_store: &LockableThresholdSigDataStore,
    registry: &dyn RegistryClient,
    csp_signer: &S,
    key_share: &VetKdEncryptedKeyShare,
    signer: NodeId,
    args: &VetKdArgs,
) -> Result<(), VetKdKeyShareVerificationError> {
    let registry_version_from_store = lockable_threshold_sig_data_store
        .read()
        .transcript_data(args.ni_dkg_id)
        .map(|transcript_data| transcript_data.registry_version())
        .ok_or_else(|| {
            VetKdKeyShareVerificationError::ThresholdSigDataNotFound(
                ThresholdSigDataNotFoundError::ThresholdSigDataNotFound {
                    dkg_id: args.ni_dkg_id.clone(),
                },
            )
        })?;

    let signature = BasicSigOf::new(BasicSig(key_share.node_signature.clone()));
    BasicSigVerifierInternal::verify_basic_sig(
        csp_signer,
        registry,
        &signature,
        &key_share.encrypted_key_share,
        signer,
        registry_version_from_store,
    )
    .map_err(VetKdKeyShareVerificationError::VerificationError)
}
