Audit Report

## Title
`is_funded()` Omits `staked_maturity_e8s_equivalent`, Permanently Stranding Staked Maturity in Inactive Dissolved Neurons — (`rs/nns/governance/src/neuron/types.rs`)

## Summary

`is_funded()` sums only `stake_e8s()` and `maturity_e8s_equivalent`, omitting `staked_maturity_e8s_equivalent`. A dissolved neuron with only staked maturity (no ICP stake, no unstaked maturity) is therefore classified as `is_inactive() == true` after 2 weeks. The `with_active_neurons_iter_sections` iterator filters it out, so `list_neurons_ready_to_unstake_maturity` never returns it, and the periodic timer task — the sole mechanism for converting staked maturity to regular maturity — can never reach it. The staked maturity is permanently inaccessible.

## Finding Description

**Root cause — `is_funded()` at `rs/nns/governance/src/neuron/types.rs:1112–1117`:**

```rust
pub fn is_funded(&self) -> bool {
    let amount_e8s = self
        .stake_e8s()
        .saturating_add(self.maturity_e8s_equivalent);
    amount_e8s > 0
}
```

`staked_maturity_e8s_equivalent` is absent. A neuron with `cached_neuron_stake_e8s = 0`, `neuron_fees_e8s = 0`, `maturity_e8s_equivalent = 0`, and `staked_maturity_e8s_equivalent = N > 0` returns `false` from `is_funded()`.

**`is_inactive()` at `rs/nns/governance/src/neuron/types.rs:1060–1098`** uses `is_funded()` as its "funded" gate (condition 2). With `is_funded()` returning `false`, and the neuron dissolved ≥ 2 weeks ago, not seed/ECT, and not a Neurons' Fund member, `is_inactive()` returns `true`.

**`with_active_neurons_iter_sections` at `rs/nns/governance/src/neuron_store.rs:519–533`** filters the stable-store range with `.filter(|n| !n.is_inactive(now))`, excluding the neuron from all active-neuron iterators.

**`list_neurons_ready_to_unstake_maturity` at `rs/nns/governance/src/neuron_store.rs:578–592`** is built on top of this filtered iterator, so the neuron is never returned.

**`unstake_maturity_of_dissolved_neurons` at `rs/nns/governance/src/neuron_store.rs:632–661`** calls `list_neurons_ready_to_unstake_maturity` and then calls `neuron.unstake_maturity()` on each result. Because the neuron is never listed, its staked maturity is never moved to `maturity_e8s_equivalent`.

**The timer task at `rs/nns/governance/src/timer_tasks/unstake_maturity_of_dissolved_neurons.rs:10–31`** fires every 60 seconds but can never reach the neuron.

**The proptest at `rs/nns/governance/src/governance/tests/mod.rs:1121–1158`** explicitly computes `net_funding_e8s` including `staked_maturity_e8s_equivalent` and asserts `neuron.is_inactive(now) == !is_funded`. This test directly contradicts the current `is_funded()` implementation and would fail for any case where `staked_maturity_e8s_equivalent > 0` and all other funding fields are zero, confirming the discrepancy between intent and implementation.

**Exploit path:**
1. Controller creates/has a neuron with `staked_maturity_e8s_equivalent > 0`.
2. Controller disburses the ICP stake, setting `cached_neuron_stake_e8s = 0`.
3. Controller has no regular maturity (`maturity_e8s_equivalent = 0`).
4. After 2 weeks post-dissolution, `is_inactive()` returns `true`.
5. The timer task runs every 60 seconds but never processes the neuron.
6. Staked maturity is permanently stranded — there is no user-callable governance method to trigger `unstake_maturity` directly.

## Impact Explanation

A neuron controller loses permanent access to their staked maturity, which represents real economic value (ICP that would be minted upon spawning). The comment at `rs/nns/governance/src/neuron_store.rs:645–646` states *"No neuron in stable storage should have staked maturity"* — this invariant is violated. This constitutes a significant NNS governance security impact with concrete, irreversible user-funds harm, qualifying as **High** severity under the "Significant NNS security impact with concrete user or protocol harm" category.

## Likelihood Explanation

The scenario is reachable by any neuron controller through entirely normal governance flows: dissolve a neuron, disburse the ICP stake, and wait 2 weeks. No privileged access, no social engineering, and no external dependencies are required. The controller harms only themselves, but the path is straightforward and repeatable for any neuron that has staked maturity and a zero ICP stake.

## Recommendation

Include `staked_maturity_e8s_equivalent` in `is_funded()`:

```rust
pub fn is_funded(&self) -> bool {
    let amount_e8s = self
        .stake_e8s()
        .saturating_add(self.staked_maturity_e8s_equivalent.unwrap_or(0))
        .saturating_add(self.maturity_e8s_equivalent);
    amount_e8s > 0
}
```

This aligns with the intent expressed in the proptest at `rs/nns/governance/src/governance/tests/mod.rs:1121–1127`, which already includes `staked_maturity_e8s_equivalent` in its `net_funding_e8s` calculation.

## Proof of Concept

```rust
#[test]
fn test_staked_maturity_stranded_in_inactive_neuron() {
    let now = 123_456_789_u64;
    // Dissolved 3 weeks ago (> 2-week inactivity threshold)
    let dissolved_at = now - 3 * 7 * ONE_DAY_SECONDS;

    let neuron = NeuronBuilder::new(
        NeuronId { id: 1 },
        Subaccount::try_from(&[0_u8; 32] as &[u8]).unwrap(),
        PrincipalId::new_user_test_id(1),
        DissolveStateAndAge::DissolvingOrDissolved {
            when_dissolved_timestamp_seconds: dissolved_at,
        },
        now,
    )
    .with_staked_maturity_e8s_equivalent(1_000_000)
    .build();
    // cached_neuron_stake_e8s = 0, maturity_e8s_equivalent = 0

    // Step 1: Confirm is_inactive() == true (bug: is_funded() ignores staked maturity)
    assert!(neuron.is_inactive(now));

    // Step 2: Add to neuron store and run unstake task repeatedly
    let mut neuron_store = NeuronStore::new(BTreeMap::new());
    neuron_store.add_neuron(neuron).unwrap();

    for _ in 0..10 {
        neuron_store.unstake_maturity_of_dissolved_neurons(now, usize::MAX);
    }

    // Step 3: Staked maturity is permanently stranded
    neuron_store.with_neuron(&NeuronId { id: 1 }, |n| {
        assert_eq!(n.staked_maturity_e8s_equivalent, Some(1_000_000));
        assert_eq!(n.maturity_e8s_equivalent, 0);
    }).unwrap();
}
```
