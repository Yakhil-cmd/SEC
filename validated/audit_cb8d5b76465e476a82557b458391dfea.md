Audit Report

## Title
Uninitialized RNG in `randomly_pick_swap_start` Silently Returns Fixed Swap Start Time — (`rs/nns/governance/src/governance.rs`)

## Summary
`CanisterRandomnessGenerator` is constructed with `rng: None` and seeded only after an async `raw_rand` round-trip completes. During the window between canister init/upgrade and the first `SeedingTask` completion, `randomly_pick_swap_start` silently substitutes `10_000` for the missing random value via `unwrap_or(10_000)`, producing a deterministic swap start time of 2:45:00 UTC for any `CreateServiceNervousSystem` proposal executed in that window.

## Finding Description
`CanisterRandomnessGenerator::new()` sets `rng: None`, and `random_u64()` returns `Err(RngError::RngNotInitialized)` when called before seeding:

```rust
// rs/nns/governance/src/canister_state.rs:136-138
pub fn new() -> Self {
    CanisterRandomnessGenerator { rng: None }
}
```

```rust
// rs/nns/governance/src/canister_state.rs:142-146
fn random_u64(&mut self) -> Result<u64, RngError> {
    match self.rng.as_mut() {
        Some(rand) => Ok(rand.next_u64()),
        None => Err(RngError::RngNotInitialized),
    }
}
```

`SeedingTask::initial_delay()` returns `Duration::from_secs(0)`, but the seeding only completes after the async `raw_rand` call returns — at minimum one IC round after `canister_init`/`canister_post_upgrade`. This creates a guaranteed uninitialized window:

```rust
// rs/nns/governance/src/timer_tasks/seeding.rs:31-41
let result: Result<Vec<u8>, (Option<i32>, String)> = env
    .call_canister_method(IC_00, "raw_rand", Encode!().unwrap())
    .await;
// seeds only on Ok(bytes)
```

`randomly_pick_swap_start` swallows the error with `unwrap_or(10_000)`:

```rust
// rs/nns/governance/src/governance.rs:7888
let time_of_day_seconds = self.randomness.random_u64().unwrap_or(10_000) % ONE_DAY_SECONDS;
```

`10_000 % 86400 = 10000`; rounded to nearest 15-minute boundary: `10000 - (10000 % 900) = 9900` seconds = **2:45:00 UTC**. This value is passed unconditionally into `make_sns_init_payload`:

```rust
// rs/nns/governance/src/governance.rs:4434
let random_swap_start_time = self.randomly_pick_swap_start();
```

```rust
// rs/nns/governance/src/governance.rs:4468
start_time.unwrap_or(random_swap_start_time),
```

The fallback only applies when the proposal omits `swap_parameters.start_time`, which is the common case. No guard prevents proposal execution during the uninitialized window.

## Impact Explanation
Any `CreateServiceNervousSystem` proposal executed during the post-upgrade uninitialized window (a routine operational event) will have its SNS swap start time fixed at the publicly known constant 2:45:00 UTC. This eliminates the unpredictability guarantee that `randomly_pick_swap_start` is designed to provide, allowing any observer who detects the governance upgrade to predict the swap start time before the proposal is executed and position themselves to participate at the exact opening moment. This constitutes a concrete SNS fairness and governance integrity impact. Severity: **Medium** — the conditions require a specific timing coincidence (adopted CSNS proposal queued across an upgrade boundary), the window is narrow, and the advantage conferred (advance knowledge of start time) is real but constrained.

## Likelihood Explanation
Governance canister upgrades are routine. A CSNS proposal adopted before an upgrade and queued for execution will be processed in the first rounds after the upgrade, squarely within the uninitialized window. The IC's deterministic execution model makes the window reproducible and observable on-chain. If `raw_rand` fails, `RETRY_SEEDING_INTERVAL` is 30 seconds, extending the window significantly.

## Recommendation
`randomly_pick_swap_start` should propagate `RngNotInitialized` rather than silently substituting a constant. The function signature should return `Result<GlobalTimeOfDay, GovernanceError>`, and `do_create_service_nervous_system` should return an error when the RNG is not yet seeded, allowing the proposal execution to be retried once seeding completes. Alternatively, seeding should be completed synchronously during `canister_init`/`canister_post_upgrade` before any proposal execution is permitted.

## Proof of Concept
1. Deploy or upgrade the NNS governance canister.
2. Before `SeedingTask` completes its first `raw_rand` call (i.e., in the same or immediately following IC round), execute a `CreateServiceNervousSystem` proposal that omits `swap_parameters.start_time`.
3. `randomly_pick_swap_start` is called with `rng = None`; `random_u64()` returns `Err(RngNotInitialized)`; `unwrap_or(10_000)` substitutes `10_000`; `10_000 % 86400 = 10000`; rounded = `9900` seconds after midnight.
4. The SNS swap `swap_start_timestamp_seconds` is set to 2:45:00 UTC — deterministic and publicly predictable.
5. A unit test can reproduce this by constructing a `Governance` instance with a fresh `CanisterRandomnessGenerator` (before calling `seed_rng`) and asserting that `randomly_pick_swap_start()` returns `GlobalTimeOfDay { seconds_after_utc_midnight: Some(9900) }`.
