Audit Report

## Title
SNS Governance `disburse_maturity` Panics on u64 Overflow, Permanently Blocking Maturity Disbursement — (`rs/sns/governance/src/governance.rs`)

## Summary
The `disburse_maturity` function in the SNS governance canister multiplies two `u64` values using `checked_mul` but immediately calls `.expect()`, causing a panic (canister trap) when `maturity_e8s_equivalent * percentage_to_disburse` exceeds `u64::MAX`. Because the trap rolls back state, the neuron's maturity is never reduced, making every subsequent `DisburseMaturity` call on that neuron also trap. The NNS governance canister already carries the correct `u128`-widened fix; the SNS path never received it.

## Finding Description
At `rs/sns/governance/src/governance.rs` lines 1643–1649, the code is:

```rust
let maturity_to_deduct = neuron
    .maturity_e8s_equivalent
    .checked_mul(disburse_maturity.percentage_to_disburse as u64)
    .expect("Overflow while processing maturity to disburse.")   // panics on None
    .checked_div(100)
    .expect("Error when processing maturity to disburse.")
    as u128;
```

Both operands are `u64`. `checked_mul` returns `None` when the product exceeds `u64::MAX`, and `.expect()` converts that `None` into a panic. The overflow threshold for `percentage_to_disburse = 100` is `u64::MAX / 100 ≈ 1.844 × 10^17 e8s`. Because the trap rolls back all state changes, `maturity_e8s_equivalent` is never decremented, so the overflow condition is permanent for that neuron until a canister upgrade.

The NNS counterpart at `rs/nns/governance/src/governance/disburse_maturity.rs` lines 229–244 correctly widens to `u128` before multiplying and propagates `None` as an `Err` rather than panicking, confirming this is a known fix that was not applied to the SNS path.

No existing guard prevents the overflow: the only pre-check (lines 1635–1640) validates that `percentage_to_disburse` is between 1 and 100, which does nothing to bound `maturity_e8s_equivalent`.

## Impact Explanation
A canister trap on an IC update call rolls back state but leaves the canister alive. However, because the neuron's maturity is never reduced, every future `DisburseMaturity` call on the affected neuron also traps. This permanently disables the maturity-disbursement path for any neuron whose maturity crosses the threshold, until an SNS upgrade is deployed. This constitutes an application-level DoS on a core SNS governance function, matching the **High ($2,000–$10,000)** impact class: "Application/platform-level DoS … or subnet availability impact not based on raw volumetric DDoS."

## Likelihood Explanation
The attacker requires only `NeuronPermissionType::DisburseMaturity` on their own neuron — a standard, non-privileged permission granted to neuron controllers by default. The overflow threshold for `percentage=100` is ~1.844 × 10^17 e8s (~1.844 billion tokens). For large SNS deployments with multi-billion token supplies, a whale neuron staked for years accumulating voting rewards can reach this threshold naturally. Additionally, `AddMaturity` proposals (used in SNS initialization and testing) can set `maturity_e8s_equivalent` directly, providing a more direct path. The condition is self-sustaining once triggered: no repeated attacker action is needed after the initial call.

## Recommendation
Replace the `u64 × u64` multiplication with `u128` arithmetic, mirroring the NNS fix already present in `rs/nns/governance/src/governance/disburse_maturity.rs`:

```rust
let maturity_to_deduct = (neuron.maturity_e8s_equivalent as u128)
    .checked_mul(disburse_maturity.percentage_to_disburse as u128)
    .and_then(|v| v.checked_div(100))
    .and_then(|v| u64::try_from(v).ok())
    .ok_or_else(|| GovernanceError::new_with_message(
        ErrorType::PreconditionFailed,
        "Overflow while calculating maturity to disburse.",
    ))?;
```

This eliminates the panic path and returns a proper `Err` to the caller.

## Proof of Concept
```rust
#[test]
fn test_disburse_maturity_panics_on_overflow() {
    // u64::MAX * 2 overflows u64, triggering the .expect() panic
    let earned_maturity_e8s = u64::MAX;
    let mut setup = prepare_setup_for_disburse_maturity_tests(earned_maturity_e8s);

    let disburse_maturity = DisburseMaturity {
        percentage_to_disburse: 2,
        to_account: None,
    };
    // Currently panics with "Overflow while processing maturity to disburse."
    // Should return Err(GovernanceError { ... }) instead.
    let result = setup.governance.disburse_maturity(
        &setup.neuron_id,
        &setup.controller,
        &disburse_maturity,
    );
    assert!(result.is_err());
}
```

Running this test against the current code at lines 1643–1649 will produce a panic rather than an `Err`, confirming the bug. The fix causes the test to pass cleanly.
