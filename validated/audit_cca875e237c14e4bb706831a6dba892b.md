Audit Report

## Title
SNS Governance Maturity Disbursement Permanently Blocked When CMC Is Unavailable - (File: rs/sns/governance/src/governance.rs)

## Summary
In SNS Governance, `disburse_maturity` immediately deducts maturity from a neuron and queues a `DisburseMaturityInProgress` entry, but the actual minting is deferred to `maybe_finalize_disburse_maturity`. That function requires `maturity_modulation.current_basis_points` to be `Some`, which is only populated by a successful CMC poll. If the CMC has never successfully responded, `current_basis_points` remains `None`, causing `maybe_finalize_disburse_maturity` to return early on every periodic tick — leaving the already-deducted maturity permanently unprocessed until a governance upgrade intervenes.

## Finding Description
**Root cause:** `GovernanceProto::effective_maturity_modulation_basis_points()` at lines 417–428 of `rs/sns/governance/src/governance.rs` returns `Err` when `self.maturity_modulation` is `None` or when `current_basis_points` is `None`. The `MaturityModulation` proto field `current_basis_points` is declared `optional int32` (proto line 1683), so it defaults to `None` on a fresh SNS deployment.

**Exploit flow:**
1. A fresh SNS is deployed (or the CMC canister ID is misconfigured, or the CMC is transiently unavailable). `proto.maturity_modulation.current_basis_points` is `None`.
2. `update_maturity_modulation` (lines 5695–5701) polls the CMC; on failure it executes `return` without updating state, leaving `current_basis_points` as `None`.
3. A neuron owner calls `manage_neuron` → `disburse_maturity`. At lines 1692–1698, `neuron.maturity_e8s_equivalent` is immediately decremented and a `DisburseMaturityInProgress` record is pushed. The call returns `Ok`.
4. After `MATURITY_DISBURSEMENT_DELAY_SECONDS` (7 days), `maybe_finalize_disburse_maturity` fires. At lines 4926–4933, it calls `effective_maturity_modulation_basis_points()`, receives `Err`, logs the error, and executes `return` — skipping all pending disbursements.
5. This repeats on every periodic tick. The maturity is gone from the neuron and never minted to the destination account.

**Why existing checks fail:** There is no fallback to a neutral `0` basis points, no retry mechanism, and no way for the user to recover without a privileged canister upgrade. NNS Governance was explicitly patched (Proposal 141779, `heap_governance_data.rs` lines 229–232) to seed `maturity_modulation` with `current_value_permyriad: Some(0)` at init for exactly this reason — but the analogous fix was never applied to SNS Governance.

## Impact Explanation
A neuron owner who calls `DisburseMaturity` on an SNS whose CMC polling has never succeeded will have their maturity permanently deducted from the neuron with no corresponding ICP minted to the destination account. The accounting invariant — "maturity deducted from a neuron must eventually be minted" — is broken. This constitutes a significant SNS security impact with concrete, demonstrable user fund harm, matching the **High ($2,000–$10,000)** bounty tier: "Significant SNS security impact with concrete user or protocol harm."

## Likelihood Explanation
Any SNS deployed with a misconfigured CMC canister ID is permanently vulnerable from day one. Beyond misconfiguration, transient CMC unavailability (subnet upgrades, CMC canister upgrades, network partitions) during the window between SNS deployment and the first successful CMC poll creates the same condition. The NNS Governance canister was explicitly patched for this exact scenario, confirming DFINITY recognized it as a real operational risk. The trigger is an unprivileged `manage_neuron` ingress call available to any neuron controller.

## Recommendation
1. **Seed at init:** In SNS Governance initialization, set `proto.maturity_modulation = Some(MaturityModulation { current_basis_points: Some(0), updated_at_timestamp_seconds: None })`, mirroring the NNS Governance fix in `heap_governance_data.rs`.
2. **Fallback in finalization:** In `maybe_finalize_disburse_maturity`, when `effective_maturity_modulation_basis_points()` returns `Err`, fall back to `0` basis points (identity modulation) rather than returning early, so that already-deducted maturity is never permanently stranded.

## Proof of Concept
1. Deploy a fresh SNS with an unreachable CMC (wrong CMC canister ID or CMC stopped).
2. Earn maturity on a neuron via voting rewards.
3. Call `manage_neuron` with `Command::DisburseMaturity { percentage_to_disburse: 100, to_account: None }`. Observe: `neuron.maturity_e8s_equivalent` drops to `0`; `neuron.disburse_maturity_in_progress` gains one entry; call returns `Ok`.
4. Advance time past `MATURITY_DISBURSEMENT_DELAY_SECONDS` (7 days).
5. Trigger `run_periodic_tasks`. Observe: `maybe_finalize_disburse_maturity` calls `effective_maturity_modulation_basis_points()`, receives `Err` (because `current_basis_points` is `None`), logs the error, and returns immediately.
6. Verify: destination account balance is unchanged; `disburse_maturity_in_progress` entry persists indefinitely; maturity is gone from the neuron.

The NNS test `test_finalize_maturity_disbursement_no_maturity_modulation` (lines 612–650 of `rs/nns/governance/src/governance/disburse_maturity_tests.rs`) provides a directly portable template for an equivalent SNS integration test confirming this failure mode.
