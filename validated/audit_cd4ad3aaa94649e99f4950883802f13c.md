### Title
Caller-Controlled `new_controller` in `disburse_to_neuron` Bypasses KYC for Child Neuron — (File: `rs/nns/governance/src/governance.rs`)

---

### Summary

The NNS Governance canister's `disburse_to_neuron` function checks that the **parent** neuron is KYC-verified but allows the child neuron to be controlled by an **arbitrary** `new_controller` principal that has never undergone KYC verification. The child neuron unconditionally inherits the parent's `kyc_verified = true` status, granting a KYC-verified neuron to a non-KYC-verified principal. This is a direct analog to the NFT report's finding that `batchMintTo()` checks `msg.sender` is KYCed but allows minting to a non-KYCed `to` recipient.

---

### Finding Description

In `disburse_to_neuron` (`rs/nns/governance/src/governance.rs`), the function enforces KYC on the **parent** neuron: [1](#0-0) 

After passing this check, the caller freely specifies `new_controller` — any arbitrary `PrincipalId` — as the controller of the newly created child neuron: [2](#0-1) 

The child neuron is then built with the **parent's** `kyc_verified` flag (always `true` at this point, since the check above passed): [3](#0-2) 

There is no check that `new_controller` (the recipient) has itself undergone KYC. The `DisburseToNeuron` command struct exposes a caller-supplied `kyc_verified: bool` field in the public Candid API: [4](#0-3) 

This field is accepted from the caller but is **never read** inside `disburse_to_neuron`; the child neuron's KYC status is set exclusively from `parent_neuron.kyc_verified`. The caller-supplied `kyc_verified` field is a dead input.

---

### Impact Explanation

A KYC-verified genesis neuron owner (Alice) can call `disburse_to_neuron` specifying `new_controller = Bob`, where Bob is a principal that has never completed KYC. The resulting child neuron will have `kyc_verified = true` and be controlled by Bob. Bob can then call `disburse_neuron` on the child neuron — which requires `kyc_verified` to be true — and convert the staked ICP to liquid tokens, fully bypassing the KYC requirement that was supposed to gate disbursement.

The KYC check in `disburse_neuron` that would block Bob directly: [5](#0-4) 

…is circumvented because the child neuron already carries `kyc_verified = true` inherited from Alice's neuron.

**Vulnerability class**: Governance authorization bug — KYC check is applied to the caller's neuron (the source) but not to the recipient principal (`new_controller`), allowing KYC-gated neuron disbursement to be performed by a non-KYC-verified party.

---

### Likelihood Explanation

- The attack requires a KYC-verified neuron owner to call `disburse_to_neuron` with a non-KYC-verified `new_controller`. This can be done by the owner themselves (self-bypass using a second identity) or by transferring to a colluding party.
- The `manage_neuron` endpoint is a standard ingress call reachable by any unprivileged user who controls a KYC-verified neuron.
- No privileged access, admin keys, or subnet-majority corruption is required.
- Genesis neurons are the primary holders of `kyc_verified = true`; the pool is finite but non-trivial.

---

### Recommendation

1. **Verify `new_controller` KYC status** inside `disburse_to_neuron` before creating the child neuron. If the NNS KYC registry is accessible, check that `new_controller` is also KYC-verified before proceeding.
2. **Remove or enforce the `kyc_verified` field** from the `DisburseToNeuron` Candid interface. Since it is currently ignored, it is misleading. If it is intended to be caller-supplied, add a validation that it matches `parent_neuron.kyc_verified` and reject mismatches.
3. Alternatively, if the KYC system is considered legacy and no longer enforced, remove the `kyc_verified` checks entirely to avoid a false sense of compliance.

---

### Proof of Concept

1. Alice holds genesis neuron N with `kyc_verified = true` and sufficient stake.
2. Bob is a principal with no KYC verification (`kyc_verified` neurons: none).
3. Alice sends an ingress `manage_neuron` call to the NNS Governance canister:
   ```
   Command::DisburseToNeuron(DisburseToNeuron {
       new_controller: Some(Bob),
       amount_e8s: <min_stake + fee>,
       dissolve_delay_seconds: 0,
       kyc_verified: true,   // ignored by governance
       nonce: 42,
   })
   ```
4. Governance checks `parent_neuron.kyc_verified == true` ✓ (Alice's neuron passes).
5. Child neuron C is created with `controller = Bob` and `kyc_verified = true` (inherited from Alice's neuron at line 3020).
6. Bob now sends `manage_neuron` → `Command::Disburse` on neuron C.
7. Governance checks `neuron.kyc_verified == true` ✓ (child neuron passes).
8. Bob receives liquid ICP, having never undergone KYC. [6](#0-5) [1](#0-0) [7](#0-6)
