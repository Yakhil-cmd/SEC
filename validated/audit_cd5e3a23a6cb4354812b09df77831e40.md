### Title
NNS Governance `disburse_neuron` Unconditionally Resets `neuron_fees_e8s` to Zero Even When Fee Burn Is Skipped - (`File: rs/nns/governance/src/governance.rs`)

### Summary
In `disburse_neuron`, the NNS governance canister resets `neuron.neuron_fees_e8s = 0` unconditionally after the fee-burn block, even when the burn is skipped because `fees_amount_e8s <= transaction_fee_e8s`. This mirrors the original report's pattern: an accounting high-water mark is advanced past the point where fees are owed, even when those fees were never actually collected/burned, permanently forgiving them without destroying the corresponding ICP.

### Finding Description

In `rs/nns/governance/src/governance.rs`, `disburse_neuron` performs two ledger operations: first burn the neuron's accumulated management fees, then transfer the disbursed stake. The burn is conditionally skipped when the fee amount is too small for the ledger to accept:

```rust
if fees_amount_e8s > transaction_fee_e8s {
    // Transfer 1: burn fees on ledger
    self.ledger.transfer_funds(fees_amount_e8s, 0, ...).await?;
}

// This block runs unconditionally — even when the burn above was skipped
self.with_neuron_mut(id, |neuron| {
    if neuron.cached_neuron_stake_e8s > fees_amount_e8s {
        neuron.cached_neuron_stake_e8s -= fees_amount_e8s;
    } else {
        neuron.cached_neuron_stake_e8s = 0;
    }
    neuron.neuron_fees_e8s = 0;   // ← always zeroed, burn or not
})
``` [1](#0-0) 

When `fees_amount_e8s <= transaction_fee_e8s`:
1. The ledger burn is **skipped** — no ICP is destroyed.
2. `cached_neuron_stake_e8s` is **reduced** by `fees_amount_e8s` in governance state.
3. `neuron_fees_e8s` is **zeroed** — the debt is forgiven.

The result: the `fees_amount_e8s` tokens remain in the neuron's ledger subaccount (never burned), while governance believes they have been accounted for. Those tokens become orphaned in the subaccount with no governance record pointing to them.

**Contrast with SNS governance**, which correctly gates the state update on whether the burn actually occurred:

```rust
if max_burnable_fee > transaction_fee_e8s {
    self.ledger.transfer_funds(max_burnable_fee, 0, ...).await?;
    // Only update state if burn succeeded
    neuron.cached_neuron_stake_e8s = neuron.cached_neuron_stake_e8s.saturating_sub(max_burnable_fee);
    neuron.neuron_fees_e8s = neuron.neuron_fees_e8s.saturating_sub(max_burnable_fee);
}
``` [2](#0-1) 

The NNS path has no such guard.

### Impact Explanation

**Ledger conservation bug**: ICP tokens that governance protocol requires to be burned (as a penalty for rejected proposals) are instead left unburned in the neuron's subaccount. The governance accounting state (`neuron_fees_e8s`) is zeroed as if the burn occurred, permanently forgiving the debt. This:

1. Inflates the effective ICP supply by the forgiven fee amount (up to `transaction_fee_e8s` = 10,000 e8s per disburse call).
2. Creates a ledger/governance state divergence: the ledger subaccount holds `fees_amount_e8s` more ICP than governance believes, orphaning those tokens.
3. Allows a neuron controller to avoid paying the full penalty for rejected proposals when accumulated fees happen to fall at or below the ledger transaction fee threshold.

### Likelihood Explanation

**Low-to-medium**. The condition `fees_amount_e8s <= transaction_fee_e8s` (≤ 10,000 e8s) is reachable in practice:

- The NNS `reject_cost_e8s` default is 1,000,000 e8s, which is above the threshold. However, governance parameters can be changed via proposal, and any future reduction of `reject_cost_e8s` to ≤ 10,000 e8s would make this trivially triggerable by any neuron controller who submits a rejected proposal and then disburses.
- Any neuron that has accumulated fees through a governance parameter change that lowered `reject_cost_e8s` mid-lifecycle could also hit this path.
- The entry path requires only a dissolved neuron with `neuron_fees_e8s <= transaction_fee_e8s` — a standard unprivileged `manage_neuron` ingress call.

### Recommendation

Mirror the SNS governance pattern: only zero out `neuron_fees_e8s` (and reduce `cached_neuron_stake_e8s`) when the burn actually executes. Move the state-update block inside the `if fees_amount_e8s > transaction_fee_e8s` branch, after the `await?`:

```rust
if fees_amount_e8s > transaction_fee_e8s {
    self.ledger.transfer_funds(fees_amount_e8s, 0, ...).await?;
    // Only update state after confirmed burn
    self.with_neuron_mut(id, |neuron| {
        if neuron.cached_neuron_stake_e8s > fees_amount_e8s {
            neuron.cached_neuron_stake_e8s -= fees_amount_e8s;
        } else {
            neuron.cached_neuron_stake_e8s = 0;
        }
        neuron.neuron_fees_e8s = 0;
    }).expect("Expected the parent neuron to exist");
}
```

When `fees_amount_e8s <= transaction_fee_e8s`, the fees should either be left in `neuron_fees_e8s` for a future burn opportunity, or explicitly handled (e.g., forgiven with a corresponding ledger burn of zero, which is a no-op). The key invariant is: `neuron_fees_e8s` must not be zeroed unless the corresponding ICP has been destroyed on the ledger.

### Proof of Concept

1. Create a neuron with `cached_neuron_stake_e8s = S` and `neuron_fees_e8s = F` where `F <= 10_000` (e.g., via a governance parameter change that sets `reject_cost_e8s = 5_000`).
2. Dissolve the neuron and call `disburse_neuron`.
3. Observe: the ledger burn is skipped (`F <= transaction_fee_e8s`), but governance sets `neuron_fees_e8s = 0` and `cached_neuron_stake_e8s = S - F`.
4. The disburse transfer sends `S - F - transaction_fee_e8s` ICP to the caller.
5. Post-condition: the neuron's ledger subaccount retains `F` ICP (never burned); governance records `cached_neuron_stake_e8s = 0` and `neuron_fees_e8s = 0`. The `F` ICP is orphaned — not burned, not accessible through governance, diverging ledger and governance state. [3](#0-2) [4](#0-3)
