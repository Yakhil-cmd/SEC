### Title
Inconsistent `minimal_gas_used` Floor in ZK Transaction Flow Allows EVM Refunds to Reduce Effective Gas Below Intrinsic Cost — (`basic_bootloader/src/bootloader/transaction_flow/zk/validation_impl.rs`)

---

### Summary

The ZK transaction flow (`ZkTransactionFlowOnlyEOA`) returns `minimal_gas_used = 0` from `compute_calldata_tokens` when the `eip_7623` feature is disabled, while the Ethereum transaction flow (`EthereumTransactionFlow`) returns `minimal_gas_used = TX_INTRINSIC_GAS = 21_000` under the same condition. This floor value is used in `compute_gas_refund` to prevent EVM refunds (EIP-3529 SSTORE clears) from reducing the effective gas charged below the intrinsic gas cost. Because the ZK flow omits this floor, an unprivileged user can craft a transaction that clears a pre-set storage slot and receives an EVM refund large enough to bring `gas_used` below 21,000, paying less than the intrinsic fee to the operator.
