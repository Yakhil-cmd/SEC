Audit Report

## Title
GTC `donate_account` Permanently Locks User Out After Partial Transfer Failure - (File: rs/nns/gtc/src/lib.rs)

## Summary
In `AccountState::transfer`, neuron IDs are unconditionally removed from `self.neuron_ids` regardless of whether the cross-canister `transfer_gtc_neuron` call succeeds or fails. After the loop, `donate_account` sets `has_donated = true` even when some neurons failed to transfer. Any subsequent attempt by the user to recover their failed neurons — via `claim_neurons` or a retry of `donate_account` — is permanently blocked by the `has_donated` flag, leaving those neurons stranded under GTC control in Governance with no recovery path.

## Finding Description
In `AccountState::transfer` (`rs/nns/gtc/src/lib.rs`, lines 188–207), the loop clones `self.neuron_ids` and iterates over each `neuron_id`. After awaiting `GovernanceCanister::transfer_gtc_neuron`, line 192 unconditionally calls `self.neuron_ids.retain(|id| id != &neuron_id)` — the neuron ID is removed from `neuron_ids` whether the call returned `Ok` or `Err`. Failed neurons are pushed to `self.failed_transferred_neurons` but their IDs are gone from `neuron_ids`. The function always returns `Ok(())` at line 209.

Back in `donate_account` (line 89–90), `account.transfer(...).await?` succeeds (since `transfer` never returns `Err` for individual neuron failures), and `account.has_donated = true` is set unconditionally. From this point:

- `claim_neurons` (lines 54–56) returns `Err("Account has previously donated its funds")` for any call on this account.
- `donate_account` → `transfer` (lines 177–178) returns `Err("Account has already donated its funds")` on any retry.

The failed neurons remain in Governance under GTC canister control but are permanently inaccessible to the user through any GTC interface.

## Impact Explanation
This results in permanent, irreversible loss of a user's genesis neuron allocation. GTC neurons represent real ICP stake allocated at genesis; SR accounts hold up to 48 neurons. A user who calls `donate_account` during any transient cross-canister failure loses all neurons that failed to transfer, with no recovery mechanism. This matches the allowed impact: **High — unauthorized/permanent loss of governance assets (neurons) controlled by an NNS canister**, with the constraint that it requires a transient failure during the specific call window.

## Likelihood Explanation
The exploit requires no attacker — it is self-triggered by a normal user calling `donate_account` during a period of transient Governance or ICP ledger unavailability. The `transfer_gtc_neuron` call on Governance itself calls the ICP ledger; any transient error (canister stopped, cycles exhausted, ledger temporarily unavailable) on any of the up to 48 sequential cross-canister calls will trigger the partial failure. This is a realistic, non-zero probability event on the IC. The user need not make any mistake; the failure is entirely environmental.

## Recommendation
1. Only remove a neuron ID from `self.neuron_ids` after a **confirmed successful** transfer (move `self.neuron_ids.retain(...)` inside the `Ok(_)` branch).
2. Only set `has_donated = true` in `donate_account` when `account.failed_transferred_neurons` is empty after `transfer` returns.
3. Allow retrying `donate_account` when `failed_transferred_neurons` is non-empty and `has_donated` is still `false`, so users can recover from partial failures.

## Proof of Concept
1. A GTC account holder with multiple SR neurons calls `donate_account`.
2. The first N neurons transfer successfully; on neuron N+1, the Governance canister returns a transient error.
3. In `AccountState::transfer` (line 192), neuron N+1's ID is removed from `self.neuron_ids` and pushed to `failed_transferred_neurons` (line 204). The loop continues; remaining neurons also fail.
4. `transfer` returns `Ok(())` (line 209).
5. `donate_account` sets `account.has_donated = true` (line 90).
6. User calls `claim_neurons` to recover failed neurons → `Err("Account has previously donated its funds")` (lines 54–56).
7. User calls `donate_account` again to retry → `Err("Account has already donated its funds")` (lines 177–178).
8. Failed neurons remain in Governance under GTC control, permanently inaccessible.

A deterministic integration test using PocketIC can reproduce this by mocking `transfer_gtc_neuron` to return an error on a specific neuron index, then asserting that `failed_transferred_neurons` is non-empty and both `claim_neurons` and `donate_account` return the blocking errors.
