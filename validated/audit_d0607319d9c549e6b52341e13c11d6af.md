Audit Report

## Title
Integer Overflow in Neuron Age Calculation During Merge Produces Inflated Voting Power - (`File: rs/nns/governance/src/governance/merge_neurons.rs`)

## Summary

`ValidSourceNeuron::try_new` and `ValidTargetNeuron::try_new` compute neuron age using plain subtraction (`now_seconds - aging_since_timestamp_seconds`). When `aging_since_timestamp_seconds > now_seconds` — a known valid legacy state explicitly covered by the test suite — the subtraction wraps to near-`u64::MAX` in production Wasm (release builds, no `overflow-checks = true` found). This corrupted age value propagates through `combine_aged_stakes` and is written back as the merged neuron's `aging_since_timestamp_seconds ≈ 0`, permanently granting it the maximum age bonus and inflated voting power in NNS governance.

## Finding Description

In `ValidSourceNeuron::try_new` (line 438) and `ValidTargetNeuron::try_new` (line 480), age is computed as:

```rust
age_seconds: now_seconds - aging_since_timestamp_seconds,
``` [1](#0-0) [2](#0-1) 

The canonical `DissolveStateAndAge::age_seconds` method uses `saturating_sub`:

```rust
now_seconds.saturating_sub(aging_since_timestamp_seconds)
``` [3](#0-2) 

The test suite explicitly confirms that `aging_since_timestamp_seconds = NOW + 1` is a valid state and that the correct age in that case is `0`: [4](#0-3) 

No `overflow-checks = true` is set anywhere in the codebase, so production Wasm release builds use Rust's default wrapping behavior. When `aging_since_timestamp_seconds = now_seconds + 1`, the subtraction produces `u64::MAX`.

This `u64::MAX` age is passed as `source.age_seconds` into `combine_aged_stakes`: [5](#0-4) 

Inside `combine_aged_stakes`, the computation uses `u128` with `saturating_mul`/`saturating_add`, then casts back to `u64`: [6](#0-5) 

With `y_age_seconds = u64::MAX`, the weighted average `new_target_age_seconds` is a very large value (proportional to `u64::MAX`). This is then used in:

```rust
aging_since_timestamp_seconds: now_seconds.saturating_sub(new_target_age_seconds),
``` [7](#0-6) 

Since `new_target_age_seconds >> now_seconds`, the `saturating_sub` clamps to `0`, writing `aging_since_timestamp_seconds = 0` into the merged neuron's state. The two `debug_assert!` guards at lines 279 and 281 that would catch this are stripped in production builds: [8](#0-7) 

## Impact Explanation

A merged neuron with `aging_since_timestamp_seconds = 0` has an effective age equal to the full IC runtime (~4+ years), which exceeds the 4-year cap for the maximum age bonus of 1.25×. This inflates both `potential_voting_power` and `deciding_voting_power` for all subsequent governance votes and reward calculations. The corruption is permanent in stable state until the neuron is dissolved. This constitutes a concrete NNS governance integrity impact — an attacker gains disproportionate voting influence and inflated neuron rewards — matching the **High** bounty tier for significant NNS security impact with concrete user and protocol harm.

## Likelihood Explanation

The precondition is controlling (or having hotkey access to) a `NotDissolving` neuron with `aging_since_timestamp_seconds > now_seconds`. The IC team's own test suite explicitly covers this case, confirming it is a known reachable state from legacy neurons. The `from_api` conversion for `DissolveStateAndAge::NotDissolving` imposes no constraint that `aging_since_timestamp_seconds <= now_seconds`. The merge is triggered via the public `manage_neuron` endpoint with no special privileges beyond neuron ownership. The constraint of requiring such a legacy neuron limits likelihood, but the attack requires no technical sophistication once the neuron is in hand.

## Recommendation

Replace the plain subtraction with `saturating_sub` in both `try_new` implementations, consistent with `DissolveStateAndAge::age_seconds`:

```diff
- age_seconds: now_seconds - aging_since_timestamp_seconds,
+ age_seconds: now_seconds.saturating_sub(aging_since_timestamp_seconds),
```

Apply at:
- `rs/nns/governance/src/governance/merge_neurons.rs` line 438 (`ValidSourceNeuron::try_new`)
- `rs/nns/governance/src/governance/merge_neurons.rs` line 480 (`ValidTargetNeuron::try_new`)

## Proof of Concept

1. Create `NotDissolving` neuron `A` with `aging_since_timestamp_seconds = now_seconds + 1` (a valid legacy state per the test suite).
2. Create a second compatible `NotDissolving` neuron `B` with the same controller and non-zero stake.
3. Call `manage_neuron` → `Merge { source_neuron_id: A }` targeting `B`.
4. In `ValidSourceNeuron::try_new`: `now_seconds - (now_seconds + 1)` wraps to `u64::MAX` in release Wasm.
5. `combine_aged_stakes(B_stake, B_age, A_stake, u64::MAX)` produces `new_target_age_seconds` near `u64::MAX`.
6. `now_seconds.saturating_sub(u64::MAX)` saturates to `0`.
7. Neuron `B` is written with `aging_since_timestamp_seconds = 0`, giving it the maximum 1.25× age bonus on its full stake for all future voting power and reward calculations.

A deterministic unit test can reproduce this by calling `calculate_merge_neurons_effect` with a mock neuron store containing a source neuron where `aging_since_timestamp_seconds = now_seconds + 1`, asserting that the resulting `target_neuron_dissolve_state_and_age` has `aging_since_timestamp_seconds = 0` (the corrupted value) rather than the correct `now_seconds` (age = 0).
