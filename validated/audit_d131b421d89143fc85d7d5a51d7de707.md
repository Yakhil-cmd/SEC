### Title
Engine Pause Blocks `exit_to_near_precompile_callback` Refund Path, Causing Permanent EVM Token Loss - (`engine/src/contract_methods/connector.rs`)

### Summary

When the Aurora Engine is paused via `pause_contract`, the callback `exit_to_near_precompile_callback` cannot execute its refund branch. Because EVM token burning happens in the initial synchronous transaction (before the pause) and the refund is deferred to an asynchronous NEAR callback, a pause applied between those two NEAR transactions permanently destroys user EVM funds with no recovery path.

### Finding Description

The `EXIT_TO_NEAR` precompile flow is split across two asynchronous NEAR transactions:

1. **Initial EVM transaction** (`submit`/`call`): The precompile burns the user's EVM tokens and schedules a NEP-141 `ft_transfer_call` promise to the destination NEAR account.
2. **Callback transaction** (`exit_to_near_precompile_callback`): Runs after the NEP-141 promise settles. If the promise **failed** (e.g., receiver not storage-registered), this callback is supposed to call `engine::refund_on_error` to restore the burned EVM tokens.

The callback unconditionally calls `require_running` as its very first check: [1](#0-0) 

```rust
pub fn exit_to_near_precompile_callback<...>(...) -> Result<Option<SubmitResult>, ContractError> {
    with_hashchain(io, env, function_name!(), |io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;          // ← blocks everything if engine is paused
        env.assert_private_call()?;
        ...
        } else if let Some(args) = args.refund {
            // Refund EVM tokens — NEVER REACHED when engine is paused
            let refund_result = engine::refund_on_error(io, env, state, &args, handler)?;
``` [2](#0-1) 

`require_running` is defined as: [3](#0-2) 

If the engine owner calls `pause_contract` between the initial EVM transaction and the callback settlement, `require_running` returns `ERR_PAUSED` and the entire callback aborts. In NEAR's async model, the base promise (NEP-141 transfer attempt) is a separate, already-committed transaction — its failure is not rolled back by the callback's abort. The EVM token burn from step 1 is also already committed. The refund branch is the only recovery mechanism, and it is now permanently unreachable for the duration of the pause. [4](#0-3) 

### Impact Explanation

**Critical — Permanent freezing of funds.**

When the engine is paused while a `EXIT_TO_NEAR` callback is pending and the underlying NEP-141 transfer has failed:
- EVM tokens are already burned (committed in the initial transaction).
- NEP-141 tokens were never delivered (the promise failed).
- The refund path inside `exit_to_near_precompile_callback` is blocked by `require_running`.
- NEAR does not retry callbacks; the callback executes exactly once.
- Result: the user's EVM tokens are permanently destroyed with no compensation.

### Likelihood Explanation

**Medium.** Emergency pauses are a realistic operational event (security incidents, bridge exploits, etc.). The window between the initial EVM transaction and the callback settlement is one NEAR block (~1 second), but under high load or during a targeted attack that forces many EXIT_TO_NEAR calls just before a pause, multiple users can be affected simultaneously. The NEP-141 transfer failing is also realistic: the destination account may not have storage registered with the NEP-141 contract.

### Recommendation

Remove `require_running` from `exit_to_near_precompile_callback`, or restructure the check so that the **refund branch executes unconditionally** regardless of the engine's pause state. The refund path only restores the user's own previously-burned EVM balance and does not introduce new state mutations that would be unsafe during a pause. A targeted fix:

```rust
pub fn exit_to_near_precompile_callback<...>(...) {
    with_hashchain(io, env, function_name!(), |io| {
        let state = state::get_state(&io)?;
        // Do NOT require_running here — refund must be possible even when paused
        env.assert_private_call()?;
        ...
        } else if let Some(args) = args.refund {
            let refund_result = engine::refund_on_error(io, env, state, &args, handler)?;
            ...
        }
    })
}
```

Alternatively, add a dedicated `require_running_or_refund` guard that only allows the refund branch through when paused.

### Proof of Concept

1. User submits an EVM transaction calling a contract that invokes the `EXIT_TO_NEAR` precompile with `N` wei. The precompile burns `N` wei from the user's EVM balance and schedules a NEP-141 `ft_transfer_call` to a NEAR account that has **not** registered storage with the NEP-141 contract.
2. The initial NEAR transaction commits. EVM balance is now reduced by `N`.
3. Engine owner calls `pause_contract`. Engine state: `is_paused = true`.
4. The NEP-141 `ft_transfer_call` promise executes and **fails** (storage not registered).
5. NEAR runtime invokes `exit_to_near_precompile_callback` as the attached callback.
6. `require_running` reads `is_paused = true` → returns `Err(ERR_PAUSED)`.
7. The callback aborts. The `refund_on_error` call is never reached.
8. User has lost `N` wei of EVM balance permanently. No NEP-141 tokens were received. [5](#0-4) [3](#0-2) [4](#0-3)

pub fn exit_to_near_precompile_callback<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<Option<SubmitResult>, ContractError> {
    with_hashchain(io, env, function_name!(), |io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;
        env.assert_private_call()?;

        // This function should only be called as the callback of
        // exactly one promise.
        if handler.promise_results_count() != 1 {
            return Err(errors::ERR_PROMISE_COUNT.into());
        }

        let args: ExitToNearPrecompileCallbackArgs = io.read_input_borsh()?;

        let maybe_result = if let Some(PromiseResult::Successful(_)) = handler.promise_result(0) {
            if let Some(args) = args.transfer_near {
                let action = PromiseAction::Transfer {
                    amount: Yocto::new(args.amount),
                };
                let promise = PromiseBatchAction {
                    target_account_id: args.target_account_id,
                    actions: vec![action],
                };

                // Safety: this call is safe because it comes from the exit to near precompile, not users.
                // The call is to transfer the unwrapped wNEAR tokens.
                let promise_id = handler.promise_create_batch(&promise);
                handler.promise_return(promise_id);
            }

            None
        } else if let Some(args) = args.refund {
            // Exit call failed; need to refund tokens
            let refund_result = engine::refund_on_error(io, env, state, &args, handler)?;

            if !refund_result.status.is_ok() {
                return Err(errors::ERR_REFUND_FAILURE.into());
            }

            Some(refund_result)
        } else {
            None
        };

        Ok(maybe_result)
    })
}

#[named]
pub fn pause_contract<I: IO + Copy, E: Env>(io: I, env: &E) -> Result<(), ContractError> {
    with_hashchain(io, env, function_name!(), |mut io| {
        let mut state = state::get_state(&io)?;
        require_owner_only(&state, &env.predecessor_account_id())?;
        require_running(&state)?;
        state.is_paused = true;
        state::set_state(&mut io, &state)?;
        Ok(())
    })
}
