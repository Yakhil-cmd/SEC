### Title
Missing Worst-Case Maturity Modulation in Minimum Disbursement Check — (`rs/nns/governance/src/governance/disburse_maturity.rs`)

### Summary
In `initiate_maturity_disbursement`, the minimum disbursement guard is applied to the raw maturity amount (`disbursement_maturity_e8s`) rather than the post-modulation minted amount. The code comment explicitly states the minimum is intended to guarantee at least `MINIMUM_DISBURSEMENT_E8S` ICP is **minted** "assuming the worst case maturity modulation," but the check never applies that modulation. A neuron owner can therefore initiate a disbursement that, after the mandatory 7-day delay and worst-case modulation (−5 %), results in minting less than the stated minimum.

### Finding Description
`MINIMUM_DISBURSEMENT_E8S` is defined as `E8 = 100_000_000` (1 ICP) and its comment reads:

> "The minimum amount of ICP that need to be **minted** when disbursing maturity. A neuron can only disburse an amount of maturity that results in minting at least this many ICP (in e8) **assuming the worst case maturity modulation**." [1](#0-0) 

The guard in `initiate_maturity_disbursement` checks the raw maturity:

```rust
if disbursement_maturity_e8s < MINIMUM_DISBURSEMENT_E8S {
    return Err(InitiateMaturityDisbursementError::DisbursementTooSmall { … });
}
``` [2](#0-1) 

Seven days later, `next_maturity_disbursement_to_finalize` applies the live maturity modulation (which can be as low as `MIN_MATURITY_MODULATION_PERMYRIAD = −500` basis points, i.e. −5 %):

```rust
let maturity_to_disburse_after_modulation_e8s = apply_maturity_modulation(
    original_maturity_e8s_equivalent,
    maturity_modulation_basis_points,
)?;
``` [3](#0-2)

The check is therefore applied to variable A (`disbursement_maturity_e8s`, the raw maturity), while the amount actually minted is variable B (`maturity_to_disburse_after_modulation_e8s`), which is never compared against `MINIMUM_DISBURSEMENT_E8S`. This is the direct IC analog of the Solana report's pattern: the guard runs before the branching transformation, so the post-transformation output is unchecked.

Contrast with SNS governance's `disburse_maturity`, which correctly applies worst-case modulation before its guard:

```rust
let worst_case_maturity_modulation =
    apply_maturity_modulation(maturity_to_deduct, MIN_MATURITY_MODULATION_PERMYRIAD)?;
if worst_case_maturity_modulation < transaction_fee_e8s { … }
``` [5](#0-4) 

NNS governance performs no equivalent worst-case pre-check.

### Impact Explanation
A neuron owner who holds exactly `MINIMUM_DISBURSEMENT_E8S` (1 ICP) of maturity passes the guard, but after the 7-day delay the minter receives only `1 ICP × 0.95 = 0.95 ICP` under worst-case modulation — 5 % below the system-stated minimum. The minted ICP is permanently below the floor the protocol advertises, violating the ledger conservation invariant the constant is meant to enforce.

### Likelihood Explanation
The maturity modulation is fetched from the CMC and can legitimately reach −500 basis points during periods of low ICP/XDR price. Any neuron owner with maturity in the range `[MINIMUM_DISBURSEMENT_E8S, MINIMUM_DISBURSEMENT_E8S / 0.95)` (i.e. between 1 ICP and ~1.053 ICP) can trigger this condition without any privileged access — they simply call `DisburseMaturity` with `percentage_to_disburse = 100`. No admin key, governance majority, or threshold attack is required.

### Recommendation
Replace the raw-maturity guard with a worst-case-modulated check, mirroring the SNS governance pattern:

```rust
let worst_case_minted = apply_maturity_modulation(
    disbursement_maturity_e8s,
    MIN_MATURITY_MODULATION_PERMYRIAD,
)?;
if worst_case_minted < MINIMUM_DISBURSEMENT_E8S {
    return Err(InitiateMaturityDisbursementError::DisbursementTooSmall {
        disbursement_maturity_e8s,
        minimum_disbursement_e8s: MINIMUM_DISBURSEMENT_E8S,
    });
}
```

This ensures the minted amount is always ≥ `MINIMUM_DISBURSEMENT_E8S` regardless of the modulation applied at finalization time, matching the stated intent of the constant.

### Proof of Concept
1. Neuron owner holds `maturity_e8s_equivalent = 100_000_000` (exactly 1 ICP).
2. Calls `manage_neuron` → `DisburseMaturity { percentage_to_disburse: 100, … }`.
3. `disbursement_maturity_e8s = 100_000_000`; guard check `100_000_000 < 100_000_000` is **false** → disbursement is accepted and maturity is deducted from the neuron.
4. Seven days later, `finalize_maturity_disbursement` runs with `maturity_modulation_basis_points = −500`.
5. `apply_maturity_modulation(100_000_000, −500)` → `100_000_000 × 9500 / 10000 = 95_000_000 e8s = 0.95 ICP`.
6. The ICP ledger mints **0.95 ICP** to the destination account — 5 % below the `MINIMUM_DISBURSEMENT_E8S` floor the protocol guarantees.
