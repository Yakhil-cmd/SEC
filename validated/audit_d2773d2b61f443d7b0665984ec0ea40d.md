### Title
Unchecked `u64`‚Üí`i64` Cast in `compute_gas_refund` Silently Bypasses Native Resource Accounting ‚ÄĒ (File: `basic_bootloader/src/bootloader/transaction_flow/refund_calculation.rs`)

---

### Summary

`compute_gas_refund` computes a `delta_gas` adjustment to ensure that native resource consumption is reflected in the gas counter, preventing users from receiving unearned gas refunds. The computation casts a `u64` quotient to `i64` with an unchecked `as i64`. If the quotient exceeds `i64::MAX`, the cast silently wraps to a negative value, the overflow-detection check `if delta_gas > 0` silently passes without triggering, and `gas_used` is never bumped up. The user then receives a larger ETH refund than they are entitled to.

---

### Finding Description

In `compute_gas_refund`:

```rust
// line 62-64
let full_native_limit = if cfg!(feature = "unlimited_native") || native_per_gas == 0 {
    u64::MAX - 1
} else {
    gas_limit.saturating_mul(native_per_gas)
};
let native_used = full_native_limit.saturating_sub(resources.native().remaining().as_u64());

// line 69-79
let delta_gas = if native_per_gas == 0 {
    0
} else {
    (native_used / native_per_gas) as i64 - (gas_used as i64)  // ‚Üź unchecked cast
};

if delta_gas > 0 {
    gas_used += delta_gas as u64;
}
```

Both `native_used` and `native_per_gas` are `u64`. The division `native_used / native_per_gas` is a `u64` result. The `as i64` cast is a **silent narrowing conversion**: if the quotient exceeds `i64::MAX` (~9.2 √ó 10¬Ļ‚Āł), the bit pattern is reinterpreted as a negative `i64` with no error, no panic, and no indication of overflow.

This is the direct structural analog of the `addSignedDelta` bug: in that case, Solidity assembly operated on 256-bit words while the logical types were 128-bit, so the overflow check `slt(output, input)` could never fire. Here, the logical type is `u64` but the overflow check is performed in the `i64` domain ‚ÄĒ if the `u64` value exceeds `i64::MAX`, the check `if delta_gas > 0` silently fails to fire.

The purpose of `delta_gas` is to prevent a user from receiving a gas refund for native resources they actually consumed. If the check is bypassed, `gas_used` is not incremented, `total_gas_refund = gas_limit - gas_used` is inflated, and the user is refunded ETH for gas they did not actually have left over. [1](#0-0) 

---

### Impact Explanation

When `native_used / native_per_gas > i64::MAX`:

1. `(native_used / native_per_gas) as i64` wraps to a large negative `i64`.
2. `delta_gas` is negative (or far below zero).
3. `if delta_gas > 0` evaluates to `false` ‚ÄĒ the guard that was supposed to catch this case silently passes.
4. `gas_used` is **not** incremented to reflect actual native consumption.
5. `total_gas_refund = gas_limit - gas_used` is larger than it should be.
6. The user receives an inflated ETH refund, effectively underpaying for native (proving) resource consumption.

This is a **resource accounting bug** with a direct financial impact: the operator/protocol absorbs the cost of native resources that the user consumed but was not charged for. [2](#0-1) 

---

### Likelihood Explanation

The overflow requires `native_used / native_per_gas > i64::MAX`. Because `native_used ‚Č§ gas_limit √ó native_per_gas` (saturating), this reduces to `gas_limit > i64::MAX` (~9.2 √ó 10¬Ļ‚Āł). `gas_limit` is a `u64` parameter with no in-function guard. Exploitability therefore depends on whether `MAX_TX_GAS_LIMIT` (defined in `zk_ee/src/system/mod.rs` and enforced during transaction validation) is strictly less than `i64::MAX`. If that constant is set to a value ‚Č§ `i64::MAX`, the overflow cannot be triggered in the current codebase. However:

- The code contains **no local assertion or checked cast** to enforce this invariant at the point of use.
- The `as i64` cast is the only such cast in all of `basic_bootloader/**/*.rs` (confirmed by grep), making it an isolated, unguarded conversion.
- Any future increase to `MAX_TX_GAS_LIMIT`, or any call path that bypasses the limit check, would immediately expose the overflow. [3](#0-2) 

---

### Recommendation

Replace the unchecked `as i64` casts with explicit checked conversions and handle the overflow case:

```rust
let delta_gas: i64 = if native_per_gas == 0 {
    0
} else {
    let native_gas_equiv = native_used / native_per_gas;
    // If native_gas_equiv > i64::MAX, the user consumed more native than
    // any gas counter can represent ‚ÄĒ treat as maximum positive delta.
    let native_gas_i64 = i64::try_from(native_gas_equiv).unwrap_or(i64::MAX);
    let gas_used_i64 = i64::try_from(gas_used).unwrap_or(i64::MAX);
    native_gas_i64.saturating_sub(gas_used_i64)
};
```

Additionally, add a `debug_assert!(gas_limit <= i64::MAX as u64)` at the entry of `compute_gas_refund` to make the implicit invariant explicit and catch regressions in tests.

---

### Proof of Concept

1. Submit a transaction with `gas_limit > i64::MAX` (requires `MAX_TX_GAS_LIMIT` to permit it, or use an L1 transaction path that saturates rather than rejects).
2. Set `gas_price = native_price` so `native_per_gas = 1`.
3. Execute a workload that exhausts all native resources (`resources.native().remaining() = 0`).
4. At refund time: `native_used = gas_limit √ó 1 = gas_limit > i64::MAX`.
5. `(native_used / 1) as i64` wraps to a negative `i64`.
6. `delta_gas < 0`; the `if delta_gas > 0` branch is skipped.
7. `gas_used` is not adjusted upward; `total_gas_refund = gas_limit - gas_used` is inflated.
8. The user receives an ETH refund proportional to the inflated `total_gas_refund √ó gas_price`, recovering funds for native resources they actually consumed. [4](#0-3)

pub(crate) fn compute_gas_refund<S: EthereumLikeTypes>(
    system: &mut System<S>,
    to_charge_for_pubdata: S::Resources,
    gas_limit: u64,
    minimal_gas_used: u64,
    native_per_gas: u64,
    resources: &mut S::Resources,
) -> Result<RefundInfo, InternalError> {
    // Already checked
    resources.charge_unchecked(&to_charge_for_pubdata);

    let mut gas_used = gas_limit
        .checked_sub(resources.ergs().0.div_floor(ERGS_PER_GAS))
        .ok_or(internal_error!("gas remaining > gas limit"))?;
    resources.exhaust_ergs();

    system_log!(system, "Gas used before refund calculations: {gas_used}\n");

    // Following EIP-3529, refunds are capped to 1/5 of the gas used
    let evm_refund = {
        let full_refund_ergs = system.io.get_refund_counter().ergs();
        let full_refund_gas = full_refund_ergs.0.div_floor(ERGS_PER_GAS);
        let max_refund = gas_used / 5;
        core::cmp::min(full_refund_gas, max_refund)
    };

    system_log!(system, "Gas refund from refund counters = {evm_refund}\n");

    gas_used -= evm_refund;

    system_log!(
        system,
        "Minimal gas used from validation = {minimal_gas_used}\n"
    );

    #[allow(unused_mut)]
    let mut gas_used = core::cmp::max(gas_used, minimal_gas_used);

    // Note: for zero gas price, we use "unlimited native"
    let full_native_limit = if cfg!(feature = "unlimited_native") || native_per_gas == 0 {
        u64::MAX - 1
    } else {
        gas_limit.saturating_mul(native_per_gas)
    };
    let native_used = full_native_limit.saturating_sub(resources.native().remaining().as_u64());

    #[cfg(not(feature = "unlimited_native"))]
    {
        // Adjust gas_used with difference with used native
        let delta_gas = if native_per_gas == 0 {
            0
        } else {
            (native_used / native_per_gas) as i64 - (gas_used as i64)
        };

        if delta_gas > 0 {
            // In this case, the native resource consumption is more than the
            // gas consumption accounted for. Consume extra gas.
            gas_used += delta_gas as u64;
        }
        // TODO: return delta_gas to gas_used?
    }

    let total_gas_refund = gas_limit - gas_used;
    system_log!(system, "Refund after accounting for unused gas, refund counters and native cost: {total_gas_refund}\n");
    require_internal!(
        total_gas_refund <= gas_limit,
        "Gas refund greater than gas limit",
        system
    )?;
    let refund_info = RefundInfo {
        gas_used,
        evm_refund,
        native_used,
    };
    system_log!(system, "Final gas used: {gas_used}\n");
    Ok(refund_info)
}
