### Title
Unprotected `new()` Initialization Allows Any Caller to Seize Engine Ownership — (`File: engine/src/contract_methods/admin.rs`)

---

### Summary

The Aurora Engine's `new()` initialization function sets the `owner_id` of the engine state without verifying who the caller is. Any NEAR account can call `new()` before the legitimate deployer does, installing themselves as `owner_id` and gaining full administrative control over the engine, including the ability to deploy arbitrary upgrade code and drain all user funds.

---

### Finding Description

`new()` in `engine/src/contract_methods/admin.rs` is the sole initialization entry point for the Aurora Engine contract. It reads caller-supplied `NewCallArgs` (which include `owner_id`, `chain_id`, `upgrade_delay_blocks`, and `key_manager`) and writes them directly into `EngineState` storage.

The only guard present is a re-initialization check:

```rust
pub fn new<I: IO + Copy, E: Env>(mut io: I, env: &E) -> Result<(), ContractError> {
    if state::get_state(&io).is_ok() {
        return Err(b"ERR_ALREADY_INITIALIZED".into());
    }
    // ...
    let state: EngineState = args.into();
    state::set_state(&mut io, &state)?;
    Ok(())
}
```

There is no check that `env.predecessor_account_id()` equals `env.current_account_id()` (the contract account itself) or any other privileged account. The `env` object is passed in but never queried for the caller's identity. [1](#0-0) 

The `EngineState` written by `new()` contains `owner_id`, which gates every privileged operation in the engine: [2](#0-1) 

`owner_id` is the sole authorization check for `upgrade()`, `stage_upgrade()`, `set_owner()`, `pause_contract()`, `resume_contract()`, `set_eth_connector_contract_account()`, `factory_update()`, `attach_full_access_key()`, and more: [3](#0-2) [4](#0-3) 

The `upgrade()` function, gated only by `require_owner_only`, deploys arbitrary WASM code to the engine contract account: [4](#0-3) 

The `new()` function is exposed as a public NEAR contract method with no access restriction: [5](#0-4) 

---

### Impact Explanation

**Critical — Direct theft of all user funds.**

An attacker who wins the race to call `new()` first becomes `owner_id`. They can then immediately call `upgrade()` to deploy arbitrary WASM to the `aurora` contract account. This arbitrary code can:
- Transfer all ETH balances held in the engine to the attacker.
- Drain all ERC-20/NEP-141 bridged token balances.
- Permanently brick the engine for all users.

All user funds at rest in the Aurora Engine (ETH, bridged ERC-20 tokens) are at risk of total, irreversible theft.

---

### Likelihood Explanation

**High.** On NEAR Protocol, contract deployment and initialization are two separate transactions. The `deploy_and_init` flow in `engine-workspace/src/lib.rs` confirms this — `deploy()` is called first, then `new()` is called as a separate transaction: [6](#0-5) 

Between these two transactions there is an observable window. Any NEAR account monitoring the network (e.g., via an indexer or RPC polling) can detect the contract deployment and submit a `new()` call targeting the freshly deployed but uninitialized contract. NEAR does not have gas-price-based ordering, but transaction submission order within a block is not guaranteed to favor the deployer. The attack requires no special privileges, no funds, and no prior relationship with the contract — only the ability to submit a NEAR transaction.

---

### Recommendation

Add a caller check at the top of `new()` that requires `env.predecessor_account_id() == env.current_account_id()`. On NEAR, the contract account itself is the only account that can be guaranteed to be the deployer's intended caller when deployment and initialization are batched atomically. Alternatively, perform deployment and initialization in a single atomic NEAR batch action so no external call to `new()` is possible before the state is set.

```rust
pub fn new<I: IO + Copy, E: Env>(mut io: I, env: &E) -> Result<(), ContractError> {
    if state::get_state(&io).is_ok() {
        return Err(b"ERR_ALREADY_INITIALIZED".into());
    }
    // Add: only the contract account itself may initialize
    if env.predecessor_account_id() != env.current_account_id() {
        return Err(b"ERR_NOT_ALLOWED".into());
    }
    // ...
}
```

### Proof of Concept

1. Aurora team deploys the Aurora Engine WASM to the `aurora` NEAR account (transaction T1).
2. Attacker observes T1 via NEAR RPC/indexer. The contract is now deployed but `EngineState` storage is empty (`get_state` returns `Err(NotFound)`).
3. Attacker submits transaction T2 calling `new()` on `aurora` with `NewCallArgs` containing `owner_id = attacker.near`, before the Aurora team's own `new()` call lands.
4. T2 succeeds: `EngineState { owner_id: "attacker.near", ... }` is written to storage.
5. Aurora team's `new()` call (T3) now returns `ERR_ALREADY_INITIALIZED` and fails.
6. Attacker calls `stage_upgrade()` with malicious WASM, waits `upgrade_delay_blocks` (which they set to 0 in their `NewCallArgs`), then calls `upgrade()`.
7. Malicious WASM is deployed to the `aurora` account. Attacker drains all user ETH and ERC-20 balances.

pub fn new<I: IO + Copy, E: Env>(mut io: I, env: &E) -> Result<(), ContractError> {
    if state::get_state(&io).is_ok() {
        return Err(b"ERR_ALREADY_INITIALIZED".into());
    }

    let input = io.read_input().to_vec();
    let args = NewCallArgs::deserialize(&input).map_err(|_| errors::ERR_BORSH_DESERIALIZE)?;

    let initial_hashchain = args.initial_hashchain();
    let state: EngineState = args.into();

    if let Some(block_hashchain) = initial_hashchain {
        let block_height = env.block_height();
        let mut hashchain = Hashchain::new(
            state.chain_id,
            env.current_account_id(),
            block_height,
            block_hashchain,
        );

        hashchain.add_block_tx(
            block_height,
            function_name!(),
            &input,
            &[],
            &Bloom::default(),
        )?;
        crate::hashchain::save_hashchain(&mut io, &hashchain)?;
    }

    state::set_state(&mut io, &state)?;
    Ok(())
}

pub fn upgrade<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<(), ContractError> {
    let state = state::get_state(&io)?;
    require_running(&state)?;
    require_owner_only(&state, &env.predecessor_account_id())?;

    let input = io.read_input().to_vec();
    let (code, state_migration_gas) = match UpgradeParams::try_from_slice(&input) {
        Ok(args) => (
            args.code,
            args.state_migration_gas
                .map_or(GAS_FOR_STATE_MIGRATION, NearGas::new),
        ),
        Err(_) => (input, GAS_FOR_STATE_MIGRATION), // Backward compatibility
    };

    let target_account_id = env.current_account_id();
    let batch = PromiseBatchAction {
        target_account_id,
        actions: vec![
            PromiseAction::DeployContract { code },
            PromiseAction::FunctionCall {
                name: "state_migration".to_string(),
                args: vec![],
                attached_yocto: ZERO_YOCTO,
                gas: state_migration_gas,
            },
        ],
    };
    let promise_id = handler.promise_create_batch(&batch);

    handler.promise_return(promise_id);

    Ok(())
