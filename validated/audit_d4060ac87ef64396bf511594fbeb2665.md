Audit Report

## Title
Stale 5% Worst-Case Maturity Modulation Pre-Check in `spawn_neuron` Allows Below-Minimum-Stake Neuron Creation - (File: `rs/nns/governance/src/governance.rs`)

## Summary

`spawn_neuron` gates neuron creation using a hardcoded 5% worst-case maturity modulation, but `maybe_spawn_neurons` now applies the Mission 70 modulation whose lower bound is −10% (−1,000 permyriad). The pre-check and execution path are mismatched: a neuron controller can pass the pre-check with maturity that produces a below-minimum-stake neuron at execution time, permanently violating the NNS protocol invariant that every neuron holds at least `neuron_minimum_stake_e8s`.

## Finding Description

In `rs/nns/governance/src/governance.rs` at line 2666, `spawn_neuron` computes the minimum possible spawned stake as:

```rust
let least_possible_stake = (maturity_to_spawn as f64 * (1_f64 - 0.05)) as u64;
``` [1](#0-0) 

This hardcodes 5% as the worst-case modulation. However, `MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70` is defined as −1,000 permyriad (−10%): [2](#0-1) 

`maybe_spawn_neurons` reads `heap_data.maturity_modulation`, validates it against `VALID_MATURITY_MODULATION_BASIS_POINTS_RANGE` (which uses the Mission 70 bounds), and passes the actual value directly to `apply_maturity_modulation`: [3](#0-2) [4](#0-3) 

`apply_maturity_modulation` computes `amount * (10_000 + basis_points) / 10_000`, so at −1,000 permyriad the spawned stake is 90% of maturity: [5](#0-4) 

The result is directly written to `cached_neuron_stake_e8s` with no post-spawn minimum-stake check: [6](#0-5) 

The CHANGELOG confirms the switch to Mission 70 modulation for spawning (Proposal 141779): [7](#0-6) 

The pre-check constant was never updated to match the new −10% floor. By contrast, SNS `disburse_maturity` correctly calls `apply_maturity_modulation` with `MIN_MATURITY_MODULATION_PERMYRIAD` for its pre-check: [8](#0-7) 

## Impact Explanation

An unprivileged neuron controller can permanently create a neuron whose `cached_neuron_stake_e8s` is below `neuron_minimum_stake_e8s`, violating the core NNS protocol invariant. With `neuron_minimum_stake_e8s = 100_000_000 e8s` (1 ICP), any maturity `M` in `[105_263_158, 111_111_111)` e8s passes the pre-check (`M * 0.95 >= 100_000_000`) but produces a sub-minimum neuron at execution (`M * 0.90 < 100_000_000`). The neuron is permanently minted in this state with no correction path. This constitutes a **High** impact: significant NNS governance security impact with concrete protocol harm — a permanently invariant-violating neuron created via a standard unprivileged `manage_neuron` ingress call.

## Likelihood Explanation

The Mission 70 modulation moves at most 30 permyriad/day. Reaching −500 permyriad (the old floor) takes approximately 17 days from zero; reaching −1,000 permyriad takes approximately 33 days. Once below −500 permyriad, any neuron controller with maturity in the ~5.5% vulnerable window can trigger the bug with a standard `manage_neuron` → `Spawn` call. No privileged access is required. The condition is realistic for active NNS participants and repeatable for any controller in the window.

## Recommendation

Replace the hardcoded `0.05` with the actual Mission 70 worst-case modulation constant, mirroring the SNS pattern:

```rust
let least_possible_stake = apply_maturity_modulation(
    maturity_to_spawn,
    MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 as i32,
).unwrap_or(0);

if least_possible_stake < economics.neuron_minimum_stake_e8s {
    return Err(GovernanceError::new_with_message(
        ErrorType::InsufficientFunds,
        "There isn't enough maturity to spawn a new neuron due to worst case maturity modulation.",
    ));
}
```

This ensures the pre-check and execution path use the same worst-case bound.

## Proof of Concept

1. Wait for Mission 70 maturity modulation to reach below −500 permyriad (e.g., −600 permyriad after ~20 days of ICP price decline).
2. As a neuron controller with `maturity_e8s_equivalent = 106_000_000` e8s (1.06 ICP):
   - Pre-check: `106_000_000 * 0.95 = 100_700_000 >= 100_000_000` → **passes**.
3. Call `manage_neuron` → `Spawn { percentage_to_spawn: 100, ... }`. The child neuron enters spawning state with `maturity_e8s_equivalent = 106_000_000`.
4. When `maybe_spawn_neurons` fires with modulation = −600 permyriad:
   - `apply_maturity_modulation(106_000_000, -600)` = `106_000_000 * 9_400 / 10_000 = 99_640_000`.
5. The child neuron is minted with `cached_neuron_stake_e8s = 99_640_000 < 100_000_000 = neuron_minimum_stake_e8s`.

A deterministic integration test can reproduce this by: (a) setting `heap_data.maturity_modulation.current_value_permyriad = -600`, (b) creating a spawning neuron with `maturity_e8s_equivalent = 106_000_000`, (c) calling `maybe_spawn_neurons`, and (d) asserting `cached_neuron_stake_e8s < neuron_minimum_stake_e8s`.

```rust
pub fn apply_maturity_modulation(
    amount_maturity_e8s: u64,
    maturity_modulation_basis_points: i32,
) -> Result<u64, String> {
    let amount_e8s = u128::from(amount_maturity_e8s);

    let adjusted_maturity_modulation_basis_points = saturating_add_or_subtract_u128_i32(
        BASIS_POINTS_PER_UNITY,
        maturity_modulation_basis_points,
    );

    let modulated_amount_e8s: u128 = amount_e8s
        .checked_mul(adjusted_maturity_modulation_basis_points)
        .ok_or_else(|| "Underflow or overflow when calculating maturity modulation".to_string())?
        .checked_div(BASIS_POINTS_PER_UNITY)
        .ok_or_else(|| "Underflow or overflow when calculating maturity modulation".to_string())?;

    u64::try_from(modulated_amount_e8s).map_err(|err| err.to_string())
}
```
