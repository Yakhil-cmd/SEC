Audit Report

## Title
SNS Governance Remains in `PreInitializationSwap` Mode After Aborted Swap, Locking Developer Neuron Holders Out of Neuron Management - (File: `rs/sns/swap/src/swap.rs`)

## Summary
When an SNS token swap is aborted, `finalize_inner` restores dapp controllers and returns early at line 1583 without calling `set_sns_governance_to_normal_mode`. SNS Governance remains permanently in `Mode::PreInitializationSwap`, blocking developer neuron holders from disbursing, splitting, configuring, or managing maturity on their neurons. There is no user-accessible path to exit this mode; the only remedy is an NNS-initiated canister upgrade.

## Finding Description
SNS Governance initializes in `Mode::PreInitializationSwap` (confirmed in `rs/sns/governance/src/init.rs` line 18). The `allows_manage_neuron_command_or_err` function in `rs/sns/governance/src/types.rs` (lines 163–211) blocks all `manage_neuron` commands except `Follow`, `MakeProposal`, `RegisterVote`, `AddNeuronPermissions`, `RemoveNeuronPermissions`, and swap-canister-only `ClaimOrRefresh` — all others, including `Disburse`, `Split`, `Configure`, `MergeMaturity`, and `DisburseMaturity`, return `Err(PreconditionFailed)`.

In `rs/sns/swap/src/swap.rs`, `finalize_inner` (lines 1544–1624) checks `should_restore_dapp_control()` at line 1572, which returns `true` whenever `lifecycle() == Lifecycle::Aborted` (line 1348–1350). On the abort path, the function restores dapp controllers and unconditionally returns at line 1583. The `set_sns_governance_to_normal_mode` call at lines 1610–1612 is only reachable on the committed (successful) path.

`set_mode` in `rs/sns/governance/src/governance.rs` (lines 785–801) can only be called by the swap canister and only to set `Normal` mode — there is no governance proposal action or user-callable method to exit `PreInitializationSwap` mode.

This is not a latent code path: integration tests in `rs/nervous_system/integration_tests/tests/sns_lifecycle.rs` (lines 1118–1138 and 1268–1289) explicitly assert that after an aborted swap, governance mode remains `PreInitializationSwap` and that `start_dissolving_neuron` returns `PreconditionFailed` with the `PreInitializationSwap` message. The system-level test helper in `rs/tests/nns/sns/lib/src/swap_finalization.rs` (lines 123–125) also asserts this post-abort state.

## Impact Explanation
Developer neuron holders who received SNS tokens at genesis via `DeveloperDistribution` have their neurons permanently unmanageable after a failed swap. They cannot disburse tokens (even after dissolve delay elapses), split neurons, start or stop dissolving, change dissolve delay, or manage maturity. The only remedy is an NNS-initiated upgrade of the SNS governance canister, which is not user-accessible. This constitutes a significant SNS security impact with concrete user harm: legitimate token holders are locked out of their own assets by a normal protocol event. This maps to **High ($2,000–$10,000): Significant SNS security impact with concrete user or protocol harm**.

## Likelihood Explanation
Any SNS swap that fails to meet `min_participants` or `min_icp_e8s` before the deadline automatically transitions to `Lifecycle::Aborted` via `try_abort` on the heartbeat, then triggers `finalize_inner` via auto-finalization. This is a routine protocol event, not an edge case. It has occurred on mainnet. No attacker action is required — the condition is triggered by ordinary swap failure. Every affected SNS project's developer neuron holders are impacted automatically and without recourse.

## Recommendation
In `finalize_inner`, on the abort path (after restoring dapp controllers and before returning at line 1583), call `set_sns_governance_to_normal_mode`. The `PreInitializationSwap` restriction exists to protect swap integrity during the swap itself; after abort, there is no integrity reason to maintain it. Alternatively, introduce a dedicated `Aborted` governance mode that lifts neuron management restrictions while still preventing actions that assume a live swap.

## Proof of Concept
1. Deploy an SNS; SNS Governance initializes in `PreInitializationSwap` mode (`rs/sns/governance/src/init.rs` line 18). Developer neurons are created with tokens.
2. The SNS swap opens but fails to reach `min_participants` or `min_icp_e8s` before the deadline.
3. `try_abort` is called on heartbeat, setting `lifecycle = Aborted`.
4. `finalize` → `finalize_inner` is called. `should_restore_dapp_control()` returns `true` (`rs/sns/swap/src/swap.rs` line 1348). Dapp controllers are restored. The function returns at line 1583 — `set_sns_governance_to_normal_mode` (line 1610) is never called.
5. A developer neuron holder calls `manage_neuron` with `Command::Disburse` or `Command::Configure(StartDissolving)`.
6. `allows_manage_neuron_command_or_err` is called with `Mode::PreInitializationSwap` (`rs/sns/governance/src/types.rs` line 173); the command falls into the `_ => false` branch (line 196) and returns `Err(PreconditionFailed, "...PreInitializationSwap mode...")`.
7. This exact failure is reproducible by running the existing integration test in `rs/nervous_system/integration_tests/tests/sns_lifecycle.rs` with `SwapFinalizationStatus::Aborted`, which already asserts this error at lines 1268–1289.
