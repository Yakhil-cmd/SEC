### Title
`API_BOUNDARY_NODE_PRINCIPALS` Heap State Silently Cleared on Canister Upgrade Causes Temporary `RestrictedRead` Downgrade for All API Boundary Nodes — (`rs/boundary_node/rate_limits/canister/storage.rs`, `canister.rs`)

---

### Summary

The rate-limits canister stores the set of authorized API boundary node principals in thread-local heap memory (`API_BOUNDARY_NODE_PRINCIPALS`). Because the IC clears heap memory on canister upgrade and the post-upgrade hook only schedules a periodic timer (not an immediate poll), there is a guaranteed window — up to `registry_polling_period_secs` seconds — during which the set is empty. Any boundary node calling `get_config` during this window receives a `RestrictedRead` (redacted) response with `rule_raw=None` for all undisclosed rules, preventing enforcement of those rules.

---

### Finding Description

**Root cause — heap-only storage:**

`API_BOUNDARY_NODE_PRINCIPALS` is declared as a plain `thread_local!` `HashSet<Principal>` with no backing stable memory: [1](#0-0) 

All other security-critical state (`CONFIGS`, `RULES`, `INCIDENTS`, `AUTHORIZED_PRINCIPAL`) is backed by `StableBTreeMap` over `VirtualMemory`, which survives upgrades: [2](#0-1) 

**Root cause — no immediate poll on upgrade:**

`post_upgrade` simply calls `init`, which calls `periodically_poll_api_boundary_nodes`: [3](#0-2) 

`periodically_poll_api_boundary_nodes` uses `set_timer_interval`, which fires **after** the full interval, not immediately: [4](#0-3) 

The production upgrade proposal confirms `registry_polling_period_secs = 300` (5 minutes): [5](#0-4) 

**Access level degradation:**

`AccessLevelResolver::get_access_level` checks `is_api_boundary_node_principal`. With an empty set, every caller — including legitimate boundary nodes — falls through to `RestrictedRead`: [6](#0-5) 

**Redaction of undisclosed rules:**

In `ConfigGetter::get`, callers without `FullAccess` or `FullRead` have the `ConfigConfidentialityFormatter` applied: [7](#0-6) 

The formatter sets `rule_raw=None` and `description=None` for every rule whose `disclosed_at` is `None`: [8](#0-7) 

The `CanisterState` implementation confirms `is_api_boundary_node_principal` reads directly from the now-empty heap set: [9](#0-8) 

---

### Impact Explanation

During the post-upgrade window (up to 300 seconds in production), all API boundary nodes receive a redacted config. Any rate-limit rule that has not been publicly disclosed (`disclosed_at=None`) is returned with `rule_raw=None`. Boundary nodes cannot parse or enforce a rule with no raw bytes, so all undisclosed active rate-limit rules are silently dropped from enforcement. Traffic that those rules are intended to block or throttle passes through unimpeded for the duration of the window.

---

### Likelihood Explanation

This condition is triggered by every routine canister upgrade — a normal governance operation that happens regularly. No attacker action is required to create the window; an attacker only needs to send traffic to the IC during the predictable post-upgrade period. The window is deterministic and observable (the upgrade proposal is public). The `inspect_message` guard at line 48–55 of `canister.rs` only applies to update (replicated-query) calls; regular query calls to `get_config` bypass it entirely and return the redacted response silently. [10](#0-9) 

---

### Recommendation

1. **Persist `API_BOUNDARY_NODE_PRINCIPALS` in stable memory** using a `StableBTreeMap` with a dedicated `MemoryId`, consistent with how `AUTHORIZED_PRINCIPAL` is stored.
2. **Alternatively**, trigger an immediate one-shot poll at the start of `post_upgrade` using `ic_cdk_timers::set_timer(Duration::ZERO, ...)` before the interval timer is registered, so the set is populated before any query can be served.

---

### Proof of Concept

State-machine test outline:

1. Install canister with `registry_polling_period_secs = 300`.
2. Simulate a successful registry poll: populate `API_BOUNDARY_NODE_PRINCIPALS` with principal `P`.
3. Add a rate-limit config containing one undisclosed rule (`disclosed_at=None`).
4. Simulate canister upgrade (`post_upgrade`): heap is cleared, `API_BOUNDARY_NODE_PRINCIPALS` becomes empty.
5. Call `get_config` as principal `P` (before the 300-second timer fires).
6. Assert: response has `is_redacted=true` and `rule_raw=None` for the undisclosed rule.
7. Assert invariant violation: a principal that was a registered API boundary node before upgrade cannot read the full config immediately after upgrade.

        (
            Some(caller_id) == authorized_principal,
            state.is_api_boundary_node_principal(&caller_id),
        )
    });

    if called_method == REPLICATED_QUERY_METHOD {
        if has_full_access || has_full_read_access {
            ic_cdk::api::call::accept_message();
        } else {
            ic_cdk::api::trap(
                "message_inspection_failed: method call is prohibited in the current context",
            );
        }
    } else if UPDATE_METHODS.contains(&called_method.as_str()) {

            ic_cdk::api::trap("message_inspection_failed: unauthorized caller");
        }
    } else {
        // All others calls are rejected
        ic_cdk::api::trap(
            "message_inspection_failed: method call is prohibited in the current context",
        );
