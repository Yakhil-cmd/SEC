Audit Report

## Title
SNS Swap Canister Permanently Locks Pre-Loaded SNS Tokens on Abort - (`rs/sns/swap/src/swap.rs`)

## Summary
The SNS Swap canister is pre-funded with SNS tokens before opening. When a swap reaches `LIFECYCLE_ABORTED`, `finalize_inner` returns early after sweeping ICP back to buyers and restoring dapp controllers, but never transfers the pre-loaded SNS tokens back to the SNS governance canister. Those tokens remain permanently locked in the swap canister's SNS ledger account with no in-protocol recovery path.

## Finding Description
The abort path in `finalize_inner` is confirmed by the code: [1](#0-0) 

When `should_restore_dapp_control()` returns `true` (lifecycle is `Aborted`), the function restores dapp controllers and returns immediately. `sweep_sns` is never invoked on this path.

`sweep_sns` itself enforces a hard guard rejecting any non-`COMMITTED` lifecycle: [2](#0-1) 

A grep search for any alternative recovery function (`sweep_sns_to_governance`, `refund_sns`, `return.*sns.*governance`) returns zero matches in `swap.rs`. The `error_refund_icp` function handles only ICP. No function in the swap canister API transfers SNS tokens back to governance on abort.

The existing test `test_finalize_swap_abort_matched_funding` explicitly confirms this behavior — the SNS ledger is never called during abort finalization: [3](#0-2) 

The protocol documentation states the swap canister may be deleted only "when all tokens registered with the swap canister have been disbursed to their rightful owners": [4](#0-3) 

The lifecycle diagram shows `ABORTED → <DELETED>`: [5](#0-4) 

If the canister is deleted after abort (as the lifecycle diagram indicates is the intended terminal state), the SNS tokens become permanently inaccessible. Even without deletion, no in-protocol mechanism exists to recover them — recovery would require an out-of-band NNS upgrade proposal to the specific swap canister, which is not guaranteed and is not part of the normal finalization flow.

## Impact Explanation
Every SNS swap that reaches `LIFECYCLE_ABORTED` results in the entire pre-loaded SNS token allocation being locked in the swap canister's SNS ledger account. For any SNS project whose decentralization swap fails, the full token supply allocated for the swap is lost. This is a concrete, demonstrable permanent loss of SNS ledger assets — matching the allowed impact: **High ($2,000–$10,000): Significant SNS or infrastructure security impact with concrete user or protocol harm**, and potentially **Critical ($10,000–$50,000): Permanent loss of in-scope ledger assets** if the token allocation is large (which it typically is for a decentralization swap).

## Likelihood Explanation
Swap abortion is a normal, expected protocol outcome triggered whenever `min_participants` or `min_icp_e8s` is not reached before the swap deadline. No special privileges are required — any unprivileged user can cause a swap to abort simply by not participating. The condition is reachable on mainnet for any SNS project that fails to attract sufficient participation, which is a realistic and recurring scenario.

## Recommendation
In `finalize_inner`, before returning early on the `ABORTED` path, add a step to transfer the remaining SNS token balance from the swap canister's SNS ledger account back to the SNS governance canister's treasury account. This mirrors how `sweep_icp` returns ICP to buyers on abort:

```rust
if self.should_restore_dapp_control() {
    // Return pre-loaded SNS tokens to SNS governance treasury
    self.sweep_sns_to_governance(now_fn, environment.sns_ledger()).await;

    finalize_swap_response.set_set_dapp_controllers_result(
        self.restore_dapp_controllers_for_finalize(environment.sns_root_mut()).await,
    );
    return finalize_swap_response;
}
```

A new `sweep_sns_to_governance` function should transfer the full SNS token balance held by the swap canister's default account on the SNS ledger back to the SNS governance canister's main account.

## Proof of Concept
1. Deploy an SNS; its swap canister is pre-loaded with `N` SNS tokens in `PENDING` state.
2. The swap opens (`OPEN` state) but fails to reach `min_icp_e8s` or `min_participants` before the deadline.
3. `try_abort` is called, transitioning the swap to `ABORTED`.
4. `finalize` is called. `finalize_inner` executes: `sweep_icp` returns ICP to buyers; `should_restore_dapp_control()` returns `true`; `restore_dapp_controllers_for_finalize` is called; function returns early — `sweep_sns` is never invoked.
5. Query the SNS ledger for the swap canister's default account balance: it still holds `N` SNS tokens.
6. Confirm no function in the swap canister API can transfer those tokens out. The existing unit test at `rs/sns/swap/tests/swap.rs:1758` already asserts `clients.sns_ledger.get_calls_snapshot() == vec![]`, confirming zero SNS ledger calls during abort finalization. [6](#0-5) [7](#0-6)
