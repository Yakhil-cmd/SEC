### Title
SNS Governance `max_number_of_neurons` Exhaustion Griefing — No Rate Limiting on Neuron Creation - (`File: rs/sns/governance/src/governance.rs`)

---

### Summary

SNS governance enforces a hard cap (`max_number_of_neurons`) on the total number of neurons. Unlike NNS governance, which has a token-bucket rate limiter on neuron creation, SNS governance has no such protection. An unprivileged attacker holding only the minimum stake per neuron can create neurons in bulk until the cap is reached, permanently blocking all other principals from staking new neurons until the attacker voluntarily dissolves their neurons or the SNS community passes a governance proposal to raise the cap.

---

### Finding Description

**NNS governance** protects its analogous `MAX_NUMBER_OF_NEURONS = 500_000` limit with an in-memory token-bucket rate limiter:

```
MAX_SUSTAINED_NEURONS_PER_HOUR = 15
MAX_NEURON_CREATION_SPIKE      = 300   (burst budget)
```

Every `claim_neuron` call in NNS first calls `rate_limiter.try_reserve(...)` before the neuron is added. [1](#0-0) 

The rate limiter is configured at canister init: [2](#0-1) 

**SNS governance** has no equivalent. Its `claim_neuron` path goes directly to `add_neuron`, which only checks heap headroom and the population cap:

```rust
self.check_heap_can_grow()?;
self.check_neuron_population_can_grow()?;
``` [3](#0-2) 

`check_neuron_population_can_grow` simply compares the current neuron count to `max_number_of_neurons`: [4](#0-3) 

The cap itself is a governance parameter with a ceiling of 200,000: [5](#0-4) 

Neurons claimed via `ClaimOrRefresh` are created with `DissolveDelaySeconds(0)`, meaning they are immediately dissolvable and the stake is fully recoverable: [6](#0-5) 

**Attack sequence:**

1. Attacker acquires `max_number_of_neurons × neuron_minimum_stake_e8s` SNS tokens (the minimum possible total cost).
2. Attacker calls `manage_neuron { ClaimOrRefresh }` in a tight loop, creating one neuron per call, each with the minimum stake and dissolve delay 0.
3. Once `proto.neurons.len() == max_number_of_neurons`, every subsequent `claim_neuron` call from any principal returns `PreconditionFailed: "Cannot add neuron. Max number of neurons reached."` [7](#0-6) 
4. The attacker's tokens remain locked only as long as the attack is sustained; they can be recovered by disbursing the dissolved neurons at any time.
5. The community's only recourse is a governance proposal to raise `max_number_of_neurons`, which requires an existing neuron with voting power and a full voting period.

---

### Impact Explanation

Any principal that has not yet staked a neuron is permanently locked out of SNS governance participation until the attacker relents or a governance proposal passes. This includes:

- New token holders who want to stake and vote.
- Participants in a future SNS swap whose neurons cannot be claimed.
- Any protocol flow that calls `claim_neuron` on behalf of users (e.g., the swap canister's `claim_swap_neurons` path also calls `add_neuron` and hits the same cap). [8](#0-7) 

The impact matches the external report's class: griefing with no profit motive, but concrete damage to users and the protocol.

---

### Likelihood Explanation

- **Entry path is fully unprivileged**: any principal can call `manage_neuron { ClaimOrRefresh }` on an SNS governance canister.
- **Cost is low and recoverable**: the attacker only needs `max_number_of_neurons × neuron_minimum_stake_e8s` tokens temporarily locked. Default SNS parameters set `neuron_minimum_stake_e8s` to a small value (e.g., 1 e8 = 0.00000001 token in some deployments). Even at higher minimums the tokens are returned on disburse.
- **No rate limiting exists** in the SNS path, unlike NNS. [9](#0-8) 
- **Repeatable**: after the community raises the cap via proposal, the attacker can immediately fill the new headroom.

---

### Recommendation

Add a per-canister token-bucket rate limiter to SNS `claim_neuron`, mirroring the NNS implementation:

- Introduce `max_neuron_creation_spike` and `max_sustained_neurons_per_hour` (or equivalent) fields to `NervousSystemParameters`, or hard-code conservative defaults.
- Gate `add_neuron` behind a `rate_limiter.try_reserve(...)` call analogous to the NNS path. [1](#0-0) 
- Alternatively, enforce a per-principal neuron creation rate limit so a single attacker cannot exhaust the global budget.

let sns_governance = ...; // target SNS governance canister
let min_stake = nervous_system_parameters.neuron_minimum_stake_e8s;
let max_neurons = nervous_system_parameters.max_number_of_neurons;

for memo in 0..max_neurons {
    // Transfer min_stake SNS tokens to the neuron subaccount
    sns_ledger.icrc1_transfer({
        to: neuron_subaccount(attacker_principal, memo),
        amount: min_stake,
    });

    // Claim the neuron — no rate limit check in SNS governance
    sns_governance.manage_neuron({
        subaccount: neuron_subaccount(attacker_principal, memo),
        command: ClaimOrRefresh { by: MemoAndController { memo, controller: None } },
    });
    // Each call succeeds; neuron is created with DissolveDelaySeconds(0)
}

// Now any call by a legitimate user fails:
// GovernanceError { PreconditionFailed: "Cannot add neuron. Max number of neurons reached." }
victim.manage_neuron({ command: ClaimOrRefresh { ... } }); // → PreconditionFailed

// Attacker recovers tokens at will:
for each attacker_neuron {
    sns_governance.manage_neuron({ command: Disburse { ... } });
}

    /// Checks whether new neurons can be added or whether the maximum number of neurons,
    /// as defined in the nervous system parameters, has already been reached.
    fn check_neuron_population_can_grow(&self) -> Result<(), GovernanceError> {
        let max_number_of_neurons = self
            .nervous_system_parameters_or_panic()
            .max_number_of_neurons
            .expect("NervousSystemParameters must have max_number_of_neurons");

        if (self.proto.neurons.len() as u64) + 1 > max_number_of_neurons {
            return Err(GovernanceError::new_with_message(
                ErrorType::PreconditionFailed,
                "Cannot add neuron. Max number of neurons reached.",
            ));
        }

        Ok(())
    }
