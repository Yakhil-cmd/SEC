Audit Report

## Title
Stale 5% Worst-Case Modulation Constant in `spawn_neuron` Pre-Check Allows Below-Minimum-Stake Neuron Creation - (File: `rs/nns/governance/src/governance.rs`)

## Summary

`spawn_neuron` gates neuron creation using a hardcoded 5% worst-case maturity modulation, but `maybe_spawn_neurons` now applies the Mission 70 modulation whose lower bound is −10% (−1,000 permyriad). When the live modulation is between −500 and −1,000 permyriad, a neuron controller can pass the pre-check with maturity that produces a spawned neuron with `cached_neuron_stake_e8s < neuron_minimum_stake_e8s`, permanently violating the protocol minimum-stake invariant.

## Finding Description

In `rs/nns/governance/src/governance.rs` at line 2666, `spawn_neuron` computes:

```rust
let least_possible_stake = (maturity_to_spawn as f64 * (1_f64 - 0.05)) as u64;
```

This hardcodes 5% as the worst-case modulation. The guard at line 2668 rejects the call only if this value falls below `economics.neuron_minimum_stake_e8s`.

However, as of Proposal 141779 (CHANGELOG lines 20–22), `maybe_spawn_neurons` reads `heap_data.maturity_modulation.current_value_permyriad` — the locally computed Mission 70 value — and passes it directly to `apply_maturity_modulation`. The Mission 70 lower bound is `MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 = -1_000` permyriad (−10%), defined in `rs/nns/governance/src/timer_tasks/update_icp_xdr_rate_related_data.rs` line 47. The sanity check in `maybe_spawn_neurons` (line 6438) accepts any value in `VALID_MATURITY_MODULATION_BASIS_POINTS_RANGE`, which spans down to −1,000 permyriad, so modulation values between −500 and −1,000 permyriad are valid and will be applied.

`apply_maturity_modulation` computes `amount * (10_000 + basis_points) / 10_000`. At −1,000 permyriad this yields 90% of maturity. The pre-check permits maturity `M` where `M * 0.95 >= neuron_minimum_stake_e8s`, but execution can produce `M * 0.90 < neuron_minimum_stake_e8s`. The pre-check was never updated when the execution path switched from the old CMC-polled modulation (bounded at −500 permyriad) to the Mission 70 modulation (bounded at −1,000 permyriad).

## Impact Explanation

Any neuron controller whose `maturity_e8s_equivalent` falls in the window `[neuron_minimum_stake_e8s / 0.95, neuron_minimum_stake_e8s / 0.90)` can create a spawned neuron whose `cached_neuron_stake_e8s` is permanently below `neuron_minimum_stake_e8s`. For the standard 1 ICP minimum (100,000,000 e8s), the vulnerable window is approximately [105,263,158, 111,111,111) e8s. The neuron is minted on-ledger and no post-spawn correction exists. This violates the NNS protocol invariant that every neuron holds at least `neuron_minimum_stake_e8s`, constituting a significant NNS governance security impact with concrete protocol harm — matching the High impact class for NNS security impact with concrete user or protocol harm.

## Likelihood Explanation

The Mission 70 modulation moves at most 30 permyriad/day. Reaching −500 permyriad (the old floor, where the gap opens) takes approximately 17 days of sustained ICP price decline from zero. Once below −500 permyriad, any neuron controller with maturity in the ~5.5% wide vulnerable window can trigger the invariant violation via a standard unprivileged `manage_neuron` → `Spawn` ingress call. No special privileges, key material, or social engineering are required. The condition is repeatable for every controller whose maturity falls in the window while modulation remains below −500 permyriad.

## Recommendation

Replace the hardcoded `0.05` with the actual Mission 70 worst-case modulation constant, mirroring the pattern already used in SNS `disburse_maturity` (`rs/sns/governance/src/governance.rs`, lines 1654–1667):

```rust
use ic_nervous_system_governance::maturity_modulation::apply_maturity_modulation;
use crate::timer_tasks::update_icp_xdr_rate_related_data::MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70;

let least_possible_stake = apply_maturity_modulation(
    maturity_to_spawn,
    MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 as i32,
).unwrap_or(0);
```

This ensures the pre-check and the execution path use the same worst-case floor.

## Proof of Concept

1. Observe or wait for Mission 70 maturity modulation to reach −600 permyriad (achievable after ~20 days of sustained ICP price decline).
2. As a neuron controller with `maturity_e8s_equivalent = 106_000_000` e8s (1.06 ICP):
   - Pre-check: `106_000_000 * 0.95 = 100_700_000 >= 100_000_000` → **passes** (line 2668).
3. Call `manage_neuron` → `Spawn { percentage_to_spawn: 100, ... }`. Child neuron enters spawning state with `maturity_e8s_equivalent = 106_000_000`.
4. When `maybe_spawn_neurons` fires with `maturity_modulation = -600`:
   - `apply_maturity_modulation(106_000_000, -600)` = `106_000_000 * 9_400 / 10_000 = 99_640_000`.
5. Child neuron is minted with `cached_neuron_stake_e8s = 99_640_000 < 100_000_000 = neuron_minimum_stake_e8s`.

A deterministic integration test or PocketIC test can reproduce this by: (a) setting `heap_data.maturity_modulation.current_value_permyriad = Some(-600)`, (b) creating a neuron with `maturity_e8s_equivalent = 106_000_000`, (c) calling `spawn_neuron`, (d) advancing time past the spawn delay, (e) calling `maybe_spawn_neurons`, and (f) asserting `cached_neuron_stake_e8s < neuron_minimum_stake_e8s`.
