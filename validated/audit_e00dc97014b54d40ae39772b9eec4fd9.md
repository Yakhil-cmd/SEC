### Title
Integer Division Truncation Zeros Out `native_per_pubdata`, Making Pubdata Free When `pubdata_price < native_price` - (`File: basic_bootloader/src/bootloader/transaction_flow/zk/validation_impl.rs`)

---

### Summary

In ZKsync OS's L2 transaction validation, `native_per_pubdata` is computed using plain integer (floor) division: `pubdata_price / native_price`. When `pubdata_price < native_price`, this truncates to **zero**. A zero `native_per_pubdata` means the system charges **no native resources** for any amount of pubdata written, allowing any user to spam storage writes at zero pubdata cost whenever this market condition holds.

---

### Finding Description

In `validate_and_compute_fee_for_transaction`, the per-pubdata-byte native cost is derived as:

```rust
// basic_bootloader/src/bootloader/transaction_flow/zk/validation_impl.rs, line 142
let native_per_pubdata = u256_try_to_u64(&pubdata_price.wrapping_div(native_price))
    .ok_or(TxError::Validation(InvalidTransaction::PubdataPriceTooHigh))?;
```

`wrapping_div` is plain integer floor division. If `pubdata_price < native_price` (e.g., `pubdata_price = 500`, `native_price = 1000`), the result is `0`.

This `native_per_pubdata = 0` value is then passed into `create_resources_for_tx` and later into `get_resources_to_charge_for_pubdata`:

```rust
// basic_bootloader/src/bootloader/transaction_flow/gas_helpers.rs, lines 430-432
let native = current_pubdata_spent
    .checked_mul(native_per_pubdata)   // = pubdata_bytes * 0 = 0
    .ok_or(out_of_native_resources!())?;
```

Any amount of pubdata costs zero native resources. The `check_enough_resources_for_pubdata` check always passes, and the post-execution pubdata charge is zero regardless of how many storage slots the transaction writes.

**Contrast with `native_per_gas`**, which correctly uses ceiling division to prevent undercharging:

```rust
// basic_bootloader/src/bootloader/transaction_flow/zk/validation_impl.rs, line 135
u256_try_to_u64(&gas_price.div_ceil(native_price))
```

The same asymmetry exists in the public API helper:

```rust
// api/src/helpers.rs, line 427
let native_per_pubdata = u256_try_to_u64(&pubdata_price.wrapping_div(native_price)).ok_or(())?;
``` [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)

### Impact Explanation

When `native_per_pubdata == 0`:

1. `intrinsic_pubdata_overhead = native_per_pubdata_byte.saturating_mul(intrinsic_pubdata)` evaluates to `0`, so no native is withheld for pubdata at resource-creation time.
2. `get_resources_to_charge_for_pubdata` returns a zero-cost resource for any number of pubdata bytes written.
3. The post-execution pubdata check (`check_enough_resources_for_pubdata`) always reports "enough resources."
4. A transaction that writes hundreds of storage slots (each costing 32+ bytes of pubdata) pays **zero** native for all of it.

The operator bears the full L1 data publication cost without receiving compensation. A malicious user can fill the block's pubdata limit (`pubdata_limit`) with storage writes at no cost, denying pubdata capacity to other users (DoS on pubdata throughput). [5](#0-4) [6](#0-5) 

---

### Likelihood Explanation

`pubdata_price` and `native_price` are independent block-level parameters set by the operator. `native_price` reflects the cost of a single RISC-V proving cycle; `pubdata_price` reflects the cost of one byte of L1 calldata. These are denominated in different units and their ratio fluctuates with L1 gas prices and proving costs. The condition `pubdata_price < native_price` is realistic during periods of low L1 gas prices or high proving demand. The default test configuration sets `pubdata_price = U256::default()` (zero), which always triggers this path. [7](#0-6) [8](#0-7) 

---

### Recommendation

Replace `wrapping_div` with `div_ceil` for `native_per_pubdata`, mirroring the existing `native_per_gas` calculation:

```rust
// Before (validation_impl.rs line 142):
let native_per_pubdata = u256_try_to_u64(&pubdata_price.wrapping_div(native_price))
    .ok_or(TxError::Validation(InvalidTransaction::PubdataPriceTooHigh))?;

// After:
let native_per_pubdata = u256_try_to_u64(&pubdata_price.div_ceil(native_price))
    .ok_or(TxError::Validation(InvalidTransaction::PubdataPriceTooHigh))?;
```

Apply the same fix in `api/src/helpers.rs` line 427. This ensures that even when `pubdata_price < native_price`, the per-pubdata charge rounds up to at least 1 native unit rather than zeroing out. [1](#0-0) [4](#0-3)

### Proof of Concept

**Setup:**
- `native_price = U256::from(1000)` (operator-set, e.g., 1000 wei per RISC-V cycle)
- `pubdata_price = U256::from(999)` (operator-set, e.g., 999 wei per pubdata byte — slightly below native_price)
- `eip1559_basefee = U256::from(1000)`

**Computation at line 142:**
```
native_per_pubdata = 999 / 1000 = 0  (integer floor division)
```

**Effect in `create_resources_for_tx` (gas_helpers.rs line 352):**
```
intrinsic_pubdata_overhead = 0 * intrinsic_pubdata = 0
```
No native is withheld for pubdata.

**Effect in `get_resources_to_charge_for_pubdata` (gas_helpers.rs line 430):**
```
native = current_pubdata_spent * 0 = 0
```

**Attacker action:** Submit a transaction that writes 20 storage slots (each 32 bytes = 640 bytes pubdata). Expected charge: `640 * 999 = 639,360` native units. Actual charge: `0`.

The transaction succeeds, the operator publishes 640 bytes to L1 at their own expense, and the attacker pays nothing for pubdata. Repeating this across many transactions fills the block's pubdata limit, denying pubdata capacity to legitimate users. [5](#0-4) [3](#0-2)

pub fn get_resources_to_charge_for_pubdata<S: EthereumLikeTypes>(
    system: &mut System<S>,
    native_per_pubdata: u64,
    base_pubdata: Option<u64>,
) -> Result<(u64, S::Resources), SystemError> {
    let current_pubdata_spent = system
        .net_pubdata_used()?
        .saturating_sub(base_pubdata.unwrap_or(0));
    let native = current_pubdata_spent
        .checked_mul(native_per_pubdata)
        .ok_or(out_of_native_resources!())?;
    let native = <S::Resources as zk_ee::system::Resources>::Native::from_computational(native);
    Ok((current_pubdata_spent, S::Resources::from_native(native)))
}
