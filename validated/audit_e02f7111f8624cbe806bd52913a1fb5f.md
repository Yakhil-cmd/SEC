### Title
`pause_precompiles` Uses Broader ACL While `resume_precompiles` Enforces Owner-Only, Creating Asymmetric Halt Authority Over Bridge Exit Precompiles - (`engine/src/contract_methods/admin.rs`)

---

### Summary

`pause_precompiles` authorizes callers via an `EngineAuthorizer` ACL that is explicitly designed and documented to be expanded to non-owner accounts, while `resume_precompiles` enforces a strict owner-only check. When the planned ACL expansion is applied (as the in-code TODO mandates), any ACL member with lower privilege than the owner can permanently pause the `EXIT_TO_NEAR` and `EXIT_TO_ETHEREUM` bridge precompiles, freezing user withdrawals, while only the owner can resume them.

---

### Finding Description

In `engine/src/contract_methods/admin.rs`, the two symmetric-looking functions have asymmetric access control:

**`pause_precompiles`** (lines 226–241) authorizes via `EngineAuthorizer`:

```rust
let authorizer: EngineAuthorizer = engine::get_authorizer(&io);
if !authorizer.is_authorized(&env.predecessor_account_id()) {
    return Err(b"ERR_UNAUTHORIZED".into());
}
``` [1](#0-0) 

**`resume_precompiles`** (lines 208–223) enforces owner-only:

```rust
require_owner_only(&state, &predecessor_account_id)?;
``` [2](#0-1) 

The `get_authorizer` function in `engine/src/engine.rs` carries an explicit TODO:

```rust
pub fn get_authorizer<I: IO + Copy>(io: &I) -> EngineAuthorizer {
    // TODO: a temporary use the owner account only until the engine adapts std with near-plugins
    state::get_state(io)
        .map(|state| EngineAuthorizer::from_accounts(once(state.owner_id)))
        .unwrap_or_default()
}
``` [3](#0-2) 

The `EngineAuthorizer` struct is explicitly designed to hold a set of multiple accounts:

```rust
pub struct EngineAuthorizer {
    /// List of [`AccountId`]s with the permission to pause precompiles.
    pub acl: BTreeSet<AccountId>,
}
``` [4](#0-3) 

When the planned expansion of `get_authorizer` is applied to include non-owner accounts (e.g., a dedicated "pauser" role), those accounts will be able to call `pause_precompiles` to set `EXIT_TO_NEAR` and/or `EXIT_TO_ETHEREUM` flags, but will be blocked from calling `resume_precompiles` (owner-only). The pause is a one-way door for any non-owner ACL member.

The precompile flags that can be paused include `EXIT_TO_NEAR` and `EXIT_TO_ETHEREUM`, which are the bridge withdrawal precompiles: [5](#0-4) 

---

### Impact Explanation

Pausing `EXIT_TO_NEAR` or `EXIT_TO_ETHEREUM` causes all user bridge-exit transactions to revert with `ERR_PAUSED`: [6](#0-5) 

Users holding ETH or ERC-20 tokens on Aurora cannot withdraw to NEAR or Ethereum. If the owner is slow to respond or the ACL member acts at a strategic moment, this constitutes a **temporary (potentially extended) freezing of user funds** — matching the High impact tier.

---

### Likelihood Explanation

The TODO comment in `get_authorizer` is an explicit, documented intent to expand the ACL to non-owner accounts. Once that upgrade is deployed, any account added to the ACL — whether compromised, malicious, or acting in self-interest (e.g., observing that the owner is about to remove them) — can trigger the freeze. This is the direct analog to M-14: a legitimately-assigned lower-privilege role exploiting the window before their removal. [7](#0-6) 

---

### Recommendation

Apply symmetric access control: either gate `resume_precompiles` with the same `EngineAuthorizer` ACL used by `pause_precompiles`, or restrict `pause_precompiles` to `require_owner_only` (matching `resume_precompiles`). The safest fix consistent with the M-14 lesson is to require the highest-privilege role (owner) for both pause and resume of bridge-critical precompiles, and use the broader ACL only for less critical operations.

---

### Proof of Concept

1. Owner upgrades `get_authorizer` to include `pauser.near` in the ACL (the planned near-plugins integration).
2. `pauser.near` calls `pause_precompiles` with `paused_mask = EXIT_TO_NEAR | EXIT_TO_ETHEREUM`.
   - Authorization passes: `authorizer.is_authorized("pauser.near")` → `true`.
   - `EnginePrecompilesPauser::pause_precompiles(flags)` writes the pause flags to storage.
3. All subsequent user calls to `withdraw` (exit to NEAR) or `exit_to_ethereum` revert with `ERR_PAUSED`.
4. `pauser.near` attempts `resume_precompiles` → rejected: `require_owner_only` fails.
5. Only the owner can call `resume_precompiles` to unfreeze. If the owner is slow or unresponsive, user funds remain frozen. [8](#0-7) [9](#0-8)

pub fn resume_precompiles<I: IO + Copy, E: Env>(io: I, env: &E) -> Result<(), ContractError> {
    with_hashchain(io, env, function_name!(), |io| {
        let state = state::get_state(&io)?;
        require_running(&state)?;
        let predecessor_account_id = env.predecessor_account_id();

        require_owner_only(&state, &predecessor_account_id)?;

        let args: PausePrecompilesCallArgs = io.read_input_borsh()?;

        let mut pauser = EnginePrecompilesPauser::from_io(io);
        pauser.resume_precompiles(flags);
        Ok(())
    })
}
