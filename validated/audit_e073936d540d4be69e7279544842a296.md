Audit Report

## Title
`AccountState::transfer` Silently Absorbs Per-Neuron Failures, Causing `has_donated`/`has_forwarded` to Be Set Unconditionally and Permanently Locking Failed-Transfer Neuron Stakes ‚Äî (`rs/nns/gtc/src/lib.rs`)

## Summary

`AccountState::transfer` iterates over all neuron IDs, removes each from `self.neuron_ids` unconditionally regardless of transfer outcome, and always returns `Ok(())`. Because `donate_account` sets `has_donated = true` only after `transfer()` returns without error ‚Äî and `transfer()` never returns an error ‚Äî `has_donated` is always written even when one or more individual `transfer_gtc_neuron` inter-canister calls fail. Any neuron whose transfer failed is removed from `neuron_ids`, placed in `failed_transferred_neurons`, and becomes permanently unreachable: every GTC recovery path (`donate_account`, `claim_neurons`, `forward_whitelisted_unclaimed_accounts`) is gated on `has_donated == false`. The staked ICP in those neurons remains locked under GTC control with no callable path to recover it.

## Finding Description

**Root cause ‚Äî `AccountState::transfer` always returns `Ok(())`:**

In `rs/nns/gtc/src/lib.rs` lines 188‚Äì209, the loop over `neuron_ids` calls `GovernanceCanister::transfer_gtc_neuron` for each neuron. Regardless of whether that call succeeds or fails, line 192 unconditionally removes the neuron from `self.neuron_ids` via `retain`. Failures are recorded in `self.failed_transferred_neurons` but the function falls through to `Ok(())` at line 209 in every case.

**`donate_account` sets `has_donated = true` unconditionally:**

In `rs/nns/gtc/src/lib.rs` lines 89‚Äì90:
```rust
account.transfer(custodian_neuron_id).await?;
account.has_donated = true;
```
Because `transfer()` never returns `Err`, the `?` never short-circuits, and `has_donated = true` is written even when some neurons failed to transfer.

**All retry paths are permanently blocked:**

- `donate_account` ‚Üí calls `account.transfer()` ‚Üí which checks `self.has_donated` at line 177‚Äì178 and returns `Err("Account has already donated its funds")`.
- `claim_neurons` ‚Üí checks `account.has_donated` at line 54‚Äì56 and returns `Err("Account has previously donated its funds")`.
- `forward_whitelisted_unclaimed_accounts` ‚Üí the loop condition at line 113‚Äì116 requires `!account.has_donated`; accounts with `has_donated = true` are skipped entirely.

The failed neurons still exist in the Governance canister with their full stake, still controlled by the GTC principal, but no GTC method can reach them.

**Same structural defect in the forwarding path:**

`forward_whitelisted_unclaimed_accounts` at lines 118‚Äì119 sets `account.has_forwarded = true` only on `Ok(_)` from `transfer()`. Since `transfer()` always returns `Ok(())`, `has_forwarded` is also set unconditionally, locking any forwarding-path failures identically.

**Failure modes for `transfer_gtc_neuron`:**

The governance implementation at `rs/nns/governance/src/governance.rs` line 1900 performs `donor_cached_neuron_stake_e8s - transaction_fee` as a plain `u64` subtraction. If the cached stake is below the transaction fee, this wraps or panics, and the ledger call at line 1902‚Äì1911 propagates an error back via `?`. Additionally, any transient IC messaging failure (queue full, canister stopped) on the inter-canister call returns an `Err` to the GTC, which is silently absorbed.

## Impact Explanation

Neurons in `failed_transferred_neurons` retain their full ICP stake in the Governance canister under GTC control, but no public GTC method can transfer, claim, or disburse them. The stake is permanently inaccessible to both the original account owner and the intended donation recipient. This constitutes permanent, irrecoverable loss of ICP staked in NNS neurons ‚Äî an in-scope NNS governance asset. Depending on the number of affected accounts and their stake sizes, aggregate losses could be substantial. This matches the **High** impact class: "Unauthorized access to neurons, governance assets, wallets, identities, ledgers, or canister-controlled funds" with permanent loss of user funds, or **Critical** if aggregate stake exceeds $1M.

## Likelihood Explanation

The trigger requires only a transient inter-canister call failure during `donate_account` or `forward_whitelisted_unclaimed_accounts`. No privileged access is needed. Realistic failure conditions include: IC message queue saturation, the Governance or Ledger canister being temporarily stopped during an upgrade, or a neuron whose `cached_neuron_stake_e8s` is below the transaction fee (causing the plain subtraction at governance line 1900 to underflow). Any GTC account owner who calls `donate_account` during a period of transient IC instability is sufficient to trigger the permanent lock. The `forward_whitelisted_unclaimed_accounts` path can be triggered by any caller after 188 days, requiring no action from the account owner at all.

## Recommendation

1. **Propagate partial failures:** `AccountState::transfer` should return `Err` if any individual `transfer_gtc_neuron` call fails, preventing `has_donated`/`has_forwarded` from being set until all transfers succeed.
2. **Do not remove a neuron from `neuron_ids` until its transfer is confirmed:** Move `self.neuron_ids.retain(...)` inside the `Ok(_)` branch so that failed neurons remain in `neuron_ids` and can be retried on the next invocation.
3. **Apply the same fix to the `forward_whitelisted_unclaimed_accounts` path** (lines 118‚Äì126), which has the identical structural defect.

## Proof of Concept

State-machine test outline (suitable for a PocketIC or local replica integration test):

1. Initialize GTC with an account holding two neurons N1 and N2, both controlled by the GTC principal.
2. Mock the Governance canister so that `transfer_gtc_neuron(N1, custodian)` succeeds and `transfer_gtc_neuron(N2, custodian)` returns a `GovernanceError`.
3. Call `donate_account` as the account owner ‚Äî assert it returns `Ok(())`.
4. Assert `account.has_donated == true`.
5. Assert `account.neuron_ids` is empty (both neurons removed unconditionally).
6. Assert `account.failed_transferred_neurons` contains N2 with a non-None error field.
7. Assert N2 still exists in the Governance canister with its full stake, controller = GTC principal.
8. Call `donate_account` again ‚Äî assert it returns `Err("Account has already donated its funds")`.
9. Call `claim_neurons` ‚Äî assert it returns `Err("Account has previously donated its funds")`.
