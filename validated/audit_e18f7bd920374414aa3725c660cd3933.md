Audit Report

## Title
Missing Proposal-Caller Binding in `settle_neurons_fund_participation` Allows Any SNS Swap to Settle Another SNS's Neurons' Fund Participation - (File: `rs/nns/governance/src/governance.rs`)

## Summary
`settle_neurons_fund_participation` verifies only that the caller is *any* deployed SNS swap canister, never that it is the swap canister *associated with the supplied `nns_proposal_id`*. An attacker controlling a legitimate SNS swap canister can call this function with a victim SNS's proposal ID and an attacker-controlled `sns_governance_canister_id`, causing NNS Governance to mint and transfer Neurons' Fund ICP to the attacker's canister and permanently lock the victim proposal's lifecycle state.

## Finding Description

The authorization check at lines 7018–7038 converts the caller to a `CanisterId` and passes it to `is_canister_id_valid_swap_canister_id`:

```rust
// Check authorization. Note that a Swap could settle each other's participation.
let target_canister_id: CanisterId = caller.try_into()...?;
if let Err(err_msg) =
    is_canister_id_valid_swap_canister_id(target_canister_id, &*self.env).await
{ return Err(...); }
```

`is_canister_id_valid_swap_canister_id` (lines 8215–8218) only checks membership in the global list of deployed SNS swap canisters:

```rust
let is_swap = list_deployed_snses_response
    .instances
    .iter()
    .any(|sns| sns.swap_canister_id == Some(target_canister_id.into()));
```

There is no subsequent check that `caller == sns_instance.swap_canister_id` for the SNS instance whose `nns_proposal_id` matches the request. The developer comment at line 7018 explicitly acknowledges the gap: *"Note that a Swap could settle each other's participation."*

After passing this check, the function uses the attacker-supplied `sns_governance_canister_id` directly in `mint_to_sns_governance` (lines 7266–7273):

```rust
let mint_icp_result = self
    .mint_to_sns_governance(
        &request.nns_proposal_id,
        sns_governance_canister_id,   // taken verbatim from the request
        swap_estimated_total_neurons_fund_participation_icp_e8s,
        amount_icp_e8s,
    )
    .await;
```

There is no cross-validation of `sns_governance_canister_id` against the governance canister recorded in the proposal. Once the mint succeeds, `final_neurons_fund_participation` is written to the proposal (line 7315) and the lifecycle is set to terminal. The idempotency path at lines 7166–7174 then returns the attacker-computed result to any subsequent caller, including the legitimate swap canister.

## Impact Explanation

**Theft / misdirection of Neurons' Fund ICP**: The attacker directs minted ICP to their own governance canister instead of the victim SNS's treasury. Neurons' Fund reserves can be substantial; with `total_direct_participation_icp_e8s` set to `u64::MAX`, the maximum reserved maturity is minted and sent to the attacker. This constitutes illegal minting and theft of ICP ledger assets, matching the Critical impact class ("Theft, permanent loss, illegal minting, or protocol insolvency involving exorbitant ICP/Cycles or in-scope chain-key/ledger assets"). Even if the Neurons' Fund balance at the time is below $1M, the impact still matches the High class ("Unauthorized access to governance assets / canister-controlled funds where exploitation requires meaningful per-target work or other constraints").

**Permanent disruption of the victim SNS swap**: The victim's proposal lifecycle is irreversibly set to `Committed` with wrong neuron data. The legitimate swap's subsequent `settle_neurons_fund_participation` call hits the idempotency path and receives the attacker-controlled result, creating incorrectly sized SNS neurons for all Neurons' Fund participants.

## Likelihood Explanation

The attacker must control a legitimate SNS swap canister, which requires an NNS governance proposal to be approved. This is a meaningful but not prohibitive barrier — any existing SNS project satisfies it. Once deployed, the attack is a single canister call with no additional setup. The attack window opens when the victim SNS swap opens (Neurons' Fund maturity is reserved) and closes when the victim's `finalize_swap` calls `settle_neurons_fund_participation`; this window can span days to weeks. The attack executes in one block and is not detectable in advance.

## Recommendation

After confirming the caller is a valid swap canister, additionally verify that the caller is the swap canister *associated with the supplied `nns_proposal_id`*:

1. In `is_canister_id_valid_swap_canister_id` (or at the call site), accept the `nns_proposal_id` as an additional parameter.
2. Find the `DeployedSns` instance whose `nns_proposal_id` matches the request.
3. Assert `target_canister_id == sns_instance.swap_canister_id` for that specific instance.

Additionally, validate that the `sns_governance_canister_id` in the `Committed` payload matches the governance canister recorded in the matched `DeployedSns` instance, rather than accepting it as a free field from the caller.

## Proof of Concept

1. SNS B has an open swap with `nns_proposal_id = 42`; Neurons' Fund has reserved 100,000 ICP of maturity.
2. Attacker controls SNS A with swap canister `swap_a` and governance canister `gov_a` (legitimately deployed).
3. `swap_a` calls `settle_neurons_fund_participation` with:
   ```
   nns_proposal_id = 42
   result = Committed {
     sns_governance_canister_id = gov_a,
     total_direct_participation_icp_e8s = u64::MAX,
   }
   ```
4. NNS Governance confirms `swap_a` is a valid swap canister via `is_canister_id_valid_swap_canister_id` — it is.
5. NNS Governance computes maximum Neurons' Fund participation, mints ICP, and sends it to `gov_a` (lines 7266–7273).
6. Proposal 42's `final_neurons_fund_participation` is set and lifecycle transitions to `Committed` (lines 7315, 7235).
7. When SNS B's legitimate swap canister calls `settle_neurons_fund_participation` for proposal 42, it hits the idempotency path at lines 7166–7174 and receives the attacker-computed result — SNS B's Neurons' Fund neurons are created with wrong amounts and the stolen ICP is unrecoverable.

A deterministic integration test can reproduce this by deploying two SNS instances in a PocketIC environment, calling `settle_neurons_fund_participation` from SNS A's swap canister with SNS B's proposal ID, and asserting that the ICP ledger balance of `gov_a` increases and that SNS B's subsequent settlement call returns the attacker-controlled snapshot.
