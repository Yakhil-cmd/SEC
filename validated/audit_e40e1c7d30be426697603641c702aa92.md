Audit Report

## Title
SNS Tokens Permanently Locked in Swap Canister After Aborted Swap Finalization — (`rs/sns/swap/src/swap.rs`, `rs/sns/swap/canister/canister.rs`)

## Summary
When an SNS swap reaches the `ABORTED` lifecycle state, the `finalize_inner` function executes `sweep_icp` to refund ICP to buyers and then returns early after restoring dapp controllers, never invoking `sweep_sns`. The SNS tokens pre-loaded into the swap canister remain locked in its ledger account with no built-in recovery path. No `error_refund_sns` or equivalent endpoint exists on the swap canister, leaving the entire `swap_distribution_sns_e8s` allocation inaccessible to the SNS treasury without an NNS-level canister upgrade.

## Finding Description
`finalize_swap` in `rs/sns/swap/canister/canister.rs` carries no caller authentication check, making it callable by any principal. [1](#0-0) 

Inside `finalize_inner` in `rs/sns/swap/src/swap.rs`, the ABORTED path executes `sweep_icp` (line 1558), then `settle_neurons_fund_participation` (line 1565), then hits the `should_restore_dapp_control()` branch (line 1572). For an aborted swap this condition is true, so the function restores dapp controllers and **returns early at line 1583**. The `sweep_sns` call at line 1595 is never reached. [2](#0-1) 

`sweep_sns` itself also guards against non-COMMITTED lifecycles, so even if called directly it would reject an ABORTED state. [3](#0-2) 

A grep across `rs/sns/swap/**` for `error_refund_sns`, `sweep_sns_on_abort`, or `refund_sns` returns no matches. The only analogous endpoint is `error_refund_icp` for ICP tokens. [4](#0-3) 

The integration test in `rs/nervous_system/integration_tests/tests/sns_lifecycle.rs` explicitly asserts that after an aborted swap the swap canister's SNS balance equals the full `swap_distribution_sns_e8s` allocation — confirming the tokens are never moved. [5](#0-4) 

The swap.proto design comment states: *"The `swap` canister can be deleted when all tokens registered with the `swap` canister have been disbursed to their rightful owners."* The ABORTED path never disburses SNS tokens, directly contradicting this stated invariant. [6](#0-5) 

## Impact Explanation
Every SNS swap that fails to meet its minimum participation threshold will abort. After finalization, the entire SNS token allocation (`swap_distribution_sns_e8s`) is locked in the swap canister's default ledger account with no canister-level mechanism to return it to the SNS governance treasury subaccount. Recovery requires an NNS governance proposal to upgrade the swap canister — a process that is not guaranteed to be timely and introduces governance risk. This constitutes a significant, concrete financial harm to the SNS project and its token holders, matching the **High** bounty impact: *"Significant SNS... security impact with concrete user or protocol harm."* The value at risk scales with the SNS token allocation, which can be substantial.

## Likelihood Explanation
An aborted swap is a normal, expected outcome for any SNS launch that fails to reach `min_participants` or `min_icp_e8s`. No special privileges, tokens, or prior relationship with the SNS are required to call `finalize_swap`. Any principal — including an automated bot — can trigger finalization once the swap is in `ABORTED` state. This condition will occur for every failed SNS swap, making likelihood **high** and the issue systematic rather than edge-case.

## Recommendation
1. Add a `sweep_sns_on_abort` step inside `finalize_inner` for the `ABORTED` lifecycle path that transfers all remaining SNS tokens from the swap canister's default account back to the SNS governance treasury subaccount, mirroring the `sweep_icp` refund logic.
2. Alternatively, add an `error_refund_sns` endpoint analogous to `error_refund_icp` that allows any caller to trigger a return of SNS tokens to the SNS governance canister after finalization completes.
3. Update the `can_finalize` guard or add a post-finalization invariant check to verify the swap canister's SNS token balance is zero before considering finalization complete, consistent with the proto-level design comment.

## Proof of Concept
1. Deploy an SNS; its swap canister is loaded with `N` SNS tokens (PENDING → OPEN transition).
2. The swap opens but fails to reach `min_participants` or `min_icp_e8s` before the deadline; the swap transitions to `ABORTED` via heartbeat.
3. Any unprivileged principal calls `finalize_swap({})` on the swap canister.
4. `finalize_inner` executes: `sweep_icp` refunds all ICP to buyers; `settle_neurons_fund_participation` runs; `should_restore_dapp_control()` returns `true`; dapp controllers are restored; the function returns early at line 1583.
5. `sweep_sns` is never called. The swap canister's SNS ledger balance remains `N` tokens.
6. No endpoint on the swap canister can move these tokens. The integration test at `rs/nervous_system/integration_tests/tests/sns_lifecycle.rs:1338` reproduces this exactly by asserting `swap_canister_balance_sns_e8s == swap_distribution_sns_e8s` after abort — a directly runnable PocketIC integration test confirming the stuck state. [5](#0-4)
