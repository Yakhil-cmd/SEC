### Title
`WSEI.withdraw()` Uses `.transfer()` with 2300 Gas Stipend, Permanently Freezing Funds for Smart Contract Depositors — (File: `contracts/src/WSEI.sol`)

### Summary
`WSEI.sol`'s `withdraw()` function uses `payable(msg.sender).transfer(wad)` to return native SEI. The 2300-gas stipend forwarded by `.transfer()` is insufficient for any smart contract recipient whose `receive()`/`fallback()` performs more than a trivial operation (e.g., multisigs, proxy wallets, contract vaults). Such callers can deposit successfully but can never withdraw, permanently freezing their funds.

### Finding Description
`WSEI.withdraw()` at line 30 calls:

```solidity
payable(msg.sender).transfer(wad);
``` [1](#0-0) 

Solidity's `.transfer()` hard-codes a 2300-gas stipend. Post-Istanbul, `SLOAD` costs 2100 gas, meaning any contract fallback that reads a single storage slot will exceed the stipend and cause the call to revert. The `withdraw()` function has no fallback path, so the revert is unrecoverable — the deposited WSEI balance is permanently locked.

The WSEI contract is not an example or test artifact. Its ABI and bytecode are embedded into the Sei EVM module:

```go
//go:embed WSEI.abi
//go:embed WSEI.bin
``` [2](#0-1) 

This confirms it is deployed as part of the chain's core EVM infrastructure.

`MultiSender.sol` has the same class of bug using `.send()`:

```solidity
bool success = payable(recipients[i]).send(amount);
require(success, "Failed to send Ether");
``` [3](#0-2) [4](#0-3) 

Here the impact is a DoS on batch transfers whenever any recipient is a smart contract — the `require` causes the entire batch to revert.

### Impact Explanation
Any user who deposits WSEI from a smart contract address (multisig, proxy wallet, contract vault) will have their funds permanently frozen. The `withdraw()` function is the only exit path, and it unconditionally uses `.transfer()` with no fallback mechanism. There is no admin recovery function. Severity maps to **Critical** (fund freeze ≥ $5k) if the WSEI contract holds meaningful TVL from smart contract depositors, or **Medium** (fund freeze < $5k) otherwise.

### Likelihood Explanation
Smart contract wallets (Gnosis Safe, account abstraction wallets) are common on EVM chains. Any such wallet interacting with WSEI — a core wrapped-native-token contract — will trigger this freeze on withdrawal. The path requires no special privileges: deposit via a contract, then call `withdraw()`.

### Recommendation
Replace `.transfer()` in `WSEI.withdraw()` with a low-level `.call()` and check the return value:

```solidity
function withdraw(uint wad) public {
    require(balanceOf[msg.sender] >= wad);
    balanceOf[msg.sender] -= wad;
    (bool success, ) = payable(msg.sender).call{value: wad}("");
    require(success, "ETH transfer failed");
    emit Withdrawal(msg.sender, wad);
}
```

Apply the same fix to both `batchTransferEqualAmount` and `batchTransfer` in `MultiSender.sol`.

### Proof of Concept
1. Deploy a smart contract wallet whose `receive()` reads one storage slot (costs >2300 gas post-Istanbul).
2. From that contract, call `WSEI.deposit{value: X}()` — succeeds, balance recorded.
3. From that contract, call `WSEI.withdraw(X)` — the inner `.transfer()` forwards 2300 gas, the recipient's `receive()` exceeds the stipend, the call reverts, and the `X` wei is permanently locked in WSEI.
