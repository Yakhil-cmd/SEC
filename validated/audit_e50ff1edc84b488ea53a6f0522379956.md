### Title
Integer Division Truncation in `native_per_pubdata` Calculation Allows Pubdata Cost Bypass When `pubdata_price < native_price` - (File: `basic_bootloader/src/bootloader/transaction_flow/zk/validation_impl.rs`)

---

### Summary

When the operator-set `pubdata_price` is less than `native_price`, the integer division `pubdata_price / native_price` truncates to zero. This causes `native_per_pubdata` to be stored as `0`, which silently disables all native-resource charging for pubdata throughout the transaction lifecycle. Any unprivileged transaction sender can then generate arbitrarily large pubdata without consuming any native resource budget for it.

---

### Finding Description

In `validate_and_compute_fee_for_transaction`, the `native_per_pubdata` ratio is computed using floor (integer) division:

```rust
// basic_bootloader/src/bootloader/transaction_flow/zk/validation_impl.rs, line 142
let native_per_pubdata = u256_try_to_u64(&pubdata_price.wrapping_div(native_price))
    .ok_or(TxError::Validation(InvalidTransaction::PubdataPriceTooHigh))?;
``` [1](#0-0) 

The same pattern appears in `api/src/helpers.rs`:

```rust
// api/src/helpers.rs, line 427
let native_per_pubdata = u256_try_to_u64(&pubdata_price.wrapping_div(native_price)).ok_or(())?;
``` [2](#0-1) 

When `pubdata_price < native_price` (e.g., `pubdata_price = 500`, `native_price = 1000`), the result is `0`. There is no guard against a zero result — the only error path is overflow (result too large for `u64`).

This zero propagates into every downstream pubdata cost calculation:

**1. Intrinsic pubdata overhead becomes zero** in `create_resources_for_tx`:
```rust
let intrinsic_pubdata_overhead = native_per_pubdata_byte.saturating_mul(intrinsic_pubdata);
// = 0 * intrinsic_pubdata = 0
``` [3](#0-2) 

**2. Execution pubdata cost becomes zero** in `get_resources_to_charge_for_pubdata`:
```rust
let native = current_pubdata_spent
    .checked_mul(native_per_pubdata)  // = pubdata_bytes * 0 = 0
    .ok_or(out_of_native_resources!())?;
``` [4](#0-3) 

**3. Post-execution pubdata check always passes** in `check_enough_resources_for_pubdata`:
```rust
let enough = resources.has_enough(&resources_for_pubdata); // resources_for_pubdata = 0 native
``` [5](#0-4) 

**4. The `delta_gas` adjustment in `compute_gas_refund` does not recover pubdata costs** — it only adjusts for `native_used / native_per_gas`, not for pubdata:
```rust
let delta_gas = if native_per_gas == 0 {
    0
} else {
    (native_used / native_per_gas) as i64 - (gas_used as i64)
};
``` [6](#0-5) 

Since `native_used` is computed as `full_native_limit - remaining_native`, and pubdata was never charged from native, the delta_gas mechanism does not compensate for the missing pubdata cost.

---

### Impact Explanation

When `pubdata_price < native_price`, any transaction can generate arbitrarily large state diffs (pubdata) without consuming any native resource budget for it. The native resource limit is derived solely from `gas_limit * native_per_gas` (computational cost), and pubdata cost is entirely zeroed out. This means:

- A transaction with a modest gas limit can write to thousands of storage slots, generating large pubdata, without running out of native resources.
- The operator bears the L1 data-availability cost of publishing this pubdata without being compensated through the native resource mechanism.
- The block-level pubdata limit (`pubdata_limit`) is the only remaining guard, but the per-transaction native resource accounting — which is the primary DDoS protection mechanism — is completely bypassed.

This is a **resource accounting bug**: the native cost of pubdata is systematically under-reported to zero due to integer division truncation, analogous to the original report's voting power being truncated to zero.

---

### Likelihood Explanation

The condition `pubdata_price < native_price` is realistic in normal operation. `native_price` reflects the cost per RISC-V proving cycle, while `pubdata_price` reflects the cost per byte of L1 data. These are independent quantities set by the operator. For example, if `native_price = 10_000` and `pubdata_price = 5_000`, then `native_per_pubdata = 0`. Any unprivileged transaction sender can observe the block context parameters (both are public) and submit pubdata-heavy transactions when this condition holds. No privileged access, leaked keys, or governance manipulation is required.

---

### Recommendation

Replace the floor division with ceiling division for `native_per_pubdata`, consistent with how `native_per_gas` is computed:

```rust
// native_per_gas uses div_ceil (correct):
u256_try_to_u64(&gas_price.div_ceil(native_price))

// native_per_pubdata should also use div_ceil:
let native_per_pubdata = u256_try_to_u64(&pubdata_price.div_ceil(native_price))
    .ok_or(TxError::Validation(InvalidTransaction::PubdataPriceTooHigh))?;
``` [7](#0-6) 

This ensures that even when `pubdata_price < native_price`, the result is at least `1` (one native cycle per pubdata byte), preventing complete bypass of pubdata cost accounting. The same fix should be applied in `api/src/helpers.rs`. [2](#0-1) 

---

### Proof of Concept

**Setup**: Block context with `native_price = 10_000`, `pubdata_price = 5_000`, `eip1559_basefee = 10_000`.

**Computation**:
- `native_per_gas = ceil(10_000 / 10_000) = 1` (correct, uses `div_ceil`)
- `native_per_pubdata = floor(5_000 / 10_000) = 0` (truncated to zero)

**Effect**:
- `intrinsic_pubdata_overhead = 0 * intrinsic_pubdata_bytes = 0` — no native deducted at validation
- `get_resources_to_charge_for_pubdata(N_bytes) = 0` for any `N_bytes` — no native charged for execution pubdata
- `check_enough_resources_for_pubdata` always returns `enough = true`

**Attack**: A transaction with `gas_limit = 100_000` and `native_per_gas = 1` has a native budget of `100_000`. It writes to 3,000 storage slots, generating ~100 KB of pubdata. Normally this would cost `100_000 * native_per_pubdata` native units. With `native_per_pubdata = 0`, the cost is zero. The transaction succeeds, the operator publishes 100 KB to L1 at their own expense, and the sender paid only for EVM gas.
