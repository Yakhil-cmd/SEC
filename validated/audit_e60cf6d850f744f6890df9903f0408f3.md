### Title
Permanently Stuck Maturity Disbursement Due to Unrecoverable Ledger Failure and Neuron Lock Retention - (`rs/nns/governance/src/governance/disburse_maturity.rs`)

---

### Summary

In the NNS Governance canister, when `try_finalize_maturity_disbursement` pops a `MaturityDisbursement` from a neuron's queue and then the ICP ledger minting call fails **and** the attempt to push the disbursement back onto the neuron also fails, the code calls `neuron_lock.retain()` to permanently hold the neuron's in-flight lock. This permanently freezes the neuron: no further maturity disbursements can ever be finalized for it, and the maturity amount that was already deducted from the neuron at initiation time is permanently lost — stuck in neither the neuron nor the user's account.

---

### Finding Description

The vulnerability class is **ledger conservation bug**: accumulated maturity (governance rewards) can be permanently destroyed/stuck due to a double-failure path in the finalization flow.

The flow is:

1. A user calls `DisburseMaturity` on their neuron. The governance canister immediately deducts `maturity_to_deduct` from `neuron.maturity_e8s_equivalent` and pushes a `DisburseMaturityInProgress` entry onto the neuron's queue. [1](#0-0) 

2. After `DISBURSEMENT_DELAY_SECONDS` (7 days), the timer task `FinalizeMaturityDisbursementsTask` calls `try_finalize_maturity_disbursement`. This function:
   - **Step 2**: Pops the `MaturityDisbursement` from the neuron's queue (mutation #1).
   - **Step 3**: Calls the ICP ledger to mint ICP to the destination account. [2](#0-1) 

3. If the ledger mint **fails**, the code attempts to reverse mutation #1 by pushing the disbursement back to the front of the neuron's queue (`push_front_maturity_disbursement_in_progress`). [3](#0-2) 

4. If **that reversal also fails** (e.g., the neuron was deleted between the pop and the push-back, or stable memory is full), the code reaches the terminal error path and calls `neuron_lock.retain()`: [4](#0-3) 

At this point:
- The maturity was already deducted from the neuron at step 1 (initiation time).
- The disbursement entry was popped from the queue and cannot be restored.
- The ICP mint never happened.
- The neuron is permanently locked via `neuron_lock.retain()`.

The maturity is gone from the neuron's balance, never minted to the user, and the neuron is permanently frozen — no future disbursements can be processed for it.

The analogous pattern to the external report is exact: in Velodrome, `RewardsDistributor.claim` tried to deposit rewards back into an expired lock and reverted, leaving rewards stuck. Here, the IC governance canister deducts maturity at initiation time, and if the finalization path encounters a double failure (ledger error + reversal error), the maturity is permanently lost and the neuron is permanently locked.

---

### Impact Explanation

- **Maturity conservation violation**: The maturity deducted from `neuron.maturity_e8s_equivalent` at `initiate_maturity_disbursement` time is permanently destroyed — it is neither in the neuron nor minted to the user.
- **Permanent neuron lock**: `neuron_lock.retain()` causes the neuron's in-flight command to never be cleared. The `FinalizeMaturityDisbursementsTask` skips locked neurons, so no future disbursements for this neuron can ever be processed.
- **No recovery path**: There is no admin function, no governance proposal, and no user-callable method to clear a retained neuron lock or restore the lost maturity. [5](#0-4) 

---

### Likelihood Explanation

The double-failure path requires:
1. The ICP ledger minting call to fail (transient ledger error, canister trap, or cycles exhaustion).
2. The `push_front_maturity_disbursement_in_progress` reversal to also fail (e.g., the neuron was concurrently deleted, or stable memory is exhausted).

Condition 1 is realistic under ledger congestion or canister upgrade timing. Condition 2 is less common but possible under memory pressure or concurrent neuron deletion. The combination is low-probability but not impossible, and the consequence is irreversible.

---

### Recommendation

1. **Do not retain the lock on double failure.** Instead of calling `neuron_lock.retain()`, release the lock and log a critical error. The maturity is already lost at this point; permanently locking the neuron compounds the damage.
2. **Restore maturity to the neuron on reversal failure.** If `push_front_maturity_disbursement_in_progress` fails, add the `original_maturity_e8s_equivalent` back to `neuron.maturity_e8s_equivalent` directly, so the user can re-initiate the disbursement.
3. **Emit a critical alert** (not just a log) when this path is reached, so operators can investigate.

The analogous fix in the Velodrome report was to send tokens directly to the owner when the normal deposit path fails. The IC analog is: if the reversal fails, restore the maturity amount directly to the neuron's `maturity_e8s_equivalent` field rather than retaining the lock.

---

### Proof of Concept

**Entry path** (unprivileged user):
1. User calls `manage_neuron` → `DisburseMaturity` on their neuron. Maturity is deducted from `neuron.maturity_e8s_equivalent` and a `DisburseMaturityInProgress` entry is queued. [6](#0-5) 

2. After 7 days, the timer fires `try_finalize_maturity_disbursement`. The disbursement is popped from the queue (Step 2). [7](#0-6) 

3. The ICP ledger `mint_icp_with_ledger` call returns an error (Step 3 failure). [8](#0-7) 

4. The reversal `push_front_maturity_disbursement_in_progress` also returns an error (e.g., neuron not found in stable storage). [9](#0-8) 

5. `neuron_lock.retain()` is called. The neuron is permanently locked. The maturity is permanently lost. [4](#0-3) 

The `FinalizeMaturityDisbursementsTask` will skip this neuron forever because it checks `!in_flight_commands.contains_key(&neuron_id.id)` before processing: [10](#0-9)

/// Initiates the maturity disbursement process for a neuron.
pub fn initiate_maturity_disbursement(
    neuron_store: &mut NeuronStore,
    caller: &PrincipalId,
    id: &NeuronId,
    disburse_maturity: &DisburseMaturity,
    now_seconds: u64,
) -> Result<u64, InitiateMaturityDisbursementError> {
    let DisburseMaturity {
        percentage_to_disburse,
        to_account,
        to_account_identifier,
    } = disburse_maturity;

    if *percentage_to_disburse == 0 || *percentage_to_disburse > 100 {
        return Err(InitiateMaturityDisbursementError::InvalidPercentage);
    }

    let destination = Destination::try_new(to_account, to_account_identifier, *caller)
        .map_err(|reason| InitiateMaturityDisbursementError::InvalidDestination { reason })?;

    let timestamp_of_disbursement_seconds = now_seconds;
    let finalize_disbursement_timestamp_seconds = now_seconds + DISBURSEMENT_DELAY_SECONDS;

    let (
        is_neuron_spawning,
        is_neuron_controlled_by_caller,
        num_disbursements,
        maturity_e8s_equivalent,
    ) = neuron_store
        .with_neuron(id, |neuron| {
            let is_neuron_spawning = neuron.state(now_seconds) == NeuronState::Spawning;
            let is_neuron_controlled_by_caller = neuron.is_controlled_by(caller);
            let num_disbursements = neuron.maturity_disbursements_in_progress().len();
            let maturity_e8s_equivalent = neuron.maturity_e8s_equivalent;
            (
                is_neuron_spawning,
                is_neuron_controlled_by_caller,
                num_disbursements,
                maturity_e8s_equivalent,
            )
        })
        .map_err(|_| InitiateMaturityDisbursementError::NeuronNotFound)?;

    let disbursement_maturity_e8s =
        percentage_of_maturity(maturity_e8s_equivalent, *percentage_to_disburse)?;
    if disbursement_maturity_e8s < MINIMUM_DISBURSEMENT_E8S {
        return Err(InitiateMaturityDisbursementError::DisbursementTooSmall {
            disbursement_maturity_e8s,
            minimum_disbursement_e8s: MINIMUM_DISBURSEMENT_E8S,
        });
    }

    if is_neuron_spawning {
        return Err(InitiateMaturityDisbursementError::NeuronSpawning);
    }
    if !is_neuron_controlled_by_caller {
        return Err(InitiateMaturityDisbursementError::CallerIsNotNeuronController);
    }
    if num_disbursements >= MAX_NUM_DISBURSEMENTS {
        return Err(InitiateMaturityDisbursementError::TooManyDisbursements);
    }

    let disbursement_in_progress = MaturityDisbursement {
        destination: Some(destination),
        amount_e8s: disbursement_maturity_e8s,
        timestamp_of_disbursement_seconds,
        finalize_disbursement_timestamp_seconds,
    };

    neuron_store
        .with_neuron_mut(id, |neuron| {
            neuron.add_maturity_disbursement_in_progress(disbursement_in_progress);
            neuron.maturity_e8s_equivalent = neuron
                .maturity_e8s_equivalent
                .saturating_sub(disbursement_maturity_e8s);
        })
        .map_err(|_| InitiateMaturityDisbursementError::Unknown {
            reason: "Failed to update neuron even though it was found before".to_string(),
        })?;

    Ok(disbursement_maturity_e8s)

    // fails, the neuron can still be unlocked as no mutations are performed yet. This is the main
    // thing the neuron lock is protecting against.
    let Ok(Some(maturity_disbursement_in_progress)) = governance.with_borrow_mut(|governance| {
        governance.with_neuron_mut(&neuron_id, |neuron| {
            neuron.pop_maturity_disbursement_in_progress()
        })
    }) else {
        // This should be impossible since we just checked that the disbursement exists in
        // `next_maturity_disbursement_to_finalize`.
        return Err(FinalizeMaturityDisbursementError::FailToPopMaturityDisbursement(neuron_id));
    };

    let mint_icp_operation = MintIcpOperation::new(account_identifier, amount_to_mint_e8s);

    tla_log_locals! {

    tla_log_label!("Disburse_Maturity_Timer");

        return Ok(());
