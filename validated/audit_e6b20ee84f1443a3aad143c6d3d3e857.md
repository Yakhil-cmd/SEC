### Title
Unguarded `new` Initializer Allows Any Caller to Seize Engine Ownership — (`File: engine/src/contract_methods/admin.rs`)

---

### Summary

The Aurora Engine's `new` initializer accepts an arbitrary `owner_id` from caller-supplied arguments and performs **no check on who the caller is**. Because WASM deployment and the `new` call are two separate NEAR transactions, any NEAR account can front-run the legitimate deployer's `new` call, install itself as owner, and then exploit owner-gated capabilities — most critically `attach_full_access_key` — to gain full control of the engine account and steal all bridged funds.

---

### Finding Description

`engine/src/contract_methods/admin.rs::new` is the sole initialization entry point for the Aurora Engine's `EngineState`. Its only guard is a check that state does not already exist:

```rust
pub fn new<I: IO + Copy, E: Env>(mut io: I, env: &E) -> Result<(), ContractError> {
    if state::get_state(&io).is_ok() {
        return Err(b"ERR_ALREADY_INITIALIZED".into());
    }
    let input = io.read_input().to_vec();
    let args = NewCallArgs::deserialize(&input)...;
    ...
    state::set_state(&mut io, &state)?;   // owner_id comes entirely from args
    Ok(())
}
``` [1](#0-0) 

`env.predecessor_account_id()` is **never consulted**. The `owner_id` field written into `EngineState` is taken verbatim from the Borsh-encoded input supplied by whoever calls `new` first. [2](#0-1) 

The public WASM entrypoint in `lib.rs` adds no additional guard:

```rust
pub extern "C" fn new() {
    let io = Runtime;
    let env = Runtime;
    contract_methods::admin::new(io, &env)
        .map_err(ContractError::msg)
        .sdk_unwrap();
}
``` [3](#0-2) 

The workspace deployment helper shows the two-step, two-transaction pattern used in practice:

```rust
// Transaction 1 — deploys WASM
let contract = account.deploy(&self.code...).await?;
// Transaction 2 — initializes state (race window here)
engine.new(self.chain_id, self.owner_id, self.upgrade_delay_blocks)
      .transact().await...
``` [4](#0-3) 

Between these two transactions there is an open window in which any NEAR account can submit a `new` call with attacker-controlled `owner_id`.

---

### Impact Explanation

**Critical — direct theft of all user funds.**

Once the attacker controls `owner_id`, they can immediately call `attach_full_access_key`:

```rust
pub fn attach_full_access_key<...>(...) -> Result<(), ContractError> {
    let state = state::get_state(&io)?;
    require_running(&state)?;
    require_owner_only(&state, &env.predecessor_account_id())?;  // passes for attacker
    ...
    let action = PromiseAction::AddFullAccessKey { public_key, nonce: 0 };
    // SAFETY comment: "only the (trusted) owner account can add a new FAK"
    // — that trust assumption is broken by the missing caller check in `new`
``` [5](#0-4) 

A full-access key on the Aurora Engine NEAR account gives the attacker the ability to:
- Deploy arbitrary replacement WASM (draining all bridged ETH and ERC-20 balances)
- Transfer the engine account's entire NEAR balance
- Call `upgrade` / `stage_upgrade` to replace contract logic [6](#0-5) 

All ETH and ERC-20 tokens bridged through the connector, as well as the engine's NEAR balance, are at risk of permanent theft.

---

### Likelihood Explanation

**High.** The two-transaction deployment pattern is observable on-chain. A NEAR account monitoring the network for a `DeployContract` action on the `aurora` account can immediately submit a `new` call in the next block with `owner_id` set to an attacker-controlled account. No special privileges, leaked keys, or social engineering are required — only the ability to submit a NEAR transaction, which any account can do.

---

### Recommendation

Add a caller check at the top of `new` so that only the contract account itself (or the account that deployed the WASM) can initialize state:

```rust
pub fn new<I: IO + Copy, E: Env>(mut io: I, env: &E) -> Result<(), ContractError> {
    if state::get_state(&io).is_ok() {
        return Err(b"ERR_ALREADY_INITIALIZED".into());
    }
    // Enforce that only the contract account itself can initialize
    if env.predecessor_account_id() != env.current_account_id() {
        return Err(b"ERR_NOT_ALLOWED".into());
    }
    ...
}
```

Alternatively, deploy the WASM and call `new` atomically in a single NEAR batch transaction so no inter-transaction race window exists. [1](#0-0) 

---

### Proof of Concept

1. Attacker monitors the NEAR blockchain for a `DeployContract` action targeting the `aurora` account.
2. In the immediately following block, attacker submits a NEAR transaction calling `aurora.new` with Borsh-encoded `NewCallArgsV4 { chain_id: ..., owner_id: "attacker.near", upgrade_delay_blocks: 0, key_manager: "attacker.near", initial_hashchain: None }`.
3. Because no state exists yet, the guard at line 57 passes; `EngineState { owner_id: "attacker.near", ... }` is written to storage.
4. The legitimate deployer's subsequent `new` call returns `ERR_ALREADY_INITIALIZED` and is silently discarded.
5. Attacker calls `aurora.attach_full_access_key` with their own public key. `require_owner_only` passes because `state.owner_id == "attacker.near" == predecessor`.
6. Attacker now holds a full-access key on the `aurora` account and can deploy arbitrary code or transfer all funds. [1](#0-0) [5](#0-4) [4](#0-3)

pub fn new<I: IO + Copy, E: Env>(mut io: I, env: &E) -> Result<(), ContractError> {
    if state::get_state(&io).is_ok() {
        return Err(b"ERR_ALREADY_INITIALIZED".into());
    }

    let input = io.read_input().to_vec();
    let args = NewCallArgs::deserialize(&input).map_err(|_| errors::ERR_BORSH_DESERIALIZE)?;

    let initial_hashchain = args.initial_hashchain();
    let state: EngineState = args.into();

    if let Some(block_hashchain) = initial_hashchain {
        let block_height = env.block_height();
        let mut hashchain = Hashchain::new(
            state.chain_id,
            env.current_account_id(),
            block_height,
            block_hashchain,
        );

        hashchain.add_block_tx(
            block_height,
            function_name!(),
            &input,
            &[],
            &Bloom::default(),
        )?;
        crate::hashchain::save_hashchain(&mut io, &hashchain)?;
    }

    state::set_state(&mut io, &state)?;
    Ok(())
}

pub fn upgrade<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<(), ContractError> {
    let state = state::get_state(&io)?;
    require_running(&state)?;
    require_owner_only(&state, &env.predecessor_account_id())?;

    let input = io.read_input().to_vec();
    let (code, state_migration_gas) = match UpgradeParams::try_from_slice(&input) {
        Ok(args) => (
            args.code,
            args.state_migration_gas
                .map_or(GAS_FOR_STATE_MIGRATION, NearGas::new),
        ),
        Err(_) => (input, GAS_FOR_STATE_MIGRATION), // Backward compatibility
    };

    let target_account_id = env.current_account_id();
    let batch = PromiseBatchAction {
        target_account_id,
        actions: vec![
            PromiseAction::DeployContract { code },
            PromiseAction::FunctionCall {
                name: "state_migration".to_string(),
                args: vec![],
                attached_yocto: ZERO_YOCTO,
                gas: state_migration_gas,
            },
        ],
    };
    let promise_id = handler.promise_create_batch(&batch);

    handler.promise_return(promise_id);

    Ok(())
}

pub fn attach_full_access_key<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<(), ContractError> {
    let state = state::get_state(&io)?;

    require_running(&state)?;
    require_owner_only(&state, &env.predecessor_account_id())?;

    let public_key = serde_json::from_slice::<FullAccessKeyArgs>(&io.read_input().to_vec())
        .map(|args| args.public_key)
        .map_err(|_| errors::ERR_JSON_DESERIALIZE)?;
    let current_account_id = env.current_account_id();
    let action = PromiseAction::AddFullAccessKey {
        public_key,
        nonce: 0, // not actually used - depends on block height
    };
    let promise = PromiseBatchAction {
        target_account_id: current_account_id,
        actions: vec![action],
    };
    // SAFETY: This action is dangerous because it adds a new full access key (FAK) to the Engine account.
    // However, it is safe to do so here because of the `require_owner_only` check above; only the
    // (trusted) owner account can add a new FAK.
    let promise_id = handler.promise_create_batch(&promise);

    handler.promise_return(promise_id);

    Ok(())
}
