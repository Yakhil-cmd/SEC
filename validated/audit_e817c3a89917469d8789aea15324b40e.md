### Title
Floor Division in `deltaGas` Calculation Causes Gas Estimation to Underestimate by 1 Gas Unit, Leading to Native Resource Exhaustion on Actual Execution - (`basic_bootloader/src/bootloader/transaction_flow/refund_calculation.rs`)

---

### Summary

ZKsync OS implements a dual-resource accounting model (EVM gas / ergs + native resource). When native resource consumption dominates, a `deltaGas` adjustment is added to `gas_used` during simulation so that `eth_estimateGas` returns a gas limit sufficient to cover native costs. The formula uses **floor (integer) division** instead of ceiling division. When `native_used` is not divisible by `native_per_gas`, the returned `gas_used` is 1 unit too low. A user who sets `gas_limit = gas_used_sim` will have `native_limit = gas_limit × native_per_gas < native_used`, causing the actual transaction to run out of native resources and revert.

---

### Finding Description

The `compute_gas_refund` function in `refund_calculation.rs` computes `deltaGas` as:

```rust
let delta_gas = if native_per_gas == 0 {
    0
} else {
    (native_used / native_per_gas) as i64 - (gas_used as i64)
};
``` [1](#0-0) 

`native_used / native_per_gas` is **floor division** in Rust integer arithmetic. The documentation explicitly states the purpose of this adjustment:

> "This ensures that gas estimation will include additional gas to cover for native resources using just base fee." [2](#0-1) 

The native resource limit for a transaction is derived as:

```
native_limit = gas_limit × native_per_gas
``` [3](#0-2) 

When `delta_gas > 0` (native-dominated case), `gas_used` is set to approximately `floor(native_used / native_per_gas)`. If the user then submits with `gas_limit = gas_used_sim`:

```
native_limit = floor(native_used / native_per_gas) × native_per_gas
```

Whenever `native_used % native_per_gas ≠ 0`, this gives `native_limit < native_used`, and the transaction runs out of native resources during actual execution.

The fix is to use ceiling division:

```rust
native_used.div_ceil(native_per_gas) as i64 - (gas_used as i64)
```

This is structurally identical to the UniV3Decomposer bug: a safety-margin formula rounds **down** instead of **up**, causing the computed buffer to be 1 unit too small, and legitimate operations to revert.

---

### Impact Explanation

Any transaction whose native resource consumption is not divisible by `native_per_gas` will have its gas estimate underestimated by exactly 1 gas unit. When the user submits the transaction with the estimated gas limit, the transaction runs out of native resources and reverts. The user must manually add a gas buffer to work around this, which is not expected behavior for `eth_estimateGas`. This breaks the core promise of the gas estimation RPC for a large fraction of transactions.

The condition `native_used % native_per_gas ≠ 0` is satisfied for almost all transactions, since `native_used` is an arbitrary integer and `native_per_gas` is a ratio derived from `gas_price / native_price`.

---

### Likelihood Explanation

The `deltaGas` path is active whenever `native_per_gas > 0` and native resource consumption exceeds EVM gas consumption — the common case for ZKsync OS transactions where proving cost dominates. The off-by-one occurs for any `native_used` not exactly divisible by `native_per_gas`, which is the overwhelming majority of real transactions. Any unprivileged user submitting a transaction via the standard `eth_estimateGas` → submit flow is affected.

---

### Recommendation

Replace the floor division with ceiling division in `compute_gas_refund`:

```rust
// Before (floor division — underestimates by 1 when native_used % native_per_gas != 0):
(native_used / native_per_gas) as i64 - (gas_used as i64)

// After (ceiling division — always sufficient):
native_used.div_ceil(native_per_gas) as i64 - (gas_used as i64)
``` [4](#0-3)

### Proof of Concept

Concrete numeric example with realistic parameters:

- `native_price = 100` (operator constant)
- `gas_price = 2000`
- `native_per_gas = ceil(2000 / 100) = 20`
- Transaction consumes `native_used = 41` native units (e.g., a simple transfer)
- EVM gas used: `gas_used_evm = 1` (negligible)

**Simulation:**
```
delta_gas = floor(41 / 20) - 1 = 2 - 1 = 1
gas_used_sim = 1 + 1 = 2
```

User submits with `gas_limit = 2`.

**Actual execution:**
```
native_limit = 2 × 20 = 40
native_used  = 41
```

`native_limit (40) < native_used (41)` → transaction runs out of native resources and reverts.

**With ceiling division:**
```
delta_gas = ceil(41 / 20) - 1 = 3 - 1 = 2
gas_used_sim = 1 + 2 = 3
native_limit = 3 × 20 = 60 ≥ 41  ✓
```

The existing test `test_simulation_gas_and_native_used` in `tests/instances/transactions/src/lib.rs` (lines 1616–1670) validates that simulation gas matches actual execution gas, but it uses a transaction where `native_used` happens to be divisible by `native_per_gas` in the test parameters, masking this off-by-one. /// Check that gas and native used is the same in simulation and actual execution
#[test]
fn test_simulation_gas_and_native_used() {
    let mut tester = TestingFramework::new();
    let wallet = tester.prefunded_random_signer();
    let target_address = common_target_address();

    let tx = TxEip1559 {
        chain_id: 37u64,
        nonce: 0,
        max_fee_per_gas: 1000,
        max_priority_fee_per_gas: 1000,
        gas_limit: 1_000_000,
        to: TxKind::Call(target_address),
        value: U256::from(100),
        input: Default::default(),
        access_list: Default::default(),
    };

    let tx_for_simulation = ZKsyncTxEnvelope::from_eth_tx(tx.clone(), wallet.clone());

    // We use a very low native per gas ratio to force the transaction to require extra gas
    let block_context = BlockContext {
        eip1559_basefee: U256::from(1000),
        native_price: U256::from(500),
        ..Default::default()
    };

    tester = tester.with_block_context(block_context.clone());
    let result_simulation = tester.simulate_block(vec![tx_for_simulation]);
    let tx_result_simulation = result_simulation.tx_results[0]
        .clone()
        .expect("Simulation must succeed");

    let tx = TxEip1559 {
        gas_limit: tx_result_simulation.gas_used,
        ..tx
    };
    let encoded = ZKsyncTxEnvelope::from_eth_tx(tx, wallet);

    tester = tester.with_block_context(block_context);
    let result_normal = tester.execute_block(vec![encoded]);
    let tx_result_normal = result_normal.tx_results[0]
        .clone()
        .expect("Normal execution must succeed");

    assert_eq!(
        tx_result_simulation.gas_used, tx_result_normal.gas_used,
        "Mismatch in gas used"
    );
    assert_eq!(
        tx_result_simulation.native_used, tx_result_normal.native_used,
        "Mismatch in native used"
    );
}
