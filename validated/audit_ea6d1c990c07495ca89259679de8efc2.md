### Title
Operator-Controlled `native_price` and `pubdata_price` Are Not Committed to On-Chain Public Input, Enabling Unconstrained Fee Extraction - (`zk_ee/src/system/metadata/zk_metadata.rs`, `basic_bootloader/src/bootloader/transaction_flow/refund_calculation.rs`)

---

### Summary

The ZKsync OS bootloader derives the `native_per_gas` ratio — which directly controls how much EVM gas is charged to users via the `deltaGas` mechanism — from `native_price` and `pubdata_price` values supplied by the operator through the block oracle. Neither of these pricing parameters is included in the batch public input committed to the settlement layer. A malicious operator can freely set these values to inflate `gas_used` up to the user's full `gas_limit`, extracting the entire pre-paid fee regardless of actual computation performed, with no on-chain enforcement or proof-level constraint.

---

### Finding Description

**Oracle input with no on-chain commitment:**

`BlockMetadataFromOracle` is deserialized from the oracle at block start in `metadata_op.rs`. The only validation performed is a bounds check on `block_gas_limit` and `individual_tx_gas_limit`:

```rust
// basic_bootloader/src/bootloader/block_flow/zk/metadata_op.rs
let block_level_metadata: BlockMetadataFromOracle =
    oracle.query_with_empty_input(BLOCK_METADATA_QUERY_ID)?;
// ...
if metadata.block_gas_limit() > MAX_BLOCK_GAS_LIMIT
    || metadata.individual_tx_gas_limit() > MAX_TX_GAS_LIMIT
{
    return Err(internal_error!("block or tx gas limit is too high"));
}
```

`native_price` and `pubdata_price` are accepted without any bounds check. [1](#0-0) 

**Fee derivation from unconstrained oracle values:**

In `validation_impl.rs`, `native_per_gas` is computed as `ceil(gas_price / native_price)`:

```rust
let native_per_gas = u256_try_to_u64(&gas_price.div_ceil(native_price))...
let native_per_pubdata = u256_try_to_u64(&pubdata_price.wrapping_div(native_price))...
``` [2](#0-1) 

**`deltaGas` inflates `gas_used` based on `native_per_gas`:**

In `refund_calculation.rs`, after execution, `gas_used` is adjusted upward by `deltaGas`:

```rust
let delta_gas = if native_per_gas == 0 {
    0
} else {
    (native_used / native_per_gas) as i64 - (gas_used as i64)
};
if delta_gas > 0 {
    gas_used += delta_gas as u64;
}
``` [3](#0-2) 

**`native_price` and `pubdata_price` are absent from the batch public input:**

`BatchOutput.hash()` commits: `chain_id`, timestamps, DA commitment, L1 tx counts, priority ops hash, L2 logs root, upgrade tx hash, interop roots hash, and settlement layer chain id. `native_price` and `pubdata_price` are not included. [4](#0-3) 

**The documentation explicitly acknowledges this gap:**

> "Currently it misses `gas_per_pubdata` and `native_price`, but we already working on design and implementation to solve this issue." [5](#0-4) 

---

### Impact Explanation

An operator who sets `native_price` to an arbitrarily large value causes `native_per_gas = gas_price / native_price` to become very small (approaching 0). This means:

- `native_used / native_per_gas` becomes very large
- `delta_gas = (native_used / native_per_gas) - gas_used` becomes a large positive number
- `gas_used` is inflated toward `gas_limit`
- The user is charged `gas_price × inflated_gas_used` instead of `gas_price × actual_gas_used`
- The operator (coinbase) collects the excess

The maximum extractable amount per transaction is `gas_price × gas_limit` — the full pre-paid fee — regardless of how little computation the transaction actually performed. Similarly, an inflated `pubdata_price` causes `native_per_pubdata` to be large, consuming native resources faster and forcing transactions to revert (DoS) or pay for pubdata they did not generate.

Since neither value is committed to the proof's public input, the settlement layer cannot detect or reject a block that used manipulated pricing.

---

### Likelihood Explanation

The operator is the entity that constructs the `BlockMetadataFromOracle` struct and feeds it to the oracle. In the forward-running system, this is done directly by the sequencer: [6](#0-5) 

The `BlockContext` struct (which maps 1:1 to `BlockMetadataFromOracle`) is fully operator-controlled with no external constraint on `native_price` or `pubdata_price`: [7](#0-6) 

Any operator running a malicious sequencer can exploit this on every block. The attack requires no special cryptographic capability, no leaked key, and no external oracle manipulation — only the ability to set block context values, which is the operator's normal function.

---

### Recommendation

1. **Include `native_price` and `pubdata_price` in the batch public input hash** (`BatchOutput.hash()`) so the settlement layer can verify these values match what was committed on-chain.
2. **Add bounds validation** on `native_price` and `pubdata_price` in `metadata_op.rs` (analogous to the existing `MAX_BLOCK_GAS_LIMIT` check), enforcing operator-agreed maximum and minimum values.
3. **Commit `native_price` and `pubdata_price` to the block header** (as the documentation already identifies as a TODO in `extra_data`).

---

### Proof of Concept

1. Operator constructs a `BlockMetadataFromOracle` with `native_price = U256::MAX` (or any very large value) and `pubdata_price = U256::ZERO`.
2. `native_per_gas = gas_price.div_ceil(U256::MAX)` → rounds to `0` or `1`.
3. With `native_per_gas = 1`, `native_used / 1 = native_used` (which can be up to `MAX_NATIVE_COMPUTATIONAL = 2^35`).
4. `delta_gas = native_used - gas_used` → a large positive number.
5. `gas_used` is inflated to `min(gas_used + delta_gas, gas_limit)`.
6. User is charged `gas_price × gas_limit` instead of `gas_price × actual_gas_used`.
7. Operator collects the full pre-paid fee.
8. The proof verifier on the settlement layer accepts the batch because `native_price` is not part of `BatchOutput.hash()` and cannot be checked. [8](#0-7) [9](#0-8) [10](#0-9)

///
/// Except for proving existence of batch(of blocks) that changes state from one to another, we want to open some info about this batch on the settlement layer:
/// - pubdata: to make sure that it's published and state is recoverable
/// - executed priority ops: to process them on the settlement layer
/// - l2 to l1 logs tree root: to be able to open them on the settlement layer
/// - extra inputs to validate on the settlement layer(timestamp and chain id)
///
#[derive(Debug)]
pub struct BatchOutput {
    /// Chain id used during execution of the blocks.
    pub chain_id: U256,
    /// First block timestamp.
    pub first_block_timestamp: u64,
    /// Last block timestamp.
    pub last_block_timestamp: u64,
    /// DA commitment scheme.
    pub da_commitment_scheme: DACommitmentScheme,
    /// Pubdata commitment.
    pub pubdata_commitment: Bytes32,
    /// Number of l1 -> l2 processed txs in the batch.
    pub number_of_layer_1_txs: U256,
    /// Number of processed L2 txs in the batch.
    pub number_of_layer_2_txs: U256,
    /// Rolling keccak256 hash of l1 -> l2 txs processed in the batch.
    pub priority_operations_hash: Bytes32,
    /// L2 logs tree root.
    /// Note that it's full root, it's keccak256 of:
    /// - merkle root of l2 -> l1 logs in the batch .
    /// - multichain root - commitment to logs emitted on chains that settle on the current.
    pub l2_logs_tree_root: Bytes32,
    /// Protocol upgrade tx hash (0 if there wasn't)
    pub upgrade_tx_hash: Bytes32,
    /// Linear keccak256 hash of interop roots
    pub interop_roots_rolling_hash: Bytes32,
    /// Settlement layer chain id.
    pub settlement_layer_chain_id: U256,
}

impl BatchOutput {
    ///
    /// Calculate keccak256 hash of public input
    ///
    pub fn hash(&self) -> [u8; 32] {
        let mut hasher = Keccak256::new();
        hasher.update(self.chain_id.to_be_bytes::<32>());
        hasher.update(&self.first_block_timestamp.to_be_bytes());
        hasher.update(&self.last_block_timestamp.to_be_bytes());
        // Encode DA commitment scheme as U256 BE
        hasher.update([0u8; 31]);
        hasher.update([self.da_commitment_scheme as u8]);
        hasher.update(self.pubdata_commitment.as_u8_ref());
        hasher.update(self.number_of_layer_1_txs.to_be_bytes::<32>());
        hasher.update(self.number_of_layer_2_txs.to_be_bytes::<32>());
        hasher.update(self.priority_operations_hash.as_u8_ref());
        hasher.update(self.l2_logs_tree_root.as_u8_ref());
        hasher.update(self.upgrade_tx_hash.as_u8_ref());
        hasher.update(self.interop_roots_rolling_hash.as_u8_ref());
        hasher.update(self.settlement_layer_chain_id.to_be_bytes::<32>());
        hasher.finalize()
    }
