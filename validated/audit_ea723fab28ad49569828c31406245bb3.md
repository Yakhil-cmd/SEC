### Title
Missing Admin Ability to Remove NEP-141 → ERC-20 Token Registration Enables Permanent Minting from Compromised Tokens - (`engine/src/map.rs`, `engine/src/engine.rs`)

---

### Summary

The `BijectionMap` used to store the NEP-141 ↔ ERC-20 token mapping has no `remove` method, and no admin-callable contract function exists to deregister a NEP-141 token. Once a NEP-141 token is registered via `deploy_erc20_token`, any NEAR account controlling that NEP-141 contract can call `ft_on_transfer` on Aurora indefinitely to mint ERC-20 tokens. If the NEP-141 contract is compromised, the Aurora admin has no granular mechanism to stop the minting short of pausing the entire engine.

---

### Finding Description

The `BijectionMap<NEP141Account, ERC20Address, I>` struct in `engine/src/map.rs` exposes only `insert`, `lookup_left`, and `lookup_right`: [1](#0-0) 

There is no `remove` method on `BijectionMap`. Correspondingly, `Engine::register_token` in `engine/src/engine.rs` writes both directions of the mapping permanently: [2](#0-1) 

No admin-callable NEAR function exists in `engine/src/lib.rs` to deregister a NEP-141 token. The full set of admin methods (`deploy_erc20_token`, `set_erc20_metadata`, `mirror_erc20_token`, etc.) contains no `unregister_erc20_token` or equivalent. [3](#0-2) 

The `ft_on_transfer` entrypoint in `engine/src/contract_methods/connector.rs` routes any call whose `predecessor_account_id` is a registered NEP-141 token directly into `receive_erc20_tokens`, which mints ERC-20 tokens on Aurora: [4](#0-3) 

Inside `receive_erc20_tokens`, the only guard is `get_erc20_from_nep141` — if the NEP-141 is registered, minting proceeds unconditionally: [5](#0-4) 

The available pause mechanisms (`PauseContract`, `PausePrecompiles`, `SetPausedFlags`) are coarse-grained and affect all users, not a single token.

---

### Impact Explanation

A compromised NEP-141 contract that is already registered can call `ft_on_transfer` on Aurora with an arbitrary `amount`, minting unbounded ERC-20 tokens for any recipient. This inflates the ERC-20 supply, allowing the attacker to dump the minted tokens for other assets, directly stealing value from existing ERC-20 holders. The Aurora admin cannot selectively block the compromised NEP-141 without halting the entire engine.

**Impact: Critical — Direct theft of user funds via ERC-20 supply inflation.**

---

### Likelihood Explanation

Any registered NEP-141 token whose NEAR contract is compromised (e.g., via a key leak, upgrade exploit, or governance attack on the NEP-141 contract itself) becomes an immediate attack vector. Aurora has bridged many NEP-141 tokens; the probability that at least one is compromised over the protocol's lifetime is non-trivial. The missing admin control means the window of exposure is unbounded once a token is registered.

---

### Recommendation

1. Add a `remove` method to `BijectionMap` in `engine/src/map.rs` that deletes both the `Nep141Erc20Map` and `Erc20Nep141Map` storage keys.
2. Add an owner-only `unregister_erc20_token` contract method in `engine/src/lib.rs` that calls this removal.
3. In `receive_erc20_tokens`, check a per-token "paused" flag before minting, so the admin can freeze a specific token without halting the entire engine.

---

### Proof of Concept

1. Admin calls `deploy_erc20_token` for `evil.near`, registering it in `nep141_erc20_map`.
2. `evil.near`'s NEAR contract is later compromised.
3. Attacker controls `evil.near` and calls `ft_on_transfer` on Aurora with `amount = u128::MAX` and `msg = <attacker_evm_address>`.
4. `ft_on_transfer` routes to `receive_erc20_tokens` because `predecessor_account_id == evil.near` is registered.
5. `get_erc20_from_nep141` succeeds, and the ERC-20 `mint` call executes, crediting `u128::MAX` tokens to the attacker.
6. Aurora admin has no function to remove `evil.near` from the mapping; the only recourse is `PauseContract`, which halts all Aurora activity. [6](#0-5) [7](#0-6)

/// A map storing a 1:1 relation between elements of types L and R.
/// The map is backed by storage of type I.
pub struct BijectionMap<L, R, I> {
    left_prefix: KeyPrefix,
    right_prefix: KeyPrefix,
    io: I,
    left_phantom: PhantomData<L>,
    right_phantom: PhantomData<R>,
}

impl<L: AsRef<[u8]> + TryFrom<Vec<u8>>, R: AsRef<[u8]> + TryFrom<Vec<u8>>, I: IO>
    BijectionMap<L, R, I>
{
    pub const fn new(left_prefix: KeyPrefix, right_prefix: KeyPrefix, io: I) -> Self {
        Self {
            left_prefix,
            right_prefix,
            io,
            left_phantom: PhantomData,
            right_phantom: PhantomData,
        }
    }

    pub fn insert(&mut self, left: &L, right: &R) {
        let key = self.left_key(left);
        self.io.write_storage(&key, right.as_ref());

        let key = self.right_key(right);
        self.io.write_storage(&key, left.as_ref());
    }

    pub fn lookup_left(&self, left: &L) -> Option<R> {
        let key = self.left_key(left);
        self.io
            .read_storage(&key)
            .and_then(|v| v.to_vec().try_into().ok())
    }

    pub fn lookup_right(&self, right: &R) -> Option<L> {
        let key = self.right_key(right);
        self.io
            .read_storage(&key)
            .and_then(|v| v.to_vec().try_into().ok())
    }

    fn left_key(&self, left: &L) -> Vec<u8> {
        bytes_to_key(self.left_prefix, left.as_ref())
    }

    fn right_key(&self, right: &R) -> Vec<u8> {
        bytes_to_key(self.right_prefix, right.as_ref())
    }

    pub fn receive_erc20_tokens<P: PromiseHandler>(
        &mut self,
        token: &AccountId,
        args: &FtOnTransferArgs,
        current_account_id: &AccountId,
        handler: &mut P,
    ) -> Result<Option<SubmitResult>, ContractError> {
        let amount = args.amount.as_u128();
        // Parse message to determine recipient
        let mut recipient = {
            // The message should contain the recipient EOA address.
            let message = args.msg.strip_prefix("0x").unwrap_or(&args.msg);
            // Recipient - 40 characters (Address in hex without '0x' prefix)
            if message.len() < 40 {
                return Err(ParseOnTransferMessageError::WrongMessageFormat.into());
            }
            let mut address_bytes = [0; 20];
            hex::decode_to_slice(&message[..40], &mut address_bytes)
                .map_err(|_| ParseOnTransferMessageError::WrongMessageFormat)?;
            Address::from_array(address_bytes)
        };

        if let Some(fallback_address) = silo::get_erc20_fallback_address(&self.io)
            && !silo::is_allow_receive_erc20_tokens(&self.io, &recipient)
        {
            recipient = fallback_address;
        }

        let erc20_token = get_erc20_from_nep141(&self.io, token)?;
        let erc20_admin_address = current_address(current_account_id);
        let result = self
            .call(
                &erc20_admin_address,
                &erc20_token,
                Wei::zero(),
                setup_receive_erc20_tokens_input(&recipient, amount),
                u64::MAX,
                Vec::new(), // TODO: are there values we should put here?
                Vec::new(),
                handler,
            )
            .and_then(submit_result_or_err)?;

        sdk::log!("Mint {amount} ERC-20 tokens for: {}", recipient.encode());

        // Return SubmitResult so that it can be accessed in standalone engine.
        // This is used to help with the indexing of bridge transactions.
        Ok(Some(result))
    }

#[named]
pub fn ft_on_transfer<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<Option<SubmitResult>, ContractError> {
    with_hashchain(io, env, function_name!(), |mut io| {
        require_running(&state::get_state(&io)?)?;
        let current_account_id = env.current_account_id();
        let predecessor_account_id = env.predecessor_account_id();
        let mut engine: Engine<_, _> = Engine::new(
            predecessor_address(&predecessor_account_id),
            current_account_id.clone(),
            io,
            env,
        )?;

        sdk::log!("Call ft_on_transfer");

        let args: FtOnTransferArgs = read_json_args(&io)?;
        let result = if predecessor_account_id == get_connector_account_id(&io)? {
            engine.receive_base_tokens(&args)
        } else {
            engine.receive_erc20_tokens(
                &predecessor_account_id,
                &args,
                &current_account_id,
                handler,
            )
        };

        #[allow(clippy::used_underscore_binding)]
        let amount_to_return = if let Err(_err) = &result {
            sdk::log!("Error in ft_on_transfer: {_err:?}");
            // An error occurred, so we need to return the amount of tokens to the sender.
            args.amount.as_u128()
        } else {
            // Everything is ok, so return 0.
            0
        };

        let output = crate::prelude::format!("\"{amount_to_return}\"");
        io.return_output(output.as_bytes());

        // In case of an error, we just return Ok(None) to avoid a panic in the contract. It's ok
        // because in case of an error, we already returned the amount of tokens to the sender.
        Ok(result.unwrap_or(None))
    })
