### Title
Single-Step Irrecoverable Dapp Controller Transfer Without New-Controller Validation — (`File: rs/sns/governance/src/proposal.rs`)

---

### Summary

The SNS `DeregisterDappCanisters` proposal action permanently transfers control of registered dapp canisters to caller-supplied `new_controllers` in a single, irreversible step. The proposal validation only checks that the list is non-empty; it does not verify that the supplied principals are valid, reachable, or have accepted the transfer. If a governance community passes a proposal with an erroneous `new_controllers` list (e.g., a mistyped principal, a burned address, or a non-existent canister ID), the dapp canisters are permanently removed from SNS control with no on-chain recovery path.

---

### Finding Description

`validate_and_render_deregister_dapp_canisters` in `rs/sns/governance/src/proposal.rs` performs the following checks before allowing a `DeregisterDappCanisters` proposal to proceed:

1. `canister_ids` must be non-empty.
2. `canister_ids.len()` must not exceed `MAX_NUMBER_OF_DAPPS_TO_MANAGE_PER_PROPOSAL`.
3. `new_controllers` must be non-empty.
4. None of the `canister_ids` may be SNS-reserved canisters. [1](#0-0) 

No check is performed to verify that the principals in `new_controllers` are:
- syntactically valid and non-zero,
- currently live canisters or self-authenticating principals,
- or willing/able to accept control (no two-step acceptance handshake).

When the proposal executes, `perform_deregister_dapp_canisters` encodes the request and calls `set_dapp_controllers` on the SNS Root canister in a single cross-canister call: [2](#0-1) 

`SnsRootCanister::set_dapp_controllers` then calls `update_settings` on the IC management canister for each dapp, replacing its controller list with the supplied principals, and removes the dapp from `dapp_canister_ids`: [3](#0-2) 

Once `update_settings` succeeds and the dapp is removed from `dapp_canister_ids`, the SNS has no further authority over those canisters. If the `new_controllers` list contains an inaccessible principal, the dapp is permanently orphaned.

---

### Impact Explanation

**Governance authorization bug / irrecoverable state change.**

An SNS dapp canister transferred to an inaccessible principal (e.g., a mistyped principal, a deleted canister, or the anonymous principal) cannot be recovered by the SNS. The SNS Root canister no longer lists the dapp in `dapp_canister_ids`, so no future SNS proposal can re-register or reclaim it. The dapp's cycles will eventually drain and the canister will be deleted. This is a permanent, protocol-level loss of assets managed by the SNS.

---

### Likelihood Explanation

SNS governance communities routinely pass `DeregisterDappCanisters` proposals to hand dapps back to developers. The `new_controllers` field is a free-form list of `PrincipalId` values supplied by the proposer. A single character error in a principal text encoding, or a copy-paste of a stale/deleted canister ID, produces a syntactically valid but unreachable principal that passes all current validation. Because the proposal rendering only displays the raw principal strings (not their on-chain status), voters have no tooling-level signal that the target is unreachable. The likelihood of an honest mistake is realistic for any active SNS. [4](#0-3) 

---

### Recommendation

1. **Short term:** Add a pre-execution check in `perform_deregister_dapp_canisters` (or in `validate_and_render_deregister_dapp_canisters`) that rejects proposals where any element of `new_controllers` is the anonymous principal (`2vxsx-fae`) or the management canister principal, and optionally verifies that each principal is either self-authenticating or a live canister via `canister_status`.

2. **Medium term:** Implement a two-step handshake: the `DeregisterDappCanisters` proposal first sets a *pending* transfer, and a second on-chain action (signed by the new controllers) confirms acceptance before the SNS Root actually relinquishes control.

3. **Long term:** Document all irrecoverable SNS governance actions and their associated risks in the SNS developer documentation, analogous to the recommendation in the original report.

---

### Proof of Concept

1. An SNS neuron holder submits a `DeregisterDappCanisters` proposal:
   ```
   canister_ids: [<live_dapp_canister_id>]
   new_controllers: [<mistyped_or_burned_principal>]
   ```
2. `validate_and_render_deregister_dapp_canisters` accepts the proposal because `new_controllers` is non-empty and none of the `canister_ids` are SNS-reserved. [5](#0-4) 
3. The governance community votes yes (the rendered proposal shows the principal string, which looks plausible).
4. `perform_deregister_dapp_canisters` calls `set_dapp_controllers` on SNS Root with `controller_principal_ids: [<mistyped_principal>]`. [6](#0-5) 
5. SNS Root calls `update_settings` on the IC management canister, replacing the dapp's controller list with the mistyped principal, and removes the dapp from `dapp_canister_ids`. [7](#0-6) 
6. The SNS permanently loses control of the dapp canister. No SNS proposal can recover it because it is no longer in `dapp_canister_ids` and the SNS is no longer a controller.

        management_canister_client: &'a impl ManagementCanisterClient,
        own_canister_id: PrincipalId,
        caller: PrincipalId,
        request: &'a SetDappControllersRequest,
    ) -> SetDappControllersResponse {
        let is_authorized = self_ref.with(|self_ref| {
            caller == self_ref.borrow().swap_canister_id()
                || caller == self_ref.borrow().governance_canister_id()
        });
        // TODO(NNS1-1993): Remove this assertion and return an error type instead.
        assert!(is_authorized, "Caller ({caller}) is not authorized.");

        // Grab a snapshot of canisters to operate on.
        let dapp_canister_ids = match &request.canister_ids {
            Some(canister_ids) => canister_ids.canister_ids.clone(),
            // If no canister list is specified, we take all the canisters controlled by root.
            None => {
                let is_authorized_to_set_all_controllers =
                    self_ref.with(|self_ref| caller == self_ref.borrow().swap_canister_id());
                if is_authorized_to_set_all_controllers {
                    self_ref.with(|self_ref| self_ref.borrow().dapp_canister_ids.clone())
                } else {
                    // TODO(NNS1-1993): Remove this panic and return an error type instead.
                    panic!("Only the swap canister is authorized to set all dapp controllers")
                }
            }
        };
