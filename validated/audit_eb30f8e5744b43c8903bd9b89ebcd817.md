Audit Report

## Title
SNS Swap `Swap::scale()` Floor Division Produces Zero SNS Tokens for Minimum-Contributing Participants When Neurons' Fund Dilutes the Rate - (File: `rs/sns/swap/src/swap.rs`)

## Summary

The SNS Swap canister's `Swap::scale()` function uses floor (integer) division to compute each participant's SNS token allocation. The swap initialization validation (`validate_participation_constraints`) checks that `min_participant_icp_e8s` is large enough using only `max_direct_participation_icp_e8s` as the denominator, but the actual `scale()` call at finalization divides by `total_participant_icp_e8s`, which includes Neurons' Fund participation. When Neurons' Fund participation is significant, `scale()` can return 0 for participants contributing the minimum allowed ICP, causing neuron recipes with `amount_e8s = 0` to be created while the participant's ICP has already been swept to SNS governance — a permanent loss of funds.

## Finding Description

**Root cause — `Swap::scale()` uses floor division with no zero-guard:**

`rs/sns/swap/src/swap.rs` lines 742–751 confirm the function computes `(amount_icp_e8s * total_sns_e8s) / total_icp_e8s` in 128-bit integer space with no guard against a zero result:

```rust
fn scale(amount_icp_e8s: u64, total_sns_e8s: u64, total_icp_e8s: NonZeroU64) -> u64 {
    assert!(amount_icp_e8s <= u64::from(total_icp_e8s));
    let r = (amount_icp_e8s as u128)
        .saturating_mul(total_sns_e8s as u128)
        .div(NonZeroU128::from(total_icp_e8s));
    assert!(r <= u64::MAX as u128);
    r as u64
}
```

**Validation uses the wrong denominator:**

`rs/sns/init/src/lib.rs` lines 1636–1638 compute the minimum SNS tokens a participant receives using only `max_direct_participation_icp_e8s`:

```rust
let min_participant_sns_e8s = min_participant_icp_e8s as u128
    * initial_swap_amount_e8s as u128
    / max_direct_participation_icp_e8s as u128;
```

But at finalization, `create_sns_neuron_recipes` (lines 818–821) uses `self.current_total_participation_e8s()` — which includes Neurons' Fund ICP — as the denominator passed to `scale()`. When Neurons' Fund participation is significant, `total_participant_icp_e8s >> max_direct_participation_icp_e8s`, and the actual SNS tokens per ICP is far lower than what validation assumed.

**Zero propagates silently through basket construction:**

Lines 848–852 call `scale()` and pass the result directly to `create_sns_neuron_basket_for_direct_participant` with no zero-check:

```rust
let amount_sns_e8s = Swap::scale(
    buyer_state.amount_icp_e8s(),
    sns_being_offered_e8s,
    total_participant_icp_e8s,
);
// ... no zero-check ...
match create_sns_neuron_basket_for_direct_participant(
    &buyer_principal,
    amount_sns_e8s,  // can be 0
    ...
```

`apportion_approximately_equally(0, count)` (lines 203–244) returns `Ok(vec![0; count])` — all zeros, no error — because `quotient = 0/count = 0`, `remainder = 0`, and the loop body is never entered. Neuron recipes are then created with `amount_e8s: 0` (lines 3331–3348).

During `sweep_sns`, the swap attempts to transfer `0 - transaction_fee_e8s` SNS tokens to each neuron account. This underflows (for non-zero fees), the transfer fails, and the recipe is marked failed. The participant's ICP has already been swept to SNS governance in `sweep_icp`, so the ICP is permanently lost with no SNS tokens received.

## Impact Explanation

Direct participants who contribute `min_participant_icp_e8s` ICP — the minimum amount explicitly validated as safe — can permanently lose their ICP without receiving any SNS tokens. This is a concrete, irreversible loss of user funds in the SNS framework, matching the allowed impact: **"Significant SNS security impact with concrete user or protocol harm"** — **High ($2,000–$10,000)**.

## Likelihood Explanation

Neurons' Fund (matched funding) participation is the standard configuration for new SNS launches. The Neurons' Fund contribution is determined by the NNS governance matching function and can be substantial. If a swap is configured with `min_participant_icp_e8s` at the minimum value that just barely passes validation (i.e., `min_participant_icp_e8s * initial_swap_amount_e8s ≈ max_direct_participation_icp_e8s`), then any Neurons' Fund participation at all causes `scale()` to return 0 for minimum-contributing direct participants. No special privileges are required; any user contributing the minimum allowed ICP amount is affected. The configuration is realistic for competitive SNS launches where the Neurons' Fund is expected to match direct participation.

## Recommendation

1. **Fix the validation denominator** in `validate_participation_constraints` (`rs/sns/init/src/lib.rs`): use `max_direct_participation_icp_e8s + max_neurons_fund_participation_icp_e8s` as the denominator when computing `min_participant_sns_e8s`, so the worst-case SNS token allocation accounts for maximum possible Neurons' Fund dilution.

2. **Add a zero-guard in `create_sns_neuron_recipes`** (`rs/sns/swap/src/swap.rs`): before calling `create_sns_neuron_basket_for_direct_participant`, check if `amount_sns_e8s == 0` and either skip the participant (counting as `invalid`) or refund their ICP.

3. **Add a minimum-per-neuron guard**: before calling `generate_vesting_schedule`, verify that `total_amount_e8s >= count * (min_stake + fee)` so no individual neuron in the basket receives 0 tokens.

## Proof of Concept

Configure a swap with matched funding:
- `max_direct_participation_icp_e8s = 100 ICP` (10,000,000,000 e8s)
- `initial_swap_amount_e8s = 1 SNS token` (100,000,000 e8s)
- `min_participant_icp_e8s = 1 ICP` (100,000,000 e8s)
- `neuron_basket_count = 1`, `neuron_minimum_stake_e8s = 1`, `transaction_fee_e8s = 0`

**Validation passes** (lines 1636–1642):
```
min_participant_sns_e8s = (100_000_000 * 100_000_000) / 10_000_000_000 = 1
1 >= 1 * (1 + 0) = 1  ✓
```

**Neurons' Fund contributes 1 ICP**, making `total_participant_icp_e8s = 101 ICP` (10,100,000,000 e8s).

**Direct participant contributes 1 ICP** (`min_participant_icp_e8s`):

`Swap::scale(100_000_000, 100_000_000, 10_100_000_000)`:
```
= (100_000_000 * 100_000_000) / 10_100_000_000
= 10_000_000_000_000_000 / 10_100_000_000
= 0  (floor division)
```

`apportion_approximately_equally(0, 1)` returns `Ok(vec![0])`. A neuron recipe with `amount_e8s: 0` is created. The participant's 1 ICP is swept to SNS governance. The `sweep_sns` transfer of 0 tokens fails (or succeeds with 0 tokens transferred). The participant permanently loses 1 ICP with no SNS tokens received.

A deterministic unit test can be written against `Swap::scale` and `create_sns_neuron_recipes` with these exact parameters to reproduce the zero-token outcome without any mainnet interaction.
