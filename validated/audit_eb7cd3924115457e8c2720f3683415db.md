Audit Report

## Title
ICRC-2 Approve Deduplication Bypass via `fee=None` vs `fee=Some(F)` — (`rs/ledger_suite/icrc1/ledger/src/main.rs`)

## Summary
In `icrc2_approve_not_async`, the caller-supplied `fee` field is preserved as `None` or `Some(expected_fee_tokens)` into `Operation::Approve`. Because `Transaction` serializes via `FlattenedTransaction` (through `#[serde(into = "FlattenedTransaction<Tokens>")]`) and `FlattenedTransaction.fee` carries `#[serde(skip_serializing_if = "Option::is_none")]`, two approve calls that are semantically identical but differ only in `fee=None` vs `fee=Some(F)` produce distinct CBOR encodings and thus distinct transaction hashes. The deduplication store admits both, debiting the full fee twice.

## Finding Description
**Root cause — line 870, `rs/ledger_suite/icrc1/ledger/src/main.rs`:**
```rust
fee: arg.fee.map(|_| expected_fee_tokens),
```
`None` stays `None`; `Some(anything_valid)` becomes `Some(expected_fee_tokens)`.

**Serialization path — lines 373–376, `rs/ledger_suite/icrc1/src/lib.rs`:**
```rust
#[serde(into = "FlattenedTransaction<Tokens>")]
pub struct Transaction<Tokens: TokensType> { … }
```
`hash()` (line 432) calls `ciborium::ser::into_writer(self, …)`, which invokes the `Into<FlattenedTransaction>` conversion.

**Fee propagation — lines 313–318, `rs/ledger_suite/icrc1/src/lib.rs`:**
```rust
fee: match &t.operation {
    Approve { fee, .. } => fee.to_owned(),   // None or Some(F)
    …
},
```

**CBOR omission — lines 116–118, `rs/ledger_suite/icrc1/src/lib.rs`:**
```rust
#[serde(skip_serializing_if = "Option::is_none")]
fee: Option<Tokens>,
```
`None` → `fee` key absent from CBOR map → hash H1.  
`Some(F)` → `fee` key present → hash H2 ≠ H1.

**Deduplication check — lines 249–253, `rs/ledger_suite/common/ledger_canister_core/src/ledger.rs`:**
```rust
if let Some(block_height) = ledger.transactions_by_hash().get(&tx_hash) {
    return Err(TransferError::TxDuplicate { … });
}
```
H1 ≠ H2, so the second call is not recognized as a duplicate.

**Fee debit — lines 537–539, `rs/ledger_suite/icrc1/src/lib.rs`:**
```rust
.burn(from, fee.clone().unwrap_or(effective_fee.clone()))?;
```
Both TX1 (`fee=None`) and TX2 (`fee=Some(F)`) debit F. Total debit: 2×F.

The ICP ledger avoids this by always storing `fee: expected_fee` unconditionally (line 1385, `rs/ledger_suite/icp/ledger/src/main.rs`), ensuring a single canonical hash regardless of what the caller supplied.

## Impact Explanation
Any user who submits an approve with `fee=None`, receives no response (network timeout, canister busy), and retries with `fee=Some(F)` — a standard SDK retry pattern — will have their balance debited by 2×F instead of F. The deduplication mechanism, which exists precisely to protect users in retry scenarios, is silently bypassed. This constitutes concrete, repeatable financial loss on the ICRC-1 ledger, a production in-scope financial integration. Severity: **High** — significant ledger security impact with concrete user financial harm.

## Likelihood Explanation
No special privileges are required; the approver triggers this themselves. Any ICRC-1 client library that retries approve calls and normalizes the `fee` field differently between attempts (omitting it on first try, including it on retry, or vice versa) will trigger the double debit. This is a common pattern in SDK retry logic. The only precondition is having sufficient balance for two fees.

## Recommendation
Normalize the `fee` field unconditionally before constructing the transaction, matching the ICP ledger's behavior. In `rs/ledger_suite/icrc1/ledger/src/main.rs` line 870, replace:
```rust
fee: arg.fee.map(|_| expected_fee_tokens),
```
with:
```rust
fee: Some(expected_fee_tokens),
```
This ensures both `fee=None` and `fee=Some(F)` produce an identical `Operation::Approve` struct, identical `FlattenedTransaction`, identical CBOR encoding, and thus identical transaction hash — restoring the deduplication invariant.

## Proof of Concept
```rust
// Same created_at_time T, same from/spender/amount/memo/expires_at
let base = ApproveArgs { created_at_time: Some(T), fee: None, ..defaults };

let r1 = icrc2_approve_not_async(caller, ApproveArgs { fee: None, ..base.clone() });
// r1 = Ok(block_0); balance = initial - F

let r2 = icrc2_approve_not_async(caller, ApproveArgs { fee: Some(F), ..base.clone() });
// r2 = Ok(block_1) — NOT TxDuplicate; balance = initial - 2*F

assert!(r1.is_ok() && r2.is_ok());
assert_eq!(balance_of(caller), initial_balance - 2 * F);
// Expected: initial_balance - F (dedup should have caught r2)
```
This can be implemented as a deterministic PocketIC integration test against any ICRC-1 ledger canister with a non-zero transfer fee.
