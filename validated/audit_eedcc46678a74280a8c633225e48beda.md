### Title
Zero `effective_gas_price` Bypasses Gas Payment in Silo Mode When `max_fee_per_gas=0` — (`engine/src/engine.rs`)

### Summary

When silo mode is active with `fixed_gas = Some(N)`, a whitelisted sender can submit an EVM transaction with `max_fee_per_gas = 0`. Because `block_base_fee_per_gas` is hardcoded to `U256::zero()`, `charge_gas` computes `effective_gas_price = 0` and `prepaid_amount = fixed_gas * 0 = 0`, executing the transaction with zero deduction from the sender's balance and zero reward to the relayer.

---

### Finding Description

`block_base_fee_per_gas` is permanently hardcoded to zero: [1](#0-0) 

The early-return guard in `charge_gas` is:

```rust
if transaction.max_fee_per_gas.is_zero()
    && fixed_gas.is_none()
    && block_base_fee_per_gas.is_zero()
{
    return Ok(GasPaymentResult::default());
}
``` [2](#0-1) 

When `fixed_gas.is_some()`, the `fixed_gas.is_none()` predicate is `false`, so the early return is **skipped** even though both `max_fee_per_gas` and `block_base_fee_per_gas` are zero. Execution continues to:

```rust
let priority_fee_per_gas = transaction
    .max_priority_fee_per_gas
    .min(transaction.max_fee_per_gas - block_base_fee_per_gas);
// = min(0, 0 - 0) = 0
let effective_gas_price = priority_fee_per_gas + block_base_fee_per_gas;
// = 0 + 0 = 0
let prepaid_amount = fixed_gas
    .map_or(transaction.gas_limit, EthGas::as_u256)
    .checked_mul(effective_gas_price)   // N * 0 = 0
    .map(Wei::new)
    .ok_or(GasPaymentError::EthAmountOverflow)?;
``` [3](#0-2) 

`prepaid_amount = Wei::zero()`. The sender's balance is decremented by zero: [4](#0-3) 

After execution, `refund_unused_gas` short-circuits immediately because `effective_gas_price.is_zero()`:

```rust
if gas_result.effective_gas_price.is_zero() {
    return Ok(());
}
``` [5](#0-4) 

The relayer receives no reward.

---

### Impact Explanation

**Impact: High — Theft of Unclaimed Yield.**

The relayer pays real NEAR gas to execute the EVM transaction but receives zero EVM gas compensation. The sender executes arbitrary EVM work for free. The `fixed_gas` mechanism exists precisely to guarantee the relayer a predictable fee; this path nullifies it entirely. The invariant `sender_balance_before - sender_balance_after == fixed_gas * effective_gas_price` is broken: the left side is zero when it should be `N * price`.

This does **not** reach Critical (direct theft of at-rest/in-motion user funds) because no existing EVM balance is transferred out of a victim account by this mechanism alone. The loss is the relayer's expected gas revenue — unclaimed yield.

---

### Likelihood Explanation

Any address whitelisted in the silo Account+Address lists can trigger this. The only preconditions are:
1. Silo mode active with `fixed_gas = Some(N > 0)` — set by the owner.
2. Sender whitelisted — a normal operational state for silo deployments.
3. Transaction submitted with `max_fee_per_gas = 0` — trivially constructable by any EIP-1559 or legacy transaction signer.

No admin compromise, key leak, or external oracle is required. The path is reachable on every silo deployment where whitelisted users are not forced off-chain to use a minimum gas price. [6](#0-5) 

---

### Recommendation

Add an explicit guard in `charge_gas` that rejects (or treats as zero-fee) when `fixed_gas.is_some()` but `effective_gas_price` resolves to zero:

```rust
// After computing effective_gas_price:
if fixed_gas.is_some() && effective_gas_price.is_zero() {
    return Err(GasPaymentError::MaxFeePerGasLessThanBaseFee);
    // or a new variant: ZeroGasPriceInSiloMode
}
```

Alternatively, enforce a minimum `max_fee_per_gas` check at the `submit` entry point when `fixed_gas.is_some()`, rejecting transactions where `max_fee_per_gas == 0`.

---

### Proof of Concept

// In engine/src/engine.rs #[cfg(test)] block:
#[test]
fn test_silo_fixed_gas_zero_max_fee_bypasses_payment() {
    let sender = Address::zero();
    let current_account_id = AccountId::default();
    let env = Fixed::default();
    let storage = RefCell::new(Storage::default());
    let mut io = StoragePointer(&storage);
    let initial_balance = Wei::new_u64(1_000_000_000);
    add_balance(&mut io, &sender, initial_balance).unwrap();

    let mut engine: Engine<_, _> =
        Engine::new_with_state(EngineState::default(), sender, current_account_id, io, &env);

    let transaction = NormalizedEthTransaction {
        address: sender,
        chain_id: None,
        nonce: U256::zero(),
        gas_limit: 2_000_000.into(),       // >= fixed_gas
        max_priority_fee_per_gas: U256::zero(),
        max_fee_per_gas: U256::zero(),      // attacker sets to 0
        to: Some(Address::zero()),
        value: Wei::zero(),
        data: vec![],
        access_list: vec![],
        authorization_list: vec![],
    };

    let fixed_gas = Some(EthGas::new(1_000_000));
    let result = engine.charge_gas(&sender, &transaction, None, fixed_gas).unwrap();

    // Invariant broken: prepaid_amount should be 1_000_000 * price, but is zero
    assert_eq!(result.prepaid_amount, Wei::zero());
    assert_eq!(result.effective_gas_price, U256::zero());

    // Sender balance unchanged — paid nothing
    let balance_after = get_balance(&engine.io, &sender);
    assert_eq!(balance_after, initial_balance);
}

```rust
    pub fn charge_gas(
        &mut self,
        sender: &Address,
        transaction: &NormalizedEthTransaction,
        max_gas_price: Option<U256>,
        fixed_gas: Option<EthGas>,
    ) -> Result<GasPaymentResult, GasPaymentError> {
        let block_base_fee_per_gas = self.block_base_fee_per_gas();
        if transaction.max_fee_per_gas.is_zero()
            && fixed_gas.is_none()
            && block_base_fee_per_gas.is_zero()
        {
            return Ok(GasPaymentResult::default());
        }

        if transaction.max_fee_per_gas < block_base_fee_per_gas {
            return Err(GasPaymentError::MaxFeePerGasLessThanBaseFee);
        }

        let priority_fee_per_gas = transaction
            .max_priority_fee_per_gas
            .min(transaction.max_fee_per_gas - block_base_fee_per_gas);
        let priority_fee_per_gas = max_gas_price.map_or(priority_fee_per_gas, |price| {
            price.min(priority_fee_per_gas)
        });
        let effective_gas_price = priority_fee_per_gas + block_base_fee_per_gas;
        // First, we try to use `fixed_gas`. At this point we already know that the `fixed_gas` is
        // less than the `gas_limit`. It allows avoiding refunding unused gas to the sender later.
        let prepaid_amount = fixed_gas
            .map_or(transaction.gas_limit, EthGas::as_u256)
            .checked_mul(effective_gas_price)
            .map(Wei::new)
            .ok_or(GasPaymentError::EthAmountOverflow)?;

        let new_balance = get_balance(&self.io, sender)
            .checked_sub(prepaid_amount)
            .ok_or(GasPaymentError::OutOfFund)?;

        set_balance(&mut self.io, sender, &new_balance);

        self.gas_price = effective_gas_price;

        Ok(GasPaymentResult {
            prepaid_amount,
            effective_gas_price,
            priority_fee_per_gas,
        })
    }
