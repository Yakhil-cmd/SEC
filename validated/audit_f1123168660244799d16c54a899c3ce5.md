Audit Report

## Title
Stale `API_BOUNDARY_NODE_PRINCIPALS` Grants Persistent `FullRead` Access to Decommissioned Boundary Nodes on Registry Poll Failure â€” (`rs/boundary_node/rate_limits/canister/canister.rs`, `storage.rs`, `access_control.rs`)

## Summary
`API_BOUNDARY_NODE_PRINCIPALS` is a non-stable, in-memory `thread_local` `HashSet` that is only replaced on a successful registry poll. When `periodically_poll_api_boundary_nodes` encounters either error branch, the set is left entirely unchanged. A boundary node decommissioned from the registry therefore retains its principal in the stale set for the full duration of any poll-failure window. Because `get_rule_by_id`, `get_config`, and `get_rules_by_incident_id` are `#[query]` methods, they bypass `inspect_message` entirely, and `AccessLevelResolver` grants `FullRead` solely on membership in the stale set, allowing the decommissioned node to receive unredacted `rule_raw` and `description` for every undisclosed rate-limit rule.

## Finding Description

**Root cause â€” non-stable set, no invalidation on failure.**
`API_BOUNDARY_NODE_PRINCIPALS` is declared as a plain `thread_local!` `RefCell<HashSet<Principal>>` with no stable-memory backing and no TTL: [1](#0-0) 

`set_api_boundary_nodes_principals` performs a full replacement only when called: [2](#0-1) 

Both error branches of `periodically_poll_api_boundary_nodes` log and return without touching the set: [3](#0-2) 

**`inspect_message` does not cover query methods.**
On the Internet Computer, `inspect_message` is invoked only for ingress *update* messages. All three read methods are `#[query]`: [4](#0-3) [5](#0-4) [6](#0-5) 

They are never evaluated by `inspect_message`, so the `REPLICATED_QUERY_METHOD` guard is irrelevant for the normal query call path.

**`AccessLevelResolver` reads directly from the stale set.** [7](#0-6) 

**`FullRead` bypasses the confidentiality formatter.**
In `RuleGetter::get`, when `is_authorized_viewer` is `true`, the raw unredacted `OutputRuleMetadata` is returned immediately: [8](#0-7) 

The formatter that would set `rule_raw = None` and `description = None` for undisclosed rules is skipped: [9](#0-8) 

The same pattern applies in `ConfigGetter::get` and `IncidentGetter::get`. [10](#0-9) [11](#0-10) 

## Impact Explanation
A decommissioned API boundary node operator who retains the node's private key can call `get_rule_by_id`, `get_config`, or `get_rules_by_incident_id` as query calls during any registry poll-failure window and receive the full unredacted `rule_raw` and `description` of every undisclosed rate-limit rule â€” including active incident-response countermeasures intentionally kept confidential to prevent evasion. This is a concrete unauthorized disclosure of confidential operational security data tied to boundary/API infrastructure, fitting the **Medium ($200â€“$2,000)** category: "Difficult attack requiring â€¦ node/boundary-node control â€¦ but still with meaningful security impact."

## Likelihood Explanation
The attacker must have previously operated an API boundary node and still hold its private key after decommissioning. Registry poll failures are a realistic operational condition (transient network partitions, registry canister upgrades). The exposure window equals the full duration of the failure. No additional privileges beyond the decommissioned node's own key are required. The attack is repeatable across every poll-failure interval.

## Recommendation
In both error branches of `periodically_poll_api_boundary_nodes`, clear the stale set:

```rust
Ok((Err(err),)) => {
    log!(P0, "...");
    canister_api.set_api_boundary_nodes_principals(vec![]);
    ("failure", "calling_canister_method_failed")
}
Err(err) => {
    log!(P0, "...");
    canister_api.set_api_boundary_nodes_principals(vec![]);
    ("failure", "canister_call_rejected")
}
```

Alternatively, store a `last_successful_poll_timestamp` per principal and reject `FullRead` grants for entries older than `2 Ă— polling_interval`.

## Proof of Concept
State-machine test outline (PocketIC or `ic-state-machine-tests`):

1. Install the rate-limit canister with a short polling interval and a mock registry.
2. Register a boundary node; advance time one interval so its principal enters `API_BOUNDARY_NODE_PRINCIPALS` via a successful poll.
3. Call `add_config` with an undisclosed rule (`disclosed_at: None`).
4. Configure the mock registry to return `Err(...)` on all subsequent calls.
5. Remove the boundary node from the mock registry's node list.
6. Advance time past several polling intervals (all polls fail; set is never updated).
7. Call `get_rule_by_id` from the decommissioned node's principal as a query.
8. Assert the response contains non-`None` `rule_raw` and `description`, confirming full unredacted access despite decommissioning.
