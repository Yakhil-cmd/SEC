### Title
Missing Upfront Access Control on `swap_node_in_subnet_directly` with Default `allow_all()` Caller Policy Enables Rate-Limit DoS - (File: rs/registry/canister/canister/canister.rs)

---

### Summary

The Registry canister exposes `swap_node_in_subnet_directly` as a publicly callable update endpoint with no upfront caller check. The caller access policy (`NODE_SWAPPING_CALLERS_POLICY`) defaults to `AccessList::allow_all()`, meaning any unprivileged principal can invoke the endpoint and reach the rate-limiter reservation step. Because the rate-limiter reservation is acquired **before** the node-operator ownership check (`validate_node_swap`), an attacker can exhaust the per-subnet rate-limit slot without owning any nodes, blocking legitimate node operators from performing swaps for up to 4 hours per subnet.

---

### Finding Description

Every other sensitive Registry canister update method calls a `check_caller_is_*` guard at the canister entry point before dispatching:

```rust
// rs/registry/canister/canister/canister.rs:857-863
fn remove_node_operators() {
    check_caller_is_governance_and_log("remove_node_operators");
    over(candid_one, |payload| remove_node_operators_(payload));
}
```

`swap_node_in_subnet_directly` has no such guard:

```rust
// rs/registry/canister/canister/canister.rs:831-842
#[unsafe(export_name = "canister_update swap_node_in_subnet_directly")]
fn swap_node_in_subnet_directly() {
    over(candid_one, |payload: SwapNodeInSubnetDirectlyPayload| {
        swap_node_in_subnet_directly_(payload)
    });
}
``` [1](#0-0) 

The internal `swap_nodes_inner` function does check a caller whitelist, but the default policy is `AccessList::allow_all()`:

```rust
// rs/registry/canister/src/flags.rs:18-20
static NODE_SWAPPING_CALLERS_POLICY: RefCell<AccessList<PrincipalId>>
    = RefCell::new(AccessList::allow_all());
static NODE_SWAPPING_SUBNETS_POLICY: RefCell<AccessList<SubnetId>>
    = RefCell::new(AccessList::allow_all());
``` [2](#0-1) 

The execution order inside `swap_nodes_inner` is:

1. Feature-enabled flag check
2. Payload validation
3. Caller whitelist check (`swapping_enabled_for_caller`) — passes for everyone with default `allow_all()`
4. Subnet whitelist check — passes for every subnet with default `allow_all()`
5. **Rate-limiter `try_reserve` — capacity consumed here**
6. `validate_node_swap` — checks caller is the actual node operator of both nodes
7. `swap_nodes_in_subnet` — performs the registry mutation
8. Rate-limiter `commit` [3](#0-2) 

If step 6 fails (caller is not the node operator), the function returns early via `?`. The `SwapReservation` is dropped without `commit` ever being called. The rate-limiter capacity consumed at step 5 is not restored, because `commit` is the only path that finalises the reservation:

```rust
// rs/registry/canister/src/mutations/do_swap_node_in_subnet_directly.rs:139-145
let reservation =
    SWAP_LIMITER.with_borrow_mut(|limiter| limiter.try_reserve(caller, subnet_id, now))?;
self.validate_node_swap(old_node_id, new_node_id, caller, subnet_id)?;
self.swap_nodes_in_subnet(subnet_id, old_node_id, new_node_id)?;
SWAP_LIMITER.with_borrow_mut(|limiter| limiter.commit(reservation, now));
``` [4](#0-3) 

The subnet rate-limiter is configured with `max_capacity = 1` and a replenishment interval of 4 hours:

```rust
// rs/registry/canister/src/mutations/do_swap_node_in_subnet_directly.rs:25-29
const NODE_SWAPS_SUBNET_CAPACITY_INTERVAL_HOURS: u64 = 4;
const NODE_SWAPS_NODE_OPERATOR_CAPACITY_INTERVAL_HOURS: u64 = 24;
``` [5](#0-4) 

---

### Impact Explanation

An unprivileged ingress sender can:

1. Read the Registry to find any node currently assigned to a target subnet (public query).
2. Call `swap_node_in_subnet_directly` with that node as `old_node_id` and any principal as `new_node_id`.
3. The call passes the `allow_all()` caller and subnet whitelist checks, reaches `try_reserve`, and consumes the subnet's single rate-limit slot.
4. The call then fails at `validate_node_swap` (attacker is not the node operator), but the slot is not restored.
5. All legitimate node operators are blocked from swapping nodes on that subnet for up to 4 hours.
6. The attacker repeats every 4 hours with zero cost (only cycles for the call).

This is a **service availability impact** on the node-rotation feature: legitimate node operators cannot perform hardware maintenance swaps on targeted subnets.

---

### Likelihood Explanation

- The endpoint is reachable by any ingress sender with no prerequisites.
- The `NODE_SWAPPING_CALLERS_POLICY` defaults to `allow_all()` in the compiled binary; no special configuration is needed to exploit this.
- The attack requires only knowledge of a node ID assigned to the target subnet, which is publicly readable from the Registry.
- The attacker needs to repeat the call every 4 hours, which is trivially automatable. [6](#0-5) 

---

### Recommendation

1. **Move the caller check to the canister entry point**, consistent with all other sensitive Registry methods:

```rust
#[unsafe(export_name = "canister_update swap_node_in_subnet_directly")]
fn swap_node_in_subnet_directly() {
    check_caller_is_node_operator_and_log("swap_node_in_subnet_directly");
    over(candid_one, |payload: SwapNodeInSubnetDirectlyPayload| {
        swap_node_in_subnet_directly_(payload)
    });
}
```

2. **Change the default `NODE_SWAPPING_CALLERS_POLICY` from `allow_all()` to `deny_all()`** and require explicit opt-in via the init payload. This enforces the intended phased-rollout access control even in the default configuration.

3. **Move the rate-limiter `try_reserve` call to after `validate_node_swap`**, so capacity is only consumed when the caller is confirmed to be the legitimate node operator.

---

    fn swap_nodes_inner(
        &mut self,
        payload: SwapNodeInSubnetDirectlyPayload,
        caller: PrincipalId,
        now: SystemTime,
    ) -> Result<(), SwapError> {
        // Check if the feature is enabled on the network.
        if !is_node_swapping_enabled() {
            return Err(SwapError::FeatureDisabled);
        }

        // Check if the payload is valid by itself.
        payload.validate()?;
        let (old_node_id, new_node_id) =
            (payload.old_node_id.unwrap(), payload.new_node_id.unwrap());

        //Check if the feature is allowed on the target subnet and for the caller
        Self::swapping_enabled_for_caller(caller)?;
        let subnet_id = self.find_subnet_for_old_node(old_node_id)?;
        Self::swapping_allowed_on_subnet(subnet_id)?;

        let reservation =
            SWAP_LIMITER.with_borrow_mut(|limiter| limiter.try_reserve(caller, subnet_id, now))?;

        self.validate_node_swap(old_node_id, new_node_id, caller, subnet_id)?;
        self.swap_nodes_in_subnet(subnet_id, old_node_id, new_node_id)?;

        SWAP_LIMITER.with_borrow_mut(|limiter| limiter.commit(reservation, now));
        Ok(())
    }
