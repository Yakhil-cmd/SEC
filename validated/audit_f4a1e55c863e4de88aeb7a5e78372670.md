Audit Report

## Title
`API_BOUNDARY_NODE_PRINCIPALS` Heap State Cleared on Upgrade Causes Post-Upgrade `RestrictedRead` Downgrade for All API Boundary Nodes — (`rs/boundary_node/rate_limits/canister/storage.rs`, `canister.rs`)

## Summary
`API_BOUNDARY_NODE_PRINCIPALS` is stored in a plain heap `thread_local! HashSet<Principal>` with no stable-memory backing, so it is wiped on every canister upgrade. The `post_upgrade` hook only schedules a periodic timer (not an immediate poll), creating a guaranteed window — up to `registry_polling_period_secs` (300 s in production) — during which the set is empty. During this window, every API boundary node calling `get_config` is treated as `RestrictedRead`, receiving a response with `is_redacted=true` and `rule_raw=None` for all undisclosed rules, making those rules unenforceable.

## Finding Description
`API_BOUNDARY_NODE_PRINCIPALS` is declared as a plain `thread_local! RefCell<HashSet<Principal>>` at `storage.rs:149`, while all other security-critical state (`CONFIGS`, `RULES`, `INCIDENTS`, `AUTHORIZED_PRINCIPAL`) is backed by `StableBTreeMap` over `VirtualMemory` and survives upgrades (`storage.rs:125–147`).

`post_upgrade` at `canister.rs:100–104` calls `init`, which calls `periodically_poll_api_boundary_nodes` at `canister.rs:85–88`. That function uses `ic_cdk_timers::set_timer_interval` (`canister.rs:254`), which fires only after the full interval elapses — not immediately.

`AccessLevelResolver::get_access_level` (`access_control.rs:38–55`) checks `is_api_boundary_node_principal`; with an empty set, every caller falls through to `AccessLevel::RestrictedRead`. `ConfigGetter::get` (`getter.rs:130–147`) then applies `ConfigConfidentialityFormatter`, which sets `is_redacted=true` and `rule_raw=None` / `description=None` for every rule whose `disclosed_at` is `None` (`confidentiality_formatting.rs:17–28`).

`get_config` is a `#[query]` method (`canister.rs:110`), so `inspect_message` does not apply; boundary nodes receive the silently redacted response without any error or rejection.

## Impact Explanation
During the post-upgrade window (up to 300 s in production), all API boundary nodes receive a redacted config. Any rate-limit rule that has not been publicly disclosed (`disclosed_at=None`) is returned with `rule_raw=None`. Boundary nodes cannot parse or enforce a rule with no raw bytes, so all undisclosed active rate-limit rules are silently dropped from enforcement. Traffic that those rules are intended to block or throttle passes through unimpeded for the duration of the window. This constitutes a significant boundary/API infrastructure security impact with concrete harm to the IC's rate-limiting posture — matching the High ($2,000–$10,000) allowed impact category for significant boundary/API infrastructure security impact.

## Likelihood Explanation
This condition is triggered by every routine canister upgrade — a normal governance operation. No attacker action is required to create the window; an attacker only needs to send traffic during the predictable post-upgrade period. The upgrade proposal is public and the window is deterministic (exactly `registry_polling_period_secs` seconds). The condition is repeatable on every upgrade.

## Recommendation
1. **Persist `API_BOUNDARY_NODE_PRINCIPALS` in stable memory** using a `StableBTreeMap` with a dedicated `MemoryId`, consistent with how `AUTHORIZED_PRINCIPAL` is stored at `storage.rs:137–141`.
2. **Alternatively**, trigger an immediate one-shot poll at the start of `post_upgrade` using `ic_cdk_timers::set_timer(Duration::ZERO, ...)` before registering the interval timer, so the set is populated before any query can be served.

## Proof of Concept
State-machine test outline:
1. Install canister with `registry_polling_period_secs = 300`.
2. Simulate a successful registry poll: populate `API_BOUNDARY_NODE_PRINCIPALS` with principal `P`.
3. Add a rate-limit config containing one undisclosed rule (`disclosed_at=None`).
4. Simulate canister upgrade (`post_upgrade`): heap is cleared, `API_BOUNDARY_NODE_PRINCIPALS` becomes empty.
5. Call `get_config` as principal `P` (before the 300-second timer fires).
6. Assert: response has `is_redacted=true` and `rule_raw=None` for the undisclosed rule.
7. Assert invariant violation: a principal that was a registered API boundary node before upgrade cannot read the full config immediately after upgrade.
