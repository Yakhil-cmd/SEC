### Title
Wrong Variable in Fee Validation Error Message Misleads SNS Parameter Configuration - (File: `rs/sns/governance/src/types.rs`)

### Summary
In `validate_neuron_minimum_stake_e8s`, the format string for the rejection error uses `neuron_minimum_stake_e8s` in the slot intended for `transaction_fee_e8s`. This is a direct analog of the reported pattern: the wrong identifier is passed to a fee-related check, causing the output of that check to misrepresent the actual fee value.

### Finding Description
In `rs/sns/governance/src/types.rs`, the function `validate_neuron_minimum_stake_e8s` checks that `neuron_minimum_stake_e8s > transaction_fee_e8s`. When the check fails, it constructs an error message:

```rust
if neuron_minimum_stake_e8s <= transaction_fee_e8s {
    Err(format!(
        "NervousSystemParameters.neuron_minimum_stake_e8s ({neuron_minimum_stake_e8s}) must be greater than \
        NervousSystemParameters.transaction_fee_e8s ({neuron_minimum_stake_e8s})"
    ))
```

The second interpolation `{neuron_minimum_stake_e8s}` is placed in the position that should display `{transaction_fee_e8s}`. The error message therefore reports the same value for both parameters, hiding the actual `transaction_fee_e8s` value from the caller. [1](#0-0) 

### Impact Explanation
When an SNS deployer or a governance proposal submitter receives this error, the message reads:

> `NervousSystemParameters.neuron_minimum_stake_e8s (X) must be greater than NervousSystemParameters.transaction_fee_e8s (X)`

Both values are shown as `X` (the value of `neuron_minimum_stake_e8s`), even when `transaction_fee_e8s` is a different value. The deployer cannot determine the actual `transaction_fee_e8s` from the error alone, which can cause them to set `neuron_minimum_stake_e8s` to an incorrect value — for example, equal to what they believe the fee is, when the real fee is higher. If the deployer then submits a governance proposal with a `neuron_minimum_stake_e8s` that is still ≤ the real `transaction_fee_e8s`, the proposal will be rejected again with the same misleading message, creating a confusing loop. In the worst case, a deployer may set `neuron_minimum_stake_e8s` to a value that is technically above the threshold shown in the error but still below the real `transaction_fee_e8s`, causing the SNS to be deployed with parameters that violate the economic invariant the check is meant to enforce — if the check itself were ever bypassed or the parameters updated through a path that does not call this validator.

### Likelihood Explanation
This code path is exercised every time `NervousSystemParameters::validate()` is called, which happens during SNS initialization and on every governance proposal that updates nervous system parameters. Any unprivileged user can submit such a proposal. The misleading error is therefore surfaced to any SNS deployer or governance participant who triggers the validation failure. [2](#0-1) 

### Recommendation
Replace the second `{neuron_minimum_stake_e8s}` with `{transaction_fee_e8s}` in the format string:

```rust
Err(format!(
    "NervousSystemParameters.neuron_minimum_stake_e8s ({neuron_minimum_stake_e8s}) must be greater than \
    NervousSystemParameters.transaction_fee_e8s ({transaction_fee_e8s})"
))
``` [3](#0-2) 

### Proof of Concept
1. Construct a `NervousSystemParameters` with `neuron_minimum_stake_e8s = Some(5_000)` and `transaction_fee_e8s = Some(10_000)`.
2. Call `params.validate()`.
3. Observe the returned error string: both the `neuron_minimum_stake_e8s` slot and the `transaction_fee_e8s` slot display `5000`, hiding the real fee of `10000`.
4. A deployer reading this error cannot determine that the actual `transaction_fee_e8s` is `10_000` and may set `neuron_minimum_stake_e8s = 5_001`, which still violates the invariant. [4](#0-3)

    /// Validates that the nervous system parameter neuron_minimum_stake_e8s is well-formed.
    fn validate_neuron_minimum_stake_e8s(&self) -> Result<(), String> {
        let transaction_fee_e8s = self.validate_transaction_fee_e8s()?;

        let neuron_minimum_stake_e8s = self.neuron_minimum_stake_e8s.ok_or_else(|| {
            "NervousSystemParameters.neuron_minimum_stake_e8s must be set".to_string()
        })?;

        if neuron_minimum_stake_e8s <= transaction_fee_e8s {
            Err(format!(
                "NervousSystemParameters.neuron_minimum_stake_e8s ({neuron_minimum_stake_e8s}) must be greater than \
                NervousSystemParameters.transaction_fee_e8s ({neuron_minimum_stake_e8s})"
            ))
        } else {
            Ok(())
        }
    }
