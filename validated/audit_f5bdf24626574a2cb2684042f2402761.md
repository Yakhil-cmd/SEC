### Title
SNS Governance Lacks Voting Power Spike Detection at Proposal Creation, Enabling Last-Minute Stake Inflation to Manipulate Ballot Weights — (File: `rs/sns/governance/src/governance.rs`)

---

### Summary

SNS governance computes ballot voting power from live neuron state at proposal creation time with no spike-detection safeguard. NNS governance recently added a `VotingPowerSnapshots` mechanism that falls back to a previous snapshot when a sudden voting-power spike (>1.5×) is detected. SNS governance has no equivalent. An unprivileged attacker who already holds an SNS neuron with sufficient dissolve delay can transfer a large amount of SNS tokens to their neuron subaccount, call `ClaimOrRefresh` to inflate `cached_neuron_stake_e8s`, then create (or time) a proposal so that the inflated voting power is captured in the ballot, potentially deciding the outcome.

---

### Finding Description

**Root cause — SNS `compute_ballots_for_new_proposal`**

`rs/sns/governance/src/governance.rs` iterates over every neuron at proposal-creation time and records the current live voting power directly into the ballot:

```rust
for (k, v) in self.proto.neurons.iter() {
    if v.dissolve_delay_seconds(now_seconds) < min_dissolve_delay_for_vote {
        continue;
    }
    let voting_power = v.voting_power(now_seconds, ...);
    electoral_roll.insert(k.clone(), Ballot { vote: Vote::Unspecified, voting_power, ... });
}
``` [1](#0-0) 

There is no snapshot history, no spike threshold, and no fallback to a prior snapshot.

**Contrast — NNS `compute_ballots_for_standard_proposal`**

NNS governance maintains periodic `VotingPowerSnapshots` (taken daily by `SnapshotVotingPowerTask`) and, at proposal creation, checks whether the current total potential voting power exceeds 1.5× the minimum in the retained snapshots. If a spike is detected, it uses the previous (lower) snapshot instead: [2](#0-1) [3](#0-2) [4](#0-3) 

**Stake inflation path**

An attacker can inflate their SNS neuron stake at any time via `ClaimOrRefresh → refresh_neuron`, which calls `update_stake` and immediately updates `cached_neuron_stake_e8s`: [5](#0-4) [6](#0-5) 

Because SNS has no spike detection, this inflated stake is captured verbatim in every ballot created after the refresh.

**Confirmed ineligibility of post-creation neurons**

Neurons that do not exist at proposal creation time cannot vote (confirmed by `test_ineligible_neuron_voting_fails`): [7](#0-6) 

The attack therefore does not require creating a new neuron — it only requires inflating an existing one's stake before the proposal is created.

---

### Impact Explanation

An attacker who controls an SNS neuron with dissolve delay ≥ `neuron_minimum_dissolve_delay_to_vote_seconds` can:

1. Transfer a large amount of SNS tokens to their neuron's subaccount.
2. Call `manage_neuron { ClaimOrRefresh }` to update `cached_neuron_stake_e8s`.
3. Create a proposal (or time the attack to coincide with an imminent proposal).
4. Their inflated voting power is captured in the ballot; no spike detection fires.
5. Vote to decide the outcome (treasury transfers, canister upgrades, parameter changes).
6. Begin dissolving the neuron; tokens are recoverable after the dissolve delay.

For high-value SNS proposals the opportunity cost of locking tokens for the dissolve delay period may be economically rational. The attack is directly analogous to the reported "buy before vote, sell after" pattern, adapted to the IC's staking model: the "sell" is replaced by a dissolve-and-disburse sequence, but the ballot manipulation is identical.

**Governance authorization bug** — an unprivileged ingress sender (neuron controller) can acquire disproportionate ballot weight at proposal creation time without any protocol-level safeguard.

---

### Likelihood Explanation

**Medium.** Prerequisites are: (a) an existing SNS neuron with sufficient dissolve delay, and (b) capital to inflate the stake. Both are achievable by any market participant. The attacker must lock funds for the dissolve delay period, but for SNS proposals controlling significant treasury assets this cost is bounded and potentially profitable. The NNS added spike detection precisely because this attack vector was considered realistic enough to warrant a protocol-level fix (NNS proposal 137252, July 2025); SNS has not received the equivalent fix.

---

### Recommendation

Implement a periodic voting-power snapshot mechanism for SNS governance, mirroring the NNS `VotingPowerSnapshots` design:

- Record a snapshot of per-neuron deciding voting power on a regular cadence (e.g., daily).
- At proposal creation, compare the current total potential voting power against the minimum in the retained snapshots.
- If the ratio exceeds a threshold (e.g., 1.5×), use the previous snapshot's voting power map to construct ballots instead of the live state.

This is the exact mitigation already deployed in NNS governance. [8](#0-7)

### Proof of Concept

```
Precondition:
  - Eve holds SNS neuron N with dissolve_delay = 1 year (≥ min_dissolve_delay_to_vote).
  - Alice and Bob each hold neurons with 10,000 SNS tokens.
  - Eve's neuron currently holds 1,000 SNS tokens.

Step 1: Eve transfers 20,001 SNS tokens to neuron N's subaccount on the SNS ledger.

Step 2: Eve calls manage_neuron { ClaimOrRefresh { by: NeuronId } }.
        → refresh_neuron reads ledger balance = 21,001 tokens.
        → neuron.update_stake(21_001 * E8, now) updates cached_neuron_stake_e8s.
        [rs/sns/governance/src/governance.rs:4287-4288]

Step 3: Eve calls manage_neuron { MakeProposal { ... } } (or waits for any proposal).
        → compute_ballots_for_new_proposal iterates neurons.
        → Eve's ballot.voting_power = f(21_001 tokens, dissolve_delay, age).
        → Alice's ballot.voting_power = f(10_000 tokens, ...).
        → Bob's ballot.voting_power  = f(10_000 tokens, ...).
        No spike detection fires.
        [rs/sns/governance/src/governance.rs:5255-5280]

Step 4: Eve votes Yes. Her voting power exceeds Alice + Bob combined.
        Proposal is adopted.

Step 5: Eve calls manage_neuron { Configure { StartDissolving } }.
        After 1 year, Eve disburses 21,001 tokens.
        Net cost: opportunity cost of locking 20,001 tokens for 1 year.
```

The same attack is impossible on NNS governance because `compute_ballots_for_standard_proposal` would detect the spike (21,001 vs. historical ~1,000 tokens = >1.5× threshold) and use a previous snapshot. [9](#0-8)

                return Err(GovernanceError::new_with_message(

                    "Cannot refresh an SNS Neuron controlled by the Neurons' Fund",

            }
        }

        // Get the balance of the neuron from the ledger canister.
        let balance = self.ledger.account_balance(account).await?;

            .neuron_minimum_stake_e8s

        if balance.get_e8s() < min_stake {
            return Err(GovernanceError::new_with_message(
                ErrorType::InsufficientFunds,
                format!(

                        Please make sure that account has at least {:?} e8s (was {:?} e8s)",
                    min_stake,
                    balance.get_e8s()

            ));

        let neuron = self.get_neuron_result_mut(nid)?;
        match neuron.cached_neuron_stake_e8s.cmp(&balance.get_e8s()) {

                    balance.get_e8s(),

                );
                neuron.update_stake(balance.get_e8s(), now);
            }
            Ordering::Less => {
                neuron.update_stake(balance.get_e8s(), now);
            }
            // If the stake is the same as the account balance,
            // just return the neuron id (this way this method
            // also serves the purpose of allowing to discover the
            // neuron id based on the memo and the controller).
            Ordering::Equal => (),
        };

        Ok(())

    /// # Voting Power Spike
    ///
    /// A voting power spike is detected when the total potential voting power is more than 1.5
    /// times the minimum total potential voting power in the snapshots. If a voting power spike is
    /// detected, then the previous ballots are returned.
    #[allow(clippy::type_complexity)]
    fn compute_ballots_for_standard_proposal(
        &self,
        now_seconds: u64,
    ) -> Result<
        (
            HashMap<u64, Ballot>,
            u64,         /*potential_voting_power*/
            Option<u64>, /*previous_ballots_timestamp_seconds*/
        ),
        GovernanceError,
    > {
        let current_voting_power_snapshot = self
            .neuron_store
            .compute_voting_power_snapshot_for_standard_proposal(
                self.voting_power_economics(),
                now_seconds,
            )?;

        // Check if there is a voting power spike. If there is, then the return value here
        // will be `Some(...)`.
        let maybe_previous_ballots_if_voting_power_spike_detected = VOTING_POWER_SNAPSHOTS
            .with_borrow(|snapshots| {
                snapshots.previous_ballots_if_voting_power_spike_detected(
                    current_voting_power_snapshot.total_potential_voting_power(),
                    now_seconds,
                )
            });

        let (voting_power_snapshot, previous_ballots_timestamp_seconds) =
            match maybe_previous_ballots_if_voting_power_spike_detected {
                // This is the extraordinary case - we have a voting power spike, and we
                // need to use the previous snapshot.
                Some((previous_snapshot_timestamp, previous_snapshot)) => {
                    (previous_snapshot, Some(previous_snapshot_timestamp))
                }
                // This is the normal case - we have no voting power spike, so we use the
                // current snapshot.
                None => (current_voting_power_snapshot, None),
            };

        let (ballots, total_potential_voting_power) =
            voting_power_snapshot.create_ballots_and_total_potential_voting_power();
        Ok((
            ballots,
            total_potential_voting_power,
            previous_ballots_timestamp_seconds,
        ))
    }

    /// Given a total potential voting power, checks if there is a voting power spike. If a spike is
    /// detected, it returns the timestamp and totals of the snapshot with the minimum total
    /// potential voting power. If no spike is detected, it returns None.
    fn totals_entry_with_minimum_total_potential_voting_power_if_voting_power_spiked(
        &self,
        now_seconds: TimestampSeconds,
        current_total_potential_voting_power: u64,
    ) -> Option<(TimestampSeconds, VotingPowerTotal)> {
        let (
            timestamp_with_minimum_total_potential_voting_power,
            totals_with_minimum_total_potential_voting_power,
        ) = self
            .voting_power_totals
            .iter()
            .filter(|(created_at, _)| {
                let age = now_seconds - created_at;
                age <= MAXIMUM_STALENESS_SECONDS
            })
            .min_by_key(|(_, snapshot)| snapshot.total_potential_voting_power)?;

        let voting_power_spike_detected = (current_total_potential_voting_power as f64)
            > (totals_with_minimum_total_potential_voting_power.total_potential_voting_power
                as f64)
                * MULTIPLIER_THRESHOLD_FOR_VOTING_POWER_SPIKE;
        if voting_power_spike_detected {
            Some((
                timestamp_with_minimum_total_potential_voting_power,
                totals_with_minimum_total_potential_voting_power,
            ))
        } else {
            None
        }
    }

    /// Updates the stake of this neuron to `new_stake` and adjust this neuron's
    /// age accordingly
    pub fn update_stake(&mut self, new_stake_e8s: u64, now: u64) {
        // If this neuron has an age and its stake is being increased, adjust the
        // neuron's age
        if self.aging_since_timestamp_seconds < now && self.cached_neuron_stake_e8s <= new_stake_e8s
        {
            let old_stake = self.cached_neuron_stake_e8s as u128;
            let old_age = now.saturating_sub(self.aging_since_timestamp_seconds) as u128;
            let new_age = (old_age * old_stake) / (new_stake_e8s as u128);

            // new_age * new_stake = old_age * old_stake -
            // (old_stake * old_age) % new_stake. That is, age is
            // adjusted in proportion to the stake, but due to the
            // discrete nature of u64 numbers, some resolution is
            // lost due to the division above. This means the age
            // bonus is derived from a constant times age times
            // stake, minus up to new_stake - 1 each time the
            // neuron is refreshed. Only if old_age * old_stake is
            // a multiple of new_stake does the age remain
            // constant after the refresh operation. However, in
            // the end, the most that can be lost due to rounding
            // from the actual age, is always less 1 second, so
            // this is not a problem.
            self.aging_since_timestamp_seconds = now.saturating_sub(new_age as u64);
            // Note that if new_stake == old_stake, then
            // new_age == old_age, and
            // now - new_age =
            // now-(now-neuron.aging_since_timestamp_seconds)
            // = neuron.aging_since_timestamp_seconds.
        }

        self.cached_neuron_stake_e8s = new_stake_e8s;
    }
