Audit Report

## Title
NNS Governance `disburse_neuron` Unconditionally Zeroes `neuron_fees_e8s` Without Burning When Fees ≤ Transaction Fee - (File: `rs/nns/governance/src/governance.rs`)

## Summary

In `disburse_neuron`, the ledger burn of accumulated neuron management fees is conditional on `fees_amount_e8s > transaction_fee_e8s`, but the governance state update that zeroes `neuron_fees_e8s` and decrements `cached_neuron_stake_e8s` executes unconditionally. When a neuron's fees are at or below the ledger transaction fee threshold, the burn is silently skipped while governance records it as complete, permanently stranding up to `transaction_fee_e8s` (10,000 e8s) of ICP in the neuron's subaccount with no recovery path.

## Finding Description

In `rs/nns/governance/src/governance.rs`, `disburse_neuron` performs two sequential operations:

**Conditional burn** (lines 2046–2065): [1](#0-0) 

**Unconditional state update** (lines 2067–2076): [2](#0-1) 

The comment on line 2068 reads *"Update the stake and the fees to reflect the burning above"*, but the block executes regardless of whether the burn occurred.

**Token accounting trace** (fees F ≤ transaction_fee T, stake S):

| Step | Ledger subaccount | `cached_neuron_stake_e8s` | `neuron_fees_e8s` |
|---|---|---|---|
| Initial | S | S | F |
| After conditional burn (skipped) | S | S | F |
| After unconditional state update | S | S − F | 0 |
| After disburse transfer (S−F−T sent, T fee) | **F** | 0 | 0 |

`disburse_amount_e8s` is computed from `neuron_minted_stake_e8s` (= S − F) captured before the state update at line 1966. [3](#0-2)  The second transfer then sends S−F−T to the user with fee T, leaving exactly F tokens stranded in the subaccount. The post-transfer state update at lines 2102–2108 subtracts `disburse_amount_e8s + transaction_fee_e8s` = S−F from the already-decremented `cached_neuron_stake_e8s`, reaching zero. [4](#0-3) 

Governance records `cached_neuron_stake_e8s = 0` and `neuron_fees_e8s = 0`, so it will never issue another transfer from this subaccount. The F tokens are permanently inaccessible.

The SNS governance explicitly avoids this by placing the state update inside the burn conditional, with the comment: *"We only update the cached_neuron_stake_e8s and neuron_fees_e8s if we actually burn fees, otherwise this leads to ledger and governance getting out of sync."* [5](#0-4) 

The TLA+ formal model at `rs/nns/governance/tla/Disburse_Neuron.tla` models the current Rust behavior (calling `update_fees` in the else branch), confirming the model was written to match the code rather than to catch this divergence. [6](#0-5) 

## Impact Explanation

**Permanent loss of user funds / ledger-governance state divergence.** ICP tokens that the protocol intends to destroy as neuron management fees are instead left in the neuron's subaccount permanently. The governance canister's view of the neuron's balance diverges from the actual ICP ledger balance by exactly `fees_amount_e8s`. No governance operation can recover or burn these tokens because the neuron's recorded stake is zero. The per-neuron loss is bounded by `transaction_fee_e8s` = 10,000 e8s = 0.0001 ICP, placing this at **Medium ($200–$2,000)** — moderate user-funds impact, accumulating across all qualifying disbursals.

## Likelihood Explanation

The trigger condition (`neuron_fees_e8s` in range `(0, transaction_fee_e8s]`) is reachable by any unprivileged neuron controller via the standard `manage_neuron` ingress endpoint with a `Disburse` command. No special access, node control, or social engineering is required. A neuron that submitted a proposal with a small `reject_cost_e8s` (≤ 10,000 e8s) that was rejected will have fees in this range. The path is deterministic and repeatable.

## Recommendation

Move the governance state update inside the conditional burn block, mirroring the SNS governance pattern at `rs/sns/governance/src/governance.rs` lines 1193–1208: [7](#0-6) 

```rust
if fees_amount_e8s > transaction_fee_e8s {
    let _result = self.ledger.transfer_funds(
        fees_amount_e8s, 0, Some(neuron_subaccount),
        governance_minting_account(), now,
    ).await?;

    self.with_neuron_mut(id, |neuron| {
        if neuron.cached_neuron_stake_e8s > fees_amount_e8s {
            neuron.cached_neuron_stake_e8s -= fees_amount_e8s;
        } else {
            neuron.cached_neuron_stake_e8s = 0;
        }
        neuron.neuron_fees_e8s = 0;
    })
    .expect("Expected the parent neuron to exist");
}
```

The TLA+ model at `rs/nns/governance/tla/Disburse_Neuron.tla` should also be updated to reflect the corrected behavior. [6](#0-5) 

## Proof of Concept

**Entry path:** Unprivileged ingress call to NNS governance `manage_neuron` → `Command::Disburse`.

**Preconditions:**
- Neuron is dissolved.
- `neuron.neuron_fees_e8s` = 5,000 (e.g., from a rejected proposal with `reject_cost_e8s` = 5,000).
- `neuron.cached_neuron_stake_e8s` = 100,000,000 (1 ICP).

**Steps:**
1. Controller calls `manage_neuron` → `Disburse` with no explicit amount.
2. `fees_amount_e8s` = 5,000 ≤ `transaction_fee_e8s` = 10,000 → burn at line 2046 is skipped.
3. Unconditional block at lines 2067–2076 executes: `cached_neuron_stake_e8s` = 99,995,000; `neuron_fees_e8s` = 0.
4. `disburse_amount_e8s` = 99,990,000 (= 99,995,000 − 10,000 tx fee).
5. Transfer 2 sends 99,990,000 to user with fee 10,000; ledger subaccount balance = 100,000,000 − 99,990,000 − 10,000 = **5,000 e8s stranded**.
6. Post-transfer update sets `cached_neuron_stake_e8s` = 0.

**Verifiable unit test:** Extend `rs/nns/governance/tests/governance.rs` with a neuron having `neuron_fees_e8s` = 5,000, call `disburse_neuron`, then assert the neuron's subaccount ledger balance equals 5,000 e8s while `cached_neuron_stake_e8s` = 0 and `neuron_fees_e8s` = 0. [8](#0-7)

#[test]
#[cfg_attr(feature = "tla", with_tla_trace_check)]
fn test_cant_disburse_without_paying_fees() {
    let (driver, mut gov, neuron) = create_mature_neuron(true);

    let id = neuron.id.unwrap();
    let from = neuron.controller.unwrap();
    let neuron_stake_e8s = neuron.cached_neuron_stake_e8s;
    let neuron_fees_e8s = neuron.neuron_fees_e8s;

    // Try to disburse more than the stake amount, this should fail.
    // and cause the neuron to be unchanged.
    let result = gov
        .disburse_neuron(
            &id,
            &from,
            &Disburse {
                amount: Some(Amount {
                    e8s: 1000 * 100_000_000,
                }),
                to_account: Some(AccountIdentifier::new(from, None).into()),
            },
        )
        .now_or_never()
        .unwrap();

    assert!(result.is_err());
    assert_eq!(result.unwrap_err().error_type, ErrorType::External as i32);

    assert_eq!(
        0,
        gov.neuron_store
            .with_neuron(&id, |neuron| neuron.neuron_fees_e8s)
            .expect("Neuron not found")
    );
    driver.assert_account_contains(
        &AccountIdentifier::new(
            GOVERNANCE_CANISTER_ID.get(),
            Some(Subaccount::try_from(&neuron.account[..]).unwrap()),
        ),
        neuron_stake_e8s - neuron_fees_e8s,
    );

    // Now try to disburse exactly the cached  amount, this should fail
    // since the fees were burned above.
    let result = gov
        .disburse_neuron(
            &id,
            &from,
            &Disburse {
                amount: Some(manage_neuron::disburse::Amount {
                    e8s: neuron_stake_e8s,
                }),
                to_account: Some(AccountIdentifier::new(from, None).into()),
            },
        )
        .now_or_never()
        .unwrap();

    assert!(result.is_err());
    assert_eq!(result.unwrap_err().error_type, ErrorType::External as i32);

    // Finally try to disburse only the current stake (the initial
    // stake - the fees);
    gov.disburse_neuron(
        &id,
        &from,
        &Disburse {
            amount: Some(manage_neuron::disburse::Amount {
                e8s: neuron_stake_e8s - neuron_fees_e8s,
            }),
            to_account: Some(AccountIdentifier::new(from, None).into()),
        },
    )
    .now_or_never()
    .unwrap()
    .unwrap();

    // The user's account should now have the amount
    driver.assert_account_contains(
        &AccountIdentifier::new(from, None),
        // In the end, the user's account should have the stake - fees
        // - transaction fees.
        neuron_stake_e8s
            - neuron_fees_e8s
            - gov
                .heap_data
                .economics
                .as_ref()
                .unwrap()
                .transaction_fee_e8s,
    );
}
