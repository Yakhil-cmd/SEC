### Title
Unbounded `transfer_fee` in `ManageLedgerParameters` Proposal Allows SNS Governance to Brick Token Transfers - (File: rs/sns/governance/src/proposal.rs)

### Summary
The `validate_and_render_manage_ledger_parameters` function in SNS governance performs no upper-bound validation on the `transfer_fee` field of a `ManageLedgerParameters` proposal. A governance majority (or a compromised/malicious SNS) can pass a proposal setting the ledger `transfer_fee` to `u64::MAX`, making every token transfer impossible and effectively bricking the SNS token economy.

### Finding Description
The `ManageLedgerParameters` proposal action allows SNS token holders to change the ledger's `transfer_fee` via governance vote. The validation function `validate_and_render_manage_ledger_parameters` only checks that at least one field is set — it performs **no range check** on `transfer_fee`:

```rust
if let Some(transfer_fee) = transfer_fee {
    render += &format!("# Set token transfer fee: {transfer_fee} token-quantums. \n",);
    change = true;
    // ← No upper-bound check here
}
``` [1](#0-0) 

Once the proposal passes, `perform_manage_ledger_parameters` upgrades the SNS ledger canister with the new fee via `LedgerUpgradeArgs`:

```rust
let ledger_upgrade_arg = candid::encode_one(Some(LedgerArgument::Upgrade(Some(
    UpgradeArgs::from(manage_ledger_parameters.clone()),
))))
``` [2](#0-1) 

The ICRC-1 ledger's `upgrade` function then blindly applies the new fee:

```rust
if let Some(transfer_fee) = args.transfer_fee {
    self.transfer_fee = Tokens::try_from(transfer_fee.clone()).unwrap_or_else(|e| {
        ic_cdk::trap(...)
    });
}
``` [3](#0-2) 

The `ManageLedgerParameters` struct itself carries `transfer_fee: Option<u64>` with no constraints: [4](#0-3) 

After the upgrade, every non-mint/burn transfer on the SNS ledger requires paying the new fee. If the fee is set to `u64::MAX`, all transfers will fail with `InsufficientFunds` since no account can hold `u64::MAX` tokens. The `NervousSystemParameters.transaction_fee_e8s` is also updated to match: [5](#0-4) 

### Impact Explanation
**High.** Setting `transfer_fee` to an extreme value (e.g., `u64::MAX` or the total token supply) makes all SNS token transfers permanently impossible. This:
- Prevents token holders from transferring, selling, or using their SNS tokens.
- Prevents neuron staking (which requires a ledger transfer), freezing governance participation.
- Effectively destroys the economic utility of the SNS token.
- Cannot be easily reversed if governance itself is frozen (neurons can't be staked to vote).

### Likelihood Explanation
**Low-to-Medium.** This requires a governance majority to pass the malicious proposal. However:
- SNS governance is permissionless — any token holder with sufficient stake can submit a `ManageLedgerParameters` proposal.
- A whale or coordinated group of token holders could pass such a proposal.
- Unlike the original M-02 (which required a single admin), this requires a governance vote, but SNS governance thresholds vary per SNS and can be low.
- The attack is irreversible if governance is subsequently frozen.

### Recommendation
Add an upper-bound validation on `transfer_fee` inside `validate_and_render_manage_ledger_parameters` in `rs/sns/governance/src/proposal.rs`. For example, cap the fee at a reasonable maximum (e.g., the current `neuron_minimum_stake_e8s` or a fixed protocol-level maximum). Additionally, validate that the new fee does not exceed `neuron_minimum_stake_e8s` (which is already enforced at SNS init time but not at fee-update time). [6](#0-5) 

### Proof of Concept

1. An SNS is deployed with `transfer_fee = 10_000` e8s and `neuron_minimum_stake_e8s = 100_000_000` e8s.
2. An attacker (or malicious governance majority) submits a `ManageLedgerParameters` proposal with `transfer_fee = u64::MAX`.
3. The proposal passes `validate_and_render_manage_ledger_parameters` — no upper-bound check exists on `transfer_fee`.
4. `perform_manage_ledger_parameters` upgrades the SNS ledger with `transfer_fee = u64::MAX`.
5. The ICRC-1 ledger's `upgrade` function sets `self.transfer_fee = u64::MAX`.
6. All subsequent `icrc1_transfer` calls fail: the ledger checks `fee == expected_fee` (i.e., `u64::MAX`), which no account can pay.
7. Token holders cannot transfer tokens, cannot stake neurons, and governance is frozen.

The attack entry path is: unprivileged SNS token holder → `manage_neuron` (submit proposal) → governance vote → `perform_manage_ledger_parameters` → ledger upgrade with unbounded fee. [7](#0-6) [8](#0-7)

fn validate_and_render_manage_ledger_parameters(
    manage_ledger_parameters: &ManageLedgerParameters,
) -> Result<String, String> {
    let mut change = false;
    let mut render = "# Proposal to change ledger parameters:\n".to_string();
    let ManageLedgerParameters {
        transfer_fee,
        token_name,
        token_symbol,
        token_logo,
    } = manage_ledger_parameters;

    if let Some(transfer_fee) = transfer_fee {
        render += &format!("# Set token transfer fee: {transfer_fee} token-quantums. \n",);
        change = true;
    }
    if let Some(token_name) = token_name {
        ledger_validation::validate_token_name(token_name)?;
        render += &format!("# Set token name: {token_name}. \n",);
        change = true;
    }
    if let Some(token_symbol) = token_symbol {
        ledger_validation::validate_token_symbol(token_symbol)?;
        render += &format!("# Set token symbol: {token_symbol}. \n",);
        change = true;
    }
    if let Some(token_logo) = token_logo {
        ledger_validation::validate_token_logo(token_logo)?;
        render += &format!("# Set token logo: {token_logo}. \n",);
        change = true;
    }
    if !change {
        Err(String::from(
            "ManageLedgerParameters must change at least one value, all values are None",
        ))
    } else {
        Ok(render)
    }
}

        &mut self,
        proposal_id: u64,
        manage_ledger_parameters: ManageLedgerParameters,
    ) -> Result<(), GovernanceError> {
        self.check_no_upgrades_in_progress(Some(proposal_id))?;

        let current_version = self.get_or_reset_deployed_version().await.map_err(|err| {
            GovernanceError::new_with_message(
                ErrorType::External,
                format!("Could not execute proposal: {err}"),
            )
        })?;

        let ledger_canister_id = self.proto.ledger_canister_id_or_panic();

        let ledger_canister_info = self.env
            .call_canister(
                CanisterId::ic_00(),
                "canister_info",
                candid::encode_one(
                    CanisterInfoRequest::new(
                        ledger_canister_id,
                        Some(1),
                    )
                ).map_err(|e| GovernanceError::new_with_message(ErrorType::External, format!("Could not execute proposal. Error encoding canister_info request.\n{e}")))?
            )
            .await
            .map(|b| {
                candid::decode_one::<CanisterInfoResponse>(&b)
                .map_err(|e| GovernanceError::new_with_message(ErrorType::External, format!("Could not execute proposal. Error decoding canister_info response.\n{e}")))
            })
            .map_err(|err: (Option<i32>, String)| GovernanceError::new_with_message(ErrorType::External, format!("Canister method call canister_info failed: {err:?}")))??;

        let ledger_canister_info_version_number_before_upgrade: u64 =
            ledger_canister_info
            .changes()
            .last().ok_or(GovernanceError::new_with_message(ErrorType::External, "Could not execute proposal. Error finding current ledger canister_info version number".to_string()))?
            .canister_version();

        let ledger_wasm = get_wasm(
            &*self.env,
            current_version.ledger_wasm_hash.clone(),
            SnsCanisterType::Ledger,
        )
        .await
        .map_err(|e| {
            GovernanceError::new_with_message(
                ErrorType::External,
                format!("Could not execute proposal. Error getting ledger canister wasm: {e}"),
            )
        })?
        .wasm;

        use ic_icrc1_ledger::{LedgerArgument, UpgradeArgs};
        let ledger_upgrade_arg = candid::encode_one(Some(LedgerArgument::Upgrade(Some(
            UpgradeArgs::from(manage_ledger_parameters.clone()),
        ))))
        .unwrap();

        self.upgrade_non_root_canister(
            ledger_canister_id,
            Wasm::Bytes(ledger_wasm),
            ledger_upgrade_arg,
            CanisterInstallMode::Upgrade,
        )
        .await?;
