### Title
SNS tokens permanently stuck in Swap canister after aborted swap — no rescue mechanism - (File: `rs/sns/swap/src/swap.rs`)

### Summary
When an SNS decentralization swap is aborted (e.g., insufficient participation), the SNS tokens pre-loaded into the swap canister are never returned. `finalize_inner` exits early for the aborted lifecycle path without calling `sweep_sns`, and no `error_refund_sns` equivalent exists in the canister's public interface.

### Finding Description
The SNS Swap canister is pre-loaded with SNS tokens in the `PENDING` state before the swap opens. These tokens sit in the swap canister's default account on the SNS ledger. In the `COMMITTED` path, `sweep_sns` distributes them to participants. However, in the `ABORTED` path, `finalize_inner` returns early after restoring dapp controllers via `should_restore_dapp_control()`, and `sweep_sns` is never reached:

```rust
// rs/sns/swap/src/swap.rs  finalize_inner()
if self.should_restore_dapp_control() {
    finalize_swap_response.set_set_dapp_controllers_result(
        self.restore_dapp_controllers_for_finalize(environment.sns_root_mut()).await,
    );
    // "finalize() need not do any more work, so always return"
    return finalize_swap_response;          // ← early return; sweep_sns never called
}
// Only reached in COMMITTED path:
finalize_swap_response
    .set_sweep_sns_result(self.sweep_sns(now_fn, environment.sns_ledger()).await);
```

The canister's public interface (`rs/sns/swap/canister/canister.rs`) exposes `error_refund_icp` for ICP tokens but provides no analogous `error_refund_sns` or `sweep_sns_on_abort` endpoint. After finalization of an aborted swap, the SNS tokens remain locked in the swap canister's SNS-ledger account indefinitely.

The only recovery path is a privileged SNS governance proposal to upgrade the swap canister and add a transfer function — there is no built-in, permissionless rescue mechanism.

### Impact Explanation
**High.** The entire SNS token allocation for the swap (potentially millions of tokens representing a large fraction of the SNS token supply) becomes permanently inaccessible to the SNS treasury and token holders after an aborted swap. The swap canister's own documentation states it "can be deleted when all tokens registered with the swap canister have been disbursed to their rightful owners" — without a rescue path, this condition can never be satisfied for the SNS token side, and the tokens are effectively burned from the perspective of the SNS project.

### Likelihood Explanation
**Medium.** Every aborted SNS swap triggers this condition. Swaps are aborted whenever minimum participation thresholds (`min_participants`, `min_icp_e8s`) are not met before the swap deadline — a realistic outcome for new or less popular SNS projects. No special attacker action is required; the condition arises from normal protocol operation.

### Recommendation
Add a mechanism to return SNS tokens to the SNS governance canister (treasury) when the swap is aborted. Options include:

1. **In `finalize_inner`**: After restoring dapp controllers in the aborted path, call a `sweep_sns_on_abort` variant that transfers the remaining SNS token balance back to the SNS governance canister's account.
2. **New public endpoint**: Add `error_refund_sns` analogous to `error_refund_icp` that, after finalization, allows the SNS governance canister to reclaim the SNS token balance from the swap canister's ledger account.

### Proof of Concept

1. Deploy an SNS with a decentralization swap; pre-load SNS tokens into the swap canister (PENDING state).
2. Open the swap with `min_participants = N` and a finite `swap_due_timestamp_seconds`.
3. Allow the swap to expire with fewer than `N` participants — `try_abort` transitions lifecycle to `ABORTED`.
4. Call `finalize_swap` — `sweep_icp` returns ICP to participants; `should_restore_dapp_control()` returns `true`; `finalize_inner` returns early.
5. Query the swap canister's default account on the SNS ledger — the full SNS token balance remains there.
6. Confirm no public endpoint exists to transfer those tokens out: `error_refund_icp` only handles ICP; `sweep_sns` is gated behind the `COMMITTED` lifecycle check in `sweep_sns()` at line 2170 (`if self.lifecycle() != Lifecycle::Committed { return SweepResult::new_with_global_failures(1); }`).
