### Title
No Mechanism to Revoke a Full Access Key Once Granted - (`engine/src/contract_methods/admin.rs`)

### Summary

`attach_full_access_key` adds a NEAR Full Access Key (FAK) to the Aurora Engine account with no corresponding revocation function. Once granted, a FAK cannot be removed through any contract-level method. A rogue or compromised FAK holder retains unrestricted control over the Aurora Engine NEAR account permanently.

### Finding Description

`attach_full_access_key` in `engine/src/contract_methods/admin.rs` issues a `PromiseAction::AddFullAccessKey` to the Aurora Engine NEAR account: [1](#0-0) 

The function itself acknowledges the danger in a comment but dismisses it by relying solely on the `require_owner_only` gate: [2](#0-1) 

The contract exposes `remove_relayer_key` as the only key-deletion path: [3](#0-2) 

`remove_relayer_key` calls `engine::remove_function_call_key`, which only operates on keys tracked under `KeyPrefix::RelayerFunctionCallKey`: [4](#0-3) 

FAKs are **not** stored under `KeyPrefix::RelayerFunctionCallKey`. They are added directly to the NEAR account's key set via the host promise API and are never recorded in contract storage: [5](#0-4) 

Therefore `remove_relayer_key` will always return `NonExistedKey` for a FAK, and there is no `remove_full_access_key` entrypoint anywhere in `lib.rs`: [6](#0-5) 

### Impact Explanation

A NEAR Full Access Key holder has unrestricted authority over the account: they can sign transactions to transfer all NEAR tokens, deploy arbitrary replacement bytecode, or delete the account entirely. The Aurora Engine account holds all bridged ETH (as EVM balances) and NEAR tokens backing the bridge. A rogue or compromised FAK holder can:

1. Transfer all NEAR tokens out of the account — **direct theft of funds at rest**.
2. Deploy malicious code that drains EVM balances — **direct theft of bridged ETH**.
3. Delete the Aurora Engine account — **permanent freeze of all user funds**.

Impact: **Critical** — direct theft of user funds and/or permanent fund freeze.

### Likelihood Explanation

The owner legitimately calls `attach_full_access_key` for operational reasons (e.g., emergency access, infrastructure management). This is a supported, tested workflow: [7](#0-6) 

Once issued, if the FAK holder's private key is leaked, the holder is compromised, or the holder turns malicious, the owner has no contract-level recourse. The owner cannot call any function to revoke the key. The only workaround — deploying a contract upgrade — is the exact mitigation the original finding's judge rejected as insufficient to reduce severity.

### Recommendation

Add a `remove_full_access_key` entrypoint (owner-only) that issues `PromiseAction::DeleteKey` for the specified public key, mirroring the existing `remove_relayer_key` pattern but without the `RelayerFunctionCallKey` storage check. Optionally, maintain an on-chain registry of all FAKs granted so the owner can enumerate and revoke them.

### Proof of Concept

1. Owner calls `attach_full_access_key` with attacker's public key (legitimate operational use).
2. Attacker's private key is later compromised or attacker turns malicious.
3. Owner attempts to call `remove_relayer_key` with the FAK's public key → transaction fails with `NonExistedKey` because the FAK is not in `KeyPrefix::RelayerFunctionCallKey` storage.
4. No other contract method exists to delete the key.
5. Attacker signs a NEAR transaction using the FAK to transfer all NEAR from the Aurora Engine account, or deploys malicious code, draining all bridged user funds. [8](#0-7) [9](#0-8) [10](#0-9)

pub fn remove_relayer_key<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,

    with_hashchain(io, env, function_name!(), |mut io| {

        require_running(&state)?;
        require_key_manager_only(&state, &env.predecessor_account_id())?;

        let args: RelayerKeyArgs = serde_json::from_slice(&io.read_input().to_vec())
            .map_err(|_| errors::ERR_JSON_DESERIALIZE)?;

        let action = PromiseAction::DeleteKey {
            public_key: args.public_key,
        };
        let promise = PromiseBatchAction {
            target_account_id: env.current_account_id(),
            actions: vec![action],
        };

        let promise_id = handler.promise_create_batch(&promise);
        handler.promise_return(promise_id);

        Ok(())
    })
}

pub fn attach_full_access_key<I: IO + Copy, E: Env, H: PromiseHandler>(
    io: I,
    env: &E,
    handler: &mut H,
) -> Result<(), ContractError> {
    let state = state::get_state(&io)?;

    require_running(&state)?;
    require_owner_only(&state, &env.predecessor_account_id())?;

    let public_key = serde_json::from_slice::<FullAccessKeyArgs>(&io.read_input().to_vec())
        .map(|args| args.public_key)
        .map_err(|_| errors::ERR_JSON_DESERIALIZE)?;
    let current_account_id = env.current_account_id();
    let action = PromiseAction::AddFullAccessKey {
        public_key,
        nonce: 0, // not actually used - depends on block height
    };
    let promise = PromiseBatchAction {
        target_account_id: current_account_id,
        actions: vec![action],
    };
    // SAFETY: This action is dangerous because it adds a new full access key (FAK) to the Engine account.
    // However, it is safe to do so here because of the `require_owner_only` check above; only the
    // (trusted) owner account can add a new FAK.
    let promise_id = handler.promise_create_batch(&promise);

    handler.promise_return(promise_id);

    Ok(())
}

/// Adds a public function call key for a relayer.
pub fn add_function_call_key<I: IO>(io: &mut I, key: &PublicKey) {
    let prefixed_key = bytes_to_key(KeyPrefix::RelayerFunctionCallKey, key.key_data());
    io.write_storage(&prefixed_key, &[1]);
}

/// Removes a public function call key for a relayer.
pub fn remove_function_call_key<I: IO>(io: &mut I, key: &PublicKey) -> Result<(), EngineError> {
    let prefixed_key = bytes_to_key(KeyPrefix::RelayerFunctionCallKey, key.key_data());
    io.remove_storage(&prefixed_key)
        .ok_or_else(|| EngineError::from(EngineErrorKind::NonExistedKey))?;

    Ok(())
}

async fn test_attach_full_access_key() {
    let aurora = deploy_engine().await;
    let secret_key = SecretKey::from_random(KeyType::ED25519);
    let public_key = public_key(&secret_key);
    let admin = aurora.create_account(&aurora.id(), secret_key);

    let err = admin
        .call(&aurora.id(), "set_upgrade_delay_blocks")
        .args_borsh(SetUpgradeDelayBlocksArgs {
            upgrade_delay_blocks: 5,
        })
        .max_gas()
        .transact()
        .await;
    assert_error_message(&err, "Failed to query access key");

    let result = aurora
        .attach_full_access_key(FullAccessKeyArgs { public_key })
        .max_gas()
        .transact()
        .await
        .unwrap();

            upgrade_delay_blocks: 5,

        .max_gas()
        .transact()
        .await
        .unwrap();
