### Title
EIP-7623 Floor Gas Enforced for Acceptance but Ignored in Refund Calculation Allows Systematic Fee Underpayment - (File: `engine/src/engine.rs`)

---

### Summary

`submit_with_alt_modexp` validates that `gas_limit >= max(intrinsic_gas, floor_gas)` per EIP-7623, but `refund_unused_gas` computes the refund using raw `gas_used` with no awareness of `floor_gas`. When `gas_used < floor_gas`, the sender is over-refunded and effectively pays less than the EIP-7623 floor gas cost. The relayer receives correspondingly less fee than it is owed.

---

### Finding Description

The transaction submission pipeline in `submit_with_alt_modexp` has three distinct gas computations that are inconsistent with each other:

**Step 1 — Gas limit check uses `floor_gas` as a hard minimum** (`engine/src/engine.rs` lines 1070–1081):

```rust
let intrinsic_gas = transaction.intrinsic_gas(CONFIG)...;
let floor_gas     = transaction.floor_gas(CONFIG)...;

if transaction.gas_limit < core::cmp::max(intrinsic_gas, floor_gas).into() {
    return Err(EngineErrorKind::IntrinsicGasNotMet.into());
}
``` [1](#0-0) 

**Step 2 — Gas is charged upfront based on `gas_limit`** (lines 1101–1106):

```rust
let prepaid_amount = match engine.charge_gas(&sender, &transaction, max_gas_price, fixed_gas) {
```

Inside `charge_gas`, when `fixed_gas` is `None`, the prepaid amount is `gas_limit * effective_gas_price`. [2](#0-1) 

**Step 3 — EVM executes with `gas_limit` and produces `gas_used`** (lines 1107–1146):

The EVM executor runs with the full `gas_limit` budget. For a data-heavy transaction calling an EOA, `gas_used` can be as low as 21,000 — far below `floor_gas`. [3](#0-2) 

**Step 4 — Refund uses raw `gas_used`, ignoring `floor_gas` entirely** (lines 1262–1302):

```rust
pub fn refund_unused_gas<I: IO>(
    io: &mut I,
    sender: &Address,
    gas_used: u64,          // ← floor_gas is never passed here
    gas_result: &GasPaymentResult,
    relayer: &Address,
    fixed_gas: Option<EthGas>,
) -> Result<(), GasPaymentError> {
    ...
    let gas_to_wei = |price: U256| {
        fixed_gas
            .map_or_else(|| gas_used.into(), EthGas::as_u256)  // ← uses gas_used, not max(gas_used, floor_gas)
            .checked_mul(price)
            ...
    };
    let spent_amount = gas_to_wei(gas_result.effective_gas_price)?;
    let refund = gas_result.prepaid_amount.checked_sub(spent_amount)...;
``` [4](#0-3) 

`floor_gas` is computed in `submit_with_alt_modexp` but is **never passed to `refund_unused_gas`**. The function signature has no `floor_gas` parameter. The refund is therefore computed as:

```
refund = gas_limit * price - gas_used * price
```

But EIP-7623 requires the sender to pay at least `floor_gas * price`. The correct refund is:

```
refund = gas_limit * price - max(gas_used, floor_gas) * price
```

The `floor_gas` function itself confirms the divergence: for a call transaction with 1,000 non-zero data bytes, `floor_gas = 61,000` while `intrinsic_gas = 37,000`. [5](#0-4) 

The Prague config is hardcoded as the active config: [6](#0-5) 

---

### Impact Explanation

**High — Theft of unclaimed yield (relayer fee revenue).**

When `gas_used < floor_gas`, the sender is over-refunded by `(floor_gas - gas_used) * effective_gas_price`. The relayer receives `gas_used * priority_fee_per_gas` instead of `floor_gas * priority_fee_per_gas`, losing `(floor_gas - gas_used) * priority_fee_per_gas` per transaction. This is a direct, repeatable extraction of fee revenue that belongs to the relayer.

---

### Likelihood Explanation

**High.** Any EVM user can craft a transaction that maximizes the gap between `floor_gas` and `gas_used`:

- Use large non-zero calldata (e.g., 1,000 non-zero bytes → `floor_gas = 61,000`)
- Target an EOA or a contract that executes cheaply (`gas_used ≈ 21,000`)
- Set `gas_limit = floor_gas` (the minimum accepted value)

No special privileges, no admin access, no oracle manipulation required. The attack is deterministic and repeatable on every block.

---

### Recommendation

Pass `floor_gas` into `refund_unused_gas` and use `max(gas_used, floor_gas)` as the effective consumed gas when `fixed_gas` is `None`:

```rust
pub fn refund_unused_gas<I: IO>(
    io: &mut I,
    sender: &Address,
    gas_used: u64,
    gas_result: &GasPaymentResult,
    relayer: &Address,
    fixed_gas: Option<EthGas>,
+   floor_gas: u64,
) -> Result<(), GasPaymentError> {
    ...
    let gas_to_wei = |price: U256| {
        fixed_gas
-           .map_or_else(|| gas_used.into(), EthGas::as_u256)
+           .map_or_else(|| core::cmp::max(gas_used, floor_gas).into(), EthGas::as_u256)
            .checked_mul(price)
            ...
    };
```

And update the call site in `submit_with_alt_modexp` to pass `floor_gas`.

### Proof of Concept

```
Setup (Prague config, total_cost_floor_per_token = 10):
  data = [0x01; 1000]  (1000 non-zero bytes)
  to   = some_eoa
  gas_limit = 61_000
  gas_price = 1 wei

Execution trace in submit_with_alt_modexp:
  intrinsic_gas = 21_000 + 16 * 1_000 = 37_000
  floor_gas     = 21_000 + 10 * (1_000 * 4) = 61_000
  max(37_000, 61_000) = 61_000 ≤ gas_limit → accepted

  charge_gas: sender balance -= 61_000 * 1 = 61_000 wei

  EVM call to EOA:
    gas_used = 21_000

  refund_unused_gas:
    spent_amount = 21_000 * 1 = 21_000 wei
    refund       = 61_000 - 21_000 = 40_000 wei → credited back to sender

Result:
  Sender net cost:          21_000 wei
  EIP-7623 required cost:   61_000 wei
  Underpayment per tx:      40_000 wei
  Relayer fee shortfall:    40_000 * priority_fee_per_gas
```

pub fn refund_unused_gas<I: IO>(
    io: &mut I,
    sender: &Address,
    gas_used: u64,
    gas_result: &GasPaymentResult,
    relayer: &Address,
    fixed_gas: Option<EthGas>,
) -> Result<(), GasPaymentError> {
    if gas_result.effective_gas_price.is_zero() {
        return Ok(());
    }

    let (refund, relayer_reward) = {
        let gas_to_wei = |price: U256| {
            fixed_gas
                .map_or_else(|| gas_used.into(), EthGas::as_u256)
                .checked_mul(price)
                .map(Wei::new)
                .ok_or(GasPaymentError::EthAmountOverflow)
        };

        let spent_amount = gas_to_wei(gas_result.effective_gas_price)?;
        let reward_amount = gas_to_wei(gas_result.priority_fee_per_gas)?;

        let refund = gas_result
            .prepaid_amount
            .checked_sub(spent_amount)
            .ok_or(GasPaymentError::EthAmountOverflow)?;

        (refund, reward_amount)
    };

    if !refund.is_zero() {
        add_balance(io, sender, refund)?;
    }

    if !relayer_reward.is_zero() {
        add_balance(io, relayer, relayer_reward)?;
    }

    Ok(())

    #[allow(clippy::naive_bytecount)]
    pub fn floor_gas(&self, config: &aurora_evm::Config) -> Result<u64, Error> {
        if config.has_floor_gas {
            let num_zero_bytes = u64::try_from(self.data.iter().filter(|b| **b == 0).count())
                .map_err(|_e| Error::IntegerConversion)?;
            let data_len = u64::try_from(self.data.len()).map_err(|_e| Error::IntegerConversion)?;
            let num_non_zero_bytes = data_len
                .checked_sub(num_zero_bytes)
                .ok_or(Error::GasOverflow)?;

            let base_gas = config.gas_transaction_call;
            let tokens_in_calldata = num_non_zero_bytes
                .checked_mul(4)
                .and_then(|gas| gas.checked_add(num_zero_bytes))
                .ok_or(Error::GasOverflow)?;

            tokens_in_calldata
                .checked_mul(config.total_cost_floor_per_token)
                .and_then(|gas| gas.checked_add(base_gas))
                .ok_or(Error::GasOverflow)
        } else {
            Ok(0)
        }
    }
