Audit Report

## Title
Unpermissioned `ClaimOrRefresh` Neuron Age Dilution via Third-Party Stake Injection - (File: rs/nns/governance/src/governance.rs)

## Summary
Any unprivileged principal can call `manage_neuron` with `Command::ClaimOrRefresh { by: By::NeuronIdOrSubaccount }` targeting any existing neuron without authorization. By first transferring ICP to the target neuron's subaccount, the subsequent `refresh_neuron` call unconditionally invokes `update_stake_adjust_age`, which dilutes the neuron's `aging_since_timestamp_seconds` toward `now` using a weighted-average formula. Because neuron age drives the age-bonus multiplier in voting power (up to 1.25× at 4 years for NNS), an attacker can permanently suppress a victim neuron's age bonus and reduce its voting reward efficiency per ICP staked.

## Finding Description

**Root cause:** `manage_neuron_internal` dispatches `By::NeuronIdOrSubaccount` to `refresh_neuron_by_id_or_subaccount` before any ownership check, and the `caller` identity is never passed into or checked within `refresh_neuron`:

```rust
// rs/nns/governance/src/governance.rs L6132-6141
Some(By::NeuronIdOrSubaccount(_)) => {
    let id = mgmt.get_neuron_id_or_subaccount()?.ok_or_else(|| { ... })?;
    self.refresh_neuron_by_id_or_subaccount(id, claim_or_refresh)
        .await
        .map(ManageNeuronResponse::claim_or_refresh_neuron_response)
}
```

`refresh_neuron_by_id_or_subaccount` (L5875) and `refresh_neuron` (L5900) accept no `caller` parameter and perform no ownership or hotkey check. When the ledger balance exceeds the cached stake (`Ordering::Less` branch at L5950-5951), `update_stake_adjust_age` is called unconditionally:

```rust
// rs/nns/governance/src/governance.rs L5950-5951
Ordering::Less => {
    neuron.update_stake_adjust_age(balance.get_e8s(), now);
}
```

`update_stake_adjust_age` (L999-1039 in `neuron/types.rs`) computes a weighted-average age where the injected tokens carry age = 0:

```rust
let (new_stake_e8s, new_age_seconds) = combine_aged_stakes(
    self.cached_neuron_stake_e8s,
    self.age_seconds(now),
    updated_stake_e8s.saturating_sub(self.cached_neuron_stake_e8s),
    0,  // injected tokens have zero age
);
```

Result: `new_age = old_age × old_stake / (old_stake + injected_stake)`. Injecting an amount equal to the current stake halves the age. Repeated injections drive age toward zero.

**Existing guards are insufficient:** The only guard in `refresh_neuron` is a minimum-stake balance check (L5924-5934). There is no check that the caller is the neuron's controller or a hotkey. The `By::Memo` and `By::MemoAndController` paths pass `caller` to `claim_or_refresh_neuron_by_memo_and_controller` but that function also performs no ownership check on refresh — however, those paths require knowledge of the memo (a weak ownership signal). The `By::NeuronIdOrSubaccount` path requires only the public neuron ID or subaccount, both of which are on-chain.

**SNS analog confirmed:** `rs/sns/governance/src/governance.rs` L4237-4298 shows `refresh_neuron` with no caller check, calling `neuron.update_stake()` (L4285, L4288) which implements the same weighted-average age dilution in `rs/sns/governance/src/neuron.rs` L649-679.

## Impact Explanation

The age bonus multiplier saturates at 1.25× at `MAX_NEURON_AGE_FOR_AGE_BONUS` (4 years). A neuron at maximum age that has its age halved loses half its age bonus contribution. Repeated injections suppress the age bonus to near zero, permanently reducing the neuron's voting reward efficiency per ICP staked. While the victim's absolute voting power increases (because stake doubles), the victim's share of voting rewards per ICP staked decreases, and the earned age property — which requires years of non-dissolving lock-up to accumulate — is destroyed without the neuron owner's consent. The victim cannot refuse the injection or prevent the age dilution. This constitutes unauthorized modification of a neuron's governance state and significant NNS/SNS governance security impact with concrete user harm, fitting the **High** impact category: unauthorized access to/modification of neurons and governance assets where exploitation requires meaningful per-target work (ICP equal to victim's stake).

## Likelihood Explanation

The attack entry point is the public `manage_neuron` ingress endpoint, callable by any principal. The neuron subaccount is deterministic and derivable from on-chain history. The economic cost scales with the desired impact: halving the age of a 100 ICP neuron requires sending 100 ICP (plus transfer fee). The injected ICP is not destroyed — it is locked in the victim's neuron — making the attacker's net cost the opportunity cost of locked ICP plus fees. A one-time significant dilution against a governance-critical neuron before a key vote is feasible. Sustained suppression is expensive but not impossible for well-funded adversaries targeting high-value neurons.

## Recommendation

1. **Preserve age on third-party refresh:** In `refresh_neuron` (NNS) and `refresh_neuron` (SNS), check whether the caller is the neuron's controller or a hotkey before calling `update_stake_adjust_age`/`update_stake`. If the caller is not authorized, update `cached_neuron_stake_e8s` directly without touching `aging_since_timestamp_seconds`.
2. **Gate `By::NeuronIdOrSubaccount` on ownership:** Require the caller to be the controller or a hotkey when using the `NeuronIdOrSubaccount` variant, since this variant requires no memo knowledge and is the easiest unauthenticated path.
3. **Apply the same fix to SNS:** `rs/sns/governance/src/governance.rs` `refresh_neuron` and `rs/sns/governance/src/neuron.rs` `update_stake` are subject to the same pattern.
