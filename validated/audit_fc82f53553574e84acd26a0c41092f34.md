Audit Report

## Title
Spawn Neuron Pre-Check Uses Stale -5% Worst-Case Modulation After Mission 70 Extended Minimum to -10% - (File: `rs/nns/governance/src/governance.rs`)

## Summary
The `spawn_neuron` pre-condition check hardcodes `-5%` as the worst-case maturity modulation when validating whether a neuron has sufficient maturity to spawn. Mission 70 extended the minimum modulation to `-10%` (`MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 = -1_000`), and the CHANGELOG confirms spawning now reads this new locally-computed value. The pre-check was never updated, allowing a neuron owner to spawn a child neuron whose actual minted stake falls below `neuron_minimum_stake_e8s`, violating a core NNS protocol invariant.

## Finding Description
At line 2666 of `rs/nns/governance/src/governance.rs`, the spawn pre-check computes:

```rust
let least_possible_stake = (maturity_to_spawn as f64 * (1_f64 - 0.05)) as u64;
``` [1](#0-0) 

This encodes a worst-case modulation of **-5% (-500 permyriad)**. However, `MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 = -1_000` (i.e., **-10%**) is the actual lower bound used by the Mission 70 computation: [2](#0-1) 

The CHANGELOG confirms that as of Proposal 141779, spawning reads the locally-computed Mission 70 modulation (not the old CMC-polled value): [3](#0-2) 

At spawn execution time in `maybe_spawn_neurons`, the code reads `maturity_modulation.current_value_permyriad` (the Mission 70 value), validates it is within `VALID_MATURITY_MODULATION_BASIS_POINTS_RANGE` (which includes values down to `-1000`), and then calls `apply_maturity_modulation(original_maturity, maturity_modulation)`: [4](#0-3) [5](#0-4) 

`apply_maturity_modulation` performs integer arithmetic: `amount * (10_000 + basis_points) / 10_000`, so at `-1000` permyriad it computes `amount * 9000 / 10000`: [6](#0-5) 

**Exploit path:**
1. Neuron owner accumulates maturity `M` in `[min_stake / 0.95, min_stake / 0.90)`. For `min_stake = 100_000_000 e8s`, this is approximately `[105_263_158, 111_111_111)` e8s.
2. Owner waits for modulation to reach `-1000` permyriad (observable via `get_maturity_modulation` query).
3. Owner calls `spawn_neuron`. Pre-check: `M * 0.95 >= 100_000_000` → **passes**.
4. `maybe_spawn_neurons` applies `M * 9000 / 10000` < `100_000_000` → child neuron minted with **below-minimum stake**.

## Impact Explanation
A child neuron is created on the ICP ledger with `cached_neuron_stake_e8s` below `neuron_minimum_stake_e8s`, violating the NNS protocol invariant that all neurons must meet the minimum stake. Such a neuron may be unable to participate in governance operations that enforce the minimum, and its existence represents an inconsistent state that could affect downstream logic (merge operations, stake checks). This constitutes a significant NNS governance security impact with concrete protocol harm, fitting the **High** impact class.

## Likelihood Explanation
Any neuron owner can trigger this — no privileged access is required. The only external dependency is the ICP market price reaching a level where the 7-day average is sufficiently below the 365-day average to push modulation to `-1000` permyriad. This is a realistic market condition during sustained price declines. The attacker can monitor the `get_maturity_modulation` query endpoint to time the `spawn_neuron` call precisely.

## Recommendation
Replace the hardcoded `-0.05` with the actual Mission 70 minimum modulation constant, or use integer arithmetic consistent with `apply_maturity_modulation`:

```rust
use crate::timer_tasks::update_icp_xdr_rate_related_data::MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70;

let least_possible_stake = apply_maturity_modulation(
    maturity_to_spawn,
    MATURITY_MODULATION_MIN_PERMYRIAD_MISSION_70 as i32,
).unwrap_or(0);
```

This ensures the pre-check uses the same arithmetic and the same bound as the actual spawn execution path.

## Proof of Concept
**Numeric example** with `neuron_minimum_stake_e8s = 100_000_000`:

| Variable | Value |
|---|---|
| `maturity_to_spawn` | `108_000_000 e8s` |
| Pre-check: `108_000_000 * 0.95` | `102_600_000` ≥ `100_000_000` → **passes** |
| Actual stake at -10%: `108_000_000 * 9000 / 10000` | `97_200_000` < `100_000_000` → **below minimum** |

A deterministic integration test can be written using PocketIC: set `neuron_minimum_stake_e8s = 100_000_000`, create a neuron with `maturity_e8s_equivalent = 108_000_000`, set `maturity_modulation.current_value_permyriad = -1000`, call `spawn_neuron`, advance time past the spawn delay, trigger `maybe_spawn_neurons`, and assert that the resulting child neuron's `cached_neuron_stake_e8s` is `97_200_000` — below the minimum.

```rust
pub fn apply_maturity_modulation(
    amount_maturity_e8s: u64,
    maturity_modulation_basis_points: i32,
) -> Result<u64, String> {
    let amount_e8s = u128::from(amount_maturity_e8s);

    let adjusted_maturity_modulation_basis_points = saturating_add_or_subtract_u128_i32(
        BASIS_POINTS_PER_UNITY,
        maturity_modulation_basis_points,
    );

    let modulated_amount_e8s: u128 = amount_e8s
        .checked_mul(adjusted_maturity_modulation_basis_points)
        .ok_or_else(|| "Underflow or overflow when calculating maturity modulation".to_string())?
        .checked_div(BASIS_POINTS_PER_UNITY)
        .ok_or_else(|| "Underflow or overflow when calculating maturity modulation".to_string())?;

    u64::try_from(modulated_amount_e8s).map_err(|err| err.to_string())
}
