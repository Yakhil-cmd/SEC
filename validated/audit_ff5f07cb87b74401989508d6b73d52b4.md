### Title
Permanent ETH Freeze When `refund_address` Is a Contract Without `receive()` in `ExitToNear` Precompile - (File: `engine-precompiles/src/native.rs`, `engine/src/engine.rs`)

### Summary

When the `error_refund` feature is enabled, the `ExitToNear` precompile accepts a user-controlled `refund_address` (bytes 1–20 of the input). If the NEAR-side `ft_transfer` fails and the `refund_address` is an EVM contract without a `receive()` or `fallback()` function, the ETH refund EVM call reverts, the callback returns `ERR_REFUND_FAILURE`, and the ETH is permanently frozen at `exit_to_near::ADDRESS` with no recovery path.

### Finding Description

**Step 1 – User-controlled `refund_address` parsed from precompile input.**

When `error_refund` is compiled in, `parse_input` extracts bytes 1–20 of the caller-supplied input as the `refund_address`: [1](#0-0) 

This address is stored verbatim in `RefundCallArgs.recipient_address`: [2](#0-1) 

**Step 2 – On NEAR-side failure, `refund_on_error` sends ETH to `refund_address` via EVM call.**

In `refund_on_error`, the ETH-exit branch performs a plain EVM value transfer (empty calldata) from `exit_to_near::ADDRESS` to `refund_address`: [3](#0-2) 

If `refund_address` is a contract without `receive()` or `fallback()`, the EVM call reverts and `engine.call` returns a non-ok `SubmitResult`.

**Step 3 – Callback treats refund failure as fatal; ETH stays at precompile address.**

`exit_to_near_precompile_callback` checks the refund result and returns `ERR_REFUND_FAILURE` if the status is not ok: [4](#0-3) 

The original ETH deduction (user → `exit_to_near::ADDRESS`) was committed in the prior NEAR receipt. The failed callback reverts only its own state changes. The ETH remains at `exit_to_near::ADDRESS` with no admin drain function or secondary recovery path.

### Impact Explanation

**Permanent freezing of funds.** ETH is moved from the user's EVM balance to `exit_to_near::ADDRESS` when the precompile is invoked. If the NEAR-side transfer fails and the refund EVM call also fails (because `refund_address` has no `receive()`), the ETH is irrecoverably locked at the precompile address. There is no privileged function to sweep `exit_to_near::ADDRESS`.

### Likelihood Explanation

DeFi contracts on Aurora routinely call the exit precompile as part of larger operations and commonly lack `receive()` functions. A contract that sets its own address as `refund_address` (to reclaim funds on failure) but has no ETH receive handler will trigger this freeze whenever the NEAR-side transfer fails (e.g., the NEAR recipient account is not registered with the NEP-141 token). The `error_refund` feature is a production compile-time flag; when enabled, this path is live.

### Recommendation

1. In `refund_on_error` for the ETH case, if the EVM call to `refund_address` fails (non-ok status), fall back to crediting the ETH back to the original EVM caller's balance rather than leaving it at `exit_to_near::ADDRESS`.
2. Alternatively, document that `refund_address` must be an EOA or a contract with a `receive()` function, and add an on-chain check (e.g., reject contracts with no code, or require a test call).

### Proof of Concept

1. Deploy a contract `C` on Aurora that has no `receive()` function.
2. From `C`, call `ExitToNear` precompile (`0xe9217bc7...`) with:
   - `flag = 0x00` (ETH exit)
   - `refund_address = address(C)` (bytes 1–20)
   - `recipient_account_id = "unregistered.near"` (not registered with the ETH connector NEP-141)
   - `msg.value = 1 ether`
3. The NEAR-side `ft_transfer` to `"unregistered.near"` fails.
4. `exit_to_near_precompile_callback` fires; `refund_on_error` calls `engine.call(exit_to_near::ADDRESS → C, 1 ether, [])`.
5. `C` has no `receive()` → EVM revert → `SubmitResult.status` is not ok.
6. Callback returns `ERR_REFUND_FAILURE`.
7. `C`'s 1 ETH is permanently frozen at `exit_to_near::ADDRESS` (`0xe9217bc70b7ed1f598ddd3199e80b093fa71124f`). [5](#0-4) [6](#0-5) [7](#0-6)

#[allow(clippy::unnecessary_wraps)]
fn refund_call_args(
    params: &ExitToNearParams,
    event: &events::ExitToNear,
) -> Option<RefundCallArgs> {
    Some(RefundCallArgs {
        recipient_address: match params {
            ExitToNearParams::BaseToken(params) => params.refund_address,
            ExitToNearParams::Erc20TokenParams(params) => params.refund_address,
        },
        erc20_address: match params {
            ExitToNearParams::BaseToken(_) => None,
            ExitToNearParams::Erc20TokenParams(_) => {
                let erc20_address = match event {
                    events::ExitToNear::Legacy(legacy) => legacy.erc20_address,
                    events::ExitToNear::Omni(omni) => omni.erc20_address,
                };
                Some(erc20_address)
            }
        },
        amount: types::u256_to_arr(&match event {
            events::ExitToNear::Legacy(legacy) => legacy.amount,
            events::ExitToNear::Omni(omni) => omni.amount,
        }),
    })
}

pub fn refund_on_error<I: IO + Copy, E: Env, P: PromiseHandler>(
    io: I,
    env: &E,
    state: EngineState,
    args: &RefundCallArgs,
    handler: &mut P,
) -> EngineResult<SubmitResult> {
    let current_account_id = env.current_account_id();
    if let Some(erc20_address) = args.erc20_address {
        // ERC-20 exit; re-mint burned tokens
        let erc20_admin_address = current_address(&current_account_id);
        let mut engine: Engine<_, _> =
            Engine::new_with_state(state, erc20_admin_address, current_account_id, io, env);

        let refund_address = args.recipient_address;
        let amount = U256::from_big_endian(&args.amount);
        let input = setup_refund_on_error_input(amount, refund_address);

        engine.call(
            &erc20_admin_address,
            &erc20_address,
            Wei::zero(),
            input,
            u64::MAX,
            Vec::new(),
            Vec::new(),
            handler,
        )
    } else {
        // ETH exit; transfer ETH back from precompile address
        let exit_address = exit_to_near::ADDRESS;
        let mut engine: Engine<_, _> =
            Engine::new_with_state(state, exit_address, current_account_id, io, env);
        let refund_address = args.recipient_address;
        let amount = Wei::new(U256::from_big_endian(&args.amount));
        engine.call(
            &exit_address,
            &refund_address,
            amount,
            Vec::new(),
            u64::MAX,
            vec![
                (exit_address.raw(), Vec::new()),
                (refund_address.raw(), Vec::new()),
            ],
            Vec::new(),
            handler,
        )
    }
}
