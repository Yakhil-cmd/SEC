### Title
CALLCODE with Non-Zero Value Incorrectly Receives CALL_STIPEND — (`evm_interpreter/src/ee_trait_impl.rs`)

### Summary

`calculate_resources_passed_in_external_call` grants the 2300-gas stipend to any call that is `!is_delegate && value > 0`. CALLCODE satisfies this condition, but the EVM Yellow Paper only grants the stipend for CALL (opcode `0xF1`), not CALLCODE (opcode `0xF2`).

### Finding Description

In `evm_interpreter/src/ee_trait_impl.rs`, the stipend logic is:

```rust
let is_delegate = call_request.is_delegate();
let is_callcode = call_request.is_callcode();
let is_callcode_or_delegate = is_callcode || is_delegate;

stipend = if !is_delegate && !call_request.nominal_token_value.is_zero() {
    ...
    Some(Ergs(CALL_STIPEND * ERGS_PER_GAS))   // 2300 gas added
} else {
    None
};
``` [1](#0-0) 

`is_callcode_or_delegate` is computed but only used for the account-creation cost guard at line 314: [2](#0-1) 

It is **never used** in the stipend condition. For a CALLCODE with `value > 0`:
- `is_delegate` = `false` → `!is_delegate` = `true`
- `nominal_token_value.is_zero()` = `false`
- → stipend branch taken → callee receives an extra `CALL_STIPEND` (2300) gas [3](#0-2) 

The `call_impl` function in `host.rs` maps CALLCODE to `CallModifier::EVMCallcode`, which is not a delegate modifier, confirming `is_delegate()` returns `false` for it: [4](#0-3) 

### Impact Explanation

Any unprivileged contract can issue `CALLCODE` with `value > 0`. The callee frame receives 2300 extra ergs it should not have per the EVM specification. This is a concrete, measurable EVM compatibility deviation. Contracts that depend on the gas-exhaustion behavior of CALLCODE (e.g., expecting the callee to be unable to perform storage writes due to the 2300-gas limit not applying) will behave differently on ZKsync OS than on Ethereum mainnet or any conformant EVM implementation.

### Likelihood Explanation

CALLCODE is deprecated but still a valid opcode. Any contract that uses it with non-zero value triggers the deviation. The path is fully unprivileged and reachable through normal transaction execution.

### Recommendation

Change the stipend condition to also exclude CALLCODE:

```rust
// Before (buggy):
stipend = if !is_delegate && !call_request.nominal_token_value.is_zero() {

// After (correct):
stipend = if !is_delegate && !is_callcode && !call_request.nominal_token_value.is_zero() {
``` [5](#0-4) 

### Proof of Concept

Deploy contract A. From A, execute `CALLCODE` targeting contract B with `value = 1 wei` and `gas = 0`. In a conformant EVM (geth/revm), the callee receives 0 gas (no stipend for CALLCODE). On ZKsync OS, the callee receives 2300 gas. A differential test asserting `gas_left_in_callee == 0` after `CALLCODE(gas=0, value=1)` will pass on geth/revm and fail on ZKsync OS, proving the deviation.
